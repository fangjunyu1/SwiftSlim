# Система типів

У попередніх уроках ми вже вивчили змінні й знаємо, що змінні можуть зберігати дані різних типів.

Наприклад:

```swift
var hello = "Hello, World"
```

Тут `hello` зберігає текст, тому його тип — `String`.

Далі ми системно вивчимо поняття типу (Type), а також явне оголошення типу й виведення типу, щоб глибше зрозуміти сутність змінних.

## Що таке тип

У Swift кожне значення має чітко визначений тип. Тип визначає, що це за значення і що з ним можна робити.

Наприклад:

* `42` — це `Int` (ціле число)
* `"Hello"` — це `String` (рядок)
* `true` — це `Bool` (булеве значення)

Вони належать до різних типів і мають різне призначення.

## Поширені базові типи в Swift

На початковому етапі найчастіше використовуються такі типи:

* `String`：рядок (текст)
* `Int`：ціле число
* `Double`：число з плаваючою комою (із десятковою частиною, вища точність)
* `Bool`：булеве значення (`true/false`)

Крім того, є ще `Float` (число з плаваючою комою), але його точність нижча за `Double`, тому він використовується рідше; також є `Character` (один символ), наприклад `"A"`, `"!"` та інші окремі символи.

### Оголошення типу (явне задання типу)

У Swift під час оголошення змінної можна вручну вказати її тип:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

Після імені змінної додається `: Тип`, щоб оголосити тип змінної.

Запис типу масиву має вигляд `: [Тип]`:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

У цьому фрагменті коду масив `scores` може зберігати лише елементи типу `Int`, а `tags` — лише елементи типу `String`.

Явне оголошення типу робить намір коду зрозумілішим і в деяких випадках допомагає уникнути помилок виведення типу.

## Виведення типу

У багатьох випадках тип можна не оголошувати явно:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"     // String
var isLogin = false  // Bool
var scores = [90, 85, 78]   // [Int]
var tags = ["Swift", "iOS"] // [String]
```

Це можливо тому, що компілятор Swift автоматично виводить тип на основі "початкового значення".

## Явне оголошення та виведення типу

У попередніх уроках ми не вводили явне оголошення типу, наприклад:

```swift
var age: Int = 18
```

Натомість ми спочатку надавали перевагу виведенню типу:

```swift
var age = 18
```

У цьому прикладі обидва записи рівнозначні: `age` в обох випадках буде визначено як тип `Int`. Порівняно з цим, запис із виведенням типу коротший і простіший.

Причина, чому на початку ми не робили акцент на явному оголошенні, полягає в тому, що виведення типу зменшує кількість додаткової інформації в коді, а отже знижує складність розуміння для початківців.

## Навіщо потрібні типи

Swift — це строго типізована мова (Strongly Typed Language).

Це означає: щойно тип змінної визначено, його не можна довільно змінювати.

Наприклад:

```swift
var age: Int = 18

age = 20    // Success
age = "20"  // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

Перше значення `20` має тип `Int`, тому його можна присвоїти `age`; друге значення `"20"` має тип `String`, який не збігається з `Int`, тому виникає помилка.

`nums` — це масив типу `[Int]`, тому він може зберігати лише цілі числа і не може містити рядки.

Типи обмежують спосіб використання даних, дозволяють виявляти проблеми невідповідності типів ще на етапі компіляції, тим самим зменшуючи кількість багів через плутанину типів і підвищуючи стабільність та супроводжуваність коду.

## Коли потрібно явно оголошувати тип

Хоча Swift може автоматично виводити тип, у деяких ситуаціях компілятор не може його визначити або визначає неправильно. У таких випадках тип потрібно задавати вручну.

**1. Порожній масив**

Під час створення порожнього масиву тип потрібно вказувати явно:

```swift
var nums: [Int] = []
```

Якщо не вказати тип:

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

компілятор не зможе зрозуміти, елементи якого типу має містити порожній масив, тому виникне помилка компіляції.

**2. Щоб уникнути помилкового виведення**

```swift
var value = 10   // Int
```

У цьому прикладі, хоча `10` також можна розглядати як число з плаваючою комою (`Double`), через відсутність десяткової крапки компілятор за замовчуванням виведе тип `Int`.

Якщо потрібно, щоб `value` було числом з плаваючою комою, слід явно оголосити тип:

```swift
var value: Double = 10   // Double
```

Коли результат виведення типу не відповідає очікуванню, потрібно або явно вказати тип, або змінити форму літерала, щоб отримати правильний тип.

**3. Власні типи**

Пізніше ми ще вивчатимемо власні типи. Під час роботи з ними також часто потрібно явно оголошувати тип:

```swift
var info: GameInfo
var users: [User] = []
```

Причина така сама, як і з порожнім масивом: компілятор не може автоматично вивести тип з порожнього значення.

## Перетворення типів

У Swift різні типи не перетворюються автоматично один в один, тому перетворення потрібно виконувати вручну.

Наприклад, додавання двох чисел різних типів:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b   // Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

У цьому коді `a` має тип `Int`, а `b` — тип `Double`. Оскільки типи різні, їх не можна безпосередньо використовувати в одній арифметичній операції.

Перед обчисленням потрібно спочатку перетворити одне зі значень до такого самого типу:

```swift
var result = Double(a) + b
```

Цей код означає, що значення `a` типу `Int` перетворюється на `Double`, після чого додається до `b`, а результат записується в `result`.

Важливо звернути увагу: перетворення типу не змінює тип початкової змінної. Наприклад, `Double(a)` не змінює тип `a` з `Int`, а лише створює нове значення типу `Double` для участі в обчисленні.

Інші типи також можна перетворювати подібним способом, тобто використовувати цільовий тип + `()` і передавати всередину значення, яке потрібно перетворити. Наприклад:

```swift
Int()、Double()、String()
```

Приклад:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

Потрібно пам’ятати, що не всі перетворення відбуваються "ідеально". Наприклад, під час перетворення `Double → Int` десяткова частина просто відкидається, а не округлюється.

## Псевдоніми типів

У Swift можна також дати вже наявному типу іншу назву, щоб зробити сенс коду зрозумілішим:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

Тут `UserID` за своєю суттю все одно є `Int`, тобто сам тип не змінюється, змінюється лише назва, яка стає змістовнішою.

У великих проєктах із великою кількістю коду та складними типами псевдоніми типів допомагають зробити код читабельнішим.

## Приклад із типами

Нижче на простому прикладі покажемо, як типи працюють у реальній розробці.

### Створення змінної

Спочатку створимо змінну `age`:

```swift
var age = 20
```

Оскільки нам потрібно відображати й змінювати цю змінну у SwiftUI, слід оголосити її через `@State`:

```swift
@State private var age = 20
```

`private` уже згадувався раніше. Це означає, що цю змінну можна використовувати лише в поточному поданні, а пізніше ми розглянемо це детальніше.

### Відображення даних у SwiftUI

Покажемо змінну `age` у SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

Тепер подання покаже:

```swift
age: 20
```

### Зміна даних

Далі додамо дві кнопки для зміни `age`:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### Результат виконання

Коли натиснути `changeAge1`, змінній `age` буде присвоєно значення `18`:

```swift
age = 18
```

Оскільки `18` має тип `Int` і цей тип збігається з типом `age`, присвоєння відбудеться успішно.

А от `changeAge2` спричинить помилку:

```swift
age = "18"
```

Повідомлення про помилку:

```swift
Cannot assign value of type 'String' to type 'Int'
```

Це тому, що `"18"` має тип `String` (рядки беруться в лапки `""`), а `age` має тип `Int`, тому типи не збігаються і присвоєння неможливе.

### Чому виникає помилка?

Ключ до цього — ось цей рядок коду:

```swift
@State private var age = 20   // Int
```

Хоча тип тут не вказано явно, оскільки початкове значення `20` має тип `Int`, компілятор автоматично виведе для `age` саме тип `Int`.

Щойно тип визначено, змінній вже не можна присвоювати значення іншого типу.

І ще важливо: кнопка `changeAge2` не "ламається під час виконання", а викликає помилку вже на етапі компіляції, тому код навіть не зможе зібратися.

У цьому й полягає одна з переваг строгої типізації Swift: проблеми з типами можна виявити ще під час написання коду.

### Правильний запис

Після видалення помилкового коду:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

Коли натиснути кнопку `changeAge1`, подання оновиться до:

```swift
age: 18
```

### Явне оголошення типу (необов’язково)

Під час оголошення також можна явно вказати тип:

```swift
@State private var age: Int = 20
```

Такий запис робить тип зрозумілішим, але в простих сценаріях це не є обов’язковим.

## Підсумок

У цьому уроці ми головним чином розглянули систему типів у Swift і її роль у реальній розробці.

У Swift кожне значення має чітко визначений тип. Тип визначає зміст даних і те, у яких операціях вони можуть брати участь.

Swift — строго типізована мова. Щойно тип змінної визначено, їй уже не можна присвоювати значення іншого типу. Таке обмеження дає змогу виявляти помилки ще на етапі компіляції, зменшувати проблеми під час виконання й підвищувати безпечність коду.

Тип змінної можна автоматично вивести з початкового значення або задати явно. У деяких випадках (наприклад, порожній масив, помилка у визначенні типу, власний тип) компілятор не може правильно вивести тип, і тоді його потрібно задавати явно.

Різні типи не перетворюються автоматично один в один — для цього потрібно явно використовувати `Int()`、`Double()`、`String()` та інші подібні способи.

Типи також підтримують псевдоніми. Особливо у складних проєктах псевдоніми типів можуть зробити код читабельнішим.

Тип — це не просто позначка даних, а й механізм обмеження, який гарантує, що дані використовуються безпечно, чітко й контрольовано.

## Додаткові знання — опціональний тип

Під час перетворення типів деякі операції можуть завершитися невдачею.

Наприклад, перетворення `String` на `Double`:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

Тут `Double(input)` повертає не `Double`, а:

```swift
Double?
```

Це і є опціональний тип (Optional).

### Чому повертається опціональний тип?

Тому що вміст рядка є невизначеним: `"3.14"` можна перетворити на `Double`, а `"Hello"` — ні.

Тобто така операція може або завершитися успішно, або завершитися невдачею.

Саме тому Swift використовує опціональний тип, щоб позначити таку невизначеність.

### Що таке опціональний тип?

Опціональний тип означає: значення може існувати, а може й не існувати.

Наприклад:

```swift
Double?
```

Цей тип означає, що якщо значення є, то під час друку воно відображатиметься як `Optional(...)`, а якщо значення немає, буде повернено `nil`.

Потрібно звернути увагу: `nil` означає "немає значення", а не `0` і не порожній рядок.

### Приклад

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Вивід:

```swift
pi:Optional(3.14)
```

Це означає, що рядок успішно перетворився на `Double` і дав `3.14`, але оскільки тип є `Double?`, значення обгортається в `Optional`.

Якщо перетворення не вдасться:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

Вивід:

```swift
pi: nil
```

Оскільки `"Hello"` не можна перетворити на тип `Double`, перетворення завершується невдачею й повертається `nil`.

### Що таке Optional(...)?

Коли ми безпосередньо друкуємо опціональний тип:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift показує його "опис для налагодження":

```swift
Optional(3.14)
```

Це не є частиною самих даних, а лише підказує, що поточне значення є "опціональним типом" і всередині містить конкретне значення.

### Розпакування опціонального типу

У реальній розробці нам зазвичай потрібно дістати справжнє значення з опціонального типу. Цей процес називається розпакуванням (Unwrapping).

Один із поширених способів — використати `??`, щоб задати значення за замовчуванням:

```swift
??
```

Це називається оператором об’єднання з `nil` (Nil-Coalescing Operator).

Наприклад:

```swift
name ?? "FangJunyu"
```

Цей код означає: якщо `name` має значення, повернути його; якщо там `nil`, використати значення за замовчуванням `"FangJunyu"`.

Потрібно звернути увагу, що тип значення за замовчуванням має збігатися з типом опціонального значення.

### Використання `??` для значення за замовчуванням

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")   // 30
```

Вивід:

```swift
pi: 30
```

Під час перетворення `input` у `Double` операція виконується успішно й виводиться значення. Якщо ж перетворення не вдасться, буде виведено значення за замовчуванням, задане через `??`.

Якщо перетворення не вдасться:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

Вивід:

```swift
pi: 10
```

Тут під час перетворення `input` у `Double` операція завершується невдачею й повертає `nil`, а `??` надає для `nil` значення за замовчуванням.

Опціональний тип використовується для позначення ситуації, коли "значення може бути, а може й не бути".

У таких потенційно невдалих операціях, як перетворення типів, Swift повертає опціональний тип, щоб забезпечити безпечність програми.

Коли нам потрібно використати значення всередині, ми можемо надати значення за замовчуванням через `??`, щоб навіть у разі появи `nil` усе одно отримати розумний результат.
