# Адаптація зовнішнього вигляду

У цьому уроці ми навчимося адаптувати інтерфейс до світлого й темного режимів.

На iPhone, iPad і Mac користувач може вибрати світлий або темний режим.

Коли система перемикається в темний режим, фон, текст і кольори іконок у багатьох App також змінюються відповідно.

Наприклад, удень інтерфейс може мати світлий фон, а вночі перемикатися на темний фон.

![view](../../../Resource/025_view9.png)

Здатність автоматично налаштовувати відображення інтерфейсу відповідно до системного вигляду називається адаптацією зовнішнього вигляду.

## Попередній перегляд адаптації вигляду в Xcode

Перед тим як вивчати адаптацію вигляду, спочатку розглянемо, як у Xcode попередньо переглядати світлий і темний режими.

У нижній частині області `Canvas` натисніть кнопку `Device Settings`, щоб відкрити спливне вікно `Canvas Device Settings`.

![canvas](../../../Resource/025_view.png)

У спливному вікні можна побачити налаштування `Color Scheme`.

![canvas1](../../../Resource/025_view1.png)

Це означає:

- `Light Appearance` означає світлий режим.
- `Dark Appearance` означає темний режим.

Тут можна перемикати вигляд попереднього перегляду й перевіряти, як поточний інтерфейс виглядає у світлому та темному режимах.

## Показ двох варіантів вигляду поруч

Якщо потрібно одночасно переглянути світлий і темний режими, можна використати функцію `Variants`.

У нижній частині області `Canvas` натисніть кнопку `Variants` і виберіть `Color Scheme Variants`.

![canvas2](../../../Resource/025_view2.png)

Після вибору `Canvas` одночасно покаже попередній перегляд світлого й темного режимів.

![canvas3](../../../Resource/025_view3.png)

Так зручніше порівнювати відмінності інтерфейсу між двома варіантами вигляду.

Простіше кажучи:

Якщо потрібно лише тимчасово перемкнути світлий/темний режим, можна використати `Canvas Device Settings`.

Якщо потрібно одночасно переглянути обидва варіанти вигляду, можна використати `Color Scheme Variants`.

## Подання за замовчуванням адаптуються до вигляду

У SwiftUI багато системних подань за замовчуванням автоматично адаптуються до світлого й темного режимів.

Наприклад:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("Світло")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("деякий текст")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Результат відображення:

![canvas](../../../Resource/025_view4.png)

Як видно, у світлому режимі іконки й заголовки зазвичай відображаються темними.

У темному режимі іконки й заголовки автоматично стають світлими.

Це відбувається тому, що такі подання SwiftUI, як `Text` і `Image`, за замовчуванням налаштовують свої кольори відповідно до системного вигляду.

Інакше кажучи, якщо ми не задаємо фіксований колір вручну, SwiftUI частково обробляє адаптацію вигляду за нас.

## Фіксовані кольори не перемикаються автоматично

Важливо пам’ятати: якщо ми вручну задаємо фіксований колір, він не буде автоматично перемикатися між світлим і темним режимами.

Наприклад:

```swift
Text("деякий текст")
	.foregroundStyle(Color.gray)
```

Тут для тексту задано `Color.gray`, тому він завжди відображатиметься сірим.

Інший приклад:

```swift
Text("Заголовок")
    .foregroundStyle(Color.white)
```

Цей код відображатиме білий текст і у світлому, і в темному режимі.

Якщо фон також світлий, білий текст може бути важко прочитати.

Тому під час адаптації інтерфейсу до різних варіантів вигляду варто уникати випадкового жорсткого задання фіксованих кольорів, таких як `Color.white` і `Color.black`.

У багатьох випадках краще надавати перевагу системним семантичним стилям, наприклад:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

Це означає:

- `.primary` зазвичай використовується для основного вмісту.
- `.secondary` зазвичай використовується для другорядного вмісту.

Вони автоматично налаштовують відображення залежно від світлого або темного режиму.

## Показ різного вмісту залежно від вигляду

Іноді ми хочемо, щоб змінювалися не лише кольори, а й щоб у різних варіантах вигляду показувався різний вміст.

Наприклад:

- У світлому режимі показувати звичайну іконку лампочки.
- У темному режимі показувати іконку лампочки, що світиться.
- У світлому режимі показувати заголовок `Світло`.
- У темному режимі показувати заголовок `Темрява`.

У такому разі потрібно визначити, чи поточна система перебуває у світлому або темному режимі.

У SwiftUI можна отримати поточний режим вигляду через значення середовища `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Повний код:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "Світло" : "Темрява"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("деякий текст")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Результат відображення:

![view](../../../Resource/025_view5.png)

З результату попереднього перегляду видно, що одне й те саме подання в різних режимах вигляду показує не зовсім однаковий вміст.

У світлому режимі `Image` показує звичайну іконку лампочки, а заголовок показує `Світло`.

У темному режимі `Image` показує іконку лампочки, що світиться, а заголовок показує `Темрява`.

Інакше кажучи, тут змінюється не лише колір. Самі назва іконки й текст заголовка також змінюються.

Це тому, що ми не записали іконку й заголовок напряму всередині `body`, а обчислюємо різний вміст на основі `colorScheme`.

У цьому коді ми використовуємо дві обчислювані властивості:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "Світло" : "Темрява"
}
```

Коли `colorScheme` має значення `.light`, це означає, що поточний режим — світлий.

Коли `colorScheme` має значення `.dark`, це означає, що поточний режим — темний.

Тому ми можемо повертати різні назви іконок і текст заголовка відповідно до різних режимів вигляду.

Це також поширене застосування обчислюваних властивостей: обчислювати вміст, який подання має показати, на основі поточного стану.

## Розуміння @Environment

Це наш перший контакт із `@Environment`.

`@Environment` можна розуміти як читання значення із середовища SwiftUI.

Коли App працює, система надає багато типів інформації середовища, наприклад:

- Поточна мова
- Поточний режим вигляду
- Поточний напрямок макета
- Поточне налаштування розміру шрифту

Коли нам потрібно отримати поточний режим вигляду, можна прочитати `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Тут:

```swift
\.colorScheme
```

означає читання значення `colorScheme` із середовища.

```swift
private var colorScheme
```

означає збереження прочитаного значення у змінну `colorScheme`.

Ім’я змінної можна вибрати самостійно. Наприклад, можна також написати:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

Поки попередній `\.colorScheme` не змінюється, це означає, що читається поточний режим вигляду.

## Два поширені значення colorScheme

`colorScheme` зазвичай має два значення:

```swift
.light
.dark
```

Вони означають:

- `.light`: світлий режим
- `.dark`: темний режим

Наприклад:

```swift
private var titleName: String {
    colorScheme == .light ? "Світло" : "Темрява"
}
```

Цей код означає:

Якщо поточний режим світлий, умова `colorScheme == .light` істинна, тому повертається `"Світло"`.

Інакше повертається `"Темрява"`.

Отже, у світлому режимі:

```swift
Text(titleName)
```

Відображається текст:

```swift
Світло
```

У темному режимі відображається текст:

```swift
Темрява
```

Таким способом ми можемо показувати різний вміст залежно від різних варіантів вигляду.

## Приклад адаптації кольорів

Окрім тексту та іконок, іноді потрібно самостійно обробляти адаптацію кольорів.

Наприклад:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Текст")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

Результат відображення:

![view](../../../Resource/025_view6.png)

У цьому коді текст білий, а фон чорний.

У світлому режимі чорний фон досить помітний.

Однак у темному режимі, якщо фон усього інтерфейсу також чорний, цей чорний фон зіллється із системним фоном і виглядатиме менш чітко.

У цей момент можна перемикати колір тексту й колір фону відповідно до режиму вигляду:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("Текст")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

Результат відображення:

![view](../../../Resource/025_view7.png)

Тепер у світлому режимі використовується чорний фон і білий текст.

У темному режимі використовується білий фон і чорний текст.

Так, незалежно від вибраного вигляду, текст і фон мають чіткий контраст, і користувач може ясно бачити вміст.

## Використання Assets для адаптації кольорів

Окрім ручної перевірки через `colorScheme`, ми також можемо використовувати кольори ресурсів `Assets` для адаптації до світлого й темного режимів.

Такий спосіб більше підходить для керування спільними кольорами в App.

У проєкті Xcode знайдіть папку ресурсів `Assets`.

![assets](../../../Resource/025_color.png)

Відкрийте папку `Assets`, клацніть правою кнопкою миші порожню область і виберіть `New Color Set`, щоб створити новий кольоровий ресурс.

![assets](../../../Resource/025_color1.png)

Тут ми створюємо колір і називаємо його `redText`.

![assets](../../../Resource/025_color2.png)

Кольоровий ресурс може окремо задавати кольори для світлого й темного режимів.

![assets](../../../Resource/025_color3.png)

Після вибору відповідної області кольору можна змінити колір в інспекторі праворуч.

Натисніть кнопку `Hide or show the Inspectors` у правому верхньому куті, щоб відкрити область інспекторів.

Потім виберіть `Show the Attributes inspector`, унизу знайдіть `Show Color Panel` і відкрийте панель кольорів.

![assets](../../../Resource/025_color4.png)

Клацніть колір на панелі кольорів, і відповідна область кольору `redText` зміниться одночасно.

Тут ми задаємо для `redText`:

- У світлому режимі показувати червоний колір
- У темному режимі показувати зелений колір

![assets](../../../Resource/025_color5.png)

Повернувшись до коду SwiftUI, можна використати цей колір так:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

Тут:

```swift
Color("redText")
```

означає читання кольорового ресурсу з назвою `redText` із `Assets`.

Результат відображення:

![assets](../../../Resource/025_view8.png)

Можна побачити, що той самий `Color("redText")` у світлому й темному режимах відображає різні кольори.

Це показує, що кольори `Assets` самі також підтримують адаптацію вигляду.

## Різниця між colorScheme і Assets

І `colorScheme`, і `Assets` можуть реалізувати адаптацію вигляду, але вони підходять для різних сценаріїв.

Якщо потрібно лише перемикати текстовий вміст, назви зображень або назви `SF Symbols` відповідно до світлого й темного режимів, можна використовувати `colorScheme`.

Наприклад:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

Якщо колір часто використовується в App, наприклад як колір теми, колір фону картки або колір тексту, більше рекомендується використовувати кольори `Assets`.

Це тому, що кольори `Assets` можна повторно використовувати в багатьох поданнях, і їх легше змінювати пізніше.

Наприклад:

```swift
Color("redText")
```

Так потрібно лише змінити колір в `Assets`, і всі місця, де використовується цей колір, зміняться разом.

## Підсумок

У цьому уроці ми вивчили адаптацію вигляду для світлого й темного режимів.

Спочатку ми навчилися попередньо переглядати різні варіанти вигляду в `Canvas` Xcode, а також використовувати `Color Scheme Variants`, щоб одночасно бачити світлий і темний режими.

Потім ми дізналися, що такі подання SwiftUI, як `Text` і `Image`, за замовчуванням автоматично адаптують свої кольори відповідно до системного вигляду.

Однак якщо вручну задати фіксовані кольори, такі як `Color.white` або `Color.black`, ці кольори не будуть автоматично перемикатися.

Далі ми вивчили `@Environment` і `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Через `colorScheme` можна визначати, чи поточний режим світлий або темний, і показувати різні іконки, текст або кольори відповідно до різних варіантів вигляду.

Нарешті ми вивчили кольори `Assets`.

Кольори `Assets` можуть окремо задавати кольори для світлого й темного режимів, тому вони підходять для керування спільними кольоровими ресурсами в App.

Адаптація вигляду може зробити App кращим і вдень, і вночі, а також допомагає уникнути проблем, наприклад нечитабельного тексту або нечіткого фону.

У реальній розробці рекомендується спочатку використовувати стандартну адаптаційну поведінку SwiftUI та кольори `Assets`.

Коли в різних варіантах вигляду потрібно перемикати текст, іконки, зображення або обробляти невелику кількість спеціальних ефектів відображення, тоді використовуйте `colorScheme` для перевірки.

## Додаткові знання: задання розміру шрифту

Коли раніше ми вивчали `font`, ми використовували деякі системні стилі шрифту:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

Окрім використання системних стилів шрифту, таких як `.largeTitle` і `.callout`, можна також вручну задати розмір шрифту.

Наприклад:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

Тут:

```swift
.font(.system(size: 100))
```

означає встановлення розміру шрифту на `100 pt`.

Для іконок `SF Symbols` `font` також впливає на розмір іконки.

Тому, якщо потрібно зробити іконку більшою, можна задати розмір через `.font(.system(size:))`.

Наприклад:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

Важливо пам’ятати: хоча ручне задання розміру шрифту гнучкіше, для звичайного тексту не рекомендується масово використовувати фіксовані розміри шрифту.

Для звичайного тексту краще надавати перевагу системним стилям шрифту, таким як `.title`, `.headline`, `.body` і `.caption`.

Так легше адаптуватися до різних пристроїв і користувацьких налаштувань шрифту.
