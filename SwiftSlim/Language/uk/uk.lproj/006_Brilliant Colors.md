# Яскраві кольори

У цьому уроці ми вивчимо поширені в SwiftUI візуальні модифікатори, зокрема:

* колір
* колір переднього плану
* колір фону
* зміщення
* прозорість
* розмиття

Також ми розглянемо Safe Area (безпечну область).

Ці модифікатори використовуються для керування зовнішнім виглядом представлень, роблячи інтерфейс більш зрозумілим і багаторівневим.

## Колір

У SwiftUI можна налаштовувати колір тексту.

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

`.blue` означає синій колір і фактично є скороченим записом `Color.blue` (завдяки виведенню типу).

До поширених кольорів належать:

```swift
.black
.green
.yellow
.pink
.gray
...
```

Усі вони є статичними властивостями `Color`.

![Color](../../RESOURCE/006_color.png)

`Color` можна розуміти як тип кольору, а `.blue`, `.red` та інші — як його конкретні значення.

### Представлення Color

У SwiftUI `Color` також може відображатися як окреме представлення.

```swift
Color.red
    .frame(width: 100, height:100)
    .cornerRadius(10)
```

![Color](../../RESOURCE/006_color5.png)

Цей код створює червоний квадрат розміром 100×100.

Також можна зробити так, щоб увесь інтерфейс мав певний колір:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
    }
}
```

![Color](../../RESOURCE/006_color1.png)

Після запуску можна побачити, що червоний колір не заповнює весь екран: верхня та нижня частини iPhone усе ще залишаються білими. Тут ідеться про існування Safe Area (безпечної області).

## Safe Area (безпечна область)

Safe Area — це область, яку система резервує, щоб вміст не перекривався, зокрема:

1. верхня панель стану (час, заряд батареї)

2. нижній індикатор Home

3. область вирізу (notch) або Dynamic Island

![Color](../../RESOURCE/006_color3.png)

За замовчуванням SwiftUI обмежує вміст межами безпечної області, тому представлення не розтягується до самих країв екрана.

### Ігнорування Safe Area (безпечної області)

Якщо потрібно, щоб колір заповнював увесь екран, можна використати `ignoresSafeArea`:

```swift
Color.red
    .ignoresSafeArea()
```

Або використати `edgesIgnoringSafeArea`:

```swift
Color.red
    .edgesIgnoringSafeArea(.all)
```

![Color](../../RESOURCE/006_color4.png)

У такому разі представлення буде розтягуватися на весь екран.

Варто звернути увагу, що `edgesIgnoringSafeArea` — це старіший запис. Починаючи з iOS 14, рекомендовано використовувати `ignoresSafeArea`.

## Колір переднього плану

### Модифікатор foregroundStyle

У попередніх уроках ми вже вивчали `foregroundStyle` для задання кольору.

Наприклад:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

`foregroundStyle` — це нова система стилів, яка підтримує кольори, градієнти, матеріали тощо.

![Color](../../RESOURCE/006_color2.png)

### Модифікатор foregroundColor

`foregroundColor` також можна використовувати для встановлення кольору:

```swift
Text("Fang Junyu")
    .foregroundColor(.blue)
```

Спосіб використання такий самий, як і у `foregroundStyle`.

У найновіших версіях Xcode Xcode попереджає, що `foregroundColor` у майбутніх версіях iOS може бути застарілим, тому пріоритетно варто використовувати `foregroundStyle`.

## Фон

Якщо потрібно додати для представлення колір фону, можна використати `background`:

```swift
background(.red)
```

Наприклад, додамо фоновий колір для тексту:

```swift
Text("FangJunyu.com")
    .background(.orange)
```

![Color](../../RESOURCE/006_color6.png)

Коли ми виділяємо текст у Word, Chrome та інших застосунках, можна побачити подібний ефект фонового кольору.

![Color](../../RESOURCE/006_color16.png)

Якщо потрібно збільшити фон, слід поєднати його з `padding`:

```swift
Text("FangJunyu.com")
    .padding(10)
    .background(Color.orange)
```

![Color](../../RESOURCE/006_color7.png)

Тут потрібно звернути увагу на одне важливе правило:

У SwiftUI модифікатори будують представлення зверху вниз. Модифікатори, написані нижче, застосовуються до результату, створеного вище.

Тому:

```swift
.padding()
.background()
```

Це означає, що фон буде обгортати представлення вже після додавання внутрішніх відступів.

Якщо ж змінити порядок:

```swift
Text("FangJunyu.com")
    .background(Color.orange)
    .padding(10)
```

![Color](../../RESOURCE/006_color6.png)

Фон не збільшиться, тому що він не обгортає `padding`, який іде після нього.

## Приклад — чотирикутна гайка

Тепер створимо просте представлення у вигляді чотирикутної гайки.

![Color](../../RESOURCE/006_color8.png)

Спочатку створимо білий квадрат розміром 50 × 50:

```swift
Color.white
    .frame(width: 50, height: 50)
```

Якщо потрібно перетворити його на коло, можна використати `cornerRadius`:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
```

Коли радіус заокруглення дорівнює половині ширини та висоти, фігура стає колом.

Тепер додамо синій фон:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
    .padding(20)
    .background(.blue)
```

![Color](../../RESOURCE/006_color8.png)

Доданий `padding` збільшує зовнішню область, а `background` малює синій колір на цій зовнішній області.

Тепер ефект чотирикутної гайки готовий.

### Інший підхід

Окрім фонового кольору, для створення чотирикутної гайки можна також використати `ZStack`.

Раніше ми вже вивчали, що `ZStack` дозволяє накладати представлення одне на одне. Чотирикутну гайку також можна розуміти як поєднання кола та прямокутника, накладених одне на одне.

```swift
ZStack {
    Color.blue
        .frame(width: 70, height: 70)
    Color.white
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color8.png)

`ZStack` накладає представлення у вказаному порядку, і представлення, додані пізніше, відображаються поверх попередніх.

## Приклад — два кола, що перекриваються

Багато іконок складаються з накладання простих фігур, наприклад двох кіл, що перекриваються.

![Color](../../RESOURCE/006_color14.png)

Спочатку створимо два кола:

```swift
Color.red
    .frame(width: 50, height:50)
    .cornerRadius(25)
Color.orange
    .frame(width: 50, height:50)
    .cornerRadius(25)
```

![Color](../../RESOURCE/006_color11.png)

Оскільки нам потрібно, щоб два кола накладалися одне на одне, використаємо контейнер `ZStack`:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color12.png)

У цей момент два кола однакового розміру повністю перекриваються.

Нам потрібно, щоб вони перекривалися лише частково, а не повністю. Для цього можна використати `offset`, щоб створити ефект зміщення.

## Зміщення

`offset` змінює лише позицію відображення представлення, але не впливає на обчислення компонування батьківського контейнера.

Спосіб використання:

```swift
.offset(x:y:)
```

`x` — це зміщення по горизонталі, `y` — зміщення по вертикалі.

Додатні значення означають зміщення вправо / вниз, від’ємні — вліво / вгору.

Використаймо `offset`, щоб два кола частково перекривалися:

```swift
ZStack {
    Color.red
        .frame(width: 50, height: 50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color13.png)

Позиція червоного кола в макеті не змінилася, але його зображення змістилося на 25 пунктів ліворуч. Тому два кола створюють візуальний ефект часткового перекриття.

## Прозорість

У SwiftUI `opacity` використовується для налаштування прозорості представлення.

Базове використання:

```swift
.opacity(0.5)
```

Параметр `opacity` має діапазон від 0.0 до 1.0, де:

* 0 означає повну прозорість
* 1 означає повну непрозорість

Можна використати `opacity`, щоб налаштувати прозорість помаранчевого кола:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
```

![Color](../../RESOURCE/006_color14.png)

Коли для помаранчевого кола встановлено `opacity(0.8)`, його непрозорість становить 80%. Коли два кола перекриваються, у зоні перекриття виникає ефект змішування кольорів.

## Розмиття

У SwiftUI можна використати `blur` для створення ефекту розмиття:

```swift
.blur(radius:10)
```

`radius` визначає радіус розмиття: що більше значення, то сильніший ефект розмиття.

Можна додати ефект розмиття до двох кіл:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x:-35)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
.blur(radius:10)
```

![Color](../../RESOURCE/006_color15.png)

У підсумку ми бачимо два дуже розмиті кола.

## Підсумок

У цьому уроці ми розглянули поширені в SwiftUI візуальні модифікатори та вивчили, як за допомогою модифікаторів керувати кольором, положенням і візуальними ефектами представлень.

На конкретних прикладах можна побачити практичну роль різних візуальних модифікаторів у побудові інтерфейсу, а також зрозуміти принцип роботи безпечної області.

Усе це є дуже базовими модифікаторами, тому варто більше практикуватися та використовувати їх, щоб у реальній розробці точніше керувати виглядом інтерфейсу.

### Післяурочні вправи

* Додайте до зображення ефекти прозорості та розмиття
* Налаштуйте три кола з різним рівнем прозорості, накладені одне на одне
* Створіть фонове зображення, яке заповнює весь екран і ігнорує безпечну область
* Використайте `offset` для налаштування положення кількох представлень

Мета вправ — не запам’ятати API, а спостерігати за зв’язком між візуальними змінами та поведінкою компонування.
