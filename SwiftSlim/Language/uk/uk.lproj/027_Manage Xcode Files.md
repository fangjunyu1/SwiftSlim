# Керування файлами Xcode

У цьому уроці ми вивчимо поширені операції з файлами в Xcode.

У попередніх уроках більшу частину коду ми писали в `ContentView`. Так зручніше спостерігати за кодом і результатом відображення.

Наприклад:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

Але в реальному App зазвичай не пишуть увесь код в одному файлі.

Коли сторінок, компонентів і функцій стає дедалі більше, потрібно розділяти різні представлення на різні файли SwiftUI.

Наприклад:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

Так структура коду стає зрозумілішою, а подальше обслуговування й редагування — зручнішими.

## Навіщо керувати файлами

У SwiftUI сторінку або компонент зазвичай можна написати як окремий файл представлення.

Наприклад:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

Тут `CardView` — це представлення SwiftUI.

Якщо всі представлення писати в `ContentView`, код ставатиме дедалі довшим і дедалі складнішим для читання.

Тому нам потрібно навчитися створювати файли, перейменовувати їх, видаляти, а також використовувати папки для керування різними файлами коду.

## Створення файлу SwiftUI

В Xcode зазвичай є два поширені способи створити файл SwiftUI:

Перший — створити порожній файл, а потім самостійно написати код.

Другий — створити файл із шаблону SwiftUI, щоб Xcode автоматично згенерував базовий код.

Для початківців більше рекомендується використовувати файл шаблону SwiftUI, тому що він автоматично створює базову структуру.

### Створення порожнього файлу

У зоні Navigator ліворуч в Xcode клацніть правою кнопкою миші порожнє місце або папку й виберіть `New Empty File`.

Місце дії показано нижче:

![xcode](../../../Resource/027_view1.png)

Після створення порожнього файлу Xcode зазвичай створює безіменний файл Swift, наприклад `Untitled.swift`.

Ми можемо перейменувати його на `TestView.swift`.

У проєкті SwiftUI, якщо цей файл використовується для показу представлення, зазвичай рекомендується додавати `View` у кінці назви.

Наприклад:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

Це не обов’язкове правило, але воно дозволяє з першого погляду зрозуміти, що це файл представлення.

Після створення порожнього файлу в ньому можуть бути лише стандартні коментарі, або він може бути порожнім. Це залежить від версії Xcode і способу створення.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

На цьому етапі потрібно вручну написати код SwiftUI:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

Так створюється простий файл представлення SwiftUI.

### Перейменування файлу

Якщо після створення файлу потрібно змінити його назву, виберіть файл у зоні Navigator ліворуч в Xcode, а потім натисніть клавішу Return `⏎`.

![xcode](../../../Resource/027_view2.png)

Після цього можна змінити назву файлу.

Варто звернути увагу: перейменування файлу не змінює автоматично назву представлення в коді.

Наприклад, після зміни назви файлу на `ProfileView.swift` у коді все ще може залишатися `struct TestView: View`.

Це не обов’язково спричинить помилку, але легко може викликати плутанину.

Тому рекомендується зберігати узгодженість між назвою файлу й назвою представлення. Наприклад, якщо файл називається `ProfileView.swift`, назву представлення також варто зробити `ProfileView`.

## Створення файлу шаблону SwiftUI

Окрім створення порожнього файлу, можна також напряму створити файл шаблону SwiftUI.

У зоні Navigator ліворуч в Xcode клацніть правою кнопкою миші порожнє місце або папку й виберіть `New File from Template...`.

Місце дії показано нижче:

![xcode](../../../Resource/027_view3.png)

У вікні, що з’явиться, зверху можна вибрати різні платформи, наприклад `iOS`, `macOS` тощо.

Тут достатньо залишити платформу, що відповідає поточному проєкту. Наприклад, якщо ми створюємо iOS App, вибираємо `iOS`.

![xcode](../../../Resource/027_view4.png)

Потім унизу вибираємо `SwiftUI View`.

![xcode](../../../Resource/027_view12.png)

Далі натисніть `Next`, введіть назву файлу й збережіть його.

Наприклад, створимо файл `Test2View.swift`.

Xcode автоматично згенерує код, схожий на наведений нижче:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

Порівняно з порожнім файлом, файл шаблону автоматично містить `import SwiftUI`, `struct`, `body` і `#Preview`.

Тому для початківців використовувати файл шаблону SwiftUI простіше, а також менша ймовірність забути написати потрібний код.

**Підказка**: окрім створення файлу через правий клік, можна також натиснути кнопку з плюсом у нижній частині Navigator ліворуч в Xcode, щоб створити файл або папку.

![xcode](../../../Resource/027_view5.png)

## Створення папки

Коли файлів стає дедалі більше, для керування ними можна використовувати папки.

У зоні Navigator ліворуч в Xcode клацніть правою кнопкою миші порожнє місце або папку й виберіть `New Folder`.

Після створення папки можна перетягнути до неї пов’язані файли.

![xcode](../../../Resource/027_view7.png)

Папки здебільшого використовуються для впорядкування структури проєкту й не змінюють спосіб використання самого представлення.

Якщо назва представлення правильна, його все одно можна використовувати в інших файлах через `TestView()`、`Test2View()` тощо.

## Видалення файлу або папки

Якщо потрібно видалити файл або папку, виберіть відповідний файл у зоні Navigator ліворуч в Xcode, потім клацніть правою кнопкою миші й виберіть `Delete`.

Також можна вибрати файл і натиснути клавішу видалення `⌫`.

Xcode покаже вікно підтвердження видалення.

![xcode](../../../Resource/027_view10.png)

Вибір `Move to Trash` означає перемістити файл у кошик.

Якщо ви бачите `Remove Reference`, це означає лише видалення посилання з проєкту Xcode, але не видалення початкового файлу з диска.

Для початківців, якщо ви просто видаляєте звичайний файл, який щойно створили самостійно, зазвичай достатньо вибрати `Move to Trash`.

## Відображення кількох представлень SwiftUI

Після того як ми навчилися створювати файли, можна розділяти різні представлення на різні файли.

Наприклад, створимо два файли SwiftUI:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` використовується для відображення імені автора:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` використовується для відображення фону картки:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

Потім можна використати ці два представлення в `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

Після запуску можна побачити, що `CardView` і `AuthorView` відображаються разом:

![xcode](../../../Resource/027_view11.png)

У цьому результаті `CardView` слугує фоном, а `AuthorView` відображається зверху, тому що вони розміщені в `ZStack`.

У `ContentView` ми використовуємо ці два представлення через `CardView()` і `AuthorView()`. `()` після назви означає створення й використання цього представлення.

Це поширене вкладення представлень у SwiftUI: в одному представленні можна продовжувати використовувати інші представлення.

## Файл входу

Насамкінець ще раз пригадаємо, звідки App починає показувати інтерфейс.

У проєкті SwiftUI зазвичай є файл входу:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Тут `@main` означає, що це місце входу App.

У `WindowGroup` ми бачимо:

```swift
ContentView()
```

Це означає, що після запуску App спочатку показує `ContentView`.

Потім у `ContentView` знову відображається:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Тому ієрархію представлень можна зрозуміти так:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

Якщо змінити представлення, що показується у файлі входу, на `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

Тоді після запуску App відображатиметься лише `CardView`.

Тобто `WindowGroup` у коді входу визначає, яке представлення буде показане першим після запуску App.

А всередині цього представлення можна продовжувати показувати інші представлення.

## Підсумок

У цьому уроці ми вивчили поширені операції керування файлами в Xcode.

Ми дізналися, як створювати файли SwiftUI, як перейменовувати файли, як створювати папки, а також як видаляти файли.

Водночас ми вивчили важливе поняття: представлення SwiftUI можна розділяти на різні файли, а потім використовувати в інших представленнях.

Наприклад:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Тут `CardView()` і `AuthorView()` — це використання інших представлень у `ContentView`.

Насамкінець ми також повторили логіку відображення файлу входу.

Коли App запускається, він спочатку переходить до коду входу, позначеного `@main`, а потім показує представлення в `WindowGroup`.

Це допомагає зрозуміти: різні файли SwiftUI не є ізольованими. Вони можуть поєднуватися між собою й зрештою формувати повний інтерфейс App.
