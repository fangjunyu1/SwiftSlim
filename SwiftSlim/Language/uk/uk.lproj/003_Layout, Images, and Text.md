# Макет, зображення і текст

У попередньому уроці ми розглядали код ContentView:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

У цьому уроці ми вивчимо механізм стандартного макета SwiftUI, а також способи використання зображень і тексту. На основі попереднього уроку ми ще глибше зрозуміємо структуру view та спосіб їх відображення. Ці знання допоможуть нам створювати базові макети інтерфейсу.

## Стандартний механізм макета в SwiftUI

Коли ми переглядаємо ContentView, можна помітити, що іконка і текст відображаються по центру, а не починаються від верхньої частини.

![Swift](../../RESOURCE/003_view.png)

За замовчуванням Stack-контейнери мають вирівнювання .center, тому дочірні view зазвичай показуються по центру.

### Вирівнювання Alignment

Очевидно, що вирівнювання по центру — це лише один із варіантів. Якщо нам потрібно вирівнювання по лівому або правому краю, слід використовувати alignment для керування розташуванням view.

```swift
alignment
```

У SwiftUI вирівнювання зазвичай з’являється у двох сценаріях:

**1. Параметр alignment у Stack-контейнері**

Наприклад, вирівняємо іконку і текст у ContentView по лівому краю:

```swift
VStack(alignment: .leading) {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
```

![Swift](../../RESOURCE/003_alignment.png)

alignment у VStack керує вирівнюванням по горизонталі.

Способи вирівнювання:

```swift
VStack(alignment: .leading)
VStack(alignment: .center)
VStack(alignment: .trailing)
```

![Swift](../../RESOURCE/003_alignment1.png)

HStack виконує горизонтальне розташування, тому alignment керує вирівнюванням по вертикалі:

```swift
HStack(alignment: .top)
HStack(alignment: .center)
HStack(alignment: .bottom)
```

![Swift](../../RESOURCE/003_alignment2.png)

ZStack виконує накладання елементів один на один, тому alignment може керувати вирівнюванням як по горизонталі, так і по вертикалі:

```swift
ZStack(alignment: .leading)
ZStack(alignment: .trailing)
ZStack(alignment: .top)
ZStack(alignment: .topLeading)
ZStack(alignment: .topTrailing)
ZStack(alignment: .center)
ZStack(alignment: .bottom)
ZStack(alignment: .bottomLeading)
ZStack(alignment: .bottomTrailing)
```

Якщо явно не вказати alignment, то VStack, HStack і ZStack за замовчуванням використовують .center.

**2. alignment усередині frame**

```swift
Text("Hello")
    .frame(width: 200, alignment: .leading)
```

Коли розмір, який задає frame, більший за власний розмір view, alignment визначає положення view всередині frame. Детальніше про frame ми поговоримо пізніше, а зараз достатньо мати базове уявлення.

### Spacer і механізм розподілу простору

alignment дозволяє впорядковувати view по горизонталі або вертикалі. Але якщо ми хочемо, щоб текст і зображення були розташовані по різних краях, одного лише вирівнювання вже недостатньо.

Наприклад, якщо ми хочемо зробити верхню частину сайту [NHK](https://www3.nhk.or.jp/nhkworld/zh/news/), де зліва розташована іконка сайту NHK, а справа — іконка меню сайту.

![Swift](../../RESOURCE/003_alignment3.png)

Якщо використовувати лише alignment, іконка NHK і іконка меню опиняться лише з одного боку. Неможливо розмістити дві іконки по лівому й правому краю одночасно, тому тут потрібен Spacer для розподілу вільного простору.

Spacer — це гнучке view для макета, яке автоматично заповнює весь залишковий простір.

Спосіб використання:

```swift
Spacer()
```

Наприклад:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Spacer()
    Text("Hello, world!")
}
```

Коли ми додаємо Spacer між Image і Text, він займає весь вільний простір і розсуває Image та Text до верхнього і нижнього краю відповідно.

![Swift](../../RESOURCE/003_view1.png)

Якщо Spacer кілька:

```swift
VStack {
    Text("Hello, world!")
    Spacer()
    Text("FangJunyu")
    Spacer()
    Text("fangjunyu.com")
    Spacer()
}
```

Тоді вільний простір буде розподілено між Spacer порівну.

![Swift](../../RESOURCE/003_spacer.png)

## Відображення зображення Image і керування його розміром

View Image переважно використовується для показу зображень. Іконки SF Symbols, які ми вивчали в минулому уроці, — це лише один із варіантів використання Image.

Спосіб використання:

```swift
Image("imageName")
```

У подвійних лапках вказується назва зображення без розширення файлу.

### Показ зображення

Спочатку підготуємо одне зображення.

![Swift](../../RESOURCE/003_img.jpg)

У Xcode виберіть папку ресурсів Assets і перетягніть зображення в Assets.

![Swift](../../RESOURCE/003_img1.png)

Потім у ContentView використайте Image для показу зображення:

```swift
struct ContentView: View {
    var body: some View {
        Image("004_img")
    }
}
```

![Swift](../../RESOURCE/003_img2.png)

Зверніть увагу: Image у SwiftUI не підтримує відтворення GIF-анімації, воно може показувати лише статичний кадр.

### Керування розміром зображення

У SwiftUI Image за замовчуванням показується у своєму оригінальному розмірі. Якщо потрібно змінити розмір відображення, спочатку слід використати resizable, щоб зробити зображення масштабованим, а потім використати frame для задання розміру в макеті.

```swift
Image("image")
    .resizable()
    .frame(width: 200, height: 200)
```

### Модифікатор resizable

Модифікатор resizable дозволяє зображенню масштабуватися в макеті, а не залишатися завжди у вихідному розмірі.

```swift
.resizable()
```

Лише після додавання resizable() frame дійсно може змінити видимий розмір зображення.

Якщо пропустити resizable:

```swift
Image("image")
    .frame(width: 50, height: 50)
```

Тоді frame лише надасть простір для макета, але фактичний розмір самого зображення не зміниться.

### Модифікатор frame

frame(width:height) використовується для задання ширини та висоти view.

Базовий спосіб використання:

```swift
.frame(width: 10,height: 10)
```

Наприклад, встановимо для зображення width = 300 і height = 100, тобто прямокутну форму.

```swift
Image("image")
    .resizable()
    .frame(width: 300, height: 100)
```

![Swift](../../RESOURCE/003_img3.png)

Можна також задавати лише ширину або лише висоту:

```swift
.frame(width: 200)
.frame(height: 100)
```

Комбінація resizable + frame дозволяє гнучко керувати тим, якого розміру зображення буде показано в інтерфейсі, зберігаючи можливість масштабування.

### Масштабування: scaledToFit і scaledToFill

Коли співвідношення ширини і висоти, задане у frame, не збігається зі співвідношенням самого зображення, картинка може розтягуватися і деформуватися.

Якщо ми хочемо, щоб зображення зберігало свої пропорції й водночас підлаштовувалося під доступний простір макета, можна використати scaledToFit або scaledToFill.

**scaledToFit**

scaledToFit зберігає початкове співвідношення сторін зображення і масштабує його так, щоб воно повністю вмістилося у доступний простір без обрізання:

```swift
.scaledToFit()
```

або

```swift
.aspectRatio(contentMode: .fit)
```

Такий спосіб підходить для випадків, коли потрібно повністю показати зображення й не допустити деформації.

Якщо для всіх зображень задавати однакові width і height, то деякі з них неминуче будуть розтягуватися.

Наприклад:

```swift
Image("image")
    .resizable()
    .frame(width: 300, height: 200)
```

Якщо не вказати спосіб масштабування, зображення не зможе відобразитися зі збереженням своїх оригінальних пропорцій.

![Swift](../../RESOURCE/003_img4.png)

Додавання scaledToFit дозволяє зберегти початкові пропорції зображення.

```swift
Image("image")
    .resizable()
    .scaledToFit()
    .frame(width: 300, height: 200)
```

![Swift](../../RESOURCE/003_img5.png)

**scaledToFill**

scaledToFill також зберігає пропорції зображення, але заповнює весь доступний простір. Якщо пропорції не збігаються, зайва частина зображення буде обрізана:

```swift
.scaledToFill()
```

або

```swift
.aspectRatio(contentMode: .fill)
```

Такий спосіб підходить для сценаріїв, де зображення має повністю покривати область, наприклад для фонових зображень або banner. Особливо добре підходить для використання зображення як фону.

## Текст

У SwiftUI Text використовується для показу тексту.

Базовий спосіб використання:

```swift
Text("FangJunyu")
```

Ми вже знайомилися з Text у попередньому уроці, а в цьому уроці продовжимо вивчати, як керувати розміром шрифту і його насиченістю, щоб текст виглядав виразніше у view.

### Розмір шрифту

За допомогою модифікатора font можна керувати розміром тексту:

```swift
.font(.title)
```

Наприклад:

```swift
Text("FangJunyu")
    .font(.largeTitle)
Text("FangJunyu")
    .font(.callout)
```

![Swift](../../RESOURCE/003_txt.png)

Поширені розміри шрифтів (від більшого до меншого):

```swift
.largeTitle
.title
.title2
.title3
.headline
.body
.subheadline
.callout
.footnote
.caption
.caption2
```

### Товщина шрифту

Якщо потрібно зробити текст жирнішим, можна використати модифікатор fontWeight:

```swift
.fontWeight(.bold)
```

Наприклад:

```swift
Text("FangJunyu")
    .font(.title)
    .fontWeight(.bold)
Text("FangJunyu")
    .font(.title)
```

![Swift](../../RESOURCE/003_txt1.png)

Поширені значення товщини шрифту (від тонкого до товстого):

```swift
.ultraLight
.thin
.light
.regular
.medium
.semibold
.bold
.heavy
.black
```

font керує розміром шрифту, а fontWeight — товщиною. Їх можна комбінувати, щоб зробити текст виразнішим.

## Підсумок і практика

На цей момент ми вже вивчили базові знання SwiftUI: стандартний макет, Spacer, Image і Text. Цього вже достатньо, щоб створювати прості view.

Наприклад: [Google](https://www.google.com/)

![Swift](../../RESOURCE/003_google.png)

Інтерфейс Google дуже чистий і простий. У ньому є зображення і текст, тому ми можемо спробувати проаналізувати його структуру з точки зору SwiftUI:

1. Увесь інтерфейс можна розділити на три частини: логотип Google, поле пошуку і підказковий текст. Для цього можна використати VStack для вертикального розташування.
2. Логотип Google — це зображення, отже для нього можна використати Image.
3. Поле пошуку містить поле вводу й іконку. Якщо поки що не враховувати поле вводу, для іконки пошуку можна використати Image.
4. Текст підказки можна показати через Text, для горизонтального розташування тексту можна використати HStack, а колір шрифту можна задати через foregroundStyle.

Практикуючи ці знання, ми вже можемо створювати прості view та глибше розуміти Image, Text і пов’язані з ними модифікатори.
