# Список фільмів

У цьому уроці ми створимо список фільмів.

![movie](../../Resource/021_movie.png)

У списку буде показано постер, назву, режисера та рейтинг кожного фільму.

У цьому прикладі ми вивчимо дуже важливе поняття: структуру `struct`. Вона допомагає нам об'єднати кілька відомостей про один фільм в одне ціле. Крім цього, ми також познайомимося з `UUID()`, `ForEach`, розділювальною лінією `Divider`, а також з тим, як використовувати власні об'єкти для керування даними.

Ці знання дуже часто трапляються в подальшій розробці на SwiftUI. Після цього уроку ви не лише зможете зробити список фільмів, а й почнете розуміти, «як відображати групу даних як інтерфейс».

## Один фільм

Спочатку ми можемо почати з інтерфейсу одного окремого фільму.

![movie](../../Resource/021_movie1.png)

Макет цього інтерфейсу в основному складається з двох частин: ліворуч постер фільму, праворуч опис фільму.

### Постер фільму

Ліворуч відображається постер фільму, і для показу зображення ми можемо використати `Image`.

Наприклад:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

Цей код означає, що буде показано зображення з назвою `"The Shawshank Redemption"`.

Тут потрібно звернути увагу на те, що назва зображення має реально існувати в папці ресурсів `Assets`, інакше інтерфейс не зможе правильно показати це зображення.

![movie](../../Resource/021_movie2.png)

Функції цих модифікаторів такі:

- `resizable()` означає, що розмір зображення можна змінювати.
- `scaledToFit()` означає, що під час масштабування зберігаються початкові пропорції, щоб зображення не розтягувалося й не деформувалося.
- `frame(height: 180)` означає, що висота зображення встановлюється на 180.
- `cornerRadius(10)` означає, що зображенню задаються заокруглені кути радіусом 10.

Таким чином ми отримаємо постер фільму відповідного розміру та із заокругленими кутами.

![movie](../../Resource/021_movie3.png)

### Опис фільму

Праворуч показується опис фільму, що містить назву фільму, режисера та рейтинг.

![movie](../../Resource/021_movie4.png)

Для відображення цієї частини ми можемо використати `Text`:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

У цьому коді `.font` означає розмір шрифту, а `.fontWeight` означає товщину шрифту.

Щоб зробити відстань між «назвою фільму», «інформацією про режисера» та «інформацією про рейтинг» більш зрозумілою, тут на найзовнішньому рівні ми використовуємо `VStack` і задаємо:

```swift
spacing: 10
```

Це означає, що відстань між кожною групою вмісту всередині цього `VStack` становить 10.

Режисер і рейтинг кожен додатково обгортаються у свій `VStack`, тому що вони обидва належать до структури типу «заголовок + вміст». Тому рівні інтерфейсу стають зрозумілішими, а згодом нам також буде зручніше налаштовувати стиль.

Крім того, `VStack` за замовчуванням вирівнюється по центру. Щоб увесь текст був вирівняний ліворуч, ми задаємо:

```swift
alignment: .leading
```

Так частина з описом фільму виглядає більш охайно.

### Розділювальна лінія

Тепер, хоча вміст опису фільму вже рознесений за допомогою `spacing`, межа між різними частинами все ще недостатньо помітна.

У цей момент ми можемо додати розділювальну лінію:

```swift
Divider()
```

Наприклад:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Таким чином частина з описом фільму матиме більш помітний ефект розділення.

![movie](../../Resource/021_movie4.png)

`Divider` — це дуже простий, але дуже часто використовуваний view, завдання якого — відокремлювати різний вміст.

У `VStack` `Divider()` відображається як горизонтальна лінія.

У `HStack` `Divider()` відображається як вертикальна лінія.

Крім того, стиль `Divider` можна змінювати за допомогою `.frame`, `.background`, `.padding` та інших способів.

Наприклад:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

Це створить синю розділювальну лінію товщиною 2 із внутрішніми відступами ліворуч і праворуч.

![divider](../../Resource/021_divider.png)

### Повний інтерфейс

Нарешті, ми можемо використати `HStack`, щоб розташувати постер фільму та опис фільму ліворуч і праворуч.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

Тут `HStack(spacing: 20)` означає, що між лівою і правою частинами залишається відстань 20.

Важливо звернути увагу, що `Divider()`, як і згадані раніше `TextField` та `Slider`, за замовчуванням намагається зайняти якомога більше доступного простору.

Тому для зовнішнього `VStack` у правій частині опису ми задаємо фіксовану ширину:

```swift
.frame(width: 200)
```

Так ширина розділювальної лінії та текстової області буде однаковою, а загальний вигляд стане охайнішим.

На цьому етапі ми завершили інтерфейс відображення «одного фільму».

![movie](../../Resource/021_movie1.png)

## Збереження фільмів у масиві

Якщо ми хочемо відображати кілька фільмів один за одним в одному стилі, це означає, що для кожного фільму доведеться знову писати схожий код.

Наприклад:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

Хоча такий код і може дати потрібний результат, його ставатиме все більше, а підтримувати його буде дуже незручно.

Наприклад, якщо потрібно змінити відстань між усіма фільмами з `20` на `15`, або змінити ширину правої частини з `200` на
`220`, доведеться вручну змінювати кожен повторюваний фрагмент коду.

Очевидно, що це неефективний спосіб.

Раніше ми вже вивчали масиви, а також те, як за допомогою `ForEach` багаторазово відображати views на основі масиву.

Наприклад:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Такий спосіб дуже добре підходить для ситуацій, коли є «лише один вид даних», наприклад група назв зображень.

Однак фільм — це не лише одне значення. Він щонайменше містить:

1. Постер фільму
2. Назву фільму
3. Режисера
4. Рейтинг

Іншими словами, один фільм насправді є групою пов'язаних даних, а не одним окремим рядком.

Якщо використовувати лише масиви, то ми зможемо зберігати цю інформацію тільки окремо:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

Під час відображення доведеться покладатися на однаковий індекс, щоб зіставити їх один до одного:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

Проблема такого способу запису полягає в тому, що вартість підтримки дуже висока.

Оскільки дані одного фільму розкидані по кількох масивах, достатньо, щоб в одному масиві був один зайвий елемент, один пропущений або інший порядок, і результат відображення буде неправильним.

Особливо під час додавання або видалення фільмів вам доведеться одночасно змінювати кілька масивів, і дуже легко щось пропустити.

То як же об'єднати такі дані, як «назва фільму, режисер, рейтинг», в одне ціле?

Для цього потрібен `struct`.

## Визначення `struct`

У Swift `struct` означає «структура».

Ви можете розуміти її як «користувацький тип даних», який може об'єднувати кілька пов'язаних полів в одне ціле.

Базове використання:

```swift
struct StructName {
	let name: String
}
```

У цьому коді `struct` — це ключове слово, яке означає, що ми визначаємо структуру. `StructName` — це назва структури.

Вміст у фігурних дужках — це поля, які містить ця структура, і для кожного поля потрібно чітко вказати назву та тип.

Зазвичай назва `struct` починається з великої літери, наприклад `Movie`, `Student`, `UserInfo`. Це поширена конвенція іменування у Swift.

Ви можете просто уявити `struct` як порожню пакувальну коробку, у якій кожне поле — це заздалегідь підготовлене порожнє місце.

Поки ці місця не заповнені вмістом, це лише порожня коробка. Лише коли всі поля заповнені відповідними значеннями, вона стає повноцінною подарунковою коробкою.

### Екземпляр `struct`

Раніше ми лише визначили саму структуру, тобто підготували форму коробки.

Нам ще потрібно заповнити її конкретним вмістом, щоб отримати справжній «екземпляр», який можна використовувати.

Під час створення екземпляра зазвичай після назви структури додають `()`:

```swift
StructName(...)
```

У круглих дужках записується вміст полів, який потрібен цій структурі.

Наприклад:

```swift
StructName(name: "Fang Junyu")
```

Цей код означає: створити новий екземпляр за форматом структури `StructName` і присвоїти полю `name` значення `"Fang Junyu"`.

Коли ми заповнюємо весь потрібний вміст полів, це рівнозначно тому, що ми наповнили коробку.

У цей момент ми отримуємо повноцінний екземпляр структури.

### Доступ до властивостей `struct`

Коли ми створили екземпляр, то можемо використовувати «точковий синтаксис», щоб звертатися до його внутрішніх властивостей.

Базовий запис:

```swift
instance.propertyName
```

Наприклад:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

Тут `st` — це екземпляр структури, а `st.name` означає читання значення `name` у цього екземпляра.

Такий спосіб читання можна уявити так: коли ми пишемо `st.name`, це схоже на читання вмісту, який лежить у певному місці коробки.

Такий спосіб запису буде дуже часто зустрічатися в наступних інтерфейсах SwiftUI.

Наприклад, якщо у нас є структура студента, що містить ім'я, вік і клас, то під час відображення в інтерфейсі можна окремо читати:

```swift
student.name
student.age
student.className
```

Перевага цього способу в тому, що дані стають зрозумілішими, і ними також легше керувати.

### Розташування `struct`

Для початківців зазвичай зручно писати структуру поза `ContentView`.

Наприклад:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

Так структура буде зрозумілішою, а читати її буде легше.

Ви помітите, що сам `ContentView` насправді теж є `struct`.

### Структура `Movie`

Маючи попередню основу, ми можемо визначити структуру фільму:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

Це структура з назвою `Movie`, яка складається з трьох полів: `name`, `director` і `rating`, що означають відповідно назву фільму, режисера та рейтинг.

Таким чином, інформацію, пов'язану з одним фільмом, більше не потрібно розбивати на кілька масивів, а можна безпосередньо об'єднати в одне ціле.

Наприклад, ми можемо створити такий екземпляр фільму:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

Цей екземпляр представляє «один повний набір даних фільму».

### Збереження структур у масиві

Раніше ми говорили, що масив може зберігати лише один і той самий тип.

Тепер у нас уже є структура `Movie`, а отже масив може зберігати кілька `Movie`.

```swift
let lists: [Movie] = []
```

Наприклад:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

У цьому масиві `lists` зберігаються не рядки, а кілька екземплярів `Movie`.

Тобто кожен елемент масиву — це один повноцінний фільм.

Таким чином, коли потрібно показати список фільмів, ми можемо за допомогою `ForEach` послідовно читати кожен фільм.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

Тут `movie in` означає, що під час кожного проходу циклу з масиву береться один фільм і тимчасово називається `movie`.

Потім ми можемо використовувати:

```swift
movie.name
movie.director
movie.rating
```

щоб окремо прочитати назву, режисера і рейтинг цього фільму та показати їх в інтерфейсі.

## Помилка `ForEach`

До цього моменту ми вже завершили базовий запис масиву фільмів і `ForEach`.

Але якщо безпосередньо запустити такий код:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

з’явиться помилка:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

Сенс цієї помилки в тому, що коли ви пишете `id: \.self`, SwiftUI потрібно використовувати кожен елемент масиву як «унікальний ідентифікатор», щоб розрізняти їх.

Такий спосіб запису підходить для простих типів, як `String` і `Int`, тому що їх самих по собі порівняно легко розрізняти.

Але `Movie` — це наша власна структура, і `ForEach` не знає, як використовувати саму структуру як унікальний ідентифікатор, тому й виникає помилка.

Найпоширеніший спосіб вирішити цю проблему — зробити так, щоб `Movie` відповідала протоколу `Identifiable`.

## Протокол

У Swift протокол можна розуміти як різновид «правила» або «вимоги».

Це можна порівняти з угодою в міжнародній організації: якщо якась країна хоче вступити до міжнародної організації, їй зазвичай потрібно спочатку виконати певні правила цієї організації, оприлюднити деякі дані та дотримуватися спільних норм. Лише після виконання цих умов вона може приєднатися або брати участь у певних справах.

Протокол працює за подібною логікою.

Коли певний тип хоче відповідати якомусь протоколу, він має виконувати вимоги, які цей протокол встановлює. Лише після виконання цих вимог тип зможе використовувати відповідну функціональність.

Базовий запис:

```swift
struct Movie: Identifiable {
    // ...
}
```

Тут `: Identifiable` означає, що `Movie` відповідає протоколу `Identifiable`.

Якщо тип відповідає кільком протоколам, їх можна розділяти комами:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

Для `Identifiable` найважливіша вимога полягає в тому, що тип повинен мати `id`, який може ідентифікувати його самого.

Наприклад:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

Тут `id` подібний до номера посвідчення особи, який використовується для розрізнення кожного фільму.

Поки `id` у кожного фільму різний, SwiftUI може правильно розрізняти кожен елемент у масиві.

Наприклад:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

А потім використати у `ForEach`:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

Тоді помилки вже не буде.

Оскільки `Movie` вже відповідає `Identifiable`, ще більш поширений запис — просто опустити `id:`:

```swift
ForEach(lists) { movie in
    // ...
}
```

Тому що SwiftUI вже знає, що для розрізнення кожного елемента потрібно використовувати `movie.id`.

## Ідентифікатор розрізнення у `ForEach`

Нижче ми глибше зрозуміємо роль `id` у `ForEach`.

Наприклад:

```swift
ForEach(lists, id: \.self)
```

Тут це означає: використовувати сам елемент як ідентифікатор розрізнення.

А:

```swift
ForEach(lists, id: \.id)
```

означає: використовувати поле `id` елемента як ідентифікатор розрізнення.

Якщо якесь поле саме по собі може гарантувати унікальність, його також можна тимчасово використати.

Наприклад, якщо назва кожного фільму різна, то наступний запис також може нормально працювати з поточними даними:

```swift
ForEach(lists, id: \.name)
```

Але тут є прихована проблема: `name` не обов’язково завжди буде унікальним.

Припустимо, що пізніше ви додасте ще один фільм з такою ж назвою — тоді `name` вже не зможе точно розрізняти кожен елемент.

У такому разі код усе ще може компілюватися, але під час оновлення view, вставки, видалення чи зміни SwiftUI може помилково розпізнавати елементи, що призведе до неправильного відображення.

Тому, коли потрібно стабільно розрізняти дані, найкраще використовувати справді унікальне поле, тобто `id`.

## UUID

Хоча ручне написання `id: 1`, `id: 2` може вирішити проблему, ризик усе ж таки залишається: можна випадково написати дубльований `id`, і унікальна ідентифікація втратить свою силу.

Наприклад:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Так з’являться два однакових `id`, і унікальна ідентифікація перестане працювати.

Щоб уникнути ручних помилок, ми зазвичай використовуємо `UUID()`.

У Swift `UUID()` випадковим чином генерує 128-бітний ідентифікатор. Зазвичай він відображається як довгий рядок із літер і цифр.

Наприклад:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

Тут `UUID()` фактично створює екземпляр типу `UUID`; ви можете розуміти його як автоматично згенерований «унікальний номер».

Тому значення, яке створюється щоразу, майже ніколи не повторюється, і воно дуже добре підходить для використання як `id`.

Ми можемо змінити `Movie` так:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Цей код означає, що полю `id` у структурі `Movie` за замовчуванням буде присвоєне значення нового екземпляра `UUID`.

Іншими словами, щоразу, коли ми створюємо новий `Movie`, система автоматично спочатку генерує для нас унікальний `id`.

У `id` вже є значення за замовчуванням, тому при подальшому створенні екземплярів `Movie` нам більше не потрібно заповнювати `id` вручну.

Наприклад:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Хоча тут `id` не написано, на практиці цей екземпляр фільму все одно має власний `id`; просто це значення вже автоматично згенероване через `UUID()`.

Після використання цього способу нам більше не потрібно вручну заповнювати `id` для кожного фільму, що не лише зменшує обсяг коду, а й дає змогу уникати помилок через дубльовані id.

Наприкінці нам достатньо лише додати зверху зображення `Banner` та використати `ScrollView`, щоб вміст можна було прокручувати — і весь вигляд списку фільмів буде завершений.

## Підсумок

У цьому уроці ми вивчили дуже важливе поняття: `struct`.

За допомогою `struct` ми можемо об'єднати кілька пов'язаних полів одного фільму в одне ціле, замість того щоб розносити назву, режисера та рейтинг по кількох масивах.

Після об'єднання даних ми можемо використати `ForEach`, щоб відображати кожен фільм з масиву один за одним у view.

Одночасно ми зрозуміли й роль «ідентифікатора розрізнення» у `ForEach`. SwiftUI має знати, як розрізняти кожен елемент масиву, щоб правильно відображати та оновлювати view.

Тому ми зробили так, щоб `Movie` відповідала протоколу `Identifiable`, і надали їй унікальний `id`.

Щоб уникнути помилок при ручному заповненні `id`, ми додатково вивчили `UUID()`, який дозволяє системі автоматично генерувати унікальний ідентифікатор.

Після проходження цього уроку ви не лише завершили список фільмів, а й почали знайомитися з важливим підходом у SwiftUI: **спочатку впорядкувати дані, а потім на їх основі генерувати view.**

## Повний код

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
