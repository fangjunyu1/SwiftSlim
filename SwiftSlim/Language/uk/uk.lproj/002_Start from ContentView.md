## Підготовка до уроку

У цьому уроці ми почнемо з файлу ContentView.swift і системно познайомимося з базовою структурою SwiftUI, зокрема з такими темами:

## Знайомство з ContentView

Код ContentView:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Хоча коду тут небагато, він уже містить основну структуру SwiftUI.

### 1. Коментарі

На початку файлу:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

Це коментарі файлу, які використовуються для запису інформації про файл, зокрема назви файлу, назви проєкту, автора та часу створення.

У Swift для однорядкового коментаря використовується //:

```swift
// Це коментар
```

Коментарі підвищують читабельність коду та допомагають розробникам розуміти логіку програми.

У реальній розробці код інколи буває складним для розуміння. Якщо не писати коментарі, можна легко зіткнутися з ситуацією, коли вже через три дні не розумієш власний код.

Тому звичка самостійно додавати коментарі під час написання коду є доброю практикою розробки. За допомогою коментарів можна фіксувати логіку коду, що полегшує подальшу підтримку.

**Тимчасове вимкнення коду**

Коментарі також можна використовувати для тимчасового вимкнення коду, щоб допомогти собі під час пошуку проблем.

Наприклад:

```
A
B
C
```

A, B і C — це три фрагменти коду, і в одному з них є помилка. Ми можемо тимчасово закоментувати код, щоб знайти причину.

Спочатку додаємо коментар до A:

```
// A
B
C
```

Якщо після коментування A код почав працювати нормально, це означає, що проблема була в A.

Якщо після коментування A проблема залишається, можна далі закоментувати B і так далі, доки не буде знайдено проблемний код.

У процесі розробки виникає багато проблем, і в більшості випадків ми шукаємо їх причину саме через тимчасове коментування коду. Це допомагає точніше знайти проблемний фрагмент і виявити BUG.

У Xcode можна використовувати комбінацію клавіш:

```
Command ⌘ + /
```

щоб швидко додавати або прибирати коментарі.

### 2. Імпорт фреймворку SwiftUI

```swift
import SwiftUI
```

Цей рядок означає імпорт фреймворку SwiftUI.

Такі типи у SwiftUI, як View, Text, Image, VStack та інші, належать саме цьому фреймворку.

Якщо не імпортувати SwiftUI, Xcode покаже помилку:

```
Cannot find type 'View' in scope
```

Це означає, що компілятор не може розпізнати тип View.

### 3. Структура View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Коли ви вперше бачите цю структуру View, вона може здатися незнайомою, тому що тут є ключові слова struct, View, var, body, some.

Ми ще не вивчали ці ключові слова, тому зараз достатньо просто розуміти, що цей код створює view з назвою ContentView.

Ви можете уявити View як полотно для малювання, на якому ми можемо щось створювати, а інструментом для цього є SwiftUI.

Наприклад:

![Swift](../../RESOURCE/002_view.png)

На зображенні вище показано три сторінки, і насправді це три окремі View.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

Коли ми розробляємо App за допомогою SwiftUI, кожна сторінка є окремим View.

### 4. Код SwiftUI

Усередині View відображається код SwiftUI:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Цей фрагмент SwiftUI означає вертикальний макет, у якому показано іконку та текст.

![Swift](../../RESOURCE/002_view1.png)

#### Макет VStack

```swift
VStack { }  // вертикальний макет
```

VStack означає контейнер вертикального розташування. Внутрішні view розміщуються зверху вниз.

![Swift](../../RESOURCE/002_view8.png)

У SwiftUI є три поширені типи макетів:

* VStack —— вертикальне розташування
* HStack —— горизонтальне розташування
* ZStack —— накладання один на один (вісь Z)

```swift
HStack { }  // горизонтальне розташування
ZStack { }  // накладання
```

Схема розташування для кожного макета:

![Swift](../../RESOURCE/002_view2.png)

Наприклад, використаємо HStack для горизонтального розташування.

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Можна побачити, що іконка глобуса і текст відображаються по горизонталі.

![Swift](../../RESOURCE/002_view3.png)

Коли потрібно розташувати елементи горизонтально, використовується HStack. Коли потрібно накладати елементи один на один, використовується ZStack.

#### Image і SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Цей код показує іконку глобуса, яка походить із системи іконок Apple SF Symbols.

![Swift](../../RESOURCE/002_view9.png)

Зміст цього коду такий: показати іконку глобуса, великого розміру, з кольором акценту.

Окрім глобуса, ми можемо показувати й інші іконки.

Наприклад, показати рюкзак:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**Як показати інші іконки?**

Для цього потрібно використовувати офіційну бібліотеку системних іконок Apple — SF Symbols.

Відкрийте офіційний сайт Apple Developer і завантажте [SF Symbols](https://developer.apple.com/sf-symbols/).

![Swift](../../RESOURCE/002_sf.png)

Відкрийте застосунок SF Symbols.

![Swift](../../RESOURCE/002_sf1.png)

Ліворуч будуть категорії символів, а праворуч — відповідні іконки.

Натисніть правою кнопкою на іконку та виберіть "Копіювати 1 назву", і ця назва буде відповідним рядком.

Наприклад:

```
"globe"
"backpack"
"heart"
```

Якщо вставити скопійовану назву іконки в Image(systemName:), можна показати різні символи.

Зверніть увагу: кожна іконка SF Symbols має мінімальну підтримувану версію системи. Якщо версія системи надто стара, іконка може не відображатися. Сумісність потрібно перевіряти в застосунку SF Symbols.

#### Модифікатори

У SwiftUI модифікатор (modifier) — це спосіб змінити зовнішній вигляд або поведінку view.

Модифікатор можна уявити як одяг: у різному одязі зовнішній вигляд теж буде різним.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

imageScale і foregroundStyle — це модифікатори для view Image. Вони змінюють Image, не змінюючи сам його вміст.

**1. imageScale**

```swift
.imageScale(.large)
```

Він керує розміром іконки SF Symbols:

* .small
* .medium
* .large

![Swift](../../RESOURCE/002_view11.png)

Ми можемо змінювати різні варіанти, щоб символи SF Symbols відображалися з різним розміром.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

foregroundStyle керує кольором переднього плану.

.tint означає поточний акцентний колір середовища (accent color), за замовчуванням в iOS це синій.

Якщо ми хочемо змінити колір переднього плану на червоний:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### View Text

Text — це текстове view, яке використовується для показу рядка.

```swift
Text("Hello, world!")
```

Наприклад, показати моє ім’я:

```swift
Text("FangJunyu")
```

Зверніть увагу: рядок обов’язково має бути взятий у подвійні лапки "".

Ви можете спробувати показати своє ім’я, номер телефону чи інший текст.

#### Відступ padding

У SwiftUI padding використовується для додавання порожнього простору між вмістом view і його межами. Це належить до "внутрішнього відступу (padding / content inset)".

```swift
HStack {
    ...
}
.padding()
```

Цей код означає, що до view HStack додається стандартний системний відступ.

**Що таке padding?**

padding означає "порожній простір між вмістом view і його межами".

На зображенні нижче після додавання padding до синього HStack синя область ніби стискається всередину й візуально "стає трохи меншою".

![Swift](../../RESOURCE/002_view6.png)

**Стандартний відступ**

Модифікатор padding() за замовчуванням використовує рекомендований системою стандартний інтервал.

```swift
.padding()
```

На різних платформах і в різних контекстах це значення може відрізнятися, наприклад:

* в iOS зазвичай це приблизно 16 pt.
* у macOS або watchOS стандартний інтервал може бути іншим — більшим або меншим, залежно від дизайнерських правил платформи.

**Власний відступ**

Також можна задати окремі відступи для view.

1. Встановлення одного напрямку

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

Пояснення напрямків:

* .top: верхній відступ
* .bottom: нижній відступ
* .leading: передній край
* .trailing: задній край

![Swift](../../RESOURCE/002_view12.png)

Зверніть увагу: leading і trailing автоматично адаптуються до напрямку мови. Наприклад, у середовищі арабської мови (RTL) вони автоматично міняються місцями.

2. Встановлення кількох напрямків

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

Через масив можна одночасно вказати кілька напрямків. Детальніше про масиви буде в наступних уроках, а зараз достатньо просто знати такий запис.

3. Встановлення горизонтального або вертикального напрямку

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

Це еквівалентно:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**Прибрати відступ**

Якщо відступ узагалі не потрібен, можна використати .padding(0):

```swift
.padding(0)
```

Або просто видалити модифікатор padding:

```swift
// .padding()
```

### 6. Код попереднього перегляду Preview

```swift
#Preview {
    ContentView()
}
```

Цей фрагмент коду означає створення попереднього перегляду ContentView у Canvas.

Зверніть увагу: #Preview — це новий синтаксис, запроваджений у Swift 5.9 / Xcode 15. До цього використовувалася структура PreviewProvider.

**Що буде, якщо закоментувати Preview?**

Якщо закоментувати Preview:

```swift
// #Preview {
//    ContentView()
// }
```

Canvas більше не показуватиме жодного вмісту для відображення.

![Swift](../../RESOURCE/002_xcode.png)

Це означає, що саме #Preview керує показом preview view у Canvas.

Коли нам потрібно переглядати SwiftUI view у Xcode, ми додаємо код #Preview. Якщо попередній перегляд не потрібен, код #Preview можна закоментувати або видалити.

## Підсумок

Хоча у файлі ContentView.swift небагато коду, насправді він уже охоплює кілька ключових концепцій SwiftUI. Для початківців цей код може виглядати незвично, але через поступовий розбір його структури можна сформувати початкове розуміння SwiftUI.

Повернімося до того, що ми вивчили в цьому уроці. Спочатку ми розглянули коментарі //, які можна використовувати для пояснення логіки коду або для тимчасового вимкнення коду.

Далі ми дізналися, що файл SwiftUI обов’язково має імпортувати фреймворк SwiftUI:

```swift
import SwiftUI
```

Якщо не імпортувати цей фреймворк, компілятор не зможе розпізнати такі типи, як View.

Потім ми познайомилися з базовою структурою SwiftUI view:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Тут ContentView — це назва view.

Ми також вивчили три поширені контейнери макетів: VStack (вертикальне розташування), HStack (горизонтальне розташування) і ZStack (накладання).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Цей код означає, що відображається контейнер із вертикальним розташуванням, усередині якого є іконка та текст.

Image може показувати іконки SF Symbols, а модифікатори керують розміром і кольором іконки.

View Text може показувати текстовий вміст.

padding — це відступ, який додає прозорий простір навколо view.

І нарешті, #Preview — це preview view, яке дозволяє показати попередній перегляд у Canvas.

### Вправи після уроку

Для початківців матеріал цього уроку може здатися дещо складним. Тому можна краще закріпити знання за допомогою додаткових вправ:

Xcode одразу показує доступні варіанти:
