# Додаток Fill Light

У цьому уроці ми створимо дуже цікавий додаток fill light. Коли настає ніч, ми можемо змусити екран телефону показувати різні кольори й використовувати його як просте заповнювальне світло.

Цей додаток fill light може перемикати кольори натисканням на екран, а також регулювати яскравість за допомогою повзунка.

У цьому прикладі ми вивчимо, як використовувати `brightness` для зміни яскравості view, `onTapGesture` для додавання жесту натискання до view, а також елемент керування `Slider`.

## Відображення кольору

Спочатку зробімо так, щоб view показував один колір.

У SwiftUI `Color` не лише означає колір, а й може відображатися як view:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

Тут `Color.red` означає червоний view. `.ignoresSafeArea()` змушує кольоровий view заповнювати весь екран, тому це більше схоже на справжній ефект fill light.

![Color](../../Resource/018_color1.png)

### Масив кольорів та index

Зараз відображається лише один колір, але fill light зазвичай не обмежується лише одним кольором. Він також може показувати синій, жовтий, фіолетовий, білий та інші кольори.

Ми хочемо, щоб під час натискання на екран можна було перемикатися між різними кольорами. Для цього можна покласти ці кольори в масив і керувати ними разом:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

Масив підходить для збереження «групи даних одного типу». Тут кожен елемент масиву є `Color`.

Якщо ми хочемо показати певний колір, можемо використати index:

```swift
colors[0]
```

Це означає, що ми читаємо колір з index `0` у масиві, тобто перший колір.

Тепер код можна записати так:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

Таким чином на екрані відображатиметься перший колір із масиву, тобто червоний.

### Керування кольором через index

Якщо нам потрібно перемикатися між різними кольорами, нам слід використовувати змінну для керування index, а не записувати його жорстко.

Можна використати `@State`, щоб оголосити змінну для збереження index:

```swift
@State private var index = 0
```

Тут `index` означає index поточного кольору.

Коли `index` змінюється, SwiftUI знову обчислює інтерфейс і оновлює вміст, що відображається.

Потім замінимо початковий `colors[0]` на:

```swift
colors[index]
```

Таким чином колір, який показується у view, визначається значенням `index`.

Тепер код стає таким:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

Коли `index` змінюється, `colors[index]` також починає показувати інший колір.

Наприклад:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

Важливо пам’ятати, що `index` не може перевищувати максимальний index масиву, інакше виникне помилка виходу за межі масиву.

## Жест натискання

Тепер ми можемо показувати різні кольори залежно від `index`, але ще не можемо перемикати їх натисканням.

У попередньому прикладі «Quote Carousel» ми використовували `Button`, щоб керувати перемиканням цитат.

А цього разу ми хочемо перемикати кольори, «натискаючи на всю кольорову область», тому `onTapGesture` підходить краще.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

Коли ми натискаємо на кольоровий view, виконується:

```swift
index += 1
```

Це означає, що `index` збільшується на `1`. Після збільшення index `colors[index]` показує наступний колір із масиву.

### onTapGesture

`onTapGesture` — це gesture modifier, який може додати до view дію натискання.

Базове використання:

```swift
.onTapGesture {
    // code
}
```

Наприклад:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

Коли ми натискаємо на цей червоний view, код у фігурних дужках виконується, а в консолі з’являється такий вивід:

```swift
Click color
```

За допомогою `onTapGesture` ми можемо задати, що має статися після натискання на певний view.

### Відмінність від `Button`

Раніше ми вже вивчали view `Button`. І `Button`, і `onTapGesture` можуть обробляти натискання, але їхні сценарії використання не зовсім однакові.

`onTapGesture` краще підходить для «додавання можливості натискання до вже існуючого view», наприклад до `Color`, `Image`, `Text` чи інших звичайних view.

`Button` же краще підходить для чітко вираженої кнопки, як-от «Підтвердити», «Надіслати» чи «Видалити».

У цьому додатку fill light ми хочемо зробити функцію перемикання кольорів максимально простою, тому для зміни кольорів натисканням на всю кольорову область можна використати `onTapGesture`.

## Проблема з index

Тепер ми можемо натискати на екран і перемикатися між різними кольорами.

Але тут є важлива проблема: **index може вийти за межі масиву**.

Наприклад:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

Якщо постійно натискати на екран, `index` зрештою стане `4`, і тоді виникне помилка “index out of range”.

Це тому, що масив `colors` має `4` елементи, але index починається з `0`, тому допустимий діапазон index — `0 - 3`, а не `4`.

Якщо звернутися до `colors[4]`, станеться «вихід index за межі».

У поточному коді після кожного натискання на екран `index` автоматично збільшується на `1`. Якщо цього не обробити, він рано чи пізно обов’язково вийде за межі.

Тому під час натискання на екран нам потрібно перевіряти index: якщо це останній колір, треба повернутися до першого, інакше збільшити його на `1`.

Це можна зробити за допомогою `if`:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

У цьому коді `colors.count` означає кількість елементів у масиві.

У поточному масиві є `4` кольори, отже:

```swift
colors.count // 4
```

Але максимальний index — не `4`, а `3`, тому що index починається з `0`.

Отже, останній index записується так:

```swift
colors.count - 1
```

Тобто:

```swift
4 - 1 = 3
```

Ця логіка означає, що якщо поточний index уже відповідає останньому кольору, ми скидаємо його на `0`, інакше збільшуємо на `1`.

Так кольори можуть перемикатися по колу.

### Оптимізація index

Якщо ми хочемо зробити код коротшим, можна використати ternary operator:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

Цей код означає: якщо `index == colors.count - 1` істинне, повертається `0`. Якщо ні, повертається `index + 1`.

Наприкінці цей результат знову присвоюється `index`.

Тепер ми можемо реалізувати ефект перемикання кольорів.

Повний код:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

Тепер ми можемо натискати на екран і перемикатися між різними кольорами, і базовий додаток fill light уже готовий.

## Відображення назви кольору

Ми також можемо додати групу текстів, що відповідають кольорам, щоб під час перемикання на екрані одночасно показувалася назва поточного кольору.

Наприклад:

- коли колір червоний, показуємо `Red`
- коли колір синій, показуємо `Blue`
- коли колір жовтий, показуємо `Yellow`
- коли колір фіолетовий, показуємо `Purple`

Тут також можна використати масив для збереження назв кольорів:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

Порядок текстів у цьому масиві має відповідати порядку кольорів у масиві кольорів.

Потім ми можемо використати `Text`, щоб показати назву кольору, яка відповідає поточному index:

```swift
Text(colorsName[index])
```

`Text` відображатиме поточну назву кольору відповідно до `index`.

Скористаємося modifier'ами, щоб покращити стиль відображення `Text`:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

Так `Text` відображатиметься білим кольором, великим заголовком і жирним шрифтом.

Зараз у нас є повноекранний view `Color`. Якщо ми хочемо, щоб `Text` показувався поверх view `Color`, нам потрібно використати контейнер компонування `ZStack`.

```swift
ZStack {
    Color
    Text
}
```

Тому код змінюється на:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

Таким чином на задньому плані буде кольоровий view, а спереду поверх нього показуватиметься текстовий view.

Результат:

![Color](../../Resource/018_color2.png)

Слід звернути увагу, що в `ZStack` view, записаний пізніше, зазвичай показується попереду. Якщо записати `Text` перед `Color`, то пізніший view `Color` може його перекрити.

## Керування яскравістю

Тепер ми можемо перемикатися між різними кольорами, але fill light має ще одну важливу функцію: **зміну яскравості**.

У SwiftUI можна використовувати modifier `brightness`, щоб змінювати яскравість view.

Наприклад:

```swift
.brightness(1)
```

Можна написати так:

```swift
colors[index]
    .brightness(0.5)
```

Так поточний колір стане яскравішим і буде більше схожим на ефект fill light.

Діапазон `brightness` — `0 - 1`. `0` означає збереження початкового кольору, чим ближче до `1`, тим яскравішим стає колір, а `1` означає максимальну білу яскравість.

Хоча ми можемо керувати `brightness` у коді, користувач ще не може змінювати його самостійно.

Нам потрібно додати елемент, який можна перетягувати: `Slider`.

## View повзунка Slider

У SwiftUI `Slider` — це елемент керування повзунком, який використовується для вибору значення в певному діапазоні. У документації Apple він описується як “a control for selecting a value from a bounded linear range”.

Базове використання:

```swift
Slider(value: $value, in: 0...1)
```

Пояснення параметрів:

1. `value: $value`: `Slider` потрібно прив’язати до змінної.

    Коли повзунок перетягують, значення змінної також змінюється. І навпаки, якщо змінюється змінна, повзунок теж оновлюється.

    Це дуже схоже на `TextField`, який ми вивчали раніше, адже в обох випадках ми «прив’язуємо елемент керування до змінної».
    
    Перед прив’язаною змінною потрібно додати символ `$`, щоб позначити binding.

2. `in: 0...1`: цей параметр означає діапазон значень повзунка.

    Тут `0...1` означає: мінімальне значення — `0`, максимальне — `1`.

    Коли повзунок перетягнути вліво до кінця, прив’язана змінна буде близькою до `0`; коли вправо до кінця — близькою до `1`.

Наприклад:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` прив’язаний до змінної `value`, тому під час перетягування повзунка значення `value` також змінюється.

Вигляд:

![Slider](../../Resource/018_slider.png)

Коли `Slider` знаходиться зліва, прив’язане значення `value` стає `0`, а коли справа — `1`.

### Діапазон значень

Діапазон значень у `Slider` не є фіксованим. Його можна записати також як:

```swift
0...100
```

або використати інший діапазон.

Але в цьому fill light нам потрібно керувати яскравістю, тому `0...1` — найкращий вибір.

## Керування яскравістю через Slider

Тепер ми з’єднаємо `Slider` і `brightness`.

Спочатку створимо змінну для збереження значення яскравості:

```swift
@State private var slider = 0.0
```

Тут `0.0` — це значення типу `Double`.

Оскільки `Slider` зазвичай прив’язують до числового типу, а тут ми хочемо безперервну зміну, `Double` підходить краще. Крім того, `brightness` також приймає лише значення типу `Double`.

Потім передамо це значення в `brightness`:

```swift
colors[index]
    .brightness(slider)
```

Коли `slider == 0`, колір залишається у звичайному стані; чим ближче `slider` до `1`, тим яскравішим виглядає колір.

### Додавання елемента Slider

Далі додамо елемент `Slider`, який змінюватиме цю змінну:

```swift
Slider(value: $slider, in: 0...1)
```

Коли повзунок змінюється, `slider` також змінюється, а `brightness(slider)` синхронно оновлює яскравість.

Це дуже типовий для SwiftUI приклад того, як «змінна керує view».

### Налаштування вигляду Slider

За замовчуванням `Slider` використовує доступний простір як свою ширину.

Ми можемо задати йому фіксовану ширину:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

Потім додамо ще кілька modifier'ів, щоб він був помітнішим:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

Так повзунок матиме білий фон і заокруглені кути, тож на view `Color` він буде помітнішим.

Наприкінці розмістимо його внизу екрана.

Оскільки ми вже використали `ZStack`, можна додати в нього `VStack` і за допомогою `Spacer()` посунути `Slider` донизу.

## Повний код

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## Підсумок

Використовуючи знання, які ми вивчили раніше, разом із базовими поняттями про кольори та масиви, ми створили дуже цікавий додаток fill light.

Завдяки цьому додатку fill light ми навчилися використовувати `brightness` для зміни яскравості, `onTapGesture` для додавання дії натискання до view та елемент керування `Slider`.

Ми додали `onTapGesture` до view `Color` і реалізували перемикання кольорів. Через елемент `Slider` ми керуємо змінною, яка відповідає за `brightness`, і таким чином змінюємо яскравість кольору. Це також приклад того, як «змінна керує view».

Ми повторили ternary operator, використали `ZStack` для накладання view та використали масив для керування групою даних одного типу. Це допомагає нам глибше зрозуміти масиви та index, а також враховувати на практиці проблему виходу index за межі.

Хоча цей приклад не є складним, він поєднує багато базових ідей, які ми вивчили раніше. Коли ми застосовуємо їх у маленькому реальному проєкті, стає легше зрозуміти, для чого потрібна кожна з них.
