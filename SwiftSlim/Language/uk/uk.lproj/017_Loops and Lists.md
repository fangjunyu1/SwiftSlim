# Цикли та списки

У цьому уроці ми вивчимо цикл `for` у Swift, а також `ForEach` у SwiftUI.

Обидва допомагають нам послідовно читати елементи масиву. У реальній розробці нам часто потрібно діставати дані з групи по одному, а потім обробляти або показувати їх, і саме цикл є важливим інструментом для цього.

Коли ми переглядаємо вебсторінки або користуємося App, ми часто бачимо багато списків або пунктів. Такий вміст зазвичай спочатку зберігається в `масиві`, потім по черзі зчитується через `цикл`, а вже після цього показується в інтерфейсі.

Наприклад, список результатів пошуку Google:

![Google](../../Resource/017_google.png)

Кожен пункт у списку результатів пошуку зазвичай відповідає одному елементу даних і виводиться послідовно через цикл.

Тому `масив` і `цикл` часто з'являються разом. `Масив` відповідає за збереження групи даних, а `цикл` відповідає за те, щоб дістати ці дані по порядку і виконати однакову дію.

Далі ми почнемо з найпростішого циклу `for`.

## Зворотний відлік

Припустімо, що ми хочемо змоделювати функцію зворотного відліку і по черзі вивести `«5, 4, 3, 2, 1»`.

Найпростіший запис виглядає так:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

Такий спосіб може змоделювати зворотний відлік, але якщо вмісту багато, код виглядатиме дуже повторюваним, і надалі його буде незручно змінювати.

У цей момент ми можемо спочатку покласти дані в масив, а потім вивести їх по черзі за допомогою циклу `for`:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Такий запис коротший і його легше розширювати. Навіть якщо чисел у зворотному відліку дуже багато, нам не потрібно рядок за рядком повторювати `print`; достатньо дозволити циклу автоматично обробити кожен елемент масиву.

Ми можемо помістити цей код у кнопку і перевірити:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

Коли ми натискаємо кнопку, цикл `for` послідовно дістає кожне ціле число з `countDown` згідно з порядком у масиві та друкує їх у консоль.

Далі подивімося, як саме працює цикл `for`.

## Цикл for

Цикл `for` використовується для послідовної обробки групи даних. Найпоширеніші об'єкти для нього - це масиви та діапазони.

Базовий запис:

```swift
for елемент in масивАбоДіапазон {
    кодДляВиконання
}
```

Наприклад, щойно показаний код зворотного відліку:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Тут `i` означає елемент, який зараз дістається, а після `in` ідуть дані для перебору, тобто масив `countDown`.

Зміст цього коду такий: програма по черзі бере кожен елемент з масиву `countDown`. Щоразу, коли дістається один елемент, він тимчасово позначається як `i`, а потім виконується код у фігурних дужках.

Процес виконання можна зрозуміти так:

- Першим дістається `5`, тому в цей момент значення `i` дорівнює `5`
- Виконується `print(i)`, і виводиться `5`
- Потім дістається наступний елемент `4`
- `print(i)` виконується ще раз, і виводиться `4`

Подальші `3`, `2` і `1` також будуть виконані таким самим способом по черзі.

![for](../../Resource/017_for.png)

Коли всі елементи масиву оброблені, цикл `for` автоматично завершується.

За допомогою циклу `for` ми можемо отримувати кожен елемент масиву та виконувати такі дії, як зворотний відлік.

**Підказка**: `i` - це дуже поширений спосіб називати змінну в циклі for для позначення поточного елемента, але це не фіксоване правило; можна використовувати й інші англійські назви.

Наприклад, можна використати `num` для позначення кожного елемента:

```swift
for num in countDown {
    print(num)
}
```

Усередині фігурних дужок також потрібно використовувати `num`, щоб позначати поточний елемент.

### Цикл по діапазону

Цикл `for` може працювати не лише з масивом, а й з діапазоном.

Наприклад, коли нам потрібно обчислити суму всіх додатних цілих чисел у межах `50`, треба послідовно додати всі цілі числа від `1` до `50`.

Найпряміший запис може бути таким:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

Проблема цього запису в тому, що нам доводиться вручну виписувати кожне число. Це і незручно, і не підходить для реальної розробки.

У такому разі можна скористатися оператором діапазону разом із циклом `for`.

У попередньому розділі про «оператори» ми вже вивчали оператор діапазону:

```swift
1...3   // 1, 2, 3
```

Тут `...` означає замкнений діапазон, тобто він включає і початок, і кінець.

Отже, тут можна використати цикл `for` для діапазону цілих чисел від `1` до `50`:

```swift
for i in 1...50 {
    
}
```

Нам потрібно скласти ці числа, тому ще потрібна змінна для збереження суми:

```swift
var count = 0
```

Потім під час кожного проходу циклу ми додаємо поточне число до `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

Цей код означає:

- `count` зберігає поточну суму
- Цикл `for` по черзі бере цілі числа від `1` до `50`
- Кожного разу, коли береться чергове число, виконується `count += i`
- Після завершення циклу в `count` зберігається підсумкова сума.

Так ми можемо отримати суму всіх додатних цілих чисел до `50`.

Так само, якщо нам потрібно обчислити суму цілих чисел від `1` до `100`, достатньо змінити діапазон:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

Цикл `for` може працювати не лише з масивом, а й з одним діапазоном. Така здатність послідовно повторно обробляти дані дуже поширена в розробці.

Вона допомагає нам автоматично виконувати багато повторюваної роботи без потреби вручну прописувати додавання для кожного числа або інші схожі повторювані дії.

## Цикл у SwiftUI

Попередній цикл `for` належить до мови Swift.

Але в SwiftUI ми не можемо безпосередньо використовувати `for` у структурі view, щоб створювати view:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

Це тому, що всередині `VStack` мають бути SwiftUI view, а звичайний цикл `for` сам по собі не є view.

Якщо ми хочемо в SwiftUI повторно створювати інтерфейс на основі вмісту масиву, потрібно використовувати `ForEach`.

Наприклад, якщо ми хочемо показати імена всіх членів групи, можна написати так:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

Результат відображення:

![for1](../../Resource/017_for1.png)

Потім додамо однакові модифікатори до кожного `Text`:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Тепер кожен `Text` відображається дрібнішим і жирнішим шрифтом.

Результат відображення:

![](../../Resource/017_for2.png)

Хоча модифікатори роблять інтерфейс красивішим, вони також роблять код довшим і більш повторюваним.

Якщо імен стане більше або надалі потрібно буде одночасно змінювати шрифт, колір та інші стилі, доведеться багато разів писати й змінювати однаковий код. Це незручно й не дуже добре для підтримки.

У такому разі ми можемо спочатку помістити ці імена в масив, а потім використати `ForEach` у SwiftUI, щоб повторно створювати view на основі даних з масиву:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

Цей код коротший. `ForEach` по черзі читає кожен елемент масиву і показує відповідний вміст в інтерфейсі.

Коли пізніше ми додаватимемо нові імена, достатньо буде змінити вміст масиву `names`, і не доведеться щоразу повторно додавати `Text` та складні модифікатори.

## ForEach

`ForEach` - це структура в SwiftUI, яка використовується для повторного створення view. Її можна розуміти як конструкцію, схожу на цикл `for` у SwiftUI.

Базове використання:

```swift
ForEach(масив, id: \.self) { елемент in
    кодSwiftUI
}
```

Наприклад, для щойно показаних імен групи:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Цей код означає:

`names` - це масив, по якому виконується перебір, тобто дані, які потрібно показати.

`id:\.self` означає, що для розрізнення кожного елемента використовується сам елемент масиву.

У поточному прикладі в масиві зберігаються рядки:

```swift
["Sam", "John", "Wathon", "Bob"]
```

Тому SwiftUI безпосередньо використовує самі рядки "Sam", "John", "Wathon", "Bob", щоб розрізняти різний вміст.

Для простих масивів на кшталт String і Int, які часто трапляються на початковому етапі, зазвичай можна просто написати:

```swift
id: \.self
```

Поки що достатньо запам'ятати саме цей запис; глибше розбирати всі правила id поки не потрібно.

`item in` означає, що `item` є поточним елементом, який дістається.

Наприклад, під час першого проходу `item` дорівнює `"Sam"`, а під час другого - `"John"`.

Тому:

```swift
Text(item)
```

По черзі перетвориться на:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

Отже, призначення цього коду таке: по черзі брати кожне ім'я з масиву `names` і створювати для кожного окремий `Text` view.

### Процес виконання

Процес виконання можна уявити так:

- `ForEach` спочатку читає перший елемент `"Sam"` у масиві `names`
- `item` тимчасово представляє `"Sam"`
- `Text(item)` показує `"Sam"`
- Потім продовжується читання наступного елемента `"John"`
- І той самий код view виконується ще раз

У підсумку всі імена з масиву будуть показані.

Перевага такого запису в тому, що коли імен стає більше, нам не потрібно знову і знову писати багато `Text`; достатньо змінити вміст масиву.

### Перебір фотографій

`ForEach` може показувати не лише текст, а й зображення, кольори або інші SwiftUI view.

Наприклад, підготуємо 4 зображення:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

Перетягніть ці чотири зображення в папку Assets у Xcode.

![image](../../Resource/017_for3.png)

Якщо не використовувати `ForEach`, нам доведеться вручну написати кожне зображення:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

Результат відображення:

![image](../../Resource/017_for4.png)

Такий запис працює, але код виходить довгим.

Що більше зображень, то більше коду `Image` доводиться повторювати.

Якщо пізніше ми захочемо однаково змінити розмір зображень, їх також доведеться змінювати по одному.

У такому разі можна помістити назви зображень у масив, а потім за допомогою `ForEach` повторно створити view для зображень:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Тут `images` - це масив рядків, і кожен елемент масиву є назвою зображення.

`ForEach(images, id: \.self)` означає, що елементи масиву `images` послідовно читаються, для розрізнення кожного пункту використовується сам елемент, а потім на основі поточного елемента створюється відповідне view.

Наприклад, якщо першим зчитується `"430F9BEF"`, тоді:

```swift
Image(item)
```

Буде еквівалентно:

```swift
Image("430F9BEF")
```

Наступні зображення також будуть показані по черзі таким самим способом.

![image](../../Resource/017_for4.png)

Тут ми все ще використовуємо:

```swift
id: \.self
```

Причина така сама, як і раніше: елементи масиву `images` теж мають простий тип рядка, тому на початковому етапі достатньо використовувати сам елемент як спосіб розрізнення.

Це можна просто запам'ятати як готову формулу:

**Коли `ForEach` перебирає простий масив на кшталт `String` або `Int`, зазвичай пишуть `id: \.self`.**

### Додавання зображень

Якщо пізніше потрібно буде додати нові зображення, достатньо просто змінити вміст масиву:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

Не потрібно знову писати новий код `Image`.

Якщо зображень багато, можна використовувати це разом із `ScrollView`, інакше після виходу вмісту за межі екрана переглянути все повністю не вийде.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### Зміна зображень

Коли нам потрібно однаково змінити стиль зображень, достатньо змінити код усередині `ForEach`.

Наприклад, змінімо ширину на `80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

Тоді всі зображення зміняться одразу, і не потрібно буде редагувати їх по одному.

Ось одна з важливих ролей `ForEach`:

**Коли структура кількох view однакова, а відрізняється лише вміст, можна написати код view лише один раз, а дані в масиві вже вирішуватимуть, що саме показувати.**

## Підсумок

У цьому уроці ми вивчили цикл `for` і `ForEach`.

Цикл `for` використовується для послідовної обробки елементів масиву або діапазону і більше орієнтований на роботу з даними.

`ForEach` використовується в SwiftUI для повторного створення view на основі вмісту масиву і більше орієнтований на відображення інтерфейсу.

На поточному початковому етапі найпоширеніший запис `ForEach` такий:

```swift
ForEach(масив, id: \.self) { елемент in
    кодSwiftUI
}
```

Тут `id: \.self` означає використання самого елемента для розрізнення кожного елемента.

Коли в масиві містяться прості типи на кшталт `String` або `Int`, зазвичай можна спочатку використовувати саме цей запис.

Надалі, коли ви переглядатимете вебсторінки або App і бачитимете групи повторюваного тексту, зображень чи іншого спискового вмісту, можна згадати, що в SwiftUI такі дані зазвичай спочатку кладуть у масив, а потім по черзі показують через `ForEach`.
