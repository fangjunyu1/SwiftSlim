# Оператори

У цьому уроці ми вивчатимемо оператори Swift (Operators).

Коли ми виконуємо обчислення, порівняння або логічні перевірки, нам потрібні оператори. Такі дії проходять через усі етапи виконання програми й є однією з базових основ програмування.

```swift
let a = 100 * 8
````

Під час вивчення програмування багато хто помічає: люди з певним математичним мисленням зазвичай легше розуміють логіку програм. Це тому, що програмування по суті неможливе без “обчислень” і “перевірки правил”, а саме оператори виконують цю ключову роль.

У реальній розробці оператори використовуються не лише для простих числових обчислень, а й широко застосовуються в різних конкретних сценаріях.

Наприклад, у грі з блоками, коли користувач перетягує блок у визначене місце, потрібно обчислити:

* координати блока
* позиції, які блок може зайняти
* чи доступна цільова позиція

Усі ці процеси “обчислення + перевірка” по суті виконуються за допомогою операторів.

![Block](../../Resource/013_block.png)

Вивчення операторів — це не просто вивчення числових обчислень, а й основа для подальшого вивчення побудови інтерфейсу, обробки даних і логіки взаємодії.

## Як виконуються обчислення у SwiftUI

У SwiftUI зазвичай обчислення запускаються через Button, який викликає функцію, а сама логіка обчислень виконується всередині цієї функції.

Наприклад:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

Коли кнопка натискається, у Console буде виведено:

```swift
c: 20
```

У цьому уроці ми будемо однаково розміщувати логіку обчислень у `func`, запускати її через кнопку й використовувати `print` для виведення результату.

У наступних прикладах, щоб спростити показ коду, ми будемо відразу позначати результати обчислень у коді й більше не повторюватимемо `print` у кожному прикладі.

## Керування виведенням у Console

Оскільки кожне натискання генерує новий рядок виводу, можна натиснути кнопку очищення в правому нижньому куті області Console, щоб очистити історію логів і зручніше переглядати поточний результат.

## Інший запис Button

Коли кнопка виконує лише одну функцію, можна використати коротший і чистіший запис:

```swift
Button(action: ) {
    // View
}
```

У цьому записі `action` вказує функцію, яку потрібно виконати після натискання кнопки (функція без параметрів), а фігурні дужки `{}` використовуються для визначення вигляду кнопки.

У поєднанні з функцією `calculate`, наведеною раніше, це можна записати так:

```swift
Button(action: calculate) {
    Text("Button")
}
```

Коли кнопка натискається, функція `calculate` виконується безпосередньо.

Тому, коли логіка кнопки проста й вона лише викликає одну функцію, запис через `action` виглядає зрозуміліше й більше відповідає хорошому стилю коду.

## Арифметичні оператори

Арифметичні оператори (Arithmetic Operators) використовуються для числових обчислень:

* `+` додавання
* `-` віднімання
* `*` множення
* `/` ділення
* `%` остача від ділення

Приклад:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

Потрібно звернути увагу, що Swift — це строго типізована мова, тому вона не змішує різні типи даних автоматично:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

Коли типи двох значень різні, тип потрібно перетворити вручну.

## Оператори порівняння

Оператори порівняння (Comparison Operators) використовуються для перевірки відношення між двома значеннями, а результатом є `Bool`:

* `==` дорівнює
* `!=` не дорівнює
* `>` більше
* `<` менше
* `>=` більше або дорівнює
* `<=` менше або дорівнює

Приклад:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

Оператори порівняння часто використовуються в умовних конструкціях. Наприклад, у “каруселі цитат” нам потрібно стежити, щоб індекс не виходив за межі масиву:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

Якщо `index` менший за `sayings.count - 1`, результат порівняння буде `true`, і код усередині `if` виконається; якщо результат буде `false`, код після `if` не виконається.

## Логічні оператори

Логічні оператори (Logical Operators) використовуються для поєднання кількох умов:

* `&&` і (AND)
* `||` або (OR)
* `!` не (NOT)

### `&&` і

Коли потрібно, щоб одночасно виконувалися кілька умов, використовується `&&`.

Наприклад:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

У цьому коді є дві умови: `age >= 18` і `age <= 100`.

Лише якщо обидві умови одночасно дорівнюють `true`, `legalAge` також буде `true`; якщо хоча б одна умова не виконується, результатом буде `false`.

### `||` або

Коли достатньо виконання хоча б однієї з кількох умов, використовується `||`.

Наприклад:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

Цей код означає: якщо виконується хоча б одна з умов `money >= 50` або `creditCard >= 50`, тоді `payAllowed` буде `true`.

Лише коли обидві умови не виконуються, результат буде `false`.

Наприклад, у сценарії оплати можна використовувати або готівку, або кредитну картку. Якщо не підходить жоден варіант, оплата не відбудеться.

### `!` не

`!` — це логічний оператор заперечення (NOT), який інвертує значення `Bool`.

Його можна розуміти так: `!` перетворює `true` на `false`, а `false` — на `true`.

Наприклад:

```swift
let allowed = true
let result = !allowed   // false
```

Тут `!allowed` означає інвертувати значення `allowed`.

Потрібно звернути увагу, що `!` і `!=` — це різні оператори: `!` інвертує значення `Bool`, а `!=` є оператором порівняння, який повертає `Bool`.

## Оператори присвоєння

Оператори присвоєння (Assignment Operators) використовуються для присвоєння або оновлення значення змінної:

* `=` присвоєння

Під час оголошення змінної або в процесі обчислення ми постійно працюємо з присвоєнням:

```swift
let a = 5
```

Цей код означає, що значення `5` присвоюється змінній `a`.

### Складені оператори присвоєння

У реальній розробці, крім базового оператора присвоєння, дуже часто використовуються “складені оператори присвоєння”, які дозволяють одразу виконати обчислення на основі поточного значення й оновити його:

* `+=`
* `-=`
* `*=`
* `/=`

У попередньому уроці про “лічильник” ми вже коротко знайомилися зі складеними операторами присвоєння.

Наприклад, якщо після натискання кнопки потрібно автоматично збільшити значення змінної:

```swift
var num = 10
num += 5   // num = 15
```

Цей код означає: до `num` додається `5`, а потім результат знову присвоюється `num`.

Тому він еквівалентний такому запису:

```swift
var num = 10
num = num + 5   // num = 15
```

## Тернарний оператор

Тернарний оператор у Swift (Ternary Operator) виконує перевірку умови й повертає одне з двох значень:

```swift
умова ? значення1 : значення2
```

Логіка його роботи така: якщо умова дорівнює `true`, повертається “значення1”; якщо умова дорівнює `false`, повертається “значення2”.

Наприклад:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

Цей код перевіряє значення `showColor`: якщо воно дорівнює `true`, повертається `Color.blue`, і фон буде синім; якщо дорівнює `false`, повертається `Color.clear`, і фоновий колір не відображатиметься.

Підказка: `Color.clear` означає прозорий колір (нічого не відображається).

### Відмінність від if-else

За логікою виконання тернарний оператор еквівалентний конструкції `if-else`.

Наприклад, наведений вище код можна записати так:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

Логіка виконання в обох випадках повністю однакова: якщо умова `true`, виконується перший варіант; якщо умова `false`, виконується другий.

З практичного погляду тернарний оператор має коротший запис і краще підходить для простих перевірок у стилі “одне з двох”, особливо коли потрібно безпосередньо повернути значення.

Під час побудови View у SwiftUI тернарний оператор трапляється дуже часто, наприклад:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

Такий запис дозволяє в одному рядку динамічно змінювати вигляд подання залежно від умови.

## Оператори діапазону

Оператори діапазону (Range Operators) використовуються для позначення інтервалів:

* `...` замкнений діапазон (включає обидві межі)
* `..<` напіввідкритий діапазон (не включає праву межу)

Наприклад:

```swift
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

Це можна розуміти так: `1...3` означає від 1 до 3 включно; `5..<8` означає від 5 до 8, але без 8.

Розширені форми (лише для ознайомлення):

```swift
5...    // >= 5
...5    // <= 5
```

Такий запис означає “односторонній діапазон”, він часто використовується в умовних перевірках або зіставленні з шаблоном.

Оператори діапазону надалі знадобляться під час вивчення циклу `for`. Поки що їх достатньо розуміти як позначення безперервного числового проміжку.

## Практика з операторами

Далі ми розглянемо кілька простих прикладів, щоб побачити, як працюють оператори у View.

### 1. Обчислення загальної кількості автомобілів

Наприклад, через певне перехрестя щодня проїжджає 500 автомобілів. Обчислимо, скільки автомобілів проїде за 30 днів.

Можемо використати арифметичні оператори:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

Логіка обчислення тут така: щодня 500 автомобілів, за 30 днів — `500 × 30`, у результаті отримуємо загалом 15000 автомобілів.

### 2. Перевірка внутрішньоігрової підписки

Можна перевірити, чи є людина внутрішньоігровим підписником: якщо в неї є або довічний доступ, або активна підписка, вважаємо її підписником.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

У цьому коді використовується оператор `||` (або): `lifeTime = false` означає, що довічного доступу немає; `subscription = true` означає, що є активна підписка.

Оскільки достатньо виконання хоча б однієї умови, `isMember` буде `true`. Отже, ця людина є внутрішньоігровим підписником.

## Підсумок

У Swift оператори — це основа обчислення даних і логічних перевірок. Так само як `Text` відповідає за відображення вмісту, оператори відповідають за обробку самих даних.

Від найпростіших числових обчислень до складних логічних перевірок, а також до керування станом і розрахунків компонування у View — усюди потрібні оператори.

Через практику в різних сценаріях можна поступово ознайомитися й опанувати різні типи операторів, представлені в цьому уроці. Це важлива основа для подальшого навчання.

### Додаткові знання — побітові оператори

Крім того, Swift також надає нижчорівневу групу операторів — побітові оператори (Bitwise Operators):

* `&` (побітове І)
* `|` (побітове АБО)
* `^` (побітове XOR)
* `~` (побітове НЕ)
* `>>` (зсув праворуч)
* `<<` (зсув ліворуч)

Наприклад:

```swift
let a = 6  // двійкове 110
let b = 3  // двійкове 011

print(a & b) // 2 (двійкове 010)
print(a | b) // 7 (двійкове 111)
print(a ^ b) // 5 (двійкове 101)
print(~a)    // -7 (інверсія)
```

У Swift цілі числа зберігаються у двійковому доповняльному коді, тому після побітової інверсії отримується відповідне від’ємне число.

Побітові оператори безпосередньо працюють із двійковими даними й зазвичай використовуються в низькорівневій обробці даних або в сценаріях оптимізації продуктивності.

Для початківців такі оператори трапляються рідко, і в звичайній розробці на iOS або SwiftUI вони також не дуже поширені, тому тут ми не будемо заглиблюватися в них.

Якщо пізніше виникне потреба працювати з низькорівневою логікою або оптимізацією продуктивності, до цієї теми можна буде повернутися окремо.
