# Як змусити кнопку працювати

У цьому уроці ми головним чином вивчимо кнопку `Button`, виведення відлагоджувальної інформації через `print`, а також функції Swift.

Ці знання є основою інтерактивного програмування у SwiftUI.

## Кнопка

Кнопка — це один із найпоширеніших UI-елементів. Коли нам потрібно щось завантажити, відкрити або вийти, ми зазвичай натискаємо кнопку.

Наприклад, в App Store натискають кнопку «Отримати», щоб завантажити App.

![Button](../../RESOURCE/008_button.png)

У SwiftUI кнопка позначається через `Button`.

Базове використання:

```swift
Button("") {
    
}
```

Структуру можна поділити на дві частини:

```swift
Button("Текст кнопки") {
    код, який виконується під час натискання кнопки
}
```

Наприклад:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

Після запуску буде показано кнопку `Start`.

![Button](../../RESOURCE/008_button1.png)

Коли користувач натискає кнопку, виконується код усередині `{}`.

Потрібно звернути увагу: `Button` належить до елементів інтерфейсу SwiftUI (`View`), тому його потрібно писати всередині `body`.

### Модифікатор buttonStyle

У SwiftUI `Button` має вбудовані системні стилі кнопок, які можна використовувати через модифікатор `buttonStyle`.

Наприклад:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

`buttonStyle` застосовує системний стиль кнопки.

До поширених варіантів належать:

* `.automatic`
* `.plain`
* `.bordered`
* `.borderedProminent`
* `.borderless`

![Button](../../RESOURCE/008_button7.png)

Різниця між цими стилями головним чином полягає в наявності рамки, фону та ступені візуального акценту.

## Виведення через print

У програмуванні часто потрібно перевірити, чи дійшла програма до певного фрагмента коду.

У Swift для цього є функція `print`, яка використовується для виведення відлагоджувальної інформації.

Базовий запис:

```swift
print("Hello")
```

Цей рядок виведе текст у `Console` (консоль).

### Перевірка, чи працює кнопка

Ми можемо помістити `print` усередину кнопки, щоб перевірити, чи вона була натиснута.

```swift
Button("Start") {
    print("Test 123")
}
```

Коли кнопку буде натиснуто, у `Console` з’явиться:

```swift
Test 123
```

Це означає, що кнопка успішно спрацювала.

![Button](../../RESOURCE/008_button3.png)

Зверніть увагу: `print` — це код Swift, а не елемент інтерфейсу. Його не можна напряму розміщувати в `body`, інакше виникне помилка.

Наприклад, ось такий запис є неправильним:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // Помилка: body повинен повертати View
    }
}
```

У SwiftUI `body` використовується для опису структури інтерфейсу, тому всередині нього можна розміщувати лише елементи інтерфейсу (`View`), наприклад `Text`, `Button`, `Image` тощо.

Звичайний код (обчислювальна логіка, `print` тощо) потрібно розміщувати в кнопках, функціях або інших обробниках подій.

### Консоль Console

Те, що виводить `print`, відображається в `Console` (консолі) в Xcode.

Якщо `Console` не видно, можна натиснути кнопку `Show the Debug Area` у правому нижньому куті, щоб відкрити область налагодження, а потім натиснути `Show the Console`, щоб показати консоль.

![](../../RESOURCE/008_button4.png)

У нижньому лівому куті `Console` є дві вкладки: `Executable` і `Previews`.

![](../../RESOURCE/008_button5.png)

Ці дві вкладки відповідають різним середовищам запуску: коли ви працюєте в Canvas Preview, виведення `print` буде показано у вкладці `Previews`; якщо ж застосунок запущено в симуляторі або на реальному пристрої, виведення буде показано у вкладці `Executable`.

Тому, якщо ви не бачите результату `print`, спочатку перевірте, чи вибрано правильну вкладку.

## Приклад — ASCII-арт

У деяких кодових проєктах часто можна побачити ASCII-арт.

ASCII-арт — це зображення, складене зі звичайних символів, наприклад:

![ASCII](../../RESOURCE/008_ascii.png)

Ми можемо використати кнопку + `print`, щоб вивести ASCII-арт.

Приклад коду:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Коли кнопку буде натиснуто, `Console` покаже:

```swift
 /\_/\
( o.o )
 > ^ <
```

Зверніть увагу: символ `\` у рядку потрібно записувати як `\\`, тому що зворотна риска в рядках є символом екранування.

## Функції

Усередині кнопки ми можемо писати код безпосередньо.

Наприклад:

```swift
Button("Start") {
    print("Hello")
}
```

Якщо коду мало, такий запис цілком нормальний. Але в реальній розробці логіка, яку запускає кнопка, може бути значно більшою.

Наприклад:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // Можливо, тут ще буде багато коду
}
```

Якщо коду стає все більше, пряме розміщення його всередині кнопки робить код інтерфейсу безладним, а читати та підтримувати його стає незручно.

Тому зазвичай ми виносимо цей код у функцію, а потім викликаємо цю функцію в кнопці.

### Що таке функція

Функція — це фрагмент коду, який можна використовувати повторно.

Коли певний код потрібно виконати, нам достатньо викликати цю функцію — і весь код усередині неї запуститься.

Такий підхід робить структуру коду чистішою, а також полегшує повторне використання.

### Базовий запис

У Swift функції визначаються за допомогою ключового слова **func**:

```swift
func randomInt() {
    // code
}
```

Цей код визначає функцію.

`randomInt` — це назва функції, яка використовується для позначення цього фрагмента коду.

`()` означає місце для параметрів — сюди можна передавати дані ззовні. Якщо дані не потрібні, дужки можна залишити порожніми.

Усередині `{}` знаходиться область коду функції — сюди записується весь код, який має виконуватися.

Наприклад:

```swift
func getName() {
    print("FangJunyu")
}
```

Призначення цієї функції — вивести текст.

### Виклик функції

Оголосити функцію — означає лише створити цей фрагмент коду. Щоб він виконався, функцію потрібно викликати.

Спосіб виклику — додати `()` після назви функції:

```swift
getName()
```

Коли програма дійде до цього рядка, вона виконає код усередині функції `getName`.

Повний приклад:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

Після запуску `Console` покаже:

```swift
FangJunyu
```

### Використання функцій у SwiftUI

У SwiftUI-представленні функції зазвичай записують поза `body`.

Наприклад:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

Коли кнопку буде натиснуто, викличеться функція `getName()`.

Такий підхід дозволяє зберігати код інтерфейсу лаконічним, а конкретну логіку обробляти всередині функцій.

### Параметри функції

Іноді функція повинна виконувати різні дії залежно від переданих даних.

Наприклад, ми хочемо, щоб функція могла виводити різні імена, а не лише фіксоване `"FangJunyu"`.

У такому випадку можна використовувати параметри. Параметри можна розуміти як дані, які передаються під час виклику функції.

Наприклад:

```swift
func getName(name: String) {
    print(name)
}
```

У цієї функції додано параметр: `name` — це назва параметра, а `String` — його тип.

Це означає, що функція повинна отримати дані типу `String`.

Оскільки функції тепер потрібні дані, під час виклику їх обов’язково потрібно передати.

```swift
getName(name: "Sam")
```

Коли під час виклику передається `"Sam"`, функція використовує це значення для виведення.

Результат виконання:

```swift
Sam
```

## Приклад — виведення ASCII-арту за допомогою функції

У попередньому прикладі ми розміщували `print` безпосередньо всередині кнопки.

Але якщо ASCII-арт складніший, код стає довшим. У такому випадку логіку можна винести у функцію, щоб код інтерфейсу був чистішим.

Приклад коду:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Після запуску, коли кнопку буде натиснуто, `Console` покаже:

```swift
 /\_/\
( o.o )
 > ^ <
```

Тут відбуваються дві дії. Після натискання кнопки користувачем:

1. кнопка викликає функцію `printCat()`
2. виконується код `print` усередині функції

Такий підхід дозволяє інтерфейсному коду відповідати лише за запуск події, а конкретну логіку обробляти у функції.

У реальній розробці більшість дій кнопок реалізуються саме через функції.

## Підсумок

У цьому уроці ми вивчили три важливі речі:

**1. Кнопка Button**

`Button` — це елемент SwiftUI, який використовується для запуску дій.

```swift
Button("Start") {

}
```

Коли користувач натискає кнопку, виконується код у фігурних дужках.

**2. Виведення через print**

`print` використовується для виведення відлагоджувальної інформації в `Console`.

```swift
print("Hello")
```

Розробник може переглядати стан роботи програми через `Console`.

**3. Функції**

Функція — це фрагмент коду, який можна використовувати повторно.

```swift
func sayHello() {
    print("Hello")
}
```

Виклик функції:

```swift
sayHello()
```

Якщо функції потрібні дані, можна використовувати параметри:

```swift
func sayHello(name: String) {
    print(name)
}
```

У наступних уроках ми продовжимо вивчати більше елементів SwiftUI, а також те, як дані взаємодіють з інтерфейсом.
