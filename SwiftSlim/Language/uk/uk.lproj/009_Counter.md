# Лічильник

У двох попередніх уроках ми вивчили змінні, константи, кнопки та методи.

У цьому уроці ми поєднаємо ці знання й створимо простий застосунок-лічильник.

У цьому прикладі ми навчимося зберігати та змінювати значення змінної, а також використовувати @State для керування змінною.

## Реалізація лічильника

Лічильник зазвичай використовується для підрахунку кількості, яка постійно збільшується. Наприклад, для підрахунку стрибків на скакалці, кіл під час бігу та інших подібних сценаріїв.

Тепер реалізуймо простий лічильник: відображатимемо число та кнопку, а після натискання кнопки число збільшуватиметься.

![Num](../../RESOURCE/009_num.png)

### Відображення числа

Спочатку використаємо Text для відображення числа.

```swift
Text("0")
````

Але якщо число має змінюватися, не можна просто жорстко записати `"0"`, натомість слід зберігати число у змінній.

```swift
var num = 0
```

Потім відобразимо цю змінну в Text:

```swift
Text("\(num)")
```

Тут використовується інтерполяція рядка `\()`, щоб перетворити число на рядок для відображення.

### Кнопка для збільшення числа

Далі додамо кнопку.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

Коли кнопку натискають, ми хочемо, щоб число збільшувалося.

Тому всередині кнопки можна змінити значення змінної:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

Цей код означає: коли кнопка натиснута, обчислюється `num + 1`, а результат присвоюється `num`.

### Коротший запис

У Swift є коротший спосіб запису:

```swift
num += 1
```

Він еквівалентний:

```swift
num = num + 1
```

Такий запис називається «складеним оператором присвоєння».

Поширені складені оператори присвоєння у Swift:

```
+=   присвоєння додавання
-=   присвоєння віднімання
*=   присвоєння множення
/=   присвоєння ділення
%=   присвоєння остачі від ділення
```

Наприклад:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### Повний код

Тепер ми можемо написати простий лічильник:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

Але в такому вигляді код не працюватиме правильно.

### Помилка

Xcode покаже повідомлення:

```
Cannot assign to property: 'self' is immutable
```

Це означає, що властивість не можна змінити, тому що представлення є незмінним.

Чому так відбувається?

## Принцип роботи SwiftUI

У SwiftUI: **інтерфейс керується даними**, а це означає, що коли дані змінюються, інтерфейс автоматично оновлюється.

Але важливо розуміти: SwiftUI відстежує зміни та оновлює представлення лише тоді, коли змінюється стан (наприклад, @State, @Binding).

Якщо використовувати звичайну змінну, SwiftUI зчитає її значення лише один раз під час створення представлення, і навіть якщо потім значення зміниться, інтерфейс не оновиться.

Наприклад:

```swift
var num = 0   // 0
num = 1   // 0
```

У наведеному прикладі `num` — це звичайна змінна. Хоч її значення і змінилося, SwiftUI не оновить представлення, і на екрані все одно залишиться початковий `0`.

![Num](../../RESOURCE/009_state.png)

Тому лише тоді, коли дані позначені як стан (наприклад, за допомогою @State, @Binding та інших property wrapper), SwiftUI автоматично відстежуватиме їхні зміни та оновлюватиме інтерфейс.

## Обгортка властивості @State

У SwiftUI, якщо потрібно змінювати змінну й оновлювати представлення, слід використовувати @State.

```swift
@State var num = 0
```

@State змушує SwiftUI стежити за цією змінною.

Коли значення змінної змінюється, SwiftUI повторно обчислює представлення та оновлює його.

Повний код:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

Тепер, коли натиснути кнопку:

```swift
num += 1
```

SwiftUI визначить, що `num` змінилося, і автоматично оновить представлення.

![Num](../../RESOURCE/009_state1.png)

## Правила використання @State

У реальній розробці для @State зазвичай дотримуються двох правил.

### 1. @State зазвичай використовується разом із private

Змінні @State зазвичай використовуються лише всередині поточного типу (наприклад, поточної структури View), тому їх зазвичай записують так:

```swift
@State private var num = 0
```

Це допомагає уникнути прямого доступу або зміни цієї змінної з інших представлень.

Ми ще не вивчали `private`, тому поки що можна просто розуміти це так:

**Вміст, позначений private, можна використовувати лише в поточному представленні, а ззовні він недоступний.**

### 2. @State використовується лише для стану поточного представлення

@State підходить для даних, якими керує саме поточний View.

Наприклад, для вмісту поля введення або стану перемикача.

```swift
@State private var isOn = false
@State private var text = ""
```

Якщо дані потрібно спільно використовувати між кількома представленнями, тоді потрібні інші типи стану, наприклад @Binding або @Observable.

Ці теми ми розглянемо в наступних уроках.

### Кнопка зменшення

Лічильник може не лише збільшувати значення, а й зменшувати його.

Можемо додати кнопку `-`.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

Потім використаємо HStack, щоб розташувати кнопки горизонтально:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

Результат:

![Num](../../RESOURCE/009_num1.png)

Коли натискається кнопка `+`, число збільшується на 1; коли натискається кнопка `-`, число зменшується на 1.

### Кнопка скидання

Ми також можемо додати кнопку скидання, яка встановлює число в 0.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

Коли натиснути кнопку скидання, `num` отримає значення `0`.

Результат:

![Num](../../RESOURCE/009_num2.png)

Коли натискається кнопка `0`, число стає `0`.

### Власний вигляд кнопки

Зараз наші кнопки можуть відображати лише текст:

```swift
Button("+") {

}
```

У такому записі кнопка може показувати лише текстовий вміст, наприклад `+` або `-`.

У реальній розробці ми зазвичай хочемо, щоб вигляд кнопки був багатшим, наприклад із використанням іконок або інших представлень.

SwiftUI дозволяє налаштовувати вміст кнопки, і для цього можна використати інший запис кнопки:

```swift
Button(action: {

}, label: {

})
```

У цьому записі `action` означає код, який виконується після натискання кнопки, а `label` — представлення, яке відображається всередині кнопки.

Наприклад, можна зробити так, щоб кнопка відображала іконку SF Symbols:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

Тоді кнопка відображатиме вже не текст, а іконку.

### Покращення інтерфейсу лічильника

Щоб інтерфейс лічильника був зрозумілішим, можна налаштувати стиль відображення числа. Наприклад, збільшити шрифт:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

Далі змінимо вміст кнопок на іконки й трохи збільшимо їхній розмір:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

Так само можна перетворити кнопку зменшення та кнопку скидання на кнопки з іконками:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

Тепер кнопки відображають іконки SF Symbols, а інтерфейс стане зрозумілішим і наочнішим.

## Підсумок

У цьому уроці, реалізувавши простий лічильник, ми комплексно попрактикували використання змінних, Text і Button. Ми використали змінну для збереження числа та керували збільшенням і зменшенням цього числа за допомогою кнопок, а також вивчили складені оператори присвоєння, наприклад `num += 1` і `num -= 1`.

У процесі реалізації ми зрозуміли, як працює SwiftUI: інтерфейс керується даними. Лише коли дані змінюються, інтерфейс оновлюється. Тому для збереження змінного стану потрібно використовувати @State. Коли змінюється змінна @State, SwiftUI автоматично оновлює представлення.

Наприкінці ми трохи покращили інтерфейс: змінили стиль шрифту для Text і використали `Button(action:label:)` для налаштування вмісту кнопок, щоб кнопки могли відображати іконки. Завдяки цьому інтерфейс лічильника став зрозумілішим і привабливішим.

Тепер ми вже познайомилися з базовим способом використання представлень у SwiftUI. У SwiftUI більшість інтерфейсів будуються на основі змінних. Коли значення змінюються, SwiftUI автоматично оновлює представлення. Тому під час розробки застосунків на SwiftUI ми зазвичай спочатку продумуємо, які дані потрібно зберігати, а вже потім на основі цих даних будуємо інтерфейс і взаємодію.

## Повний код

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
