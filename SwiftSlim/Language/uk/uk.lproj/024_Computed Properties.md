# Обчислювана властивість

У цьому уроці ми головним чином вивчимо обчислювані властивості.

Обчислювана властивість використовується для обчислення нового результату на основі вже наявних даних. Вона може обчислювати числові значення, а також вміст, який потрібно показати у SwiftUI view.

Наприклад:

```swift
let a = 10
let b = 20
let c = a + b
```

Тут `c` означає результат після додавання `a` і `b`.

У звичайному коді такий спосіб запису дуже поширений.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

Порада: у події натискання `Button` можна виконувати звичайний Swift-код.

Але якщо подібний код написати безпосередньо у властивостях SwiftUI view, виникне проблема.

Наприклад:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

Цей код спричинить помилку.

На перший погляд здається, що `a` і `b` уже написані перед `c`, тому значення `c` нібито можна обчислити.

Але під час оголошення властивостей у struct такий спосіб запису не можна використовувати безпосередньо.

## Чому c не можна безпосередньо обчислити як a + b?

Тому що `a`, `b` і `c` — це не тимчасові константи всередині події натискання кнопки. Це властивості view `ContentView`.

У події натискання кнопки цей код може виконуватися нормально:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

Тому що після натискання кнопки код виконується послідовно. Спочатку створюється `a`, потім створюється `b`, і в кінці за допомогою `a + b` обчислюється `c`.

Але під час оголошення властивостей усередині view ситуація інша:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

Тут `a`, `b` і `c` — це всі властивості view `ContentView`.

Коли створюється view `ContentView`, Swift спочатку має підготувати ці властивості. Щоб процес створення був безпечним, Swift не дозволяє значенню за замовчуванням однієї stored property екземпляра безпосередньо читати інші instance properties того самого екземпляра.

Тому цей рядок спричинить помилку:

```swift
let c = a + b
```

Простіше кажучи: **під час оголошення властивостей усередині view не можна безпосередньо використовувати одну звичайну властивість для обчислення іншої звичайної властивості.**

Така властивість, яка безпосередньо зберігає значення, називається “stored property”. Для простішого розуміння тимчасово можна вважати її звичайною властивістю.

Наприклад:

```swift
let a = 10
```

`a` зберігає `10`.

```swift
let b = 20
```

`b` зберігає `20`.

Але:

```swift
let c = a + b
```

Тут `c` — це не фіксоване значення, написане безпосередньо. Очікується, що воно буде обчислене через `a + b`.

Для ситуацій, коли “результат отримується на основі вже наявних даних”, краще підходить обчислювана властивість.

Код можна змінити так:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

Тут `c` — це обчислювана властивість.

```swift
var c: Int {
    return a + b
}
```

Її значення таке: **коли потрібно використати `c`, тоді читаються `a` і `b`, а результат `a + b` повертається назовні.**

Наприклад:

```swift
Text("c: \(c)")
```

Коли `Text` показує `c`, лише тоді запускається обчислення `c`.

## Що таке обчислювана властивість?

Обчислювана властивість виглядає як змінна, але сама не зберігає дані.

Наприклад:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

Тут `c` — це обчислювана властивість.

Вона не зберігає фіксоване значення, як звичайна властивість. Щоразу, коли `c` читається, код у `{}` виконується знову, і повертається обчислений результат.

Це можна зрозуміти так:

```swift
var c: Int {
    return a + b
}
```

Коли потрібне `c`, обчислюється `a + b`.

Тому обчислювана властивість підходить для такої ситуації: **результат не потрібно зберігати окремо, оскільки його можна обчислити з уже наявних даних.**

## Базовий синтаксис

Обчислювана властивість зазвичай складається з трьох частин:

```swift
var c: Int {
    return a + b
}
```

### 1. Оголошення за допомогою `var`

```swift
var c
```

Обчислювану властивість потрібно оголошувати саме через `var`; `let` використовувати не можна.

Тому що обчислювана властивість — це не фіксоване збережене значення, а результат, який динамічно обчислюється щоразу під час читання.

### 2. Вказування return type

```swift
var c: Int
```

Обчислювана властивість має вказувати return type.

Тут `c` у результаті повертає ціле число, тому type — `Int`.

### 3. Написання логіки обчислення у {}

```swift
{
    return a + b
}
```

У `{}` пишеться логіка обчислення. Тут повертається результат `a + b`.

## Ключове слово return

Обчислювана властивість має повернути результат.

Наприклад:

```swift
var c: Int {
    return a + b
}
```

Тут `return` означає: повернути результат обчислення `a + b`.

Якщо всередині обчислюваної властивості є лише один expression, який безпосередньо створює результат, `return` можна опустити:

```swift
var c: Int {
    a + b
}
```

Але якщо всередині обчислюваної властивості є кілька рядків коду, потрібно використовувати `return`, щоб чітко вказати результат, який повертається.

Наприклад:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Тут процес обчислення поділений на два кроки.

Перший крок — обчислити загальну ціну:

```swift
let total = count * price
```

Другий крок — об’єднати загальну ціну в текст і повернути її:

```swift
return "Total：\(total) $"
```

Якщо прибрати `return`:

```swift
var totalPriceText: String {
    let total = count * price
    "Total：\(total) $"
}
```

цей код спричинить помилку.

Причина така: **у цій обчислюваній властивості вже є кілька рядків коду, тому Swift не може автоматично визначити, який рядок є фінальним результатом, що має бути повернений.**

Тому, якщо в обчислюваній властивості є лише один expression, який створює результат, `return` можна опустити.

```swift
var c: Int {
    a + b
}
```

Якщо в обчислюваній властивості є кілька рядків коду, краще явно писати `return`.

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

## Різниця між обчислюваною властивістю і звичайною властивістю

Звичайна властивість зберігає дані.

```swift
var c = 30
```

Тут `c` зберігає конкретне значення: `30`.

Обчислювана властивість не зберігає дані.

```swift
var c: Int {
    a + b
}
```

Тут `c` не зберігає `30`. Вона лише надає спосіб обчислення.

Коли `c` читається, Swift виконує:

```swift
a + b
```

Потім повертає обчислений результат.

Тому обчислювана властивість підходить для ситуацій, коли результат потрібно обчислювати на основі інших даних.

## body також є обчислюваною властивістю

Після вивчення обчислюваних властивостей ми можемо заново зрозуміти найпоширеніший код у SwiftUI:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Тут `body` також є обчислюваною властивістю.

Вона оголошена через `var`:

```swift
var body
```

Вона має return type:

```swift
some View
```

У її `{}` повертається вміст SwiftUI view:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

Тому `body` можна зрозуміти так: коли SwiftUI потрібно показати цей view, SwiftUI читає `body` і на основі вмісту, який повертає `body`, створює interface.

Якщо написати `return`, це можна зрозуміти так:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Але у SwiftUI зазвичай ми опускаємо `return` і пишемо безпосередньо:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Коли дані `@State` змінюються, SwiftUI знову читає `body` і оновлює interface відповідно до нових даних.

Наприклад:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("count: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

Після натискання кнопки `count` зміниться.

```swift
count += 1
```

Після зміни `count` SwiftUI знову обчислює `body`, тому текст в interface також оновлюється.

```swift
Text("count: \(count)")
```

Це також причина, чому у SwiftUI interface може автоматично оновлюватися після зміни даних.

### Не рекомендується писати складні обчислення в body

Оскільки `body` є обчислюваною властивістю, вона може читатися і переобчислюватися багато разів.

Наприклад:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("num: \(num)")
            Text("name: \(name)")
        }
    }
}
```

Коли `name` змінюється, SwiftUI знову обчислює `body`.

У цей момент код усередині `body` також виконується знову, і `num` також створюється заново.

```swift
let num = 10
```

У цьому прикладі `num` дуже простий, тому вплив невеликий.

Але якщо виконувати складні обчислення всередині `body`, наприклад filtering великої кількості даних, sorting, image processing тощо, це може вплинути на плавність interface.

Тому у SwiftUI `body` має головним чином відповідати за опис структури interface.

Прості тимчасові дані можна писати всередині `body`.

Складні обчислення краще винести за межі `body`, наприклад в обчислювану властивість, method або окрему частину обробки даних.

## Приклад: кількість і загальна ціна

Тепер розберемо обчислювану властивість на простому прикладі.

Припустімо, що одна морква коштує 2 $. Користувач може натискати кнопки, щоб змінювати кількість покупки, а interface має показувати загальну ціну.

Формула обчислення загальної ціни:

```text
Кількість * Ціна за одиницю
```

Якщо зберігати загальну ціну у звичайній variable, це буде доволі незручно.

Тому що щоразу, коли кількість змінюється, загальну ціну потрібно оновлювати вручну.

Кращий спосіб — використовувати обчислювану властивість:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Результат відображення:

![view](../../Resource/024_view.png)

У цьому прикладі:

```swift
@State private var count = 1
```

`count` означає кількість моркви. Коли натискається кнопка, `count` змінюється.

```swift
private let price = 2
```

`price` означає ціну однієї моркви. Тут це фіксоване значення, тому використовується `let`.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` означає загальну ціну.

Її не потрібно зберігати окремо, тому що загальну ціну завжди можна обчислити через `count * price`.

Коли `count` дорівнює `1`:

```swift
totalPrice = 1 * 2
```

Результат, що відображається:

```swift
totalPrice: 2 $
```

Після натискання кнопки `+` `count` стає `2`.

У цей момент, коли `totalPrice` читається знову, вона обчислюється повторно:

```swift
totalPrice = 2 * 2
```

Результат, що відображається:

```swift
totalPrice: 4 $
```

Саме в цьому полягає роль обчислюваної властивості: динамічно обчислювати новий результат на основі вже наявних даних.

## Обчислювану властивість також можна використовувати для перевірки умов

Обчислювана властивість може не лише обчислювати числа, а й повертати результат перевірки.

Наприклад, коли ми хочемо, щоб мінімальна кількість була 1.

Коли кількість уже дорівнює 1, кнопка `-` не повинна зменшувати значення далі.

Можна додати обчислювану властивість:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Повний код:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Результат відображення:

![view](../../Resource/024_view1.png)

Тут:

```swift
private var canDecrease: Bool {
    count > 1
}
```

цей код означає, чи можна зараз ще зменшувати кількість.

Коли `count` більше за `1`:

```swift
canDecrease == true
```

це означає, що можна зменшувати.

Коли `count` дорівнює `1`:

```swift
canDecrease == false
```

це означає, що далі зменшувати не можна.

### Перевірка умови всередині кнопки

У кнопці використовується:

```swift
if canDecrease {
    count -= 1
}
```

Лише коли `canDecrease` дорівнює `true`, можна зменшити `count`.

### Керування foreground color view

Також можна використовувати `canDecrease`, щоб керувати foreground color кнопки:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Тут використовується ternary operator:

```swift
canDecrease ? Color.primary : Color.gray
```

Значення цього коду таке: якщо `canDecrease` дорівнює `true`, foreground color використовує `Color.primary`; якщо `canDecrease` дорівнює `false`, foreground color використовує `Color.gray`.

`Color.primary` — це system semantic color, який надає SwiftUI. Він означає основний колір тексту в поточному interface.

У Light Mode `Color.primary` зазвичай близький до чорного; у Dark Mode зазвичай близький до білого.

Тому перевага використання `Color.primary` полягає в тому, що він автоматично адаптується до Light Mode і Dark Mode.

### Керування disabled-станом view

`disabled` використовується для керування тим, чи перебуває view у disabled-стані:

```swift
.disabled(!canDecrease)
```

Коли `disabled` дорівнює `false`, view можна натискати.

Коли `disabled` дорівнює `true`, view перебуває у disabled-стані, і його не можна натискати.

Тут `canDecrease` використовується як умова, тому код легше зрозуміти.

Побачивши `canDecrease`, можна одразу зрозуміти, що це означає “чи можна зараз ще зменшувати”.

### Додаткове пояснення: чому перевірка виконується двічі?

У кнопці `-`:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Тут використовується і `.disabled(!canDecrease)`, і `if canDecrease` всередині кнопки.

`.disabled(!canDecrease)` відповідає за вимкнення кнопки на рівні interface, щоб користувач не міг її натиснути.

`if canDecrease` ще раз перевіряє умову перед виконанням коду. Лише коли зменшення можливе, виконується `count -= 1`.

Це подвійний захист. У реальній розробці, якщо кнопка вже disabled, внутрішню перевірку можна опустити. Але в навчальному прикладі її можна залишити, щоб роль `canDecrease` була зрозумілішою.

## Підсумок

У цьому уроці ми головним чином вивчили обчислювані властивості.

Обчислювана властивість не зберігає значення безпосередньо. Коли вона читається, вона обчислює результат на основі вже наявних даних.

Наприклад:

```swift
var c: Int {
    a + b
}
```

Тут `c` не потрібно зберігати окремо, тому що його можна обчислити через `a + b`.

Обчислювану властивість потрібно оголошувати через `var`, і потрібно вказувати return type.

```swift
var canDecrease: Bool {
    count > 1
}
```

Обчислювана властивість може повертати не лише числове значення, а й результат перевірки, текстовий вміст і навіть вміст SwiftUI view.

У цьому уроці ми також вивчили `return`.

`return` означає повернення результату:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Якщо в обчислюваній властивості є лише один expression, який безпосередньо створює результат, `return` можна опустити.

```swift
var totalPrice: Int {
    count * price
}
```

Крім того, ми також дізналися про `Color.primary` і `disabled`.

`Color.primary` — це system semantic color у SwiftUI. Він автоматично адаптує відображення відповідно до Light Mode і Dark Mode.

```swift
.foregroundStyle(Color.primary)
```

`disabled` використовується для керування тим, чи view вимкнене.

```swift
.disabled(true)
```

Це означає disabled, не можна натискати.

```swift
.disabled(false)
```

Це означає доступне, можна натискати.

Тому обчислювані властивості дуже часто використовуються у SwiftUI. Вони допомагають чіткіше організовувати результати обчислень, умови та вміст, що відображається.
