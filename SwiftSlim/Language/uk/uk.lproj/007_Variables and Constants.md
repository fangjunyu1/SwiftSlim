# Змінні та константи

У цьому уроці ми головним чином вивчимо змінні та константи в Swift, а також познайомимося з деякими поширеними типами даних і базовими операторами.

Це найосновніші знання в програмуванні, а також важлива база для подальшого вивчення SwiftUI.

## Змінні та константи в повсякденному житті

Ми можемо зрозуміти змінні та константи через звичайні речі з життя.

Змінні в повсякденному житті:

* у телевізорі вміст кожної програми різний
* погода щодня змінюється
* годинник змінюється щосекунди

Спільна риса цих речей у тому, що вони змінюються.

Якби на телевізорі завжди було лише одне зображення, погода завжди була сонячною, а стрілки годинника ніколи не рухалися, тоді ці речі були б константами.

Одне може змінюватися, а інше — ні.

## Розуміння змінних і констант

Під час розробки App зазвичай потрібно, щоб користувач заповнював або зберігав певну інформацію.

Наприклад:

* назву облікового запису
* дату народження
* контактні дані
* адресу

Ця інформація буде збережена і відображена.

Наприклад, користувач вводить у App ім’я:

```swift
FangJunyu
```

Нам потрібно зберегти це ім’я, щоб потім показати його в App.

Процес збереження можна спрощено уявити так: ми кладемо річ у шухляду.

Коли ми зберігаємо ім’я, це схоже на те, як ми кладемо його в шухляду.

Речей, які потрібно зберігати, може бути багато, тому і шухляд теж буде багато. Щоб розуміти, що лежить у кожній шухляді, нам потрібно дати кожній із них назву.

Наприклад:

```swift
name
```

У цьому прикладі `name` — це назва, а `FangJunyu` — це збережена інформація.

![Var](../../RESOURCE/007_var.png)

**У Swift, якщо потрібно зберігати дані, обов’язково слід оголосити змінну (`var`) або константу (`let`).**

Оскільки ім’я зазвичай можна змінювати, тут слід використовувати змінну.

```swift
var name = "FangJunyu"
```

Тут оголошено змінну з назвою `name`, тип цієї змінної — `String`, а значення — `"FangJunyu"`.

### Різниця між змінною та константою

Змінна оголошується за допомогою `var`:

```swift
var
```

Константа оголошується за допомогою `let`:

```swift
let
```

Наприклад:

```swift
var name = "FangJunyu"
let id = 123456
```

Якщо оголошено змінну, це означає, що значення можна змінювати; якщо оголошено константу, значення більше не можна змінити.

Отже, головна різниця між змінною і константою полягає в тому, чи дозволено змінювати значення.

### Зміна константи

Якщо код намагається змінити значення константи:

```swift
let name = "Sam"
name = "Bob"
```

Swift повідомить, що присвоєння неможливе, тому що це константа.

```swift
Cannot assign to property: 'name' is a 'let' constant
```

Такий механізм допомагає розробникам уникати випадкової зміни важливих даних.

### Відображення у SwiftUI

Відкрийте файл `ContentView.swift` і оголосіть змінну всередині `View`:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

Тут оголошено змінну з назвою `name`, і через `Text` вона відображається на екрані.

Якщо змінити значення змінної:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

Текст, який відображає `Text`, також зміниться.

Таким чином, ми можемо використовувати змінні для керування тим, що показується в інтерфейсі, і не змінювати щоразу текст безпосередньо всередині `Text`.

Якщо цю інформацію потрібно передати в інше місце, ми також можемо передати змінну далі — це схоже на те, ніби ми передаємо вміст “шухляди” для використання в іншому місці.

### Розташування змінних і констант у SwiftUI

У SwiftUI змінні та константи зазвичай записують поза `body`:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

Звичайно, змінні та константи також можна записати всередині `body`:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

Однак між цими двома способами є важлива різниця:

* якщо змінні та константи записані всередині `body`, вони створюватимуться заново щоразу під час повторного обчислення представлення
* якщо вони записані поза `body`, вони існуватимуть як властивості структури представлення, і код буде мати чіткішу організацію

Тому в реальній розробці змінні та константи зазвичай записують поза `body`.

## Типи даних

Змінні можуть зберігати багато різних видів даних, і ці види називаються типами даних.

Наприклад, раніше ми використовували `Text` для показу тексту:

```swift
Text("Hello, World")
```

Тут `"Hello, World"` належить до типу рядка (`String`).

Рядок використовується для представлення текстового вмісту, і в Swift його потрібно брати в подвійні лапки `""`.

Наприклад:

```swift
var hello = "Hello, World"
```

Окрім рядків, у Swift є багато інших типів даних.

На початковому етапі найпоширенішими є чотири типи:

* `String`
* `Int`
* `Double`
* `Bool`

Цих чотирьох типів достатньо для виконання більшості базових операцій з даними.

**String**

`String` означає текстовий вміст, наприклад:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

Рядки зазвичай використовуються для зберігання: імені користувача, заголовка, текстового вмісту.

**Int**

`Int` означає ціле число, тобто число без дробової частини.

Наприклад:

```swift
var age = 26
var count = 100
```

Цілі числа зазвичай використовуються для зберігання: віку, кількості, лічильників.

**Double**

`Double` означає число з дробовою частиною.

Наприклад:

```swift
var weight = 74.5
var height = 185.0
```

У Swift дробові числа зазвичай за замовчуванням мають тип `Double`.

Є ще схожий тип `Float`, але він має менший діапазон значень, тому в реальній розробці частіше використовують `Double`.

**Bool**

Тип `Bool` використовується для представлення двох станів: `true` і `false`.

Наприклад:

```swift
var isShowAlert = true
var isLogin = false
```

Тип `Bool` часто використовується для умовних перевірок. Наприклад, чи потрібно показувати вікно-підказку.

Для ситуацій, де можливі лише два результати, `Bool` підходить найкраще.

### Відображення у SwiftUI

Коли ми показуємо значення типу `String`, його можна безпосередньо передати в `Text`:

```swift
Text(name)
```

Але `Int`, `Double` та інші типи не є рядками, тому їх не можна напряму вставити як частину тексту.

Якщо потрібно показати такі дані в `Text`, можна використати інтерполяцію рядка (`String Interpolation`):

```swift
\()
```

Інтерполяцію рядка потрібно писати всередині рядка і обгортати змінну або константу в `\()`.

Наприклад:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

У наведеному коді `""` означає рядок, а `\()` використовується для вставлення змінної або константи в рядок.

Наприклад:

```swift
"DoubleNum: \(num * 2)"
```

На екрані буде показано:

```swift
DoubleNum: 2
```

Таким чином, ми можемо вставляти змінні або константи в рядок, а потім показувати їх за допомогою `Text`.

Підказка: `\()` можна використовувати лише всередині рядка `""`.

## Оператори

Коли ми оголошуємо змінну або константу, можна побачити такий запис:

```swift
var num = 1
```

Тут `=` називається оператором присвоєння.

Його роль полягає в тому, щоб присвоїти значення справа змінній зліва.

У цьому прикладі значення `1` присвоюється змінній `num`.

Окрім оператора присвоєння, є ще кілька поширених числових операторів:

* `+`
* `-`
* `*`
* `/`

Коли ми виконуємо обчислення з числами, наприклад із типами `Int` або `Double`, використовуються саме ці оператори.

Наприклад:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

Після виконання обчислення результат зберігається у змінній.

## Підсумок

Змінні, константи та оператори — це найбазовіші поняття в програмуванні.

За допомогою змінних і констант ми можемо зберігати в програмі різні дані; за допомогою типів даних можна чітко визначити вид цих даних; за допомогою операторів можна виконувати обчислення й обробку даних.

Ці знання схожі на базові інструменти світу програмування. Їхнє розуміння створює міцну основу для подальшого вивчення Swift і SwiftUI.
