# Власні кольори

У цьому уроці ми навчимося створювати власні кольори у SwiftUI.

У попередніх уроках ми вже використовували стандартні кольори, які надає SwiftUI, наприклад:

```swift
Color.blue
Color.red
Color.green
```

Ці кольори дуже зручно використовувати, але в реальній розробці App стандартні кольори часто недостатньо точні.

Наприклад, у дизайн-макеті можуть використовуватися такі кольори:

```text
#2c54c2
#4875ed
#213e8d
```

Такий колір називається Hex-кольором.

У цьому уроці ми спочатку додамо підтримку Hex-кольорів у SwiftUI, а потім використаємо `static`, щоб упорядкувати часто вживані кольори.

Насамкінець ми застосуємо власні кольори до подання енциклопедії тварин і додатково використаємо градієнтний фон, щоб кнопки виглядали більш об’ємно.

## Навіщо потрібні власні кольори?

У SwiftUI ми можемо напряму використовувати системні кольори.

Наприклад:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

Тут `.blue` означає синій колір. Насправді це скорочений запис `Color.blue`.

Перевага стандартних кольорів у тому, що вони прості й зручні, але вибір кольорів доволі обмежений.

Наприклад:

```swift
Color.blue
```

може означати лише стандартний синій колір, який надає SwiftUI.

![Color.blue](../../../Resource/031_color6.png)

Але в реальній розробці нам часто потрібні точніші кольори.

Наприклад, навіть синій колір може мати різні варіанти: світло-синій, темно-синій, сіро-синій, яскраво-синій тощо.

![More Blue](../../../Resource/031_color5.png)

У такому разі, якщо використовувати тільки `Color.blue`, буде складно відтворити ефект із дизайну.

Тому нам потрібно додати у SwiftUI підтримку власних кольорів.

## Що таке Hex-колір?

Колір на екрані зазвичай складається з трьох каналів: червоного, зеленого й синього, тобто RGB.

RGB відповідно означає:

```text
Red     // червоний
Green   // зелений
Blue    // синій
```

Hex-колір — це один зі способів запису RGB-кольору.

Наприклад:

```swift
#5479FF
```

Це значення кольору можна спрощено розуміти як три частини:

```text
54  // означає червоний канал
79  // означає зелений канал
FF  // означає синій канал
```

У цьому уроці нам не потрібно обчислювати ці значення і не потрібно глибоко розбиратися в правилах шістнадцяткової системи.

Зараз достатньо знати: `#5479FF` означає конкретний колір.

Надалі, коли ми бачитимемо записи на кшталт `#2c54c2`, `#4875ed`, їх можна спочатку сприймати як значення кольору.

У таких дизайн-інструментах, як Sketch, Figma, Photoshop, також часто можна побачити подібні значення кольорів.

![color](../../../Resource/031_color.png)

Але за замовчуванням у SwiftUI не можна напряму написати так:

```swift
Color(hex: "#5479FF")
```

Тому нам потрібно самостійно розширити тип `Color`, щоб він підтримував створення кольору через Hex-рядок.

## Створення файлу Color+Hex.swift

Спочатку створимо новий Swift-файл.

Ім’я файлу можна записати так:

```text
Color.swift
```

Також можна використати зрозумілішу назву:

```text
Color+Hex.swift
```

Тут більше рекомендовано використовувати:

```text
Color+Hex.swift
```

У Swift-проєктах назви файлів на кшталт `Color+Hex.swift` трапляються дуже часто.

Вона означає, що цей файл є файлом розширення, який додає `Color` можливість працювати з Hex.

Назва файлу не впливає напряму на виконання коду. Вона лише допомагає легше зрозуміти призначення цього файлу.

## Додавання коду Color(hex:)

У файлі `Color+Hex.swift` напишіть наведений нижче код:

```swift
import SwiftUI

extension Color {
    init(hex: String) {
        let scanner = Scanner(string: hex)
        
        if hex.hasPrefix("#") {
            scanner.currentIndex = hex.index(after: hex.startIndex)
        }
        
        var rgbValue: UInt64 = 0
        scanner.scanHexInt64(&rgbValue)
        
        let red = Double((rgbValue & 0xFF0000) >> 16) / 255.0
        let green = Double((rgbValue & 0x00FF00) >> 8) / 255.0
        let blue = Double(rgbValue & 0x0000FF) / 255.0
        
        self.init(red: red, green: green, blue: blue)
    }
}
```

Цей код використовує `extension`, щоб розширити тип `Color`, і додає до `Color` новий ініціалізатор:

```swift
init(hex: String)
```

Після появи цього ініціалізатора ми зможемо передавати Hex-рядок у `Color` і створювати власний колір:

```swift
Color(hex: "#5479FF")
```

Усередині цього розширення Hex-рядок буде перетворено на RGB-колір, який SwiftUI може розпізнати.

На цьому етапі не потрібно детально розуміти логіку перетворення в кожному рядку. Достатньо знати, що доданий метод `Color(hex:)` дозволяє створювати власні кольори через Hex-значення.

## Використання власних кольорів

Тепер ми можемо протестувати власні кольори у `ContentView`.

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Color.blue")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color.blue)
            
            Text("#2c54c2")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#2c54c2"))
            
            Text("#4875ed")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#4875ed"))
            
            Text("#213e8d")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#213e8d"))
        }
        .padding()
    }
}
```

Результат відображення:

![color](../../../Resource/031_color1.png)

У цьому прикладі перший рядок використовує системний колір:

```swift
.foregroundStyle(Color.blue)
```

Наступні три рядки використовують власні Hex-кольори:

```swift
.foregroundStyle(Color(hex: "#2c54c2"))
.foregroundStyle(Color(hex: "#4875ed"))
.foregroundStyle(Color(hex: "#213e8d"))
```

Порівнявши їх, можна помітити, що системний синій має лише один стандартний ефект.

А Hex-кольори можуть показувати тонші зміни синього.

У цьому й полягає цінність власних кольорів: вони дозволяють зробити кольори інтерфейсу ближчими до реального дизайну й зручніше контролювати візуальний стиль App.

## Використання static для впорядкування кольорів

Тепер ми вже можемо створювати кольори через Hex-рядок:

```swift
Color(hex: "#2c54c2")
```

Такий запис працює нормально, але якщо один і той самий колір повторюється в багатьох місцях, подальше обслуговування стане менш зручним.

Якщо це значення кольору написане в 10 місцях, то в майбутньому для зміни цього синього кольору доведеться змінювати кожне місце окремо.

У такій ситуації можна використати `static`, щоб централізовано впорядкувати часто вживані кольори.

Унизу файлу `Color+Hex.swift` продовжте додавати такий код:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Тут ми додали до `Color` три статичні властивості: `animalBlue`, `animalLightBlue` і `animalDarkBlue`. Вони відповідно означають сині кольори різної світлоти й темноти.

Оскільки ці властивості використовують `static`, вони належать самому типу `Color`.

Під час використання до них можна звертатися напряму через `Color.`:

```swift
Color.animalBlue
```

Такий запис зрозуміліший, ніж пряме написання Hex-рядка.

Коли ми бачимо `Color.animalBlue`, можна зрозуміти, що це синій колір, який використовується в енциклопедії тварин.

А коли бачимо `Color(hex: "#2c54c2")`, ми лише знаємо, що це значення кольору, але не так легко зрозуміти його конкретне призначення.

Є ще одна перевага централізованого керування кольорами: подальші зміни стають зручнішими.

Якщо потрібно змінити основний колір енциклопедії тварин, достатньо змінити місце його визначення:

```swift
static let animalBlue = Color(hex: "#1f5cff")
```

Усі місця, де використовується цей колір, оновляться разом.

У цьому сенс використання `static` для впорядкування кольорів: назви кольорів стають зрозумілішими, а подальше обслуговування — зручнішим.

## Застосування до енциклопедії тварин

Тепер ми можемо застосувати власні кольори до попереднього подання енциклопедії тварин.

Раніше фон кнопки тварини був білим:

```swift
.background(Color.white)
```

![single white button](../../../Resource/031_color7.png)

Тепер його можна змінити на власний колір:

```swift
.background(Color.animalBlue)
```

![single blue button](../../../Resource/031_color8.png)

Тут `Color.animalBlue` — це статичний колір, який ми щойно визначили у `Color+Hex.swift`.

Текст кнопки використовує білий колір:

```swift
.foregroundStyle(Color.white)
```

Фон emoji тварини використовує напівпрозорий білий колір:

```swift
.background(Color.white.opacity(0.15))
```

Так кнопка сформує єдиний синій візуальний стиль.

Головне в цьому кроці — не додати складний код, а справді застосувати власні кольори, які ми вивчили раніше, в інтерфейсі.

## Використання градієнтного фону

Окрім одного кольору, ми також можемо поєднати кілька кольорів у градієнтний ефект.

Наприклад, раніше ми визначили такі кольори:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Ці кольори можна використовувати не лише окремо, а й об’єднувати в градієнтний фон.

У SwiftUI для створення лінійного градієнта можна використовувати `LinearGradient`.

Наприклад:

```swift
LinearGradient(
    colors: [Color.animalBlue, Color.animalLightBlue],
    startPoint: .leading,
    endPoint: .trailing
)
```

Цей код створює градієнтний фон зліва направо: колір поступово переходить від `Color.animalBlue` до `Color.animalLightBlue`.

Серед них `colors` задає кольори, що беруть участь у градієнті, а `startPoint` і `endPoint` керують напрямком градієнта.

### Тестування ефекту градієнта

Ми можемо протестувати градієнтний фон на простому `Text`.

```swift
Text("Linear Gradient")
    .font(.title)
    .fontWeight(.bold)
    .padding()
    .foregroundStyle(Color.white)
    .background {
        LinearGradient(
            colors: [Color.animalBlue, Color.animalLightBlue],
            startPoint: .leading,
            endPoint: .trailing
        )
    }
    .cornerRadius(12)
```

Результат відображення:

![color](../../../Resource/031_color3.png)

У цьому прикладі фон `Text` уже не є одним кольором, а поступово змінюється зліва направо.

Порівняно зі звичайним фоновим кольором, градієнтний фон має більше шарів і легше створює візуальний акцент в інтерфейсі.

## Використання градієнтного фону в енциклопедії тварин

Тепер ми можемо взяти фоновий колір кнопки тварини:

```swift
.background(Color.animalBlue)
```

і змінити його на градієнтний фон:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .leading,
        endPoint: .trailing
    )
}
```

Повний код наведено нижче:

```swift
import SwiftUI

struct ContentView: View {
    @State private var selectedAnimal: Animal? = nil
    
    let animals = Animal.animals
    
    var body: some View {
        VStack(alignment: .leading, spacing: 20) {
            ForEach(animals) { animal in
                Button(action: {
                    selectedAnimal = animal
                }, label: {
                    HStack(spacing: 10) {
                        Text(animal.avatarEmoji)
                            .font(.title2)
                            .frame(width: 40, height: 40)
                            .background(Color.white.opacity(0.15))
                            .cornerRadius(20)
                        
                        Text(animal.name)
                            .fontWeight(.bold)
                            .foregroundStyle(Color.white)
                        
                        Spacer()
                    }
                    .padding(15)
                    .frame(width: 200, height: 60)
                    .background {
                        LinearGradient(
                            colors: [Color.animalBlue, Color.animalLightBlue],
                            startPoint: .leading,
                            endPoint: .trailing
                        )
                    }
                    .cornerRadius(12)
                    .shadow(radius: 5)
                })
                .buttonStyle(.plain)
            }
        }
        .padding()
        .sheet(item: $selectedAnimal) { animal in
            AnimalDetailView(animal: animal)
        }
    }
}
```

Результат відображення:

![color](../../../Resource/031_color4.png)

Тепер кнопка тварини вже не має лише однотонного фону, а отримала градієнтний ефект зліва направо.

Порівняно з однотонним фоном, градієнтний фон робить інтерфейс більш багатошаровим і ближчим до візуального дизайну реального App.

## Різниця між однотонним і градієнтним фоном

Однотонний фон використовує лише один колір.

Наприклад:

```swift
.background(Color.animalBlue)
```

Такий запис простий і зрозумілий, він підходить для більшості базових інтерфейсів.

Градієнтний фон використовує кілька кольорів.

Наприклад:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .topLeading,
        endPoint: .bottomTrailing
    )
}
```

Такий запис може зробити інтерфейс більш багатошаровим, але також легше збільшує візуальну складність.

Тому в реальній розробці це можна розуміти так:

Однотонний фон підходить для звичайного тексту, звичайних кнопок, звичайних карток і звичайного фону сторінки.

Градієнтний фон підходить для ключових кнопок, верхніх областей, обкладинок-карток, входів до функцій та інших місць, які потрібно підкреслити.

У цьому уроці ми навчилися використовувати власні кольори у SwiftUI.

Спочатку ми ознайомилися з Hex-кольорами.

```text

```

Він означає конкретний колір.

Це дозволяє SwiftUI створювати колір таким способом:

```swift
Color(hex: "#2c54c2")
```

```swift
extension Color {

```

Так в інших поданнях можна напряму використовувати:

Color.animalBlue
```

Порівняно з повторним написанням Hex-рядка щоразу, цей спосіб зрозуміліший і зручніший для подальшого централізованого редагування кольорів.

LinearGradient(

```

Через цей урок ми пройшли процес від «створення власного кольору» до «використання кольору в реальному інтерфейсі».

Надалі, коли певний колір потрібно використовувати повторно в багатьох місцях, можна спочатку подумати про те, щоб упорядкувати його в розширенні `Color`.

Так код буде зрозумілішим, а стиль інтерфейсу — легше уніфікувати.
