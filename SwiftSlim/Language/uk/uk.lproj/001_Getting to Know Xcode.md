# Знайомство з Xcode

## Що таке Xcode

Xcode — це офіційне інтегроване середовище розробки Apple (IDE, Integrated Development Environment), яке використовується для створення застосунків для платформ Apple, таких як iOS, macOS, watchOS і tvOS.

![Swift](../../RESOURCE/001_xcode.png)

Упродовж усього навчального курсу ми будемо використовувати Xcode для написання коду та запуску програм.

У цьому уроці ми дізнаємося, як завантажити Xcode, створити свій перший застосунок для iOS і початково ознайомитися зі структурою інтерфейсу Xcode.

## Завантаження Xcode

Наразі Xcode можна встановити двома способами:

1. Завантажити через [Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12)

2. Завантажити через [офіційний сайт Apple Developer](https://developer.apple.com/xcode/)

Обидва способи дозволяють встановити Xcode. Різниця полягає в тому, що на офіційному сайті Apple Developer, окрім стабільної версії, також можна завантажити [Beta](https://developer.apple.com/download/all/?q=Xcode) версію або старіші версії.

Якщо ви лише навчаєтеся розробці, рекомендується використовувати стабільну версію. Beta-версія в основному призначена для тестування нових функцій і може працювати нестабільно.

## Створення першого проєкту

Після відкриття Xcode ви побачите стартовий екран.

![Swift](../../RESOURCE/001_xcode1.png)

Ліворуч містяться три поширені варіанти:

* Create New Project —— створити новий проєкт
* Clone Git Repository —— клонувати Git-репозиторій
* Open Existing Project —— відкрити наявний проєкт

Праворуч відображаються нещодавно відкриті проєкти Xcode.

Ми обираємо варіант "Create New Project".

### Вибір шаблону проєкту

![Swift](../../RESOURCE/001_xcode2.png)

Після цього відкриється екран вибору шаблону проєкту, де показані різні платформи (iOS, macOS тощо) та різні типи шаблонів.

Пояснення поширених шаблонів:

* App —— найпростіший тип застосунку (рекомендовано)
* Document App —— застосунок, побудований на моделі документів
* Game —— для розробки ігор
* Framework —— модуль для повторного використання

На початковому етапі ми обираємо лише шаблон "iOS" - "App", оскільки це тип застосунку з інтерфейсом.

Натисніть "Next".

### Заповнення інформації про проєкт

Далі відкриється екран заповнення інформації про проєкт, де потрібно ввести дані відповідно до призначення кожного поля.

![Swift](../../RESOURCE/001_xcode3.png)

Значення полів:

* Product Name —— назва проєкту, наприклад: SwiftSlimTest
* Team —— команда розробника; якщо у вас немає платного акаунта Apple Developer, це поле можна не вибирати
* Organization Identifier —— ідентифікатор організації, зазвичай використовується зворотний формат домену, наприклад:

  * com.yourname
  * com.yourcompany
  * якщо у вас немає домену, можна використати своє ім’я англійською як ідентифікатор
* Bundle Identifier —— унікальний ідентифікатор застосунку, який автоматично формується на основі Organization Identifier і Product Name
* Interface —— технологія інтерфейсу, оберіть SwiftUI
* Language —— мова розробки, оберіть Swift
* Testing System —— за замовчуванням створюється ціль для тестування (Unit Tests); на початковому етапі це можна проігнорувати
* Storage —— фреймворк для локального збереження даних, який може автоматично інтегрувати SwiftData або Core Data; на початковому етапі це також можна проігнорувати

Після заповнення натисніть "Next".

### Збереження проєкту

![Swift](../../RESOURCE/001_xcode4.png)

Оберіть відповідну папку для збереження.

Натисніть кнопку "Create", і Xcode автоматично згенерує структуру проєкту.

## Структура проєкту Xcode

Після збереження проєкту Xcode у Finder можна побачити створену папку проєкту.

![Swift](../../RESOURCE/001_xcode5.png)

* Assets.xcassets —— папка для зберігання зображень, іконок застосунку та інших ресурсів

* ContentView.swift —— стандартний файл SwiftUI-вигляду, у якому ми надалі писатимемо код інтерфейсу

* SwiftSlimTestApp.swift —— файл входу в застосунок (App Entry Point), який відповідає за запуск застосунку; поки що нам не потрібно розбирати його детально

* SwiftSlimTest.xcodeproj —— файл проєкту Xcode; подвійне натискання по ньому відкриває проєкт

## Знайомство з інтерфейсом Xcode

Подвійно клацніть файл .xcodeproj, щоб відкрити проєкт, і в лівій області Navigator оберіть файл ContentView.swift.

Інтерфейс Xcode головним чином складається з п’яти частин:

* Navigator —— область навігації, використовується для перегляду структури файлів і пошуку
* Editor —— область редагування, використовується для написання коду Swift або SwiftUI
* Canvas —— область полотна, використовується для попереднього перегляду SwiftUI
* Inspector —— область перевірки властивостей, використовується для перегляду та зміни властивостей файлу
* Debug Area —— область налагодження, використовується для перегляду журналів виводу

![Swift](../../RESOURCE/001_xcode6.png)

Зверніть увагу: Canvas головним чином використовується для попереднього перегляду SwiftUI-виглядів. З точки зору офіційної архітектури Xcode, це допоміжна панель попереднього перегляду (Preview Pane) для Editor. Canvas можна ввімкнути лише тоді, коли відкрито файл SwiftUI-вигляду; для інших типів файлів функція попереднього перегляду не показується.

У наступних уроках ми головним чином писатимемо код Swift і SwiftUI в області Editor.

Під час першого відкриття проєкту Xcode області Inspector і Debug Area за замовчуванням можуть бути приховані; їх можна показати за допомогою кнопок у правому верхньому та правому нижньому куті.

![Swift](../../RESOURCE/001_xcode7.png)

Підказка: область Inspector головним чином використовується для перегляду та зміни властивостей файлів. У реальній розробці цю область зазвичай приховують, щоб отримати більше простору для редагування.

## Підсумок

У цьому уроці ми дізналися, як завантажити Xcode, створити свій перший проєкт Xcode, а також ознайомилися з базовою структурою інтерфейсу Xcode.

Далі — час для вільної практики:

* Ви можете спробувати створювати й видаляти файли в області Navigator.
* Перегляньте код в області Editor і зверніть увагу на його структуру.
* Перегляньте кнопки в області Canvas і спробуйте збільшити або зменшити вікно попереднього перегляду.

На наступному уроці ми почнемо писати простий код, і це допоможе нам ще краще зрозуміти Xcode.
