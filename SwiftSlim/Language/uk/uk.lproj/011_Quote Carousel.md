# Карусель цитат

У цьому уроці ми реалізуємо функцію "каруселі цитат" і детальніше вивчимо базові знання Swift, зокрема масиви (Array) та умовні оператори (if-else).

Ми покажемо, як зберігати кілька цитат і за допомогою взаємодії з кнопкою реалізувати їх циклічне переключення.

## Відображення цитати

Спочатку нам потрібно відобразити одну цитату в SwiftUI.

Найпростіший спосіб — використати подання Text:

```swift
Text("Slow progress is still progress.")
```

Цей код може відобразити лише одну фіксовану цитату. Якщо потрібно показувати кілька цитат і реалізувати перемикання між ними, ми маємо спочатку зберегти ці цитати.

Але звичайна рядкова змінна може зберігати лише одну цитату:

```swift
let sayings = "Slow progress is still progress."
```

Якщо потрібно зберігати кілька цитат, доведеться оголосити окрему змінну для кожної цитати:

```swift
let saying1 = "Slow progress is still progress."
let saying2 = "Difficult roads lead to beautiful destinations."
let saying3 = "Awake minds are the strongest."
let saying4 = "Long-term thinking wins over short-term tricks."
let saying5 = "Master one thing, and do it well."
```

Але в реальній розробці такий підхід не лише громіздкий, а й незручний: оскільки кожна змінна є незалежною, ми не можемо гнучко реалізувати перемикання в каруселі.

Щоб зручно керувати кількома цитатами, потрібно використати структуру даних, яка дозволяє зберігати їх разом. Саме для цього існує масив (Array).

Після використання масиву наведений вище код можна записати так:

```swift
let sayings = [
    "Slow progress is still progress.",
    "Difficult roads lead to beautiful destinations.",
    "Awake minds are the strongest.",
    "Long-term thinking wins over short-term tricks.",
    "Master one thing, and do it well."
    ]
```

**Підказка: у програмуванні змінні масиву, що зберігають кілька елементів, зазвичай мають назву в множині, наприклад sayings, щоб підкреслити їхню природу як колекції.**

## Масив

У Swift масив — це колекція впорядкованих елементів, яка позначається квадратними дужками `[]`.

```swift
[]
```

Усередині масиву може бути кілька елементів одного типу, а самі елементи розділяються англійською комою `,`.

Наприклад:

```swift
[101, 102, 103, 104, 105]
```

Масив можна просто уявити як потяг:

![Array](../../RESOURCE/011_array1.png)

Увесь потяг представляє об’єкт масиву, а кожен вагон розташований у певному порядку один за одним.

### Індекс і доступ до елементів

Оскільки масив є впорядкованим, система може знаходити конкретні елементи за їхнім порядком. Такий механізм позиціонування називається індексом (Index).

У Swift (як і в більшості мов програмування) індексація масиву починається з 0, а не з 1. Це означає, що перший елемент масиву має індекс 0, другий — 1, і так далі.

![Array](../../RESOURCE/011_array2.png)

Щоб отримати доступ до певного елемента масиву, достатньо після назви масиву поставити квадратні дужки й указати в них індекс потрібного елемента.

Наприклад:

```swift
sayings[0]   // 101
sayings[1]   // 102
```

Якщо спробувати звернутися до індексу, який виходить за межі допустимого діапазону масиву, виникне проблема "виходу індексу за межі масиву" (Index Out of Range). Тому під час доступу до масиву потрібно бути уважним і стежити, щоб індекс залишався в допустимих межах.

**Вихід індексу за межі масиву**

Наприклад, якщо масив містить лише 5 елементів, допустимий діапазон індексів — від 0 до 4. Якщо ми спробуємо звернутися до `sayings[5]`, програма не зможе знайти відповідний "вагон", і це спричинить помилку "вихід індексу за межі масиву", що призведе до аварійного завершення застосунку.

![Array](../../RESOURCE/011_array3.png)

### Операції з масивом

Масив підтримує не лише статичне визначення, а й операції додавання, видалення, зміни елементів, а також отримання його довжини.

Підказка: якщо потрібно змінювати масив, його обов’язково слід оголошувати через `var`, а не через константу `let`.

**1. Додавання елемента**

Для додавання елемента в кінець масиву можна використати метод `append`:

```swift
var num = [0, 1, 2]
num.append(3)   // [0, 1, 2, 3]
```

**2. Видалення елемента**

За допомогою методу `remove(at:)` можна видалити конкретний елемент масиву:

```swift
var num = [0, 1, 2]
num.remove(at: 0)   // [1, 2]
```

**3. Зміна елемента**

Елемент масиву можна змінити напряму через індекс:

```swift
var num = [0, 1, 2]
num[1] = 4   // [0, 4, 2]
```

**4. Підрахунок довжини масиву**

Властивість `count` використовується для отримання кількості елементів у масиві:

```swift
var num = [0, 1, 2]
num.count   // 3
```

### Використання масиву для відображення цитат

Щоб відображати кілька цитат, ми можемо зберегти їх у масиві, а потім отримувати й показувати потрібну цитату через індекс.

Спочатку створимо в `ContentView` масив `sayings` для зберігання цитат, а потім у поданні `Text` зчитуватимемо та показуватимемо відповідну цитату через індекс:

```swift
struct ContentView: View {
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[0])
        }
        .padding()
    }
}
```

Тут `sayings[0]` означає першу цитату в масиві.

Якщо потрібно показати іншу цитату, достатньо змінити значення індексу в квадратних дужках:

```swift
Text(sayings[0]) // "Slow progress is still progress."
Text(sayings[1]) // "Difficult roads lead to beautiful destinations."
```

### Оголошення індексу цитати

Щоб реалізувати динамічне перемикання цитат, ми не можемо "жорстко" записувати індекс безпосередньо в `Text`.

Нам потрібно створити змінну, яка окремо зберігатиме значення поточного індексу.

У SwiftUI для цього можна використати `@State`, щоб оголосити змінний індекс:

```swift
@State private var index = 0
```

SwiftUI буде стежити за змінною, обгорнутою в `@State`, і коли `index` зміниться, SwiftUI повторно відрендерить подання та покаже відповідну цитату.

Далі ми використовуємо `sayings[index]`, щоб динамічно отримувати цитату з масиву:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[index])
        }
        .padding()
    }
}
```

Коли значення `index` змінюється, `Text` показує іншу цитату.

### Використання кнопки для керування індексом

Щоб керувати перемиканням цитат, можна використати `Button` для зміни значення `index`. Кожного разу після натискання кнопки `index` збільшуватиметься на 1:

```swift
Button("Next") {
    index += 1
}
```

Коли ми натискаємо кнопку, значення `index` змінюється з 0 на 1, що запускає оновлення подання, і `Text(sayings[index])` зчитує наступну цитату.

Однак тут є потенційна проблема: якщо натискати кнопку багато разів поспіль, `index` збільшиться до значення, яке виходить за межі масиву, і це призведе до помилки індексації. Наприклад, коли `index` стане 5 (а допустимі індекси масиву — від 0 до 4), програма аварійно завершиться.

Щоб запобігти виходу індексу за межі масиву, потрібно додати умову, яка гарантуватиме, що `index` не перевищить межі масиву. Це можна реалізувати за допомогою конструкції `if-else`, перевіряючи, чи менший `index` за довжину масиву.

## Керування умовами: оператор if-else

Оператор `if-else` — одна з найуживаніших умовних конструкцій у Swift. Він використовується для перевірки умови та виконання різних блоків коду залежно від того, істинна вона чи ні.

Базова структура:

```swift
if condition {
    // Код, який виконується, якщо condition == true
} else {
    // Код, який виконується, якщо condition == false
}
```

У конструкції `if` умова `condition` є булевим значенням (тип `Bool`) і може бути `true` або `false`. Якщо умова дорівнює `true`, виконується код у блоці `if`, інакше — код у блоці `else`.

Наприклад:

```swift
let age = 25
if age > 18 {
    print("Big Boy")
} else {
    print("Little Body")
}
```

У цьому прикладі значення `age` дорівнює 25, тому оператор `if` перевіряє, чи виконується умова `age > 18`. Оскільки умова істинна, буде виведено `"Big Boy"`.

Якщо блок `else` не потрібен, його можна опустити:

```swift
if condition {
    // Код, який виконується, якщо condition == true
}
```

### Використання умови для контролю діапазону індексу

Щоб уникнути виходу індексу за межі масиву, можна використати `if`, який перевірятиме, чи не перевищує `index` допустимий діапазон:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    }
}
```

Розбір логіки: значення `sayings.count` дорівнює 5 (тобто всього є 5 цитат), отже `sayings.count - 1` дорівнює 4, а це і є останній допустимий індекс масиву.

Коли `index` менший за 4, натискання кнопки й збільшення на 1 є безпечним; щойно `index` досягає 4, умова перестає виконуватись, і натискання кнопки вже не дає жодного ефекту.

Тепер код уже реалізує перемикання цитат:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack {
            Text(sayings[index])
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                }
            }
        }
        .padding()
    }
}
```

### Циклічне відображення цитат

Якщо ми хочемо, щоб після показу останньої цитати при наступному натисканні кнопки знову відображалась перша цитата, тобто щоб цитати показувалися по колу, це можна реалізувати через блок `else`:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    } else {
        index = 0
    }
}
```

Коли натискається кнопка, якщо `index` досяг останнього елемента масиву, він буде скинутий до 0, і показ цитат піде по колу.

## Оптимізація подання цитат

Тепер логіка каруселі цитат уже реалізована, але ми ще можемо покращити інтерфейс, щоб він виглядав привабливіше.

Повний код:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack(spacing: 10) {
            Text(sayings[index])
                .padding(20)
                .background(.white)
                .cornerRadius(10)
                .opacity(0.8)
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                } else {
                    index = 0
                }
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .background(
            Image("001")
        )
    }
}
```

У цьому прикладі до `Text` додано напівпрозоре біле тло й заокруглення кутів, кнопка `Button` використовує стиль `.borderedProminent`, а для `VStack` встановлено фонове зображення.

Додаткове пояснення: коли для обробки фонового зображення використовується модифікатор `background()`, його стандартна поведінка полягає в тому, щоб максимально заповнити область компонування поточного подання. У багатьох випадках він також природно розтягується до безпечної області (`Safe Area`).

## Додаткові знання — множинні умови: конструкція if-else if-else

У реальній розробці часто доводиться працювати з кількома умовами. Наприклад, у певній грі, якщо рахунок дорівнює 1, спрацьовує подія A; якщо дорівнює 2 — подія B; якщо дорівнює 3 — подія C, і так далі.

Коли умовних гілок більше ніж дві, нам потрібна конструкція `if-else if-else` для множинної перевірки умов.

Базовий синтаксис:

```swift
if conditionA {
    // Код, який виконується, якщо conditionA == true
} else if conditionB {
    // Код, який виконується, якщо conditionB == true
} else if conditionC {
    // Код, який виконується, якщо conditionC == true
} else {
    // Код, який виконується, якщо жодна умова не виконалась
}
```

У такій ситуації програма послідовно перевіряє умови й виконує код першої умови, яка виявиться істинною. Якщо жодна умова не підходить, виконується код у блоці `else`.

У каруселі цитат ми також можемо використати конструкцію `if-else if-else` для множинної перевірки:

```swift
Button("Next") {
    if index == 0 {
        index += 1
    } else if index == 1 {
        index += 1
    } else if index == 2 {
        index += 1
    } else if index == 3 {
        index += 1
    } else if index == 4 {
        index = 0
    } else {
        index = 0
    }
}
```

Коли `index` дорівнює 0, 1, 2 або 3, кожне натискання кнопки виконує `index += 1`; коли `index` дорівнює 4 (останній елемент), значення `index` скидається до 0, і таким чином реалізується цикл.

Гілка `else` використовується як запасний варіант, щоб уникнути проблем у разі появи некоректного значення (наприклад, якщо `index` було змінено помилково).

Зверни увагу, що тут оператор `==` означає перевірку "чи дорівнює". Якщо в операторі `if` перевірка показує, що `index` дорівнює певному числу, результатом буде `true`, і виконається відповідний блок коду. Якщо значення не збігається, програма перейде до перевірки наступного `if`.

Така множинна перевірка корисна тоді, коли в різних умовних сценаріях потрібно виконувати різний код.
