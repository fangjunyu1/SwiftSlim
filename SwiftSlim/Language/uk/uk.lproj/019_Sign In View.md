# Екран входу

У цьому уроці ми вивчимо дуже практичний сценарій — екран входу.

Незалежно від того, чи це вебсайт, чи застосунок, у багатьох випадках під час використання користувачу потрібно увійти в обліковий запис і ввести пароль.

Наприклад, сторінка входу GitHub:

![WordPress](../../Resource/019_github.png)

У цьому уроці ми створимо схожий екран входу, щоб користувач міг вручну ввести обліковий запис і пароль та перевірити, чи немає проблем у введеному вмісті.

## Верхнє компонування

Ми будемо писати код екрана входу у файлі `ContentView`.

Спочатку створимо область ідентифікації у верхній частині екрана входу. Ви можете заздалегідь підготувати відповідне зображення іконки й помістити його в папку `Assets`.

![icon](../../Resource/019_icon1.png)

Потім використайте `Image` і модифікатори, щоб показати зображення:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

Ці модифікатори означають:

- `resizable()`: дозволяє змінювати розмір зображення.
- `scaledToFit()`: масштабує зображення зі збереженням початкових пропорцій.
- `frame(width: 100)`: задає ширину відображення зображення `100`.

Далі використайте `Text` і модифікатори, щоб показати заголовок входу:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

Потім використайте `VStack`, щоб розташувати зображення і текст вертикально:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

Результат:

![view](../../Resource/019_view.png)

На цьому ми завершили верхню область заголовка екрана входу.

### Оптимізація компонування

Зараз зображення та заголовок за замовчуванням розташовані в центральній частині всього макета.

Якщо ми хочемо, щоб вони більше нагадували «позначку входу у верхній частині сторінки», можна використати `Spacer()`, щоб розтягнути решту простору та змістити вміст ближче до верху.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

Результат:

Але тепер `Image` і `Text` розташовані надто близько до верхнього краю екрана і виглядають дещо тісно.

У цей момент ми можемо використати `padding`, щоб додати верхній відступ для всього `VStack`.

## Ім'я користувача та пароль

Сторінка входу зазвичай вимагає, щоб користувач ввів ім'я користувача та пароль.

У SwiftUI ми можемо використовувати `TextField`, щоб отримувати вміст, який вводить користувач.

Але зверніть увагу: `TextField` сам по собі не зберігає введені дані надовго, це лише елемент введення. Насправді ці дані зберігають змінні, які ми до нього прив'язуємо.

Тому спочатку нам потрібно створити дві змінні `@State`, щоб зберігати ім'я користувача та пароль:

```swift
@State private var user = ""
@State private var password = ""
```

Коли значення змінної `@State` змінюється, SwiftUI автоматично оновлює пов'язане представлення.

Далі використайте `TextField`, щоб прив'язати ці дві змінні:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

Тут `$user` і `$password` означають «прив'язку».

Тобто коли користувач вводить текст у поле, змінна теж синхронно змінюється; а коли змінюється змінна, синхронно змінюється й вміст, що відображається в полі.

Такий зв'язок, у якому «представлення і дані синхронізуються одне з одним», і є прив'язкою.

Зверніть увагу, що тут потрібно використовувати запис із `$`:

```swift
$user
```

Тому що `TextField` потрібен не звичайний рядок, а значення прив'язки, яке «може змінювати дані в обох напрямках».

### Відображення полів введення

Розмістімо їх у `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

```swift

```

То поле введення спочатку покаже текст-підказку, наприклад:

```swift
input user
```

Цей текст лише підказує користувачу, «що тут потрібно ввести», і не є реальним введеним вмістом.

### Оптимізація полів введення

Тепер поля введення вже можна використовувати, але стиль за замовчуванням досить простий.

Щоб зробити інтерфейс зрозумілішим, ми можемо додати заголовок перед полем введення та трохи покращити стиль самого поля.

Наприклад, спочатку додаймо заголовок:

```swift
Text("Username")
    .fontWeight(.bold)
```

Потім використаймо `HStack`, щоб розмістити заголовок і поле введення в одному рядку:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

Тут використовується `HStack`, тому що ми хочемо «показати заголовок зліва, а поле введення справа».

Слід звернути увагу, що `TextField` за замовчуванням займає весь доступний простір, що залишився.

![color](../../Resource/019_view6.png)

Щоб розміри двох полів введення були більш однаковими, можна використати `frame(width:)`, щоб задати їм ширину і зробити їх вигляд охайнішим.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

Потім додайте рамку, щоб поле введення було помітнішим:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Тепер додаймо і ім'я користувача, і пароль у представлення `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

## Логіка входу

Тепер додамо кнопці входу найпростішу логіку перевірки: **визначити, чи ввів користувач ім'я користувача і пароль**.

Ми хочемо реалізувати такі ситуації:

- Якщо користувач нічого не ввів, вивести `Empty`
- Якщо користувач ввів лише одне з двох, підказати, чого бракує
- Якщо введено і ім'я користувача, і пароль, вивести `Success`

### Не введено жодної інформації

Оскільки `user` і `password` за замовчуванням є порожніми рядками:

```swift
@State private var user = ""
@State private var password = ""
```

Коли користувач нічого не вводить, вони так і залишаються порожніми.

У Swift такі типи, як `String`, можуть використовувати `isEmpty`, щоб перевіряти, чи є вміст порожнім.

### Властивість isEmpty

`isEmpty` часто використовується для перевірки того, чи є порожніми рядки, масиви та інший вміст.

Наприклад:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

Якщо вміст порожній, `isEmpty` повертає `true`; якщо вміст не порожній, `isEmpty` повертає `false`.

Тому ми можемо використовувати його, щоб визначити, чи введено ім'я користувача і пароль.

### Використання isEmpty для перевірки змінних

Якщо користувач не ввів жодної інформації, тоді:

```swift
user.isEmpty // true
password.isEmpty    // true
```

У такому разі ми можемо записати умову так:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

Цей код означає: якщо `user` порожній і `password` теж порожній, тоді вивести:

```swift
Empty
```

Тут `&&` — це логічний оператор, що означає «і».

Тобто вся умова є істинною лише тоді, коли істинна умова зліва і також істинна умова справа.

Тому цей код виконається лише тоді, коли і ім'я користувача, і пароль порожні.

## Користувач заповнив лише частину

Тепер розгляньмо іншу ситуацію: користувач ввів лише ім'я користувача або лише пароль.

Наприклад:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

У цьому випадку ім'я користувача не порожнє, але пароль порожній.

Для перевірки ми можемо й далі використовувати `isEmpty`:

### Ситуація, коли користувач ввів обліковий запис або пароль

Якщо користувач ввів лише ім'я користувача або пароль, нам потрібно показати відповідну підказку про те, чого бракує.

Наприклад, якщо користувач ввів лише ім'я користувача:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Логіка цього коду така: якщо ім'я користувача порожнє, попросити ввести ім'я користувача. Інакше перевірити, чи пароль порожній; якщо пароль порожній, попросити ввести пароль.

### Користувач ввів усю інформацію

Якщо і ім'я користувача, і пароль уже введені, тоді `isEmpty` для обох поверне `false`.

У такому разі це можна записати так:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

Тут `!` означає «заперечення».

Наприклад:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

Тобто `user.isEmpty` використовується для перевірки, «чи порожнє ім'я користувача», а `!user.isEmpty` — для перевірки, «чи ім'я користувача не порожнє».

Для пароля логіка така сама.

Отже, зміст цього коду такий: ім'я користувача не порожнє, і пароль теж не порожній.

Коли обидві ці умови виконуються, виводиться:

```swift
Success
```

## Повна логіка перевірки

Тепер об'єднаємо ці три ситуації й запишемо їх у `Button`:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

Порядок виконання цього коду можна зрозуміти так:

Першим кроком спочатку перевіряємо:

```swift
if user.isEmpty && password.isEmpty
```

Якщо і ім'я користувача, і пароль порожні, відразу виводиться `Empty`.

Якщо обидва не порожні, програма переходить до перевірки наступного рядка:

```swift
else if user.isEmpty || password.isEmpty
```

Тут `||` означає «або».

Тобто достатньо, щоб хоча б одна умова була істинною, і весь вираз буде істинним.

Тому, доки ім'я користувача порожнє або пароль порожній, виконується блок коду `else if`.

Після входу в блок коду ми додатково перевіряємо, яке саме поле не було заповнене:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Якщо дві попередні умови не виконуються, це означає, що ім'я користувача не порожнє і пароль теж не порожній.

Тоді програма переходить в останній `else`:

```swift
print("Success")
```

Так ми завершили найпростішу логіку перевірки введення для входу.

## Перевірка імені користувача та пароля

Наведена вище логіка перевіряє лише, «чи було введено хоч щось».

Але в реальному застосунку просто ввести вміст недостатньо. Зазвичай ім'я користувача та пароль ще потрібно надіслати на сервер для перевірки.

Якщо сервер підтвердить, що ім'я користувача існує і пароль правильний, вхід буде дозволено; інакше з'явиться повідомлення про помилку входу.

Щоб потренувати цей процес, ми можемо спочатку задати в коді тимчасовий обліковий запис і пароль, імітуючи «правильні дані для входу»:

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

Коли і ім'я користувача, і пароль не порожні, далі ми порівнюємо їх, щоб перевірити, чи правильні вони:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

Тут `==` означає «чи рівні вони».

Наприклад:

```swift
user == userID
```

Тобто чи повністю збігається ім'я користувача, введене користувачем, із правильним ім'ям користувача, яке ми задали; для пароля логіка така сама.

Тому, коли введені ім'я користувача та пароль дорівнюють заданим значенням, виводиться `Success`; якщо хоча б одне з них не збігається, виводиться `Input error`.

Так ми завершили просту «імітацію перевірки входу».

Хоча це ще не справжня функція входу, підключена до сервера, цього вже достатньо, щоб допомогти нам зрозуміти: **після натискання кнопки може виконуватися різний код залежно від різних умов**.

## Підсумок

У цьому уроці ми завершили базовий екран входу й зосередилися насамперед на вивченні «логіки оцінки введеного вмісту на основі умов».

Основний зміст цього уроку — використання `isEmpty` для перевірки, чи є рядок порожнім, а також використання інструкцій `if` разом із логічними операторами для обробки різних ситуацій.

Серед них два дуже важливі логічні оператори:

- `&&`: означає «і», обидві умови мають виконуватися
- `||`: означає «або», достатньо виконання хоча б однієї умови

Коли ці умови зрештою дають `true` або `false`, інструкція `if` виконує різний код залежно від результату.

Завдяки цьому екрану входу ми вже почали знайомитися з підходом до розробки, який поєднує «інтерфейс + дані + логічне судження».

```swift
                .resizable()
                .scaledToFit()
                .frame(width: 60)

                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Button("Sign in") {
                if user.isEmpty && password.isEmpty {
                    print("Empty")
                } else if user.isEmpty || password.isEmpty {

                    } else if password.isEmpty {

                    }
                } else {

}

```
