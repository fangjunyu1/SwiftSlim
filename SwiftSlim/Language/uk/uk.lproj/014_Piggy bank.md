# Скарбничка

У цьому уроці ми створимо простий проєкт “Скарбничка”. Функціональність у ньому нескладна, але містить повну логіку взаємодії, тому він дуже добре підходить для початківців.

Через цей проєкт ми вивчимо `TextField` (поле введення), `border` (рамку), а також зрозуміємо, що таке прив’язка даних (`$`).

Наша кінцева мета — створити “скарбничку”, у яку можна ввести суму, натиснути кнопку збереження й накопичувати загальну суму грошей.

## Відображення загальної суми

Спочатку нам потрібно показати в інтерфейсі загальну суму грошей у “скарбничці”.

У `ContentView` оголосимо змінну для збереження загальної суми в “скарбничці”.

```swift
@State private var amount = 0
````

Потім використаємо `Text`, щоб показати це значення:

```swift
Text("\(amount)")
	.font(.largeTitle)
	.fontWeight(.bold)
```

Тут `.font` і `.fontWeight` використовуються для налаштування розміру та товщини шрифту.

```swift
struct ContentView: View {
    
    @State private var amount = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
			.font(.largeTitle)
			.fontWeight(.bold)
		}
    }
}
```

Коли `amount` змінюється, значення загальної суми в `Text` також автоматично оновлюється.

## Введення суми

Окрім відображення загальної суми, “скарбничка” повинна дозволяти користувачеві “вводити” суму для збереження. Для цього потрібно використати елемент інтерфейсу SwiftUI `TextField`.

### TextField

`TextField` — це елемент SwiftUI для введення тексту, який зазвичай використовується для однорядкового введення.

`TextField` переважно підтримує два формати прив’язки:

**1. Прив’язка до рядкового типу (`String`)**

```swift
TextField("Placeholder", text: $text)
```

Підходить для введення імені, заголовка та іншого текстового вмісту.

**2. Прив’язка до числового типу (`Int/Double`)**

```swift
TextField("Amount", value: $number, format: .number)
```

Підходить для введення віку, зарплати, суми та інших числових значень.

`format: .number` означає, що це поле введення буде аналізувати й відображати значення у “числовому форматі”.

#### Текст підказки

Перший параметр `TextField` — це текст підказки (`Placeholder`), який підказує користувачеві, що потрібно ввести:

```swift
TextField("input your name", text: $text)
```

Коли поле введення порожнє, у ньому показується сіра підказка.

![textfield](../../RESOURCE/014_textfield.png)

#### Прив’язка даних

`TextField` сам по собі не відповідає за постійне збереження введеного вмісту, тому даними потрібно керувати через прив’язану змінну.

```swift
@State private var text = ""
TextField("Placeholder", text: $text)
```

Текст, який вводить користувач, напряму зберігатиметься у змінній `text`, а `TextField` відповідає лише за інтерфейс введення.

`$` означає прив’язку (`Binding`):

```swift
$text
```

Це не звичайна змінна, а значення типу `Binding`.

Його роль — створити зв’язок між поданням і даними:

коли користувач вводить щось у `TextField`, `text` автоматично оновлюється; коли `text` змінюється, `TextField` теж синхронно оновлюється.

Наприклад:

```swift
struct ContentView: View {
    @State private var text = ""
    
    var body: some View {
        VStack {
			Text("text:\(text)")
        	TextField("input your name", text: $text)
            	.frame(width: 150)
		}
    }
}
```

Під час введення тексту в `TextField` верхній `Text` буде відображати його в реальному часі.

![textField](../../RESOURCE/014_textfield1.png)

### Прив’язка числового типу

Коли потрібно вводити число, слід прив’язати змінну відповідного типу:

```swift
@State private var number = 0

TextField("Amount", value: $number, format: .number)
```

Коли користувач вводить суму, `TextField` аналізує введене значення й автоматично оновлює `number`.

## Додавання поля введення

Після ознайомлення з базовим використанням `TextField` застосуймо його до проєкту “Скарбничка”.

У “скарбничці” потрібно вводити “суму”, тому ми використаємо спосіб із “прив’язкою числового типу (`Int/Double`)” і додамо ще одну змінну `number` для збереження введеної користувачем суми:

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
            	.font(.largeTitle)
            	.fontWeight(.bold)
        	TextField("Amount", value: $number, format: .number)
		}
    }
}
```

Тепер `TextField` за замовчуванням відображає значення `number` (а його початкове значення — `0`).

Коли вміст очищається, знову показується підказка `Amount`.

Це ще раз показує, що `TextField` відображає значення прив’язаної змінної, а не зберігає вміст самостійно.

### Проблема з розміром поля введення

У поточному поданні можна помітити, що `TextField` не має рамки, показує лише `0`, а сам текст ніби вирівняний по лівому краю.

Ефект виглядає так:

![textfield](../../RESOURCE/014_textfield2.png)

Це відбувається тому, що за замовчуванням `TextField` займає всю ширину батьківського подання.

Можемо підтвердити це, додавши фоновий колір:

```swift
TextField("Amount", value: $number, format: .number)
    .background(Color.red)
```

Ефект буде таким:

![textfield](../../RESOURCE/014_textfield3.png)

Можна побачити, що `TextField` фактично займає всю ширину рядка.

### Встановлення фіксованої ширини

Якщо ми хочемо, щоб поле введення було компактнішим, можна використати `frame`, щоб обмежити його ширину:

```swift
TextField("Amount", value: $number, format: .number)
	.frame(width: 100)
```

Ефект буде таким:

![textfield](../../RESOURCE/014_textfield4.png)

### Додавання рамки та внутрішніх відступів

Оскільки `TextField` за замовчуванням не має рамки, ми можемо вручну додати стиль:

```swift
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Тепер це поле введення виглядає більше схожим на “стандартне поле введення”.

![textfield](../../RESOURCE/014_textfield5.png)

## Рамка

У SwiftUI `border` використовується для додавання рамки до подання.

Базове використання:

```swift
border(Color.black, width: 1)
```

Тут `Color.black` означає колір рамки, а `width` — її товщину.

У `TextField` ми використовуємо:

```swift
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Це означає, що до поля введення додається чорна рамка товщиною `1 pt`.

Також можна спробувати інші кольори, наприклад `Color.blue`, `Color.green`, або збільшити товщину рамки, наприклад `width: 2`.

## Кнопка збереження

Тепер у нас є змінна `amount` для відображення загальної суми, `TextField` для введення суми, а також змінна `number`, прив’язана до поля введення.

Поточний код такий:

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
            	.font(.largeTitle)
            	.fontWeight(.bold)
        	TextField("Amount", value: $number, format: .number)
            	.frame(width: 100)
            	.padding(5)
            	.border(Color.black, width: 1)
		}
    }
}
```

### Додавання кнопки

Далі нам потрібно додати кнопку, яка запускатиме збереження грошей.

```swift
Button("Save") {
    
}
```

### Реалізація логіки збереження

Ми хочемо реалізувати таку поведінку: користувач вводить суму, натискає кнопку, і ця сума автоматично додається до загальної суми.

```swift
Button("Save") {
    amount += number
}
```

Тут використовується “складений оператор присвоєння”, який додає введену користувачем суму `number` до загальної суми `amount`.

### Скидання введеної суми

Зараз є одна проблема: після натискання кнопки сума в полі введення не очищується.

Якщо користувач введе `10`, натисне зберегти, у полі введення все одно залишиться `10`, і це заважатиме наступному введенню.

Це відбувається тому, що `TextField` прив’язаний до `number`, а при натисканні кнопки ми змінили лише `amount`, тоді як `number` не змінився, тому поле введення й далі показує старе значення.

Потрібно додати логіку очищення: після натискання кнопки спочатку додати введену суму до загальної, а потім скинути введену суму:

```swift
Button("Save") {
    amount += number
	number = 0
}
```

Тепер, коли користувач вводить суму й натискає кнопку, ця сума додається до загальної. Після цього `number` скидається до `0`, щоб підготувати поле до наступного введення.

Таким чином ми реалізували всю логіку збереження в “скарбничці”.

## Оптимізація подання

Після завершення функціональності ми можемо трохи покращити інтерфейс, додавши фонове зображення та стиль для кнопки.

Додати стиль кнопки:

```swift
.buttonStyle(.borderedProminent)
```

Додати фонове зображення:

```swift
.background {
    Image("1")
}
```

Тепер ми завершили всю розробку “скарбнички”.

## Повний код

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

Або запустити “скарбничку” в симуляторі чи на реальному пристрої, щоб спробувати власноруч створений нами застосунок.

### Налагоджувальний вивід

Щоб перевірити, чи логіка працює правильно, ми також можемо додати до кнопки налагоджувальний код і перевірити, чи правильно обробляється введена сума:

```swift
Button("Save") {
    print("---Saving---")
    amount += number
    number = 0
    print("amount:\(amount)")
    print("number:\(number)")
}
```

Щоразу, коли ми зберігаємо гроші й натискаємо кнопку, у Console з’являтиметься відповідний налагоджувальний вивід, за яким можна перевірити, чи немає помилок у логіці коду.

```swift
---Saving---
amount:11
number:0
```
