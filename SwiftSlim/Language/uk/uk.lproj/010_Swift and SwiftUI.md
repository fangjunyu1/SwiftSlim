# Swift і SwiftUI

Під час вивчення розробки для платформ Apple багато навчальних матеріалів схильні пояснювати Swift і SwiftUI окремо. Через це в початківців легко виникає відчуття розриву, ніби це дві повністю незалежні технології.

Насправді Swift і SwiftUI тісно інтегровані між собою: Swift — це мова програмування, яка відповідає за логіку та дані; SwiftUI — це фреймворк користувацького інтерфейсу, побудований на Swift, який відповідає за рендеринг і взаємодію з поданнями.

Цей урок допоможе тобі сформувати чітку картину: що таке Swift, що таке SwiftUI, і як вони співпрацюють між собою в коді.

## Що таке Swift?

Swift — це сучасна мова програмування, створена Apple для розробки застосунків на платформах iOS, macOS, watchOS і tvOS.

![swift](../../RESOURCE/010_swift.png)

Swift — це строго типізована статична мова, у дизайні якої особлива увага приділена безпеці. Вона зменшує кількість поширених помилок програмування (наприклад, null pointer, вихід за межі масиву тощо) і підтримує безшовну інтеграцію з C та Objective-C.

У реальній розробці Swift переважно відповідає за логіку застосунку, наприклад за обробку даних, мережеві запити, операції збереження тощо.

## Що таке SwiftUI?

SwiftUI — це новий фреймворк для побудови інтерфейсів, представлений Apple у 2019 році. SwiftUI використовує декларативну модель програмування, яка дозволяє розробнику за допомогою простішого коду описувати структуру інтерфейсу та поведінку взаємодії.

![swiftui](../../RESOURCE/010_swiftui.png)

Декларативне програмування означає, що розробнику достатньо сказати системі, що саме потрібно показати, а система сама автоматично оновлює подання відповідно до змін даних. Більше не потрібно вручну оновлювати стан інтерфейсу, і це суттєво зменшує складність UI-розробки.

Основу SwiftUI становлять компоненти подань (наприклад, Text, Image, Button тощо) і контейнери для компонування (наприклад, VStack, HStack, ZStack). Ці компоненти взаємодіють через прив’язку даних і керування станом, завдяки чому інтерфейс може реагувати на зміни даних і автоматично оновлюватися.

## Як Swift і SwiftUI співпрацюють?

Обов’язки Swift і SwiftUI можна коротко підсумувати так:

**1. Swift: обробка логіки та даних**

Swift головним чином використовується для керування даними, збереження стану та виконання логіки. Такий код зазвичай не впливає безпосередньо на рендеринг інтерфейсу, а відповідає за обробку даних і поведінки.

Наприклад, обробка даних у Swift:

```swift
let markdown = try? String(contentsOf: url)
````

Цей код включає оголошення змінних, констант, функцій, керування потоком виконання тощо й не впливає безпосередньо на відображення подання.

**2. SwiftUI: оголошення та рендеринг інтерфейсу**

SwiftUI використовується для побудови структури та вмісту інтерфейсу користувача, головним чином відповідає за рендеринг екрана та взаємодію з поданнями.

Наприклад, створення текстового подання за допомогою SwiftUI:

```swift
Text("SwiftSlim")
```

Усе це — код для побудови та керування елементами інтерфейсу, який безпосередньо взаємодіє з UI.

### Базовий приклад

Нижче наведено простий приклад SwiftUI:

```swift
// ContentView.swift

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

У цьому прикладі:

**Код SwiftUI**: VStack, Image, Text, padding — усе це належить до SwiftUI й відповідає за відображення та компонування інтерфейсу.

**Код Swift**: коментар `//` у верхній частині файлу та `import SwiftUI` належать до базової структури Swift-коду; `#Preview` унизу — це макрос для попереднього перегляду в Xcode, який не бере участі в реальному рендерингу інтерфейсу.

### Розширений приклад

У реальних проєктах Swift і SwiftUI часто використовуються разом, щоб кожен виконував ті завдання, у яких він найсильніший:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    func printName() {
        print("name: \(name)")
    }

    var body: some View {
        VStack {
            Text(name)
            Button("Print") {
                printName()
            }
        }
        .padding()
    }
}
```

У цьому прикладі:

**Код SwiftUI**: `@State` — це спеціальний property wrapper SwiftUI, який використовується для оголошення змінного стану подання; `Button` і `Text` — це компоненти подань SwiftUI, що відповідають за відображення інтерфейсу та взаємодію з користувачем.

**Код Swift**: `private var name` і `func printName()` — це Swift-код, який відповідає за зберігання даних і виконання логіки та не впливає безпосередньо на рендеринг подання.

Коли користувач натискає кнопку, SwiftUI запускає код дії, зазначений у кнопці:

```swift
Button("Print") {
    printName()
}
```

Тут `Button` — це компонент SwiftUI, а виклик функції `printName()` — це вже Swift-код, який відповідає за конкретну логіку.

Завдяки такій взаємодії Swift і SwiftUI можна безшовно поєднувати: Swift відповідає за обробку даних і логіки, а SwiftUI — за відображення користувацького інтерфейсу.

## Де зазвичай пишеться код Swift і SwiftUI?

У SwiftUI інтерфейс будується з подання, яке повертається через властивість `body`. Тому весь код, який описує інтерфейс, зазвичай пишеться саме в `body`.

Наприклад:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

У цьому прикладі `Text` є компонентом подання SwiftUI, тому він має бути розміщений у `body`, оскільки саме через `body` SwiftUI зчитує й генерує інтерфейс.

Код, який не пов’язаний безпосередньо з інтерфейсом, наприклад змінні, функції або логіка обробки даних, зазвичай пишеться поза `body`. Наприклад:

```swift
struct ContentView: View {

    // Swift: дані або логіка
    var name = "Fang"

    func printName() {
        print(name)
    }

    // SwiftUI: інтерфейс
    var body: some View {
        Text(name)
    }
}
```

Важливо звернути увагу, що в `body` усе одно можна використовувати синтаксис Swift, наприклад `if`, `for` та інші конструкції керування потоком, але вони лише керують тим, як генеруються подання SwiftUI:

```swift
if isLogin {
    Text("Welcome")
}
```

Тому у SwiftUI-розробці це можна спрощено розуміти так: код подань (Text, Image, Button тощо) зазвичай пишеться в `body`; код даних і логіки (змінні, функції тощо) зазвичай пишеться поза `body`.

## Swift-файли

У міру поглиблення навчання ми також познайомимося з архітектурою MVVM, де шари ViewModel і Model зазвичай складаються з чистого Swift-коду та повністю відокремлені від шару подань (SwiftUI).

Наприклад, клас для керування станом застосунку:

```swift
class AppStorage: ObservableObject {
    static var shared = AppStorage()
    private var isLoading = false
}
```

Або структура, яка описує модель даних:

```swift
struct SponsorStruct: Identifiable {
    var id:String
    var icon: String
}
```

Завдання таких файлів полягає в керуванні та збереженні даних, вони не відображаються безпосередньо в інтерфейсі, тому повністю належать до Swift.

## Історичний контекст

Щоб зрозуміти зв’язок між Swift і SwiftUI, також варто знати, як еволюціонували технології розробки Apple. Станом на 2026 рік цей стек технологій уже пройшов через кілька етапів розвитку.

### Історія Swift

До появи Swift основною мовою розробки для платформ Apple була Objective-C, яка також підтримувала змішане використання з мовою C. Її синтаксис був доволі багатослівним, а поріг входу для новачків — вищим:

```Obj-c
// Запис в Objective-C
NSString *name = @"Fang";
NSLog(@"Hello，%@", name);
```

У 2014 році Apple представила мову Swift на WWDC. Swift має сучасніший синтаксис, вищу безпеку типів і поступово замінила Objective-C як основну мову розробки:

```swift
// Запис у Swift
let name = "Fang"
print("Hello, \(name)")
```

Однак Objective-C не зникла повністю. Вона й надалі широко використовується в багатьох існуючих проєктах і в низькорівневих фреймворках. Базове розуміння її синтаксису досі корисне для підтримки старих проєктів і розуміння внутрішніх механізмів системи.

### Історія SwiftUI

До появи SwiftUI на iOS використовувався **UIKit**, а на macOS — **AppKit**. Обидва ці фреймворки працювали за "імперативною" моделлю програмування. Розробнику потрібно було або перетягувати компоненти в Storyboard, або вручну писати код для керування станом подань. Це призводило до великого обсягу коду та високої вартості підтримки, особливо коли логіка інтерфейсу ставала складною.

![storyboard](../../RESOURCE/010_xcode.png)

У 2019 році Apple офіційно представила SwiftUI на WWDC. SwiftUI приніс "декларативну" парадигму програмування, яка значно спростила процес розробки інтерфейсів.

![storyboard](../../RESOURCE/010_xcode1.png)

Варто зазначити, що SwiftUI не є повністю незалежною низькорівневою реалізацією. На різних платформах він по суті працює через міст і взаємодію з UIKit (на iOS) або AppKit (на macOS), а на нижньому рівні все ще спирається на ці два фреймворки.

### Зв’язок Swift з UIKit/AppKit

Хоча Swift є універсальною мовою програмування, яка може працювати на різних платформах Apple, вона не може повністю замінити UIKit або AppKit. Для деяких складних вимог до інтерфейсу або для функціональності, яку SwiftUI ще не охоплює, усе ще потрібно звертатися до UIKit чи AppKit.

Наприклад, UIKit уже дуже зрілий у питаннях керування складними view controller, анімацій та розпізнавання жестів і має велику кількість перевірених у продакшені практик. Хоча можливості SwiftUI в цих напрямах постійно зростають, у деяких крайових сценаріях він усе ще має обмеження.

Тому багато розробників поєднують у своїх проєктах SwiftUI з UIKit (або AppKit), щоб максимально використати сильні сторони кожного інструмента.

З цього погляду SwiftUI можна розглядати як високорівневу обгортку над UIKit / AppKit. Паралельно з вивченням SwiftUI базове розуміння UIKit і AppKit допомагає приймати обґрунтованіші технічні рішення під час підтримки старих проєктів або реалізації складних функцій.

## Підсумок

**Swift**: переважно використовується для написання логіки, обробки даних, керування потоком виконання тощо й не пов’язаний безпосередньо з компонуванням інтерфейсу.

**SwiftUI**: використовується для декларативної побудови користувацького інтерфейсу; код подань і компонування належить до SwiftUI.

У реальній розробці Swift і SwiftUI зазвичай використовуються разом: Swift відповідає за логіку, а SwiftUI — за інтерфейс.

Від Objective-C і UIKit до Swift і SwiftUI середовище розробки Apple поступово перейшло до сучаснішого й лаконічнішого стилю програмування, але UIKit і AppKit у багатьох традиційних проєктах і досі залишаються важливими.

Розуміння взаємозв’язку між Swift і SwiftUI допомагає нам ефективніше розробляти застосунки для iOS/macOS, а також приймати раціональніші технічні рішення під час підтримки старих проєктів.
