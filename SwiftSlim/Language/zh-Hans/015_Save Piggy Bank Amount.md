# 保存存钱罐金额

在上一节课中，我们实现了一个简单的“存钱罐”。这节课，主要学习如何将存钱罐的金额持久化保存。

我们将学习 UserDefaults 以及 @AppStorage 属性包装器，还会延伸的讲一下 onAppear 和变量为 nil 的用法。

## 持久化保存问题

现在这个“存钱罐”应用，界面非常简单，使用也非常方便。但是，存在一个问题。

那就是，我们每次刷新视图或者关闭应用时，“存钱罐”总金额就会变成 0，无法持久化保存。

这样，“存钱罐”只能保存临时的金钱，无法满足实际的使用需求。

**为什么存钱罐金额不能保存呢？**

这是因为 @State 封装的变量，只能保存在当前视图。它的生命周期和视图绑定的，当我们在 Xcode 中刷新预览视图或者关闭应用时，视图就会被销毁，变量也会笑会。

我们可以简单的理解为，我们存钱罐的金额是临时保存的，只在视图运行时临时保存，关闭视图就会清空。

因此，每次重新打开视图时，变量都会重新初始化，之前存入的金额都会丢失。

如果我们希望存钱罐的金额可以一直保存，而不是与视图的生命周期捆绑在一起，就需要将数据持久化保存。或者说，保存到设备上。

我们可以使用 UserDefaults 保存简单的数据，复杂的数据需要使用 SwiftData 保存。当然，这也是我们以后学习的知识。

## UserDefaults 保存

UserDefaults 保存存钱罐总金额。

最简单的一个用法就是使用 @AppStorage 封装 amount 变量：

```swift
@AppStorage("amount") private var amount = 0
```

使用 @AppStorage 替代 @State 后，amount 就可以实现持久化保存。
