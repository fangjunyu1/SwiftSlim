# दिसण्याचे अनुकूलन

या धड्यात आपण हलका मोड आणि गडद मोड यांचे दिसण्याचे अनुकूलन शिकू.

iPhone, iPad आणि Mac वर वापरकर्ता हलका मोड निवडू शकतो, तसेच गडद मोडही निवडू शकतो.

सिस्टम गडद मोडमध्ये बदलली की, अनेक App चे पार्श्वभूमी, मजकूर आणि आयकॉनचे रंगही त्यानुसार बदलतात.

उदाहरणार्थ, दिवसा इंटरफेस हलक्या पार्श्वभूमीत असू शकतो, तर रात्री तो गडद पार्श्वभूमीत बदलू शकतो.

![view](../../../Resource/025_view9.png)

सिस्टमच्या दिसण्याप्रमाणे इंटरफेसचे प्रदर्शन आपोआप समायोजित करण्याची ही क्षमता म्हणजे दिसण्याचे अनुकूलन.

## Xcode मध्ये दिसण्याचे अनुकूलन पूर्वावलोकन करणे

दिसण्याचे अनुकूलन शिकण्यापूर्वी, आपण आधी Xcode मध्ये हलका मोड आणि गडद मोड कसे पूर्वावलोकन करायचे ते पाहू.

`Canvas` क्षेत्राच्या तळाशी `Device Settings` बटणावर क्लिक केल्यास `Canvas Device Settings` पॉपअप उघडता येतो.

![canvas](../../../Resource/025_view.png)

पॉपअपमध्ये `Color Scheme` सेटिंग दिसेल.

![canvas1](../../../Resource/025_view1.png)

त्यापैकी:

- `Light Appearance` म्हणजे हलका मोड.
- `Dark Appearance` म्हणजे गडद मोड.

येथून आपण पूर्वावलोकनाचे दिसणे बदलू शकतो आणि सध्याचा इंटरफेस हलका मोड व गडद मोडमध्ये कसा दिसतो ते पाहू शकतो.

## दोन प्रकारचे दिसणे एकत्र दाखवणे

हलका मोड आणि गडद मोड एकाच वेळी पाहायचे असल्यास `Variants` सुविधा वापरू शकता.

`Canvas` क्षेत्राच्या तळाशी `Variants` बटणावर क्लिक करा आणि `Color Scheme Variants` निवडा.

![canvas2](../../../Resource/025_view2.png)

निवडल्यानंतर `Canvas` हलका मोड आणि गडद मोडचे पूर्वावलोकन परिणाम एकाच वेळी दाखवेल.

![canvas3](../../../Resource/025_view3.png)

यामुळे दोन प्रकारच्या दिसण्यात इंटरफेसमध्ये असलेला फरक तुलना करणे अधिक सोपे होते.

सोप्या भाषेत:

फक्त तात्पुरते हलका/गडद मोड बदलायचा असल्यास `Canvas Device Settings` वापरू शकता.

दोन्ही प्रकारचे दिसणे एकाच वेळी पाहायचे असल्यास `Color Scheme Variants` वापरू शकता.

## दृश्ये डीफॉल्टने दिसण्याशी जुळवून घेतात

SwiftUI मध्ये अनेक सिस्टम दृश्ये डीफॉल्टने हलका मोड आणि गडद मोडशी आपोआप जुळवून घेतात.

उदाहरणार्थ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("प्रकाश")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("काही काही काही")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

दिसण्याचा परिणाम:

![canvas](../../../Resource/025_view4.png)

आपण पाहू शकता की, हलक्या मोडमध्ये आयकॉन आणि शीर्षक साधारणपणे गडद रंगात दिसतात.

गडद मोडमध्ये आयकॉन आणि शीर्षक आपोआप हलक्या रंगात बदलतात.

याचे कारण म्हणजे `Text`, `Image` यांसारखी SwiftUI दृश्ये डीफॉल्टने सिस्टमच्या दिसण्याप्रमाणे रंग समायोजित करतात.

म्हणजेच, आपण हाताने निश्चित रंग दिला नाही, तर SwiftUI आपल्यासाठी दिसण्याच्या अनुकूलनाचा काही भाग हाताळते.

## निश्चित रंग आपोआप उलटत नाहीत

लक्षात ठेवण्यासारखे म्हणजे, आपण हाताने एखादा निश्चित रंग दिला तर तो हलका/गडद मोडनुसार आपोआप बदलत नाही.

उदाहरणार्थ:

```swift
Text("काही काही काही")
	.foregroundStyle(Color.gray)
```

येथील मजकूर `Color.gray` म्हणून ठरवला आहे, म्हणून तो नेहमी राखाडीच दिसेल.

आणखी एक उदाहरण:

```swift
Text("शीर्षक")
    .foregroundStyle(Color.white)
```

ही कोड हलका मोड असो किंवा गडद मोड, नेहमी पांढरा मजकूर दाखवेल.

जर पार्श्वभूमीही हलकी असेल, तर पांढरा मजकूर स्पष्ट दिसणार नाही.

म्हणून दिसण्याचे अनुकूलन करताना `Color.white`, `Color.black` अशा निश्चित रंगांना मनाने लिहिणे शक्यतो टाळावे.

बर्‍याच वेळा सिस्टम सेमॅंटिक शैली प्रथम वापरू शकतो, उदाहरणार्थ:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

त्यापैकी:

- `.primary` साधारणपणे मुख्य सामग्रीसाठी वापरले जाते.
- `.secondary` साधारणपणे दुय्यम सामग्रीसाठी वापरले जाते.

हे हलका/गडद मोडनुसार प्रदर्शनाचा परिणाम आपोआप समायोजित करतात.

## दिसण्याप्रमाणे वेगळी सामग्री दाखवणे

कधी कधी आपल्याला फक्त रंग बदलावा असे नसते, तर वेगवेगळ्या दिसण्यात वेगळी सामग्री दाखवायची असते.

उदाहरणार्थ:

- हलक्या मोडमध्ये सामान्य बल्ब आयकॉन दाखवणे.
- गडद मोडमध्ये चमकणारा बल्ब आयकॉन दाखवणे.
- हलक्या मोडमध्ये शीर्षक `Light` दाखवणे.
- गडद मोडमध्ये शीर्षक `Dark` दाखवणे.

अशा वेळी सध्याची सिस्टम हलक्या मोडमध्ये आहे की गडद मोडमध्ये आहे हे तपासावे लागते.

SwiftUI मध्ये `colorScheme` या पर्यावरण मूल्यातून सध्याचा दिसण्याचा मोड मिळवता येतो:

```swift
@Environment(\.colorScheme) private var colorScheme
```

पूर्ण कोड:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "प्रकाश" : "गडद"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("काही काही काही")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

दिसण्याचा परिणाम:

![view](../../../Resource/025_view5.png)

पूर्वावलोकनातून दिसते की, एकच दृश्य वेगवेगळ्या दिसण्याच्या मोडमध्ये पूर्णपणे सारखे दिसत नाही.

हलक्या मोडमध्ये `Image` सामान्य बल्ब आयकॉन दाखवते, आणि शीर्षक `Light` म्हणून दिसते.

गडद मोडमध्ये `Image` चमकणारा बल्ब आयकॉन दाखवते, आणि शीर्षक `Dark` म्हणून दिसते.

म्हणजेच, येथे बदल फक्त रंगात नाही, तर आयकॉनचे नाव आणि शीर्षकाचा मजकूर स्वतःही बदलतो.

याचे कारण म्हणजे आपण आयकॉन आणि शीर्षक थेट `body` मध्ये लिहून ठेवले नाहीत, तर `colorScheme` नुसार वेगळी सामग्री मोजून काढली.

या कोडमध्ये आपण दोन computed properties वापरल्या आहेत:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "प्रकाश" : "गडद"
}
```

जेव्हा `colorScheme` `.light` असते, तेव्हा सध्याचा मोड हलका आहे.

जेव्हा `colorScheme` `.dark` असते, तेव्हा सध्याचा मोड गडद आहे.

म्हणून आपण वेगवेगळ्या दिसण्याच्या मोडनुसार वेगवेगळी आयकॉन नावे आणि शीर्षक मजकूर परत करू शकतो.

ही computed property च्या सामान्य वापरांपैकी एक आहे: सध्याच्या स्थितीनुसार दृश्याला दाखवायची सामग्री मोजून काढणे.

## @Environment समजून घेणे

आपण `@Environment` ला पहिल्यांदा येथे भेटत आहोत.

`@Environment` असे समजू शकतो: SwiftUI च्या environment मधून एखादे मूल्य वाचणे.

App चालू असताना सिस्टम अनेक पर्यावरण माहिती पुरवते, उदाहरणार्थ:

- सध्याची भाषा
- सध्याचा दिसण्याचा मोड
- सध्याची layout direction
- सध्याची font size सेटिंग

सध्याचा दिसण्याचा मोड मिळवायचा असल्यास आपण `colorScheme` वाचू शकतो:

```swift
@Environment(\.colorScheme) private var colorScheme
```

त्यात

```swift
\.colorScheme
```

म्हणजे environment मधून `colorScheme` हे मूल्य वाचणे.

```swift
private var colorScheme
```

म्हणजे वाचलेले मूल्य `colorScheme` या variable मध्ये साठवणे.

variable चे नाव आपण स्वतः ठरवू शकतो, उदाहरणार्थ असेही लिहू शकतो:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

जोपर्यंत पुढील `\.colorScheme` बदलत नाही, तोपर्यंत वाचले जाणारे मूल्य सध्याचा दिसण्याचा मोडच असतो.

## colorScheme ची दोन सामान्य मूल्ये

`colorScheme` ची सामान्यपणे दोन मूल्ये असतात:

```swift
.light
.dark
```

त्यांचा अर्थ:

- `.light`: हलका मोड
- `.dark`: गडद मोड

उदाहरणार्थ:

```swift
private var titleName: String {
    colorScheme == .light ? "प्रकाश" : "गडद"
}
```

या कोडचा अर्थ असा आहे:

जर सध्याचा मोड हलका असेल, तर `colorScheme == .light` खरे ठरेल आणि `"प्रकाश"` परत केले जाईल.

नाहीतर `"गडद"` परत केले जाईल.

म्हणून हलक्या मोडमध्ये:

```swift
Text(titleName)
```

दाखवले जाणारे म्हणजे:

```swift
प्रकाश
```

गडद मोडमध्ये दाखवले जाणारे म्हणजे:

```swift
गडद
```

या पद्धतीने आपण वेगवेगळ्या दिसण्याप्रमाणे वेगळी सामग्री दाखवू शकतो.

## रंग अनुकूलन उदाहरण

मजकूर आणि आयकॉन व्यतिरिक्त, कधी कधी आपल्याला रंगांचे अनुकूलन स्वतः हाताळावे लागते.

उदाहरणार्थ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("मजकूर")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

दिसण्याचा परिणाम:

![view](../../../Resource/025_view6.png)

या कोडमध्ये मजकूर पांढरा आहे आणि पार्श्वभूमी काळी आहे.

हलक्या मोडमध्ये काळी पार्श्वभूमी स्पष्ट दिसते.

परंतु गडद मोडमध्ये, जर संपूर्ण इंटरफेसची पार्श्वभूमीही काळी असेल, तर ही काळी पार्श्वभूमी सिस्टम पार्श्वभूमीत मिसळते आणि पुरेशी स्पष्ट दिसत नाही.

अशा वेळी आपण दिसण्याच्या मोडनुसार मजकूराचा रंग आणि पार्श्वभूमीचा रंग बदलू शकतो:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("मजकूर")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

दिसण्याचा परिणाम:

![view](../../../Resource/025_view7.png)

आता हलक्या मोडमध्ये काळी पार्श्वभूमी आणि पांढरा मजकूर आहे.

गडद मोडमध्ये पांढरी पार्श्वभूमी आणि काळा मजकूर आहे.

अशा प्रकारे कोणत्याही दिसण्यात मजकूर आणि पार्श्वभूमीमध्ये स्पष्ट contrast राहतो, आणि वापरकर्ता सामग्री स्पष्टपणे पाहू शकतो.

## Assets वापरून रंग अनुकूल करणे

`colorScheme` वापरून हाताने तपासण्याव्यतिरिक्त, आपण `Assets` मधील resource color वापरूनही हलका/गडद मोडशी जुळवून घेऊ शकतो.

ही पद्धत App मधील सर्वसाधारण रंग व्यवस्थापित करण्यासाठी अधिक योग्य आहे.

Xcode प्रकल्पात `Assets` resource folder शोधा.

![assets](../../../Resource/025_color.png)

`Assets` folder उघडा, रिकाम्या भागावर उजवे क्लिक करा, `New Color Set` निवडा आणि नवीन color resource तयार करा.

![assets](../../../Resource/025_color1.png)

येथे आपण एक रंग तयार करतो आणि त्याचे नाव `redText` ठेवतो.

![assets](../../../Resource/025_color2.png)

Color resource मध्ये हलका मोड आणि गडद मोडसाठी वेगवेगळे रंग सेट करता येतात.

![assets](../../../Resource/025_color3.png)

संबंधित रंग क्षेत्र निवडल्यानंतर, उजव्या बाजूच्या attributes inspector मध्ये रंग बदलू शकता.

वरच्या उजव्या कोपर्‍यातील `Hide or show the Inspectors` बटणावर क्लिक करून inspector क्षेत्र उघडा.

मग `Show the Attributes inspector` निवडा, तळाशी `Show Color Panel` शोधा आणि color panel उघडा.

![assets](../../../Resource/025_color4.png)

color panel मधील रंगावर क्लिक केल्यावर `redText` शी संबंधित रंग क्षेत्रही एकाच वेळी बदलते.

येथे आपण `redText` असे सेट करतो:

- हलक्या मोडमध्ये लाल रंग दाखवणे
- गडद मोडमध्ये हिरवा रंग दाखवणे

![assets](../../../Resource/025_color5.png)

SwiftUI कोडमध्ये परत आल्यावर हा रंग असा वापरू शकतो:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

त्यापैकी:

```swift
Color("redText")
```

म्हणजे `Assets` मधून `redText` नावाचा color resource वाचणे.

दिसण्याचा परिणाम:

![assets](../../../Resource/025_view8.png)

आपण पाहू शकता की, तोच `Color("redText")` हलका मोड आणि गडद मोडमध्ये वेगवेगळे रंग दाखवतो.

यावरून `Assets` रंग स्वतःही दिसण्याचे अनुकूलन समर्थित करतात हे कळते.

## colorScheme आणि Assets मधील फरक

`colorScheme` आणि `Assets` दोन्ही दिसण्याचे अनुकूलन करू शकतात, पण त्यांना योग्य असलेल्या परिस्थिती वेगळ्या आहेत.

फक्त हलका/गडद मोडनुसार मजकूर सामग्री, image name किंवा `SF Symbols` नाव बदलायचे असल्यास `colorScheme` वापरू शकता.

उदाहरणार्थ:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

जर App मध्ये वारंवार वापरले जाणारे रंग असतील, उदाहरणार्थ theme color, card background color, text color, तर `Assets` रंग वापरणे अधिक शिफारसीय आहे.

कारण `Assets` रंग अनेक दृश्यांमध्ये पुन्हा वापरता येतात, आणि नंतर बदल करणेही अधिक सोपे असते.

उदाहरणार्थ:

```swift
Color("redText")
```

असे केल्यास फक्त `Assets` मध्ये रंग बदलावा लागतो, आणि हा रंग वापरणाऱ्या सर्व ठिकाणी बदल एकत्र लागू होतो.

## सारांश

या धड्यात आपण हलका मोड आणि गडद मोड यांचे दिसण्याचे अनुकूलन शिकलो.

सर्वप्रथम, आपण Xcode च्या `Canvas` मध्ये वेगवेगळे दिसणे कसे पूर्वावलोकन करायचे ते शिकलो, तसेच `Color Scheme Variants` वापरून हलका मोड आणि गडद मोड एकाच वेळी कसे पाहायचे तेही शिकलो.

यानंतर, SwiftUI मधील `Text`, `Image` यांसारखी दृश्ये डीफॉल्टने सिस्टमच्या दिसण्याप्रमाणे रंग आपोआप जुळवून घेतात हे आपण पाहिले.

परंतु, आपण हाताने `Color.white` किंवा `Color.black` सारखे निश्चित रंग दिल्यास, हे रंग आपोआप उलटत नाहीत.

यानंतर, आपण `@Environment` आणि `colorScheme` शिकलो:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` द्वारे आपण सध्याचा मोड हलका आहे की गडद आहे हे तपासू शकतो, आणि वेगवेगळ्या दिसण्याप्रमाणे वेगवेगळे आयकॉन, मजकूर किंवा रंग दाखवू शकतो.

शेवटी, आपण `Assets` रंग शिकलो.

`Assets` रंगांमध्ये हलका मोड आणि गडद मोडसाठी वेगवेगळे रंग सेट करता येतात, त्यामुळे ते App मधील सर्वसाधारण color resources व्यवस्थापित करण्यासाठी योग्य असतात.

दिसण्याचे अनुकूलन App ला दिवसा आणि रात्री दोन्ही वेळा अधिक चांगला प्रदर्शन परिणाम देते, तसेच मजकूर अस्पष्ट दिसणे किंवा पार्श्वभूमी स्पष्ट न दिसणे यांसारख्या समस्या टाळते.

प्रत्यक्ष विकासात, SwiftUI ची डीफॉल्ट अनुकूलन क्षमता आणि `Assets` रंगांना प्रथम प्राधान्य देण्याची शिफारस केली जाते.

जेव्हा वेगवेगळ्या दिसण्यात मजकूर, आयकॉन, image बदलायचे असतात किंवा काही विशेष display effects हाताळायचे असतात, तेव्हाच `colorScheme` वापरून तपासणी करा.

## विस्तार ज्ञान: font size निर्दिष्ट करणे

आधी `font` शिकताना आपण काही system font styles वापरल्या होत्या:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` अशा system font styles वापरण्याव्यतिरिक्त, font size हातानेही निर्दिष्ट करता येतो.

उदाहरणार्थ:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

येथील:

```swift
.font(.system(size: 100))
```

म्हणजे font size `100 pt` म्हणून सेट करणे.

`SF Symbols` आयकॉनसाठी `font` आयकॉनच्या आकारावरही परिणाम करते.

म्हणून आयकॉन मोठा दाखवायचा असल्यास `.font(.system(size:))` वापरून आकार निर्दिष्ट करू शकता.

उदाहरणार्थ:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

लक्षात ठेवण्यासारखे म्हणजे, font size हाताने निर्दिष्ट करणे अधिक लवचिक असले तरी, सामान्य मजकुरात निश्चित font size मोठ्या प्रमाणात वापरण्याची शिफारस नाही.

सामान्य मजकुरासाठी `.title`, `.headline`, `.body`, `.caption` अशा system font styles प्रथम वापराव्यात.

यामुळे वेगवेगळ्या device आणि वापरकर्त्यांच्या font settings शी अधिक चांगले जुळवून घेता येते.
