# लूप आणि यादी

या धड्यात, आपण Swift मधील `for` लूप आणि SwiftUI मधील `ForEach` शिकणार आहोत.

ही दोन्ही साधने अॅरेमधील घटक क्रमाने वाचण्यास मदत करतात. प्रत्यक्ष विकासात, आपल्याला अनेकदा एका गटातील डेटा एक-एक करून काढून त्यावर प्रक्रिया करावी लागते किंवा तो दाखवावा लागतो, आणि लूप हे हे काम पूर्ण करण्यासाठीचे एक महत्त्वाचे साधन आहे.

जेव्हा आपण वेबपेज ब्राउझ करतो किंवा App वापरतो, तेव्हा आपल्याला अनेक यादी किंवा नोंदी दिसतात. अशी सामग्री साधारणपणे आधी `अॅरे` मध्ये साठवली जाते, आणि नंतर `लूप` च्या मदतीने एक-एक करून काढली जाते, आणि शेवटी इंटरफेसमध्ये दाखवली जाते.

उदाहरणार्थ, Google च्या शोध परिणामांची यादी:

![Google](../../Resource/017_google.png)

शोध परिणामांच्या यादीतील प्रत्येक नोंद साधारणपणे एका डेटाशी संबंधित असते, आणि ती लूपद्वारे क्रमाने दाखवली जाते.

म्हणूनच, `अॅरे` आणि `लूप` हे अनेकदा एकत्र दिसतात. `अॅरे` एका गटातील डेटा साठवण्याचे काम करते, आणि `लूप` हा डेटा क्रमाने काढून त्याच प्रकारची कृती करण्याचे काम करतो.

आता, आपण सर्वात मूलभूत `for` लूपपासून सुरुवात करूया.

## काउंटडाउन

समजा, आपल्याला एक काउंटडाउन फीचर तयार करायचे आहे, आणि क्रमाने `“5、4、3、2、1”` आउटपुट करायचे आहे.

सर्वात थेट लिहिण्याची पद्धत अशी आहे:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

ही पद्धत काउंटडाउनची नक्कल करू शकते, पण जर सामग्री जास्त असेल, तर ती खूपच पुनरावृत्तीची वाटते, आणि नंतर बदल करणेही सोयीचे राहत नाही.

अशावेळी, आपण आधी डेटा अॅरेमध्ये ठेवू शकतो, आणि मग `for` लूप वापरून तो क्रमाने प्रिंट करू शकतो:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ही पद्धत अधिक संक्षिप्त आहे, आणि विस्तार करणेही सोपे आहे. काउंटडाउनमध्ये अनेक संख्या असल्या, तरी आपल्याला `print` पुन्हा पुन्हा लिहिण्याची गरज नसते; फक्त लूपला अॅरेमधील प्रत्येक घटक आपोआप हाताळू द्यायचा असतो.

हा कोड आपण बटणामध्ये ठेवून तपासू शकतो:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

बटणावर क्लिक केल्यावर, `for` लूप अॅरेच्या क्रमाने `countDown` मधील प्रत्येक पूर्णांक एक-एक करून काढेल, आणि तो कन्सोलमध्ये प्रिंट करेल.

आता, `for` लूप नेमका कसा काम करतो ते पाहूया.

## for लूप

`for` लूप एखाद्या गटातील डेटा क्रमाने हाताळण्यासाठी वापरला जातो. सर्वात सामान्य उदाहरणे म्हणजे अॅरे आणि श्रेणी.

मूलभूत स्वरूप:

```swift
for घटक in अॅरे किंवा श्रेणी {
    चालवायचा कोड
}
```

उदाहरणार्थ, आत्ताचा काउंटडाउन कोड:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

इथे `i` हा सध्या काढलेला घटक दर्शवतो, आणि `in` नंतर लूप करायचा डेटा येतो, म्हणजे `countDown` अॅरे.

या कोडचा अर्थ असा आहे: प्रोग्राम `countDown` अॅरेमधून प्रत्येक घटक क्रमाने काढेल. प्रत्येक वेळी एक घटक काढल्यावर, तो तात्पुरता `i` या नावाने दर्शवला जाईल, आणि मग कंसांच्या आतला कोड चालवला जाईल.

ही प्रक्रिया आपण अशी समजू शकतो:

* पहिल्यांदा काढलेला घटक `5` आहे, त्यामुळे त्या क्षणी `i` ची किंमत `5` आहे
* `print(i)` चालवले जाते, आणि `5` आउटपुट होते
* मग पुढचा घटक `4` काढला जातो
* पुन्हा `print(i)` चालवले जाते, आणि `4` आउटपुट होते

यानंतरचे `3`、`2`、`1` हेदेखील याच पद्धतीने एक-एक करून चालतात.

![for](../../Resource/017_for.png)

अॅरेमधील सर्व घटक हाताळून झाले की, `for` लूप आपोआप संपतो.

`for` लूपच्या मदतीने आपण अॅरेमधील प्रत्येक घटक मिळवू शकतो, आणि काउंटडाउनसारखी फीचर्स तयार करू शकतो.

**सूचना**: `i` हे for लूपमध्ये खूप सामान्यपणे वापरले जाणारे नाव आहे. ते सध्या काढलेल्या घटकासाठी वापरले जाते, पण ते काही ठरलेले नाव नाही; आपण इतर इंग्रजी नावेही वापरू शकतो.

उदाहरणार्थ, प्रत्येक काढलेल्या घटकासाठी `num` वापरू शकतो:

```swift
for num in countDown {
    print(num)
}
```

कर्ली ब्रेसेसमध्येही सध्याच्या घटकासाठी `num` हाच वापरावा लागेल.

### लूप श्रेणी

`for` लूप फक्त अॅरेवरच नाही, तर श्रेणीवरही चालू शकतो.

उदाहरणार्थ, जर आपल्याला `50` पर्यंतच्या धन पूर्णांकांची बेरीज काढायची असेल, तर `1` ते `50` मधील सर्व पूर्णांक क्रमाने जोडावे लागतील.

सर्वात थेट पद्धत अशी असू शकते:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

या पद्धतीची अडचण अशी की, प्रत्येक संख्या आपल्याला हाताने लिहावी लागते. ते त्रासदायक आहे, आणि प्रत्यक्ष विकासाच्या परिस्थितीत योग्यही नाही.

अशावेळी, श्रेणी ऑपरेटर आणि `for` लूप वापरून हे काम करता येते.

यापूर्वीच्या “ऑपरेटर” धड्यात आपण श्रेणी ऑपरेटर आधीच शिकलो आहोत:

```swift
1...3   // 1, 2, 3
```

इथे `...` हे बंद श्रेणी दर्शवते, म्हणजे सुरुवात आणि शेवट दोन्ही समाविष्ट होतात.

म्हणून येथे `1` ते `50` या पूर्णांक श्रेणीवर `for` लूप वापरता येईल:

```swift
for i in 1...50 {
    
}
```

आपल्याला या संख्यांची बेरीज करायची असल्याने, एकूण बेरीज साठवण्यासाठी एक व्हेरिएबलही लागेल:

```swift
var count = 0
```

मग प्रत्येक लूपमध्ये, सध्याची संख्या `count` मध्ये जोडायची:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

या कोडचा अर्थ असा आहे:

* `count` सध्याची एकूण बेरीज साठवण्यासाठी वापरला जातो
* `for` लूप `1` ते `50` मधील पूर्णांक एक-एक करून काढतो
* प्रत्येक वेळी एक संख्या काढल्यावर `count += i` चालते
* लूप संपल्यानंतर, `count` मध्ये अंतिम बेरीज साठवलेली असते

अशा प्रकारे, आपल्याला `50` पर्यंतच्या सर्व धन पूर्णांकांची बेरीज मिळते.

तसेच, जर आपल्याला `1` ते `100` पर्यंतच्या पूर्णांकांची बेरीज काढायची असेल, तर फक्त श्रेणी बदलली की पुरेसे आहे:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` लूप फक्त अॅरेवरच नाही, तर एखाद्या श्रेणीवरही चालतो. डेटा क्रमाने पुन्हा पुन्हा हाताळण्याची ही क्षमता विकासात खूप सामान्य आहे.

यामुळे आपल्याला अनेक पुनरावृत्तीची कामे आपोआप हाताळता येतात; प्रत्येक संख्या जोडण्याची प्रक्रिया किंवा इतर तशाच प्रकारची पुनरावृत्तीची कामे हाताने लिहावी लागत नाहीत.

## SwiftUI मधील लूप

याआधीचा `for` लूप हा Swift भाषेचा भाग आहे.

पण SwiftUI मध्ये, आपण view च्या रचनेत थेट `for` वापरून views तयार करू शकत नाही:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

कारण `VStack` च्या आत SwiftUI views लागतात, पण सामान्य `for` लूप स्वतः view नसतो.

जर आपल्याला SwiftUI मध्ये अॅरेच्या आधारे इंटरफेसमधील views पुन्हा पुन्हा तयार करायचे असतील, तर `ForEach` वापरावे लागते.

उदाहरणार्थ, जर आपल्याला एखाद्या गटातील सर्व सदस्यांची नावे दाखवायची असतील, तर असे लिहू शकतो:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

दाखवलेला परिणाम:

![for1](../../Resource/017_for1.png)

आता प्रत्येक `Text` ला समान modifiers जोडूया:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

आता प्रत्येक `Text` लहान आणि जाड फॉन्टमध्ये दिसेल.

दाखवलेला परिणाम:

![](../../Resource/017_for2.png)

modifiers मुळे इंटरफेस अधिक सुंदर दिसतो, पण कोडही अधिक लांब आणि पुनरावृत्तीचा होतो.

जर नावे वाढली, किंवा नंतर फॉन्ट, रंग इत्यादी style एकाचवेळी बदलायच्या असतील, तर अनेक सारख्या ओळी पुन्हा पुन्हा लिहाव्या आणि बदलाव्या लागतील. त्यामुळे ते त्रासदायक होईल आणि देखभाल करणेही अवघड होईल.

अशावेळी, आपण आधी ही नावे अॅरेमध्ये ठेवू शकतो, आणि मग SwiftUI मधील `ForEach` वापरून अॅरेमधील डेटाच्या आधारे views पुन्हा पुन्हा तयार करू शकतो:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

हा कोड अधिक संक्षिप्त आहे. `ForEach` अॅरेमधील प्रत्येक घटक क्रमाने वाचतो आणि त्यानुसार संबंधित सामग्री इंटरफेसमध्ये दाखवतो.

नंतर नवीन नावे जोडायची असतील, तर फक्त `names` अॅरेमधील नावे बदलली की झाले; प्रत्येक वेळी `Text` आणि त्याचे modifiers पुन्हा जोडण्याची गरज नसते.

## ForEach

`ForEach` ही SwiftUI मधील views पुन्हा पुन्हा तयार करण्यासाठीची रचना आहे. तिला SwiftUI मधील `for` लूपसारखी रचना म्हणून समजू शकतो.

मूलभूत वापर:

```swift
ForEach(अॅरे, id: \.self) { घटक in
    SwiftUI कोड
}
```

उदाहरणार्थ, आत्ताच्या गटातील नावे:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

या कोडचा अर्थ:

`names` हा लूप होणारा अॅरे आहे, म्हणजे दाखवायचा डेटा.

`id:\.self` म्हणजे अॅरेमधील घटक स्वतःच प्रत्येक नोंद ओळखण्यासाठी वापरला जातो.

सध्याच्या उदाहरणात, अॅरेमध्ये strings आहेत:

```swift
["Sam", "John", "Wathon", "Bob"]
```

म्हणून SwiftUI थेट "Sam"、"John"、"Wathon"、"Bob" या strings वापरून वेगवेगळ्या सामग्रीची ओळख करतो.

सुरुवातीच्या टप्प्यात सामान्यपणे दिसणाऱ्या String, Int सारख्या साध्या अॅरेसाठी, आपण सहसा असे लिहू शकतो:

```swift
id: \.self
```

आत्ता तरी ही एकच पद्धत लक्षात ठेवली तरी पुरेसे आहे; `id` च्या अधिक नियमांमध्ये खोलात जाण्याची गरज नाही.

`item in` मधील `item` सध्या काढलेल्या घटकाला दर्शवतो.

उदाहरणार्थ, पहिल्या लूपमध्ये `item` हा `"Sam"` असेल, आणि दुसऱ्या लूपमध्ये `item` हा `"John"` असेल.

म्हणून:

```swift
Text(item)
```

हे क्रमाने असे होईल:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

म्हणजे, या कोडचे काम असे आहे: `names` अॅरेमधून प्रत्येक नाव एक-एक करून काढणे, आणि प्रत्येक नावासाठी एक `Text` view तयार करणे.

### चालण्याची प्रक्रिया

ही प्रक्रिया अशी समजू शकतो:

* `ForEach` आधी `names` अॅरेमधील पहिला घटक `"Sam"` वाचतो
* `item` तात्पुरता `"Sam"` दर्शवतो
* `Text(item)` `"Sam"` दाखवतो
* मग पुढचा घटक `"John"` वाचतो
* आणि पुन्हा तोच view कोड चालवतो

शेवटी, अॅरेमधील सर्व नावे दाखवली जातात.

अशी पद्धत वापरण्याचा फायदा असा की: नावे वाढली तरी आपल्याला अनेक `Text` पुन्हा पुन्हा लिहावे लागत नाहीत; फक्त अॅरेमधील सामग्री बदलली की पुरेसे होते.

### फोटोवर लूप

`ForEach` फक्त मजकूरच नाही, तर चित्रे, रंग, किंवा इतर SwiftUI views देखील दाखवू शकतो.

उदाहरणार्थ, आपण 4 चित्रे तयार केली आहेत:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

ही चार चित्रे Xcode च्या Assets फोल्डरमध्ये drag करा.

![image](../../Resource/017_for3.png)

जर `ForEach` वापरला नाही, तर आपल्याला प्रत्येक चित्र हाताने लिहावे लागेल:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

दाखवलेला परिणाम:

![image](../../Resource/017_for4.png)

ही पद्धत काम करते, पण कोड तुलनेने मोठा होतो.

चित्रे वाढत गेली की, `Image` कोडही अधिकाधिक वेळा पुन्हा लिहावा लागतो.

नंतर सर्व चित्रांचे आकार एकत्र बदलायचे असले, तरी प्रत्येक एकेक करून बदलावे लागेल.

अशावेळी, चित्रांची नावे अॅरेमध्ये ठेवता येतात, आणि मग `ForEach` वापरून image views पुन्हा पुन्हा तयार करता येतात:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

इथे `images` हा strings चा अॅरे आहे, आणि अॅरेतील प्रत्येक घटक हे चित्राचे नाव आहे.

`ForEach(images, id: \.self)` याचा अर्थ, `images` अॅरेमधील प्रत्येक घटक क्रमाने वाचणे, आणि त्या घटकालाच प्रत्येक नोंद ओळखण्यासाठी वापरणे, नंतर सध्याच्या घटकाच्या आधारे संबंधित view तयार करणे.

उदाहरणार्थ, पहिल्यांदा वाचलेले मूल्य `"430F9BEF"` असेल, तर:

```swift
Image(item)
```

हे यासारखेच आहे:

```swift
Image("430F9BEF")
```

पुढील चित्रेही याच पद्धतीने क्रमाने दाखवली जातील.

![image](../../Resource/017_for4.png)

इथेही आपण अजूनही वापरतो:

```swift
id: \.self
```

कारण आधीसारखेच आहे. `images` अॅरेमधील घटकही साधे string प्रकार आहेत, त्यामुळे सुरुवातीच्या टप्प्यात आपण थेट त्या घटकालाच ओळख म्हणून वापरू शकतो.

हे एका स्थिर सूत्रासारखे सोप्या शब्दांत समजू शकतो:

**जेव्हा `ForEach` `String` किंवा `Int` सारख्या साध्या अॅरेवर चालतो, तेव्हा आपण सहसा `id: \.self` असे लिहितो.**

### नवीन चित्रे जोडणे

नंतर आणखी चित्रे जोडायची असतील, तर फक्त अॅरेमधील सामग्री बदलली की झाले:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

नवीन `Image` कोड पुन्हा लिहिण्याची गरज नाही.

जर चित्रे खूप असतील, तर `ScrollView` सोबत वापरता येईल. नाहीतर सामग्री स्क्रीनच्या बाहेर जाईल, आणि पूर्णपणे preview करता येणार नाही.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### चित्रे बदलणे

जर सर्व चित्रांची style एकसारखी बदलायची असेल, तर `ForEach` च्या आतला कोडच बदलला की झाले.

उदाहरणार्थ, रुंदी `80` करणे:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

असे केल्यावर सर्व चित्रे एकत्र बदलतील; एकेक करून बदलण्याची गरज नाही.

हीच `ForEach` ची एक महत्त्वाची भूमिका आहे:

**जेव्हा अनेक views ची रचना सारखी असते, पण सामग्री वेगळी असते, तेव्हा आपण एकच view कोड लिहू शकतो, आणि अॅरेमधील डेटा काय दाखवायचे ते ठरवतो.**

## सारांश

या धड्यात, आपण `for` लूप आणि `ForEach` शिकलो.

`for` लूप अॅरे किंवा श्रेणीतील घटक क्रमाने हाताळण्यासाठी वापरला जातो; तो अधिक डेटा हाताळण्याकडे झुकतो.

`ForEach` हा SwiftUI मध्ये अॅरेमधील डेटाच्या आधारे views पुन्हा पुन्हा तयार करण्यासाठी वापरला जातो; तो अधिक view display कडे झुकतो.

सुरुवातीच्या टप्प्यात, `ForEach` चे सर्वात सामान्य स्वरूप असे आहे:

```swift
ForEach(अॅरे, id: \.self) { घटक in
    SwiftUI कोड
}
```

इथे `id: \.self` म्हणजे घटकालाच प्रत्येक नोंदीची ओळख म्हणून वापरणे.

जेव्हा अॅरेमधील सामग्री `String`、`Int` सारख्या साध्या प्रकारांची असते, तेव्हा आपण साधारणपणे ही एकच पद्धत आधी वापरू शकतो.

पुढे, जेव्हा आपण वेबपेज किंवा App मध्ये पुनरावृत्तीचा मजकूर, चित्रे किंवा इतर यादीसारखी सामग्री पाहू, तेव्हा हे लक्षात येईल की SwiftUI मध्ये अशा प्रकारची सामग्री बहुधा आधी अॅरेमध्ये ठेवली जाते, आणि नंतर `ForEach` च्या मदतीने क्रमाने दाखवली जाते.
