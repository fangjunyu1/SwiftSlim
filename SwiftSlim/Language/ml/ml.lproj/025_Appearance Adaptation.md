# രൂപഭാവത്തിന് അനുസരിച്ചുള്ള ക്രമീകരണം

ഈ പാഠത്തിൽ, ലൈറ്റ് മോഡിനും ഡാർക്ക് മോഡിനും അനുസരിച്ച് ഇന്റർഫേസ് ക്രമീകരിക്കുന്നതിനെക്കുറിച്ച് പഠിക്കും.

iPhone, iPad, Mac എന്നിവയിൽ, ഉപയോക്താക്കൾക്ക് ലൈറ്റ് മോഡോ ഡാർക്ക് മോഡോ തിരഞ്ഞെടുക്കാം.

സിസ്റ്റം ഡാർക്ക് മോഡിലേക്ക് മാറുമ്പോൾ, പല App-കളുടെയും പശ്ചാത്തലം, ടെക്സ്റ്റ്, ഐക്കൺ നിറങ്ങൾ അതനുസരിച്ച് മാറും.

ഉദാഹരണത്തിന്, പകൽ ഇന്റർഫേസിൽ ഇളം നിറത്തിലുള്ള പശ്ചാത്തലം ആയിരിക്കാം, രാത്രിയിൽ അത് ഇരുണ്ട പശ്ചാത്തലമായി മാറാം.

![view](../../../Resource/025_view9.png)

സിസ്റ്റത്തിന്റെ രൂപഭാവത്തിന് അനുസരിച്ച് ഇന്റർഫേസ് പ്രദർശനം സ്വയമായി ക്രമീകരിക്കുന്ന ഈ കഴിവിനെയാണ് രൂപഭാവത്തിന് അനുസരിച്ചുള്ള ക്രമീകരണം എന്ന് പറയുന്നത്.

## Xcode-ൽ രൂപഭാവ ക്രമീകരണം പ്രിവ്യൂ ചെയ്യുക

രൂപഭാവ ക്രമീകരണം പഠിക്കുന്നതിന് മുമ്പ്, Xcode-ൽ ലൈറ്റ് മോഡും ഡാർക്ക് മോഡും എങ്ങനെ പ്രിവ്യൂ ചെയ്യാമെന്ന് ആദ്യം നോക്കാം.

`Canvas` മേഖലയിലെ താഴെയുള്ള ഭാഗത്ത്, `Device Settings` ബട്ടൺ ക്ലിക്ക് ചെയ്താൽ `Canvas Device Settings` പോപ്പ്-അപ്പ് തുറക്കാം.

![canvas](../../../Resource/025_view.png)

പോപ്പ്-അപ്പിൽ `Color Scheme` ക്രമീകരണം കാണാം.

![canvas1](../../../Resource/025_view1.png)

ഇതിന്റെ അർത്ഥം:

- `Light Appearance` ലൈറ്റ് മോഡിനെയാണ് സൂചിപ്പിക്കുന്നത്.
- `Dark Appearance` ഡാർക്ക് മോഡിനെയാണ് സൂചിപ്പിക്കുന്നത്.

ഇവിടെ പ്രിവ്യൂ രൂപഭാവം മാറ്റി, നിലവിലെ ഇന്റർഫേസ് ലൈറ്റ് മോഡിലും ഡാർക്ക് മോഡിലും എങ്ങനെ കാണുന്നുവെന്ന് പരിശോധിക്കാം.

## രണ്ട് രൂപഭാവങ്ങൾ ചേർത്ത് കാണിക്കുക

ലൈറ്റ് മോഡും ഡാർക്ക് മോഡും ഒരേസമയം കാണണമെങ്കിൽ, `Variants` ഫീച്ചർ ഉപയോഗിക്കാം.

`Canvas` മേഖലയിലെ താഴെയുള്ള ഭാഗത്ത്, `Variants` ബട്ടൺ ക്ലിക്ക് ചെയ്ത് `Color Scheme Variants` തിരഞ്ഞെടുക്കുക.

![canvas2](../../../Resource/025_view2.png)

തിരഞ്ഞെടുത്തതിന് ശേഷം, `Canvas` ലൈറ്റ് മോഡിന്റെയും ഡാർക്ക് മോഡിന്റെയും പ്രിവ്യൂകൾ ഒരേസമയം കാണിക്കും.

![canvas3](../../../Resource/025_view3.png)

ഇത് രണ്ട് രൂപഭാവങ്ങളിലുമുള്ള ഇന്റർഫേസ് വ്യത്യാസങ്ങൾ താരതമ്യം ചെയ്യുന്നത് കൂടുതൽ സൗകര്യപ്രദമാക്കുന്നു.

ലളിതമായി പറഞ്ഞാൽ:

ലൈറ്റ്/ഡാർക്ക് മോഡുകൾ താൽക്കാലികമായി മാത്രം മാറണമെങ്കിൽ, `Canvas Device Settings` ഉപയോഗിക്കാം.

രണ്ട് രൂപഭാവങ്ങളും ഒരേസമയം കാണണമെങ്കിൽ, `Color Scheme Variants` ഉപയോഗിക്കാം.

## View-കൾ സാധാരണയായി രൂപഭാവത്തിന് അനുസരിക്കും

SwiftUI-യിൽ, പല സിസ്റ്റം view-ുകളും സാധാരണയായി ലൈറ്റ് മോഡിനും ഡാർക്ക് മോഡിനും സ്വയമായി പൊരുത്തപ്പെടും.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("വെളിച്ചം")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("ഉദാഹരണ വാചകം")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

പ്രദർശന ഫലം:

![canvas](../../../Resource/025_view4.png)

കാണാൻ കഴിയുന്നതുപോലെ, ലൈറ്റ് മോഡിൽ ഐക്കണുകളും ശീർഷകങ്ങളും സാധാരണയായി ഇരുണ്ട നിറത്തിൽ കാണിക്കും.

ഡാർക്ക് മോഡിൽ, ഐക്കണുകളും ശീർഷകങ്ങളും സ്വയമായി ഇളം നിറത്തിലേക്ക് മാറും.

ഇതിന് കാരണം `Text`, `Image` പോലുള്ള SwiftUI view-ുകൾ സാധാരണയായി സിസ്റ്റം രൂപഭാവത്തിന് അനുസരിച്ച് നിറങ്ങൾ ക്രമീകരിക്കുന്നതാണ്.

അഥവാ, നാം ഒരു സ്ഥിരമായ നിറം കൈമാറി നിർദ്ദിഷ്ടമാക്കാത്ത പക്ഷം, SwiftUI രൂപഭാവ ക്രമീകരണത്തിന്റെ ഒരു ഭാഗം നമ്മുവേണ്ടി കൈകാര്യം ചെയ്യും.

## സ്ഥിര നിറങ്ങൾ സ്വയമായി മാറില്ല

ശ്രദ്ധിക്കേണ്ട കാര്യം, നാം ഒരു സ്ഥിര നിറം കൈമാറി നിർദ്ദിഷ്ടമാക്കിയാൽ, അത് ലൈറ്റ്/ഡാർക്ക് മോഡുകൾക്കിടയിൽ സ്വയമായി മാറില്ല.

ഉദാഹരണത്തിന്:

```swift
Text("ഉദാഹരണ വാചകം")
	.foregroundStyle(Color.gray)
```

ഇവിടെ ടെക്സ്റ്റ് `Color.gray` ആയി സജ്ജമാക്കിയതിനാൽ, അത് എല്ലായ്പ്പോഴും ചാരനിറത്തിൽ തന്നെ പ്രദർശിപ്പിക്കും.

മറ്റൊരു ഉദാഹരണം:

```swift
Text("ശീർഷകം")
    .foregroundStyle(Color.white)
```

ഈ കോഡ് ലൈറ്റ് മോഡിലും ഡാർക്ക് മോഡിലും വെള്ള നിറത്തിലുള്ള ടെക്സ്റ്റ് പ്രദർശിപ്പിക്കും.

പശ്ചാത്തലവും ഇളം നിറമാണെങ്കിൽ, വെള്ള ടെക്സ്റ്റ് വായിക്കാൻ ബുദ്ധിമുട്ടാകാം.

അതിനാൽ രൂപഭാവ ക്രമീകരണം ചെയ്യുമ്പോൾ, `Color.white`, `Color.black` പോലുള്ള സ്ഥിര നിറങ്ങൾ അനാവശ്യമായി hard-code ചെയ്യുന്നത് കഴിയുന്നതും ഒഴിവാക്കണം.

പല സാഹചര്യങ്ങളിലും, താഴെപ്പറയുന്ന സിസ്റ്റം semantic styles ആദ്യം ഉപയോഗിക്കാം:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

ഇതിന്റെ അർത്ഥം:

- `.primary` സാധാരണയായി പ്രധാന ഉള്ളടക്കത്തിനായി ഉപയോഗിക്കുന്നു.
- `.secondary` സാധാരണയായി ഉപ ഉള്ളടക്കത്തിനായി ഉപയോഗിക്കുന്നു.

ഇവ ലൈറ്റ് മോഡിനും ഡാർക്ക് മോഡിനും അനുസരിച്ച് പ്രദർശനം സ്വയമായി ക്രമീകരിക്കും.

## രൂപഭാവം അനുസരിച്ച് വ്യത്യസ്ത ഉള്ളടക്കം കാണിക്കുക

ചിലപ്പോൾ നിറം മാത്രമല്ല മാറേണ്ടത്; വ്യത്യസ്ത രൂപഭാവങ്ങളിൽ വ്യത്യസ്ത ഉള്ളടക്കവും കാണിക്കേണ്ടിവരും.

ഉദാഹരണത്തിന്:

- ലൈറ്റ് മോഡിൽ സാധാരണ lightbulb ഐക്കൺ കാണിക്കുക.
- ഡാർക്ക് മോഡിൽ പ്രകാശിക്കുന്ന lightbulb ഐക്കൺ കാണിക്കുക.
- ലൈറ്റ് മോഡിൽ ശീർഷകമായി `വെളിച്ചം` കാണിക്കുക.
- ഡാർക്ക് മോഡിൽ ശീർഷകമായി `ഇരുട്ട്` കാണിക്കുക.

ഇത്തരം സാഹചര്യത്തിൽ, നിലവിലെ സിസ്റ്റം ലൈറ്റ് മോഡിലാണോ ഡാർക്ക് മോഡിലാണോ എന്ന് പരിശോധിക്കണം.

SwiftUI-യിൽ, `colorScheme` environment value വഴി നിലവിലെ രൂപഭാവ മോഡ് ലഭ്യമാക്കാം:

```swift
@Environment(\.colorScheme) private var colorScheme
```

പൂർണ്ണ കോഡ്:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "വെളിച്ചം" : "ഇരുട്ട്"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("ഉദാഹരണ വാചകം")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

പ്രദർശന ഫലം:

![view](../../../Resource/025_view5.png)

പ്രിവ്യൂ ഫലത്തിൽ നിന്ന് കാണാൻ കഴിയുന്നതുപോലെ, ഒരേ view വ്യത്യസ്ത രൂപഭാവ മോഡുകളിൽ പൂർണ്ണമായും ഒരേ ഉള്ളടക്കം കാണിക്കുന്നില്ല.

ലൈറ്റ് മോഡിൽ, `Image` സാധാരണ lightbulb ഐക്കൺ കാണിക്കും, ശീർഷകം `വെളിച്ചം` എന്ന് കാണിക്കും.

ഡാർക്ക് മോഡിൽ, `Image` പ്രകാശിക്കുന്ന lightbulb ഐക്കൺ കാണിക്കും, ശീർഷകം `ഇരുട്ട്` എന്ന് കാണിക്കും.

അഥവാ, ഇവിടെ മാറുന്നത് നിറം മാത്രം അല്ല. ഐക്കൺ പേരും ശീർഷക ടെക്സ്റ്റും തന്നെ മാറിയിരിക്കുന്നു.

ഇതിന് കാരണം നാം ഐക്കണും ശീർഷകവും `body`-ക്കുള്ളിൽ നേരിട്ട് hard-code ചെയ്തിട്ടില്ല. പകരം, `colorScheme` അടിസ്ഥാനമാക്കി വ്യത്യസ്ത ഉള്ളടക്കം കണക്കാക്കുന്നു.

ഈ കോഡിൽ, നാം രണ്ട് computed properties ഉപയോഗിച്ചു:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "വെളിച്ചം" : "ഇരുട്ട്"
}
```

`colorScheme` `.light` ആണെങ്കിൽ, നിലവിലെ മോഡ് ലൈറ്റ് മോഡാണെന്ന് അർത്ഥം.

`colorScheme` `.dark` ആണെങ്കിൽ, നിലവിലെ മോഡ് ഡാർക്ക് മോഡാണെന്ന് അർത്ഥം.

അതുകൊണ്ട്, വ്യത്യസ്ത രൂപഭാവ മോഡുകൾ അനുസരിച്ച് വ്യത്യസ്ത ഐക്കൺ പേരുകളും ശീർഷക ടെക്സ്റ്റും തിരികെ നൽകാം.

ഇതും computed properties-ന്റെ ഒരു സാധാരണ ഉപയോഗമാണ്: നിലവിലെ state അടിസ്ഥാനമാക്കി view പ്രദർശിപ്പിക്കേണ്ട ഉള്ളടക്കം കണക്കാക്കുക.

## @Environment മനസ്സിലാക്കുക

ഇതാണ് `@Environment`-നെ നാം ആദ്യമായി കാണുന്നത്.

`@Environment` എന്നത് SwiftUI-യുടെ environment-ൽ നിന്ന് ഒരു മൂല്യം വായിക്കുന്നതായി മനസ്സിലാക്കാം.

App പ്രവർത്തിക്കുമ്പോൾ, സിസ്റ്റം പല തരത്തിലുള്ള environment വിവരങ്ങൾ നൽകും, ഉദാഹരണത്തിന്:

- നിലവിലെ ഭാഷ
- നിലവിലെ രൂപഭാവ മോഡ്
- നിലവിലെ layout direction
- നിലവിലെ font size ക്രമീകരണം

നിലവിലെ രൂപഭാവ മോഡ് ലഭ്യമാക്കേണ്ടപ്പോൾ, `colorScheme` വായിക്കാം:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ഇവിടെ:

```swift
\.colorScheme
```

environment-ൽ നിന്ന് `colorScheme` മൂല്യം വായിക്കുന്നതിനെ സൂചിപ്പിക്കുന്നു.

```swift
private var colorScheme
```

വായിച്ച മൂല്യം `colorScheme` എന്ന variable-ൽ സൂക്ഷിക്കുന്നതിനെ സൂചിപ്പിക്കുന്നു.

variable-ന്റെ പേര് നിങ്ങൾക്ക് തന്നെ തീരുമാനിക്കാം. ഉദാഹരണത്തിന്, ഇങ്ങനെ എഴുതുകയും ചെയ്യാം:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

മുൻപിലെ `\.colorScheme` മാറാത്തിടത്തോളം, വായിക്കുന്നത് നിലവിലെ രൂപഭാവ മോഡാണെന്ന് അർത്ഥം.

## colorScheme-ന്റെ രണ്ട് സാധാരണ മൂല്യങ്ങൾ

`colorScheme`-ന് സാധാരണയായി രണ്ട് മൂല്യങ്ങളുണ്ട്:

```swift
.light
.dark
```

അവയുടെ അർത്ഥം:

- `.light`: ലൈറ്റ് മോഡ്
- `.dark`: ഡാർക്ക് മോഡ്

ഉദാഹരണത്തിന്:

```swift
private var titleName: String {
    colorScheme == .light ? "വെളിച്ചം" : "ഇരുട്ട്"
}
```

ഈ കോഡിന്റെ അർത്ഥം:

നിലവിലെ മോഡ് ലൈറ്റ് മോഡാണെങ്കിൽ, `colorScheme == .light` true ആയതിനാൽ, അത് `"വെളിച്ചം"` തിരികെ നൽകും.

അല്ലെങ്കിൽ, അത് `"ഇരുട്ട്"` തിരികെ നൽകും.

അതിനാൽ, ലൈറ്റ് മോഡിൽ:

```swift
Text(titleName)
```

പ്രദർശിപ്പിക്കുന്ന ടെക്സ്റ്റ്:

```swift
വെളിച്ചം
```

ഡാർക്ക് മോഡിൽ, പ്രദർശിപ്പിക്കുന്ന ടെക്സ്റ്റ്:

```swift
ഇരുട്ട്
```

ഈ രീതിയിൽ, വ്യത്യസ്ത രൂപഭാവങ്ങൾക്ക് അനുസരിച്ച് വ്യത്യസ്ത ഉള്ളടക്കം കാണിക്കാം.

## നിറ ക്രമീകരണ ഉദാഹരണം

ടെക്സ്റ്റും ഐക്കണുകളും കൂടാതെ, ചിലപ്പോൾ നിറ ക്രമീകരണം നാം തന്നെ കൈകാര്യം ചെയ്യേണ്ടിവരും.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("വാചകം")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

പ്രദർശന ഫലം:

![view](../../../Resource/025_view6.png)

ഈ കോഡിൽ, ടെക്സ്റ്റ് വെള്ള നിറമാണ്, പശ്ചാത്തലം കറുപ്പാണ്.

ലൈറ്റ് മോഡിൽ, കറുപ്പ് പശ്ചാത്തലം വളരെ വ്യക്തമാണ്.

എന്നാൽ ഡാർക്ക് മോഡിൽ, മുഴുവൻ ഇന്റർഫേസ് പശ്ചാത്തലവും കറുപ്പാണെങ്കിൽ, ഈ കറുപ്പ് പശ്ചാത്തലം സിസ്റ്റം പശ്ചാത്തലവുമായി ലയിച്ച് വ്യക്തമല്ലാത്തതായി തോന്നാം.

ഈ സമയത്ത്, രൂപഭാവ മോഡ് അനുസരിച്ച് ടെക്സ്റ്റ് നിറവും പശ്ചാത്തല നിറവും മാറ്റാം:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("വാചകം")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

പ്രദർശന ഫലം:

![view](../../../Resource/025_view7.png)

ഇപ്പോൾ, ലൈറ്റ് മോഡിൽ കറുപ്പ് പശ്ചാത്തലവും വെള്ള ടെക്സ്റ്റും ആണ്.

ഡാർക്ക് മോഡിൽ വെള്ള പശ്ചാത്തലവും കറുപ്പ് ടെക്സ്റ്റും ആണ്.

ഇങ്ങനെ, ഏത് രൂപഭാവം ഉപയോഗിച്ചാലും, ടെക്സ്റ്റിനും പശ്ചാത്തലത്തിനും വ്യക്തമായ contrast ഉണ്ടായിരിക്കും, ഉപയോക്താക്കൾക്ക് ഉള്ളടക്കം വ്യക്തമായി കാണാം.

## നിറങ്ങൾ ക്രമീകരിക്കാൻ Assets ഉപയോഗിക്കുക

`colorScheme` ഉപയോഗിച്ച് കൈമാറി പരിശോധിക്കുന്നതിനു പുറമേ, ലൈറ്റ്/ഡാർക്ക് മോഡുകൾക്ക് അനുസരിക്കാൻ `Assets` resource colors ഉപയോഗിക്കാനും കഴിയും.

ഈ രീതി App-യിലെ പൊതുവായി ഉപയോഗിക്കുന്ന നിറങ്ങൾ കൈകാര്യം ചെയ്യാൻ കൂടുതൽ അനുയോജ്യമാണ്.

## colorScheme-നും Assets-നും ഉള്ള വ്യത്യാസം

`colorScheme`യും `Assets`യും രൂപഭാവ ക്രമീകരണം നടപ്പാക്കാൻ കഴിയും, പക്ഷേ അവ അനുയോജ്യമായ സാഹചര്യങ്ങൾ വ്യത്യസ്തമാണ്.

ലൈറ്റ്/ഡാർക്ക് മോഡ് അനുസരിച്ച് ടെക്സ്റ്റ് ഉള്ളടക്കം, image names, അല്ലെങ്കിൽ `SF Symbols` names മാത്രം മാറ്റേണ്ടതാണെങ്കിൽ, `colorScheme` ഉപയോഗിക്കാം.

ഉദാഹരണത്തിന്:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

App-ൽ പതിവായി ഉപയോഗിക്കുന്ന നിറമാണെങ്കിൽ, ഉദാഹരണത്തിന് theme color, card background color, text color എന്നിവയ്ക്ക്, `Assets` നിറങ്ങൾ ഉപയോഗിക്കുന്നത് കൂടുതൽ ശുപാർശ ചെയ്യുന്നു.

കാരണം `Assets` നിറങ്ങൾ പല view-ുകളിലും വീണ്ടും ഉപയോഗിക്കാം, പിന്നീട് മാറ്റാനും കൂടുതൽ എളുപ്പമാണ്.

ഉദാഹരണത്തിന്:

```swift
Color("redText")
```

ഇങ്ങനെ ചെയ്താൽ, `Assets`-ൽ നിറം മാറ്റുന്നത് മാത്രം മതി; ഈ നിറം ഉപയോഗിക്കുന്ന എല്ലാ സ്ഥലങ്ങളും ഒരുമിച്ച് മാറും.

## സംഗ്രഹം

ഈ പാഠത്തിൽ, ലൈറ്റ് മോഡിനും ഡാർക്ക് മോഡിനും വേണ്ട രൂപഭാവ ക്രമീകരണം പഠിച്ചു.

ആദ്യം, Xcode-ന്റെ `Canvas`-ൽ വ്യത്യസ്ത രൂപഭാവങ്ങൾ എങ്ങനെ പ്രിവ്യൂ ചെയ്യാമെന്ന് പഠിച്ചു, കൂടാതെ `Color Scheme Variants` ഉപയോഗിച്ച് ലൈറ്റ് മോഡും ഡാർക്ക് മോഡും ഒരേസമയം കാണുന്നതും പഠിച്ചു.

തുടർന്ന്, `Text`, `Image` പോലുള്ള SwiftUI view-ുകൾ സാധാരണയായി സിസ്റ്റം രൂപഭാവത്തിന് അനുസരിച്ച് നിറങ്ങൾ സ്വയമായി ക്രമീകരിക്കുന്നതായി മനസ്സിലാക്കി.

എന്നാൽ, `Color.white` അല്ലെങ്കിൽ `Color.black` പോലുള്ള സ്ഥിര നിറങ്ങൾ കൈമാറി നിർദ്ദിഷ്ടമാക്കിയാൽ, ഈ നിറങ്ങൾ സ്വയമായി മാറില്ല.

അടുത്തതായി, `@Environment`യും `colorScheme`യും പഠിച്ചു:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` വഴി, നിലവിലെ മോഡ് ലൈറ്റ് മോഡാണോ ഡാർക്ക് മോഡാണോ എന്ന് തിരിച്ചറിയാനും, വ്യത്യസ്ത രൂപഭാവങ്ങൾക്ക് അനുസരിച്ച് വ്യത്യസ്ത ഐക്കണുകൾ, ടെക്സ്റ്റ്, അല്ലെങ്കിൽ നിറങ്ങൾ കാണിക്കാനും കഴിയും.

അവസാനമായി, `Assets` നിറങ്ങളെക്കുറിച്ച് പഠിച്ചു.

`Assets` നിറങ്ങൾ ലൈറ്റ് മോഡിനും ഡാർക്ക് മോഡിനും നിറങ്ങൾ വേർതിരിച്ച് സജ്ജമാക്കാൻ കഴിയും, അതിനാൽ App-ിലെ പൊതുവായ color resources കൈകാര്യം ചെയ്യാൻ ഇവ അനുയോജ്യമാണ്.

രൂപഭാവ ക്രമീകരണം App-നെ പകലിലും രാത്രിയിലും കൂടുതൽ നല്ലതായി കാണിക്കാൻ സഹായിക്കും, വായിക്കാൻ ബുദ്ധിമുട്ടുള്ള ടെക്സ്റ്റ് അല്ലെങ്കിൽ വ്യക്തമല്ലാത്ത പശ്ചാത്തലം പോലുള്ള പ്രശ്നങ്ങളും ഒഴിവാക്കാം.

യഥാർത്ഥ വികസനത്തിൽ, SwiftUI-യുടെ default adaptation behavior-നും `Assets` നിറങ്ങൾക്കും മുൻഗണന നൽകുന്നത് ശുപാർശ ചെയ്യുന്നു.

വ്യത്യസ്ത രൂപഭാവങ്ങളിൽ ടെക്സ്റ്റ്, ഐക്കണുകൾ, ചിത്രങ്ങൾ എന്നിവ മാറ്റേണ്ടതായാലോ, കുറച്ച് പ്രത്യേക display effects കൈകാര്യം ചെയ്യേണ്ടതായാലോ, അപ്പോൾ `colorScheme` ഉപയോഗിച്ച് പരിശോധിക്കാം.

## വിപുലമായ അറിവ്: Font Size നിർദ്ദിഷ്ടമാക്കുക

മുമ്പ് `font` പഠിക്കുമ്പോൾ, നാം ചില system font styles ഉപയോഗിച്ചിരുന്നു:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` പോലുള്ള system font styles ഉപയോഗിക്കുന്നതിനു പുറമേ, font size കൈമാറി നിർദ്ദിഷ്ടമാക്കാനും കഴിയും.

ഉദാഹരണത്തിന്:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

ഇവിടെ:

```swift
.font(.system(size: 100))
```

font size `100 pt` ആയി സജ്ജമാക്കുന്നതിനെ സൂചിപ്പിക്കുന്നു.

`SF Symbols` ഐക്കണുകൾക്കായി, `font` ഐക്കണിന്റെ വലുപ്പത്തെയും ബാധിക്കും.

അതുകൊണ്ട് ഐക്കൺ കൂടുതൽ വലുതായി കാണിക്കണമെങ്കിൽ, `.font(.system(size:))` ഉപയോഗിച്ച് വലുപ്പം നിർദ്ദിഷ്ടമാക്കാം.

ഉദാഹരണത്തിന്:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

ശ്രദ്ധിക്കേണ്ടത്, font size കൈമാറി നിർദ്ദിഷ്ടമാക്കുന്നത് കൂടുതൽ flexible ആണെങ്കിലും, സാധാരണ ടെക്സ്റ്റിന് fixed font sizes വ്യാപകമായി ഉപയോഗിക്കുന്നത് ശുപാർശ ചെയ്യുന്നില്ല.

സാധാരണ ടെക്സ്റ്റിന് `.title`, `.headline`, `.body`, `.caption` പോലുള്ള system font styles ആദ്യം ഉപയോഗിക്കുക.

ഇങ്ങനെ ചെയ്താൽ വ്യത്യസ്ത ഉപകരണങ്ങൾക്കും ഉപയോക്താവിന്റെ font settings-നും കൂടുതൽ നന്നായി പൊരുത്തപ്പെടാൻ കഴിയും.
