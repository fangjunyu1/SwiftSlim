# പരിചയം

## Swift എന്താണ്?

Swift പഠിക്കാൻ തുടങ്ങുന്നതിന് മുമ്പ്, ആദ്യം Swift പ്രോഗ്രാമിംഗ് ഭാഷയെ കുറിച്ച് ഒന്ന് ലളിതമായി മനസിലാക്കാം. Swift എന്നത് Apple വികസിപ്പിച്ച ഒരു ആധുനിക പ്രോഗ്രാമിംഗ് ഭാഷയാണ്. ഇത് പ്രധാനമായും iOS, macOS, iPadOS, watchOS പോലുള്ള Apple പ്ലാറ്റ്‌ഫോം ആപ്പുകൾ നിർമ്മിക്കാൻ ഉപയോഗിക്കുന്നു.

![Swift](../../RESOURCE/000_swift.png)

മറ്റ് പ്രോഗ്രാമിംഗ് ഭാഷകളുമായി താരതമ്യപ്പെടുത്തുമ്പോൾ Swift-ന്റെ സിന്റാക്സ് കൂടുതൽ ലളിതമാണ്. സുരക്ഷിതത്വത്തിനും വായനാസൗകര്യത്തിനും കൂടുതൽ പ്രാധാന്യം നൽകുന്നു. SwiftUI ഒപ്പമുണ്ടാകുമ്പോൾ ഇത് പഠിക്കാനും മനസ്സിലാക്കാനും കൂടുതൽ എളുപ്പമാണ്. തീർച്ചയായും, ഞാൻ ഇതിന് മുമ്പ് മറ്റു പ്രോഗ്രാമിംഗ് ഭാഷകൾ പഠിച്ചിട്ടില്ല, അതിനാൽ ഇത് HTML-നേക്കാൾ അല്പം ബുദ്ധിമുട്ടാകാം.

Swift Apple പ്ലാറ്റ്‌ഫോം ആപ്പുകൾ വികസിപ്പിക്കാൻ മാത്രം ഉപയോഗിക്കുന്നതല്ല. Swift കമ്മ്യൂണിറ്റി Swift-നെ [Android](https://www.swift.org/blog/nightly-swift-sdk-for-android/) പ്ലാറ്റ്‌ഫോമിലേക്കും, പോലും [സെർവർ](https://github.com/awslabs/swift-aws-lambda-runtime) പോലുള്ള മറ്റ് പരിസരങ്ങളിലേക്കും കൊണ്ടുപോകാൻ പ്രവർത്തിച്ചു കൊണ്ടിരിക്കുന്നു. ദീർഘകാല വികസന സാധ്യത നോക്കുമ്പോൾ Swift-ന് കൂടുതൽ വളർച്ചാ സാധ്യതയുണ്ട്, ഭാവിയിൽ Flutter, React Native പോലുള്ള ക്രോസ്-പ്ലാറ്റ്‌ഫോം ഭാഷകളോട് സാമ്യമുള്ള നിലയിലേക്ക് എത്താനും സാധ്യതയുണ്ട്.

## പഠന ഉള്ളടക്കം

ഈ ട്യൂട്ടോറിയൽ പ്രധാനമായും Swift, SwiftUI എന്നിവയുടെ അടിസ്ഥാന അറിവുകൾ ചുറ്റിപ്പറ്റിയാണ്.

നാം ലളിതമായ വേരിയബിളുകളും ഡാറ്റാ ടൈപ്പുകളും കൊണ്ട് തുടങ്ങും, SwiftUI ഉപയോഗിച്ച് UI നിർമ്മിക്കും, ഡാറ്റ എങ്ങനെ സംഭരിക്കപ്പെടുന്നു, എങ്ങനെ വായിക്കപ്പെടുന്നു എന്നിവ മനസിലാക്കും, പിന്നെ SwiftData, iCloud sync പോലുള്ള മുന്നേറ്റ വിഷയങ്ങളിലേക്കും നീങ്ങാം.

ഓരോ അധ്യായവും ഒരു യാഥാർത്ഥ്യ ഉദാഹരണത്തെ ആധാരമാക്കി Swift അറിവുകൾ പഠിപ്പിക്കും.

ചില ട്യൂട്ടോറിയലുകൾ ആദ്യം വലിയ അളവിൽ syntax, API എന്നിവ വിശദീകരിച്ച് ശേഷം മാത്രമേ project practice-ലേക്ക് കടക്കാറുള്ളൂ. തുടങ്ങുന്നവർക്ക് ഇത് ഡിക്ഷണറിയിൽ നിന്ന് അക്ഷരങ്ങൾ പഠിക്കുന്ന കുട്ടിയെപ്പോലെയാണ്. ഉപയോഗിക്കാവുന്ന ഒരു സാഹചര്യവുമില്ലാതെ ഏറെ കാര്യങ്ങൾ മനപ്പാഠമാക്കേണ്ടിവന്നാൽ, ഒടുവിൽ പഠന താൽപര്യം തന്നെ നഷ്ടപ്പെടും.

ഈ ട്യൂട്ടോറിയലിൽ യഥാർത്ഥ development സാഹചര്യങ്ങൾക്കുള്ളിൽ നിന്നാണ് അറിവുകൾ അവതരിപ്പിക്കുക. ആവശ്യകതകളിൽ നിന്നാണ് ബന്ധപ്പെട്ട concepts ഉരുത്തിരിയുന്നത്.

ഈ ട്യൂട്ടോറിയൽ പൂർത്തിയാക്കിയാൽ, നിങ്ങൾക്ക് ഒരു അടിസ്ഥാന ആപ്പ് സ്വയം വികസിപ്പിക്കാനും പ്രവർത്തിപ്പിക്കാനും, അത് സ്വന്തം iOS / macOS ഉപകരണങ്ങളിൽ ഇൻസ്റ്റാൾ ചെയ്യാനും, App Store-ലേക്ക് പ്രസിദ്ധീകരിക്കാനും കഴിയും.

## പൂർണ്ണമായും പുതുമുഖമായാൽ എങ്ങനെ പഠിക്കണം?

ഞാനും തുടക്കത്തിൽ പൂർണ്ണമായും പുതുമുഖമായിരുന്നു. ആദ്യം YouTube-ലുള്ള ചില [Swift പഠന വീഡിയോകൾ](https://www.youtube.com/watch?v=4SaddkGm5yA&list=PLliocbKHJNwuLHJlVv6q5tKJxJltjBspC&index=3) ഒന്ന് നോക്കിയപ്പോൾ iOS ആപ്പ് ഡെവലപ്പ്മെന്റ് അത്രയും സങ്കീർണ്ണമല്ലെന്ന് തോന്നി; അതാണ് താൽപര്യം വളരാൻ കാരണമായത്. തുടർന്ന് ഞാൻ കൂടുതൽ Swift പഠിക്കാൻ തുടങ്ങി. ഉദാഹരണത്തിന്, [Swift Playgrounds](https://apps.apple.com/cn/app/swift-playgrounds/id1496833156?mt=12) എന്നതിലെ coding games പരീക്ഷിച്ചു, [Swift-ന്റെ ഔദ്യോഗിക ഡോക്യുമെന്റേഷൻ](https://docs.swift.org/swift-book/documentation/the-swift-programming-language/) വായിച്ചു, കൂടാതെ [100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui) പഠിച്ചു.

ഇപ്പോൾ തിരിഞ്ഞുനോക്കുമ്പോൾ, ശരിയായ പഠന ഘടന തന്നത് പ്രധാനമായും [100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui) ആണെന്ന് തോന്നുന്നു. അതുവഴി SwiftUI ക്രമബദ്ധമായി പഠിക്കാനായി. പക്ഷേ, ട്യൂട്ടോറിയൽ ഇംഗ്ലീഷിലായിരുന്നു, കൂടാതെ പ്രോഗ്രാമിംഗ് അടിസ്ഥാനമൊന്നും ഇല്ലാത്തതിനാൽ മനസ്സിലാക്കുന്ന പ്രക്രിയയിൽ ഏറെ ബുദ്ധിമുട്ടുകൾ ഉണ്ടായിരുന്നു. അതിനാലാണ് ഞാൻ [ChatGPT](https://chatgpt.com/)യും [Claude](https://claude.ai/?redirect=claude.com) പോലുള്ള AI ഉപകരണങ്ങൾ ഉപയോഗിച്ച് ആശയങ്ങൾ മനസിലാക്കിയത്.

പഠനത്തിനിടെ, മനസ്സിലാക്കാൻ ബുദ്ധിമുട്ടുള്ള അറിവുകൾ വരിക അനിവാര്യമാണ്. ഉദാഹരണത്തിന്, UserDefaults പഠിക്കുമ്പോൾ ഡാറ്റ എങ്ങനെ save ചെയ്യാമെന്നു മാത്രമേ എനിക്ക് അറിയുമായിരുന്നുള്ളു; ആ data manually read ചെയ്യണമെന്നും അറിയില്ലായിരുന്നു. അതുകൊണ്ട് തന്നെ എന്റെ code-ലാണ് പ്രശ്നമെന്ന് ഞാൻ സംശയിച്ചു. ഇത്ര ലളിതമായ ഒരു concept പോലും എന്നെ കുറെ ദിവസങ്ങൾ തടഞ്ഞുവെച്ചിരുന്നു.

Closure, main thread, generics പോലുള്ള ആശയങ്ങളിലും ഇതേ അനുഭവം ഉണ്ടായിരുന്നു. തുടക്കത്തിൽ ആഴത്തിൽ മനസിലാക്കാനായില്ല. പിന്നീട് യഥാർത്ഥ development സമയത്ത് ഇവ വീണ്ടും വന്നപ്പോൾ, അതിനെ വീണ്ടും പഠിക്കാൻ സമയം മാറ്റിവെക്കേണ്ടിവന്നു.

അതുകൊണ്ട് എന്റെ നിർദേശം:

**1. അടിസ്ഥാന അറിവുകൾ പൂർത്തിയാക്കുക**

ചൈനയിൽ ഒരു പറമാർശമുണ്ട്: “എല്ലാ തുടക്കവും ബുദ്ധിമുട്ടാണ്.” Swift പഠനം പൂജ്യത്തിൽ നിന്ന് തുടങ്ങുന്നവർക്ക് പഠനത്തിനിടെ ബുദ്ധിമുട്ടായി തോന്നുന്ന പല കാര്യങ്ങളും ഉണ്ടായേക്കും. ഇത് അധികമായി ആ വിഷയങ്ങൾ അസാധാരണമായി സങ്കീർണ്ണമാണെന്നതിനാൽ അല്ല; അറിവിന്റെ പിരമിഡിന്റെ അടിത്തട്ടിൽ ഇപ്പോഴും തുടങ്ങുന്നവർ കാണാത്ത ചില അടിസ്ഥാന ആശയങ്ങൾ കിടക്കുന്നതിനാലാണ്.

അത്തരമൊരു സാഹചര്യമുണ്ടായാൽ, [ChatGPT](https://chatgpt.com/)യും [Claude](https://claude.ai/?redirect=claude.com) പോലുള്ള AI ഉപകരണങ്ങൾ ഉപയോഗിച്ച് നഷ്ടമായ അടിസ്ഥാന അറിവുകൾ വേഗത്തിൽ പൂരിപ്പിക്കാം.

**2. ഓരോ ആശയവും മനസ്സിലാക്കുക**

പഠനത്തിൽ ഏറ്റവും പ്രധാനപ്പെട്ടത്, സമയം എത്ര എടുത്താലും, ഓരോ concept-ും സാധ്യമായത്ര മനസ്സിലാക്കാൻ ശ്രമിക്കുകയാണ്.

മനസ്സിലാകാത്ത ഭാഗങ്ങൾ skip ചെയ്താൽ, അവ അടുത്ത development ഘട്ടത്തിൽ വീണ്ടും വരാനുള്ള സാധ്യത ഏറെ കൂടുതലാണ്. അപ്പോൾ അവ വീണ്ടും progress തടയും. ഒടുവിൽ വീണ്ടും സമയം ചെലവഴിച്ച് അത് പഠിക്കേണ്ടിവരും. പിന്നീടാണ് തിരുത്തുന്നതിനെക്കാൾ പഠന ഘട്ടത്തിലും തന്നെ കാര്യങ്ങൾ മനസ്സിലാക്കുന്നത് നല്ലത്.

ഓരോ ആശയവും യഥാർത്ഥത്തിൽ ഒരു tool ആണെന്ന് കരുതാം. കൂടുതൽ tools കൈവശമുണ്ടെങ്കിൽ, പ്രശ്നപരിഹാര ശേഷിയും കൂടുതൽ ആയിരിക്കും.

ഒരു വീട് പണിയുന്നതുപോലെ. കൈയിൽ ഒരു tool മാത്രം ഉണ്ടെങ്കിൽ ഭൂരിഭാഗം ജോലിയും അല്പം പ്രയാസപ്പെട്ടെങ്കിലും തീർക്കാം. പക്ഷേ കൂടുതൽ quality ആവശ്യമായപ്പോൾ അത് മതിയാകില്ല. കൂടുതൽ tools കൈവശമുണ്ടെങ്കിൽ മാത്രമേ കൂടുതൽ ഉറപ്പുള്ള വീട് പണിയാനാകൂ.

**3. തുടർച്ചയായ പഠനം നിലനിർത്തുക**

പഠനം ദീർഘകാലം തുടരേണ്ട ഒന്നാണ്. ഒരു ട്യൂട്ടോറിയൽ ശ്രദ്ധാപൂർവ്വം പഠിക്കാൻ ഏതാനും മാസങ്ങൾ മാറ്റിവെക്കാൻ വളരെ കുറച്ച് പേർ മാത്രമേ ശരിക്കും തയ്യാറാകൂ. കുറച്ചുകാലം കഠിനമായി പഠിക്കുന്നതിനേക്കാൾ, തുടർച്ചയായി സമയം നിക്ഷേപിക്കുന്നത് ഏറെ പ്രധാനമാണ്.

പ്രതിദിനം വെറും ഒരു ശതമാനം മാത്രം പൂർത്തിയാക്കിയാലും, നൂറ് ദിവസത്തിൽ മുഴുവൻ പഠനലക്ഷ്യവും പൂർത്തിയാക്കാം.

## വിദ്യാഭ്യാസ ദർശനം

ഈ കോഴ്‌സ് Swift, SwiftUI എന്നിവയിലെ എല്ലാം പഠിപ്പിക്കില്ല. പകരം, പുതുമുഖങ്ങളെ Swift എന്ന പ്രോഗ്രാമിംഗ് വഴിയിലേക്ക് കൊണ്ടുപോകുകയാണ് ചെയ്യുന്നത്. ഇത് building blocks ചേർക്കുന്നതുപോലെയാണ് — അവ എങ്ങനെ കൂട്ടിച്ചേർക്കാം എന്ന് ട്യൂട്ടോറിയൽ കാണിച്ചുതരും; ശേഷം നിങ്ങളുടെ blocks നിങ്ങൾ തന്നെയാണ് നിർമ്മിക്കേണ്ടത്.

ഈ project [GitHub](https://github.com/fangjunyu1/SwiftSlim) ൽ open source ആയി ലഭ്യമാണ്. അത് സൗജന്യമായി download ചെയ്യുകയും ഉപയോഗിക്കുകയും ചെയ്യാം. അനുഭവിക്കാൻ App Store പതിപ്പും ലഭ്യമാണ്.

സാങ്കേതിക പശ്ചാത്തലം ഇല്ലാത്തെങ്കിലും സ്വന്തം career വഴിയെ മാറ്റാൻ ആഗ്രഹിക്കുന്ന കൂടുതൽ ആളുകൾക്ക് programming പഠിച്ചും apps വികസിപ്പിച്ചും പുതിയ സാധ്യതകൾ കണ്ടെത്താൻ സഹായിക്കാനാണ് ഞങ്ങളുടെ ആഗ്രഹം.

## ഉള്ളടക്ക പുതുക്കൽ

ഈ ട്യൂട്ടോറിയൽ നിലവിലെ Swift, SwiftUI പതിപ്പുകളെ അടിസ്ഥാനമാക്കി എഴുതിയതാണ്.

Apple പ്ലാറ്റ്‌ഫോങ്ങളും tools-ഉം പുതുക്കപ്പെടുന്നതിനനുസരിച്ച് ചില APIകൾ മാറാം. ഏതെങ്കിലും പ്രശ്നമോ update വേണ്ടതോ കണ്ടാൽ, ഔദ്യോഗിക documentation അല്ലെങ്കിൽ open-source repository-യിലെ ഏറ്റവും പുതിയ പതിപ്പ് പരിശോധിക്കാം.
