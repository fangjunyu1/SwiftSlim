# Xcode ഫയലുകൾ കൈകാര്യം ചെയ്യൽ

ഈ പാഠത്തിൽ, Xcode-ൽ സാധാരണയായി ഉപയോഗിക്കുന്ന ഫയൽ പ്രവർത്തനങ്ങൾ പഠിക്കാം.

മുൻപത്തെ പാഠങ്ങളിൽ, നാം ഭൂരിഭാഗം കോഡും `ContentView`-ൽ എഴുതി. അങ്ങനെ കോഡും പ്രദർശന ഫലവും കാണാൻ എളുപ്പമാണ്.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

എന്നാൽ ഒരു യഥാർത്ഥ App-ൽ, സാധാരണയായി എല്ലാ കോഡും ഒരു ഫയലിൽ എഴുതാറില്ല.

പേജുകളും ഘടകങ്ങളും ഫംഗ്ഷനുകളും കൂടുതലാകുമ്പോൾ, വ്യത്യസ്ത കാഴ്ചകളെ വ്യത്യസ്ത SwiftUI ഫയലുകളിലായി വേർതിരിക്കേണ്ടിവരും.

ഉദാഹരണത്തിന്:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

ഇത് കോഡിന്റെ ഘടന കൂടുതൽ വ്യക്തമായി കാണാൻ സഹായിക്കുന്നു, പിന്നീടുള്ള പരിപാലനവും മാറ്റങ്ങളും എളുപ്പമാക്കുന്നു.

## ഫയലുകൾ എന്തിന് കൈകാര്യം ചെയ്യണം

SwiftUI-യിൽ, ഒരു പേജോ ഒരു ഘടകമോ സാധാരണയായി ഒരു സ്വതന്ത്ര കാഴ്ച ഫയലായി എഴുതാം.

ഉദാഹരണത്തിന്:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ഇവിടെ `CardView` ഒരു SwiftUI കാഴ്ചയാണ്.

എല്ലാ കാഴ്ചകളും `ContentView`-ൽ എഴുതിയാൽ, കോഡ് ക്രമേണ നീളുകയും വായിക്കാൻ കൂടുതൽ ബുദ്ധിമുട്ടാകുകയും ചെയ്യും.

അതിനാൽ, ഫയലുകൾ സൃഷ്ടിക്കുക, പേരുമാറ്റുക, ഇല്ലാതാക്കുക, കൂടാതെ വ്യത്യസ്ത കോഡ് ഫയലുകൾ കൈകാര്യം ചെയ്യാൻ ഫോൾഡറുകൾ ഉപയോഗിക്കുക എന്നിവ പഠിക്കണം.

## SwiftUI ഫയൽ സൃഷ്ടിക്കൽ

Xcode-ൽ SwiftUI ഫയൽ സൃഷ്ടിക്കാൻ സാധാരണയായി രണ്ട് രീതികളുണ്ട്:

ഒന്ന്, ഒരു ശൂന്യ ഫയൽ സൃഷ്ടിച്ച് കോഡ് സ്വയം എഴുതുന്നതാണ്.

മറ്റൊന്ന്, SwiftUI ടെംപ്ലേറ്റ് ഫയൽ സൃഷ്ടിച്ച് Xcode അടിസ്ഥാന കോഡ് സ്വയം സൃഷ്ടിക്കുന്നതാണ്.

ആരംഭക്കാർക്ക് SwiftUI ടെംപ്ലേറ്റ് ഫയൽ ഉപയോഗിക്കുന്നതാണ് കൂടുതൽ ശുപാർശ ചെയ്യുന്നത്, കാരണം അത് അടിസ്ഥാന ഘടന സ്വയം സൃഷ്ടിക്കും.

### ശൂന്യ ഫയൽ സൃഷ്ടിക്കൽ

Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator പ്രദേശത്ത്, ശൂന്യ ഇടത്തിലോ ഫോൾഡറിലോ വലത്-ക്ലിക്ക് ചെയ്ത് `New Empty File` തിരഞ്ഞെടുക്കുക.

പ്രവർത്തന സ്ഥാനം താഴെ കാണിക്കുന്നു:

![xcode](../../../Resource/027_view1.png)

ശൂന്യ ഫയൽ സൃഷ്ടിച്ച ശേഷം, Xcode സാധാരണയായി `Untitled.swift` പോലുള്ള പേരില്ലാത്ത ഒരു Swift ഫയൽ സൃഷ്ടിക്കും.

അത് `TestView.swift` എന്ന് പേരുമാറ്റാം.

SwiftUI പ്രോജക്റ്റിൽ, ഈ ഫയൽ ഒരു കാഴ്ച കാണിക്കാൻ ഉപയോഗിക്കുന്നതാണെങ്കിൽ, സാധാരണയായി പേരിന്റെ അവസാനത്തിൽ `View` ചേർക്കാൻ ശുപാർശ ചെയ്യുന്നു.

ഉദാഹരണത്തിന്:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

ഇത് നിർബന്ധമായ നിയമമല്ല, പക്ഷേ ഇത് ഒരു കാഴ്ച ഫയലാണെന്ന് ഒരു നോക്കിൽ മനസ്സിലാക്കാൻ സഹായിക്കും.

ശൂന്യ ഫയൽ സൃഷ്ടിച്ച ശേഷം, ഫയലിൽ ഡിഫോൾട്ട് കമന്റുകൾ മാത്രം ഉണ്ടായിരിക്കാം, അല്ലെങ്കിൽ ശൂന്യമായിരിക്കാം. ഇത് Xcode പതിപ്പിനും സൃഷ്ടിക്കുന്ന രീതിക്കും ആശ്രയിച്ചിരിക്കുന്നു.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ഈ സമയത്ത്, SwiftUI കോഡ് കൈകൊണ്ട് എഴുതേണ്ടതുണ്ട്:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

ഇങ്ങനെ ഒരു ലളിതമായ SwiftUI കാഴ്ച ഫയൽ സൃഷ്ടിച്ചു കഴിഞ്ഞു.

### ഫയലിന്റെ പേരുമാറ്റൽ

ഫയൽ സൃഷ്ടിച്ചതിന് ശേഷം ഫയൽ പേര് മാറ്റണമെങ്കിൽ, Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator പ്രദേശത്ത് ഫയൽ തിരഞ്ഞെടുക്കുക, തുടർന്ന് എന്റർ കീ `⏎` അമർത്തുക.

![xcode](../../../Resource/027_view2.png)

ഇങ്ങനെ ഫയൽ പേര് മാറ്റാം.

ശ്രദ്ധിക്കേണ്ടത്, ഫയൽ പേര് മാറ്റുന്നത് കോഡിലുള്ള കാഴ്ചയുടെ പേര് സ്വയം മാറ്റുകയില്ല.

ഉദാഹരണത്തിന്, ഫയൽ പേര് `ProfileView.swift` ആയി മാറ്റിയ ശേഷവും, കോഡിൽ ഇപ്പോഴും `struct TestView: View` ആയിരിക്കാം.

ഇത് നിർബന്ധമായും പിശക് ഉണ്ടാക്കണമെന്നില്ല, പക്ഷേ ആളുകൾക്ക് ആശയക്കുഴപ്പം ഉണ്ടാക്കാൻ ഇടയുണ്ട്.

അതിനാൽ, ഫയൽ പേരും കാഴ്ചയുടെ പേരും ഒരുപോലെ നിലനിർത്താൻ ശുപാർശ ചെയ്യുന്നു. ഉദാഹരണത്തിന്, ഫയൽ പേര് `ProfileView.swift` ആണെങ്കിൽ, കാഴ്ചയുടെ പേരും `ProfileView` ഉപയോഗിക്കുക.

## SwiftUI ടെംപ്ലേറ്റ് ഫയൽ സൃഷ്ടിക്കൽ

ശൂന്യ ഫയൽ സൃഷ്ടിക്കുന്നതിനു പുറമെ, SwiftUI ടെംപ്ലേറ്റ് ഫയൽ നേരിട്ട് സൃഷ്ടിക്കാനും കഴിയും.

Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator പ്രദേശത്ത്, ശൂന്യ ഇടത്തിലോ ഫോൾഡറിലോ വലത്-ക്ലിക്ക് ചെയ്ത് `New File from Template...` തിരഞ്ഞെടുക്കുക.

പ്രവർത്തന സ്ഥാനം താഴെ കാണിക്കുന്നു:

![xcode](../../../Resource/027_view3.png)

തുറക്കുന്ന വിൻഡോയിൽ, മുകളിലായി `iOS`, `macOS` എന്നിവ പോലെ വ്യത്യസ്ത പ്ലാറ്റ്ഫോമുകൾ തിരഞ്ഞെടുക്കാം.

ഇവിടെ നിലവിലെ പ്രോജക്റ്റിനൊത്ത പ്ലാറ്റ്ഫോം തന്നെ തിരഞ്ഞെടുക്കുക. ഉദാഹരണത്തിന്, നാം iOS App സൃഷ്ടിക്കുകയാണെങ്കിൽ, `iOS` തിരഞ്ഞെടുക്കുക.

![xcode](../../../Resource/027_view4.png)

തുടർന്ന് താഴെ `SwiftUI View` തിരഞ്ഞെടുക്കുക.

![xcode](../../../Resource/027_view12.png)

പിന്നെ `Next` ക്ലിക്ക് ചെയ്ത്, ഫയൽ പേര് നൽകി സേവ് ചെയ്യുക.

ഉദാഹരണത്തിന്, നാം `Test2View.swift` എന്നൊരു ഫയൽ സൃഷ്ടിക്കുന്നു.

Xcode താഴെ കാണിക്കുന്നതുപോലുള്ള കോഡ് സ്വയം സൃഷ്ടിക്കും:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

ശൂന്യ ഫയലുമായി താരതമ്യം ചെയ്യുമ്പോൾ, ടെംപ്ലേറ്റ് ഫയലിൽ `import SwiftUI`, `struct`, `body`, `#Preview` എന്നിവ സ്വയം ഉൾപ്പെടും.

അതിനാൽ ആരംഭക്കാർക്ക് SwiftUI ടെംപ്ലേറ്റ് ഫയൽ ഉപയോഗിക്കുന്നത് കൂടുതൽ ലളിതമാണ്, കോഡ് വിട്ടുപോകാൻ സാധ്യതയും കുറവാണ്.

**സൂചന**: വലത്-ക്ലിക്ക് ചെയ്ത് ഫയൽ സൃഷ്ടിക്കുന്നതിനു പുറമെ, Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator-ന്റെ താഴെയുള്ള പ്ലസ് ബട്ടൺ ക്ലിക്ക് ചെയ്തും ഫയലോ ഫോൾഡറോ സൃഷ്ടിക്കാം.

![xcode](../../../Resource/027_view5.png)

## ഫോൾഡർ സൃഷ്ടിക്കൽ

ഫയലുകൾ കൂടുതലാകുമ്പോൾ, അവ കൈകാര്യം ചെയ്യാൻ ഫോൾഡറുകൾ ഉപയോഗിക്കാം.

Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator പ്രദേശത്ത്, ശൂന്യ ഇടത്തിലോ ഫോൾഡറിലോ വലത്-ക്ലിക്ക് ചെയ്ത് `New Folder` തിരഞ്ഞെടുക്കുക.

ഫോൾഡർ സൃഷ്ടിച്ച ശേഷം, ബന്ധപ്പെട്ട ഫയലുകൾ ഫോൾഡറിലേക്ക് ഡ്രാഗ് ചെയ്യാം.

![xcode](../../../Resource/027_view7.png)

ഫോൾഡർ പ്രധാനമായും പ്രോജക്റ്റ് ഘടന ക്രമീകരിക്കാൻ ഉപയോഗിക്കുന്നു; അത് കാഴ്ചയുടെ ഉപയോഗരീതിയെ തന്നെ മാറ്റുകയില്ല.

കാഴ്ചയുടെ പേര് ശരിയായിരിക്കുന്നിടത്തോളം, മറ്റ് ഫയലുകളിലും `TestView()`, `Test2View()` പോലുള്ള രീതിയിൽ ഉപയോഗിക്കാം.

## ഫയലോ ഫോൾഡറോ ഇല്ലാതാക്കൽ

ഫയലോ ഫോൾഡറോ ഇല്ലാതാക്കണമെങ്കിൽ, Xcode-ന്റെ ഇടത് വശത്തുള്ള Navigator പ്രദേശത്ത് ബന്ധപ്പെട്ട ഫയൽ തിരഞ്ഞെടുക്കുക, തുടർന്ന് വലത്-ക്ലിക്ക് ചെയ്ത് `Delete` തിരഞ്ഞെടുക്കുക.

ഫയൽ തിരഞ്ഞെടുക്കിയ ശേഷം delete കീ `⌫` അമർത്താനും കഴിയും.

Xcode ഇല്ലാതാക്കൽ സ്ഥിരീകരണ വിൻഡോ കാണിക്കും.

![xcode](../../../Resource/027_view10.png)

`Move to Trash` തിരഞ്ഞെടുക്കുന്നത് ഫയൽ Trash-ലേക്ക് മാറ്റുന്നതാണ്.

`Remove Reference` കണ്ടാൽ, അത് Xcode പ്രോജക്റ്റിൽ നിന്ന് റഫറൻസ് മാത്രം നീക്കുന്നതാണെന്ന് അർത്ഥം; ഡിസ്കിലുള്ള യഥാർത്ഥ ഫയൽ ഇല്ലാതാക്കുകയില്ല.

ആരംഭക്കാർക്ക്, നിങ്ങൾ ഇപ്പോൾ സൃഷ്ടിച്ച സാധാരണ ഫയൽ മാത്രം ഇല്ലാതാക്കുന്നതാണെങ്കിൽ, സാധാരണയായി `Move to Trash` തിരഞ്ഞെടുക്കുന്നതു മതി.

## നിരവധി SwiftUI കാഴ്ചകൾ കാണിക്കൽ

ഫയൽ സൃഷ്ടിക്കൽ പഠിച്ച ശേഷം, വ്യത്യസ്ത കാഴ്ചകളെ വ്യത്യസ്ത ഫയലുകളിലായി വേർതിരിക്കാം.

ഉദാഹരണത്തിന്, നാം രണ്ട് SwiftUI ഫയലുകൾ സൃഷ്ടിക്കുന്നു:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` എഴുത്തുകാരന്റെ പേര് കാണിക്കാൻ ഉപയോഗിക്കുന്നു:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` കാർഡ് പശ്ചാത്തലം കാണിക്കാൻ ഉപയോഗിക്കുന്നു:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

തുടർന്ന്, `ContentView`-ൽ ഈ രണ്ട് കാഴ്ചകളും ഉപയോഗിക്കാം:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

റൺ ചെയ്ത ശേഷം, `CardView`യും `AuthorView`യും ഒന്നിച്ച് കൂട്ടിച്ചേർത്ത് കാണിക്കുന്നതായി കാണാം:

![xcode](../../../Resource/027_view11.png)

ഈ ഫലത്തിൽ, `CardView` പശ്ചാത്തലമായി പ്രവർത്തിക്കുന്നു, `AuthorView` മുകളിൽ കാണിക്കുന്നു. കാരണം അവ `ZStack`-ൽ വച്ചിരിക്കുന്നു.

`ContentView`-ൽ, `CardView()`യും `AuthorView()`യും ഉപയോഗിച്ച് ഈ രണ്ട് കാഴ്ചകളും ഉപയോഗിക്കുന്നു. പേരിനുശേഷമുള്ള `()` ഈ കാഴ്ച സൃഷ്ടിച്ച് ഉപയോഗിക്കുന്നു എന്നതിനെ സൂചിപ്പിക്കുന്നു.

ഇതാണ് SwiftUI-യിലെ സാധാരണ കാഴ്ച നെസ്റ്റിംഗ്: ഒരു കാഴ്ചയ്ക്കുള്ളിൽ മറ്റ് കാഴ്ചകൾ തുടർന്നും ഉപയോഗിക്കാം.

## പ്രവേശന ഫയൽ

അവസാനമായി, App എവിടെ നിന്നാണ് ഇന്റർഫേസ് കാണിക്കാൻ തുടങ്ങുന്നതെന്ന് വീണ്ടും നോക്കാം.

SwiftUI പ്രോജക്റ്റിൽ, സാധാരണയായി ഒരു പ്രവേശന ഫയൽ ഉണ്ടാകും:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ഇവിടെ `@main` ഇത് App-ന്റെ പ്രവേശന സ്ഥാനമാണെന്ന് സൂചിപ്പിക്കുന്നു.

`WindowGroup`-ൽ, നാം കാണുന്നത്:

```swift
ContentView()
```

ഇത് App ആരംഭിച്ച ശേഷം ആദ്യം `ContentView` കാണിക്കുന്നുവെന്ന് അർത്ഥമാക്കുന്നു.

തുടർന്ന് `ContentView`-നുള്ളിൽ വീണ്ടും കാണിക്കുന്നത്:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

അതിനാൽ കാഴ്ചകളുടെ നിലക്രമം ഇങ്ങനെ മനസ്സിലാക്കാം:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

പ്രവേശന ഫയലിൽ കാണിക്കുന്ന കാഴ്ച `CardView` ആയി മാറ്റിയാൽ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

അപ്പോൾ App ആരംഭിച്ച ശേഷം `CardView` മാത്രം കാണിക്കും.

അഥവാ, പ്രവേശന കോഡിലെ `WindowGroup` App ആരംഭിച്ചതിനു ശേഷം ആദ്യം ഏത് കാഴ്ച കാണിക്കണമെന്നത് തീരുമാനിക്കുന്നു.

ആ കാഴ്ചയുടെ ഉള്ളിൽ, മറ്റ് കാഴ്ചകൾ തുടർന്നും കാണിക്കാം.

## സംഗ്രഹം

ഈ പാഠത്തിൽ, Xcode-ൽ സാധാരണ ഉപയോഗിക്കുന്ന ഫയൽ മാനേജ്മെന്റ് പ്രവർത്തനങ്ങൾ പഠിച്ചു.

SwiftUI ഫയൽ എങ്ങനെ സൃഷ്ടിക്കാം, ഫയൽ എങ്ങനെ പേരുമാറ്റാം, ഫോൾഡർ എങ്ങനെ സൃഷ്ടിക്കാം, ഫയൽ എങ്ങനെ ഇല്ലാതാക്കാം എന്നിവ നാം മനസ്സിലാക്കി.

അതോടൊപ്പം, ഒരു പ്രധാന ആശയവും നാം പഠിച്ചു: SwiftUI കാഴ്ചകളെ വ്യത്യസ്ത ഫയലുകളിലായി വേർതിരിച്ച്, മറ്റ് കാഴ്ചകളിൽ ഉപയോഗിക്കാം.

ഉദാഹരണത്തിന്:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ഇവിടെ `CardView()`യും `AuthorView()`യും `ContentView`-ൽ മറ്റ് കാഴ്ചകൾ ഉപയോഗിക്കുന്നതിന്റെ ഉദാഹരണങ്ങളാണ്.

അവസാനമായി, പ്രവേശന ഫയലിന്റെ പ്രദർശന തർക്കവും നാം വീണ്ടും നോക്കി.

App ആരംഭിക്കുമ്പോൾ, ആദ്യം `@main` അടയാളപ്പെടുത്തിയ പ്രവേശന കോഡിലേക്ക് പ്രവേശിക്കും, തുടർന്ന് `WindowGroup`-ലുള്ള കാഴ്ച കാണിക്കും.

ഇത് നമുക്ക് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു: വ്യത്യസ്ത SwiftUI ഫയലുകൾ പരസ്പരം വേർപ്പെട്ടതല്ല; അവ പരസ്പരം കൂട്ടിച്ചേർന്ന് ഒടുവിൽ സമ്പൂർണ്ണ App ഇന്റർഫേസ് രൂപപ്പെടുത്താം.
