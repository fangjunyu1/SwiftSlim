# ലൂപുകളും ലിസ്റ്റുകളും

ഈ പാഠത്തിൽ, Swift ലെയുള്ള `for` ലൂപ്പും SwiftUI ലെയുള്ള `ForEach` ഉം നാം പഠിക്കും.

ഇവ രണ്ടും അറേയിലുള്ള ഘടകങ്ങളെ ക്രമമായി വായിക്കാൻ സഹായിക്കുന്നു. യാഥാർത്ഥ്യ വികസനത്തിൽ, ഒരു കൂട്ടം ഡാറ്റ ഒന്നൊന്നായി എടുത്ത് തുടർന്ന് പ്രോസസ് ചെയ്യുകയോ പ്രദർശിപ്പിക്കുകയോ ചെയ്യേണ്ടി വരും. അതിനായി ലൂപ് വളരെ പ്രധാനപ്പെട്ട ഒരു ഉപകരണമാണ്.

നാം വെബ്‌പേജുകൾ ബ്രൗസ് ചെയ്യുമ്പോഴും App ഉപയോഗിക്കുമ്പോഴും, പല ലിസ്റ്റുകളും ഇനങ്ങളും പതിവായി കാണാറുണ്ട്. ഇത്തരം ഉള്ളടക്കങ്ങൾ സാധാരണ ആദ്യം `അറേ`യിൽ സംഭരിക്കും, തുടർന്ന് `ലൂപ്` വഴി ഓരോന്നായി എടുത്ത് ഒടുവിൽ ഇന്റർഫേസിൽ പ്രദർശിപ്പിക്കും.

ഉദാഹരണത്തിന്, Google ന്റെ തിരയൽ ഫലങ്ങളുടെ ലിസ്റ്റ്:

![Google](../../Resource/017_google.png)

തിരയൽ ഫലങ്ങളുടെ ലിസ്റ്റിലെ ഓരോ ഇനവും സാധാരണ ഒരു ഡാറ്റാ ഇനത്തോട് തുല്യമായിരിക്കും, അത് ലൂപ് വഴി ഒന്നൊന്നായി പ്രദർശിപ്പിക്കുന്നു.

അതിനാൽ, `അറേ`യും `ലൂപ്`വും പലപ്പോഴും ഒന്നിച്ച് പ്രത്യക്ഷപ്പെടുന്നു. `അറേ` ഒരു കൂട്ടം ഡാറ്റ സംഭരിക്കാൻ ഉപയോഗിക്കുന്നു; `ലൂപ്` ആ ഡാറ്റ ക്രമമായി എടുത്ത് അതേ പ്രവർത്തനം നിർവഹിക്കാൻ ഉപയോഗിക്കുന്നു.

ഇപ്പോൾ, ഏറ്റവും അടിസ്ഥാനപരമായ `for` ലൂപിൽ നിന്ന് തുടങ്ങാം.

## കൗണ്ട്ഡൗൺ

നമുക്ക് ഒരു കൗണ്ട്ഡൗൺ ഫീച്ചർ അനുകരിക്കണമെന്നു കരുതാം, അതായത് `“5、4、3、2、1”` എന്ന ക്രമത്തിൽ ഔട്ട്പുട്ട് ചെയ്യണം.

ഏറ്റവും നേരിട്ടുള്ള എഴുതുന്ന രീതി:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

ഈ രീതി കൗണ്ട്ഡൗൺ അനുകരിക്കാനാകുമെങ്കിലും, ഉള്ളടക്കം കൂടുതലായാൽ ഇത് വളരെ ആവർത്തനപരമായി തോന്നും. പിന്നീട് തിരുത്താനും ഇത് സൗകര്യപ്രദമല്ല.

അപ്പോൾ, ആദ്യം ഡാറ്റ ഒരു അറേയിൽ വെച്ച്, തുടർന്ന് `for` ലൂപ് ഉപയോഗിച്ച് ഒന്നൊന്നായി പ്രിന്റ് ചെയ്യാം:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ഈ എഴുതൽ കൂടുതൽ ലളിതവും വികസിപ്പിക്കാൻ എളുപ്പവുമാണ്. കൗണ്ട്ഡൗണിലെ സംഖ്യകൾ വളരെ കൂടുതലായാലും, ഓരോ `print` ഉം വരി വരിയായി ആവർത്തിച്ച് എഴുതേണ്ടതില്ല; അറേയിലെ ഓരോ ഘടകവും ലൂപ് സ്വയം കൈകാര്യം ചെയ്യും.

ഈ കോഡ് ഒരു ബട്ടണിൽ വെച്ച് പരീക്ഷിക്കാം:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

ബട്ടണിൽ ക്ലിക്ക് ചെയ്താൽ, `for` ലൂപ് അറേയിലെ ക്രമത്തിൽ `countDown` ലുള്ള ഓരോ പൂർണ്ണസംഖ്യയും ഒന്നൊന്നായി എടുത്ത് കോൺസോളിൽ പ്രിന്റ് ചെയ്യും.

ഇനി `for` ലൂപ് എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്ന് നോക്കാം.

## for ലൂപ്

`for` ലൂപ് ഒരു കൂട്ടം ഡാറ്റ ക്രമത്തിൽ പ്രോസസ് ചെയ്യാൻ ഉപയോഗിക്കുന്നു. സാധാരണയായി ഇത് അറേകളും റേഞ്ചുകളും ഉപയോഗിച്ചാണ് പ്രവർത്തിക്കുന്നത്.

അടിസ്ഥാന രൂപം:

```swift
for ഘടകം in അറേ അല്ലെങ്കിൽ റേഞ്ച് {
    നിർവഹിക്കേണ്ട കോഡ്
}
```

ഉദാഹരണത്തിന്, ഇപ്പോൾ കണ്ട കൗണ്ട്ഡൗൺ കോഡ്:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ഇവിടെ `i` നിലവിൽ എടുത്ത ഘടകത്തെ സൂചിപ്പിക്കുന്നു. `in` ന് ശേഷം വരുന്ന ഭാഗം ലൂപ്പ് ചെയ്യേണ്ട ഡാറ്റയാണ്, അതായത് `countDown` അറേ.

ഈ കോഡിന്റെ അർത്ഥം: പ്രോഗ്രാം `countDown` അറേയിൽ നിന്ന് ഓരോ ഘടകവും ക്രമമായി എടുത്തുകൊണ്ടിരിക്കും. ഓരോ തവണയും ഒരു ഘടകം എടുത്താൽ, അതിനെ താൽക്കാലികമായി `i` എന്ന് വിളിച്ച്, തുടർന്ന് കർളി ബ്രേസുകൾക്കുള്ളിലെ കോഡ് നിർവഹിക്കും.

പ്രവർത്തനക്രമം ഇങ്ങനെ മനസ്സിലാക്കാം:

* ആദ്യമായി എടുത്തത് `5` ആകും; അതിനാൽ ആ സമയത്ത് `i` യുടെ മൂല്യം `5` ആണ്
* `print(i)` നിർവഹിച്ച് `5` പ്രിന്റ് ചെയ്യും
* തുടർന്ന് അടുത്ത ഘടകമായ `4` എടുക്കും
* വീണ്ടും `print(i)` നിർവഹിച്ച് `4` പ്രിന്റ് ചെയ്യും

അടുത്തുള്ള `3`、`2`、`1` എന്നിവയും ഇതേ രീതിയിൽ ഒന്നൊന്നായി നിർവഹിക്കും.

![for](../../Resource/017_for.png)

അറേയിലെ എല്ലാ ഘടകങ്ങളും പ്രോസസ് ചെയ്ത് കഴിഞ്ഞാൽ, `for` ലൂപ് സ്വയമേവ അവസാനിക്കും.

`for` ലൂപ് ഉപയോഗിച്ച് നമുക്ക് അറേയിലെ ഓരോ ഘടകവും ലഭിക്കും, കൂടാതെ കൗണ്ട്ഡൗൺ പോലുള്ള ഫീച്ചറുകളും നിർമ്മിക്കാം.

**ശ്രദ്ധിക്കുക**: `i` എന്നത് for ലൂപിൽ വളരെ സാധാരണമായി ഉപയോഗിക്കുന്ന ഒരു പേരാണ്. ഇത് നിലവിൽ എടുത്ത ഘടകത്തെ സൂചിപ്പിക്കാനാണ് ഉപയോഗിക്കുന്നത്. എന്നാൽ ഇത് നിർബന്ധമായ ഒരു പേര് അല്ല; മറ്റേതെങ്കിലും ഇംഗ്ലീഷ് പേര് ഉപയോഗിക്കാം.

ഉദാഹരണത്തിന്, ഓരോ എടുത്ത ഘടകത്തെയും സൂചിപ്പിക്കാൻ `num` ഉപയോഗിക്കാം:

```swift
for num in countDown {
    print(num)
}
```

കർളി ബ്രേസുകൾക്കുള്ളിലും നിലവിലെ ഘടകത്തെ സൂചിപ്പിക്കാൻ `num` തന്നെ ഉപയോഗിക്കണം.

### ലൂപ്പ് റേഞ്ച്

`for` ലൂപ് അറേകൾ മാത്രമല്ല, റേഞ്ചുകളും ലൂപ്പ് ചെയ്യാൻ കഴിയും.

ഉദാഹരണത്തിന്, `50` വരെയുള്ള ധന പൂർണ്ണസംഖ്യകളുടെ മൊത്തം കണക്കാക്കണമെങ്കിൽ, `1` മുതൽ `50` വരെയുള്ള എല്ലാ പൂർണ്ണസംഖ്യകളും ഒന്നൊന്നായി ചേർക്കണം.

ഏറ്റവും നേരിട്ടുള്ള എഴുതുന്ന രീതി ഇങ്ങനെ ആയിരിക്കും:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

ഈ എഴുതുന്ന രീതിയിലെ പ്രശ്നം, ഓരോ സംഖ്യയും കൈകൊണ്ട് എഴുതണം എന്നതാണ്. ഇത് ബുദ്ധിമുട്ടുള്ളതും യഥാർത്ഥ വികസന സാഹചര്യങ്ങൾക്ക് അനുയോജ്യമല്ലാത്തതുമാണ്.

അപ്പോൾ, റേഞ്ച് ഓപ്പറേറ്ററും `for` ലൂപും ചേർത്ത് ഇത് പൂർത്തിയാക്കാം.

മുമ്പത്തെ “ഓപ്പറേറ്ററുകൾ” എന്ന പാഠത്തിൽ, നാം റേഞ്ച് ഓപ്പറേറ്റർ പഠിച്ചിരുന്നു:

```swift
1...3   // 1, 2, 3
```

ഇവിടെ `...` എന്നത് അടഞ്ഞ ഇടവേളയാണ്, അതായത് ആരംഭവും അവസാനവും ഉൾപ്പെടും.

അതിനാൽ ഇവിടെ `1` മുതൽ `50` വരെയുള്ള പൂർണ്ണസംഖ്യകളുടെ റേഞ്ച് `for` ലൂപിൽ ഉപയോഗിക്കാം:

```swift
for i in 1...50 {
    
}
```

ഈ സംഖ്യകളെ ചേർക്കേണ്ടതിനാൽ, മൊത്തം സംഭരിക്കാൻ ഒരു വേറിയബിൾ വേണം:

```swift
var count = 0
```

തുടർന്ന് ഓരോ ലൂപ്പിലും, നിലവിലെ സംഖ്യ `count` ലേക്ക് ചേർക്കുക:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

ഈ കോഡിന്റെ അർത്ഥം:

* `count` നിലവിലെ മൊത്തം സംഭരിക്കാൻ ഉപയോഗിക്കുന്നു
* `for` ലൂപ് `1` മുതൽ `50` വരെ ഉള്ള പൂർണ്ണസംഖ്യകൾ ഒന്നൊന്നായി എടുക്കും
* ഓരോ തവണയും ഒരു സംഖ്യ എടുത്താൽ, `count += i` നിർവഹിക്കും
* ലൂപ്പ് അവസാനിക്കുമ്പോൾ, `count` ൽ അന്തിമ മൊത്തം ഉണ്ടായിരിക്കും

ഇങ്ങനെ നമുക്ക് `50` വരെയുള്ള എല്ലാ ധന പൂർണ്ണസംഖ്യകളുടെയും മൊത്തം ലഭിക്കും.

അതുപോലെ, `1` മുതൽ `100` വരെ ഉള്ള സംഖ്യകളുടെ മൊത്തം കണക്കാക്കണമെങ്കിൽ, റേഞ്ച് മാത്രം മാറ്റിയാൽ മതി:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` ലൂപ് അറേകൾ മാത്രമല്ല, ഒരു റേഞ്ചും ലൂപ് ചെയ്യാൻ കഴിയും. ഡാറ്റ ക്രമത്തിൽ ആവർത്തിച്ച് പ്രോസസ് ചെയ്യാനുള്ള ഈ കഴിവ് വികസനത്തിൽ വളരെ സാധാരണമാണ്.

ഓരോ സംഖ്യയും കൈകൊണ്ട് ചേർക്കുന്നതുപോലുള്ള, അല്ലെങ്കിൽ മറ്റ് സമാനമായ ആവർത്തനപ്രവർത്തനങ്ങൾ എഴുതാതെ തന്നെ, നിരവധി ആവർത്തന ജോലികൾ സ്വയം നിർവഹിക്കാൻ ഇത് സഹായിക്കുന്നു.

## SwiftUI യിലെ ലൂപ്പ്

മുന്‍പ് കണ്ട `for` ലൂപ് Swift ഭാഷയുടേതാണ്.

എന്നാൽ SwiftUI യിൽ, view ഘടനയ്ക്കുള്ളിൽ നമുക്ക് `for` നേരിട്ട് ഉപയോഗിച്ച് view സൃഷ്ടിക്കാൻ കഴിയില്ല:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

കാരണം `VStack` നുള്ളിൽ വേണ്ടത് SwiftUI views ആണ്; സാധാരണ `for` ലൂപ് സ്വയം ഒരു view അല്ല.

SwiftUI യിൽ, അറേയുടെ ഉള്ളടക്കത്തെ അടിസ്ഥാനമാക്കി ഇന്റർഫേസ് ആവർത്തിച്ച് സൃഷ്ടിക്കണമെങ്കിൽ, `ForEach` ഉപയോഗിക്കണം.

ഉദാഹരണത്തിന്, ഒരു ഗ്രൂപ്പിലെ എല്ലാ അംഗങ്ങളുടെയും പേര് പ്രദർശിപ്പിക്കണമെങ്കിൽ, ഇങ്ങനെ എഴുതാം:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

പ്രദർശന ഫലം:

![for1](../../Resource/017_for1.png)

ഇപ്പോൾ ഓരോ `Text` നും ഒരേ modifiers ചേർക്കാം:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ഇപ്പോൾ ഓരോ `Text` ഉം ചെറുതും കട്ടിയുമുള്ള ഫോണ്ടിൽ പ്രദർശിപ്പിക്കും.

പ്രദർശന ഫലം:

![](../../Resource/017_for2.png)

modifiers ഇന്റർഫേസ് കൂടുതൽ മനോഹരമാക്കുന്നുവെങ്കിലും, കോഡ് കൂടുതൽ നീളമുള്ളതും ആവർത്തനപരവുമായിത്തീരുന്നു.

പേരുകൾ കൂടുതൽ കൂടിയാലോ, പിന്നീട് ഫോണ്ട്, നിറം തുടങ്ങിയ ശൈലികൾ ഒറ്റത്തവണ മാറ്റണമെന്നുണ്ടായാലോ, നിരവധി സമാനമായ കോഡുകൾ ആവർത്തിച്ച് എഴുതുകയും തിരുത്തുകയും വേണം. അതുകൊണ്ട് ഇത് ബുദ്ധിമുട്ടുള്ളതും പരിപാലിക്കാൻ അനുകൂലമല്ലാത്തതുമാണ്.

അപ്പോൾ, ആദ്യം ഈ പേരുകൾ അറേയിൽ വെച്ച്, SwiftUI യിലെ `ForEach` ഉപയോഗിച്ച് അറേയിലെ ഡാറ്റ അനുസരിച്ച് view കളെ ആവർത്തിച്ച് സൃഷ്ടിക്കാം:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

ഈ കോഡ് കൂടുതൽ ലളിതമാണ്. `ForEach` അറേയിലെ ഓരോ ഘടകവും ക്രമത്തിൽ വായിച്ച് അതനുസരിച്ചുള്ള ഉള്ളടക്കം ഇന്റർഫേസിൽ പ്രദർശിപ്പിക്കും.

പിന്നീട് പുതിയ പേരുകൾ ചേർക്കണമെങ്കിൽ, `names` അറേയിലെ ഉള്ളടക്കം മാത്രം മാറ്റിയാൽ മതി; ഓരോ തവണയും `Text` ഉം സങ്കീർണ്ണമായ modifiers ഉം വീണ്ടും ചേർക്കേണ്ടതില്ല.

## ForEach

`ForEach` എന്നത് SwiftUI യിൽ view കളെ ആവർത്തിച്ച് സൃഷ്ടിക്കാൻ ഉപയോഗിക്കുന്ന ഒരു ഘടനയാണ്. ഇത് SwiftUI യിലെ `for` ലൂപിനോട് സാമ്യമുള്ള ഘടനയായി മനസ്സിലാക്കാം.

അടിസ്ഥാന ഉപയോഗം:

```swift
ForEach(അറേ, id: \.self) { ഘടകം in
    SwiftUI കോഡ്
}
```

ഉദാഹരണത്തിന്, ഇപ്പോൾ കണ്ട ഗ്രൂപ്പിലെ പേരുകൾ:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ഈ കോഡ് സൂചിപ്പിക്കുന്നത്:

`names` ആണ് ലൂപ്പ് ചെയ്യുന്ന അറേ, അഥവാ പ്രദർശിപ്പിക്കേണ്ട ഡാറ്റ.

`id:\.self` എന്നത് അറേയിലുള്ള ഘടകത്തെ തന്നെയാണ് ഓരോ ഇനവും തിരിച്ചറിയാൻ ഉപയോഗിക്കുന്നത്.

ഇപ്പോഴത്തെ ഉദാഹരണത്തിൽ, അറേയിൽ strings ആണ് ഉള്ളത്:

```swift
["Sam", "John", "Wathon", "Bob"]
```

അതിനാൽ SwiftUI നേരിട്ട് "Sam"、"John"、"Wathon"、"Bob" എന്നീ string കളെ വ്യത്യസ്ത ഉള്ളടക്കങ്ങൾ തിരിച്ചറിയാൻ ഉപയോഗിക്കും.

ആരംഭ ഘട്ടത്തിൽ സാധാരണ കാണുന്ന String, Int പോലുള്ള ലളിതമായ അറേകൾക്കായി, നാം സാധാരണ ഇങ്ങനെ എഴുതാം:

```swift
id: \.self
```

ഇപ്പോൾ ഈ ഒരു എഴുതുന്ന രീതി ഓർക്കുന്നത് മതി; കൂടുതൽ `id` നിയമങ്ങൾ ആഴത്തിൽ പഠിക്കേണ്ടതില്ല.

`item in` ലുള്ള `item` നിലവിൽ എടുത്ത ഘടകത്തെ സൂചിപ്പിക്കുന്നു.

ഉദാഹരണത്തിന്, ആദ്യ ലൂപ്പിൽ `item` `"Sam"` ആയിരിക്കും; രണ്ടാമത്തെ ലൂപ്പിൽ `item` `"John"` ആയിരിക്കും.

അതിനാൽ:

```swift
Text(item)
```

ഇത് ക്രമത്തിൽ ഇങ്ങനെ മാറും:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

അതായത്, ഈ കോഡിന്റെ പ്രവർത്തനം: `names` അറേയിൽ നിന്ന് ഓരോ പേരും ഒന്നൊന്നായി എടുത്ത്, ഓരോ പേരിനും ഒരു `Text` view സൃഷ്ടിക്കുക.

### പ്രവർത്തനക്രമം

പ്രവർത്തനക്രമം ഇങ്ങനെ മനസ്സിലാക്കാം:

* `ForEach` ആദ്യം `names` അറേയിലെ ആദ്യ ഘടകമായ `"Sam"` വായിക്കും
* `item` താൽക്കാലികമായി `"Sam"` നെ സൂചിപ്പിക്കും
* `Text(item)` `"Sam"` പ്രദർശിപ്പിക്കും
* തുടർന്ന് അടുത്ത ഘടകമായ `"John"` വായിക്കും
* പിന്നെയും അതേ view കോഡ് ആവർത്തിച്ച് നിർവഹിക്കും

അവസാനം, അറേയിലെ എല്ലാ പേരുകളും പ്രദർശിപ്പിക്കും.

ഇങ്ങനെ എഴുതുന്നതിന്റെ ഗുണം: പേരുകൾ കൂടുതലായാലും, നിരവധി `Text` വരികൾ ആവർത്തിച്ച് എഴുതേണ്ടതില്ല; അറേയിലെ ഉള്ളടക്കം മാത്രം മാറ്റിയാൽ മതി.

### ഫോട്ടോകൾ ലൂപ്പ് ചെയ്യുക

`ForEach` ടെക്സ്റ്റ് മാത്രം പ്രദർശിപ്പിക്കുന്നതല്ല, ചിത്രങ്ങൾ, നിറങ്ങൾ, അല്ലെങ്കിൽ മറ്റു SwiftUI views എന്നിവയും പ്രദർശിപ്പിക്കാം.

ഉദാഹരണത്തിന്, നമുക്ക് 4 ചിത്രങ്ങൾ തയ്യാറാക്കിയതായി കരുതാം:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

ഈ നാല് ചിത്രങ്ങളും Xcode ലെയുള്ള Assets ഫോൾഡറിലേക്ക് drag ചെയ്യുക.

![image](../../Resource/017_for3.png)

`ForEach` ഉപയോഗിക്കാതെ ഇരുന്നാൽ, ഓരോ ചിത്രവും കൈകൊണ്ട് എഴുതണം:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

പ്രദർശന ഫലം:

![image](../../Resource/017_for4.png)

ഈ രീതിയിൽ ഫീച്ചർ സാധ്യമാകുമെങ്കിലും, കോഡ് താരതമ്യേന കൂടുതൽ നീളമുള്ളതായിരിക്കും.

ചിത്രങ്ങൾ കൂടുതൽ കൂടുന്നതോറും, കൂടുതൽ `Image` കോഡ് ആവർത്തിച്ച് എഴുതേണ്ടിവരും.

പിന്നീട് എല്ലാ ചിത്രങ്ങളുടെയും വലുപ്പം ഒറ്റത്തവണ മാറ്റണമെങ്കിലും, ഓരോന്നായി തിരുത്തേണ്ടിവരും.

അപ്പോൾ, ചിത്രങ്ങളുടെ പേരുകൾ അറേയിൽ വെച്ച്, തുടർന്ന് `ForEach` ഉപയോഗിച്ച് image views ആവർത്തിച്ച് സൃഷ്ടിക്കാം:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

ഇവിടെ `images` ഒരു string അറേയാണ്, അറേയിലെ ഓരോ ഘടകവും ഒരു ചിത്രത്തിന്റെ പേരാണ്.

`ForEach(images, id: \.self)` എന്നത് `images` അറേയിലെ ഓരോ ഘടകവും ഒന്നൊന്നായി വായിച്ച്, ഘടകത്തെ തന്നെയാണ് ഓരോ ഇനവും തിരിച്ചറിയാൻ ഉപയോഗിക്കുന്നത്. ശേഷം നിലവിലെ ഘടകത്തെ അടിസ്ഥാനമാക്കി അനുയോജ്യമായ view സൃഷ്ടിക്കും.

ഉദാഹരണത്തിന്, ആദ്യമായി വായിച്ചതാണ് `"430F9BEF"` എങ്കിൽ:

```swift
Image(item)
```

അതായത്:

```swift
Image("430F9BEF")
```

മറ്റ് ചിത്രങ്ങളും ഇതേ രീതിയിൽ ഒന്നൊന്നായി പ്രദർശിപ്പിക്കും.

![image](../../Resource/017_for4.png)

ഇവിടെയും നാം ഇപ്പോഴും ഉപയോഗിക്കുന്നത്:

```swift
id: \.self
```

കാരണം മുമ്പത്തെ പോലെ തന്നെയാണ്. `images` അറേയിലെ ഘടകങ്ങളും ലളിതമായ string type ആണല്ലോ. അതിനാൽ ആരംഭ ഘട്ടത്തിൽ ഘടകത്തെ തന്നെ തിരിച്ചറിയൽ രീതിയായി ഉപയോഗിക്കാം.

ഇത് ഒരു സ്ഥിരസൂത്രവാക്യമായി ലളിതമായി മനസ്സിലാക്കാം:

**`ForEach` ലൂപ്പ് ചെയ്യുന്നത് `String` അല്ലെങ്കിൽ `Int` പോലുള്ള ലളിതമായ അറേകളാണെങ്കിൽ, സാധാരണ `id: \.self` എന്ന് എഴുതും.**

### പുതിയ ചിത്രങ്ങൾ ചേർക്കുക

പിന്നീട് കൂടുതൽ ചിത്രങ്ങൾ ചേർക്കണമെങ്കിൽ, അറേയിലെ ഉള്ളടക്കം മാത്രം തിരുത്തിയാൽ മതി:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

പുതിയ `Image` കോഡ് വീണ്ടും ആവർത്തിച്ച് എഴുതേണ്ടതില്ല.

ചിത്രങ്ങൾ കൂടുതലാണെങ്കിൽ, `ScrollView` നൊപ്പം ഉപയോഗിക്കാം. അല്ലെങ്കിൽ ഉള്ളടക്കം സ്ക്രീനിന് പുറത്തേക്ക് കടന്നുപോകും; അപ്പോൾ മുഴുവനായി preview ചെയ്യാൻ കഴിയില്ല.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### ചിത്രങ്ങൾ തിരുത്തുക

എല്ലാ ചിത്രങ്ങളുടെയും ശൈലി ഒരുമിച്ച് മാറ്റണമെങ്കിൽ, `ForEach` നുള്ളിലെ കോഡ് മാത്രം മാറ്റിയാൽ മതി.

ഉദാഹരണത്തിന്, വീതി `80` ആക്കുക:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

ഇങ്ങനെ എല്ലാ ചിത്രങ്ങളും ഒരുമിച്ച് മാറും; ഓരോന്നായി തിരുത്തേണ്ടതില്ല.

ഇതാണ് `ForEach` ന്റെ ഒരു പ്രധാന പങ്ക്:

**പല views ന്റെയും ഘടന ഒരുപോലെയായിരിക്കുമ്പോൾ, ഉള്ളടക്കം മാത്രം വ്യത്യസ്തമാണെങ്കിൽ, ഒരു view കോഡ് മാത്രം എഴുതിയാൽ മതി; തുടർന്ന് അറേയിലെ ഡാറ്റ എന്ത് പ്രദർശിപ്പിക്കണമെന്ന് തീരുമാനിക്കും.**

## സംഗ്രഹം

ഈ പാഠത്തിൽ, `for` ലൂപും `ForEach` ഉം നാം പഠിച്ചു.

`for` ലൂപ് അറേകളിലോ റേഞ്ചുകളിലോ ഉള്ള ഘടകങ്ങളെ ക്രമത്തിൽ പ്രോസസ് ചെയ്യാൻ ഉപയോഗിക്കുന്നു; ഇത് കൂടുതൽ ഡാറ്റ പ്രോസസ്സിംഗിനെയാണ് ലക്ഷ്യമിടുന്നത്.

`ForEach` SwiftUI യിൽ അറേയിലെ ഉള്ളടക്കത്തെ അടിസ്ഥാനമാക്കി view കളെ ആവർത്തിച്ച് സൃഷ്ടിക്കാൻ ഉപയോഗിക്കുന്നു; ഇത് കൂടുതൽ view പ്രദർശനത്തിനെയാണ് ലക്ഷ്യമിടുന്നത്.

ഇപ്പോൾ ആരംഭ ഘട്ടത്തിൽ, `ForEach` ന്റെ ഏറ്റവും സാധാരണമായ എഴുതുന്ന രീതി ഇങ്ങനെയാണ്:

```swift
ForEach(അറേ, id: \.self) { ഘടകം in
    SwiftUI കോഡ്
}
```

ഇവിടെ `id: \.self` എന്നത് ഘടകത്തെ തന്നെയാണ് ഓരോ ഇനവും തിരിച്ചറിയാൻ ഉപയോഗിക്കുന്നത് എന്നർത്ഥം.

അറേയിലുള്ള ഉള്ളടക്കം `String`、`Int` പോലുള്ള ലളിതമായ തരങ്ങളാണെങ്കിൽ, നാം സാധാരണ ആദ്യം ഈ ഒരു എഴുതുന്ന രീതി തന്നെ ഉപയോഗിക്കും.

ഇനി വെബ്‌പേജുകളോ Appകളോ ബ്രൗസ് ചെയ്യുമ്പോൾ, ആവർത്തിച്ച് വരുന്ന ടെക്സ്റ്റുകൾ, ചിത്രങ്ങൾ, അല്ലെങ്കിൽ മറ്റ് ലിസ്റ്റ് ഉള്ളടക്കങ്ങൾ കണ്ടാൽ, SwiftUI യിൽ ഇത്തരം ഉള്ളടക്കങ്ങൾ സാധാരണ ആദ്യം അറേയിൽ വെച്ച്, തുടർന്ന് `ForEach` വഴി ഒന്നൊന്നായി പ്രദർശിപ്പിക്കുമെന്ന് ഓർക്കാം.
