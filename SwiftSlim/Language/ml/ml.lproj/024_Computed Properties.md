# കണക്കാക്കുന്ന പ്രോപ്പർട്ടി

ഈ പാഠത്തിൽ പ്രധാനമായി കണക്കാക്കുന്ന പ്രോപ്പർട്ടി പഠിക്കാം.

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി നിലവിലുള്ള ഡാറ്റയെ അടിസ്ഥാനമാക്കി പുതിയൊരു ഫലം കണക്കാക്കാൻ ഉപയോഗിക്കുന്നു. ഇത് സംഖ്യാ മൂല്യങ്ങൾ കണക്കാക്കാനും SwiftUI വ്യൂയിൽ കാണിക്കേണ്ട ഉള്ളടക്കം കണക്കാക്കാനും കഴിയും.

ഉദാഹരണത്തിന്:

```swift
let a = 10
let b = 20
let c = a + b
```

ഇവിടെ `c` എന്നത് `a`യും `b`യും ചേർത്ത ശേഷമുള്ള ഫലമാണ്.

സാധാരണ കോഡിൽ ഇത്തരം എഴുത്ത് വളരെ സാധാരണമാണ്.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

സൂചന: `Button` ക്ലിക്ക് ഇവന്റിനുള്ളിൽ സാധാരണ Swift കോഡ് പ്രവർത്തിപ്പിക്കാം.

എന്നാൽ സമാനമായ കോഡ് SwiftUI വ്യൂയുടെ പ്രോപ്പർട്ടികളിൽ നേരിട്ട് എഴുതിയാൽ പ്രശ്നം ഉണ്ടാകും.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

ഈ കോഡ് പിശക് കാണിക്കും.

കാണുമ്പോൾ `a`യും `b`യും `c`-യ്ക്ക് മുമ്പ് എഴുതിയിട്ടുണ്ട്, അതിനാൽ `c`-യുടെ മൂല്യം കണക്കാക്കാമെന്ന് തോന്നാം.

പക്ഷേ struct-ന്റെ പ്രോപ്പർട്ടികൾ പ്രഖ്യാപിക്കുമ്പോൾ, ഈ രീതിയിൽ നേരിട്ട് എഴുതാൻ കഴിയില്ല.

## എന്തുകൊണ്ട് c നേരിട്ട് a + b ആയി കണക്കാക്കാൻ കഴിയില്ല?

കാരണം `a`, `b`, `c` എന്നിവ ബട്ടൺ ക്ലിക്ക് ഇവന്റിനുള്ളിലെ താൽക്കാലിക constants അല്ല. അവ `ContentView` എന്ന വ്യൂവിലെ പ്രോപ്പർട്ടികളാണ്.

ബട്ടൺ ക്ലിക്ക് ഇവന്റിനുള്ളിൽ ഈ കോഡ് സാധാരണയായി പ്രവർത്തിക്കും:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

കാരണം ബട്ടൺ ക്ലിക്ക് ചെയ്ത ശേഷം കോഡ് ക്രമമായി പ്രവർത്തിക്കുന്നു. ആദ്യം `a` സൃഷ്ടിക്കുന്നു, തുടർന്ന് `b` സൃഷ്ടിക്കുന്നു, അവസാനം `a + b` ഉപയോഗിച്ച് `c` കണക്കാക്കുന്നു.

എന്നാൽ വ്യൂവിനുള്ളിൽ പ്രോപ്പർട്ടികൾ പ്രഖ്യാപിക്കുമ്പോൾ സാഹചര്യം വ്യത്യസ്തമാണ്:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

ഇവിടെ `a`, `b`, `c` എല്ലാം `ContentView` എന്ന വ്യൂവിലെ പ്രോപ്പർട്ടികളാണ്.

`ContentView` വ്യൂ സൃഷ്ടിക്കുമ്പോൾ Swift ആദ്യം ഈ പ്രോപ്പർട്ടികൾ തയ്യാറാക്കണം. സൃഷ്ടിക്കൽ പ്രക്രിയ സുരക്ഷിതമാക്കാൻ, ഒരു instance stored property-യുടെ default value-ൽ അതേ instance-ലെ മറ്റൊരു instance property നേരിട്ട് വായിക്കാൻ Swift അനുവദിക്കുന്നില്ല.

അതിനാൽ ഈ വരി പിശക് കാണിക്കും:

```swift
let c = a + b
```

ലളിതമായി മനസ്സിലാക്കാം: **വ്യൂവിനുള്ളിൽ പ്രോപ്പർട്ടികൾ പ്രഖ്യാപിക്കുമ്പോൾ, ഒരു സാധാരണ പ്രോപ്പർട്ടി ഉപയോഗിച്ച് മറ്റൊരു സാധാരണ പ്രോപ്പർട്ടി നേരിട്ട് കണക്കാക്കാൻ കഴിയില്ല.**

മൂല്യം നേരിട്ട് സംഭരിക്കുന്ന ഇത്തരത്തിലുള്ള പ്രോപ്പർട്ടിയെ “stored property” എന്ന് വിളിക്കുന്നു. എളുപ്പത്തിൽ മനസ്സിലാക്കാൻ, താൽക്കാലികമായി ഇത് സാധാരണ പ്രോപ്പർട്ടി എന്ന് കരുതാം.

ഉദാഹരണത്തിന്:

```swift
let a = 10
```

`a` `10` സംഭരിക്കുന്നു.

```swift
let b = 20
```

`b` `20` സംഭരിക്കുന്നു.

പക്ഷേ:

```swift
let c = a + b
```

ഇവിടെ `c` നേരിട്ട് എഴുതിയ സ്ഥിരമായ മൂല്യമല്ല. അത് `a + b` വഴി കണക്കാക്കപ്പെടാൻ ഉദ്ദേശിക്കുന്നു.

ഇത്തരത്തിലുള്ള “നിലവിലുള്ള ഡാറ്റയിൽ നിന്ന് ഫലം നേടുന്ന” സാഹചര്യത്തിന് കണക്കാക്കുന്ന പ്രോപ്പർട്ടി കൂടുതൽ അനുയോജ്യമാണ്.

ഇത് ഇങ്ങനെ മാറ്റാം:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

ഇവിടെ `c` ആണ് കണക്കാക്കുന്ന പ്രോപ്പർട്ടി.

```swift
var c: Int {
    return a + b
}
```

ഇതിന്റെ അർത്ഥം: **`c` ഉപയോഗിക്കേണ്ടപ്പോൾ മാത്രമേ `a`യും `b`യും വായിച്ച് `a + b` ഫലം തിരികെ നൽകൂ.**

ഉദാഹരണത്തിന്:

```swift
Text("c: \(c)")
```

`Text` `c` കാണിക്കുമ്പോഴാണ് `c`-യുടെ കണക്കുകൂട്ടൽ നടക്കുന്നത്.

## കണക്കാക്കുന്ന പ്രോപ്പർട്ടി എന്താണ്?

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ഒരു variable പോലെ തോന്നും, പക്ഷേ അത് സ്വയം ഡാറ്റ സംഭരിക്കില്ല.

ഉദാഹരണത്തിന്:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

ഇവിടെ `c` ആണ് കണക്കാക്കുന്ന പ്രോപ്പർട്ടി.

ഇത് സാധാരണ പ്രോപ്പർട്ടിപോലെ ഒരു സ്ഥിരമായ മൂല്യം സംഭരിക്കില്ല. ഓരോ തവണയും `c` വായിക്കുമ്പോൾ `{}` ഉള്ളിലെ കോഡ് വീണ്ടും പ്രവർത്തിക്കുകയും കണക്കാക്കിയ ഫലം തിരികെ നൽകുകയും ചെയ്യും.

ഇത് ഇങ്ങനെ മനസ്സിലാക്കാം:

```swift
var c: Int {
    return a + b
}
```

`c` ആവശ്യമുള്ളപ്പോൾ `a + b` കണക്കാക്കുന്നു.

അതുകൊണ്ട് കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ഈ സാഹചര്യത്തിന് അനുയോജ്യമാണ്: **ഒരു ഫലം പ്രത്യേകം സംഭരിക്കേണ്ടതില്ല, അത് നിലവിലുള്ള ഡാറ്റയിൽ നിന്ന് കണക്കാക്കാനാകുന്നു.**

## അടിസ്ഥാന എഴുത്ത് രീതി

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി സാധാരണയായി മൂന്ന് ഭാഗങ്ങൾ ഉൾക്കൊള്ളുന്നു:

```swift
var c: Int {
    return a + b
}
```

### 1. `var` ഉപയോഗിച്ച് പ്രഖ്യാപിക്കുക

```swift
var c
```

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി നിർബന്ധമായും `var` ഉപയോഗിച്ച് പ്രഖ്യാപിക്കണം. `let` ഉപയോഗിക്കാൻ കഴിയില്ല.

കാരണം കണക്കാക്കുന്ന പ്രോപ്പർട്ടി സ്ഥിരമായി സംഭരിച്ചിരിക്കുന്ന മൂല്യമല്ല. ഓരോ തവണയും വായിക്കുമ്പോൾ dynamic ആയി കണക്കാക്കപ്പെടുന്ന ഫലമാണ്.

### 2. return type സൂചിപ്പിക്കുക

```swift
var c: Int
```

കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്ക് return type സൂചിപ്പിക്കണം.

ഇവിടെ `c` അവസാനം ഒരു integer തിരികെ നൽകും, അതിനാൽ type `Int` ആണ്.

### 3. {} ഉള്ളിൽ കണക്കാക്കുന്ന logic എഴുതുക

```swift
{
    return a + b
}
```

`{}` ഉള്ളിൽ കണക്കാക്കുന്ന logic എഴുതുന്നു. ഇവിടെ തിരികെ നൽകുന്നത് `a + b` ആണ്.

## return കീവേഡ്

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ഒരു ഫലം തിരികെ നൽകണം.

ഉദാഹരണത്തിന്:

```swift
var c: Int {
    return a + b
}
```

ഇവിടെ `return` എന്നത് `a + b` കണക്കാക്കിയ ഫലം പുറത്തേക്ക് തിരികെ നൽകുക എന്നർത്ഥമാണ്.

കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ നേരിട്ട് ഫലം നൽകുന്ന ഒരു expression മാത്രം ഉണ്ടെങ്കിൽ, `return` ഒഴിവാക്കാം:

```swift
var c: Int {
    a + b
}
```

പക്ഷേ കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ പല വരി കോഡ് ഉണ്ടെങ്കിൽ, ഫലം വ്യക്തമായി തിരികെ നൽകാൻ `return` ഉപയോഗിക്കണം.

ഉദാഹരണത്തിന്:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

ഇവിടെ കണക്കുകൂട്ടൽ രണ്ട് ഘട്ടങ്ങളായി വിഭജിച്ചിരിക്കുന്നു.

ആദ്യ ഘട്ടം, മൊത്തം വില കണക്കാക്കുന്നു:

```swift
let total = count * price
```

രണ്ടാം ഘട്ടം, മൊത്തം വിലയെ ടെക്സ്റ്റാക്കി ചേർത്ത് തിരികെ നൽകുന്നു:

```swift
return "Total：\(total) $"
```

`return` നീക്കിയാൽ:

```swift
var totalPriceText: String {
    let total = count * price
    "Total：\(total) $"
}
```

ഈ കോഡ് പിശക് കാണിക്കും.

കാരണം: **ഈ കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ ഇതിനകം പല വരി കോഡ് ഉണ്ട്. ഏത് വരിയാണ് അവസാനം തിരികെ നൽകേണ്ട ഫലം എന്ന് Swift സ്വയം തീരുമാനിക്കാൻ കഴിയില്ല.**

അതിനാൽ, കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ ഫലം നൽകുന്ന ഒരു expression മാത്രം ഉണ്ടെങ്കിൽ `return` ഒഴിവാക്കാം.

```swift
var c: Int {
    a + b
}
```

കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ പല വരി കോഡ് ഉണ്ടെങ്കിൽ, `return` വ്യക്തമായി എഴുതുന്നത് നല്ലതാണ്.

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

## കണക്കാക്കുന്ന പ്രോപ്പർട്ടിയും സാധാരണ പ്രോപ്പർട്ടിയും തമ്മിലുള്ള വ്യത്യാസം

സാധാരണ പ്രോപ്പർട്ടി ഡാറ്റ സംഭരിക്കുന്നു.

```swift
var c = 30
```

ഇവിടെ `c` ഒരു നിർദ്ദിഷ്ട മൂല്യം സംഭരിക്കുന്നു: `30`.

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ഡാറ്റ സംഭരിക്കില്ല.

```swift
var c: Int {
    a + b
}
```

ഇവിടെ `c` `30` സംഭരിക്കുന്നില്ല. അത് വെറും കണക്കാക്കുന്ന രീതിയാണ് നൽകുന്നത്.

`c` വായിക്കുമ്പോൾ Swift ഇത് പ്രവർത്തിപ്പിക്കും:

```swift
a + b
```

ശേഷം കണക്കാക്കിയ ഫലം തിരികെ നൽകും.

അതുകൊണ്ട് മറ്റുള്ള ഡാറ്റയുടെ അടിസ്ഥാനത്തിൽ ഫലം കണക്കാക്കേണ്ട സാഹചര്യങ്ങളിൽ കണക്കാക്കുന്ന പ്രോപ്പർട്ടി അനുയോജ്യമാണ്.

## body-യും കണക്കാക്കുന്ന പ്രോപ്പർട്ടിയാണ്

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി പഠിച്ചതിന് ശേഷം, SwiftUI-യിൽ ഏറ്റവും പതിവായി കാണുന്ന ഈ കോഡ് വീണ്ടും മനസ്സിലാക്കാം:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

ഇവിടെ `body`-യും ഒരു കണക്കാക്കുന്ന പ്രോപ്പർട്ടിയാണ്.

ഇത് `var` ഉപയോഗിച്ച് പ്രഖ്യാപിച്ചിരിക്കുന്നു:

```swift
var body
```

ഇതിന് return type ഉണ്ട്:

```swift
some View
```

ഇതിന്റെ `{}` ഉള്ളിൽ തിരികെ നൽകുന്നത് SwiftUI വ്യൂ ഉള്ളടക്കമാണ്:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

അതിനാൽ `body` ഇങ്ങനെ മനസ്സിലാക്കാം: SwiftUI ഈ വ്യൂ കാണിക്കേണ്ടപ്പോൾ `body` വായിക്കുന്നു, പിന്നെ `body` തിരികെ നൽകുന്ന ഉള്ളടക്കത്തെ അടിസ്ഥാനമാക്കി interface സൃഷ്ടിക്കുന്നു.

`return` എഴുതിയാൽ, ഇങ്ങനെ മനസ്സിലാക്കാം:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

എന്നാൽ SwiftUI-യിൽ സാധാരണയായി `return` ഒഴിവാക്കി നേരിട്ട് ഇങ്ങനെ എഴുതുന്നു:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

`@State` ഡാറ്റ മാറുമ്പോൾ SwiftUI `body` വീണ്ടും വായിച്ച് പുതിയ ഡാറ്റ അനുസരിച്ച് interface അപ്‌ഡേറ്റ് ചെയ്യും.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("count: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

ബട്ടൺ ക്ലിക്ക് ചെയ്ത ശേഷം `count` മാറും.

```swift
count += 1
```

`count` മാറിയാൽ SwiftUI `body` വീണ്ടും കണക്കാക്കും. അതിനാൽ interface-ലുള്ള ടെക്സ്റ്റും അപ്‌ഡേറ്റ് ചെയ്യും.

```swift
Text("count: \(count)")
```

SwiftUI-യിൽ ഡാറ്റ മാറിയ ശേഷം interface സ്വയം refresh ആകുന്നതിന് ഇതും ഒരു കാരണം ആണ്.

### body ഉള്ളിൽ സങ്കീർണ്ണമായ കണക്കുകൂട്ടൽ എഴുതുന്നത് ശുപാർശ ചെയ്യുന്നതല്ല

`body` കണക്കാക്കുന്ന പ്രോപ്പർട്ടിയായതിനാൽ, അത് പലതവണ വായിക്കപ്പെടുകയും വീണ്ടും കണക്കാക്കപ്പെടുകയും ചെയ്യാം.

ഉദാഹരണത്തിന്:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("num: \(num)")
            Text("name: \(name)")
        }
    }
}
```

`name` മാറുമ്പോൾ SwiftUI `body` വീണ്ടും കണക്കാക്കും.

അപ്പോൾ `body` ഉള്ളിലെ കോഡും വീണ്ടും പ്രവർത്തിക്കും, `num`-വും വീണ്ടും സൃഷ്ടിക്കപ്പെടും.

```swift
let num = 10
```

ഈ ഉദാഹരണത്തിലെ `num` വളരെ ലളിതമാണ്, അതിനാൽ വലിയ ബാധകതയില്ല.

പക്ഷേ `body` ഉള്ളിൽ വലിയ ഡാറ്റ filtering, sorting, image processing പോലെയുള്ള സങ്കീർണ്ണമായ കണക്കുകൂട്ടലുകൾ നടത്തിയാൽ, interface-ന്റെ സുതാര്യമായ പ്രവർത്തനത്തെ ബാധിക്കാം.

അതിനാൽ SwiftUI-യിൽ `body` പ്രധാനമായും interface ഘടന വിവരിക്കുന്നതിൽ മാത്രം ശ്രദ്ധിക്കണം.

ലളിതമായ താൽക്കാലിക ഡാറ്റ `body` ഉള്ളിൽ എഴുതാം.

സങ്കീർണ്ണമായ കണക്കുകൂട്ടലുകൾ `body`-യ്ക്കു പുറത്തുള്ള കണക്കാക്കുന്ന പ്രോപ്പർട്ടി, method, അല്ലെങ്കിൽ പ്രത്യേക data processing ഭാഗത്തിലേക്ക് മാറ്റുന്നത് നല്ലതാണ്.

## ഉദാഹരണം: എണ്ണംയും മൊത്തം വിലയും

ഇപ്പോൾ ഒരു ലളിതമായ ഉദാഹരണത്തിലൂടെ കണക്കാക്കുന്ന പ്രോപ്പർട്ടി മനസ്സിലാക്കാം.

ഒരു carrot-ന്റെ വില 2 $ ആണെന്ന് കരുതുക. ഉപയോക്താവിന് ബട്ടൺ ക്ലിക്ക് ചെയ്ത് വാങ്ങുന്ന എണ്ണം മാറ്റാം, interface-ൽ മൊത്തം വില കാണിക്കണം.

മൊത്തം വില കണക്കാക്കുന്ന രീതി:

```text
എണ്ണം * യൂണിറ്റ് വില
```

മൊത്തം വില സാധാരണ variable-ൽ സംഭരിച്ചാൽ അത് അല്പം ബുദ്ധിമുട്ടായിരിക്കും.

കാരണം ഓരോ തവണയും എണ്ണം മാറുമ്പോൾ മൊത്തം വില കൈമാറി അപ്‌ഡേറ്റ് ചെയ്യണം.

മികച്ച മാർഗം കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ഉപയോഗിക്കുകയാണ്:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

കാണിക്കുന്ന ഫലം:

![view](../../Resource/024_view.png)

ഈ ഉദാഹരണത്തിൽ:

```swift
@State private var count = 1
```

`count` carrot-ന്റെ എണ്ണമാണ്. ബട്ടൺ ക്ലിക്ക് ചെയ്യുമ്പോൾ `count` മാറും.

```swift
private let price = 2
```

`price` carrot-ന്റെ യൂണിറ്റ് വിലയാണ്. ഇവിടെ ഇത് സ്ഥിരമായ മൂല്യമാണ്, അതിനാൽ `let` ഉപയോഗിക്കുന്നു.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` മൊത്തം വിലയാണ്.

ഇത് പ്രത്യേകം സംഭരിക്കേണ്ടതില്ല, കാരണം മൊത്തം വില എപ്പോഴും `count * price` വഴി കണക്കാക്കാം.

`count` `1` ആണെങ്കിൽ:

```swift
totalPrice = 1 * 2
```

കാണിക്കുന്ന ഫലം:

```swift
totalPrice: 2 $
```

`+` ബട്ടൺ ക്ലിക്ക് ചെയ്ത ശേഷം `count` `2` ആകും.

അപ്പോൾ `totalPrice` വീണ്ടും വായിക്കുമ്പോൾ, അത് വീണ്ടും കണക്കാക്കപ്പെടും:

```swift
totalPrice = 2 * 2
```

കാണിക്കുന്ന ഫലം:

```swift
totalPrice: 4 $
```

ഇതാണ് കണക്കാക്കുന്ന പ്രോപ്പർട്ടിയുടെ ജോലി: നിലവിലുള്ള ഡാറ്റയെ അടിസ്ഥാനമാക്കി പുതിയ ഫലം dynamic ആയി കണക്കാക്കുക.

## കണക്കാക്കുന്ന പ്രോപ്പർട്ടി判断ത്തിനും ഉപയോഗിക്കാം

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി സംഖ്യ മാത്രമല്ല കണക്കാക്കുന്നത്,判断 ഫലവും തിരികെ നൽകാം.

ഉദാഹരണത്തിന്, എണ്ണം ഏറ്റവും കുറഞ്ഞത് 1 ആയിരിക്കണമെന്ന് നാം ആഗ്രഹിക്കുന്നു.

എണ്ണം ഇതിനകം 1 ആണെങ്കിൽ, `-` ബട്ടൺ വീണ്ടും കുറയ്ക്കരുത്.

ഇതിനായി ഒരു കണക്കാക്കുന്ന പ്രോപ്പർട്ടി ചേർക്കാം:

```swift
private var canDecrease: Bool {
    count > 1
}
```

പൂർണ്ണ കോഡ്:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

കാണിക്കുന്ന ഫലം:

![view](../../Resource/024_view1.png)

ഇവിടെ:

```swift
private var canDecrease: Bool {
    count > 1
}
```

ഈ കോഡ് ഇപ്പോൾ കൂടി കുറയ്ക്കാനാകുമോ എന്ന് സൂചിപ്പിക്കുന്നു.

`count` `1`-നെക്കാൾ വലുതാണെങ്കിൽ:

```swift
canDecrease == true
```

അതാണ് കുറയ്ക്കാൻ കഴിയും എന്നർത്ഥം.

`count` `1`-ന് തുല്യമായാൽ:

```swift
canDecrease == false
```

അതാണ് ഇനി കുറയ്ക്കാൻ കഴിയില്ല എന്നർത്ഥം.

### ബട്ടണിലെ condition check

ബട്ടണിൽ ഇത് ഉപയോഗിക്കുന്നു:

```swift
if canDecrease {
    count -= 1
}
```

`canDecrease` `true` ആണെങ്കിൽ മാത്രമേ `count` കുറയ്ക്കൂ.

### വ്യൂയുടെ foreground color നിയന്ത്രിക്കുക

`canDecrease` ഉപയോഗിച്ച് ബട്ടണിന്റെ foreground color നിയന്ത്രിക്കാനും കഴിയും:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

ഇവിടെ ternary operator ഉപയോഗിക്കുന്നു:

```swift
canDecrease ? Color.primary : Color.gray
```

ഈ കോഡിന്റെ അർത്ഥം: `canDecrease` `true` ആണെങ്കിൽ foreground color `Color.primary` ഉപയോഗിക്കും; `canDecrease` `false` ആണെങ്കിൽ foreground color `Color.gray` ഉപയോഗിക്കും.

`Color.primary` SwiftUI നൽകുന്ന system semantic color ആണ്. ഇത് നിലവിലെ interface-ലെ പ്രധാന text color സൂചിപ്പിക്കുന്നു.

Light Mode-ൽ `Color.primary` സാധാരണയായി കറുപ്പിനോട് അടുത്തിരിക്കും; Dark Mode-ൽ സാധാരണയായി വെള്ളയോട് അടുത്തിരിക്കും.

അതിനാൽ `Color.primary` ഉപയോഗിക്കുന്നതിന്റെ ഗുണം: ഇത് Light Mode-നും Dark Mode-നും സ്വയം പൊരുത്തപ്പെടും.

### വ്യൂയുടെ disabled നില നിയന്ത്രിക്കുക

`disabled` വ്യൂ disabled നിലയിലാണോ എന്ന് നിയന്ത്രിക്കാൻ ഉപയോഗിക്കുന്നു:

```swift
.disabled(!canDecrease)
```

`disabled` `false` ആണെങ്കിൽ വ്യൂ ക്ലിക്ക് ചെയ്യാം.

`disabled` `true` ആണെങ്കിൽ വ്യൂ disabled നിലയിലാകും, ക്ലിക്ക് ചെയ്യാൻ കഴിയില്ല.

ഇവിടെ `canDecrease` condition ആയി ഉപയോഗിക്കുന്നതിനാൽ, കോഡ് മനസ്സിലാക്കാൻ എളുപ്പമാകും.

`canDecrease` കാണുമ്പോൾ തന്നെ, അത് “ഇപ്പോൾ കൂടി കുറയ്ക്കാനാകുമോ” എന്നർത്ഥമാണെന്ന് അറിയാം.

### അധിക വിശദീകരണം: എന്തുകൊണ്ട് രണ്ടുതവണ പരിശോധിക്കുന്നു?

`-` ബട്ടണിൽ:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

ഇവിടെ `.disabled(!canDecrease)` ഉപയോഗിച്ചിട്ടുണ്ട്, കൂടാതെ ബട്ടണിനുള്ളിൽ `if canDecrease` ഉപയോഗിച്ചിട്ടുണ്ട്.

`.disabled(!canDecrease)` interface-ൽ ബട്ടൺ disabled ആക്കി ഉപയോക്താവിന് അത് ക്ലിക്ക് ചെയ്യാൻ കഴിയാതാക്കുന്നു.

`if canDecrease` കോഡ് പ്രവർത്തിക്കുന്നതിന് മുമ്പ് വീണ്ടും പരിശോധിക്കുന്നു. കുറയ്ക്കാൻ കഴിയുന്ന സാഹചര്യത്തിൽ മാത്രമേ `count -= 1` പ്രവർത്തിക്കൂ.

ഇത് double protection ആണ്. യഥാർത്ഥ development-ൽ ബട്ടൺ ഇതിനകം disabled ആണെങ്കിൽ, ഉള്ളിലെ判断 ഒഴിവാക്കാം. പക്ഷേ teaching example-ൽ ഇത് നിലനിർത്തുന്നത് `canDecrease`-ന്റെ പങ്ക് കൂടുതൽ വ്യക്തമായി കാണിക്കുന്നു.

## സംഗ്രഹം

ഈ പാഠത്തിൽ പ്രധാനമായി കണക്കാക്കുന്ന പ്രോപ്പർട്ടി പഠിച്ചു.

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി മൂല്യം നേരിട്ട് സംഭരിക്കില്ല. അത് വായിക്കുമ്പോൾ നിലവിലുള്ള ഡാറ്റയെ അടിസ്ഥാനമാക്കി ഫലം കണക്കാക്കും.

ഉദാഹരണത്തിന്:

```swift
var c: Int {
    a + b
}
```

ഇവിടെ `c` പ്രത്യേകം സംഭരിക്കേണ്ടതില്ല, കാരണം അത് `a + b` വഴി കണക്കാക്കാം.

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി നിർബന്ധമായും `var` ഉപയോഗിച്ച് പ്രഖ്യാപിക്കണം, കൂടാതെ return type സൂചിപ്പിക്കണം.

```swift
var canDecrease: Bool {
    count > 1
}
```

കണക്കാക്കുന്ന പ്രോപ്പർട്ടി സംഖ്യാ മൂല്യം മാത്രമല്ല,判断 ഫലം, text content, പോലും SwiftUI view content തിരികെ നൽകാനും കഴിയും.

ഈ പാഠത്തിൽ നാം `return`-വും പഠിച്ചു.

`return` എന്നത് ഫലം തിരികെ നൽകുക എന്നർത്ഥമാണ്:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

കണക്കാക്കുന്ന പ്രോപ്പർട്ടിക്കുള്ളിൽ ഫലം നൽകുന്ന ഒരു expression മാത്രം ഉണ്ടെങ്കിൽ, `return` ഒഴിവാക്കാം.

```swift
var totalPrice: Int {
    count * price
}
```

കൂടാതെ നാം `Color.primary`യും `disabled`യും അറിഞ്ഞു.

`Color.primary` SwiftUI-യുടെ system semantic color ആണ്. ഇത് Light Mode-നും Dark Mode-നും അനുസരിച്ച് display effect സ്വയം ക്രമീകരിക്കും.

```swift
.foregroundStyle(Color.primary)
```

`disabled` വ്യൂ disabled ആണോ എന്ന് നിയന്ത്രിക്കാൻ ഉപയോഗിക്കുന്നു.

```swift
.disabled(true)
```

ഇത് disabled ആണ്, ക്ലിക്ക് ചെയ്യാൻ കഴിയില്ല എന്ന് സൂചിപ്പിക്കുന്നു.

```swift
.disabled(false)
```

ഇത് available ആണ്, ക്ലിക്ക് ചെയ്യാം എന്ന് സൂചിപ്പിക്കുന്നു.

അതുകൊണ്ട് കണക്കാക്കുന്ന പ്രോപ്പർട്ടി SwiftUI-യിൽ വളരെ സാധാരണമാണ്. ഇത് കണക്കാക്കിയ ഫലങ്ങൾ, conditions, display content എന്നിവ കൂടുതൽ വ്യക്തമായി ക്രമീകരിക്കാൻ സഹായിക്കുന്നു.
