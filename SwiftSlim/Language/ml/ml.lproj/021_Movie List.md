# മൂവി പട്ടിക

ഈ പാഠത്തിൽ നാം ഒരു മൂവി പട്ടിക തയ്യാറാക്കും.

![movie](../../Resource/021_movie.png)

പട്ടികയിൽ ഓരോ സിനിമയുടെയും പോസ്റ്റർ, പേര്, സംവിധായകൻ, റേറ്റിംഗ് എന്നിവ കാണിക്കും.

ഈ ഉദാഹരണത്തിൽ നാം വളരെ പ്രധാനപ്പെട്ട ഒരു ആശയം പഠിക്കും: `struct` ഘടന. ഒരു സിനിമയുടെ പല വിവരങ്ങളും ഒരൊറ്റ സമഗ്ര രൂപത്തിലേക്ക് കൂട്ടിക്കൊണ്ടുവരാൻ ഇത് സഹായിക്കുന്നു. ഇതിനൊപ്പം `UUID()`‌, `ForEach`, `Divider` വിഭജക രേഖ, കൂടാതെ കസ്റ്റം ഒബ്ജക്റ്റുകൾ ഉപയോഗിച്ച് ഡാറ്റ എങ്ങനെ കൈകാര്യം ചെയ്യാമെന്നും നാം കാണും.

ഈ അറിവുകൾ പിന്നീട് SwiftUI വികസനത്തിൽ വളരെ സാധാരണമാണ്. ഈ പാഠം പഠിച്ചു കഴിഞ്ഞാൽ, നിങ്ങൾക്ക് ഒരു മൂവി പട്ടിക ഉണ്ടാക്കാനാകുന്നതുപോലെ തന്നെ, “ഒരു കൂട്ടം ഡാറ്റയെ സ്ക്രീനിലെ ഇന്റർഫേസായി എങ്ങനെ കാണിക്കാം” എന്നും മനസ്സിലാക്കാൻ തുടങ്ങും.

## ഒറ്റ സിനിമ

ആദ്യം ഒരു സിനിമയുടെ ഇന്റർഫേസിൽ നിന്ന് തുടങ്ങാം.

![movie](../../Resource/021_movie1.png)

ഈ ഇന്റർഫേസിന്റെ ലേഔട്ട് പ്രധാനമായും രണ്ട് ഭാഗങ്ങളാണ്: ഇടത് വശത്ത് സിനിമയുടെ പോസ്റ്റർ, വലത് വശത്ത് സിനിമയുടെ പരിചയം.

### സിനിമാ പോസ്റ്റർ

ഇടത് വശത്ത് സിനിമാ പോസ്റ്റർ കാണിക്കുന്നു; ചിത്രം കാണിക്കാൻ നാം `Image` ഉപയോഗിക്കാം.

ഉദാഹരണത്തിന്:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

ഈ കോഡ് `"The Shawshank Redemption"` എന്ന പേരുള്ള ഒരു ചിത്രം പ്രദർശിപ്പിക്കുന്നു എന്നതാണ് അർത്ഥം.

ഇവിടെ ശ്രദ്ധിക്കേണ്ടത്, ചിത്രത്തിന്റെ പേര് `Assets` റിസോഴ്സ് ഫോൾഡറിൽ യഥാർത്ഥത്തിൽ ഉണ്ടായിരിക്കണം; ഇല്ലെങ്കിൽ ഇന്റർഫേസിൽ ഈ ചിത്രം ശരിയായി കാണിക്കാനാവില്ല.

![movie](../../Resource/021_movie2.png)

ഈ മോഡിഫയറുകളുടെ പ്രവർത്തനം ഇപ്രകാരം:

- `resizable()` ചിത്രം വലുപ്പം മാറ്റാൻ കഴിയും എന്ന് സൂചിപ്പിക്കുന്നു.
- `scaledToFit()` സ്കെയിൽ ചെയ്യുമ്പോൾ യഥാർത്ഥ അനുപാതം നിലനിർത്തുകയും ചിത്രം വലിച്ചു വിരൂപമാകുന്നത് ഒഴിവാക്കുകയും ചെയ്യുന്നു.
- `frame(height: 180)` ചിത്രം 180 എന്ന ഉയരത്തിൽ ക്രമീകരിക്കുന്നു.
- `cornerRadius(10)` ചിത്രം 10 റേഡിയസുള്ള വൃത്താകൃതിയിലുള്ള കോണുകൾ നൽകുന്നു.

ഇങ്ങനെ നമുക്ക് ശരിയായ വലുപ്പത്തിലും വൃത്താകൃതിയിലുള്ള കോണുകളോടുകൂടിയ ഒരു സിനിമാ പോസ്റ്റർ ചിത്രം ലഭിക്കും.

![movie](../../Resource/021_movie3.png)

### സിനിമയുടെ പരിചയം

വലത് വശത്ത് സിനിമയുടെ പേര്, സംവിധായകൻ, റേറ്റിംഗ് എന്നിവ ഉൾപ്പെടുന്ന സിനിമാ പരിചയം കാണിക്കുന്നു.

![movie](../../Resource/021_movie4.png)

ഈ ഭാഗം കാണിക്കാൻ നാം `Text` ഉപയോഗിക്കാം:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

ഈ കോഡിൽ `.font` ഫോണ്ടിന്റെ വലുപ്പവും `.fontWeight` ഫോണ്ടിന്റെ കട്ടിയും സൂചിപ്പിക്കുന്നു.

“സിനിമയുടെ പേര്”, “സംവിധായകന്റെ വിവരം”, “റേറ്റിംഗ് വിവരം” എന്നിവയ്ക്കിടയിലെ സ്പേസിംഗ് കൂടുതൽ വ്യക്തമായിരിക്കാനായി, ഇവിടെ ഏറ്റവും പുറത്തുള്ള ലെയറിൽ `VStack` ഉപയോഗിച്ച് ഇത് ക്രമീകരിക്കുന്നു:

```swift
spacing: 10
```

ഇത് ഈ `VStack` ഉള്ളിലെ ഓരോ ഗ്രൂപ്പുകൾക്കുമിടയിൽ 10 എന്ന അകലം ഉണ്ടാകുമെന്ന് സൂചിപ്പിക്കുന്നു.

സംവിധായകനും റേറ്റിംഗും ഓരോന്നും വേർതിരിച്ച് `VStack` കൊണ്ട് മൂടുന്നതിന്റെ കാരണം, അവ രണ്ടും “തലക്കെട്ട് + ഉള്ളടക്കം” എന്ന ഘടനയിലാണെന്നതാണ്. അതിനാൽ ഇന്റർഫേസിന്റെ തലങ്ങൾ കൂടുതൽ വ്യക്തമായിരിക്കും, പിന്നീടുള്ള സ്റ്റൈൽ മാറ്റങ്ങളും എളുപ്പമാകും.

കൂടാതെ `VStack` ഡീഫോൾട്ടായി മദ്ധ്യത്തിൽ അലൈൻ ചെയ്യും. മുഴുവൻ വാചകവും ഇടത്തോട്ട് അലൈൻ ചെയ്യാൻ നാം ഇങ്ങനെ ക്രമീകരിക്കുന്നു:

```swift
alignment: .leading
```

ഇതോടെ സിനിമയുടെ പരിചയഭാഗം കൂടുതൽ വൃത്തിയായി കാണപ്പെടുന്നു.

### വിഭജക രേഖ

ഇപ്പോൾ, സിനിമാ പരിചയഭാഗത്തിലെ ഉള്ളടക്കങ്ങൾ `spacing` കാരണം അകലം നേടിയെങ്കിലും, വ്യത്യസ്ത ഉള്ളടക്കങ്ങളിലെ അതിരുകൾ ഇനിയും വ്യക്തമായിട്ടില്ല.

ഈ സമയം നമുക്ക് ഒരു വിഭജക രേഖ ചേർക്കാം:

```swift
Divider()
```

ഉദാഹരണത്തിന്:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

ഇങ്ങനെ സിനിമയുടെ പരിചയഭാഗത്തിൽ വ്യക്തമായ വിഭജന പ്രഭാവം ലഭിക്കും.

![movie](../../Resource/021_movie4.png)

`Divider` വളരെ ലളിതമായെങ്കിലും ഏറെ ഉപയോഗിക്കുന്ന ഒരു view ആണ്; ഇതിന്റെ ജോലി വിവിധ ഉള്ളടക്കങ്ങളെ വേർതിരിക്കുക എന്നതാണ്.

`VStack` ൽ `Divider()` ഒരു കിടപ്പ് രേഖയായി കാണിക്കും.

`HStack` ൽ `Divider()` ഒരു നിവർന്ന രേഖയായി കാണിക്കും.

അതുകൂടാതെ `.frame`, `.background`, `.padding` തുടങ്ങിയ മാർഗങ്ങൾ ഉപയോഗിച്ച് `Divider` ന്റെ സ്റ്റൈൽ മാറ്റാനും കഴിയും.

ഉദാഹരണത്തിന്:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

ഇത് 2 തടി, നീല നിറം, ഇടത്-വലത് ഇൻറർ പാഡിങ്ങ് എന്നിവയുള്ള ഒരു വിഭജക രേഖ സൃഷ്ടിക്കും.

![divider](../../Resource/021_divider.png)

### പൂർണ്ണ ഇന്റർഫേസ്

അവസാനമായി, സിനിമാ പോസ്റ്ററും സിനിമാ പരിചയവും തമ്മിൽ ഇടത്-വലത് ക്രമത്തിൽ വയ്ക്കാൻ നമുക്ക് `HStack` ഉപയോഗിക്കാം.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

ഇവിടെ `HStack(spacing: 20)` എന്ന് പറയുന്നത് ഇടത്തും വലത്തും ഉള്ള രണ്ട് ഭാഗങ്ങൾക്കിടയിൽ 20 എന്ന അകലം നിലനിർത്തുക എന്നതാണ്.

ശ്രദ്ധിക്കേണ്ടത്, `Divider()` മുമ്പ് പറഞ്ഞ `TextField`, `Slider` പോലെത്തന്നെ ഡീഫോൾട്ടായി ലഭ്യമായിടത്തോളം സ്ഥലം കൈവശപ്പെടുത്താൻ ശ്രമിക്കുന്നു.

അതുകൊണ്ട് വലത് വശത്തെ പരിചയഭാഗത്തിന്റെ പുറത്തുള്ള `VStack` ന് നാം ഒരു സ്ഥിര വീതി നിശ്ചയിക്കുന്നു:

```swift
.frame(width: 200)
```

ഇങ്ങനെ ചെയ്താൽ വിഭജക രേഖയുടെയും ടെക്സ്റ്റ് ഏരിയയുടെയും വീതി ഒരേപോലെ നിലനിൽക്കും, മുഴുവൻ കാഴ്ചയും കൂടുതൽ വൃത്തിയായിരിക്കും.

ഇത്ര വരെ നാം “ഒറ്റ സിനിമ” കാണിക്കുന്ന ഇന്റർഫേസ് പൂർത്തിയാക്കി.

![movie](../../Resource/021_movie1.png)

## സിനിമകളെ അറേയിലായി സൂക്ഷിക്കുക

പല സിനിമകളെയും ഒരേ സ്റ്റൈലിൽ ഒന്നിന് പിന്നാലെ മറ്റൊന്ന് പ്രദർശിപ്പിക്കണമെങ്കിൽ, ഓരോ സിനിമയ്ക്കുമായി സമാനമായ കോഡ് ആവർത്തിച്ച് എഴുതേണ്ടി വരും.

ഉദാഹരണത്തിന്:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

ഇങ്ങനെ എഴുതിയ കോഡ് ഫലം ലഭ്യമാക്കുമെങ്കിലും, കോഡ് കൂടി കൂടി പോകും, പരിപാലിക്കാനും ബുദ്ധിമുട്ടാകും.

ഉദാഹരണത്തിന്, എല്ലാ സിനിമകളുടെയും സ്പേസിംഗ് `20` നിന്ന് `15` ആക്കണം, അല്ലെങ്കിൽ വലത് വശത്തിന്റെ വീതി `200` നിന്ന്
`220` ആക്കണം എങ്കിൽ, ഓരോ ആവർത്തന കോഡ് ഭാഗവും കൈകൊണ്ട് തിരുത്തേണ്ടി വരും.

ഇത് ഒരു കാര്യക്ഷമമായ മാർഗമല്ലെന്നത് വ്യക്തമാണ്.

മുമ്പ് നാം അറേകളെക്കുറിച്ചും, അറേയെ അടിസ്ഥാനമാക്കി view ആവർത്തിച്ച് കാണിക്കാൻ `ForEach` ഉപയോഗിക്കുന്നതിനെക്കുറിച്ചും പഠിച്ചിരുന്നു.

ഉദാഹരണത്തിന്:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

ഒരു മാത്രം ഡാറ്റയുള്ള സാഹചര്യം കൈകാര്യം ചെയ്യാൻ ഈ രീതി വളരെ അനുയോജ്യമാണ്, ഉദാഹരണത്തിന് ഒരു കൂട്ടം ചിത്രം പേരുകൾ.

എന്നാൽ ഒരു സിനിമയ്ക്ക് ഒരു മാത്രം മൂല്യം മാത്രമല്ല. കുറഞ്ഞത് ഇവയെങ്കിലും അതിലുണ്ട്:

1. സിനിമാ പോസ്റ്റർ
2. സിനിമയുടെ പേര്
3. സംവിധായകൻ
4. റേറ്റിംഗ്

അതു പോലെ, ഒരു സിനിമ എന്നത് പരസ്പരം ബന്ധപ്പെട്ട ഡാറ്റയുടെ ഒരു കൂട്ടമാണ്; ഒറ്റ സ്ട്രിംഗ് അല്ല.

നാം അറേ മാത്രം ഉപയോഗിച്ചാൽ, ഈ വിവരങ്ങൾ വേർതിരിച്ച് മാത്രം സൂക്ഷിക്കാനാകും:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

പ്രദർശിപ്പിക്കുമ്പോൾ ഒരേ ഇൻഡക്സ് ആശ്രയിച്ചാണ് അവ ഒന്നൊന്നായി പൊരുത്തപ്പെടുത്തേണ്ടത്:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

ഈ രീതിയുടെ പ്രശ്നം, പരിപാലനച്ചെലവ് വളരെ കൂടുതലാണ് എന്നതാണ്.

കാരണം, ഒരു സിനിമയുടെ ഡാറ്റ പല അറേകളിലായി ചിതറിക്കിടക്കുന്നു. ഏതെങ്കിലും ഒരു അറേയിൽ ഒരു ഇനം കൂടുതലോ കുറവോ ആയാലും, അല്ലെങ്കിൽ ക്രമം പൊരുത്തപ്പെടാതിരുന്നാലും, പ്രദർശനഫലം തെറ്റിപ്പോകും.

പ്രത്യേകിച്ച് പുതിയ സിനിമ ചേർക്കുമ്പോഴും നീക്കുമ്പോഴും, നിങ്ങൾ പല അറേകളും ഒരേസമയം തിരുത്തണം; എന്തെങ്കിലും വിട്ടുപോകാൻ വളരെ എളുപ്പമാണ്.

അപ്പോൾ “സിനിമയുടെ പേര്, സംവിധായകൻ, റേറ്റിംഗ്” പോലുള്ള വിവരങ്ങളെ എങ്ങനെ ഒരു സമഗ്ര രൂപത്തിലാക്കാം?

അതിന് `struct` ഉപയോഗിക്കണം.

## `struct` ഘടന നിർവചിക്കൽ

Swift ൽ `struct` എന്നത് “ഘടന” എന്നാണ്.

ഇത് നിങ്ങൾക്ക് “കസ്റ്റം ഡാറ്റ ടൈപ്പ്” എന്ന നിലയിൽ കരുതാം; പല ബന്ധപ്പെട്ട ഫീൽഡുകളെയും ഒന്നായി കൂട്ടിച്ചേർക്കാൻ ഇത് സഹായിക്കുന്നു.

അടിസ്ഥാന ഉപയോഗം:

```swift
struct StructName {
	let name: String
}
```

ഈ കോഡിൽ `struct` എന്നത് നാം ഒരു ഘടന നിർവചിക്കുന്നു എന്ന് കാണിക്കുന്ന കീവേഡ് ആണ്. `StructName` ആണ് ഘടനയുടെ പേര്.

കൗതുകക്കുറ്റികളിലുള്ള ഉള്ളടക്കമാണ് ഈ ഘടനയിൽ ഉൾപ്പെടുന്ന ഫീൽഡുകൾ; ഓരോ ഫീൽഡിനും പേര്, ടൈപ്പ് എന്നിവ വ്യക്തമായി എഴുതണം.

സാധാരണയായി `struct` ന്റെ പേര് വലിയ അക്ഷരത്തിൽ തുടങ്ങും, ഉദാഹരണത്തിന് `Movie`, `Student`, `UserInfo`. ഇത് Swift ൽ സാധാരണമായ നാമകരണം രീതിയാണ്.

`struct` നെ ഒരു ശൂന്യ പാക്കിംഗ് ബോക്‌സായി നിങ്ങൾ ലളിതമായി ചിന്തിക്കാം; അതിലെ ഓരോ ഫീൽഡും ആ ബോക്‌സിൽ മുൻകൂട്ടി നൽകിയ ഒരു ഒഴിവു സ്ഥാനത്തെപ്പോലെയാണ്.

ഈ സ്ഥാനങ്ങളിൽ ഇപ്പോഴും ഒന്നും നിറച്ചിട്ടില്ലെങ്കിൽ, അത് വെറും ഒരു ശൂന്യ ബോക്‌സ് മാത്രമാണ്. എല്ലാ ഫീൽഡുകളിലും ശരിയായ മൂല്യങ്ങൾ നിറച്ചാൽ മാത്രമേ അത് ഒരു പൂർണ്ണമായ സമ്മാന പെട്ടിയായി മാറൂ.

### `struct` ഇൻസ്റ്റൻസ്

മുമ്പ് നാം ഘടനയെ മാത്രമാണ് നിർവചിച്ചത്; അതായത് ഒരു പാക്കിംഗ് ബോക്‌സിന്റെ മാതൃക തയ്യാറാക്കി.

ഇപ്പോൾ അതിൽ വ്യക്തമായ ഉള്ളടക്കം നിറച്ച്, യഥാർത്ഥത്തിൽ ഉപയോഗിക്കാവുന്ന ഒരു “ഇൻസ്റ്റൻസ്” നേടേണ്ടതുണ്ട്.

ഇൻസ്റ്റൻസ് സൃഷ്ടിക്കുമ്പോൾ സാധാരണയായി ഘടനയുടെ പേരിന് ശേഷം `()` ചേർക്കും:

```swift
StructName(...)
```

ചെറിയ വളയകക്കൂട്ടുകളിൽ എഴുതുന്നത് ഈ ഘടനയ്ക്കാവശ്യമായ ഫീൽഡ് ഉള്ളടക്കങ്ങളാണ്.

ഉദാഹരണത്തിന്:

```swift
StructName(name: "Fang Junyu")
```

ഈ കോഡ് `StructName` എന്ന ഘടനയുടെ ഫോർമാറ്റ് അനുസരിച്ച് ഒരു പുതിയ ഇൻസ്റ്റൻസ് സൃഷ്ടിച്ച്, `name` ഫീൽഡിന് `"Fang Junyu"` എന്ന മൂല്യം നൽകുന്നു എന്ന് സൂചിപ്പിക്കുന്നു.

ആവശ്യമായ എല്ലാ ഫീൽഡ് ഉള്ളടക്കങ്ങളും നാം നിറച്ചുകഴിഞ്ഞാൽ, അത് ബോക്‌സ് നിറച്ചതിനു തുല്യമാണ്.

അപ്പോൾ നമുക്ക് ഒരു പൂർണ്ണ ഘടന ഇൻസ്റ്റൻസ് ലഭിക്കുന്നു.

### `struct` പ്രോപ്പർട്ടികളിലേക്ക് പ്രവേശിക്കുക

ഒരു ഇൻസ്റ്റൻസ് സൃഷ്ടിച്ചുകഴിഞ്ഞാൽ, അതിന്റെ ഉള്ളിലെ പ്രോപ്പർട്ടികൾക്ക് “dot syntax” ഉപയോഗിച്ച് പ്രവേശിക്കാം.

അടിസ്ഥാന രീതി:

```swift
instance.propertyName
```

ഉദാഹരണത്തിന്:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

ഇവിടെ `st` ഒരു ഘടന ഇൻസ്റ്റൻസാണ്; `st.name` എന്നത് ആ ഇൻസ്റ്റൻസിലെ `name` മൂല്യം വായിക്കുക എന്നാണ്.

ഈ വായന രീതി നിങ്ങൾ ഇങ്ങനെ മനസ്സിലാക്കാം: `st.name` എന്ന് എഴുതുമ്പോൾ, ആ ബോക്‌സിലെ ഒരു പ്രത്യേക സ്ഥാനത്ത് സൂക്ഷിച്ചിരിക്കുന്ന ഉള്ളടക്കം വായിക്കുന്നതുപോലെയാണ്.

ഈ രീതി പിന്നീട് SwiftUI ഇന്റർഫേസുകളിൽ വളരെ സാധാരണമായി കാണാം.

ഉദാഹരണത്തിന്, പേര്, വയസ്, ക്ലാസ് എന്നിവ അടങ്ങിയ ഒരു വിദ്യാർത്ഥി ഘടന ഉണ്ടെങ്കിൽ, ഇന്റർഫേസിൽ കാണിക്കുമ്പോൾ നാം ഇങ്ങനെ പ്രത്യേകം വായിക്കാം:

```swift
student.name
student.age
student.className
```

ഇങ്ങനെ ചെയ്യുന്നതിന്റെ ഗുണം ഡാറ്റ കൂടുതൽ വ്യക്തമായിരിക്കും, കൂടാതെ അത് കൈകാര്യം ചെയ്യാനും എളുപ്പമാകും.

### `struct` ന്റെ സ്ഥാനം

ആരംഭകർക്കായി സാധാരണയായി ഘടന `ContentView` യ്ക്ക് പുറത്തായി എഴുതാം.

ഉദാഹരണത്തിന്:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

ഇങ്ങനെ ചെയ്താൽ ഘടന കൂടുതൽ വ്യക്തമായിരിക്കും, വായിക്കാനും എളുപ്പമാകും.

`ContentView` തന്നെയും യഥാർത്ഥത്തിൽ ഒരു `struct` ആണെന്ന് നിങ്ങൾ കാണും.

### `Movie` ഘടന

മുമ്പത്തെ അടിസ്ഥാനത്തോടെ, നമുക്ക് ഒരു സിനിമ ഘടന നിർവചിക്കാം:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

ഇത് `Movie` എന്ന പേരിലുള്ള ഒരു ഘടനയാണ്; `name`, `director`, `rating` എന്ന മൂന്ന് ഫീൽഡുകൾ ഉൾക്കൊള്ളുന്നു. അവ യഥാക്രമം സിനിമയുടെ പേര്, സംവിധായകൻ, റേറ്റിംഗ് എന്നിവയാണ് സൂചിപ്പിക്കുന്നത്.

ഇങ്ങനെ ഒരു സിനിമയുമായി ബന്ധപ്പെട്ട വിവരങ്ങൾ ഒന്നിലധികം അറേകളായി വേർതിരിച്ച് സൂക്ഷിക്കേണ്ടതില്ല; നേരിട്ട് ഒരൊറ്റ സമഗ്ര രൂപത്തിലാക്കാം.

ഉദാഹരണത്തിന്, നമുക്ക് ഇങ്ങനെ ഒരു സിനിമ ഇൻസ്റ്റൻസ് സൃഷ്ടിക്കാം:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

ഈ ഇൻസ്റ്റൻസ് “ഒരു പൂർണ്ണ സിനിമാ ഡാറ്റ”യെ പ്രതിനിധീകരിക്കുന്നു.

### ഘടനകളെ അറേയിലാക്കി സൂക്ഷിക്കുക

മുമ്പ് പറഞ്ഞതുപോലെ, ഒരു അറേയ്ക്ക് ഒരേ ടൈപ്പിലുള്ള ഡാറ്റ മാത്രമേ സൂക്ഷിക്കാനാകൂ.

ഇപ്പോൾ നമുക്ക് `Movie` എന്ന ഘടനയുണ്ട്; അതിനാൽ അറേയ്ക്ക് പല `Movie` ഇനങ്ങളും സൂക്ഷിക്കാനാകും.

```swift
let lists: [Movie] = []
```

ഉദാഹരണത്തിന്:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

ഈ `lists` അറേയിൽ സൂക്ഷിക്കുന്നത് സ്ട്രിംഗുകൾ അല്ല, പല `Movie` ഇൻസ്റ്റൻസുകളാണ്.

അഥവാ, അറേയിലെ ഓരോ ഘടകവും ഒരു പൂർണ്ണ സിനിമയാണ്.

ഇങ്ങനെ, സിനിമാ പട്ടിക പ്രദർശിപ്പിക്കേണ്ടപ്പോൾ `ForEach` ഉപയോഗിച്ച് ഓരോ സിനിമയെയും ക്രമത്തിൽ വായിക്കാം.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

ഇവിടെ `movie in` എന്നത് ഓരോ ലൂപ്പ് സമയത്തും അറേയിൽ നിന്ന് ഒരു സിനിമ എടുത്ത് താൽക്കാലികമായി `movie` എന്ന് പേരിടുന്നു എന്നതാണ് അർത്ഥം.

പിന്നീട് നമുക്ക് ഇത് ഉപയോഗിക്കാം:

```swift
movie.name
movie.director
movie.rating
```

ഈ ഒരു സിനിമയുടെ പേര്, സംവിധായകൻ, റേറ്റിംഗ് എന്നിവ പ്രത്യേകം വായിച്ച് ഇന്റർഫേസിൽ കാണിക്കാൻ.

## `ForEach` പിശക്

ഇതുവരെ നാം സിനിമാ അറേയുടെയും `ForEach` ന്റെയും അടിസ്ഥാന എഴുത്ത് പൂർത്തിയാക്കി.

പക്ഷേ താഴെ കാണുന്ന കോഡ് നേരിട്ട് പ്രവർത്തിപ്പിച്ചാൽ:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

ഒരു പിശക് ലഭിക്കും:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

ഈ പിശകിന്റെ അർത്ഥം, നിങ്ങൾ `id: \.self` എന്ന് എഴുതുമ്പോൾ SwiftUI ക്ക് അറേയിലെ ഓരോ ഘടകത്തെയും “യൂണിക് ഐഡന്റിഫയർ” ആയി വേർതിരിച്ചു കാണേണ്ടതുണ്ട് എന്നതാണ്.

ഇത്തരം രീതി `String`, `Int` പോലുള്ള ലളിതമായ ടൈപ്പുകൾക്ക് അനുയോജ്യമാണ്, കാരണം അവയെ സ്വയം വേർതിരിക്കാനാവും.

എന്നാൽ `Movie` നാം നിർവചിച്ച ഒരു കസ്റ്റം ഘടനയാണ്; അതിനെ തന്നെയാണ് യൂണിക് ഐഡന്റിഫയർ ആക്കേണ്ടത് എന്ന് `ForEach` അറിയില്ല, അതിനാലാണ് പിശക് വരുന്നത്.

ഈ പ്രശ്നം പരിഹരിക്കാനുള്ള ഏറ്റവും സാധാരണമായ മാർഗം `Movie` നെ `Identifiable` പ്രോട്ടോക്കോൾ പിന്തുടരുന്നതാക്കുക എന്നതാണ്.

## പ്രോട്ടോക്കോൾ

Swift ൽ പ്രോട്ടോക്കോൾ എന്നത് ഒരു തരത്തിലുള്ള “നിയമം” അല്ലെങ്കിൽ “ആവശ്യം” ആയി മനസ്സിലാക്കാം.

അത് ഒരു അന്തർദേശീയ സംഘടനയിലെ കരാറുപോലെ ചിന്തിക്കാം: ഒരു രാജ്യം ഒരു സംഘടനയിൽ ചേരണമെങ്കിൽ, ആ സംഘടന നിർദേശിക്കുന്ന ചില നിയമങ്ങൾ പാലിക്കണം, ചില ഡാറ്റ പുറത്തുവിടണം, പൊതുവായ ചില ചട്ടങ്ങൾ അനുസരിക്കണം. ഈ നിബന്ധനകൾ പാലിച്ച ശേഷമേ അതിൽ ചേരാനോ, ചില കാര്യങ്ങളിൽ പങ്കെടുക്കാനോ സാധിക്കൂ.

പ്രോട്ടോക്കോളും ഇതേപോലെയാണ്.

ഒരു ടൈപ്പ് ഒരു പ്രോട്ടോക്കോൾ പിന്തുടരണമെങ്കിൽ, ആ പ്രോട്ടോക്കോൾ നിർദേശിക്കുന്ന ഉള്ളടക്കം അത് നിറവേറ്റണം. ഈ ആവശ്യങ്ങൾ നിറവേറ്റിയ ശേഷമേ ആ ടൈപ്പിന് അനുബന്ധമായ ഫംഗ്ഷനാലിറ്റി ഉപയോഗിക്കാൻ കഴിയൂ.

അടിസ്ഥാന എഴുത്ത്:

```swift
struct Movie: Identifiable {
    // ...
}
```

ഇവിടെ `: Identifiable` എന്ന് പറഞ്ഞത് `Movie` `Identifiable` പ്രോട്ടോക്കോൾ പിന്തുടരുന്നു എന്നാണ്.

ഒരു ടൈപ്പ് ഒരിലധികം പ്രോട്ടോക്കോളുകൾ പിന്തുടരുന്നുണ്ടെങ്കിൽ, കോമ ഉപയോഗിച്ച് വേർതിരിക്കാം:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

`Identifiable` നു വേണ്ടി ഏറ്റവും പ്രധാനപ്പെട്ട ആവശ്യം, ആ ടൈപ്പിനുള്ളിൽ സ്വയം തിരിച്ചറിയാനുള്ള `id` ഉണ്ടായിരിക്കണം എന്നതാണ്.

ഉദാഹരണത്തിന്:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

ഇവിടെ `id` ഓരോ സിനിമയെയും വേർതിരിക്കാൻ ഉപയോഗിക്കുന്ന ഐഡന്റിറ്റി നമ്പറിനെപ്പോലെയാണ്.

ഓരോ സിനിമയുടെയും `id` വ്യത്യസ്തമായിരിക്കുമ്പോൾ, SwiftUI ക്ക് അറേയിലെ ഓരോ ഘടകവും ശരിയായി വേർതിരിക്കാം.

ഉദാഹരണത്തിന്:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

പിന്നീട് `ForEach` ൽ ഇങ്ങനെ ഉപയോഗിക്കുന്നു:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

ഇങ്ങനെ ചെയ്താൽ പിശക് ഉണ്ടാകില്ല.

`Movie` ഇതിനകം `Identifiable` പിന്തുടരുന്നതിനാൽ, കൂടുതൽ സാധാരണമായ രീതി `id:` നേരിട്ട് ഒഴിവാക്കുന്നതാണ്:

```swift
ForEach(lists) { movie in
    // ...
}
```

കാരണം ഓരോ ഇനവും വേർതിരിക്കാൻ `movie.id` ഉപയോഗിക്കണമെന്ന് SwiftUI ഇതിനകം അറിയുന്നു.

## `ForEach` ന്റെ തിരിച്ചറിയൽ അടയാളം

ഇപ്പോൾ `ForEach` ൽ `id` യുടെ പങ്ക് കൂടുതൽ ആഴത്തിൽ മനസ്സിലാക്കാം.

ഉദാഹരണത്തിന്:

```swift
ForEach(lists, id: \.self)
```

ഇവിടെ അർത്ഥം: ഘടകത്തെ തന്നെയാണ് തിരിച്ചറിയൽ അടയാളമായി ഉപയോഗിക്കുന്നത്.

അതേസമയം:

```swift
ForEach(lists, id: \.id)
```

എന്ന് പറഞ്ഞാൽ ഘടകത്തിലെ `id` ഫീൽഡാണ് തിരിച്ചറിയൽ അടയാളമായി ഉപയോഗിക്കുന്നത്.

ഒരു ഫീൽഡിന് തന്നെ യുണീക്നസ് ഉറപ്പാക്കാനാകുമെങ്കിൽ, താൽക്കാലികമായി അതും ഉപയോഗിക്കാം.

ഉദാഹരണത്തിന് ഓരോ സിനിമയുടെയും പേര് വ്യത്യസ്തമാണെങ്കിൽ, താഴെ കാണുന്ന എഴുത്ത് നിലവിലെ ഡാറ്റയിൽ പ്രവർത്തിച്ചേക്കാം:

```swift
ForEach(lists, id: \.name)
```

പക്ഷേ ഇവിടെ ഒരു മറഞ്ഞ പ്രശ്നമുണ്ട്: `name` എല്ലായ്പ്പോഴും യുണീക് ആയിരിക്കണമെന്നില്ല.

കരുതുക, പിന്നെ നിങ്ങൾ ഒരേ പേരിലുള്ള മറ്റൊരു സിനിമ ചേർത്തു; അപ്പോൾ `name` ഓരോ ഘടകത്തെയും കൃത്യമായി വേർതിരിക്കാൻ മതിയാകില്ല.

അത്തരം സാഹചര്യത്തിൽ, കോഡ് കമ്പൈൽ ചെയ്യാമെങ്കിലും, view refresh ചെയ്യുമ്പോഴും update, insert, delete ചെയ്യുമ്പോഴും SwiftUI തെറ്റായി തിരിച്ചറിയാൻ ഇടയുണ്ട്; അതിലൂടെ പ്രദർശനത്തിൽ അസാധാരണതകൾ ഉണ്ടാകും.

അതിനാൽ ഡാറ്റയെ സ്ഥിരമായി വേർതിരിക്കേണ്ട സാഹചര്യങ്ങളിൽ, യഥാർത്ഥത്തിൽ യുണീക് ആയ ഫീൽഡ്, അതായത് `id`, തന്നെ ഉപയോഗിക്കുന്നതാണ് നല്ലത്.

## UUID

`id: 1`, `id: 2` എന്നിവ കൈകൊണ്ട് എഴുതുന്നത് പ്രശ്നം പരിഹരിച്ചാലും, ഒരു അപകടസാധ്യത ഇപ്പോഴും നിലനിൽക്കും: തെറ്റിയായി ഒരേ `id` വീണ്ടും എഴുതാം; അപ്പോൾ യൂണിക് ഐഡന്റിഫയർ പ്രവർത്തനരഹിതമാകും.

ഉദാഹരണത്തിന്:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

ഇങ്ങനെ രണ്ട് ഒരേ `id` ഉണ്ടാകും; അപ്പോൾ യൂണിക് തിരിച്ചറിയൽ പ്രവർത്തനരഹിതമാകും.

കൈകൊണ്ട് ചെയ്യുന്ന പിശകുകൾ ഒഴിവാക്കാൻ നാം സാധാരണയായി `UUID()` ഉപയോഗിക്കുന്നു.

Swift ൽ `UUID()` ഒരു 128-ബിറ്റ് ഐഡന്റിഫയർ റാൻഡമായി സൃഷ്ടിക്കുന്നു. സാധാരണയായി ഇത് അക്ഷരങ്ങളും അക്കങ്ങളും അടങ്ങിയ ഒരു നീളമുള്ള സ്ട്രിംഗായി കാണപ്പെടും.

ഉദാഹരണത്തിന്:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

ഇവിടത്തെ `UUID()` യഥാർത്ഥത്തിൽ `UUID` ടൈപ്പിന്റെ ഒരു ഇൻസ്റ്റൻസ് സൃഷ്ടിക്കുന്നതാണ്; ഇത് സ്വയം സൃഷ്ടിക്കപ്പെടുന്ന “യൂണിക് നമ്പർ” ആയി കരുതാം.

അതിനാൽ ഓരോ തവണ സൃഷ്ടിക്കുന്ന മൂല്യവും ഏകദേശം ആവർത്തിക്കാറില്ല; അതുകൊണ്ട് `id` ആയി ഉപയോഗിക്കാൻ ഇത് വളരെ അനുയോജ്യമാണ്.

നമുക്ക് `Movie` ഇങ്ങനെ മാറ്റാം:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

ഈ കോഡ് `Movie` ഘടനയിലെ `id` ഫീൽഡിന് ഡീഫോൾട്ടായി ഒരു പുതിയ `UUID` ഇൻസ്റ്റൻസ് നൽകപ്പെടും എന്ന് സൂചിപ്പിക്കുന്നു.

അതായത്, ഓരോ പുതിയ `Movie` സൃഷ്ടിക്കുമ്പോഴും സിസ്റ്റം ആദ്യം തന്നെ അതിന് ഒരു യൂണിക് `id` സ്വയം സൃഷ്ടിക്കും.

`id` ക്ക് ഇതിനകം ഒരു ഡീഫോൾട്ട് മൂല്യമുള്ളതിനാൽ, പിന്നെ `Movie` ഇൻസ്റ്റൻസുകൾ സൃഷ്ടിക്കുമ്പോൾ `id` കൈകൊണ്ട് നൽകേണ്ടതില്ല.

ഉദാഹരണത്തിന്:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

ഇവിടെ `id` എഴുതിയിട്ടില്ലെങ്കിലും, യഥാർത്ഥത്തിൽ ഈ സിനിമ ഇൻസ്റ്റൻസിന് സ്വന്തം `id` ഉണ്ട്; ആ മൂല്യം `UUID()` വഴി സ്വയം സൃഷ്ടിക്കപ്പെട്ടതാണ്.

ഈ രീതി ഉപയോഗിച്ചാൽ, ഓരോ സിനിമയ്ക്കും `id` കൈകൊണ്ട് നൽകേണ്ടതില്ല; അതിലൂടെ കോഡിന്റെ അളവ് കുറയും, ഒരേ `id` ആവർത്തിച്ച് നൽകിയതിലൂടെ വരുന്ന പിശകുകളും ഒഴിവാക്കാം.

അവസാനമായി, മുകളിലായി ഒരു `Banner` ചിത്രം ചേർത്ത്, ഉള്ളടക്കം സ്ക്രോൾ ചെയ്യാൻ `ScrollView` ഉപയോഗിച്ചാൽ മുഴുവൻ മൂവി പട്ടിക view പൂർത്തിയാകും.

## സംഗ്രഹം

ഈ പാഠത്തിൽ നാം വളരെ പ്രധാനപ്പെട്ട ഒരു ആശയം പഠിച്ചു: `struct`.

`struct` ഉപയോഗിച്ച് ഒരു സിനിമയുടെ നിരവധി ബന്ധപ്പെട്ട ഫീൽഡുകളെ ഒരൊറ്റ സമഗ്ര രൂപത്തിലാക്കാം; പേര്, സംവിധായകൻ, റേറ്റിംഗ് എന്നിവ പല അറേകളിലായി വേർതിരിച്ച് സൂക്ഷിക്കേണ്ടതില്ല.

ഡാറ്റ ഈ രീതിയിൽ ക്രമീകരിച്ച ശേഷം, `ForEach` ഉപയോഗിച്ച് അറേയിലെ ഓരോ സിനിമയെയും ക്രമത്തിൽ view ൽ കാണിക്കാം.

ഇതിനൊപ്പം `ForEach` ലെ “തിരിച്ചറിയൽ അടയാളത്തിന്റെ” പങ്കും നമുക്ക് മനസ്സിലായി. SwiftUI ക്ക് അറേയിലെ ഓരോ ഘടകവും എങ്ങനെ വേർതിരിക്കണമെന്ന് അറിയണം; അപ്പോഴാണ് view ശരിയായി പ്രദർശിപ്പിക്കാനും അപ്ഡേറ്റ് ചെയ്യാനും കഴിയുന്നത്.

അതുകൊണ്ട് `Movie` നെ `Identifiable` പ്രോട്ടോക്കോൾ പിന്തുടരുന്നതാക്കി, അതിന് ഒരു യൂണിക് `id` നൽകുകയും ചെയ്തു.

`id` കൈകൊണ്ട് നൽകുമ്പോൾ പിശകുകൾ വരാതിരിക്കാനായി, സിസ്റ്റം സ്വയം യൂണിക് ഐഡന്റിഫയർ സൃഷ്ടിക്കുന്ന `UUID()` നെയും നാം പിന്നീട് പഠിച്ചു.

ഈ പാഠം പഠിച്ചു കഴിഞ്ഞാൽ, നിങ്ങൾ ഒരു മൂവി പട്ടിക പൂർത്തിയാക്കുക മാത്രമല്ല, SwiftUI യിലെ ഒരു പ്രധാന ചിന്താഗതിയെയും സമീപിക്കാൻ തുടങ്ങും: **ആദ്യം ഡാറ്റ ക്രമീകരിക്കുക, ശേഷം ആ ഡാറ്റയുടെ അടിസ്ഥാനത്തിൽ view സൃഷ്ടിക്കുക.**

## പൂർണ്ണ കോഡ്

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
