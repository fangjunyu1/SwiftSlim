# ഫിൽ ലൈറ്റ് ആപ്പ്

ഈ പാഠത്തിൽ, വളരെ രസകരമായ ഒരു ഫിൽ ലൈറ്റ് ആപ്പ് നിർമ്മിക്കാം. രാത്രി വന്നാൽ, ഫോൺ സ്ക്രീനിൽ വിവിധ നിറങ്ങൾ പ്രദർശിപ്പിച്ച് അത് ഒരു ലളിതമായ ഫിൽ ലൈറ്റായി ഉപയോഗിക്കാം.

ഈ ഫിൽ ലൈറ്റ് ആപ്പിൽ സ്ക്രീൻ ടാപ്പ് ചെയ്ത് നിറങ്ങൾ മാറ്റാനും, സ്ലൈഡർ ഉപയോഗിച്ച് ബ്രൈറ്റ്‌നസ് ക്രമപ്പെടുത്താനും കഴിയും.

ഈ ഉദാഹരണത്തിൽ, `brightness` ഉപയോഗിച്ച് വ്യൂയുടെ ബ്രൈറ്റ്‌നസ് ക്രമപ്പെടുത്തുന്നത്, `onTapGesture` ഉപയോഗിച്ച് വ്യൂയിൽ ടാപ്പ് ജെസ്ചർ ചേർക്കുന്നത്, കൂടാതെ `Slider` വ്യൂ കൺട്രോൾ എന്നിവയെക്കുറിച്ച് പഠിക്കും.

## നിറം കാണിക്കൽ

ആദ്യം, ഒരു നിറം പ്രദർശിപ്പിക്കുന്ന വ്യൂ ഉണ്ടാക്കാം.

SwiftUI ൽ, `Color` ഒരു നിറം മാത്രമല്ല, ഒരു വ്യൂ ആയി തന്നെ പ്രദർശിപ്പിക്കാനും കഴിയും:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

ഇവിടെ `Color.red` ഒരു ചുവപ്പ് വ്യൂയെ സൂചിപ്പിക്കുന്നു. `.ignoresSafeArea()` നിറമുള്ള വ്യൂയെ മുഴുവൻ സ്ക്രീനിലാകെ വിരിച്ച്, യഥാർത്ഥ ഫിൽ ലൈറ്റ് പോലെയുള്ള ഒരു അനുഭവം നൽകുന്നു.

### നിറങ്ങളുടെ അറേയും ഇൻഡക്സും

ഇപ്പോൾ ഒരു നിറം മാത്രമാണ് കാണിക്കുന്നത്. പക്ഷേ ഫിൽ ലൈറ്റ് സാധാരണയായി ഒരു നിറത്തിൽ മാത്രം ഒതുങ്ങുന്നില്ല. ഇത് നീല, മഞ്ഞ, വയലറ്റ്, വെള്ള തുടങ്ങിയ നിറങ്ങളും കാണിക്കാം.

സ്ക്രീൻ ടാപ്പ് ചെയ്യുമ്പോൾ വ്യത്യസ്ത നിറങ്ങൾ മാറിക്കാണിക്കണമെന്നാണ് നമ്മൾ ആഗ്രഹിക്കുന്നത്. ഈ നിറങ്ങളെ ഒരു അറേയിൽ വെച്ച് ഒരുമിച്ച് നിയന്ത്രിക്കാം:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

ഒരു അറേ “ഒരേ തരത്തിലുള്ള ഡാറ്റയുടെ ഒരു കൂട്ടം” സൂക്ഷിക്കാൻ അനുയോജ്യമാണ്. ഇവിടെ അറേയിലെ ഓരോ ഘടകവും ഒരു `Color` ആണ്.

ഒരു പ്രത്യേക നിറം കാണിക്കണമെങ്കിൽ, നമുക്ക് ഇൻഡക്സ് ഉപയോഗിക്കാം:

```swift
colors[0]
```

ഇതിന്റെ അർത്ഥം അറേയിലെ `0` ഇൻഡക്സിലുള്ള നിറം വായിക്കുക, അതായത് ആദ്യ നിറം.

ഇപ്പോൾ കോഡ് ഇങ്ങനെ എഴുതാം:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

ഇങ്ങനെ ചെയ്താൽ സ്ക്രീനിൽ അറേയിലെ ആദ്യ നിറമായ ചുവപ്പ് കാണിക്കും.

### ഇൻഡക്സ് ഉപയോഗിച്ച് നിറം നിയന്ത്രിക്കൽ

വ്യത്യസ്ത നിറങ്ങൾക്കിടയിൽ മാറേണ്ടതുണ്ടെങ്കിൽ, ഇൻഡക്സ് സ്ഥിരമായി എഴുതുന്നതിനുപകരം അത് നിയന്ത്രിക്കാൻ ഒരു വേരിയബിൾ വേണം.

ഇൻഡക്സ് സൂക്ഷിക്കാൻ `@State` ഉപയോഗിച്ച് ഒരു വേരിയബിൾ പ്രഖ്യാപിക്കാം:

```swift
@State private var index = 0
```

ഇവിടെ `index` നിലവിലെ നിറത്തിന്റെ ഇൻഡക്സിനെ സൂചിപ്പിക്കുന്നു.

`index` മാറുമ്പോൾ SwiftUI വീണ്ടും ഇന്റർഫേസ് കണക്കാക്കി പ്രദർശിപ്പിക്കുന്ന ഉള്ളടക്കം പുതുക്കും.

ശേഷം പഴയ `colors[0]` മാറ്റി ഇതാക്കി എഴുതാം:

```swift
colors[index]
```

ഇങ്ങനെ വ്യൂയിൽ കാണുന്ന നിറം `index` അനുസരിച്ച് നിശ്ചയിക്കപ്പെടും.

ഇപ്പോൾ കോഡ് ഇങ്ങനെ മാറുന്നു:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

`index` മാറുമ്പോൾ `colors[index]` മറ്റൊരു നിറവും പ്രദർശിപ്പിക്കും.

ഉദാഹരണത്തിന്:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

ശ്രദ്ധിക്കേണ്ട കാര്യം, `index` അറേയിലെ ഏറ്റവും വലിയ ഇൻഡക്സിനെ കടക്കരുത്, അല്ലെങ്കിൽ out-of-range പിശക് ഉണ്ടാകും.

## ടാപ്പ് ജെസ്ചർ

ഇപ്പോൾ നമുക്ക് `index` അനുസരിച്ച് വ്യത്യസ്ത നിറങ്ങൾ കാണിക്കാം, പക്ഷേ ടാപ്പ് ചെയ്ത് നിറം മാറ്റാൻ ഇപ്പോഴും കഴിയില്ല.

മുമ്പത്തെ “Quote Carousel” പാഠത്തിൽ quote മാറ്റാൻ നാം `Button` ഉപയോഗിച്ചിരുന്നു.

എന്നാൽ ഈ തവണ, “മുഴുവൻ നിറമുള്ള ഭാഗത്ത് ടാപ്പ്” ചെയ്ത് നിറം മാറണമെന്നാണ് ആഗ്രഹിക്കുന്നത്, അതിനാൽ `onTapGesture` കൂടുതൽ അനുയോജ്യമാണ്.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

നാം നിറമുള്ള വ്യൂ ടാപ്പ് ചെയ്യുമ്പോൾ, ഈ കോഡ് പ്രവർത്തിക്കും:

```swift
index += 1
```

ഇതിന്റെ അർത്ഥം `index` `1` കൊണ്ട് കൂട്ടുക എന്നാണ്. ഇൻഡക്സ് കൂടിയാൽ, `colors[index]` അറേയിലെ അടുത്ത നിറം കാണിക്കും.

### `onTapGesture`

`onTapGesture` ഒരു gesture modifier ആണ്. ഇത് ഒരു വ്യൂവിലേക്ക് ടാപ്പ് പ്രവർത്തനം ചേർക്കുന്നു.

അടിസ്ഥാന ഉപയോഗം:

```swift
.onTapGesture {
    // code
}
```

ഉദാഹരണത്തിന്:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

ഈ ചുവപ്പ് വ്യൂ ടാപ്പ് ചെയ്താൽ, braces ഉള്ളിലെ കോഡ് പ്രവർത്തിക്കുകയും console ൽ ഇത് പ്രിന്റ് ചെയ്യുകയും ചെയ്യും:

```swift
Click color
```

`onTapGesture` വഴി ഒരു വ്യൂ ടാപ്പ് ചെയ്ത ശേഷം എന്ത് സംഭവിക്കണം എന്ന് നമുക്ക് നിശ്ചയിക്കാം.

### `Button` നോടുള്ള വ്യത്യാസം

മുമ്പ് നാം `Button` വ്യൂ പഠിച്ചിരുന്നു. `Button` ഉം `onTapGesture` ഉം രണ്ടും ടാപ്പ് പ്രവർത്തനം കൈകാര്യം ചെയ്യാം, പക്ഷേ അവ ഉപയോഗിക്കുന്ന സാഹചര്യങ്ങൾ ഒരുപോലെയല്ല.

`onTapGesture` “ഇപ്പോഴുള്ള ഒരു വ്യൂവിൽ ടാപ്പ് ഫംഗ്ഷൻ ചേർക്കാൻ” കൂടുതൽ അനുയോജ്യമാണ്, ഉദാഹരണത്തിന് `Color`, `Image`, `Text`, അല്ലെങ്കിൽ മറ്റേതെങ്കിലും സാധാരണ വ്യൂകൾ.

അതേസമയം, `Button` “Confirm”, “Submit”, “Delete” പോലെയുള്ള വ്യക്തമായ ഒരു ബട്ടൺ സൂചിപ്പിക്കാൻ കൂടുതൽ അനുയോജ്യമാണ്.

ഈ ഫിൽ ലൈറ്റ് ആപ്പിൽ നിറം മാറുന്ന ഫംഗ്ഷൻ കൂടുതൽ ലളിതമാക്കാനാണ് നമ്മൾ ആഗ്രഹിക്കുന്നത്. നിറമുള്ള മുഴുവൻ ഭാഗവും ടാപ്പ് ചെയ്‌താൽ നിറം മാറുന്നതിനാൽ, ഇവിടെ `onTapGesture` ഒരു നല്ല തിരഞ്ഞെടുപ്പാണ്.

## ഇൻഡക്സ് പ്രശ്നം

ഇപ്പോൾ നമുക്ക് സ്ക്രീൻ ടാപ്പ് ചെയ്ത് വ്യത്യസ്ത നിറങ്ങൾക്കിടയിൽ മാറാൻ കഴിയും.

എന്നാൽ ഇവിടെ ഒരു പ്രധാന പ്രശ്നമുണ്ട്: **ഇൻഡക്സ് അറേയുടെ പരിധി കടന്ന് പോകാം**.

ഉദാഹരണത്തിന്:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

നാം സ്ക്രീൻ നിരന്തരം ടാപ്പ് ചെയ്താൽ, `index` ഒടുവിൽ `4` ആകും, അപ്പോൾ “index out of range” എന്ന പിശക് ഉണ്ടാകും.

ഇതിന് കാരണം `colors` അറേയിൽ `4` ഘടകങ്ങൾ ഉണ്ടെങ്കിലും ഇൻഡക്സിംഗ് `0` മുതൽ തുടങ്ങുന്നതാണ്. അതിനാൽ സാധുവായ ഇൻഡക്സ് പരിധി `0 - 3` ആണ്, `4` അല്ല.

`colors[4]` ആക്‌സസ് ചെയ്താൽ “index out of range” പിശക് ഉണ്ടാകും.

നിലവിലെ കോഡിൽ, ഓരോ ടാപ്പിലും `index` സ്വയം `1` കൂടി കൊണ്ടിരിക്കുന്നു. ഇത് നാം കൈകാര്യം ചെയ്യാത്ത പക്ഷം, ഒടുവിൽ അത് പരിധി കടക്കും.

അതുകൊണ്ട്, സ്ക്രീൻ ടാപ്പ് ചെയ്യുമ്പോൾ ഇൻഡക്സ് പരിശോധിക്കണം: അത് അവസാന നിറമായാൽ ആദ്യ നിറത്തിലേക്ക് മടങ്ങുക; ഇല്ലെങ്കിൽ `1` കൂട്ടിക്കൊണ്ടിരിക്കുക.

ഇത് `if` സ്റ്റേറ്റ്മെന്റ് ഉപയോഗിച്ച് ചെയ്യാം:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

ഈ കോഡിൽ `colors.count` എന്നത് അറേയിലെ ഘടകങ്ങളുടെ എണ്ണം എന്നാണ്.

നിലവിലെ അറേയിൽ 4 നിറങ്ങളുള്ളതിനാൽ:

```swift
colors.count // 4
```

പക്ഷേ ഏറ്റവും വലിയ ഇൻഡക്സ് `4` അല്ല, അത് `3` ആണ്, കാരണം ഇൻഡക്സിംഗ് `0` മുതൽ തുടങ്ങുന്നു.

അതിനാൽ അവസാന ഇൻഡക്സ് ഇങ്ങനെ എഴുതണം:

```swift
colors.count - 1
```

അതായത്:

```swift
4 - 1 = 3
```

ഈ തർക്കത്തിന്റെ അർത്ഥം: നിലവിലെ ഇൻഡക്സ് ഇതിനകം അവസാന നിറത്തെ സൂചിപ്പിക്കുന്നുവെങ്കിൽ, ഇൻഡക്സ് `0` ആയി റീസെറ്റ് ചെയ്യുക; അല്ലെങ്കിൽ `1` കൂട്ടുക.

ഈ രീതിയിൽ നിറങ്ങൾ ചുറ്റി മാറിക്കൊണ്ടിരിക്കും.

### ഇൻഡക്സ് ലൊജിക്ക് കൂടുതൽ ലളിതമാക്കൽ

കോഡ് കൂടുതൽ ചുരുക്കണമെങ്കിൽ, ternary operator ഉപയോഗിക്കാം:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

ഈ കോഡിന്റെ അർത്ഥം: `index == colors.count - 1` ശരിയാണെങ്കിൽ `0` തിരികെ നൽകുക. അല്ലെങ്കിൽ `index + 1` തിരികെ നൽകുക.

അവസാനം ഈ ഫലം `index` ലേക്ക് വീണ്ടും assign ചെയ്യും.

ഇപ്പോൾ നമുക്ക് നിറം മാറുന്ന ഇഫക്റ്റ് പ്രവർത്തിപ്പിക്കാം.

പൂർണ്ണ കോഡ്:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

ഇപ്പോൾ നമുക്ക് സ്ക്രീൻ ടാപ്പ് ചെയ്ത് വ്യത്യസ്ത നിറങ്ങൾക്കിടയിൽ മാറാൻ കഴിയും. ഒരു അടിസ്ഥാന ഫിൽ ലൈറ്റ് ആപ്പ് തയ്യാറായി കഴിഞ്ഞു.

## നിറത്തിന്റെ പേര് കാണിക്കൽ

നിറങ്ങൾക്ക് അനുയോജ്യമായ മറ്റൊരു ടെക്സ്റ്റ് ഗ്രൂപ്പ് കൂടി ചേർക്കാം. അതിനാൽ നിറം മാറുമ്പോൾ, ഇപ്പോഴുള്ള നിറത്തിന്റെ പേരും സ്ക്രീനിൽ കാണിക്കും.

ഉദാഹരണത്തിന്:

- ചുവപ്പായാൽ `Red`
- നീലയായാൽ `Blue`
- മഞ്ഞയായാൽ `Yellow`
- വയലറ്റായാൽ `Purple`

ഇവിടെയും നിറങ്ങളുടെ പേരുകൾ സൂക്ഷിക്കാൻ ഒരു അറേ ഉപയോഗിക്കാം:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

ഈ അറേയിലെ ടെക്സ്റ്റിന്റെ ക്രമം, നിറങ്ങളുടെ അറേയിലെ നിറങ്ങളുടെ ക്രമവുമായി കൃത്യമായി പൊരുത്തപ്പെടണം.

ശേഷം നിലവിലെ ഇൻഡക്സ് അനുസരിച്ച് നിറത്തിന്റെ പേര് കാണിക്കാൻ `Text` ഉപയോഗിക്കാം:

```swift
Text(colorsName[index])
```

`Text` `index` അനുസരിച്ച് നിലവിലെ നിറത്തിന്റെ പേര് കാണിക്കും.

`Text` ന്റെ രൂപം മെച്ചപ്പെടുത്താൻ modifiers ഉപയോഗിക്കാം:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

ഇതിലൂടെ `Text` വെള്ള നിറത്തിൽ, title വലുപ്പത്തിൽ, bold font weight സഹിതം കാണിക്കും.

ഇപ്പോൾ നമ്മുക്കൊരു full-screen `Color` വ്യൂ ഉണ്ട്. `Text` നെ `Color` വ്യൂയുടെ മുകളിൽ കാണിക്കണമെങ്കിൽ, രണ്ടിനെയും ഒന്നിന്മീതെ ഒന്നായി വെക്കാൻ `ZStack` layout container ഉപയോഗിക്കണം.

```swift
ZStack {
    Color
    Text
}
```

അപ്പോൾ കോഡ് ഇങ്ങനെ മാറും:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

ഇങ്ങനെ പശ്ചാത്തലത്തിൽ നിറമുള്ള വ്യൂ ഇരിക്കും, അതിന്റെ മുകളിൽ ടെക്സ്റ്റ് വ്യൂ ലെയർ ചെയ്ത് കാണിക്കും.

ഫലപ്രദർശനം:

![Color](../../Resource/018_color2.png)

ശ്രദ്ധിക്കേണ്ട മറ്റൊരു കാര്യം, `ZStack` ൽ ശേഷം എഴുതിയ വ്യൂകൾ സാധാരണയായി മുന്നിൽ കാണിക്കും. `Text` നെ `Color` ന് മുമ്പ് എഴുതിയാൽ, പിന്നീട് വരുന്ന `Color` വ്യൂ `Text` നെ മറയ്ക്കാം.

## ബ്രൈറ്റ്‌നസ് നിയന്ത്രിക്കൽ

ഇപ്പോൾ നമുക്ക് വ്യത്യസ്ത നിറങ്ങൾ മാറ്റാം, പക്ഷേ ഫിൽ ലൈറ്റിന് മറ്റൊരു പ്രധാന സവിശേഷതയുണ്ട്: **ബ്രൈറ്റ്‌നസ് ക്രമപ്പെടുത്തൽ**.

SwiftUI ൽ, `brightness` modifier ഉപയോഗിച്ച് വ്യൂയുടെ പ്രകാശം ക്രമപ്പെടുത്താം.

ഉദാഹരണത്തിന്:

```swift
.brightness(1)
```

ഇങ്ങനെ എഴുതാം:

```swift
colors[index]
    .brightness(0.5)
```

ഇതിലൂടെ നിലവിലെ നിറം കൂടുതൽ പ്രകാശമുള്ളതാകും, അതോടെ ഫിൽ ലൈറ്റ് ഫലത്തിന് കൂടുതൽ അടുത്തതായി തോന്നും.

ബ്രൈറ്റ്‌നസിന്റെ പരിധി `0 - 1` ആണ്. `0` എന്നാൽ മൂല നിറം അതേപടി നിലനിർത്തുക, കൂടാതെ മൂല്യം `1` ന് അടുത്താകുമ്പോൾ നിറം കൂടുതൽ തെളിയും. `1` ഏറ്റവും തെളിച്ചമുള്ള വെള്ള ഫലമാണ്.

കോഡിൽ `brightness` നിയന്ത്രിക്കാമെങ്കിലും, ഉപയോക്താവിന് അത് നേരിട്ട് ക്രമപ്പെടുത്താൻ ഇപ്പോഴും കഴിയില്ല.

അതിനാൽ വലിച്ചുനീക്കാൻ കഴിയുന്ന ഒരു നിയന്ത്രണം കൂടി ചേർക്കണം: `Slider`.

## `Slider` വ്യൂ

SwiftUI ൽ, `Slider` ഒരു നിശ്ചിത പരിധിക്കുള്ളിൽ മൂല്യം തിരഞ്ഞെടുക്കാൻ ഉപയോഗിക്കുന്ന നിയന്ത്രണമാണ്. Apple ഇതിനെ “പരിമിതമായ ലീനിയർ പരിധിയിൽ നിന്ന് മൂല്യം തിരഞ്ഞെടുക്കാനുള്ള നിയന്ത്രണം” എന്നു വിശദീകരിക്കുന്നു.

മൂല ഉപയോഗം:

```swift
Slider(value: $value, in: 0...1)
```

പാരാമീറ്ററുകളുടെ വിശദീകരണം:

1. `value: $value`: `Slider` ഒരു വ്യരിയബിളുമായി bind ചെയ്യണം.

    സ്ലൈഡർ നീക്കുമ്പോൾ, അതുമായി ബന്ധിപ്പിച്ചിരിക്കുന്ന വ്യരിയബിളിന്റെ മൂല്യവും ഒരേസമയം മാറും. മറിച്ച്, വ്യരിയബിൾ മാറുമ്പോൾ സ്ലൈഡറും പുതുക്കപ്പെടും.

    ഇത് നാം മുമ്പ് പഠിച്ച `TextField` നോട് വളരെ സാമ്യമുള്ളതാണ്. രണ്ടും “നിയന്ത്രണത്തെ ഒരു വ്യരിയബിളുമായി bind ചെയ്യുന്നു”.

    binding കാണിക്കാൻ bind ചെയ്തിരിക്കുന്ന വ്യരിയബിളിന് മുന്നിൽ `$` അടയാളം വേണം.

2. `in: 0...1`: ഈ പാരാമീറ്റർ സ്ലൈഡറിന്റെ മൂല്യപരിധിയെ സൂചിപ്പിക്കുന്നു.

    ഇവിടെ `0...1` എന്നത് കുറഞ്ഞ മൂല്യം `0` ആണെന്നും, ഏറ്റവും വലിയ മൂല്യം `1` ആണെന്നും അർത്ഥം.

    സ്ലൈഡർ പൂർണ്ണമായി ഇടത്തോട്ട് നീക്കുമ്പോൾ, bind ചെയ്തിരിക്കുന്ന വ്യരിയബിൾ `0` ന് അടുത്തായിരിക്കും; വലത്തോട്ട് നീക്കുമ്പോൾ `1` ന് അടുത്തായിരിക്കും.

ഉദാഹരണത്തിന്:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` `value` വ്യരിയബിളുമായി bind ചെയ്തിരിക്കുന്നു. സ്ലൈഡർ നീക്കുമ്പോൾ `value` വ്യരിയബിളിന്റെ മൂല്യവും ഒരേസമയം മാറുന്നു.

ദൃശ്യരൂപം:

![Slider](../../Resource/018_slider.png)

`Slider` ഇടത്തോട്ട് നീക്കുമ്പോൾ bind ചെയ്തിരിക്കുന്ന `value` `0` ആകും. വലത്തോട്ട് നീക്കുമ്പോൾ അത് `1` ആകും.

### മൂല്യപരിധി

`Slider` ന്റെ മൂല്യപരിധി സ്ഥിരമായതല്ല. ഇത് ഇങ്ങനെ കൂടി എഴുതാം:

```swift
0...100
```

അല്ലെങ്കിൽ വേറെ range ആയി.

പക്ഷേ ഈ ഫിൽ ലൈറ്റ് ആപ്പിൽ ബ്രൈറ്റ്‌നസ് നിയന്ത്രിക്കേണ്ടതിനാൽ, `0...1` ഏറ്റവും അനുയോജ്യമായതാണ്.

## `Slider` ഉപയോഗിച്ച് ബ്രൈറ്റ്‌നസ് നിയന്ത്രിക്കൽ

ഇപ്പോൾ `Slider` നെ `brightness` നോട് ബന്ധിപ്പിക്കണം.

ആദ്യം ബ്രൈറ്റ്‌നസ് മൂല്യം സൂക്ഷിക്കാൻ ഒരു വ്യരിയബിൾ സൃഷ്ടിക്കാം:

```swift
@State private var slider = 0.0
```

ഇവിടെ `0.0` എന്നത് `Double` വിധത്തിലുള്ള ഒരു മൂല്യമാണ്.

`Slider` സാധാരണയായി സംഖ്യാത്മക തരത്തിലുള്ളവയുമായി bind ചെയ്യപ്പെടുന്നു. ഇവിടെ ഇത് തുടർച്ചയായി മാറണമെന്നാണ് നമുക്ക് വേണ്ടത്, അതിനാൽ `Double` കൂടുതൽ യോജിച്ചിരിക്കുന്നു. കൂടാതെ `brightness` `Double` മൂല്യങ്ങൾ മാത്രമേ സ്വീകരിക്കൂ.

പിന്നീട് ഈ മൂല്യം `brightness` ന് നൽകാം:

```swift
colors[index]
    .brightness(slider)
```

`slider == 0` ആണെങ്കിൽ നിറം തന്റെ default നിലയിൽ തന്നെ നിലനിൽക്കും. `slider` `1` ന് എത്ര അടുത്തുനിൽക്കുന്നു, നിറം അത്രയും കൂടുതൽ തെളിഞ്ഞതായി തോന്നും.

### `Slider` നിയന്ത്രണം ചേർക്കൽ

ശേഷം ഈ വ്യരിയബിളിനെ മാറ്റാൻ `Slider` നിയന്ത്രണം ചേർക്കാം:

```swift
Slider(value: $slider, in: 0...1)
```

സ്ലൈഡർ മാറുമ്പോൾ `slider` മൂല്യവും മാറും, കൂടാതെ `brightness(slider)` അതേ സമയം ബ്രൈറ്റ്‌നസ് പുതുക്കും.

ഇത് SwiftUI ൽ “വ്യരിയബിൾ വ്യൂയെ ഓടിക്കുന്നു” എന്ന ആശയത്തിന്റെ വളരെ സാധാരണമായ ഉദാഹരണമാണ്.

### `Slider` ന്റെ രൂപം ക്രമപ്പെടുത്തൽ

ഡിഫോൾട്ട് ആയി, `Slider` ലഭ്യമായ മുഴുവൻ വീതിയും ഉപയോഗിക്കും.

നമുക്ക് അതിന് ഒരു നിശ്ചിത വീതി കൊടുക്കാം:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

ശേഷം അത് കൂടുതൽ വ്യക്തമായി കാണാൻ ചില modifiers കൂടി ചേർക്കാം:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

ഇത് സ്ലൈഡറിന് വെള്ള പശ്ചാത്തലവും വട്ടമൂലകളും നൽകും, അതിനാൽ `Color` വ്യൂയുടെ മുകളിൽ ഇത് കൂടുതൽ വ്യക്തമായി കാണാം.

അവസാനം, അത് സ്ക്രീൻയുടെ താഴെ വയ്ക്കുക.

നാം ഇതിനകം `ZStack` ഉപയോഗിക്കുന്നതിനാൽ, അതിനകത്ത് `VStack` വെച്ച് `Spacer()` ഉപയോഗിച്ച് `Slider` നെ താഴേക്ക് തള്ളാം.

## പൂർണ്ണ കോഡ്

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## സംഗ്രഹം

മുമ്പ് പഠിച്ച അറിവും നിറങ്ങൾ, അറേകൾ തുടങ്ങിയ അടിസ്ഥാന ആശയങ്ങളും ഉപയോഗിച്ച്, വളരെ രസകരമായ ഒരു ഫിൽ ലൈറ്റ് ആപ്പ് നാം നിർമ്മിച്ചു.

ഈ ഫിൽ ലൈറ്റ് ആപ്പിലൂടെ, `brightness` ഉപയോഗിച്ച് ബ്രൈറ്റ്‌നസ് ക്രമപ്പെടുത്തൽ, `onTapGesture` ഉപയോഗിച്ച് വ്യൂകളിലേക്ക് ടാപ്പ് പ്രവർത്തനങ്ങൾ ചേർക്കൽ, `Slider` നിയന്ത്രണം ഉപയോഗിക്കൽ എന്നിവ പഠിച്ചു.

നിറങ്ങൾ മാറ്റാൻ `Color` വ്യൂവിലേക്ക് `onTapGesture` ചേർത്തു. `brightness` നിയന്ത്രിക്കുന്ന ഒരു വ്യരിയബിള്‍ മാറ്റാൻ `Slider` ഉപയോഗിക്കുകയും ചെയ്തു. ഇത് “വ്യരിയബിൾ വ്യൂയെ നിയന്ത്രിക്കുന്നു” എന്ന ആശയത്തിന്റെ മറ്റൊരു ഉദാഹരണമാണ്.

ternary operator നെയും നമ്മൾ വീണ്ടും കണ്ടു, വ്യൂകളെ ഒന്നിന്മീതെ ഒന്നു കാണിക്കാൻ `ZStack` ഉപയോഗിച്ചു, ഒരേ തരത്തിലുള്ള ഡാറ്റാ ഗ്രൂപ്പുകൾ കൈകാര്യം ചെയ്യാൻ അറേകളും ഉപയോഗിച്ചു. ഇതിലൂടെ അറേകളെയും ഇൻഡക്സുകളെയും കുറിച്ചുള്ള മനസ്സിലാക്കൽ കൂടുതൽ ആഴത്തിലാകുന്നു, പ്രത്യേകിച്ച് out-of-range പിശകുകൾ ഒഴിവാക്കാനുള്ള പ്രായോഗിക പ്രശ്നത്തിൽ.

ഈ ഉദാഹരണം സങ്കീർണ്ണമല്ലെങ്കിലും, നാം മുമ്പ് പഠിച്ച പല അടിസ്ഥാന ആശയങ്ങളെയും ഇത് ബന്ധിപ്പിക്കുന്നു. അവയെ ഒരു ചെറിയ യാഥാർത്ഥ്യ പ്രോജക്ടിൽ ഉപയോഗിക്കുമ്പോൾ, ഓരോ ആശയവും എന്തിനാണ് ഉപയോഗിക്കുന്നത് എന്നത് കൂടുതൽ എളുപ്പത്തിൽ മനസ്സിലാക്കാം.

### യഥാർത്ഥ ഉപയോഗസാഹചാര്യങ്ങൾ

ഒരു പഴയ iPhone മേശപ്പുറത്ത് വെച്ച്, നിങ്ങൾ തന്നെ വികസിപ്പിച്ച ഫിൽ ലൈറ്റ് ആപ്പ് ഉപയോഗിച്ച് വെളിച്ചത്തിന്റെ നിറം നിയന്ത്രിക്കുന്നതായി കരുതൂ. അതൊരു വളരെ നല്ല അനുഭവമായിരിക്കും.

App Store ൽ അനേകം “ഫിൽ ലൈറ്റ്” ആപ്പുകൾ കാണാം, അവയും അത്ര സങ്കീർണ്ണമെന്ന് തോന്നുന്നില്ല.

![AppStore](../../Resource/018_appStore.PNG)

നമുക്ക് ലളിതമായ ആപ്പുകളിൽ നിന്ന് വികസനം തുടങ്ങാം, പിന്നെ അവ App Store ൽ പ്രസിദ്ധീകരിക്കാൻ ശ്രമിക്കാം. ഇത് വികസനത്തിലേക്കുള്ള താൽപ്പര്യം വർധിപ്പിക്കുന്നതുമാത്രമല്ല, നമ്മുടെ വളർച്ചയുടെ രേഖയുമാകും.

### പാഠത്തിന് ശേഷമുള്ള അഭ്യാസം

ഈ ഫിൽ ലൈറ്റ് ആപ്പിനെ ഇനി എങ്ങനെ വികസിപ്പിക്കാം എന്ന് നിങ്ങൾക്ക് ചിന്തിച്ചുകൊണ്ടിരിക്കാം, ഉദാഹരണത്തിന്:

- കൂടുതൽ നിറങ്ങൾ ചേർക്കുക
- നിലവിലെ ബ്രൈറ്റ്‌നസ് മൂല്യം കാണിക്കുക
- താഴെയുള്ള സ്ലൈഡർ ഭാഗത്തിന്റെ ഡിസൈൻ കൂടുതൽ മെച്ചപ്പെടുത്തുക

ഈ അറിവ് നിങ്ങൾ യഥാർത്ഥത്തിൽ ഉപയോഗിക്കാൻ തുടങ്ങുമ്പോൾ, നാം പഠിക്കുന്ന ഓരോ ആശയവും ആപ്പ് വികസനത്തിനുള്ള ഒരു ഉപകരണമാണെന്ന് മനസ്സിലാകും.

നാം എത്ര കൂടുതൽ ഉപകരണങ്ങൾ കൈകാര്യം ചെയ്യുന്നു, അത്ര കൂടുതൽ സവിശേഷതകൾ സൃഷ്ടിക്കാനാകും.
