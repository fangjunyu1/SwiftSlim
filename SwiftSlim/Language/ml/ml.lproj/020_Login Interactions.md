# ലോഗിൻ ഇന്ററാക്ഷനുകൾ

മുൻ പാഠത്തിൽ, നാം ഒരു ലോഗിൻ വ്യൂ സൃഷ്ടിക്കുകയും ബട്ടണുകളും നിർണയ ലജിക്കും വഴി ഇൻപുട്ട് ഉള്ളടക്കത്തിന്റെ ലളിതമായ പരിശോധന നടപ്പിലാക്കുകയും ചെയ്തു.

ഈ പാഠത്തിൽ, മുൻ പാഠത്തിലെ കോഡിനെ അടിസ്ഥാനമാക്കി `SecureField` പാസ്‌വേഡ് ഇൻപുട്ട് ബോക്സ്, `Alert` മുന്നറിയിപ്പ് ബോക്സ്, കൂടാതെ വേരിയബിളുകളുടെ അടിസ്ഥാനത്തിൽ വ്യത്യസ്ത വ്യൂകൾ എങ്ങനെ പ്രദർശിപ്പിക്കാമെന്ന് പഠിക്കാം.

ഈ പാഠത്തിന്റെ വഴി, ലോഗിൻ വ്യൂയുടെ അടിസ്ഥാന ഇന്ററാക്ഷൻ പ്രവാഹം നിങ്ങൾക്ക് കൂടുതൽ ആഴത്തിൽ മനസ്സിലാക്കാൻ കഴിയും. ഇത് നിരവധി ആപ്പുകളിൽ ഉപയോഗിക്കുന്ന ഒരു അടിസ്ഥാന ലജിക്കും കൂടിയാണ്.

## പാസ്‌വേഡ് ഇൻപുട്ട് ബോക്സ്

കഴിഞ്ഞ പാഠത്തിൽ, പാസ്‌വേഡ് നൽകാൻ നാം `TextField` ഉപയോഗിച്ചു.

പക്ഷേ യഥാർത്ഥ ലോഗിൻ ഇന്റർഫേസിൽ, പാസ്‌വേഡ് സാധാരണയായി നേരിട്ട് കാണിക്കാറില്ല; പകരം അത് ഡോട്ടുകളായോ മറ്റേതെങ്കിലും മറച്ചുവെച്ച ഇഫക്റ്റായോ കാണിക്കും, ഇതിലൂടെ ഉപയോക്താവിന്റെ സ്വകാര്യത സംരക്ഷിക്കാം.

SwiftUI-ൽ, പാസ്‌വേഡ് നൽകാൻ പ്രത്യേകമായി ഉപയോഗിക്കുന്ന കൺട്രോൾ `SecureField` ആണ്.

ഉദാഹരണത്തിന്:

```swift
SecureField("Password", text: $password)
```

ഇത് എഴുതുന്ന രീതി `TextField` നോട് വളരെ സാമ്യമുണ്ട്. അതുപോലെ തന്നെ ഇതിന്നും ഒരു സൂചനാ എഴുത്തും ഒരു ബൗണ്ട് ചെയ്ത വേരിയബിളും ആവശ്യമാണ്.

ഇത് ലളിതമായി ഇങ്ങനെ മനസ്സിലാക്കാം: `SecureField`-വും ഒരു ഇൻപുട്ട് ബോക്‌സാണ്, പക്ഷേ അത് നൽകുന്ന ഉള്ളടക്കം മറച്ചുവയ്ക്കും.

മുൻപ് പാസ്‌വേഡ് നൽകാൻ ഉപയോഗിച്ചിരുന്ന `TextField` നെയെ നേരിട്ട് `SecureField` ആയി മാറ്റാം:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

ഉപയോക്താവ് പാസ്‌വേഡ് നൽകുമ്പോൾ, പാസ്‌വേഡിന്റെ ഉള്ളടക്കം ഇന്റർഫേസിൽ നേരിട്ട് കാണിക്കപ്പെടില്ല.

ഇപ്പോൾ നമുക്ക് ഉപയോക്തൃനാമവും പാസ്‌വേഡും നൽകാം, പിന്നെ ബട്ടൺ ക്ലിക്ക് ചെയ്ത ശേഷം ഇൻപുട്ട് ശരിയാണോ എന്ന് പരിശോധിക്കാം.

### ഫീഡ്‌ബാക്ക് സന്ദേശങ്ങൾ കാണിക്കുക

ഇതിന് മുമ്പ് നാം സ്ഥിരമായി ഫലങ്ങൾ പുറത്തുകൊണ്ടുവരാൻ `print` ഉപയോഗിച്ചു.

`print` ന്റെ ഉള്ളടക്കം കോൺസോളിൽ മാത്രം കാണിക്കും. ഡെവലപ്പർമാർക്ക് ഇത് Xcode-ൽ പ്രോഗ്രാം ഓടിക്കുമ്പോൾ കാണാൻ കഴിയും, പക്ഷേ സാധാരണ ഉപയോക്താക്കൾക്ക് ഫോണിൽ ഈ സന്ദേശങ്ങൾ കാണാൻ കഴിയില്ല.

അതിനർത്ഥം, പ്രോഗ്രാം പരിശോധനം പൂർത്തിയാക്കിയിട്ടുണ്ടെങ്കിലും, ഉപയോക്താവിന് നൽകിയ ഉള്ളടക്കം ശരിയാണോ എന്ന് അറിയാൻ സാധിക്കില്ല.

അപ്പോൾ, സ്ക്രീനിൽ നേരിട്ട് സന്ദേശങ്ങൾ കാണിക്കുന്ന ഒരു മാർഗം നമുക്ക് വേണം.

## Alert മുന്നറിയിപ്പ് ബോക്സ്

SwiftUI-ൽ, `Alert` ഉപയോഗിച്ച് ഒരു പോപ്പ്-അപ്പ് സന്ദേശ ബോക്സ് കാണിക്കാം.

അടിസ്ഥാന ഉപയോഗം:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

ഈ കോഡ് മുൻപ് പഠിച്ച കൺട്രോളുകളെക്കാൾ അല്പം കൂടുതൽ സങ്കീർണ്ണമായി തോന്നാം, പക്ഷേ നമുക്ക് ഇത് വിഭജിച്ച് മനസ്സിലാക്കാം.

`Alert` പ്രധാനമായും താഴെ പറയുന്ന ഭാഗങ്ങൾ ഉൾക്കൊള്ളുന്നു:

- `"Title"`: മുന്നറിയിപ്പ് ബോക്സിന്റെ തലക്കെട്ട്
- `isPresented`: മുന്നറിയിപ്പ് ബോക്സ് പ്രദർശിപ്പിക്കണമോ എന്ന് നിയന്ത്രിക്കുന്ന ഒരു ബൗണ്ട് ചെയ്‌ത ബൂളിയൻ മൂല്യം
- `Button(...)`: മുന്നറിയിപ്പ് ബോക്സിലെ ബട്ടൺ
- `message:`: മുന്നറിയിപ്പ് ബോക്സിൽ പ്രദർശിപ്പിക്കുന്ന സന്ദേശത്തിന്റെ ഉള്ളടക്കം

ഇത് ലളിതമായി ഇങ്ങനെ മനസ്സിലാക്കാം: ബൗണ്ട് ചെയ്ത വേരിയബിൾ `true` ആകുമ്പോൾ, സിസ്റ്റം ഈ മുന്നറിയിപ്പ് ബോക്സ് പോപ്പ് അപ് ചെയ്യും.

### Alert ഒരു modifier ആണ്

നാം മുൻപ് പഠിച്ച നിരവധി modifier-ുകളെപ്പോലെ തന്നെ, `Alert`-ഉം ഒരു നിർദ്ദിഷ്ട വ്യൂവിന് ശേഷം ചേർക്കപ്പെടുന്നു.

ഉദാഹരണത്തിന്, ഇത് ബട്ടണിന് ശേഷം ചേർക്കാം:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

ഇത് പുറം `VStack`-ന് ശേഷവും ചേർക്കാം:

```swift
VStack {
    // ...
}
.alert() {...}
```

അതായത്, `Alert` എന്നത് ഇന്റർഫേസിൽ ഒറ്റയ്ക്കു വെക്കുന്ന ഒരു സ്വതന്ത്ര വ്യൂ അല്ല; അത് ഒരു പ്രത്യേക വ്യൂവിനോട് ചേർന്നിരിക്കുന്ന ഒരു modifier ആണ്.

ലളിതമായി പറഞ്ഞാൽ, `Alert` സാധാരണയായി ബട്ടൺ അല്ലെങ്കിൽ പുറം layout ന് ശേഷം എഴുതപ്പെടുന്നു, അതുവഴി ഒരു നിർദ്ദിഷ്ട നിബന്ധന നിറവേറ്റുമ്പോൾ ഒരു സന്ദേശം പോപ്പ് അപ് ചെയ്യാൻ സാധിക്കും.

### Alert കാണിക്കുക

`Alert` കാണിക്കണമെങ്കിൽ, അത് നിയന്ത്രിക്കാൻ ഒരു വേരിയബിൾ ആവശ്യമുണ്ട്.

ഉദാഹരണത്തിന്:

```swift
@State private var showAlert = false
```

ഈ വേരിയബിൾ `Bool` തരം ആണ്, അതായത് ഇതിന് രണ്ട് മൂല്യങ്ങൾ മാത്രമേ ഉണ്ടാകൂ: `true` അല്ലെങ്കിൽ `false`.

തുടർന്ന് ഇത് `Alert` ന്റെ `isPresented`-നോട് bind ചെയ്യുന്നു:

```swift
isPresented: $showAlert
```

`showAlert` `false` ആണെങ്കിൽ, മുന്നറിയിപ്പ് ബോക്സ് കാണിക്കുകയില്ല; `showAlert` `true` ആണെങ്കിൽ, മുന്നറിയിപ്പ് ബോക്സ് പോപ്പ് അപ് ചെയ്യും.

### ബട്ടൺ വഴി മുന്നറിയിപ്പ് ബോക്സ് കാണിക്കുക

ഇപ്പോൾ, ആദ്യം ഏറ്റവും ലളിതമായ സാഹചര്യം നടപ്പിലാക്കാം:

ഉപയോക്താവ് ഒന്നും നൽകാതെ നേരിട്ട് `Sign in` ബട്ടൺ ക്ലിക്ക് ചെയ്യുമ്പോൾ, “നിങ്ങൾ ഇതുവരെ ഒന്നും നൽകിയിട്ടില്ല” എന്ന് പറയുന്ന ഒരു സന്ദേശ ബോക്സ് പോപ്പ് അപ് ചെയ്യാം.

ആദ്യം, ഈ സന്ദേശ ബോക്‌സ് നിയന്ത്രിക്കാൻ ഒരു വേരിയബിൾ സൃഷ്ടിക്കുക:

```swift
@State private var showEmpty = false
```

തുടർന്ന്, `Button` നു ശേഷം ഒരു `Alert` ചേർക്കുക:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

അതിനുശേഷം, ബട്ടൺ ക്ലിക്ക് ചെയ്യുന്ന കോഡിൽ പരിശോധിക്കുക:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

`user` ഉം `password` ഉം രണ്ടും ശൂന്യമായാൽ, നാം `showEmpty` നെ `true` ആക്കും.

അതേസമയം `Alert` ഈ വേരിയബിളിനോട് bind ചെയ്തിരിക്കുന്നതിനാൽ, അത് `true` ആകുമ്പോൾ സിസ്റ്റം സ്വയം മുന്നറിയിപ്പ് ബോക്സ് പ്രദർശിപ്പിക്കും.

പ്രദർശന ഫലം:

![alert](../../Resource/020_alert1.png)

ഇങ്ങനെ ചെയ്താൽ, ഉപയോക്താവിന് ബട്ടൺ ക്ലിക്ക് ചെയ്ത ഉടൻ സന്ദേശം കാണാൻ സാധിക്കും, ഡെവലപ്പർക്ക് `print` output കാണാൻ കോൺസോളിൽ പോകേണ്ടതില്ല.

### ബട്ടണും മുന്നറിയിപ്പ് ബോക്സും മെച്ചപ്പെടുത്തുക

മുൻപ്, “ഒന്നും നിറച്ചിട്ടില്ല” എന്ന സാഹചര്യത്തെ മാത്രമാണ് നാം കൈകാര്യം ചെയ്തത്.

ഇനിയും താഴെ പറയുന്ന സാഹചര്യങ്ങളും കൈകാര്യം ചെയ്യേണ്ടതുണ്ട്:

- ഉപയോക്തൃനാമം നൽകിയിട്ടില്ല
- പാസ്‌വേഡ് നൽകിയിട്ടില്ല
- ഉപയോക്തൃനാമവും പാസ്‌വേഡും നൽകിയിട്ടുണ്ട്, പക്ഷേ ഇൻപുട്ട് തെറ്റാണ്

അതുകൊണ്ട്, ഈ സാഹചര്യങ്ങൾക്കായുള്ള അനുയോജ്യമായ സന്ദേശങ്ങളും തയ്യാറാക്കണം.

ആദ്യം, മൂന്ന് പുതിയ വേരിയബിളുകൾ ചേർക്കുക:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

തുടർന്ന്, ബട്ടണിന് ശേഷം ബന്ധപ്പെട്ട `Alert`-കൾ ചേർക്കുക:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

തുടർന്ന്, ബട്ടണിനുള്ളിലെ തീരുമാന ലാജിക് പൂർത്തിയാക്കുക:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

ബട്ടൺ ഉപയോക്താവ് യാതൊരു വിവരവും നൽകിയിട്ടില്ല, ഒരു വിവരം വിട്ടുപോയിട്ടുണ്ട്, അല്ലെങ്കിൽ ഇൻപുട്ട് സാധൂകരണം പരാജയപ്പെട്ടു എന്ന് നിർണയിക്കുമ്പോൾ, അതിന് ബന്ധപ്പെട്ട Boolean വേരിയബിൾ `true` ആക്കും, പിന്നെ അതിനു ചേരുന്ന `Alert` പോപ്പ് അപ് ചെയ്യും.

ഇവിടെ ഞാൻ ചില സാഹചര്യങ്ങൾ നിങ്ങൾ സ്വയം പരീക്ഷിക്കണമെന്ന് നിർദേശിക്കുന്നു:

- ഒന്നും നിറയ്ക്കാതിരിക്കുക
- ഉപയോക്തൃനാമം മാത്രം നൽകുക
- പാസ്‌വേഡ് മാത്രം നൽകുക
- ഉപയോക്തൃനാമവും പാസ്‌വേഡും രണ്ടും നൽകുക, പക്ഷേ തെറ്റായ ഉള്ളടക്കത്തോടെ
- ഉപയോക്തൃനാമവും പാസ്‌വേഡും രണ്ടും നൽകുക, ഉള്ളടക്കം ശരിയായിരിക്കണം

ഇതിലൂടെ നമ്മുടെ കോഡിൽ പ്രശ്നമുണ്ടോ എന്ന് പരിശോധിക്കാം. ഇത് ആപ്പ് വികസനത്തിലെ വളരെ പ്രധാനപ്പെട്ട ഒരു “testing” ഘട്ടവുമാണ്.

ഏതെങ്കിലും സാഹചര്യത്തിൽ പ്രോഗ്രാം ശരിയായി പ്രവർത്തിക്കാത്തപക്ഷം, ലാജിക്കിൽ പ്രശ്നമുണ്ടാകാമെന്നും പ്രോഗ്രാമിൽ ഒരു `Bug` ഉണ്ടായിരിക്കാമെന്നും അത് സൂചിപ്പിക്കുന്നു.

വികസനത്തിൽ സാധാരണയായി `Bug` എന്നു പറയുന്നത്, ഒരു പിശക്, പോരായ്മ, അല്ലെങ്കിൽ പ്രതീക്ഷിച്ചതിനൊത്തല്ലാത്ത പെരുമാറ്റം എന്നിവയെ സൂചിപ്പിക്കുന്നു.

## ലോഗിൻ വ്യൂ പൂർത്തിയാക്കുക

മുൻപ് പറഞ്ഞ `Alert` ഉദാഹരണങ്ങൾ എല്ലാം “ഇൻപുട്ട് തെറ്റായപ്പോൾ” ഉപയോക്താവിന് എങ്ങനെ സന്ദേശം നൽകണം എന്നതിനെക്കുറിച്ചായിരുന്നു.

ഉപയോക്താവ് ശരിയായ ഉപയോക്തൃനാമവും പാസ്‌വേഡും നൽകിയാൽ, നാം സാധാരണയായി “മുന്നറിയിപ്പ് ബോക്സ്” കാണിക്കാറില്ല; പകരം നേരിട്ട് ലോഗിൻ കഴിഞ്ഞുള്ള ഇന്റർഫേസിലേക്ക് കടക്കും.

ഇവിടെ വളരെ പ്രധാനപ്പെട്ട ഒരു ആശയമുണ്ട്: **വേരിയബിളിന്റെ മൂല്യത്തിനനുസരിച്ച് വ്യത്യസ്ത വ്യൂകൾ കാണിക്കുക.**

ഉദാഹരണത്തിന്:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

ഇവിടെ, വ്യൂയുടെ പ്രദർശനം നിയന്ത്രിക്കാൻ നാം `Bool` തരം ഉപയോഗിക്കുന്നു:

- `login` `true` ആണെങ്കിൽ, അതിന് ലോഗിൻ ഇതിനകം വിജയിച്ചു എന്നും ലോഗിൻ ശേഷമുള്ള ഉള്ളടക്കം കാണിക്കപ്പെടുന്നു എന്നും അർത്ഥം
- `login` `false` ആണെങ്കിൽ, ലോഗിൻ വിജയിച്ചിട്ടില്ല എന്നും ലോഗിൻ ഇന്റർഫേസ് തുടര്ന്നും പ്രദർശിപ്പിക്കപ്പെടുന്നു എന്നും അർത്ഥം.

`if` സ്റ്റേറ്റ്മെന്റ് Swift-ലുള്ള ഒരു conditional statement ആണെങ്കിലും, SwiftUI യുടെ `body`-യിൽ അതിനെ വ്യൂ പ്രദർശന ലാജിക് നിയന്ത്രിക്കാൻ ഉപയോഗിക്കാം.

അതാണ്, നിബന്ധനയുടെ ഫലത്തിന്റെ അടിസ്ഥാനത്തിൽ ഇപ്പോൾ ഏത് വ്യൂ കാണിക്കണമെന്ന് SwiftUI തീരുമാനിക്കും.

ഈ പ്രദർശന ലാജിക് നിലവിലെ `ContentView`-ലേക്ക് ഇടാം:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Original login view code
		}
	}
}
```

ഇവിടെ, ഉപയോക്താവ് വിജയകരമായി ലോഗിൻ ചെയ്തോയെന്ന് സൂചിപ്പിക്കാൻ `login` എന്ന പുതിയ വേരിയബിൾ ചേർത്തു.

ശേഷം, മുൻപ് എഴുതിയ ലോഗിൻ വ്യൂയുടെ കോഡ് `else` സ്റ്റേറ്റ്മെന്റിന് ശേഷം വെച്ചു.

വ്യൂ പ്രദർശന ലാജിക്:

- `login` `false` ആയിരിക്കുമ്പോൾ, ലോഗിൻ ഇപ്പോഴും വിജയിച്ചിട്ടില്ല എന്നതാണ് അർത്ഥം, അതിനാൽ പഴയ ലോഗിൻ വ്യൂ തുടർന്നും കാണിക്കും.
- `login` `true` ആയിരിക്കുമ്പോൾ, അത് ലോഗിൻ വിജയിച്ചുവെന്ന് സൂചിപ്പിക്കുന്നു; ആ സമയത്ത് ലോഗിൻ വ്യൂ ഇനി കാണിക്കില്ല, പകരം `Login successful` കാണിക്കും.

അതിനർത്ഥം, `ContentView` `login` ന്റെ മൂല്യത്തിനനുസരിച്ച് വ്യത്യസ്തമായ വ്യൂ ഉള്ളടക്കം പ്രദർശിപ്പിക്കും.

ഇപ്പോൾ, ബട്ടണിനുള്ളിലെ ലോഗിൻ വിജയ ലാജിക് നമുക്ക് തിരുത്താം:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

ഉപയോക്താവ് നൽകിയ ഉപയോക്തൃനാമവും പാസ്‌വേഡും രണ്ടും ശരിയാണെങ്കിൽ, നാം `login` നെ `true` ആക്കും.

`login` ന്റെ മൂല്യം മാറുന്ന മുറയ്ക്ക്, `ContentView` ഉള്ളിലെ `if` പരിശോധനയുടെ ഫലവും മാറും, അതിനാൽ ഇന്റർഫേസ് പഴയ ലോഗിൻ വ്യൂയിൽ നിന്ന് `Login successful` എന്നതിലേക്ക് മാറും.

പ്രദർശന ഫലം താഴെപറയുന്നതുപോലെയാണ്:

![view](../../Resource/020_view.png)

ഇപ്പോൾ, “നിബന്ധനയെ ആശ്രയിച്ച് വ്യത്യസ്ത വ്യൂകൾ പ്രദർശിപ്പിക്കൽ” എന്ന അറിവ് നാം ഈ നിലവിലെ ലോഗിൻ പേജിൽ യഥാർത്ഥത്തിൽ ഉപയോഗിച്ചിരിക്കുന്നു.

യഥാർത്ഥ ആപ്പുകളിലും സാധാരണയായി ഇതേ രീതിയാണ്: ഒരു നിർദ്ദിഷ്ട മൂല്യത്തിന്റെ അടിസ്ഥാനത്തിൽ വ്യത്യസ്ത വ്യൂകൾ പ്രദർശിപ്പിക്കുന്നു.

ഉദാഹരണത്തിന്, ഉപയോക്താവ് ലോഗിൻ ചെയ്തിട്ടില്ലെങ്കിൽ ലോഗിൻ പേജ് കാണിക്കും; ഉപയോക്താവ് വിജയകരമായി ലോഗിൻ ചെയ്ത ശേഷം മറ്റൊരു ഇന്റർഫേസ് കാണിക്കും.

പ്രധാന ആശയം ഇതാണ്: **വേരിയബിളുകളുടെ മാറ്റങ്ങളിലൂടെ ഇന്റർഫേസിൽ എന്ത് ഉള്ളടക്കം കാണിക്കണം എന്ന് നിയന്ത്രിക്കൽ.**

## സംഗ്രഹം

ഈ പാഠത്തിൽ, നാം മൂന്ന് പ്രധാന കാര്യങ്ങൾ പഠിച്ചു:

1. പാസ്‌വേഡ് നൽകാൻ `SecureField` ഉപയോഗിക്കുക, അതുവഴി പാസ്‌വേഡ് ഉള്ളടക്കം ഇന്റർഫേസിൽ നേരിട്ട് കാണിക്കപ്പെടില്ല.
2. കോൺസോളിൽ മാത്രം output കാണിക്കുന്നതിനുപകരം, ഉപയോക്താവിനെ സന്ദേശങ്ങൾ കാണിക്കാൻ `Alert` ഉപയോഗിക്കുക.
3. വ്യത്യസ്ത വ്യൂകളുടെ പ്രദർശനം നിയന്ത്രിക്കാനും, ലോഗിൻ ചെയ്യുന്നതിനുമുൻപും ശേഷവും ഉള്ള ഇന്റർഫേസുകൾ തമ്മിൽ സ്വിച്ച് ചെയ്യാനും വേരിയബിളുകളും `if` നിബന്ധനകളും ഉപയോഗിക്കുക.

ഈ അറിവുകൾ പഠിച്ചതിലൂടെ, ഇനി നമുക്ക് അടിസ്ഥാനപരമായ ഇന്ററാക്ഷൻ ഫലങ്ങളുള്ള ഒരു ലോഗിൻ വ്യൂ പൂർത്തിയാക്കാൻ കഴിയും, കൂടാതെ SwiftUI യിലെ “state മാറുമ്പോൾ interface പുതുക്കപ്പെടുന്നു” എന്ന വികസന ആശയവും കൂടുതൽ വ്യക്തമായി മനസ്സിലാക്കാം.

```swift
struct ContentView: View {

    @State private var showMissPassword = false
    @State private var showWrongInfo = false

    let userID = "FangJunyu"

                        .frame(width: 200)

                        .fontWeight(.bold)

                        .frame(width: 200)
                        .border(Color.gray,width: 1)

                Button("Sign in") {

                        print("Empty")

                    } else if user.isEmpty || password.isEmpty {

                            print("Please enter your username.")

                        } else if password.isEmpty {
                            print("Please enter your password.")

                        }

                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}

                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {

                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}

        }

```

## വിപുലീകരിച്ച അറിവ് - ഒരിലധികം ബട്ടണുകളുള്ള Alert

ഈ പാഠത്തിന്റെ മുൻ ഭാഗത്തിൽ, നാം ഏറ്റവും അടിസ്ഥാനപരമായ `Alert` ആണ് ഉപയോഗിച്ചത്, അതിൽ ഒരു ബട്ടൺ മാത്രമേ ഉണ്ടായിരുന്നുള്ളൂ, പ്രധാനമായും ഉപയോക്താവിനെ അറിയിക്കാൻ അത് ഉപയോഗിച്ചിരുന്നു.

വാസ്തവത്തിൽ, `Alert`-ൽ ഒരിലധികം ബട്ടണുകളും ഉൾക്കൊള്ളാം, ഇതിലൂടെ ഉപയോക്താവിന് വ്യത്യസ്ത തീരുമാനങ്ങൾ എടുക്കാനാകും.

ഉദാഹരണത്തിന്:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

ബട്ടൺ ക്ലിക്ക് ചെയ്ത ശേഷം, `Ok`, `delete`, `cancel` എന്ന മൂന്ന് ഓപ്ഷനുകളുള്ള ഒരു സന്ദേശ ബോക്സ് പോപ്പ് അപ് ചെയ്യും.

പ്രദർശന ഫലം:

![alert](../../Resource/020_alert2.png)

വ്യത്യസ്ത ബട്ടണുകൾ ക്ലിക്ക് ചെയ്താൽ അനുയോജ്യമായ കോഡ് പ്രവർത്തിക്കും.

### Button ന്റെ `role` പ്രോപ്പർട്ടി

ഇവിടെ നീ ശ്രദ്ധിക്കും, `Alert` ന്റെ ഉള്ളിലുള്ള `Button`, മുമ്പ് പഠിച്ച സാധാരണ ബട്ടൺ syntax-നേക്കാൾ അല്പം വ്യത്യസ്തമാണ്:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

ഇവിടെ അധികമായി ഒരു `role` parameter ഉണ്ട്.

SwiftUI-ൽ, ഈ ബട്ടൺ ഏത് തരത്തിലുള്ള പ്രവർത്തനത്തെ പ്രതിനിധീകരിക്കുന്നു എന്ന് സിസ്റ്റത്തിനോട് പറയാൻ `role` ഉപയോഗിക്കുന്നു.

ചില സാധാരണ roles ഇവയാണ്:

- `.cancel`: റദ്ദാക്കൽ പ്രവർത്തനം
- `.destructive`: അപകടകരമായ പ്രവർത്തനം, ഉദാഹരണത്തിന് delete
- `nil`: സാധാരണ പ്രവർത്തനം

വ്യത്യസ്ത `role` മൂല്യങ്ങൾ സാധാരണയായി ഇന്റർഫേസിൽ വ്യത്യസ്ത visual styles കൊണ്ടുവരും.

ഉദാഹരണത്തിന്, അപകടകരമായ പ്രവർത്തനങ്ങൾ സാധാരണയായി കൂടുതൽ ശ്രദ്ധാകർഷകമായ രീതിയിൽ കാണിക്കും, ഉപയോക്താവിനെ ശ്രദ്ധാപൂർവ്വം ക്ലിക്ക് ചെയ്യാൻ ഓർമ്മിപ്പിക്കാൻ.

കൂടാതെ, `role` ബട്ടണിന് അധിക semantic വിവരങ്ങളും നൽകുന്നു, ഇതിലൂടെ screen readers-ന് (ഉദാഹരണത്തിന് Apple ഉപകരണങ്ങളിലെ VoiceOver) ഉപയോക്താവിന് ബട്ടണിന്റെ ഉദ്ദേശ്യം വിശദീകരിക്കാൻ എളുപ്പമാകും.

അതുകൊണ്ട്, അനുയോജ്യമായ സാഹചര്യങ്ങളിൽ ഒരു ബട്ടണിൽ `role` ചേർക്കുന്നത് വളരെ നല്ലൊരു ശീലമാണ്.
