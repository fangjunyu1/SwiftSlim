# دورة حياة واجهة SwiftUI

في الدرس السابق، عرفنا أن متغيرات `@State` ترتبط بدورة حياة الواجهة، كما تعلمنا الاستخدام الأساسي لـ `onAppear`.

في هذا الدرس، سنفهم بشكل أعمق دورة حياة واجهات SwiftUI، ونتعرف على كيفية إنشاء الواجهة وعرضها، وكيف يتم إنشاء المتغيرات داخل الواجهة وإعادة إنشائها، كما سنتعلم `onAppear` و`onDisappear` والدالة البانية `init`.

وبناءً على هذه المعرفة، سنكوّن فهمًا أشمل لعملية بناء واجهات SwiftUI بالكامل.

## ترتيب تشغيل التطبيق

عند إنشاء مشروع iOS، يقوم Xcode افتراضيًا بإنشاء ملفين:

```text
ContentView.swift
اسم_المشروع + App.swift
```

ومن بينهما، يُعد الملف `"اسم المشروع + App".swift` ملف الدخول الرئيسي للتطبيق بأكمله.

على سبيل المثال:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

### مسار تنفيذ الكود

عندما نقوم بتشغيل التطبيق (على المحاكي أو على جهاز حقيقي)، يبحث النظام أولًا عن الكلمة المفتاحية `@main`:

```swift
@main
struct SwiftSlimTestApp: App {...}
```

وبعد تحديد ملف الدخول، يبدأ تنفيذ الكود.

أولًا يدخل إلى هيكل `App`، وينفذ الكود داخل `body`، ثم ينشئ `WindowGroup` ويحمّل بداخله `ContentView`.

### دور WindowGroup

يُستخدم `WindowGroup` لإدارة النوافذ:

```swift
WindowGroup {
    ContentView()
}
```

في أنظمة iPad وmacOS توجد إمكانية دعم تعدد النوافذ، أما على iPhone فعادةً توجد نافذة واحدة فقط.

لذلك، يقوم `WindowGroup` على iPhone بشكل أساسي بإدارة أول واجهة يتم عرضها.

### عملية تحميل الواجهة

بعد أن يجد النظام `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
    }
}
```

ستقوم SwiftUI بتنفيذ الكود الموجود داخل `body`، ثم تبني الواجهة وتعرضها على الشاشة وفقًا للبنية المعادة من الواجهة، مثل `VStack` و`Text` وغير ذلك.

بعد اكتمال هذه الخطوات، يمكننا رؤية `ContentView` على الشاشة.

![ContentView](../../Resource/016_view1.png)

ومن المهم الانتباه إلى أن وظيفة `body` هي إنشاء الواجهة، وليست تخزين الواجهة.

وهذا يعني أنه في كل مرة يتم فيها تحديث الواجهة، تعيد SwiftUI حساب `body` وتُنشئ بنية واجهة جديدة.

### منطق واجهة المعاينة

المعاينة في Xcode (`Canvas`) وآلية التشغيل الفعلية هما آليتان مختلفتان.

على سبيل المثال، إذا أضفنا مخرجات تصحيح في ملف الدخول:

```swift
@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .onAppear {
                    print("Go to ContentView")
                }
        }
    }
}
```

عند معاينة واجهة `ContentView` داخل Xcode، لن يبدأ التنفيذ من مدخل `@main`، ولذلك لن يتم تشغيل `onAppear` الموجود في ملف الدخول.

لكن إذا قمنا بتشغيل التطبيق على المحاكي أو على جهاز فعلي، فسيبدأ النظام التنفيذ من `@main`، ويتبع المسار الكامل لتشغيل التطبيق، وعندها سيتم استدعاء `onAppear` الموجود في المدخل، كما ستظهر معلومات التصحيح.

السبب الأساسي هو أن معاينة Xcode ليست إلا "تصييرًا جزئيًا"، فهي تُستخدم فقط لعرض الواجهة الحالية، بينما يوفّر المحاكي والجهاز الحقيقي "بيئة تشغيل كاملة" تقوم بتنفيذ التطبيق بأكمله.

لذلك، عند اختبار وظائف التطبيق، يجب الاعتماد على المحاكي أو الجهاز الحقيقي، لأن معاينة Xcode لا توفّر بيئة تشغيل كاملة.

## دورة حياة الواجهة

في المرحلة الحالية، تتركز جميع أكوادنا داخل `ContentView`. لكن في التطبيق الحقيقي، يحتوي التطبيق عادةً على عدة واجهات، ويتم التبديل بينها في سيناريوهات مختلفة.

على سبيل المثال، في "الحصالة":

![PiggyBank](../../Resource/016_view.png)

عند الضغط على "الإيداع"، يتم عرض واجهة `SaveView`، وعند انتهاء الإيداع أو إغلاقها، تتم إزالة واجهة `SaveView`.

وهذه العملية تمثل دورة حياة الواجهة: إنشاء الواجهة، ثم عرضها، ثم إزالتها في النهاية.

### إغلاق التطبيق

عندما يتم إغلاق التطبيق، تتم إزالة جميع الواجهات، كما تختفي الحالات المرتبطة بها.

لذلك سيتم إزالة `ContentView` والواجهات الأخرى من الذاكرة، وسيتم أيضًا مسح حالة تشغيل التطبيق بالكامل.

## دورة حياة المتغيرات داخل الواجهة

في SwiftUI، ترتبط دورة حياة المتغيرات غالبًا بالواجهة نفسها.

على سبيل المثال:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"
    var body: some View {
        var num = 10
        VStack {
            Text("num:\(num)")
            Text("name:\(name)")
        }
    }
}
```

### متغيرات @State

المتغيرات المعلنة باستخدام `@State` ترتبط بدورة حياة الواجهة.

```swift
@State private var name = "FangJunyu"
```

عندما يتم إنشاء الواجهة، يتم أيضًا إنشاء `name`، وعندما تتم إزالة الواجهة، يتم التخلص من `name`.

ولهذا السبب نحتاج إلى استخدام `UserDefaults` أو وسائل مشابهة لتنفيذ الحفظ الدائم للبيانات.

### المتغيرات داخل body

إذا عرّفنا متغيرًا داخل `body`:

```swift
var num = 10
```

فإن دورة حياته ترتبط بعملية تنفيذ `body`.

وعندما تتغير حالة SwiftUI، مثلًا:

```swift
@State private var name = "FangJunyu"
name = "Hu"
```

عندما تتغير قيمة `name`، يكتشف `@State` هذا التغيير في البيانات، ويقوم بإبلاغ SwiftUI لإعادة حساب الواجهة، فيُعاد حساب `body`.

وعندما تتم إعادة حساب `body`، يُعاد تنفيذ جميع الأكواد الموجودة بداخله، كما تُعاد أيضًا إنشاء المتغيرات الموجودة داخله، مثل `num`.

ولهذا السبب أيضًا لا يُنصح بتعريف متغيرات معقدة داخل `body`.

لأن المتغيرات الموجودة في `body` ستُعاد إنشاؤها في كل مرة يتم فيها تحديث الواجهة، وهذا يزيد من تكلفة الحساب ويؤثر على الأداء.

في SwiftUI، يجب إدارة أنواع البيانات المختلفة بطرق مختلفة. البيانات التي ينبغي أن ترتبط بدورة حياة الواجهة يمكن حفظها باستخدام `@State`، أما البيانات المؤقتة المستخدمة في الحساب فيمكن وضعها داخل `body`.

## onAppear و onDisappear

في الدرس السابق، تعلمنا بالفعل `onAppear`، وهي تُستدعى عندما تظهر الواجهة.

```swift
.onAppear {}
```

على سبيل المثال:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
        .onAppear {
            print("Show ContentView")
        }
    }
}
```

في معاينة Xcode أو أثناء التشغيل، يمكن رؤية مخرجات التصحيح الموجودة داخل `onAppear`.

### onDisappear

والمقابل لـ `onAppear` هو `onDisappear`:

```swift
.onDisappear {}
```

عندما يتم إغلاق الواجهة، يتم استدعاء `onDisappear`.

على سبيل المثال:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

عند إزالة الواجهة، سيتم تنفيذ الكود الموجود داخل `onDisappear`.

ملاحظة: في هذه المرحلة، تُعد `ContentView` الواجهة الجذرية للتطبيق، ولا يمكن إغلاقها أو إزالتها مباشرة. لذلك، لا يمكننا ملاحظة تأثير تنفيذ `onDisappear` في هذه المرحلة.

وفي الدروس اللاحقة، عندما نتعلم الانتقال بين الصفحات أو فتح واجهات جديدة، سنرى استدعاء `onDisappear` بشكل فعلي.

## منطق الإنشاء والعرض

من المهم ملاحظة أن إنشاء الواجهة وعرضها هما مرحلتان مختلفتان.

عند إنشاء الواجهة، يتم استدعاء الدالة البانية الخاصة بالهيكل:

```swift
init() {}
```

ولأن واجهات SwiftUI هي هياكل `struct`:

```swift
struct ContentView: View { ... }
```

فعندما يتم إنشاء نسخة من الواجهة، يتم تنفيذ الدالة البانية `init`. أما عندما يتم عرض الواجهة، فعندها فقط يتم استدعاء `onAppear`.

على سبيل المثال:

```swift
struct ContentView: View {
    init() {
        print("Create ContentView")
    }
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

عند معاينة الواجهة في Xcode، يمكننا رؤية مخرجات التصحيح التالية:

```swift
Create ContentView
Show ContentView
```

وهذا يوضح أنه يتم أولًا تنفيذ `init` لإنشاء الواجهة، ثم حساب الكود الموجود داخل `body`، ثم عرض الواجهة وتنفيذ `onAppear`.

لذلك يجب فهم أن مرحلتي إنشاء الواجهة وعرضها منفصلتان عن بعضهما.

### الدالة البانية init

تُعد `init` من القواعد الأساسية في Swift، ويمكن تعريفها في كل من `struct` و`class`، وتُستخدم لتهيئة الكائنات.

```swift
init() {}
```

في SwiftUI، عندما يتم إنشاء الواجهة، يستدعي النظام الدالة `init` الخاصة بها، ويمكن استخدامها لاستقبال المعاملات أو تهيئة البيانات.

إذا لم نكتب `init` يدويًا، فستقوم Swift بإنشاء دالة بانية افتراضية لـ `struct` تلقائيًا.

وبالنسبة لواجهات SwiftUI مثل `ContentView`، فعند إنشاء الواجهة يتم تنفيذ `init`، وعند عرض الواجهة يتم تنفيذ `onAppear`.

لذلك فإن `init` هي الدالة البانية التي تُنفذ عند إنشاء الواجهة، وسنستخدمها لاحقًا عند تمرير المعاملات أو تهيئة البيانات.

## ملخص

في هذا الدرس تعلمنا ترتيب تشغيل التطبيق، من ملف الدخول وحتى مسار التنفيذ إلى ملف `ContentView`.

كما تعرفنا على دورة حياة واجهات SwiftUI: فعند إنشاء الواجهة يتم تنفيذ `init`، وعند ظهورها على الشاشة يتم تنفيذ `onAppear`، وعند إزالة الواجهة أو إغلاقها يتم تنفيذ `onDisappear`.

وتعلمنا أيضًا آلية تحديث الواجهة: فالواجهة مدفوعة بالحالة، وعندما تتغير حالات مثل `@State`، تقوم SwiftUI بتحديث الواجهة، وإعادة حساب `body`، كما تُعاد أيضًا إنشاء المتغيرات الموجودة داخله.

وترتبط دورة حياة المتغيرات بدورة حياة الواجهة، أما المتغيرات المؤقتة داخل `body` فتُعاد إنشاؤها في كل مرة يتم فيها التحديث.

إن فهم هذه الدورات السلوكية للواجهة والمتغيرات يساعدنا على تنظيم الكود بشكل أفضل، ويجعل منطق التطبيق أوضح.
