# التكيف مع المظهر

في هذا الدرس، سنتعلم التكيف مع مظهر الوضع الفاتح والوضع الداكن.

على iPhone و iPad و Mac، يمكن للمستخدم اختيار الوضع الفاتح أو الوضع الداكن.

عندما يتحول النظام إلى الوضع الداكن، تتغير أيضًا ألوان الخلفية والنصوص والأيقونات في كثير من التطبيقات.

على سبيل المثال، قد تكون الواجهة في النهار بخلفية فاتحة، وقد تتحول في الليل إلى خلفية داكنة.

![view](../../../Resource/025_view9.png)

تسمى هذه القدرة على تعديل عرض الواجهة تلقائيًا وفقًا لمظهر النظام باسم التكيف مع المظهر.

## معاينة التكيف مع المظهر في Xcode

قبل تعلم التكيف مع المظهر، نحتاج أولًا إلى فهم كيفية معاينة الوضع الفاتح والوضع الداكن في Xcode.

في أسفل منطقة `Canvas`، انقر زر `Device Settings` لفتح نافذة `Canvas Device Settings`.

![canvas](../../../Resource/025_view.png)

في النافذة، يمكنك رؤية إعداد `Color Scheme`.

![canvas1](../../../Resource/025_view1.png)

وفيه:

- يشير `Light Appearance` إلى الوضع الفاتح.
- يشير `Dark Appearance` إلى الوضع الداكن.

يمكننا التبديل من هنا بين مظاهر المعاينة، ومشاهدة تأثير عرض الواجهة الحالية في الوضع الفاتح والوضع الداكن.

## عرض المظهرين جنبًا إلى جنب

إذا أردت مشاهدة الوضع الفاتح والوضع الداكن في الوقت نفسه، يمكنك استخدام ميزة `Variants`.

في أسفل منطقة `Canvas`، انقر زر `Variants`، ثم اختر `Color Scheme Variants`.

![canvas2](../../../Resource/025_view2.png)

بعد الاختيار، سيعرض `Canvas` معاينة الوضع الفاتح والوضع الداكن في الوقت نفسه.

![canvas3](../../../Resource/025_view3.png)

بهذه الطريقة يمكن مقارنة اختلافات الواجهة بين المظهرين بسهولة أكبر.

ببساطة:

إذا أردت التبديل مؤقتًا بين الوضع الفاتح/الداكن فقط، يمكنك استخدام `Canvas Device Settings`.

إذا أردت مشاهدة المظهرين في الوقت نفسه، يمكنك استخدام `Color Scheme Variants`.

## تتكيف العروض افتراضيًا مع المظهر

في SwiftUI، تتكيف كثير من عروض النظام تلقائيًا بشكل افتراضي مع الوضع الفاتح والوضع الداكن.

على سبيل المثال:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("فاتح")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("نص نص نص")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

تأثير العرض:

![canvas](../../../Resource/025_view4.png)

يمكنك أن ترى أنه في الوضع الفاتح، تظهر الأيقونة والعنوان عادةً بلون داكن.

وفي الوضع الداكن، تتحول الأيقونة والعنوان تلقائيًا إلى لون فاتح.

وذلك لأن عروض SwiftUI مثل `Text` و `Image` تضبط ألوانها افتراضيًا وفقًا لمظهر النظام.

بمعنى آخر، إذا لم نحدد لونًا ثابتًا يدويًا، فسيساعدنا SwiftUI في التعامل مع جزء من التكيف مع المظهر.

## الألوان الثابتة لا تتبدل تلقائيًا

يجب الانتباه إلى أنه إذا حددنا لونًا ثابتًا يدويًا، فلن يتبدل تلقائيًا وفقًا للوضع الفاتح/الداكن.

على سبيل المثال:

```swift
Text("نص نص نص")
	.foregroundStyle(Color.gray)
```

هنا تم تحديد النص على أنه `Color.gray`، لذلك سيظل يظهر دائمًا باللون الرمادي.

ومثال آخر:

```swift
Text("عنوان")
    .foregroundStyle(Color.white)
```

سيعرض هذا الكود النص باللون الأبيض سواء كان في الوضع الفاتح أو الوضع الداكن.

إذا كانت الخلفية فاتحة أيضًا، فقد يصبح النص الأبيض غير واضح.

لذلك عند تنفيذ التكيف مع المظهر، حاول قدر الإمكان تجنب تثبيت ألوان مثل `Color.white` و `Color.black` عشوائيًا.

في كثير من الحالات، يمكن إعطاء الأولوية لاستخدام أنماط النظام الدلالية، مثل:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

وفيها:

- تُستخدم `.primary` عادةً للمحتوى الرئيسي.
- تُستخدم `.secondary` عادةً للمحتوى الثانوي.

ستضبط هذه الأنماط تأثير العرض تلقائيًا وفقًا للوضع الفاتح/الداكن.

## عرض محتوى مختلف حسب المظهر

أحيانًا لا نريد تغيير اللون فقط، بل نريد أيضًا عرض محتوى مختلف في كل مظهر.

على سبيل المثال:

- في الوضع الفاتح، اعرض أيقونة مصباح عادية.
- في الوضع الداكن، اعرض أيقونة مصباح مضيء.
- في الوضع الفاتح، يعرض العنوان `فاتح`.
- في الوضع الداكن، يعرض العنوان `داكن`.

في هذه الحالة، نحتاج إلى تحديد ما إذا كان النظام الحالي في الوضع الفاتح أم الوضع الداكن.

في SwiftUI، يمكن الحصول على وضع المظهر الحالي من خلال قيمة البيئة `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

الكود الكامل:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "فاتح" : "داكن"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("نص نص نص")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

تأثير العرض:

![view](../../../Resource/025_view5.png)

من تأثير المعاينة، يمكننا رؤية أن العرض نفسه لا يعرض محتوى مطابقًا تمامًا في أنماط المظهر المختلفة.

في الوضع الفاتح، يعرض `Image` أيقونة المصباح العادية، ويعرض العنوان `فاتح`.

في الوضع الداكن، يعرض `Image` أيقونة المصباح المضيء، ويعرض العنوان `داكن`.

أي أن التغيير هنا ليس مجرد تغيير في اللون، بل تغير اسم الأيقونة ونص العنوان نفسيهما أيضًا.

وذلك لأننا لم نكتب الأيقونة والعنوان مباشرةً بشكل ثابت داخل `body`، بل حسبنا محتوى مختلفًا وفقًا لـ `colorScheme`.

في هذا الكود، استخدمنا خاصيتين محسوبتين:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "فاتح" : "داكن"
}
```

عندما تكون قيمة `colorScheme` هي `.light`، فهذا يعني أن الوضع الحالي هو الوضع الفاتح.

وعندما تكون قيمة `colorScheme` هي `.dark`، فهذا يعني أن الوضع الحالي هو الوضع الداكن.

لذلك يمكننا إرجاع اسم أيقونة ونص عنوان مختلفين وفقًا لأنماط المظهر المختلفة.

وهذا أيضًا استخدام شائع للخاصية المحسوبة: حساب المحتوى الذي يحتاج العرض إلى إظهاره وفقًا للحالة الحالية.

## فهم @Environment

هذه هي المرة الأولى التي نتعامل فيها مع `@Environment`.

يمكن فهم `@Environment` على أنه: قراءة قيمة من بيئة SwiftUI.

أثناء تشغيل App، يوفر النظام كثيرًا من معلومات البيئة، مثل:

- اللغة الحالية
- وضع المظهر الحالي
- اتجاه التخطيط الحالي
- إعداد حجم الخط الحالي

عندما نحتاج إلى الحصول على وضع المظهر الحالي، يمكننا قراءة `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

وفيه

```swift
\.colorScheme
```

يعني قراءة قيمة `colorScheme` من البيئة.

```swift
private var colorScheme
```

يعني حفظ القيمة المقروءة في المتغير `colorScheme`.

يمكنك اختيار اسم المتغير بنفسك، مثل كتابته أيضًا على النحو التالي:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

طالما أن `\.colorScheme` في البداية لم يتغير، فهذا يعني أننا نقرأ وضع المظهر الحالي.

## القيمتان الشائعتان لـ colorScheme

القيمتان الشائعتان لـ `colorScheme` هما:

```swift
.light
.dark
```

وهما تعنيان:

- `.light`: الوضع الفاتح
- `.dark`: الوضع الداكن

على سبيل المثال:

```swift
private var titleName: String {
    colorScheme == .light ? "فاتح" : "داكن"
}
```

معنى هذا الكود هو:

إذا كان الوضع الحالي هو الوضع الفاتح، فسيكون الشرط `colorScheme == .light` صحيحًا، وسيُرجع `"فاتح"`.

وإلا فسيُرجع `"داكن"`.

لذلك، في الوضع الفاتح:

```swift
Text(titleName)
```

سيعرض:

```swift
فاتح
```

وفي الوضع الداكن، سيعرض:

```swift
داكن
```

بهذه الطريقة، يمكننا عرض محتوى مختلف حسب المظهر المختلف.

## مثال على تكييف الألوان

إلى جانب النصوص والأيقونات، نحتاج أحيانًا إلى التعامل مع تكييف الألوان بأنفسنا.

على سبيل المثال:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("نص")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

تأثير العرض:

![view](../../../Resource/025_view6.png)

في هذا الكود، النص أبيض، والخلفية سوداء.

في الوضع الفاتح، تكون الخلفية السوداء واضحة نسبيًا.

لكن في الوضع الداكن، إذا كانت خلفية الواجهة كلها سوداء أيضًا، فستندمج هذه الخلفية السوداء مع خلفية النظام، وستبدو أقل وضوحًا.

في هذه الحالة، يمكننا تبديل لون النص ولون الخلفية وفقًا لوضع المظهر:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("نص")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

تأثير العرض:

![view](../../../Resource/025_view7.png)

الآن، في الوضع الفاتح لدينا خلفية سوداء ونص أبيض.

وفي الوضع الداكن لدينا خلفية بيضاء ونص أسود.

بهذا الشكل، سيكون هناك تباين واضح بين النص والخلفية في أي مظهر، وسيتمكن المستخدم من رؤية المحتوى بوضوح.

## استخدام Assets لتكييف الألوان

إلى جانب استخدام `colorScheme` للحكم اليدوي، يمكننا أيضًا استخدام ألوان موارد `Assets` للتكيف مع الوضع الفاتح/الداكن.

هذه الطريقة أنسب لإدارة الألوان العامة داخل App.

في مشروع Xcode، ابحث عن مجلد موارد `Assets`.

![assets](../../../Resource/025_color.png)

افتح مجلد `Assets`، وانقر بزر الفأرة الأيمن على المساحة الفارغة، ثم اختر `New Color Set` لإنشاء مورد لون جديد.

![assets](../../../Resource/025_color1.png)

هنا ننشئ لونًا ونسميه `redText`.

![assets](../../../Resource/025_color2.png)

يمكن لمورد اللون تعيين ألوان مختلفة للوضع الفاتح والوضع الداكن بشكل منفصل.

![assets](../../../Resource/025_color3.png)

بعد اختيار منطقة اللون المناسبة، يمكنك تعديل اللون من مفتش الخصائص على اليمين.

انقر زر `Hide or show the Inspectors` في الزاوية العلوية اليمنى لفتح منطقة المفتش.

ثم اختر `Show the Attributes inspector`، وابحث في الأسفل عن `Show Color Panel` لفتح لوحة الألوان.

![assets](../../../Resource/025_color4.png)

عند النقر على اللون داخل لوحة الألوان، ستتغير منطقة اللون المقابلة لـ `redText` بالتزامن.

هنا نضبط `redText` على النحو التالي:

- يظهر باللون الأحمر في الوضع الفاتح
- يظهر باللون الأخضر في الوضع الداكن

![assets](../../../Resource/025_color5.png)

عند الرجوع إلى كود SwiftUI، يمكن استخدام هذا اللون بهذا الشكل:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

وفيه:

```swift
Color("redText")
```

يعني قراءة مورد اللون المسمى `redText` من `Assets`.

تأثير العرض:

![assets](../../../Resource/025_view8.png)

يمكنك أن ترى أن `Color("redText")` نفسه يعرض ألوانًا مختلفة في الوضع الفاتح والوضع الداكن.

وهذا يوضح أن لون `Assets` نفسه يدعم التكيف مع المظهر أيضًا.

## الفرق بين colorScheme و Assets

يمكن لكل من `colorScheme` و `Assets` تنفيذ التكيف مع المظهر، لكن السيناريوهات المناسبة لكل منهما مختلفة.

إذا كنت تريد فقط تبديل نص، أو اسم صورة، أو اسم `SF Symbols` وفقًا للوضع الفاتح/الداكن، فيمكنك استخدام `colorScheme`.

على سبيل المثال:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

إذا كان اللون يُستخدم كثيرًا داخل App، مثل لون السمة، أو خلفية البطاقة، أو لون النص، فمن الأفضل استخدام ألوان `Assets`.

لأن ألوان `Assets` يمكن إعادة استخدامها في عدة عروض، كما أن تعديلها لاحقًا يكون أسهل.

على سبيل المثال:

```swift
Color("redText")
```

بهذه الطريقة، يكفي تعديل اللون داخل `Assets`، وستتغير كل الأماكن التي تستخدم هذا اللون معًا.

## الملخص

في هذا الدرس، تعلمنا التكيف مع مظهر الوضع الفاتح والوضع الداكن.

أولًا، تعلمنا كيفية معاينة المظاهر المختلفة داخل `Canvas` في Xcode، وتعلمنا أيضًا كيفية استخدام `Color Scheme Variants` لمشاهدة الوضع الفاتح والوضع الداكن في الوقت نفسه.

بعد ذلك، فهمنا أن عروض SwiftUI مثل `Text` و `Image` تتكيف تلقائيًا بشكل افتراضي مع لون مظهر النظام.

لكن إذا حددنا لونًا ثابتًا يدويًا، مثل `Color.white` أو `Color.black`، فلن تتبدل هذه الألوان تلقائيًا.

ثم تعلمنا `@Environment` و `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

من خلال `colorScheme`، يمكننا تحديد ما إذا كان الوضع الحالي فاتحًا أم داكنًا، ثم عرض أيقونات أو نصوص أو ألوان مختلفة حسب المظهر.

أخيرًا، تعلمنا ألوان `Assets`.

يمكن لألوان `Assets` تعيين ألوان مختلفة للوضع الفاتح والوضع الداكن، وهي مناسبة لإدارة موارد الألوان العامة داخل App.

يساعد التكيف مع المظهر على جعل App يمتلك تأثير عرض أفضل في النهار والليل، كما يمكنه تجنب مشكلات مثل عدم وضوح النص أو عدم تميّز الخلفية.

في التطوير الفعلي، يُنصح بإعطاء الأولوية لقدرات التكيف الافتراضية في SwiftUI وألوان `Assets`.

وعندما تحتاج إلى تبديل النصوص أو الأيقونات أو الصور بين المظاهر المختلفة، أو التعامل مع عدد قليل من تأثيرات العرض الخاصة، استخدم `colorScheme` للحكم.

## معرفة إضافية: تحديد حجم الخط

عند تعلم `font` سابقًا، استخدمنا بعض أنماط خطوط النظام:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

إلى جانب استخدام أنماط خطوط النظام مثل `.largeTitle` و `.callout`، يمكن أيضًا تحديد حجم الخط يدويًا.

على سبيل المثال:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

وهنا:

```swift
.font(.system(size: 100))
```

يعني ضبط حجم الخط على `100 pt`.

بالنسبة إلى أيقونات `SF Symbols`، يؤثر `font` أيضًا في حجم الأيقونة.

لذلك، إذا أردت جعل الأيقونة تظهر بحجم أكبر، يمكنك تحديد الحجم من خلال `.font(.system(size:))`.

على سبيل المثال:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

يجب الانتباه إلى أنه رغم أن تحديد حجم الخط يدويًا أكثر مرونة، فلا يُنصح باستخدام أحجام ثابتة بكثرة للنصوص العادية.

يفضل أن تستخدم النصوص العادية أنماط خطوط النظام مثل `.title` و `.headline` و `.body` و `.caption`.

بهذا يمكن التكيف بشكل أفضل مع الأجهزة المختلفة وإعدادات حجم الخط لدى المستخدم.
