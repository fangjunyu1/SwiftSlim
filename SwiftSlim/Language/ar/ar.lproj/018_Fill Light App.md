# تطبيق ضوء التعبئة

في هذا الدرس، سنبني تطبيقًا ممتعًا جدًا لضوء التعبئة. عندما يحل الليل، يمكننا جعل شاشة الهاتف تعرض ألوانًا مختلفة واستخدامها كضوء تعبئة بسيط.

يمكن لهذا التطبيق تبديل الألوان عند النقر على الشاشة، كما يمكنه أيضًا ضبط السطوع باستخدام شريط تمرير.

في هذا المثال، سنتعلم كيفية استخدام `brightness` لضبط سطوع العرض، و`onTapGesture` لإضافة إيماءة نقر إلى العرض، وعنصر التحكم `Slider`.

النتيجة:

![Color](../../Resource/018_color.png)

## عرض لون

أولًا، دعنا نجعل العرض يُظهر لونًا.

في SwiftUI، لا يمثّل `Color` لونًا فقط، بل يمكن عرضه أيضًا كواجهة:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

هنا، يمثّل `Color.red` عرضًا باللون الأحمر. تجعل `.ignoresSafeArea()` عرض اللون يملأ الشاشة بالكامل، لذلك يبدو أشبه بتأثير ضوء تعبئة حقيقي.

النتيجة:

![Color](../../Resource/018_color1.png)

### مصفوفة الألوان والفهرس

حاليًا، يتم عرض لون واحد فقط. لكن ضوء التعبئة عادة لا يقتصر على لون واحد. فقد يعرض أيضًا الأزرق والأصفر والبنفسجي والأبيض وغيرها.

نريد التبديل بين ألوان مختلفة عند النقر على الشاشة. يمكننا وضع هذه الألوان في مصفوفة لإدارتها بشكل موحّد:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

المصفوفة مناسبة لتخزين "مجموعة من البيانات من النوع نفسه". هنا، كل عنصر في المصفوفة هو `Color`.

إذا أردنا عرض لون معيّن، فيمكننا استخدام الفهرس:

```swift
colors[0]
```

وهذا يعني قراءة اللون عند الفهرس `0` في المصفوفة، أي اللون الأول.

يمكن الآن كتابة الكود بهذا الشكل:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

بهذه الطريقة، ستعرض الشاشة أول لون في المصفوفة، وهو الأحمر.

### استخدام الفهرس للتحكم في اللون

إذا احتجنا إلى التبديل بين ألوان مختلفة، فنحن بحاجة إلى متغيّر لإدارة الفهرس بدلًا من كتابة الفهرس بشكل ثابت.

يمكننا استخدام `@State` للتصريح عن متغيّر يخزّن الفهرس:

```swift
@State private var index = 0
```

هنا، يمثّل `index` فهرس اللون الحالي.

عندما يتغيّر `index`، تعيد SwiftUI حساب الواجهة وتحدّث ما يتم عرضه.

ثم نغيّر `colors[0]` الأصلية إلى:

```swift
colors[index]
```

بهذه الطريقة، يصبح اللون الذي يعرضه المشهد محددًا بواسطة `index`.

الآن يصبح الكود:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

عندما يتغيّر `index`، سيعرض `colors[index]` لونًا مختلفًا أيضًا.

على سبيل المثال:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

هناك نقطة يجب الانتباه إليها، وهي أن `index` لا يمكن أن يتجاوز أكبر فهرس في المصفوفة، وإلا سيحدث خطأ تجاوز النطاق.

## إيماءة النقر

الآن يمكننا عرض ألوان مختلفة بناءً على `index`، لكن لا يمكننا بعد النقر للتبديل بينها.

في درس "عرض الاقتباسات"، استخدمنا `Button` للتحكم في تبديل الاقتباسات.

لكن هذه المرة، نريد "النقر على مساحة اللون بالكامل" لتبديل الألوان، لذلك يكون `onTapGesture` أنسب.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

عندما ننقر على عرض اللون، سيتم تنفيذ هذا الكود:

```swift
index += 1
```

وهذا يعني زيادة `index` بمقدار `1`. وبعد زيادة الفهرس، سيعرض `colors[index]` اللون التالي في المصفوفة.

### `onTapGesture`

`onTapGesture` هو مُعدِّل إيماءة يضيف إجراء نقر إلى العرض.

الاستخدام الأساسي:

```swift
.onTapGesture {
    // code
}
```

على سبيل المثال:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

عند النقر على هذا العرض الأحمر، سيتم تنفيذ الكود داخل الأقواس وسيظهر هذا في وحدة التحكم:

```swift
Click color
```

من خلال `onTapGesture`، يمكننا تحديد ما الذي يحدث بعد النقر على عرض ما.

### الفرق عن `Button`

في وقت سابق، تعلّمنا العرض `Button`. يمكن لكلٍّ من `Button` و`onTapGesture` التعامل مع عمليات النقر، لكن سيناريوهات استخدامهما ليست متطابقة تمامًا.

يكون `onTapGesture` أنسب لـ "إضافة وظيفة النقر إلى عرض موجود مسبقًا"، مثل `Color` أو `Image` أو `Text` أو أي عرض عادي آخر.

أما `Button` فهو أنسب عندما نمثّل زرًا واضحًا مثل "تأكيد" أو "إرسال" أو "حذف".

في تطبيق ضوء التعبئة هذا، نريد أن يكون تبديل الألوان أبسط. فالنقر على مساحة اللون بالكامل يبدّل اللون، لذلك يعد `onTapGesture` خيارًا مناسبًا هنا.

## مشكلة الفهرس

الآن يمكننا النقر على الشاشة للتبديل بين ألوان مختلفة.

لكن توجد هنا مشكلة مهمة: **قد يتجاوز الفهرس نطاق المصفوفة**.

على سبيل المثال:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

إذا واصلنا النقر على الشاشة، فسيصبح `index` في النهاية `4`، وعندها سيحدث خطأ "الفهرس خارج النطاق".

وذلك لأن مصفوفة `colors` تحتوي على `4` عناصر، لكن الفهرسة تبدأ من `0`، لذا فإن نطاق الفهارس الصالح هو `0 - 3` وليس `4`.

إذا حاولنا الوصول إلى `colors[4]`، فسيحدث خطأ "الفهرس خارج النطاق".

في الكود الحالي، تزيد كل نقرة `index` بمقدار `1`. وإذا لم نتعامل مع ذلك، فسيتجاوز المجال الصالح في النهاية.

لذلك نحتاج إلى التحقق من الفهرس عند النقر على الشاشة: إذا كان بالفعل عند اللون الأخير، فارجع إلى اللون الأول؛ وإلا فاستمر في إضافة `1`.

يمكننا تنفيذ ذلك باستخدام عبارة `if`:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

في هذا الكود، يعني `colors.count` عدد العناصر في المصفوفة.

هناك 4 ألوان في المصفوفة الحالية، لذا:

```swift
colors.count // 4
```

لكن أكبر فهرس ليس `4`، بل `3`، لأن الفهرسة تبدأ من `0`.

لذلك يجب كتابة آخر فهرس على الشكل:

```swift
colors.count - 1
```

أي:

```swift
4 - 1 = 3
```

يعني هذا المنطق: إذا كان الفهرس الحالي يشير بالفعل إلى آخر لون، فأعد الفهرس إلى `0`؛ وإلا فزده بمقدار `1`.

وبهذه الطريقة، يمكن تبديل الألوان بشكل دائري.

### تحسين منطق الفهرس

إذا أردنا أن يكون الكود أكثر اختصارًا، فيمكننا أيضًا استخدام العامل الثلاثي:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

يعني هذا الكود: إذا كانت `index == colors.count - 1` صحيحة، فأرجِع `0`. وإلا فأرجِع `index + 1`.

وأخيرًا، أعد إسناد النتيجة إلى `index`.

الآن يمكننا تحقيق تأثير تبديل الألوان.

الكود الكامل:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

الآن يمكننا النقر على الشاشة والتبديل بين ألوان مختلفة. لقد اكتمل تطبيق ضوء تعبئة أساسي.

## عرض اسم اللون

يمكننا إضافة مجموعة أخرى من النصوص المقابلة للألوان، بحيث عند تغيّر اللون، يظهر أيضًا اسم اللون الحالي على الشاشة.

على سبيل المثال:

- يعرض الأحمر `Red`
- يعرض الأزرق `Blue`
- يعرض الأصفر `Yellow`
- يعرض البنفسجي `Purple`

يمكننا هنا أيضًا استخدام مصفوفة لتخزين أسماء الألوان:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

يجب أن يتطابق ترتيب النصوص في هذه المصفوفة تمامًا مع ترتيب الألوان في مصفوفة الألوان.

ثم يمكننا استخدام `Text` لعرض اسم اللون الموافق للفهرس الحالي:

```swift
Text(colorsName[index])
```

يعرض `Text` اسم اللون الحالي وفقًا لـ `index`.

استخدم المُعدِّلات لتحسين مظهر `Text`:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

هذا يجعل `Text` يظهر باللون الأبيض، وبحجم عنوان، وبخط عريض.

لدينا الآن عرض `Color` بملء الشاشة. إذا أردنا أن يظهر `Text` فوق عرض `Color`، فنحن بحاجة إلى استخدام حاوية التخطيط `ZStack` لتكديسهما معًا.

```swift
ZStack {
    Color
    Text
}
```

لذلك يصبح الكود:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

بهذه الطريقة، تكون الخلفية هي عرض اللون، ويتم تكديس عرض نصي فوقها.

النتيجة:

![Color](../../Resource/018_color2.png)

هناك نقطة يجب ملاحظتها، وهي أنه في `ZStack`، تظهر العروض المكتوبة لاحقًا عادة في الأمام. إذا كُتب `Text` قبل `Color`، فقد يغطي عرض `Color` اللاحق عنصر `Text`.

## التحكم في السطوع

الآن يمكننا التبديل بين ألوان مختلفة، لكن لضوء التعبئة ميزة أخرى مهمة: **ضبط السطوع**.

في SwiftUI، يمكننا استخدام المُعدِّل `brightness` لضبط سطوع العرض.

على سبيل المثال:

```swift
.brightness(1)
```

يمكننا كتابة:

```swift
colors[index]
    .brightness(0.5)
```

هذا يجعل اللون الحالي أكثر سطوعًا وأقرب إلى تأثير ضوء التعبئة.

نطاق السطوع هو `0 - 1`. تعني `0` الإبقاء على اللون الأصلي، وكلما اقتربت القيمة من `1` أصبح اللون أكثر سطوعًا. أما `1` فتعني النتيجة البيضاء الأكثر سطوعًا.

على الرغم من أنه يمكننا التحكم في `brightness` في الكود، فإن المستخدم لا يزال غير قادر على ضبطه مباشرة.

لذلك نحتاج إلى إضافة عنصر تحكم قابل للسحب: `Slider`.

## عنصر العرض Slider

في SwiftUI، يُعد `Slider` عنصر تحكم يُستخدم لاختيار قيمة ضمن نطاق. وتصفه Apple بأنه "عنصر تحكم لاختيار قيمة من نطاق خطي محدود".

الاستخدام الأساسي:

```swift
Slider(value: $value, in: 0...1)
```

شرح المعاملات:

1. `value: $value`: يجب ربط `Slider` بمتغيّر.

    عند سحب شريط التمرير، تتغيّر قيمة المتغيّر في الوقت نفسه. وبالمقابل، إذا تغيّرت قيمة المتغيّر، فسيُحدَّث شريط التمرير أيضًا.

    هذا مشابه جدًا لـ `TextField` الذي تعلمناه سابقًا. فكلاهما "يربط عنصر تحكم بمتغيّر".

    يحتاج المتغيّر المرتبط إلى الرمز `$` قبله لتمثيل الربط.

2. `in: 0...1`: يمثّل هذا المعامل نطاق القيم لشريط التمرير.

    هنا، `0...1` تعني أن القيمة الدنيا هي `0` والقيمة العليا هي `1`.

    عندما يتم سحب شريط التمرير إلى أقصى اليسار، تكون قيمة المتغيّر المرتبط قريبة من `0`؛ وعند سحبه إلى أقصى اليمين، تكون قريبة من `1`.

على سبيل المثال:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

يتم ربط `Slider` بالمتغيّر `value`. وعند سحب شريط التمرير، تتغيّر قيمة المتغيّر `value` في الوقت نفسه.

المظهر:

![Slider](../../Resource/018_slider.png)

عندما يتم تحريك `Slider` إلى الجهة اليسرى، تصبح قيمة `value` المرتبطة `0`. وعندما يُحرَّك إلى الجهة اليمنى، تصبح `1`.

### نطاق القيم

نطاق قيم `Slider` ليس ثابتًا. ويمكن أيضًا كتابته بهذا الشكل:

```swift
0...100
```

أو أي نطاق آخر.

لكن في تطبيق ضوء التعبئة هذا، نحتاج إلى التحكم في السطوع، لذلك فإن `0...1` هو الخيار الأنسب.

## استخدام `Slider` للتحكم في السطوع

الآن نحتاج إلى ربط `Slider` مع `brightness`.

أولًا، أنشئ متغيّرًا لتخزين قيمة السطوع:

```swift
@State private var slider = 0.0
```

هنا، `0.0` هي قيمة من النوع `Double`.

لأن `Slider` عادة ما يرتبط بنوع رقمي، ولأننا هنا نريده أن يتغيّر بشكل مستمر، فإن `Double` هو الأنسب. كما أن `brightness` لا يقبل إلا قيمة من النوع `Double`.

ثم مرّر هذه القيمة إلى `brightness`:

```swift
colors[index]
    .brightness(slider)
```

عندما تكون `slider == 0`، يبقى اللون في حالته الافتراضية. وكلما اقتربت `slider` من `1` بدا اللون أكثر سطوعًا.

### إضافة عنصر التحكم `Slider`

بعد ذلك، أضف عنصر التحكم `Slider` لتعديل هذا المتغيّر:

```swift
Slider(value: $slider, in: 0...1)
```

عندما يتغيّر شريط التمرير، تتغيّر أيضًا قيمة `slider`، وتقوم `brightness(slider)` بتحديث السطوع في الوقت نفسه.

وهذا مثال شائع جدًا على فكرة "المتغيّرات تقود العرض" في SwiftUI.

### ضبط مظهر `Slider`

بشكل افتراضي، يستخدم `Slider` العرض المتاح.

يمكننا منحه عرضًا ثابتًا:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

ثم نضيف بعض المُعدِّلات الأخرى ليصبح أوضح:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

هذا يمنح شريط التمرير خلفية بيضاء وزوايا مستديرة، مما يجعله أكثر بروزًا فوق عرض `Color`.

وأخيرًا، ضعه في أسفل الشاشة.

وبما أننا استخدمنا بالفعل `ZStack`، فيمكننا وضع `VStack` داخله واستخدام `Spacer()` لدفع `Slider` إلى الأسفل.

## الكود الكامل

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## الملخص

باستخدام المعرفة التي تعلمناها سابقًا مع المفاهيم الأساسية مثل الألوان والمصفوفات، بنينا تطبيق ضوء تعبئة ممتعًا جدًا.

من خلال هذا التطبيق، تعلّمنا كيفية استخدام `brightness` لضبط السطوع، و`onTapGesture` لإضافة إجراءات النقر إلى العروض، وعنصر التحكم `Slider`.

أضفنا `onTapGesture` إلى عرض `Color` لتبديل الألوان. كما استخدمنا عنصر التحكم `Slider` لتغيير متغيّر يدير `brightness`، وهو مثال آخر على فكرة "المتغيّرات تقود العرض".

كما راجعنا العامل الثلاثي، واستخدمنا `ZStack` لتكديس العروض، واستخدمنا المصفوفات لإدارة مجموعة من البيانات من النوع نفسه. وهذا يساعد على تعميق فهمنا للمصفوفات والفهارس، بما في ذلك المشكلة العملية المتمثلة في تجنّب أخطاء تجاوز الفهرس للنطاق.

على الرغم من أن هذا المثال ليس معقدًا، فإنه يربط العديد من الأفكار الأساسية التي تعلمناها من قبل. وعندما يتم جمعها في مشروع صغير حقيقي، يصبح من الأسهل فهم ما يُستخدم له كل مفهوم.

### سيناريو استخدام واقعي

تخيّل وضع iPhone قديم على المكتب واستخدام تطبيق ضوء التعبئة الذي طوّرته بنفسك للتحكم في لون الضوء. ستكون تلك تجربة جميلة جدًا.

هناك الكثير من تطبيقات "ضوء التعبئة" على App Store، ولا تبدو معقدة جدًا أيضًا.

![AppStore](../../Resource/018_appStore.PNG)

يمكننا البدء بتطوير تطبيقات بسيطة ومحاولة نشرها على App Store. وهذا لا يزيد اهتمامنا بالتطوير فحسب، بل يوثّق أيضًا مسيرة نموّنا.

### تدريب بعد الدرس

يمكنك الاستمرار في التفكير في طرق لتوسيع تطبيق ضوء التعبئة هذا، مثل:

- إضافة المزيد من الألوان
- عرض قيمة السطوع الحالية
- تحسين تصميم منطقة شريط التمرير السفلية

عندما تستخدم هذه المعرفة فعلًا، ستكتشف أن كل مفهوم نتعلمه هو في الحقيقة أداة لبناء التطبيقات.

وكلما أتقنّا أدوات أكثر، استطعنا إنشاء ميزات أكثر.
