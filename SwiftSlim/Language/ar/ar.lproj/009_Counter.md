# العدّاد

في الفصلين السابقين تعرّفنا إلى المتغيرات والثوابت والأزرار والدوال.

وفي هذا الفصل سنجمع هذه المعرفة معاً لننشئ تطبيق عدّاد بسيطاً.

ومن خلال هذا المثال سنتعلّم كيف نحفظ قيمة متغيرة ونغيّرها، وكيف نستخدم `@State` لإدارة هذه القيمة.

## تنفيذ العدّاد

يُستخدم العدّاد عادة لتسجيل عدد يتزايد باستمرار، مثل عدد القفزات أو عدد لفّات الجري.

سننشئ الآن عدّاداً بسيطاً: يعرض رقماً مع زر، وعند الضغط على الزر يزداد الرقم.

![Num](../../RESOURCE/009_num.png)

### عرض الرقم

أولاً، نستخدم `Text` لعرض رقم.

```swift
Text("0")
```

لكن إذا كان الرقم سيتغير، فلا ينبغي أن نكتب `"0"` مباشرة، بل يجب أن نخزّنه في متغير.

```swift
var num = 0
```

ثم نعرض المتغير داخل `Text`:

```swift
Text("\(num)")
```

هنا نستخدم إدراج السلاسل النصية `\()` لتحويل الرقم إلى نص لعرضه.

### زيادة الرقم بالزر

بعد ذلك نضيف زرّاً.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

وعند الضغط على الزر نريد أن يزداد الرقم.

لذلك يمكننا تعديل المتغير داخل الزر:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

هذه الشيفرة تعني: عند الضغط على الزر، احسب `num + 1` ثم أعد إسناد الناتج إلى `num`.

### صيغة أكثر اختصاراً

توفّر Swift طريقة مختصرة لكتابة ذلك:

```swift
num += 1
```

وهي مكافئة تماماً لـ:

```swift
num = num + 1
```

وهذه الصيغة تُسمى عامل الإسناد المركب.

ومن أمثلة عوامل الإسناد المركبة الشائعة في Swift:

```text
+=   إسناد مع الجمع
-=   إسناد مع الطرح
*=   إسناد مع الضرب
/=   إسناد مع القسمة
%=   إسناد مع باقي القسمة
```

مثلاً:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### الشيفرة الكاملة

الآن يمكننا كتابة عداد بسيط:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
            Text("\(num)")
            Button("+") {
                num += 1
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
```

لكن هذه الشيفرة لن تعمل كما نتوقع.

### ظهور خطأ

سيعرض Xcode الرسالة:

```text
Cannot assign to property: 'self' is immutable
```

أي أنه لا يمكن تعديل الخاصية لأن الواجهة غير قابلة للتغيير بهذه الطريقة.

لكن لماذا؟

## كيف يعمل SwiftUI؟

في SwiftUI: **الواجهة مدفوعة بالبيانات**. وهذا يعني أن الواجهة تتحدث تلقائياً عندما تتغير البيانات.

لكن يجب الانتباه إلى نقطة مهمة: SwiftUI لا يراقب التغييرات إلا عندما تكون القيمة جزءاً من حالة `state`، مثل `@State` أو `@Binding`.

أما إذا استخدمنا متغيراً عادياً، فإن SwiftUI يقرأ قيمته مرة واحدة فقط عند إنشاء الواجهة، وبعد ذلك حتى لو تغيرت قيمته فلن يتم تحديث العرض.

مثلاً:

```swift
var num = 0   // 0
num = 1       // 0
```

في المثال أعلاه، `num` متغير عادي. وعلى الرغم من أن قيمته تغيرت، فإن SwiftUI لا يحدّث الواجهة، ولذلك سيظل المعروض هو `0`.

![Num](../../RESOURCE/009_state.png)

لذلك، فقط عندما تُعلَّم البيانات على أنها حالة، مثل استخدام `@State` أو `@Binding`، يستطيع SwiftUI مراقبتها وتحديث الواجهة تلقائياً.

## مغلّف الخاصية `@State`

في SwiftUI، إذا كنت تريد تعديل متغير مع تحديث الواجهة، فعليك استخدام `@State`.

```swift
@State var num = 0
```

يجعل `@State` SwiftUI يراقب هذا المتغير.

وعندما تتغير القيمة، يعيد SwiftUI حساب الواجهة ويحدّث العرض.

الشيفرة الكاملة:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
            Text("\(num)")
            Button("+") {
                num += 1
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
```

الآن، عندما نضغط على الزر:

```swift
num += 1
```

سيكتشف SwiftUI أن `num` تغيرت، ثم يحدّث الواجهة تلقائياً.

![Num](../../RESOURCE/009_state1.png)

## قواعد استخدام `@State`

في التطوير العملي، هناك قاعدتان شائعتان عند استخدام `@State`.

### 1. غالباً ما تُستخدم `private`

عادةً ما تكون متغيرات `@State` مخصصة للاستخدام داخل الـ `View` نفسه، ولذلك تُكتب غالباً بالشكل التالي:

```swift
@State private var num = 0
```

وبهذا نمنع الواجهات الأخرى من الوصول إليها أو تعديلها مباشرة.

لم نتعلم `private` بعد بشكل رسمي، لكن يمكنك أن تفهمها الآن ببساطة على أنها:

**أي شيء يحمل `private` لا يمكن استخدامه إلا داخل الواجهة الحالية.**

### 2. `@State` مخصّص لحالة الواجهة الحالية فقط

يصلح `@State` للبيانات التي تديرها الواجهة نفسها.

مثلاً، محتوى حقل الإدخال أو حالة مفتاح التبديل:

```swift
@State private var isOn = false
@State private var text = ""
```

أما إذا كانت البيانات بحاجة إلى أن تُشارك بين عدة واجهات، فنحتاج إلى أنواع حالة أخرى مثل `@Binding` أو `@Observable`.

وسنتعلّم هذه الأشياء في دروس لاحقة.

### زر التقليل

العدّاد لا يقتصر على الزيادة فقط، بل يمكنه أيضاً أن ينقص.

لنضف زر `-`.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

ثم نستخدم `HStack` لترتيب الأزرار أفقياً:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

النتيجة:

![Num](../../RESOURCE/009_num1.png)

عند الضغط على زر `+` يزداد الرقم بمقدار 1، وعند الضغط على زر `-` ينقص الرقم بمقدار 1.

### زر إعادة الضبط

يمكننا أيضاً إضافة زر يعيد الرقم إلى `0`.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

عند الضغط على زر إعادة الضبط، ستصبح قيمة `num` هي `0`.

النتيجة:

![Num](../../RESOURCE/009_num2.png)

عند الضغط على الزر `0` يعود الرقم إلى `0`.

### أزرار مخصّصة

حتى الآن، لا تستطيع أزرارنا سوى عرض نص:

```swift
Button("+") {

}
```

وهذه الصيغة مناسبة للنصوص البسيطة مثل `+` أو `-`.

لكن في التطوير العملي، نرغب غالباً في مظهر أغنى للأزرار، مثل عرض أيقونات أو عناصر أخرى.

يسمح SwiftUI بتخصيص محتوى الزر، وعندها يمكن استخدام الصيغة الأخرى لـ `Button`:

```swift
Button(action: {

}, label: {

})
```

في هذه الصيغة:

- `action` تمثل الشيفرة التي تُنفذ عند الضغط
- `label` تمثل الواجهة التي يعرضها الزر

مثلاً، يمكننا أن نجعل الزر يعرض أيقونة من SF Symbols:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

وبهذه الطريقة لن يعرض الزر نصاً، بل أيقونة.

### تحسين واجهة العدّاد

لجعل واجهة العدّاد أوضح، يمكننا أيضاً تحسين طريقة عرض الرقم، مثلاً بجعل النص أكبر:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

ثم يمكننا أيضاً استبدال نصوص الأزرار بأيقونات، مع تكبير حجم الأيقونة:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

وبالمثل يمكننا تحويل زر التقليل وزر إعادة الضبط إلى أزرار أيقونات:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

الآن أصبحت الأزرار تعرض أيقونات SF Symbols، وأصبحت الواجهة أوضح وأكثر مباشرة.

![Num](../../RESOURCE/009_num3.png)

## الخلاصة

في هذا الفصل، ومن خلال بناء عدّاد بسيط، راجعنا معاً استخدام المتغيرات و`Text` و`Button`. فقد استخدمنا متغيراً لتخزين الرقم، وتحكمنا في زيادته ونقصانه بواسطة الأزرار، كما تعرّفنا إلى عوامل الإسناد المركبة مثل `num += 1` و`num -= 1`.

وأثناء التنفيذ، فهمنا آلية عمل SwiftUI: فالواجهة مدفوعة بالبيانات. وعندما تتغير البيانات تتحدث الواجهة. ولذلك نحتاج إلى `@State` لحفظ الحالة القابلة للتغيير. وعندما تتغير قيمة متغير من نوع `@State`، فإن SwiftUI يحدّث الواجهة تلقائياً.

وفي النهاية، أجرينا بعض التحسينات البصرية البسيطة، مثل تعديل نمط الخط في `Text` واستخدام `Button(action:label:)` لتخصيص محتوى الأزرار، بحيث أصبحت تعرض أيقونات وجعلت واجهة العدّاد أوضح وأجمل.

وهكذا بدأنا نلامس الطريقة الأساسية التي تعمل بها واجهات SwiftUI. ففي SwiftUI تكون معظم الواجهات مدفوعة بالمتغيرات. وعندما تتغير المتغيرات، تقوم SwiftUI بتحديث العرض تلقائياً. ولهذا، ففي تطوير تطبيقات SwiftUI نحتاج عادة إلى التفكير أولاً في البيانات التي يجب حفظها، ثم نبني على أساسها الواجهة ومنطق التفاعل.

## الشيفرة الكاملة

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
