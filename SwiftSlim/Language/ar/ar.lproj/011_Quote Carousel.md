# عرض الأقوال المتعاقب

في هذا الفصل، سننفّذ ميزة "عرض الأقوال بشكل متعاقب"، كما سنتعمق في بعض أساسيات Swift مثل المصفوفات (`Array`) وجمل الشرط (`if-else`).

سنرى كيف نحتفظ بعدة أقوال مأثورة، ثم نستخدم تفاعل الزر لجعل هذه الأقوال تتغير بشكل دوري.

![alt text](../../RESOURCE/011_word.png)

## عرض قول مأثور

أولاً، نحتاج إلى عرض قول مأثور في SwiftUI.

أبسط طريقة هي استخدام عنصر `Text`:

```swift
Text("Slow progress is still progress.")
```

لكن هذه الشيفرة لا تستطيع إلا عرض قول واحد ثابت. وإذا أردنا عرض عدة أقوال مع إمكانية التنقل بينها، فعلينا أولاً حفظ هذه الأقوال.

إلا أن متغير النص العادي لا يمكنه سوى حفظ قول واحد:

```swift
let sayings = "Slow progress is still progress."
```

وإذا أردنا حفظ عدة أقوال، فسنحتاج إلى تعريف متغير مستقل لكل قول:

```swift
let saying1 = "Slow progress is still progress."
let saying2 = "Difficult roads lead to beautiful destinations."
let saying3 = "Awake minds are the strongest."
let saying4 = "Long-term thinking wins over short-term tricks."
let saying5 = "Master one thing, and do it well."
```

لكن في التطوير الفعلي، هذه الطريقة ليست متعبة فقط، بل إنها تجعل التبديل بين الأقوال غير مرن، لأن كل متغير مستقل عن الآخر.

ولكي نتمكن من إدارة عدة أقوال بسهولة، نحتاج إلى بنية بيانات تجمعها معاً، وهذه البنية هي المصفوفة (`Array`).

بعد استخدام المصفوفة، يمكن كتابة الشيفرة السابقة بالشكل التالي:

```swift
let sayings = [
    "Slow progress is still progress.",
    "Difficult roads lead to beautiful destinations.",
    "Awake minds are the strongest.",
    "Long-term thinking wins over short-term tricks.",
    "Master one thing, and do it well."
    ]
```

**ملاحظة: في أسلوب كتابة الشيفرة، غالباً ما تُستخدم الصيغة الجمع لاسم متغير المصفوفة التي تحتوي على عدة عناصر، مثل `sayings`، حتى تعبّر عن طبيعتها بوصفها مجموعة.**

## المصفوفات

في Swift، المصفوفة هي مجموعة مرتبة من العناصر، وتُكتب باستخدام الأقواس المربعة `[]`.

```swift
[]
```

يمكن أن تحتوي المصفوفة على عدة عناصر من النوع نفسه، وتُفصل العناصر فيما بينها بفاصلة إنجليزية `,`.

مثلاً:

```swift
[101, 102, 103, 104, 105]
```

يمكنك أن تتخيل المصفوفة ببساطة على أنها قطار:

![Array](../../RESOURCE/011_array1.png)

يمثل القطار كله كائن المصفوفة، بينما تمثل كل عربة عنصراً داخلها، مرتبة حسب التسلسل.

### الفهرس والوصول إلى العناصر

بما أن المصفوفة مرتبة، فيمكن للنظام تحديد عنصر معين بحسب موقعه في التسلسل. وتسمى هذه الآلية بـ "الفهرس" (`Index`).

في Swift، كما في معظم لغات البرمجة، يبدأ فهرس المصفوفة من `0` وليس من `1`. وهذا يعني أن العنصر الأول فهرسه `0`، والثاني فهرسه `1`، وهكذا.

![Array](../../RESOURCE/011_array2.png)

وإذا أردنا الوصول إلى عنصر معين داخل المصفوفة، يكفي أن نكتب اسم المصفوفة ثم نضيف بعده أقواساً مربعة نضع بداخلها قيمة الفهرس المطلوب.

مثلاً:

```swift
sayings[0]   // 101
sayings[1]   // 102
```

أما إذا حاولنا الوصول إلى فهرس خارج النطاق الصحيح للمصفوفة، فسنواجه مشكلة "تجاوز حدود الفهرس" (`Index Out of Range`). لذلك يجب التأكد دائماً من أن قيمة الفهرس تقع ضمن النطاق الصحيح.

**تجاوز حدود الفهرس**

مثلاً، إذا كانت المصفوفة تحتوي على 5 عناصر فقط، فإن الفهارس الصحيحة تكون من `0` إلى `4`. وإذا حاولنا الوصول إلى `sayings[5]`، فلن يتمكن البرنامج من العثور على "العربة" المقابلة، وسيؤدي ذلك إلى خطأ تجاوز حدود الفهرس وقد يتسبب في انهيار التطبيق.

![Array](../../RESOURCE/011_array3.png)

### العمليات على المصفوفة

المصفوفات لا تدعم فقط التعريف الثابت، بل تدعم أيضاً إضافة العناصر وحذفها وتعديلها، كما يمكن معرفة عدد العناصر الموجودة داخلها.

ملاحظة: إذا كنت تحتاج إلى تعديل المصفوفة، فيجب أن تعلن عنها باستخدام `var` وليس `let`.

**1. إضافة عنصر**

يمكن استخدام الدالة `append` لإضافة عنصر إلى نهاية المصفوفة:

```swift
var num = [0, 1, 2]
num.append(3)   // [0, 1, 2, 3]
```

**2. حذف عنصر**

باستخدام `remove(at:)` يمكن حذف عنصر معين من المصفوفة:

```swift
var num = [0, 1, 2]
num.remove(at: 0)   // [1, 2]
```

**3. تعديل عنصر**

يمكن تعديل عنصر مباشرة باستخدام الفهرس:

```swift
var num = [0, 1, 2]
num[1] = 4   // [0, 4, 2]
```

**4. معرفة طول المصفوفة**

استخدم الخاصية `count` للحصول على عدد عناصر المصفوفة:

```swift
var num = [0, 1, 2]
num.count   // 3
```

### استخدام المصفوفة لعرض الأقوال

لكي نعرض عدة أقوال، يمكننا حفظها داخل مصفوفة، ثم الوصول إليها وعرضها بالاعتماد على الفهرس.

أولاً، ننشئ مصفوفة `sayings` داخل `ContentView` لتخزين الأقوال، ثم نستخدم `Text` لقراءة القول المناسب من المصفوفة بواسطة الفهرس:

```swift
struct ContentView: View {
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[0])
        }
        .padding()
    }
}
```

هنا، `sayings[0]` تعني أول قول داخل المصفوفة.

وإذا أردنا عرض قول مختلف، يكفي تغيير قيمة الفهرس داخل الأقواس:

```swift
Text(sayings[0]) // "Slow progress is still progress."
Text(sayings[1]) // "Difficult roads lead to beautiful destinations."
```

### تعريف فهرس القول

ولكي نتمكن من تبديل الأقوال بشكل ديناميكي، لا ينبغي أن نكتب الفهرس بشكل ثابت داخل `Text`.

بل نحتاج إلى إنشاء متغير مستقل يحفظ الفهرس الحالي للقول المعروض.

في SwiftUI يمكننا استخدام `@State` لتعريف فهرس قابل للتغيير:

```swift
@State private var index = 0
```

يقوم SwiftUI بمراقبة المتغيرات المغلّفة بـ `@State`. وعندما تتغير قيمة `index`، يعيد SwiftUI رسم الواجهة ويعرض القول المناسب.

ثم نستخدم `sayings[index]` للحصول على القول بشكل ديناميكي:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[index])
        }
        .padding()
    }
}
```

وعندما تتغير قيمة `index`، سيعرض `Text` قولاً مختلفاً.

### استخدام زر للتحكم في الفهرس

لكي نتحكم في تبديل الأقوال، يمكننا استخدام `Button` لتعديل قيمة `index`. وفي كل مرة نضغط فيها على الزر، تزداد قيمة `index` بمقدار 1:

```swift
Button("Next") {
    index += 1
}
```

وعندما نضغط على الزر، تنتقل قيمة `index` من `0` إلى `1`، فيُعاد تحديث الواجهة، ويقوم `Text(sayings[index])` بقراءة القول التالي.

لكن توجد هنا مشكلة محتملة: إذا استمر المستخدم في الضغط على الزر، فستزداد قيمة `index` حتى تتجاوز حدود المصفوفة، مما يؤدي إلى خطأ في الفهرس. فعلى سبيل المثال، عندما تصبح قيمة `index` هي `5`، بينما نطاق الفهارس في المصفوفة هو من `0` إلى `4`، سينهار التطبيق.

ولمنع هذا، نحتاج إلى إدخال تحكم شرطي يضمن أن `index` لن يتجاوز النطاق الصحيح للمصفوفة. ويمكننا استخدام جملة `if-else` للتحقق مما إذا كانت قيمة `index` أصغر من طول المصفوفة.

## التحكم الشرطي: جملة `if-else`

جملة `if-else` من أكثر جمل التفرّع الشرطي استخداماً في Swift. وهي تُستخدم للتحقق مما إذا كان الشرط متحققاً، ثم تنفيذ شيفرة مختلفة بحسب كون النتيجة صحيحة أو خاطئة.

البنية الأساسية:

```swift
if condition {
    // الشيفرة التي تُنفذ عندما تكون condition صحيحة
} else {
    // الشيفرة التي تُنفذ عندما تكون condition خاطئة
}
```

في جملة `if`، يكون الشرط `condition` قيمة من نوع `Bool`، أي إما `true` أو `false`. فإذا كانت القيمة `true`، تُنفذ الشيفرة داخل `if`، وإلا تُنفذ الشيفرة داخل `else`.

مثلاً:

```swift
let age = 25
if age > 18 {
    print("Big Boy")
} else {
    print("Little Body")
}
```

في هذا المثال، قيمة `age` هي `25`، وتتحقق جملة `if` مما إذا كان `age > 18`. وبما أن الشرط صحيح، فسيتم طباعة `"Big Boy"`.

وإذا لم نكن بحاجة إلى جزء `else`، فيمكننا حذفه:

```swift
if condition {
    // الشيفرة التي تُنفذ عندما تكون condition صحيحة
}
```

### استخدام الشرط للتحكم في نطاق الفهرس

لمنع تجاوز حدود فهرس المصفوفة، يمكننا استخدام `if` للتأكد من أن `index` لا يتجاوز النطاق الصحيح:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    }
}
```

تحليل المنطق: قيمة `sayings.count` هي `5`، لأن لدينا 5 أقوال. وبالتالي فإن `sayings.count - 1` تساوي `4`، وهي آخر فهرس صحيح في المصفوفة.

عندما تكون قيمة `index` أصغر من `4`، يكون من الآمن زيادة الفهرس بمقدار 1 عند الضغط على الزر. أما إذا أصبحت قيمة `index` تساوي `4`، فلن يتحقق الشرط بعد الآن، ولن يؤدي الضغط على الزر إلى أي تغيير.

الآن أصبحت الشيفرة قادرة بالفعل على تبديل الأقوال:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack {
            Text(sayings[index])
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                }
            }
        }
        .padding()
    }
}
```

### جعل الأقوال تدور بشكل دائري

إذا أردنا أنه بعد عرض آخر قول، يؤدي الضغط على الزر إلى العودة إلى أول قول من جديد، فيمكننا تحقيق ذلك باستخدام جزء `else`:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    } else {
        index = 0
    }
}
```

عند الضغط على الزر، إذا وصلت قيمة `index` إلى آخر عنصر في المصفوفة، فسيتم إرجاعها إلى `0`، وبذلك يبدأ عرض الأقوال من البداية مرة أخرى.

## تحسين واجهة عرض الأقوال

الآن أصبح منطق عرض الأقوال المتعاقب جاهزاً، لكن لا يزال بإمكاننا تحسين شكل الواجهة لتصبح أجمل.

الشيفرة الكاملة:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack(spacing: 10) {
            Text(sayings[index])
                .padding(20)
                .background(.white)
                .cornerRadius(10)
                .opacity(0.8)
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                } else {
                    index = 0
                }
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .background(
            Image("001")
        )
    }
}
```

في هذا المثال، أضفنا إلى عنصر `Text` خلفية بيضاء شبه شفافة مع زوايا دائرية. واستخدمنا لزر `Button` النمط `.borderedProminent`، كما أضفنا صورة خلفية إلى `VStack`.

معلومة إضافية: عند استخدام المعدّل `background()` مع صور الخلفية، فإن سلوكه الافتراضي هو محاولة ملء المساحة التخطيطية الحالية قدر الإمكان. وفي كثير من الحالات، يتمدد هذا بشكل طبيعي حتى يغطي المنطقة الآمنة (`Safe Area`) أيضاً.

والآن أصبح لدينا عرض أقوال متعاقب كامل.

## الخلاصة

من خلال هذا الفصل، تعلّمنا كيف نستخدم المصفوفة لتخزين عدة أقوال، وكيف نستخدم `if` و`if-else` لتنفيذ التبديل الدوري بين الأقوال.

كما تعرّفنا إلى العمليات الأساسية على المصفوفات، مثل إضافة العناصر وحذفها وتعديلها، وإلى كيفية منع أخطاء تجاوز الفهرس.

هذا الفصل لم يشرح فقط كيفية تنفيذ عرض الأقوال المتعاقب، بل جمع أيضاً بين المصفوفات والجمل الشرطية في سياق عملي، مما يساعدنا على إتقان التعامل مع البيانات والتحكم في تدفق البرنامج.

## معرفة إضافية - الشروط المتعددة: `if-else if-else`

في التطوير العملي، نحتاج كثيراً إلى معالجة أكثر من شرط واحد. فمثلاً في لعبة ما، إذا كانت النقاط تساوي `1` نُطلق الحدث A، وإذا كانت تساوي `2` نُطلق الحدث B، وإذا كانت تساوي `3` نُطلق الحدث C، وهكذا.

وعندما يكون لدينا أكثر من احتمالين، نستخدم بنية `if-else if-else`.

الصيغة الأساسية:

```swift
if conditionA {
    // الشيفرة التي تُنفذ عندما تكون conditionA صحيحة
} else if conditionB {
    // الشيفرة التي تُنفذ عندما تكون conditionB صحيحة
} else if conditionC {
    // الشيفرة التي تُنفذ عندما تكون conditionC صحيحة
} else {
    // الشيفرة التي تُنفذ عندما لا يتحقق أي شرط
}
```

في هذه الحالة، يفحص البرنامج الشروط بالترتيب، وينفذ أول شرط يتحقق. وإذا لم يتحقق أي شرط، تُنفذ شيفرة `else`.

وفي عرض الأقوال المتعاقب، يمكننا أيضاً استخدام هذه الصيغة:

```swift
Button("Next") {
    if index == 0 {
        index += 1
    } else if index == 1 {
        index += 1
    } else if index == 2 {
        index += 1
    } else if index == 3 {
        index += 1
    } else if index == 4 {
        index = 0
    } else {
        index = 0
    }
}
```

عندما تكون قيمة `index` هي `0` أو `1` أو `2` أو `3`، فإن كل ضغطة على الزر تنفذ `index += 1`. وعندما تكون القيمة `4`، أي آخر عنصر، يتم إرجاع `index` إلى `0` لبدء الحلقة من جديد.

يُستخدم فرع `else` هنا كحل احتياطي في حال ظهرت قيمة غير صحيحة، مثل أن تتغير `index` بطريقة خاطئة.

ومن المهم الانتباه إلى أن العامل `==` هنا يعني "هل القيمتان متساويتان". فإذا كانت قيمة `index` مساوية للرقم الذي نتحقق منه، ستكون نتيجة الشرط `true` وتُنفذ الكتلة التالية. وإذا لم تتساوَ، ينتقل البرنامج إلى الشرط التالي.

هذا النوع من التفرع المتعدد مفيد جداً عندما نريد تنفيذ شيفرات مختلفة في حالات شرطية مختلفة.
