# الحلقات والقوائم

في هذا الدرس، سنتعلّم حلقة `for` في Swift، و `ForEach` في SwiftUI.

كلتاهما تساعداننا على قراءة العناصر الموجودة في المصفوفة بالترتيب. في التطوير الفعلي، نحتاج كثيرًا إلى استخراج مجموعة من البيانات عنصرًا بعد عنصر، ثم معالجتها أو عرضها، وتُعدّ الحلقات الأداة المهمة لإنجاز ذلك.

عندما نتصفّح الويب أو نستخدم أحد التطبيقات، نرى كثيرًا من القوائم أو العناصر. هذا النوع من المحتوى يُخزَّن عادةً أولًا في `مصفوفة`، ثم يُقرأ بالتتابع من خلال `حلقة`، وأخيرًا يُعرَض على الواجهة.

على سبيل المثال، قائمة نتائج بحث Google:

![Google](../../Resource/017_google.png)

كل عنصر في قائمة نتائج البحث يمثّل عادةً جزءًا من البيانات، ثم يُعرَض واحدًا بعد الآخر من خلال حلقة.

لذلك، كثيرًا ما تظهر `المصفوفات` و`الحلقات` معًا. فـ `المصفوفة` مسؤولة عن حفظ مجموعة من البيانات، و`الحلقة` مسؤولة عن استخراج هذه البيانات بالترتيب وتنفيذ العملية نفسها عليها.

والآن، لنبدأ بأبسط حلقة، وهي `for`.

## العدّ التنازلي

لنفترض أننا نريد محاكاة وظيفة العدّ التنازلي، بحيث نطبع `“5، 4، 3، 2، 1”` بالتتابع.

أبسط كتابة مباشرة هي:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

هذه الطريقة يمكنها محاكاة العدّ التنازلي، لكن إذا كان المحتوى كثيرًا فسيصبح التكرار واضحًا جدًا، كما أن التعديل لاحقًا لن يكون مريحًا.

في هذه الحالة، يمكننا أولًا وضع البيانات داخل مصفوفة، ثم استخدام حلقة `for` لطباعة العناصر واحدًا بعد الآخر:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

هذه الكتابة أكثر اختصارًا وأسهل في التوسّع. وحتى لو كان هناك الكثير من الأرقام في العدّ التنازلي، فلن نحتاج إلى كتابة `print` سطرًا بعد سطر، بل يكفي أن نجعل الحلقة تتعامل تلقائيًا مع كل عنصر في المصفوفة.

يمكننا وضع هذا الكود داخل زر لتجربته:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

عند الضغط على الزر، ستأخذ حلقة `for` كل عدد صحيح داخل `countDown` بالترتيب، ثم تطبعه في وحدة التحكّم.

والآن، لنلقِ نظرة على كيفية عمل حلقة `for`.

## حلقة for

تُستخدم حلقة `for` لمعالجة مجموعة من البيانات بالترتيب، وأكثر ما تُستخدم مع المصفوفات والمديات.

الكتابة الأساسية:

```swift
for عنصر in مصفوفة_أو_مدى {
    الكود_الذي_يتم_تنفيذه
}
```

على سبيل المثال، كود العدّ التنازلي السابق:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

هنا، يمثّل `i` العنصر الحالي الذي تم استخراجه، وما بعد `in` هو البيانات التي نكرّر عليها، وهي مصفوفة `countDown`.

معنى هذا الكود هو: سيأخذ البرنامج كل عنصر من مصفوفة `countDown` بالترتيب. وفي كل مرة يأخذ عنصرًا، يستخدم `i` مؤقتًا لتمثيله، ثم ينفّذ الكود الموجود داخل الأقواس المعقوفة.

يمكن فهم عملية التنفيذ على هذا النحو:

* أوّل عنصر يتم أخذه هو `5`، لذلك تكون قيمة `i` في هذه اللحظة هي `5`
* يتم تنفيذ `print(i)` فتتم طباعة `5`
* ثم يتم أخذ العنصر التالي `4`
* ثم يتم تنفيذ `print(i)` مرة أخرى فتتم طباعة `4`

أما `3` و`2` و`1` فسيتم تنفيذها بالطريقة نفسها بالتتابع.

![for](../../Resource/017_for.png)

عندما تتم معالجة جميع العناصر الموجودة في المصفوفة، تنتهي حلقة `for` تلقائيًا.

يمكننا باستخدام حلقة `for` الحصول على كل عنصر في المصفوفة، وتنفيذ وظائف مثل العدّ التنازلي.

**ملاحظة**: الاسم `i` شائع جدًا في حلقات `for`، ويُستخدم لتمثيل العنصر الحالي الذي تم استخراجه، لكنه ليس اسمًا ثابتًا، ويمكنك استخدام أسماء إنجليزية أخرى.

على سبيل المثال، يمكن استخدام `num` لتمثيل كل عنصر يتم استخراجه:

```swift
for num in countDown {
    print(num)
}
```

وداخل الأقواس المعقوفة، يجب أيضًا استخدام `num` لتمثيل العنصر الحالي.

### التكرار على المدى

لا يمكن لحلقة `for` التكرار على المصفوفات فقط، بل يمكنها أيضًا التكرار على المديات.

على سبيل المثال، عندما نريد حساب مجموع الأعداد الصحيحة الموجبة حتى `50`، نحتاج إلى جمع جميع الأعداد من `1` إلى `50` واحدًا بعد الآخر.

أبسط كتابة مباشرة قد تكون:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

المشكلة في هذه الكتابة أننا نحتاج إلى كتابة كل رقم يدويًا، وهذا متعب وغير مناسب لسيناريوهات التطوير الحقيقية.

في هذه الحالة، يمكننا استخدام معامل المدى مع حلقة `for` لإنجاز ذلك.

في درس “المعاملات” السابق، تعلّمنا بالفعل معامل المدى:

```swift
1...3   // 1, 2, 3
```

هنا، `...` تعني مدى مغلق، أي إنه يشمل قيمة البداية وقيمة النهاية.

لذلك، يمكننا هنا استخدام حلقة `for` على المدى الصحيح من `1` إلى `50`:

```swift
for i in 1...50 {
    
}
```

ونحن نريد جمع هذه الأعداد الصحيحة، لذلك نحتاج أيضًا إلى متغيّر لحفظ المجموع:

```swift
var count = 0
```

ثم في كل مرة تتكرّر فيها الحلقة، نضيف الرقم الحالي الذي تم أخذه إلى `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

هذا الكود يعني:

* يُستخدم `count` لحفظ المجموع الحالي
* ستأخذ حلقة `for` الأعداد الصحيحة من `1` إلى `50` بالتتابع
* في كل مرة يتم أخذ عدد صحيح، يتم تنفيذ `count += i`
* بعد انتهاء الحلقة، ستكون القيمة النهائية للمجموع محفوظة داخل `count`

وبهذه الطريقة يمكننا الحصول على مجموع جميع الأعداد الصحيحة الموجبة حتى `50`.

وبالمثل، إذا أردنا حساب مجموع الأعداد الصحيحة من `1` إلى `100`، فما علينا سوى تعديل المدى:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

حلقة `for` لا تكرّر على المصفوفات فقط، بل يمكنها أيضًا التكرار على مدى.

وهذه القدرة على معالجة البيانات المتكررة بالترتيب شائعة جدًا في التطوير.

فهي تساعدنا على تنفيذ كثير من الأعمال المتكررة تلقائيًا، دون الحاجة إلى كتابة جمع كل رقم يدويًا، أو غير ذلك من العمليات المتكررة المشابهة.

## التكرار في SwiftUI

حلقة `for` السابقة تنتمي إلى لغة Swift نفسها.

لكن في SwiftUI، لا يمكننا استخدام `for` مباشرةً داخل بنية الواجهة لإنشاء العروض:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

وذلك لأن ما بداخل `VStack` يجب أن يكون عروض SwiftUI، بينما حلقة `for` العادية ليست عرضًا بحدّ ذاتها.

إذا أردنا في SwiftUI إنشاء الواجهة بشكل متكرر اعتمادًا على محتوى المصفوفة، فعلينا استخدام `ForEach`.

على سبيل المثال، إذا أردنا عرض أسماء جميع أعضاء مجموعة صغيرة، يمكن كتابة ذلك هكذا:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

نتيجة العرض:

![for1](../../Resource/017_for1.png)

ثم نضيف المعدّلات نفسها إلى كل `Text`:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

الآن، سيظهر كل `Text` بخط أصغر وأكثر سماكة.

نتيجة العرض:

![](../../Resource/017_for2.png)

على الرغم من أن المعدّلات تجعل الواجهة أجمل، فإنها تجعل الكود أطول وأكثر تكرارًا أيضًا.

إذا زاد عدد الأسماء، أو احتجنا لاحقًا إلى تعديل الخط أو اللون أو غير ذلك من الأنماط بشكل موحّد، فسنضطر إلى كتابة وتعديل الكثير من الكود المتشابه، وهذا سيكون مزعجًا وغير مناسب للصيانة.

هنا يمكننا وضع هذه الأسماء أولًا داخل مصفوفة، ثم استخدام `ForEach` في SwiftUI لإنشاء العروض بشكل متكرر بناءً على البيانات الموجودة في المصفوفة:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

هذه المجموعة من الأكواد أكثر اختصارًا، إذ سيقوم `ForEach` بقراءة كل عنصر في المصفوفة بالتتابع، ثم عرض المحتوى المقابل له على الواجهة.

وعندما نضيف أسماء جديدة لاحقًا، يكفي تعديل أسماء المصفوفة `names`، ولا نحتاج في كل مرة إلى تكرار إضافة `Text` ومعدّلاته الكثيرة.

## ForEach

`ForEach` هو تركيب في SwiftUI يُستخدم لإنشاء العروض بشكل متكرر. ويمكنك فهمه على أنه تركيب يشبه حلقة `for` داخل SwiftUI.

الكتابة الأساسية:

```swift
ForEach(المصفوفة, id: \.self) { العنصر in
    كود SwiftUI
}
```

على سبيل المثال، أسماء المجموعة السابقة:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

هذا الكود يعني:

`names` هي المصفوفة التي يتم التكرار عليها، أي البيانات التي نريد عرضها.

و`id:\.self` تعني استخدام العنصر نفسه داخل المصفوفة للتمييز بين كل عنصر.

في هذا المثال، تحتوي المصفوفة على سلاسل نصية:

```swift
["Sam", "John", "Wathon", "Bob"]
```

لذلك ستستخدم SwiftUI مباشرةً السلاسل `"Sam"` و`"John"` و`"Wathon"` و`"Bob"` نفسها للتمييز بين العناصر المختلفة.

وبالنسبة للمصفوفات البسيطة الشائعة في مرحلة المبتدئين مثل `String` و`Int`، فيمكننا غالبًا أن نكتب مباشرةً:

```swift
id: \.self
```

في الوقت الحالي، يكفي أن تتذكّر هذه الكتابة فقط، ولا حاجة إلى التعمّق في قواعد `id` الأخرى.

أما `item in`، فـ `item` تمثّل العنصر الحالي الذي تم استخراجه.

فعلى سبيل المثال، في أوّل تكرار تكون قيمة `item` هي `"Sam"`، وفي التكرار الثاني تكون `"John"`.

لذلك:

```swift
Text(item)
```

سيتحوّل بالتتابع إلى:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

إذًا، وظيفة هذا الكود هي: استخراج كل اسم من مصفوفة `names` بالتتابع، ثم إنشاء عرض `Text` لكل اسم.

### عملية التنفيذ

يمكن فهم عملية التنفيذ على النحو التالي:

* يقرأ `ForEach` أولًا العنصر الأول `"Sam"` من مصفوفة `names`
* تمثّل `item` مؤقتًا القيمة `"Sam"`
* يقوم `Text(item)` بعرض `"Sam"`
* ثم يقرأ العنصر التالي `"John"`
* ثم يكرّر تنفيذ كود العرض نفسه

وفي النهاية، ستظهر جميع الأسماء الموجودة في المصفوفة.

فائدة هذه الكتابة هي أنه عندما يزيد عدد الأسماء، لا نحتاج إلى تكرار كتابة عدد كبير من عناصر `Text`، بل يكفي تعديل محتوى المصفوفة فقط.

### التكرار على الصور

لا يقتصر `ForEach` على عرض النصوص فقط، بل يمكنه أيضًا عرض الصور أو الألوان أو غيرها من عروض SwiftUI.

على سبيل المثال، لنفترض أننا جهّزنا 4 صور:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

اسحب هذه الصور الأربع إلى مجلد Assets في Xcode.

![image](../../Resource/017_for3.png)

إذا لم نستخدم `ForEach`، فسنحتاج إلى كتابة كل صورة يدويًا:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

نتيجة العرض:

![image](../../Resource/017_for4.png)

هذه الطريقة يمكنها تنفيذ الوظيفة، لكن الكود سيكون طويلًا نسبيًا.

وإذا ازداد عدد الصور، فسوف نحتاج إلى تكرار كتابة المزيد من كود `Image`.

وإذا أردنا لاحقًا تعديل حجم الصور بشكل موحّد، فسنحتاج إلى تعديلها واحدةً واحدة.

في هذه الحالة، يمكننا وضع أسماء الصور داخل مصفوفة، ثم استخدام `ForEach` لإنشاء عروض الصور بشكل متكرر:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

هنا، `images` هي مصفوفة من السلاسل النصية، وكل عنصر فيها هو اسم صورة.

أما `ForEach(images, id: \.self)` فتعني: اقرأ كل عنصر في مصفوفة `images` بالتتابع، واستخدم العنصر نفسه للتمييز بين كل عنصر، ثم أنشئ العرض المناسب بناءً على العنصر الحالي.

على سبيل المثال، إذا كانت أول قيمة مقروءة هي `"430F9BEF"`، فإن:

```swift
Image(item)
```

تعادل:

```swift
Image("430F9BEF")
```

أما الصور التالية فستُعرض بالتتابع بالطريقة نفسها.

![image](../../Resource/017_for4.png)

وما زلنا هنا نستخدم:

```swift
id: \.self
```

والسبب هو نفسه كما في السابق، لأن عناصر مصفوفة `images` هي أيضًا سلاسل نصية بسيطة، لذلك يمكننا في مرحلة المبتدئين أن نستخدم العنصر نفسه مباشرةً كطريقة للتمييز.

ويمكنك فهم هذا بشكل مبسّط على أنه صيغة ثابتة:

**عندما يستخدم `ForEach` مصفوفة بسيطة من نوع `String` أو `Int`، فعادةً نكتب `id: \.self`.**

### إضافة صور جديدة

إذا احتجنا لاحقًا إلى إضافة صور جديدة، فما علينا سوى تعديل محتوى المصفوفة:

```swift
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

ولا نحتاج إلى تكرار كتابة كود `Image` جديد.

إذا كان عدد الصور كبيرًا، فيمكن استخدامه مع `ScrollView`، وإلا فلن يمكن معاينة كل المحتوى كاملًا بعد تجاوزه حدود الشاشة.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### تعديل الصور

عندما نحتاج إلى تعديل نمط الصور بشكل موحّد، يكفي أن نعدّل الكود داخل `ForEach` فقط.

على سبيل المثال، إذا غيّرنا العرض إلى `80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

فعندها ستتغيّر جميع الصور معًا، ولن نحتاج إلى تعديلها واحدةً واحدة.

وهذا أحد الأدوار المهمة لـ `ForEach`:

**عندما تكون بنية عدة عروض متشابهة، ويكون الاختلاف فقط في المحتوى، يمكننا كتابة كود العرض مرة واحدة، ثم نجعل البيانات الموجودة في المصفوفة تحدّد ما الذي سيتم عرضه.**

## الخلاصة

في هذا الدرس، تعلّمنا حلقة `for` و `ForEach`.

حلقة `for` تُستخدم لمعالجة العناصر الموجودة في المصفوفة أو المدى بالترتيب، وهي تميل أكثر إلى معالجة البيانات.

أما `ForEach` فتُستخدم في SwiftUI لإنشاء العروض بشكل متكرر اعتمادًا على محتوى المصفوفة، وهي تميل أكثر إلى عرض الواجهة.

وفي مرحلة المبتدئين الحالية، فإن أكثر كتابة شائعة لـ `ForEach` هي:

```swift
ForEach(المصفوفة, id: \.self) { العنصر in
    كود SwiftUI
}
```

وفيها، `id: \.self` تعني استخدام العنصر نفسه للتمييز بين كل عنصر.

وعندما يكون محتوى المصفوفة من الأنواع البسيطة مثل `String` و`Int`، فيمكننا عادةً أن نستخدم هذه الكتابة فقط في البداية.

لاحقًا، عندما تتصفّح الويب أو تستخدم تطبيقًا وترى مجموعة من النصوص أو الصور أو غيرها من العناصر المتكررة، يمكنك أن تتذكّر أن هذه العناصر في SwiftUI تكون غالبًا محفوظة أولًا داخل مصفوفة، ثم تُعرض بالتتابع من خلال `ForEach`.
