# جعل الزر يبدأ بالعمل

يركّز هذا الفصل على ثلاثة عناصر مهمة: زر `Button` في SwiftUI، والطباعة باستخدام `print` لأغراض التصحيح، والدوال في Swift.

وهذه كلها تشكّل أساس البرمجة التفاعلية في SwiftUI.

## الزر Button

الزر من أكثر عناصر الواجهة استخداماً. فعندما نحتاج إلى تنزيل شيء أو فتحه أو الخروج منه، فإننا غالباً نضغط زراً.

مثلاً، في App Store نضغط زر `Get` لتنزيل تطبيق.

![Button](../../RESOURCE/008_button.png)

في SwiftUI يُمثّل الزر باستخدام `Button`.

الصيغة الأساسية:

```swift
Button("") {
    
}
```

ويمكن تقسيم بنية الزر إلى جزأين:

```swift
Button("نص الزر") {
    الشيفرة التي تُنفذ عند الضغط على الزر
}
```

مثلاً:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

بعد التشغيل سيظهر زر بعنوان `Start`.

![Button](../../RESOURCE/008_button1.png)

وعندما يضغط المستخدم على الزر، تُنفَّذ الشيفرة الموجودة داخل الأقواس المعقوفة.

من المهم الانتباه إلى أن `Button` عنصر واجهة من نوع `View`، لذلك يجب أن يُكتب داخل `body`.

### المعدّل `buttonStyle`

في SwiftUI، يوفّر `Button` أنماطاً جاهزة من النظام، ويمكن استخدامها من خلال المعدّل `buttonStyle`.

مثلاً:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

يطبّق `buttonStyle` أسلوب زر يوفّره النظام.

ومن الخيارات الشائعة:

- `.automatic`
- `.plain`
- `.bordered`
- `.borderedProminent`
- `.borderless`

![Button](../../RESOURCE/008_button7.png)

والفرق بينها يظهر أساساً في وجود الحدود أو الخلفية ودرجة بروز الزر بصرياً.

## إخراج `print`

في البرمجة نحتاج كثيراً إلى معرفة ما إذا كان التنفيذ قد وصل إلى موضع معيّن من الشيفرة.

وتوفّر Swift الدالة `print` لإخراج معلومات تصحيح.

الصيغة الأساسية:

```swift
print("Hello")
```

هذا السطر يطبع النص إلى الـ Console.

### اختبار ما إذا كان الزر يعمل

يمكننا وضع `print` داخل الزر لنتحقق من أن الضغط عليه يُفعّل التنفيذ.

```swift
Button("Start") {
    print("Test 123")
}
```

عند الضغط على الزر، ستظهر في الـ Console النتيجة:

```text
Test 123
```

وهذا يدل على أن الزر قد تم تفعيله بنجاح.

![Button](../../RESOURCE/008_button3.png)

ملاحظة: `print` هي شيفرة Swift عادية وليست عنصراً من عناصر الواجهة. لذلك لا يمكن وضعها مباشرة داخل `body`، وإلا سيظهر خطأ.

مثلاً، هذا الاستخدام غير صحيح:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // خطأ: body يجب أن يعيد View
    }
}
```

في SwiftUI، يُستخدم `body` لوصف بنية الواجهة فقط، لذلك لا يمكن أن يحتوي إلا على عناصر عرض مثل `Text` و`Button` و`Image`.

أما الشيفرة العادية، مثل العمليات الحسابية أو `print`، فيجب أن توضع داخل الأزرار أو الدوال أو معالجات الأحداث الأخرى.

### الـ Console

سيظهر الناتج الذي تطبعه `print` داخل Console الخاص بـ Xcode.

إذا لم تكن منطقة Console ظاهرة، فيمكنك الضغط على زر `Show the Debug Area` في أسفل اليمين لإظهار منطقة التصحيح، ثم الضغط على `Show the Console`.

![](../../RESOURCE/008_button4.png)

في أسفل يسار الـ Console يوجد تبويبان: `Executable` و`Previews`.

![](../../RESOURCE/008_button5.png)

هاتان العلامتان تمثلان بيئتين مختلفتين للتشغيل:

- إذا كنت تستخدم Canvas للمعاينة، فسيظهر ناتج `print` تحت تبويب `Previews`
- وإذا كنت تشغّل التطبيق على المحاكي أو الجهاز الحقيقي، فسيظهر الناتج تحت تبويب `Executable`

لذلك إذا شعرت أن `print` لا يعرض شيئاً، فتحقق أولاً من أنك تنظر إلى التبويب الصحيح.

## مثال - رسومات ASCII

في كثير من المشاريع البرمجية نرى رسومات ASCII.

وهي رسومات تُنشأ من الأحرف العادية، مثل:

![ASCII](../../RESOURCE/008_ascii.png)

يمكننا استخدام زر + `print` لطباعة رسم ASCII.

مثال:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

عند الضغط على الزر، سيكون الناتج:

```text
 /\_/\
( o.o )
 > ^ <
```

ملاحظة: داخل السلاسل النصية يحتاج الرمز `\` إلى الهروب المناسب، لأن الشرطة المائلة العكسية تُعد محرف escape في النصوص.

## الدوال Functions

يمكننا بالطبع كتابة الشيفرة مباشرة داخل الزر.

مثلاً:

```swift
Button("Start") {
    print("Hello")
}
```

إذا كانت الشيفرة قصيرة فلا توجد مشكلة. لكن في التطوير الفعلي قد تتضمن عملية الزر منطقاً أطول بكثير.

مثلاً:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // قد يكون هناك المزيد من الشيفرة
}
```

إذا أصبحت الشيفرة أطول فأطول، فكتابة كل شيء داخل الزر ستجعل شيفرة الواجهة مزدحمة وصعبة القراءة والصيانة.

لهذا السبب نقوم عادة بجمع هذا المنطق داخل دالة، ثم نستدعي تلك الدالة من الزر.

### ما هي الدالة؟

الدالة عبارة عن كتلة من الشيفرة يمكن إعادة استخدامها.

وعندما نحتاج إلى تنفيذ تلك الشيفرة، يكفي أن نستدعي الدالة، فتعمل التعليمات الموجودة بداخلها.

هذا يجعل بنية الشيفرة أوضح، ويسهّل إعادة الاستخدام.

### الصيغة الأساسية

في Swift تُعرّف الدالة باستخدام الكلمة المفتاحية `func`:

```swift
func randomInt() {
    // code
}
```

هذا السطر يعرّف دالة.

`randomInt` هو اسم الدالة، ويُستخدم لتمييز هذه الكتلة من الشيفرة.

أما `()` فهي مكان المعاملات، ويمكن للدالة من خلالها استقبال بيانات من الخارج. وإذا لم تكن هناك حاجة إلى بيانات، فتبقى فارغة.

أما `{}` فهي جسم الدالة، وتُكتب بداخلها الشيفرة التي نريد تنفيذها.

مثلاً:

```swift
func getName() {
    print("FangJunyu")
}
```

وظيفة هذه الدالة هي طباعة نص معيّن.

### استدعاء الدالة

تعريف الدالة وحده لا يعني تنفيذها. فإذا أردنا تشغيلها، فعلينا استدعاؤها.

وطريقة الاستدعاء تكون بكتابة اسم الدالة متبوعاً بـ `()`:

```swift
getName()
```

عندما يصل التنفيذ إلى هذا السطر، ستُنفذ الشيفرة داخل `getName`.

المثال الكامل:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

بعد التشغيل سيكون الناتج:

```text
FangJunyu
```

### استخدام الدوال داخل SwiftUI

في واجهات SwiftUI، تُكتب الدوال عادة خارج `body`.

مثلاً:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

عندما يضغط المستخدم على الزر، تُستدعى الدالة `getName()`.

وبذلك تبقى شيفرة الواجهة نظيفة، بينما يُترك المنطق الفعلي للدالة.

### معاملات الدالة

أحياناً تحتاج الدالة إلى أن تنفّذ سلوكاً مختلفاً اعتماداً على البيانات التي تستقبلها.

مثلاً، قد نريد أن تطبع الدالة أسماء مختلفة بدلاً من طباعة `"FangJunyu"` دائماً.

في هذه الحالة نستخدم المعاملات. ويمكن فهم المعامل على أنه بيانات تُمرر إلى الدالة عند استدعائها.

مثلاً:

```swift
func getName(name: String) {
    print(name)
}
```

في هذا المثال أضفنا معاملاً جديداً:

- `name` هو اسم المعامل
- `String` هو نوعه

وهذا يعني أن الدالة تحتاج إلى استقبال قيمة من نوع `String`.

وبما أن الدالة تحتاج الآن إلى بيانات، فلا بد من تزويدها بها عند الاستدعاء:

```swift
getName(name: "Sam")
```

عند تمرير `"Sam"`، ستستخدمه الدالة وتطبعه.

الناتج:

```text
Sam
```

## مثال - طباعة ASCII باستخدام دالة

في المثال السابق كتبنا `print` مباشرة داخل الزر.

لكن إذا كان الرسم النصي أكثر تعقيداً، فستصبح الشيفرة طويلة. وهنا يكون من الأنسب نقل هذا المنطق إلى دالة حتى تبقى شيفرة الواجهة أوضح.

مثال:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

بعد تشغيل التطبيق والضغط على الزر، سيكون الناتج في الـ Console:

```text
 /\_/\
( o.o )
 > ^ <
```

وهنا يحدث أمران عند ضغط المستخدم على الزر:

1. الزر يستدعي الدالة `printCat()`
2. الدالة تنفّذ أوامر `print` الموجودة بداخلها

وبهذه الطريقة تصبح شيفرة الواجهة مسؤولة فقط عن إطلاق الحدث، بينما تنتقل التفاصيل المنطقية إلى الدالة.

وفي التطوير العملي، تُنفَّذ معظم أفعال الأزرار بهذه الطريقة.

## الخلاصة

في هذا الفصل تعلّمنا ثلاثة أمور أساسية:

**1. الزر Button**

`Button` هو عنصر SwiftUI المستخدم لتنفيذ الأفعال.

```swift
Button("Start") {

}
```

وعندما يضغط المستخدم على الزر، تُنفذ الشيفرة داخل الأقواس.

**2. إخراج print**

تُستخدم `print` لطباعة معلومات التصحيح إلى الـ Console.

```swift
print("Hello")
```

وبهذا يستطيع المطور مراقبة حالة تشغيل البرنامج.

**3. الدوال**

الدالة عبارة عن كتلة قابلة لإعادة الاستخدام من الشيفرة.

```swift
func sayHello() {
    print("Hello")
}
```

واستدعاؤها يكون بالشكل:

```swift
sayHello()
```

وإذا احتاجت الدالة إلى بيانات، فيمكن استخدام المعاملات:

```swift
func sayHello(name: String) {
    print(name)
}
```

في الفصول القادمة سنواصل التعرف إلى مزيد من عناصر SwiftUI وكيفية تفاعل البيانات مع الواجهة.

## تمارين

جرّب إنجاز التمارين التالية:

1. أنشئ زرّاً بعنوان `Hello`
2. عند الضغط على الزر، اطبع `Hello Swift` في الـ Console
3. انقل الشيفرة الخاصة بالطباعة إلى دالة، ثم استدعِ هذه الدالة من الزر
