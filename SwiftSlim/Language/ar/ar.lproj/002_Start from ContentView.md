# البدء من ContentView

## الاستعداد قبل الدرس

في هذا الفصل، سنبدأ من ملف `ContentView.swift` ونتعرّف بشكل منهجي إلى البنية الأساسية في SwiftUI، بما في ذلك:

- التعليقات
- بنية `View`
- تخطيطات `VStack` و`HStack` و`ZStack`
- أيقونات SF Symbols
- معدّلات العرض `modifier`
- شيفرة المعاينة `#Preview`

أولاً، ابحث عن مشروع Xcode الذي أنشأته سابقاً، ثم انقر مرتين على ملف `.xcodeproj`.

في منطقة Navigator على اليسار، اختر ملف `ContentView.swift`.

ملاحظة: عند كل مرة تفتح فيها المشروع، قد تظهر في Canvas عبارة `Preview paused`. يمكنك الضغط على زر التحديث لاستعادة المعاينة.

![Swift](../../RESOURCE/002_view7.png)

## التعرّف إلى ContentView

شيفرة `ContentView`:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

على الرغم من أن الشيفرة قصيرة، فإنها تتضمن البنية الجوهرية لـ SwiftUI.

### 1. التعليقات

في أعلى الملف:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/2/23.
//
```

هذه تعليقات ملف، تُستخدم لتسجيل معلومات مثل اسم الملف واسم المشروع واسم المنشئ ووقت الإنشاء.

في Swift نستخدم `//` للتعليق أحادي السطر:

```swift
// هذا تعليق
```

التعليقات تساعد على تحسين قابلية قراءة الشيفرة وتسهّل على المطوّر فهم المنطق.

في التطوير العملي قد تصبح الشيفرة صعبة الفهم. وإذا لم تكتب تعليقات، فقد تجد نفسك بعد بضعة أيام غير قادر على فهم ما كتبته أنت بنفسك.

لذلك، فإن إضافة التعليقات أثناء كتابة الشيفرة تُعد عادة جيدة. فهي تساعدك في تسجيل المنطق وتسهّل صيانة الشيفرة لاحقاً.

**تعطيل الشيفرة مؤقتاً**

يمكن استخدام التعليقات أيضاً لتعطيل جزء من الشيفرة مؤقتاً من أجل تتبّع مشكلة ما.

مثلاً:

```text
A
B
C
```

إذا كانت هناك مشكلة في أحد هذه الأجزاء الثلاثة، فيمكننا التعليق على كل جزء مؤقتاً لاكتشاف مكان الخطأ.

لنبدأ بالتعليق على `A`:

```text
// A
B
C
```

إذا عادت الشيفرة إلى العمل بشكل طبيعي بعد تعطيل `A`، فهذا يعني أن المشكلة غالباً في `A`.

أما إذا استمرت المشكلة، فننتقل إلى تعطيل `B`، وهكذا حتى نعثر على الجزء الذي يسبب الخطأ.

في عملية التطوير نواجه كثيراً من المشكلات، وغالباً ما نحتاج إلى تعليق الشيفرة مؤقتاً لتحديد السبب. وهذا يساعدنا على تحديد موضع الخطأ وإيجاد الـ BUG.

في Xcode يمكن استخدام الاختصار:

```text
Command ⌘ + /
```

لإضافة التعليقات أو حذفها بسرعة.

### 2. استيراد إطار SwiftUI

```swift
import SwiftUI
```

هذا السطر يعني استيراد إطار SwiftUI.

أنواع مثل `View` و`Text` و`Image` و`VStack` كلها تأتي من هذا الإطار.

إذا لم تستورد SwiftUI، فسيعرض Xcode الخطأ التالي:

```text
Cannot find type 'View' in scope
```

أي أن المترجم لا يستطيع التعرّف إلى النوع `View`.

### 3. بنية View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

عند رؤية هذه البنية لأول مرة، قد تبدو غريبة قليلاً لأنها تتضمن كلمات مثل `struct` و`View` و`var` و`body` و`some`.

نحن لم نتعلّم هذه الكلمات بعد، لكن يكفي الآن أن نفهم أن هذه الشيفرة تنشئ واجهة باسم `ContentView`.

يمكنك أن تتخيّل `View` كلوحة رسم، بينما SwiftUI هي أدوات الرسم التي نستخدمها عليها.

مثلاً:

![Swift](../../RESOURCE/002_view.png)

في الصورة أعلاه تظهر ثلاث صفحات، وفي الحقيقة كل صفحة منها هي `View`.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

عندما نطوّر تطبيقاً باستخدام SwiftUI، تكون كل صفحة في التطبيق عبارة عن `View`.

### 4. شيفرة SwiftUI

داخل الـ `View` نجد شيفرة SwiftUI التالية:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

هذه الشيفرة تعني: حاوية تخطيط عمودي تعرض أيقونة ونصاً.

![Swift](../../RESOURCE/002_view1.png)

#### تخطيط VStack

```swift
VStack { }  // تخطيط عمودي
```

`VStack` تمثل حاوية تخطيط عمودي. حيث تُرتّب العناصر بداخلها من الأعلى إلى الأسفل.

![Swift](../../RESOURCE/002_view8.png)

أشهر ثلاثة أنواع من التخطيط في SwiftUI هي:

- `VStack` — ترتيب عمودي
- `HStack` — ترتيب أفقي
- `ZStack` — ترتيب متراكب على المحور Z

```swift
HStack { }  // ترتيب أفقي
ZStack { }  // ترتيب متراكب
```

الشكل التوضيحي لترتيب هذه الحاويات:

![Swift](../../RESOURCE/002_view2.png)

مثلاً، إذا استخدمنا `HStack` فسنحصل على ترتيب أفقي:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

سنلاحظ أن أيقونة الكرة الأرضية والنص أصبحا معروضين جنباً إلى جنب.

![Swift](../../RESOURCE/002_view3.png)

عندما نحتاج إلى ترتيب أفقي نستخدم `HStack`، وعندما نحتاج إلى التراكب نستخدم `ZStack`.

#### Image وSF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

هذه الشيفرة تعرض أيقونة كرة أرضية من مكتبة أيقونات Apple المعروفة باسم SF Symbols.

![Swift](../../RESOURCE/002_view9.png)

ومعنى الشيفرة هو: عرض أيقونة كرة أرضية بحجم كبير ولون مميّز.

ولا يقتصر الأمر على الكرة الأرضية فقط، بل يمكننا عرض أيقونات أخرى.

مثلاً، لعرض حقيبة:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**كيف نعرض أيقونات أخرى؟**

نحتاج إلى استخدام مكتبة الأيقونات الرسمية من Apple، وهي SF Symbols.

انتقل إلى موقع Apple Developer وقم بتنزيل [SF Symbols](../../RESOURCE/https://developer.apple.com/sf-symbols/).

![Swift](../../RESOURCE/002_sf.png)

ثم افتح تطبيق SF Symbols.

![Swift](../../RESOURCE/002_sf1.png)

في الجهة اليسرى ستجد تصنيفات الرموز، وفي الجهة اليمنى ستظهر الأيقونات المطابقة.

انقر بزر الفأرة الأيمن على أيقونة، ثم اختر نسخ الاسم. وسيكون الاسم هو النص الذي نستخدمه داخل الشيفرة.

مثلاً:

```text
"globe"
"backpack"
"heart"
```

ضع الاسم المنسوخ داخل `Image(systemName:)` لتعرض أيقونة مختلفة.

ملاحظة: لكل أيقونة في SF Symbols حد أدنى من إصدار النظام المدعوم. وإذا كان إصدار النظام منخفضاً، فقد لا تظهر الأيقونة. ويمكنك رؤية معلومات التوافق داخل تطبيق SF Symbols.

#### الـ Modifiers

في SwiftUI، المعدّل `modifier` هو وسيلة لتغيير مظهر الواجهة أو سلوكها.

يمكنك أن تتخيل المعدّلات مثل الملابس: المظهر الأساسي هو نفسه، لكن النتيجة النهائية تختلف بحسب ما تضيفه عليه.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

هنا `imageScale` و`foregroundStyle` هما معدّلان مطبّقان على `Image`.

**1. imageScale**

```swift
.imageScale(.large)
```

تتحكم هذه الخاصية في حجم أيقونات SF Symbols:

- `.small`
- `.medium`
- `.large`

![Swift](../../RESOURCE/002_view11.png)

يمكننا تعديل هذه الخيارات لرؤية اختلاف الحجم.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

تتحكم `foregroundStyle` في لون المقدمة.

القيمة `.tint` تعني لون التمييز الحالي في البيئة، والذي يكون أزرق افتراضياً على iOS.

وإذا أردنا تغيير اللون إلى الأحمر:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Text

`Text` هو عنصر النص، ويُستخدم لعرض السلاسل النصية.

```swift
Text("Hello, world!")
```

مثلاً، إذا أردت عرض اسمي:

```swift
Text("FangJunyu")
```

ملاحظة: يجب أن يكون النص داخل علامتي اقتباس مزدوجتين `""`.

يمكنك أن تجرّب عرض اسمك أو رقم هاتفك أو أي محتوى نصي آخر.

#### padding

في SwiftUI، تُستخدم `padding` لإضافة مساحة فارغة بين محتوى العنصر وحدوده.

```swift
HStack {
    ...
}
.padding()
```

هذه الشيفرة تعني إضافة هوامش داخلية افتراضية حول عنصر `HStack`.

**ما المقصود بـ padding؟**

المقصود بها هو المسافة الفارغة بين محتوى العنصر وحدوده.

في الشكل التالي، بعد إضافة `padding` إلى الـ `HStack` الأزرق، ستلاحظ أن المنطقة الزرقاء تبدو وكأنها انكمشت إلى الداخل.

![Swift](../../RESOURCE/002_view6.png)

**الهامش الافتراضي**

المعدّل `padding()` يستخدم المسافة القياسية الموصى بها من النظام.

```swift
.padding()
```

وقد تختلف هذه القيمة بحسب المنصة والسياق:

- في iOS تكون غالباً نحو `16 pt`
- في macOS أو watchOS قد تختلف وفقاً لإرشادات التصميم لكل منصة

**تخصيص الهوامش**

يمكن أيضاً تحديد الحافة التي نريد إضافة الهامش إليها:

1. تحديد اتجاه واحد

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

معاني الاتجاهات:

- `.top` — هامش علوي
- `.bottom` — هامش سفلي
- `.leading` — هامش البداية
- `.trailing` — هامش النهاية

![Swift](../../RESOURCE/002_view12.png)

ملاحظة: `leading` و`trailing` يتكيّفان تلقائياً مع اتجاه اللغة. ففي اللغات من اليمين إلى اليسار، مثل العربية، ينعكس الاتجاه تلقائياً.

2. تحديد عدة اتجاهات

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

يمكنك استخدام مصفوفة لتحديد أكثر من اتجاه في وقت واحد. وسنتعلّم المصفوفات لاحقاً بشكل أعمق، أما الآن فيكفي التعرف إلى هذه الصيغة.

3. تحديد الاتجاه الأفقي أو العمودي

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

وهذا يعادل:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**إزالة الهوامش**

إذا كنت لا تريد أي هامش، يمكنك استخدام `.padding(0)`:

```swift
.padding(0)
```

أو حذف المعدّل بالكامل:

```swift
// .padding()
```

### 6. شيفرة المعاينة Preview

```swift
#Preview {
    ContentView()
}
```

هذا يعني إنشاء معاينة لـ `ContentView` داخل Canvas.

ملاحظة: `#Preview` هي صيغة جديدة أضيفت في Swift 5.9 وXcode 15. قبل ذلك، كانت SwiftUI تستخدم بنية `PreviewProvider`.

**ماذا يحدث إذا علّقنا `#Preview`؟**

إذا قمنا بالتعليق على شيفرة المعاينة:

```swift
// #Preview {
//    ContentView()
// }
```

فلن يعرض Canvas أي محتوى قابل للمعاينة.

![Swift](../../RESOURCE/002_xcode.png)

وذلك لأن `#Preview` هو الذي يحدد للـ Canvas ما الذي يجب أن يعرضه.

عندما نحتاج إلى معاينة واجهة SwiftUI في Xcode، نضيف `#Preview`. وإذا لم نحتج إلى المعاينة، يمكننا تعليق هذا الجزء أو حذفه.

## الخلاصة

على الرغم من أن ملف `ContentView.swift` لا يحتوي على كثير من الشيفرة، فإنه يغطي بالفعل عدة مفاهيم أساسية في SwiftUI. وقد يبدو غريباً للمبتدئين في البداية، لكن عند تفكيكه جزءاً جزءاً، يمكننا بناء فهم أولي واضح.

في هذا الفصل تعلمنا أولاً التعليقات `//`، والتي يمكن استخدامها لشرح المنطق أو تعطيل الشيفرة مؤقتاً.

ثم فهمنا أن ملف SwiftUI يجب أن يستورد إطار SwiftUI:

```swift
import SwiftUI
```

من دون استيراد هذا الإطار لن يستطيع المترجم التعرف إلى أنواع مثل `View`.

بعد ذلك تعرّفنا إلى البنية الأساسية لواجهة SwiftUI:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

حيث إن `ContentView` هو اسم الواجهة.

كما تعرّفنا إلى ثلاثة أنواع شائعة من حاويات التخطيط: `VStack` للترتيب العمودي، و`HStack` للترتيب الأفقي، و`ZStack` للتراكب.

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

في هذه الشيفرة نعرض حاوية ترتيب عمودي تحتوي على أيقونة ونص.

يمكن لـ `Image` عرض أيقونات SF Symbols، ويمكن التحكم في حجم الأيقونة ولونها باستخدام المعدّلات.

أما `Text` فيُستخدم لعرض النصوص.

وتعمل `padding` على إضافة مساحة فارغة حول العنصر.

وفي النهاية، تعرّفنا إلى `#Preview` الذي يعرض المعاينة في Canvas.

### تمارين بعد الدرس

بالنسبة للمبتدئين، قد تبدو مفاهيم هذا الفصل كثيرة بعض الشيء. لذلك يمكننا تعميق الفهم عبر بعض التمارين:

- غيّر اسم أيقونة SF Symbols
- غيّر لون الأيقونة إلى الأسود
- استبدل `VStack` بـ `HStack`
- علّق `Image` أو `Text` وراقب كيف تتغيّر المعاينة

### معلومة إضافية: الإكمال التلقائي Code Completion

أثناء كتابة الشيفرة، قد تكون لاحظت أن Xcode يعرض تلقائياً قائمة بالخيارات المتاحة.

مثلاً عند تعديل `imageScale`:

```swift
.imageScale(.)
```

فإن Xcode يعرض الخيارات الممكنة:

![Swift](../../RESOURCE/002_view10.png)

وهذه هي آلية الإكمال التلقائي `Code Completion`. وهي تعتمد على استنتاج النوع واقتراح أعضاء الـ enum، وتساعد على تسريع الكتابة وتقليل الأخطاء.

سنتعرّف لاحقاً بشكل رسمي إلى `enum`. أما الآن فيكفي أن تأخذ عنها فكرة أولية بسيطة.
