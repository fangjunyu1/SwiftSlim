# المتغيرات والثوابت

يركز هذا الفصل على المتغيرات والثوابت في Swift، مع التعرف إلى بعض أنواع البيانات الشائعة والعوامل الأساسية.

هذه المفاهيم من أكثر أسس البرمجة أهمية، كما أنها تشكل قاعدة مهمة لفهم SwiftUI لاحقاً.

## المتغيرات والثوابت في الحياة اليومية

يمكننا فهم المتغيرات والثوابت من خلال أمثلة من الحياة اليومية.

أمثلة على المتغيرات:

- محتوى كل برنامج في التلفاز يختلف
- الطقس يتغير كل يوم
- الساعة تتغير كل ثانية

القاسم المشترك بين هذه الأشياء أنها تتغير.

أما إذا كان التلفاز يعرض الصورة نفسها دائماً، والطقس مشمساً دائماً، وعقارب الساعة لا تتحرك أبداً، فحينها ستكون هذه الأشياء ثوابت.

إذن: هناك ما يمكن أن يتغير، وهناك ما لا يتغير.

## فهم المتغيرات والثوابت

في تطوير التطبيقات، نحتاج كثيراً إلى أن يُدخل المستخدم أو يحفظ بعض المعلومات.

مثلاً:

- اسم الحساب
- تاريخ الميلاد
- معلومات التواصل
- العنوان

يجب حفظ هذه المعلومات ثم عرضها.

مثلاً، إذا أدخل المستخدم في التطبيق اسماً:

```text
FangJunyu
```

فنحن نحتاج إلى حفظ هذا الاسم لعرضه لاحقاً في التطبيق.

يمكنك أن تتخيل هذه العملية على أنها وضع شيء داخل درج.

فعندما نحفظ الاسم، فكأننا نضعه داخل درج.

وبما أن هناك الكثير مما قد نحتاج إلى حفظه، فسيكون لدينا الكثير من الأدراج. ولكي نعرف ما بداخل كل درج، علينا أن نعطيه اسماً.

مثلاً:

```text
name
```

في هذا المثال، `name` هو اسم الدرج، و`FangJunyu` هي القيمة المحفوظة داخله.

![Var](../../RESOURCE/007_var.png)

**في Swift، إذا أردنا حفظ بيانات، فلا بد من التصريح عنها باستخدام متغير `var` أو ثابت `let`.**

وبما أن الاسم عادة يمكن أن يتغير، فمن المناسب استخدام متغير:

```swift
var name = "FangJunyu"
```

هذا يعرّف متغيراً اسمه `name`، ونوعه `String`، وقيمته `"FangJunyu"`.

### الفرق بين المتغير والثابت

يُعلن عن المتغير باستخدام:

```swift
var
```

ويُعلن عن الثابت باستخدام:

```swift
let
```

مثلاً:

```swift
var name = "FangJunyu"
let id = 123456
```

إذا كانت القيمة معلنة كمتغير، فهذا يعني أنه يمكن تعديلها لاحقاً. أما إذا كانت معلنة كثابت، فلا يمكن تغييرها بعد ذلك.

إذن الفرق الجوهري بينهما هو: هل يُسمح للقيمة أن تتغير أم لا؟

### تعديل ثابت

إذا حاولت الشيفرة تعديل قيمة ثابت:

```swift
let name = "Sam"
name = "Bob"
```

فإن Swift ستعرض رسالة تفيد بعدم إمكانية الإسناد لأن `name` ثابت.

```text
Cannot assign to property: 'name' is a 'let' constant
```

هذا يساعد المطور على منع تعديل بعض البيانات المهمة عن طريق الخطأ.

### العرض داخل SwiftUI

افتح ملف `ContentView.swift` وصرّح عن متغير داخل الـ `View`:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

في هذا المثال، أعلنا عن متغير اسمه `name` ثم عرضناه بواسطة `Text`.

إذا قمنا بتعديل قيمة المتغير:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

فإن النص المعروض بواسطة `Text` سيتغير أيضاً.

وبهذه الطريقة يمكننا استخدام المتغيرات للتحكم في المحتوى المعروض، بدلاً من تعديل النص مباشرة داخل `Text` في كل مرة.

وإذا احتجنا إلى استخدام هذه المعلومة في مكان آخر، فيمكننا أيضاً تمرير المتغير إليه، وكأننا أخذنا ما في "الدرج" لنستخدمه في مكان آخر.

### موضع المتغيرات والثوابت في SwiftUI

في SwiftUI، تُكتب المتغيرات والثوابت عادة خارج `body`:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

وبالطبع يمكن أيضاً كتابتها داخل `body`:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

لكن هناك فرقاً مهماً بين الحالتين:

- إذا كُتبت المتغيرات والثوابت داخل `body`، فسيُعاد إنشاؤها في كل مرة يُعاد فيها حساب العرض
- أما إذا كُتبت خارج `body`، فستكون خصائص تابعة لبنية الـ `View`، وسيكون تنظيم الشيفرة أوضح

لذلك، ففي التطوير العملي تُكتب المتغيرات والثوابت عادة خارج `body`.

## أنواع البيانات

يمكن للمتغيرات أن تخزن أنواعاً مختلفة من القيم، وتُسمى هذه الأنواع بأنواع البيانات.

على سبيل المثال، استخدمنا سابقاً `Text` لعرض نص:

```swift
Text("Hello, World")
```

القيمة `"Hello, World"` هي من نوع `String`.

ويمثّل `String` النصوص، وفي Swift يجب وضع النص داخل علامتي اقتباس مزدوجتين.

مثلاً:

```swift
var hello = "Hello, World"
```

إلى جانب النصوص، توجد أنواع أخرى كثيرة في Swift.

في مرحلة البداية، أكثر أربعة أنواع شيوعاً هي:

- `String`
- `Int`
- `Double`
- `Bool`

وهذه الأنواع الأربعة تكفي لمعالجة الجزء الأكبر من البيانات الأساسية.

**String**

يمثل النصوص، مثل:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

ويُستخدم عادةً لأسماء المستخدمين، والعناوين، والمحتوى النصي.

**Int**

يمثل الأعداد الصحيحة، أي الأعداد من دون كسور.

مثلاً:

```swift
var age = 26
var count = 100
```

ويُستخدم كثيراً لتمثيل العمر، والعدد، والعدّاد.

**Double**

يمثل الأعداد ذات الفاصلة العشرية.

مثلاً:

```swift
var weight = 74.5
var height = 185.0
```

في Swift، الأعداد العشرية تُفهم غالباً افتراضياً على أنها من نوع `Double`.

وهناك نوع مشابه اسمه `Float`، لكنه أقل دقة وأضيق نطاقاً، لذلك يُستخدم `Double` أكثر في التطوير العملي.

**Bool**

يُستخدم `Bool` للتعبير عن حالتين فقط: `true` و`false`.

مثلاً:

```swift
var isShowAlert = true
var isLogin = false
```

ويُستخدم هذا النوع كثيراً في الجمل الشرطية، مثل تحديد ما إذا كان يجب عرض نافذة تنبيه أم لا.

وفي كل حالة يكون لها احتمالان فقط، يكون `Bool` مناسباً جداً.

### العرض داخل SwiftUI

عندما نعرض قيمة من نوع `String`، يمكننا تمريرها مباشرة إلى `Text`:

```swift
Text(name)
```

لكن الأنواع مثل `Int` و`Double` ليست نصوصاً، لذلك لا يمكن عرضها مباشرة داخل النص بالشكل نفسه.

وإذا أردنا عرض هذه القيم داخل `Text`، نستخدم إدراج السلاسل النصية `String Interpolation`:

```swift
\()
```

ويجب أن يُكتب هذا الإدراج داخل سلسلة نصية وأن توضع القيمة أو المتغير بين `\(` و`)`.

مثلاً:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

في المثال أعلاه، تمثل `""` سلسلة نصية، بينما تُستخدم `\()` لإدراج متغيرات أو ثوابت داخل هذه السلسلة.

مثلاً:

```swift
"DoubleNum: \(num * 2)"
```

سيظهر المحتوى بالشكل:

```text
DoubleNum: 2
```

وبهذه الطريقة، نستطيع إدراج المتغيرات والثوابت داخل النص ثم عرضها بواسطة `Text`.

ملاحظة: لا يمكن استخدام `\()` إلا داخل سلسلة نصية.

## العوامل

عند التصريح عن متغير أو ثابت، كثيراً ما نرى الصيغة التالية:

```swift
var num = 1
```

الرمز `=` هنا يُسمى عامل الإسناد.

ومهمته هي إسناد القيمة الموجودة في اليمين إلى المتغير الموجود في اليسار.

في هذا المثال، أُسندت القيمة `1` إلى المتغير `num`.

إلى جانب عامل الإسناد، هناك أيضاً عدة عوامل حسابية شائعة:

- `+`
- `-`
- `*`
- `/`

عندما نجري حسابات على الأعداد، مثل `Int` أو `Double`، فإننا نستخدم هذه العوامل.

مثلاً:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

بعد تنفيذ العملية الحسابية، تُحفَظ النتيجة داخل المتغير.

## الخلاصة

تُعد المتغيرات والثوابت والعوامل من أكثر مفاهيم البرمجة أساسية.

فباستخدام المتغيرات والثوابت يمكننا حفظ أنواع متعددة من البيانات داخل البرنامج. وباستخدام أنواع البيانات يمكننا تحديد نوع القيمة بوضوح. وباستخدام العوامل يمكننا إجراء العمليات الحسابية ومعالجة البيانات.

هذه المفاهيم تشبه الأدوات الأساسية في عالم البرمجة. وإتقانها يمنحنا أساساً متيناً لمتابعة تعلّم Swift وSwiftUI لاحقاً.
