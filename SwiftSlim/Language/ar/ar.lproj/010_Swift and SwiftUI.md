# Swift وSwiftUI

عند تعلم تطوير تطبيقات منصات Apple، تميل كثير من الدروس إلى شرح Swift وSwiftUI بشكل منفصل. وهذا قد يجعل المبتدئ يشعر بأنهما تقنيتان منفصلتان تماماً.

لكن في الحقيقة، Swift وSwiftUI مترابطتان بشدة: فـ Swift هي لغة البرمجة المسؤولة عن المنطق والبيانات، بينما SwiftUI هي إطار الواجهة المبني فوق Swift والمسؤول عن رسم الواجهات والتفاعل معها.

في هذا الفصل سنبني إطاراً ذهنياً واضحاً: ما هي Swift؟ وما هي SwiftUI؟ وكيف يتعاون الاثنان داخل الشيفرة؟

## ما هي Swift؟

Swift هي لغة برمجة حديثة أطلقتها Apple، وتُستخدم لتطوير التطبيقات على iOS وmacOS وwatchOS وtvOS وغيرها من منصات Apple.

![swift](../../RESOURCE/010_swift.png)

Swift لغة قوية من حيث الأنواع `Strongly Typed` وثابتة الأنواع `Statically Typed`. وقد صُممت مع تركيز كبير على الأمان، لتقليل كثير من الأخطاء البرمجية الشائعة مثل المؤشرات الفارغة أو تجاوز الحدود، كما أنها تدعم التكامل السلس مع C وObjective-C.

في التطوير العملي، تتولى Swift غالباً منطق التطبيق، مثل معالجة البيانات، وطلبات الشبكة، وعمليات التخزين.

## ما هي SwiftUI؟

SwiftUI هي إطار جديد لبناء الواجهات أطلقته Apple في عام 2019. وتعتمد SwiftUI على أسلوب برمجة تصريحي `Declarative`، مما يسمح للمطور بوصف بنية الواجهة وسلوكها بشيفرة أبسط وأكثر وضوحاً.

![swiftui](../../RESOURCE/010_swiftui.png)

البرمجة التصريحية تعني أن المطور يوضح للنظام ما الذي يريد عرضه، بينما يتولى النظام تحديث الواجهة تلقائياً بحسب تغيّر البيانات. ولم يعد من الضروري تحديث حالة الواجهة يدوياً، وهذا يبسّط تطوير الواجهة بشكل كبير.

يتمحور SwiftUI حول عناصر الواجهة مثل `Text` و`Image` و`Button`، وحاويات التخطيط مثل `VStack` و`HStack` و`ZStack`. وتتفاعل هذه العناصر من خلال ربط البيانات وإدارة الحالة، مما يجعل الواجهة تستجيب لتغيّرات البيانات وتُحدَّث تلقائياً.

## كيف يتعاون Swift وSwiftUI؟

يمكن تلخيص دور كل من Swift وSwiftUI في النقاط التالية:

**1. Swift: معالجة المنطق والبيانات**

تُستخدم Swift أساساً لإدارة البيانات، وتخزين الحالة، وتنفيذ المنطق. وهذه الشيفرة لا تؤثر عادة بشكل مباشر في رسم الواجهة، بل تتولى معالجة السلوك والبيانات.

مثلاً، معالجة البيانات في Swift:

```swift
let markdown = try? String(contentsOf: url)
```

هذه الشيفرة تتعلق بالمتغيرات، والثوابت، والدوال، والتحكم في التدفق، لكنها لا تُستخدم مباشرة لرسم عناصر الواجهة.

**2. SwiftUI: التصريح عن الواجهة ورسمها**

أما SwiftUI فتُستخدم لبناء محتوى الواجهة وتخطيطها، وهي المسؤولة عن عرض العناصر والتفاعل معها.

مثلاً، إنشاء نص باستخدام SwiftUI:

```swift
Text("SwiftSlim")
```

هذا النوع من الشيفرة مخصص لبناء عناصر الواجهة والتحكم في شكلها.

### مثال أساسي

إليك مثالاً بسيطاً لـ SwiftUI:

```swift
// ContentView.swift

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

في هذا المثال:

**شيفرة SwiftUI**: `VStack` و`Image` و`Text` و`padding` كلها عناصر SwiftUI، وهي مسؤولة عن العرض والتخطيط.

**شيفرة Swift**: التعليقات `//` في أعلى الملف، و`import SwiftUI`، وبنية الشيفرة العامة، كلها تندرج ضمن هيكل شيفرة Swift. أما `#Preview` فهو ماكرو خاص بالمعاينة في Xcode ولا يشارك في رسم الواجهة الفعلية.

### مثال أكثر تقدماً

في المشاريع الحقيقية، غالباً ما تُستخدم Swift وSwiftUI معاً، بحيث يؤدي كل منهما المهمة التي يجيدها:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    func printName() {
        print("name: \(name)")
    }

    var body: some View {
        VStack {
            Text(name)
            Button("Print") {
                printName()
            }
        }
        .padding()
    }
}
```

في هذا المثال:

**شيفرة SwiftUI**: `@State` هو مغلّف خاصية خاص بـ SwiftUI لإعلان الحالة القابلة للتغيير داخل الواجهة. و`Button` و`Text` عنصران من عناصر SwiftUI لعرض المحتوى والتفاعل مع المستخدم.

**شيفرة Swift**: مثل `private var name` و`func printName()`، وهي شيفرة مخصّصة لتخزين البيانات وتنفيذ المنطق، ولا ترسم الواجهة مباشرة.

عندما يضغط المستخدم على الزر، يقوم SwiftUI بتشغيل الشيفرة الموجودة داخله:

```swift
Button("Print") {
    printName()
}
```

في هذا المثال، `Button` عنصر من SwiftUI، بينما `printName()` دالة مكتوبة بلغة Swift لتنفيذ المنطق الفعلي.

وهذا التعاون السلس هو ما يجعل Swift مسؤولة عن المنطق والبيانات، بينما تتولى SwiftUI بناء الواجهة.

## أين تُكتب شيفرة Swift وأين تُكتب شيفرة SwiftUI؟

في SwiftUI، تُبنى الواجهة من خلال الـ `body`، ولذلك فإن كل الشيفرة التي تصف الواجهة تُكتب غالباً داخله.

مثلاً:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

في هذا المثال، `Text` عنصر واجهة، لذلك يجب أن يُكتب داخل `body`، لأن SwiftUI يقرأ `body` ليكوّن الواجهة.

أما الشيفرة غير المتعلقة مباشرة بالرسم، مثل المتغيرات والدوال ومنطق معالجة البيانات، فتُكتب عادة خارج `body`.

مثلاً:

```swift
struct ContentView: View {

    // Swift: بيانات أو منطق
    var name = "Fang"

    func printName() {
        print(name)
    }

    // SwiftUI: واجهة
    var body: some View {
        Text(name)
    }
}
```

ومن المهم ملاحظة أن `body` يمكن أن يحتوي مع ذلك على بناءات Swift مثل `if` و`for`، لكنها تُستخدم هنا للتحكم في إنشاء عناصر SwiftUI:

```swift
if isLogin {
    Text("Welcome")
}
```

لذلك يمكننا تبسيط الفكرة على النحو التالي:

- شيفرة الواجهة مثل `Text` و`Image` و`Button` تُكتب عادة داخل `body`
- وشيفرة البيانات والمنطق مثل المتغيرات والدوال تُكتب غالباً خارج `body`

## ملفات Swift

مع التقدم في التعلم، سنتعرّف أيضاً إلى بنية MVVM، حيث تتكون طبقتا `ViewModel` و`Model` غالباً من شيفرة Swift خالصة ومنفصلة تماماً عن طبقة الواجهة `SwiftUI`.

مثلاً، صنف يستخدم لإدارة حالة التطبيق:

```swift
class AppStorage: ObservableObject {
    static var shared = AppStorage()
    private var isLoading = false
}
```

أو بنية تصف نموذج بيانات:

```swift
struct SponsorStruct: Identifiable {
    var id:String
    var icon: String
}
```

وظيفة هذه الملفات هي إدارة البيانات وتخزينها، وهي لا تُعرض مباشرة على الشاشة، ولذلك فهي تنتمي بالكامل إلى شيفرة Swift.

## الخلفية التاريخية

لفهم العلاقة بين Swift وSwiftUI بشكل أفضل، من المفيد أيضاً معرفة تطور تقنيات Apple عبر الزمن. وحتى عام 2026، مرّت هذه المنظومة بعدة مراحل تطور.

### تاريخ Swift

قبل ظهور Swift، كانت Objective-C هي اللغة الرئيسية لتطوير تطبيقات منصات Apple، وكانت تدعم الدمج مع C. وكانت صيغتها أطول وأكثر صعوبة بالنسبة للمبتدئين:

```Obj-c
// أسلوب Objective-C
NSString *name = @"Fang";
NSLog(@"Hello，%@", name);
```

في عام 2014، أعلنت Apple عن Swift في WWDC. وقد جاءت Swift بصياغة أحدث وأمان أعلى في التعامل مع الأنواع، ثم بدأت تدريجياً تحل محل Objective-C كلغة التطوير الأساسية:

```swift
// أسلوب Swift
let name = "Fang"
print("Hello, \(name)")
```

ومع ذلك، لم تختفِ Objective-C تماماً. فما زالت مستخدمة في كثير من المشاريع القديمة وفي بعض الأطر منخفضة المستوى. لذلك تبقى معرفة أساسياتها مفيدة لمن يعمل على صيانة المشاريع القديمة أو يريد فهم الجوانب الأعمق من النظام.

### تاريخ SwiftUI

قبل ظهور SwiftUI، كان iOS يعتمد على **UIKit** بينما كان macOS يعتمد على **AppKit**. وكانت هاتان المنظومتان تتبعان نمطاً برمجياً أمْرياً `Imperative`، حيث كان على المطور إما سحب المكونات بصرياً في Storyboard أو كتابة كثير من الشيفرة للتحكم في حالة الواجهة.

![storyboard](../../RESOURCE/010_xcode.png)

وفي عام 2019، أطلقت Apple SwiftUI رسمياً في WWDC، وقدّمت من خلالها نموذج البرمجة التصريحية، مما بسّط كثيراً من عملية تطوير الواجهات.

![storyboard](../../RESOURCE/010_xcode1.png)

لكن من المهم الانتباه إلى أن SwiftUI ليست تنفيذاً مستقلاً بالكامل على المستوى الأدنى. ففي المنصات المختلفة، تعمل SwiftUI أساساً بالتعاون مع UIKit في iOS أو AppKit في macOS، وما زالت تعتمد عليهما في الطبقة السفلية.

### العلاقة بين Swift وUIKit / AppKit

على الرغم من أن Swift لغة برمجة عامة يمكنها العمل على مختلف منصات Apple، فإنها لا تستطيع وحدها أن تحل محل UIKit أو AppKit بالكامل. ففي بعض متطلبات الواجهة المعقدة، أو في الميزات التي لم تغطها SwiftUI بعد، ما زال المطور بحاجة إلى UIKit أو AppKit.

فعلى سبيل المثال، UIKit ناضجة جداً في إدارة متحكمات العرض المعقدة، والرسوم المتحركة، والإيماءات، وقد خضعت لتجارب إنتاج واسعة. أما SwiftUI، فرغم تطورها المستمر، فما زالت هناك بعض الحالات الطرفية التي تظهر فيها محدودياتها.

ولهذا، يختار كثير من المطورين استخدام SwiftUI مع UIKit أو AppKit في المشروع نفسه، حتى يستفيدوا من مزايا كل طرف.

ومن هذا المنظور يمكننا فهم SwiftUI بوصفها طبقة عالية المستوى فوق UIKit / AppKit. ولذلك فإن معرفة بعض أساسيات UIKit وAppKit أثناء تعلم SwiftUI ستكون مفيدة لاحقاً عند صيانة المشاريع القديمة أو تنفيذ الوظائف المعقدة.

## الخلاصة

**Swift**: تُستخدم أساساً لكتابة المنطق، ومعالجة البيانات، والتحكم في التدفق، وهي ليست مسؤولة مباشرة عن تخطيط الواجهة.

**SwiftUI**: تُستخدم لبناء واجهات المستخدم بطريقة تصريحية، وكل ما يتعلق بعرض العناصر وتخطيطها يندرج ضمنها.

في التطوير العملي، تُستخدم Swift وSwiftUI معاً غالباً: Swift تتولى المنطق، وSwiftUI تتولى الواجهة.

ومن Objective-C وUIKit إلى Swift وSwiftUI، اتجهت بيئة تطوير Apple تدريجياً نحو أساليب أحدث وأكثر بساطة. ومع ذلك، تظل UIKit وAppKit مهمتين في كثير من المشاريع التقليدية.

ومن خلال فهم العلاقة بين Swift وSwiftUI، يمكننا تطوير تطبيقات iOS وmacOS بكفاءة أكبر، كما يمكننا اتخاذ قرارات تقنية أفضل عند صيانة المشاريع القديمة.
