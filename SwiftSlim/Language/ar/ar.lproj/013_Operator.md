# العوامل

في هذا الفصل، سنتعلّم عوامل Swift (`Operators`).

عندما نجري عمليات حسابية، أو مقارنات، أو أحكاماً منطقية، فإننا نحتاج إلى استخدام العوامل. وهذه العمليات تمتد عبر جميع مراحل تنفيذ البرنامج، وهي من أكثر المهارات الأساسية في البرمجة.

```swift
let a = 100 * 8
```

أثناء تعلّم البرمجة، يلاحظ كثير من الناس أن من يملكون قدراً من التفكير الرياضي يفهمون منطق البرامج بسهولة أكبر. وهذا لأن البرمجة في جوهرها تعتمد على "الحساب" و"الحكم وفق القواعد"، وهذان الأمران هما بالضبط ما تنهض به العوامل.

وفي التطوير العملي، لا تُستخدم العوامل فقط في الحسابات الرقمية البسيطة، بل تُستخدم أيضاً في كثير من السيناريوهات الفعلية.

مثلاً، في لعبة تعتمد على القطع، عندما يسحب المستخدم قطعة إلى موضع معين، قد نحتاج إلى حساب:

- الإحداثيات الخاصة بالقطعة
- الموضع الذي يمكن أن تملأه القطعة
- ما إذا كان الموضع المستهدف صالحاً للاستخدام

هذه العمليات من "الحساب + الحكم" تعتمد في جوهرها على العوامل.

![Block](../../Resource/013_block.png)

لذلك فإن تعلم العوامل لا يعني تعلّم الحسابات الرقمية فقط، بل يعني أيضاً وضع أساس مهم للتخطيط اللاحق للواجهات، ومعالجة البيانات، ومنطق التفاعل.

## كيف تُنفّذ العمليات الحسابية في SwiftUI

في SwiftUI، تُنفّذ العمليات الحسابية غالباً عن طريق زر `Button` يقوم باستدعاء دالة، ثم يجري المنطق الحسابي داخل هذه الدالة.

مثلاً:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

عند الضغط على الزر، سيكون الإخراج في الـ Console:

```text
c: 20
```

في هذا الفصل، سنوحّد أسلوبنا بحيث نكتب منطق العمليات داخل `func`، ثم نستخدم الزر لتشغيله، ونستخدم `print` لإظهار النتائج.

وفي الأمثلة اللاحقة، ومن أجل تبسيط عرض الشيفرة، سنكتب النتائج مباشرة داخل الشيفرة بدلاً من تكرار استخدام `print` في كل مرة.

## إدارة مخرجات Console

بما أن كل ضغطة على الزر ستنتج مخرجات جديدة، فيمكنك الضغط على زر المسح في أسفل يمين منطقة Console لتنظيف السجل القديم، حتى يكون من الأسهل رؤية النتيجة الحالية.

![Console](../../Resource/013_console.png)

## صيغة أخرى لكتابة Button

عندما يكون الزر مسؤولاً فقط عن تنفيذ دالة واحدة، يمكن استخدام صيغة أكثر اختصاراً:

```swift
Button(action: ) {
    // View
}
```

في هذه الصيغة، تحدد `action` الدالة التي يجب تنفيذها عند الضغط على الزر، بينما تُستخدم الأقواس المعقوفة `{}` لتعريف الواجهة التي يعرضها الزر.

وبالاعتماد على الدالة `calculate` التي رأيناها سابقاً، يمكن كتابة الزر بالشكل التالي:

```swift
Button(action: calculate) {
    Text("Button")
}
```

عند الضغط على الزر، ستُنفّذ الدالة `calculate` مباشرة.

ولذلك، عندما يكون منطق الزر بسيطاً ويقتصر على استدعاء دالة واحدة فقط، فإن استخدام صيغة `action` يكون أوضح وأكثر توافقاً مع تنظيم الشيفرة.

## العوامل الحسابية

تُستخدم العوامل الحسابية (`Arithmetic Operators`) لإجراء الحسابات الرقمية:

- `+` الجمع
- `-` الطرح
- `*` الضرب
- `/` القسمة
- `%` باقي القسمة

أمثلة:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

ومن المهم الانتباه إلى أن Swift لغة قوية من حيث الأنواع، لذلك فهي لا تدمج تلقائياً بين أنواع رقمية مختلفة:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

عندما يكون النوعان مختلفين، يجب تحويل أحدهما يدوياً.

## عوامل المقارنة

تُستخدم عوامل المقارنة (`Comparison Operators`) للمقارنة بين قيمتين، والنتيجة تكون من نوع `Bool`:

- `==` يساوي
- `!=` لا يساوي
- `>` أكبر من
- `<` أصغر من
- `>=` أكبر من أو يساوي
- `<=` أصغر من أو يساوي

أمثلة:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

تُستخدم عوامل المقارنة كثيراً في الجمل الشرطية. فمثلاً في فصل "عرض الأقوال المتعاقب"، نحتاج إلى التأكد من أن الفهرس لا يتجاوز حدود المصفوفة:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

إذا كانت قيمة `index` أصغر من `sayings.count - 1`، فستكون نتيجة المقارنة `true`، وبالتالي تُنفّذ الشيفرة داخل `if`. أما إذا كانت النتيجة `false`، فلن تُنفّذ الشيفرة التالية.

## العوامل المنطقية

تُستخدم العوامل المنطقية (`Logical Operators`) لدمج عدة شروط معاً:

- `&&` و (AND)
- `||` أو (OR)
- `!` نفي (NOT)

### `&&` — و

عندما يجب أن تتحقق عدة شروط كلها معاً، نستخدم `&&`.

مثلاً:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

في هذا المثال لدينا شرطان: `age >= 18` و`age <= 100`.

ولا تكون قيمة `legalAge` صحيحة (`true`) إلا إذا كان الشرطان معاً صحيحين. وإذا لم يتحقق أحدهما، فستكون النتيجة `false`.

### `||` — أو

عندما يكفي تحقق شرط واحد من عدة شروط، نستخدم `||`.

مثلاً:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

هذا يعني أن `payAllowed` ستكون `true` إذا تحقق أحد الشرطين: `money >= 50` أو `creditCard >= 50`.

ولا تكون النتيجة `false` إلا إذا لم يتحقق الشرطان معاً.

وفي سيناريو الدفع مثلاً، يمكن استخدام النقد أو البطاقة الائتمانية. وإذا لم يصلح أي منهما، فإن الدفع يفشل.

### `!` — النفي

الرمز `!` هو عامل النفي المنطقي (`NOT`)، ويُستخدم لعكس قيمة `Bool`.

يمكننا أن نفهمه ببساطة هكذا:

- يحوّل `true` إلى `false`
- ويحّول `false` إلى `true`

مثلاً:

```swift
let allowed = true
let result = !allowed   // false
```

هنا، `!allowed` تعني أخذ القيمة العكسية لـ `allowed`.

ومن المهم الانتباه إلى أن `!` و`!=` ليسا العامل نفسه. فـ `!` تعكس قيمة منطقية، أما `!=` فهو عامل مقارنة يعيد قيمة منطقية.

## عوامل الإسناد

تُستخدم عوامل الإسناد (`Assignment Operators`) لإسناد القيم إلى المتغيرات أو لتحديثها:

- `=` الإسناد

في أثناء تعريف المتغيرات أو إجراء الحسابات، يظهر الإسناد دائماً:

```swift
let a = 5
```

هذه الشيفرة تعني إسناد القيمة `5` إلى المتغير `a`.

### عوامل الإسناد المركبة

في التطوير العملي، إلى جانب عامل الإسناد الأساسي، نستخدم كثيراً ما يسمى بـ "عوامل الإسناد المركبة"، والتي تسمح بإجراء عملية على القيمة الحالية ثم إعادة إسناد الناتج مباشرة:

- `+=`
- `-=`
- `*=`
- `/=`

وقد تعرّفنا بشكل بسيط إلى هذه العوامل سابقاً في فصل العدّاد.

مثلاً، إذا أردنا أن تزيد قيمة متغير عند الضغط على زر:

```swift
var num = 10
num += 5   // num = 15
```

تعني هذه الشيفرة: أضف `5` إلى `num`، ثم أعد إسناد الناتج إلى `num`.

ولذلك فهي مكافئة تماماً لـ:

```swift
var num = 10
num = num + 5   // num = 15
```

## العامل الثلاثي

في Swift، يُستخدم العامل الثلاثي (`Ternary Operator`) للتحقق من شرط معين، ثم إرجاع واحدة من نتيجتين:

```swift
condition ? value1 : value2
```

ومنطقه كالتالي:

- إذا كان الشرط `true`، تُعاد `value1`
- وإذا كان الشرط `false`، تُعاد `value2`

مثلاً:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

ستتحقق هذه الشيفرة من قيمة `showColor`:

- إذا كانت `true`، فسترجع `Color.blue`، ويظهر لون الخلفية بالأزرق
- وإذا كانت `false`، فسترجع `Color.clear`، أي لن تظهر خلفية

ملاحظة: `Color.clear` تعني لوناً شفافاً.

### الفرق بينه وبين `if-else`

من حيث المنطق، العامل الثلاثي يكافئ جملة `if-else`.

فمثلاً، يمكن كتابة المثال السابق بهذه الصورة:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

المنطق هنا مطابق تماماً: إذا كان الشرط صحيحاً نستخدم النتيجة الأولى، وإذا كان خاطئاً نستخدم النتيجة الثانية.

لكن في الاستخدام العملي، العامل الثلاثي أكثر اختصاراً، ويُعد أنسب للحالات البسيطة التي تعتمد على الاختيار بين احتمالين، خصوصاً عندما نريد إرجاع قيمة مباشرة.

وهو شائع جداً في بناء واجهات SwiftUI، مثلاً:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

هذه الصيغة تسمح لنا بتغيير شكل الواجهة بشكل ديناميكي بحسب الشرط، وكل ذلك في سطر واحد.

## عوامل النطاق

تُستخدم عوامل النطاق (`Range Operators`) للتعبير عن مجال من القيم:

- `...` نطاق مغلق، يشمل الطرفين
- `..<` نطاق نصف مفتوح، لا يشمل الحد الأيمن

مثلاً:

```text
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

يمكن فهم `1...3` على أنه "من 1 إلى 3 شاملاً 3"، بينما `5..<8` تعني "من 5 إلى 8 من دون تضمين 8".

وتوجد أيضاً صيغ موسعة، فقط للمعرفة:

```text
5...    // >= 5
...5    // <= 5
```

وتعبر هذه الصيغ عن "نطاق أحادي الجانب"، وتظهر غالباً في الشروط أو مطابقة الأنماط.

وسنستخدم عوامل النطاق لاحقاً عند تعلّم حلقات `for`. أما الآن، فيكفي أن نفهمها بوصفها تمثل مجالاً مستمراً من القيم.

## التدرّب على العوامل

الآن، دعنا نطبّق بعض العوامل من خلال أمثلة بسيطة داخل الواجهة.

### 1. حساب العدد الكلي للمركبات

مثلاً، إذا كان يمر عبر تقاطع معيّن 500 سيارة يومياً، فكيف نحسب إجمالي عدد السيارات خلال 30 يوماً؟

يمكننا استخدام العوامل الحسابية:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

المعنى هنا بسيط: 500 سيارة يومياً × 30 يوماً = 15000 سيارة.

### 2. تحديد ما إذا كان المستخدم عضواً مدفوعاً

يمكننا أيضاً تحديد ما إذا كان شخص ما عضواً مدفوعاً داخل التطبيق. فإذا كان لديه اشتراك دائم أو اشتراك متكرر، فإننا نعدّه عضواً.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

في هذا المثال استخدمنا العامل `||`:

- `lifeTime = false` تعني أنه لا يملك اشتراكاً دائماً
- `subscription = true` تعني أنه يملك اشتراكاً متكرراً

وبما أن تحقق أحد الشرطين يكفي، فإن `isMember` تكون `true`، وبالتالي يُعتبر هذا الشخص عضواً مدفوعاً.

## الخلاصة

في Swift، تُعد العوامل أساس الحساب ومعالجة المنطق. وكما أن `Text` تُستخدم لعرض المحتوى، فإن العوامل تُستخدم لمعالجة البيانات نفسها.

ومن أبسط العمليات الرقمية إلى الأحكام المنطقية المعقدة، ثم إلى التحكم في الحالة وحسابات التخطيط داخل الواجهة، نجد أن العوامل حاضرة في كل مكان.

ومن خلال التمرّن على سيناريوهات متنوعة، يمكننا أن نألف هذه العوامل ونتقنها تدريجياً، وهي من الركائز المهمة لما سنتعلمه لاحقاً.

### معرفة إضافية - العوامل البتّية

إلى جانب ما سبق، توفّر Swift أيضاً مجموعة من العوامل الأدنى مستوى، وهي العوامل البتّية (`Bitwise Operators`):

- `&` و بتّية
- `|` أو بتّية
- `^` XOR بتّي
- `~` نفي بتّي
- `>>` إزاحة إلى اليمين
- `<<` إزاحة إلى اليسار

مثلاً:

```swift
let a = 6  // binary 110
let b = 3  // binary 011

print(a & b) // 2 (binary 010)
print(a | b) // 7 (binary 111)
print(a ^ b) // 5 (binary 101)
print(~a)    // -7 (bitwise NOT)
```

في Swift، تمثل الأعداد الصحيحة داخلياً باستخدام نظام المكمل الثنائي، ولذلك فإن النفي البتّي ينتج قيمة سالبة مقابلة.

تتعامل هذه العوامل مباشرة مع التمثيل الثنائي، ولذلك تظهر غالباً في المعالجة منخفضة المستوى أو في سيناريوهات تحسين الأداء.

وبالنسبة للمبتدئين، فإن استخدام هذه العوامل ليس شائعاً كثيراً في تطوير iOS أو SwiftUI اليومي، ولذلك لن نتوسع فيها هنا.

وعندما نحتاج في المستقبل إلى التعمق في المنطق منخفض المستوى أو تحسين الأداء، يمكننا العودة إليها وفهمها بشكل أعمق.
