# الخصائص المحسوبة

في هذا الدرس سنتعلم بشكل أساسي الخصائص المحسوبة.

تُستخدم الخصائص المحسوبة لحساب نتيجة جديدة اعتمادًا على بيانات موجودة. يمكنها حساب القيم العددية، ويمكنها أيضًا حساب المحتوى الذي يجب عرضه في واجهات SwiftUI.

على سبيل المثال:

```swift
let a = 10
let b = 20
let c = a + b
```

هنا يمثّل `c` نتيجة جمع `a` و`b`.

في الكود العادي، هذا الأسلوب شائع جدًا.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

ملاحظة: يمكن تنفيذ كود Swift عادي داخل حدث النقر الخاص بـ `Button`.

لكن إذا كتبنا كودًا مشابهًا مباشرة داخل خصائص واجهة SwiftUI، فسنواجه مشكلة.

على سبيل المثال:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

هذا الكود سيؤدي إلى خطأ.

يبدو أن `a` و`b` كُتبا قبل `c`، وكأن قيمة `c` يمكن حسابها.

لكن في تصريح خصائص البنية، لا يمكن استخدام هذه الكتابة مباشرة.

## لماذا لا يستطيع c حساب a + b مباشرة؟

لأن `a` و`b` و`c` ليست ثوابت مؤقتة داخل حدث نقر زر، بل هي خصائص داخل الواجهة `ContentView`.

داخل حدث نقر الزر، يمكن تنفيذ هذا الكود بشكل طبيعي:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

لأنه بعد النقر على الزر، يُنفَّذ الكود بالترتيب. يُنشأ `a` أولًا، ثم يُنشأ `b`، وأخيرًا تُحسب `c` باستخدام `a + b`.

لكن عند التصريح عن الخصائص داخل الواجهة، يكون الوضع مختلفًا:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

هنا `a` و`b` و`c` كلها خصائص داخل الواجهة `ContentView`.

عند إنشاء الواجهة `ContentView`، يحتاج Swift إلى تجهيز هذه الخصائص أولًا. ولضمان أمان عملية الإنشاء، لا يسمح Swift للقيمة الافتراضية لخاصية مخزنة في كائن ما بأن تقرأ مباشرة خصائص أخرى من الكائن نفسه.

لذلك سيؤدي هذا السطر إلى خطأ:

```swift
let c = a + b
```

يمكن فهم الأمر ببساطة هكذا: **عند التصريح عن الخصائص داخل الواجهة، لا يمكن استخدام خاصية عادية مباشرة لحساب خاصية عادية أخرى.**

الخاصية التي تحفظ قيمة مباشرة تُسمى "خاصية مخزنة". ولتسهيل الفهم، يمكن مؤقتًا اعتبارها خاصية عادية.

على سبيل المثال:

```swift
let a = 10
```

`a` تحفظ القيمة `10`.

```swift
let b = 20
```

`b` تحفظ القيمة `20`.

لكن:

```swift
let c = a + b
```

هنا `c` ليست قيمة مكتوبة مباشرة، بل نريد حسابها من خلال `a + b`.

في مثل هذه الحالة، أي "الحصول على نتيجة اعتمادًا على بيانات موجودة"، تكون الخصائص المحسوبة أنسب.

يمكن تعديلها إلى:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

هنا `c` هي خاصية محسوبة.

```swift
var c: Int {
    return a + b
}
```

معناها هو: **عند الحاجة إلى استخدام `c`، اقرأ `a` و`b` عندها، ثم أعد نتيجة `a + b`.**

على سبيل المثال:

```swift
Text("c: \(c)")
```

عندما يعرض `Text` قيمة `c`، عندها فقط تُشغَّل عملية حساب `c`.

## ما هي الخاصية المحسوبة؟

تبدو الخاصية المحسوبة كأنها متغير، لكنها لا تخزن البيانات بنفسها.

على سبيل المثال:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

هنا `c` هي خاصية محسوبة.

هي لا تحفظ قيمة ثابتة مثل الخاصية العادية، بل تعيد تنفيذ الكود داخل `{}` في كل مرة تُقرأ فيها `c`، ثم تعيد نتيجة الحساب.

يمكن فهمها هكذا:

```swift
var c: Int {
    return a + b
}
```

عندما نحتاج إلى استخدام `c`، نذهب لحساب `a + b`.

لذلك، تناسب الخاصية المحسوبة هذا النوع من الحالات: **نتيجة لا تحتاج إلى حفظ منفصل، ويمكن حسابها من البيانات الموجودة.**

## الصيغة الأساسية

تتكون الخاصية المحسوبة عادةً من ثلاثة أجزاء:

```swift
var c: Int {
    return a + b
}
```

### 1. التصريح باستخدام `var`

```swift
var c
```

يجب التصريح عن الخاصية المحسوبة باستخدام `var`، ولا يمكن استخدام `let`.

لأن الخاصية المحسوبة ليست قيمة ثابتة مخزنة، بل نتيجة تُحسب ديناميكيًا في كل مرة تُقرأ فيها.

### 2. تحديد نوع القيمة المعادة

```swift
var c: Int
```

تحتاج الخاصية المحسوبة إلى تحديد نوع القيمة المعادة.

هنا ستعيد `c` في النهاية عددًا صحيحًا، لذلك نوعها هو `Int`.

### 3. استخدام {} لكتابة منطق الحساب

```swift
{
    return a + b
}
```

ما داخل `{}` هو منطق الحساب، وهنا القيمة المعادة هي `a + b`.

## الكلمة المفتاحية return

تحتاج الخاصية المحسوبة إلى إرجاع نتيجة.

على سبيل المثال:

```swift
var c: Int {
    return a + b
}
```

هنا تعني `return`: أعد نتيجة حساب `a + b` إلى الخارج.

إذا كانت الخاصية المحسوبة تحتوي على تعبير واحد ينتج النتيجة مباشرة، فيمكن حذف `return`:

```swift
var c: Int {
    a + b
}
```

لكن إذا احتوت الخاصية المحسوبة على عدة أسطر من الكود، فيجب استخدام `return` لتحديد النتيجة المعادة بوضوح.

على سبيل المثال:

```swift
var totalPriceText: String {
    let total = count * price
    return "الإجمالي: \(total) $"
}
```

تنقسم عملية الحساب هنا إلى خطوتين.

الخطوة الأولى، حساب السعر الإجمالي:

```swift
let total = count * price
```

الخطوة الثانية، دمج السعر الإجمالي في نص ثم إرجاعه:

```swift
return "الإجمالي: \(total) $"
```

إذا حذفنا `return`:

```swift
var totalPriceText: String {
    let total = count * price
    "الإجمالي: \(total) $"
}
```

هذا الكود سيؤدي إلى خطأ.

السبب هو: **هذه الخاصية المحسوبة تحتوي بالفعل على عدة أسطر من الكود، ولم يعد Swift قادرًا على تحديد أي سطر هو النتيجة النهائية التي يجب إرجاعها.**

لذلك، عندما تحتوي الخاصية المحسوبة على سطر نتيجة واحد فقط، يمكن حذف `return`.

```swift
var c: Int {
    a + b
}
```

وعندما تحتوي الخاصية المحسوبة على عدة أسطر من الكود، يُنصح بكتابة `return` بوضوح.

```swift
var totalPriceText: String {
    let total = count * price
    return "الإجمالي: \(total) $"
}
```

## الفرق بين الخاصية المحسوبة والخاصية العادية

الخاصية العادية تخزن البيانات.

```swift
var c = 30
```

هنا تحفظ `c` قيمة محددة: `30`.

الخاصية المحسوبة لا تخزن البيانات.

```swift
var c: Int {
    a + b
}
```

هنا لا تحفظ `c` القيمة `30`. إنها فقط تقدم طريقة حساب.

عند قراءة `c`، ينفذ Swift:

```swift
a + b
```

ثم يعيد نتيجة الحساب.

لذلك، تناسب الخاصية المحسوبة الحالات التي تُحسب فيها النتيجة من بيانات أخرى.

## body هي أيضًا خاصية محسوبة

بعد تعلم الخصائص المحسوبة، يمكننا إعادة فهم أكثر جزء شائع في SwiftUI:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

هنا `body` هي أيضًا خاصية محسوبة.

تُصرَّح باستخدام `var`:

```swift
var body
```

ولها نوع قيمة معادة:

```swift
some View
```

وما داخل `{}` يعيد محتوى واجهة SwiftUI:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

لذلك يمكن فهم `body` هكذا: عندما يحتاج SwiftUI إلى عرض هذه الواجهة، يقرأ `body` وينشئ الواجهة وفقًا للمحتوى الذي تعيده.

إذا كتبنا `return`، يمكن فهمها بهذا الشكل:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

لكن في SwiftUI، عادةً ما نحذف `return` ونكتب مباشرة:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

عندما تتغير بيانات `@State`، سيعيد SwiftUI قراءة `body` ويحدّث الواجهة وفقًا للبيانات الجديدة.

على سبيل المثال:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("العدد: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

بعد النقر على الزر، ستتغير قيمة `count`.

```swift
count += 1
```

بعد تغير `count`، سيعيد SwiftUI حساب `body`، لذلك سيتحدث النص في الواجهة أيضًا.

```swift
Text("العدد: \(count)")
```

وهذا أيضًا سبب أن الواجهة في SwiftUI يمكن أن تتجدد تلقائيًا بعد تغير البيانات.

### لا يُنصح بكتابة حسابات معقدة داخل body

بما أن `body` خاصية محسوبة، فقد تُقرأ وتُعاد حسابها مرات متعددة.

على سبيل المثال:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("الرقم: \(num)")
            Text("الاسم: \(name)")
        }
    }
}
```

عندما تتغير `name`، سيعيد SwiftUI حساب `body`.

في هذه اللحظة، سيُنفَّذ الكود داخل `body` مرة أخرى، وسيُنشأ `num` من جديد أيضًا.

```swift
let num = 10
```

في هذا المثال، `num` بسيط جدًا، لذلك لا يؤثر كثيرًا.

لكن إذا نُفذت داخل `body` حسابات معقدة، مثل تصفية كمية كبيرة من البيانات، أو ترتيبها، أو معالجة الصور، فقد يؤثر ذلك في سلاسة الواجهة.

لذلك، في SwiftUI، ينبغي أن تكون `body` مسؤولة أساسًا عن وصف بنية الواجهة.

يمكن كتابة البيانات المؤقتة البسيطة داخل `body`.

أما الحسابات المعقدة فيمكن وضعها في خاصية محسوبة خارج `body`، أو في دالة، أو في معالجة بيانات مستقلة.

## مثال: الكمية والسعر الإجمالي

فيما يلي، سنفهم الخاصية المحسوبة من خلال مثال بسيط.

لنفترض أن سعر الجزرة الواحدة هو 2 $، ويمكن للمستخدم النقر على الأزرار لتغيير كمية الشراء، وتحتاج الواجهة إلى عرض السعر الإجمالي.

طريقة حساب السعر الإجمالي:

```
الكمية * سعر الوحدة
```

إذا استخدمنا متغيرًا عاديًا لحفظ السعر الإجمالي، فسيكون الأمر مزعجًا بعض الشيء.

لأنه في كل مرة تتغير فيها الكمية، نحتاج إلى تحديث السعر الإجمالي يدويًا.

الطريقة الأفضل هي استخدام خاصية محسوبة:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("السعر: \(price) $")
                Text("الكمية: \(count)")
                Text("السعر الإجمالي: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

تأثير العرض:

![view](../../Resource/024_view.png)

في هذا المثال:

```swift
@State private var count = 1
```

`count` تمثل كمية الجزر. عند النقر على الزر، ستتغير قيمة `count`.

```swift
private let price = 2
```

`price` تمثل سعر الجزرة الواحدة. هنا القيمة ثابتة، لذلك نستخدم `let`.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` تمثل السعر الإجمالي.

لا تحتاج إلى حفظ منفصل، لأن السعر الإجمالي يمكن دائمًا حسابه من خلال `count * price`.

عندما تكون `count` هي `1`:

```swift
totalPrice = 1 * 2
```

تكون نتيجة العرض:

```swift
السعر الإجمالي: 2 $
```

بعد النقر على زر `+`، تصبح `count` مساوية لـ `2`.

عند قراءة `totalPrice` مرة أخرى في هذه اللحظة، سيُعاد حسابها:

```swift
totalPrice = 2 * 2
```

تكون نتيجة العرض:

```swift
السعر الإجمالي: 4 $
```

هذا هو دور الخاصية المحسوبة: حساب نتيجة جديدة ديناميكيًا اعتمادًا على بيانات موجودة.

## يمكن استخدام الخاصية المحسوبة للحكم

لا تستطيع الخاصية المحسوبة حساب القيم العددية فقط، بل يمكنها أيضًا إرجاع نتائج حكم.

على سبيل المثال، عندما نريد أن تكون الكمية الدنيا 1.

عندما تكون الكمية بالفعل 1، لا ينبغي أن يستمر زر `-` في الإنقاص.

يمكن إضافة خاصية محسوبة:

```swift
private var canDecrease: Bool {
    count > 1
}
```

الكود الكامل:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("السعر: \(price) $")
                Text("الكمية: \(count)")
                Text("السعر الإجمالي: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

تأثير العرض:

![view](../../Resource/024_view1.png)

هذا الجزء:

```swift
private var canDecrease: Bool {
    count > 1
}
```

يعني هذا الكود: هل يمكن الإنقاص حاليًا أم لا.

عندما تكون `count` أكبر من `1`:

```swift
canDecrease == true
```

يعني أنه يمكن الإنقاص.

عندما تكون `count` مساوية لـ `1`:

```swift
canDecrease == false
```

يعني أنه لا يمكن الاستمرار في الإنقاص.

### الحكم الشرطي داخل الزر

يستخدم الزر:

```swift
if canDecrease {
    count -= 1
}
```

فقط عندما تكون نتيجة `canDecrease` هي `true`، يمكن إنقاص `count`.

### التحكم في لون مقدمة الواجهة

يمكن أيضًا استخدام `canDecrease` للتحكم في لون مقدمة الزر:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

هنا استخدمنا العامل الثلاثي:

```swift
canDecrease ? Color.primary : Color.gray
```

معنى هذا الكود هو: إذا كانت `canDecrease` تساوي `true`، فاستخدم `Color.primary` كلون المقدمة؛ وإذا كانت `canDecrease` تساوي `false`، فاستخدم `Color.gray`.

`Color.primary` هو لون دلالي نظامي يوفره SwiftUI، ويمثل لون النص الرئيسي في الواجهة الحالية.

في النمط الفاتح، يكون `Color.primary` عادة قريبًا من الأسود؛ وفي النمط الداكن، يكون `Color.primary` عادة قريبًا من الأبيض.

لذلك، فائدة استخدام `Color.primary` هي أنه يتكيف تلقائيًا مع النمط الفاتح والنمط الداكن.

### التحكم في حالة تعطيل الواجهة

يُستخدم `disabled` للتحكم فيما إذا كانت الواجهة في حالة تعطيل:

```swift
.disabled(!canDecrease)
```

عندما تكون قيمة `disabled` هي `false`، يمكن النقر على الواجهة.

عندما تكون قيمة `disabled` هي `true`، تكون الواجهة معطلة ولا يمكن النقر عليها.

استخدام `canDecrease` كشرط هنا يجعل الكود أسهل في الفهم.

عند رؤية `canDecrease`، يمكننا أن نعرف أنه يعني "هل يمكن الإنقاص حاليًا أم لا".

### ملاحظة إضافية: لماذا يوجد حكمان؟

داخل زر `-`:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

هنا استخدمنا `.disabled(!canDecrease)`، واستخدمنا أيضًا `if canDecrease` داخل الزر.

`.disabled(!canDecrease)` مسؤولة عن تعطيل الزر من الواجهة، بحيث لا يستطيع المستخدم النقر عليه.

`if canDecrease` مسؤولة عن التحقق مرة أخرى قبل تنفيذ الكود، ولا تنفذ `count -= 1` إلا عندما يكون الإنقاص ممكنًا.

هذا نوع من الحماية المزدوجة. في التطوير الفعلي، إذا كان الزر قد عُطّل بالفعل، يمكن حذف الحكم الداخلي؛ لكن في مثال تعليمي، نُبقيه ليصبح دور `canDecrease` أوضح.

## الخلاصة

في هذا الدرس تعلمنا أساسًا الخصائص المحسوبة.

الخاصية المحسوبة لا تحفظ القيمة مباشرة، بل تحسب نتيجة اعتمادًا على بيانات موجودة عند قراءتها.

على سبيل المثال:

```swift
var c: Int {
    a + b
}
```

هنا لا تحتاج `c` إلى حفظ منفصل، لأنها يمكن أن تُحسب من خلال `a + b`.

يجب التصريح عن الخاصية المحسوبة باستخدام `var`، ويجب تحديد نوع القيمة المعادة.

```swift
var canDecrease: Bool {
    count > 1
}
```

لا تستطيع الخاصية المحسوبة إرجاع القيم العددية فقط، بل يمكنها أيضًا إرجاع نتائج الحكم، ومحتوى النصوص، وحتى محتوى واجهات SwiftUI.

في هذا الدرس، تعلمنا أيضًا `return`.

تعني `return` إرجاع نتيجة:

```swift
var totalPriceText: String {
    let total = count * price
    return "الإجمالي: \(total) $"
}
```

إذا كانت الخاصية المحسوبة تحتوي على تعبير واحد ينتج النتيجة مباشرة، فيمكن حذف `return`.

```swift
var totalPrice: Int {
    count * price
}
```

بالإضافة إلى ذلك، تعرفنا أيضًا على `Color.primary` و`disabled`.

`Color.primary` هو لون دلالي نظامي في SwiftUI، وسيعدّل تأثير العرض تلقائيًا وفقًا للنمط الفاتح والنمط الداكن.

```swift
.foregroundStyle(Color.primary)
```

يُستخدم `disabled` للتحكم فيما إذا كانت الواجهة معطلة.

```swift
.disabled(true)
```

يعني أنها معطلة ولا يمكن النقر عليها.

```swift
.disabled(false)
```

يعني أنها متاحة ويمكن النقر عليها.

لذلك، الخصائص المحسوبة شائعة جدًا في SwiftUI. يمكنها مساعدتنا على تنظيم نتائج الحساب، وشروط الحكم، ومحتوى العرض بشكل أوضح.
