# حفظ مبلغ الحصالة

في الدرس السابق، قمنا بإنشاء "حصالة" بسيطة. في هذا الدرس، سنتعلم كيف نجعل مبلغ الحصالة محفوظًا بشكل دائم.

يركز هذا الدرس بشكل أساسي على `UserDefaults` وغطاء الخصائص `@AppStorage`، وسنشرح أيضًا دور `onAppear` وطريقة استخدام النوع الاختياري (`nil`).

من المهم ملاحظة أن هذا الدرس يعتمد على كود "الحصالة" الذي أنشأناه في الدرس السابق.

## مشكلة الحفظ الدائم

واجهة "الحصالة" الآن بسيطة جدًا وسهلة الاستخدام.

![Piggy Bank](../../Resource/014_view.png)

لكن توجد مشكلة واضحة: في كل مرة يتم فيها تحديث الواجهة أو إغلاق التطبيق، يعود إجمالي مبلغ "الحصالة" إلى 0، ويضيع كل المبلغ السابق.

هذا يعني أن البيانات لم يتم حفظها فعليًا. وبعبارة أخرى، فإن "الحصالة" الحالية لا تحفظ إلا البيانات المؤقتة.

### لماذا لا يمكن حفظها بشكل دائم؟

السبب هو أننا استخدمنا متغيرًا مُعلَنًا باستخدام `@State`:

```swift
@State private var amount = 0
```

المتغير المُعلَن باستخدام `@State` يعتمد في دورة حياته بالكامل على الواجهة.

عند إنشاء الواجهة، يتم تهيئة `amount` بالقيمة 0، وعند تدمير الواجهة يتم التخلص من `amount`.

البيانات الموجودة في `amount` هنا تبقى في الذاكرة فقط، ولا يتم حفظها على الجهاز.

إذا كنا لا نريد أن يكون `amount` مرتبطًا بدورة حياة الواجهة، فنحن بحاجة إلى تنفيذ حفظ دائم للبيانات. أو بعبارة أخرى، نحتاج إلى حفظ البيانات على الجهاز.

### ما المقصود بـ "استمرارية البيانات"؟

يمكن فهم استمرارية البيانات على أنها حفظ البيانات من "الذاكرة المؤقتة" إلى "تخزين الجهاز".

عند إغلاق الواجهة أو إغلاق التطبيق، لا تضيع البيانات.

في تطوير Swift، يمكن استخدام `UserDefaults` لحفظ البيانات البسيطة بشكل دائم، أما البيانات الأكثر تعقيدًا فتحتاج إلى `SwiftData` أو `CoreData`.

في هذا الدرس، سنتعلم أولًا أبسط طريقة، وهي `UserDefaults`.

## UserDefaults

يُستخدم `UserDefaults` لحفظ بيانات خفيفة بصيغة أزواج مفتاح-قيمة، وغالبًا ما يُستخدم لتخزين الأنواع الأساسية مثل `String` و`Int` و`Double` و`Bool`.

### حفظ البيانات

يستخدم `UserDefaults` الدالة `set` لحفظ البيانات:

```swift
UserDefaults.standard.set("FangJunyu", forKey: "name")
UserDefaults.standard.set(28, forKey: "age")
UserDefaults.standard.set(true, forKey: "isMember")
```

المعامل الأول في `UserDefaults` يمثل البيانات المراد حفظها، وهنا مررنا بيانات من النوع `String` و`Int` وغير ذلك.

أما المعامل الثاني `forKey` فهو يشبه اسم المتغير، ويُستخدم لتحديد البيانات المحفوظة داخل `UserDefaults`.

### قراءة البيانات

يستخدم `UserDefaults` دوالًا مقابلة لقراءة البيانات:

```swift
let name = UserDefaults.standard.string(forKey: "name")	// FangJunyu
let age = UserDefaults.standard.integer(forKey: "age")	// 28
let isMember = UserDefaults.standard.bool(forKey: "isMember")	// true
```

عند القراءة يجب استخدام الدالة المناسبة لنوع البيانات، وإلا فقد يحدث خطأ أو تصبح البيانات غير صحيحة.

### ملاحظات مهمة حول UserDefaults

#### 1. إرجاع نوع اختياري

عند قراءة البيانات باستخدام `UserDefaults`، فإن بعض الدوال تُرجع نوعًا اختياريًا (`Optional`).

على سبيل المثال:

```swift
let name = UserDefaults.standard.string(forKey: "name")
```

سيكون نوع `name` كما يلي:

```swift
String?
```

وهذا يعني أنه قد يحتوي على قيمة، وقد لا يحتوي على أي قيمة.

**لماذا يعيد نوعًا اختياريًا؟**

لأن مفتاحًا معينًا داخل `UserDefaults` قد لا تكون له بيانات محفوظة أصلًا.

مثلًا:

```swift
let name = UserDefaults.standard.string(forKey: "City") // nil
```

إذا لم يتم حفظ المفتاح `"City"` من قبل، فستكون نتيجة القراءة:

```swift
nil
```

ومن المهم التأكيد أن `nil` تعني عدم وجود بيانات، وليست سلسلة نصية فارغة.

في درس "نظام الأنواع"، واجهنا حالة مشابهة، فعندما يفشل تحويل نوع البيانات يتم إرجاع نوع اختياري.

في هذه الحالة نحتاج إلى استخدام `??` لتقديم قيمة افتراضية وفك التغليف:

```swift
let name = UserDefaults.standard.string(forKey: "City") ?? "Rizhao"
```

إذا تمكّن `UserDefaults` من قراءة البيانات، فسيتم استخدام القيمة المقروءة، وإذا لم يتمكن من ذلك (`nil`) فستُستخدم القيمة الافتراضية.

#### 2. إرجاع نوع غير اختياري

عند قراءة الأنواع `Int` و`Double` و`Bool` من `UserDefaults`، يتم إرجاع نوع غير اختياري.

على سبيل المثال:

```swift
let num = UserDefaults.standard.integer(forKey: "num")	// 0
let height = UserDefaults.standard.double(forKey: "height")	// 0.0
let isBuyCar = UserDefaults.standard.bool(forKey: "isBuyCar")	// false
```

حتى لو لم يتم حفظ أي بيانات لهذه المفاتيح من قبل، فلن تكون القيمة المعادة `nil`.

بل سيتم إرجاع قيمة افتراضية:

```swift
Int → 0
Double → 0.0
Bool → false
```

#### 3. قيود كمية البيانات

`UserDefaults` مناسب فقط لحفظ كمية صغيرة من البيانات، مثل إعدادات التطبيق أو الحالات البسيطة.

في التطبيقات العملية، لا يناسب حفظ كميات كبيرة من البيانات أو البيانات التي تتطلب قراءة وكتابة متكررة جدًا.

## حفظ مبلغ "الحصالة"

الآن يمكننا استخدام `UserDefaults` لتنفيذ منطق حفظ إجمالي مبلغ "الحصالة".

عندما يضغط المستخدم على الزر، تتم إضافة المبلغ المُدخل إلى الإجمالي، ثم يتم حفظ الإجمالي داخل `UserDefaults`.

```swift
Button("Save") {
    amount += number
    number = 0
	UserDefaults.standard.set(amount, forKey: "amount")	// UserDefaults
}
.buttonStyle(.borderedProminent)
```

بهذا نكون قد أنجزنا منطق كتابة البيانات.

## شرح منطق استمرارية البيانات

مع أننا أكملنا منطق كتابة البيانات بشكل دائم، إلا أننا سنلاحظ عند التشغيل الفعلي أنه في كل مرة يتم فيها تحديث الواجهة أو إعادة فتح التطبيق، يبقى المتغير `amount` مساويًا لـ 0.

والسبب هو:

```swift
@State private var amount = 0
```

في SwiftUI، تتم إعادة تهيئة متغيرات `@State` عند إنشاء الواجهة من جديد.

حتى لو كانت قيمة `amount` محفوظة داخل `UserDefaults`، فإن `amount` سيُعاد ضبطه إلى 0 عند إعادة تحميل الواجهة.

وهذا يوضح أن البيانات حُفظت فقط داخل `UserDefaults`، لكنها لم تُسترجع داخل الواجهة.

لذلك، عند تحميل الواجهة، نحتاج أيضًا إلى قراءة البيانات يدويًا من `UserDefaults` ثم إعادة إسنادها إلى `amount`، وبهذه الطريقة فقط يكتمل منطق الحفظ الدائم.

### تشبيه منطقي

يمكن تشبيه هذه العملية بـ "السبورة في الفصل الدراسي":

أثناء الدرس، يكتب المعلم المحتوى على السبورة، وهذا يشبه الحالة الحالية داخل الواجهة (`@State`).

عندما ينتهي الدرس، تُمسح السبورة استعدادًا للحصة التالية. وهذا يشبه تدمير الواجهة، حيث يتم أيضًا مسح البيانات الموجودة في `@State`.

ولتجنّب ضياع المحتوى التعليمي، يقوم المعلم بحفظه مسبقًا في ملف الدرس. وهذا الملف يشبه البيانات المحفوظة داخل `UserDefaults`.

وعندما تبدأ الحصة التالية من جديد، تكون السبورة فارغة في البداية (`@State` بعد التهيئة)، ويحتاج المعلم إلى إعادة كتابة المحتوى السابق على السبورة استنادًا إلى ملف الدرس (أي قراءة `UserDefaults`).

من المهم فهم أن السبورة لا تستعيد المحتوى السابق تلقائيًا، بل يجب إعادة الكتابة من خلال "قراءة ملف الدرس" (أي قراءة `UserDefaults`).

## استخدام onAppear لقراءة البيانات

عند ظهور الواجهة، نحتاج إلى قراءة القيمة المخزنة داخل `UserDefaults` يدويًا وإسنادها إلى المتغير `amount` المُعلَن باستخدام `@State`، وبذلك نحقق الحفظ الدائم الحقيقي للبيانات.

في SwiftUI، يمكن استخدام `onAppear` لتنفيذ منطق التهيئة عند ظهور الواجهة:

```swift
.onAppear {}
```

ضع كود قراءة `UserDefaults` داخل `onAppear`:

```swift
.onAppear {
    amount = UserDefaults.standard.integer(forKey: "amount")
}
```

عند إنشاء الواجهة، تتم تهيئة `amount` بالقيمة 0. وعندما تظهر الواجهة، يتم تشغيل `onAppear` لقراءة القيمة من `UserDefaults` وإسنادها إلى `amount`. وبهذه الطريقة تستطيع الواجهة عرض إجمالي المبلغ الذي تم حفظه آخر مرة.

الآن أصبح منطق الحفظ الدائم بالكامل في "الحصالة" مكتملًا.

## الكود الكامل

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
                UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
        .onAppear {
            amount = UserDefaults.standard.integer(forKey: "amount")
        }
    }
}
```

## غطاء الخصائص @AppStorage

توفّر SwiftUI غطاء الخصائص `@AppStorage` لتبسيط منطق الحفظ باستخدام `UserDefaults`.

```swift
@AppStorage("amount") private var amount = 0
```

`@AppStorage` هو تغليف لـ `UserDefaults`، و`"amount"` هنا هو المفتاح المقابل داخل `UserDefaults`.

عند إنشاء الواجهة، تتم قراءة البيانات من `UserDefaults`، وعندما تتغير قيمة المتغير، يتم حفظها تلقائيًا داخل `UserDefaults`.

وهو يدعم أيضًا آلية تحديث الحالة في SwiftUI، ومثل `@State` تمامًا، سيتم تحديث واجهة SwiftUI عند تغيّر القيمة.

بمعنى آخر، يمكن لـ `@AppStorage` تنفيذ منطق قراءة البيانات وحفظها معًا، دون الحاجة إلى استدعاء `UserDefaults` يدويًا أو استخدام `onAppear` لقراءة البيانات.

لذلك يمكننا استخدام `@AppStorage` بدلًا من `UserDefaults` الموجود في الكود:

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

## ملخص

في هذا الدرس تعلمنا بشكل أساسي النقاط التالية:

أولًا، يُستخدم `UserDefaults` لحفظ البيانات الخفيفة، وقد استخدمناه لحفظ إجمالي المبلغ على الجهاز. ويوفّر `UserDefaults` دوالًا للقراءة والكتابة، كما يتطلب منا التعامل مع الأنواع الاختيارية في بعض الحالات.

ثانيًا، يُستخدم متغير `@State` فقط لإدارة الحالة المؤقتة داخل الواجهة، كما أن دورة حياته مرتبطة بالواجهة نفسها. وعند إعادة إنشاء الواجهة، تتم إعادة تهيئة `@State`.

وعند استرجاع بيانات `UserDefaults`، تعلمنا استخدام `onAppear` لقراءة البيانات عند ظهور الواجهة واستكمال عملية الاستعادة.

وأخيرًا، استخدمنا `@AppStorage` كبديل لـ `UserDefaults`، بحيث تتم قراءة البيانات تلقائيًا عند تحميل الواجهة، وكتابتها تلقائيًا عند تغيّر القيمة، مما يلغي الحاجة إلى الاستدعاء اليدوي لـ `UserDefaults` ومنطق `onAppear`.

من خلال هذه المعرفة، يمكننا تنفيذ وظيفة أساسية لتخزين البيانات داخل التطبيق، بحيث يمكن حفظ البيانات بشكل دائم.

## تمارين بعد الدرس

الآن أصبحت "الحصالة" تدعم حفظ المبلغ، لكن الوظائف ما زالت غير مكتملة.

1. عندما يحتاج المستخدم إلى سحب مبلغ، يجب تصميم منطق مناسب للسحب.
2. عندما يريد المستخدم تصفير المبلغ، نحتاج أيضًا إلى تنفيذ منطق لإعادة الضبط.
3. إذا أردنا إضافة اسم للحصالة، فعلينا أيضًا إدخال حفظ بيانات من النوع `String`.

يمكنك تحسين هذه الوظائف بناءً على الأساس الحالي، لتصبح "الحصالة" أكثر اكتمالًا.

## معرفة إضافية - مشكلة القيمة الافتراضية في مربع الإدخال

الآن يمكننا كذلك تحسين تجربة التفاعل مع مربع الإدخال، بحيث تصبح عملية الإدخال أكثر سهولة.

![textfield](../../Resource/015_view.png)

في التنفيذ الحالي، القيمة الافتراضية داخل مربع الإدخال هي 0. وهذا يعني أن المستخدم يحتاج في كل مرة إلى حذف 0 أولًا ثم إدخال الرقم الجديد، وهي تجربة استخدام غير مناسبة.

يجب أن نجعل القيمة الابتدائية لمربع الإدخال فارغة، بدلًا من أن تكون 0.

والسبب هو أن `TextField` مرتبط بـ `number` بشكل ثنائي الاتجاه:

```swift
@State private var number = 0
TextField("Amount", value: $number, format: .number)
```

عندما تتم تهيئة `number` بالقيمة 0، سيعرض مربع الإدخال أيضًا 0.

لذلك نحتاج إلى جعل `number` قيمة فارغة، أي `nil`.

```swift
@State private var number = nil
```

في هذه اللحظة سيظهر الخطأ التالي:

```swift
Generic parameter 'Value' could not be inferred
```

وهذه هي الحالة التي تحدثنا عنها في "نظام الأنواع": عندما يتعذّر على المتغير استنتاج نوعه تلقائيًا، نحتاج إلى التصريح عن نوعه بشكل صريح.

لأننا عيّنا `number` إلى `nil`، و`nil` بحد ذاته لا يحمل أي معلومات عن النوع، فلن يتمكن المترجم من معرفة ما إذا كان هذا المتغير من النوع `String` أو `Int` أو غير ذلك، ولذلك سيظهر الخطأ.

لذلك نحتاج إلى التصريح عن نوع `number` بشكل صريح:

```swift
@State private var number:Int? = nil
```

هنا تعني `Int?` نوعًا اختياريًا (`Optional`)، أي أن قيمة `number` يمكن أن تكون `Int` أو يمكن أن تكون `nil`.

وبهذه الطريقة، عندما نهيئه بـ `nil`، سيظهر مربع الإدخال فارغًا بدلًا من 0.

### فك تغليف النوع الاختياري

بعد أن نجعل `number` نوعًا اختياريًا، سيظهر خطأ في كود الحساب الأصلي:

```swift
amount += number
```

والرسالة ستكون:

```swift
Value of optional type 'Int?' must be unwrapped to a value of type 'Int'
```

وذلك لأن `number` الآن قد تكون `nil`. وإذا شاركت مباشرة في العملية الحسابية، فسيصبح الأمر كالتالي:

```swift
amount += nil
```

ومن الواضح أن `nil` لا يمكن جمعها مع `amount`، وهذا غير مسموح.

لذلك، عند استخدام نوع اختياري في العمليات الحسابية، يجب أولًا فك تغليفه.

يمكننا هنا استخدام `??` لتقديم قيمة افتراضية:

```swift
amount += number ?? 0
```

عندما تكون `number` مساوية لـ `nil`، سيتم استخدام القيمة الافتراضية 0، وعندما تحتوي على قيمة فسيتم استخدام القيمة الفعلية التي أدخلها المستخدم.

وهذا يضمن صحة الحساب، كما يحقق في الوقت نفسه جعل مربع الإدخال فارغًا افتراضيًا، بحيث لا يحتاج المستخدم إلى حذف 0 يدويًا في كل مرة.

```swift
                amount += number ?? 0

            }
            .buttonStyle(.borderedProminent)

        }
    }
}
```
