# إدارة ملفات Xcode

في هذا الدرس، سنتعلّم عمليات الملفات الشائعة في Xcode.

في الدروس السابقة، كتبنا معظم الشيفرة داخل `ContentView`. هذا يسهّل ملاحظة الشيفرة وتأثير العرض.

على سبيل المثال:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

لكن في تطبيق حقيقي، لا نضع عادةً كل الشيفرة داخل ملف واحد.

عندما تزداد الصفحات والمكوّنات والوظائف، نحتاج إلى فصل العروض المختلفة في ملفات SwiftUI مختلفة.

على سبيل المثال:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

هذا يجعل بنية الشيفرة أوضح، ويسهّل الصيانة والتعديل لاحقًا.

## لماذا نحتاج إلى إدارة الملفات

في SwiftUI، يمكن عادةً كتابة الصفحة أو المكوّن كملف عرض مستقل.

على سبيل المثال:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

هنا، `CardView` هو عرض SwiftUI.

إذا كتبنا كل العروض داخل `ContentView`، فستصبح الشيفرة أطول تدريجيًا وأصعب في القراءة.

لذلك، نحتاج إلى تعلّم إنشاء الملفات وإعادة تسميتها وحذفها، واستخدام المجلدات لإدارة ملفات الشيفرة المختلفة.

## إنشاء ملف SwiftUI

في Xcode، توجد عادةً طريقتان شائعتان لإنشاء ملف SwiftUI:

الأولى هي إنشاء ملف فارغ، ثم كتابة الشيفرة بأنفسنا.

والثانية هي إنشاء ملف قالب SwiftUI، بحيث ينشئ Xcode الشيفرة الأساسية تلقائيًا.

بالنسبة إلى المبتدئين، يُنصح أكثر باستخدام ملف قالب SwiftUI، لأنه ينشئ البنية الأساسية تلقائيًا.

### إنشاء ملف فارغ

في منطقة Navigator على يسار Xcode، انقر بزر الفأرة الأيمن على مساحة فارغة أو على مجلد، ثم اختر `New Empty File`.

موضع العملية كما يلي:

![xcode](../../../Resource/027_view1.png)

بعد إنشاء ملف فارغ، ينشئ Xcode عادةً ملف Swift بدون اسم، مثل `Untitled.swift`.

يمكننا إعادة تسميته إلى `TestView.swift`.

في مشروع SwiftUI، إذا كان هذا الملف يُستخدم لعرض View، فمن المستحسن عادةً إضافة `View` في نهاية الاسم.

على سبيل المثال:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

هذه ليست قاعدة إلزامية، لكنها تجعلنا نعرف من النظرة الأولى أن هذا ملف عرض.

بعد إنشاء ملف فارغ، قد يحتوي الملف على تعليقات افتراضية فقط، وقد يكون فارغًا، وذلك يعتمد على إصدار Xcode وطريقة الإنشاء.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

في هذه المرحلة، نحتاج إلى كتابة شيفرة SwiftUI يدويًا:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

بهذا، نكون قد أنشأنا ملف عرض SwiftUI بسيطًا.

### إعادة تسمية الملف

إذا أردت تعديل اسم الملف بعد إنشائه، يمكنك اختيار الملف في منطقة Navigator على يسار Xcode، ثم الضغط على مفتاح الإدخال `⏎`.

![xcode](../../../Resource/027_view2.png)

بهذا يمكن تعديل اسم الملف.

يجب الانتباه إلى أن تعديل اسم الملف لا يعدّل تلقائيًا اسم العرض داخل الشيفرة.

على سبيل المثال، بعد تغيير اسم الملف إلى `ProfileView.swift`، قد يظل داخل الشيفرة `struct TestView: View`.

هذا لا يؤدي بالضرورة إلى خطأ، لكنه قد يسبب ارتباكًا.

لذلك، يُنصح بالحفاظ على اتساق اسم الملف واسم العرض. على سبيل المثال، إذا كان اسم الملف `ProfileView.swift`، فيُستخدم أيضًا `ProfileView` كاسم للعرض.

## إنشاء ملف قالب SwiftUI

إلى جانب إنشاء ملف فارغ، يمكننا أيضًا إنشاء ملف قالب SwiftUI مباشرةً.

في منطقة Navigator على يسار Xcode، انقر بزر الفأرة الأيمن على مساحة فارغة أو على مجلد، ثم اختر `New File from Template...`.

موضع العملية كما يلي:

![xcode](../../../Resource/027_view3.png)

في النافذة المنبثقة، يمكن اختيار منصات مختلفة من الأعلى، مثل `iOS` و `macOS` وغيرهما.

هنا نُبقي الاختيار على المنصة المطابقة للمشروع الحالي. على سبيل المثال، إذا كنا ننشئ تطبيق iOS، فنختار `iOS`.

![xcode](../../../Resource/027_view4.png)

ثم نختار `SwiftUI View` في الأسفل.

![xcode](../../../Resource/027_view12.png)

بعد ذلك، انقر على `Next`، ثم أدخل اسم الملف واحفظه.

على سبيل المثال، ننشئ ملفًا باسم `Test2View.swift`.

سيُنشئ Xcode تلقائيًا شيفرة مشابهة لما يلي:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

بالمقارنة مع الملف الفارغ، يحتوي ملف القالب تلقائيًا على `import SwiftUI` و `struct` و `body` و `#Preview`.

لذلك بالنسبة إلى المبتدئين، يكون استخدام ملف قالب SwiftUI أبسط، كما يقلّل احتمال نسيان كتابة الشيفرة.

**تلميح**: إلى جانب إنشاء ملف بالنقر بزر الفأرة الأيمن، يمكنك أيضًا النقر على زر الإضافة في أسفل منطقة Navigator على يسار Xcode لإنشاء ملف أو مجلد.

![xcode](../../../Resource/027_view5.png)

## إنشاء مجلد

عندما تصبح الملفات كثيرة، يمكننا استخدام المجلدات لإدارتها.

في منطقة Navigator على يسار Xcode، انقر بزر الفأرة الأيمن على مساحة فارغة أو على مجلد، ثم اختر `New Folder`.

بعد إنشاء المجلد، يمكنك سحب الملفات المرتبطة إليه.

![xcode](../../../Resource/027_view7.png)

تُستخدم المجلدات أساسًا لتنظيم بنية المشروع، ولا تغيّر طريقة استخدام العرض نفسه.

طالما أن اسم العرض صحيح، يمكن استخدامه من ملفات أخرى بالطريقة نفسها، مثل `TestView()` و `Test2View()`.

## حذف ملف أو مجلد

إذا أردت حذف ملف أو مجلد، يمكنك اختيار الملف المقابل في منطقة Navigator على يسار Xcode، ثم النقر بزر الفأرة الأيمن واختيار `Delete`.

يمكنك أيضًا اختيار الملف ثم الضغط على مفتاح الحذف `⌫`.

سيعرض Xcode نافذة تأكيد الحذف.

![xcode](../../../Resource/027_view10.png)

اختيار `Move to Trash` يعني نقل الملف إلى سلة المهملات.

إذا رأيت `Remove Reference`، فهذا يعني إزالة المرجع من مشروع Xcode فقط، دون حذف الملف الأصلي من القرص.

بالنسبة إلى المبتدئين، إذا كنت تحذف فقط ملفًا عاديًا أنشأته للتو، فعادةً يكفي اختيار `Move to Trash`.

## عرض عدة عروض SwiftUI

بعد تعلّم إنشاء الملفات، يمكننا فصل العروض المختلفة في ملفات مختلفة.

على سبيل المثال، ننشئ ملفي SwiftUI:

```swift
AuthorView.swift
CardView.swift
```

يُستخدم `AuthorView` لعرض اسم الكاتب:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

ويُستخدم `CardView` لعرض خلفية البطاقة:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

بعد ذلك، يمكننا استخدام هذين العرضين داخل `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

بعد التشغيل، يمكننا رؤية `CardView` و `AuthorView` معروضين معًا بشكل مركّب:

![xcode](../../../Resource/027_view11.png)

في هذا التأثير، يعمل `CardView` كخلفية، ويظهر `AuthorView` في الأعلى، وذلك لأنهما وُضعا داخل `ZStack`.

في `ContentView`، نستخدم هذين العرضين من خلال `CardView()` و `AuthorView()`، وتشير `()` بعد الاسم إلى إنشاء هذا العرض واستخدامه.

هذا هو تداخل العروض الشائع في SwiftUI: يمكن للعرض الواحد أن يستخدم عروضًا أخرى داخله.

## ملف نقطة الدخول

أخيرًا، لنراجع من أين يبدأ التطبيق في عرض الواجهة.

في مشروع SwiftUI، يوجد عادةً ملف نقطة دخول:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

يشير `@main` هنا إلى أن هذا هو موضع دخول التطبيق.

داخل `WindowGroup`، نرى:

```swift
ContentView()
```

هذا يعني أن التطبيق يعرض `ContentView` أولًا بعد بدء التشغيل.

ثم يعرض `ContentView` داخله أيضًا:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

لذلك يمكن فهم طبقة العروض كما يلي:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

إذا غيّرنا العرض المعروض في ملف نقطة الدخول إلى `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

عندها، بعد بدء تشغيل التطبيق، سيعرض `CardView` فقط.

أي أن `WindowGroup` في شيفرة نقطة الدخول يحدد أي عرض سيظهر أولًا بعد بدء تشغيل التطبيق.

أما داخل هذا العرض، فيمكنه بدوره الاستمرار في عرض عروض أخرى.

## الخلاصة

في هذا الدرس، تعلّمنا عمليات إدارة الملفات الشائعة في Xcode.

تعرّفنا على كيفية إنشاء ملفات SwiftUI، وكيفية إعادة تسمية الملفات، وكيفية إنشاء المجلدات، وكذلك كيفية حذف الملفات.

وفي الوقت نفسه، تعلّمنا مفهومًا مهمًا: يمكن فصل عروض SwiftUI في ملفات مختلفة، ثم استخدامها داخل عروض أخرى.

على سبيل المثال:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

هنا، `CardView()` و `AuthorView()` هما استخدام لعروض أخرى داخل `ContentView`.

أخيرًا، راجعنا أيضًا منطق العرض في ملف نقطة الدخول.

عند بدء تشغيل التطبيق، يدخل أولًا إلى شيفرة نقطة الدخول المعلّمة بـ `@main`، ثم يعرض العرض الموجود داخل `WindowGroup`.

يساعدنا هذا على فهم أن ملفات SwiftUI المختلفة ليست معزولة عن بعضها، بل يمكنها أن تتراكب وتتكوّن معًا في النهاية لتشكيل واجهة تطبيق كاملة.
