# ಲಾಗಿನ್ ಪರಸ್ಪರ ಕ್ರಿಯೆಗಳು

ಹಿಂದಿನ ಪಾಠದಲ್ಲಿ ನಾವು ಒಂದು ಲಾಗಿನ್ ವ್ಯೂ ರಚಿಸಿದ್ದೇವೆ ಮತ್ತು ಬಟನ್‌ಗಳು ಹಾಗೂ ನಿರ್ಧಾರ ಲಾಜಿಕ್ ಮೂಲಕ ಇನ್‌ಪುಟ್ ಮಾಡಿದ ವಿಷಯದ ಸರಳ ಪರಿಶೀಲನೆಯನ್ನು ಜಾರಿಗೆ ತಂದಿದ್ದೇವೆ.

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ಹಿಂದಿನ ಪಾಠದ ಕೋಡ್ ಅನ್ನು ಆಧಾರವಾಗಿ ತೆಗೆದುಕೊಂಡು `SecureField` ಪಾಸ್‌ವರ್ಡ್ ಇನ್‌ಪುಟ್ ಬಾಕ್ಸ್, `Alert` ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್, ಮತ್ತು ಚರಗಳ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವ್ಯೂಗಳನ್ನು ಹೇಗೆ ತೋರಿಸುವುದು ಎಂಬುದನ್ನು ಕಲಿಯುತ್ತೇವೆ.

ಈ ಪಾಠದ ಮೂಲಕ, ಲಾಗಿನ್ ವ್ಯೂನ ಮೂಲಭೂತ ಪರಸ್ಪರ ಕ್ರಿಯಾ ಪ್ರವಾಹವನ್ನು ಇನ್ನಷ್ಟು ಚೆನ್ನಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು. ಇದು ಅನೇಕ ಆಪ್‌ಗಳಲ್ಲಿ ಬಳಸಲಾಗುವ ಮೂಲಭೂತ ಲಾಜಿಕ್ ಕೂಡ ಆಗಿದೆ.

## ಪಾಸ್‌ವರ್ಡ್ ಇನ್‌ಪುಟ್ ಬಾಕ್ಸ್

ಹಿಂದಿನ ಪಾಠದಲ್ಲಿ ನಾವು ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಲು `TextField` ಅನ್ನು ಬಳಸಿದ್ದೇವೆ.

ಆದರೆ ನಿಜವಾದ ಲಾಗಿನ್ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ, ಪಾಸ್‌ವರ್ಡ್ ಸಾಮಾನ್ಯವಾಗಿ ನೇರವಾಗಿ ತೋರಿಸಲಾಗುವುದಿಲ್ಲ. ಅದರ ಬದಲು ಅದು ಬಿಂದುಗಳು ಅಥವಾ ಬೇರೆ ಮರೆಮಾಚಿದ ಪರಿಣಾಮದ ರೂಪದಲ್ಲಿ ತೋರುತ್ತದೆ, ಇದರಿಂದ ಬಳಕೆದಾರರ ಗೌಪ್ಯತೆ ರಕ್ಷಿತವಾಗುತ್ತದೆ.

SwiftUI ಯಲ್ಲಿ, ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಲು ವಿಶೇಷವಾಗಿ ಬಳಸುವ ನಿಯಂತ್ರಣ `SecureField` ಆಗಿದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
SecureField("Password", text: $password)
```

ಇದನ್ನು ಬರೆಯುವ ವಿಧಾನವು `TextField` ಗೆ ಬಹಳ ಹೋಲುತ್ತದೆ. ಇದಕ್ಕೂ ಸೂಚನಾ ಪಠ್ಯ ಮತ್ತು ಬೌಂಡ್ ಮಾಡಲಾದ ಚರ ಅಗತ್ಯವಿರುತ್ತದೆ.

ಇದನ್ನು ಸರಳವಾಗಿ ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು: `SecureField` ಕೂಡ ಒಂದು ಇನ್‌ಪುಟ್ ಬಾಕ್ಸ್, ಆದರೆ ಇದು ನಮೂದಿಸಿದ ವಿಷಯವನ್ನು ಮರೆಮಾಡುತ್ತದೆ.

ಹಿಂದೆ ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಲು ಬಳಸಿದ್ದ `TextField` ಅನ್ನು ನೇರವಾಗಿ `SecureField` ಆಗಿ ಬದಲಾಯಿಸಬಹುದು:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

ಬಳಕೆದಾರನು ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಿದಾಗ, ಪಾಸ್‌ವರ್ಡ್‌ನ ವಿಷಯ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ನೇರವಾಗಿ ತೋರಿಸುವುದಿಲ್ಲ.

ತೋರಿಸುವ ಪರಿಣಾಮ:

![password](../../Resource/020_password.png)

ಈಗ, ನಾವು ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಿ, ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ ಇನ್‌ಪುಟ್ ಸರಿಯೇ ಎಂದು ಪರಿಶೀಲಿಸಬಹುದು.

### ಪ್ರತಿಕ್ರಿಯಾ ಸಂದೇಶಗಳನ್ನು ತೋರಿಸುವುದು

ಈ ಮೊದಲು ನಾವು ಫಲಿತಾಂಶಗಳನ್ನು ಔಟ್‌ಪುಟ್ ಮಾಡಲು ನಿರಂತರವಾಗಿ `print` ಅನ್ನು ಬಳಸುತ್ತಿದ್ದೇವೆ.

`print` ನ ವಿಷಯವು ಕೇವಲ ಕನ್ಸೋಲ್‌ನಲ್ಲಿ ಮಾತ್ರ ಕಾಣಿಸುತ್ತದೆ. ಡೆವಲಪರ್‌ಗಳು Xcode ನಲ್ಲಿ ಪ್ರೋಗ್ರಾಂ ರನ್ ಮಾಡುವಾಗ ಅದನ್ನು ನೋಡಬಹುದು, ಆದರೆ ಸಾಮಾನ್ಯ ಬಳಕೆದಾರರು ಫೋನ್‌ನಲ್ಲಿ ಈ ಸಂದೇಶಗಳನ್ನು ನೋಡಲು ಸಾಧ್ಯವಿಲ್ಲ.

ಅಂದರೆ, ಪ್ರೋಗ್ರಾಂ ಈಗಾಗಲೇ ಪರಿಶೀಲನೆ ಪೂರ್ಣಗೊಳಿಸಿದ್ದರೂ, ಬಳಕೆದಾರರಿಗೆ ನಮೂದಿಸಿದ ವಿಷಯ ಸರಿಯೇ ಎಂಬುದು ತಿಳಿಯುವುದಿಲ್ಲ.

ಈ ಸಮಯದಲ್ಲಿ, ನಾವು ಸಂದೇಶಗಳನ್ನು ನೇರವಾಗಿ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ತೋರಿಸುವ ವಿಧಾನವೊಂದನ್ನು ಅಗತ್ಯಪಡುತ್ತೇವೆ.

## Alert ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್

SwiftUI ಯಲ್ಲಿ, `Alert` ಅನ್ನು ಬಳಸಿಕೊಂಡು ಒಂದು ಪಾಪ್-ಅಪ್ ಸಂದೇಶ ಬಾಕ್ಸ್ ತೋರಿಸಬಹುದು.

ಮೂಲ ಬಳಕೆ:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

ಈ ಕೋಡ್ ಹಿಂದಿನ ಕಂಟ್ರೋಲ್‌ಗಳಿಗೆ ಹೋಲಿಸಿದರೆ ಸ್ವಲ್ಪ ಹೆಚ್ಚು ಸಂಕೀರ್ಣವಾಗಿ ಕಾಣಬಹುದು, ಆದರೆ ನಾವು ಅದನ್ನು ಭಾಗಗಳಾಗಿ ವಿಭಜಿಸಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು.

`Alert` ಮುಖ್ಯವಾಗಿ ಕೆಳಗಿನ ಭಾಗಗಳನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ:

- `"Title"`: ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್‌ನ ಶೀರ್ಷಿಕೆ
- `isPresented`: ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ತೋರಿಸಬೇಕೇ ಬೇಡವೇ ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸುವ ಬೌಂಡ್ ಮಾಡಿದ Boolean ಮೌಲ್ಯ
- `Button(...)`: ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್‌ನ ಬಟನ್
- `message:`: ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್‌ನಲ್ಲಿ ತೋರಿಸಲಾಗುವ ಸಂದೇಶದ ವಿಷಯ

ಇದನ್ನು ಸರಳವಾಗಿ ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು: ಬೌಂಡ್ ಮಾಡಲಾದ ಚರವು `true` ಆದಾಗ, ಸಿಸ್ಟಮ್ ಈ ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ಅನ್ನು ಪಾಪ್ ಅಪ್ ಮಾಡುತ್ತದೆ.

ತೋರಿಸುವ ಪರಿಣಾಮ:

ಸಂಬಂಧಿತ ರಚನೆ:

![alert](../../Resource/020_alert.png)

### Alert ಒಂದು modifier ಆಗಿದೆ

ನಾವು ಹಿಂದೆ ಕಲಿತ ಅನೇಕ modifiers ಗಳಂತೆ, `Alert` ಕೂಡ ಒಂದು ನಿರ್ದಿಷ್ಟ ವ್ಯೂನ ನಂತರ ಸೇರಿಸಲಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ಇದನ್ನು ಬಟನ್‌ನ ನಂತರ ಜೋಡಿಸಬಹುದು:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

ಇದನ್ನು ಹೊರಗಿನ `VStack` ನ ನಂತರವೂ ಜೋಡಿಸಬಹುದು:

```swift
VStack {
    // ...
}
.alert() {...}
```

ಅಂದರೆ, `Alert` ಎನ್ನುವುದು ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ಪ್ರತ್ಯೇಕವಾಗಿ ಇಡಲಾದ ಸ್ವತಂತ್ರ ವ್ಯೂ ಅಲ್ಲ; ಬದಲಿಗೆ ಅದು ನಿರ್ದಿಷ್ಟ ವ್ಯೂಗೆ ಅಂಟಿಕೊಂಡಿರುವ modifier ಆಗಿದೆ.

ಇದನ್ನು ಸರಳವಾಗಿ ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು: `Alert` ಸಾಮಾನ್ಯವಾಗಿ ಬಟನ್ ಅಥವಾ ಹೊರಗಿನ layout ನಂತರ ಬರೆಯಲ್ಪಡುತ್ತದೆ, ಮತ್ತು ನಿರ್ದಿಷ್ಟ ಷರತ್ತು ಪೂರ್ತಿಯಾದಾಗ ಒಂದು ಸಂದೇಶವನ್ನು ಪಾಪ್ ಅಪ್ ಮಾಡಲು ಬಳಸಲಾಗುತ್ತದೆ.

### Alert ತೋರಿಸುವುದು

ನಾವು `Alert` ತೋರಿಸಬೇಕೆಂದರೆ, ಅದನ್ನು ನಿಯಂತ್ರಿಸಲು ಒಂದು ಚರ ಅಗತ್ಯವಿರುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
@State private var showAlert = false
```

ಈ ಚರವು `Bool` ಪ್ರಕಾರದಾಗಿದೆ, ಅಂದರೆ ಇದಕ್ಕೆ ಕೇವಲ ಎರಡು ಮೌಲ್ಯಗಳಿರುತ್ತವೆ: `true` ಅಥವಾ `false`.

ನಂತರ ಇದನ್ನು `Alert` ನ `isPresented` ಗೆ bind ಮಾಡಲಾಗುತ್ತದೆ:

```swift
isPresented: $showAlert
```

`showAlert` `false` ಆಗಿರುವಾಗ, ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ತೋರಿಸುವುದಿಲ್ಲ; `showAlert` `true` ಆಗಿರುವಾಗ, ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ಕಾಣಿಸುತ್ತದೆ.

### ಬಟನ್ ಮೂಲಕ ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ತೋರಿಸುವುದು

ಈಗ ಮೊದಲು ಅತ್ಯಂತ ಸರಳ ಪರಿಸ್ಥಿತಿಯನ್ನು ಜಾರಿಗೆ ತರುವುದನ್ನು ನೋಡೋಣ:

ಬಳಕೆದಾರನು ಏನೂ ನಮೂದಿಸದೆ ನೇರವಾಗಿ `Sign in` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ, “ನೀನು ಇನ್ನೂ ಯಾವುದೇ ವಿಷಯವನ್ನು ನಮೂದಿಸಿಲ್ಲ” ಎಂದು ಹೇಳುವ ಒಂದು ಸಂದೇಶ ಬಾಕ್ಸ್ ಪಾಪ್ ಅಪ್ ಆಗಬೇಕು.

ಮೊದಲು, ಈ ಸಂದೇಶ ಬಾಕ್ಸ್ ಅನ್ನು ನಿಯಂತ್ರಿಸಲು ಒಂದು ಚರವನ್ನು ರಚಿಸಿ:

```swift
@State private var showEmpty = false
```

ನಂತರ, `Button` ನಂತರ ಒಂದು `Alert` ಸೇರಿಸಿ:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

ನಂತರ, ಬಟನ್ ಕ್ಲಿಕ್ ಕೋಡ್‌ನಲ್ಲಿ ಪರಿಶೀಲನೆ ಮಾಡಿ:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

`user` ಮತ್ತು `password` ಎರಡೂ ಖಾಲಿಯಾಗಿದ್ದರೆ, `showEmpty` ಅನ್ನು `true` ಆಗಿ ಬದಲಾಯಿಸುತ್ತೇವೆ.

ಮತ್ತೆ `Alert` ಈ ಚರಕ್ಕೆ bind ಆಗಿರುವುದರಿಂದ, ಅದು `true` ಆದಾಗ ಸಿಸ್ಟಮ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ಅನ್ನು ತೋರಿಸುತ್ತದೆ.

ತೋರಿಸುವ ಪರಿಣಾಮ:

![alert](../../Resource/020_alert1.png)

ಈ ರೀತಿಯಲ್ಲಿ, ಬಳಕೆದಾರನು ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ತಕ್ಷಣ ಸಂದೇಶವನ್ನು ನೋಡಬಹುದು, ಮತ್ತು ಡೆವಲಪರ್‌ಗೆ `print` ಔಟ್‌ಪುಟ್ ನೋಡಲು ಕನ್ಸೋಲ್‌ಗೆ ಹೋಗುವ ಅಗತ್ಯವಿರುವುದಿಲ್ಲ.

### ಬಟನ್ ಮತ್ತು ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್‌ಗಳನ್ನು ಇನ್ನಷ್ಟು ಪೂರ್ಣಗೊಳಿಸುವುದು

ಹಿಂದೆ ನಾವು “ಏನನ್ನೂ ತುಂಬಿಲ್ಲ” ಎಂಬ ಪರಿಸ್ಥಿತಿಯನ್ನು ಮಾತ್ರ ನಿರ್ವಹಿಸಿದ್ದೇವೆ.

ಇನ್ನೂ ಕೆಳಗಿನ ಪರಿಸ್ಥಿತಿಗಳನ್ನೂ ನಿರ್ವಹಿಸಬೇಕಾಗಿದೆ:

- ಬಳಕೆದಾರ ಹೆಸರು ತುಂಬಲಾಗಿಲ್ಲ
- ಪಾಸ್‌ವರ್ಡ್ ತುಂಬಲಾಗಿಲ್ಲ
- ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಎರಡನ್ನೂ ತುಂಬಲಾಗಿದೆ, ಆದರೆ ಇನ್‌ಪುಟ್ ತಪ್ಪಾಗಿದೆ

ಆದ್ದರಿಂದ, ಈ ಸಂದರ್ಭಗಳಿಗೆ ಸಂಬಂಧಿಸಿದ ಸಂದೇಶಗಳನ್ನೂ ಸಿದ್ಧಪಡಿಸಬೇಕಾಗುತ್ತದೆ.

ಮೊದಲು, ಮೂರು ಹೊಸ ಚರಗಳನ್ನು ಸೇರಿಸಿ:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

ನಂತರ, ಬಟನ್ ನಂತರ ಸಂಬಂಧಿತ `Alert` ಗಳನ್ನು ಸೇರಿಸಿ:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

ಮುಂದೆ, ಬಟನ್ ಒಳಗಿನ ನಿರ್ಧಾರ ಲಾಜಿಕ್ ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

ಬಟನ್ ಬಳಕೆದಾರನು ಯಾವುದೇ ಮಾಹಿತಿಯನ್ನು ತುಂಬಿಲ್ಲ, ಒಂದು ಮಾಹಿತಿಯನ್ನು ಬಿಟ್ಟುಬಿಟ್ಟಿದ್ದಾನೆ, ಅಥವಾ ಇನ್‌ಪುಟ್ ಮಾನ್ಯೀಕರಣ ವಿಫಲವಾಗಿದೆ ಎಂದು ತೀರ್ಮಾನಿಸಿದಾಗ, ಅದು ಸಂಬಂಧಿತ Boolean ಚರವನ್ನು `true` ಆಗಿ ಬದಲಾಯಿಸುತ್ತದೆ, ನಂತರ ಸಂಬಂಧಿತ `Alert` ಪಾಪ್ ಅಪ್ ಆಗುತ್ತದೆ.

ಇಲ್ಲಿ ನಾನು ಕೆಲವು ಪರಿಸ್ಥಿತಿಗಳನ್ನು ನೀನೇ ಪರೀಕ್ಷಿಸಬೇಕು ಎಂದು ಸಲಹೆ ನೀಡುತ್ತೇನೆ:

- ಏನನ್ನೂ ತುಂಬಬೇಡ
- ಕೇವಲ ಬಳಕೆದಾರ ಹೆಸರನ್ನು ಮಾತ್ರ ತುಂಬು
- ಕೇವಲ ಪಾಸ್‌ವರ್ಡ್ ಅನ್ನು ಮಾತ್ರ ತುಂಬು
- ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಎರಡನ್ನೂ ತುಂಬು, ಆದರೆ ವಿಷಯ ತಪ್ಪಾಗಿರಲಿ
- ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಎರಡನ್ನೂ ತುಂಬು, ಮತ್ತು ವಿಷಯ ಸರಿಯಾಗಿರಲಿ

ಇದರಿಂದ ನಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಸಮಸ್ಯೆಯಿದೆಯೇ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸಬಹುದು. ಇದು ಆಪ್ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಅತ್ಯಂತ ಪ್ರಮುಖವಾದ “testing” ಹಂತವೂ ಆಗಿದೆ.

ಯಾವುದೇ ನಿರ್ದಿಷ್ಟ ಪರಿಸ್ಥಿತಿಯಲ್ಲಿ ಪ್ರೋಗ್ರಾಂ ಸರಿಯಾಗಿ ವರ್ತಿಸದಿದ್ದರೆ, ಲಾಜಿಕ್‌ನಲ್ಲಿ ಸಮಸ್ಯೆಯಿರಬಹುದು ಮತ್ತು ಪ್ರೋಗ್ರಾಂನಲ್ಲಿ ಒಂದು `Bug` ಕಾಣಿಸಿಕೊಂಡಿದೆ ಎಂಬರ್ಥ.

ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ನಾವು ಸಾಮಾನ್ಯವಾಗಿ `Bug` ಎಂದು ಕರೆಯುವುದೇನೆಂದರೆ ದೋಷ, ಕೊರತೆ, ಅಥವಾ ನಿರೀಕ್ಷಿತಕ್ಕೆ ಹೊಂದದ ವರ್ತನೆ.

## ಲಾಗಿನ್ ವ್ಯೂ ಪೂರ್ಣಗೊಳಿಸುವುದು

ಹಿಂದೆ ಹೇಳಿದ `Alert` ಉದಾಹರಣೆಗಳೆಲ್ಲವೂ “ಇನ್‌ಪುಟ್ ತಪ್ಪಾಗಿದೆ” ಎಂಬ ಸಂದರ್ಭದಲ್ಲಿ ಬಳಕೆದಾರರಿಗೆ ಹೇಗೆ ಸಂದೇಶ ನೀಡುವುದು ಎಂಬುದರ ಬಗ್ಗೆ ಇವೆ.

ಬಳಕೆದಾರನು ಸರಿಯಾದ ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಅನ್ನು ನಮೂದಿಸಿದಾಗ, ನಾವು ಸಾಮಾನ್ಯವಾಗಿ “ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್” ಅನ್ನು ಪಾಪ್ ಅಪ್ ಮಾಡುವುದಿಲ್ಲ; ಬದಲಿಗೆ ನೇರವಾಗಿ ಲಾಗಿನ್ ನಂತರದ ಇಂಟರ್‌ಫೇಸ್‌ಗೆ ಹೋಗುತ್ತೇವೆ.

ಇಲ್ಲಿ ಒಂದು ಬಹಳ ಮುಖ್ಯವಾದ ಕಲ್ಪನೆ ಇದೆ: **ಚರದ ಮೌಲ್ಯದ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವ್ಯೂಗಳನ್ನು ತೋರಿಸುವುದು.**

ಉದಾಹರಣೆಗೆ:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

ಇಲ್ಲಿ ನಾವು ವ್ಯೂ ಪ್ರದರ್ಶನವನ್ನು ನಿರ್ವಹಿಸಲು `Bool` ಪ್ರಕಾರವನ್ನು ಬಳಸುತ್ತೇವೆ:

- `login` `true` ಆಗಿದ್ದರೆ, ಲಾಗಿನ್ ಈಗಾಗಲೇ ಯಶಸ್ವಿಯಾಗಿದೆ ಮತ್ತು ಲಾಗಿನ್ ನಂತರದ ವಿಷಯ ತೋರಿಸಲಾಗುತ್ತದೆ ಎಂದರ್ಥ
- `login` `false` ಆಗಿದ್ದರೆ, ಲಾಗಿನ್ ಯಶಸ್ವಿಯಾಗಿಲ್ಲ ಮತ್ತು ಲಾಗಿನ್ ಇಂಟರ್‌ಫೇಸ್ ಮುಂದುವರಿದು ತೋರುತ್ತದೆ ಎಂದರ್ಥ.

`if` ಸ್ಟೇಟ್ಮೆಂಟ್ Swift ನಲ್ಲಿ ಒಂದು conditional statement ಆಗಿದ್ದರೂ, SwiftUI ಯ `body` ಒಳಗೆ ಅದನ್ನು ವ್ಯೂ ಪ್ರದರ್ಶನ ಲಾಜಿಕ್ ನಿಯಂತ್ರಿಸಲು ಬಳಸಬಹುದು.

ಅಂದರೆ, SwiftUI ಷರತ್ತಿನ ಫಲಿತಾಂಶದ ಆಧಾರದ ಮೇಲೆ ಈಗ ಯಾವ ವ್ಯೂ ತೋರಿಸಬೇಕು ಎಂದು ನಿರ್ಧರಿಸುತ್ತದೆ.

ನಾವು ಈ ಪ್ರದರ್ಶನ ಲಾಜಿಕ್ ಅನ್ನು ಪ್ರಸ್ತುತ `ContentView` ಒಳಗೆ ಇರಿಸಬಹುದು:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Original login view code
		}
	}
}
```

ಇಲ್ಲಿ, ಬಳಕೆದಾರನು ಯಶಸ್ವಿಯಾಗಿ ಲಾಗಿನ್ ಆಗಿದ್ದಾನೆಯೇ ಎಂದು ಸೂಚಿಸಲು `login` ಎಂಬ ಹೊಸ ಚರವನ್ನು ಸೇರಿಸಿದ್ದೇವೆ.

ನಂತರ, ಈಗಾಗಲೇ ಬರೆದ ಲಾಗಿನ್ ವ್ಯೂ ಕೋಡ್ ಅನ್ನು `else` ಸ್ಟೇಟ್ಮೆಂಟ್ ನಂತರ ಇಟ್ಟಿದ್ದೇವೆ.

ವ್ಯೂ ಪ್ರದರ್ಶನ ಲಾಜಿಕ್:

- `login` `false` ಆಗಿದ್ದರೆ, ಲಾಗಿನ್ ಇನ್ನೂ ಯಶಸ್ವಿಯಾಗಿಲ್ಲ ಎಂದರ್ಥ, ಆದ್ದರಿಂದ ಮೂಲ ಲಾಗಿನ್ ವ್ಯೂ ಮುಂದುವರಿದು ತೋರಿಸಲಾಗುತ್ತದೆ.
- `login` `true` ಆಗಿದ್ದರೆ, ಲಾಗಿನ್ ಯಶಸ್ವಿಯಾಗಿದೆ ಎಂದರ್ಥ; ಈ ಸಮಯದಲ್ಲಿ ಲಾಗಿನ್ ವ್ಯೂ ತೋರಿಸಲಾಗುವುದಿಲ್ಲ, ಬದಲಿಗೆ `Login successful` ತೋರಿಸಲಾಗುತ್ತದೆ.

ಅಂದರೆ, `ContentView` `login` ಮೌಲ್ಯದ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವ್ಯೂ ವಿಷಯವನ್ನು ತೋರಿಸುತ್ತದೆ.

ಮುಂದೆ, ಬಟನ್ ಒಳಗಿನ ಲಾಗಿನ್ ಯಶಸ್ಸಿನ ಲಾಜಿಕ್ ಅನ್ನು ತಿದ್ದುಪಡಿ ಮಾಡೋಣ:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

ಬಳಕೆದಾರನು ನಮೂದಿಸಿದ ಬಳಕೆದಾರ ಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಎರಡೂ ಸರಿಯಾಗಿದ್ದರೆ, ನಾವು `login` ಅನ್ನು `true` ಆಗಿ ಬದಲಾಯಿಸುತ್ತೇವೆ.

`login` ಮೌಲ್ಯ ಬದಲಾಗುತ್ತಿದ್ದಂತೆಯೇ, `ContentView` ಒಳಗಿನ `if` ಪರಿಶೀಲನೆಯ ಫಲಿತಾಂಶವೂ ಬದಲಾಗುತ್ತದೆ, ಮತ್ತು ಇಂಟರ್‌ಫೇಸ್ ಮೂಲ ಲಾಗಿನ್ ವ್ಯೂನಿಂದ `Login successful` ಗೆ ಬದಲಾಗುತ್ತದೆ.

ತೋರಿಸುವ ಪರಿಣಾಮ ಹೀಗಿದೆ:

![view](../../Resource/020_view.png)

ಈಗ ನಾವು “ಷರತ್ತಿನ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವ್ಯೂಗಳನ್ನು ತೋರಿಸುವುದು” ಎಂಬ ತಿಳುವಳಿಕೆಯನ್ನು ನಿಜವಾಗಿಯೂ ಈ ಪ್ರಸ್ತುತ ಲಾಗಿನ್ ಪುಟದಲ್ಲಿ ಬಳಸಿದ್ದೇವೆ.

ನಿಜವಾದ ಆಪ್‌ಗಳಲ್ಲಿಯೂ ಸಾಮಾನ್ಯವಾಗಿ ಇದೇ ರೀತಿ ಮಾಡಲಾಗುತ್ತದೆ: ನಿರ್ದಿಷ್ಟ ಮೌಲ್ಯದ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವ್ಯೂಗಳು ತೋರಿಸಲಾಗುತ್ತವೆ.

ಉದಾಹರಣೆಗೆ, ಬಳಕೆದಾರನು ಲಾಗಿನ್ ಆಗಿಲ್ಲದಿದ್ದರೆ ಲಾಗಿನ್ ಪುಟ ತೋರಿಸಲಾಗುತ್ತದೆ; ಮತ್ತು ಲಾಗಿನ್ ಯಶಸ್ವಿಯಾದ ನಂತರ ಮತ್ತೊಂದು ಇಂಟರ್‌ಫೇಸ್ ತೋರಿಸಲಾಗುತ್ತದೆ.

ಮೂಲ ಕಲ್ಪನೆ ಏನೆಂದರೆ: **ಚರಗಳ ಬದಲಾವಣೆಗಳ ಮೂಲಕ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ಯಾವ ವಿಷಯ ತೋರಿಸಬೇಕು ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸುವುದು.**

## ಸಾರಾಂಶ

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ಮೂರು ಮುಖ್ಯ ವಿಷಯಗಳನ್ನು ಕಲಿತಿದ್ದೇವೆ:

1. ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಲು `SecureField` ಬಳಸುವುದು, ಇದರಿಂದ ಪಾಸ್‌ವರ್ಡ್ ವಿಷಯ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ನೇರವಾಗಿ ತೋರಿಸುವುದಿಲ್ಲ.
2. ಕೇವಲ ಕನ್ಸೋಲ್‌ನಲ್ಲಿ ಔಟ್‌ಪುಟ್ ತೋರಿಸುವ ಬದಲು, ಬಳಕೆದಾರರಿಗೆ ಸಂದೇಶ ತೋರಿಸಲು `Alert` ಬಳಸುವುದು.
3. ವಿಭಿನ್ನ ವ್ಯೂಗಳ ಪ್ರದರ್ಶನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಮತ್ತು ಲಾಗಿನ್ ಮುಂಚೆ ಹಾಗೂ ನಂತರದ ಇಂಟರ್‌ಫೇಸ್ ನಡುವೆ ಬದಲಾವಣೆ ಜಾರಿಗೆ ತರಲು ಚರಗಳು ಮತ್ತು `if` ಷರತ್ತುಗಳನ್ನು ಬಳಸುವುದು.

ಈ ಜ್ಞಾನವನ್ನು ಕಲಿಯುವ ಮೂಲಕ, ನಾವು ಈಗ ಮೂಲಭೂತ ಪರಸ್ಪರ ಕ್ರಿಯೆಗಳ ಪರಿಣಾಮಗಳಿರುವ ಒಂದು ಲಾಗಿನ್ ವ್ಯೂ ಪೂರ್ಣಗೊಳಿಸಲು ಸಮರ್ಥರಾಗಿದ್ದೇವೆ, ಮತ್ತು SwiftUI ಯ “ಸ್ಥಿತಿ ಬದಲಾಗಿದರೆ ಇಂಟರ್‌ಫೇಸ್ ನವೀಕರಿಸುತ್ತದೆ” ಎಂಬ ಅಭಿವೃದ್ಧಿ ಚಿಂತನೆಗೂ ಇನ್ನಷ್ಟು ಆಳವಾಗಿ ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳಬಹುದು.

## ವಿಸ್ತರಿತ ಜ್ಞಾನ - ಹಲವು ಬಟನ್‌ಗಳಿರುವ Alert

ಈ ಪಾಠದ ಹಿಂದಿನ ಭಾಗದಲ್ಲಿ, ನಾವು ಅತ್ಯಂತ ಮೂಲಭೂತ `Alert` ಅನ್ನು ಬಳಸಿದ್ದೇವೆ, ಅದರಲ್ಲಿ ಕೇವಲ ಒಂದು ಬಟನ್ ಮಾತ್ರ ಇತ್ತು ಮತ್ತು ಅದು ಮುಖ್ಯವಾಗಿ ಬಳಕೆದಾರರಿಗೆ ಸೂಚನೆ ನೀಡಲು ಬಳಸಲ್ಪಟ್ಟಿತು.

ವಾಸ್ತವವಾಗಿ, `Alert` ನಲ್ಲಿ ಹಲವು ಬಟನ್‌ಗಳೂ ಇರಬಹುದು, ಇದರಿಂದ ಬಳಕೆದಾರರು ವಿಭಿನ್ನ ಆಯ್ಕೆಗಳನ್ನು ಮಾಡಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, `Ok`, `delete`, ಮತ್ತು `cancel` ಎಂಬ ಮೂರು ಆಯ್ಕೆಗಳಿರುವ ಸಂದೇಶ ಬಾಕ್ಸ್ ಪಾಪ್ ಅಪ್ ಆಗುತ್ತದೆ.

ತೋರಿಸುವ ಪರಿಣಾಮ:

![alert](../../Resource/020_alert2.png)

ಬೇರೆ ಬೇರೆ ಬಟನ್‌ಗಳನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ ಸಂಬಂಧಿತ ಕೋಡ್ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ.

### Button ನ `role` ಗುಣಲಕ್ಷಣ

ಇಲ್ಲಿ ನೀನು ಗಮನಿಸುವುದೇನೆಂದರೆ, `Alert` ಒಳಗಿನ `Button` ನಾವು ಹಿಂದೆ ಕಲಿತ ಸಾಮಾನ್ಯ ಬಟನ್ syntax ಗಿಂತ ಸ್ವಲ್ಪ ಭಿನ್ನವಾಗಿದೆ:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

ಇಲ್ಲಿ ಹೆಚ್ಚುವರಿ `role` parameter ಇದೆ.

SwiftUI ಯಲ್ಲಿ, ಈ ಬಟನ್ ಯಾವ ರೀತಿಯ ಕ್ರಿಯೆಯನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಸಿಸ್ಟಮ್‌ಗೆ ತಿಳಿಸಲು `role` ಬಳಸಲಾಗುತ್ತದೆ.

ಕೆಲವು ಸಾಮಾನ್ಯ roles ಹೀಗಿವೆ:

- `.cancel`: ರದ್ದು ಮಾಡುವ ಕ್ರಿಯೆ
- `.destructive`: ಅಪಾಯಕಾರಿಯಾದ ಕ್ರಿಯೆ, ಉದಾಹರಣೆಗೆ delete
- `nil`: ಸಾಮಾನ್ಯ ಕ್ರಿಯೆ

ವಿಭಿನ್ನ `role` ಮೌಲ್ಯಗಳು ಸಾಮಾನ್ಯವಾಗಿ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲೂ ವಿಭಿನ್ನ ದೃಶ್ಯ ಶೈಲಿಗಳನ್ನು ನೀಡುತ್ತವೆ.

ಉದಾಹರಣೆಗೆ, ಅಪಾಯಕಾರಿಯಾದ ಕ್ರಿಯೆಗಳು ಹೆಚ್ಚಾಗಿ ಕಣ್ಣಿಗೆ ಬೀಳುವ ರೀತಿಯಲ್ಲಿ ತೋರಿಸಲಾಗುತ್ತವೆ, ಇದರಿಂದ ಬಳಕೆದಾರರು ಎಚ್ಚರಿಕೆಯಿಂದ ಕ್ಲಿಕ್ ಮಾಡುವಂತೆ ನೆನಪಾಗುತ್ತದೆ.

ಅದರ ಜೊತೆಗೆ, `role` ಬಟನ್‌ಗೆ ಹೆಚ್ಚುವರಿ semantic ಮಾಹಿತಿ ನೀಡುತ್ತದೆ, ಇದರಿಂದ screen readers (ಉದಾಹರಣೆಗೆ Apple ಸಾಧನಗಳಲ್ಲಿನ VoiceOver) ಬಳಕೆದಾರರಿಗೆ ಬಟನ್‌ನ ಉದ್ದೇಶವನ್ನು ವಿವರಿಸಲು ಸುಲಭವಾಗುತ್ತದೆ.

ಆದ್ದರಿಂದ, ಸೂಕ್ತ ಸಂದರ್ಭಗಳಲ್ಲಿ ಬಟನ್‌ಗೆ `role` ಸೇರಿಸುವುದು ಬಹಳ ಒಳ್ಳೆಯ ಅಭ್ಯಾಸವಾಗಿದೆ。
