# ರೂಪದ ಹೊಂದಾಣಿಕೆ

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ಗಾಗಿ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಹೇಗೆ ಹೊಂದಿಸಬೇಕು ಎಂಬುದನ್ನು ಕಲಿಯುತ್ತೇವೆ.

iPhone, iPad ಮತ್ತು Mac ನಲ್ಲಿ, ಬಳಕೆದಾರರು ಬೆಳಕು ಮೋಡ್ ಅಥವಾ ಕತ್ತಲೆ ಮೋಡ್ ಆಯ್ಕೆ ಮಾಡಬಹುದು.

ಸಿಸ್ಟಮ್ ಕತ್ತಲೆ ಮೋಡ್‌ಗೆ ಬದಲಾದಾಗ, ಅನೇಕ App‌ಗಳ ಹಿನ್ನೆಲೆ, ಪಠ್ಯ ಮತ್ತು ಐಕಾನ್ ಬಣ್ಣಗಳೂ ಅದಕ್ಕೆ ಅನುಗುಣವಾಗಿ ಬದಲಾಗುತ್ತವೆ.

ಉದಾಹರಣೆಗೆ, ಹಗಲು ಸಮಯದಲ್ಲಿ ಇಂಟರ್ಫೇಸ್ ಬೆಳಕು ಹಿನ್ನೆಲೆಯನ್ನು ಬಳಸಬಹುದು, ರಾತ್ರಿ ಅದು ಕತ್ತಲೆ ಹಿನ್ನೆಲೆಯಾಗಿ ಬದಲಾಗಬಹುದು.

![view](../../../Resource/025_view9.png)

ಸಿಸ್ಟಮ್ ರೂಪದ ಪ್ರಕಾರ ಇಂಟರ್ಫೇಸ್ ಪ್ರದರ್ಶನವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸರಿಹೊಂದಿಸುವ ಈ ಸಾಮರ್ಥ್ಯವನ್ನು ರೂಪದ ಹೊಂದಾಣಿಕೆ ಎಂದು ಕರೆಯಲಾಗುತ್ತದೆ.

## Xcode ನಲ್ಲಿ ರೂಪದ ಹೊಂದಾಣಿಕೆಯನ್ನು ಪೂರ್ವವೀಕ್ಷಣೆ ಮಾಡುವುದು

ರೂಪದ ಹೊಂದಾಣಿಕೆಯನ್ನು ಕಲಿಯುವ ಮೊದಲು, Xcode ನಲ್ಲಿ ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್ ಅನ್ನು ಹೇಗೆ ಪೂರ್ವವೀಕ್ಷಣೆ ಮಾಡುವುದು ಎಂದು ಮೊದಲು ತಿಳಿದುಕೊಳ್ಳೋಣ.

`Canvas` ಪ್ರದೇಶದ ಕೆಳಭಾಗದಲ್ಲಿ, `Device Settings` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದರೆ `Canvas Device Settings` ಪಾಪ್-ಅಪ್ ತೆರೆಯುತ್ತದೆ.

![canvas](../../../Resource/025_view.png)

ಪಾಪ್-ಅಪ್‌ನಲ್ಲಿ `Color Scheme` ಸೆಟ್ಟಿಂಗ್ ಅನ್ನು ನೋಡಬಹುದು.

![canvas1](../../../Resource/025_view1.png)

ಇದರ ಅರ್ಥ:

- `Light Appearance` ಎಂದರೆ ಬೆಳಕು ಮೋಡ್.
- `Dark Appearance` ಎಂದರೆ ಕತ್ತಲೆ ಮೋಡ್.

ಇಲ್ಲಿ ಪೂರ್ವವೀಕ್ಷಣೆ ರೂಪವನ್ನು ಬದಲಿಸಿ, ಪ್ರಸ್ತುತ ಇಂಟರ್ಫೇಸ್ ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ ಹೇಗೆ ಕಾಣುತ್ತದೆ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸಬಹುದು.

## ಎರಡು ರೂಪಗಳನ್ನು ಪಕ್ಕಪಕ್ಕದಲ್ಲಿ ತೋರಿಸುವುದು

ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್ ಅನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ನೋಡಲು ಬಯಸಿದರೆ, `Variants` ವೈಶಿಷ್ಟ್ಯವನ್ನು ಬಳಸಬಹುದು.

`Canvas` ಪ್ರದೇಶದ ಕೆಳಭಾಗದಲ್ಲಿ, `Variants` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ `Color Scheme Variants` ಆಯ್ಕೆಮಾಡಿ.

![canvas2](../../../Resource/025_view2.png)

ಆಯ್ಕೆ ಮಾಡಿದ ನಂತರ, `Canvas` ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ನ ಪೂರ್ವವೀಕ್ಷಣೆಗಳನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ತೋರಿಸುತ್ತದೆ.

![canvas3](../../../Resource/025_view3.png)

ಇದರಿಂದ ಎರಡು ರೂಪಗಳಲ್ಲಿನ ಇಂಟರ್ಫೇಸ್ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಹೋಲಿಸುವುದು ಇನ್ನಷ್ಟು ಸುಲಭವಾಗುತ್ತದೆ.

ಸರಳವಾಗಿ ಹೇಳುವುದಾದರೆ:

ಬೆಳಕು/ಕತ್ತಲೆ ಮೋಡ್‌ಗಳನ್ನು ತಾತ್ಕಾಲಿಕವಾಗಿ ಮಾತ್ರ ಬದಲಾಯಿಸಲು ಬಯಸಿದರೆ, `Canvas Device Settings` ಅನ್ನು ಬಳಸಬಹುದು.

ಎರಡೂ ರೂಪಗಳನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ನೋಡಲು ಬಯಸಿದರೆ, `Color Scheme Variants` ಅನ್ನು ಬಳಸಬಹುದು.

## View‌ಗಳು ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ರೂಪಕ್ಕೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತವೆ

SwiftUI ನಲ್ಲಿ, ಅನೇಕ ಸಿಸ್ಟಮ್ view‌ಗಳು ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ಗೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೊಂದಿಕೊಳ್ಳುತ್ತವೆ.

ಉದಾಹರಣೆಗೆ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("ಬೆಳಕು")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ಪ್ರದರ್ಶನ ಫಲಿತಾಂಶ:

![canvas](../../../Resource/025_view4.png)

ನೀವು ನೋಡಬಹುದು, ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ ಐಕಾನ್‌ಗಳು ಮತ್ತು ಶೀರ್ಷಿಕೆಗಳು ಸಾಮಾನ್ಯವಾಗಿ ಕತ್ತಲೆ ಬಣ್ಣದಲ್ಲಿ ಕಾಣುತ್ತವೆ.

ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ, ಐಕಾನ್‌ಗಳು ಮತ್ತು ಶೀರ್ಷಿಕೆಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬೆಳಕು ಬಣ್ಣಕ್ಕೆ ಬದಲಾಗುತ್ತವೆ.

ಇದಕ್ಕೆ ಕಾರಣ, `Text`, `Image` ಮುಂತಾದ SwiftUI view‌ಗಳು ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸಿಸ್ಟಮ್ ರೂಪದ ಪ್ರಕಾರ ತಮ್ಮ ಬಣ್ಣಗಳನ್ನು ಸರಿಹೊಂದಿಸುತ್ತವೆ.

ಅಂದರೆ, ನಾವು ಸ್ಥಿರ ಬಣ್ಣವನ್ನು ಕೈಯಾರೆ ಸೂಚಿಸದಿದ್ದರೆ, SwiftUI ರೂಪದ ಹೊಂದಾಣಿಕೆಯ ಒಂದು ಭಾಗವನ್ನು ನಮ್ಮಿಗಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ.

## ಸ್ಥಿರ ಬಣ್ಣಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಾಗುವುದಿಲ್ಲ

ಗಮನಿಸಬೇಕಾದದ್ದು ಏನೆಂದರೆ, ನಾವು ಸ್ಥಿರ ಬಣ್ಣವನ್ನು ಕೈಯಾರೆ ಸೂಚಿಸಿದರೆ, ಅದು ಬೆಳಕು/ಕತ್ತಲೆ ಮೋಡ್‌ಗಳ ನಡುವೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಾಗುವುದಿಲ್ಲ.

ಉದಾಹರಣೆಗೆ:

```swift
Text("ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ")
	.foregroundStyle(Color.gray)
```

ಇಲ್ಲಿ ಪಠ್ಯವನ್ನು `Color.gray` ಎಂದು ಹೊಂದಿಸಲಾಗಿದೆ, ಆದ್ದರಿಂದ ಅದು ಯಾವಾಗಲೂ ಬೂದು ಬಣ್ಣದಲ್ಲೇ ಕಾಣುತ್ತದೆ.

ಮತ್ತೊಂದು ಉದಾಹರಣೆ:

```swift
Text("ಶೀರ್ಷಿಕೆ")
    .foregroundStyle(Color.white)
```

ಈ ಕೋಡ್ ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್ ಎರಡರಲ್ಲಿಯೂ ಬಿಳಿ ಪಠ್ಯವನ್ನು ತೋರಿಸುತ್ತದೆ.

ಹಿನ್ನೆಲೆಯೂ ಬೆಳಕು ಬಣ್ಣದ್ದಾಗಿದ್ದರೆ, ಬಿಳಿ ಪಠ್ಯವನ್ನು ಓದಲು ಕಷ್ಟವಾಗಬಹುದು.

ಆದ್ದರಿಂದ ವಿಭಿನ್ನ ರೂಪಗಳಿಗೆ ಇಂಟರ್ಫೇಸ್ ಹೊಂದಿಸುವಾಗ, `Color.white`, `Color.black` ಮುಂತಾದ ಸ್ಥಿರ ಬಣ್ಣಗಳನ್ನು ಸುಲಭವಾಗಿ hard-code ಮಾಡುವುದನ್ನು ಸಾಧ್ಯವಾದಷ್ಟು ತಪ್ಪಿಸಬೇಕು.

ಅನೇಕ ಸಂದರ್ಭಗಳಲ್ಲಿ, ಸಿಸ್ಟಮ್ semantic style‌ಗಳನ್ನು ಆದ್ಯತೆ ನೀಡಬಹುದು, ಉದಾಹರಣೆಗೆ:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

ಇದರ ಅರ್ಥ:

- `.primary` ಸಾಮಾನ್ಯವಾಗಿ ಮುಖ್ಯ ವಿಷಯಕ್ಕೆ ಬಳಸಲಾಗುತ್ತದೆ.
- `.secondary` ಸಾಮಾನ್ಯವಾಗಿ ದ್ವಿತೀಯ ವಿಷಯಕ್ಕೆ ಬಳಸಲಾಗುತ್ತದೆ.

ಅವು ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ನ ಆಧಾರದ ಮೇಲೆ ತಮ್ಮ ಪ್ರದರ್ಶನವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸರಿಹೊಂದಿಸುತ್ತವೆ.

## ರೂಪದ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವಿಷಯ ತೋರಿಸುವುದು

ಕೆಲವೊಮ್ಮೆ, ನಾವು ಬಣ್ಣ ಮಾತ್ರ ಬದಲಾಗಬೇಕು ಎಂದು ಬಯಸುವುದಿಲ್ಲ; ವಿಭಿನ್ನ ರೂಪಗಳಲ್ಲಿ ವಿಭಿನ್ನ ವಿಷಯವೂ ಕಾಣಬೇಕು ಎಂದು ಬಯಸುತ್ತೇವೆ.

ಉದಾಹರಣೆಗೆ:

- ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ ಸಾಮಾನ್ಯ ದೀಪದ ಐಕಾನ್ ತೋರಿಸಬೇಕು.
- ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ ಹೊಳೆಯುವ ದೀಪದ ಐಕಾನ್ ತೋರಿಸಬೇಕು.
- ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ ಶೀರ್ಷಿಕೆ `ಬೆಳಕು` ಎಂದು ತೋರಿಸಬೇಕು.
- ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ ಶೀರ್ಷಿಕೆ `ಕತ್ತಲೆ` ಎಂದು ತೋರಿಸಬೇಕು.

ಈ ಸಂದರ್ಭದಲ್ಲಿ, ಪ್ರಸ್ತುತ ಸಿಸ್ಟಮ್ ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿದೆಯೇ ಅಥವಾ ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿದೆಯೇ ಎಂದು ನಿರ್ಧರಿಸಬೇಕು.

SwiftUI ನಲ್ಲಿ, ಪ್ರಸ್ತುತ ರೂಪ ಮೋಡ್ ಅನ್ನು `colorScheme` environment value ಮೂಲಕ ಪಡೆಯಬಹುದು:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ಪೂರ್ಣ ಕೋಡ್:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "ಬೆಳಕು" : "ಕತ್ತಲೆ"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ ಸ್ವಲ್ಪ")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ಪ್ರದರ್ಶನ ಫಲಿತಾಂಶ:

![view](../../../Resource/025_view5.png)

ಪೂರ್ವವೀಕ್ಷಣೆ ಫಲಿತಾಂಶದಿಂದ, ಒಂದೇ view ವಿಭಿನ್ನ ರೂಪ ಮೋಡ್‌ಗಳಲ್ಲಿ ಸಂಪೂರ್ಣವಾಗಿ ಒಂದೇ ವಿಷಯವನ್ನು ತೋರಿಸುವುದಿಲ್ಲ ಎಂದು ಕಾಣಬಹುದು.

ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ, `Image` ಸಾಮಾನ್ಯ ದೀಪದ ಐಕಾನ್ ತೋರಿಸುತ್ತದೆ, ಮತ್ತು ಶೀರ್ಷಿಕೆ `ಬೆಳಕು` ಎಂದು ತೋರಿಸುತ್ತದೆ.

ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ, `Image` ಹೊಳೆಯುವ ದೀಪದ ಐಕಾನ್ ತೋರಿಸುತ್ತದೆ, ಮತ್ತು ಶೀರ್ಷಿಕೆ `ಕತ್ತಲೆ` ಎಂದು ತೋರಿಸುತ್ತದೆ.

ಅಂದರೆ, ಇಲ್ಲಿ ಬದಲಾವಣೆ ಕೇವಲ ಬಣ್ಣದ ಬದಲಾವಣೆ ಮಾತ್ರವಲ್ಲ. ಐಕಾನ್ ಹೆಸರು ಮತ್ತು ಶೀರ್ಷಿಕೆ ಪಠ್ಯವೂ ಬದಲಾಗಿದೆ.

ಇದಕ್ಕೆ ಕಾರಣ, ನಾವು ಐಕಾನ್ ಮತ್ತು ಶೀರ್ಷಿಕೆಯನ್ನು ನೇರವಾಗಿ `body` ಒಳಗೆ hard-code ಮಾಡಿಲ್ಲ. ಬದಲಾಗಿ, `colorScheme` ಆಧರಿಸಿ ವಿಭಿನ್ನ ವಿಷಯವನ್ನು ಲೆಕ್ಕಿಸುತ್ತೇವೆ.

ಈ ಕೋಡ್‌ನಲ್ಲಿ, ನಾವು ಎರಡು computed property‌ಗಳನ್ನು ಬಳಸಿದ್ದೇವೆ:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "ಬೆಳಕು" : "ಕತ್ತಲೆ"
}
```

`colorScheme` `.light` ಆಗಿದ್ದರೆ, ಪ್ರಸ್ತುತ ಮೋಡ್ ಬೆಳಕು ಮೋಡ್ ಎಂದರ್ಥ.

`colorScheme` `.dark` ಆಗಿದ್ದರೆ, ಪ್ರಸ್ತುತ ಮೋಡ್ ಕತ್ತಲೆ ಮೋಡ್ ಎಂದರ್ಥ.

ಹೀಗಾಗಿ, ವಿಭಿನ್ನ ರೂಪ ಮೋಡ್‌ಗಳ ಪ್ರಕಾರ ವಿಭಿನ್ನ ಐಕಾನ್ ಹೆಸರುಗಳು ಮತ್ತು ಶೀರ್ಷಿಕೆ ಪಠ್ಯವನ್ನು ಹಿಂತಿರುಗಿಸಬಹುದು.

ಇದು computed property‌ಗಳ ಸಾಮಾನ್ಯ ಬಳಕೆಯೂ ಹೌದು: ಪ್ರಸ್ತುತ ಸ್ಥಿತಿಯ ಆಧಾರದ ಮೇಲೆ view ತೋರಿಸಬೇಕಾದ ವಿಷಯವನ್ನು ಲೆಕ್ಕಿಸುವುದು.

## @Environment ಅನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ನಾವು `@Environment` ಅನ್ನು ಎದುರಿಸುತ್ತಿರುವುದು ಇದೇ ಮೊದಲ ಬಾರಿ.

`@Environment` ಅನ್ನು SwiftUI ಯ environment‌ನಿಂದ ಒಂದು ಮೌಲ್ಯವನ್ನು ಓದುವುದು ಎಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು.

App ಓಡುತ್ತಿರುವಾಗ, ಸಿಸ್ಟಮ್ ಅನೇಕ ರೀತಿಯ environment ಮಾಹಿತಿಯನ್ನು ಒದಗಿಸುತ್ತದೆ, ಉದಾಹರಣೆಗೆ:

- ಪ್ರಸ್ತುತ ಭಾಷೆ
- ಪ್ರಸ್ತುತ ರೂಪ ಮೋಡ್
- ಪ್ರಸ್ತುತ layout ದಿಕ್ಕು
- ಪ್ರಸ್ತುತ font size ಸೆಟ್ಟಿಂಗ್

ಪ್ರಸ್ತುತ ರೂಪ ಮೋಡ್ ಪಡೆಯಬೇಕಾದಾಗ, ನಾವು `colorScheme` ಓದಬಹುದು:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ಇಲ್ಲಿ:

```swift
\.colorScheme
```

environment‌ನಿಂದ `colorScheme` ಮೌಲ್ಯವನ್ನು ಓದುವುದು ಎಂದರ್ಥ.

```swift
private var colorScheme
```

ಓದಿದ ಮೌಲ್ಯವನ್ನು `colorScheme` variable‌ನಲ್ಲಿ ಉಳಿಸುವುದು ಎಂದರ್ಥ.

variable ಹೆಸರನ್ನು ನೀವು ಸ್ವತಃ ನಿರ್ಧರಿಸಬಹುದು. ಉದಾಹರಣೆಗೆ, ಹೀಗೆ ಕೂಡ ಬರೆಯಬಹುದು:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

ಮುಂದಿನ `\.colorScheme` ಬದಲಾಗದಿರುವವರೆಗೆ, ಓದುತ್ತಿರುವುದು ಪ್ರಸ್ತುತ ರೂಪ ಮೋಡ್ ಎಂದರ್ಥ.

## colorScheme ನ ಎರಡು ಸಾಮಾನ್ಯ ಮೌಲ್ಯಗಳು

`colorScheme` ಸಾಮಾನ್ಯವಾಗಿ ಎರಡು ಮೌಲ್ಯಗಳನ್ನು ಹೊಂದಿರುತ್ತದೆ:

```swift
.light
.dark
```

ಅವುಗಳ ಅರ್ಥ:

- `.light`: ಬೆಳಕು ಮೋಡ್
- `.dark`: ಕತ್ತಲೆ ಮೋಡ್

ಉದಾಹರಣೆಗೆ:

```swift
private var titleName: String {
    colorScheme == .light ? "ಬೆಳಕು" : "ಕತ್ತಲೆ"
}
```

ಈ ಕೋಡ್‌ನ ಅರ್ಥ:

ಪ್ರಸ್ತುತ ಮೋಡ್ ಬೆಳಕು ಮೋಡ್ ಆಗಿದ್ದರೆ, `colorScheme == .light` ಸತ್ಯವಾಗುತ್ತದೆ, ಆದ್ದರಿಂದ ಅದು `"ಬೆಳಕು"` ಅನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ಇಲ್ಲದಿದ್ದರೆ, ಅದು `"ಕತ್ತಲೆ"` ಅನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ಆದ್ದರಿಂದ ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ:

```swift
Text(titleName)
```

ತೋರಿಸಲಾದ ಪಠ್ಯ:

```swift
ಬೆಳಕು
```

ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ, ತೋರಿಸಲಾದ ಪಠ್ಯ:

```swift
ಕತ್ತಲೆ
```

ಈ ರೀತಿಯಲ್ಲಿ, ವಿಭಿನ್ನ ರೂಪಗಳ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ವಿಷಯ ತೋರಿಸಬಹುದು.

## ಬಣ್ಣ ಹೊಂದಾಣಿಕೆಯ ಉದಾಹರಣೆ

ಪಠ್ಯ ಮತ್ತು ಐಕಾನ್‌ಗಳ ಹೊರತಾಗಿ, ಕೆಲವೊಮ್ಮೆ ನಾವು ಬಣ್ಣ ಹೊಂದಾಣಿಕೆಯನ್ನು ಸ್ವತಃ ನಿರ್ವಹಿಸಬೇಕಾಗಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("ಪಠ್ಯ")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

ಪ್ರದರ್ಶನ ಫಲಿತಾಂಶ:

![view](../../../Resource/025_view6.png)

ಈ ಕೋಡ್‌ನಲ್ಲಿ, ಪಠ್ಯ ಬಿಳಿ ಬಣ್ಣದಲ್ಲಿದೆ ಮತ್ತು ಹಿನ್ನೆಲೆ ಕಪ್ಪು ಬಣ್ಣದಲ್ಲಿದೆ.

ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ, ಕಪ್ಪು ಹಿನ್ನೆಲೆ ತುಂಬಾ ಸ್ಪಷ್ಟವಾಗಿ ಕಾಣುತ್ತದೆ.

ಆದರೆ ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ, ಸಂಪೂರ್ಣ ಇಂಟರ್ಫೇಸ್ ಹಿನ್ನೆಲೆಯೂ ಕಪ್ಪಾಗಿದ್ದರೆ, ಈ ಕಪ್ಪು ಹಿನ್ನೆಲೆ ಸಿಸ್ಟಮ್ ಹಿನ್ನೆಲೆಯೊಂದಿಗೇ ಬೆರೆತು ಕಡಿಮೆ ಸ್ಪಷ್ಟವಾಗಿ ಕಾಣುತ್ತದೆ.

ಈ ಸಮಯದಲ್ಲಿ, ರೂಪ ಮೋಡ್‌ನ ಪ್ರಕಾರ ಪಠ್ಯ ಬಣ್ಣ ಮತ್ತು ಹಿನ್ನೆಲೆ ಬಣ್ಣವನ್ನು ಬದಲಾಯಿಸಬಹುದು:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("ಪಠ್ಯ")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

ಪ್ರದರ್ಶನ ಫಲಿತಾಂಶ:

![view](../../../Resource/025_view7.png)

ಈಗ, ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿ ಕಪ್ಪು ಹಿನ್ನೆಲೆ ಮತ್ತು ಬಿಳಿ ಪಠ್ಯ ಇರುತ್ತದೆ.

ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿ ಬಿಳಿ ಹಿನ್ನೆಲೆ ಮತ್ತು ಕಪ್ಪು ಪಠ್ಯ ಇರುತ್ತದೆ.

ಹೀಗೆ ಯಾವ ರೂಪ ಬಳಸಿದರೂ, ಪಠ್ಯ ಮತ್ತು ಹಿನ್ನೆಲೆಗೆ ಸ್ಪಷ್ಟ contrast ಇರುತ್ತದೆ, ಮತ್ತು ಬಳಕೆದಾರರು ವಿಷಯವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ನೋಡಬಹುದು.

## colorScheme ಮತ್ತು Assets ನಡುವಿನ ವ್ಯತ್ಯಾಸ

`colorScheme` ಮತ್ತು `Assets` ಎರಡೂ ರೂಪದ ಹೊಂದಾಣಿಕೆಯನ್ನು ಜಾರಿಗೆ ತರುವುದಕ್ಕೆ ಬಳಸಬಹುದು, ಆದರೆ ಅವು ವಿಭಿನ್ನ ಸಂದರ್ಭಗಳಿಗೆ ಸೂಕ್ತವಾಗಿವೆ.

ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್ ಪ್ರಕಾರ ಪಠ್ಯ ವಿಷಯ, image ಹೆಸರುಗಳು ಅಥವಾ `SF Symbols` ಹೆಸರುಗಳನ್ನು ಮಾತ್ರ ಬದಲಾಯಿಸಬೇಕಾದರೆ, `colorScheme` ಬಳಸಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

App ನಲ್ಲಿ ಬಣ್ಣವನ್ನು ಆಗಾಗ್ಗೆ ಬಳಸುತ್ತಿದ್ದರೆ, ಉದಾಹರಣೆಗೆ theme color, card background color ಅಥವಾ text color, `Assets` ಬಣ್ಣಗಳನ್ನು ಬಳಸುವುದು ಹೆಚ್ಚು ಶಿಫಾರಸು ಮಾಡಲಾಗುತ್ತದೆ.

ಏಕೆಂದರೆ `Assets` ಬಣ್ಣಗಳನ್ನು ಅನೇಕ view‌ಗಳಲ್ಲಿ ಮರುಬಳಕೆ ಮಾಡಬಹುದು, ಮತ್ತು ನಂತರ ಬದಲಾಯಿಸುವುದೂ ಹೆಚ್ಚು ಸುಲಭ.

ಉದಾಹರಣೆಗೆ:

```swift
Color("redText")
```

ಈ ರೀತಿಯಲ್ಲಿ, `Assets` ನಲ್ಲಿ ಬಣ್ಣವನ್ನು ಮಾತ್ರ ಬದಲಾಯಿಸಿದರೆ ಸಾಕು, ಈ ಬಣ್ಣ ಬಳಸುವ ಎಲ್ಲ ಸ್ಥಳಗಳೂ ಒಟ್ಟಿಗೆ ಬದಲಾಗುತ್ತವೆ.

## ಸಾರಾಂಶ

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ಗಾಗಿ ರೂಪದ ಹೊಂದಾಣಿಕೆಯನ್ನು ಕಲಿತೆವು.

ಮೊದಲು, Xcode ನ `Canvas` ನಲ್ಲಿ ವಿಭಿನ್ನ ರೂಪಗಳನ್ನು ಹೇಗೆ ಪೂರ್ವವೀಕ್ಷಣೆ ಮಾಡುವುದು, ಮತ್ತು `Color Scheme Variants` ಬಳಸಿ ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್ ಅನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಹೇಗೆ ನೋಡುವುದು ಎಂದು ಕಲಿತೆವು.

ನಂತರ, SwiftUI ಯ `Text`, `Image` ಮುಂತಾದ view‌ಗಳು ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸಿಸ್ಟಮ್ ರೂಪದ ಪ್ರಕಾರ ಬಣ್ಣಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೊಂದಿಸುತ್ತವೆ ಎಂದು ತಿಳಿದುಕೊಂಡೆವು.

ಆದರೆ, `Color.white` ಅಥವಾ `Color.black` ಮುಂತಾದ ಸ್ಥಿರ ಬಣ್ಣಗಳನ್ನು ಕೈಯಾರೆ ಸೂಚಿಸಿದರೆ, ಈ ಬಣ್ಣಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಾಗುವುದಿಲ್ಲ.

ಮುಂದಾಗಿ, ನಾವು `@Environment` ಮತ್ತು `colorScheme` ಕಲಿತೆವು:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` ಮೂಲಕ, ಪ್ರಸ್ತುತ ಮೋಡ್ ಬೆಳಕು ಮೋಡ್‌ನಲ್ಲಿದೆಯೇ ಅಥವಾ ಕತ್ತಲೆ ಮೋಡ್‌ನಲ್ಲಿದೆಯೇ ಎಂದು ನಿರ್ಧರಿಸಬಹುದು, ಮತ್ತು ವಿಭಿನ್ನ ರೂಪಗಳ ಪ್ರಕಾರ ವಿಭಿನ್ನ ಐಕಾನ್‌ಗಳು, ಪಠ್ಯ ಅಥವಾ ಬಣ್ಣಗಳನ್ನು ತೋರಿಸಬಹುದು.

ಕೊನೆಯಲ್ಲಿ, ನಾವು `Assets` ಬಣ್ಣಗಳ ಬಗ್ಗೆ ಕಲಿತೆವು.

`Assets` ಬಣ್ಣಗಳು ಬೆಳಕು ಮೋಡ್ ಮತ್ತು ಕತ್ತಲೆ ಮೋಡ್‌ಗಾಗಿ ಪ್ರತ್ಯೇಕವಾಗಿ ಬಣ್ಣಗಳನ್ನು ಹೊಂದಿಸಬಹುದು, ಆದ್ದರಿಂದ App ನಲ್ಲಿನ ಸಾಮಾನ್ಯ color resource‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಅವು ಸೂಕ್ತವಾಗಿವೆ.

ರೂಪದ ಹೊಂದಾಣಿಕೆ App ಅನ್ನು ಹಗಲು ಮತ್ತು ರಾತ್ರಿ ಎರಡೂ ಸಮಯದಲ್ಲೂ ಉತ್ತಮವಾಗಿ ಕಾಣುವಂತೆ ಮಾಡುತ್ತದೆ, ಜೊತೆಗೆ ಪಠ್ಯ ಓದಲು ಕಷ್ಟವಾಗುವುದು ಅಥವಾ ಹಿನ್ನೆಲೆ ಸ್ಪಷ್ಟವಾಗದಿರುವಂತಹ ಸಮಸ್ಯೆಗಳನ್ನು ತಪ್ಪಿಸಬಹುದು.

ವಾಸ್ತವಿಕ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ, SwiftUI ಯ default adaptation behavior ಮತ್ತು `Assets` ಬಣ್ಣಗಳನ್ನು ಆದ್ಯತೆ ನೀಡಲು ಶಿಫಾರಸು ಮಾಡಲಾಗುತ್ತದೆ.

ವಿಭಿನ್ನ ರೂಪಗಳಲ್ಲಿ ಪಠ್ಯ, ಐಕಾನ್, ಚಿತ್ರಗಳನ್ನು ಬದಲಾಯಿಸಬೇಕಾದಾಗ, ಅಥವಾ ಕೆಲವು ವಿಶೇಷ ಪ್ರದರ್ಶನ ಪರಿಣಾಮಗಳನ್ನು ನಿರ್ವಹಿಸಬೇಕಾದಾಗ, ಆಗ `colorScheme` ಬಳಸಿ ಪರಿಶೀಲಿಸಿ.

## ವಿಸ್ತೃತ ಜ್ಞಾನ: font size ಸೂಚಿಸುವುದು

ಹಿಂದೆ `font` ಕಲಿತಾಗ, ನಾವು ಕೆಲವು system font style‌ಗಳನ್ನು ಬಳಸಿದ್ದೇವೆ:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` ಮುಂತಾದ system font style‌ಗಳನ್ನು ಬಳಸುವುದರ ಜೊತೆಗೆ, font size ಅನ್ನು ಕೈಯಾರೆ ಕೂಡ ಸೂಚಿಸಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

ಇಲ್ಲಿ:

```swift
.font(.system(size: 100))
```

font size ಅನ್ನು `100 pt` ಗೆ ಹೊಂದಿಸುವುದು ಎಂದರ್ಥ.

`SF Symbols` ಐಕಾನ್‌ಗಳಿಗೆ, `font` ಐಕಾನ್ ಗಾತ್ರದ ಮೇಲೆಯೂ ಪರಿಣಾಮ ಬೀರುತ್ತದೆ.

ಆದ್ದರಿಂದ ಐಕಾನ್ ದೊಡ್ಡದಾಗಿ ಕಾಣಬೇಕು ಎಂದರೆ, `.font(.system(size:))` ಮೂಲಕ ಗಾತ್ರವನ್ನು ಸೂಚಿಸಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

ಗಮನಿಸಬೇಕಾದದ್ದು ಏನೆಂದರೆ, font size ಅನ್ನು ಕೈಯಾರೆ ಸೂಚಿಸುವುದು ಹೆಚ್ಚು flexible ಆಗಿದ್ದರೂ, ಸಾಮಾನ್ಯ ಪಠ್ಯಕ್ಕೆ fixed font size‌ಗಳನ್ನು ಹೆಚ್ಚು ಬಳಸುವುದನ್ನು ಶಿಫಾರಸು ಮಾಡುವುದಿಲ್ಲ.

ಸಾಮಾನ್ಯ ಪಠ್ಯಕ್ಕೆ `.title`, `.headline`, `.body`, `.caption` ಮುಂತಾದ system font style‌ಗಳನ್ನು ಆದ್ಯತೆ ನೀಡಿ.

ಇದರಿಂದ ವಿಭಿನ್ನ ಸಾಧನಗಳು ಮತ್ತು ಬಳಕೆದಾರರ font ಸೆಟ್ಟಿಂಗ್‌ಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳುವುದು ಸುಲಭವಾಗುತ್ತದೆ.
