# Xcode ಫೈಲ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು

ಈ ಪಾಠದಲ್ಲಿ, Xcode ನಲ್ಲಿ ಸಾಮಾನ್ಯವಾಗಿ ಬಳಸುವ ಫೈಲ್ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಕಲಿಯುತ್ತೇವೆ.

ಹಿಂದಿನ ಪಾಠಗಳಲ್ಲಿ, ನಾವು ಹೆಚ್ಚಿನ ಕೋಡ್ ಅನ್ನು `ContentView` ಒಳಗೆ ಬರೆದಿದ್ದೇವೆ. ಇದರಿಂದ ಕೋಡ್ ಮತ್ತು ಪ್ರದರ್ಶನ ಪರಿಣಾಮವನ್ನು ಗಮನಿಸುವುದು ಸುಲಭವಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

ಆದರೆ ನಿಜವಾದ App ನಲ್ಲಿ, ಸಾಮಾನ್ಯವಾಗಿ ಎಲ್ಲಾ ಕೋಡ್ ಅನ್ನು ಒಂದೇ ಫೈಲ್‌ನಲ್ಲಿ ಬರೆಯುವುದಿಲ್ಲ.

ಪುಟಗಳು, ಘಟಕಗಳು ಮತ್ತು ಕಾರ್ಯಗಳು ಹೆಚ್ಚಾಗುತ್ತಾ ಹೋದಂತೆ, ನಾವು ಬೇರೆ ಬೇರೆ ದೃಶ್ಯಗಳನ್ನು ಬೇರೆ ಬೇರೆ SwiftUI ಫೈಲ್‌ಗಳಲ್ಲಿ ವಿಭಜಿಸಬೇಕು.

ಉದಾಹರಣೆಗೆ:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

ಇದರಿಂದ ಕೋಡ್ ರಚನೆ ಹೆಚ್ಚು ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ, ಹಾಗೆಯೇ ನಂತರದ ನಿರ್ವಹಣೆ ಮತ್ತು ತಿದ್ದುಪಡಿ ಸುಲಭವಾಗುತ್ತದೆ.

## ಫೈಲ್‌ಗಳನ್ನು ಏಕೆ ನಿರ್ವಹಿಸಬೇಕು

SwiftUI ನಲ್ಲಿ, ಒಂದು ಪುಟ ಅಥವಾ ಒಂದು ಘಟಕವನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಪ್ರತ್ಯೇಕ ದೃಶ್ಯ ಫೈಲ್ ಆಗಿ ಬರೆಯಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ಇಲ್ಲಿನ `CardView` ಒಂದು SwiftUI ದೃಶ್ಯವಾಗಿದೆ.

ಎಲ್ಲಾ ದೃಶ್ಯಗಳನ್ನು `ContentView` ಒಳಗೆ ಬರೆದರೆ, ಕೋಡ್ ಕ್ರಮೇಣ ಉದ್ದವಾಗುತ್ತದೆ ಮತ್ತು ಓದಲು ಹೆಚ್ಚು ಕಷ್ಟವಾಗುತ್ತದೆ.

ಆದ್ದರಿಂದ, ನಾವು ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸುವುದು, ಮರುಹೆಸರಿಸುವುದು, ಅಳಿಸುವುದು, ಮತ್ತು ಬೇರೆ ಬೇರೆ ಕೋಡ್ ಫೈಲ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಬಳಸುವುದು ಕಲಿಯಬೇಕು.

## SwiftUI ಫೈಲ್ ರಚಿಸುವುದು

Xcode ನಲ್ಲಿ SwiftUI ಫೈಲ್ ರಚಿಸಲು ಸಾಮಾನ್ಯವಾಗಿ ಎರಡು ವಿಧಾನಗಳಿವೆ:

ಒಂದು ಖಾಲಿ ಫೈಲ್ ರಚಿಸಿ, ನಂತರ ಕೋಡ್ ಅನ್ನು ಸ್ವತಃ ಬರೆಯುವುದು.

ಮತ್ತೊಂದು SwiftUI ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್ ರಚಿಸುವುದು; Xcode ಮೂಲ ಕೋಡ್ ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸುತ್ತದೆ.

ಆರಂಭಿಕರಿಗಾಗಿ, SwiftUI ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್ ಬಳಸುವುದನ್ನು ಹೆಚ್ಚು ಶಿಫಾರಸು ಮಾಡಲಾಗುತ್ತದೆ, ಏಕೆಂದರೆ ಅದು ಮೂಲ ರಚನೆಯನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸುತ್ತದೆ.

### ಖಾಲಿ ಫೈಲ್ ರಚಿಸುವುದು

Xcode ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ, ಖಾಲಿ ಜಾಗ ಅಥವಾ ಫೋಲ್ಡರ್ ಮೇಲೆ ರೈಟ್-ಕ್ಲಿಕ್ ಮಾಡಿ, `New Empty File` ಆಯ್ಕೆಮಾಡಿ.

ಕಾರ್ಯಾಚರಣೆಯ ಸ್ಥಳ ಈ ಕೆಳಗಿನಂತಿದೆ:

![xcode](../../../Resource/027_view1.png)

ಖಾಲಿ ಫೈಲ್ ರಚಿಸಿದ ನಂತರ, Xcode ಸಾಮಾನ್ಯವಾಗಿ ಹೆಸರಿಲ್ಲದ Swift ಫೈಲ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ, ಉದಾಹರಣೆಗೆ `Untitled.swift`.

ನಾವು ಅದನ್ನು `TestView.swift` ಎಂದು ಮರುಹೆಸರಿಸಬಹುದು.

SwiftUI ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ, ಈ ಫೈಲ್ ಒಂದು ದೃಶ್ಯವನ್ನು ಪ್ರದರ್ಶಿಸಲು ಬಳಸಿದರೆ, ಸಾಮಾನ್ಯವಾಗಿ ಹೆಸರಿನ ಕೊನೆಯಲ್ಲಿ `View` ಸೇರಿಸುವಂತೆ ಶಿಫಾರಸು ಮಾಡಲಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

ಇದು ಕಡ್ಡಾಯ ನಿಯಮವಲ್ಲ, ಆದರೆ ಇದು ಒಂದು ದೃಶ್ಯ ಫೈಲ್ ಎಂದು ಒಂದೇ ನೋಟದಲ್ಲಿ ತಿಳಿಯಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

ಖಾಲಿ ಫೈಲ್ ರಚಿಸಿದ ನಂತರ, ಫೈಲ್‌ನಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಟಿಪ್ಪಣಿಗಳು ಮಾತ್ರ ಇರಬಹುದು, ಅಥವಾ ಅದು ಖಾಲಿಯಾಗಿರಬಹುದು. ಇದು Xcode ಆವೃತ್ತಿ ಮತ್ತು ರಚಿಸುವ ವಿಧಾನಕ್ಕೆ ಅವಲಂಬಿತವಾಗಿರುತ್ತದೆ.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ಈ ಸಮಯದಲ್ಲಿ, ನಾವು SwiftUI ಕೋಡ್ ಅನ್ನು ಕೈಯಾರೆ ಬರೆಯಬೇಕು:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

ಇದರಿಂದ, ಸರಳವಾದ SwiftUI ದೃಶ್ಯ ಫೈಲ್ ರಚನೆ ಪೂರ್ಣಗೊಳ್ಳುತ್ತದೆ.

### ಫೈಲ್ ಮರುಹೆಸರಿಸುವುದು

ಫೈಲ್ ರಚಿಸಿದ ನಂತರ ಫೈಲ್ ಹೆಸರನ್ನು ಬದಲಾಯಿಸಲು ಬಯಸಿದರೆ, Xcode ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ ಫೈಲ್ ಆಯ್ಕೆಮಾಡಿ, ನಂತರ `⏎` Enter ಕೀಲಿಯನ್ನು ಒತ್ತಬಹುದು.

![xcode](../../../Resource/027_view2.png)

ಇದರಿಂದ ಫೈಲ್ ಹೆಸರನ್ನು ಬದಲಾಯಿಸಬಹುದು.

ಗಮನಿಸಬೇಕಾದ ವಿಷಯವೆಂದರೆ, ಫೈಲ್ ಹೆಸರನ್ನು ಬದಲಾಯಿಸುವುದರಿಂದ ಕೋಡ್‌ನಲ್ಲಿರುವ ದೃಶ್ಯದ ಹೆಸರು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಾಗುವುದಿಲ್ಲ.

ಉದಾಹರಣೆಗೆ, ಫೈಲ್ ಹೆಸರನ್ನು `ProfileView.swift` ಎಂದು ಬದಲಾಯಿಸಿದ ನಂತರವೂ, ಕೋಡ್‌ನಲ್ಲಿ ಇನ್ನೂ `struct TestView: View` ಇರಬಹುದು.

ಇದು ಖಂಡಿತವಾಗಿ ದೋಷಕ್ಕೆ ಕಾರಣವಾಗದೇ ಇರಬಹುದು, ಆದರೆ ಗೊಂದಲ ಉಂಟುಮಾಡಬಹುದು.

ಆದ್ದರಿಂದ, ಫೈಲ್ ಹೆಸರು ಮತ್ತು ದೃಶ್ಯ ಹೆಸರನ್ನು ಒಂದೇ ರೀತಿಯಾಗಿ ಇರಿಸುವುದು ಉತ್ತಮ. ಉದಾಹರಣೆಗೆ ಫೈಲ್ ಹೆಸರು `ProfileView.swift` ಆಗಿದ್ದರೆ, ದೃಶ್ಯ ಹೆಸರನ್ನೂ `ProfileView` ಆಗಿ ಬಳಸಬಹುದು.

## SwiftUI ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್ ರಚಿಸುವುದು

ಖಾಲಿ ಫೈಲ್ ರಚಿಸುವುದರ ಜೊತೆಗೆ, ನಾವು ನೇರವಾಗಿ SwiftUI ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್‌ನನ್ನೂ ರಚಿಸಬಹುದು.

Xcode ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ, ಖಾಲಿ ಜಾಗ ಅಥವಾ ಫೋಲ್ಡರ್ ಮೇಲೆ ರೈಟ್-ಕ್ಲಿಕ್ ಮಾಡಿ, `New File from Template...` ಆಯ್ಕೆಮಾಡಿ.

ಕಾರ್ಯಾಚರಣೆಯ ಸ್ಥಳ ಈ ಕೆಳಗಿನಂತಿದೆ:

![xcode](../../../Resource/027_view3.png)

ತೆರೆಯುವ ವಿಂಡೋದಲ್ಲಿ, ಮೇಲ್ಭಾಗದಲ್ಲಿ `iOS`, `macOS` ಮುಂತಾದ ಬೇರೆ ಬೇರೆ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳನ್ನು ಆಯ್ಕೆಮಾಡಬಹುದು.

ಇಲ್ಲಿ ಪ್ರಸ್ತುತ ಪ್ರಾಜೆಕ್ಟ್‌ಗೆ ಹೊಂದುವ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಅನ್ನು ಹಾಗೆಯೇ ಆಯ್ಕೆಮಾಡಿದರೆ ಸಾಕು. ಉದಾಹರಣೆಗೆ, ನಾವು iOS App ರಚಿಸುತ್ತಿದ್ದರೆ, `iOS` ಆಯ್ಕೆಮಾಡಿ.

![xcode](../../../Resource/027_view4.png)

ನಂತರ ಕೆಳಭಾಗದಲ್ಲಿ `SwiftUI View` ಆಯ್ಕೆಮಾಡಿ.

![xcode](../../../Resource/027_view12.png)

ನಂತರ `Next` ಕ್ಲಿಕ್ ಮಾಡಿ, ಫೈಲ್ ಹೆಸರನ್ನು ನಮೂದಿಸಿ ಮತ್ತು ಉಳಿಸಿ.

ಉದಾಹರಣೆಗೆ, ನಾವು `Test2View.swift` ಎಂಬ ಫೈಲ್ ರಚಿಸುತ್ತೇವೆ.

Xcode ಕೆಳಗಿನಂತಿರುವ ಕೋಡ್ ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸುತ್ತದೆ:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

ಖಾಲಿ ಫೈಲ್‌ನೊಂದಿಗೆ ಹೋಲಿಸಿದರೆ, ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ `import SwiftUI`, `struct`, `body` ಮತ್ತು `#Preview` ಒಳಗೊಂಡಿರುತ್ತದೆ.

ಆದ್ದರಿಂದ ಆರಂಭಿಕರಿಗೆ, SwiftUI ಟೆಂಪ್ಲೇಟ್ ಫೈಲ್ ಬಳಸುವುದು ಹೆಚ್ಚು ಸರಳ, ಮತ್ತು ಕೋಡ್ ಮರೆತಿಹೋಗುವ ಸಾಧ್ಯತೆಯೂ ಕಡಿಮೆ.

**ಸೂಚನೆ**: ರೈಟ್-ಕ್ಲಿಕ್ ಮೂಲಕ ಫೈಲ್ ರಚಿಸುವುದರ ಜೊತೆಗೆ, Xcode ಎಡಭಾಗದ Navigator ಕೆಳಭಾಗದಲ್ಲಿರುವ ಪ್ಲಸ್ ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ ಫೈಲ್ ಅಥವಾ ಫೋಲ್ಡರ್ ರಚಿಸಬಹುದು.

![xcode](../../../Resource/027_view5.png)

## ಫೋಲ್ಡರ್ ರಚಿಸುವುದು

ಫೈಲ್‌ಗಳು ಹೆಚ್ಚಾಗುತ್ತಾ ಹೋದಾಗ, ಅವುಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಬಳಸಬಹುದು.

Xcode ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ, ಖಾಲಿ ಜಾಗ ಅಥವಾ ಫೋಲ್ಡರ್ ಮೇಲೆ ರೈಟ್-ಕ್ಲಿಕ್ ಮಾಡಿ, `New Folder` ಆಯ್ಕೆಮಾಡಿ.

ಫೋಲ್ಡರ್ ರಚಿಸಿದ ನಂತರ, ಸಂಬಂಧಿತ ಫೈಲ್‌ಗಳನ್ನು ಫೋಲ್ಡರ್‌ಗೆ ಎಳೆದು ಹಾಕಬಹುದು.

![xcode](../../../Resource/027_view7.png)

ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಮುಖ್ಯವಾಗಿ ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಯನ್ನು ಸರಿಪಡಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ; ಅವು ದೃಶ್ಯವನ್ನು ಬಳಸುವ ವಿಧಾನವನ್ನು ಬದಲಾಯಿಸುವುದಿಲ್ಲ.

ದೃಶ್ಯ ಹೆಸರೇ ಸರಿಯಾಗಿದ್ದರೆ, ಇತರೆ ಫೈಲ್‌ಗಳಲ್ಲಿಯೂ `TestView()` ಮತ್ತು `Test2View()` ರೀತಿಯಲ್ಲಿ ಬಳಸಬಹುದು.

## ಫೈಲ್ ಅಥವಾ ಫೋಲ್ಡರ್ ಅಳಿಸುವುದು

ಫೈಲ್ ಅಥವಾ ಫೋಲ್ಡರ್ ಅಳಿಸಲು ಬಯಸಿದರೆ, Xcode ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ ಸಂಬಂಧಿತ ಫೈಲ್ ಆಯ್ಕೆಮಾಡಿ, ನಂತರ ರೈಟ್-ಕ್ಲಿಕ್ ಮಾಡಿ `Delete` ಆಯ್ಕೆಮಾಡಬಹುದು.

ಫೈಲ್ ಆಯ್ಕೆ ಮಾಡಿದ ನಂತರ `⌫` Delete ಕೀಲಿಯನ್ನು ಒತ್ತುವುದೂ ಸಾಧ್ಯ.

Xcode ಅಳಿಸುವಿಕೆಯ ದೃಢೀಕರಣ ವಿಂಡೋವನ್ನು ತೋರಿಸುತ್ತದೆ.

![xcode](../../../Resource/027_view10.png)

`Move to Trash` ಆಯ್ಕೆಮಾಡುವುದರಿಂದ ಫೈಲ್ ಅನ್ನು Trash ಗೆ ಸರಿಸಲಾಗುತ್ತದೆ.

`Remove Reference` ಕಂಡುಬಂದರೆ, ಅದು Xcode ಪ್ರಾಜೆಕ್ಟ್‌ನಿಂದ ಉಲ್ಲೇಖವನ್ನು ಮಾತ್ರ ತೆಗೆದುಹಾಕುತ್ತದೆ, ಆದರೆ ಡಿಸ್ಕ್‌ನಲ್ಲಿರುವ ಮೂಲ ಫೈಲ್ ಅನ್ನು ಅಳಿಸುವುದಿಲ್ಲ.

ಆರಂಭಿಕರಿಗಾಗಿ, ನೀವು ಇತ್ತೀಚೆಗೆ ರಚಿಸಿದ ಸಾಮಾನ್ಯ ಫೈಲ್ ಅನ್ನು ಮಾತ್ರ ಅಳಿಸುತ್ತಿದ್ದರೆ, ಸಾಮಾನ್ಯವಾಗಿ `Move to Trash` ಆಯ್ಕೆಮಾಡಿದರೆ ಸಾಕು.

## ಹಲವು SwiftUI ದೃಶ್ಯಗಳನ್ನು ಪ್ರದರ್ಶಿಸುವುದು

ಫೈಲ್ ರಚಿಸುವುದನ್ನು ಕಲಿತ ನಂತರ, ನಾವು ಬೇರೆ ಬೇರೆ ದೃಶ್ಯಗಳನ್ನು ಬೇರೆ ಬೇರೆ ಫೈಲ್‌ಗಳಲ್ಲಿ ವಿಭಜಿಸಬಹುದು.

ಉದಾಹರಣೆಗೆ, ನಾವು ಎರಡು SwiftUI ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸುತ್ತೇವೆ:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` ಅನ್ನು ಲೇಖಕರ ಹೆಸರನ್ನು ಪ್ರದರ್ಶಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` ಅನ್ನು ಕಾರ್ಡ್ ಹಿನ್ನೆಲೆಯನ್ನು ಪ್ರದರ್ಶಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

ನಂತರ, ನಾವು `ContentView` ಒಳಗೆ ಈ ಎರಡು ದೃಶ್ಯಗಳನ್ನು ಬಳಸಬಹುದು:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

ಚಲಾಯಿಸಿದ ನಂತರ, `CardView` ಮತ್ತು `AuthorView` ಒಂದಾಗಿ ಸಂಯೋಜಿತವಾಗಿ ಪ್ರದರ್ಶಿಸುವುದನ್ನು ನೋಡಬಹುದು:

![xcode](../../../Resource/027_view11.png)

ಈ ಪರಿಣಾಮದಲ್ಲಿ, `CardView` ಹಿನ್ನೆಲೆಯಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ, ಮತ್ತು `AuthorView` ಮೇಲ್ಭಾಗದಲ್ಲಿ ಕಾಣಿಸುತ್ತದೆ. ಇದು ಅವುಗಳನ್ನು `ZStack` ಒಳಗೆ ಇಟ್ಟಿರುವುದರಿಂದ ಆಗುತ್ತದೆ.

`ContentView` ಒಳಗೆ, ನಾವು `CardView()` ಮತ್ತು `AuthorView()` ಮೂಲಕ ಈ ಎರಡು ದೃಶ್ಯಗಳನ್ನು ಬಳಸುತ್ತೇವೆ. ಹೆಸರಿನ ನಂತರದ `()` ಈ ದೃಶ್ಯವನ್ನು ರಚಿಸಿ ಬಳಸುವುದನ್ನು ಸೂಚಿಸುತ್ತದೆ.

ಇದೇ SwiftUI ಯಲ್ಲಿ ಸಾಮಾನ್ಯವಾದ ದೃಶ್ಯ ನೆಸ್ಟಿಂಗ್: ಒಂದು ದೃಶ್ಯದ ಒಳಗೆ ಇತರೆ ದೃಶ್ಯಗಳನ್ನು ಮುಂದುವರಿಸಿ ಬಳಸಬಹುದು.

## ಪ್ರವೇಶ ಫೈಲ್

ಕೊನೆಯಲ್ಲಿ, App ಎಲ್ಲಿ നിന്ന് ಇಂಟರ್ಫೇಸ್ ಪ್ರದರ್ಶನವನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಮತ್ತೆ ಪರಿಶೀಲಿಸೋಣ.

SwiftUI ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ, ಸಾಮಾನ್ಯವಾಗಿ ಒಂದು ಪ್ರವೇಶ ಫೈಲ್ ಇರುತ್ತದೆ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ಇಲ್ಲಿನ `@main` ಇದು App ನ ಪ್ರವೇಶ ಸ್ಥಳ ಎಂದು ಸೂಚಿಸುತ್ತದೆ.

`WindowGroup` ಒಳಗೆ, ನಾವು ಇದನ್ನು ನೋಡುತ್ತೇವೆ:

```swift
ContentView()
```

ಇದು App ಆರಂಭವಾದ ನಂತರ ಮೊದಲಿಗೆ `ContentView` ಪ್ರದರ್ಶಿಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ಸೂಚಿಸುತ್ತದೆ.

ನಂತರ `ContentView` ಒಳಗೆ ಮತ್ತೊಮ್ಮೆ ಇದನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ಆದ್ದರಿಂದ ದೃಶ್ಯ ಹಂತವನ್ನು ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

ಪ್ರವೇಶ ಫೈಲ್‌ನಲ್ಲಿ ಪ್ರದರ್ಶಿಸುವ ದೃಶ್ಯವನ್ನು ನಾವು `CardView` ಆಗಿ ಬದಲಿಸಿದರೆ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

ಆಗ App ಆರಂಭವಾದ ನಂತರ, ಅದು `CardView` ಅನ್ನು ಮಾತ್ರ ಪ್ರದರ್ಶಿಸುತ್ತದೆ.

ಅಂದರೆ, ಪ್ರವೇಶ ಕೋಡ್‌ನಲ್ಲಿರುವ `WindowGroup` App ಆರಂಭವಾದ ನಂತರ ಮೊದಲು ಯಾವ ದೃಶ್ಯವನ್ನು ಪ್ರದರ್ಶಿಸಬೇಕು ಎಂಬುದನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ.

ಮತ್ತು ಆ ದೃಶ್ಯದ ಒಳಗೆ, ಇತರೆ ದೃಶ್ಯಗಳನ್ನು ಮುಂದುವರಿಸಿ ಪ್ರದರ್ಶಿಸಬಹುದು.

## ಸಾರಾಂಶ

ಈ ಪಾಠದಲ್ಲಿ, Xcode ನಲ್ಲಿ ಸಾಮಾನ್ಯವಾಗಿ ಬಳಸುವ ಫೈಲ್ ನಿರ್ವಹಣಾ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಕಲಿತೇವೆ.

SwiftUI ಫೈಲ್‌ಗಳನ್ನು ಹೇಗೆ ರಚಿಸುವುದು, ಫೈಲ್‌ಗಳನ್ನು ಹೇಗೆ ಮರುಹೆಸರಿಸುವುದು, ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಹೇಗೆ ರಚಿಸುವುದು, ಹಾಗೂ ಫೈಲ್‌ಗಳನ್ನು ಹೇಗೆ ಅಳಿಸುವುದು ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಂಡೆವು.

ಅದೇ ಸಮಯದಲ್ಲಿ, ನಾವು ಒಂದು ಮುಖ್ಯ ಕಲ್ಪನೆಯನ್ನು ಕಲಿತೇವೆ: SwiftUI ದೃಶ್ಯಗಳನ್ನು ಬೇರೆ ಬೇರೆ ಫೈಲ್‌ಗಳಲ್ಲಿ ವಿಭಜಿಸಿ, ನಂತರ ಇತರೆ ದೃಶ್ಯಗಳಲ್ಲಿ ಬಳಸಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ಇಲ್ಲಿನ `CardView()` ಮತ್ತು `AuthorView()` ಎಂದರೆ `ContentView` ಒಳಗೆ ಇತರೆ ದೃಶ್ಯಗಳನ್ನು ಬಳಸುವುದು.

ಕೊನೆಯಲ್ಲಿ, ಪ್ರವೇಶ ಫೈಲ್‌ನ ಪ್ರದರ್ಶನ ಲಾಜಿಕ್‌ನ್ನೂ ನಾವು ಮರುಪರಿಶೀಲಿಸಿದೆವು.

App ಆರಂಭವಾದಾಗ, ಅದು ಮೊದಲು `@main` ಗುರುತಿನ ಪ್ರವೇಶ ಕೋಡ್‌ಗೆ ಪ್ರವೇಶಿಸಿ, ನಂತರ `WindowGroup` ಒಳಗಿನ ದೃಶ್ಯವನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ.

ಇದು ನಮಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ: ಬೇರೆ ಬೇರೆ SwiftUI ಫೈಲ್‌ಗಳು ಪ್ರತ್ಯೇಕವಾಗಿರುವುದಿಲ್ಲ; ಅವು ಪರಸ್ಪರ ಸಂಯೋಜಿಸಬಹುದು ಮತ್ತು ಕೊನೆಯಲ್ಲಿ ಸಂಪೂರ್ಣ App ಇಂಟರ್ಫೇಸ್ ಆಗಿ ರೂಪಗೊಳ್ಳಬಹುದು.
