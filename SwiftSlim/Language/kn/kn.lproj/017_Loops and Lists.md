# ಲೂಪ್‌ಗಳು ಮತ್ತು ಪಟ್ಟಿಗಳು

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು Swift ನಲ್ಲಿನ `for` ಲೂಪ್ ಹಾಗೂ SwiftUI ನಲ್ಲಿನ `ForEach` ಅನ್ನು ಕಲಿಯುತ್ತೇವೆ.

ಇವು ಎರಡೂ ಕೂಡ ಅರೆದಲ್ಲಿರುವ ಅಂಶಗಳನ್ನು ಕ್ರಮವಾಗಿ ಓದಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ. ನೈಜ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ, ನಾವು ಅನೇಕ ಬಾರಿ ಒಂದು ಗುಂಪಿನ ಡೇಟಾವನ್ನು ಒಂದೊಂದಾಗಿ ತೆಗೆದು, ನಂತರ ಅದನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಲು ಅಥವಾ ತೋರಿಸಲು ಬೇಕಾಗುತ್ತದೆ; ಲೂಪ್ ಎನ್ನುವುದು ಈ ಕೆಲಸವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವ ಪ್ರಮುಖ ಸಾಧನವಾಗಿದೆ.

ನಾವು ವೆಬ್‌ಪುಟಗಳನ್ನು ವೀಕ್ಷಿಸುವಾಗ ಅಥವಾ App ಬಳಸುವಾಗ, ಅನೇಕ ಪಟ್ಟಿಗಳು ಅಥವಾ ಐಟಂಗಳನ್ನು ನೋಡುತ್ತೇವೆ. ಇಂತಹ ವಿಷಯಗಳನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಮೊದಲು `ಅರೆ`ಯಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ, ನಂತರ `ಲೂಪ್` ಮೂಲಕ ಒಂದೊಂದಾಗಿ ಓದಿ, ಕೊನೆಯಲ್ಲಿ ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ತೋರಿಸಲಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, Google ನ ಹುಡುಕಾಟ ಫಲಿತಾಂಶಗಳ ಪಟ್ಟಿ:

![Google](../../Resource/017_google.png)

ಹುಡುಕಾಟ ಫಲಿತಾಂಶಗಳ ಪಟ್ಟಿಯಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಐಟಂ ಸಹಜವಾಗಿ ಒಂದೊಂದು ಡೇಟಾ ಅಂಶಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗಿರುತ್ತದೆ, ಮತ್ತು ಲೂಪ್ ಮೂಲಕ ಕ್ರಮವಾಗಿ ತೋರಿಸಲಾಗುತ್ತದೆ.

ಆದ್ದರಿಂದ, `ಅರೆ` ಮತ್ತು `ಲೂಪ್`ಗಳು ಬಹಳ ಬಾರಿ ಜೊತೆಯಾಗಿ ಕಾಣಿಸುತ್ತವೆ. `ಅರೆ` ಒಂದು ಗುಂಪಿನ ಡೇಟಾವನ್ನು ಸಂಗ್ರಹಿಸುವ ಜವಾಬ್ದಾರಿ ವಹಿಸುತ್ತದೆ, ಮತ್ತು `ಲೂಪ್` ಆ ಡೇಟಾವನ್ನು ಕ್ರಮವಾಗಿ ತೆಗೆದು, ಅದೇ ಕಾರ್ಯವನ್ನು ನಿರ್ವಹಿಸುವ ಜವಾಬ್ದಾರಿ ವಹಿಸುತ್ತದೆ.

ಈಗ, ನಾವು ಅತ್ಯಂತ ಮೂಲಭೂತವಾದ `for` ಲೂಪ್‌ನಿಂದ ಪ್ರಾರಂಭಿಸೋಣ.

## ಕೌಂಟ್‌ಡೌನ್

ನಾವು ಒಂದು ಕೌಂಟ್‌ಡೌನ್ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಅನುಕರಿಸಲು, ಕ್ರಮವಾಗಿ `“5、4、3、2、1”` ಅನ್ನು ಮುದ್ರಿಸಬೇಕೆಂದು ಕಲ್ಪಿಸೋಣ.

ಅತ್ಯಂತ ನೇರವಾದ ಬರವಣಿಗೆಯ ವಿಧಾನ ಹೀಗಿದೆ:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

ಈ ವಿಧಾನದಿಂದ ಕೌಂಟ್‌ಡೌನ್ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಅನುಕರಿಸಬಹುದು, ಆದರೆ ವಿಷಯ ಹೆಚ್ಚು ಇದ್ದರೆ, ಇದು ಬಹಳ ಪುನರಾವರ್ತಿತವಾಗಿ ಕಾಣುತ್ತದೆ, ಮತ್ತು ನಂತರ ತಿದ್ದುಪಡಿ ಮಾಡಲೂ ಅನುಕೂಲಕರವಾಗಿರುವುದಿಲ್ಲ.

ಈ ಸಂದರ್ಭದಲ್ಲಿ, ನಾವು ಮೊದಲು ಡೇಟಾವನ್ನು ಅರೆನಲ್ಲಿ ಇಟ್ಟು, ನಂತರ `for` ಲೂಪ್ ಬಳಸಿ ಕ್ರಮವಾಗಿ ಮುದ್ರಿಸಬಹುದು:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ಈ ಬರವಣಿಗೆಯ ವಿಧಾನ ಹೆಚ್ಚು ಸರಳವಾಗಿದ್ದು, ವಿಸ್ತರಿಸಲು ಕೂಡ ಸುಲಭ. ಕೌಂಟ್‌ಡೌನ್‌ನಲ್ಲಿನ ಸಂಖ್ಯೆಗಳು ಬಹಳ ಹೆಚ್ಚಿದ್ದರೂ ಸಹ, ನಾವು `print` ಅನ್ನು ಸಾಲುಸಾಲಾಗಿ ಪುನರಾವರ್ತಿಸಿ ಬರೆಯಬೇಕಾಗಿಲ್ಲ; ಅರೆದಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನೂ ಲೂಪ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಿಭಾಯಿಸಿದರೆ ಸಾಕು.

ಈ ಕೋಡ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಲು ನಾವು ಬಟನ್‌ನೊಳಗೆ ಇಡಬಹುದು:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ, `for` ಲೂಪ್ ಅರೆದಲ್ಲಿನ ಕ್ರಮದಂತೆ `countDown` ಒಳಗಿನ ಪ್ರತಿಯೊಂದು ಪೂರ್ಣಾಂಕವನ್ನು ಒಂದೊಂದಾಗಿ ತೆಗೆದು, ಅವನ್ನು ಕನ್‌ಸೋಲ್‌ನಲ್ಲಿ ಮುದ್ರಿಸುತ್ತದೆ.

ಮುಂದಾಗಿ, `for` ಲೂಪ್ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ನೋಡೋಣ.

## for ಲೂಪ್

`for` ಲೂಪ್ ಒಂದು ಗುಂಪಿನ ಡೇಟಾವನ್ನು ಕ್ರಮವಾಗಿ ಸಂಸ್ಕರಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ; ಸಾಮಾನ್ಯವಾಗಿ ಬಳಸುವ ವಸ್ತುಗಳು ಅರೆಗಳು ಮತ್ತು ವ್ಯಾಪ್ತಿಗಳು.

ಮೂಲಭೂತ ಬರವಣಿಗೆ:

```swift
for ಅಂಶ in ಅರೆ ಅಥವಾ ವ್ಯಾಪ್ತಿ {
    ಕಾರ್ಯಗತಗೊಳ್ಳುವ ಕೋಡ್
}
```

ಉದಾಹರಣೆಗೆ, ಈಗಷ್ಟೇ ನೋಡಿದ ಕೌಂಟ್‌ಡೌನ್ ಕೋಡ್:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ಇಲ್ಲಿನ `i` ಪ್ರಸ್ತುತ ತೆಗೆದುಕೊಳ್ಳಲಾದ ಅಂಶವನ್ನು ಸೂಚಿಸುತ್ತದೆ; `in` ನಂತರ ಬರುವದ್ದು ಲೂಪ್ ಮಾಡಬೇಕಾದ ಡೇಟಾ, ಅಂದರೆ `countDown` ಅರೆ.

ಈ ಕೋಡ್‌ನ ಅರ್ಥ ಏನೆಂದರೆ: ಕಾರ್ಯಕ್ರಮವು `countDown` ಅರೆದಿಂದ ಕ್ರಮವಾಗಿ ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ಪ್ರತಿ ಬಾರಿ ಒಂದು ಅಂಶವನ್ನು ತೆಗೆದುಕೊಂಡಾಗ, ಅದನ್ನು ತಾತ್ಕಾಲಿಕವಾಗಿ `i` ಮೂಲಕ ಸೂಚಿಸಿ, ನಂತರ ಕರ್ಳಿ ಬ್ರೇಸ್‌ಗಳೊಳಗಿನ ಕೋಡ್ ಅನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತದೆ.

ಕಾರ್ಯಗತ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು:

* ಮೊದಲ ಬಾರಿ ತೆಗೆದುಕೊಳ್ಳಲಾಗುವುದು `5`, ಆದ್ದರಿಂದ ಆ ಸಮಯದಲ್ಲಿ `i` ಯ ಮೌಲ್ಯ `5`
* `print(i)` ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ, `5` ಮುದ್ರಣವಾಗುತ್ತದೆ
* ನಂತರ ಮುಂದಿನ ಅಂಶವಾದ `4` ತೆಗೆದುಕೊಳ್ಳಲಾಗುತ್ತದೆ
* ಮತ್ತೆ `print(i)` ಒಂದು ಬಾರಿ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ, `4` ಮುದ್ರಣವಾಗುತ್ತದೆ

ಮುಂದಿನ `3`、`2`、`1` ಕೂಡ ಇದೇ ರೀತಿಯಲ್ಲಿ ಕ್ರಮವಾಗಿ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತವೆ.

![for](../../Resource/017_for.png)

ಅರೆದಲ್ಲಿನ ಎಲ್ಲಾ ಅಂಶಗಳ ಸಂಸ್ಕರಣೆ ಪೂರ್ಣಗೊಂಡ ಬಳಿಕ, `for` ಲೂಪ್ ಸ್ವಯಂ ಅಂತ್ಯಗೊಳ್ಳುತ್ತದೆ.

ನಾವು `for` ಲೂಪ್ ಸಹಾಯದಿಂದ ಅರೆದಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ಪಡೆಯಬಹುದು, ಮತ್ತು ಕೌಂಟ್‌ಡೌನ್‌ಗಳಂತಹ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಬಹುದು.

**ಸೂಚನೆ**: `i` ಎಂಬುದು for ಲೂಪ್‌ನಲ್ಲಿ ಬಹಳ ಸಾಮಾನ್ಯವಾದ ಹೆಸರಿಡುವ ವಿಧಾನ; ಇದು ಪ್ರಸ್ತುತ ತೆಗೆದುಕೊಳ್ಳಲಾದ ಅಂಶವನ್ನು ಸೂಚಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ. ಆದರೆ ಇದು ಸ್ಥಿರವಾದ ಬರವಣಿಗೆ ಅಲ್ಲ; ಬೇರೆ ಇಂಗ್ಲಿಷ್ ಹೆಸರುಗಳನ್ನು ಕೂಡ ಬಳಸಬಹುದು.

ಉದಾಹರಣೆಗೆ, ಪ್ರತಿಯೊಂದು ತೆಗೆದುಕೊಳ್ಳಲಾದ ಅಂಶವನ್ನು ಸೂಚಿಸಲು `num` ಅನ್ನು ಬಳಸಿ:

```swift
for num in countDown {
    print(num)
}
```

ಕರ್ಳಿ ಬ್ರೇಸ್‌ಗಳೊಳಗೂ, ಪ್ರಸ್ತುತ ತೆಗೆದುಕೊಳ್ಳಲಾದ ಅಂಶವನ್ನು ಸೂಚಿಸಲು `num` ಅನ್ನು ಬಳಸಬೇಕು.

### ಲೂಪ್ ವ್ಯಾಪ್ತಿ

`for` ಲೂಪ್ ಅರೆಗಳನ್ನು ಮಾತ್ರವಲ್ಲ, ವ್ಯಾಪ್ತಿಗಳನ್ನೂ ಲೂಪ್ ಮಾಡಬಹುದು.

ಉದಾಹರಣೆಗೆ, `50` ಒಳಗಿನ ಧನ ಪೂರ್ಣಾಂಕಗಳ ಮೊತ್ತವನ್ನು ಲೆಕ್ಕ ಹಾಕಬೇಕಾದರೆ, `1` ರಿಂದ `50` ವರೆಗೆ ಇರುವ ಎಲ್ಲಾ ಪೂರ್ಣಾಂಕಗಳನ್ನು ಕ್ರಮವಾಗಿ ಸೇರಿಸಬೇಕು.

ಅತ್ಯಂತ ನೇರವಾದ ಬರವಣಿಗೆ ಹೀಗಿರಬಹುದು:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

ಈ ಬರವಣಿಗೆಯ ಸಮಸ್ಯೆ ಏನೆಂದರೆ, ಪ್ರತಿಯೊಂದು ಸಂಖ್ಯೆಯನ್ನೂ ನಾವು ಕೈಯಾರೆ ಬರೆಯಬೇಕು; ಇದು ಕಷ್ಟಕರವೂ ಆಗಿದೆ, ಮತ್ತು ನೈಜ ಅಭಿವೃದ್ಧಿ ಸಂದರ್ಭಗಳಿಗೆ ಸೂಕ್ತವೂ ಅಲ್ಲ.

ಈ ಸಂದರ್ಭದಲ್ಲಿ, ವ್ಯಾಪ್ತಿ ಆಪರೇಟರ್ ಅನ್ನು `for` ಲೂಪ್ ಜೊತೆಗೆ ಬಳಸುವುದರಿಂದ ಇದನ್ನು ಪೂರ್ಣಗೊಳಿಸಬಹುದು.

ಹಿಂದಿನ “ಆಪರೇಟರ್‌ಗಳು” ಅಧ್ಯಾಯದಲ್ಲಿ, ನಾವು ವ್ಯಾಪ್ತಿ ಆಪರೇಟರ್ ಅನ್ನು ಈಗಾಗಲೇ ಕಲಿತಿದ್ದೇವೆ:

```swift
1...3   // 1, 2, 3
```

ಇಲ್ಲಿನ `...` ಮುಚ್ಚಿದ ವ್ಯಾಪ್ತಿಯನ್ನು ಸೂಚಿಸುತ್ತದೆ, ಅಂದರೆ ಪ್ರಾರಂಭ ಬಿಂದು ಮತ್ತು ಅಂತ್ಯ ಬಿಂದು ಎರಡನ್ನೂ ಒಳಗೊಂಡಿರುತ್ತದೆ.

ಆದ್ದರಿಂದ, ಇಲ್ಲಿ `1` ರಿಂದ `50` ವರೆಗೆ ಇರುವ ಪೂರ್ಣಾಂಕ ವ್ಯಾಪ್ತಿಯನ್ನು `for` ಲೂಪ್ ಮೂಲಕ ಬಳಸಬಹುದು:

```swift
for i in 1...50 {
    
}
```

ಈ ಸಂಖ್ಯೆಗಳನ್ನು ಸೇರಿಸಬೇಕಾದ್ದರಿಂದ, ಒಟ್ಟು ಮೊತ್ತವನ್ನು ಉಳಿಸಿಕೊಳ್ಳಲು ಇನ್ನೊಂದು ಚರ ಅವಶ್ಯಕ:

```swift
var count = 0
```

ನಂತರ, ಪ್ರತಿಯೊಂದು ಲೂಪ್‌ನಲ್ಲಿ ಪ್ರಸ್ತುತ ತೆಗೆದುಕೊಳ್ಳಲಾದ ಸಂಖ್ಯೆಯನ್ನು `count` ಗೆ ಸೇರಿಸಬೇಕು:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

ಈ ಕೋಡ್‌ನ ಅರ್ಥ:

* `count` ಪ್ರಸ್ತುತ ಒಟ್ಟು ಮೊತ್ತವನ್ನು ಉಳಿಸುತ್ತದೆ
* `for` ಲೂಪ್ `1` ರಿಂದ `50` ವರೆಗೆ ಇರುವ ಪೂರ್ಣಾಂಕಗಳನ್ನು ಕ್ರಮವಾಗಿ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ
* ಪ್ರತಿಯೊಂದು ಬಾರಿ ಒಂದು ಪೂರ್ಣಾಂಕವನ್ನು ತೆಗೆದುಕೊಂಡಾಗ, `count += i` ಒಂದು ಬಾರಿ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ
* ಲೂಪ್ ಮುಗಿದ ನಂತರ, `count` ಒಳಗೆ ಅಂತಿಮ ಮೊತ್ತ ಉಳಿದಿರುತ್ತದೆ

ಈ ರೀತಿಯಾಗಿ, ನಾವು `50` ಒಳಗಿನ ಎಲ್ಲಾ ಧನ ಪೂರ್ಣಾಂಕಗಳ ಮೊತ್ತವನ್ನು ಪಡೆಯಬಹುದು.

ಅದೇ ರೀತಿ, `1` ರಿಂದ `100` ವರೆಗೆ ಇರುವ ಪೂರ್ಣಾಂಕಗಳ ಮೊತ್ತವನ್ನು ಲೆಕ್ಕ ಹಾಕಬೇಕಾದರೆ, ವ್ಯಾಪ್ತಿಯನ್ನು ಮಾತ್ರ ಬದಲಿಸಿದರೆ ಸಾಕು:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` ಲೂಪ್ ಅರೆಗಳನ್ನು ಮಾತ್ರವಲ್ಲ, ಒಂದು ವ್ಯಾಪ್ತಿಯನ್ನೂ ಲೂಪ್ ಮಾಡಬಹುದು. ಡೇಟಾವನ್ನು ಕ್ರಮವಾಗಿ ಪುನರಾವರ್ತಿಸಿ ಸಂಸ್ಕರಿಸುವ ಈ ಸಾಮರ್ಥ್ಯ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಬಹಳ ಸಾಮಾನ್ಯವಾಗಿದೆ.

ಇದು ನಮಗೆ ಬಹಳಷ್ಟು ಪುನರಾವರ್ತಿತ ಕೆಲಸಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಡೆಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ; ಪ್ರತಿಯೊಂದು ಸಂಖ್ಯೆಯ ಸೇರ್ಪಡೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅಥವಾ ಇತರ ಸಮಾನವಾದ ಪುನರಾವರ್ತಿತ ಕಾರ್ಯಗಳನ್ನು ನಾವು ಕೈಯಾರೆ ಬರೆಯಬೇಕಾಗಿಲ್ಲ.

## SwiftUI ಯಲ್ಲಿನ ಲೂಪ್

ಮುಂದೆ ನೋಡಿದ `for` ಲೂಪ್ Swift ಭಾಷೆಗೆ ಸೇರಿದ್ದು.

ಆದರೆ SwiftUI ಯಲ್ಲಿ, ದೃಶ್ಯ ರಚನೆ ಒಳಗೆ ನಾವು ನೇರವಾಗಿ `for` ಅನ್ನು ಬಳಸಿ view ಅನ್ನು ರಚಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

ಇದು ಯಾಕೆಂದರೆ `VStack` ಒಳಗೆ ಬೇಕಾಗಿರುವುದು SwiftUI views; ಸಾಮಾನ್ಯ `for` ಲೂಪ್ ಸ್ವತಃ ಒಂದು view ಅಲ್ಲ.

ನಾವು SwiftUI ಯಲ್ಲಿ ಅರೆದಲ್ಲಿನ ವಿಷಯದ ಆಧಾರದ ಮೇಲೆ ಇಂಟರ್‌ಫೇಸ್ ಅನ್ನು ಪುನರಾವರ್ತಿಸಿ ರಚಿಸಬೇಕಾದರೆ, `ForEach` ಅನ್ನು ಬಳಸಬೇಕು.

ಉದಾಹರಣೆಗೆ, ನಾವು ಒಂದು ಗುಂಪಿನ ಎಲ್ಲಾ ಸದಸ್ಯರ ಹೆಸರುಗಳನ್ನು ತೋರಿಸಬೇಕೆಂದರೆ, ಹೀಗೆ ಬರೆಯಬಹುದು:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

ತೋರಿಸುವ ಪರಿಣಾಮ:

![for1](../../Resource/017_for1.png)

ನಂತರ, ಪ್ರತಿಯೊಂದು `Text` ಗೆ ಒಂದೇ modifiers ಅನ್ನು ಸೇರಿಸೋಣ:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ಈಗ, ಪ್ರತಿಯೊಂದು `Text` ಕೂಡ ಇನ್ನಷ್ಟು ಚಿಕ್ಕದು ಮತ್ತು ದಪ್ಪವಾದ ಅಕ್ಷರ ರೂಪದಲ್ಲಿ ತೋರುತ್ತದೆ.

ತೋರಿಸುವ ಪರಿಣಾಮ:

![](../../Resource/017_for2.png)

modifiers ಇಂಟರ್‌ಫೇಸ್ ಅನ್ನು ಹೆಚ್ಚು ಸುಂದರವಾಗಿಸಿದರೂ, ಕೋಡ್ ಅನ್ನು ಇನ್ನಷ್ಟು ಉದ್ದವಾಗಿಯೂ, ಪುನರಾವರ್ತಿತವಾಗಿಯೂ ಮಾಡುತ್ತವೆ.

ಹೆಸರುಗಳು ಇನ್ನಷ್ಟು ಹೆಚ್ಚಾದರೆ, ಅಥವಾ ನಂತರ font, color ಮುಂತಾದ style‌ಗಳನ್ನು ಒಟ್ಟಾಗಿ ತಿದ್ದುಪಡಿ ಮಾಡಬೇಕಾದರೆ, ಅನೇಕ ಒಂದೇ ರೀತಿಯ ಕೋಡ್‌ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ಬರೆಯಬೇಕಾಗುತ್ತದೆ ಮತ್ತು ಬದಲಾಯಿಸಬೇಕಾಗುತ್ತದೆ; ಇದು ಕಷ್ಟಕರವೂ ಆಗುತ್ತದೆ, ನಿರ್ವಹಣೆಯೂ ಸುಲಭವಿರುವುದಿಲ್ಲ.

ಈ ಸಂದರ್ಭದಲ್ಲಿ, ನಾವು ಮೊದಲು ಈ ಹೆಸರುಗಳನ್ನು ಅರೆದಲ್ಲಿ ಇಟ್ಟು, ನಂತರ SwiftUI ಯ `ForEach` ಬಳಸಿ ಅರೆದಲ್ಲಿನ ಡೇಟಾ ಆಧಾರಿತವಾಗಿ view ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ರಚಿಸಬಹುದು:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

ಈ ಕೋಡ್ ಗುಂಪು ಹೆಚ್ಚು ಸರಳವಾಗಿದೆ; `ForEach` ಅರೆದಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ಕ್ರಮವಾಗಿ ಓದಿ, ಸಂಬಂಧಿತ ವಿಷಯವನ್ನು ಇಂಟರ್‌ಫೇಸ್‌ನಲ್ಲಿ ತೋರಿಸುತ್ತದೆ.

ನಂತರ ನಾವು ಹೊಸ ಹೆಸರನ್ನು ಸೇರಿಸಬೇಕಾದರೆ, `names` ಅರೆದಲ್ಲಿನ ಹೆಸರನ್ನೇ ಬದಲಿಸಿದರೆ ಸಾಕು; ಪ್ರತಿ ಬಾರಿ `Text` ಮತ್ತು ಅದರ ಸಂಕೀರ್ಣ modifiers ಗಳನ್ನು ಮರು ಸೇರಿಸುವ ಅಗತ್ಯವಿಲ್ಲ.

## ForEach

`ForEach` ಎಂಬುದು SwiftUI ಯಲ್ಲಿ views ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ರಚಿಸಲು ಬಳಸುವ ಒಂದು ರಚನೆ. ಇದನ್ನು SwiftUI ಯಲ್ಲಿನ `for` ಲೂಪ್‌ಗೆ ಹೋಲುವ ಒಂದು ರಚನೆ ಎಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು.

ಮೂಲಭೂತ ಬಳಕೆ:

```swift
ForEach(ಅರೆ, id: \.self) { ಅಂಶ in
    SwiftUI ಕೋಡ್
}
```

ಉದಾಹರಣೆಗೆ, ಈಗಷ್ಟೇ ನೋಡಿದ ಗುಂಪಿನ ಹೆಸರುಗಳು:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ಈ ಕೋಡ್‌ನ ಅರ್ಥ:

`names` ಎಂಬುದು ಲೂಪ್ ಮಾಡಲಾಗುವ ಅರೆ, ಅಂದರೆ ತೋರಿಸಬೇಕಾದ ಡೇಟಾ.

`id:\.self` ಅರೆದಲ್ಲಿನ ಅಂಶವನ್ನೇ ಬಳಸಿಕೊಂಡು ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ವಿಭೇದಿಸುವುದನ್ನು ಸೂಚಿಸುತ್ತದೆ.

ಪ್ರಸ್ತುತ ಉದಾಹರಣೆಯಲ್ಲಿ, ಅರೆವು ಸ್ಟ್ರಿಂಗ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸಿದೆ:

```swift
["Sam", "John", "Wathon", "Bob"]
```

ಆದ್ದರಿಂದ SwiftUI ನೇರವಾಗಿ "Sam"、"John"、"Wathon"、"Bob" ಎಂಬ ಈ ಸ್ಟ್ರಿಂಗ್‌ಗಳನ್ನೇ ಬೇರೆಬೇರೆ ವಿಷಯಗಳನ್ನು ಗುರುತಿಸಲು ಬಳಸುತ್ತದೆ.

ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ಸಾಮಾನ್ಯವಾಗಿರುವ String, Int ಮುಂತಾದ ಸರಳ ಅರೆಗಳಿಗಾಗಿ, ನಾವು ಸಾಮಾನ್ಯವಾಗಿ ಹೀಗೆ ಬರೆಯಬಹುದು:

```swift
id: \.self
```

ಈಗಕ್ಕೆ, ಈ ಒಂದು ಬರವಣಿಗೆಯ ವಿಧಾನವನ್ನು ಮೊದಲು ನೆನಪಿಟ್ಟುಕೊಳ್ಳುವುದೇ ಸಾಕು; `id` ನಿಯಮಗಳ ಬಗ್ಗೆ ಇನ್ನೂ ಆಳವಾಗಿ ತಿಳಿಯಬೇಕಾಗಿಲ್ಲ.

`item in` ಒಳಗಿನ `item` ಪ್ರಸ್ತುತ ತೆಗೆದುಕೊಳ್ಳಲಾದ ಅಂಶವನ್ನು ಸೂಚಿಸುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ಮೊದಲ ಲೂಪ್‌ನಲ್ಲಿ `item` `"Sam"` ಆಗಿರುತ್ತದೆ; ಎರಡನೇ ಲೂಪ್‌ನಲ್ಲಿ `item` `"John"` ಆಗಿರುತ್ತದೆ.

ಆದ್ದರಿಂದ:

```swift
Text(item)
```

ಇದು ಕ್ರಮವಾಗಿ ಹೀಗೆ ಆಗುತ್ತದೆ:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

ಹೀಗಾಗಿ, ಈ ಕೋಡ್‌ನ ಕೆಲಸ ಏನೆಂದರೆ: `names` ಅರೆದಿಂದ ಕ್ರಮವಾಗಿ ಪ್ರತಿಯೊಂದು ಹೆಸರನ್ನೂ ತೆಗೆದು, ಪ್ರತಿಯೊಂದು ಹೆಸರಿಗಾಗಿ ಒಂದು `Text` view ಅನ್ನು ರಚಿಸುವುದು.

### ಕಾರ್ಯಗತ ಪ್ರಕ್ರಿಯೆ

ಕಾರ್ಯಗತ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಹೀಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು:

* `ForEach` ಮೊದಲು `names` ಅರೆದಲ್ಲಿನ ಮೊದಲ ಅಂಶ `"Sam"` ಅನ್ನು ಓದುತ್ತದೆ
* `item` ತಾತ್ಕಾಲಿಕವಾಗಿ `"Sam"` ಅನ್ನು ಸೂಚಿಸುತ್ತದೆ
* `Text(item)` `"Sam"` ಅನ್ನು ತೋರಿಸುತ್ತದೆ
* ನಂತರ ಮುಂದಿನ ಅಂಶ `"John"` ಅನ್ನು ಓದುತ್ತದೆ
* ಮತ್ತೆ ಅದೇ view ಕೋಡ್ ಅನ್ನು ಪುನರಾವರ್ತಿಸಿ ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತದೆ

ಕೊನೆಯಲ್ಲಿ, ಅರೆದಲ್ಲಿನ ಎಲ್ಲಾ ಹೆಸರುಗಳೂ ತೋರಿಸಲಾಗುತ್ತವೆ.

ಈ ರೀತಿಯಲ್ಲಿ ಬರೆಯುವುದರಿಂದ ದೊರೆಯುವ ಪ್ರಯೋಜನವೆಂದರೆ: ಹೆಸರುಗಳು ಹೆಚ್ಚಾದಾಗ, ನಾವು ಅನೇಕ `Text` ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ಬರೆಯಬೇಕಾಗುವುದಿಲ್ಲ; ಅರೆದಲ್ಲಿನ ವಿಷಯವನ್ನು ಮಾತ್ರ ಬದಲಿಸಿದರೆ ಸಾಕು.

### ಫೋಟೋಗಳನ್ನು ಲೂಪ್ ಮಾಡುವುದು

`ForEach` ಕೇವಲ ಪಠ್ಯವನ್ನೇ ಅಲ್ಲ, ಚಿತ್ರಗಳು, ಬಣ್ಣಗಳು, ಅಥವಾ ಇತರ SwiftUI views ಗಳನ್ನೂ ತೋರಿಸಬಹುದು.

ಉದಾಹರಣೆಗೆ, ನಾವು 4 ಚಿತ್ರಗಳನ್ನು ಸಿದ್ಧಪಡಿಸಿದ್ದೇವೆ:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

ಈ ನಾಲ್ಕು ಚಿತ್ರಗಳನ್ನು Xcode ನ Assets ಫೋಲ್ಡರ್‌ಗೆ drag ಮಾಡಿ.

![image](../../Resource/017_for3.png)

`ForEach` ಬಳಸದೆ ಇದ್ದರೆ, ನಾವು ಪ್ರತಿಯೊಂದು ಚಿತ್ರವನ್ನೂ ಕೈಯಾರೆ ಬರೆಯಬೇಕಾಗುತ್ತದೆ:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

ತೋರಿಸುವ ಪರಿಣಾಮ:

![image](../../Resource/017_for4.png)

ಈ ಬರವಣಿಗೆಯ ವಿಧಾನದಿಂದ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಸಾಕಾರಗೊಳಿಸಬಹುದು, ಆದರೆ ಕೋಡ್ ತುಸು ಉದ್ದವಾಗಿರುತ್ತದೆ.

ಚಿತ್ರಗಳು ಹೆಚ್ಚು ಹೆಚ್ಚು ಆಗುತ್ತಿದ್ದಂತೆ, ನಾವು ಹೆಚ್ಚು ಹೆಚ್ಚು `Image` ಕೋಡ್ ಅನ್ನು ಪುನರಾವರ್ತಿಸಿ ಬರೆಯಬೇಕಾಗುತ್ತದೆ.

ನಂತರ ಎಲ್ಲಾ ಚಿತ್ರಗಳ ಗಾತ್ರವನ್ನು ಒಟ್ಟಾಗಿ ಬದಲಾಯಿಸಬೇಕಾದರೂ, ಒಂದೊಂದಾಗಿ ತಿದ್ದುಪಡಿ ಮಾಡಬೇಕಾಗುತ್ತದೆ.

ಈ ಸಂದರ್ಭದಲ್ಲಿ, ನಾವು ಚಿತ್ರಗಳ ಹೆಸರನ್ನು ಅರೆದಲ್ಲಿ ಇಟ್ಟು, ನಂತರ `ForEach` ಬಳಸಿ ಚಿತ್ರ view ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ರಚಿಸಬಹುದು:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

ಇಲ್ಲಿನ `images` ಒಂದು ಸ್ಟ್ರಿಂಗ್ ಅರೆ; ಅರೆದಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಅಂಶವೂ ಒಂದು ಚಿತ್ರದ ಹೆಸರು.

`ForEach(images, id: \.self)` ಅಂದರೆ, `images` ಅರೆದಲ್ಲಿನ ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ಕ್ರಮವಾಗಿ ಓದಿ, ಅಂಶವನ್ನೇ ಬಳಸಿಕೊಂಡು ಪ್ರತಿಯೊಂದು ಐಟಂ ಅನ್ನು ವಿಭೇದಿಸಿ, ನಂತರ ಪ್ರಸ್ತುತ ಅಂಶದ ಆಧಾರದ ಮೇಲೆ ಸಂಬಂಧಿತ view ಅನ್ನು ರಚಿಸುವುದು.

ಉದಾಹರಣೆಗೆ, ಮೊದಲ ಬಾರಿ ಓದಲಾಗುವುದು `"430F9BEF"` ಆಗಿದ್ದರೆ, ಆಗ:

```swift
Image(item)
```

ಇದು ಹೀಗೇ ಸಮಾನವಾಗುತ್ತದೆ:

```swift
Image("430F9BEF")
```

ಮುಂದಿನ ಚಿತ್ರಗಳೂ ಇದೇ ರೀತಿಯಲ್ಲಿ ಕ್ರಮವಾಗಿ ತೋರಿಸಲಾಗುತ್ತವೆ.

![image](../../Resource/017_for4.png)

ಇಲ್ಲಿಯೂ ಸಹ ನಾವು ಬಳಸುತ್ತಿರುವುದು:

```swift
id: \.self
```

ಕಾರಣವೂ ಹಿಂದಿನಂತೆಯೇ, ಏಕೆಂದರೆ `images` ಅರೆದಲ್ಲಿನ ಅಂಶಗಳೂ ಸರಳವಾದ ಸ್ಟ್ರಿಂಗ್ ಪ್ರಕಾರವೇ ಆಗಿವೆ; ಆದ್ದರಿಂದ ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ಅಂಶವನ್ನೇ ವಿಭೇದಿಸುವ ವಿಧಾನವಾಗಿ ನೇರವಾಗಿ ಬಳಸಬಹುದು.

ಇದನ್ನು ಸರಳವಾಗಿ ಒಂದು ಸ್ಥಿರ ಸೂತ್ರವೆಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು:

**`ForEach` ಲೂಪ್ ಮಾಡುವದ್ದು `String` ಅಥವಾ `Int` ಮೊದಲಾದ ಸರಳ ಅರೆಗಳಾಗಿದ್ದರೆ, ಸಾಮಾನ್ಯವಾಗಿ `id: \.self` ಎಂದು ಬರೆಯಲಾಗುತ್ತದೆ.**

### ಹೊಸ ಚಿತ್ರಗಳನ್ನು ಸೇರಿಸುವುದು

ನಂತರ ಹೊಸ ಚಿತ್ರಗಳನ್ನು ಸೇರಿಸಬೇಕಾದರೆ, ಅರೆದಲ್ಲಿನ ವಿಷಯವನ್ನು ಮಾತ್ರ ಬದಲಿಸಿದರೆ ಸಾಕು:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

ಹೊಸ `Image` ಕೋಡ್ ಅನ್ನು ಮರು ಬರೆಯುವ ಅಗತ್ಯವಿಲ್ಲ.

ಚಿತ್ರಗಳು ಹೆಚ್ಚು ಇದ್ದರೆ, `ScrollView` ಜೊತೆಗೂಡಿ ಬಳಸಬಹುದು; ಇಲ್ಲವಾದರೆ ವಿಷಯವು ಪರದೆಯ ಮಿತಿಯನ್ನು ಮೀರುತ್ತದೆ ಮತ್ತು ಪೂರ್ಣವಾಗಿ ಪೂರ್ವದರ್ಶನ ಮಾಡಲು ಸಾಧ್ಯವಾಗುವುದಿಲ್ಲ.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### ಚಿತ್ರಗಳನ್ನು ತಿದ್ದುಪಡಿ ಮಾಡುವುದು

ನಾವು ಎಲ್ಲ ಚಿತ್ರಗಳ ಶೈಲಿಯನ್ನು ಒಟ್ಟಿಗೆ ತಿದ್ದುಪಡಿ ಮಾಡಬೇಕಾದರೆ, `ForEach` ಒಳಗಿನ ಕೋಡ್ ಅನ್ನು ಮಾತ್ರ ಬದಲಿಸಿದರೆ ಸಾಕು.

ಉದಾಹರಣೆಗೆ, ಅಗಲವನ್ನು `80`ಕ್ಕೆ ಬದಲಿಸುವುದು:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

ಈ ರೀತಿ, ಎಲ್ಲಾ ಚಿತ್ರಗಳೂ ಒಂದೇ ವೇಳೆ ಬದಲಾಗುತ್ತವೆ; ಒಂದೊಂದಾಗಿ ತಿದ್ದುಪಡಿ ಮಾಡುವ ಅಗತ್ಯವಿಲ್ಲ.

ಇದೇ `ForEach` ನ ಒಂದು ಪ್ರಮುಖ ಪಾತ್ರ:

**ಅನೇಕ views ಗಳ ರಚನೆ ಒಂದೇ ಆಗಿದ್ದು, ವಿಷಯ ಮಾತ್ರ ಭಿನ್ನವಾಗಿದ್ದರೆ, ಒಂದು view ಕೋಡ್ ಮಾತ್ರ ಬರೆದು, ನಂತರ ಅರೆದಲ್ಲಿನ ಡೇಟಾ ಯಾವ ವಿಷಯ ತೋರಿಸಬೇಕು ಎಂಬುದನ್ನು ನಿರ್ಧರಿಸಲು ಬಿಡಬಹುದು.**

## ಸಾರಾಂಶ

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು `for` ಲೂಪ್ ಮತ್ತು `ForEach` ಅನ್ನು ಕಲಿತೆವು.

`for` ಲೂಪ್ ಅರೆ ಅಥವಾ ವ್ಯಾಪ್ತಿಯಲ್ಲಿನ ಅಂಶಗಳನ್ನು ಕ್ರಮವಾಗಿ ಸಂಸ್ಕರಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ; ಇದು ಹೆಚ್ಚು ಡೇಟಾ ಸಂಸ್ಕರಣೆಯ ಕಡೆ ಒಲವು ಹೊಂದಿರುತ್ತದೆ.

`ForEach` ಅನ್ನು SwiftUI ಯಲ್ಲಿ ಅರೆದಲ್ಲಿನ ವಿಷಯದ ಆಧಾರದ ಮೇಲೆ views ಗಳನ್ನು ಪುನರಾವರ್ತಿಸಿ ರಚಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ; ಇದು ಹೆಚ್ಚು view ಪ್ರದರ್ಶನದ ಕಡೆ ಒಲವು ಹೊಂದಿದೆ.

ಪ್ರಸ್ತುತ ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ, `ForEach` ನ ಅತ್ಯಂತ ಸಾಮಾನ್ಯ ಬರವಣಿಗೆಯ ವಿಧಾನ ಹೀಗಿದೆ:

```swift
ForEach(ಅರೆ, id: \.self) { ಅಂಶ in
    SwiftUI ಕೋಡ್
}
```

ಇಲ್ಲಿನ `id: \.self` ಅಂದರೆ ಅಂಶವನ್ನೇ ಬಳಸಿಕೊಂಡು ಪ್ರತಿಯೊಂದು ಅಂಶವನ್ನು ವಿಭೇದಿಸುವುದು.

ಅರೆದಲ್ಲಿನ ವಿಷಯವು `String`、`Int` ಮೊದಲಾದ ಸರಳ ಪ್ರಕಾರಗಳಾಗಿದ್ದರೆ, ಸಾಮಾನ್ಯವಾಗಿ ನಾವು ಮೊದಲು ಇದೇ ಒಂದು ಬರವಣಿಗೆಯ ವಿಧಾನವನ್ನು ಬಳಸಬಹುದು.

ಮುಂದೆ, ವೆಬ್‌ಪುಟ ಅಥವಾ App ನೋಡುತ್ತಿರುವಾಗ, ಒಂದು ಗುಂಪಿನ ಪುನರಾವರ್ತಿತ ಪಠ್ಯ, ಚಿತ್ರಗಳು, ಅಥವಾ ಇತರ ಪಟ್ಟಿಯ ವಿಷಯಗಳನ್ನು ಕಂಡರೆ, SwiftUI ಯಲ್ಲಿ ಇಂತಹ ವಿಷಯಗಳನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಮೊದಲು ಅರೆದಲ್ಲಿ ಇಟ್ಟು, ನಂತರ `ForEach` ಮೂಲಕ ಕ್ರಮವಾಗಿ ತೋರಿಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ನೆನಪಿಸಿಕೊಳ್ಳಬಹುದು.
