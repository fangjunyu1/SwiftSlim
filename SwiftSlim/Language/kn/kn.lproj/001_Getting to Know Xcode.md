# Xcode ಪರಿಚಯ

## Xcode ಎಂದರೆ ಏನು?

Xcode ಎನ್ನುವುದು Apple ಅಧಿಕೃತವಾಗಿ ಒದಗಿಸುವ ಸಂಯೋಜಿತ ಅಭಿವೃದ್ಧಿ ಪರಿಸರ (IDE, Integrated Development Environment), ಮತ್ತು ಇದು iOS, macOS, watchOS, tvOS ಮುಂತಾದ Apple ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ.

![Swift](../../RESOURCE/001_xcode.png)

ಪೂರ್ತಿ ಕಲಿಕೆ ಪ್ರಕ್ರಿಯೆಯಲ್ಲೂ ನಾವು ಕೋಡ್ ಬರೆಯಲು ಮತ್ತು ಪ್ರೋಗ್ರಾಂ ಚಾಲನೆಗೊಳಿಸಲು Xcode ಅನ್ನು ಬಳಸುತ್ತೇವೆ.

ಈ ಪಾಠದಲ್ಲಿ ನಾವು Xcode ಅನ್ನು ಹೇಗೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡುವುದು, ಮೊದಲ iOS ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಹೇಗೆ ರಚಿಸುವುದು, ಮತ್ತು Xcode ಇಂಟರ್ಫೇಸ್‌ನ ಮೂಲಭೂತ ರಚನೆಯನ್ನು ಹೇಗೆ ಗುರುತಿಸುವುದು ಎಂಬುದನ್ನು ಕಲಿಯುತ್ತೇವೆ.

## Xcode ಡೌನ್‌ಲೋಡ್ ಮಾಡುವುದು

ಈಗ Xcode ಅನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಲು ಎರಡು ವಿಧಾನಗಳಿವೆ:

1、[Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12) ಮೂಲಕ ಡೌನ್‌ಲೋಡ್ ಮಾಡುವುದು

2、[Apple Developer ಅಧಿಕೃತ ವೆಬ್‌ಸೈಟ್](https://developer.apple.com/xcode/) ಮೂಲಕ ಡೌನ್‌ಲೋಡ್ ಮಾಡುವುದು

ಎರಡೂ ವಿಧಾನಗಳಿಂದ Xcode ಅನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಬಹುದು. ವ್ಯತ್ಯಾಸವೆಂದರೆ Apple Developer ಅಧಿಕೃತ ತಾಣದಲ್ಲಿ ಸ್ಥಿರ ಆವೃತ್ತಿಯ ಜೊತೆಗೆ [Beta](https://developer.apple.com/download/all/?q=Xcode) ಆವೃತ್ತಿ ಅಥವಾ ಹಳೆಯ ಆವೃತ್ತಿಗಳನ್ನು ಸಹ ಪಡೆಯಬಹುದು.

ನೀವು ಕೇವಲ ಕಲಿಕೆಯ ಉದ್ದೇಶಕ್ಕೆ ಬಳಸುತ್ತಿದ್ದರೆ, ಅಧಿಕೃತ ಸ್ಥಿರ ಆವೃತ್ತಿಯನ್ನು ಬಳಸುವುದು ಉತ್ತಮ. Beta ಆವೃತ್ತಿ ಮುಖ್ಯವಾಗಿ ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ, ಮತ್ತು ಅದು ಸ್ಥಿರವಾಗಿರದೇ ಇರಬಹುದು.

## ಮೊದಲ ಪ್ರಾಜೆಕ್ಟ್ ನಿರ್ಮಾಣ

Xcode ತೆರೆಯುತ್ತಿದ್ದಂತೆ, ಸ್ಟಾರ್ಟ್ ಸ್ಕ್ರೀನ್ ಕಾಣುತ್ತದೆ.

![Swift](../../RESOURCE/001_xcode1.png)

ಎಡಭಾಗದಲ್ಲಿ ಮೂರು ಸಾಮಾನ್ಯ ಆಯ್ಕೆಗಳು ಇರುತ್ತವೆ:

- Create New Project —— ಹೊಸ ಪ್ರಾಜೆಕ್ಟ್ ರಚಿಸು  
- Clone Git Repository —— Git ರಿಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡು  
- Open Existing Project —— ಈಗಿರುವ ಪ್ರಾಜೆಕ್ಟ್ ತೆರೆಯು

ಬಲಭಾಗದಲ್ಲಿ ಇತ್ತೀಚೆಗೆ ತೆರೆಯಲಾದ Xcode ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು ಕಾಣುತ್ತವೆ.

ನಾವು `"Create New Project"` ಆಯ್ಕೆಯನ್ನು ಆರಿಸುತ್ತೇವೆ.

### ಪ್ರಾಜೆಕ್ಟ್ ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ

![Swift](../../RESOURCE/001_xcode2.png)

ಪ್ರಾಜೆಕ್ಟ್ ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ ಪರದೆಗೆ ಬಂದಾಗ, ಇಲ್ಲಿ ಹಲವು ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳು (iOS, macOS ಮುಂತಾದವು) ಮತ್ತು ಬೇರೆ ಬೇರೆ ಪ್ರಾಜೆಕ್ಟ್ ಟೆಂಪ್ಲೇಟ್ ಪ್ರಕಾರಗಳು ಕಾಣುತ್ತವೆ.

ಸಾಮಾನ್ಯ ಟೆಂಪ್ಲೇಟ್ ವಿವರಣೆ:
- App —— ಅತ್ಯಂತ ಮೂಲಭೂತ ಅಪ್ಲಿಕೇಶನ್ ಪ್ರಕಾರ (ಶಿಫಾರಸು)
- Document App —— ಡಾಕ್ಯುಮೆಂಟ್ ಮಾದರಿಯ ಅಪ್ಲಿಕೇಶನ್
- Game —— ಗೇಮ್ ಅಭಿವೃದ್ಧಿಗಾಗಿ
- Framework —— ಮರುಬಳಕೆ ಮಾಡಬಹುದಾದ ಮಾದರಿ

ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ನಾವು ಕೇವಲ `"iOS" - "App"` ಟೆಂಪ್ಲೇಟ್ ಅನ್ನು ಮಾತ್ರ ಆರಿಸುತ್ತೇವೆ, ಏಕೆಂದರೆ ಇದು ಇಂಟರ್ಫೇಸ್ ಇರುವ ಅಪ್ಲಿಕೇಶನ್ ಪ್ರಕಾರವಾಗಿದೆ.

`"Next"` ಕ್ಲಿಕ್ ಮಾಡಿ.

### ಪ್ರಾಜೆಕ್ಟ್ ಮಾಹಿತಿ ತುಂಬುವುದು

ಈಗ ಪ್ರಾಜೆಕ್ಟ್ ಮಾಹಿತಿ ತುಂಬುವ ಪರದೆಗೆ ಬರುತ್ತೇವೆ. ಇಲ್ಲಿ ಪ್ರತಿ ಫೀಲ್ಡ್‌ನ ಪ್ರಕಾರ ಸೂಕ್ತ ಮಾಹಿತಿಯನ್ನು ನಮೂದಿಸಬೇಕು.

![Swift](../../RESOURCE/001_xcode3.png)

ಪ್ರತಿ ಫೀಲ್ಡ್‌ನ ಅರ್ಥ:

- Product Name —— ಪ್ರಾಜೆಕ್ಟ್ ಹೆಸರು, ಉದಾಹರಣೆ: `SwiftSlimTest`
- Team —— ಡೆವಲಪರ್ ತಂಡ; ನೀವು ಪೇಡ್ Apple Developer ಖಾತೆ ಹೊಂದಿರದಿದ್ದರೆ, ಇದನ್ನು ಖಾಲಿ ಬಿಡಬಹುದು
- Organization Identifier —— ಸಂಸ್ಥೆಯ ಗುರುತು; ಸಾಮಾನ್ಯವಾಗಿ reverse domain ರೂಪವನ್ನು ಬಳಸಲಾಗುತ್ತದೆ, ಉದಾಹರಣೆಗೆ:
  - `com.yourname`
  - `com.yourcompany`
  - ನಿಮ್ಮ ಬಳಿ domain ಇಲ್ಲದಿದ್ದರೆ, ನಿಮ್ಮ ಇಂಗ್ಲಿಷ್ ಹೆಸರನ್ನೇ ಗುರುತಾಗಿ ಬಳಸಬಹುದು
- Bundle Identifier —— ಅಪ್ಲಿಕೇಶನ್‌ನ ವಿಶಿಷ್ಟ ಗುರುತು, ಇದು Organization Identifier ಮತ್ತು Product Name ಆಧರಿಸಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸೃಷ್ಟಿಯಾಗುತ್ತದೆ
- Interface —— ಇಂಟರ್ಫೇಸ್ ತಂತ್ರಜ್ಞಾನ, SwiftUI ಆಯ್ಕೆಮಾಡಿ
- Language —— ಅಭಿವೃದ್ಧಿ ಭಾಷೆ, Swift ಆಯ್ಕೆಮಾಡಿ
- Testing System —— ಡೀಫಾಲ್ಟ್ ಆಗಿ test target (Unit Tests) ರಚಿಸುತ್ತದೆ, ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ಇದನ್ನು ಕಡೆಗಣಿಸಬಹುದು
- Storage —— ಲೋಕಲ್ ಡೇಟಾ ಪರ್ಸಿಸ್ಟೆನ್ಸ್ ಫ್ರೇಮ್‌ವರ್ಕ್, ಇದು SwiftData ಅಥವಾ Core Data ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸೇರಿಸಬಹುದು; ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ಇದನ್ನೂ ಕಡೆಗಣಿಸಬಹುದು

ಎಲ್ಲ ಮಾಹಿತಿಯನ್ನು ತುಂಬಿದ ನಂತರ `"Next"` ಕ್ಲಿಕ್ ಮಾಡಿ.

### ಪ್ರಾಜೆಕ್ಟ್ ಸೇವ್ ಮಾಡುವುದು

![Swift](../../RESOURCE/001_xcode4.png)

ಉಚಿತವಾದ ಒಂದು ಫೋಲ್ಡರ್ ಆಯ್ಕೆ ಮಾಡಿ ಸೇವ್ ಮಾಡಿ.

`"Create"` ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, Xcode ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಯನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ.

## Xcode ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆ

Xcode ಪ್ರಾಜೆಕ್ಟ್ ಸೇವ್ ಆದ ನಂತರ, Finder ನಲ್ಲಿ ರಚಿಸಲಾದ ಫೋಲ್ಡರ್ ಕಾಣುತ್ತದೆ.

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets —— ಚಿತ್ರಗಳು, App ಐಕಾನ್‌ಗಳು ಮತ್ತು ಇತರ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಇಡುವ ಫೋಲ್ಡರ್

- ContentView.swift —— ಡೀಫಾಲ್ಟ್ ಆಗಿ ರಚಿಸಲಾದ SwiftUI view ಫೈಲ್; ಮುಂದಿನ ಹಂತಗಳಲ್ಲಿ ನಾವು ಇಂಟರ್ಫೇಸ್ ಕೋಡ್ ಅನ್ನು ಇಲ್ಲಿ ಬರೆಯುತ್ತೇವೆ

- SwiftSlimTestApp.swift —— ಅಪ್ಲಿಕೇಶನ್ ಪ್ರವೇಶ ಫೈಲ್ (App Entry Point), ಇದು ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆಗೆ ಹೊಣೆಗಾರ; ತಾತ್ಕಾಲಿಕವಾಗಿ ಇದನ್ನು ಆಳವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ಅಗತ್ಯವಿಲ್ಲ

- SwiftSlimTest.xcodeproj —— Xcode ಪ್ರಾಜೆಕ್ಟ್ ಫೈಲ್; ಇದನ್ನು ಡಬಲ್ ಕ್ಲಿಕ್ ಮಾಡಿದರೆ ಪ್ರಾಜೆಕ್ಟ್ ತೆರೆಯಬಹುದು

## Xcode ಇಂಟರ್ಫೇಸ್ ಪರಿಚಯ

`.xcodeproj` ಫೈಲ್ ಅನ್ನು ಡಬಲ್ ಕ್ಲಿಕ್ ಮಾಡಿ ಪ್ರಾಜೆಕ್ಟ್ ತೆರೆಯಿರಿ, ನಂತರ ಎಡಭಾಗದ Navigator ಪ್ರದೇಶದಲ್ಲಿ `ContentView.swift` ಫೈಲ್ ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿ.

Xcode ಇಂಟರ್ಫೇಸ್ ಮುಖ್ಯವಾಗಿ ಐದು ಭಾಗಗಳಿಂದ ಕೂಡಿದೆ:

- Navigator —— ಫೈಲ್ ರಚನೆ ನೋಡಲು ಮತ್ತು ಹುಡುಕಲು ಬಳಸುವ ನ್ಯಾವಿಗೇಶನ್ ಪ್ರದೇಶ
- Editor —— Swift ಅಥವಾ SwiftUI ಕೋಡ್ ಬರೆಯುವ ಸಂಪಾದನಾ ಪ್ರದೇಶ
- Canvas —— SwiftUI ಪ್ರಿವ್ಯೂಗಾಗಿ ಬಳಸುವ ಕ್ಯಾನ್ವಾಸ್ ಪ್ರದೇಶ
- Inspector —— ಫೈಲ್ ಗುಣಲಕ್ಷಣಗಳನ್ನು ವೀಕ್ಷಿಸಲು ಮತ್ತು ತಿದ್ದುಪಡಿ ಮಾಡಲು ಬಳಸುವ ಪರಿಶೀಲನಾ ಪ್ರದೇಶ
- Debug Area —— ಲಾಗ್ ಔಟ್‌ಪುಟ್ ನೋಡಲು ಬಳಸುವ ಡಿಬಗ್ ಪ್ರದೇಶ

![Swift](../../RESOURCE/001_xcode6.png)

ಗಮನಿಸಿ: Canvas ಮುಖ್ಯವಾಗಿ SwiftUI view ಪ್ರಿವ್ಯೂಗಾಗಿ ಬಳಸಲಾಗುತ್ತದೆ. Xcode ಅಧಿಕೃತ ಆರ್ಕಿಟೆಕ್ಚರ್ ದೃಷ್ಟಿಯಿಂದ, ಇದು Editor ಗೆ ಸಹಾಯಕ Preview Pane ಆಗಿದೆ. SwiftUI view ಫೈಲ್ ತೆರೆಯಲಾಗಿದ್ದಾಗ ಮಾತ್ರ Canvas ಬಳಸಬಹುದು; ಇತರ ಫೈಲ್ ಪ್ರಕಾರಗಳಿಗೆ ಪ್ರಿವ್ಯೂ ಸೌಲಭ್ಯ ಇರುವುದಿಲ್ಲ.

ಮುಂದಿನ ಪಾಠಗಳಲ್ಲಿ ನಾವು ಮುಖ್ಯವಾಗಿ Editor ಪ್ರದೇಶದಲ್ಲಿ Swift ಮತ್ತು SwiftUI ಕೋಡ್ ಬರೆಯುತ್ತೇವೆ.

ಮೊದಲ ಬಾರಿ Xcode ಪ್ರಾಜೆಕ್ಟ್ ತೆರೆಯುವಾಗ Inspector ಮತ್ತು Debug Area ಡೀಫಾಲ್ಟ್ ಆಗಿ ಮರೆಮಾಡಲ್ಪಟ್ಟಿರಬಹುದು. ಅವನ್ನು ಮೇಲಿನ ಬಲ ಕೋನ ಮತ್ತು ಕೆಳಗಿನ ಬಲ ಕೋನದ ಬಟನ್‌ಗಳ ಮೂಲಕ ತೋರಿಸಬಹುದು.

![Swift](../../RESOURCE/001_xcode7.png)

ಸೂಚನೆ: Inspector ಪ್ರದೇಶವು ಮುಖ್ಯವಾಗಿ ಫೈಲ್ ಗುಣಲಕ್ಷಣಗಳನ್ನು ವೀಕ್ಷಿಸಲು ಮತ್ತು ಬದಲಾಯಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ. ನೈಜ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ, ಹೆಚ್ಚುವರಿ ಎಡಿಟಿಂಗ್ ಸ್ಪೇಸ್ ಪಡೆಯಲು ಸಾಮಾನ್ಯವಾಗಿ ಈ ಭಾಗವನ್ನು ಮರೆಮಾಡಲಾಗುತ್ತದೆ.

## ಸಾರಾಂಶ

ಈ ಪಾಠದಲ್ಲಿ ನಾವು Xcode ಅನ್ನು ಹೇಗೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡುವುದು, ಮೊದಲ Xcode ಪ್ರಾಜೆಕ್ಟ್ ಹೇಗೆ ರಚಿಸುವುದು, ಮತ್ತು Xcode ಇಂಟರ್ಫೇಸ್‌ನ ಮೂಲಭೂತ ವಿನ್ಯಾಸವನ್ನು ಕಲಿತೇವೆ.

ಈಗ ಸ್ವತಂತ್ರ ಅಭ್ಯಾಸ ಸಮಯ:
- Navigator ಪ್ರದೇಶದಲ್ಲಿ ಫೈಲ್ ರಚಿಸಿ ಅಥವಾ ಅಳಿಸಿ ನೋಡಿ  
- Editor ಪ್ರದೇಶದ ಕೋಡ್ ನೋಡಿ, ಅದರ ರಚನೆಯನ್ನು ಗಮನಿಸಿ
- Canvas ಪ್ರದೇಶದಲ್ಲಿನ ಬಟನ್‌ಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ, ಪ್ರಿವ್ಯೂ ಅನ್ನು zoom in / zoom out ಮಾಡಿ ನೋಡಿ

ಮುಂದಿನ ಪಾಠದಲ್ಲಿ ನಾವು ಕೆಲವು ಸರಳ ಕೋಡ್ ಬರೆಯಲು ಪ್ರಾರಂಭಿಸುತ್ತೇವೆ, ಇದರಿಂದ Xcode ಕುರಿತು ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟ ಅರಿವು ಬರುತ್ತದೆ。
