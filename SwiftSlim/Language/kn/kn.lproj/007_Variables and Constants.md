# ವ್ಯೇರಿಯಬಲ್‌ಗಳು ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್‌ಗಳು

ಈ ಪಾಠದಲ್ಲಿ ನಾವು Swift ನಲ್ಲಿರುವ ವ್ಯೇರಿಯಬಲ್‌ಗಳು ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್‌ಗಳನ್ನು, ಹಾಗೆಯೇ ಕೆಲವು ಸಾಮಾನ್ಯ data type ಗಳು ಮತ್ತು ಮೂಲ operators ಗಳನ್ನು ಕಲಿಯುತ್ತೇವೆ.

ಇವು ಪ್ರೋಗ್ರಾಮಿಂಗ್‌ನ ಅತ್ಯಂತ ಮೂಲಭೂತ ಜ್ಞಾನಗಳಾಗಿದ್ದು, ಮುಂದಿನ SwiftUI ಕಲಿಕೆಗೆ ಪ್ರಮುಖ ಆಧಾರವಾಗಿವೆ.

## ಜೀವನದಲ್ಲಿನ ವ್ಯೇರಿಯಬಲ್‌ಗಳು ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್‌ಗಳು

ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ಕಲ್ಪನೆಗಳನ್ನು ದಿನನಿತ್ಯದ ಉದಾಹರಣೆಗಳ ಮೂಲಕ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು.

ಜೀವನದಲ್ಲಿನ ವ್ಯೇರಿಯಬಲ್‌ಗಳು:

- ಟಿವಿಯಲ್ಲಿ ಪ್ರತಿಯೊಂದು ಕಾರ್ಯಕ್ರಮದಲ್ಲೂ ಬರುವ ವಿಷಯ ವಿಭಿನ್ನವಾಗಿರುತ್ತದೆ
- ಪ್ರತಿದಿನದ ಹವಾಮಾನ ಒಂದೇ ತರಹ ಇರುವುದಿಲ್ಲ
- ಗಡಿಯಾರ ಪ್ರತಿಯೊಂದು ಕ್ಷಣವೂ ಬದಲಾಗುತ್ತದೆ

ಈ ಎಲ್ಲದರ ಸಾಮಾನ್ಯ ಲಕ್ಷಣ ಏನೆಂದರೆ: ಇವುಗಳಲ್ಲಿ ಬದಲಾವಣೆ ಬರುತ್ತದೆ.

ಒಂದು ವೇಳೆ ಟಿವಿಯಲ್ಲಿ ಸದಾ ಒಂದೇ ದೃಶ್ಯ ತೋರಿಸುತ್ತಿದ್ದರೆ, ಹವಾಮಾನ ಎಂದಿಗೂ ಬದಲಾಗದಿದ್ದರೆ, ಗಡಿಯಾರ ಸೂಜಿಗಳು ಎಂದಿಗೂ ಚಲಿಸದಿದ್ದರೆ, ಆಗ ಅವೆಲ್ಲವೂ ಕನ್ಸ್ಟೆಂಟ್‌ಗಳಾಗುತ್ತವೆ.

ಒಂದು ಬದಲಾಗಬಹುದು, ಮತ್ತೊಂದು ಬದಲಾಗುವುದಿಲ್ಲ.

## ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

App ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ, ಸಾಮಾನ್ಯವಾಗಿ ಬಳಕೆದಾರನು ಕೆಲವು ಮಾಹಿತಿಯನ್ನು ತುಂಬಬೇಕು ಅಥವಾ ಉಳಿಸಬೇಕು.

ಉದಾಹರಣೆಗೆ:

- ಖಾತೆ ಹೆಸರು
- ಹುಟ್ಟುಹಬ್ಬ
- ಸಂಪರ್ಕ ಮಾಹಿತಿ
- ವಿಳಾಸ

ಈ ಮಾಹಿತಿಗಳು ಉಳಿದು ನಂತರ ತೋರಿಸಲಾಗುತ್ತವೆ.

ಉದಾಹರಣೆಗೆ, ಬಳಕೆದಾರನು App ಒಳಗೆ ಒಂದು ಹೆಸರು ನಮೂದಿಸಿದ್ದಾನೆ:

```
FangJunyu
```

ನಾವು ಈ ಹೆಸರನ್ನು ಉಳಿಸಬೇಕಿದೆ, ಹೀಗಾಗಿ App ನಲ್ಲಿ ಅದನ್ನು ತೋರಿಸಬಹುದು.

ಈ ಉಳಿಸುವ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಸರಳವಾಗಿ ಒಂದು drawer ಒಳಗೆ ವಸ್ತುವನ್ನು ಇಡುವಂತೆ ಊಹಿಸಬಹುದು.

ನಾವು ಹೆಸರನ್ನು ಉಳಿಸುವಾಗ, ಅದು ಒಂದು drawer ಒಳಗೆ ಇಟ್ಟಂತಾಗುತ್ತದೆ.

ಉಳಿಸಬೇಕಾದ ವಸ್ತುಗಳು ತುಂಬಾ ಇರಬಹುದು; ಹಾಗೆಯೇ drawer ಗಳು ಕೂಡ ಅನೇಕವಾಗಿರುತ್ತವೆ. ಯಾವ drawer ನಲ್ಲಿ ಏನು ಇದೆ ಎಂದು ತಿಳಿಯಲು, ಪ್ರತಿಯೊಂದಕ್ಕೂ ಹೆಸರು ಬೇಕಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```
name
```

ಈ ಉದಾಹರಣೆಯಲ್ಲಿ `name` ಎನ್ನುವುದು drawer ಹೆಸರಿನಂತೆ, ಮತ್ತು `FangJunyu` ಉಳಿಸಲಾದ ಮಾಹಿತಿ.

![Var](../../RESOURCE/007_var.png)

**Swift ನಲ್ಲಿ ಡೇಟಾ ಉಳಿಸಬೇಕಾದರೆ, ಅದನ್ನು ತಪ್ಪದೇ ವ್ಯೇರಿಯಬಲ್ (`var`) ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ (`let`) ರೂಪದಲ್ಲಿ ಘೋಷಿಸಬೇಕು.**

ಹೆಸರು ಸಾಮಾನ್ಯವಾಗಿ ಬದಲಾಯಿಸಬಹುದಾದ್ದರಿಂದ, ಇಲ್ಲಿ ನಾವು ವ್ಯೇರಿಯಬಲ್ ಬಳಸಬೇಕು.

```swift
var name = "FangJunyu"
```

ಇಲ್ಲಿ `name` ಎಂಬ ಹೆಸರಿನ ವ್ಯೇರಿಯಬಲ್ ಘೋಷಿಸಲಾಗಿದೆ. ಅದರ type `String`, ಮತ್ತು ಮೌಲ್ಯ `"FangJunyu"`.

### ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ನಡುವಿನ ವ್ಯತ್ಯಾಸ

ವ್ಯೇರಿಯಬಲ್ ಅನ್ನು `var` ಮೂಲಕ ಘೋಷಿಸಲಾಗುತ್ತದೆ:

```swift
var
```

ಕನ್ಸ್ಟೆಂಟ್ ಅನ್ನು `let` ಮೂಲಕ ಘೋಷಿಸಲಾಗುತ್ತದೆ:

```swift
let
```

ಉದಾಹರಣೆಗೆ:

```swift
var name = "FangJunyu"
let id = 123456
```

ವ್ಯೇರಿಯಬಲ್ ಆಗಿ ಘೋಷಿಸಿದರೆ ಅದರ ಮೌಲ್ಯವನ್ನು ನಂತರ ಬದಲಾಯಿಸಬಹುದು; ಕನ್ಸ್ಟೆಂಟ್ ಆಗಿದ್ದರೆ, ಮರುಬದಲಾವಣೆ ಸಾಧ್ಯವಿಲ್ಲ.

ಹೀಗಾಗಿ ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ನಡುವಿನ ಮುಖ್ಯ ವ್ಯತ್ಯಾಸ: ಮೌಲ್ಯ ಬದಲಾಯಿಸಲು ಅನುಮತಿ ಇದೆಯೇ ಇಲ್ಲವೇ?

### ಕನ್ಸ್ಟೆಂಟ್ ಅನ್ನು ಬದಲಾಯಿಸುವುದು

ಒಂದು ವೇಳೆ ಕೋಡ್ ಕನ್ಸ್ಟೆಂಟ್‌ನ ಮೌಲ್ಯ ಬದಲಾಯಿಸಲು ಪ್ರಯತ್ನಿಸಿದರೆ:

```swift
let name = "Sam"
name = "Bob"
```

Swift ಇದು ಸಾಧ್ಯವಿಲ್ಲ ಎಂದು ಸೂಚಿಸುತ್ತದೆ, ಏಕೆಂದರೆ ಅದು ಕನ್ಸ್ಟೆಂಟ್ ಆಗಿದೆ.

```
Cannot assign to property: 'name' is a 'let' constant
```

ಈ ರೀತಿ ವ್ಯವಸ್ಥೆಯು ಪ್ರಮುಖ ಡೇಟಾ ಅಜಾಗರೂಕತೆಯಿಂದ ಬದಲಾಗುವುದನ್ನು ತಪ್ಪಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

### SwiftUI ನಲ್ಲಿ ತೋರಿಸುವುದು

`ContentView.swift` ಫೈಲ್ ತೆರೆಯಿರಿ ಮತ್ತು View ಒಳಗೆ ವ್ಯೇರಿಯಬಲ್ ಘೋಷಿಸಿ:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

ಇಲ್ಲಿ `name` ಎಂಬ ವ್ಯೇರಿಯಬಲ್ ಘೋಷಿಸಲಾಗಿದೆ ಮತ್ತು `Text` ಮೂಲಕ ಅದನ್ನು ಇಂಟರ್ಫೇಸ್‌ನಲ್ಲಿ ತೋರಿಸಲಾಗಿದೆ.

ನಾವು ವ್ಯೇರಿಯಬಲ್‌ನ ಮೌಲ್ಯವನ್ನು ಬದಲಿಸಿದರೆ:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

`Text` ತೋರಿಸುವ ವಿಷಯವೂ ಅದಕ್ಕೆ ಅನುಗುಣವಾಗಿ ಬದಲಾಗುತ್ತದೆ.

ಈ ರೀತಿಯಲ್ಲಿ ನಾವು view ನಲ್ಲಿ ತೋರಿಸುವ ವಿಷಯವನ್ನು ವ್ಯೇರಿಯಬಲ್ ಮೂಲಕ ನಿಯಂತ್ರಿಸಬಹುದು; ಪ್ರತಿಸಾರಿ `Text` ಒಳಗಿನ literal text ಬದಲಿಸಬೇಕಾಗುವುದಿಲ್ಲ.

ಇದೇ ಮಾಹಿತಿಯನ್ನು ಬೇರೆಡೆ ಬಳಸಬೇಕಾದರೂ, ನಾವು ಈ ವ್ಯೇರಿಯಬಲ್ ಅನ್ನು pass ಮಾಡಬಹುದು — ಅಂದರೆ drawer ಒಳಗಿನ ವಿಷಯವನ್ನು ಇನ್ನೊಂದು ಜಾಗಕ್ಕೆ ಕೊಟ್ಟಂತಾಗುತ್ತದೆ.

### SwiftUI ಯಲ್ಲಿ ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ಬರೆಯುವ ಸ್ಥಳ

SwiftUI ಯಲ್ಲಿ ವ್ಯೇರಿಯಬಲ್‌ಗಳು ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್‌ಗಳು ಸಾಮಾನ್ಯವಾಗಿ `body` ಹೊರಗೆ ಬರೆಯಲಾಗುತ್ತವೆ:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

ಖಂಡಿತವಾಗಿಯೂ, ಅವನ್ನು `body` ಒಳಗೆಯೂ ಬರೆಯಬಹುದು:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

ಆದರೆ ಇವೆರಡರ ನಡುವೆ ಒಂದು ಪ್ರಮುಖ ವ್ಯತ್ಯಾಸ ಇದೆ:

- ವ್ಯೇರಿಯಬಲ್ ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ `body` ಒಳಗೆ ಇದ್ದರೆ, view ಮರುಗಣನೆಗೊಂಡ ಪ್ರತಿ ಸಲ ಅದು ಹೊಸದಾಗಿ ರಚಿಸಲಾಗುತ್ತದೆ
- `body` ಹೊರಗೆ ಇದ್ದರೆ, ಅದು view structure ನ property ಆಗಿರುತ್ತದೆ, ಇದರಿಂದ ಕೋಡ್ ರಚನೆ ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟವಾಗಿರುತ್ತದೆ

ಹೀಗಾಗಿ ನೈಜ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಸಾಮಾನ್ಯವಾಗಿ ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್‌ಗಳನ್ನು `body` ಹೊರಗೇ ಬರೆಯಲಾಗುತ್ತದೆ.

## ಡೇಟಾ ಟೈಪ್‌ಗಳು

ವ್ಯೇರಿಯಬಲ್‌ಗಳು ಹಲವಾರು ವಿಧದ ಡೇಟಾವನ್ನು ಉಳಿಸಬಹುದು. ಇವುಗಳನ್ನು data types ಎಂದು ಕರೆಯುತ್ತಾರೆ.

ಉದಾಹರಣೆಗೆ, ನಾವು ಮೊದಲು `Text` ಬಳಸಿ ಪಠ್ಯ ತೋರಿಸಿದ್ದೇವೆ:

```swift
Text("Hello, World")
```

ಇಲ್ಲಿನ `"Hello, World"` ಒಂದು String (ಪಠ್ಯ string) type ಆಗಿದೆ.

String ಪಠ್ಯ ವಿಷಯವನ್ನು ಸೂಚಿಸುತ್ತದೆ, ಮತ್ತು Swift ನಲ್ಲಿ ಅದನ್ನು `""` ಒಳಗೆ ಬರೆಯಬೇಕು.

ಉದಾಹರಣೆಗೆ:

```swift
var hello = "Hello, World"
```

String ಹೊರತಾಗಿಯೂ Swift ನಲ್ಲಿ ಇನ್ನೂ ಅನೇಕ data types ಇವೆ.

ಆರಂಭಿಕ ಹಂತದಲ್ಲಿ ಸಾಮಾನ್ಯವಾಗಿ ಕಾಣುವ ನಾಲ್ಕು ಮುಖ್ಯ types ಇವು:

- String
- Int
- Double
- Bool

ಈ ನಾಲ್ಕು data types ಬಳಸಿ ಬಹುತೇಕ ಮೂಲಭೂತ ಡೇಟಾ ಸಂಸ್ಕರಣೆಯನ್ನು ಮಾಡಬಹುದು.

**String**

String ಅಂದರೆ ಪಠ್ಯ. ಉದಾಹರಣೆಗೆ:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

String ಸಾಮಾನ್ಯವಾಗಿ ಬಳಕೆದಾರ ಹೆಸರು, ಶೀರ್ಷಿಕೆ, ಪಠ್ಯ ವಿಷಯ ಮುಂತಾದವುಗಳನ್ನು ಸೂಚಿಸುತ್ತದೆ.

**Int**

`Int` ಅಂದರೆ ಪೂರ್ಣಾಂಕ — ದಶಾಂಶ ಭಾಗವಿಲ್ಲದ ಸಂಖ್ಯೆ.

ಉದಾಹರಣೆಗೆ:

```swift
var age = 26
var count = 100
```

ಪೂರ್ಣಾಂಕವನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ವಯಸ್ಸು, ಸಂಖ್ಯೆ, ಎಣಿಕೆ ಮುಂತಾದವುಗಳಿಗೆ ಬಳಸುತ್ತಾರೆ.

**Double**

`Double` ದಶಾಂಶ ಹೊಂದಿರುವ ಸಂಖ್ಯೆಯನ್ನು ಸೂಚಿಸುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
var weight = 74.5
var height = 185.0
```

Swift ನಲ್ಲಿ ದಶಾಂಶ ಸಂಖ್ಯೆಗಳು ಸಾಮಾನ್ಯವಾಗಿ ಡೀಫಾಲ್ಟ್ ಆಗಿ `Double` ಆಗಿರುತ್ತವೆ.

ಇದಕ್ಕೆ ಸಮಾನವಾದ `Float` ಎಂಬ type ಕೂಡ ಇದೆ, ಆದರೆ ಅದರ precision ಕಡಿಮೆ. ಆದ್ದರಿಂದ ನೈಜ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ `Double` ಹೆಚ್ಚು ಬಳಸಲಾಗುತ್ತದೆ.

**Bool**

`Bool` type ಎರಡು ಸ್ಥಿತಿಗಳನ್ನು ತೋರಿಸಲು ಬಳಸಲಾಗುತ್ತದೆ: `true` ಮತ್ತು `false`.

ಉದಾಹರಣೆಗೆ:

```swift
var isShowAlert = true
var isLogin = false
```

`Bool` ಸಾಮಾನ್ಯವಾಗಿ condition check ಗಳಲ್ಲಿ ಬಳಸಲಾಗುತ್ತದೆ. ಉದಾಹರಣೆಗೆ, alert ತೋರಿಸಬೇಕೇ ಬೇಡವೇ.

ಎರಡು ಫಲಿತಾಂಶಗಳಲ್ಲಿ ಒಂದೇ ಸಾಧ್ಯವಾಗುವ ಸಂದರ್ಭಗಳಲ್ಲಿ `Bool` ತುಂಬಾ ಸೂಕ್ತವಾಗಿದೆ.

### SwiftUI ನಲ್ಲಿ ತೋರಿಸುವುದು

ನಾವು String type ತೋರಿಸುವಾಗ, ಅದನ್ನು ನೇರವಾಗಿ `Text` ನಲ್ಲಿ ಬಳಸಬಹುದು:

```swift
Text(name)
```

ಆದರೆ `Int`, `Double` ಮುಂತಾದವು string ಅಲ್ಲ; ಹೀಗಾಗಿ ಅವನ್ನು ನೇರವಾಗಿ text content ಭಾಗವಾಗಿ ಬಳಸಲು ಸಾಧ್ಯವಿಲ್ಲ.

ಅವುಗಳನ್ನು `Text` ನಲ್ಲಿ ತೋರಿಸಬೇಕಾದರೆ, string interpolation ಬಳಸಬೇಕು:

```swift
\()
```

String interpolation ಎಂದರೆ string ಒಳಗೆ `\()` ಬಳಸಿ ವ್ಯೇರಿಯಬಲ್ ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ ಸೇರಿಸುವ ವಿಧಾನ.

ಉದಾಹರಣೆಗೆ:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

ಈ ಕೋಡ್‌ನಲ್ಲಿ `""` string ಅನ್ನು ಸೂಚಿಸುತ್ತದೆ, ಮತ್ತು `\()` ಅದರೊಳಗೆ ವ್ಯೇರಿಯಬಲ್ ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ ಸೇರಿಸಲು ಬಳಕೆಯಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
"DoubleNum: \(num * 2)"
```

ತೋರಿಸುವ ವಿಷಯ:

```swift
DoubleNum: 2
```

ಈ ವಿಧಾನದಿಂದ ನಾವು string ಒಳಗೆ ವ್ಯೇರಿಯಬಲ್ ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ ಸೇರಿಸಿ, ನಂತರ ಅವನ್ನು `Text` ಮೂಲಕ view ನಲ್ಲಿ ತೋರಿಸಬಹುದು.

ಸೂಚನೆ: `\()` ಅನ್ನು ಕೇವಲ string `""` ಒಳಗಷ್ಟೇ ಬಳಸಬಹುದು.

## Operators

ವ್ಯೇರಿಯಬಲ್ ಅಥವಾ ಕನ್ಸ್ಟೆಂಟ್ ಘೋಷಿಸುವಾಗ ನಾವು ಈ ರೀತಿಯ ಬರಹ ನೋಡುತ್ತೇವೆ:

```swift
var num = 1
```

ಇಲ್ಲಿನ `=` ಅನ್ನು assignment operator ಎಂದು ಕರೆಯುತ್ತಾರೆ.

ಇದರ ಕೆಲಸ: ಬಲಭಾಗದ ಮೌಲ್ಯವನ್ನು ಎಡಭಾಗದ ವ್ಯೇರಿಯಬಲ್‌ಗೆ ನೀಡುವುದು.

ಈ ಉದಾಹರಣೆಯಲ್ಲಿ `1` ಮೌಲ್ಯವು `num` ವ್ಯೇರಿಯಬಲ್‌ಗೆ assign ಆಗಿದೆ.

Assignment operator ಹೊರತಾಗಿಯೂ, ಇನ್ನೂ ಕೆಲವು ಸಾಮಾನ್ಯ ಸಂಖ್ಯಾ operators ಇವೆ:

- `+`
- `-`
- `*`
- `/`

ಸಂಖ್ಯೆಗಳ ಮೇಲೆ ಲೆಕ್ಕಾಚಾರ ಮಾಡಬೇಕಾದಾಗ, ಉದಾಹರಣೆಗೆ `Int` ಅಥವಾ `Double` data ಮೇಲೆ, ಇವುಗಳನ್ನು ಬಳಸುತ್ತೇವೆ.

ಉದಾಹರಣೆಗೆ:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

ಲೆಕ್ಕಾಚಾರವಾದ ನಂತರ ಫಲಿತಾಂಶವು ವ್ಯೇರಿಯಬಲ್‌ನಲ್ಲಿ ಉಳಿಯುತ್ತದೆ.

## ಸಾರಾಂಶ

ವ್ಯೇರಿಯಬಲ್‌ಗಳು, ಕನ್ಸ್ಟೆಂಟ್‌ಗಳು ಮತ್ತು operators — ಇವು ಪ್ರೋಗ್ರಾಮಿಂಗ್‌ನ ಅತ್ಯಂತ ಮೂಲಭೂತ ಕಲ್ಪನೆಗಳು.

ವ್ಯೇರಿಯಬಲ್ ಮತ್ತು ಕನ್ಸ್ಟೆಂಟ್ ಮೂಲಕ ನಾವು ವಿವಿಧ ರೀತಿಯ ಡೇಟಾವನ್ನು ಉಳಿಸಬಹುದು; data type ಮೂಲಕ ಆ ಡೇಟಾದ ಸ್ವಭಾವ ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ; operators ಮೂಲಕ ಆ ಡೇಟಾವನ್ನು ಲೆಕ್ಕ ಹಾಕಬಹುದು ಮತ್ತು ಸಂಸ್ಕರಿಸಬಹುದು.

ಈ ಜ್ಞಾನವು ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಲೋಕದ ಮೂಲ ಸಾಧನಗಳಂತಿದೆ. ಇದನ್ನು ಹಿಡಿತಕ್ಕೆ ತರುತ್ತಿದ್ದಂತೆ, ಮುಂದಿನ Swift ಮತ್ತು SwiftUI ಕಲಿಕೆಗೆ ಗಟ್ಟಿಯಾದ ನೆಲೆ ಸಿಗುತ್ತದೆ.
