# ಫಿಲ್ ಲೈಟ್ ಆಪ್

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ತುಂಬಾ ಆಸಕ್ತಿದಾಯಕವಾದ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಅನ್ನು ನಿರ್ಮಿಸುತ್ತೇವೆ. ರಾತ್ರಿ ಬಂದಾಗ, ನಾವು ಫೋನ್ ಪರದೆಯಲ್ಲಿ ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳನ್ನು ತೋರಿಸಿ ಅದನ್ನು ಸರಳವಾದ ಫಿಲ್ ಲೈಟ್ ಆಗಿ ಬಳಸಬಹುದು.

ಈ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಪರದೆಯನ್ನು ಟ್ಯಾಪ್ ಮಾಡುವ ಮೂಲಕ ಬಣ್ಣಗಳನ್ನು ಬದಲಾಯಿಸಬಹುದು, ಹಾಗೆಯೇ ಸ್ಲೈಡರ್ ಬಳಸಿ ಪ್ರಕಾಶಮಾನತೆಯನ್ನೂ ಹೊಂದಿಸಬಹುದು.

ಈ ಉದಾಹರಣೆಯಲ್ಲಿ, `brightness` ಬಳಸಿ ವ್ಯೂನ ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ಹೊಂದಿಸುವುದು, `onTapGesture` ಬಳಸಿ ವ್ಯೂಗೆ ಟ್ಯಾಪ್ ಜೇಶ್ಚರ್ ಸೇರಿಸುವುದು, ಮತ್ತು `Slider` ನಿಯಂತ್ರಣದ ಬಗ್ಗೆ ನಾವು ಕಲಿಯುತ್ತೇವೆ.

## ಬಣ್ಣವನ್ನು ತೋರಿಸುವುದು

ಮೊದಲು, ವ್ಯೂನಲ್ಲಿ ಒಂದು ಬಣ್ಣವನ್ನು ತೋರಿಸೋಣ.

SwiftUI ನಲ್ಲಿ, `Color` ಕೇವಲ ಒಂದು ಬಣ್ಣವನ್ನು ಮಾತ್ರ ಸೂಚಿಸುವುದಿಲ್ಲ, ಅದನ್ನು ವ್ಯೂ ಆಗಿಯೂ ತೋರಿಸಬಹುದು:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

ಇಲ್ಲಿ `Color.red` ಕೆಂಪು ಬಣ್ಣದ ವ್ಯೂ ಅನ್ನು ಸೂಚಿಸುತ್ತದೆ. `.ignoresSafeArea()` ಬಣ್ಣದ ವ್ಯೂವನ್ನು ಸಂಪೂರ್ಣ ಪರದೆಗೆ ಹರಡುತ್ತದೆ, ಅದರಿಂದ ಇದು ನಿಜವಾದ ಫಿಲ್ ಲೈಟ್ ಪರಿಣಾಮದಂತೆ ಕಾಣುತ್ತದೆ.

ಫಲಿತಾಂಶ:

### ಬಣ್ಣಗಳ ಅರೆ ಮತ್ತು ಸೂಚ್ಯಂಕ

ಈಗ ಕೇವಲ ಒಂದು ಬಣ್ಣ ಮಾತ್ರ ತೋರಿಸಲಾಗುತ್ತಿದೆ. ಆದರೆ ಫಿಲ್ ಲೈಟ್ ಸಾಮಾನ್ಯವಾಗಿ ಒಂದೇ ಬಣ್ಣ ಹೊಂದಿರುವುದಿಲ್ಲ. ಇದು ನೀಲಿ, ಹಳದಿ, ನೇರಳೆ, ಬಿಳಿ ಮತ್ತು ಇತರ ಬಣ್ಣಗಳನ್ನೂ ತೋರಿಸಬಹುದು.

ಪರದೆಯನ್ನು ಟ್ಯಾಪ್ ಮಾಡಿದಾಗ ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳಿಗೆ ಬದಲಾಯಿಸಬೇಕೆಂದು ನಾವು ಬಯಸುತ್ತೇವೆ. ಈ ಬಣ್ಣಗಳನ್ನು ನಾವು ಒಂದೇ ಅರೆನಲ್ಲಿ ಇಟ್ಟು ಸುಲಭವಾಗಿ ನಿರ್ವಹಿಸಬಹುದು:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

ಅರೆ "ಒಂದೇ ತರದ ಡೇಟಾ ಗುಂಪನ್ನು" ಸಂಗ್ರಹಿಸಲು ಸೂಕ್ತವಾಗಿದೆ. ಇಲ್ಲಿ ಅರೆಯ ಪ್ರತಿಯೊಂದು ಅಂಶವೂ ಒಂದು `Color` ಆಗಿದೆ.

ನಾವು ಯಾವುದಾದರೂ ನಿರ್ದಿಷ್ಟ ಬಣ್ಣವನ್ನು ತೋರಿಸಬೇಕಾದರೆ, ಸೂಚ್ಯಂಕವನ್ನು ಬಳಸಬಹುದು:

```swift
colors[0]
```

ಇದರ ಅರ್ಥ ಅರೆದಲ್ಲಿನ `0` ಸೂಚ್ಯಂಕದ ಬಣ್ಣವನ್ನು ಓದುವುದು, ಅಂದರೆ ಮೊದಲ ಬಣ್ಣವನ್ನು ಪಡೆಯುವುದು.

ಈಗ ಕೋಡ್ ಹೀಗೆ ಬರೆಯಬಹುದು:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

ಈ ರೀತಿಯಲ್ಲಿ ಪರದೆ ಅರೆದಲ್ಲಿನ ಮೊದಲ ಬಣ್ಣವಾದ ಕೆಂಪನ್ನು ತೋರಿಸುತ್ತದೆ.

### ಸೂಚ್ಯಂಕದಿಂದ ಬಣ್ಣವನ್ನು ನಿಯಂತ್ರಿಸುವುದು

ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳಿಗೆ ಬದಲಾಯಿಸಬೇಕಾದರೆ, ಸೂಚ್ಯಂಕವನ್ನು ನೇರವಾಗಿ ಬರೆಯುವುದಕ್ಕೆ ಬದಲು ಅದನ್ನು ನಿಯಂತ್ರಿಸಲು ಒಂದು ವ್ಯರಿಯಬಲ್ ಬೇಕಾಗುತ್ತದೆ.

ಸೂಚ್ಯಂಕವನ್ನು ಉಳಿಸಲು `@State` ಬಳಸಿ ಒಂದು ವ್ಯರಿಯಬಲ್ ಘೋಷಿಸಬಹುದು:

```swift
@State private var index = 0
```

ಇಲ್ಲಿ `index` ಪ್ರಸ್ತುತ ಬಣ್ಣದ ಸೂಚ್ಯಂಕವನ್ನು ಸೂಚಿಸುತ್ತದೆ.

`index` ಬದಲಾಗುವಾಗ, SwiftUI ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಮತ್ತೆ ಲೆಕ್ಕ ಹಾಕಿ ತೋರಿಸಲಾಗುತ್ತಿರುವ ವಿಷಯವನ್ನು ನವೀಕರಿಸುತ್ತದೆ.

ನಂತರ ಹಳೆಯ `colors[0]` ಅನ್ನು ಬದಲಿಸಿ ಹೀಗೆ ಬರೆಯಿರಿ:

```swift
colors[index]
```

ಈ ರೀತಿಯಲ್ಲಿ ವ್ಯೂನಲ್ಲಿ ತೋರಿಸುವ ಬಣ್ಣ `index` ಮೂಲಕ ನಿರ್ಧಾರವಾಗುತ್ತದೆ.

ಈಗ ಕೋಡ್ ಹೀಗಾಗುತ್ತದೆ:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

`index` ಬದಲಾಗುವಾಗ, `colors[index]` ಕೂಡ ಬೇರೆ ಬಣ್ಣವನ್ನು ತೋರಿಸುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

ಗಮನಿಸಬೇಕಾದ ವಿಷಯವೆಂದರೆ, `index` ಅರೆದಲ್ಲಿನ ಗರಿಷ್ಠ ಸೂಚ್ಯಂಕಕ್ಕಿಂತ ಹೆಚ್ಚಾಗಬಾರದು, ಇಲ್ಲವಾದರೆ out-of-range ದೋಷ ಬರುತ್ತದೆ.

## ಟ್ಯಾಪ್ ಜೇಶ್ಚರ್

ಈಗ ನಾವು `index` ಪ್ರಕಾರ ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳನ್ನು ತೋರಿಸಬಹುದು, ಆದರೆ ಟ್ಯಾಪ್ ಮಾಡಿ ಬಣ್ಣವನ್ನು ಬದಲಾಯಿಸಲು ಇನ್ನೂ ಸಾಧ್ಯವಾಗಿಲ್ಲ.

ಹಿಂದಿನ “Quote Carousel” ಪಾಠದಲ್ಲಿ, quote ಬದಲಾಯಿಸಲು ನಾವು `Button` ಬಳಸಿದ್ದೆವು.

ಆದರೆ ಈ ಬಾರಿ, “ಪೂರ್ಣ ಬಣ್ಣದ ಪ್ರದೇಶವನ್ನು ಟ್ಯಾಪ್” ಮಾಡಿ ಬಣ್ಣ ಬದಲಾಯಿಸಲು ಬಯಸುತ್ತೇವೆ, ಆದ್ದರಿಂದ `onTapGesture` ಇನ್ನೂ ಹೆಚ್ಚು ಸೂಕ್ತವಾಗಿದೆ.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

ನಾವು ಬಣ್ಣದ ವ್ಯೂ ಮೇಲೆ ಟ್ಯಾಪ್ ಮಾಡಿದಾಗ, ಈ ಕೋಡ್ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ:

```swift
index += 1
```

ಇದರ ಅರ್ಥ `index` ಅನ್ನು `1` ರಿಂದ ಹೆಚ್ಚಿಸುವುದು. ಸೂಚ್ಯಂಕ ಹೆಚ್ಚಾದ ನಂತರ, `colors[index]` ಅರೆದಲ್ಲಿನ ಮುಂದಿನ ಬಣ್ಣವನ್ನು ತೋರಿಸುತ್ತದೆ.

### `onTapGesture`

`onTapGesture` ಒಂದು gesture modifier ಆಗಿದ್ದು, ಅದು ವ್ಯೂಗೆ ಟ್ಯಾಪ್ ಕ್ರಿಯೆಯನ್ನು ಸೇರಿಸುತ್ತದೆ.

ಮೂಲ ಬಳಕೆ:

```swift
.onTapGesture {
    // code
}
```

ಉದಾಹರಣೆಗೆ:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

ಈ ಕೆಂಪು ವ್ಯೂ ಮೇಲೆ ಟ್ಯಾಪ್ ಮಾಡಿದಾಗ, braces ಒಳಗಿನ ಕೋಡ್ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ ಮತ್ತು console ನಲ್ಲಿ ಇದು ಮುದ್ರಿತವಾಗುತ್ತದೆ:

```swift
Click color
```

`onTapGesture` ಮೂಲಕ, ವ್ಯೂ ಮೇಲೆ ಟ್ಯಾಪ್ ಮಾಡಿದ ನಂತರ ಏನು ನಡೆಯಬೇಕು ಎಂಬುದನ್ನು ನಾವು ನಿರ್ಧರಿಸಬಹುದು.

### `Button` ಜೊತೆಯಲ್ಲಿನ ವ್ಯತ್ಯಾಸ

ಹಿಂದೆ ನಾವು `Button` ವ್ಯೂ ಬಗ್ಗೆ ಕಲಿತಿದ್ದೇವೆ. `Button` ಮತ್ತು `onTapGesture` ಎರಡೂ ಟ್ಯಾಪ್ ಕ್ರಿಯೆಗಳನ್ನು ನಿರ್ವಹಿಸಬಹುದು, ಆದರೆ ಅವುಗಳ ಬಳಕೆಯ ಸಂದರ್ಭಗಳು ಸಂಪೂರ್ಣ ಒಂದೇ ರೀತಿಯಾಗಿಲ್ಲ.

`onTapGesture` “ಈಗಾಗಲೇ ಇರುವ ವ್ಯೂಗೆ ಟ್ಯಾಪ್ ಕಾರ್ಯಕ್ಷಮತೆ ಸೇರಿಸಲು” ಹೆಚ್ಚು ಸೂಕ್ತವಾಗಿದೆ, ಉದಾಹರಣೆಗೆ `Color`, `Image`, `Text`, ಅಥವಾ ಇತರ ಸಾಮಾನ್ಯ ವ್ಯೂಗಳು.

ಇನ್ನೊಂದೆಡೆ `Button` “Confirm”, “Submit”, ಅಥವಾ “Delete” போன்ற ಸ್ಪಷ್ಟ ಬಟನ್ ಅನ್ನು ಸೂಚಿಸಲು ಹೆಚ್ಚು ಸೂಕ್ತವಾಗಿದೆ.

ಈ ಫಿಲ್ ಲೈಟ್ ಆಪ್‌ನಲ್ಲಿ, ಬಣ್ಣ ಬದಲಿಸುವ ಕಾರ್ಯವನ್ನು ಇನ್ನೂ ಸರಳಗೊಳಿಸಲು ನಾವು ಬಯಸುತ್ತೇವೆ. ಸಂಪೂರ್ಣ ಬಣ್ಣದ ಪ್ರದೇಶವನ್ನು ಟ್ಯಾಪ್ ಮಾಡಿದರೆ ಬಣ್ಣ ಬದಲಾಗುವುದರಿಂದ, ಇಲ್ಲಿ `onTapGesture` ಒಳ್ಳೆಯ ಆಯ್ಕೆಯಾಗಿದೆ.

## ಸೂಚ್ಯಂಕದ ಸಮಸ್ಯೆ

ಈಗ ನಾವು ಪರದೆಯನ್ನು ಟ್ಯಾಪ್ ಮಾಡಿ ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳಿಗೆ ಬದಲಾಯಿಸಬಹುದು.

ಆದರೆ ಇಲ್ಲಿ ಒಂದು ಪ್ರಮುಖ ಸಮಸ್ಯೆಯಿದೆ: **ಸೂಚ್ಯಂಕ ಅರೆ ಮಿತಿಯನ್ನು ಮೀರಿ ಹೋಗಬಹುದು**.

ಉದಾಹರಣೆಗೆ:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

ನಾವು ಪರದೆಯನ್ನು ಮತ್ತೆ ಮತ್ತೆ ಟ್ಯಾಪ್ ಮಾಡುತ್ತಿದ್ದರೆ, `index` ಕೊನೆಯಲ್ಲಿ `4` ಆಗುತ್ತದೆ, ಮತ್ತು ಆಗ “index out of range” ದೋಷ ಉಂಟಾಗುತ್ತದೆ.

ಇದರ ಕಾರಣ `colors` ಅರೆದಲ್ಲಿ `4` ಅಂಶಗಳಿವೆ, ಆದರೆ ಸೂಚ್ಯಂಕ `0` ರಿಂದ ಆರಂಭವಾಗುತ್ತದೆ. ಆದ್ದರಿಂದ ಮಾನ್ಯ ಸೂಚ್ಯಂಕ ಮಿತಿ `0 - 3`, `4` ಅಲ್ಲ.

ನಾವು `colors[4]` ಅನ್ನು ಪ್ರವೇಶಿಸಿದರೆ, “index out of range” ದೋಷ ಬರುತ್ತದೆ.

ಪ್ರಸ್ತುತ ಕೋಡ್‌ನಲ್ಲಿ, ಪ್ರತಿ ಟ್ಯಾಪ್ `index` ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ `1` ಹೆಚ್ಚಿಸುತ್ತದೆ. ನಾವು ಇದನ್ನು ನಿರ್ವಹಿಸದಿದ್ದರೆ, ಅದು ಕೊನೆಯಲ್ಲಿ ಮಿತಿಯನ್ನು ಮೀರುತ್ತದೆ.

ಆದ್ದರಿಂದ, ಪರದೆಯನ್ನು ಟ್ಯಾಪ್ ಮಾಡಿದಾಗ ಸೂಚ್ಯಂಕವನ್ನು ಪರಿಶೀಲಿಸಬೇಕು: ಅದು ಕೊನೆಯ ಬಣ್ಣವಾಗಿದ್ದರೆ, ಮೊದಲ ಬಣ್ಣಕ್ಕೆ ಹಿಂದಿರುಗಬೇಕು; ಇಲ್ಲದಿದ್ದರೆ `1` ಹೆಚ್ಚಿಸಬೇಕು.

ಇದನ್ನು ನಾವು `if` statement ಮೂಲಕ ಮಾಡಬಹುದು:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

ಈ ಕೋಡ್‌ನಲ್ಲಿ, `colors.count` ಅಂದರೆ ಅರೆದಲ್ಲಿನ ಅಂಶಗಳ ಸಂಖ್ಯೆ.

ಪ್ರಸ್ತುತ ಅರೆದಲ್ಲಿ 4 ಬಣ್ಣಗಳಿವೆ, ಆದ್ದರಿಂದ:

```swift
colors.count // 4
```

ಆದರೆ ಗರಿಷ್ಠ ಸೂಚ್ಯಂಕ `4` ಅಲ್ಲ, `3`, ಏಕೆಂದರೆ ಸೂಚ್ಯಂಕ `0` ರಿಂದ ಆರಂಭವಾಗುತ್ತದೆ.

ಆದ್ದರಿಂದ ಕೊನೆಯ ಸೂಚ್ಯಂಕವನ್ನು ಹೀಗೆ ಬರೆಯಬೇಕು:

```swift
colors.count - 1
```

ಅಂದರೆ:

```swift
4 - 1 = 3
```

ಈ ತರ್ಕದ ಅರ್ಥ: ಪ್ರಸ್ತುತ ಸೂಚ್ಯಂಕ ಈಗಾಗಲೇ ಕೊನೆಯ ಬಣ್ಣವನ್ನು ಸೂಚಿಸುತ್ತಿದ್ದರೆ, ಸೂಚ್ಯಂಕವನ್ನು `0` ಗೆ ಮರುಹೊಂದಿಸಿ; ಇಲ್ಲದಿದ್ದರೆ ಅದನ್ನು `1` ಹೆಚ್ಚಿಸಿ.

ಈ ರೀತಿಯಲ್ಲಿ, ಬಣ್ಣಗಳು ಚಕ್ರಾಕಾರವಾಗಿ ಬದಲಾಗುತ್ತವೆ.

### ಸೂಚ್ಯಂಕ ಲಾಜಿಕ್‌ನ್ನು ಇನ್ನೂ ಸಂಕ್ಷಿಪ್ತಗೊಳಿಸುವುದು

ಕೋಡ್ ಅನ್ನು ಇನ್ನೂ ಚಿಕ್ಕದಾಗಿಸಬೇಕೆಂದರೆ, ternary operator ನ್ನೂ ಬಳಸಬಹುದು:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

ಈ ಕೋಡ್‌ನ ಅರ್ಥ: `index == colors.count - 1` ಸರಿ ಇದ್ದರೆ, `0` ಅನ್ನು ಹಿಂತಿರುಗಿಸಿ. ಇಲ್ಲದಿದ್ದರೆ `index + 1` ಅನ್ನು ಹಿಂತಿರುಗಿಸಿ.

ಕೊನೆಯಲ್ಲಿ, ಆ ಫಲಿತಾಂಶವನ್ನು ಮತ್ತೆ `index` ಗೆ assign ಮಾಡಲಾಗುತ್ತದೆ.

ಈಗ ನಾವು ಬಣ್ಣ ಬದಲಾವಣೆ ಪರಿಣಾಮವನ್ನು ಜಾರಿಗೆ ತರುವುದು ಸಾಧ್ಯ.

ಪೂರ್ಣ ಕೋಡ್:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

ಈಗ ನಾವು ಪರದೆಯನ್ನು ಟ್ಯಾಪ್ ಮಾಡಿ ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳಿಗೆ ಬದಲಾಯಿಸಬಹುದು. ಒಂದು ಮೂಲಭೂತ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಸಿದ್ಧವಾಗಿದೆ.

## ಬಣ್ಣದ ಹೆಸರನ್ನು ತೋರಿಸುವುದು

ಬಣ್ಣಗಳಿಗೆ ಹೊಂದುವ ಮತ್ತೊಂದು ಪಠ್ಯ ಗುಂಪನ್ನು ಸೇರಿಸಬಹುದು, ಹೀಗಾಗಿ ಬಣ್ಣ ಬದಲಾದಾಗ ಪ್ರಸ್ತುತ ಬಣ್ಣದ ಹೆಸರೂ ಪರದೆಯಲ್ಲಿ ತೋರುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

- ಕೆಂಪಾಗಿದ್ದರೆ `Red`
- ನೀಲಾಗಿದ್ದರೆ `Blue`
- ಹಳದಿಯಾಗಿದ್ದರೆ `Yellow`
- ನೇರಳೆಯಾಗಿದ್ದರೆ `Purple`

ಇಲ್ಲಿಯೂ ನಾವು ಬಣ್ಣಗಳ ಹೆಸರನ್ನು ಸಂಗ್ರಹಿಸಲು ಅರೆ ಬಳಸಬಹುದು:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

ಈ ಅರೆದಲ್ಲಿನ ಪಠ್ಯದ ಕ್ರಮವು ಬಣ್ಣಗಳ ಅರೆದಲ್ಲಿನ ಬಣ್ಣಗಳ ಕ್ರಮಕ್ಕೆ ಸರಿಯಾಗಿ ಹೊಂದಿರಬೇಕು.

ನಂತರ ನಾವು ಪ್ರಸ್ತುತ ಸೂಚ್ಯಂಕಕ್ಕೆ ಹೊಂದುವ ಬಣ್ಣದ ಹೆಸರನ್ನು ತೋರಿಸಲು `Text` ಅನ್ನು ಬಳಸಬಹುದು:

```swift
Text(colorsName[index])
```

`Text` `index` ಆಧಾರದ ಮೇಲೆ ಪ್ರಸ್ತುತ ಬಣ್ಣದ ಹೆಸರನ್ನು ತೋರಿಸುತ್ತದೆ.

`Text` ನ ದೃಷ್ಯವನ್ನು ಉತ್ತಮಗೊಳಿಸಲು modifiers ಬಳಸಿ:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

ಇದರಿಂದ `Text` ಬಿಳಿ ಬಣ್ಣದಲ್ಲಿ, ದೊಡ್ಡ ಶೀರ್ಷಿಕೆ ಗಾತ್ರದಲ್ಲಿ ಮತ್ತು bold font weight ಜೊತೆ ಕಾಣಿಸುತ್ತದೆ.

ಈಗ ನಮ್ಮ ಬಳಿ full-screen `Color` ವ್ಯೂ ಇದೆ. `Text` ಅನ್ನು `Color` ವ್ಯೂ ಮೇಲಾಗಿ ತೋರಿಸಬೇಕಾದರೆ, ಎರಡನ್ನೂ ಒಂದರ ಮೇಲೊಂದು ಇಡಲು `ZStack` layout container ಬಳಸಬೇಕು.

```swift
ZStack {
    Color
    Text
}
```

ಹೀಗಾಗಿ ಕೋಡ್ ಹೀಗೆ ಆಗುತ್ತದೆ:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

ಈ ರೀತಿಯಾಗಿ, ಹಿನ್ನೆಲೆಯಾಗಿ ಬಣ್ಣದ ವ್ಯೂ ಇರುತ್ತದೆ ಮತ್ತು ಅದರ ಮೇಲೆ ಪಠ್ಯದ ವ್ಯೂ ಮೇರೆಯಾಗಿ ಕಾಣುತ್ತದೆ.

ಫಲಿತಾಂಶ:

![Color](../../Resource/018_color2.png)

ಒಂದು ವಿಷಯ ಗಮನಿಸಬೇಕಾದದ್ದು, `ZStack` ನಲ್ಲಿ ನಂತರ ಬರೆಯಲಾದ ವ್ಯೂ ಸಾಮಾನ್ಯವಾಗಿ ಮುಂದೆ ಕಾಣಿಸುತ್ತದೆ. `Text` ಅನ್ನು `Color` ಗಿಂತ ಮೊದಲು ಬರೆಯುವುದಾದರೆ, ನಂತರದ `Color` ವ್ಯೂ `Text` ಅನ್ನು ಮುಚ್ಚಬಹುದು.

## ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ನಿಯಂತ್ರಿಸುವುದು

ಈಗ ನಾವು ಬೇರೆ ಬೇರೆ ಬಣ್ಣಗಳನ್ನು ಬದಲಾಯಿಸಬಹುದು, ಆದರೆ ಫಿಲ್ ಲೈಟ್‌ಗೆ ಇನ್ನೊಂದು ಮುಖ್ಯ ವೈಶಿಷ್ಟ್ಯವಿದೆ: **ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ಹೊಂದಿಸುವುದು**.

SwiftUI ನಲ್ಲಿ, ವ್ಯೂನ ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ಹೊಂದಿಸಲು `brightness` modifier ಅನ್ನು ಬಳಸಬಹುದು.

ಉದಾಹರಣೆಗೆ:

```swift
.brightness(1)
```

ನಾವು ಹೀಗೆ ಬರೆಯಬಹುದು:

```swift
colors[index]
    .brightness(0.5)
```

ಇದರಿಂದ ಪ್ರಸ್ತುತ ಬಣ್ಣ ಇನ್ನೂ ಹೆಚ್ಚು ಬೆಳಗುತ್ತದೆ ಮತ್ತು ಫಿಲ್ ಲೈಟ್ ಪರಿಣಾಮಕ್ಕೆ ಇನ್ನೂ ಹತ್ತಿರವಾಗುತ್ತದೆ.

ಪ್ರಕಾಶಮಾನತೆಯ ವ್ಯಾಪ್ತಿ `0 - 1` ಆಗಿದೆ. `0` ಅಂದರೆ ಮೂಲ ಬಣ್ಣವನ್ನು ಹಾಗೆಯೇ ಉಳಿಸುವುದು, ಮತ್ತು ಮೌಲ್ಯ `1` ಗೆ ಹತ್ತಿರವಾಗುವಷ್ಟೂ ಬಣ್ಣ ಇನ್ನೂ ಹೆಚ್ಚು ಬೆಳಗುತ್ತದೆ. `1` ಅಂದರೆ ಅತ್ಯಂತ ಪ್ರಕಾಶಮಾನವಾದ ಬಿಳಿ ಫಲಿತಾಂಶ.

ನಾವು ಕೋಡ್ ಮೂಲಕ `brightness` ನಿಯಂತ್ರಿಸಬಹುದಾದರೂ, ಬಳಕೆದಾರರು ಇನ್ನೂ ಅದನ್ನು ನೇರವಾಗಿ ಹೊಂದಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ.

ಆದ್ದರಿಂದ ನಾವು ಎಳೆಯಬಹುದಾದ ನಿಯಂತ್ರಣವನ್ನು ಸೇರಿಸಬೇಕು: `Slider`.

## `Slider` ವ್ಯೂ

SwiftUI ನಲ್ಲಿ, `Slider` ಒಂದು ನಿಯಂತ್ರಣವಾಗಿದ್ದು, ನಿರ್ದಿಷ್ಟ ವ್ಯಾಪ್ತಿಯೊಳಗಿನ ಮೌಲ್ಯವನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಬಳಸಲಾಗುತ್ತದೆ. Apple ಇದನ್ನು “ನಿರ್ದಿಷ್ಟ ರೇಖೀಯ ವ್ಯಾಪ್ತಿಯೊಳಗಿಂದ ಮೌಲ್ಯವನ್ನು ಆಯ್ಕೆ ಮಾಡುವ ನಿಯಂತ್ರಣ” ಎಂದು ವಿವರಿಸುತ್ತದೆ.

ಮೂಲ ಬಳಕೆ:

```swift
Slider(value: $value, in: 0...1)
```

ಪ್ಯಾರಾಮಿಟರ್ ವಿವರಣೆ:

1. `value: $value`: `Slider` ಒಂದು ವ್ಯರಿಯಬಲ್‌ಗೆ bind ಆಗಿರಬೇಕು.

    ಸ್ಲೈಡರ್ ಅನ್ನು ಎಳೆಯುವಾಗ, ವ್ಯರಿಯಬಲ್‌ನ ಮೌಲ್ಯವೂ ಅದೇ ಸಮಯದಲ್ಲಿ ಬದಲಾಗುತ್ತದೆ. ಹಾಗೆಯೇ, ವ್ಯರಿಯಬಲ್ ಬದಲಾಗಿದರೆ, ಸ್ಲೈಡರ್ ಕೂಡ ನವೀಕರಿಸುತ್ತದೆ.

    ಇದು ನಾವು ಹಿಂದಿನಲ್ಲೇ ಕಲಿತ `TextField` ಜೊತೆ ತುಂಬಾ ಹೋಲುತ್ತದೆ. ಎರಡೂ “ನಿಯಂತ್ರಣವನ್ನು ವ್ಯರಿಯಬಲ್‌ಗೆ bind ಮಾಡುತ್ತವೆ”.

    binding ಸೂಚಿಸಲು bind ಆಗಿರುವ ವ್ಯರಿಯಬಲ್‌ಗಿಂತ ಮೊದಲು `$` ಚಿಹ್ನೆ ಇರಬೇಕು.

2. `in: 0...1`: ಈ ಪ್ಯಾರಾಮಿಟರ್ ಸ್ಲೈಡರ್‌ನ ಮೌಲ್ಯದ ವ್ಯಾಪ್ತಿಯನ್ನು ಸೂಚಿಸುತ್ತದೆ.

    ಇಲ್ಲಿ `0...1` ಅಂದರೆ ಕನಿಷ್ಠ ಮೌಲ್ಯ `0` ಮತ್ತು ಗರಿಷ್ಠ ಮೌಲ್ಯ `1`.

    ಸ್ಲೈಡರ್ ಅನ್ನು ಸಂಪೂರ್ಣ ಎಡಕ್ಕೆ ಎಳೆದಾಗ, bind ಆಗಿರುವ ವ್ಯರಿಯಬಲ್ `0` ಗೆ ಹತ್ತಿರವಾಗುತ್ತದೆ; ಸಂಪೂರ್ಣ ಬಲಕ್ಕೆ ಎಳೆದಾಗ ಅದು `1` ಗೆ ಹತ್ತಿರವಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` `value` ವ್ಯರಿಯಬಲ್‌ಗೆ bind ಆಗಿದೆ. ಸ್ಲೈಡರ್ ಅನ್ನು ಎಳೆಯುವಾಗ, `value` ವ್ಯರಿಯಬಲ್‌ನ ಮೌಲ್ಯವೂ ಅದೇ ಸಮಯದಲ್ಲಿ ಬದಲಾಗುತ್ತದೆ.

ದೃಶ್ಯ:

![Slider](../../Resource/018_slider.png)

`Slider` ಅನ್ನು ಎಡಕ್ಕೆ ಸರಿಸಿದಾಗ, bind ಆಗಿರುವ `value` `0` ಆಗುತ್ತದೆ. ಬಲಕ್ಕೆ ಸರಿಸಿದಾಗ ಅದು `1` ಆಗುತ್ತದೆ.

### ಮೌಲ್ಯದ ವ್ಯಾಪ್ತಿ

`Slider` ನ ಮೌಲ್ಯದ ವ್ಯಾಪ್ತಿ ಸ್ಥಿರವಾಗಿರುವುದಿಲ್ಲ. ಅದನ್ನು ಹೀಗೆ ಕೂಡ ಬರೆಯಬಹುದು:

```swift
0...100
```

ಅಥವಾ ಬೇರೆ ಯಾವದಾದರೂ range ಆಗಿರಬಹುದು.

ಆದರೆ ಈ ಫಿಲ್ ಲೈಟ್ ಆಪ್‌ನಲ್ಲಿ, ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ನಿಯಂತ್ರಿಸಬೇಕಿರುವುದರಿಂದ `0...1` ಅತ್ಯಂತ ಸೂಕ್ತ ಆಯ್ಕೆ.

## `Slider` ಬಳಸಿ ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ನಿಯಂತ್ರಿಸುವುದು

ಈಗ `Slider` ಅನ್ನು `brightness` ಜೊತೆ ಕನೆಕ್ಟ್ ಮಾಡಬೇಕು.

ಮೊದಲು, ಪ್ರಕಾಶಮಾನತೆಯ ಮೌಲ್ಯವನ್ನು ಉಳಿಸಲು ಒಂದು ವ್ಯರಿಯಬಲ್ ಸೃಷ್ಟಿಸೋಣ:

```swift
@State private var slider = 0.0
```

ಇಲ್ಲಿ `0.0` ಒಂದು `Double` ವಿಧದ ಮೌಲ್ಯವಾಗಿದೆ.

`Slider` ಸಾಮಾನ್ಯವಾಗಿ ಸಂಖ್ಯಾತ್ಮಕ ವಿಧಕ್ಕೆ bind ಆಗಿರುತ್ತದೆ, ಮತ್ತು ಇಲ್ಲಿ ಅದು ನಿರಂತರವಾಗಿ ಬದಲಾಗಬೇಕು ಎಂದು ನಾವು ಬಯಸುವುದರಿಂದ `Double` ಹೆಚ್ಚು ಸೂಕ್ತವಾಗಿದೆ. ಜೊತೆಗೆ `brightness` ಕೇವಲ `Double` ಮೌಲ್ಯವನ್ನೇ ಸ್ವೀಕರಿಸುತ್ತದೆ.

ನಂತರ ಈ ಮೌಲ್ಯವನ್ನು `brightness` ಗೆ ನೀಡಿ:

```swift
colors[index]
    .brightness(slider)
```

`slider == 0` ಆಗಿದ್ದರೆ, ಬಣ್ಣ ತನ್ನ default ಸ್ಥಿತಿಯಲ್ಲೇ ಇರುತ್ತದೆ. `slider` `1` ಗೆ ಎಷ್ಟು ಹತ್ತಿರವಾಗುತ್ತದೆಯೋ, ಬಣ್ಣ ಅಷ್ಟು ಹೆಚ್ಚು ಬೆಳಗುತ್ತದೆ.

### `Slider` ನಿಯಂತ್ರಣವನ್ನು ಸೇರಿಸುವುದು

ಮುಂದೆ, ಈ ವ್ಯರಿಯಬಲ್ ಅನ್ನು ಬದಲಿಸಲು `Slider` ನಿಯಂತ್ರಣವನ್ನು ಸೇರಿಸಿ:

```swift
Slider(value: $slider, in: 0...1)
```

ಸ್ಲೈಡರ್ ಬದಲಾಗುವಾಗ, `slider` ಮೌಲ್ಯವೂ ಬದಲಾಗುತ್ತದೆ, ಮತ್ತು `brightness(slider)` ಅದೇ ಸಮಯದಲ್ಲಿ ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ನವೀಕರಿಸುತ್ತದೆ.

ಇದು SwiftUI ನಲ್ಲಿ “ವ್ಯರಿಯಬಲ್ ವ್ಯೂ ಅನ್ನು ಚಾಲನೆ ಮಾಡುತ್ತದೆ” ಎಂಬುದರ ಅತ್ಯಂತ ಸಾಮಾನ್ಯ ಉದಾಹರಣೆ.

### `Slider` ನ ದೃಷ್ಯವನ್ನು ಹೊಂದಿಸುವುದು

ಡಿಫಾಲ್ಟ್ ಆಗಿ, `Slider` ಲಭ್ಯವಿರುವ ಅಗಲವನ್ನು ಬಳಸುತ್ತದೆ.

ನಾವು ಅದಕ್ಕೆ ಒಂದು ಸ್ಥಿರ ಅಗಲವನ್ನು ಕೊಡಬಹುದು:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

ನಂತರ, ಅದನ್ನು ಇನ್ನೂ ಸ್ಪಷ್ಟವಾಗಿ ಕಾಣಲು ಕೆಲವು modifiers ಅನ್ನು ಸೇರಿಸೋಣ:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

ಇದರಿಂದ ಸ್ಲೈಡರ್‌ಗೆ ಬಿಳಿ ಹಿನ್ನೆಲೆ ಮತ್ತು ಗೋಳಾದ ಮೂಲೆಗಳು ಸಿಗುತ್ತವೆ, ಹಾಗಾಗಿ ಅದು `Color` ವ್ಯೂ ಮೇಲ್ಮೇಲೆ ಇನ್ನೂ ಸ್ಪಷ್ಟವಾಗಿ ಕಾಣಿಸುತ್ತದೆ.

ಕೊನೆಯಲ್ಲಿ, ಅದನ್ನು ಪರದೆಯ ಕೆಳಭಾಗದಲ್ಲಿ ಇರಿಸಿ.

ನಾವು ಈಗಾಗಲೇ `ZStack` ಬಳಸಿದ್ದರಿಂದ, ಅದರೊಳಗೆ `VStack` ಇಟ್ಟು `Spacer()` ಬಳಸಿ `Slider` ಅನ್ನು ಕೆಳಗೆ ತಳ್ಳಬಹುದು.

## ಸಂಪೂರ್ಣ ಕೋಡ್

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## ಸಾರಾಂಶ

ಹಿಂದೆ ಕಲಿತ ಜ್ಞಾನವನ್ನು ಬಣ್ಣಗಳು ಮತ್ತು ಅರೆಗಳಂತಹ ಮೂಲಭೂತ ಕಲ್ಪನೆಗಳೊಂದಿಗೆ ಬಳಸಿ, ನಾವು ತುಂಬಾ ಆಸಕ್ತಿದಾಯಕವಾದ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ನಿರ್ಮಿಸಿದ್ದೇವೆ.

ಈ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಮೂಲಕ, `brightness` ಬಳಸಿ ಪ್ರಕಾಶಮಾನತೆಯನ್ನು ಹೊಂದಿಸುವುದು, `onTapGesture` ಬಳಸಿ ವ್ಯೂಗಳಿಗೆ ಟ್ಯಾಪ್ ಕ್ರಿಯೆಗಳನ್ನು ಸೇರಿಸುವುದು, ಮತ್ತು `Slider` ನಿಯಂತ್ರಣವನ್ನು ಬಳಸುವುದು ಎಂಬುದನ್ನು ಕಲಿತಿದ್ದೇವೆ.

ಬಣ್ಣ ಬದಲಿಸಲು ನಾವು `Color` ವ್ಯೂಗೆ `onTapGesture` ಸೇರಿಸಿದ್ದೇವೆ. `brightness` ಅನ್ನು ನಿರ್ವಹಿಸುವ ವ್ಯರಿಯಬಲ್ ಅನ್ನು ಬದಲಿಸಲು `Slider` ನಿಯಂತ್ರಣವನ್ನೂ ಬಳಸಿದ್ದೇವೆ, ಇದು “ವ್ಯರಿಯಬಲ್ ವ್ಯೂ ಅನ್ನು ಚಾಲನೆ ಮಾಡುತ್ತದೆ” ಎಂಬುದಕ್ಕೆ ಮತ್ತೊಂದು ಉದಾಹರಣೆ.

ನಾವು ternary operator ಅನ್ನು ಮರುಪರಿಶೀಲಿಸಿದ್ದೇವೆ, ವ್ಯೂಗಳನ್ನು ಒಂದರ ಮೇಲೊಂದು ಇರಿಸಲು `ZStack` ಬಳಸಿದ್ದೇವೆ, ಮತ್ತು ಒಂದೇ ವಿಧದ ಡೇಟಾ ಗುಂಪನ್ನು ನಿರ್ವಹಿಸಲು ಅರೆಗಳನ್ನು ಬಳಸಿದ್ದೇವೆ. ಇದರಿಂದ ಅರೆಗಳು ಮತ್ತು ಸೂಚ್ಯಂಕಗಳ ಬಗ್ಗೆ ನಮ್ಮ ತಿಳುವಳಿಕೆ ಇನ್ನೂ ಆಳವಾಗುತ್ತದೆ, ವಿಶೇಷವಾಗಿ out-of-range ದೋಷಗಳನ್ನು ತಪ್ಪಿಸುವ ವ್ಯವಹಾರಿಕ ಸಮಸ್ಯೆಯ ವಿಷಯದಲ್ಲಿ.

ಈ ಉದಾಹರಣೆ ತುಂಬಾ ಜಟಿಲವಲ್ಲದಿದ್ದರೂ, ಇದು ನಾವು ಮೊದಲು ಕಲಿತ ಅನೇಕ ಮೂಲಭೂತ ಕಲ್ಪನೆಗಳನ್ನು ಜೋಡಿಸುತ್ತದೆ. ಅವುಗಳನ್ನು ಒಂದು ಸಣ್ಣ ನೈಜ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಬಳಸಿದಾಗ, ಪ್ರತಿಯೊಂದು ಕಲ್ಪನೆ ಯಾವ ಕೆಲಸಕ್ಕೆ ಉಪಯೋಗವಾಗುತ್ತದೆ ಎಂಬುದು ಇನ್ನೂ ಸುಲಭವಾಗಿ ಅರ್ಥವಾಗುತ್ತದೆ.

### ನೈಜ ಬಳಕೆಯ ದೃಶ್ಯ

ಒಂದು ಹಳೆಯ iPhone ಅನ್ನು ಮೇಜಿನ ಮೇಲೆ ಇಟ್ಟು, ನೀವು ತಯಾರಿಸಿದ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಮೂಲಕ ಬೆಳಕಿನ ಬಣ್ಣವನ್ನು ನಿಯಂತ್ರಿಸುತ್ತಿರುವುದನ್ನು ಊಹಿಸಿ. ಅದು ತುಂಬಾ ಒಳ್ಳೆಯ ಅನುಭವವಾಗಿರುತ್ತದೆ.

App Store ನಲ್ಲಿ ಅನೇಕ “ಫಿಲ್ ಲೈಟ್” ಆಪ್‌ಗಳು ಇವೆ, ಮತ್ತು ಅವುಗಳೂ ಹೆಚ್ಚಿನ ಜಟಿಲತೆ ಇರುವಂತಿಲ್ಲ.

![AppStore](../../Resource/018_appStore.PNG)

ನಾವು ಸರಳ ಆಪ್‌ಗಳಿಂದ ಆರಂಭಿಸಿ, ಅವನ್ನು App Store ಗೆ ಪ್ರಕಟಿಸಲು ಪ್ರಯತ್ನಿಸಬಹುದು. ಇದು ಅಭಿವೃದ್ಧಿಯ ಮೇಲಿನ ನಮ್ಮ ಆಸಕ್ತಿಯನ್ನು ಹೆಚ್ಚಿಸುವುದರ ಜೊತೆಗೆ, ನಮ್ಮ ಬೆಳವಣಿಗೆಯನ್ನೂ ದಾಖಲಿಸುತ್ತದೆ.

### ಪಾಠದ ನಂತರದ ಅಭ್ಯಾಸ

ಈ ಫಿಲ್ ಲೈಟ್ ಆಪ್ ಅನ್ನು ಇನ್ನೂ ಹೇಗೆ ವಿಸ್ತರಿಸಬಹುದು ಎಂದು ನೀವು ಯೋಚಿಸಬಹುದು, ಉದಾಹರಣೆಗೆ:

- ಇನ್ನಷ್ಟು ಬಣ್ಣಗಳನ್ನು ಸೇರಿಸಿ
- ಪ್ರಸ್ತುತ ಪ್ರಕಾಶಮಾನತೆಯ ಮೌಲ್ಯವನ್ನು ತೋರಿಸಿ
- ಕೆಳಭಾಗದ ಸ್ಲೈಡರ್ ಪ್ರದೇಶದ ವಿನ್ಯಾಸವನ್ನು ಸುಧಾರಿಸಿ

ನೀವು ಈ ಜ್ಞಾನವನ್ನು ನಿಜವಾಗಿಯೂ ಬಳಸಲು ಪ್ರಾರಂಭಿಸಿದಾಗ, ನಾವು ಕಲಿಯುವ ಪ್ರತಿಯೊಂದು ಕಲ್ಪನೆಯೂ ವಾಸ್ತವವಾಗಿ ಆಪ್ ನಿರ್ಮಾಣದ ಒಂದು ಸಾಧನ ಎಂಬುದು ನಿಮಗೆ ಗೊತ್ತಾಗುತ್ತದೆ.

ನಾವು ಎಷ್ಟು ಹೆಚ್ಚು ಸಾಧನಗಳನ್ನು ಕಲಿತಿದ್ದೇವೋ, ಅಷ್ಟು ಹೆಚ್ಚು ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ನಿರ್ಮಿಸಬಹುದು.
