# ଲୁପ୍ ଏବଂ ତାଲିକା

ଏହି ପାଠରେ, ଆମେ Swift ର `for` ଲୁପ୍ ଏବଂ SwiftUI ର `ForEach` ବିଷୟରେ ଶିଖିବୁ।

ଏହି ଦୁଇଟି ମଧ୍ୟ ଆମକୁ ଏକ ଆରେର ଉପାଦାନଗୁଡ଼ିକୁ କ୍ରମକ୍ରମେ ପଢ଼ିବାରେ ସାହାଯ୍ୟ କରେ। ବାସ୍ତବ ଡିଭେଲପ୍‌ମେଣ୍ଟରେ, ଆମେ ପ୍ରାୟତଃ ଏକ ଗୋଷ୍ଠୀ ଡାଟାକୁ ଏକେକରେ ନେଇ, ପୁଣି ସେଗୁଡ଼ିକୁ ପ୍ରୋସେସ କିମ୍ବା ଦେଖାଇବାକୁ ପଡ଼େ, ଏବଂ ଲୁପ୍ ହେଉଛି ଏହି କାମ ସମାପ୍ତ କରିବା ପାଇଁ ଗୁରୁତ୍ୱପୁର୍ଣ୍ଣ ଉପକରଣ।

ଆମେ ଯେତେବେଳେ ୱେବପେଜ ବ୍ରାଉଜ କରୁ କିମ୍ବା କୌଣସି App ବ୍ୟବହାର କରୁ, ପ୍ରାୟତଃ ବହୁତ ତାଲିକା କିମ୍ବା ଆଇଟମ୍ ଦେଖୁଥାଉ। ଏପରି ବିଷୟବସ୍ତୁ ସାଧାରଣତଃ ପ୍ରଥମେ `ଆରେ` ଭିତରେ ସଞ୍ଚୟ ହୁଏ, ତାପରେ `ଲୁପ୍` ଦ୍ୱାରା ଏକେକରେ ପଢ଼ାଯାଏ, ଏବଂ ଶେଷରେ ଇଣ୍ଟରଫେସ୍‌ରେ ଦେଖାଯାଏ।

ଉଦାହରଣ ସ୍ୱରୂପ, Google ର ସର୍ଚ୍ଚ ଫଳାଫଳ ତାଲିକା:

![Google](../../Resource/017_google.png)

ସର୍ଚ୍ଚ ଫଳାଫଳ ତାଲିକାର ପ୍ରତ୍ୟେକ ଆଇଟମ୍ ସାଧାରଣତଃ ଗୋଟିଏ ଡାଟା ସହ ସମ୍ବନ୍ଧିତ ହୁଏ, ଏବଂ ଲୁପ୍ ମାଧ୍ୟମରେ ଏକେକରେ ଦେଖାଯାଏ।

ସେହିହେତୁ, `ଆରେ` ଏବଂ `ଲୁପ୍` ପ୍ରାୟତଃ ସହିତେ ଦେଖାଯାଏ। `ଆରେ` ଗୋଟିଏ ଗୋଷ୍ଠୀ ଡାଟାକୁ ସଞ୍ଚୟ କରେ, ଏବଂ `ଲୁପ୍` ସେହି ଡାଟାକୁ କ୍ରମକ୍ରମେ ନେଇ ସମାନ କାର୍ଯ୍ୟ କରେ।

ଏବେ, ଆମେ ସବୁଠାରୁ ମୂଳଭୂତ `for` ଲୁପ୍ ଠାରୁ ଆରମ୍ଭ କରିବା।

## କାଉଣ୍ଟଡାଉନ୍

ଧରନ୍ତୁ ଆମେ ଗୋଟିଏ କାଉଣ୍ଟଡାଉନ୍ ଫଙ୍କ୍ସନ୍ ଅନୁକରଣ କରିବାକୁ ଚାହୁଁଛୁ ଏବଂ କ୍ରମକ୍ରମେ `“5、4、3、2、1”` ଆଉଟପୁଟ୍ କରିବାକୁ ଚାହୁଁଛୁ।

ସବୁଠାରୁ ସହଜ ଭାବରେ ଏହା ଏଭଳି ଲେଖାଯାଏ:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

ଏହି ପ୍ରକ୍ରିୟାରେ କାଉଣ୍ଟଡାଉନ୍ ଫଙ୍କ୍ସନ୍ ଦେଖାଯାଇପାରେ, କିନ୍ତୁ ବିଷୟବସ୍ତୁ ଅଧିକ ହେଲେ ଏହା ବହୁତ ପୁନରାବୃତ୍ତିଶୀଳ ଲାଗେ, ଏବଂ ପରେ ସମ୍ପାଦନ କରିବାକୁ ମଧ୍ୟ ସୁବିଧାଜନକ ନୁହେଁ।

ଏହି ସମୟରେ, ଆମେ ପ୍ରଥମେ ଡାଟାକୁ ଆରେରେ ରଖି, ପରେ `for` ଲୁପ୍ ବ୍ୟବହାର କରି କ୍ରମକ୍ରମେ ଆଉଟପୁଟ୍ କରିପାରିବା:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ଏହି ଲେଖା ପ୍ରକ୍ରିୟା ଅଧିକ ସଂକ୍ଷିପ୍ତ ଏବଂ ବିସ୍ତାର କରିବାକୁ ମଧ୍ୟ ସହଜ। କାଉଣ୍ଟଡାଉନ୍‌ରେ ଯେତେ ନମ୍ବର ଥାଉନାହିଁ, ଆମକୁ `print` କୁ ଲାଇନ୍ ପରେ ଲାଇନ୍ ଲେଖିବାକୁ ପଡ଼ିବ ନାହିଁ; କେବଳ ଲୁପ୍‌କୁ ଆରେର ପ୍ରତ୍ୟେକ ଉପାଦାନକୁ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ପ୍ରୋସେସ କରିବାକୁ ଦେବା ଯଥେଷ୍ଟ।

ଆମେ ଏହି କୋଡ୍‌କୁ ବଟନ୍‌ର ଭିତରେ ରଖି ପରୀକ୍ଷା କରିପାରିବା:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

ବଟନ୍‌କୁ ଦବାଇଲେ, `for` ଲୁପ୍ ଆରେର କ୍ରମ ଅନୁଯାୟୀ `countDown` ଭିତରୁ ପ୍ରତ୍ୟେକ ପୁର୍ଣ୍ଣସଂଖ୍ୟାକୁ ନେବ ଏବଂ ସେଗୁଡ଼ିକୁ କନସୋଲ୍‌ରେ ପ୍ରିଣ୍ଟ୍ କରିବ।

ଏବେ ଆମେ `for` ଲୁପ୍ କିପରି କାମ କରେ ତାହା ଦେଖିବା।

## for ଲୁପ୍

`for` ଲୁପ୍ ଏକ ଗୋଷ୍ଠୀ ଡାଟାକୁ କ୍ରମକ୍ରମେ ପ୍ରୋସେସ କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ। ସବୁଠାରୁ ସାଧାରଣ ବସ୍ତୁ ହେଉଛି ଆରେ ଏବଂ ରେଞ୍ଜ୍।

ମୂଳ ଲେଖନ ଶୈଳୀ:

```swift
for ଉପାଦାନ in ଆରେ କିମ୍ବା ରେଞ୍ଜ୍ {
    ଚାଲାଇବାକୁଥିବା କୋଡ୍
}
```

ଉଦାହରଣ ସ୍ୱରୂପ, ଆଗର କାଉଣ୍ଟଡାଉନ୍ କୋଡ୍:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ଏଠାରେ `i` ର ଅର୍ଥ ହେଉଛି ବର୍ତ୍ତମାନ ନିଆଯାଇଥିବା ଉପାଦାନ, ଏବଂ `in` ପରେ ଥିବା ଡାଟା ହେଉଛି ଯାହା ଉପରେ ଲୁପ୍ ଚାଲିବ, ଅର୍ଥାତ୍ `countDown` ଆରେ।

ଏହି କୋଡ୍‌ର ଅର୍ଥ ହେଉଛି: ପ୍ରୋଗ୍ରାମ୍ `countDown` ଆରେରୁ କ୍ରମକ୍ରମେ ପ୍ରତ୍ୟେକ ଉପାଦାନକୁ ନେବ। ପ୍ରତ୍ୟେକ ଥର ଗୋଟିଏ ଉପାଦାନ ନେଲେ, ତାହାକୁ ଅସ୍ଥାୟୀ ଭାବରେ `i` ଦ୍ୱାରା ପ୍ରତିନିଧିତ୍ୱ କରିବ, ତାପରେ ବଡ଼ ବ୍ରାକେଟ୍‌ଭିତରର କୋଡ୍‌କୁ ଚାଲାଇବ।

ଚାଳନ ପ୍ରକ୍ରିୟାକୁ ଏଭଳି ବୁଝିପାରିବା:

* ପ୍ରଥମେ ନିଆଯାଏ `5`, ସେହିପରି ଏହି ସମୟରେ `i` ର ମୂଲ୍ୟ `5`
* `print(i)` ଚାଲିବ ଏବଂ `5` ଆଉଟପୁଟ୍ ହେବ
* ପରେ ପରବର୍ତ୍ତୀ ଉପାଦାନ `4` କୁ ନିଆଯିବ
* ପୁଣିଥରେ `print(i)` ଚାଲିବ, ଏବଂ `4` ଆଉଟପୁଟ୍ ହେବ

ପରବର୍ତ୍ତୀ `3`、`2`、`1` ମଧ୍ୟ ଏହିଭଳି କ୍ରମକ୍ରମେ ଚାଲିବ।

![for](../../Resource/017_for.png)

ଆରେର ସମସ୍ତ ଉପାଦାନ ପ୍ରୋସେସ୍ ହୋଇଗଲେ, `for` ଲୁପ୍ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ଶେଷ ହେବ।

ଆମେ `for` ଲୁପ୍ ବ୍ୟବହାର କରି ଆରେର ପ୍ରତ୍ୟେକ ଉପାଦାନକୁ ନେଇ, କାଉଣ୍ଟଡାଉନ୍ ପରି ଫଙ୍କ୍ସନ୍ ସମ୍ପୂର୍ଣ୍ଣ କରିପାରିବା।

**ସୂଚନା**: `i` ହେଉଛି for ଲୁପ୍‌ରେ ବହୁତ ସାଧାରଣ ନାମକରଣ ଶୈଳୀ, ଯାହା ବର୍ତ୍ତମାନ ନିଆଯାଇଥିବା ଉପାଦାନକୁ ସୂଚାଏ, କିନ୍ତୁ ଏହା ନିଶ୍ଚିତ ଲେଖନ ନୁହେଁ; ଅନ୍ୟ ଇଂରାଜୀ ନାମ ମଧ୍ୟ ବ୍ୟବହାର କରିପାରିବେ।

ଉଦାହରଣ ସ୍ୱରୂପ, ପ୍ରତ୍ୟେକ ନିଆଯାଇଥିବା ଉପାଦାନକୁ ସୂଚାଇବା ପାଇଁ `num` ବ୍ୟବହାର କରାଯାଇପାରେ:

```swift
for num in countDown {
    print(num)
}
```

ବଡ଼ ବ୍ରାକେଟ୍‌ର ଭିତରେ ମଧ୍ୟ, ବର୍ତ୍ତମାନ ନିଆଯାଇଥିବା ଉପାଦାନକୁ ସୂଚିତ କରିବା ପାଇଁ `num` ବ୍ୟବହାର କରିବାକୁ ପଡ଼ିବ।

### ରେଞ୍ଜ୍ ଉପରେ ଲୁପ୍

`for` ଲୁପ୍ କେବଳ ଆରେ ଉପରେ ନୁହେଁ, ରେଞ୍ଜ୍ ଉପରେ ମଧ୍ୟ ଚାଲିପାରେ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଯେତେବେଳେ ଆମକୁ `50` ଭିତରର ସମସ୍ତ ଧନାତ୍ମକ ପୁର୍ଣ୍ଣସଂଖ୍ୟାର ଯୋଗଫଳ ଗଣନା କରିବାକୁ ଥାଏ, ସେତେବେଳେ `1` ଠାରୁ `50` ପର୍ଯ୍ୟନ୍ତ ସମସ୍ତ ପୁର୍ଣ୍ଣସଂଖ୍ୟାକୁ ଏକେକରେ ଯୋଗ କରିବାକୁ ପଡ଼େ।

ସବୁଠାରୁ ସିଧାସଳଖ ଲେଖା ଏଭଳି ହୋଇପାରେ:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

ଏହି ଲେଖାର ସମସ୍ୟା ହେଉଛି, ଆମେ ପ୍ରତ୍ୟେକ ସଂଖ୍ୟାକୁ ହାତରେ ଲେଖିବାକୁ ପଡ଼େ, ଯାହା ଜଟିଳ ଏବଂ ପ୍ରାକୃତିକ ଡିଭେଲପ୍‌ମେଣ୍ଟ ପରିସ୍ଥିତି ପାଇଁ ଉଚିତ ନୁହେଁ।

ଏହି ସମୟରେ, ଆମେ ରେଞ୍ଜ୍ ଅପରେଟର୍ ସହ `for` ଲୁପ୍‌କୁ ଏକାସାଥିରେ ବ୍ୟବହାର କରି ଏହାକୁ ସମାପ୍ତ କରିପାରିବା।

ପୂର୍ବରୁ “ଅପରେଟର୍” ଅଧ୍ୟାୟରେ, ଆମେ ରେଞ୍ଜ୍ ଅପରେଟର୍ ବିଷୟରେ ଶିଖିଥିଲୁ:

```swift
1...3   // 1, 2, 3
```

ଏଠାରେ `...` ର ଅର୍ଥ ହେଉଛି ଏକ ବନ୍ଦ ରେଞ୍ଜ୍, ଅର୍ଥାତ୍ ଆରମ୍ଭ ଏବଂ ଶେଷ ଉଭୟକୁ ସମେତ କରେ।

ତେଣୁ, ଏଠାରେ `1` ଠାରୁ `50` ପର୍ଯ୍ୟନ୍ତ ପୁର୍ଣ୍ଣସଂଖ୍ୟାର ରେଞ୍ଜ୍ ଉପରେ `for` ଲୁପ୍ ବ୍ୟବହାର କରାଯାଇପାରେ:

```swift
for i in 1...50 {
    
}
```

ଏହି ସଂଖ୍ୟାଗୁଡ଼ିକୁ ଯୋଗ କରିବା ପାଇଁ, ଆମକୁ ମୋଟ ଫଳକୁ ସଞ୍ଚୟ କରିବା ପାଇଁ ଆଉ ଗୋଟିଏ ଭେରିଏବଲ୍ ଆବଶ୍ୟକ:

```swift
var count = 0
```

ତାପରେ ପ୍ରତ୍ୟେକ ଲୁପ୍‌ରେ, ବର୍ତ୍ତମାନ ନିଆଯାଇଥିବା ସଂଖ୍ୟାକୁ `count` ସହ ଯୋଗ କରିବା:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ:

* `count` ବର୍ତ୍ତମାନର ମୋଟ ଯୋଗଫଳକୁ ସଞ୍ଚୟ କରିବା ପାଇଁ
* `for` ଲୁପ୍ `1` ଠାରୁ `50` ପର୍ଯ୍ୟନ୍ତ ପୁର୍ଣ୍ଣସଂଖ୍ୟାଗୁଡ଼ିକୁ କ୍ରମକ୍ରମେ ନେବ
* ପ୍ରତ୍ୟେକଥର ଗୋଟିଏ ପୁର୍ଣ୍ଣସଂଖ୍ୟା ନିଆଯାଇଲେ, `count += i` ଏକଥର ଚାଲିବ
* ଲୁପ୍ ଶେଷ ହେଲେ, `count` ଭିତରେ ଶେଷ ମୋଟ ଫଳ ରହିବ।

ଏଭଳି, ଆମେ `50` ଭିତରର ସମସ୍ତ ଧନାତ୍ମକ ପୁର୍ଣ୍ଣସଂଖ୍ୟାର ଯୋଗଫଳ ପାଇପାରିବା।

ସେହିଭଳି, ଯଦି ଆମକୁ `1` ଠାରୁ `100` ପର୍ଯ୍ୟନ୍ତ ପୁର୍ଣ୍ଣସଂଖ୍ୟାର ଯୋଗଫଳ ଗଣନା କରିବାକୁ ଥାଏ, ତେବେ କେବଳ ରେଞ୍ଜ୍ ପରିବର୍ତ୍ତନ କରିଲେ ହେବ:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` ଲୁପ୍ କେବଳ ଆରେ ଉପରେ ନୁହେଁ, ଗୋଟିଏ ରେଞ୍ଜ୍ ଉପରେ ମଧ୍ୟ ଚାଲିପାରେ। ଡାଟାକୁ କ୍ରମକ୍ରମେ ପୁନରାବୃତ୍ତିଭାବେ ପ୍ରୋସେସ୍ କରିବାର ଏହି କ୍ଷମତା ଡିଭେଲପ୍‌ମେଣ୍ଟରେ ବହୁତ ସାଧାରଣ।

ଏହା ଆମକୁ ବହୁତ ପୁନରାବୃତ୍ତିଶୀଳ କାମ ସ୍ୱୟଂଚାଳିତ ଭାବରେ କରିବାରେ ସାହାଯ୍ୟ କରେ, ଏବଂ ପ୍ରତ୍ୟେକ ସଂଖ୍ୟାର ଯୋଗ ପ୍ରକ୍ରିୟା କିମ୍ବା ଏହାପରି ଅନ୍ୟ କାମଗୁଡ଼ିକୁ ହାତରେ ଲେଖିବାକୁ ପଡ଼େ ନାହିଁ।

## SwiftUI ରେ ଲୁପ୍

ଆଗର `for` ଲୁପ୍ Swift ଭାଷାର ଅଂଶ।

କିନ୍ତୁ SwiftUI ରେ, ଆମେ view ଗଠନର ଭିତରେ ସିଧାସଳଖ `for` ବ୍ୟବହାର କରି view ସୃଷ୍ଟି କରିପାରିବୁ ନାହିଁ:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

ଏହାର କାରଣ ହେଉଛି `VStack` ଭିତରେ SwiftUI view ଆବଶ୍ୟକ, କିନ୍ତୁ ସାଧାରଣ `for` ଲୁପ୍ ନିଜେ ଗୋଟିଏ view ନୁହେଁ।

ଯଦି ଆମେ SwiftUI ରେ ଆରେର ବିଷୟବସ୍ତୁ ଅନୁସାରେ ପୁନରାବୃତ୍ତିଭାବେ UI ସୃଷ୍ଟି କରିବାକୁ ଚାହୁଁ, ତେବେ `ForEach` ବ୍ୟବହାର କରିବାକୁ ପଡ଼ିବ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଯଦି ଆମେ ଗୋଟିଏ ଦଳର ସମସ୍ତ ସଦସ୍ୟଙ୍କ ନାମ ଦେଖାଇବାକୁ ଚାହୁଁ, ଏଭଳି ଲେଖିପାରିବା:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

ଦେଖାଯାଇଥିବା ଫଳାଫଳ:

![for1](../../Resource/017_for1.png)

ତାପରେ, ଆମେ ପ୍ରତ୍ୟେକ `Text` କୁ ସମାନ modifier ଯୋଡ଼ିବା:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ଏବେ, ପ୍ରତ୍ୟେକ `Text` ଛୋଟ ଏବଂ ମୋଟା ଫଣ୍ଟ୍‌ରେ ଦେଖାଯାଉଛି।

ଦେଖାଯାଇଥିବା ଫଳାଫଳ:

![](../../Resource/017_for2.png)

modifier ଗୁଡ଼ିକ ଇଣ୍ଟରଫେସ୍‌କୁ ଅଧିକ ସୁନ୍ଦର କରେ, କିନ୍ତୁ ସେଗୁଡ଼ିକ କୋଡ୍‌କୁ ଅଧିକ ଲମ୍ବା ଏବଂ ପୁନରାବୃତ୍ତିଶୀଳ କରିଦେଏ।

ଯଦି ନାମ ଅଧିକ ହେବ, କିମ୍ବା ପରେ ଫଣ୍ଟ୍, ରଙ୍ଗ ଇତ୍ୟାଦି ଶୈଳୀକୁ ଏକାସାଥିରେ ପରିବର୍ତ୍ତନ କରିବାକୁ ଥାଏ, ତେବେ ବହୁତ ସମାନ କୋଡ୍ ପୁଣିଥରେ ଲେଖିବା ଏବଂ ବଦଳାଇବାକୁ ପଡ଼ିବ, ଯାହା ଅସୁବିଧାଜନକ ଏବଂ ରକ୍ଷାଣାବେକ୍ଷଣ ପାଇଁ ମଧ୍ୟ ଭଲ ନୁହେଁ।

ଏହି ସମୟରେ, ଆମେ ପ୍ରଥମେ ଏହି ନାମଗୁଡ଼ିକୁ ଆରେରେ ରଖିପାରିବା, ତାପରେ SwiftUI ର `ForEach` ବ୍ୟବହାର କରି, ଆରେର ଡାଟା ଅନୁଯାୟୀ ପୁନରାବୃତ୍ତିଭାବେ view ସୃଷ୍ଟି କରିପାରିବା:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

ଏହି କୋଡ୍ ଅଧିକ ସଂକ୍ଷିପ୍ତ। `ForEach` ଆରେର ପ୍ରତ୍ୟେକ ଉପାଦାନକୁ କ୍ରମକ୍ରମେ ପଢ଼ିବ ଏବଂ ସମ୍ବନ୍ଧିତ ବିଷୟବସ୍ତୁକୁ ଇଣ୍ଟରଫେସ୍‌ରେ ଦେଖାଇବ।

ପରେ ଯେତେବେଳେ ଆମେ ନୂତନ ନାମ ଯୋଡ଼ିବୁ, କେବଳ `names` ଆରେର ନାମଗୁଡ଼ିକୁ ବଦଳାଇଲେ ହେବ; ପ୍ରତିଥର `Text` ଏବଂ ଜଟିଳ modifier ଯୋଡ଼ିବାକୁ ପଡ଼ିବ ନାହିଁ।

## ForEach

`ForEach` ହେଉଛି SwiftUI ରେ ପୁନରାବୃତ୍ତିଭାବେ view ସୃଷ୍ଟି କରିବା ପାଇଁ ବ୍ୟବହୃତ ଗଠନ। ଏହାକୁ SwiftUI ର ଏକ `for` ଲୁପ୍ ସଦୃଶ ଗଠନ ଭାବେ ବୁଝିପାରିବେ।

ମୂଳ ବ୍ୟବହାର:

```swift
ForEach(ଆରେ, id: \.self) { ଉପାଦାନ in
    SwiftUI କୋଡ୍
}
```

ଉଦାହରଣ ସ୍ୱରୂପ, ଏମିତି ସମୁହର ନାମଗୁଡ଼ିକ:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ:

`names` ହେଉଛି ଲୁପ୍ ହେବାକୁଥିବା ଆରେ, ଅର୍ଥାତ୍ ଯେଉଁ ଡାଟା ଦେଖାଯିବ।

`id:\.self` ର ଅର୍ଥ ହେଉଛି ଆରେର ଉପାଦାନକୁ ନିଜକୁ ହିଁ ପ୍ରତ୍ୟେକ ଆଇଟମ୍‌କୁ ପୃଥକ କରିବା ପାଇଁ ବ୍ୟବହାର କରାଯିବ।

ବର୍ତ୍ତମାନ ଉଦାହରଣରେ, ଆରେରେ string ସଞ୍ଚୟ ହୋଇଛି:

```swift
["Sam", "John", "Wathon", "Bob"]
```

ତେଣୁ SwiftUI ସିଧାସଳଖ "Sam"、"John"、"Wathon"、"Bob" ଏହି string ଗୁଡ଼ିକୁ ନିଜେ ପୃଥକ କରିବା ପାଇଁ ବ୍ୟବହାର କରିବ।

ପ୍ରାରମ୍ଭିକ ଚର୍ଯ୍ୟାରେ ସାଧାରଣ `String`、`Int` ପରି ସରଳ ଆରେ ପାଇଁ, ଆମେ ସାଧାରଣତଃ ପ୍ରଥମେ ଏଭଳି ଲେଖିପାରିବା:

```swift
id: \.self
```

ଏବେ ପାଇଁ କେବଳ ଏହି ଲେଖନ ଶୈଳୀକୁ ମନେ ରଖିଲେ ହେବ; id ବିଷୟରେ ଆଉ ଅଧିକ ନିୟମ ଗଭୀରରେ ବୁଝିବା ଆବଶ୍ୟକ ନାହିଁ।

`item in` ରେ `item` ମାନେ ବର୍ତ୍ତମାନ ନିଆଯାଇଥିବା ଉପାଦାନ।

ଉଦାହରଣ ସ୍ୱରୂପ, ପ୍ରଥମ ଲୁପ୍‌ରେ `item` ହେଉଛି `"Sam"` ଏବଂ ଦ୍ୱିତୀୟ ଲୁପ୍‌ରେ `item` ହେଉଛି `"John"`।

ସେହିପରି:

```swift
Text(item)
```

କ୍ରମକ୍ରମେ ଏହା ହେବ:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

ତେଣୁ, ଏହି କୋଡ୍‌ର କାମ ହେଉଛି: `names` ଆରେରୁ କ୍ରମକ୍ରମେ ପ୍ରତ୍ୟେକ ନାମକୁ ନେଇ, ପ୍ରତ୍ୟେକ ନାମ ପାଇଁ ଗୋଟିଏ `Text` view ସୃଷ୍ଟି କରିବା।

### ଚାଳନ ପ୍ରକ୍ରିୟା

ଚାଳନ ପ୍ରକ୍ରିୟାକୁ ଏଭଳି ବୁଝିପାରିବା:

* `ForEach` ପ୍ରଥମେ `names` ଆରେର ପ୍ରଥମ ଉପାଦାନ `"Sam"` କୁ ପଢ଼େ
* `item` ଅସ୍ଥାୟୀ ଭାବରେ `"Sam"` କୁ ପ୍ରତିନିଧିତ୍ୱ କରେ
* `Text(item)` `"Sam"` କୁ ଦେଖାଏ
* ପରେ ପରବର୍ତ୍ତୀ ଉପାଦାନ `"John"` କୁ ପଢ଼େ
* ତାପରେ ସେହି ଏକେଇ view କୋଡ୍‌କୁ ପୁଣି ଚାଲାଏ

ଶେଷରେ, ଆରେର ସମସ୍ତ ନାମ ଦେଖାଯିବ।

ଏଭଳି ଲେଖିବାର ଲାଭ ହେଉଛି: ଯେତେବେଳେ ନାମର ସଂଖ୍ୟା ବଢ଼େ, ଆମକୁ ବହୁତ `Text` ପୁଣିଥରେ ଲେଖିବାକୁ ପଡ଼େ ନାହିଁ; କେବଳ ଆରେର ବିଷୟବସ୍ତୁକୁ ବଦଳାଇଲେ ହେବ।

### ଫଟୋ ଉପରେ ଲୁପ୍

`ForEach` କେବଳ ଟେକ୍ସ୍ଟ୍ ନୁହେଁ, ଛବି, ରଙ୍ଗ କିମ୍ବା ଅନ୍ୟ SwiftUI view ମଧ୍ୟ ଦେଖାଇପାରେ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଆମେ 4ଟି ଛବି ପ୍ରସ୍ତୁତ କରିଛୁ:

FEE2AB8D.jpeg

ଏହି ଚାରିଟି ଛବିକୁ Xcode ର Assets ଫୋଲ୍ଡରକୁ ଟାଣି ଆଣନ୍ତୁ।

![image](../../Resource/017_for3.png)

ଯଦି `ForEach` ବ୍ୟବହାର ନକରିବା, ତେବେ ଆମକୁ ପ୍ରତ୍ୟେକ ଛବିକୁ ହାତରେ ଲେଖିବାକୁ ପଡ଼ିବ:

```swift
struct ContentView: View {
    var body: some View {
        VStack {

                .resizable()
                .scaledToFit()

            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

ଯଦି ଛବି ଆଉ ଅଧିକ ହେବ, ତେବେ ଆମକୁ ଆଉ ଅଧିକ `Image` କୋଡ୍ ପୁଣିଥରେ ଲେଖିବାକୁ ପଡ଼ିବ।

ପରେ ଯଦି ଆମେ ସମାନ ଭାବରେ ଛବିର ଆକାର ବଦଳାଇବାକୁ ଚାହୁଁ, ତେବେ ଏକେକରେ ପରିବର୍ତ୍ତନ କରିବାକୁ ହେବ।

ଏହି ସମୟରେ, ଆମେ ଛବିର ନାମଗୁଡ଼ିକୁ ଆରେରେ ରଖି, ପରେ `ForEach` ବ୍ୟବହାର କରି ପୁନରାବୃତ୍ତିଭାବେ ଛବି view ସୃଷ୍ଟି କରିପାରିବା:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

ଏଠାରେ `images` ହେଉଛି ଗୋଟିଏ string ଆରେ, ଏବଂ ଆରେର ପ୍ରତ୍ୟେକ ଉପାଦାନ ହେଉଛି ଗୋଟିଏ ଛବିର ନାମ।

`ForEach(images, id: \.self)` ର ଅର୍ଥ ହେଉଛି, `images` ଆରେର ପ୍ରତ୍ୟେକ ଉପାଦାନକୁ କ୍ରମକ୍ରମେ ପଢ଼ିବା, ଉପାଦାନକୁ ନିଜକୁ ହିଁ ପ୍ରତ୍ୟେକ ଆଇଟମ୍‌କୁ ପୃଥକ କରିବା ପାଇଁ ବ୍ୟବହାର କରିବା, ଏବଂ ବର୍ତ୍ତମାନ ଉପାଦାନ ଅନୁଯାୟୀ ସମ୍ବନ୍ଧିତ view ସୃଷ୍ଟି କରିବା।

ଉଦାହରଣ ସ୍ୱରୂପ, ପ୍ରଥମେ ପଢ଼ାଯାଇଥିବା `"430F9BEF"` ହେଲେ:

```swift
Image(item)
```

ଏହାର ଅର୍ଥ ହେଉଛି:

```swift
Image("430F9BEF")
```

ପରବର୍ତ୍ତୀ ଛବିଗୁଡ଼ିକ ମଧ୍ୟ ଏହିଭଳି କ୍ରମକ୍ରମେ ଦେଖାଯିବ।

![image](../../Resource/017_for4.png)

ଏଠାରେ ମଧ୍ୟ ଆମେ ଏହାକୁ ବ୍ୟବହାର କରୁଛୁ:

```swift
id: \.self
```

କାରଣ ପୂର୍ବବତ୍ ସେଇ, `images` ଆରେର ଉପାଦାନମାନେ ମଧ୍ୟ ସରଳ string ପ୍ରକାର, ତେଣୁ ପ୍ରାରମ୍ଭିକ ଚର୍ଯ୍ୟାରେ ସିଧାସଳଖ ଉପାଦାନକୁ ନିଜକୁ ପୃଥକ କରିବାର ଉପାୟ ଭାବରେ ବ୍ୟବହାର କରିପାରିବା।

ଏହାକୁ ଗୋଟିଏ ନିଶ୍ଚିତ ସୂତ୍ର ଭାବରେ ସରଳ ଭାବରେ ବୁଝିପାରିବେ:

**ଯେତେବେଳେ `ForEach` ଗୋଟିଏ `String` କିମ୍ବା `Int` ପରି ସରଳ ଆରେ ଉପରେ ଲୁପ୍ କରେ, ସାଧାରଣତଃ ଆମେ `id: \.self` ଲେଖୁ।**

### ନୂତନ ଛବି ଯୋଡ଼ିବା

ପରେ ଯଦି ଆମକୁ ନୂତନ ଛବି ଯୋଡ଼ିବାକୁ ଥାଏ, ତେବେ କେବଳ ଆରେର ବିଷୟବସ୍ତୁକୁ ବଦଳାଇଲେ ହେବ:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

ଆମକୁ ପୁଣିଥରେ ନୂତନ `Image` କୋଡ୍ ଲେଖିବାକୁ ପଡ଼ିବ ନାହିଁ।

ଯଦି ଛବି ଅଧିକ ଥାଏ, ତେବେ `ScrollView` ସହିତ ଏହାକୁ ବ୍ୟବହାର କରିପାରିବେ; ନହେଲେ ବିଷୟବସ୍ତୁ ସ୍କ୍ରିନ୍‌ର ସୀମା ଛାଡ଼ିଦେଲେ, ପୂର୍ଣ୍ଣ ଭାବେ ଦେଖିପାରିବେ ନାହିଁ।

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### ଛବି ପରିବର୍ତ୍ତନ

ଯେତେବେଳେ ଆମକୁ ସମାନ ଭାବରେ ଛବିର ଶୈଳୀ ବଦଳାଇବାକୁ ଥାଏ, କେବଳ `ForEach` ଭିତରର କୋଡ୍‌କୁ ବଦଳାଇଲେ ହେବ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଚଉଡ଼ାକୁ `80` କରିବା:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

ଏଭଳି, ସମସ୍ତ ଛବି ଏକାସାଥିରେ ପରିବର୍ତ୍ତିତ ହେବ, ଏକେକରେ ବଦଳାଇବାକୁ ହେବ ନାହିଁ।

ଏହାହିଁ `ForEach` ର ଗୁରୁତ୍ୱପୁର୍ଣ୍ଣ କାମ ମଧ୍ୟରୁ ଗୋଟିଏ:

**ଯେତେବେଳେ ଅନେକ view ର ଗଠନ ସମାନ ଥାଏ ଏବଂ କେବଳ ବିଷୟବସ୍ତୁ ଅଲଗା ଥାଏ, ସେତେବେଳେ ଆପଣ କେବଳ ଗୋଟିଏ view କୋଡ୍ ଲେଖିପାରିବେ, ଏବଂ ଆରେର ଡାଟା କଣ ଦେଖାଯିବ ତାହା ନିର୍ଣ୍ଣୟ କରିବ।**

## ସାରାଂଶ

ଏହି ପାଠରେ, ଆମେ `for` ଲୁପ୍ ଏବଂ `ForEach` ବିଷୟରେ ଶିଖିଲୁ।

`for` ଲୁପ୍ ଆରେ କିମ୍ବା ରେଞ୍ଜ୍‌ର ଉପାଦାନଗୁଡ଼ିକୁ କ୍ରମକ୍ରମେ ପ୍ରୋସେସ୍ କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ଏବଂ ଏହା ଡାଟା ପ୍ରୋସେସିଂ ପ୍ରତି ଅଧିକ ଝୁକିତ।

`ForEach` SwiftUI ରେ ଆରେର ବିଷୟବସ୍ତୁ ଅନୁଯାୟୀ ପୁନରାବୃତ୍ତିଭାବେ view ସୃଷ୍ଟି କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ଏବଂ ଏହା view ଦେଖାଇବା ପ୍ରତି ଅଧିକ ଝୁକିତ।

ବର୍ତ୍ତମାନର ପ୍ରାରମ୍ଭିକ ଚର୍ଯ୍ୟାରେ, `ForEach` ର ସବୁଠାରୁ ସାଧାରଣ ଲେଖନ ଶୈଳୀ ହେଉଛି:

```swift
ForEach(ଆରେ, id: \.self) { ଉପାଦାନ in
    SwiftUI କୋଡ୍
}
```

ଏଠାରେ, `id: \.self` ର ଅର୍ଥ ହେଉଛି ଉପାଦାନକୁ ନିଜକୁ ପ୍ରତ୍ୟେକ ଆଇଟମ୍‌କୁ ପୃଥକ କରିବା ପାଇଁ ବ୍ୟବହାର କରିବା।

ଯେତେବେଳେ ଆରେର ବିଷୟବସ୍ତୁ `String`、`Int` ପରି ସରଳ ପ୍ରକାରର ହୁଏ, ଆମେ ସାଧାରଣତଃ ପ୍ରଥମେ ଏହି ଏକମାତ୍ର ଲେଖନ ଶୈଳୀକୁ ବ୍ୟବହାର କରିପାରିବା।

ଭବିଷ୍ୟତ୍‌ରେ, ୱେବପେଜ୍ କିମ୍ବା App ବ୍ରାଉଜ କରିବା ବେଳେ ଯଦି ଆପଣ ଗୋଟିଏ ଗୋଷ୍ଠୀ ପୁନରାବୃତ୍ତିଶୀଳ ଟେକ୍ସ୍ଟ୍, ଛବି କିମ୍ବା ଅନ୍ୟ ତାଲିକା ବିଷୟବସ୍ତୁ ଦେଖନ୍ତି, ତେବେ ମନେ ପକାଇପାରିବେ ଯେ SwiftUI ରେ ଏପରି ବିଷୟବସ୍ତୁ ସାଧାରଣତଃ ପ୍ରଥମେ ଆରେରେ ରଖାଯାଏ, ତାପରେ `ForEach` ଦ୍ୱାରା କ୍ରମକ୍ରମେ ଦେଖାଯାଏ।
