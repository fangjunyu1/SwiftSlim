# Xcode ଫାଇଲ୍ ପରିଚାଳନା

ଏହି ପାଠରେ, ଆମେ Xcode ରେ ସାଧାରଣ ଫାଇଲ୍ ଅପରେସନ୍ ଶିଖିବା।

ପୂର୍ବ ପାଠଗୁଡ଼ିକରେ, ଆମେ ଅଧିକାଂଶ କୋଡ୍ `ContentView` ଭିତରେ ଲେଖିଥିଲୁ। ଏହା କୋଡ୍ ଏବଂ ଦେଖାଯାଉଥିବା ଫଳାଫଳକୁ ଦେଖିବାକୁ ସହଜ କରେ।

ଉଦାହରଣ ସ୍ୱରୂପ:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

କିନ୍ତୁ ବାସ୍ତବ App ରେ, ସାଧାରଣତଃ ସମସ୍ତ କୋଡ୍ ଗୋଟିଏ ଫାଇଲ୍ ମଧ୍ୟରେ ଲେଖାଯାଏ ନାହିଁ।

ପୃଷ୍ଠା, ଘଟକ ଏବଂ ଫଙ୍କସନ୍ ବଢ଼ିବା ସହିତ, ଆମକୁ ଭିନ୍ନ ଭିନ୍ନ ଦୃଶ୍ୟଗୁଡ଼ିକୁ ଭିନ୍ନ SwiftUI ଫାଇଲ୍‌ରେ ଭାଗ କରିବାକୁ ପଡ଼େ।

ଉଦାହରଣ ସ୍ୱରୂପ:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

ଏହା କୋଡ୍‌ର ଗଠନକୁ ଅଧିକ ସ୍ପଷ୍ଟ କରେ, ଏବଂ ପରେ ରକ୍ଷାବେକ୍ଷଣ ଓ ସଂଶୋଧନକୁ ମଧ୍ୟ ସହଜ କରେ।

## କାହିଁକି ଫାଇଲ୍ ପରିଚାଳନା କରିବା ଦରକାର

SwiftUI ରେ, ଗୋଟିଏ ପୃଷ୍ଠା କିମ୍ବା ଗୋଟିଏ ଘଟକକୁ ସାଧାରଣତଃ ଅଲଗା ଦୃଶ୍ୟ ଫାଇଲ୍ ଭାବେ ଲେଖିହେବ।

ଉଦାହରଣ ସ୍ୱରୂପ:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ଏଠାରେ `CardView` ହେଉଛି ଗୋଟିଏ SwiftUI ଦୃଶ୍ୟ।

ଯଦି ସମସ୍ତ ଦୃଶ୍ୟ `ContentView` ଭିତରେ ଲେଖାଯାଏ, ତେବେ କୋଡ୍ ଧୀରେ ଧୀରେ ଲମ୍ବା ହେବ, ଏବଂ ପଢ଼ିବା ମଧ୍ୟ ଅଧିକ କଷ୍ଟକର ହେବ।

ସେହିପରି, ଆମେ ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା, ନାମ ବଦଳାଇବା, ଫାଇଲ୍ ହଟାଇବା, ଏବଂ ଭିନ୍ନ କୋଡ୍ ଫାଇଲ୍‌ଗୁଡ଼ିକୁ ପରିଚାଳନା ପାଇଁ ଫୋଲ୍ଡର ବ୍ୟବହାର କରିବା ଶିଖିବା ଦରକାର।

## SwiftUI ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା

Xcode ରେ SwiftUI ଫାଇଲ୍ ସୃଷ୍ଟି କରିବାର ସାଧାରଣତଃ ଦୁଇଟି ପ୍ରଚଳିତ ପଦ୍ଧତି ଅଛି:

ଗୋଟିଏ ହେଉଛି ଖାଲି ଫାଇଲ୍ ସୃଷ୍ଟି କରି, ତାପରେ ନିଜେ କୋଡ୍ ଲେଖିବା।

ଅନ୍ୟଟି ହେଉଛି SwiftUI ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା, ଯେଉଁଠାରେ Xcode ସ୍ୱୟଂଚାଳିତ ଭାବେ ମୂଳ କୋଡ୍ ତିଆରି କରେ।

ଆରମ୍ଭିକ ଶିକ୍ଷାର୍ଥୀଙ୍କ ପାଇଁ SwiftUI ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ବ୍ୟବହାର କରିବା ଅଧିକ ସୁପାରିଶ କରାଯାଏ, କାରଣ ଏହା ମୂଳ ଗଠନକୁ ସ୍ୱୟଂଚାଳିତ ଭାବେ ତିଆରି କରେ।

### ଖାଲି ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା

Xcode ର ବାମ ପଟିର Navigator ଅଞ୍ଚଳରେ, ଖାଲି ସ୍ଥାନ କିମ୍ବା ଫୋଲ୍ଡର ଉପରେ ଡାହାଣ କ୍ଲିକ୍ କରି `New Empty File` ବାଛନ୍ତୁ।

ଅପରେସନ୍ ଅବସ୍ଥାନ ନିମ୍ନରୂପ:

![xcode](../../../Resource/027_view1.png)

ଖାଲି ଫାଇଲ୍ ସୃଷ୍ଟି ହେବା ପରେ, Xcode ସାଧାରଣତଃ ଗୋଟିଏ ନାମହୀନ Swift ଫାଇଲ୍ ସୃଷ୍ଟି କରେ, ଯେପରି `Untitled.swift`।

ଆମେ ଏହାକୁ `TestView.swift` ଭାବେ ପୁନର୍ନାମକରଣ କରିପାରିବା।

SwiftUI ପ୍ରୋଜେକ୍ଟରେ, ଯଦି ଏହି ଫାଇଲ୍ ଗୋଟିଏ ଦୃଶ୍ୟ ଦେଖାଇବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ତେବେ ନାମର ଶେଷରେ ସାଧାରଣତଃ `View` ଯୋଡ଼ିବା ସୁପାରିଶ କରାଯାଏ।

ଉଦାହରଣ ସ୍ୱରୂପ:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

ଏହା କୌଣସି ବାଧ୍ୟତାମୂଳକ ନିୟମ ନୁହେଁ, କିନ୍ତୁ ଏହା ଆମକୁ ଗୋଟିଏ ଦୃଷ୍ଟିରେ ଏହା ଦୃଶ୍ୟ ଫାଇଲ୍ ବୋଲି ବୁଝିବାକୁ ସାହାଯ୍ୟ କରେ।

ଖାଲି ଫାଇଲ୍ ସୃଷ୍ଟି ପରେ, ଫାଇଲ୍‌ରେ କେବଳ ଡିଫଲ୍ଟ ଟୀକା ଥାଇପାରେ, କିମ୍ବା ଏହା ଖାଲି ମଧ୍ୟ ଥାଇପାରେ। ଏହା Xcode ସଂସ୍କରଣ ଏବଂ ସୃଷ୍ଟି ପଦ୍ଧତି ଉପରେ ନିର୍ଭର କରେ।

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ଏହି ସମୟରେ, ଆମକୁ SwiftUI କୋଡ୍ ହାତରେ ଲେଖିବାକୁ ପଡ଼େ:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

ଏଭଳି ଭାବେ, ଗୋଟିଏ ସରଳ SwiftUI ଦୃଶ୍ୟ ଫାଇଲ୍ ସୃଷ୍ଟି ସମାପ୍ତ ହେଲା।

### ଫାଇଲ୍ ନାମ ବଦଳାଇବା

ଯଦି ଫାଇଲ୍ ସୃଷ୍ଟି ପରେ ଫାଇଲ୍ ନାମ ବଦଳାଇବାକୁ ଚାହୁଁଛନ୍ତି, Xcode ର ବାମ ପଟିର Navigator ଅଞ୍ଚଳରେ ଫାଇଲ୍ ବାଛନ୍ତୁ, ତାପରେ ଏଣ୍ଟର କୀ `⏎` ଦବାନ୍ତୁ।

![xcode](../../../Resource/027_view2.png)

ଏହାଦ୍ୱାରା ଫାଇଲ୍ ନାମ ବଦଳାଇହେବ।

ଧ୍ୟାନ ଦେବା ଦରକାର, ଫାଇଲ୍ ନାମ ବଦଳାଇଲେ କୋଡ୍‌ରେ ଥିବା ଦୃଶ୍ୟ ନାମ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବଦଳାଯାଏ ନାହିଁ।

ଉଦାହରଣକୁ, ଫାଇଲ୍ ନାମ `ProfileView.swift` କରାଯାଇବା ପରେ, କୋଡ୍‌ରେ ତଥାପି `struct TestView: View` ଥାଇପାରେ।

ଏହା ନିଶ୍ଚୟ ଭାବେ ତ୍ରୁଟି କରାଇବ ବୋଲି ନୁହେଁ, କିନ୍ତୁ ଏହା ଭ୍ରମ ସୃଷ୍ଟି କରିପାରେ।

ତେଣୁ, ଫାଇଲ୍ ନାମ ଓ ଦୃଶ୍ୟ ନାମକୁ ଏକରୂପ ରଖିବା ସୁପାରିଶ କରାଯାଏ। ଉଦାହରଣକୁ, ଫାଇଲ୍ ନାମ `ProfileView.swift` ହେଲେ, ଦୃଶ୍ୟ ନାମ ମଧ୍ୟ `ProfileView` ବ୍ୟବହାର କରନ୍ତୁ।

## SwiftUI ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା

ଖାଲି ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା ଛଡ଼ା, ଆମେ ସିଧାସଳଖ SwiftUI ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ମଧ୍ୟ ସୃଷ୍ଟି କରିପାରିବା।

Xcode ର ବାମ ପଟିର Navigator ଅଞ୍ଚଳରେ, ଖାଲି ସ୍ଥାନ କିମ୍ବା ଫୋଲ୍ଡର ଉପରେ ଡାହାଣ କ୍ଲିକ୍ କରି `New File from Template...` ବାଛନ୍ତୁ।

ଅପରେସନ୍ ଅବସ୍ଥାନ ନିମ୍ନରୂପ:

![xcode](../../../Resource/027_view3.png)

ଖୋଲିଥିବା ୱିଣ୍ଡୋରେ, ଉପର ଭାଗରେ ଭିନ୍ନ ପ୍ଲାଟଫର୍ମ ବାଛିହେବ, ଯେପରି `iOS`、`macOS` ଇତ୍ୟାଦି।

ଏଠାରେ ବର୍ତ୍ତମାନ ପ୍ରୋଜେକ୍ଟ ସହିତ ସମ୍ବନ୍ଧିତ ପ୍ଲାଟଫର୍ମକୁ ବାଛି ରଖିଲେ ହେବ। ଉଦାହରଣକୁ, ଆମେ iOS App ସୃଷ୍ଟି କରୁଛୁ, ତେବେ `iOS` ବାଛିବା।

![xcode](../../../Resource/027_view4.png)

ତାପରେ ତଳ ଭାଗରେ `SwiftUI View` ବାଛନ୍ତୁ।

![xcode](../../../Resource/027_view12.png)

ତାପରେ `Next` କ୍ଲିକ୍ କରନ୍ତୁ, ଫାଇଲ୍ ନାମ ପ୍ରବେଶ କରାଇ ସେଭ୍ କରନ୍ତୁ।

ଉଦାହରଣକୁ, ଆମେ `Test2View.swift` ନାମର ଗୋଟିଏ ଫାଇଲ୍ ସୃଷ୍ଟି କରୁଛୁ।

Xcode ସ୍ୱୟଂଚାଳିତ ଭାବେ ନିମ୍ନରୂପ କୋଡ୍ ସୃଷ୍ଟି କରିବ:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

ଖାଲି ଫାଇଲ୍ ସହିତ ତୁଳନା କଲେ, ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ସ୍ୱୟଂଚାଳିତ ଭାବେ `import SwiftUI`、`struct`、`body` ଏବଂ `#Preview` ସମ୍ମିଳିତ କରେ।

ସେହିପରି, ଆରମ୍ଭିକ ଶିକ୍ଷାର୍ଥୀଙ୍କ ପାଇଁ SwiftUI ଟେମ୍ପଲେଟ୍ ଫାଇଲ୍ ବ୍ୟବହାର କରିବା ଅଧିକ ସରଳ, ଏବଂ କୋଡ୍ ଛାଡ଼ିଯିବାର ସମ୍ଭାବନା ମଧ୍ୟ କମ୍।

**ଟିପ୍ପଣୀ**: ଡାହାଣ କ୍ଲିକ୍ କରି ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା ଛଡ଼ା, Xcode ର ବାମ ପଟିର Navigator ତଳ ଭାଗରେ ଥିବା ପ୍ଲସ୍ ବଟନ୍ କ୍ଲିକ୍ କରି ଫାଇଲ୍ କିମ୍ବା ଫୋଲ୍ଡର ମଧ୍ୟ ସୃଷ୍ଟି କରିପାରିବେ।

![xcode](../../../Resource/027_view5.png)

## ଫୋଲ୍ଡର ସୃଷ୍ଟି କରିବା

ଫାଇଲ୍ ବଢ଼ିବା ସହିତ, ଆମେ ସେଗୁଡ଼ିକୁ ପରିଚାଳନା ପାଇଁ ଫୋଲ୍ଡର ବ୍ୟବହାର କରିପାରିବା।

Xcode ର ବାମ ପଟିର Navigator ଅଞ୍ଚଳରେ, ଖାଲି ସ୍ଥାନ କିମ୍ବା ଫୋଲ୍ଡର ଉପରେ ଡାହାଣ କ୍ଲିକ୍ କରି `New Folder` ବାଛନ୍ତୁ।

ଫୋଲ୍ଡର ସୃଷ୍ଟି ହେବା ପରେ, ସମ୍ବନ୍ଧିତ ଫାଇଲ୍‌ଗୁଡ଼ିକୁ ଫୋଲ୍ଡର ଭିତରକୁ ଟାଣି ନେଇପାରିବେ।

![xcode](../../../Resource/027_view7.png)

ଫୋଲ୍ଡରଗୁଡ଼ିକ ପ୍ରଧାନତଃ ପ୍ରୋଜେକ୍ଟ ଗଠନ ସଜାଇବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ଏହା ଦୃଶ୍ୟ ନିଜର ବ୍ୟବହାର ପଦ୍ଧତିକୁ ବଦଳାଏ ନାହିଁ।

ଦୃଶ୍ୟ ନାମ ଠିକ୍ ଥିଲେ, ଅନ୍ୟ ଫାଇଲ୍‌ରେ ଏହାକୁ ତଥାପି `TestView()`、`Test2View()` ପରି ବ୍ୟବହାର କରିପାରିବେ।

## ଫାଇଲ୍ କିମ୍ବା ଫୋଲ୍ଡର ହଟାଇବା

ଯଦି ଫାଇଲ୍ କିମ୍ବା ଫୋଲ୍ଡର ହଟାଇବାକୁ ଚାହୁଁଛନ୍ତି, Xcode ର ବାମ ପଟିର Navigator ଅଞ୍ଚଳରେ ସମ୍ବନ୍ଧିତ ଫାଇଲ୍ ବାଛି, ତାପରେ ଡାହାଣ କ୍ଲିକ୍ କରି `Delete` ବାଛନ୍ତୁ।

ଫାଇଲ୍ ବାଛିବା ପରେ, ଡିଲିଟ୍ କୀ `⌫` ମଧ୍ୟ ଦବାଇପାରିବେ।

Xcode ହଟାଇବା ପୁଷ୍ଟିକରଣ ୱିଣ୍ଡୋ ଦେଖାଇବ।

![xcode](../../../Resource/027_view10.png)

`Move to Trash` ବାଛିବାର ଅର୍ଥ ହେଉଛି ଫାଇଲ୍‌କୁ ଟ୍ରାଶ୍‌କୁ ସରାଇବା।

ଯଦି `Remove Reference` ଦେଖନ୍ତି, ତାହାର ଅର୍ଥ କେବଳ Xcode ପ୍ରୋଜେକ୍ଟରୁ ରେଫରେନ୍ସ୍‌କୁ ହଟାଇବା, କିନ୍ତୁ ଡିସ୍କ୍‌ରେ ଥିବା ମୂଳ ଫାଇଲ୍ ହଟିବ ନାହିଁ।

ଆରମ୍ଭିକ ଶିକ୍ଷାର୍ଥୀଙ୍କ ପାଇଁ, ଯଦି ନିଜେ ନୂଆ ସୃଷ୍ଟି କରିଥିବା ସାଧାରଣ ଫାଇଲ୍ ହଟାଇବା ମାତ୍ର ହେଉଛି, ସାଧାରଣତଃ `Move to Trash` ବାଛିଲେ ହେବ।

## ଅନେକ SwiftUI ଦୃଶ୍ୟ ଦେଖାଇବା

ଫାଇଲ୍ ସୃଷ୍ଟି କରିବା ଶିଖିଲା ପରେ, ଆମେ ଭିନ୍ନ ଦୃଶ୍ୟଗୁଡ଼ିକୁ ଭିନ୍ନ ଫାଇଲ୍‌ରେ ଭାଗ କରିପାରିବା।

ଉଦାହରଣକୁ, ଆମେ ଦୁଇଟି SwiftUI ଫାଇଲ୍ ସୃଷ୍ଟି କରୁଛୁ:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` ଲେଖକଙ୍କ ନାମ ଦେଖାଇବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` କାର୍ଡ ପୃଷ୍ଠଭୂମି ଦେଖାଇବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

ତାପରେ, ଆମେ `ContentView` ଭିତରେ ଏହି ଦୁଇଟି ଦୃଶ୍ୟ ବ୍ୟବହାର କରିପାରିବା:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

ଚାଲୁ କରିବା ପରେ, `CardView` ଏବଂ `AuthorView` ଏକାସାଥିରେ ଯୋଗ ହୋଇ ଦେଖାଯାଉଥିବାକୁ ଦେଖିପାରିବେ:

![xcode](../../../Resource/027_view11.png)

ଏହି ପ୍ରଭାବରେ, `CardView` ପୃଷ୍ଠଭୂମି ଭାବେ ରହେ, ଏବଂ `AuthorView` ଉପରେ ଦେଖାଯାଏ, କାରଣ ସେମାନଙ୍କୁ `ZStack` ଭିତରେ ରଖାଯାଇଛି।

`ContentView` ଭିତରେ, ଆମେ `CardView()` ଏବଂ `AuthorView()` ଦ୍ୱାରା ଏହି ଦୁଇଟି ଦୃଶ୍ୟ ବ୍ୟବହାର କରୁଛୁ। ନାମ ପରେ ଥିବା `()` ର ଅର୍ଥ ହେଉଛି ଏହି ଦୃଶ୍ୟକୁ ସୃଷ୍ଟି କରି ବ୍ୟବହାର କରିବା।

ଏହା SwiftUI ରେ ସାଧାରଣ ଦୃଶ୍ୟ ନେଷ୍ଟିଂ: ଗୋଟିଏ ଦୃଶ୍ୟ ଭିତରେ ଅନ୍ୟ ଦୃଶ୍ୟଗୁଡ଼ିକୁ ଆଉ ବ୍ୟବହାର କରାଯାଇପାରେ।

## ପ୍ରବେଶ ଫାଇଲ୍

ଶେଷରେ, App କେଉଁଠାରୁ ଇଣ୍ଟରଫେସ୍ ଦେଖାଇବା ଆରମ୍ଭ କରେ ତାହା ପୁନଃ ଦେଖିବା।

SwiftUI ପ୍ରୋଜେକ୍ଟରେ, ସାଧାରଣତଃ ଗୋଟିଏ ପ୍ରବେଶ ଫାଇଲ୍ ଥାଏ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ଏଠାରେ `@main` ର ଅର୍ଥ ହେଉଛି ଏହା App ର ପ୍ରବେଶ ସ୍ଥାନ।

`WindowGroup` ଭିତରେ, ଆମେ ଦେଖୁଛୁ:

```swift
ContentView()
```

ଏହାର ଅର୍ଥ App ଆରମ୍ଭ ହେବା ପରେ, ପ୍ରଥମେ `ContentView` ଦେଖାଯାଏ।

ତାପରେ `ContentView` ଭିତରେ ଆଉ ଦେଖାଯାଏ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ସେହିପରି ଦୃଶ୍ୟ ସ୍ତରଗୁଡ଼ିକୁ ଏଭଳି ବୁଝିହେବ:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

ଯଦି ଆମେ ପ୍ରବେଶ ଫାଇଲ୍‌ରେ ଦେଖାଯାଉଥିବା ଦୃଶ୍ୟକୁ `CardView` କରିଦେଉ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

ତେବେ App ଆରମ୍ଭ ହେବା ପରେ, କେବଳ `CardView` ଦେଖାଯିବ।

ଅର୍ଥାତ୍, ପ୍ରବେଶ କୋଡ୍‌ରେ ଥିବା `WindowGroup` App ଆରମ୍ଭ ହେବା ପରେ ପ୍ରଥମେ କେଉଁ ଦୃଶ୍ୟ ଦେଖାଯିବ ତାହା ନିର୍ଦ୍ଧାରଣ କରେ।

ଏବଂ ଏହି ଦୃଶ୍ୟ ଭିତରେ, ଆଉ ଅନ୍ୟ ଦୃଶ୍ୟଗୁଡ଼ିକୁ ଚାଲୁ ରଖି ଦେଖାଇପାରେ।

## ସାରାଂଶ

ଏହି ପାଠରେ, ଆମେ Xcode ରେ ସାଧାରଣ ଫାଇଲ୍ ପରିଚାଳନା ଅପରେସନ୍‌ଗୁଡ଼ିକ ଶିଖିଲୁ।

ଆମେ SwiftUI ଫାଇଲ୍ କିପରି ସୃଷ୍ଟି କରିବା, ଫାଇଲ୍ ନାମ କିପରି ବଦଳାଇବା, ଫୋଲ୍ଡର କିପରି ସୃଷ୍ଟି କରିବା, ଏବଂ ଫାଇଲ୍ କିପରି ହଟାଇବା ବୁଝିଲୁ।

ସେହି ସମୟରେ, ଆମେ ଗୋଟିଏ ଗୁରୁତ୍ୱପୂର୍ଣ୍ଣ ଧାରଣା ମଧ୍ୟ ଶିଖିଲୁ: SwiftUI ଦୃଶ୍ୟଗୁଡ଼ିକୁ ଭିନ୍ନ ଫାଇଲ୍‌ରେ ଭାଗ କରି, ପରେ ଅନ୍ୟ ଦୃଶ୍ୟରେ ବ୍ୟବହାର କରିପାରିବା।

ଉଦାହରଣ ସ୍ୱରୂପ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ଏଠାରେ `CardView()` ଏବଂ `AuthorView()` ହେଉଛି `ContentView` ଭିତରେ ଅନ୍ୟ ଦୃଶ୍ୟଗୁଡ଼ିକୁ ବ୍ୟବହାର କରିବା।

ଶେଷରେ, ଆମେ ପ୍ରବେଶ ଫାଇଲ୍‌ର ଦେଖାଇବା ଲୋଜିକ୍‌କୁ ମଧ୍ୟ ପୁନଃଦେଖିଲୁ।

App ଆରମ୍ଭ ହେବା ସମୟରେ, ଏହା ପ୍ରଥମେ `@main` ଚିହ୍ନ ଥିବା ପ୍ରବେଶ କୋଡ୍‌କୁ ପ୍ରବେଶ କରେ, ତାପରେ `WindowGroup` ଭିତରେ ଥିବା ଦୃଶ୍ୟକୁ ଦେଖାଏ।

ଏହା ଆମକୁ ବୁଝିବାରେ ସାହାଯ୍ୟ କରେ: ଭିନ୍ନ SwiftUI ଫାଇଲ୍‌ଗୁଡ଼ିକ ଏକାକୀ ନୁହେଁ, ସେମାନେ ପରସ୍ପରକୁ ଯୋଡ଼ିପାରନ୍ତି, ଏବଂ ଶେଷରେ ଏକ ସମ୍ପୂର୍ଣ୍ଣ App ଇଣ୍ଟରଫେସ୍ ତିଆରି କରନ୍ତି।
