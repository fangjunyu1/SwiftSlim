# ଦୃଶ୍ୟ ଅନୁକଳନ

ଏହି ପାଠରେ, ଆମେ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ପାଇଁ ଦୃଶ୍ୟ ଅନୁକଳନ ଶିଖିବୁ।

iPhone, iPad ଏବଂ Mac ରେ, ବ୍ୟବହାରକାରୀ ହାଲୁକା ମୋଡ୍ ବା ଗାଢ଼ ମୋଡ୍ ବାଛିପାରନ୍ତି।

ସିଷ୍ଟମ୍ ଗାଢ଼ ମୋଡ୍ କୁ ବଦଳିଲେ, ଅନେକ App ର ପୃଷ୍ଠଭୂମି, ଟେକ୍ସ୍ଟ ଏବଂ ଆଇକନ୍ ରଙ୍ଗ ମଧ୍ୟ ସେହିପରି ବଦଳିଯାଏ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଦିନେ ଇଣ୍ଟରଫେସ୍ ହାଲୁକା ପୃଷ୍ଠଭୂମି ହୋଇପାରେ, ରାତିରେ ଏହା ଗାଢ଼ ପୃଷ୍ଠଭୂମିକୁ ବଦଳିଯାଇପାରେ।

![view](../../../Resource/025_view9.png)

ସିଷ୍ଟମ୍ ଦୃଶ୍ୟ ଅନୁସାରେ ଇଣ୍ଟରଫେସ୍ ପ୍ରଦର୍ଶନକୁ ସ୍ୱୟଂଚାଳିତ ଭାବେ ସମଯୋଜନ କରିବାର ଏହି କ୍ଷମତାକୁ ଦୃଶ୍ୟ ଅନୁକଳନ କୁହାଯାଏ।

## Xcode ରେ ଦୃଶ୍ୟ ଅନୁକଳନ ପ୍ରିଭ୍ୟୁ

ଦୃଶ୍ୟ ଅନୁକଳନ ଶିଖିବା ପୂର୍ବରୁ, ପ୍ରଥମେ Xcode ରେ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ କିପରି ପ୍ରିଭ୍ୟୁ କରିବା ଜାଣିବା।

`Canvas` ଅଞ୍ଚଳର ତଳଭାଗରେ, `Device Settings` ବଟନ୍ କ୍ଲିକ୍ କଲେ `Canvas Device Settings` ପପ୍-ଅପ୍ ଖୋଲିବ।

![canvas](../../../Resource/025_view.png)

ପପ୍-ଅପ୍ ଭିତରେ `Color Scheme` ସେଟିଂ ଦେଖିପାରିବେ।

![canvas1](../../../Resource/025_view1.png)

ଏଥିରେ:

- `Light Appearance` ର ଅର୍ଥ ହେଉଛି ହାଲୁକା ମୋଡ୍।
- `Dark Appearance` ର ଅର୍ଥ ହେଉଛି ଗାଢ଼ ମୋଡ୍।

ଏଠାରୁ ଆମେ ପ୍ରିଭ୍ୟୁ ଦୃଶ୍ୟ ବଦଳାଇ, ବର୍ତ୍ତମାନ ଇଣ୍ଟରଫେସ୍ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ରେ କେମିତି ଦେଖାଯାଏ ଦେଖିପାରିବା।

## ଦୁଇ ପ୍ରକାର ଦୃଶ୍ୟକୁ ପାଖାପାଖି ଦେଖାଇବା

ଯଦି ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ କୁ ଏକେ ସମୟରେ ଦେଖିବାକୁ ଚାହୁଁଛନ୍ତି, `Variants` ବୈଶିଷ୍ଟ୍ୟ ବ୍ୟବହାର କରିପାରିବେ।

`Canvas` ଅଞ୍ଚଳର ତଳଭାଗରେ, `Variants` ବଟନ୍ କ୍ଲିକ୍ କରି `Color Scheme Variants` ବାଛନ୍ତୁ।

![canvas2](../../../Resource/025_view2.png)

ବାଛିବା ପରେ, `Canvas` ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ର ପ୍ରିଭ୍ୟୁ ପ୍ରଭାବକୁ ଏକେ ସମୟରେ ଦେଖାଇବ।

![canvas3](../../../Resource/025_view3.png)

ଏହା ଦୁଇ ପ୍ରକାର ଦୃଶ୍ୟ ତଳେ ଇଣ୍ଟରଫେସ୍ ତଫାତକୁ ଅଧିକ ସହଜରେ ତୁଳନା କରିବାରେ ସାହାଯ୍ୟ କରେ।

ସହଜ ଭାବେ କହିଲେ:

ଯଦି କେବଳ ଅସ୍ଥାୟୀ ଭାବେ ହାଲୁକା/ଗାଢ଼ ମୋଡ୍ ବଦଳାଇବାକୁ ଚାହୁଁଛନ୍ତି, `Canvas Device Settings` ବ୍ୟବହାର କରିପାରିବେ।

ଯଦି ଦୁଇ ପ୍ରକାର ଦୃଶ୍ୟକୁ ଏକେ ସମୟରେ ଦେଖିବାକୁ ଚାହୁଁଛନ୍ତି, `Color Scheme Variants` ବ୍ୟବହାର କରିପାରିବେ।

## ଭ୍ୟୁଗୁଡ଼ିକ ଡିଫଲ୍ଟ ଭାବେ ଦୃଶ୍ୟକୁ ଅନୁକଳନ କରେ

SwiftUI ରେ, ଅନେକ ସିଷ୍ଟମ୍ ଭ୍ୟୁ ଡିଫଲ୍ଟ ଭାବେ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ସହ ସ୍ୱୟଂଚାଳିତ ଭାବେ ଅନୁକୂଳ ହୋଇଯାଏ।

ଉଦାହରଣ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("ଆଲୋକ")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("କିଛି କିଛି କିଛି")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ପ୍ରଦର୍ଶନ ପ୍ରଭାବ:

![canvas](../../../Resource/025_view4.png)

ଦେଖିପାରିବେ, ହାଲୁକା ମୋଡ୍ ରେ ଆଇକନ୍ ଏବଂ ଶୀର୍ଷକ ସାଧାରଣତଃ ଗାଢ଼ ରଙ୍ଗରେ ଦେଖାଯାଏ।

ଗାଢ଼ ମୋଡ୍ ରେ, ଆଇକନ୍ ଏବଂ ଶୀର୍ଷକ ସ୍ୱୟଂଚାଳିତ ଭାବେ ହାଲୁକା ରଙ୍ଗକୁ ବଦଳିଯାଏ।

ଏହାର କାରଣ ହେଉଛି `Text`, `Image` ପରି SwiftUI ଭ୍ୟୁଗୁଡ଼ିକ ଡିଫଲ୍ଟ ଭାବେ ସିଷ୍ଟମ୍ ଦୃଶ୍ୟ ଅନୁସାରେ ରଙ୍ଗ ସମଯୋଜନ କରେ।

ଅର୍ଥାତ୍, ଯଦି ଆମେ ହାତେ ସ୍ଥିର ରଙ୍ଗ ନିର୍ଦ୍ଦିଷ୍ଟ ନକରୁ, SwiftUI ଆମ ପାଇଁ ଦୃଶ୍ୟ ଅନୁକଳନର ଏକ ଅଂଶ ସମ୍ଭାଳିଦେବ।

## ସ୍ଥିର ରଙ୍ଗ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବଦଳେ ନାହିଁ

ଧ୍ୟାନ ଦେବାକୁ ପଡ଼େ, ଯଦି ଆମେ ହାତେ ସ୍ଥିର ରଙ୍ଗ ନିର୍ଦ୍ଦିଷ୍ଟ କରୁ, ସେହି ରଙ୍ଗ ହାଲୁକା/ଗାଢ଼ ମୋଡ୍ ଅନୁସାରେ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବଦଳିବ ନାହିଁ।

ଉଦାହରଣ:

```swift
Text("କିଛି କିଛି କିଛି")
	.foregroundStyle(Color.gray)
```

ଏଠାରେ ଟେକ୍ସ୍ଟକୁ `Color.gray` ଭାବେ ନିର୍ଦ୍ଦିଷ୍ଟ କରାଯାଇଛି, ତେଣୁ ଏହା ସବୁବେଳେ ଧୂସର ରଙ୍ଗରେ ଦେଖାଯିବ।

ଆଉ ଏକ ଉଦାହରଣ:

```swift
Text("ଶୀର୍ଷକ")
    .foregroundStyle(Color.white)
```

ଏହି କୋଡ୍ ହାଲୁକା ମୋଡ୍ କିମ୍ବା ଗାଢ଼ ମୋଡ୍, ଦୁଇଟିରେ ମଧ୍ୟ ଧଳା ଟେକ୍ସ୍ଟ ଦେଖାଇବ।

ଯଦି ପୃଷ୍ଠଭୂମି ମଧ୍ୟ ହାଲୁକା ହୁଏ, ଧଳା ଟେକ୍ସ୍ଟ ସ୍ପଷ୍ଟ ଦେଖାଯାଇନପାରେ।

ସେହିପାଇଁ ଦୃଶ୍ୟ ଅନୁକଳନ କରିବାବେଳେ, `Color.white`, `Color.black` ପରି ସ୍ଥିର ରଙ୍ଗକୁ ଇଚ୍ଛାମତେ ଲେଖିଦେବାରୁ ସମ୍ଭବତଃ ଦୂରେ ରହିବା ଭଲ।

ଅନେକ ସ୍ଥିତିରେ, ପ୍ରଥମେ ସିଷ୍ଟମ୍ ସେମାଣ୍ଟିକ୍ ଶୈଳୀ ବ୍ୟବହାର କରିପାରିବେ, ଯେପରି:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

ଏଥିରେ:

- `.primary` ସାଧାରଣତଃ ପ୍ରଧାନ ବିଷୟବସ୍ତୁ ପାଇଁ ବ୍ୟବହୃତ ହୁଏ।
- `.secondary` ସାଧାରଣତଃ ଦ୍ୱିତୀୟ ବିଷୟବସ୍ତୁ ପାଇଁ ବ୍ୟବହୃତ ହୁଏ।

ସେଗୁଡ଼ିକ ହାଲୁକା/ଗାଢ଼ ମୋଡ୍ ଅନୁସାରେ ପ୍ରଦର୍ଶନ ପ୍ରଭାବକୁ ସ୍ୱୟଂଚାଳିତ ଭାବେ ସମଯୋଜନ କରିବ।

## ଦୃଶ୍ୟ ଅନୁସାରେ ଭିନ୍ନ ବିଷୟବସ୍ତୁ ଦେଖାଇବା

କେତେବେଳେ, ଆମେ କେବଳ ରଙ୍ଗ ବଦଳିବାକୁ ନୁହେଁ, ଭିନ୍ନ ଦୃଶ୍ୟ ତଳେ ଭିନ୍ନ ବିଷୟବସ୍ତୁ ଦେଖାଇବାକୁ ମଧ୍ୟ ଚାହୁଁଥାଉ।

ଉଦାହରଣ:

- ହାଲୁକା ମୋଡ୍ ରେ ସାଧାରଣ ବଲ୍ବ ଆଇକନ୍ ଦେଖାନ୍ତୁ।
- ଗାଢ଼ ମୋଡ୍ ରେ ଆଲୋକିତ ବଲ୍ବ ଆଇକନ୍ ଦେଖାନ୍ତୁ।
- ହାଲୁକା ମୋଡ୍ ରେ ଶୀର୍ଷକ `ଆଲୋକ` ଦେଖାନ୍ତୁ।
- ଗାଢ଼ ମୋଡ୍ ରେ ଶୀର୍ଷକ `ଅନ୍ଧାର` ଦେଖାନ୍ତୁ।

ଏବେ, ବର୍ତ୍ତମାନ ସିଷ୍ଟମ୍ ହାଲୁକା ମୋଡ୍ ରେ ଅଛି କି ଗାଢ଼ ମୋଡ୍ ରେ ଅଛି, ତାହା ବିଚାର କରିବା ଦରକାର।

SwiftUI ରେ, `colorScheme` ପରିବେଶ ମୂଲ୍ୟ ମାଧ୍ୟମରେ ବର୍ତ୍ତମାନ ଦୃଶ୍ୟ ମୋଡ୍ ନେଇପାରିବେ:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ପୂର୍ଣ୍ଣ କୋଡ୍:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "ଆଲୋକ" : "ଅନ୍ଧାର"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("କିଛି କିଛି କିଛି")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ପ୍ରଦର୍ଶନ ପ୍ରଭାବ:

![view](../../../Resource/025_view5.png)

ପ୍ରିଭ୍ୟୁ ପ୍ରଭାବରୁ ଦେଖିପାରିବେ, ଭିନ୍ନ ଦୃଶ୍ୟ ମୋଡ୍ ତଳେ ସେହି ଏକେ ଭ୍ୟୁରେ ଦେଖାଯାଉଥିବା ବିଷୟବସ୍ତୁ ପୂରାପୁରି ଏକରକମ ନୁହେଁ।

ହାଲୁକା ମୋଡ୍ ରେ, `Image` ସାଧାରଣ ବଲ୍ବ ଆଇକନ୍ ଦେଖାଏ, ଶୀର୍ଷକ `ଆଲୋକ` ଭାବେ ଦେଖାଯାଏ।

ଗାଢ଼ ମୋଡ୍ ରେ, `Image` ଆଲୋକିତ ବଲ୍ବ ଆଇକନ୍ ଦେଖାଏ, ଶୀର୍ଷକ `ଅନ୍ଧାର` ଭାବେ ଦେଖାଯାଏ।

ଅର୍ଥାତ୍, ଏଠାରେ ପରିବର୍ତ୍ତନ କେବଳ ରଙ୍ଗରେ ନୁହେଁ, ଆଇକନ୍ ନାମ ଏବଂ ଶୀର୍ଷକ ଟେକ୍ସ୍ଟ ନିଜେ ମଧ୍ୟ ବଦଳିଯାଉଛି।

ଏହାର କାରଣ ହେଉଛି ଆମେ ଆଇକନ୍ ଏବଂ ଶୀର୍ଷକକୁ ସିଧାସଳଖ `body` ଭିତରେ ଲେଖିନାହିଁ, ବରଂ `colorScheme` ଅନୁସାରେ ଭିନ୍ନ ବିଷୟବସ୍ତୁ ଗଣନା କରିଛୁ।

ଏହି କୋଡ୍ ଭିତରେ, ଆମେ ଦୁଇଟି କମ୍ପ୍ୟୁଟେଡ୍ ପ୍ରୋପର୍ଟି ବ୍ୟବହାର କରିଛୁ:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "ଆଲୋକ" : "ଅନ୍ଧାର"
}
```

ଯେତେବେଳେ `colorScheme` `.light` ହୁଏ, ଏହାର ଅର୍ଥ ବର୍ତ୍ତମାନ ହାଲୁକା ମୋଡ୍।

ଯେତେବେଳେ `colorScheme` `.dark` ହୁଏ, ଏହାର ଅର୍ଥ ବର୍ତ୍ତମାନ ଗାଢ଼ ମୋଡ୍।

ସେହିପରି ଆମେ ଭିନ୍ନ ଦୃଶ୍ୟ ମୋଡ୍ ଅନୁସାରେ ଭିନ୍ନ ଆଇକନ୍ ନାମ ଏବଂ ଶୀର୍ଷକ ଟେକ୍ସ୍ଟ ଫେରାଇପାରୁ।

ଏହା କମ୍ପ୍ୟୁଟେଡ୍ ପ୍ରୋପର୍ଟିର ଏକ ସାଧାରଣ ବ୍ୟବହାର: ବର୍ତ୍ତମାନ ସ୍ଥିତି ଅନୁସାରେ ଭ୍ୟୁରେ ଦେଖାଇବାକୁ ଥିବା ବିଷୟବସ୍ତୁ ଗଣନା କରିବା।

## @Environment କୁ ବୁଝିବା

ଏହା ଆମର `@Environment` ସହ ପ୍ରଥମ ପରିଚୟ।

`@Environment` କୁ ଏଭଳି ବୁଝିପାରିବେ: SwiftUI ର ପରିବେଶରୁ ଏକ ମୂଲ୍ୟ ପଢ଼ିବା।

App ଚାଲୁଥିବାବେଳେ, ସିଷ୍ଟମ୍ ଅନେକ ପରିବେଶ ସୂଚନା ପ୍ରଦାନ କରେ, ଯେପରି:

- ବର୍ତ୍ତମାନ ଭାଷା
- ବର୍ତ୍ତମାନ ଦୃଶ୍ୟ ମୋଡ୍
- ବର୍ତ୍ତମାନ ଲେଆଉଟ୍ ଦିଗ
- ବର୍ତ୍ତମାନ ଫଣ୍ଟ ଆକାର ସେଟିଂ

ଯେତେବେଳେ ଆମକୁ ବର୍ତ୍ତମାନ ଦୃଶ୍ୟ ମୋଡ୍ ନେବାକୁ ପଡ଼େ, ଆମେ `colorScheme` ପଢ଼ିପାରୁ:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ଏଥିରେ

```swift
\.colorScheme
```

ମାନେ ପରିବେଶରୁ `colorScheme` ଏହି ମୂଲ୍ୟ ପଢ଼ିବା।

```swift
private var colorScheme
```

ମାନେ ପଢ଼ାଯାଇଥିବା ମୂଲ୍ୟକୁ `colorScheme` ଏହି ଭେରିଏବଲ୍ ରେ ସଞ୍ଚୟ କରିବା।

ଭେରିଏବଲ୍ ନାମ ଆପଣ ନିଜେ ନିର୍ଣ୍ଣୟ କରିପାରିବେ, ଉଦାହରଣ ସ୍ୱରୂପ ଏଭଳି ମଧ୍ୟ ଲେଖିପାରିବେ:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

ସାମ୍ନାର `\.colorScheme` ଅପରିବର୍ତ୍ତିତ ରହିଲେ, ଏହାର ଅର୍ଥ ହେଉଛି ପଢ଼ାଯାଉଥିବା ମୂଲ୍ୟ ବର୍ତ୍ତମାନ ଦୃଶ୍ୟ ମୋଡ୍।

## colorScheme ର ଦୁଇଟି ସାଧାରଣ ମୂଲ୍ୟ

`colorScheme` ର ସାଧାରଣତଃ ଦୁଇଟି ମୂଲ୍ୟ ଥାଏ:

```swift
.light
.dark
```

ସେଗୁଡ଼ିକର ଅର୍ଥ:

- `.light`: ହାଲୁକା ମୋଡ୍
- `.dark`: ଗାଢ଼ ମୋଡ୍

ଉଦାହରଣ:

```swift
private var titleName: String {
    colorScheme == .light ? "ଆଲୋକ" : "ଅନ୍ଧାର"
}
```

ଏହି କୋଡ୍ ର ଅର୍ଥ ହେଉଛି:

ଯଦି ବର୍ତ୍ତମାନ ହାଲୁକା ମୋଡ୍, `colorScheme == .light` ସତ୍ୟ ହେଲେ, `"ଆଲୋକ"` ଫେରାଇବ।

ନହେଲେ, `"ଅନ୍ଧାର"` ଫେରାଇବ।

ତେଣୁ, ହାଲୁକା ମୋଡ୍ ରେ:

```swift
Text(titleName)
```

ଦେଖାଯାଉଥିବା ହେଉଛି:

```swift
ଆଲୋକ
```

ଗାଢ଼ ମୋଡ୍ ରେ, ଦେଖାଯାଉଥିବା ହେଉଛି:

```swift
ଅନ୍ଧାର
```

ଏହି ପ୍ରକାରରେ, ଆମେ ଭିନ୍ନ ଦୃଶ୍ୟ ଅନୁସାରେ ଭିନ୍ନ ବିଷୟବସ୍ତୁ ଦେଖାଇପାରୁ।

## ରଙ୍ଗ ଅନୁକଳନ ଉଦାହରଣ

ଟେକ୍ସ୍ଟ ଏବଂ ଆଇକନ୍ ବ୍ୟତୀତ, କେତେବେଳେ ଆମକୁ ନିଜେ ରଙ୍ଗ ଅନୁକଳନ ସମ୍ଭାଳିବାକୁ ପଡ଼େ।

ଉଦାହରଣ:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("ପାଠ୍ୟ")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

ପ୍ରଦର୍ଶନ ପ୍ରଭାବ:

![view](../../../Resource/025_view6.png)

ଏହି କୋଡ୍ ଭିତରେ, ଟେକ୍ସ୍ଟ ଧଳା ଏବଂ ପୃଷ୍ଠଭୂମି କଳା।

ହାଲୁକା ମୋଡ୍ ରେ, କଳା ପୃଷ୍ଠଭୂମି ଅଧିକ ସ୍ପଷ୍ଟ ଲାଗେ।

କିନ୍ତୁ ଗାଢ଼ ମୋଡ୍ ରେ, ଯଦି ସମ୍ପୂର୍ଣ୍ଣ ଇଣ୍ଟରଫେସ୍ ପୃଷ୍ଠଭୂମି ମଧ୍ୟ କଳା ହୁଏ, ଏହି କଳା ପୃଷ୍ଠଭୂମି ସିଷ୍ଟମ୍ ପୃଷ୍ଠଭୂମି ସହ ମିଶିଯିବ, ଦେଖିବାକୁ ପର୍ଯ୍ୟାପ୍ତ ସ୍ପଷ୍ଟ ଲାଗିବ ନାହିଁ।

ଏବେ, ଆମେ ଦୃଶ୍ୟ ମୋଡ୍ ଅନୁସାରେ ଟେକ୍ସ୍ଟ ରଙ୍ଗ ଏବଂ ପୃଷ୍ଠଭୂମି ରଙ୍ଗ ବଦଳାଇପାରୁ:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("ପାଠ୍ୟ")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

ପ୍ରଦର୍ଶନ ପ୍ରଭାବ:

![view](../../../Resource/025_view7.png)

ଏବେ, ହାଲୁକା ମୋଡ୍ ରେ କଳା ପୃଷ୍ଠଭୂମି ଏବଂ ଧଳା ଟେକ୍ସ୍ଟ ଅଛି।

ଗାଢ଼ ମୋଡ୍ ରେ ଧଳା ପୃଷ୍ଠଭୂମି ଏବଂ କଳା ଟେକ୍ସ୍ଟ ଅଛି।

ଏଭଳି କଲେ କୌଣସି ଦୃଶ୍ୟରେ ମଧ୍ୟ, ଟେକ୍ସ୍ଟ ଏବଂ ପୃଷ୍ଠଭୂମିର ସ୍ପଷ୍ଟ ବିପରୀତତା ରହେ, ଯାହାଦ୍ୱାରା ବ୍ୟବହାରକାରୀ ବିଷୟବସ୍ତୁକୁ ସ୍ପଷ୍ଟ ଭାବେ ଦେଖିପାରନ୍ତି।

## Assets ବ୍ୟବହାର କରି ରଙ୍ଗ ଅନୁକଳନ

`colorScheme` ବ୍ୟବହାର କରି ହାତେ ବିଚାର କରିବା ବ୍ୟତୀତ, ଆମେ `Assets` ରିସୋର୍ସ ରଙ୍ଗ ବ୍ୟବହାର କରି ହାଲୁକା/ଗାଢ଼ ମୋଡ୍ ଅନୁକଳନ କରିପାରୁ।

ଏହି ପଦ୍ଧତି App ଭିତରେ ସାଧାରଣ ରଙ୍ଗ ପରିଚାଳନା ପାଇଁ ଅଧିକ ଉପଯୁକ୍ତ।

![assets](../../../Resource/025_color.png)

`Assets` ଫୋଲ୍ଡର୍ ଖୋଲନ୍ତୁ, ଖାଲି ଅଞ୍ଚଳରେ ରାଇଟ୍-କ୍ଲିକ୍ କରନ୍ତୁ, `New Color Set` ବାଛନ୍ତୁ, ଏବଂ ଏକ ନୂତନ ରଙ୍ଗ ରିସୋର୍ସ ସୃଷ୍ଟି କରନ୍ତୁ।

## colorScheme ଏବଂ Assets ର ତଫାତ

`colorScheme` ଏବଂ `Assets` ଦୁଇଟି ମଧ୍ୟ ଦୃଶ୍ୟ ଅନୁକଳନ କରିପାରେ, କିନ୍ତୁ ସେମାନଙ୍କ ପାଇଁ ଉପଯୁକ୍ତ ସ୍ଥିତି ଭିନ୍ନ।

ଯଦି କେବଳ ହାଲୁକା/ଗାଢ଼ ମୋଡ୍ ଅନୁସାରେ ଟେକ୍ସ୍ଟ ବିଷୟବସ୍ତୁ, ଛବି ନାମ କିମ୍ବା `SF Symbols` ନାମ ବଦଳାଇବାକୁ ଥାଏ, `colorScheme` ବ୍ୟବହାର କରିପାରିବେ।

ଉଦାହରଣ:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

ଯଦି App ଭିତରେ ପ୍ରାୟସଃ ବ୍ୟବହୃତ ରଙ୍ଗ, ଯେପରି ଥିମ୍ ରଙ୍ଗ, କାର୍ଡ ପୃଷ୍ଠଭୂମି ରଙ୍ଗ, ଟେକ୍ସ୍ଟ ରଙ୍ଗ ଥାଏ, ତେବେ `Assets` ରଙ୍ଗ ବ୍ୟବହାର କରିବାକୁ ଅଧିକ ସୁପାରିଶ କରାଯାଏ।

କାରଣ `Assets` ରଙ୍ଗ ଅନେକ ଭ୍ୟୁ ଭିତରେ ପୁନର୍ବ୍ୟବହାର କରାଯାଇପାରେ, ପରେ ବଦଳାଇବା ମଧ୍ୟ ଅଧିକ ସହଜ।

ଉଦାହରଣ:

```swift
Color("redText")
```

ଏଭଳି କରି କେବଳ `Assets` ଭିତରେ ରଙ୍ଗ ବଦଳାଇଲେ, ଏହି ରଙ୍ଗ ବ୍ୟବହାର କରୁଥିବା ସମସ୍ତ ସ୍ଥାନ ଏକାସାଥିରେ ବଦଳିଯିବ।

## ସାରାଂଶ

ଏହି ପାଠରେ, ଆମେ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ର ଦୃଶ୍ୟ ଅନୁକଳନ ଶିଖିଲୁ।

ପ୍ରଥମେ, ଆମେ Xcode ର `Canvas` ଭିତରେ ଭିନ୍ନ ଦୃଶ୍ୟ କିପରି ପ୍ରିଭ୍ୟୁ କରିବା ଶିଖିଲୁ, ଏବଂ `Color Scheme Variants` ବ୍ୟବହାର କରି ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ କୁ ଏକେ ସମୟରେ କିପରି ଦେଖିବା ମଧ୍ୟ ଶିଖିଲୁ।

ତାପରେ, ଆମେ ଜାଣିଲୁ SwiftUI ଭିତରେ `Text`, `Image` ପରି ଭ୍ୟୁଗୁଡ଼ିକ ଡିଫଲ୍ଟ ଭାବେ ସିଷ୍ଟମ୍ ଦୃଶ୍ୟ ଅନୁସାରେ ରଙ୍ଗକୁ ସ୍ୱୟଂଚାଳିତ ଭାବେ ଅନୁକୂଳ କରେ।

କିନ୍ତୁ, ଯଦି ହାତେ ସ୍ଥିର ରଙ୍ଗ ନିର୍ଦ୍ଦିଷ୍ଟ କରାଯାଏ, ଯେପରି `Color.white` କିମ୍ବା `Color.black`, ଏହି ରଙ୍ଗଗୁଡ଼ିକ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବଦଳିବ ନାହିଁ।

ପରେ, ଆମେ `@Environment` ଏବଂ `colorScheme` ଶିଖିଲୁ:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` ମାଧ୍ୟମରେ, ଆମେ ବର୍ତ୍ତମାନ ହାଲୁକା ମୋଡ୍ କି ଗାଢ଼ ମୋଡ୍ ବିଚାର କରିପାରୁ, ଏବଂ ଭିନ୍ନ ଦୃଶ୍ୟ ଅନୁସାରେ ଭିନ୍ନ ଆଇକନ୍, ଟେକ୍ସ୍ଟ କିମ୍ବା ରଙ୍ଗ ଦେଖାଇପାରୁ।

ଶେଷରେ, ଆମେ `Assets` ରଙ୍ଗ ଶିଖିଲୁ।

`Assets` ରଙ୍ଗରେ ହାଲୁକା ମୋଡ୍ ଏବଂ ଗାଢ଼ ମୋଡ୍ ତଳେ ଅଲଗା ରଙ୍ଗ ସେଟ୍ କରିପାରିବେ, ଯାହା App ଭିତରେ ସାଧାରଣ ରଙ୍ଗ ରିସୋର୍ସ ପରିଚାଳନା ପାଇଁ ଉପଯୁକ୍ତ।

ଦୃଶ୍ୟ ଅନୁକଳନ App କୁ ଦିନେ ଏବଂ ରାତିରେ ଅଧିକ ଭଲ ପ୍ରଦର୍ଶନ ପ୍ରଭାବ ଦେଇପାରେ, ଏବଂ ଟେକ୍ସ୍ଟ ଅସ୍ପଷ୍ଟ, ପୃଷ୍ଠଭୂମି ଅସ୍ପଷ୍ଟ ଇତ୍ୟାଦି ସମସ୍ୟା ଏଡ଼ାଇପାରେ।

ବାସ୍ତବ ଡିଭେଲପମେଣ୍ଟରେ, ପ୍ରଥମେ SwiftUI ର ଡିଫଲ୍ଟ ଅନୁକଳନ କ୍ଷମତା ଏବଂ `Assets` ରଙ୍ଗ ବ୍ୟବହାର କରିବାକୁ ପରାମର୍ଶ ଦିଆଯାଏ।

ଯେତେବେଳେ ଭିନ୍ନ ଦୃଶ୍ୟ ତଳେ ଟେକ୍ସ୍ଟ, ଆଇକନ୍, ଛବି ବଦଳାଇବାକୁ ପଡ଼େ, କିମ୍ବା କିଛି ସ୍ୱଳ୍ପ ବିଶେଷ ପ୍ରଦର୍ଶନ ପ୍ରଭାବ ସମ୍ଭାଳିବାକୁ ପଡ଼େ, ସେତେବେଳେ ପୁଣି `colorScheme` ବ୍ୟବହାର କରି ବିଚାର କରନ୍ତୁ।

## ବିସ୍ତୃତ ଜ୍ଞାନ: ଫଣ୍ଟ ଆକାର ନିର୍ଦ୍ଦିଷ୍ଟ କରିବା

ପୂର୍ବରୁ `font` ଶିଖିବାବେଳେ, ଆମେ କିଛି ସିଷ୍ଟମ୍ ଫଣ୍ଟ ଶୈଳୀ ବ୍ୟବହାର କରିଥିଲୁ:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` ପରି ସିଷ୍ଟମ୍ ଫଣ୍ଟ ଶୈଳୀ ବ୍ୟବହାର କରିବା ବ୍ୟତୀତ, ହାତେ ଫଣ୍ଟ ଆକାର ମଧ୍ୟ ନିର୍ଦ୍ଦିଷ୍ଟ କରିପାରିବେ।

ଉଦାହରଣ:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

ଏଠାରେ:

```swift
.font(.system(size: 100))
```

ମାନେ ଫଣ୍ଟ ଆକାରକୁ `100 pt` କରି ସେଟ୍ କରିବା।

`SF Symbols` ଆଇକନ୍ ପାଇଁ, `font` ମଧ୍ୟ ଆଇକନ୍ ଆକାରକୁ ପ୍ରଭାବିତ କରେ।

ସେହିପାଇଁ, ଯଦି ଆଇକନ୍ ଅଧିକ ବଡ଼ ଦେଖାଇବାକୁ ଚାହୁଁଛନ୍ତି, `.font(.system(size:))` ମାଧ୍ୟମରେ ଆକାର ନିର୍ଦ୍ଦିଷ୍ଟ କରିପାରିବେ।

ଉଦାହରଣ:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

ଧ୍ୟାନ ଦେବା ଦରକାର, ହାତେ ଫଣ୍ଟ ଆକାର ନିର୍ଦ୍ଦିଷ୍ଟ କରିବା ଅଧିକ ଲଚିଳା ହେଲେ ମଧ୍ୟ, ସାଧାରଣ ଟେକ୍ସ୍ଟ ପାଇଁ ବହୁ ପରିମାଣରେ ସ୍ଥିର ଫଣ୍ଟ ଆକାର ବ୍ୟବହାର କରିବା ପରାମର୍ଶ ନୁହେଁ।

ସାଧାରଣ ଟେକ୍ସ୍ଟ ପାଇଁ ପ୍ରଥମେ `.title`, `.headline`, `.body`, `.caption` ପରି ସିଷ୍ଟମ୍ ଫଣ୍ଟ ଶୈଳୀ ବ୍ୟବହାର କରନ୍ତୁ।

ଏହା ଭିନ୍ନ ଡିଭାଇସ୍ ଏବଂ ବ୍ୟବହାରକାରୀଙ୍କ ଫଣ୍ଟ ସେଟିଂ ସହ ଅଧିକ ଭଲ ଭାବେ ଅନୁକୂଳ ହେବ।
