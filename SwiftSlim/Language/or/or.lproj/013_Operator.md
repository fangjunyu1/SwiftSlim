# ଅପରେଟର

ଏହି ପାଠରେ, ଆମେ Swift ର ଅପରେଟର (Operators) ବିଷୟରେ ଶିଖିବୁ।

ଯେତେବେଳେ ଆମେ ଗଣନା, ତୁଳନା କିମ୍ବା ତାର୍କିକ ନିଷ୍ପତ୍ତି କରୁଥାଉ, ସେତେବେଳେ ଅପରେଟର ଆବଶ୍ୟକ ହୋଇଥାଏ। ଏହି କାର୍ଯ୍ୟଗୁଡ଼ିକ ପ୍ରୋଗ୍ରାମ ଚାଲୁଥିବା ସମୟର ସମସ୍ତ ପର୍ଯ୍ୟାୟରେ ରହେ, ଏବଂ ଏହା ପ୍ରୋଗ୍ରାମିଙ୍ଗର ସବୁଠାରୁ ମୌଳିକ କ୍ଷମତାମାନଙ୍କ ମଧ୍ୟରୁ ଗୋଟିଏ।

```swift
let a = 100 * 8
````

ପ୍ରୋଗ୍ରାମିଙ୍ଗ ଶିଖିବା ପ୍ରକ୍ରିୟାରେ, ଅନେକେ ଦେଖନ୍ତି ଯେ ଯେଉଁମାନଙ୍କର ଗଣିତ ଭିତ୍ତିକ ଚିନ୍ତାଧାରା ଭଲ, ସେମାନେ ପ୍ରୋଗ୍ରାମର ଲଜିକ୍‌କୁ ଅଧିକ ସହଜରେ ବୁଝନ୍ତି। କାରଣ ପ୍ରୋଗ୍ରାମିଙ୍ଗର ମୂଳରେ “ଗଣନା” ଏବଂ “ନିୟମ ଆଧାରିତ ନିଷ୍ପତ୍ତି” ରହିଛି, ଏବଂ ଏହି ଦୁଇଟି କାମ ଅପରେଟର ଦ୍ୱାରା କରାଯାଏ।

ବାସ୍ତବିକ ବିକାଶରେ, ଅପରେଟର କେବଳ ସରଳ ସଂଖ୍ୟା ଗଣନା ପାଇଁ ନୁହେଁ, ବହୁତ ନିର୍ଦ୍ଦିଷ୍ଟ ପରିସ୍ଥିତିରେ ମଧ୍ୟ ବ୍ୟାପକ ଭାବରେ ବ୍ୟବହୃତ ହୁଏ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଗୋଟିଏ ବ୍ଲକ୍ ଗେମ୍‌ରେ, ବ୍ୟବହାରକାରୀ ଯେତେବେଳେ ଗୋଟିଏ ବ୍ଲକ୍‌କୁ ନିର୍ଦ୍ଦିଷ୍ଟ ସ୍ଥାନକୁ ଟାଣି ନେଇଯାଆନ୍ତି, ସେତେବେଳେ ଗଣନା କରିବାକୁ ପଡ଼େ:

* ବ୍ଲକ୍‌ର କୋଅର୍ଡିନେଟ୍ ସ୍ଥିତି
* ବ୍ଲକ୍ ଯେଉଁ ସ୍ଥାନଗୁଡ଼ିକୁ ପୂରଣ କରିପାରିବ
* ଲକ୍ଷ୍ୟ ସ୍ଥାନ ବ୍ୟବହାରଯୋଗ୍ୟ କି ନୁହେଁ ବିଚାର କରିବା

ଏହି “ଗଣନା + ବିଚାର” ପ୍ରକ୍ରିୟାମାନେ ମୂଳତଃ ଅପରେଟର ଉପରେ ନିର୍ଭର କରେ।

![Block](../../Resource/013_block.png)

ଅପରେଟର ଶିଖିବା ମାନେ କେବଳ ସଂଖ୍ୟା ଗଣନା ଶିଖିବା ନୁହେଁ, ପରବର୍ତ୍ତୀ UI layout, data processing ଏବଂ interaction logic ପାଇଁ ଭିତି ତିଆରି କରିବା ମଧ୍ୟ ଅଟେ।

## SwiftUI ଭିତରେ ଗଣନା କିପରି ଚାଲିଥାଏ

SwiftUI ଭିତରେ, ସାଧାରଣତଃ Button ଦ୍ୱାରା function କୁ trigger କରାଯାଏ, ଏବଂ function ଭିତରେ ଗଣନା logic ଲେଖାଯାଏ।

ଉଦାହରଣ:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

Button ଉପରେ କ୍ଲିକ୍ କଲେ, Console ରେ ଏହା ଦେଖାଯିବ:

```
c: 20
```

ଏହି ପାଠରେ, ଆମେ ସମସ୍ତ ଗଣନା logic କୁ func ଭିତରେ ଲେଖିବୁ, Button ଦ୍ୱାରା ତାହାକୁ trigger କରିବୁ, ଏବଂ print ବ୍ୟବହାର କରି ଫଳାଫଳ ଦେଖିବୁ।

ପରବର୍ତ୍ତୀ ଉଦାହରଣଗୁଡ଼ିକରେ, କୋଡ୍‌କୁ ସରଳ ଭାବେ ଦେଖାଇବା ପାଇଁ ଆମେ ପ୍ରତ୍ୟକ୍ଷ କୋଡ୍‌ରେ ଗଣନାର ଫଳ ଟୀକା ଭାବରେ ଲେଖିବୁ, ଏବଂ ପୁଣିଥରେ print ବ୍ୟବହାର କରିବୁ ନାହିଁ।

## Console ଆଉଟପୁଟ ପରିଚାଳନା

ପ୍ରତ୍ୟେକ କ୍ଲିକ୍‌ରେ Console ଭିତରେ ଆଉଟପୁଟ ସୃଷ୍ଟି ହେବାରୁ, Console ଅଞ୍ଚଳର ଡାହାଣତଳ କୋଣରେ ଥିବା clear button କୁ କ୍ଲିକ୍ କରି ଇତିହାସ ଲଗ୍‌ଗୁଡ଼ିକୁ ସଫା କରିପାରିବେ, ଯାହାରୁ ନବୀନତମ ଫଳାଫଳ ସୁବିଧାରେ ଦେଖିହେବ।

![Console](../../Resource/013_console.png)

## Button ର ଆଉ ଗୋଟିଏ ଲେଖନ ଶୈଳୀ

ଯଦି Button କେବଳ ଗୋଟିଏ function କୁ ଚଲାଏ, ତେବେ ଅଧିକ ସରଳ ଶୈଳୀ ବ୍ୟବହାର କରାଯାଇପାରେ:

```swift
Button(action: ) {
    // View
}
```

ଏହି ଲେଖନ ଶୈଳୀରେ, action ବୋଲିଲେ Button କ୍ଲିକ୍ ସମୟରେ ଚାଲୁଥିବା function (ଯାହାର କୌଣସି parameter ନାହିଁ) କୁ ବୁଝାଯାଏ, ଏବଂ curly braces {} Button ର view କୁ ନିର୍ଦ୍ଦିଷ୍ଟ କରେ।

ପୂର୍ବରୁ ଥିବା calculate function ସହିତ ଏହାକୁ ଏଭଳି ଲେଖିପାରିବା:

```swift
Button(action: calculate) {
    Text("Button")
}
```

Button ଉପରେ କ୍ଲିକ୍ କଲେ, calculate function ସିଧାସଳଖ ଭାବେ ଚାଲିବ।

ଏହି କାରଣରୁ, ଯେତେବେଳେ Button ର logic ସରଳ ଥାଏ ଏବଂ କେବଳ ଗୋଟିଏ function କୁ call କରିଥାଏ, action ଶୈଳୀ ଅଧିକ ସ୍ପଷ୍ଟ ଏବଂ code convention ସହିତ ଅଧିକ ସମଞ୍ଜସ।

## ଅଙ୍କଗତ ଅପରେଟର

ଅଙ୍କଗତ ଅପରେଟର (Arithmetic Operators) ସଂଖ୍ୟା ଗଣନା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ:

* `+` ଯୋଗ
* `-` ବିଯୋଗ
* `*` ଗୁଣନ
* `/` ଭାଗ
* `%` ଅବଶେଷ (ରିମେଣ୍ଡର)

ଉଦାହରଣ:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

ଏଠାରେ ଧ୍ୟାନ ଦେବା ଯୋଗ୍ୟ କଥା, Swift ଗୋଟିଏ strong typed language, ତେଣୁ ଏହା ଭିନ୍ନ data type ମାନଙ୍କୁ ସ୍ୱତଃସ୍ପୃତ ଭାବେ ମିଶାଏ ନାହିଁ:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

ଯେତେବେଳେ ଦୁଇଟି ମୂଲ୍ୟର type ଭିନ୍ନ ହୁଏ, ସେତେବେଳେ type କୁ ହାତେ ପରିବର୍ତ୍ତନ କରିବାକୁ ପଡ଼େ।

## ତୁଳନାତ୍ମକ ଅପରେଟର

ତୁଳନାତ୍ମକ ଅପରେଟର (Comparison Operators) ଦୁଇଟି ମୂଲ୍ୟର ସମ୍ବନ୍ଧକୁ ବିଚାର କରିବାକୁ ବ୍ୟବହୃତ ହୁଏ, ଏବଂ ଫଳ Bool ରୂପେ ମିଳେ:

* `==` ସମାନ
* `!=` ସମାନ ନୁହେଁ
* `>` ବଡ଼
* `<` ଛୋଟ
* `>=` ବଡ଼ କିମ୍ବା ସମାନ
* `<=` ଛୋଟ କିମ୍ବା ସମାନ

ଉଦାହରଣ:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

ତୁଳନାତ୍ମକ ଅପରେଟର ସାଧାରଣତଃ ନିଷ୍ପତ୍ତିକାରୀ statement ମାନଙ୍କରେ ବ୍ୟବହୃତ ହୁଏ। ଉଦାହରଣ ସ୍ୱରୂପ, “quotation carousel” ଭିତରେ ଆମେ index ଆରେର ସୀମାକୁ ଅତିକ୍ରମ ନ କରୁଥିବାକୁ ନିୟନ୍ତ୍ରଣ କରିବାକୁ ପଡ଼େ:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

ଯଦି index, sayings.count - 1 ଠାରୁ ଛୋଟ, ତେବେ ତୁଳନାର ଫଳ true ହେବ ଏବଂ if statement ଭିତରର କୋଡ୍ ଚାଲିବ; ଯଦି ଫଳ false ହୁଏ, ତେବେ ପରବର୍ତ୍ତୀ କୋଡ୍ ଚାଲିବ ନାହିଁ।

## ତାର୍କିକ ଅପରେଟର

ତାର୍କିକ ଅପରେଟର (Logical Operators) ଅନେକ ଶର୍ତ୍ତକୁ ଏକତ୍ର କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ:

* `&&` ଏବଂ (AND)
* `||` କିମ୍ବା (OR)
* `!` ନୁହେଁ (NOT)

### && ଏବଂ

ଯେତେବେଳେ ଅନେକ ଶର୍ତ୍ତ ସମସ୍ତେ ପୂରଣ ହେବା ଆବଶ୍ୟକ, ସେତେବେଳେ && ବ୍ୟବହାର କରାଯାଏ।

ଉଦାହରଣ:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

ଏହି କୋଡ୍‌ରେ ଦୁଇଟି ଶର୍ତ୍ତ ଅଛି: age >= 18 ଏବଂ age <= 100।

ଏହି ଦୁଇଟି ଶର୍ତ୍ତ ଦୁହିଁଟି true ହେଲେମାତ୍ର legalAge true ହେବ; ଯେକୌଣସି ଗୋଟିଏ ଶର୍ତ୍ତ false ହେଲେ, ଫଳ false ହେବ।

### || କିମ୍ବା

ଯେତେବେଳେ ଅନେକ ଶର୍ତ୍ତ ମଧ୍ୟରୁ କେବଳ ଗୋଟିଏ ପୂରଣ ହେଲେ ଚାଲିବ, ସେତେବେଳେ || ବ୍ୟବହାର କରାଯାଏ।

ଉଦାହରଣ:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ ହେଉଛି, money >= 50 କିମ୍ବା creditCard >= 50 ମଧ୍ୟରୁ ଯେକୌଣସି ଗୋଟିଏ ଠିକ୍ ହେଲେ, payAllowed true ହେବ।

ଦୁଇଟି ଶର୍ତ୍ତ ଉଭୟ false ହେଲେମାତ୍ର ଫଳ false ହେବ।

ଉଦାହରଣ ସ୍ୱରୂପ, payment ପରିସ୍ଥିତିରେ ନଗଦ କିମ୍ବା credit card ଯେକୌଣସି ଗୋଟିଏରେ ଦେୟ ଦେଇପାରିବା। ଯଦି ଉଭୟରେ ଦେଇପାରାଯାଏ ନାହିଁ, ତେବେ payment ବିଫଳ ହେବ।

### ! ନୁହେଁ

! ହେଉଛି logical NOT operator, ଯାହା Bool ମୂଲ୍ୟକୁ ପ୍ରତ୍ୟାବର୍ତ୍ତନ କରେ।

ସରଳ ଭାବରେ କହିଲେ, ! true କୁ false କରେ ଏବଂ false କୁ true କରେ।

ଉଦାହରଣ:

```swift
let allowed = true
let result = !allowed   // false
```

ଏଠାରେ !allowed ର ଅର୍ଥ ହେଉଛି allowed ର ମୂଲ୍ୟକୁ ପ୍ରତିକୂଳ କରିଦେବା।

ଏଠାରେ ଧ୍ୟାନ ରଖିବାକୁ ପଡ଼ିବ, ! ଏବଂ != ଦୁଇଟି ଭିନ୍ନ ଅପରେଟର; ! Bool ମୂଲ୍ୟକୁ ପ୍ରତ୍ୟାବର୍ତ୍ତନ କରେ, ଯେତେବେଳେ != ଗୋଟିଏ comparison operator ଏବଂ ସେହିଠାରେ Bool ଫଳ ମିଳେ।

## ନିୟୋଜନ ଅପରେଟର

ନିୟୋଜନ ଅପରେଟର (Assignment Operators) ମୂଲ୍ୟ ଦେବା କିମ୍ବା variable ଅଦ୍ୟତନ କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ:

* `=` ନିୟୋଜନ

Variable declaration କିମ୍ବା ଗଣନା ପ୍ରକ୍ରିୟାରେ, assignment operation ସବୁବେଳେ ଥାଏ:

```swift
let a = 5
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ, ମୂଲ୍ୟ 5 କୁ variable a କୁ ଦିଆଯାଇଛି।

### ସଂଯୁକ୍ତ ନିୟୋଜନ ଅପରେଟର

ବାସ୍ତବିକ development ରେ, ମୌଳିକ assignment operator ଛଡ଼ା, “compound assignment operators” ବହୁତ ବେଳେ ବ୍ୟବହୃତ ହୁଏ, ଯେଉଁମାନେ ପୂର୍ବତନ ମୂଲ୍ୟ ଉପରେ ସିଧାସଳଖ ଗଣନା କରି ତାହାକୁ ଅପଡେଟ୍ କରନ୍ତି:

* `+=`
* `-=`
* `*=`
* `/=`

ଆଗରୁ “counter” ପାଠରେ ଆମେ compound assignment operator ବିଷୟରେ ସରଳ ଭାବରେ ଦେଖିଛୁ।

ଉଦାହରଣ ସ୍ୱରୂପ, Button କ୍ଲିକ୍ କଲେ variable କୁ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ବଢ଼ାଇବା:

```swift
var num = 10
num += 5   // num = 15
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ, num ସହିତ 5 ଯୋଗ କରି, ତାହାର ଫଳକୁ ପୁଣି num କୁ ଦିଆଯାଉଛି।

ଏହି କାରଣରୁ, ଏହା ନିମ୍ନଲିଖିତ କୋଡ୍‌ ସମାନ:

```swift
var num = 10
num = num + 5   // num = 15
```

## ଟର୍ନାରି ଅପରେଟର

Swift ଭିତରେ ternary operator (ଟର୍ନାରି ଅପରେଟର) ଗୋଟିଏ ଶର୍ତ୍ତ ଆଧାରରେ ନିଷ୍ପତ୍ତି ନେଇ ଦୁଇଟି ଫଳାଫଳ ମଧ୍ୟରୁ ଗୋଟିଏକୁ ଫେରାଇଦିଏ:

```swift
ଶର୍ତ୍ତ ? ମୂଲ୍ୟ1 : ମୂଲ୍ୟ2
```

ଏହାର କାର୍ଯ୍ୟପ୍ରଣାଳୀ ହେଉଛି, ଯଦି ଶର୍ତ୍ତ true ହୁଏ, “ମୂଲ୍ୟ1” ଫେରେ; ଯଦି ଶର୍ତ୍ତ false ହୁଏ, “ମୂଲ୍ୟ2” ଫେରେ।

ଉଦାହରଣ:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

ଏହି କୋଡ୍‌ showColor ର ମୂଲ୍ୟକୁ ବିଚାର କରିବ: ଯଦି true, ତେବେ Color.blue ଫେରିବ ଏବଂ ପୃଷ୍ଠଭୂମି ନୀଳ ହେବ; ଯଦି false, ତେବେ Color.clear ଫେରିବ, ଅର୍ଥାତ୍ ପୃଷ୍ଠଭୂମି ରଙ୍ଗ ଦେଖାଯିବ ନାହିଁ।

ସୂଚନା: Color.clear ର ଅର୍ଥ ହେଉଛି transparent color (ସ୍ପଷ୍ଟ/ଅଦୃଶ୍ୟ ରଙ୍ଗ)।

### if-else ସହିତ ପାର୍ଥକ୍ୟ

ଟର୍ନାରି ଅପରେଟର ତାର ତାର୍କିକ ରୂପରେ if-else statement ସହିତ ସମାନ।

ଉପରୋକ୍ତ କୋଡ୍‌କୁ ଏଭଳି ଲେଖାଯାଇପାରେ:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

ଦୁହିଁଟିର କାର୍ଯ୍ୟ ଲଜିକ୍ ପୂରାପୂରି ସମାନ; ଶର୍ତ୍ତ true ହେଲେ ପ୍ରଥମ ଫଳ ମିଳେ, ଶର୍ତ୍ତ false ହେଲେ ଦ୍ୱିତୀୟ ଫଳ ମିଳେ।

ବ୍ୟବହାରରେ, ଟର୍ନାରି ଅପରେଟରର ଲେଖନ ଅଧିକ ସଂକ୍ଷିପ୍ତ ଏବଂ “ଦୁଇଟିରେ ଗୋଟିଏ” ପରି ସରଳ ନିଷ୍ପତ୍ତିକୁ ଦେଖାଇବାରେ ଅଧିକ ଉପଯୁକ୍ତ, ବିଶେଷକରି ସିଧାସଳଖ ଗୋଟିଏ ମୂଲ୍ୟ ଫେରାଇବା ସମୟରେ।

SwiftUI ର View ତିଆରି କରିବା ସମୟରେ, ternary operator ବହୁତ ସାଧାରଣ, ଯେପରି:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

ଏହି ଲେଖନ ଶୈଳୀ ଗୋଟିଏ ଲାଇନ୍ କୋଡ୍‌ରେ ଶର୍ତ୍ତ ଆଧାରରେ View ର ଦେଖାଯାଉଥିବା ପ୍ରଭାବକୁ ବଦଳାଇଦିଏ।

## ରେଞ୍ଜ ଅପରେଟର

ରେଞ୍ଜ ଅପରେଟର (Range Operators) ଗୋଟିଏ ଅବଧି କିମ୍ବା interval କୁ ପ୍ରଦର୍ଶନ କରିବାକୁ ବ୍ୟବହୃତ ହୁଏ:

* `...` ବନ୍ଦ interval (ଦୁଇ ପାର୍ଶ୍ୱକୁ ସମେତ)
* `..<` half-open interval (ଡାହାଣ ପାର୍ଶ୍ୱ ଅନ୍ତର୍ଭୁକ୍ତ ନୁହେଁ)

ଉଦାହରଣ:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

ସରଳ ଭାବେ ବୁଝିଲେ, 1...3 ର ଅର୍ଥ 1 ଠାରୁ 3 ପର୍ଯ୍ୟନ୍ତ, ଯେଉଁଠି 3 ଅନ୍ତର୍ଭୁକ୍ତ; 5..<8 ର ଅର୍ଥ 5 ଠାରୁ 8 ପର୍ଯ୍ୟନ୍ତ, କିନ୍ତୁ 8 ଅନ୍ତର୍ଭୁକ୍ତ ନୁହେଁ।

ବିସ୍ତାରିତ ରୂପ (କେବଳ ଜାଣିବା ପାଇଁ):

```
5...    // >= 5
...5    // <= 5
```

ଏହି ଲେଖନ ଶୈଳୀ “ଏକପାର୍ଶ୍ୱିକ range” କୁ ପ୍ରଦର୍ଶନ କରେ, ଯାହା ସାଧାରଣତଃ condition checking କିମ୍ବା pattern matching ରେ ବ୍ୟବହୃତ ହୁଏ।

ରେଞ୍ଜ ଅପରେଟର ପରେ for loop ଶିଖିବା ସମୟରେ ବ୍ୟବହୃତ ହେବ; ବର୍ତ୍ତମାନ ପାଇଁ ଏହାକୁ ଲଗାତାର ସଂଖ୍ୟାର ଗୋଟିଏ ଅବଧି ବୋଲି ବୁଝିଲେ ପର୍ଯ୍ୟାପ୍ତ।

## ଅପରେଟର ଅଭ୍ୟାସ

ଏବେ, କିଛି ସରଳ ଉଦାହରଣ ଦ୍ୱାରା ଆମେ View ଭିତରେ ଅପରେଟରର କାମକୁ ଅଭ୍ୟାସ କରିବା।

### 1. ଯାନବାହନର ମୋଟ ସଂଖ୍ୟା ଗଣନା

ଉଦାହରଣ ସ୍ୱରୂପ, ଗୋଟିଏ ରାସ୍ତାଚୌମୁହାନୀରୁ ପ୍ରତିଦିନ 500 ଟି ଗାଡ଼ି ଯାଏ, ତେବେ 30 ଦିନରେ ମୋଟ କେତେ ଗାଡ଼ି ଯିବ ତାହା ଗଣନା କରିବାକୁ ପଡ଼ିବ।

ଆମେ arithmetic operator ବ୍ୟବହାର କରିପାରିବା:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

ଏଠାରେ ଗଣନା ପ୍ରକ୍ରିୟା ହେଉଛି, ପ୍ରତିଦିନ 500 ଟି ଗାଡ଼ି, 30 ଦିନର ମୋଟ ହେଉଛି 500 × 30, ଏବଂ ଶେଷରେ 15000 ଟି ଗାଡ଼ିର ସମୁଦାୟ ମିଳିଲା।

### 2. ଇନ୍-ଆପ୍ ପର୍ଚେଜ୍ ସଦସ୍ୟ କି ନୁହେଁ ବିଚାର କରିବା

ଆମେ ଏକ ବ୍ୟକ୍ତି ଇନ୍-ଆପ୍ ପର୍ଚେଜ୍ ସଦସ୍ୟ କି ନୁହେଁ ବିଚାର କରିପାରିବା। ସେ permanent membership କିମ୍ବା subscription membership ମଧ୍ୟରୁ ଯେକୌଣସି ଗୋଟିଏ ରଖିଥିଲେ, ସେଠାରେ ତାଙ୍କୁ member ବୋଲି ଧରାଯିବ।

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

ଏହି କୋଡ୍‌ରେ || (OR) operator ବ୍ୟବହୃତ ହୋଇଛି: lifeTime = false ମାନେ permanent membership ନାହିଁ; subscription = true ମାନେ subscription membership ଅଛି।

ଯେତେବେଳେ ଦୁଇଟି ଶର୍ତ୍ତରୁ ଯେକୌଣସି ଗୋଟିଏ ସତ୍ୟ ହେଲେ ଚାଲିବ, isMember true ହେବ। ତେଣୁ, ଏହି ବ୍ୟକ୍ତି ଗୋଟିଏ ଇନ୍-ଆପ୍ ପର୍ଚେଜ୍ ସଦସ୍ୟ।

## ସାରାଂଶ

Swift ଭିତରେ, ଅପରେଟର data calculation ଏବଂ logic judgment ର ଭିତି, ଯେପରି Text content କୁ ଦେଖାଏ, ସେପରି ଅପରେଟର data କୁ ପ୍ରକ୍ରିୟାକରଣ କରେ।

ସରଳ ସଂଖ୍ୟା ଗଣନାଠାରୁ ଜଟିଳ ତାର୍କିକ ନିଷ୍ପତ୍ତି ପର୍ଯ୍ୟନ୍ତ, ଏବଂ View ର state control ଠାରୁ layout calculation ପର୍ଯ୍ୟନ୍ତ, ସବୁଠି ଅପରେଟର ଅଂଶଗ୍ରହଣ କରେ।

ଭିନ୍ନ ଭିନ୍ନ ପରିସ୍ଥିତିର ଅଭ୍ୟାସ ଦ୍ୱାରା, ଏହି ପାଠରେ ଉଲ୍ଲେଖ କରାଯାଇଥିବା ବିଭିନ୍ନ ଅପରେଟରକୁ ଧୀରେ ଧୀରେ ଚେନା ଓ ଆତ୍ମସାତ୍ କରାଯାଇପାରିବ, ଏବଂ ଏହା ପରବର୍ତ୍ତୀ ଶିକ୍ଷା ପାଇଁ ଗୁରୁତ୍ୱପୂର୍ଣ୍ଣ ଭିତି ହେବ।

### ଅତିରିକ୍ତ ଜ୍ଞାନ - ବିଟ୍‌ୱାଇଜ୍ ଅପରେଟର

ଏହାଛଡ଼ା, Swift ଆଉ ଗୋଟିଏ ଅଧିକ ନିମ୍ନସ୍ତରୀୟ operator group ମଧ୍ୟ ଦେଇଥାଏ — bitwise operators:

* `&`（ବିଟ୍‌ୱାଇଜ୍ AND）
* `|`（ବିଟ୍‌ୱାଇଜ୍ OR）
* `^`（ବିଟ୍‌ୱାଇଜ୍ XOR）
* `~`（ବିଟ୍‌ୱାଇଜ୍ NOT）
* `>>`（ଡାହାଣକୁ ସ୍ଥାନାନ୍ତର）
* `<<`（ବାମକୁ ସ୍ଥାନାନ୍ତର）

ଉଦାହରଣ:

```swift
let a = 6  // ବାଇନେରି 110
let b = 3  // ବାଇନେରି 011

print(a & b) // 2 (ବାଇନେରି 010)
print(a | b) // 7 (ବାଇନେରି 111)
print(a ^ b) // 5 (ବାଇନେରି 101)
print(~a)    // -7 (ପ୍ରତିକୂଳ)
```

Swift ଭିତରେ, integer ମାନେ binary two's complement ରେ ପ୍ରତିନିଧିତ୍ୱ କରାଯାଆନ୍ତି, ତେଣୁ bitwise NOT ପରେ ସମ୍ବନ୍ଧିତ ନେଗେଟିଭ୍ ଫଳ ମିଳେ।

Bitwise operator ଗୁଡ଼ିକ ସିଧାସଳଖ binary ଉପରେ କାମ କରେ, ଏବଂ ସାଧାରଣତଃ lower-level data processing କିମ୍ବା performance optimization ପରି ପରିସ୍ଥିତିରେ ବ୍ୟବହୃତ ହୁଏ।

ଆରମ୍ଭିକ ଶିକ୍ଷାର୍ଥୀଙ୍କ ପାଇଁ, ଏହି ପ୍ରକାର operator ର ବ୍ୟବହାର ପରିସ୍ଥିତି କମ୍ ଥାଏ, ଏବଂ ସାଧାରଣ iOS କିମ୍ବା SwiftUI development ଭିତରେ ମଧ୍ୟ ଏଗୁଡ଼ିକ ବହୁତ ସାଧାରଣ ନୁହେଁ, ତେଣୁ ଏଠାରେ ଏହାକୁ ଗଭୀରରେ ବ୍ୟାଖ୍ୟା କରାଯାଉନାହିଁ।

ପରେ ଯଦି lower-level logic କିମ୍ବା performance optimization ବିଷୟ ଆସେ, ସେତେବେଳେ ଏହାକୁ ଆଉ ଗଭୀରରେ ଶିଖିବା ଏବଂ ବୁଝିବାକୁ ପାରିବେ।
