# ବଟନ୍‌କୁ କାମ କରାନ୍ତୁ

ଏହି ପାଠରେ ଆମେ ପ୍ରଧାନତଃ SwiftUI ର `Button` ବଟନ୍‌, `print` ଦ୍ୱାରା ଡିବଗ୍ ସୂଚନା ଆଉଟପୁଟ୍ କରିବା, ଏବଂ Swift ଫଙ୍କସନ୍‌ ବିଷୟରେ ଶିଖିବୁ।

ଏହି ସମସ୍ତ ବିଷୟ SwiftUI ଇଣ୍ଟର୍ଆକ୍ଟିଭ୍ ପ୍ରୋଗ୍ରାମିଙ୍ଗର ମୂଳ ଆଧାର ଅଟେ।

## ବଟନ୍

ବଟନ୍ ହେଉଛି ଆମେ ସବୁଠାରୁ ସାଧାରଣଭାବେ ଦେଖୁଥିବା UI କଣ୍ଟ୍ରୋଲ୍‌। ଯେତେବେଳେ ଆମକୁ ଡାଉନଲୋଡ୍‌, ଖୋଲିବା କିମ୍ବା ବନ୍ଦ କରିବା ପରି କାମ କରିବାକୁ ପଡେ, ସାଧାରଣତଃ ଆମେ ବଟନ୍‌କୁ କ୍ଲିକ୍ କରୁଥାଉ।

ଉଦାହରଣ ସ୍ୱରୂପ, App Store ରେ “获取” ବଟନ୍‌କୁ କ୍ଲିକ୍ କରି App ଡାଉନଲୋଡ୍ କରାଯାଏ।

![Button](../../RESOURCE/008_button.png)

SwiftUI ରେ ବଟନ୍‌କୁ `Button` ବୋଲି ପ୍ରତିନିଧିତ୍ୱ କରାଯାଏ।

ମୂଳ ଲେଖା ପ୍ରକ୍ରିୟା:

```swift
Button("") {
    
}
````

ଏହାର ଗଠନକୁ ଦୁଇଟି ଅଂଶରେ ବିଭକ୍ତ କରିହେବ:

```swift
Button("ବଟନ୍ ଟେକ୍ସ୍ଟ") {
    ବଟନ୍ କ୍ଲିକ୍ କଲେ ଚାଲିବା କୋଡ୍
}
```

ଉଦାହରଣ:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

ରନ୍ କରିଲେ ଏକ `Start` ବଟନ୍ ଦେଖାଯିବ।

![Button](../../RESOURCE/008_button1.png)

ଯେତେବେଳେ ୟୁଜର୍ ବଟନ୍‌କୁ କ୍ଲିକ୍ କରିବେ, `{}` ମଧ୍ୟରେ ଥିବା କୋଡ୍ ଚାଲିବ।

ଏଠାରେ ଧ୍ୟାନ ଦେବାକୁ ଯୋଗ୍ୟ କଥା ହେଲା: `Button` ହେଉଛି SwiftUI ର ଇଣ୍ଟରଫେସ୍ କଣ୍ଟ୍ରୋଲ୍ (`View`)। ସେହିପରି ଏହାକୁ ନିଶ୍ଚୟ ଭାବରେ `body` ଭିତରେ ଲେଖିବାକୁ ପଡିବ।

### `buttonStyle` ମୋଡିଫାୟର୍

SwiftUI ରେ `Button` ପାଇଁ ସିଷ୍ଟମ୍‌ ଭିତରେ ନିର୍ମିତ କିଛି ବଟନ୍ ଷ୍ଟାଇଲ୍ ଥାଏ, ଯାହାକୁ `buttonStyle` ମୋଡିଫାୟର୍ ଦ୍ୱାରା ବ୍ୟବହାର କରିହେବ।

ଉଦାହରଣ:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

`buttonStyle` ସିଷ୍ଟମ୍‌ ଦ୍ୱାରା ଦିଆଯାଇଥିବା ବଟନ୍ ଷ୍ଟାଇଲ୍‌କୁ ବ୍ୟବହାର କରେ।

ସାଧାରଣ ବିକଳ୍ପଗୁଡିକ:

* `.automatic`
* `.plain`
* `.bordered`
* `.borderedProminent`
* `.borderless`

![Button](../../RESOURCE/008_button7.png)

ଭିନ୍ନ ଭିନ୍ନ ଷ୍ଟାଇଲ୍‌ର ମୁଖ୍ୟ ତଫାତ ହେଉଛି ବଟନ୍‌ରେ ବୋର୍ଡର୍‌, ପୃଷ୍ଠଭୂମି ଏବଂ ଭିଜୁଆଲ୍ ଜୋର କେତେ ଅଛି।

## `print` ଆଉଟପୁଟ୍

ପ୍ରୋଗ୍ରାମିଙ୍ଗରେ, ଅନେକ ସମୟ ଆମକୁ ଦେଖିବାକୁ ପଡେ ଯେ ପ୍ରୋଗ୍ରାମ୍ କୌଣସି ନିର୍ଦ୍ଦିଷ୍ଟ କୋଡ୍ ଅଂଶକୁ ପହଞ୍ଚିଛି କି ନାହିଁ।

Swift ରେ ଡିବଗ୍ ସୂଚନା ଆଉଟପୁଟ୍ ପାଇଁ `print` ଫଙ୍କସନ୍ ଦିଆଯାଇଛି।

ମୂଳ ଲେଖା:

```swift
print("Hello")
```

ଏହି କୋଡ୍ ଟେକ୍ସ୍ଟକୁ Console (କନ୍ଟ୍ରୋଲ୍ କନ୍ସୋଲ୍) ରେ ଆଉଟପୁଟ୍ କରିବ।

### ବଟନ୍ କାମ କରୁଛି କି ନାହିଁ ଯାଞ୍ଚ

ଆମେ `print` କୁ ବଟନ୍‌ ମଧ୍ୟରେ ଲେଖି ବଟନ୍ କ୍ଲିକ୍ ହୋଇଛି କି ନାହିଁ ଯାଞ୍ଚ କରିପାରିବା।

```swift
Button("Start") {
    print("Test 123")
}
```

ବଟନ୍ କ୍ଲିକ୍ କଲେ, Console ରେ ଏହା ଦେଖାଯିବ:

```
Test 123
```

ଏହାର ଅର୍ଥ ହେଉଛି ବଟନ୍ ସଫଳତାର ସହ ଟ୍ରିଗର୍ ହୋଇଛି।

![Button](../../RESOURCE/008_button3.png)

ଧ୍ୟାନ ଦିଅନ୍ତୁ: `print` ହେଉଛି Swift କୋଡ୍‌, UI କଣ୍ଟ୍ରୋଲ୍‌ ନୁହେଁ। ଏହାକୁ ସିଧାସଳଖ `body` ଭିତରେ ଲେଖିହେବ ନାହିଁ, ନହେଲେ ଏରର୍ ଆସିବ।

ଉଦାହରଣ ସ୍ୱରୂପ, ନିମ୍ନପ୍ରକାର ଲେଖା ଭୁଲ୍:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // ଭୁଲ୍: body ନିଶ୍ଚୟ ଭାବରେ ଏକ View ଫେରାଇବାକୁ ପଡିବ
    }
}
```

SwiftUI ରେ `body` ଇଣ୍ଟରଫେସ୍‌ର ଗଠନ ବର୍ଣ୍ଣନା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ତେଣୁ ଏଠାରେ କେବଳ UI କଣ୍ଟ୍ରୋଲ୍ (`View`) ମାନେ ଥାଇପାରିବେ, ଯଥା `Text`, `Button`, `Image` ଇତ୍ୟାଦି।

ସାଧାରଣ କୋଡ୍‌ (ଗଣନା ଲଜିକ୍‌, `print` ଇତ୍ୟାଦି) ବଟନ୍‌, ଫଙ୍କସନ୍ କିମ୍ବା ଅନ୍ୟ କୌଣସି ଇଭେଣ୍ଟ ହ୍ୟାଣ୍ଡଲର୍‌ ଭିତରେ ରଖିବାକୁ ପଡେ।

### Console କନ୍ସୋଲ୍

`print` ଦ୍ୱାରା ଆଉଟପୁଟ୍ ହୋଇଥିବା ସମଗ୍ରୀ Xcode ର Console (କନ୍ସୋଲ୍) ରେ ଦେଖାଯାଏ।

ଯଦି Console ଦେଖାଯାଉନାହିଁ, ତେବେ ଡାହାଣ ପଟି ତଳେ ଥିବା `Show the Debug Area` ବଟନ୍‌କୁ କ୍ଲିକ୍ କରି ଡିବଗ୍ ଏରିଆ ଖୋଲନ୍ତୁ, ପରେ `Show the Console` ବଟନ୍‌କୁ କ୍ଲିକ୍ କରି କନ୍ସୋଲ୍ ଦେଖାନ୍ତୁ।

![](../../RESOURCE/008_button4.png)

Console ର ବାମ ତଳ କୋଣରେ ଦୁଇଟି ଟ୍ୟାବ୍ ଥାଏ: `Executable` ଏବଂ `Previews`।

![](../../RESOURCE/008_button5.png)

ଏହି ଦୁଇଟି ଟ୍ୟାବ୍ ଭିନ୍ନ ରନ୍‌ ପରିବେଶକୁ ସୂଚିତ କରେ: Canvas preview ଭ୍ୟୁରେ ଥିଲେ `print` ର ଆଉଟପୁଟ୍ `Previews` ଟ୍ୟାବ୍ ତଳେ ଦେଖାଯିବ; ଯଦି ଆପ୍‌କୁ simulator କିମ୍ବା ଆସଲି ଡିଭାଇସ୍‌ରେ ଚଳାଉଛନ୍ତି, ଆଉଟପୁଟ୍ `Executable` ଟ୍ୟାବ୍ ତଳେ ଦେଖାଯିବ।

ସେହିପରି, ଯଦି ଆପଣଙ୍କୁ ଲାଗେ `print` ର କୌଣସି ଆଉଟପୁଟ୍ ନାହିଁ, ପ୍ରଥମେ ସଠିକ୍ ଟ୍ୟାବ୍ ଚୟନ କରିଛନ୍ତି କି ନାହିଁ ଯାଞ୍ଚ କରନ୍ତୁ।

## କେସ୍ - ASCII ଚରିତ୍ର ଚିତ୍ର

କିଛି କୋଡ୍ ପ୍ରୋଜେକ୍ଟରେ ଆମେ ପ୍ରାୟତଃ ASCII ଚରିତ୍ର ଚିତ୍ର ଦେଖୁଥାଉ।

ଚରିତ୍ର ଚିତ୍ର ମାନେ ସାଧାରଣ ଚରିତ୍ର ଦ୍ୱାରା ଗଠିତ ଚିତ୍ର। ଉଦାହରଣ:

![ASCII](../../RESOURCE/008_ascii.png)

ଆମେ ବଟନ୍ + `print` ଦ୍ୱାରା ଗୋଟିଏ ASCII ଚରିତ୍ର ଚିତ୍ର ଆଉଟପୁଟ୍ କରିପାରିବା।

ଉଦାହରଣ କୋଡ୍:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

ବଟନ୍ କ୍ଲିକ୍ କଲେ, Console ରେ ଏହା ଆଉଟପୁଟ୍ ହେବ:

```
 /\_/\
( o.o )
 > ^ <
```

ଧ୍ୟାନ ଦିଅନ୍ତୁ: ଷ୍ଟ୍ରିଙ୍ଗ ମଧ୍ୟରେ `\` ଲେଖିବାକୁ `\\` ବ୍ୟବହାର କରିବାକୁ ପଡେ, କାରଣ backslash ଷ୍ଟ୍ରିଙ୍ଗ ଭିତରେ ଏକ escape character ଅଟେ।

## ଫଙ୍କସନ୍

ବଟନ୍‌ ଭିତରେ ଆମେ ସିଧାସଳଖ କୋଡ୍ ଲେଖିପାରୁ।

ଉଦାହରଣ:

```swift
Button("Start") {
    print("Hello")
}
```

କୋଡ୍ କମ୍ ଥିଲେ ଏଭଳି ଲେଖିବାରେ କୌଣସି ସମସ୍ୟା ନାହିଁ। କିନ୍ତୁ ବାସ୍ତବ ଡିଭେଲପମେଣ୍ଟରେ, ଗୋଟିଏ ବଟନ୍ ଟ୍ରିଗର୍ କରୁଥିବା ଲଜିକ୍ ଅନେକ ହୋଇପାରେ।

ଉଦାହରଣ:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // ସମ୍ଭବତଃ ଆଉ ଅଧିକ କୋଡ୍ ଥାଇପାରେ
}
```

ଯେତେବେଳେ କୋଡ୍ ବଢ଼ିବ, ସେତେବେଳେ ବଟନ୍‌ ଭିତରେ ସିଧାସଳଖ ଲେଖିବାରୁ UI କୋଡ୍ ବହୁତ ଅବ୍ୟବସ୍ଥିତ ହୋଇଯିବ, ଏବଂ ପଢିବା ଓ ରକ୍ଷଣାବେକ୍ଷଣ କରିବା ଅସୁବିଧା ହେବ।

ତେଣୁ, ଆମେ ସାଧାରଣତଃ ଏହି କୋଡ୍‌ଗୁଡିକୁ ଗୋଟିଏ ଫଙ୍କସନ୍‌ରେ ରଖିବା ସହିତ, ବଟନ୍‌ ଭିତରେ ସେହି ଫଙ୍କସନ୍‌କୁ କଲ୍ କରୁଥାଉ।

### ଫଙ୍କସନ୍ କ’ଣ

ଫଙ୍କସନ୍ ହେଉଛି ଏକ ପୁନଃବ୍ୟବହାରଯୋଗ୍ୟ କୋଡ୍ ଖଣ୍ଡ।

ଯେତେବେଳେ କୌଣସି କୋଡ୍ ଚଲାଇବାକୁ ଥାଏ, ଆମକୁ କେବଳ ସେହି ଫଙ୍କସନ୍‌କୁ କଲ୍ କରିବାକୁ ପଡେ, ତେବେ ଫଙ୍କସନ୍‌ ଭିତରର କୋଡ୍ ଚାଲିଯିବ।

ଏହା କୋଡ୍‌ର ଗଠନକୁ ଅଧିକ ସ୍ପଷ୍ଟ କରେ, ଏବଂ ପୁଣିପୁଣି ବ୍ୟବହାର ପାଇଁ ସହଜ କରେ।

### ମୂଳ ଲେଖା ପ୍ରକ୍ରିୟା

Swift ରେ ଫଙ୍କସନ୍‌କୁ `func` କୀୱାର୍ଡ୍‌ ଦ୍ୱାରା ଡିଫାଇନ୍ କରାଯାଏ:

```swift
func randomInt() {
    // code
}
```

ଏହି କୋଡ୍ ଗୋଟିଏ ଫଙ୍କସନ୍ ଡିଫାଇନ୍ କରୁଛି।

`randomInt` ହେଉଛି ଫଙ୍କସନ୍‌ର ନାମ, ଯାହା ଏହି କୋଡ୍‌ ଖଣ୍ଡକୁ ପରିଚୟ କରେ।

`() `ହେଉଛି ପ୍ୟାରାମିଟର୍ ଅଂଶ; ଏଠାରେ ବାହାରରୁ ଆସୁଥିବା ଡାଟା ନିଆଯାଇପାରେ। ଯଦି ଡାଟା ଦରକାର ନାହିଁ, ତେବେ ଏହାକୁ ଖାଲି ରଖାଯାଇପାରେ।

`{}` ଭିତରେ ହେଉଛି ଫଙ୍କସନ୍‌ର କୋଡ୍ ଅଂଶ, ଯେଉଁଠାରେ ଚଲାଇବାକୁ ଥିବା ସମସ୍ତ କୋଡ୍ ଲେଖାଯାଏ।

ଉଦାହରଣ:

```swift
func getName() {
    print("FangJunyu")
}
```

ଏହି ଫଙ୍କସନ୍‌ର କାମ ହେଉଛି ଗୋଟିଏ ଟେକ୍ସ୍ଟ ଆଉଟପୁଟ୍ କରିବା।

### ଫଙ୍କସନ୍ କଲ୍ କରିବା

ଫଙ୍କସନ୍ ଡିଫାଇନ୍ କରିବାର ଅର୍ଥ କେବଳ ସେହି କୋଡ୍ ତିଆରି କରିବା। ଯଦି ଏହାକୁ ଚଲାଇବାକୁ ଚାହୁଁଛନ୍ତି, ତେବେ ସେହି ଫଙ୍କସନ୍‌କୁ କଲ୍ କରିବାକୁ ପଡିବ।

କଲ୍ କରିବା ପ୍ରକ୍ରିୟା ହେଉଛି ଫଙ୍କସନ୍‌ର ନାମ ପରେ `()` ଯୋଡିବା:

```swift
getName()
```

ପ୍ରୋଗ୍ରାମ୍ ଏହି ଲାଇନ୍‌କୁ ଚାଲାଇଲେ, `getName` ଫଙ୍କସନ୍‌ର ଭିତରେ ଥିବା କୋଡ୍ ଚାଲିଯିବ।

ସମ୍ପୂର୍ଣ୍ଣ ଉଦାହରଣ:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

ରନ୍ କରିଲେ, Console ରେ ଏହା ଆଉଟପୁଟ୍ ହେବ:

```
FangJunyu
```

### SwiftUI ରେ ଫଙ୍କସନ୍ ବ୍ୟବହାର

SwiftUI view ଭିତରେ, ଫଙ୍କସନ୍ ସାଧାରଣତଃ `body` ବାହାରେ ଲେଖାଯାଏ।

ଉଦାହରଣ:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

ବଟନ୍ କ୍ଲିକ୍ କଲେ, `getName()` ଫଙ୍କସନ୍ କଲ୍ ହେବ।

ଏହା ଦ୍ୱାରା ଇଣ୍ଟରଫେସ୍ କୋଡ୍ ସଂକ୍ଷିପ୍ତ ରହିଥାଏ, ଏବଂ ନିର୍ଦ୍ଦିଷ୍ଟ ଲଜିକ୍‌କୁ ଫଙ୍କସନ୍‌ ମାଧ୍ୟମରେ ହ୍ୟାଣ୍ଡଲ୍ କରାଯାଏ।

### ଫଙ୍କସନ୍ ପ୍ୟାରାମିଟର୍

କେତେକ ସମୟରେ, ଗୋଟିଏ ଫଙ୍କସନ୍‌କୁ ଭିନ୍ନ ଡାଟା ଅନୁଯାୟୀ ଭିନ୍ନ କାମ କରିବାକୁ ପଡେ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଆମେ ଚାହୁଁଛୁ ଫଙ୍କସନ୍‌ଟି କେବଳ ନିର୍ଦ୍ଦିଷ୍ଟ `"FangJunyu"` ନୁହେଁ, ବରଂ ଭିନ୍ନ ନାମ ଆଉଟପୁଟ୍ କରୁ।

ସେତେବେଳେ ପ୍ୟାରାମିଟର୍ ବ୍ୟବହାର କରାଯାଏ। ପ୍ୟାରାମିଟର୍‌କୁ ଏଭଳି ବୁଝି ପାରିବେ: ଫଙ୍କସନ୍ କଲ୍ କରିବା ସମୟରେ ଭିତରକୁ ପଠାଯାଉଥିବା ଡାଟା।

ଉଦାହରଣ:

```swift
func getName(name: String) {
    print(name)
}
```

ଏହି ଫଙ୍କସନ୍‌ରେ ନୂତନ ଗୋଟିଏ ପ୍ୟାରାମିଟର୍ ଯୋଡାଯାଇଛି। `name` ହେଉଛି ପ୍ୟାରାମିଟର୍‌ର ନାମ, ଏବଂ `String` ହେଉଛି ତାହାର ପ୍ରକାର।

ଏହାର ଅର୍ଥ, ଫଙ୍କସନ୍‌ଟିକୁ `String` ପ୍ରକାରର ଡାଟା ଗ୍ରହଣ କରିବାକୁ ପଡିବ।

ଯେହେତୁ ଏବେ ଫଙ୍କସନ୍‌ଟିକୁ ଡାଟା ଦରକାର, ସେଥିପାଇଁ କଲ୍ କରିବା ବେଳେ ନିଶ୍ଚୟ ସେହି ଡାଟା ଦେବାକୁ ପଡିବ।

```swift
getName(name: "Sam")
```

ଏଠାରେ କଲ୍ କରିବାବେଳେ `"Sam"` ଦିଆଯାଇଛି, ତେଣୁ ଫଙ୍କସନ୍‌ ଭିତରେ ସେହି ମୂଲ୍ୟକୁ ଆଉଟପୁଟ୍ କରାଯିବ।

ରନ୍ କରିବାର ଫଳାଫଳ:

```
Sam
```

## କେସ୍ - ଫଙ୍କସନ୍ ଦ୍ୱାରା ASCII ଚରିତ୍ର ଚିତ୍ର ଆଉଟପୁଟ୍

ଆଗରୁ ଥିବା ଉଦାହରଣରେ, ଆମେ `print` କୁ ସିଧାସଳଖ ବଟନ୍‌ ଭିତରେ ଲେଖିଥିଲୁ।

କିନ୍ତୁ ଯଦି ଚରିତ୍ର ଚିତ୍ରଟି ଅଧିକ ଜଟିଳ ହୋଇଥାଏ, କୋଡ୍‌ ବହୁତ ଲମ୍ବା ହୋଇଯିବ। ସେତେବେଳେ ଲଜିକ୍‌କୁ ଫଙ୍କସନ୍‌ ଭିତରେ ରଖିବା ଦ୍ୱାରା UI କୋଡ୍‌କୁ ଅଧିକ ସ୍ପଷ୍ଟ କରିହେବ।

ଉଦାହରଣ କୋଡ୍:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

ରନ୍ କରି ବଟନ୍ କ୍ଲିକ୍ କଲେ, Console ରେ ଏହା ଆଉଟପୁଟ୍ ହେବ:

```
 /\_/\
( o.o )
 > ^ <
```

ଏଠାରେ ଦୁଇଟି କାମ ଘଟୁଛି। ଯେତେବେଳେ ୟୁଜର୍ ବଟନ୍‌କୁ କ୍ଲିକ୍ କରନ୍ତି:

1. ବଟନ୍ `printCat()` ଫଙ୍କସନ୍‌କୁ କଲ୍ କରେ
2. ଫଙ୍କସନ୍‌ ଭିତରେ ଥିବା `print` କୋଡ୍ ଚାଲିଯାଏ

ଏହା ଦ୍ୱାରା UI କୋଡ୍‌ କେବଳ ଇଭେଣ୍ଟ ଟ୍ରିଗର୍ କରେ, ଏବଂ ନିର୍ଦ୍ଦିଷ୍ଟ ଲଜିକ୍‌କୁ ଫଙ୍କସନ୍‌ ହ୍ୟାଣ୍ଡଲ୍ କରେ।

ବାସ୍ତବ ଡିଭେଲପମେଣ୍ଟରେ, ଅଧିକାଂଶ ବଟନ୍ କାର୍ଯ୍ୟ ଫଙ୍କସନ୍ ଦ୍ୱାରା ହିଁ କରାଯାଏ।

## ସାରାଂଶ

ଏହି ପାଠରେ ଆମେ ତିନିଟି ମୁଖ୍ୟ ବିଷୟ ଶିଖିଲୁ:

**1. `Button` ବଟନ୍**

`Button` ହେଉଛି SwiftUI ରେ କାର୍ଯ୍ୟ ଟ୍ରିଗର୍ କରିବା ପାଇଁ ବ୍ୟବହୃତ କଣ୍ଟ୍ରୋଲ୍।

```swift
Button("Start") {

}
```

ଯେତେବେଳେ ୟୁଜର୍ ବଟନ୍ କ୍ଲିକ୍ କରନ୍ତି, curly braces ଭିତରର କୋଡ୍ ଚାଲିଯାଏ।

**2. `print` ଆଉଟପୁଟ୍**

`print` କନ୍ସୋଲ୍‌ରେ ଡିବଗ୍ ସୂଚନା ଆଉଟପୁଟ୍ କରିବାକୁ ବ୍ୟବହୃତ ହୁଏ।

```swift
print("Hello")
```

ଡିଭେଲପର୍ Console ଦ୍ୱାରା ପ୍ରୋଗ୍ରାମ୍‌ର ଚାଲୁଥିବା ଅବସ୍ଥା ଦେଖିପାରନ୍ତି।

**3. ଫଙ୍କସନ୍**

ଫଙ୍କସନ୍ ହେଉଛି ଏକ ପୁନଃବ୍ୟବହାରଯୋଗ୍ୟ କୋଡ୍ ଖଣ୍ଡ।

```swift
func sayHello() {
    print("Hello")
}
```

ଫଙ୍କସନ୍ କଲ୍ କରିବା:

```swift
sayHello()
```

ଯଦି ଫଙ୍କସନ୍‌କୁ ଡାଟା ଦରକାର, ତେବେ ପ୍ୟାରାମିଟର୍ ବ୍ୟବହାର କରିପାରିବେ:

```swift
func sayHello(name: String) {
    print(name)
}
```

ପରବର୍ତ୍ତୀ ପାଠଗୁଡିକରେ ଆମେ ଆଉ ଅଧିକ SwiftUI କଣ୍ଟ୍ରୋଲ୍ ଏବଂ ଡାଟା କିପରି UI ସହ ଇଣ୍ଟରାକ୍ଟ କରେ ସେଥି ପାଇଁ ଶିଖିବୁ।

## ଅଭ୍ୟାସ

ନିମ୍ନଲିଖିତ ଅଭ୍ୟାସଗୁଡିକ କରିବାକୁ ଚେଷ୍ଟା କରନ୍ତୁ:

1. ଗୋଟିଏ ବଟନ୍ ତିଆରି କରନ୍ତୁ, ବଟନ୍‌ର ଟେକ୍ସ୍ଟ `"Hello"` ହେଉ
2. ବଟନ୍ କ୍ଲିକ୍ କଲେ Console ରେ `Hello Swift` ଆଉଟପୁଟ୍ କରନ୍ତୁ
3. ଆଉଟପୁଟ୍ କୋଡ୍‌କୁ ଗୋଟିଏ ଫଙ୍କସନ୍‌ରେ ରଖନ୍ତୁ, ତାପରେ ବଟନ୍‌ରେ ସେହି ଫଙ୍କସନ୍‌କୁ କଲ୍ କରନ୍ତୁ
