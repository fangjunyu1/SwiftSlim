# କାଉଣ୍ଟର

ପୂର୍ବର ଦୁଇଟି ପାଠରେ ଆମେ ଭେରିଏବଲ୍, କନ୍ଷ୍ଟାଣ୍ଟ, ବଟନ୍ ଏବଂ ମେଥଡ୍ ବିଷୟରେ ଶିଖିଥିଲୁ।

ଏହି ପାଠରେ ଆମେ ସେହି ଜ୍ଞାନଗୁଡିକୁ ଏକସାଥିରେ ବ୍ୟବହାର କରି, ଗୋଟିଏ ସରଳ କାଉଣ୍ଟର ଆପ୍ ତିଆରି କରିବୁ।

ଏହି ଉଦାହରଣରେ, ଆମେ ଭେରିଏବଲ୍‌କୁ କିପରି ସଞ୍ଚୟ ଏବଂ ପରିବର୍ତ୍ତନ କରିବା, ସହିତ `@State` ବ୍ୟବହାର କରି ଭେରିଏବଲ୍‌କୁ କିପରି ପରିଚାଳନା କରିବା ଶିଖିବୁ।

## କାଉଣ୍ଟର ତିଆରି କରିବା

କାଉଣ୍ଟର ସାଧାରଣତଃ ନିରନ୍ତର ଭାବେ ବୃଦ୍ଧି ହେଉଥିବା ସଂଖ୍ୟାକୁ ରେକର୍ଡ କରିବା ପାଇଁ ବ୍ୟବହୃତ ହୁଏ। ଯଥା, ଦାଡ଼ିଲାଫର ସଂଖ୍ୟା, ଦୌଡ଼ର ଚକ୍ର ସଂଖ୍ୟା ପରି ଗଣନା ସମ୍ବନ୍ଧୀୟ ପରିସ୍ଥିତି।

ଏବେ ଆମେ ଗୋଟିଏ ସରଳ କାଉଣ୍ଟର ତିଆରି କରିବୁ: ଗୋଟିଏ ସଂଖ୍ୟା ଏବଂ ଗୋଟିଏ ବଟନ୍ ଦେଖାଇବା, ଏବଂ ବଟନ୍‌କୁ କ୍ଲିକ୍ କଲେ ସଂଖ୍ୟା ବଢ଼ିଯିବ।

![Num](../../RESOURCE/009_num.png)

### ସଂଖ୍ୟା ଦେଖାଇବା

ପ୍ରଥମେ, ଆମେ `Text` ବ୍ୟବହାର କରି ସଂଖ୍ୟାକୁ ଦେଖାଇବୁ।

```swift
Text("0")
```

କିନ୍ତୁ, ଯଦି ସଂଖ୍ୟାଟି ପରିବର୍ତ୍ତନଶୀଳ ହୋଇଥାଏ, ତେବେ `"0"` କୁ ସିଧାସଳଖ ଲେଖିବା ଠିକ୍ ହେବ ନାହିଁ; ବରଂ ଏହାକୁ ଏକ ଭେରିଏବଲ୍‌ରେ ସଞ୍ଚୟ କରିବା ଉଚିତ।

```swift
var num = 0
```

ତାପରେ `Text` ଭିତରେ ସେହି ଭେରିଏବଲ୍‌କୁ ଦେଖାଇବା:

```swift
Text("\(num)")
```

ଏଠାରେ `\()` ଷ୍ଟ୍ରିଙ୍ଗ୍ ଇଣ୍ଟରପୋଲେସନ୍ ବ୍ୟବହାର ହୋଇଛି, ଯାହା ସଂଖ୍ୟାକୁ ଷ୍ଟ୍ରିଙ୍ଗ୍ କରି ଦେଖାଏ।

### ବଟନ୍‌ରେ ସଂଖ୍ୟା ବଢ଼ାଇବା

ଏବେ, ଆମେ ଗୋଟିଏ ବଟନ୍ ଯୋଡିବୁ।

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

ବଟନ୍‌ଟି କ୍ଲିକ୍ ହେଲେ, ଆମେ ଚାହୁଁଛୁ ସଂଖ୍ୟା ବଢ଼ୁ।

ସେହିପାଇଁ ବଟନ୍‌ର ଭିତରେ ଭେରିଏବଲ୍‌କୁ ପରିବର୍ତ୍ତନ କରିପାରିବା:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

ଏହି କୋଡ୍‌ର ଅର୍ଥ ହେଉଛି: ବଟନ୍ କ୍ଲିକ୍ କଲେ, `num + 1` ଗଣନା କରାଯିବ ଏବଂ ଫଳାଫଳକୁ `num` କୁ ଅସାଇନ୍ କରାଯିବ।

### ଅଧିକ ସଂକ୍ଷିପ୍ତ ଲେଖା ପ୍ରକ୍ରିୟା

Swift ଏହା ପାଇଁ ଗୋଟିଏ ଅଧିକ ସଂକ୍ଷିପ୍ତ ଲେଖା ପ୍ରକ୍ରିୟା ଦେଇଛି:

```swift
num += 1
```

ଏହା ନିମ୍ନଲିଖିତ କୋଡ୍ ସହ ସମାନ:

```swift
num = num + 1
```

ଏହି ଲେଖା ପ୍ରକ୍ରିୟାକୁ “କମ୍ପାଉଣ୍ଡ ଆସାଇନମେଣ୍ଟ ଅପରେଟର୍” କୁହାଯାଏ।

Swift ରେ ସାଧାରଣ କମ୍ପାଉଣ୍ଡ ଆସାଇନମେଣ୍ଟ ଅପରେଟର୍‌ଗୁଡିକ ହେଲେ:

```
+=   ଯୋଗ ଆସାଇନମେଣ୍ଟ
-=   ବିୟୋଗ ଆସାଇନମେଣ୍ଟ
*=   ଗୁଣନ ଆସାଇନମେଣ୍ଟ
/=   ଭାଗ ଆସାଇନମେଣ୍ଟ
%=   ଶେଷଫଳ ଆସାଇନମେଣ୍ଟ
```

ଉଦାହରଣ:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### ସମ୍ପୂର୍ଣ୍ଣ କୋଡ୍

ଏବେ, ଆମେ ଗୋଟିଏ ସରଳ କାଉଣ୍ଟର ଲେଖିପାରିବୁ:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

କିନ୍ତୁ, ଏହି କୋଡ୍ ଏବେ ସଠିକ୍ ଭାବେ ଚାଲିବ ନାହିଁ।

### ଏରର୍ ଦେଖାଯାଏ

Xcode ଏହିପରି ସୂଚନା ଦେବ:

```swift
Cannot assign to property: 'self' is immutable
```

ଏହାର ଅର୍ଥ, ପ୍ରୋପର୍ଟିକୁ ପରିବର୍ତ୍ତନ କରିହେବ ନାହିଁ, କାରଣ view ଅପରିବର୍ତ୍ତନଶୀଳ (immutable) ଅଟେ।

ଏହା କାହିଁକି ହେଉଛି?

## SwiftUI ର କାମ କରିବା ଉପାୟ

SwiftUI ରେ: **ଇଣ୍ଟରଫେସ୍ ଡାଟା ଦ୍ୱାରା ଚାଳିତ ହୁଏ**, ଅର୍ଥାତ୍ ଯେତେବେଳେ ଡାଟା ପରିବର୍ତ୍ତନ ହୁଏ, ଇଣ୍ଟରଫେସ୍ ସ୍ୱୟଂଚାଳିତ ଭାବେ ରିଫ୍ରେଶ୍ ହୁଏ।

କିନ୍ତୁ ଧ୍ୟାନ ଦିଅନ୍ତୁ, କେବଳ ଷ୍ଟେଟ୍ (`@State`, `@Binding` ଇତ୍ୟାଦି) ପରିବର୍ତ୍ତନ ହେଲେମାତ୍ର SwiftUI ସେହି ପରିବର୍ତ୍ତନକୁ ନଜର ରଖି view କୁ ରିଫ୍ରେଶ୍ କରେ।

ଯଦି ସାଧାରଣ ଭେରିଏବଲ୍ ବ୍ୟବହାର କରାଯାଏ, SwiftUI view ସୃଷ୍ଟି ସମୟରେ ସେହି ଭେରିଏବଲ୍‌ର ମୂଲ୍ୟକୁ କେବଳ ଗୋଟେଥର ପଢ଼େ। ତାପରେ ଭେରିଏବଲ୍‌ର ମୂଲ୍ୟ ବଦଳିଲେ ମଧ୍ୟ, ଇଣ୍ଟରଫେସ୍ ଅପଡେଟ୍ ହୁଏନି।

ଉଦାହରଣ:

```swift
var num = 0   // 0
num = 1   // 0
```

ଉପରୋକ୍ତ ଉଦାହରଣରେ, `num` ଏକ ସାଧାରଣ ଭେରିଏବଲ୍। ଯଦିଓ ତାହାର ମୂଲ୍ୟ ବଦଳିଯାଇଛି, SwiftUI view କୁ ଅପଡେଟ୍ କରିବ ନାହିଁ, ଏବଂ ଇଣ୍ଟରଫେସ୍ ଏଭଳି ଆରମ୍ଭିକ `0` କୁ ହିଁ ଦେଖାଇବ।

![Num](../../RESOURCE/009_state.png)

ତେଣୁ, କେବଳ ଯେତେବେଳେ ଡାଟାକୁ ଷ୍ଟେଟ୍ ଭାବେ ଚିହ୍ନିତ କରାଯାଏ (`@State`, `@Binding` ପରି ପ୍ରୋପର୍ଟି ର୍ୟାପର୍ ବ୍ୟବହାର କରି), SwiftUI ତାହାର ପରିବର୍ତ୍ତନକୁ ନଜର ରଖି ସ୍ୱୟଂଚାଳିତ ଭାବେ view କୁ ଅପଡେଟ୍ କରିଥାଏ।

## `@State` ପ୍ରୋପର୍ଟି ର୍ୟାପର୍

SwiftUI ରେ, ଯଦି ଭେରିଏବଲ୍‌କୁ ପରିବର୍ତ୍ତନ କରିବା ସହ view କୁ ମଧ୍ୟ ଅପଡେଟ୍ କରିବାକୁ ଥାଏ, ତେବେ ନିଶ୍ଚୟ `@State` ବ୍ୟବହାର କରିବାକୁ ପଡିବ।

```swift
@State var num = 0
```

`@State` SwiftUI କୁ ଏହି ଭେରିଏବଲ୍‌କୁ ନଜର ରଖିବାକୁ କହେ।

ଯେତେବେଳେ ଭେରିଏବଲ୍‌ର ମୂଲ୍ୟ ପରିବର୍ତ୍ତନ ହୁଏ, SwiftUI view କୁ ପୁନର୍ଗଣନା କରେ ଏବଂ ଏହାକୁ ଅପଡେଟ୍ କରେ।

ସମ୍ପୂର୍ଣ୍ଣ କୋଡ୍:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

ଏବେ, ବଟନ୍ କ୍ଲିକ୍ କଲେ:

```swift
num += 1
```

SwiftUI ବୁଝିପାରିବ `num` ବଦଳିଛି, ଏବଂ ସ୍ୱୟଂଚାଳିତ ଭାବେ view କୁ ଅପଡେଟ୍ କରିଦେବ।

## `@State` ବ୍ୟବହାର ନିୟମ

ବାସ୍ତବ ଡିଭେଲପମେଣ୍ଟରେ, `@State` ସାଧାରଣତଃ ଦୁଇଟି ନିୟମ ଅନୁସରଣ କରେ।

### 1. `@State` ସାଧାରଣତଃ `private` ସହ ବ୍ୟବହୃତ ହୁଏ

`@State` ଭେରିଏବଲ୍‌ଗୁଡିକ ସାଧାରଣତଃ ବର୍ତ୍ତମାନ ଟାଇପ୍‌ର (ଯଥା ବର୍ତ୍ତମାନର `View` struct) ଭିତରେ ମାତ୍ର ବ୍ୟବହୃତ ହୁଏ, ସେଥିପାଇଁ ଏଭଳି ଲେଖାଯାଏ:

```swift
@State private var num = 0
```

ଏହା ଅନ୍ୟ view ମାନଙ୍କୁ ଏହି ଭେରିଏବଲ୍‌କୁ ସିଧାସଳଖ ଆକ୍ସେସ୍ କିମ୍ବା ପରିବର୍ତ୍ତନ କରିବାରୁ ରୋକେ।

ଏବେ ଆମେ `private` ବିଷୟରେ ଅଧ୍ୟୟନ କରିନାହୁଁ, ତେଣୁ ସରଳ ଭାବେ ଏଭଳି ବୁଝନ୍ତୁ:

**`private` ଦ୍ୱାରା ସଂଶୋଧିତ ବିଷୟବସ୍ତୁ କେବଳ ବର୍ତ୍ତମାନ view ଭିତରେ ବ୍ୟବହାର ହୋଇପାରେ, ବାହାରର view ଗୁଡିକ ଏହାକୁ ବ୍ୟବହାର କରିପାରିବ ନାହିଁ।**

### 2. `@State` କେବଳ ବର୍ତ୍ତମାନ view ର ଷ୍ଟେଟ୍ ପାଇଁ ବ୍ୟବହୃତ ହୁଏ

`@State` ବର୍ତ୍ତମାନ `View` ନିଜେ ପରିଚାଳନା କରୁଥିବା ଡାଟା ପାଇଁ ଉପଯୁକ୍ତ।

ଯଥା, ଇନପୁଟ୍ ବକ୍ସର ବିଷୟବସ୍ତୁ, ସ୍ୱିଚ୍‌ର ଅବସ୍ଥା।

```swift
@State private var isOn = false
@State private var text = ""
```

ଯଦି ଡାଟାକୁ ଅନେକ view ମଧ୍ୟରେ ଶେୟାର୍ କରିବାକୁ ପଡେ, ତେବେ `@Binding` କିମ୍ବା `@Observable` ପରି ଅନ୍ୟ ଷ୍ଟେଟ୍ ଟାଇପ୍ ବ୍ୟବହାର କରିବାକୁ ପଡେ।

ଏହି ବିଷୟଗୁଡିକୁ ପରବର୍ତ୍ତୀ ପାଠରେ ଶିଖିବା।

### କମାଇବା ବଟନ୍

କାଉଣ୍ଟରରେ କେବଳ ବଢ଼ାଇବା ନୁହେଁ, କମାଇବା ମଧ୍ୟ ଥାଇପାରେ।

ଆମେ ଗୋଟିଏ `-` ବଟନ୍ ଯୋଡିପାରିବୁ।

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

ତାପରେ `HStack` ବ୍ୟବହାର କରି ବଟନ୍‌ଗୁଡିକୁ ଅଣ୍ଡାଠାରେ ବ୍ୟବସ୍ଥା କରିବା:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

ଫଳାଫଳ:

![Num](../../RESOURCE/009_num1.png)

`+` ବଟନ୍ କ୍ଲିକ୍ କଲେ, ସଂଖ୍ୟା 1 ବଢ଼ିବ; `-` ବଟନ୍ କ୍ଲିକ୍ କଲେ, ସଂଖ୍ୟା 1 କମିବ।

### ରିସେଟ୍ ବଟନ୍

ଆମେ ଗୋଟିଏ ରିସେଟ୍ ବଟନ୍ ମଧ୍ୟ ଯୋଡିପାରିବୁ, ଯାହା ସଂଖ୍ୟାକୁ 0 କରିଦେବ।

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

ଯେତେବେଳେ ରିସେଟ୍ ବଟନ୍‌କୁ କ୍ଲିକ୍ କରାଯାଏ, `num` କୁ `0` ଅସାଇନ୍ କରାଯାଏ।

ଫଳାଫଳ:

![Num](../../RESOURCE/009_num2.png)

`0` ବଟନ୍ କ୍ଲିକ୍ କଲେ, ସଂଖ୍ୟା 0 ହୋଇଯାଏ।

### କଷ୍ଟମ୍ ବଟନ୍

ଏବେ ଆମର ବଟନ୍‌ଗୁଡିକ କେବଳ ଟେକ୍ସ୍ଟ ଦେଖାଇପାରୁଛି:

```swift
Button("+") {

}
```

ଏହି ଲେଖା ପ୍ରକ୍ରିୟାର ବଟନ୍ କେବଳ ଟେକ୍ସ୍ଟ ବିଷୟବସ୍ତୁ ଦେଖାଇପାରେ, ଯଥା `+` କିମ୍ବା `-`।

ବାସ୍ତବ ଡିଭେଲପମେଣ୍ଟରେ, ଆମେ ସାଧାରଣତଃ ବଟନ୍‌ର ଦୃଶ୍ୟକୁ ଅଧିକ ସମୃଦ୍ଧ କରିବାକୁ ଚାହୁଁ, ଯଥା ଆଇକନ୍ କିମ୍ବା ଅନ୍ୟ view ବ୍ୟବହାର କରି।

SwiftUI ଆମକୁ ବଟନ୍‌ର ଦେଖାଯାଉଥିବା ବିଷୟବସ୍ତୁକୁ କଷ୍ଟମାଇଜ୍ କରିବାର ସୁଯୋଗ ଦେଇଥାଏ। ଏଥି ପାଇଁ ବଟନ୍‌ର ଅନ୍ୟ ଗୋଟିଏ ଲେଖା ପ୍ରକ୍ରିୟା ବ୍ୟବହାର କରାଯାଏ:

```swift
Button(action: {

}, label: {

})
```

ଏହି ଲେଖା ପ୍ରକ୍ରିୟାରେ, `action` ବଟନ୍ କ୍ଲିକ୍ କଲେ ଚାଲିବା କୋଡ୍‌କୁ ସୂଚାଏ, ଏବଂ `label` ବଟନ୍‌ରେ ଦେଖାଯାଉଥିବା view କୁ ସୂଚାଏ।

ଉଦାହରଣସ୍ୱରୂପ, ଆମେ ବଟନ୍‌ରେ ଗୋଟିଏ SF Symbols ଆଇକନ୍ ଦେଖାଇପାରିବୁ:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

ଏବେ ବଟନ୍‌ରେ ଟେକ୍ସ୍ଟ ନୁହେଁ, ଗୋଟିଏ ଆଇକନ୍ ଦେଖାଯିବ।

### କାଉଣ୍ଟର ଇଣ୍ଟରଫେସ୍‌କୁ ଆହୁରି ଭଲ କରିବା

କାଉଣ୍ଟର ଇଣ୍ଟରଫେସ୍‌କୁ ଆହୁରି ସ୍ପଷ୍ଟ କରିବା ପାଇଁ, ଆମେ ସଂଖ୍ୟାର ଦେଖାଯିବା ଷ୍ଟାଇଲ୍‌କୁ ବଦଳାଇପାରିବା। ଯଥା, ସଂଖ୍ୟାକୁ ବଡ଼ କରିବା:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

ତାପରେ, ଆମେ ବଟନ୍‌ର ଦେଖାଯାଉଥିବା ବିଷୟବସ୍ତୁକୁ ମଧ୍ୟ ଆଇକନ୍ କରିଦେବା, ଏବଂ ଆଇକନ୍‌ର ଆକାରକୁ କିଛି ବଡ଼ କରିବା:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

ସେହିପରି, ଆମେ କମାଇବା ବଟନ୍ ଏବଂ ରିସେଟ୍ ବଟନ୍‌କୁ ମଧ୍ୟ ଆଇକନ୍ ରୂପରେ ବଦଳାଇପାରିବୁ:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

ଏବେ ବଟନ୍‌ଗୁଡିକ SF Symbols ଆଇକନ୍‌ ଦେଖାଇବ, ଏବଂ ଇଣ୍ଟରଫେସ୍ ଆହୁରି ସ୍ପଷ୍ଟ ହେବ।

## ସାରାଂଶ

ଏହି ପାଠରେ ଆମେ ଗୋଟିଏ ସରଳ କାଉଣ୍ଟର ତିଆରି କରି, ଭେରିଏବଲ୍‌, `Text` ଏବଂ `Button` ବ୍ୟବହାରକୁ ସମନ୍ୱିତ ଭାବରେ ଅଭ୍ୟାସ କଲୁ। ଆମେ ଭେରିଏବଲ୍ ବ୍ୟବହାର କରି ସଂଖ୍ୟା ସଞ୍ଚୟ କଲୁ, ଏବଂ ବଟନ୍‌ ଦ୍ୱାରା ସଂଖ୍ୟାକୁ ବଢ଼ାଇଲୁ କିମ୍ବା କମାଇଲୁ, ସହିତ `num += 1` ଏବଂ `num -= 1` ପରି କମ୍ପାଉଣ୍ଡ ଆସାଇନମେଣ୍ଟ ଅପରେଟର୍‌ଗୁଡିକୁ ମଧ୍ୟ ଶିଖିଲୁ।

ଏହି ପ୍ରକ୍ରିୟାରେ, ଆମେ SwiftUI ର କାମ କରିବା ଉପାୟକୁ ମଧ୍ୟ ବୁଝିଲୁ: ଇଣ୍ଟରଫେସ୍ ଡାଟା ଦ୍ୱାରା ଚାଳିତ ହୁଏ। ଯେତେବେଳେ ଡାଟା ପରିବର୍ତ୍ତନ ହୁଏ, ଇଣ୍ଟରଫେସ୍ ରିଫ୍ରେଶ୍ ହୁଏ। ସେହିପାଇଁ ପରିବର୍ତ୍ତନଶୀଳ ଷ୍ଟେଟ୍‌କୁ ସଞ୍ଚୟ କରିବା ପାଇଁ `@State` ବ୍ୟବହାର କରିବା ଆବଶ୍ୟକ। ଯେତେବେଳେ `@State` ଭେରିଏବଲ୍‌ର ମୂଲ୍ୟ ବଦଳେ, SwiftUI ସ୍ୱୟଂଚାଳିତ ଭାବେ view କୁ ଅପଡେଟ୍ କରେ।

ଶେଷରେ, ଆମେ ଇଣ୍ଟରଫେସ୍‌ର ସାଧାରଣ ସୁଧାର କଲୁ: `Text` ର ଫଣ୍ଟ ଷ୍ଟାଇଲ୍ ବଦଳାଇଲୁ, ଏବଂ `Button(action:label:)` ବ୍ୟବହାର କରି ବଟନ୍‌ର ବିଷୟବସ୍ତୁକୁ କଷ୍ଟମାଇଜ୍ କଲୁ, ଯାହା ଫଳରେ ବଟନ୍‌ଗୁଡିକ ଆଇକନ୍ ଦେଖାଇପାରୁଛନ୍ତି, ଏବଂ କାଉଣ୍ଟର ଇଣ୍ଟରଫେସ୍ ଆହୁରି ସ୍ପଷ୍ଟ ଏବଂ ସୁନ୍ଦର ହୋଇଛି।

ଏବେ, ଆମେ SwiftUI view ର ସବୁଠାରୁ ମୂଳ ବ୍ୟବହାର ଉପାୟକୁ ସ୍ପର୍ଶ କରିଛୁ। SwiftUI ରେ, ଅଧିକାଂଶ ଇଣ୍ଟରଫେସ୍ ଭେରିଏବଲ୍‌ ଦ୍ୱାରା ଚାଳିତ ହୁଏ। ଯେତେବେଳେ ଭେରିଏବଲ୍‌ର ମୂଲ୍ୟ ପରିବର୍ତ୍ତନ ହୁଏ, SwiftUI ସ୍ୱୟଂଚାଳିତ ଭାବେ view କୁ ଅପଡେଟ୍ କରେ। ସେଥିପାଇଁ SwiftUI ଆପ୍ ଡିଭେଲପ କରିବାବେଳେ, ଆମେ ସାଧାରଣତଃ ପ୍ରଥମେ କଣ କଣ ଡାଟା ସଞ୍ଚୟ କରିବାକୁ ପଡିବ ସେଥିକି ଡିଜାଇନ୍ କରୁଥାଉ, ତାପରେ ସେହି ଡାଟା ଉପରେ ଭିତ୍ତି କରି ଇଣ୍ଟରଫେସ୍ ଏବଂ ଇଣ୍ଟର୍ଆକ୍ସନ୍ ଲଜିକ୍ ଗଠନ କରୁଥାଉ।

## ସମ୍ପୂର୍ଣ୍ଣ କୋଡ୍

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
