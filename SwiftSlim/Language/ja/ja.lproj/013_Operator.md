# 演算子

このレッスンでは、Swift の演算子（Operators）を学びます。

計算、比較、論理判定を行うときには、どれも演算子が必要です。これらの操作はプログラム実行のあらゆる段階に関わっており、プログラミングにおける最も基本的な能力の一つです。

```swift
let a = 100 * 8
```

プログラミングを学ぶ過程で、多くの人は「ある程度の数学的思考力を持つ人の方が、プログラムのロジックを理解しやすい」と感じます。なぜなら、プログラミングの本質は「計算」と「ルール判定」から切り離せず、それこそが演算子の担う中心的な役割だからです。

実際の開発では、演算子は単純な数値計算だけでなく、さまざまな具体的な場面で広く使われています。

たとえば、ブロックゲームでユーザーがブロックを指定位置へドラッグしたときには、次のようなことを計算しなければなりません：

- ブロックの座標位置
- ブロックを埋められる位置
- その位置が使用可能かどうかの判定

こうした「計算 + 判定」の過程は、本質的にすべて演算子に依存しています。

![Block](../../Resource/013_block.png)

演算子を学ぶことは、単に数字の計算を学ぶだけではなく、その後の UI レイアウト、データ処理、インタラクションロジックの土台を築くことでもあります。

## SwiftUI における計算実行の方法

SwiftUI では、通常 `Button` によって関数を発火し、その関数の中で演算ロジックを実行します。

たとえば：

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

ボタンをクリックすると、Console には次のように出力されます：

```
c: 20
```

このレッスンでは、演算ロジックを一律に `func` の中に書き、ボタンで実行し、`print` を使って結果を出力する形に統一します。

後続の例では、コードを見やすくするために、計算結果をコード内に直接注記し、毎回 `print` を繰り返し使うことはしません。

## Console 出力管理

クリックするたびに出力が増えるため、Console 領域右下のクリアボタンを押して履歴ログを消すと、現在の結果を見やすくできます。

## Button のもう一つの書き方

ボタンが 1 つの関数だけを実行する場合は、もっと簡潔な書き方ができます：

```swift
Button(action: ) {
    // View
}
```

この書き方では、`action` がボタンクリック時に実行する関数（引数なし関数）を指定し、中括弧 `{}` はボタンのビューを定義します。

前の `calculate` 関数と組み合わせると、次のように書けます：

```swift
Button(action: calculate) {
    Text("Button")
}
```

ボタンを押すと、`calculate` 関数が直接実行されます。

そのため、ボタンロジックが単純で、しかも 1 つの関数だけを呼び出す場合には、`action` を使う書き方の方が分かりやすく、コード規約にもより適しています。

## 算術演算子

算術演算子（Arithmetic Operators）は数値計算に使われます：

- `+` 加算
- `-` 減算
- `*` 乗算
- `/` 除算
- `%` 剰余（余り）

例：

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

注意すべき点として、Swift は強い型付けを持つ言語なので、異なるデータ型を自動で混在させることはありません：

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

型が異なる場合は、手動で型変換する必要があります。

## 比較演算子

比較演算子（Comparison Operators）は、2 つの値の関係を判定し、結果は `Bool` になります：

- `==` 等しい
- `!=` 等しくない
- `>` より大きい
- `<` より小さい
- `>=` 以上
- `<=` 以下

例：

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

比較演算子は条件文でよく使われます。たとえば「名言カルーセル」では、インデックスが配列範囲を超えないように制御する必要がありました：

```swift
if index < sayings.count - 1 {
    index += 1
}
```

もし `index` が `sayings.count - 1` より小さければ、比較結果は `true` となり、`if` 文の中のコードが実行されます。比較結果が `false` なら、後続のコードは実行されません。

## 論理演算子

論理演算子（Logical Operators）は、複数の条件を組み合わせるために使います：

- `&&` AND
- `||` OR
- `!` NOT

### && AND

複数の条件が同時に満たされる必要がある場合は、`&&` を使います。

たとえば：

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

このコードには、`age >= 18` と `age <= 100` の 2 つの条件が含まれています。

この 2 つが同時に `true` のときだけ、`legalAge` は `true` になります。どちらか一つでも成立しなければ、結果は `false` です。

### || OR

複数の条件のうち、どれか一つだけ満たせばよい場合は、`||` を使います。

たとえば：

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

このコードの意味は、`money >= 50` または `creditCard >= 50` のどちらか一方でも成立すれば、`payAllowed` は `true` になるということです。

両方とも成立しない場合にだけ、結果は `false` になります。

たとえば支払いの場面では、現金またはクレジットカードのどちらかで支払えます。両方とも使えない場合は、支払い失敗になります。

### ! NOT

`!` は論理否定演算子（NOT）で、`Bool` 値を反転させるために使います。

簡単に言えば、`!` は `true` を `false` に、`false` を `true` に変えます。

たとえば：

```swift
let allowed = true
let result = !allowed   // false
```

ここでの `!allowed` は、`allowed` の値を反転することを意味します。

注意すべきなのは、`!` と `!=` は別の演算子だということです。`!` は `Bool` 値を反転し、`!=` は比較演算子で `Bool` 値を返します。

## 代入演算子

代入演算子（Assignment Operators）は、値の代入や変数の更新に使います：

- `=` 代入

変数宣言でも計算過程でも、代入操作は必ず登場します：

```swift
let a = 5
```

このコードは、数値 5 を変数 `a` に代入することを意味します。

### 複合代入演算子

実際の開発では、基本の代入演算子に加えて、「複合代入演算子」もよく使われます。これは、元の値をもとに計算を行い、その結果で直接更新するためのものです：

- `+=`
- `-=`
- `*=`
- `/=`

前の「カウンター」のレッスンでも、複合代入演算子には簡単に触れました。

たとえば、ボタンを押したときに変数を自動的に増やしたい場合：

```swift
var num = 10
num += 5   // num = 15
```

このコードの意味は、`num` と 5 を加算し、その結果を再び `num` に代入するということです。

したがって、これは次と同じ意味です：

```swift
var num = 10
num = num + 5   // num = 15
```

## 三項演算子

Swift の三項演算子（Ternary Operator）は、条件に応じて 2 つの結果のうち 1 つを返します：

```swift
条件 ? 値1 : 値2
```

その実行ロジックは、条件が `true` なら「値1」を返し、条件が `false` なら「値2」を返す、というものです。

たとえば：

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

このコードは `showColor` の値を判定します。もし `true` なら `Color.blue` を返して背景は青色になり、もし `false` なら `Color.clear` を返して背景色は表示されません。

ヒント：`Color.clear` は透明色（内容を表示しない）を表します。

### if-else との違い

三項演算子は、ロジック上 `if-else` 文と等価です。

たとえば、上のコードは次のようにも書けます：

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

実行ロジックは完全に一致していて、条件が `true` のときは最初の結果、`false` のときは 2 番目の結果が実行されます。

実用上、三項演算子はより簡潔で、「二者択一」のシンプルな判定を表現するのに向いており、値を直接返す場面でよく使われます。

SwiftUI の View 構築でも、三項演算子は非常によく使われます。たとえば：

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

この書き方なら、1 行のコードで条件に応じてビュー効果を動的に変更できます。

## 範囲演算子

範囲演算子（Range Operators）は、区間を表すために使われます：

- `...` 閉区間（両端を含む）
- `..<` 半開区間（右端を含まない）

たとえば：

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

これは、`1...3` が 1 から 3 まで（3 を含む）、`5..<8` が 5 から 8 まで（8 は含まない）ことを意味します。

拡張形式（参考程度）：

```
5...    // >= 5
...5    // <= 5
```

これは「片側範囲」を表し、条件判定やパターンマッチングでよく使われます。

範囲演算子は今後 `for` ループを学ぶときに使います。現時点では、連続した数値範囲を表すものとして理解しておけば十分です。

## 演算子を練習する

ここでは、いくつかの簡単な例を通して、ビュー内で演算子の役割を練習してみます。

### 1. 車両総数を計算する

たとえば、ある交差点を毎日 500 台の車が通るとして、30 日間の通行車両総数を計算してみましょう。

計算演算子を使えば：

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

ここでの計算過程は、1 日 500 台、30 日間で合計 `500 × 30` となり、最終的に 15000 台になるということです。

### 2. アプリ内課金会員かどうかを判定する

ある人がアプリ内課金会員かどうかは、永久会員またはサブスクリプション会員のどちらか一方でも有効であれば、会員とみなすことができます。

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

このコードでは `||`（OR）演算子を使っています。`lifeTime = false` は永久会員ではないことを意味し、`subscription = true` はサブスクリプション会員であることを意味します。

どちらか一方の条件を満たせばよいので、`isMember` は `true` になります。したがって、この人はアプリ内課金会員です。

## まとめ

Swift において、演算子はデータ計算と論理判定の基礎です。`Text` が内容を表示するためのものだとすれば、演算子はデータそのものを処理するためのものです。

単純な数値計算から、複雑な論理判定、さらにビュー内の状態制御やレイアウト計算に至るまで、どれも演算子抜きには成り立ちません。

さまざまな場面で練習を重ねることで、このレッスンで紹介した各種演算子に徐々に慣れ、使いこなせるようになります。これは今後の学習にとって重要な基礎です。

### 拡張知識 - ビット演算子

さらに、Swift にはもっと低レベルな演算子もあります。それがビット演算子（Bitwise Operators）です：

- `&`（ビット AND）
- `|`（ビット OR）
- `^`（ビット XOR）
- `~`（ビット NOT）
- `>>`（右シフト）
- `<<`（左シフト）

たとえば：

```swift
let a = 6  // 2進数 110
let b = 3  // 2進数 011

print(a & b) // 2 (2進数 010)
print(a | b) // 7 (2進数 111)
print(a ^ b) // 5 (2進数 101)
print(~a)    // -7 (ビット反転)
```

Swift では整数は 2 の補数形式で表現されるため、ビット反転すると対応する負数結果が得られます。

ビット演算子は直接 2 進数に対して操作を行い、通常は低レベルなデータ処理やパフォーマンス最適化の場面で使われます。

初心者にとっては、こうした演算子の使用場面は比較的少なく、一般的な iOS や SwiftUI 開発でもあまり見かけません。そのため、ここでは詳しく掘り下げません。

今後、低レベルロジックやパフォーマンス最適化に関わるようになったら、さらに深く学んで理解していけば十分です。
