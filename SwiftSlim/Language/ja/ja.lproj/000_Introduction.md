# はじめに

## Swift とは？

Swift を学ぶ前に、まず Swift というプログラミング言語について簡単に理解しておきましょう。Swift は Apple が開発した現代的なプログラミング言語で、主に iOS、macOS、iPadOS、watchOS など Apple プラットフォーム向けのアプリケーションを構築するために使われます。

![Swift](../../RESOURCE/000_swift.png)

Swift は他のプログラミング言語と比べて文法が簡潔で、安全性と可読性をより重視しています。SwiftUI と組み合わせることで、さらに学びやすく理解しやすくなります。もちろん、私はそれ以前に他のプログラミング言語を学んだことがなかったので、HTML よりは少し難しいかもしれません。

Swift は Apple プラットフォーム向けアプリの開発だけに使えるわけではありません。Swift コミュニティは Swift を [Android](https://www.swift.org/blog/nightly-swift-sdk-for-android/) プラットフォームへ移植しようと取り組んでおり、さらに [サーバー](https://github.com/awslabs/swift-aws-lambda-runtime) など他の環境でも動作させようとしています。長期的な発展という観点から見ると、Swift にはより大きな成長余地があり、将来的には Flutter や React Native のようなクロスプラットフォーム言語になる可能性もあります。

## 学習内容

このチュートリアルは、Swift と SwiftUI の基礎知識を中心に進めます。

まずは簡単な変数とデータ型から学び、SwiftUI を使って UI を構築し、データの保存と読み取りの方法を理解し、最後には SwiftData や iCloud 同期などの発展的な内容にも触れるかもしれません。

各章は実際のケースを中心に展開し、実際の利用場面の中で Swift を学んでいきます。

チュートリアルによっては、最初に大量の文法や API をまとめて説明してから、プロジェクト実践に入るものもあります。しかし、初心者にとってそれは、子どもが辞書だけで文字を覚えるようなものです。使う場面がないまま大量の内容を暗記しても、最終的には学習意欲を失ってしまうだけです。

このチュートリアルでは、実際の開発シーンの中で知識を学び、必要性から関連する知識ポイントを引き出していきます。

このチュートリアルを終えるころには、基本的なアプリを自力で開発して動かし、自分の iOS / macOS デバイスにインストールできるようになり、さらに App Store に公開することも可能になります。

## 初心者はどう学べばよい？

私も最初はまったくの初心者でした。最初は YouTube にあるいくつかの [Swift 学習動画](https://www.youtube.com/watch?v=4SaddkGm5yA&list=PLliocbKHJNwuLHJlVv6q5tKJxJltjBspC&index=3) を見ただけでしたが、iOS アプリ開発がそれほど複雑ではないと感じ、興味を持つようになりました。その後、さらに多くの Swift の知識を学ぼうとし始めました。たとえば、[Swift Playgrounds](https://apps.apple.com/cn/app/swift-playgrounds/id1496833156?mt=12) のプログラミングゲームを体験したり、[Swift 公式チュートリアル文書](https://docs.swift.org/swift-book/documentation/the-swift-programming-language/) を読んだり、[100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui) を学んだりしました。

今振り返ると、主に [100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui) が学習のフレームワークを提供してくれたおかげで、体系的に SwiftUI を学べるようになったと思います。ただし、チュートリアルが英語であること、そして自分にプログラミングの基礎がなかったこともあり、理解の過程では非常に多くの困難がありました。そのため、[ChatGPT](https://chatgpt.com/) や [Claude](https://claude.ai/?redirect=claude.com) のような AI を使って知識点の理解を補助していました。

学習の過程では、どうしても理解が難しい知識が出てきます。たとえば UserDefaults を学んでいたとき、私はデータの保存方法しか知らず、データを手動で読み取る必要があることを知らなかったため、自分のコードに問題があるのではないかと疑っていました。このような単純な知識ポイントでさえ、私は数日を無駄にしてしまいました。

また、クロージャ、メインスレッド、ジェネリクスなどもそうでした。学習初期には深く理解できなかったため、実際の開発で再び出会ったときに、改めて時間と労力をかけて学び直す必要がありました。

そのため、私のアドバイスは次の通りです。

**1. 基礎知識を補う**

中国には「万事はじめ難し」という言葉があります。プログラミング未経験から Swift を学ぶ初心者にとって、学習の過程で理解しにくい知識に出会うのは避けられません。しかもそれは、多くの場合その知識そのものが難しいからではなく、その知識の土台にあるさらに基礎的な概念に、まだ触れていないからです。

このような場合は、[ChatGPT](https://chatgpt.com/) や [Claude](https://claude.ai/?redirect=claude.com) などの AI ツールを活用して、不足している基礎知識をタイムリーに補うことをおすすめします。

**2. すべての知識点を理解する**

学習の過程で最も大切なのは、時間が数日かかっても、できるだけ一つひとつの知識点を理解しようとすることです。

理解できない内容を飛ばしてしまうと、その後の開発で高い確率で再び出会い、進捗の妨げになります。結局はまた時間をかけて学び直すことになるのです。後から補うよりも、学習段階でしっかり理解しておく方がよいのです。

あらゆる知識点は本質的に一つの道具です。身につけた道具が多いほど、問題を解決する力も強くなります。

これは家を建てることに似ています。手元に道具が一つしかなければ、なんとか大部分の作業はできても、より高い品質を求められたときには力不足になります。より多くの道具を身につけてこそ、より堅牢な家を建てられるのです。

**3. 継続して学び続ける**

学習には長期的な継続が必要です。数か月という時間を本当に真剣に一つのチュートリアル学習に使える人は、実はそれほど多くありません。短期間に高強度で学ぶことよりも、継続して時間を投資することの方が重要です。

たとえ毎日 1% しか進めなくても、100 日続ければ最終的に学習目標全体を達成できます。

## 教育的な願い

このコースでは、Swift や SwiftUI の知識をあまり多く教え込むのではなく、初心者を Swift というプログラミングの道へ導くことを目指しています。積み木を組み立てるのと同じように、チュートリアルは組み方を説明し、残りの時間はあなた自身が自分の積み木を組み上げていく時間になります。

このプロジェクトはすでに [GitHub](https://github.com/fangjunyu1/SwiftSlim) でオープンソース化されており、無料でダウンロードして利用できます。また、体験用の App Store 版も提供しています。

私たちは、技術的な背景がなくてもキャリアを変えたいと願うより多くの人が、プログラミングやアプリ開発を学ぶことで新たな可能性を得られるよう支援したいと考えています。
