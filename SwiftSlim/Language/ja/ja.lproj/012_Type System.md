# 型システム

これまでのレッスンで、私たちはすでに変数を学び、変数にはさまざまな型のデータを保存できることを知りました。

たとえば：

```swift
var hello = "Hello, World"
```

ここでの `hello` は 1 つのテキストを保存しているため、その型は `String` です。

次に、型（Type）の概念、明示的な型宣言、型推論を体系的に学び、変数の本質をより深く理解していきます。

## 型とは何か

Swift では、あらゆる値には明確な型があります。型は、その値が何であり、何ができるかを決定します。

たとえば：

- 42 は `Int`（整数）
- `"Hello"` は `String`（文字列）
- `true` は `Bool`（真偽値）

これらは異なる型に属しており、用途も異なります。

## Swift でよく使う基本型

初心者段階で最もよく使う型は次の通りです：

- `String`：文字列（テキスト）
- `Int`：整数
- `Double`：浮動小数点数（小数あり、精度高め）
- `Bool`：真偽値（`true/false`）

そのほかに `Float`（浮動小数点数）もありますが、精度は `Double` より低いため一般的にはあまり使いません。また、`Character`（単一文字）もあり、たとえば `"A"`、`"!"` のような 1 文字を表します。

### 型宣言（明示的な型）

Swift では、変数宣言時に手動で型を指定できます：

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

変数名の後ろに `: 型` を付けて、その変数の型を宣言します。

配列の型の書き方は `: [型]` です：

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

このコードでは、`scores` 配列には `Int` 型の要素しか保存できず、`tags` には `String` 型の要素しか保存できません。

明示的に型を宣言すると、コードの意図がより明確になり、場合によっては型推論の誤りも防げます。

## 型推論

多くの場合、明示的に型を宣言しなくてもかまいません：

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"    // String
var isLogin = false  // Bool
var scores= [90, 85, 78]	// [Int]
var tags = ["Swift", "iOS"]	// [String]
```

これは、Swift コンパイラが「初期値」に基づいて自動的に型を推論するからです。

## 明示的宣言と型推論

前のレッスンでは、たとえば次のような明示的宣言：

```swift
var age: Int = 18
```

は導入せず、代わりに型推論を優先しました：

```swift
var age = 18
```

この例では、どちらの書き方も同じ意味で、`age` はどちらも `Int` 型と確定されます。それに比べて、型推論の書き方の方がより簡潔です。

初期段階で明示的宣言をあまり強調しなかったのは、型推論によってコード中の余計な情報を減らし、初心者が理解する負担を軽くできるからです。

## なぜ型が必要なのか

Swift は強い型付けを持つ言語（`Strongly Typed Language`）です。

つまり、一度変数の型が決まると、勝手に別の型へ変えることはできません。

たとえば：

```swift
var age: Int = 18

age = 20	// Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

最初の `20` は `Int` 型なので `age` に代入できます。2 つ目の `"20"` は `String` 型で、`Int` 型と一致しないためエラーになります。

`nums` は `[Int]` 型の配列であり、整数しか保存できず、文字列を混在させることはできません。

型はデータの使い方を制約し、コンパイル段階で型不一致の問題を発見できるため、型の混乱によるバグを減らし、コードの安定性と保守性を高めます。

## どんなときに明示的な型宣言が必要か

Swift は自動的に型を推論できますが、コンパイラが推論できない場合や、推論を誤る場合もあります。そのときは手動で宣言する必要があります。

**1. 空配列**

空配列を作るときは、必ず明示的に型を宣言しなければなりません：

```swift
var nums: [Int] = []
```

型を書かないと：

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

コンパイラは空配列から要素型を推論できないため、コンパイルエラーになります。

**2. 推論ミスを避ける**

```swift
var value = 10   // Int
```

この例では、10 は `Double`（小数）として解釈することもできますが、小数点がないため、コンパイラはデフォルトで `Int` と推論します。

もし `value` を浮動小数点数として扱いたいなら、明示的に型を宣言する必要があります：

```swift
var value: Double = 10   // Double
```

型推論の結果が期待と異なるときは、明示的な型宣言またはリテラル形式の調整によって、正しい型を保証するべきです。

**3. カスタム型**

今後はカスタム型も学んでいきます。カスタム型を使う場合も、通常は明示的な宣言が必要です：

```swift
var info: GameInfo
var users: [User] = []
```

これも空配列と同じ理由で、コンパイラが空の値から型を自動推論できないからです。

## 型変換

Swift では、異なる型の間は自動変換されず、必ず手動で変換する必要があります。

たとえば、異なる型の数字を足し算する場合：

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b	// Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

このコードでは、`a` は `Int` 型、`b` は `Double` 型であり、型が異なるため、そのままでは演算できません。

演算する前に、どちらか一方を同じ型へ変換する必要があります：

```swift
var result = Double(a) + b
```

このコードは、`Int` 型の `a` を `Double` 型へ変換し、その後 `b` と足し算し、その計算結果を `result` に代入していることを意味します。

注意すべきなのは、型変換は元の変数の型を変更しないということです。たとえば `Double(a)` は `a` の `Int` 型を変えるのではなく、演算に使うための新しい `Double` 値を作るだけです。

他の型も同様の方法で変換できます。つまり、目標型 + `()` の形式で変換したい値を渡します。たとえば：

```
Int()、Double()、String()
```

例：

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

注意すべきなのは、すべての型変換が完全に成功するわけではないということです。たとえば `Double → Int` の場合、小数部分は単純に切り捨てられ、四捨五入はされません。

## 型エイリアス

Swift では、既存の型に別名を付けて、コードの意味をより明確にすることもできます：

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

ここでの `UserId` は本質的には依然として `Int` であり、型自体は変わっていません。ただ名前がより意味のあるものになっただけです。

コード量が多く型が複雑なプロジェクトでは、型エイリアスによってコードをより読みやすくできます。

## 型の例

次に、シンプルな例を通して、実際の開発において型がどのような役割を果たすかを示します。

### 変数を作る

まず、`age` という変数を作ります：

```swift
var age = 20
```

この変数は SwiftUI 上で表示し、さらに変更もしたいので、`@State` を使って宣言する必要があります：

```swift
@State private var age = 20
```

`private` については以前少し触れましたが、この変数は現在のビューの中でしか使えないことを意味します。今後さらに学びます。

### SwiftUI でデータを表示する

SwiftUI の中で `age` 変数を表示してみます：

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

今、ビューには次のように表示されます：

```swift
age: 20
```

### データを変更する

続いて、`age` を変更するために 2 つのボタンを追加します：

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### 実行結果

`changeAge1` を押すと、`age` に 18 が代入されます：

```swift
age = 18
```

18 は `Int` 型であり、`age` の型と一致するため、代入は成功します。

一方、`changeAge2` はエラーになります：

```swift
age = "18"
```

エラーメッセージ：

```
Cannot assign value of type 'String' to type 'Int'
```

これは、`"18"` が `String` 型（文字列は `""` で囲む必要があります）であり、`age` が `Int` 型なので、型が一致せず代入できないためです。

### なぜエラーになるのか？

ポイントは次のコードです：

```swift
@State private var age = 20   // Int
```

明示的に型を宣言していなくても、初期値 20 が `Int` 型であるため、コンパイラは `age` を自動的に `Int` 型と推論します。

一度型が決まると、別の型を代入することはできません。

しかも `changeAge2` ボタンは「実行時に失敗する」のではなく、コンパイル段階でエラーとなり、コードはコンパイルを通過できません。

これも Swift の強い型付けの利点の一つです：コードを書いている段階で、型の問題を先に発見できます。

### 正しい書き方

誤ったコードを削除すると：

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

`changeAge1` ボタンを押すと、ビューは次のように更新されます：

```
age: 18
```

### 明示的な型宣言（任意）

定義時に、明確に型を指定することもできます：

```swift
@State private var age: Int = 20
```

この書き方は型をより明確にできますが、単純な場面では必須ではありません。

## まとめ

このレッスンでは主に、Swift の型システムとその実際の開発における役割を学びました。

Swift では、すべての値に明確な型があり、型はデータの意味や参加できる操作を決定します。

Swift は強い型付けを持っているため、変数の型が一度決まると、別の型を代入することはできません。この制約によって、コンパイル段階でエラーを発見でき、実行時の問題を減らし、コードの安全性を高められます。

変数の型は初期値から自動推論することもでき、明示的に宣言することもできます。特定の場面（空配列、型推論ミス、カスタム型など）では、コンパイラが正しく型を推論できないため、明示的な宣言が必要になります。

異なる型の間では自動変換は行われず、`Int()`、`Double()`、`String()` などを使って明示的に変換しなければなりません。

型は別名（エイリアス）にも対応しています。特に複雑なプロジェクトでは、型エイリアスが可読性を高めます。

型は単なるデータラベルではなく、安全かつ明確にデータを使うための制約機構でもあります。

## 拡張知識 - Optional 型

型変換を行うとき、失敗する可能性のある操作があります。

たとえば、`String` を `Double` に変換する場合：

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

ここで `Double(input)` が返すのは `Double` ではなく：

```
Double?
```

これが Optional 型（`Optional`）です。

### なぜ Optional 型が返るのか？

文字列の内容は不確定だからです。`"3.14"` は `Double` に変換できますが、`"Hello"` は `Double` に変換できません。

つまり、この操作は成功することもあれば失敗することもあります。

そのため Swift は、この不確実性を表すために Optional 型を使います。

### Optional 型とは何か？

Optional 型とは、「値が存在するかもしれないし、存在しないかもしれない」ことを表します。

たとえば：

```
Double?
```

この型は、値があれば `Optional(...)` と表示され、値がなければ `nil` を返すことを意味します。

注意すべきなのは、`nil` は「値がない」ことを意味するのであって、0 や空文字列ではないということです。

### 例

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

出力：

```
pi:Optional(3.14)
```

これは、文字列が `Double` に正常変換されて 3.14 になったものの、型が `Double?` なので `Optional` で包まれていることを意味します。

もし変換に失敗すると：

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

出力：

```
pi: nil
```

`"Hello"` は `Double` 型に変換できないため、変換に失敗し、`nil` が返ります。

### Optional(...) とは何か？

Optional 型をそのまま出力すると：

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift はその「デバッグ記述」を表示します：

```
Optional(3.14)
```

これは実際のデータの一部ではなく、現在の値が「Optional 型」であり、その中に値が含まれていることを示しているだけです。

### Optional をアンラップする

実際の開発では、通常 Optional 型の中から本当の値を取り出す必要があります。この過程をアンラップ（Unwrapping）と呼びます。

よく使う方法の一つは、`??` を使ってデフォルト値を与える方法です：

```
??
```

これは Nil 合体演算子（`Nil-Coalescing Operator`）と呼ばれます。

たとえば：

```swift
name ?? "FangJunyu"
```

このコードは、もし `name` に値があればその値を返し、`nil` であればデフォルト値 `"FangJunyu"` を使う、という意味です。

注意すべき点は、デフォルト値の型は Optional 型と一致していなければならないことです。

### ?? を使ってデフォルト値を与える

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")	// 30
```

出力：

```
pi: 30
```

`input` が `Double` 型に変換されると、変換に成功した場合はその値が出力されます。もし失敗した場合は、`??` が与えたデフォルト値が出力されます。

もし変換に失敗すると：

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

出力：

```
pi: 10
```

ここでは、`input` を `Double` に変換しようとして失敗し、`nil` が返されます。そのとき `??` が `nil` に対してデフォルト値を提供します。

Optional 型は、「値があるかもしれないし、ないかもしれない」状況を表すために使われます。

型変換のような失敗する可能性のある操作では、Swift はプログラムの安全性を保つために Optional 型を返します。

中の値を使いたい場合は、`??` などを使ってデフォルト値を与えることで、`nil` が現れても妥当な結果を得られるようにできます。
