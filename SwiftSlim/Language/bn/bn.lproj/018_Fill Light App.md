# ফিল লাইট অ্যাপ

এই পাঠে আমরা একটি খুবই মজার ফিল লাইট অ্যাপ তৈরি করব। রাত হলে আমরা ফোনের স্ক্রিনে বিভিন্ন রঙ দেখাতে পারি এবং এটিকে একটি সাধারণ ফিল লাইট হিসেবে ব্যবহার করতে পারি।

এই ফিল লাইট অ্যাপ স্ক্রিনে ট্যাপ করে রঙ বদলাতে পারে, এবং স্লাইডার ব্যবহার করে উজ্জ্বলতাও নিয়ন্ত্রণ করতে পারে।

এই উদাহরণে আমরা `brightness` দিয়ে ভিউয়ের উজ্জ্বলতা সমন্বয় করা, `onTapGesture` দিয়ে ভিউতে ট্যাপ জেসচার যোগ করা, এবং `Slider` কন্ট্রোল সম্পর্কে শিখব।

## রঙ দেখানো

প্রথমে, আমরা ভিউতে একটি রঙ দেখাই।

SwiftUI-তে `Color` শুধু একটি রঙ নির্দেশ করে না, এটি একটি ভিউ হিসেবেও প্রদর্শিত হতে পারে:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

এখানে `Color.red` একটি লাল ভিউ নির্দেশ করে। `.ignoresSafeArea()` রঙের ভিউকে পুরো স্ক্রিন জুড়ে ছড়িয়ে দেয়, তাই এটি বাস্তব ফিল লাইটের মতো দেখায়।

### রঙের অ্যারে এবং ইনডেক্স

এখন শুধু একটি রঙ দেখানো হচ্ছে। কিন্তু ফিল লাইট সাধারণত শুধু এক রঙের হয় না। এটি নীল, হলুদ, বেগুনি, সাদা ইত্যাদি রঙও দেখাতে পারে।

আমরা চাই স্ক্রিনে ট্যাপ করলে বিভিন্ন রঙের মধ্যে বদল হোক। এই রঙগুলোকে আমরা একটি অ্যারেতে রেখে একসাথে ম্যানেজ করতে পারি:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

একটি অ্যারে "একই ধরনের একগুচ্ছ ডেটা" সংরক্ষণের জন্য উপযুক্ত। এখানে অ্যারের প্রতিটি উপাদানই একটি `Color`।

যদি আমরা একটি নির্দিষ্ট রঙ দেখাতে চাই, তাহলে ইনডেক্স ব্যবহার করতে পারি:

```swift
colors[0]
```

এর মানে হলো অ্যারের `0` ইনডেক্সের রঙটি নেওয়া, অর্থাৎ প্রথম রঙটি।

এখন কোডটি এভাবে লেখা যেতে পারে:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

এভাবে স্ক্রিনে অ্যারের প্রথম রঙটি দেখাবে, অর্থাৎ লাল।

### ইনডেক্স দিয়ে রঙ নিয়ন্ত্রণ

যদি আমাদের বিভিন্ন রঙের মধ্যে বদল করতে হয়, তাহলে ইনডেক্সটি স্থির করে না লিখে, সেটি পরিচালনা করার জন্য একটি ভ্যারিয়েবল দরকার।

ইনডেক্স সংরক্ষণের জন্য আমরা `@State` দিয়ে একটি ভ্যারিয়েবল ঘোষণা করতে পারি:

```swift
@State private var index = 0
```

এখানে `index` বর্তমান রঙের ইনডেক্স নির্দেশ করে।

যখন `index` পরিবর্তন হবে, SwiftUI ইন্টারফেস আবার গণনা করবে এবং প্রদর্শিত কন্টেন্ট আপডেট করবে।

তারপর আগের `colors[0]` পরিবর্তন করে লিখুন:

```swift
colors[index]
```

এভাবে ভিউতে কোন রঙ দেখানো হবে, তা `index` দ্বারা নির্ধারিত হবে।

এখন কোডটি হয়ে যায়:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

যখন `index` বদলাবে, `colors[index]`-ও অন্য রঙ দেখাবে।

উদাহরণ হিসেবে:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

একটি বিষয় খেয়াল রাখতে হবে, `index` অ্যারের সর্বোচ্চ ইনডেক্সের বেশি হতে পারবে না, নইলে out-of-range ত্রুটি হবে।

## ট্যাপ জেসচার

এখন আমরা `index` অনুযায়ী বিভিন্ন রঙ দেখাতে পারি, কিন্তু এখনও ট্যাপ করে রঙ বদলানো যাচ্ছে না।

আগের "Quote Carousel" পাঠে আমরা উদ্ধৃতি বদলানোর জন্য `Button` ব্যবহার করেছি।

কিন্তু এবার আমরা চাই "পুরো রঙের অংশে ট্যাপ" করে রঙ বদলাতে, তাই `onTapGesture` এখানে বেশি উপযুক্ত।

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

যখন আমরা রঙের ভিউতে ট্যাপ করি, তখন এই কোডটি চলবে:

```swift
index += 1
```

এর মানে `index`-এর মান `1` করে বাড়ানো। ইনডেক্স বাড়লে `colors[index]` অ্যারের পরের রঙটি দেখাবে।

### `onTapGesture`

`onTapGesture` হলো একটি জেসচার মডিফায়ার, যা কোনো ভিউতে ট্যাপ করার কাজ যোগ করে।

মৌলিক ব্যবহার:

```swift
.onTapGesture {
    // code
}
```

উদাহরণ:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

এই লাল ভিউতে ট্যাপ করলে, ব্রেসের ভিতরের কোড চলবে এবং কনসোলে এটি আউটপুট হবে:

```swift
Click color
```

`onTapGesture` ব্যবহার করে আমরা নির্ধারণ করতে পারি, কোনো ভিউতে ট্যাপ করার পর কী হবে।

### `Button` থেকে পার্থক্য

এর আগে আমরা `Button` ভিউ শিখেছি। `Button` এবং `onTapGesture` দুটোই ট্যাপ অ্যাকশন সামলাতে পারে, কিন্তু এদের ব্যবহারের পরিস্থিতি পুরোপুরি এক নয়।

`onTapGesture` বেশি উপযুক্ত "আগে থেকে থাকা কোনো ভিউতে ট্যাপ করার ফাংশন যোগ করার" জন্য, যেমন `Color`, `Image`, `Text`, বা অন্য কোনো সাধারণ ভিউ।

অন্যদিকে `Button` বেশি উপযুক্ত যখন আমরা "Confirm", "Submit", বা "Delete"-এর মতো স্পষ্ট একটি বোতাম বোঝাতে চাই।

এই ফিল লাইট অ্যাপে আমরা চাই রঙ বদলের ফাংশনটি আরও সহজ হোক। পুরো রঙের অংশে ট্যাপ করলেই রঙ বদলে যাবে, তাই এখানে `onTapGesture` ব্যবহার করা ভালো।

## ইনডেক্স সমস্যা

এখন আমরা স্ক্রিনে ট্যাপ করে বিভিন্ন রঙ বদলাতে পারি।

কিন্তু এখানে একটি গুরুত্বপূর্ণ সমস্যা আছে: **ইনডেক্স অ্যারের সীমার বাইরে চলে যেতে পারে**।

উদাহরণ:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

যদি আমরা বারবার স্ক্রিনে ট্যাপ করি, তাহলে `index` শেষ পর্যন্ত `4` হয়ে যাবে, আর তখন "index out of range" ত্রুটি হবে।

এর কারণ হলো `colors` অ্যারেতে `4`টি উপাদান আছে, কিন্তু ইনডেক্স `0` থেকে শুরু হয়, তাই বৈধ ইনডেক্সের পরিসর `0 - 3`, `4` নয়।

যদি আমরা `colors[4]` ব্যবহার করি, তাহলে "index out of range" ত্রুটি হবে।

বর্তমান কোডে, প্রতিবার ট্যাপ করলে `index`-এ `1` যোগ হয়। যদি আমরা এটি সামলাই না, তাহলে এটি শেষ পর্যন্ত বৈধ সীমার বাইরে চলে যাবে।

তাই স্ক্রিনে ট্যাপ করার সময় আমাদের ইনডেক্স পরীক্ষা করতে হবে: যদি এটি শেষ রঙে পৌঁছে যায়, তাহলে আবার প্রথম রঙে ফিরে যেতে হবে; না হলে `1` যোগ করতে হবে।

এটি `if` স্টেটমেন্ট দিয়ে করা যায়:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

এই কোডে `colors.count` মানে অ্যারেতে মোট উপাদানের সংখ্যা।

বর্তমান অ্যারেতে 4টি রঙ আছে, তাই:

```swift
colors.count // 4
```

কিন্তু সর্বোচ্চ ইনডেক্স `4` নয়, `3`, কারণ ইনডেক্স `0` থেকে শুরু হয়।

তাই শেষ ইনডেক্সটি এভাবে লিখতে হবে:

```swift
colors.count - 1
```

অর্থাৎ:

```swift
4 - 1 = 3
```

এই লজিকের মানে হলো: যদি বর্তমান ইনডেক্স ইতিমধ্যে শেষ রঙের হয়, তাহলে ইনডেক্সকে `0` করুন; না হলে `1` বাড়ান।

এভাবে রঙগুলো চক্রাকারে বদলানো যাবে।

### ইনডেক্স লজিককে আরও সংক্ষিপ্ত করা

যদি আমরা কোড আরও সংক্ষিপ্ত করতে চাই, তাহলে ternary operator-ও ব্যবহার করতে পারি:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

এই কোডের মানে হলো: যদি `index == colors.count - 1` সত্য হয়, তাহলে `0` ফেরত দাও। না হলে `index + 1` ফেরত দাও।

সবশেষে, ফলাফলটি আবার `index`-এ অ্যাসাইন করা হয়।

এখন আমরা রঙ বদলানোর ইফেক্টটি বাস্তবায়ন করতে পারি।

সম্পূর্ণ কোড:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

এখন আমরা স্ক্রিনে ট্যাপ করে বিভিন্ন রঙের মধ্যে বদলাতে পারি। একটি সাধারণ ফিল লাইট অ্যাপ তৈরি হয়ে গেছে।

## রঙের নাম দেখানো

আমরা রঙগুলোর সঙ্গে মিলিয়ে আরেকটি টেক্সটের সেট যোগ করতে পারি, যাতে রঙ বদলালে স্ক্রিনে বর্তমান রঙের নামও একসাথে দেখা যায়।

উদাহরণ:

- লাল হলে `Red` দেখাবে
- নীল হলে `Blue` দেখাবে
- হলুদ হলে `Yellow` দেখাবে
- বেগুনি হলে `Purple` দেখাবে

এখানেও আমরা রঙের নামগুলো সংরক্ষণের জন্য একটি অ্যারে ব্যবহার করতে পারি:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

এই অ্যারের টেক্সটগুলোর ক্রম অবশ্যই রঙের অ্যারের রঙগুলোর ক্রমের সঙ্গে পুরোপুরি মিলতে হবে।

তারপর আমরা বর্তমান ইনডেক্সের রঙের নাম দেখাতে `Text` ব্যবহার করতে পারি:

```swift
Text(colorsName[index])
```

`Text` `index` অনুযায়ী বর্তমান রঙের নাম দেখাবে।

`Text`-এর চেহারা উন্নত করতে modifier ব্যবহার করুন:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

এতে `Text` সাদা রঙে, বড় শিরোনাম সাইজে, এবং bold font weight-এ দেখাবে।

এখন আমাদের একটি full-screen `Color` ভিউ আছে। যদি আমরা চাই `Text` `Color` ভিউয়ের উপর দেখাক, তাহলে দুটিকে একের উপর এক বসাতে `ZStack` layout container ব্যবহার করতে হবে।

```swift
ZStack {
    Color
    Text
}
```

তাই কোডটি হয়ে যায়:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

এভাবে ব্যাকগ্রাউন্ডে থাকবে রঙের ভিউ, আর তার উপর স্তর হিসেবে থাকবে একটি টেক্সট ভিউ।

ফলাফল:

![Color](../../Resource/018_color2.png)

একটি বিষয় খেয়াল রাখতে হবে, `ZStack`-এ পরে লেখা ভিউ সাধারণত সামনে দেখা যায়। যদি `Text`-কে `Color`-এর আগে লেখা হয়, তাহলে পরে লেখা `Color` ভিউ `Text`-কে ঢেকে দিতে পারে।

## উজ্জ্বলতা নিয়ন্ত্রণ

এখন আমরা বিভিন্ন রঙ বদলাতে পারি, কিন্তু ফিল লাইটের আরেকটি গুরুত্বপূর্ণ ফিচার আছে: **উজ্জ্বলতা সমন্বয়**।

SwiftUI-তে আমরা `brightness` modifier ব্যবহার করে একটি ভিউয়ের উজ্জ্বলতা সমন্বয় করতে পারি।

উদাহরণ:

```swift
.brightness(1)
```

আমরা লিখতে পারি:

```swift
colors[index]
    .brightness(0.5)
```

এতে বর্তমান রঙটি আরও উজ্জ্বল হয় এবং ফিল লাইটের প্রভাবের আরও কাছাকাছি লাগে।

উজ্জ্বলতার পরিসর `0 - 1`। `0` মানে আসল রঙ অপরিবর্তিত রাখা, আর মান যত `1`-এর কাছে যায়, রঙ তত উজ্জ্বল হয়। `1` মানে সবচেয়ে উজ্জ্বল সাদা ফলাফল।

যদিও আমরা কোডে `brightness` নিয়ন্ত্রণ করতে পারি, ব্যবহারকারী এখনও এটি সরাসরি নিয়ন্ত্রণ করতে পারে না।

তাই আমাদের একটি টেনে সরানো যায় এমন কন্ট্রোল যোগ করতে হবে: `Slider`।

## `Slider` ভিউ

SwiftUI-তে `Slider` হলো একটি কন্ট্রোল যা একটি নির্দিষ্ট পরিসরের মধ্যে একটি মান বেছে নিতে ব্যবহৃত হয়। Apple একে বলে "একটি bounded linear range থেকে মান নির্বাচন করার কন্ট্রোল"।

মৌলিক ব্যবহার:

```swift
Slider(value: $value, in: 0...1)
```

প্যারামিটার ব্যাখ্যা:

1. `value: $value`: `Slider`-কে একটি ভ্যারিয়েবলের সঙ্গে bind করতে হয়।

    স্লাইডার টানা হলে ভ্যারিয়েবলের মানও একই সঙ্গে বদলায়। আবার ভ্যারিয়েবলের মান বদলালে স্লাইডারও আপডেট হয়।

    এটি আগের শেখা `TextField`-এর মতোই। দুটোই "কন্ট্রোলকে ভ্যারিয়েবলের সঙ্গে bind করে"।

    binding বোঝাতে bind করা ভ্যারিয়েবলের আগে `$` চিহ্ন দিতে হয়।

2. `in: 0...1`: এই প্যারামিটারটি স্লাইডারের মানের পরিসর বোঝায়।

    এখানে `0...1` মানে সর্বনিম্ন মান `0` এবং সর্বোচ্চ মান `1`।

    স্লাইডার একেবারে বামে গেলে bind করা ভ্যারিয়েবলের মান `0`-এর কাছাকাছি হবে; একেবারে ডানে গেলে `1`-এর কাছাকাছি হবে।

উদাহরণ:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` `value` ভ্যারিয়েবলের সঙ্গে bind করা আছে। স্লাইডার টানলে `value` ভ্যারিয়েবলের মানও একই সঙ্গে বদলায়।

চেহারা:

![Slider](../../Resource/018_slider.png)

`Slider` বাম দিকে সরালে bind করা `value` হয়ে যায় `0`। ডান দিকে সরালে হয়ে যায় `1`।

### মানের পরিসর

`Slider`-এর মানের পরিসর স্থির নয়। এটিকে এভাবেও লেখা যায়:

```swift
0...100
```

অথবা অন্য কোনো range।

কিন্তু এই ফিল লাইট অ্যাপে আমাদের উজ্জ্বলতা নিয়ন্ত্রণ করতে হবে, তাই `0...1`-ই সবচেয়ে উপযুক্ত।

## `Slider` দিয়ে উজ্জ্বলতা নিয়ন্ত্রণ

এখন আমাদের `Slider`-কে `brightness`-এর সঙ্গে যুক্ত করতে হবে।

প্রথমে, উজ্জ্বলতার মান সংরক্ষণের জন্য একটি ভ্যারিয়েবল তৈরি করুন:

```swift
@State private var slider = 0.0
```

এখানে `0.0` হলো একটি `Double` মান।

কারণ `Slider` সাধারণত একটি numeric type-এর সঙ্গে bind হয়, আর এখানে আমরা চাই এটি ধারাবাহিকভাবে পরিবর্তিত হোক, তাই `Double` বেশি উপযুক্ত। এছাড়া `brightness` শুধুমাত্র `Double` মান নিতে পারে।

তারপর এই মানটি `brightness`-এ পাঠান:

```swift
colors[index]
    .brightness(slider)
```

যখন `slider == 0`, রঙটি তার default অবস্থায় থাকে। `slider` যত `1`-এর কাছে যায়, রঙটি তত উজ্জ্বল দেখায়।

### `Slider` কন্ট্রোল যোগ করা

এরপর, এই ভ্যারিয়েবল পরিবর্তনের জন্য একটি `Slider` কন্ট্রোল যোগ করুন:

```swift
Slider(value: $slider, in: 0...1)
```

স্লাইডার পরিবর্তন হলে `slider`-এর মানও পরিবর্তিত হয়, এবং `brightness(slider)` একই সঙ্গে উজ্জ্বলতা আপডেট করে।

এটি SwiftUI-তে "ভ্যারিয়েবল ভিউকে চালিত করে" ধারণার খুবই সাধারণ একটি উদাহরণ।

### `Slider`-এর চেহারা সমন্বয়

ডিফল্টভাবে, `Slider` পাওয়া জায়গার পুরো প্রস্থ ব্যবহার করে।

আমরা এটিকে একটি স্থির প্রস্থ দিতে পারি:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

তারপর এটিকে আরও স্পষ্ট করতে কিছু modifier যোগ করি:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

এতে স্লাইডারের চারপাশে সাদা ব্যাকগ্রাউন্ড এবং গোলাকার কোণা থাকবে, ফলে `Color` ভিউয়ের উপর এটি আরও স্পষ্টভাবে দেখা যাবে।

সবশেষে, এটিকে স্ক্রিনের নিচে রাখুন।

যেহেতু আমরা আগে থেকেই `ZStack` ব্যবহার করেছি, তাই এর ভেতরে একটি `VStack` রেখে `Spacer()` ব্যবহার করে `Slider`-কে নিচে ঠেলে দিতে পারি।

## সম্পূর্ণ কোড

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## সারসংক্ষেপ

আগে শেখা জ্ঞান এবং রঙ ও অ্যারের মতো মৌলিক ধারণা ব্যবহার করে আমরা একটি খুবই মজার ফিল লাইট অ্যাপ তৈরি করেছি।

এই ফিল লাইট অ্যাপের মাধ্যমে আমরা `brightness` দিয়ে উজ্জ্বলতা নিয়ন্ত্রণ করা, `onTapGesture` দিয়ে ভিউতে ট্যাপ অ্যাকশন যোগ করা, এবং `Slider` কন্ট্রোল ব্যবহার করা শিখেছি।

আমরা `Color` ভিউতে `onTapGesture` যোগ করেছি রঙ বদলানোর জন্য। `brightness` নিয়ন্ত্রণ করা একটি ভ্যারিয়েবল পরিবর্তনের জন্য `Slider`-ও ব্যবহার করেছি, যা "ভ্যারিয়েবল ভিউকে চালিত করে"-র আরেকটি উদাহরণ।

আমরা ternary operator-ও আবার দেখেছি, ভিউগুলোকে স্তরবিন্যাস করতে `ZStack` ব্যবহার করেছি, এবং একই ধরনের একগুচ্ছ ডেটা পরিচালনা করতে অ্যারে ব্যবহার করেছি। এতে অ্যারে এবং ইনডেক্স সম্পর্কে আমাদের ধারণা আরও গভীর হয়, বিশেষ করে index out-of-range ত্রুটি এড়িয়ে চলার বাস্তব সমস্যাটি।

যদিও এই উদাহরণটি খুব জটিল নয়, এটি আমাদের আগে শেখা অনেক মৌলিক ধারণাকে একসাথে যুক্ত করে। এগুলো যখন একটি ছোট বাস্তব প্রকল্পে ব্যবহার করা হয়, তখন প্রতিটি ধারণা কোথায় কাজে লাগে তা বোঝা সহজ হয়।

### বাস্তব ব্যবহারের দৃশ্য

ভাবুন, ডেস্কে একটি পুরনো iPhone রেখে, নিজের তৈরি করা ফিল লাইট অ্যাপ দিয়ে আলোর রঙ নিয়ন্ত্রণ করছেন। এটি হবে খুবই ভালো একটি অভিজ্ঞতা।

App Store-এ অনেক "ফিল লাইট" অ্যাপ আছে, আর সেগুলোকেও খুব জটিল মনে হয় না।

![AppStore](../../Resource/018_appStore.PNG)

আমরা সহজ অ্যাপ দিয়ে শুরু করতে পারি, এবং সেগুলো App Store-এ প্রকাশ করার চেষ্টা করতে পারি। এতে শুধু ডেভেলপমেন্টের প্রতি আগ্রহই বাড়ে না, আমাদের বেড়ে ওঠার পথও নথিভুক্ত হয়।

### পাঠ-পরবর্তী অনুশীলন

আপনি এই ফিল লাইট অ্যাপটি আরও কীভাবে বাড়ানো যায় তা ভাবতে পারেন, যেমন:

- আরও রঙ যোগ করা
- বর্তমান উজ্জ্বলতার মান দেখানো
- নিচের স্লাইডার অংশের ডিজাইন আরও সুন্দর করা

যখন আপনি সত্যিই এই জ্ঞান ব্যবহার করতে শুরু করবেন, তখন বুঝতে পারবেন যে আমরা যে প্রতিটি ধারণা শিখি, তা আসলে অ্যাপ তৈরি করার একটি টুল।

আমরা যত বেশি টুল আয়ত্ত করব, তত বেশি ফিচার তৈরি করতে পারব।
