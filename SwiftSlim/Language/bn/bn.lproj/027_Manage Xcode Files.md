# Xcode ফাইল পরিচালনা

এই পাঠে আমরা Xcode-এ সাধারণ ফাইল অপারেশন শিখব।

আগের পাঠগুলোতে আমরা বেশিরভাগ কোড `ContentView`-এর মধ্যে লিখেছি। এতে কোড এবং প্রদর্শনের ফলাফল দেখা সহজ হয়।

উদাহরণস্বরূপ:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

তবে বাস্তব App-এ সাধারণত সব কোড একটি ফাইলের মধ্যে রাখা হয় না।

যখন পৃষ্ঠা, কম্পোনেন্ট এবং ফিচার বাড়তে থাকে, তখন আমাদের বিভিন্ন ভিউ আলাদা SwiftUI ফাইলে ভাগ করতে হয়।

উদাহরণস্বরূপ:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

এতে কোডের গঠন আরও পরিষ্কার হয়, এবং পরবর্তী রক্ষণাবেক্ষণ ও পরিবর্তনও সহজ হয়।

## কেন ফাইল পরিচালনা করতে হয়

SwiftUI-তে একটি পৃষ্ঠা বা একটি কম্পোনেন্ট সাধারণত আলাদা একটি ভিউ ফাইল হিসেবে লেখা যায়।

উদাহরণস্বরূপ:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

এখানে `CardView` একটি SwiftUI ভিউ।

যদি সব ভিউ `ContentView`-এর মধ্যে লেখা হয়, তাহলে কোড ধীরে ধীরে দীর্ঘ হবে এবং পড়াও কঠিন হবে।

তাই আমাদের ফাইল তৈরি, নাম পরিবর্তন, মুছে ফেলা এবং ফোল্ডার ব্যবহার করে বিভিন্ন কোড ফাইল পরিচালনা করা শিখতে হবে।

## SwiftUI ফাইল তৈরি করা

Xcode-এ SwiftUI ফাইল তৈরি করার সাধারণত দুটি প্রচলিত উপায় আছে:

একটি হলো খালি ফাইল তৈরি করে নিজে কোড লেখা।

অন্যটি হলো SwiftUI টেমপ্লেট ফাইল তৈরি করা, যেখানে Xcode স্বয়ংক্রিয়ভাবে মৌলিক কোড তৈরি করে।

নতুনদের জন্য SwiftUI টেমপ্লেট ফাইল ব্যবহার করা বেশি প্রস্তাবিত, কারণ এটি স্বয়ংক্রিয়ভাবে মৌলিক গঠন তৈরি করে।

### খালি ফাইল তৈরি করা

Xcode-এর বাম পাশের Navigator এলাকায়, খালি জায়গা বা ফোল্ডারে ডান-ক্লিক করে `New Empty File` নির্বাচন করুন।

অপারেশনের অবস্থান নিচে দেখানো হয়েছে:

![xcode](../../../Resource/027_view1.png)

খালি ফাইল তৈরি করার পর, Xcode সাধারণত নামহীন একটি Swift ফাইল তৈরি করে, যেমন `Untitled.swift`।

আমরা এটি `TestView.swift` নামে পরিবর্তন করতে পারি।

SwiftUI প্রকল্পে, যদি এই ফাইলটি একটি ভিউ প্রদর্শনের জন্য ব্যবহার করা হয়, তাহলে সাধারণত নামের শেষে `View` যোগ করার পরামর্শ দেওয়া হয়।

উদাহরণস্বরূপ:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

এটি বাধ্যতামূলক নিয়ম নয়, তবে এতে এক নজরেই বোঝা যায় এটি একটি ভিউ ফাইল।

খালি ফাইল তৈরি করার পর, ফাইলের মধ্যে শুধু ডিফল্ট মন্তব্য থাকতে পারে, আবার ফাঁকাও থাকতে পারে। এটি Xcode সংস্করণ এবং তৈরির পদ্ধতির উপর নির্ভর করে।

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

এই সময় আমাদের হাতে SwiftUI কোড লিখতে হবে:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

এভাবেই একটি সহজ SwiftUI ভিউ ফাইল তৈরি হয়ে যায়।

### ফাইলের নাম পরিবর্তন করা

ফাইল তৈরি করার পর যদি ফাইলের নাম পরিবর্তন করতে চান, তাহলে Xcode-এর বাম পাশের Navigator এলাকায় ফাইলটি নির্বাচন করে Enter কী `⏎` চাপতে পারেন।

![xcode](../../../Resource/027_view2.png)

এভাবে ফাইলের নাম পরিবর্তন করা যায়।

খেয়াল রাখতে হবে, ফাইলের নাম পরিবর্তন করলে কোডের মধ্যে থাকা ভিউর নাম স্বয়ংক্রিয়ভাবে পরিবর্তন হয় না।

উদাহরণস্বরূপ, ফাইলের নাম `ProfileView.swift` করার পরও কোডের মধ্যে এখনও `struct TestView: View` থাকতে পারে।

এতে অগত্যা ত্রুটি হবে এমন নয়, তবে সহজেই বিভ্রান্তি তৈরি হতে পারে।

তাই ফাইলের নাম এবং ভিউর নাম একই রাখা ভালো। যেমন, ফাইলের নাম `ProfileView.swift` হলে ভিউর নামও `ProfileView` ব্যবহার করা উচিত।

## SwiftUI টেমপ্লেট ফাইল তৈরি করা

খালি ফাইল তৈরি করার পাশাপাশি, আমরা সরাসরি SwiftUI টেমপ্লেট ফাইলও তৈরি করতে পারি।

Xcode-এর বাম পাশের Navigator এলাকায়, খালি জায়গা বা ফোল্ডারে ডান-ক্লিক করে `New File from Template...` নির্বাচন করুন।

অপারেশনের অবস্থান নিচে দেখানো হয়েছে:

![xcode](../../../Resource/027_view3.png)

পপ-আপ উইন্ডোর উপরের অংশে বিভিন্ন প্ল্যাটফর্ম নির্বাচন করা যায়, যেমন `iOS`, `macOS` ইত্যাদি।

এখানে বর্তমান প্রকল্পের সঙ্গে মিল থাকা প্ল্যাটফর্ম নির্বাচন করলেই হবে। যেমন, আমরা যদি iOS App তৈরি করি, তাহলে `iOS` নির্বাচন করি।

![xcode](../../../Resource/027_view4.png)

তারপর নিচে `SwiftUI View` নির্বাচন করুন।

![xcode](../../../Resource/027_view12.png)

তারপর `Next` ক্লিক করে ফাইলের নাম লিখে সংরক্ষণ করুন।

উদাহরণস্বরূপ, আমরা একটি `Test2View.swift` তৈরি করি।

Xcode স্বয়ংক্রিয়ভাবে নিচের মতো কোড তৈরি করবে:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

খালি ফাইলের তুলনায়, টেমপ্লেট ফাইলে স্বয়ংক্রিয়ভাবে `import SwiftUI`, `struct`, `body` এবং `#Preview` থাকে।

তাই নতুনদের জন্য SwiftUI টেমপ্লেট ফাইল ব্যবহার করা আরও সহজ, এবং কোড বাদ পড়ার সম্ভাবনাও কম।

**টিপস**: ডান-ক্লিক করে ফাইল তৈরি করার পাশাপাশি, Xcode-এর বাম পাশের Navigator-এর নিচের প্লাস বোতামেও ক্লিক করে ফাইল বা ফোল্ডার তৈরি করা যায়।

![xcode](../../../Resource/027_view5.png)

## ফোল্ডার তৈরি করা

যখন ফাইলের সংখ্যা বাড়তে থাকে, তখন আমরা ফোল্ডার ব্যবহার করে সেগুলো পরিচালনা করতে পারি।

Xcode-এর বাম পাশের Navigator এলাকায়, খালি জায়গা বা ফোল্ডারে ডান-ক্লিক করে `New Folder` নির্বাচন করুন।

ফোল্ডার তৈরি করার পর, সম্পর্কিত ফাইলগুলো ফোল্ডারের মধ্যে টেনে নিয়ে যেতে পারেন।

![xcode](../../../Resource/027_view7.png)

ফোল্ডার মূলত প্রকল্পের গঠন সাজানোর জন্য ব্যবহৃত হয়, এটি ভিউ ব্যবহারের পদ্ধতি নিজে থেকে বদলায় না।

যতক্ষণ ভিউর নাম সঠিক থাকে, অন্য ফাইলেও এখনও `TestView()`, `Test2View()`-এর মতো করে ব্যবহার করা যায়।

## ফাইল বা ফোল্ডার মুছে ফেলা

যদি ফাইল বা ফোল্ডার মুছে ফেলতে চান, Xcode-এর বাম পাশের Navigator এলাকায় সংশ্লিষ্ট ফাইল নির্বাচন করে ডান-ক্লিক করুন, তারপর `Delete` নির্বাচন করুন।

ফাইল নির্বাচন করার পর Delete কী `⌫` চাপলেও হয়।

Xcode একটি মুছে ফেলার নিশ্চিতকরণ উইন্ডো দেখাবে।

![xcode](../../../Resource/027_view10.png)

`Move to Trash` নির্বাচন করার অর্থ হলো ফাইলটি Trash-এ সরিয়ে দেওয়া।

যদি `Remove Reference` দেখা যায়, এর অর্থ হলো শুধু Xcode প্রকল্প থেকে রেফারেন্স সরানো হবে, কিন্তু ডিস্কের আসল ফাইল মুছে ফেলা হবে না।

নতুনদের জন্য, যদি আপনি শুধু নিজে সদ্য তৈরি করা সাধারণ ফাইল মুছতে চান, সাধারণত `Move to Trash` নির্বাচন করলেই হবে।

## একাধিক SwiftUI ভিউ প্রদর্শন করা

ফাইল তৈরি করা শেখার পর, আমরা বিভিন্ন ভিউ আলাদা ফাইলে ভাগ করতে পারি।

উদাহরণস্বরূপ, আমরা দুটি SwiftUI ফাইল তৈরি করি:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` লেখকের নাম প্রদর্শনের জন্য ব্যবহৃত হয়:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` কার্ডের পটভূমি প্রদর্শনের জন্য ব্যবহৃত হয়:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

তারপর আমরা `ContentView`-এর মধ্যে এই দুটি ভিউ ব্যবহার করতে পারি:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

চালানোর পর দেখা যাবে, `CardView` এবং `AuthorView` একসঙ্গে মিলিয়ে প্রদর্শিত হয়েছে:

![xcode](../../../Resource/027_view11.png)

এই ফলাফলে `CardView` পটভূমি হিসেবে কাজ করছে, আর `AuthorView` উপরের দিকে প্রদর্শিত হচ্ছে। এর কারণ হলো এগুলো `ZStack`-এর মধ্যে রাখা হয়েছে।

`ContentView`-এর মধ্যে আমরা `CardView()` এবং `AuthorView()` দিয়ে এই দুটি ভিউ ব্যবহার করি। নামের পরের `()` বোঝায় এই ভিউটি তৈরি করে ব্যবহার করা।

এটাই SwiftUI-তে সাধারণ ভিউ নেস্টিং: একটি ভিউর মধ্যে অন্য ভিউও ব্যবহার করা যায়।

## এন্ট্রি ফাইল

শেষে, আমরা আবার দেখে নিই App কোথা থেকে ইন্টারফেস দেখানো শুরু করে।

SwiftUI প্রকল্পে সাধারণত একটি এন্ট্রি ফাইল থাকে:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

এখানে `@main` বোঝায় এটি App-এর এন্ট্রি অবস্থান।

`WindowGroup`-এর মধ্যে আমরা দেখি:

```swift
ContentView()
```

এর অর্থ হলো App চালু হওয়ার পর প্রথমে `ContentView` প্রদর্শিত হয়।

তারপর `ContentView`-এর মধ্যে আবার প্রদর্শিত হয়:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

তাই ভিউ স্তরকে এভাবে বোঝা যায়:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

যদি আমরা এন্ট্রি ফাইলে প্রদর্শিত ভিউটি `CardView`-এ পরিবর্তন করি:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

তাহলে App চালু হওয়ার পর শুধু `CardView` প্রদর্শিত হবে।

অর্থাৎ, এন্ট্রি কোডের `WindowGroup` নির্ধারণ করে App চালু হওয়ার পর প্রথমে কোন ভিউ প্রদর্শিত হবে।

আর এই ভিউর ভিতরে আবার অন্য ভিউও প্রদর্শন করা যেতে পারে।

## সারাংশ

এই পাঠে আমরা Xcode-এ সাধারণ ফাইল পরিচালনা অপারেশন শিখেছি।

আমরা শিখেছি কীভাবে SwiftUI ফাইল তৈরি করতে হয়, কীভাবে ফাইলের নাম পরিবর্তন করতে হয়, কীভাবে ফোল্ডার তৈরি করতে হয়, এবং কীভাবে ফাইল মুছে ফেলতে হয়।

একই সঙ্গে আমরা একটি গুরুত্বপূর্ণ ধারণাও শিখেছি: SwiftUI ভিউগুলো আলাদা ফাইলে ভাগ করা যায়, তারপর অন্য ভিউর মধ্যে ব্যবহার করা যায়।

উদাহরণস্বরূপ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

এখানে `CardView()` এবং `AuthorView()` হলো `ContentView`-এর মধ্যে অন্য ভিউ ব্যবহার করা।

শেষে আমরা এন্ট্রি ফাইলের প্রদর্শন যুক্তিও আবার দেখেছি।

App চালু হলে প্রথমে `@main` চিহ্নিত এন্ট্রি কোডে প্রবেশ করে, তারপর `WindowGroup`-এর মধ্যে থাকা ভিউ প্রদর্শন করে।

এটি আমাদের বুঝতে সাহায্য করে: বিভিন্ন SwiftUI ফাইল একে অপরের থেকে বিচ্ছিন্ন নয়; এগুলো একে অপরের সঙ্গে মিলিয়ে শেষ পর্যন্ত একটি সম্পূর্ণ App ইন্টারফেস তৈরি করতে পারে।
