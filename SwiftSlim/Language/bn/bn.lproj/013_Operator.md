# অপারেটর

এই পাঠে আমরা Swift এর অপারেটর (Operators) সম্পর্কে জানব।

যখন আমরা হিসাব-নিকাশ, তুলনা বা যৌক্তিক (logical) সিদ্ধান্ত গ্রহণ করি, তখন আমাদের অপারেটর ব্যবহার করতে হয়। এই কাজগুলো প্রোগ্রাম এক্সিকিউশনের প্রতিটি ধাপে জড়িয়ে থাকে, যা প্রোগ্রামিংয়ের অন্যতম মৌলিক একটি দক্ষতা।

```swift
let a = 100 * 8
```

প্রোগ্রামিং শেখার সময় অনেকেই খেয়াল করেন: যাদের গাণিতিক চিন্তাধারা ভালো, তারা সাধারণত প্রোগ্রামের লজিক সহজে বুঝতে পারেন। এর কারণ হলো প্রোগ্রামিং মূলত "হিসাব" এবং "নিয়ম যাচাই" ছাড়া অচল, আর এই মূল কাজগুলোই অপারেটর করে থাকে।

বাস্তব ডেভেলপমেন্টে, অপারেটর শুধু সাধারণ সংখ্যার হিসাব-নিকাশেই ব্যবহৃত হয় না, বরং বিভিন্ন নির্দিষ্ট ক্ষেত্রেও এর ব্যাপক ব্যবহার রয়েছে।

উদাহরণস্বরূপ, একটি ব্লক গেমে, ব্যবহারকারী যখন কোনো ব্লক টেনে নির্দিষ্ট জায়গায় নিয়ে যান, তখন হিসাব করতে হয়:

- ব্লকের কোঅর্ডিনেট বা অবস্থান
- ব্লকটি কোথায় বসানো যেতে পারে
- লক্ষ্যস্থানটি (target position) ব্যবহারযোগ্য কি না তা যাচাই করা

এই "হিসাব + যাচাই"-এর প্রক্রিয়াগুলো মূলত অপারেটরের ওপর নির্ভর করেই সম্পন্ন হয়।

![Block](../../Resource/013_block.png)

অপারেটর শেখা মানে শুধু সংখ্যার হিসাব শেখা নয়, বরং এটি ভবিষ্যতের ইউজার ইন্টারফেস লেআউট, ডেটা প্রসেসিং এবং ইন্টারেক্টিভ লজিকের জন্য একটি শক্ত ভিত্তি তৈরি করে।

## SwiftUI-তে হিসাব এক্সিকিউট করার উপায়

SwiftUI-তে সাধারণত Button-এর মাধ্যমে ফাংশন ট্রিগার করা হয় এবং এরপর সেই ফাংশনের ভেতরে গাণিতিক লজিক এক্সিকিউট করা হয়।

উদাহরণস্বরূপ:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

যখন বাটন ক্লিক করা হবে, কনসোলে (Console) আউটপুট আসবে:

```
c: 20
```

এই পাঠে আমরা সব গাণিতিক লজিক func-এর ভেতরে লিখব, বাটনের মাধ্যমে তা এক্সিকিউট করব এবং ফলাফল দেখাতে print ব্যবহার করব।

পরবর্তী উদাহরণগুলোতে, কোড সহজভাবে দেখানোর জন্য আমরা সরাসরি কোডের পাশেই হিসাবের ফলাফল উল্লেখ করব এবং বারবার print আউটপুট ব্যবহার করব না।

## কনসোল (Console) আউটপুট ব্যবস্থাপনা

যেহেতু প্রতি ক্লিকেই আউটপুট ইনফরমেশন তৈরি হয়, তাই আপনি কনসোল এরিয়ার নিচের ডান কোণায় থাকা ক্লিয়ার (clear) বাটনে ক্লিক করে পুরনো লগ মুছে ফেলতে পারেন, যাতে বর্তমান ফলাফল দেখতে সুবিধা হয়।

## Button লেখার আরেকটি নিয়ম

বাটন যখন শুধুমাত্র একটি ফাংশন এক্সিকিউট করে, তখন আপনি আরও পরিচ্ছন্ন একটি পদ্ধতি ব্যবহার করতে পারেন:

```swift
Button(action: ) {
    // View
}
```

এই লেখায়, action নির্ধারণ করে বাটন ক্লিক করার পর কোন ফাংশনটি (প্যারামিটার ছাড়া ফাংশন) চলবে, আর দ্বিতীয় বন্ধনী {} বাটনের ভিউ (View) তৈরি করতে ব্যবহৃত হয়।

আগের calculate ফাংশনটির সাথে মিলিয়ে এটিকে এভাবে লেখা যায়:

```swift
Button(action: calculate) {
    Text("Button")
}
```

বাটন ক্লিক করা হলে, সরাসরি calculate ফাংশনটি এক্সিকিউট হবে।

তাই, যখন বাটনের লজিক খুব সহজ হয় এবং এটি শুধু একটি ফাংশন কল করে, তখন action দিয়ে লেখাটি বেশি পরিষ্কার দেখায় এবং এটি কোডিং স্ট্যান্ডার্ডের সাথে মানানসই।

## গাণিতিক অপারেটর (Arithmetic Operators)

গাণিতিক অপারেটর (Arithmetic Operators) সংখ্যার হিসাব-নিকাশে ব্যবহৃত হয়:

- \+ যোগ
- \- বিয়োগ
- \* গুণ
- / ভাগ
- % মডিউলো বা ভাগশেষ

উদাহরণ:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

একটি বিষয় খেয়াল রাখতে হবে, Swift একটি স্ট্রংলি-টাইপড (strongly-typed) ভাষা, তাই এটি স্বয়ংক্রিয়ভাবে ভিন্ন ধরনের ডেটা টাইপ মেশায় না:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

যখন দুটি টাইপ ভিন্ন হয়, তখন ম্যানুয়ালি টাইপ কনভার্ট করে নিতে হয়।

## তুলনামূলক অপারেটর (Comparison Operators)

তুলনামূলক অপারেটর (Comparison Operators) দুটি ভ্যালুর সম্পর্ক যাচাই করতে ব্যবহৃত হয়, যার ফলাফল হয় Bool:

- == সমান
- != অসমান
- \> বৃহত্তর
- < ক্ষুদ্রতর
- \>= বৃহত্তর বা সমান
- <= ক্ষুদ্রতর বা সমান

উদাহরণ:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

তুলনামূলক অপারেটর সাধারণত কন্ডিশনাল স্টেটমেন্টে (if-else) ব্যবহৃত হয়। যেমন "উক্তি স্লাইডার (Quotes Carousel)" বানানোর সময় আমাদের খেয়াল রাখতে হয় যেন ইনডেক্সটি অ্যারের (array) সীমার বাইরে না যায়:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

যদি index এর মান sayings.count - 1 এর চেয়ে ছোট হয়, তবে তুলনার ফলাফল হবে true, এবং if স্টেটমেন্টের ভেতরের কোড চলবে; আর ফলাফল false হলে, ভেতরের কোডটি চলবে না।

## লজিক্যাল অপারেটর (Logical Operators)

লজিক্যাল অপারেটর (Logical Operators) একাধিক শর্ত বা কন্ডিশন যোগ করতে ব্যবহৃত হয়:

- && এবং (AND)
- || অথবা (OR)
- ! না (NOT)

### && এবং (AND)

যখন একাধিক শর্ত একই সাথে পূরণ হতে হবে, তখন && ব্যবহৃত হয়।

উদাহরণ:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

এই কোডে দুটি শর্ত রয়েছে: age >= 18 এবং age <= 100।

শুধুমাত্র যখন এই দুটি শর্ত একযোগে true হবে, তখনই legalAge এর মান true হবে; এর মধ্যে যেকোনো একটি শর্ত পূরণ না হলে, ফলাফল হবে false।

### || অথবা (OR)

যখন একাধিক শর্তের মধ্যে কেবল যেকোনো একটি পূরণ হলেই চলবে, তখন || ব্যবহৃত হয়।

উদাহরণ:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

এই কোডটি বোঝাচ্ছে যে, money >= 50 অথবা creditCard >= 50 এর মধ্যে যেকোনো একটি সত্য হলে, payAllowed এর মান true হবে।

কেবলমাত্র দুটি শর্তই পূরণ না হলে ফলাফল false হবে।

উদাহরণস্বরূপ, পেমেন্টের ক্ষেত্রে আপনি নগদ অর্থ বা ক্রেডিট কার্ড ব্যবহার করে পেমেন্ট করতে পারেন। যদি দুটোর কোনোটি দিয়েই পেমেন্ট সম্ভব না হয়, তবেই পেমেন্ট ব্যর্থ হবে।

### ! না (NOT)

! হলো লজিক্যাল NOT অপারেটর, যা কোনো Bool ভ্যালুকে উল্টে দিতে (reverse) ব্যবহৃত হয়।

সহজ কথায়, ! অপারেটর true-কে false বানায়, এবং false-কে true বানায়।

উদাহরণ:

```swift
let allowed = true
let result = !allowed   // false
```

এখানে !allowed দ্বারা allowed এর মানকে উল্টে দেওয়া বোঝানো হয়েছে।

মনে রাখা জরুরি, ! এবং != দুটি সম্পূর্ণ ভিন্ন অপারেটর। ! কোনো Bool ভ্যালুকে উল্টে দেয়, আর != হলো তুলনামূলক অপারেটর যা তুলনা করে একটি Bool ভ্যালু প্রদান করে।

## অ্যাসাইনমেন্ট অপারেটর (Assignment Operators)

অ্যাসাইনমেন্ট অপারেটর (Assignment Operators) ভেরিয়েবলে মান বসাতে বা মান আপডেট করতে ব্যবহৃত হয়:

- = অ্যাসাইনমেন্ট (মান বসানো)

ভেরিয়েবল ডিক্লেয়ার করার সময় বা হিসাব-নিকাশের প্রক্রিয়ায় অ্যাসাইনমেন্টের কাজ জড়িত থাকে:

```swift
let a = 5
```

এই কোডটি বোঝাচ্ছে যে, ৫ মানটি ভেরিয়েবল a-এর ভেতরে রাখা (assign) হলো।

### কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটর (Compound Assignment Operators)

বাস্তব ডেভেলপমেন্টে, সাধারণ অ্যাসাইনমেন্ট অপারেটরের পাশাপাশি "কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটর" প্রায়ই ব্যবহৃত হয়, যা কোনো ভেরিয়েবলের মূল মানের সাথে সরাসরি হিসাব করে মান আপডেট করে:

- +=
- -=
- *=
- /=

আগের "কাউন্টার (Counter)" নামক পাঠে আমরা কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটর সম্পর্কে কিছুটা জেনেছি।

উদাহরণস্বরূপ, বাটনে ক্লিক করলে ভেরিয়েবলের মান স্বয়ংক্রিয়ভাবে বাড়তে থাকে:

```swift
var num = 10
num += 5   // num = 15
```

এই কোডটি বোঝাচ্ছে যে, num এর সাথে ৫ যোগ করা হবে, এবং সেই যোগফলটি আবার num-এর ভেতরেই রাখা হবে।

তাই, এটি নিচের কোডটির সমতুল্য:

```swift
var num = 10
num = num + 5   // num = 15
```

## টার্নারি অপারেটর (Ternary Operator)

Swift-এর টার্নারি অপারেটর (Ternary Operator) শর্তের ওপর ভিত্তি করে সিদ্ধান্ত নেয় এবং দু'টি ফলাফলের মধ্যে একটি প্রদান করে:

```swift
Condition ? Value1 : Value2
```

এর কাজ করার নিয়ম হলো, যদি শর্তটি true হয়, তবে "Value1" প্রদান করবে; আর যদি শর্তটি false হয়, তবে "Value2" প্রদান করবে।

উদাহরণ:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

এই কোডটি showColor-এর মান যাচাই করবে: যদি true হয়, তবে Color.blue প্রদান করবে এবং ব্যাকগ্রাউন্ড নীল দেখাবে; আর false হলে Color.clear প্রদান করবে, ফলে কোনো ব্যাকগ্রাউন্ড কালার দেখাবে না।

টিপস: Color.clear বলতে স্বচ্ছ (transparent) রঙ বোঝায় (যা কিছু প্রদর্শন করে না)।

### if-else এর সাথে পার্থক্য

টার্নারি অপারেটর লজিক্যাল দিক থেকে if-else স্টেটমেন্টের সমতুল্য।

যেমন, ওপরের কোডটিকে এভাবেও লেখা যায়:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

উভয়ের এক্সিকিউশন লজিক একদম একই, শর্ত true হলে প্রথম ফলাফলটি কাজ করবে; আর false হলে দ্বিতীয় ফলাফলটি কাজ করবে।

বাস্তব ব্যবহারে, টার্নারি অপারেটর লেখা অনেক সহজ এবং পরিচ্ছন্ন। এটি "দুটির মধ্যে একটি বাছাই" করার মতো সাধারণ যাচাইয়ের ক্ষেত্রে বেশি উপযোগী এবং সরাসরি কোনো মান রিটার্ন করার জন্য সাধারণত ব্যবহৃত হয়।

SwiftUI-তে ভিউ (View) তৈরির সময় টার্নারি অপারেটর খুবই পরিচিত একটি পদ্ধতি, যেমন:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

এই লেখার মাধ্যমে আপনি মাত্র এক লাইন কোড দিয়ে শর্ত অনুযায়ী ভিউ-এর আউটপুট ডায়নামিকভাবে পরিবর্তন করতে পারেন।

## রেঞ্জ অপারেটর (Range Operators)

রেঞ্জ অপারেটর (Range Operators) কোনো নির্দিষ্ট সীমা বা রেঞ্জ বোঝাতে ব্যবহৃত হয়:

- ... ক্লোজড রেঞ্জ (উভয় প্রান্ত অন্তর্ভুক্ত)
- ..< হাফ-ওপেন রেঞ্জ (ডান দিকের মান অন্তর্ভুক্ত নয়)

উদাহরণ:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

সহজভাবে বলতে গেলে, 1...3 মানে ১ থেকে শুরু করে ৩ পর্যন্ত, যেখানে ৩ অন্তর্ভুক্ত; আর 5..<8 মানে ৫ থেকে শুরু করে ৮ পর্যন্ত, তবে এতে ৮ অন্তর্ভুক্ত নয় (অর্থাৎ ৭ পর্যন্ত)।

বর্ধিত রূপ (শুধু জানার জন্য):

```
5...    // >= 5
...5    // <= 5
```

এই ধরনটি "ওয়ান-সাইডেড রেঞ্জ (One-Sided Range)" বোঝায়, যা সাধারণত কন্ডিশন চেক করতে বা প্যাটার্ন ম্যাচিংয়ে ব্যবহৃত হয়।

পরবর্তীতে for লুপ (for-loop) শেখার সময় রেঞ্জ অপারেটর ব্যবহৃত হবে, তবে আপাতত এটিকে একটি টানা বা ধারাবাহিক সংখ্যার সীমা হিসেবে মনে রাখতে পারেন।

## অপারেটর অনুশীলন

নিচে কয়েকটি সহজ উদাহরণের মাধ্যমে আমরা ভিউ-এর (View) ভেতরে অপারেটরের কাজ অনুশীলন করব।

### ১. মোট গাড়ির সংখ্যা হিসাব করা

ধরা যাক, একটি মোড় দিয়ে প্রতিদিন ৫০০টি গাড়ি চলাচল করে, তাহলে ৩০ দিনে মোট কতগুলো গাড়ি চলাচল করবে তার হিসাব বের করা।

আমরা এর জন্য গাণিতিক অপারেটর ব্যবহার করতে পারি:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

এখানে হিসাবের প্রক্রিয়াটি হলো, প্রতিদিন ৫০০ গাড়ি, ৩০ দিনে মোট পরিমাণ ৫০০ × ৩০, যার ফলে মোট ১৫,০০০ গাড়ির সংখ্যা পাওয়া যায়।

### ২. ইন-অ্যাপ পারচেস মেম্বারশিপ যাচাই করা

আমরা যাচাই করতে পারি কোনো ব্যক্তি ইন-অ্যাপ মেম্বার কি না। যদি তিনি লাইফটাইম মেম্বারশিপ বা সাবস্ক্রিপশন মেম্বারশিপের যেকোনো একটি কিনে থাকেন, তবে তাকে মেম্বার হিসেবে ধরা হবে।

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

এই কোডে || (অথবা/OR) অপারেটর ব্যবহার করা হয়েছে: lifeTime = false মানে তার লাইফটাইম মেম্বারশিপ নেই; আর subscription = true মানে তার সাবস্ক্রিপশন আছে।

যেহেতু যেকোনো একটি শর্ত পূরণ হলেই চলবে, তাই isMember এর মান true হবে। এর ফলে, ওই ব্যক্তি একজন ইন-অ্যাপ মেম্বার হিসেবে গণ্য হবেন।

## সারসংক্ষেপ

Swift-এ অপারেটর হলো ডেটা হিসাব-নিকাশ এবং লজিক্যাল সিদ্ধান্তের মূল ভিত্তি। Text যেমন কন্টেন্ট দেখাতে কাজ করে, অপারেটর তেমনি ডেটা প্রসেস করতে কাজ করে।

সহজ সংখ্যা হিসাব থেকে শুরু করে, জটিল লজিক যাচাই এবং ভিউ-এর স্টেট (State) কন্ট্রোল ও লেআউট নির্ধারণ পর্যন্ত প্রতিটি কাজেই অপারেটরের উপস্থিতি অপরিহার্য।

বিভিন্ন পরিস্থিতির ওপর ভিত্তি করে অনুশীলন করার মাধ্যমে আপনি এই পাঠে শেখা বিভিন্ন ধরনের অপারেটরগুলোতে অভ্যস্ত হতে পারবেন, যা ভবিষ্যতের শেখার জন্য একটি অত্যন্ত গুরুত্বপূর্ণ ভিত্তি হিসেবে কাজ করবে।

### অতিরিক্ত জ্ঞান - বিটওয়াইজ অপারেটর (Bitwise Operators)

এছাড়া, Swift আরও এক ধরনের লো-লেভেল (low-level) অপারেটর প্রদান করে — বিটওয়াইজ অপারেটর (Bitwise Operators):

- & (বিটওয়াইজ অ্যান্ড/AND)
- | (বিটওয়াইজ অর/OR)
- ^ (বিটওয়াইজ এক্স-অর/XOR)
- ~ (বিটওয়াইজ নট/NOT)
- \>> (রাইট শিফট)
- << (লেফট শিফট)

উদাহরণ:

```swift
let a = 6  // বাইনারি 110
let b = 3  // বাইনারি 011

print(a & b) // 2 (বাইনারি 010)
print(a | b) // 7 (বাইনারি 111)
print(a ^ b) // 5 (বাইনারি 101)
print(~a)    // -7 (ইনভার্ট/নট)
```

Swift-এ, পূর্ণসংখ্যা বা ইন্টিজার (integer) বাইনারি টু'স কমপ্লিমেন্ট (two's complement) ব্যবহার করে বোঝানো হয়, তাই নট (NOT) বা invert করার পর তার বিপরীত ঋণাত্মক (negative) ফলাফল পাওয়া যায়।

বিটওয়াইজ অপারেটর সরাসরি বাইনারি ডেটার ওপর কাজ করে। এগুলো সাধারণত লো-লেভেল ডেটা প্রসেসিং এবং পারফরম্যান্স অপ্টিমাইজেশনের ক্ষেত্রে ব্যবহৃত হয়।

নতুনদের জন্য এই অপারেটরগুলোর ব্যবহারের ক্ষেত্র তুলনামূলক কম, এবং সাধারণ iOS বা SwiftUI ডেভেলপমেন্টেও এগুলো খুব একটা দেখা যায় না। তাই এখানে এগুলো নিয়ে বিস্তারিত আলোচনা করা হলো না।

পরবর্তীতে যদি লো-লেভেল লজিক বা পারফরম্যান্স অপ্টিমাইজেশন নিয়ে কাজ করার প্রয়োজন হয়, তখন এগুলো সম্পর্কে আরও বিস্তারিত শেখা এবং বোঝা যাবে।
