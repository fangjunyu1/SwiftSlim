# SwiftUI ভিউ লাইফসাইকেল

আগের অধ্যায়ে, আমরা জেনেছিলাম যে @State ভেরিয়েবল ভিউর লাইফসাইকেলের সাথে চলে, এবং আমরা onAppear-এর মৌলিক ব্যবহারও শিখেছিলাম।

এই পাঠে, আমরা SwiftUI ভিউর লাইফসাইকেল আরও গভীরভাবে বুঝব, জানব ভিউ কীভাবে তৈরি এবং প্রদর্শিত হয়, ভিউর ভেরিয়েবলগুলো কীভাবে তৈরি ও পুনর্নির্মিত হয়, এবং onAppear, onDisappear ও init কনস্ট্রাক্টর শিখব।

এই জ্ঞানের ভিত্তিতে, আমরা SwiftUI ভিউর সম্পূর্ণ নির্মাণ প্রক্রিয়া সম্পর্কে সামগ্রিক ধারণা গড়ে তুলব।

## অ্যাপ চালু হওয়ার ক্রম

iOS প্রজেক্ট তৈরি করার সময়, Xcode ডিফল্টরূপে দুটি ফাইল তৈরি করে:

```
ContentView.swift
প্রজেক্টের নাম + App.swift
```

এর মধ্যে, "প্রজেক্টের নাম + App".swift হলো সম্পূর্ণ অ্যাপের এন্ট্রি ফাইল।

উদাহরণ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

### কোড এক্সিকিউশন প্রবাহ

যখন আমরা অ্যাপ চালাই (সিমুলেটর বা রিয়েল ডিভাইসে), সিস্টেম @main কীওয়ার্ড খোঁজে:

```swift
@main
struct SwiftSlimTestApp: App {...}
```

এন্ট্রি ফাইল নিশ্চিত হওয়ার পর, কোড এক্সিকিউশন শুরু হয়।

প্রথমে App স্ট্রাকচারে প্রবেশ করে, `body`-র ভেতরের কোড এক্সিকিউট হয়, তারপর WindowGroup তৈরি হয় এবং এর মধ্যে থাকা ContentView লোড হয়।

### WindowGroup-এর কাজ

WindowGroup উইন্ডো পরিচালনার জন্য ব্যবহৃত হয়:

```swift
WindowGroup {
    ContentView()
}
```

iPad / macOS সিস্টেম মাল্টি-উইন্ডো সমর্থন করে, iPhone সাধারণত একটি উইন্ডো থাকে।

তাই, iPhone-এ WindowGroup মূলত প্রদর্শিত প্রথম ইন্টারফেস পরিচালনা করে।

### ভিউ লোড হওয়ার প্রক্রিয়া

সিস্টেম ContentView খোঁজার পর:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
    }
}
```

SwiftUI `body`-র কোড এক্সিকিউট করবে, তারপর রিটার্ন করা ভিউ স্ট্রাকচার (যেমন VStack, Text ইত্যাদি) অনুযায়ী ইন্টারফেস তৈরি করে স্ক্রিনে প্রদর্শন করবে।

এই ধাপগুলো সম্পন্ন হওয়ার পর, আমরা ভিউতে ContentView দেখতে পাব।

![ContentView](../../Resource/016_view1.png)

উল্লেখ্য, `body`-র কাজ হলো ভিউ তৈরি করা, ভিউ সংরক্ষণ করা নয়।

এর মানে হলো, প্রতিবার ভিউ রিফ্রেশ হওয়ার সময়, SwiftUI পুনরায় `body` ক্যালকুলেট করবে এবং নতুন ভিউ স্ট্রাকচার তৈরি করবে।

### প্রিভিউ ভিউ লজিক

Xcode প্রিভিউ (Canvas) এবং বাস্তব রান দুটি ভিন্ন মেকানিজম।

উদাহরণস্বরূপ, এন্ট্রি ফাইলে একটি ডিবাগ আউটপুট যোগ করুন:

```swift
@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .onAppear {
                    print("Go to ContentView")
                }
        }
    }
}
```

Xcode-এ ContentView ভিউ প্রিভিউ করলে, @main এন্ট্রি থেকে এক্সিকিউশন শুরু হয় না, তাই এন্ট্রি ফাইলে থাকা onAppear ট্রিগার হবে না।

কিন্তু, সিমুলেটর বা রিয়েল ডিভাইসে চালালে, সিস্টেম @main থেকে এক্সিকিউশন শুরু করে, সম্পূর্ণ প্রবাহ অনুযায়ী পুরো অ্যাপ চালায়, এন্ট্রিতে থাকা onAppear ট্রিগার হবে এবং ডিবাগ তথ্য আউটপুট হবে।

মূল বিষয় হলো, Xcode প্রিভিউ শুধু "আংশিক রেন্ডারিং", শুধুমাত্র বর্তমান ভিউ প্রদর্শনের জন্য ব্যবহৃত হয়; সিমুলেটর এবং রিয়েল ডিভাইস "সম্পূর্ণ রান পরিবেশ" প্রদান করে, সম্পূর্ণ অ্যাপ এক্সিকিউট করে।

তাই, অ্যাপের ফিচার পরীক্ষা করার সময়, সিমুলেটর বা রিয়েল ডিভাইসকে মানদণ্ড হিসেবে ধরতে হবে, Xcode প্রিভিউ সম্পূর্ণ রান পরিবেশ প্রদান করতে পারে না।

## ভিউর লাইফসাইকেল

বর্তমান পর্যায়ে, আমাদের সমস্ত কোড ContentView-এ কেন্দ্রীভূত। কিন্তু বাস্তব অ্যাপে, সাধারণত একটি অ্যাপে একাধিক ভিউ থাকে এবং ভিন্ন পরিস্থিতিতে তাদের মধ্যে সুইচ করা হয়।

উদাহরণস্বরূপ, "সঞ্চয় বাক্স"-এ:

![PiggyBank](../../Resource/016_view.png)

"সঞ্চয়" ক্লিক করলে, SaveView ভিউ প্রদর্শিত হয়; সঞ্চয় শেষ হলে বা বন্ধ করলে, SaveView ভিউ সরিয়ে দেওয়া হয়।

এই প্রক্রিয়াটিই একটি ভিউর লাইফসাইকেল, ভিউ তৈরি থেকে শুরু করে প্রদর্শন, সবশেষে সরিয়ে দেওয়া পর্যন্ত।

### অ্যাপ বন্ধ করা

যখন অ্যাপ বন্ধ করা হয়, সমস্ত ভিউ সরিয়ে দেওয়া হয়, সংশ্লিষ্ট স্টেটও অদৃশ্য হয়ে যায়।

তাই, ContentView এবং অন্যান্য ভিউ সবই মেমরি থেকে সরিয়ে দেওয়া হবে, সম্পূর্ণ অ্যাপের রান স্টেটও মুছে যাবে।

## ভিউর ভেরিয়েবলের লাইফসাইকেল

SwiftUI-তে, ভেরিয়েবলের লাইফসাইকেল সাধারণত ভিউর সাথে বাঁধা।

উদাহরণ:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"
    var body: some View {
        var num = 10
        VStack {
            Text("num:\(num)")
            Text("name:\(name)")
        }
    }
}
```

### @State ভেরিয়েবল

@State দিয়ে ঘোষিত ভেরিয়েবল ভিউর লাইফসাইকেলের সাথে বাঁধা।

```swift
@State private var name = "FangJunyu"
```

যখন ভিউ তৈরি হয়, `name`-ও তৈরি হয়; যখন ভিউ সরিয়ে দেওয়া হয়, `name`-ও ধ্বংস হয়।

এজন্যই UserDefaults ইত্যাদি পদ্ধতি ব্যবহার করে ডেটা পার্সিসটেন্স করার প্রয়োজন হয়।

### body-র ভেতরের ভেরিয়েবল

`body`-তে সংজ্ঞায়িত ভেরিয়েবল:

```swift
var num = 10
```

এর লাইফসাইকেল `body`-র এক্সিকিউশন প্রক্রিয়ার সাথে বাঁধা।

যখন SwiftUI-এর স্টেট পরিবর্তিত হয়, উদাহরণস্বরূপ:

```swift
@State private var name = "FangJunyu"
name = "Hu"
```

যখন `name` পরিবর্তিত হয়, @State ডেটা পরিবর্তন শনাক্ত করে, SwiftUI-কে ভিউ পুনরায় ক্যালকুলেট করতে জানায়, `body` পুনরায় ক্যালকুলেট হয়।

`body` পুনরায় ক্যালকুলেট হওয়ার সময়, `body`-র ভেতরের সমস্ত কোড পুনরায় এক্সিকিউট হবে, `body`-র ভেতরের ভেরিয়েবল (যেমন `num`) পুনরায় তৈরি হবে।

এজন্যই `body`-তে জটিল ভেরিয়েবল সংজ্ঞায়িত করার পরামর্শ দেওয়া হয় না।

কারণ, প্রতিবার ভিউ রিফ্রেশ হওয়ার সময়, `body`-র ভেতরের ভেরিয়েবলগুলো পুনরায় তৈরি হবে, এটি ক্যালকুলেশন খরচ বাড়ায় এবং পারফরম্যান্সে প্রভাব ফেলে।

SwiftUI-তে, ভিন্ন ধরনের ডেটা ভিন্ন পদ্ধতিতে পরিচালনা করা উচিত, ভিউর লাইফসাইকেলের সাথে থাকার প্রয়োজন এমন ডেটা @State দিয়ে সংরক্ষণ করা যায়; অস্থায়ী ক্যালকুলেশনের ডেটা `body`-তে রাখা যায়।

## onAppear এবং onDisappear

আগের পাঠে, আমরা ইতোমধ্যে onAppear শিখেছি, যখন ভিউ প্রদর্শিত হয় তখন onAppear কল করা হয়।

```swift
.onAppear {}
```

উদাহরণ:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
        .onAppear {
            print("Show ContentView")
        }
    }
}
```

Xcode প্রিভিউ বা রান করার সময়, onAppear-এর ডিবাগ আউটপুট দেখা যাবে।

### onDisappear

onAppear-এর বিপরীত হলো onDisappear:

```swift
.onDisappear {}
```

যখন ভিউ বন্ধ করা হয়, onDisappear কল করা হয়।

উদাহরণ:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

যখন ভিউ সরিয়ে দেওয়া হয়, onDisappear-এর ভেতরের কোড কল করা হবে।

টিপস: বর্তমান পর্যায়ে ব্যবহৃত ContentView হলো অ্যাপের রুট ভিউ, সরাসরি বন্ধ বা সরানো যায় না। তাই, এই পর্যায়ে onDisappear-এর এক্সিকিউশন দেখা সম্ভব নয়।

পরবর্তীতে পেজ ট্রানজিশন এবং নতুন ভিউ খোলা শেখার সময়, onDisappear কল হতে দেখা যাবে।

## তৈরি এবং প্রদর্শন লজিক

উল্লেখ্য যে, ভিউ তৈরি এবং প্রদর্শন দুটি ভিন্ন পর্যায়।

যখন ভিউ তৈরি হয়, struct-এর কনস্ট্রাক্টর মেথড কল করা হয়:

```swift
init() {}
```

যেহেতু SwiftUI ভিউ হলো struct স্ট্রাকচার:

```swift
struct ContentView: View { ... }
```

তাই, যখন ভিউ ইনস্ট্যান্স তৈরি হয়, `init` কনস্ট্রাক্টর মেথড এক্সিকিউট হয়। যখন ভিউ প্রদর্শিত হয়, তখন `onAppear` মেথড কল করা হয়।

উদাহরণ:

```swift
struct ContentView: View {
    init() {
        print("Create ContentView")
    }
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

Xcode-এ ভিউ প্রিভিউ করলে, ডিবাগ আউটপুট দেখা যাবে:

```swift
Create ContentView
Show ContentView
```

এটি প্রমাণ করে যে, প্রথমে `init` এক্সিকিউট হয়ে ভিউ তৈরি হয়, তারপর `body`-র কোড ক্যালকুলেট হয়, সবশেষে ভিউ প্রদর্শিত হয় এবং `onAppear` মেথড এক্সিকিউট হয়।

তাই, ভিউ তৈরি এবং প্রদর্শন পর্যায় আলাদা এটি বুঝতে হবে।

### init কনস্ট্রাক্টর মেথড

`init` হলো Swift-এর মৌলিক সিনট্যাক্স, struct এবং class উভয়ই সংজ্ঞায়িত করতে পারে, অবজেক্ট ইনিশিয়ালাইজ করার জন্য ব্যবহৃত হয়।

```swift
init() {}
```

SwiftUI-তে, যখন ভিউ তৈরি হয়, সিস্টেম ভিউর `init` মেথড কল করে, প্যারামিটার গ্রহণ বা ডেটা ইনিশিয়ালাইজ করার জন্য ব্যবহার করা যায়।

যদি ম্যানুয়ালি `init` না লেখা হয়, Swift স্বয়ংক্রিয়ভাবে struct-এর জন্য একটি ডিফল্ট কনস্ট্রাক্টর মেথড তৈরি করবে।

ContentView ইত্যাদি SwiftUI ভিউর ক্ষেত্রে, ভিউ তৈরি হওয়ার সময় `init` এক্সিকিউট হয়, ভিউ প্রদর্শিত হওয়ার সময় `onAppear` এক্সিকিউট হয়।

তাই, `init` হলো ভিউ তৈরি হওয়ার সময় এক্সিকিউট হওয়া কনস্ট্রাক্টর মেথড, পরবর্তীতে প্যারামিটার পাস বা ডেটা ইনিশিয়ালাইজ করার সময় ব্যবহার করা হবে।

## সারসংক্ষেপ

এই পাঠে আমরা অ্যাপ চালু হওয়ার ক্রম শিখেছি, এন্ট্রি ফাইল থেকে ContentView ফাইল পর্যন্ত এক্সিকিউশন প্রবাহ।

আমরা SwiftUI ভিউর লাইফসাইকেল বুঝেছি: ভিউ তৈরি হওয়ার সময় `init` এক্সিকিউট হয়, স্ক্রিনে প্রদর্শিত হওয়ার সময় `onAppear` এক্সিকিউট হয়, ভিউ সরিয়ে বা বন্ধ করার সময় `onDisappear` এক্সিকিউট হয়।

ভিউ আপডেট মেকানিজম শিখেছি: ভিউ স্টেট দ্বারা চালিত হয়, যখন @State ইত্যাদি স্টেট পরিবর্তিত হয়, SwiftUI ভিউ রিফ্রেশ করে, `body` পুনরায় ক্যালকুলেট হয়, `body`-র ভেতরের ভেরিয়েবলগুলোও পুনরায় তৈরি হয়।

ভেরিয়েবলের লাইফসাইকেল ভিউর লাইফসাইকেলের সাথে বাঁধা, এবং `body`-র ভেতরের অস্থায়ী ভেরিয়েবল প্রতিবার রিফ্রেশের সময় পুনরায় তৈরি হয়।

এই লাইফসাইকেল এবং ভেরিয়েবলের আচরণ বোঝা আমাদের কোড আরও ভালোভাবে সংগঠিত করতে এবং অ্যাপ লজিক আরও স্পষ্ট করতে সাহায্য করবে।
