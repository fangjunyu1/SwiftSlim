# সঞ্চয় বাক্সের পরিমাণ সংরক্ষণ করা

আগের পাঠে, আমরা একটি সহজ "সঞ্চয় বাক্স" তৈরি করেছিলাম। এই পাঠে, আমরা শিখব কীভাবে "সঞ্চয় বাক্স"-এর পরিমাণ স্থায়ীভাবে সংরক্ষণ করা যায়।

এই পাঠে মূলত UserDefaults এবং @AppStorage প্রপার্টি র‍্যাপার শেখা হবে, সেই সাথে onAppear-এর কাজ এবং অপশনাল টাইপ (nil)-এর ব্যবহার পদ্ধতিও বিস্তারিত আলোচনা করা হবে।

উল্লেখ্য যে, এই পাঠে আগের পাঠে তৈরি করা "সঞ্চয় বাক্স" কোড ব্যবহার করতে হবে।

## স্থায়ী সংরক্ষণের সমস্যা

এখন, "সঞ্চয় বাক্স"-এর ইন্টারফেস খুবই সংক্ষিপ্ত এবং ব্যবহার করাও খুব সহজ।

![Piggy Bank](../../Resource/014_view.png)

কিন্তু, একটি স্পষ্ট সমস্যা রয়েছে: প্রতিবার ভিউ রিফ্রেশ করলে বা অ্যাপ বন্ধ করলে, "সঞ্চয় বাক্স"-এর মোট পরিমাণ সবসময় ০ হয়ে যায়, এবং আগের সমস্ত মোট পরিমাণও হারিয়ে যায়।

এর মানে হলো ডেটা আসলে সংরক্ষিত হচ্ছে না, অন্যভাবে বলতে গেলে, বর্তমান "সঞ্চয় বাক্স" শুধুমাত্র অস্থায়ী ডেটা সংরক্ষণ করতে পারে।

### কেন স্থায়ীভাবে সংরক্ষণ করা যাচ্ছে না?

এর কারণ হলো আমরা @State দিয়ে ভেরিয়েবল ঘোষণা করেছি:

```swift
@State private var amount = 0
```

@State দিয়ে ঘোষিত ভেরিয়েবলের জীবনকাল সম্পূর্ণভাবে ভিউর উপর নির্ভরশীল।

যখন ভিউ তৈরি হয়, `amount` ০-তে ইনিশিয়ালাইজ হয়, ভিউ ধ্বংস হলে, `amount`-ও ধ্বংস হয়।

এখানে `amount`-এ সংরক্ষিত ডেটা শুধুমাত্র মেমরিতে থাকে, ডিভাইসে সংরক্ষিত হয় না।

আমরা চাই না `amount` ভিউর জীবনকালের সাথে বাঁধা থাকুক, তাই ডেটা স্থায়ীভাবে সংরক্ষণ করতে হবে। অর্থাৎ, ডেটা ডিভাইসে সংরক্ষণ করতে হবে।

### "ডেটা পার্সিসটেন্স" কী?

ডেটা পার্সিসটেন্স বলতে বোঝায় ডেটাকে "অস্থায়ী মেমরি" থেকে "ডিভাইস স্টোরেজ"-এ সংরক্ষণ করা।

ভিউ বন্ধ করলে বা অ্যাপ বন্ধ করলেও ডেটা হারিয়ে যাবে না।

Swift ডেভেলপমেন্টে, সহজ "ডেটা পার্সিসটেন্স" UserDefaults দিয়ে সংরক্ষণ করা যায়, এবং জটিল ডেটার জন্য SwiftData বা CoreData ব্যবহার করতে হয়।

এই পাঠে, আমরা প্রথমে সবচেয়ে সহজ UserDefaults শিখব।

## UserDefaults

UserDefaults হালকা কী-ভ্যালু পেয়ার ডেটা সংরক্ষণের জন্য ব্যবহৃত হয়, সাধারণত String, Int, Double, Bool ইত্যাদি মৌলিক টাইপ সংরক্ষণে ব্যবহার করা হয়।

### ডেটা সংরক্ষণ করা

UserDefaults ডেটা সংরক্ষণের জন্য `set` মেথড ব্যবহার করে:

```swift
UserDefaults.standard.set("FangJunyu", forKey: "name")
UserDefaults.standard.set(28, forKey: "age")
UserDefaults.standard.set(true, forKey: "isMember")
```

UserDefaults-এর প্রথম প্যারামিটার সংরক্ষিত ডেটা বোঝায়, এখানে String, Int ইত্যাদি টাইপের ডেটা পাস করা হয়।

দ্বিতীয় প্যারামিটার `forKey` ভেরিয়েবলের নামের মতো, UserDefaults-এ সংরক্ষিত ডেটা শনাক্ত করতে ব্যবহৃত হয়।

### ডেটা পড়া

UserDefaults সংশ্লিষ্ট মেথড ব্যবহার করে পড়া হয়:

```swift
let name = UserDefaults.standard.string(forKey: "name")    // FangJunyu
let age = UserDefaults.standard.integer(forKey: "age")    // 28
let isMember = UserDefaults.standard.bool(forKey: "isMember")    // true
```

পড়ার সময় অবশ্যই সংশ্লিষ্ট টাইপের মেথড ব্যবহার করতে হবে, অন্যথায় ত্রুটি বা ভুল ডেটা পাওয়া যাবে।

### UserDefaults সতর্কতা

#### ১. অপশনাল টাইপ রিটার্ন করা

UserDefaults দিয়ে ডেটা পড়ার সময়, কিছু মেথড অপশনাল টাইপ (Optional) রিটার্ন করে।

উদাহরণ:

```swift
let name = UserDefaults.standard.string(forKey: "name")
```

রিটার্ন হওয়া `name`-এর টাইপ হবে:

```swift
String?
```

এর মানে হলো মান থাকতেও পারে, নাও থাকতে পারে।

**কেন অপশনাল টাইপ রিটার্ন করে?**

এর কারণ হলো, UserDefaults-এ কোনো Key-এর সাথে সংশ্লিষ্ট ডেটা নাও থাকতে পারে।

উদাহরণ:

```swift
let name = UserDefaults.standard.string(forKey: "City") // nil
```

যদি আগে কখনো "City" Key সংরক্ষণ না হয়ে থাকে, তাহলে পড়ার ফলাফল হবে:

```swift
nil
```

জোর দিয়ে বলা দরকার, nil মানে ডেটা নেই, খালি স্ট্রিং নয়।

"টাইপ সিস্টেম" অধ্যায়ে আমরা একই ধরনের সমস্যার মুখোমুখি হয়েছিলাম, যখন ডেটা টাইপ কনভার্সন ব্যর্থ হয় তখন অপশনাল টাইপ রিটার্ন হয়।

এক্ষেত্রে, আমাদের `??` ব্যবহার করে ডিফল্ট মান প্রদান করতে হবে এবং আনর্যাপ করতে হবে:

```swift
let name = UserDefaults.standard.string(forKey: "City") ?? "Rizhao"
```

যদি UserDefaults ডেটা পড়তে পারে, পড়া মান ব্যবহার করা হবে; যদি পড়তে না পারে (nil), ডিফল্ট মান ব্যবহার করা হবে।

#### ২. নন-অপশনাল টাইপ রিটার্ন করা

UserDefaults Int, Double, Bool টাইপ পড়ার সময় নন-অপশনাল টাইপ রিটার্ন করে।

উদাহরণ:

```swift
let num = UserDefaults.standard.integer(forKey: "num")    // 0
let height = UserDefaults.standard.double(forKey: "height")    // 0.0
let isBuyCar = UserDefaults.standard.bool(forKey: "isBuyCar")    // false
```

এমনকি যদি এই Key-গুলোতে কখনো ডেটা সংরক্ষিত না হয়ে থাকে, রিটার্ন মান nil হবে না।

বরং ডিফল্ট মান রিটার্ন হবে:

```swift
Int → 0
Double → 0.0
Bool → false
```

#### ৩. ডেটার পরিমাণ সীমাবদ্ধতা

UserDefaults শুধুমাত্র অল্প পরিমাণ ডেটা সংরক্ষণের জন্য উপযুক্ত, যেমন অ্যাপ কনফিগারেশন, সহজ স্টেটাস ইত্যাদি ডেটা।

বাস্তবে, বেশি পরিমাণ ডেটা বা ঘন ঘন পড়া-লেখার ডেটা সংরক্ষণের জন্য উপযুক্ত নয়।

## "সঞ্চয় বাক্স"-এর পরিমাণ সংরক্ষণ করা

এখন, আমরা UserDefaults ব্যবহার করে "সঞ্চয় বাক্স"-এর মোট পরিমাণ সংরক্ষণের কোড লজিক বাস্তবায়ন করতে পারি।

যখন ব্যবহারকারী বাটন ক্লিক করেন, ইনপুট করা পরিমাণ মোট পরিমাণে যোগ হয়, এবং মোট পরিমাণ UserDefaults-এ সংরক্ষণ করা হয়।

```swift
Button("Save") {
    amount += number
    number = 0
    UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
}
.buttonStyle(.borderedProminent)
```

এখন, আমরা ডেটা লেখার লজিক বাস্তবায়ন করেছি।

## ডেটা পার্সিসটেন্স লজিক ব্যাখ্যা

যদিও আমরা ডেটা পার্সিসটেন্সের লেখার লজিক সম্পন্ন করেছি, কিন্তু বাস্তবে চালানোর সময় দেখা যাবে: প্রতিবার ভিউ রিফ্রেশ করলে বা অ্যাপ পুনরায় খুললে, `amount` ভেরিয়েবল এখনও ০।

এর কারণ হলো:

```swift
@State private var amount = 0
```

SwiftUI-তে, @State ভেরিয়েবল ভিউ তৈরি হওয়ার সময় পুনরায় ইনিশিয়ালাইজ হয়।

যদিও UserDefaults-এ `amount`-এর মান সংরক্ষিত আছে, ভিউ পুনরায় লোড হওয়ার সময় `amount` আবার ০-তে রিসেট হয়।

এর মানে হলো, এখন পর্যন্ত শুধু UserDefaults-এ ডেটা সংরক্ষণ করা হয়েছে, কিন্তু ভিউতে ডেটা পুনরুদ্ধার করা হয়নি।

তাই, ভিউ লোড হওয়ার সময়, UserDefaults থেকে ম্যানুয়ালি ডেটা পড়ে `amount`-এ পুনরায় অ্যাসাইন করতে হবে, তবেই সম্পূর্ণ ডেটা পার্সিসটেন্সের কোড লজিক বাস্তবায়িত হবে।

### লজিক উপমা

এই প্রক্রিয়াটিকে "শ্রেণিকক্ষের ব্ল্যাকবোর্ড" হিসেবে বোঝা যায়:

ক্লাস চলাকালীন, শিক্ষক ব্ল্যাকবোর্ডে বিষয়বস্তু লেখেন, এই বিষয়বস্তুগুলো বর্তমান ভিউতে থাকা স্টেটের (@State) মতো।

ক্লাস শেষ হওয়ার পর, পরবর্তী ক্লাসের জন্য ব্ল্যাকবোর্ড মুছে ফেলা হয়। এটি ভিউ ধ্বংস হওয়ার সমতুল্য, @State-এর ডেটাও মুছে যায়।

শিক্ষণীয় বিষয়বস্তু হারিয়ে না যাওয়ার জন্য, শিক্ষক আগেই বিষয়বস্তু কোর্স ম্যাটেরিয়ালে সংরক্ষণ করেন। এই কোর্স ম্যাটেরিয়াল UserDefaults-এ সংরক্ষিত ডেটার মতো।

পরের বার ক্লাস শুরু হলে, ব্ল্যাকবোর্ড প্রথমে ফাঁকা থাকে (@State ইনিশিয়ালাইজ), শিক্ষককে কোর্স ম্যাটেরিয়াল অনুযায়ী আগের জ্ঞান আবার ব্ল্যাকবোর্ডে লিখতে হয় (UserDefaults পড়া)।

বোঝা দরকার যে, ব্ল্যাকবোর্ড স্বয়ংক্রিয়ভাবে আগের বিষয়বস্তু পুনরুদ্ধার করে না, "কোর্স ম্যাটেরিয়াল পড়ে" (UserDefaults পড়ে) পুনরায় লিখতে হবে।

## onAppear ব্যবহার করে ডেটা পড়া

ভিউ প্রদর্শিত হওয়ার সময়, UserDefaults থেকে মান ম্যানুয়ালি পড়তে হবে এবং @State-এর `amount` ভেরিয়েবলে অ্যাসাইন করতে হবে, যাতে সত্যিকারের ডেটা পার্সিসটেন্স বাস্তবায়িত হয়।

SwiftUI-তে, ভিউ প্রদর্শিত হওয়ার সময় ইনিশিয়ালাইজেশন লজিক চালানোর জন্য `onAppear` ব্যবহার করা যায়:

```swift
.onAppear {}
```

UserDefaults পড়ার কোড `onAppear`-এ রাখুন:

```swift
.onAppear {
    amount = UserDefaults.standard.integer(forKey: "amount")
}
```

যখন ভিউ তৈরি হয়, `amount` ০-তে ইনিশিয়ালাইজ হয়; যখন ভিউ প্রদর্শিত হয়, `onAppear` ট্রিগার হয়, UserDefaults থেকে পড়ে `amount`-এ অ্যাসাইন করা হয়। এভাবে, ভিউ আগের বার সংরক্ষিত মোট পরিমাণ প্রদর্শন করতে পারবে।

এখন, সম্পূর্ণ "সঞ্চয় বাক্স"-এর ডেটা পার্সিসটেন্স লজিক সম্পন্ন হলো।

## সম্পূর্ণ কোড

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
                UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
        .onAppear {
            amount = UserDefaults.standard.integer(forKey: "amount")
        }
    }
}
```

## @AppStorage প্রপার্টি র‍্যাপার

SwiftUI @AppStorage প্রপার্টি র‍্যাপার প্রদান করে, যা UserDefaults-এর সংরক্ষণ লজিক সহজ করতে পারে।

```swift
@AppStorage("amount") private var amount = 0
```

@AppStorage হলো UserDefaults-এর এনক্যাপসুলেশন, "amount" UserDefaults-এর Key-এর সাথে সংশ্লিষ্ট।

যখন ভিউ তৈরি হয়, UserDefaults থেকে ডেটা পড়া হবে; যখন ভেরিয়েবল পরিবর্তিত হয়, স্বয়ংক্রিয়ভাবে UserDefaults-এ লেখা হবে।

SwiftUI-এর স্টেট আপডেট মেকানিজম সমর্থন করে, @State-এর মতো, পরিবর্তনের সময় SwiftUI ভিউ রিফ্রেশ হবে।

অর্থাৎ, @AppStorage ডেটা পড়া এবং সংরক্ষণের লজিক বাস্তবায়ন করতে পারে, আর ম্যানুয়ালি UserDefaults কল করার বা `onAppear` দিয়ে ম্যানুয়ালি ডেটা পড়ার প্রয়োজন নেই।

তাই, আমরা কোডে UserDefaults-এর পরিবর্তে @AppStorage ব্যবহার করতে পারি:

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

## সারসংক্ষেপ

এই পাঠে মূলত নিম্নলিখিত জ্ঞানবিন্দুগুলো শেখা হয়েছে:

প্রথমত, UserDefaults হালকা ডেটা সংরক্ষণের জন্য ব্যবহৃত হয়, আমরা UserDefaults ব্যবহার করে মোট পরিমাণ ডিভাইসে সংরক্ষণ করেছি। UserDefaults পড়া ও লেখার মেথড প্রদান করে, এবং অপশনাল টাইপ ইত্যাদি পরিস্থিতিও সামলাতে হয়।

দ্বিতীয়ত, @State ভেরিয়েবল শুধুমাত্র ভিউতে অস্থায়ী স্টেট পরিচালনার জন্য, এর জীবনকাল ভিউর সাথে সংগতিপূর্ণ। ভিউ পুনরায় তৈরি হলে, @State পুনরায় ইনিশিয়ালাইজ হয়।

UserDefaults ডেটা পুনরুদ্ধারের সময়, `onAppear` ব্যবহার করে ভিউ প্রদর্শনের সময় UserDefaults পড়া এবং ডেটা পুনরুদ্ধার সম্পন্ন করতে শেখা হয়েছে।

সবশেষে, UserDefaults-এর পরিবর্তে @AppStorage ব্যবহার করা হয়েছে, যাতে ভিউ লোড হওয়ার সময় স্বয়ংক্রিয়ভাবে ডেটা পড়া যায় এবং ডেটা পরিবর্তনের সময় স্বয়ংক্রিয়ভাবে লেখা যায়, ফলে ম্যানুয়ালি UserDefaults এবং onAppear কল করার লজিক প্রতিস্থাপিত হয়।

এই জ্ঞানের মাধ্যমে, আমরা একটি মৌলিক অ্যাপ ডেটা স্টোরেজ ফিচার বাস্তবায়ন করতে পারি, যাতে ডেটা স্থায়ীভাবে সংরক্ষিত হতে পারে।

## পাঠ-পরবর্তী অনুশীলন

এখন, "সঞ্চয় বাক্স" পরিমাণ সংরক্ষণ সমর্থন করে, কিন্তু ফিচার এখনও অসম্পূর্ণ।

১. যখন ব্যবহারকারীকে পরিমাণ তুলতে হবে, তখন সংশ্লিষ্ট উত্তোলন লজিক ডিজাইন করতে হবে।
২. যখন ব্যবহারকারী পরিমাণ মুছে ফেলতে চাইবেন, তখন রিসেট লজিকও বাস্তবায়ন করতে হবে।
৩. যদি সঞ্চয় বাক্সে নাম যোগ করতে চান, তাহলে String টাইপের ডেটা সংরক্ষণও চালু করতে হবে।

বিদ্যমান ভিত্তিতে এই ফিচারগুলো সম্পন্ন করা যায়, যাতে "সঞ্চয় বাক্স"-এর ফিচার আরও সম্পূর্ণ হয়।

## বিস্তারিত জ্ঞান - ইনপুট বক্সের ডিফল্ট মানের সমস্যা

এখন, আমরা ইনপুট বক্সের ইন্টারেকশন আরও অপ্টিমাইজ করতে পারি, ইনপুট অপারেশনকে আরও সুবিধাজনক করতে।

![textfield](../../Resource/015_view.png)

বর্তমান বাস্তবায়নে, ইনপুট বক্সের ডিফল্ট মান ০। প্রতিবার পরিমাণ ইনপুট করার সময়, ব্যবহারকারীকে প্রথমে ০ মুছতে হয়, তারপর নতুন মান ইনপুট করতে হয়, এই ইন্টারেকশন অভিজ্ঞতা যুক্তিসঙ্গত নয়।

আমাদের ইনপুট বক্সের প্রাথমিক মান খালি হওয়া উচিত, ০ নয়।

এর কারণ হলো, TextField এবং `number` দ্বি-মুখী বাইন্ডিং করা:

```swift
@State private var number = 0
TextField("Amount", value: $number, format: .number)
```

যখন `number` ০-তে ইনিশিয়ালাইজ হয়, ইনপুট বক্সেও ০ প্রদর্শিত হবে।

তাই, আমাদের `number`-কে খালি মানে পরিবর্তন করতে হবে, অর্থাৎ `nil`।

```swift
@State private var number = nil
```

এই সময় একটি ত্রুটি দেখা দেবে:

```
Generic parameter 'Value' could not be inferred
```

এটি "টাইপ সিস্টেম" অধ্যায়ে আলোচিত পরিস্থিতি: যখন ভেরিয়েবল স্বয়ংক্রিয়ভাবে টাইপ অনুমান করতে না পারে, তখন স্পষ্টভাবে ভেরিয়েবলের টাইপ ঘোষণা করতে হয়।

যেহেতু আমরা `number`-কে `nil` করেছি, এবং `nil` নিজে কোনো টাইপ তথ্য ধারণ করে না, কম্পাইলার নির্ধারণ করতে পারে না যে এই ভেরিয়েবলটি String, Int নাকি অন্য কোনো টাইপ, তাই ত্রুটি দেখা দেয়।

তাই, আমাদের `number`-এর জন্য স্পষ্টভাবে টাইপ ঘোষণা করতে হবে:

```swift
@State private var number:Int? = nil
```

এখানে `Int?` অপশনাল টাইপ (Optional) বোঝায়, মানে `number`-এর মান Int হতে পারে, অথবা nil হতে পারে।

এভাবে, ইনিশিয়ালাইজেশনের সময় এটি nil করা হলে, ইনপুট বক্সে ০-এর পরিবর্তে খালি প্রদর্শিত হবে।

### অপশনাল টাইপ আনর্যাপ করা

যখন আমরা `number`-কে অপশনাল টাইপে পরিবর্তন করি, আগের ক্যালকুলেশন কোডে ত্রুটি দেখা দেবে:

```swift
amount += number
```

প্রম্পট:

```swift
Value of optional type 'Int?' must be unwrapped to a value of type 'Int'
```

এর কারণ হলো, `number` এখন nil হতে পারে। যদি সরাসরি ক্যালকুলেশনে অংশ নেয়, তাহলে হবে:

```swift
amount += nil
```

স্পষ্টতই, nil `amount`-এর সাথে যোগ করা যায় না, এটি একটি অবৈধ অপারেশন।

তাই, অপশনাল টাইপ ক্যালকুলেশনে ব্যবহার করার আগে, আনর্যাপ করতে হবে।

এখানে `??` ব্যবহার করে ডিফল্ট মান প্রদান করা যায়:

```swift
amount += number ?? 0
```

যখন `number` nil হয়, ডিফল্ট মান ০ ব্যবহার হবে; যখন `number`-এ মান থাকে, ইনপুট করা প্রকৃত ডেটা ব্যবহার হবে।

এভাবে ক্যালকুলেশনের সঠিকতা নিশ্চিত হয়, এবং ইনপুট বক্সের ডিফল্ট মান খালি হওয়ার প্রভাবও বাস্তবায়িত হয়, ব্যবহারকারীকে আর ম্যানুয়ালি ইনপুট বক্সের ০ মুছতে হবে না।
