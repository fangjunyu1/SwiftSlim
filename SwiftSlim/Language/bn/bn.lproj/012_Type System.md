# টাইপ সিস্টেম

আগের ক্লাসগুলোতে, আমরা ভেরিয়েবল সম্পর্কে শিখেছি এবং জেনেছি যে ভেরিয়েবল বিভিন্ন ধরণের ডেটা ধারণ করতে পারে।

উদাহরণস্বরূপ:

```swift
var hello = "Hello, World"
```

এখানে hello একটি টেক্সট ধারণ করছে, তাই এর টাইপ হলো String।

এরপর, আমরা টাইপ (Type) এর ধারণা, এক্সপ্লিসিট টাইপ ডিক্লেয়ারেশন এবং টাইপ ইনফারেন্স সম্পর্কে ধারাবাহিকভাবে শিখব, যাতে ভেরিয়েবলের মূল প্রকৃতি আরও গভীরভাবে বোঝা যায়।

## টাইপ কী

Swift-এ, প্রতিটি ভ্যালুর একটি নির্দিষ্ট টাইপ থাকে। টাইপ নির্ধারণ করে ভ্যালুটি কী এবং এটি কী করতে পারে।

উদাহরণস্বরূপ:

- 42 হলো একটি Int (পূর্ণসংখ্যা)
- "Hello" হলো String (স্ট্রিং/টেক্সট)
- true হলো Bool (বুলিয়ান ভ্যালু)

এগুলো ভিন্ন ভিন্ন টাইপের অন্তর্ভুক্ত এবং এদের ব্যবহারও আলাদা।

## Swift-এর সাধারণ বেসিক টাইপ

প্রাথমিক পর্যায়ে, সবচেয়ে সাধারণ টাইপগুলো নিচে দেওয়া হলো:

- String: স্ট্রিং (টেক্সট)
- Int: পূর্ণসংখ্যা
- Double: ফ্লোটিং-পয়েন্ট নম্বর (দশমিক যুক্ত, উচ্চ নির্ভুলতা সম্পন্ন)
- Bool: বুলিয়ান ভ্যালু (true/false)

এছাড়া, Float (ফ্লোটিং-পয়েন্ট নম্বর) আছে, তবে এর নির্ভুলতা Double-এর চেয়ে কম, যা সাধারণত কম ব্যবহৃত হয়; এবং Character (সিঙ্গেল ক্যারেক্টার) রয়েছে, যেমন "A", "!" ইত্যাদি একক অক্ষর।

### টাইপ ডিক্লেয়ারেশন (এক্সপ্লিসিট টাইপ)

Swift-এ, ভেরিয়েবল ঘোষণার সময় আপনি ম্যানুয়ালি টাইপ নির্দিষ্ট করতে পারেন:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

ভেরিয়েবল নামের পর ": টাইপ" যোগ করে ভেরিয়েবলের টাইপ ঘোষণা করা হয়।

অ্যারের (Array) টাইপ লেখার নিয়ম হলো ": [টাইপ]":

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

এই কোডে, scores অ্যারে শুধুমাত্র Int টাইপের উপাদান ধারণ করতে পারে এবং tags শুধুমাত্র String টাইপের উপাদান ধারণ করতে পারে।

এক্সপ্লিসিট টাইপ ডিক্লেয়ারেশন কোডের উদ্দেশ্যকে আরও পরিষ্কার করে এবং কিছু ক্ষেত্রে টাইপ ইনফারেন্স ত্রুটি এড়াতে সাহায্য করে।

## টাইপ ইনফারেন্স (Type Inference)

অনেক ক্ষেত্রেই, টাইপ এক্সপ্লিসিটভাবে ঘোষণা করার প্রয়োজন হয় না:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"    // String
var isLogin = false  // Bool
var scores= [90, 85, 78]	// [Int]
var tags = ["Swift", "iOS"]	// [String]
```

এর কারণ হলো, Swift কম্পাইলার "প্রাথমিক ভ্যালু" এর উপর ভিত্তি করে স্বয়ংক্রিয়ভাবে টাইপ অনুমান বা ইনফার (infer) করতে পারে।

## এক্সপ্লিসিট ডিক্লেয়ারেশন এবং টাইপ ইনফারেন্স

আগের ক্লাসগুলোতে, আমরা এক্সপ্লিসিট ডিক্লেয়ারেশন ব্যবহার করিনি, যেমন:

```swift
var age: Int = 18
```

বরং আমরা টাইপ ইনফারেন্স ব্যবহারে অগ্রাধিকার দিয়েছি:

```swift
var age = 18
```

এই উদাহরণে, উভয় লেখার ধরনই সমতুল্য, age-কে Int টাইপ হিসেবেই নির্ধারণ করা হবে। তুলনামূলকভাবে, টাইপ ইনফারেন্সের লেখার ধরন অনেক বেশি সংক্ষিপ্ত।

শুরুতে এক্সপ্লিসিট ডিক্লেয়ারেশনের উপর জোর না দেওয়ার কারণ হলো, টাইপ ইনফারেন্স কোডের অতিরিক্ত তথ্য কমিয়ে দেয়, যা প্রাথমিক পর্যায়ে শেখার ক্ষেত্রে বোঝার কষ্ট কমায়।

## টাইপ কেন প্রয়োজন

Swift একটি স্ট্রংলি টাইপড ল্যাঙ্গুয়েজ (Strongly Typed Language)।

এর মানে হলো: একবার কোনো ভেরিয়েবলের টাইপ নির্ধারিত হয়ে গেলে, তা ইচ্ছামতো পরিবর্তন করা যায় না।

উদাহরণস্বরূপ:

```swift
var age: Int = 18

age = 20	// Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

প্রথম 20 হলো Int টাইপ, যা age-এ অ্যাসাইন করা যেতে পারে; দ্বিতীয় "20" হলো String টাইপ, যা Int টাইপের সাথে অসামঞ্জস্যপূর্ণ, তাই এটি এরর (error) দেখাবে।

nums হলো [Int] টাইপের অ্যারে, এটি শুধুমাত্র পূর্ণসংখ্যা সংরক্ষণ করতে পারে, স্ট্রিং মেশানো যাবে না।

টাইপ ডেটার ব্যবহারকে সীমাবদ্ধ করতে পারে এবং কম্পাইল পর্যায়ে টাইপ অমিল জনিত সমস্যাগুলি ধরতে পারে, যার ফলে টাইপ সংক্রান্ত বিভ্রান্তি থেকে সৃষ্ট বাগ (BUG) হ্রাস পায় এবং কোডের স্থিতিশীলতা ও রক্ষণাবেক্ষণযোগ্যতা বৃদ্ধি পায়।

## কখন অবশ্যই এক্সপ্লিসিট টাইপ ডিক্লেয়ার করতে হবে

যদিও Swift স্বয়ংক্রিয়ভাবে টাইপ অনুমান করতে পারে, তবে কিছু পরিস্থিতিতে কম্পাইলার অনুমান করতে পারে না, বা ভুল অনুমান করতে পারে, তখন ম্যানুয়ালি ডিক্লেয়ার করা প্রয়োজন।

**১. এম্পটি অ্যারে (খালি অ্যারে)**

একটি খালি অ্যারে তৈরি করার সময়, অবশ্যই এক্সপ্লিসিট টাইপ ডিক্লেয়ার করতে হবে:

```swift
var nums: [Int] = []
```

যদি টাইপ না লেখেন:

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

খালি অ্যারে থেকে কম্পাইলার উপাদানগুলোর টাইপ অনুমান করতে পারে না, তাই কম্পাইল এরর তৈরি হবে।

**২. ইনফারেন্স এরর এড়ানো**

```swift
var value = 10   // Int
```

এই উদাহরণে, যদিও 10 ফ্লোটিং-পয়েন্ট নম্বর (Double) হিসেবেও প্রকাশ করা যেতে পারে, তবে দশমিক বিন্দু না থাকায়, কম্পাইলার ডিফল্টভাবে একে Int টাইপ হিসেবে অনুমান করবে।

যদি আপনি চান value ফ্লোটিং-পয়েন্ট নম্বর প্রকাশ করুক, তবে এক্সপ্লিসিট টাইপ ডিক্লেয়ার করতে হবে:

```swift
var value: Double = 10   // Double
```

যখন টাইপ ইনফারেন্সের ফলাফল প্রত্যাশা অনুযায়ী হয় না, তখন এক্সপ্লিসিট টাইপ ডিক্লেয়ারেশনের মাধ্যমে বা লিটারেল ফর্ম সমন্বয় করে টাইপ সঠিক আছে কিনা তা নিশ্চিত করতে হবে।

**৩. কাস্টম টাইপ**

আমরা পরবর্তীতে কাস্টম টাইপ সম্পর্কেও শিখব। কাস্টম টাইপ ব্যবহার করার সময়ও সাধারণত এক্সপ্লিসিট ডিক্লেয়ারেশনের প্রয়োজন হয়:

```swift
var info: GameInfo
var users: [User] = []
```

এটি খালি অ্যারের কারণের মতই, কম্পাইলার খালি ভ্যালু থেকে স্বয়ংক্রিয়ভাবে টাইপ অনুমান করতে পারে না।

## টাইপ কনভার্সন (টাইপ রূপান্তর)

Swift-এ, বিভিন্ন টাইপের মধ্যে স্বয়ংক্রিয়ভাবে রূপান্তর হয় না, অবশ্যই ম্যানুয়ালি রূপান্তর করতে হবে।

উদাহরণস্বরূপ, দুটি ভিন্ন টাইপের সংখ্যার যোগফল:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b	// Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

এই কোডে, a হলো Int টাইপ এবং b হলো Double টাইপ। টাইপ ভিন্ন হওয়ায়, সরাসরি কোনো গাণিতিক অপারেশন করা যাবে না।

অপারেশন করার আগে, এদের মধ্যে যেকোনো একটিকে একই টাইপে রূপান্তর করতে হবে:

```swift
var result = Double(a) + b
```

এই কোডটি বোঝায় যে, Int টাইপ a-কে Double টাইপে রূপান্তর করে, তারপর b-এর সাথে যোগ করা হচ্ছে এবং হিসাব করা ফলাফল result-এ অ্যাসাইন করা হচ্ছে।

মনে রাখবেন যে, টাইপ কনভার্সন মূল ভেরিয়েবলের টাইপ পরিবর্তন করে না। উদাহরণস্বরূপ, Double(a) a-এর Int টাইপকে পরিবর্তন করবে না, বরং অপারেশনে অংশ নেওয়ার জন্য একটি নতুন Double ভ্যালু তৈরি করবে।

অন্যান্য টাইপগুলোও রূপান্তরের জন্য অনুরূপ পদ্ধতি ব্যবহার করতে পারে, অর্থাৎ টার্গেট টাইপ + () ফর্ম্যাট ব্যবহার করে রূপান্তর করতে চাওয়া ভ্যালুটি এর ভেতরে পাস করতে হয়। উদাহরণস্বরূপ:

```
Int()、Double()、String()
```

উদাহরণ:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

মনে রাখবেন যে, সব টাইপ সফলভাবে রূপান্তর করা যায় না। যেমন Double → Int করার সময়, এটি দশমিক অংশটি সরাসরি কেটে ফেলবে এবং রাউন্ডিং (round off) করবে না।

## টাইপ এলিয়াস (Type Alias)

Swift-এ, কোডের অর্থকে আরও পরিষ্কার করার জন্য আপনি বিদ্যমান কোনো টাইপের একটি ভিন্ন নাম বা এলিয়াস (alias) দিতে পারেন:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

এখানকার UserId মূলত Int-ই। টাইপ পরিবর্তন হয়নি, শুধুমাত্র নামটির অর্থ আরও পরিষ্কার হয়েছে।

বিশাল কোডবেস এবং জটিল টাইপ সম্পন্ন প্রোজেক্টে, টাইপ এলিয়াস কোডকে আরও পাঠযোগ্য করে তোলে।

## টাইপের উদাহরণ

নিচে একটি সাধারণ উদাহরণের মাধ্যমে বাস্তব ডেভেলপমেন্টে টাইপের ভূমিকা দেখানো হলো।

### ভেরিয়েবল তৈরি করা

প্রথমে, আমরা একটি age ভেরিয়েবল তৈরি করি:

```swift
var age = 20
```

যেহেতু আমাদের এই ভেরিয়েবলটি SwiftUI-তে দেখানো এবং পরিবর্তন করা প্রয়োজন, তাই একে ঘোষণা করার জন্য @State ব্যবহার করতে হবে:

```swift
@State private var age = 20
```

private-এর কথা আগেই বলা হয়েছে, এর মানে হলো এই ভেরিয়েবলটি শুধুমাত্র বর্তমান ভিউতেই ব্যবহার করা যাবে, যা নিয়ে ভবিষ্যতে আরও শেখা হবে।

### SwiftUI-তে ডেটা প্রদর্শন

SwiftUI-তে age ভেরিয়েবলটি প্রদর্শন করা:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

এখন ভিউতে দেখাবে:

```swift
age: 20
```

### ডেটা পরিবর্তন

এরপর, age পরিবর্তন করার জন্য আমরা দুটি বাটন যোগ করব:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### রান করার ফলাফল (Execution Result)

যখন changeAge1-এ ক্লিক করা হবে, age-এ 18 অ্যাসাইন হবে:

```swift
age = 18
```

যেহেতু 18 হলো Int টাইপ, যা age-এর টাইপের সাথে মিলে যায়, তাই সফলভাবে অ্যাসাইন হবে।

changeAge2-তে ক্লিক করলে এরর আসবে:

```swift
age = "18"
```

এরর মেসেজ:

```
Cannot assign value of type 'String' to type 'Int'
```

এর কারণ হলো "18" একটি String টাইপ (স্ট্রিংকে "" দিয়ে আবৃত করতে হয়), এবং age একটি Int টাইপ। টাইপ অসামঞ্জস্যপূর্ণ হওয়ায় এটি অ্যাসাইন করা যাবে না।

### কেন এরর দেখাবে?

মূল বিষয় হলো এই লাইনের কোডটি:

```swift
@State private var age = 20   // Int
```

যদিও এক্সপ্লিসিট টাইপ ডিক্লেয়ার করা হয়নি, তবে প্রাথমিক ভ্যালু 20 হলো Int টাইপ, তাই কম্পাইলার স্বয়ংক্রিয়ভাবে age-কে Int টাইপ হিসেবে অনুমান করবে।

একবার টাইপ নির্ধারিত হয়ে গেলে, এটিকে অন্য কোনো টাইপে অ্যাসাইন করা যাবে না।

তাছাড়া, changeAge2 বাটনটি "রানটাইম"-এ ফেইল করবে না, বরং এটি কম্পাইল পর্যায়েই এরর দেখাবে এবং কোডটি কম্পাইল হবে না।

এটি Swift-এর স্ট্রং টাইপের অন্যতম সুবিধা: কোড লেখার সময়ই আপনি অগ্রিম টাইপ সংক্রান্ত সমস্যাগুলো খুঁজে পেতে পারেন।

### সঠিক লেখার নিয়ম

ভুল কোড মুছে ফেলার পর:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

changeAge1 বাটনে ক্লিক করলে, ভিউ আপডেট হয়ে দেখাবে:

```
age: 18
```

### এক্সপ্লিসিট টাইপ ডিক্লেয়ারেশন (ঐচ্ছিক)

আপনি ডিক্লেয়ার করার সময় পরিষ্কারভাবে টাইপ নির্দিষ্ট করেও দিতে পারেন:

```swift
@State private var age: Int = 20
```

এই ধরনের লেখায় টাইপ আরও পরিষ্কারভাবে বোঝা যায়, তবে সাধারণ পরিস্থিতিতে এটি অপরিহার্য নয়।

## সারসংক্ষেপ

এই ক্লাসে মূলত Swift-এর টাইপ সিস্টেম এবং বাস্তব ডেভেলপমেন্টে এর ভূমিকা নিয়ে আলোচনা করা হয়েছে।

Swift-এ, প্রতিটি ভ্যালুর একটি স্পষ্ট টাইপ থাকে, যা ডেটার অর্থ নির্ধারণ করে এবং এটি কোন অপারেশনগুলোতে অংশ নিতে পারবে তা ঠিক করে।

Swift হলো স্ট্রং টাইপ। একবার ভেরিয়েবলের টাইপ নির্ধারিত হলে একে আর অন্য টাইপে অ্যাসাইন করা যায় না। এই সীমাবদ্ধতা কম্পাইল পর্যায়ে এরর ধরতে সাহায্য করে, যার ফলে রানটাইমের সমস্যা কমে এবং কোডের নিরাপত্তা বাড়ে।

ভেরিয়েবলের টাইপ প্রাথমিক ভ্যালু থেকে স্বয়ংক্রিয়ভাবে অনুমান করা যেতে পারে, বা এক্সপ্লিসিটভাবে ঘোষণা করা যেতে পারে। কিছু নির্দিষ্ট ক্ষেত্রে (যেমন খালি অ্যারে, টাইপ ইনফারেন্স এরর, কাস্টম টাইপ), কম্পাইলার সঠিকভাবে টাইপ অনুমান করতে পারে না, তখন অবশ্যই এক্সপ্লিসিটভাবে ডিক্লেয়ার করতে হবে।

বিভিন্ন টাইপের মধ্যে স্বয়ংক্রিয়ভাবে রূপান্তর হয় না, অবশ্যই Int(), Double(), String() ইত্যাদি পদ্ধতির মাধ্যমে এক্সপ্লিসিট রূপান্তর করতে হবে।

টাইপ এলিয়াসও সাপোর্ট করে, বিশেষ করে জটিল প্রোজেক্টে, টাইপ এলিয়াস কোডের পঠনযোগ্যতা বাড়াতে পারে।

টাইপ শুধুমাত্র ডেটার মার্কিং নয়, এটি একটি কনস্ট্রেইন্ট (সীমাবদ্ধতা) মেকানিজমও, যা নিশ্চিত করে যে ডেটার ব্যবহার নিরাপদ, সুস্পষ্ট এবং নিয়ন্ত্রণযোগ্য।

## অতিরিক্ত জ্ঞান - অপশনাল টাইপ (Optional Types)

টাইপ কনভার্সন করার সময়, কিছু অপারেশন ব্যর্থ হতে পারে।

উদাহরণস্বরূপ, String-কে Double-এ রূপান্তর করা:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

এখানকার Double(input) একটি Double রিটার্ন করে না, বরং রিটার্ন করে:

```
Double?
```

এটিকেই বলা হয় অপশনাল টাইপ (Optional)।

### কেন এটি অপশনাল টাইপ রিটার্ন করে?

কারণ স্ট্রিংয়ের কন্টেন্ট অনিশ্চিত, "3.14"-কে Double-এ রূপান্তর করা সম্ভব, কিন্তু "Hello"-কে Double-এ রূপান্তর করা সম্ভব নয়।

তার মানে হলো, এই অপারেশনটি সফলও হতে পারে, আবার ব্যর্থও হতে পারে।

তাই, এই অনিশ্চয়তা প্রকাশ করতে Swift অপশনাল টাইপ ব্যবহার করে।

### অপশনাল টাইপ কী?

অপশনাল টাইপ বোঝায়: একটি ভ্যালুর অস্তিত্ব থাকতেও পারে, আবার নাও থাকতে পারে।

উদাহরণস্বরূপ:

```
Double?
```

এই টাইপটি বোঝায় যে, যদি ভ্যালু থাকে, তবে প্রিন্ট করার সময় এটি Optional(...) দেখাবে, আর যদি কোনো ভ্যালু না থাকে, তবে এটি nil রিটার্ন করবে।

মনে রাখবেন যে, nil মানে হলো "কোনো ভ্যালু নেই", এটি 0 বা খালি স্ট্রিং নয়।

### উদাহরণ

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

আউটপুট:

```
pi:Optional(3.14)
```

এটি নির্দেশ করে যে, স্ট্রিংটি সফলভাবে Double-এ রূপান্তরিত হয়েছে এবং 3.14 পাওয়া গেছে, কিন্তু টাইপটি Double? হওয়ার কারণে, এটি Optional-এর ভেতর মোড়ানো বা র‍্যাপ করা (wrapped) আছে।

যদি রূপান্তর ব্যর্থ হয়:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

আউটপুট:

```
pi: nil
```

যেহেতু "Hello"-কে Double টাইপে রূপান্তর করা যায় না, তাই রূপান্তরটি ব্যর্থ হয় এবং nil রিটার্ন করে।

### Optional(...) আসলে কী?

যখন আমরা সরাসরি একটি অপশনাল টাইপ প্রিন্ট করি:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift এর "ডিবাগ ডেসক্রিপশন" দেখায়:

```
Optional(3.14)
```

এটি প্রকৃত ডেটার কোনো অংশ নয়, বরং এটি নির্দেশ করে যে বর্তমান ভ্যালুটি একটি "অপশনাল টাইপ" এবং এর ভেতরে একটি ভ্যালু আছে।

### অপশনাল টাইপ আনর‍্যাপ (Unwrap) করা

বাস্তব ডেভেলপমেন্টে, আমাদের প্রায়শই অপশনাল টাইপ থেকে আসল ভ্যালুটি বের করার প্রয়োজন হয়, এই প্রক্রিয়াটিকে বলা হয় আনর‍্যাপিং (Unwrapping)।

একটি সাধারণ পদ্ধতি হলো ?? ব্যবহার করে ডিফল্ট ভ্যালু প্রদান করা:

```
??
```

একে বলা হয়: নিল-কোয়ালেসিং অপারেটর (Nil-Coalescing Operator)।

উদাহরণস্বরূপ:

```swift
name ?? "FangJunyu"
```

এই কোডটি বোঝায় যে, যদি name-এ কোনো ভ্যালু থাকে, তবে সেই ভ্যালুটি রিটার্ন করবে; আর যদি এটি nil হয়, তবে ডিফল্ট ভ্যালু "FangJunyu" ব্যবহার করবে।

মনে রাখতে হবে যে, ডিফল্ট ভ্যালুর টাইপটি অবশ্যই অপশনাল টাইপের সাথে মিলতে হবে।

### ?? ব্যবহার করে ডিফল্ট ভ্যালু প্রদান করা

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")	// 30
```

আউটপুট:

```
pi: 30
```

input-কে Double টাইপে রূপান্তর করার সময়, রূপান্তর সফল হয় এবং ভ্যালু আউটপুট দেয়। যদি রূপান্তর ব্যর্থ হয়, তবে ?? দ্বারা প্রদত্ত ডিফল্ট ভ্যালুটি আউটপুট হবে।

যদি রূপান্তর ব্যর্থ হয়:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

আউটপুট:

```
pi: 10
```

এখানে, input-কে Double টাইপে রূপান্তর করার সময়, রূপান্তর ব্যর্থ হয় এবং nil রিটার্ন করে, তখন ?? এই nil-এর জন্য একটি ডিফল্ট ভ্যালু প্রদান করবে।

অপশনাল টাইপ এমন পরিস্থিতিতে ব্যবহৃত হয় যা বোঝায় "ভ্যালু থাকতেও পারে, আবার নাও থাকতে পারে"।

টাইপ কনভার্সনের মতো এমন অপারেশনে যা ব্যর্থ হতে পারে, সেখানে প্রোগ্রামের নিরাপত্তা নিশ্চিত করতে Swift অপশনাল টাইপ রিটার্ন করে।

যখন আমাদের এর ভেতরের ভ্যালুটি ব্যবহার করার প্রয়োজন হয়, আমরা ?? এর মাধ্যমে ডিফল্ট ভ্যালু প্রদান করতে পারি, যাতে nil আসলে আমরা যেন একটি যৌক্তিক ফলাফল পেতে পারি।
