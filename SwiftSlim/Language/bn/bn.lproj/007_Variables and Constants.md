# ভেরিয়েবল এবং কনস্ট্যান্ট

এই পাঠে আমরা মূলত Swift-এর ভেরিয়েবল এবং কনস্ট্যান্ট সম্পর্কে শিখব, এবং কিছু সাধারণ ডেটা টাইপ ও মৌলিক অপারেটরের সাথে পরিচিত হব।

এই বিষয়গুলো প্রোগ্রামিংয়ের সবচেয়ে মৌলিক জ্ঞান, এবং পরবর্তীতে SwiftUI শেখার জন্য একটি গুরুত্বপূর্ণ ভিত্তি।

## দৈনন্দিন জীবনে ভেরিয়েবল এবং কনস্ট্যান্ট

আমরা দৈনন্দিন জীবনের বিভিন্ন জিনিসের মাধ্যমে ভেরিয়েবল এবং কনস্ট্যান্ট বুঝতে পারি।

আমাদের জীবনের ভেরিয়েবলগুলো হলো:

- টিভিতে সম্প্রচারিত প্রতিটি অনুষ্ঠানের বিষয়বস্তু আলাদা হয়
- প্রতিদিনের আবহাওয়া ভিন্ন থাকে
- ঘড়ির কাঁটা প্রতি সেকেন্ডে পরিবর্তিত হয়

এই জিনিসগুলোর একটি সাধারণ বৈশিষ্ট্য হলো: এগুলো পরিবর্তিত হয়।

যদি টিভিতে সবসময় শুধু একটি ছবি থাকে, আবহাওয়া সবসময় রৌদ্রোজ্জ্বল থাকে এবং ঘড়ির কাঁটা কখনো না ঘোরে, তবে এই জিনিসগুলো হলো কনস্ট্যান্ট।

একটি পরিবর্তিত হতে পারে, এবং অন্যটি কখনো পরিবর্তিত হয় না।

## ভেরিয়েবল এবং কনস্ট্যান্ট বোঝা

অ্যাপ ডেভেলপমেন্টে সাধারণত ব্যবহারকারীদের কিছু তথ্য পূরণ বা সংরক্ষণ করতে হয়।

উদাহরণস্বরূপ:

- অ্যাকাউন্টের নাম
- জন্মদিন
- যোগাযোগের তথ্য
- ঠিকানা

এই তথ্যগুলো সংরক্ষণ করা হবে এবং প্রদর্শন করা হবে।

উদাহরণস্বরূপ, ব্যবহারকারী অ্যাপে একটি নাম ইনপুট হিসেবে দিলে:

```
FangJunyu
```

অ্যাপে দেখানোর জন্য আমাদের এই নামটি সংরক্ষণ করতে হবে।

এই সংরক্ষণের প্রক্রিয়াটিকে সহজভাবে বোঝা যেতে পারে: একটি ড্রয়ারে কিছু রাখা।

যখন আমরা একটি নাম সংরক্ষণ করি, তখন মনে হয় যেন আমরা নামটি একটি ড্রয়ারে রাখছি।

সংরক্ষণ করার মতো অনেক কিছু থাকতে পারে, এবং ড্রয়ারও অনেক থাকতে পারে। কোন ড্রয়ারে কী আছে তা জানার জন্য, আমাদের প্রতিটি ড্রয়ারের একটি নাম দিতে হবে।

উদাহরণস্বরূপ:

```
name
```

এই উদাহরণে, name হলো ড্রয়ারের নাম, এবং FangJunyu হলো সংরক্ষিত তথ্য।

![Var](../../RESOURCE/007_var.png)

**Swift-এ ডেটা সংরক্ষণ করার প্রয়োজন হলে, অবশ্যই ভেরিয়েবল (var) অথবা কনস্ট্যান্ট (let) ব্যবহার করে ডিক্লেয়ার (ঘোষণা) করতে হবে।**

যেহেতু নাম সাধারণত পরিবর্তনযোগ্য, তাই আমাদের ভেরিয়েবল ব্যবহার করা উচিত।

```swift
var name = "FangJunyu"
```

এখানে name নামক একটি ভেরিয়েবল ঘোষণা করা হয়েছে, যার ডেটা টাইপ হলো String এবং এর মান হলো "FangJunyu"।

### ভেরিয়েবল এবং কনস্ট্যান্ট এর মধ্যে পার্থক্য

ভেরিয়েবল ঘোষণা করা হয় var ব্যবহার করে:

```swift
var
```

কনস্ট্যান্ট ঘোষণা করা হয় let ব্যবহার করে:

```swift
let
```

উদাহরণস্বরূপ:

```swift
var name = "FangJunyu"
let id = 123456
```

যদি ভেরিয়েবল হিসেবে ঘোষণা করা হয়, তার মানে হলো এই মানটি পরিবর্তন করা যাবে; আর যদি কনস্ট্যান্ট হিসেবে ঘোষণা করা হয়, তবে এটি আর পরিবর্তন করা যাবে না।

সুতরাং, ভেরিয়েবল এবং কনস্ট্যান্ট এর মূল পার্থক্য হলো: এদের মান পরিবর্তন করার অনুমতি আছে কি নেই।

### কনস্ট্যান্ট পরিবর্তন করা

যদি কোড কোনো কনস্ট্যান্টের মান পরিবর্তন করার চেষ্টা করে:

```swift
let name = "Sam"
name = "Bob"
```

Swift তখন জানাবে যে এই মানটি অ্যাসাইন (assign) করা সম্ভব নয়, কারণ এটি একটি কনস্ট্যান্ট।

```
Cannot assign to property: 'name' is a 'let' constant
```

এই প্রক্রিয়া ডেভেলপারদের কিছু গুরুত্বপূর্ণ ডেটা ভুলবশত পরিবর্তিত হওয়া থেকে রক্ষা করতে সাহায্য করে।

### SwiftUI-তে প্রদর্শন করা

ContentView.swift ফাইলটি খুলুন এবং View এর ভেতরে একটি ভেরিয়েবল ঘোষণা করুন:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

এখানে name নামক একটি ভেরিয়েবল ঘোষণা করা হয়েছে এবং Text এর মাধ্যমে এটিকে ইন্টারফেসে প্রদর্শন করা হয়েছে।

যদি আমরা ভেরিয়েবলের কন্টেন্ট পরিবর্তন করি:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

তাহলে Text দ্বারা প্রদর্শিত টেক্সট কন্টেন্টও পরিবর্তিত হবে।

এই পদ্ধতির মাধ্যমে, আমরা ইন্টারফেসে প্রদর্শিত কন্টেন্ট নিয়ন্ত্রণ করতে ভেরিয়েবল ব্যবহার করতে পারি, প্রতিবার সরাসরি Text এর ভেতরের টেক্সট পরিবর্তন করার প্রয়োজন হয় না।

অন্য কোনো জায়গায় যদি এই তথ্যের প্রয়োজন হয়, তবে আমরা ভেরিয়েবলটি সেখানে পাঠিয়ে দিতে পারি, যা অনেকটা "ড্রয়ার"-এর ভেতরের জিনিস অন্য কোথাও ব্যবহারের জন্য দেওয়ার মতো।

### SwiftUI-তে ভেরিয়েবল এবং কনস্ট্যান্ট এর অবস্থান

SwiftUI-তে, ভেরিয়েবল এবং কনস্ট্যান্ট সাধারণত body এর বাইরে লেখা হয়:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

অবশ্যই, ভেরিয়েবল এবং কনস্ট্যান্ট body এর ভেতরেও লেখা যেতে পারে:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

তবে এই দুটির মধ্যে একটি গুরুত্বপূর্ণ পার্থক্য রয়েছে:

- যদি ভেরিয়েবল এবং কনস্ট্যান্ট body এর ভেতরে লেখা হয়, তবে প্রতিবার ভিউ (view) রিক্যালকুলেট হওয়ার সময় এই ভেরিয়েবলটি নতুন করে তৈরি হবে।
- যদি body এর বাইরে লেখা হয়, তবে ভেরিয়েবলটি ভিউ স্ট্রাকচারের একটি প্রপার্টি হিসেবে থাকবে এবং কোডের গঠন আরও পরিষ্কার হবে।

তাই, বাস্তব ডেভেলপমেন্টে, ভেরিয়েবল এবং কনস্ট্যান্ট সাধারণত body এর বাইরেই লেখা হয়।

## ডেটা টাইপ (Data Types)

ভেরিয়েবল বিভিন্ন ধরনের ডেটা সংরক্ষণ করতে পারে, এই ধরনগুলোকে ডেটা টাইপ বলা হয়।

উদাহরণস্বরূপ, এর আগে আমরা টেক্সট দেখানোর জন্য Text ব্যবহার করেছি:

```swift
Text("Hello, World")
```

এখানের "Hello, World" হলো স্ট্রিং (String) টাইপের অংশ।

স্ট্রিং ব্যবহার করা হয় টেক্সট কন্টেন্ট বোঝাতে, এবং Swift-এ একে ডাবল কোটেশন "" দিয়ে আবৃত করে লিখতে হয়।

উদাহরণস্বরূপ:

```swift
var hello = "Hello, World"
```

স্ট্রিং ছাড়াও, Swift-এ আরও অনেক ডেটা টাইপ রয়েছে।

প্রাথমিক পর্যায়ে সবচেয়ে সাধারণ চারটি টাইপ হলো:

- `String`
- `Int`
- `Double`
- `Bool`

এই চারটি টাইপ দিয়ে বেশিরভাগ প্রাথমিক ডেটা প্রসেসিংয়ের কাজ সম্পন্ন করা যায়।

**String**

String টেক্সট কন্টেন্ট বোঝায়, যেমন:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

স্ট্রিং সাধারণত ব্যবহৃত হয়: ব্যবহারকারীর নাম (username), শিরোনাম এবং টেক্সট কন্টেন্ট বোঝাতে।

**Int**

Int মানে পূর্ণসংখ্যা, অর্থাৎ দশমিক ছাড়া সংখ্যা।

উদাহরণস্বরূপ:

```swift
var age = 26
var count = 100
```

পূর্ণসংখ্যা সাধারণত ব্যবহৃত হয়: বয়স, পরিমাণ এবং গণনা বোঝাতে।

**Double**

Double বলতে দশমিক যুক্ত সংখ্যা বোঝায়।

উদাহরণস্বরূপ:

```swift
var weight = 74.5
var height = 185.0
```

Swift-এ, দশমিক সংখ্যা সাধারণত ডিফল্টরূপে Double টাইপ হিসেবে বিবেচিত হয়।

Float নামেও একটি অনুরূপ টাইপ রয়েছে, তবে এর সাংখ্যিক পরিসর তুলনামূলক ছোট হওয়ায়, বাস্তব ডেভেলপমেন্টে Double বেশি ব্যবহৃত হয়।

**Bool**

Bool টাইপটি দুটি অবস্থা বোঝাতে ব্যবহৃত হয়: true (সত্য) এবং false (মিথ্যা)।

উদাহরণস্বরূপ:

```swift
var isShowAlert = true
var isLogin = false
```

Bool টাইপ প্রায়ই শর্ত যাচাই (condition checking) এর জন্য ব্যবহৃত হয়। যেমন, সতর্কতামূলক বার্তা (alert) দেখানো হবে কি না।

যেসব ক্ষেত্রে শুধুমাত্র দুটি ফলাফল থাকে, সেখানে Bool টাইপ ব্যবহার করা সবচেয়ে উপযুক্ত।

### SwiftUI-তে প্রদর্শন করা

যখন আমরা String টাইপ প্রদর্শন করি, তখন আমরা সরাসরি Text ব্যবহার করতে পারি:

```swift
Text(name)
```

কিন্তু Int, Double ইত্যাদি টাইপ স্ট্রিং নয়, তাই এগুলোকে সরাসরি টেক্সট কন্টেন্টের অংশ হিসেবে ব্যবহার করা যায় না।

Text এর ভেতরে যদি এই ডেটাগুলো প্রদর্শন করতে হয়, তবে স্ট্রিং ইন্টারপোলেশন (String Interpolation) ব্যবহার করা যেতে পারে:

```swift
\()
```

স্ট্রিং ইন্টারপোলেশন স্ট্রিং কন্টেন্টের ভেতরে লিখতে হয়, এবং ভেরিয়েবল বা কনস্ট্যান্টকে \() এর মধ্যে আবৃত করতে হয়।

উদাহরণস্বরূপ:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

উপরের কোডে, "" স্ট্রিং নির্দেশ করে, এবং \() ব্যবহৃত হয়েছে ভেরিয়েবল বা কনস্ট্যান্টকে স্ট্রিংয়ের মধ্যে প্রবেশ করাতে।

উদাহরণস্বরূপ:

```swift
"DoubleNum: \(num * 2)"
```

প্রদর্শিত কন্টেন্টটি হলো:

```swift
DoubleNum: 2
```

এইভাবে, আমরা একটি স্ট্রিংয়ের মধ্যে ভেরিয়েবল বা কনস্ট্যান্ট যুক্ত করতে পারি, এবং তারপর Text ব্যবহার করে সেগুলোকে ইন্টারফেসে প্রদর্শন করতে পারি।

টিপস: \() শুধুমাত্র স্ট্রিং "" এর ভেতরেই ব্যবহার করা যায়।

## অপারেটর (Operators)

ভেরিয়েবল বা কনস্ট্যান্ট ঘোষণা করার সময়, আমরা নিচের মতো লেখা দেখতে পাই:

```swift
var num = 1
```

এখানের = চিহ্নটিকে অ্যাসাইনমেন্ট অপারেটর (Assignment Operator) বলা হয়।

এর কাজ হলো: ডানদিকের মানটিকে বামদিকের ভেরিয়েবলে অ্যাসাইন করে দেওয়া।

এই উদাহরণে, 1 মানটিকে num ভেরিয়েবলে অ্যাসাইন করা হয়েছে।

অ্যাসাইনমেন্ট অপারেটর ছাড়াও, আরও কিছু সাধারণ গাণিতিক অপারেটর রয়েছে:

- `+`
- `-`
- `*`
- `/`

যখন আমরা সংখ্যা নিয়ে হিসাব-নিকাশ করি, যেমন Int বা Double টাইপের ডেটা, তখন এই অপারেটরগুলো ব্যবহৃত হয়।

উদাহরণস্বরূপ:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

হিসাব সম্পন্ন হওয়ার পর, ফলাফলটি ভেরিয়েবলে সংরক্ষিত হবে।

## সারসংক্ষেপ (Summary)

ভেরিয়েবল, কনস্ট্যান্ট এবং অপারেটর হলো প্রোগ্রামিংয়ের সবচেয়ে মৌলিক ধারণা।

ভেরিয়েবল এবং কনস্ট্যান্টের মাধ্যমে আমরা প্রোগ্রামে বিভিন্ন ডেটা সংরক্ষণ করতে পারি; ডেটা টাইপের মাধ্যমে ডেটার ধরন নির্দিষ্ট করতে পারি; এবং অপারেটরের সাহায্যে ডেটা হিসাব ও প্রসেস করতে পারি।

এই জ্ঞান প্রোগ্রামিং জগতের প্রাথমিক টুলের মতো। এই বিষয়গুলো আয়ত্ত করতে পারলে পরবর্তীতে Swift এবং SwiftUI শেখার জন্য একটি শক্ত ভিত্তি তৈরি হবে।
