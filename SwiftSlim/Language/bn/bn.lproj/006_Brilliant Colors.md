# চমৎকার রঙ

এই পাঠে আমরা SwiftUI-এর সাধারণ ভিজ্যুয়াল মডিফায়ার (visual modifiers) সম্পর্কে শিখব, যার মধ্যে রয়েছে:

- রঙ (Color)
- ফোরগ্রাউন্ড কালার (Foreground Color)
- ব্যাকগ্রাউন্ড কালার (Background Color)
- অফসেট (Offset)
- অপাসিটি বা স্বচ্ছতা (Opacity)
- ব্লার বা অস্পষ্টতা (Blur)

এছাড়াও আমরা Safe Area (নিরাপদ অঞ্চল) সম্পর্কে শিখব।

এই মডিফায়ারগুলো ভিউয়ের বাহ্যিক রূপ নিয়ন্ত্রণ করতে ব্যবহৃত হয়, যা ইন্টারফেসকে আরও স্পষ্ট ও স্তরায়িত (layered) করে তোলে।

## রঙ (Color)

SwiftUI-তে, আপনি টেক্সটের রঙ সেট করতে পারেন।

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

.blue নীল রঙ নির্দেশ করে, যা মূলত Color.blue-এর একটি সংক্ষিপ্ত রূপ (টাইপ ইনফারেন্স)।

সাধারণ রঙগুলোর মধ্যে রয়েছে:

```
.black
.green
.yellow
.pink
.gray
...
```

এগুলো সবই Color-এর স্ট্যাটিক প্রপার্টি (static properties)।

![Color](../../RESOURCE/006_color.png)

আপনি Color-কে রঙের ধরন হিসেবে বিবেচনা করতে পারেন, যেখানে .blue, .red হলো এর নির্দিষ্ট রঙ।

### Color ভিউ (Color View)

SwiftUI-তে, Color-কে একটি ভিউ (view) হিসেবেও প্রদর্শন করা যেতে পারে।

```swift
Color.red
    .frame(width: 100, height:100)
    .cornerRadius(10)
```

![Color](../../RESOURCE/006_color5.png)

এই কোডটি একটি ১০০×১০০ সাইজের লাল রঙের বর্গাকার ব্লক তৈরি করবে।

আপনি চাইলে পুরো ইন্টারফেসটি নির্দিষ্ট কোনো রঙে প্রদর্শন করতে পারেন:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
    }
}
```

![Color](../../RESOURCE/006_color1.png)

রান করার পর দেখতে পাবেন যে লাল রঙটি পুরো স্ক্রিন জুড়ে নেই, iPhone-এর ওপর ও নিচের অংশ এখনও সাদা রয়ে গেছে, এখানেই Safe Area (নিরাপদ অঞ্চল)-এর উপস্থিতি জড়িত।

## Safe Area (নিরাপদ অঞ্চল)

Safe Area হলো সিস্টেম কর্তৃক সংরক্ষিত একটি অঞ্চল, যা কনটেন্ট ঢেকে যাওয়া রোধ করতে ব্যবহৃত হয়, এর মধ্যে রয়েছে:

১. ওপরের স্ট্যাটাস বার (সময়, ব্যাটারি)

২. নিচের হোম ইন্ডিকেটর বার

৩. নচ (notch) অথবা ডায়নামিক আইল্যান্ড (Dynamic Island) অঞ্চল

![Color](../../RESOURCE/006_color3.png)

SwiftUI ডিফল্টভাবে কনটেন্টকে নিরাপদ অঞ্চলের মধ্যে সীমাবদ্ধ রাখে, তাই ভিউ স্ক্রিনের একেবারে প্রান্ত পর্যন্ত প্রসারিত হয় না।

### Safe Area (নিরাপদ অঞ্চল) এড়িয়ে চলা

আপনি যদি পুরো স্ক্রিন জুড়ে রঙটি ছড়িয়ে দিতে চান, তবে ignoresSafeArea ব্যবহার করতে পারেন:

```swift
Color.red
    .ignoresSafeArea()
```

অথবা edgesIgnoringSafeArea ব্যবহার করতে পারেন:

```swift
Color.red
    .edgesIgnoringSafeArea(.all)
```

![Color](../../RESOURCE/006_color4.png)

এভাবে ভিউটি পুরো স্ক্রিন জুড়ে প্রসারিত হবে।

মনে রাখবেন, edgesIgnoringSafeArea হলো পুরোনো লেখার পদ্ধতি, iOS 14 থেকে ignoresSafeArea ব্যবহার করার পরামর্শ দেওয়া হয়।

## ফোরগ্রাউন্ড কালার

### foregroundStyle মডিফায়ার

আগের পাঠগুলোতে আমরা রঙ সেট করতে foregroundStyle শিখেছি।

উদাহরণস্বরূপ:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

foregroundStyle হলো একটি নতুন স্টাইল সিস্টেম, যা রঙ, গ্রেডিয়েন্ট, ম্যাটেরিয়াল (materials) ইত্যাদি সাপোর্ট করে।

![Color](../../RESOURCE/006_color2.png)

### foregroundColor মডিফায়ার

foregroundColor ব্যবহার করেও রঙ সেট করা যায়:

```swift
Text("Fang Junyu")
    .foregroundColor(.blue)
```

এর ব্যবহারের পদ্ধতি foregroundStyle-এর মতোই।

নতুন সংস্করণের Xcode-এ, এটি সতর্ক করে যে ভবিষ্যতের iOS সংস্করণে foregroundColor বাতিল হয়ে যেতে পারে, তাই foregroundStyle ব্যবহার করাকে অগ্রাধিকার দেওয়ার পরামর্শ দেওয়া হয়।

## ব্যাকগ্রাউন্ড

আপনি যদি ভিউতে ব্যাকগ্রাউন্ড কালার যুক্ত করতে চান, তবে background ব্যবহার করতে পারেন:

```swift
background(.red)
```

উদাহরণস্বরূপ, টেক্সটে ব্যাকগ্রাউন্ড কালার যুক্ত করতে:

```swift
Text("FangJunyu.com")
    .background(.orange)
```

![Color](../../RESOURCE/006_color6.png)

আমরা যখন Word, Chrome ইত্যাদি অ্যাপে টেক্সট সিলেক্ট করি, তখন এই ধরনের ব্যাকগ্রাউন্ড কালার ইফেক্ট দেখতে পাই।

![Color](../../RESOURCE/006_color16.png)

যদি ব্যাকগ্রাউন্ডটি সম্প্রসারিত করতে চান, তবে padding যুক্ত করতে হবে:

```swift
Text("FangJunyu.com")
    .padding(10)
    .background(Color.orange)
```

![Color](../../RESOURCE/006_color7.png)

এখানে একটি গুরুত্বপূর্ণ নিয়ম মনে রাখতে হবে:

SwiftUI মডিফায়ারগুলো ওপর থেকে নিচে ভিউ তৈরি করে। পরে লেখা মডিফায়ারটি তার আগের ফলাফলের ওপর কাজ করে।

তাই:

```swift
.padding()
.background()
```

এর মানে হলো ব্যাকগ্রাউন্ডটি প্যাডিং যুক্ত হওয়ার পর পুরো ভিউটিকে আবৃত করবে।

যদি ক্রম উল্টে লেখা হয়:

```swift
Text("FangJunyu.com")
    .background(Color.orange)
    .padding(10)
```

![Color](../../RESOURCE/006_color6.png)

ব্যাকগ্রাউন্ড প্রসারিত হবে না, কারণ ব্যাকগ্রাউন্ডটি এর পরে থাকা প্যাডিংকে আবৃত করেনি।

## উদাহরণ - চারকোণা নাট (Square Nut)

এখন, আমরা একটি সাধারণ চারকোণা নাটের ভিউ তৈরি করব।

![Color](../../RESOURCE/006_color8.png)

প্রথমে একটি ৫০ × ৫০ সাইজের সাদা রঙের বর্গাকার ব্লক তৈরি করুন:

```swift
Color.white
    .frame(width: 50, height: 50)
```

যদি একে বৃত্তে রূপান্তর করতে চান, তবে cornerRadius ব্যবহার করতে পারেন:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
```

যখন কর্নার রেডিয়াস (corner radius) প্রস্থ ও উচ্চতার অর্ধেকের সমান হয়, তখন এটি বৃত্তে পরিণত হয়।

এখন, একটি নীল ব্যাকগ্রাউন্ড যুক্ত করুন:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
    .padding(20)
    .background(.blue)
```

![Color](../../RESOURCE/006_color8.png)

নতুন যুক্ত করা padding বাইরের অংশটিকে প্রসারিত করবে, এবং background বাইরের অংশে নীল রঙ আঁকবে।

এখন চারকোণা নাটের ইফেক্টটি সম্পন্ন হয়েছে।

### বিকল্প একটি পদ্ধতি

ব্যাকগ্রাউন্ড কালার ব্যবহারের পাশাপাশি, আপনি ZStack ব্যবহার করেও এই চারকোণা নাট তৈরি করতে পারেন।

আমরা আগেই শিখেছি যে ZStack ব্যবহার করে ভিউগুলোকে স্তরে স্তরে সাজানো যায়, আর চারকোণা নাটকেও একটি বৃত্ত ও একটি আয়তক্ষেত্রের ওভারল্যাপ হিসেবে ভাবা যেতে পারে।

```swift
ZStack {
    Color.blue
        .frame(width: 70, height: 70)
    Color.white
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color8.png)

ZStack ভিউগুলোকে ক্রমানুসারে একটির ওপর আরেকটি সাজায়, পরে যুক্ত করা ভিউটি ওপরের স্তরে প্রদর্শিত হয়।

## উদাহরণ - দুটি ওভারল্যাপ করা বৃত্ত

অনেক আইকনই সাধারণ জ্যামিতিক আকার ওভারল্যাপ করে তৈরি করা হয়, উদাহরণস্বরূপ দুটি ওভারল্যাপ করা বৃত্ত।

![Color](../../RESOURCE/006_color14.png)

প্রথমে, দুটি বৃত্ত তৈরি করুন:

```swift
Color.red
    .frame(width: 50, height:50)
    .cornerRadius(25)
Color.orange
    .frame(width: 50, height:50)
    .cornerRadius(25)
```

![Color](../../RESOURCE/006_color11.png)

যেহেতু আমাদের দুটি বৃত্তকে একে অপরের ওপর প্রদর্শন করতে হবে, তাই ZStack লেআউট ব্যবহার করব:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color12.png)

এই মুহূর্তে, সমান আকারের দুটি বৃত্ত একে অপরকে সম্পূর্ণ ঢেকে রেখেছে।

আমাদের দুটি বৃত্ত আংশিক ওভারল্যাপ করানো দরকার, সম্পূর্ণ নয়, এর জন্য অফসেট (offset) ইফেক্ট ব্যবহার করতে পারি।

## অফসেট (Offset)

offset শুধুমাত্র ভিউ আঁকার (drawing) অবস্থান পরিবর্তন করে, এটি প্যারেন্ট ভিউয়ের লেআউট হিসাবকে প্রভাবিত করে না।

ব্যবহারের পদ্ধতি:

```swift
.offset(x:y:)
```

এখানে x হলো আনুভূমিক (horizontal) অফসেটের পরিমাণ, আর y হলো উলম্ব (vertical) অফসেটের পরিমাণ।

ধনাত্মক মান ডানদিকে/নিচের দিকে স্থানান্তরিত হওয়া বোঝায়, আর ঋণাত্মক মান বামদিকে/ওপরের দিকে স্থানান্তরিত হওয়া বোঝায়।

দুটি বৃত্তকে আংশিকভাবে ওভারল্যাপ করতে offset ব্যবহার করি:

```swift
ZStack {
    Color.red
        .frame(width: 50, height: 50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color13.png)

লেআউটে লাল বৃত্তের অবস্থান পরিবর্তন হয়নি, তবে আঁকার অবস্থানটি ২৫ পয়েন্ট বামে সরে গেছে। ফলে দুটি বৃত্তের মাঝে আংশিক ওভারল্যাপের ভিজ্যুয়াল ইফেক্ট তৈরি হয়েছে।

## অপাসিটি বা স্বচ্ছতা (Opacity)

SwiftUI-তে, ভিউয়ের স্বচ্ছতা নির্ধারণের জন্য opacity ব্যবহৃত হয়।

সাধারণ ব্যবহার:

```swift
.opacity(0.5)
```

opacity এর প্যারামিটার রেঞ্জ ০.০ থেকে ১.০ পর্যন্ত, যেখানে:

- ০ বলতে সম্পূর্ণ স্বচ্ছ (transparent) বোঝায়
- ১ বলতে অস্বচ্ছ (opaque) বোঝায়

কমলা বৃত্তটির স্বচ্ছতা সেট করতে opacity ব্যবহার করা যেতে পারে:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
```

![Color](../../RESOURCE/006_color14.png)

কমলা বৃত্তের opacity ০.৮ এ সেট করার পর, এর অস্বচ্ছতা ৮০% হয়, যখন দুটি বৃত্ত ওভারল্যাপ করে, তখন ওভারল্যাপ অংশে রঙ মিশ্রিত হওয়ার ইফেক্ট তৈরি হয়।

## ব্লার বা অস্পষ্টতা (Blur)

SwiftUI-তে, অস্পষ্ট ইফেক্ট তৈরি করতে blur ব্যবহার করা যেতে পারে:

```swift
.blur(radius:10)
```

radius বলতে ব্লারের রেডিয়াস বোঝায়, মান যত বড় হবে, ব্লার ইফেক্ট তত বেশি স্পষ্ট হবে।

আমরা বৃত্ত দুটিতে ব্লার ইফেক্ট যোগ করতে পারি:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x:-35)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
.blur(radius:10)
```

## সারসংক্ষেপ

এই পাঠটি SwiftUI-এর সাধারণ ভিজ্যুয়াল মডিফায়ারগুলোকে কেন্দ্র করে পরিচালিত হয়েছে, যেখানে আমরা শিখেছি কীভাবে মডিফায়ারের মাধ্যমে ভিউয়ের রঙ, অবস্থান এবং ভিজ্যুয়াল ইফেক্ট নিয়ন্ত্রণ করতে হয়।

নির্দিষ্ট উদাহরণের সাহায্যে, আমরা ইন্টারফেসে বিভিন্ন ভিজ্যুয়াল মডিফায়ারের প্রকৃত কাজগুলো দেখেছি এবং নিরাপদ অঞ্চল (Safe Area) সম্পর্কে জ্ঞান অর্জন করেছি।

এগুলো অত্যন্ত মৌলিক মডিফায়ার, আমরা যত বেশি এগুলো অনুশীলন ও ব্যবহার করব, বাস্তব ডেভেলপমেন্টে ইন্টারফেস ইফেক্টগুলো তত নিখুঁতভাবে নিয়ন্ত্রণ করতে পারব।

### পাঠ শেষের অনুশীলন

- একটি ছবিতে অপাসিটি এবং ব্লার ইফেক্ট যোগ করুন
- বিভিন্ন অপাসিটি লেভেলের তিনটি ওভারল্যাপিং বৃত্ত সেট করুন
- স্ক্রিন জুড়ে থাকা একটি ব্যাকগ্রাউন্ড ছবি তৈরি করুন, এবং নিরাপদ অঞ্চল এড়িয়ে চলুন (ignore safe area)
- একাধিক ভিউয়ের অবস্থান সামঞ্জস্য করতে offset ব্যবহার করুন

এই অনুশীলনের উদ্দেশ্য এপিআই (API) মুখস্থ করা নয়, বরং ভিজ্যুয়াল পরিবর্তন এবং লেআউট আচরণের মধ্যে সম্পর্ক পর্যবেক্ষণ করা।
