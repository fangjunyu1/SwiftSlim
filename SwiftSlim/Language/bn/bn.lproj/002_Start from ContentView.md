# ContentView থেকে শুরু

## ক্লাসের প্রস্তুতি

এই পাঠে, আমরা ContentView.swift ফাইল থেকে শুরু করে SwiftUI-এর মৌলিক কাঠামো সম্পর্কে পদ্ধতিগতভাবে জানব, যার মধ্যে রয়েছে:

- মন্তব্য (Comments)
- View কাঠামো
- VStack / HStack / ZStack লেআউট
- SF Symbols আইকন
- ভিউ মডিফায়ার (View modifiers)
- প্রিভিউ কোড (#Preview)

প্রথমে, আগে তৈরি করা Xcode প্রজেক্টটি খুঁজে বের করুন এবং .xcodeproj ফাইলে ডাবল-ক্লিক করুন।

বাম পাশের Navigator নেভিগেশন এরিয়া থেকে ContentView.swift ফাইলটি নির্বাচন করুন।

দ্রষ্টব্য: প্রতিবার প্রজেক্ট খোলার সময়, Canvas-এ "Preview paused" দেখাতে পারে। প্রিভিউ সচল করতে রিফ্রেশ বাটনে ক্লিক করুন।

![Swift](../../RESOURCE/002_view7.png)

## ContentView চেনা

ContentView কোড:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

কোডটি খুব অল্প মনে হলেও, এর মধ্যে SwiftUI-এর মূল কাঠামো অন্তর্ভুক্ত রয়েছে।

### ১. মন্তব্য (Comments)

ফাইলের শীর্ষে:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/2/23.
//
```

এটি হলো ফাইল কমেন্ট বা মন্তব্য, যা ফাইলের তথ্য রেকর্ড করার জন্য ব্যবহৃত হয়। এতে ফাইলের নাম, প্রজেক্টের নাম, নির্মাতার নাম এবং তৈরির সময় উল্লেখ থাকে।

Swift-এ এক লাইনের মন্তব্যের জন্য `//` ব্যবহার করা হয়:

```swift
// এটি একটি মন্তব্য
```

মন্তব্য কোডের পঠনযোগ্যতা বাড়ায় এবং ডেভেলপারকে কোডের লজিক বুঝতে সাহায্য করে।

বাস্তব ডেভেলপমেন্টে, কোড অনেক সময় জটিল হতে পারে। যদি মন্তব্য না লেখা হয়, তবে তিন দিন পরে নিজের লেখা কোড নিজেই বুঝতে না পারার সম্ভাবনা থাকে।

তাই, কোড লেখার সময় সক্রিয়ভাবে মন্তব্য যোগ করা একটি ভালো অভ্যাস। মন্তব্যের মাধ্যমে কোডের লজিক লিখে রাখলে পরবর্তীতে কোড রক্ষণাবেক্ষণ করা সহজ হয়।

**সাময়িকভাবে কোড বন্ধ রাখা**

কোনো সমস্যা খুঁজে বের করার জন্য মন্তব্য ব্যবহার করে সাময়িকভাবে কোড বন্ধ রাখা যায়।

উদাহরণস্বরূপ:

```
A
B
C
```

A, B, এবং C হলো কোডের তিনটি অংশ। যদি কোনো একটি অংশে ভুল থাকে, তবে আমরা কোডটি মন্তব্য (comment out) করে পরীক্ষা করতে পারি।

প্রথমে A-তে মন্তব্য যোগ করি:

```
// A
B
C
```

যদি A মন্তব্য করার পর কোড ঠিকভাবে কাজ করে, তবে বোঝা যাবে সমস্যাটি A কোডে ছিল।

যদি A মন্তব্য করার পরেও সমস্যা থেকে যায়, তবে আমরা একইভাবে B-কে মন্তব্য করতে পারি। এভাবে ধাপে ধাপে সমস্যাযুক্ত কোডটি খুঁজে বের করা যায়।

ডেভেলপমেন্টের সময় অনেক সমস্যা দেখা দেয়, যেগুলো চিহ্নিত করতে আমরা কোড কমেন্ট আউট করি। এটি বাগ (BUG) খুঁজে বের করতে সাহায্য করে।

Xcode-এ শর্টকাট ব্যবহার করতে পারেন:

```
Command ⌘ + /
```

এটি দ্রুত মন্তব্য যোগ করতে বা মুছতে সাহায্য করে।

### ২. SwiftUI ফ্রেমওয়ার্ক ইম্পোর্ট করা

```swift
import SwiftUI
```

এই কোডটি SwiftUI ফ্রেমওয়ার্ক ইম্পোর্ট করা বোঝায়।

SwiftUI-এর View, Text, Image, VStack ইত্যাদি টাইপগুলো এই ফ্রেমওয়ার্ক থেকেই আসে।

যদি SwiftUI ইম্পোর্ট করা না হয়, তবে Xcode ত্রুটি (error) দেখাবে:

```
Cannot find type 'View' in scope
```

এর মানে হলো কম্পাইলার View টাইপটি চিনতে পারছে না।

### ৩. View কাঠামো

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

প্রথমবার এই View কাঠামোটি দেখে আপনার অপরিচিত মনে হতে পারে, কারণ এতে struct, View, var, body, some-এর মতো কি-ওয়ার্ড (keywords) রয়েছে।

আমরা এখনো এই কি-ওয়ার্ডগুলো শিখিনি, তবে আপাতত শুধু এটুকুই জেনে রাখুন যে, এই কোডটির অর্থ হলো `ContentView` নামে একটি View তৈরি করা হয়েছে।

আপনি View-কে একটি ড্রয়িং বোর্ড বা ক্যানভাস হিসেবে কল্পনা করতে পারেন, যেখানে আমরা চিত্রাঙ্কন করি, আর চিত্রাঙ্কনের সরঞ্জাম হলো SwiftUI।

উদাহরণস্বরূপ:

![Swift](../../RESOURCE/002_view.png)

উপরের ছবিতে তিনটি পেজ দেখানো হয়েছে, যা মূলত তিনটি ভিন্ন View।

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

যখন আমরা SwiftUI দিয়ে অ্যাপ তৈরি করি, তখন প্রতিটি পেজ এক একটি View হয়।

### ৪. SwiftUI কোড

View কাঠামোর ভেতরে SwiftUI কোড প্রদর্শিত হয়:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

এই SwiftUI কোডটি নির্দেশ করে যে এটি একটি ভার্টিকাল (উল্লম্ব) লেআউট, যা একটি আইকন এবং একটি টেক্সট প্রদর্শন করছে।

![Swift](../../RESOURCE/002_view1.png)

#### VStack লেআউট

```swift
VStack { }  // ভার্টিকাল লেআউট
```

VStack মানে ভার্টিকাল (উল্লম্ব) লেআউট কন্টেইনার। এর ভিতরের ভিউগুলো উপর থেকে নিচে সাজানো থাকে।

![Swift](../../RESOURCE/002_view8.png)

SwiftUI-তে সাধারণত তিন ধরণের লেআউট থাকে:

- VStack —— উল্লম্বভাবে সাজানো (Vertical)
- HStack —— অনুভূমিকভাবে সাজানো (Horizontal)
- ZStack —— একটির ওপর অন্যটি সাজানো (Z-axis / Overlay)

```swift
HStack { }  // অনুভূমিকভাবে সাজানো
ZStack { }  // ওভারলে বা স্তরীভূতভাবে সাজানো
```

লেআউটগুলোর বিন্যাস চিত্র:

![Swift](../../RESOURCE/002_view2.png)

উদাহরণস্বরূপ: HStack ব্যবহার করে অনুভূমিক বিন্যাস।

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

এখানে দেখা যাচ্ছে, গ্লোব আইকন এবং টেক্সটটি পাশাপাশি (横向) প্রদর্শিত হচ্ছে।

![Swift](../../RESOURCE/002_view3.png)

যখন আমরা পাশাপাশি কিছু সাজাতে চাই, তখন HStack ব্যবহার করি। আর যখন একটির ওপর অন্যটি রাখতে চাই, তখন ZStack ব্যবহার করি।

#### Image এবং SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

এই কোডটি একটি গ্লোব (পৃথিবী) আইকন প্রদর্শন করে। এই আইকনটি অ্যাপলের SF Symbols আইকন সিস্টেম থেকে নেওয়া হয়েছে।

![Swift](../../RESOURCE/002_view9.png)

কোডটির অর্থ হলো একটি গ্লোব আইকন দেখাও, যার সাইজ হবে বড় (large) এবং রঙ হবে সিস্টেমের টিন্ট কালার।

আমরা গ্লোব ছাড়াও অন্য আইকন দেখাতে পারি।

যেমন, একটি ব্যাকপ্যাক (ব্যাগ) দেখানো:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**কিভাবে অন্য আইকন দেখাবেন?**

আমাদের অ্যাপল থেকে সরবরাহকৃত অফিসিয়াল সিস্টেম আইকন লাইব্রেরি SF Symbols ব্যবহার করতে হবে।

Apple Developer ওয়েবসাইট থেকে [SF Symbols](../../RESOURCE/https://developer.apple.com/sf-symbols/) ডাউনলোড করুন।

![Swift](../../RESOURCE/002_sf.png)

SF Symbols অ্যাপটি ওপেন করুন।

![Swift](../../RESOURCE/002_sf1.png)

বাম দিকে আইকনের ক্যাটাগরি এবং ডান দিকে সংশ্লিষ্ট আইকনগুলো থাকে।

যেকোনো আইকনের ওপর রাইট ক্লিক করে "Copy Name" সিলেক্ট করুন। এই নামটি একটি স্ট্রিং।

উদাহরণস্বরূপ:

```
"globe"
"backpack"
"heart"
```

কপি করা নামটি `Image(systemName:)`-এর ভেতর বসালে সেই আইকনটি প্রদর্শিত হবে।

দ্রষ্টব্য: প্রতিটি SF Symbols আইকনের জন্য একটি নির্দিষ্ট ন্যূনতম সিস্টেম ভার্সন প্রয়োজন। যদি আপনার সিস্টেম ভার্সন পুরনো হয়, তবে আইকনটি নাও দেখাতে পারে। SF Symbols অ্যাপ থেকে এর কম্প্যাটিবিলিটি চেক করে নিন।

#### মডিফায়ার (Modifiers)

SwiftUI-তে মডিফায়ার (modifier) হলো ভিউয়ের চেহারা বা আচরণ পরিবর্তন করার একটি পদ্ধতি।

মডিফায়ারকে পোশাকের মতো কল্পনা করতে পারেন; যেমন আলাদা আলাদা পোশাকে মানুষকে আলাদা দেখায়, তেমনি মডিফায়ার ভিউয়ের রূপ বদলে দেয়।

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

এখানে `imageScale` এবং `foregroundStyle` হলো Image ভিউয়ের মডিফায়ার। এগুলো Image-এর মূল কন্টেন্ট পরিবর্তন না করে এর বাহ্যিক রূপ পরিবর্তন করে।

**১. imageScale**

```swift
.imageScale(.large)
```

এটি SF Symbols আইকনের আকার নিয়ন্ত্রণ করে:

- .small (ছোট)
- .medium (মাঝারি)
- .large (বড়)

![Swift](../../RESOURCE/002_view11.png)

আমরা বিভিন্ন অপশন বেছে নিয়ে আইকনটি ছোট বা বড় করে দেখাতে পারি।

**২. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

`foregroundStyle` ভিউয়ের সামনের রঙ (foreground color) নিয়ন্ত্রণ করে।

`.tint` মানে হলো বর্তমান এনভায়রনমেন্টের অ্যাকসেন্ট কালার (accent color), যা iOS-এ ডিফল্টভাবে নীল হয়।

আমরা যদি রঙ লাল করতে চাই:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Text ভিউ

Text হলো টেক্সট ভিউ, যা স্ট্রিং (লেখা) প্রদর্শনের জন্য ব্যবহৃত হয়।

```swift
Text("Hello, world!")
```

যেমন, আমার নাম দেখানো:

```swift
Text("FangJunyu")
```

দ্রষ্টব্য: স্ট্রিং বা টেক্সট অবশ্যই `""` ডাবল কোটেশনের ভেতরে লিখতে হবে।

আপনি আপনার নিজের নাম বা ফোন নম্বর লিখে পরীক্ষা করে দেখতে পারেন।

#### padding (মার্জিন বা প্যাডিং)

SwiftUI-তে `padding` ভিউয়ের কন্টেন্ট এবং তার সীমানার (border) মাঝে অতিরিক্ত খালি জায়গা যোগ করতে ব্যবহৃত হয়। একে "ইনার মার্জিন" বা ভেতরের ফাঁকা জায়গা বলা হয়।

```swift
HStack {
    ...
}
.padding()
```

উপরের কোডটি HStack ভিউতে সিস্টেমের ডিফল্ট প্যাডিং যোগ করে।

**প্যাডিং কী?**

প্যাডিং হলো "ভিউয়ের কন্টেন্ট এবং তার বর্ডারের মাঝখানের ফাঁকা জায়গা"।

নিচের ছবিতে, নীল রঙের HStack-এ প্যাডিং যোগ করার পর নীল এরিয়াটি ভেতরের দিকে সংকুচিত হয়ে আসে, ফলে ভিউটি দেখতে একটু ছোট মনে হয়।

![Swift](../../RESOURCE/002_view6.png)

**ডিফল্ট প্যাডিং**

`padding()` মডিফায়ারটি ডিফল্টভাবে সিস্টেমের স্ট্যান্ডার্ড স্পেসিং ব্যবহার করে।

```swift
.padding()
```

বিভিন্ন প্ল্যাটফর্মে এর মান ভিন্ন হতে পারে, যেমন:

- iOS-এ সাধারণত এটি ১৬ pt হয়।
- macOS বা watchOS-এ সিস্টেমের স্ট্যান্ডার্ড গ্যাপ ভিন্ন হতে পারে।

**কাস্টম প্যাডিং**

ভিউয়ের নির্দিষ্ট দিকেও প্যাডিং সেট করা যায়।

১. একটি নির্দিষ্ট দিকে সেট করা:

```swift
.padding(.top, 20)      // উপরে
.padding(.leading, 20)  // বামে (শুরুতে)
.padding(.trailing, 20) // ডানে (শেষে)
.padding(.bottom, 20)   // নিচে
```

দিকের বর্ণনা:

- .top: উপরের দিক
- .bottom: নিচের দিক
- .leading: সামনের দিক (বাম থেকে ডানে লেখা ভাষার জন্য বাম দিক)
- .trailing: পেছনের দিক (ডান দিক)

![Swift](../../RESOURCE/002_view12.png)

দ্রষ্টব্য: `leading` এবং `trailing` ভাষার দিক অনুযায়ী স্বয়ংক্রিয়ভাবে পরিবর্তিত হয়। যেমন আরবি ভাষার (RTL) ক্ষেত্রে এটি উল্টে যায়।

২. একাধিক দিকে সেট করা:

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

একটি অ্যাররে (array) ব্যবহার করে একসাথে একাধিক দিকে প্যাডিং দেওয়া যায়। অ্যাররের ব্যবহার আমরা পরের ক্লাসে বিস্তারিত শিখব।

৩. অনুভূমিক বা উল্লম্ব দিকে সেট করা:

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

এটি নিচের কোডের সমতুল্য:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**প্যাডিং মুছে ফেলা**

যদি কোনো প্যাডিং না চান, তবে `.padding(0)` ব্যবহার করতে পারেন অথবা মডিফায়ারটি মুছে ফেলতে পারেন:

```swift
.padding(0)
```

## সারাংশ

ContentView.swift ফাইলের কোড কম হলেও, এটি SwiftUI-এর অনেকগুলো মূল কনসেপ্ট কভার করে। নতুনদের কাছে এটি কিছুটা অপরিচিত মনে হতে পারে, তবে কোডটি ধাপে ধাপে বিশ্লেষণ করলে SwiftUI সম্পর্কে একটি প্রাথমিক ধারণা তৈরি হয়।

এই ক্লাসের সারাংশ হলো, আমরা প্রথমে মন্তব্য `//` সম্পর্কে শিখেছি যা লজিক লিখে রাখতে বা কোড বন্ধ রাখতে ব্যবহৃত হয়।

এরপর আমরা জেনেছি যে প্রতিটি SwiftUI ফাইলে অবশ্যই SwiftUI ফ্রেমওয়ার্ক ইম্পোর্ট করতে হবে:

```swift
import SwiftUI
```

তা না হলে কম্পাইলার View টাইপ চিনতে পারবে না।

এরপর আমরা SwiftUI ভিউয়ের বেসিক স্ট্রাকচার দেখেছি:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

এখানে `ContentView` হলো আমাদের ভিউয়ের নাম।

আমরা তিনটি প্রধান লেআউট কন্টেইনার শিখেছি: VStack (উল্লম্ব), HStack (অনুভূমিক) এবং ZStack (স্তরীভূত)।

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

এই কোডটি একটি উল্লম্ব লেআউটে একটি আইকন এবং একটি টেক্সট দেখাচ্ছে।

Image দিয়ে SF Symbols আইকন দেখানো হয় এবং মডিফায়ার দিয়ে এর সাইজ ও রঙ পরিবর্তন করা হয়।

Text ভিউ টেক্সট প্রদর্শনের জন্য ব্যবহৃত হয়।

padding হলো ভিউয়ের চারপাশের খালি জায়গা।

সবশেষে আমরা দেখেছি `#Preview`, যা দিয়ে ক্যানভাসে কোডের ফলাফল দেখা যায়।

### ক্লাস পরবর্তী অনুশীলন

নতুনদের জন্য এই বিষয়গুলো একটু জটিল মনে হতে পারে। ভালো করে বোঝার জন্য নিচের অনুশীলনগুলো করুন:

- SF Symbols আইকনের নাম পরিবর্তন করে দেখুন।
- আইকনের রঙ কালো (black) করে দেখুন।
- VStack-কে HStack-এ পরিবর্তন করে দেখুন।
- Image বা Text-কে মন্তব্য (comment out) করে দেখুন ক্যানভাসে কী পরিবর্তন হয়।

### বোনাস: কোড কমপ্লিশন (Code Completion)

কোড টাইপ করার সময় আপনি হয়তো লক্ষ্য করেছেন যে Xcode স্বয়ংক্রিয়ভাবে কিছু অপশন দেখায়।

যেমন, যখন আমরা `imageScale` মডিফায়ার পরিবর্তন করি:

```swift
.imageScale(.)
```

Xcode সাথে সাথে কিছু অপশন দেখাবে:

![Swift](../../RESOURCE/002_view10.png)

একে বলা হয় কোড কমপ্লিশন (Code Completion)। এটি টাইপ ইনফারেন্স এবং এনাম (enum) মেম্বারদের ওপর ভিত্তি করে সাজেস্ট করে, যা দ্রুত কোড লিখতে এবং ভুল কমাতে সাহায্য করে।

পরবর্তী ক্লাসগুলোতে আমরা এনাম (enum) সম্পর্কে বিস্তারিত জানব। আপাতত জেনে রাখুন যে এটি ডেভেলপারদের কাজ অনেক সহজ করে দেয়।
