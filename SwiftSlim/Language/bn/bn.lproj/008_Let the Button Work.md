# বাটনগুলো কাজ করানো

এই পাঠে আমরা মূলত Button (বাটন), ডিবাগিং তথ্য আউটপুট করার জন্য print, এবং Swift এর ফাংশন (Functions) সম্পর্কে শিখব।

এই বিষয়গুলো SwiftUI এর ইন্টারেক্টিভ প্রোগ্রামিংয়ের ভিত্তি।

## বাটন (Button)

বাটন হল আমাদের সবচেয়ে সাধারণ UI কন্ট্রোল। যখন আমাদের কোনো কিছু ডাউনলোড, ওপেন বা প্রস্থান করার প্রয়োজন হয়, তখন আমরা সাধারণত বাটনে ক্লিক করি।

উদাহরণস্বরূপ, App Store-এ অ্যাপ ডাউনলোড করার জন্য "Get" (পান) বাটনে ক্লিক করা হয়।

![Button](../../RESOURCE/008_button.png)

SwiftUI তে, বাটনকে Button দিয়ে প্রকাশ করা হয়।

প্রাথমিক ব্যবহার:

```swift
Button("") {
    
}
```

এর গঠনকে দুটি ভাগে ভাগ করা যায়:

```swift
Button("বাটনের টেক্সট") {
    বাটনে ক্লিক করলে যে কোড এক্সিকিউট হবে
}
```

উদাহরণস্বরূপ:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

রান করার পর একটি Start বাটন প্রদর্শিত হবে।

![Button](../../RESOURCE/008_button1.png)

যখন ব্যবহারকারী বাটনে ক্লিক করেন, তখন {} এর ভেতরের কোড এক্সিকিউট বা রান হয়।

একটি বিষয় খেয়াল রাখতে হবে: Button হল SwiftUI-এর একটি ইন্টারফেস কন্ট্রোল (View), তাই একে অবশ্যই body-র ভেতরে লিখতে হবে।

### buttonStyle মডিফায়ার (Modifier)

SwiftUI-তে, Button সিস্টেমে আগে থেকেই তৈরি করা বাটনের স্টাইল প্রদান করে, যা buttonStyle মডিফায়ারের মাধ্যমে ব্যবহার করা যেতে পারে।

উদাহরণস্বরূপ:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

buttonStyle সিস্টেমের দেওয়া বাটনের স্টাইল ব্যবহার করবে।

সাধারণ অপশনগুলোর মধ্যে রয়েছে:

- .automatic
- .plain
- .bordered
- .borderedProminent
- .borderless

![Button](../../RESOURCE/008_button7.png)

বিভিন্ন স্টাইলের পার্থক্য মূলত বাটনে বর্ডার, ব্যাকগ্রাউন্ড এবং ভিজ্যুয়াল এমফাসিস বা দৃষ্টি আকর্ষণের মাত্রার উপর নির্ভর করে।

## print আউটপুট

প্রোগ্রামিংয়ে, প্রায়ই আমাদের দেখতে হয় প্রোগ্রামটি কোনো নির্দিষ্ট কোডের অংশে রান করেছে কিনা।

ডিবাগিং তথ্য আউটপুট করার জন্য Swift print ফাংশন প্রদান করে।

প্রাথমিক লেখার নিয়ম:

```swift
print("Hello")
```

এই লাইনটির কোড টেক্সটটিকে Console-এ (কনসোলে) আউটপুট করবে।

### বাটনের ব্যবহারযোগ্যতা পরীক্ষা করা

বাটনটি ক্লিক করা হয়েছে কিনা তা পরীক্ষা করতে আমরা বাটনের ভেতরে print লিখতে পারি।

```swift
Button("Start") {
    print("Test 123")
}
```

বাটনে ক্লিক করা হলে, Console-এ আউটপুট হবে:

```
Test 123
```

এর মানে হলো বাটনটি সফলভাবে ট্রিগার বা কাজ করেছে।

![Button](../../RESOURCE/008_button3.png)

লক্ষ্য করুন: print হলো Swift কোড, কোনো ইন্টারফেস কন্ট্রোল নয়। এটিকে সরাসরি body-র ভেতরে রাখা যাবে না, রাখলে এরর (error) দেখাবে।

উদাহরণস্বরূপ, নিচের মতো করে লেখাটি ভুল:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // ভুল: body কে অবশ্যই একটি View রিটার্ন করতে হবে
    }
}
```

SwiftUI-তে, body ইন্টারফেসের স্ট্রাকচার বা গঠন বর্ণনা করতে ব্যবহৃত হয়, এবং এতে কেবল ইন্টারফেস কন্ট্রোল (View) রাখা যায়, যেমন Text, Button, Image ইত্যাদি।

সাধারণ কোড (ক্যালকুলেশন লজিক, print ইত্যাদি) অবশ্যই বাটন, ফাংশন বা অন্য কোনো ইভেন্ট হ্যান্ডলারের ভেতরে রাখতে হবে।

### Console কনসোল

print এর আউটপুটের বিষয়বস্তু Xcode এর Console (কনসোল)-এ প্রদর্শিত হবে।

যদি Console দেখতে না পান, তাহলে নিচের ডানদিকের Show the Debug Area বাটনে ক্লিক করে ডিবাগিং এরিয়া খুলুন এবং তারপরে Show the Console বাটনে ক্লিক করে কনসোলটি প্রদর্শন করুন।

![](../../RESOURCE/008_button4.png)

Console এর নিচের বাম কোণে দুটি ট্যাব আছে: Executable এবং Previews।

![](../../RESOURCE/008_button5.png)

এই দুটি ট্যাব ভিন্ন ভিন্ন রানটাইম এনভায়রনমেন্টের জন্য: যখন Canvas-এ প্রিভিউ দেখা হয়, তখন print এর আউটপুট Previews ট্যাবের নিচে দেখাবে; আর যদি সিমুলেটর বা আসল ডিভাইসে অ্যাপটি রান করা হয়, তবে আউটপুট Executable ট্যাবের নিচে প্রদর্শিত হবে।

তাই, যখন আপনি দেখবেন print এর কোনো আউটপুট আসছে না, তখন প্রথমেই নিশ্চিত করুন যে আপনি সঠিক ট্যাবটি নির্বাচন করেছেন কিনা।

## উদাহরণ - ASCII ক্যারেক্টার আর্ট

কিছু কোড প্রোজেক্টে, প্রায়ই ASCII ক্যারেক্টার আর্ট দেখা যায়।

ক্যারেক্টার আর্ট হলো সাধারণ অক্ষর দিয়ে তৈরি করা প্যাটার্ন বা ছবি, যেমন:

![ASCII](../../RESOURCE/008_ascii.png)

আমরা বাটন + print ব্যবহার করে একটি ASCII ক্যারেক্টার আর্ট আউটপুট করতে পারি।

উদাহরণের কোড:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

বাটনে ক্লিক করলে, Console-এ আউটপুট হবে:

```
 /\_/\
( o.o )
 > ^ <
```

লক্ষ্য করুন: স্ট্রিংয়ের ভেতরে \ কে অবশ্যই \\\ হিসেবে লিখতে হবে, কারণ ব্যাকস্ল্যাশ হলো স্ট্রিংয়ের মধ্যে একটি এস্কেপ ক্যারেক্টার (escape character)।

## ফাংশন (Function)

বাটনের ভেতরে, আমরা সরাসরি কোড লিখতে পারি।

উদাহরণস্বরূপ:

```swift
Button("Start") {
    print("Hello")
}
```

যদি কোড তুলনামূলক কম হয়, তবে এভাবে লিখলে কোনো সমস্যা নেই। কিন্তু বাস্তব ডেভেলপমেন্টে, একটি বাটন অনেকগুলো লজিক ট্রিগার করতে পারে।

উদাহরণস্বরূপ:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // আরো অনেক কোড থাকতে পারে
}
```

যদি কোড ধীরে ধীরে বাড়তে থাকে, তবে সরাসরি বাটনের ভেতরে লিখলে ইন্টারফেসের কোড খুব এলোমেলো হয়ে যাবে, এবং এটি পড়া ও মেইনটেইন করা সুবিধাজনক হবে না।

অতএব, আমরা সাধারণত এই কোডগুলোকে গুছিয়ে একটি ফাংশনের মধ্যে রাখি, এবং তারপর বাটনের ভেতর থেকে সেই ফাংশনটি কল করি।

### ফাংশন কী?

ফাংশন হলো এমন এক টুকরো কোড যা বারবার ব্যবহার করা যায়।

যখন নির্দিষ্ট কোনো কোড এক্সিকিউট বা রান করার প্রয়োজন হয়, তখন আমাদের শুধু এই ফাংশনটি কল করতে হয়, তাহলেই ফাংশনের ভেতরের কোড রান হবে।

এটি কোডের গঠনকে আরও পরিষ্কার করে তোলে এবং পুনরায় ব্যবহারের জন্য সুবিধাজনক করে।

### প্রাথমিক লেখার নিয়ম

Swift-এ, ফাংশন সংজ্ঞায়িত বা ডিফাইন করতে **func** কীওয়ার্ড ব্যবহার করা হয়:

```swift
func randomInt() {
    // code
}
```

এই কোডটি একটি ফাংশন ডিফাইন করে।

randomInt হলো ফাংশনটির নাম, যা এই কোডটিকে চিহ্নিত করতে ব্যবহৃত হয়।

() প্যারামিটারের স্থান নির্দেশ করে, যেখানে বাহির থেকে আসা ডেটা গ্রহণ করা যায়। যদি কোনো ডেটার প্রয়োজন না হয়, তবে এটি ফাঁকা রাখা যেতে পারে।

{} এর ভেতরটা হলো ফাংশনের কোড এরিয়া, যেখানে রান করার জন্য প্রয়োজনীয় সব কোড লেখা হয়।

উদাহরণস্বরূপ:

```swift
func getName() {
    print("FangJunyu")
}
```

এই ফাংশনটির কাজ হলো একটি টেক্সট আউটপুট করা।

### ফাংশন কল করা

ফাংশন ডিফাইন করা মানে শুধুমাত্র কোডটি তৈরি করা, যদি এটি রান করতে চান, তবে ফাংশনটি কল করতে হবে।

কল করার নিয়ম হলো ফাংশনের নামের শেষে () যুক্ত করা:

```swift
getName()
```

যখন প্রোগ্রামটি এই লাইনের কোড পর্যন্ত পৌঁছায়, তখন এটি getName ফাংশনের ভেতরের কোড এক্সিকিউট করে।

সম্পূর্ণ উদাহরণ:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

রান করার পর, Console-এ আউটপুট হবে:

```
FangJunyu
```

### SwiftUI-তে ফাংশনের ব্যবহার

SwiftUI ভিউতে, ফাংশন সাধারণত body-র বাইরে লেখা হয়।

উদাহরণস্বরূপ:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

বাটনে ক্লিক করা হলে, getName() ফাংশনটি কল হবে।

এর ফলে ইন্টারফেসের কোড পরিচ্ছন্ন থাকে এবং নির্দিষ্ট লজিক ফাংশনের ভেতরে প্রসেস করা যায়।

### ফাংশন প্যারামিটার

মাঝে মাঝে, ভিন্ন ভিন্ন ডেটার উপর ভিত্তি করে একটি ফাংশনকে ভিন্ন ভিন্ন কাজ করতে হয়।

উদাহরণস্বরূপ, আমরা চাই ফাংশনটি শুধু নির্দিষ্ট "FangJunyu" আউটপুট না করে, বিভিন্ন নাম আউটপুট করুক।

এই ক্ষেত্রে প্যারামিটার ব্যবহার করা যেতে পারে, প্যারামিটারকে এভাবে বোঝা যায়: ফাংশন কল করার সময় যে ডেটা প্রদান করা হয়।

উদাহরণস্বরূপ:

```swift
func getName(name: String) {
    print(name)
}
```

এই ফাংশনটিতে একটি নতুন প্যারামিটার যোগ করা হয়েছে, name হলো প্যারামিটারের নাম এবং String হলো প্যারামিটারের টাইপ।

এর মানে হলো ফাংশনটির একটি String টাইপের ডেটা গ্রহণ করা প্রয়োজন।

যেহেতু ফাংশনটির এখন ডেটা প্রয়োজন, তাই কল করার সময় অবশ্যই এই ডেটা প্রদান করতে হবে।

```swift
getName(name: "Sam")
```

যখন কল করার সময় "Sam" পাঠানো হয়, ফাংশনটি আউটপুট করার জন্য ভেতরে এই মানটি ব্যবহার করবে।

রান করার ফলাফল:

```
Sam
```

## উদাহরণ - ফাংশন ব্যবহার করে ASCII ক্যারেক্টার আর্ট আউটপুট করা

আগের উদাহরণে, আমরা print সরাসরি বাটনের ভেতরে লিখেছিলাম।

কিন্তু যদি ক্যারেক্টার আর্টটি বেশি জটিল হয়, তবে কোডটি অনেক বড় হয়ে যাবে। তখন আপনি লজিকটি একটি ফাংশনের মধ্যে রাখতে পারেন, যাতে ইন্টারফেসের কোড আরও পরিষ্কার থাকে।

উদাহরণের কোড:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

রান করার পর বাটনে ক্লিক করলে, Console-এ আউটপুট হবে:

```
 /\_/\
( o.o )
 > ^ <
```

এখানে দুটি জিনিস ঘটেছে, যখন ব্যবহারকারী বাটনে ক্লিক করেন:

1. বাটনটি printCat() ফাংশনকে কল করে
2. ফাংশনের ভেতরের print কোডগুলো এক্সিকিউট হয়

এভাবে ইন্টারফেসের কোড শুধুমাত্র ইভেন্ট ট্রিগার করার দায়িত্ব পালন করে, আর নির্দিষ্ট লজিক ফাংশনে প্রসেস হয়।

বাস্তব ডেভেলপমেন্টে, অধিকাংশ বাটনের অপারেশন বা কাজ ফাংশনের মাধ্যমেই বাস্তবায়িত করা হয়।

## সারাংশ

এই পাঠে আমরা তিনটি গুরুত্বপূর্ণ বিষয় শিখেছি:

**১. Button (বাটন)**

Button হলো SwiftUI-এর একটি কন্ট্রোল যা কোনো কাজ ট্রিগার করতে ব্যবহৃত হয়।

```swift
Button("Start") {

}
```

যখন ব্যবহারকারী বাটনে ক্লিক করেন, তখন কোঁকড়ানো বন্ধনীর (curly braces) ভেতরের কোড রান হবে।

**২. print আউটপুট**

print ব্যবহার করা হয় Console-এ ডিবাগিং তথ্য আউটপুট করার জন্য।

```swift
print("Hello")
```

ডেভেলপাররা Console-এর মাধ্যমে প্রোগ্রামের রানিং স্টেট বা অবস্থা দেখতে পারেন।

**৩. ফাংশন (Functions)**

ফাংশন হলো এমন এক টুকরো কোড যা বারবার ব্যবহার করা যায়।

```swift
func sayHello() {
    print("Hello")
}
```

ফাংশন কল করা:

```swift
sayHello()
```

যদি ফাংশনের ডেটা প্রয়োজন হয়, তবে প্যারামিটার ব্যবহার করা যেতে পারে:

```swift
func sayHello(name: String) {
    print(name)
}
```

পরবর্তী পাঠগুলোতে, আমরা আরও অনেক SwiftUI কন্ট্রোল এবং কীভাবে ডেটা ইন্টারফেসের সাথে যোগাযোগ বা ইন্টারেক্ট করে তা শিখতে থাকব।

## অনুশীলন

নিচের অনুশীলনটি সম্পন্ন করার চেষ্টা করুন:

1. একটি বাটন তৈরি করুন, বাটনের টেক্সট হবে "Hello"
2. বাটনে ক্লিক করা হলে, Console-এ Hello Swift আউটপুট করুন
3. আউটপুট কোডটি একটি ফাংশনের মধ্যে লিখুন, এবং তারপর বাটনের ভেতর থেকে ফাংশনটিকে কল করুন
