# কম্পিউটেড প্রপার্টি

এই পাঠে আমরা মূলত কম্পিউটেড প্রপার্টি শিখব।

কম্পিউটেড প্রপার্টি বিদ্যমান ডেটার উপর ভিত্তি করে নতুন একটি ফলাফল হিসাব করতে ব্যবহৃত হয়। এটি সংখ্যাগত মান হিসাব করতে পারে, আবার SwiftUI ভিউতে দেখাতে হবে এমন বিষয়বস্তুও হিসাব করতে পারে।

উদাহরণ:

```swift
let a = 10
let b = 20
let c = a + b
```

এখানে `c` মানে `a` এবং `b` যোগ করার পরের ফলাফল।

সাধারণ কোডে এই ধরনের লেখা খুবই প্রচলিত।

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

টিপ: `Button`-এর ট্যাপ ইভেন্টের মধ্যে সাধারণ Swift কোড চালানো যায়।

কিন্তু একই ধরনের কোড যদি সরাসরি SwiftUI ভিউর প্রপার্টির মধ্যে লিখি, তাহলে সমস্যা হবে।

উদাহরণ:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

এই কোডে ত্রুটি হবে।

দেখতে মনে হয় `a` এবং `b` তো `c`-এর আগে লেখা হয়েছে, তাই যেন `c`-এর মান হিসাব করা সম্ভব।

কিন্তু স্ট্রাকচারের প্রপার্টি ঘোষণার মধ্যে এইভাবে সরাসরি ব্যবহার করা যায় না।

## কেন c সরাসরি a + b হিসাব করতে পারে না

কারণ `a`, `b`, `c` বোতামের ট্যাপ ইভেন্টের অস্থায়ী কনস্ট্যান্ট নয়; এগুলো `ContentView` ভিউর প্রপার্টি।

বোতামের ট্যাপ ইভেন্টের মধ্যে এই কোড স্বাভাবিকভাবে চলতে পারে:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

কারণ বোতাম ট্যাপ করার পর কোড ক্রমানুসারে চলে। আগে `a` তৈরি হয়, তারপর `b` তৈরি হয়, শেষে `a + b` দিয়ে `c` হিসাব করা হয়।

কিন্তু ভিউর ভেতরে প্রপার্টি ঘোষণা করার সময় পরিস্থিতি আলাদা:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

এখানে `a`, `b`, `c` সবই `ContentView` ভিউর প্রপার্টি।

`ContentView` ভিউ তৈরি করার সময় Swift-কে আগে এই প্রপার্টিগুলো প্রস্তুত করতে হয়। তৈরি হওয়ার প্রক্রিয়াটি নিরাপদ রাখতে Swift কোনো ইনস্ট্যান্স স্টোরড প্রপার্টির ডিফল্ট মানকে একই ইনস্ট্যান্সের অন্য ইনস্ট্যান্স প্রপার্টি সরাসরি পড়তে দেয় না।

তাই এই লাইনটি ত্রুটি দেবে:

```swift
let c = a + b
```

সহজভাবে বোঝা যায়: **ভিউর ভেতরে প্রপার্টি ঘোষণা করার সময়, একটি সাধারণ প্রপার্টি দিয়ে সরাসরি আরেকটি সাধারণ প্রপার্টি হিসাব করা যায় না।**

যে প্রপার্টি সরাসরি মান সংরক্ষণ করে, তাকে "স্টোরড প্রপার্টি" বলা হয়। বোঝার সুবিধার জন্য আপাতত এটিকে সাধারণ প্রপার্টি হিসেবেও ভাবতে পারি।

উদাহরণ:

```swift
let a = 10
```

`a` সংরক্ষণ করে `10`।

```swift
let b = 20
```

`b` সংরক্ষণ করে `20`।

কিন্তু:

```swift
let c = a + b
```

এখানে `c` সরাসরি লিখে দেওয়া মান নয়; বরং `a + b` দিয়ে হিসাব করে পেতে চাই।

এই ধরনের “বিদ্যমান ডেটা থেকে ফলাফল পাওয়া” অবস্থায় কম্পিউটেড প্রপার্টি বেশি উপযুক্ত।

এভাবে পরিবর্তন করা যায়:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

এখানে `c` হলো কম্পিউটেড প্রপার্টি।

```swift
var c: Int {
    return a + b
}
```

এর অর্থ হলো: **যখন `c` ব্যবহার করতে হবে, তখন `a` এবং `b` পড়ে `a + b`-এর ফলাফল ফেরত দাও।**

উদাহরণ:

```swift
Text("c: \(c)")
```

`Text` যখন `c` দেখায়, তখনই `c`-এর হিসাব চালু হয়।

## কম্পিউটেড প্রপার্টি কী?

কম্পিউটেড প্রপার্টি দেখতে একটি ভেরিয়েবলের মতো, কিন্তু এটি নিজে ডেটা সংরক্ষণ করে না।

উদাহরণ:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

এখানে `c` হলো কম্পিউটেড প্রপার্টি।

এটি সাধারণ প্রপার্টির মতো নির্দিষ্ট একটি মান সংরক্ষণ করে না; বরং প্রতিবার `c` পড়া হলে `{}`-এর কোড আবার চালায় এবং হিসাবের ফলাফল ফেরত দেয়।

এভাবে বোঝা যায়:

```swift
var c: Int {
    return a + b
}
```

যখন `c` ব্যবহার করতে হবে, তখন `a + b` হিসাব করা হবে।

তাই কম্পিউটেড প্রপার্টি এই ধরনের অবস্থার জন্য উপযুক্ত: **একটি ফলাফল আলাদা করে সংরক্ষণ করার দরকার নেই, বিদ্যমান ডেটা থেকে হিসাব করা যায়।**

## মৌলিক লেখার ধরন

কম্পিউটেড প্রপার্টিতে সাধারণত তিনটি অংশ থাকে:

```swift
var c: Int {
    return a + b
}
```

### ১. `var` দিয়ে ঘোষণা করা

```swift
var c
```

কম্পিউটেড প্রপার্টি অবশ্যই `var` দিয়ে ঘোষণা করতে হয়, `let` ব্যবহার করা যায় না।

কারণ কম্পিউটেড প্রপার্টি স্থিরভাবে সংরক্ষিত কোনো মান নয়; বরং প্রতিবার পড়ার সময় ডায়নামিকভাবে হিসাব করা ফলাফল।

### ২. ফেরত আসা টাইপ লিখে দেওয়া

```swift
var c: Int
```

কম্পিউটেড প্রপার্টিতে ফেরত আসা টাইপ লিখতে হয়।

এখানে `c` শেষ পর্যন্ত একটি পূর্ণসংখ্যা ফেরত দেবে, তাই টাইপ হলো `Int`।

### ৩. {} ব্যবহার করে হিসাবের লজিক লেখা

```swift
{
    return a + b
}
```

`{}`-এর মধ্যে লেখা থাকে হিসাবের লজিক; এখানে ফেরত আসছে `a + b`।

## return কীওয়ার্ড

কম্পিউটেড প্রপার্টিকে একটি ফলাফল ফেরত দিতে হয়।

উদাহরণ:

```swift
var c: Int {
    return a + b
}
```

এখানে `return` মানে: `a + b`-এর হিসাবের ফলাফল বাইরে ফেরত দাও।

যদি কম্পিউটেড প্রপার্টিতে শুধু একটি সরাসরি ফলাফল তৈরি করা এক্সপ্রেশন থাকে, তাহলে `return` বাদ দেওয়া যায়:

```swift
var c: Int {
    a + b
}
```

কিন্তু যদি কম্পিউটেড প্রপার্টিতে একাধিক লাইনের কোড থাকে, তাহলে ফলাফল স্পষ্টভাবে ফেরত দিতে `return` ব্যবহার করতে হয়।

উদাহরণ:

```swift
var totalPriceText: String {
    let total = count * price
    return "মোট: \(total) $"
}
```

এখানে হিসাবের প্রক্রিয়া দুই ধাপে ভাগ করা হয়েছে।

প্রথম ধাপ, আগে মোট দাম হিসাব করা:

```swift
let total = count * price
```

দ্বিতীয় ধাপ, মোট দামকে টেক্সট হিসেবে জোড়া লাগিয়ে ফেরত দেওয়া:

```swift
return "মোট: \(total) $"
```

যদি `return` সরিয়ে দিই:

```swift
var totalPriceText: String {
    let total = count * price
    "মোট: \(total) $"
}
```

এই কোডে ত্রুটি হবে।

কারণ হলো: **এই কম্পিউটেড প্রপার্টিতে ইতিমধ্যে একাধিক লাইনের কোড আছে, Swift আর স্বয়ংক্রিয়ভাবে বুঝতে পারে না কোন লাইনটি শেষ পর্যন্ত ফেরত দেওয়ার ফলাফল।**

তাই, যখন কম্পিউটেড প্রপার্টিতে শুধু এক লাইনের ফলাফল থাকে, তখন `return` বাদ দেওয়া যায়।

```swift
var c: Int {
    a + b
}
```

যখন কম্পিউটেড প্রপার্টিতে একাধিক লাইনের কোড থাকে, তখন স্পষ্টভাবে `return` লিখতে পরামর্শ দেওয়া হয়।

```swift
var totalPriceText: String {
    let total = count * price
    return "মোট: \(total) $"
}
```

## কম্পিউটেড প্রপার্টি এবং সাধারণ প্রপার্টির পার্থক্য

সাধারণ প্রপার্টি ডেটা সংরক্ষণ করে।

```swift
var c = 30
```

এখানে `c` একটি নির্দিষ্ট মান সংরক্ষণ করে: `30`।

কম্পিউটেড প্রপার্টি ডেটা সংরক্ষণ করে না।

```swift
var c: Int {
    a + b
}
```

এখানে `c` `30` সংরক্ষণ করে না। এটি শুধু একটি হিসাবের পদ্ধতি দেয়।

`c` পড়ার সময় Swift চালায়:

```swift
a + b
```

তারপর হিসাবের ফলাফল ফেরত দেয়।

তাই কম্পিউটেড প্রপার্টি অন্য ডেটা থেকে ফলাফল হিসাব করার অবস্থার জন্য উপযুক্ত।

## body-ও কম্পিউটেড প্রপার্টি

কম্পিউটেড প্রপার্টি শেখার পর আমরা SwiftUI-র সবচেয়ে সাধারণ একটি কোড নতুনভাবে বুঝতে পারি:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

এখানে `body`-ও একটি কম্পিউটেড প্রপার্টি।

এটি `var` দিয়ে ঘোষণা করা হয়েছে:

```swift
var body
```

এর একটি ফেরত আসা টাইপ আছে:

```swift
some View
```

এর `{}`-এর মধ্যে ফেরত আসে SwiftUI ভিউর বিষয়বস্তু:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

তাই `body`-কে এভাবে বোঝা যায়: যখন SwiftUI এই ভিউ দেখাতে চায়, তখন এটি `body` পড়ে এবং `body` যে বিষয়বস্তু ফেরত দেয় তার ভিত্তিতে ইন্টারফেস তৈরি করে।

যদি `return` লিখি, তাহলে এভাবে বোঝা যায়:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

তবে SwiftUI-তে আমরা সাধারণত `return` বাদ দিয়ে সরাসরি লিখি:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

যখন `@State` ডেটা পরিবর্তিত হয়, SwiftUI আবার `body` পড়ে এবং নতুন ডেটা অনুযায়ী ইন্টারফেস আপডেট করে।

উদাহরণ:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("সংখ্যা: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

বোতাম ট্যাপ করার পর `count` বদলাবে।

```swift
count += 1
```

`count` বদলানোর পর SwiftUI আবার `body` হিসাব করবে, তাই ইন্টারফেসের টেক্সটও আপডেট হবে।

```swift
Text("সংখ্যা: \(count)")
```

এ কারণেই SwiftUI-তে ডেটা বদলানোর পর ইন্টারফেস স্বয়ংক্রিয়ভাবে রিফ্রেশ হতে পারে।

### body-তে জটিল হিসাব লেখা সুপারিশ করা হয় না

যেহেতু `body` একটি কম্পিউটেড প্রপার্টি, এটি বহুবার পড়া এবং আবার হিসাব করা হতে পারে।

উদাহরণ:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("সংখ্যা: \(num)")
            Text("নাম: \(name)")
        }
    }
}
```

`name` বদলালে SwiftUI আবার `body` হিসাব করবে।

তখন `body`-এর ভেতরের কোডও আবার চলবে, এবং `num`-ও আবার তৈরি হবে।

```swift
let num = 10
```

এই উদাহরণের `num` খুব সহজ, তাই প্রভাব বেশি নয়।

কিন্তু যদি `body`-তে জটিল হিসাব চালানো হয়, যেমন অনেক ডেটা ফিল্টার করা, সাজানো, ছবি প্রসেস করা ইত্যাদি, তাহলে ইন্টারফেসের মসৃণতা প্রভাবিত হতে পারে।

তাই SwiftUI-তে `body`-র প্রধান কাজ হওয়া উচিত ইন্টারফেসের গঠন বর্ণনা করা।

সহজ অস্থায়ী ডেটা `body`-র মধ্যে লেখা যায়।

জটিল হিসাব `body`-র বাইরের কম্পিউটেড প্রপার্টি, মেথড, অথবা আলাদা ডেটা প্রসেসিংয়ে রাখা যেতে পারে।

## উদাহরণ: পরিমাণ এবং মোট দাম

নিচে একটি সহজ উদাহরণের মাধ্যমে আমরা কম্পিউটেড প্রপার্টি বুঝব।

ধরা যাক একটি গাজরের একক দাম 2 $, ব্যবহারকারী বোতাম ট্যাপ করে কেনার পরিমাণ পরিবর্তন করতে পারে, আর ইন্টারফেসে মোট দাম দেখাতে হবে।

মোট দাম হিসাবের পদ্ধতি:

```
পরিমাণ * একক দাম
```

যদি মোট দাম সংরক্ষণ করতে সাধারণ ভেরিয়েবল ব্যবহার করি, তাহলে ব্যাপারটি কিছুটা ঝামেলাপূর্ণ হবে।

কারণ প্রতি বার পরিমাণ বদলালে মোট দাম হাতে করে আপডেট করতে হবে।

ভালো পদ্ধতি হলো কম্পিউটেড প্রপার্টি ব্যবহার করা:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("দাম: \(price) $")
                Text("পরিমাণ: \(count)")
                Text("মোট দাম: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

দেখানোর ফলাফল:

![view](../../Resource/024_view.png)

এই উদাহরণে:

```swift
@State private var count = 1
```

`count` গাজরের পরিমাণ বোঝায়। বোতাম ট্যাপ করার সময় `count` বদলাবে।

```swift
private let price = 2
```

`price` গাজরের একক দাম বোঝায়। এখানে এটি স্থির মান, তাই `let` ব্যবহার করা হয়েছে।

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` মোট দাম বোঝায়।

এটি আলাদা করে সংরক্ষণ করার দরকার নেই, কারণ মোট দাম সবসময় `count * price` দিয়ে হিসাব করা যায়।

যখন `count` হলো `1`:

```swift
totalPrice = 1 * 2
```

দেখানো ফলাফল:

```swift
মোট দাম: 2 $
```

`+` বোতাম ট্যাপ করার পর `count` হয়ে যায় `2`।

এই সময় আবার `totalPrice` পড়লে সেটি আবার হিসাব হবে:

```swift
totalPrice = 2 * 2
```

দেখানো ফলাফল:

```swift
মোট দাম: 4 $
```

এটাই কম্পিউটেড প্রপার্টির কাজ: বিদ্যমান ডেটা অনুযায়ী ডায়নামিকভাবে নতুন ফলাফল হিসাব করা।

## কম্পিউটেড প্রপার্টি শর্ত নির্ধারণেও ব্যবহার করা যায়

কম্পিউটেড প্রপার্টি শুধু সংখ্যাগত মান হিসাব করতে পারে না, শর্তের ফলাফলও ফেরত দিতে পারে।

উদাহরণস্বরূপ, যখন আমরা চাই পরিমাণের সর্বনিম্ন মান 1 হোক।

পরিমাণ ইতিমধ্যেই 1 হলে `-` বোতাম আর কমানো উচিত নয়।

একটি কম্পিউটেড প্রপার্টি যোগ করা যায়:

```swift
private var canDecrease: Bool {
    count > 1
}
```

সম্পূর্ণ কোড:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("দাম: \(price) $")
                Text("পরিমাণ: \(count)")
                Text("মোট দাম: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

দেখানোর ফলাফল:

![view](../../Resource/024_view1.png)

এখানের:

```swift
private var canDecrease: Bool {
    count > 1
}
```

এই কোডের অর্থ হলো, এখন কমানো যাবে কি না।

যখন `count` `1`-এর চেয়ে বড়:

```swift
canDecrease == true
```

মানে কমানো যাবে।

যখন `count` `1`-এর সমান:

```swift
canDecrease == false
```

মানে আর কমানো যাবে না।

### বোতামের মধ্যে শর্ত যাচাই

বোতামে ব্যবহার করা হয়েছে:

```swift
if canDecrease {
    count -= 1
}
```

শুধু `canDecrease`-এর ফলাফল `true` হলে `count` কমানো যাবে।

### ভিউর foreground color নিয়ন্ত্রণ

`canDecrease` ব্যবহার করে বোতামের foreground color-ও নিয়ন্ত্রণ করা যায়:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

এখানে ternary operator ব্যবহার করা হয়েছে:

```swift
canDecrease ? Color.primary : Color.gray
```

এই কোডের অর্থ হলো, যদি `canDecrease` `true` হয়, foreground color হিসেবে `Color.primary` ব্যবহার করো; যদি `canDecrease` `false` হয়, foreground color হিসেবে `Color.gray` ব্যবহার করো।

`Color.primary` হলো SwiftUI দেওয়া সিস্টেম semantic color, যা বর্তমান ইন্টারফেসের প্রধান টেক্সটের রং বোঝায়।

লাইট মোডে `Color.primary` সাধারণত কালোর কাছাকাছি; ডার্ক মোডে `Color.primary` সাধারণত সাদার কাছাকাছি।

তাই `Color.primary` ব্যবহারের সুবিধা হলো: এটি লাইট মোড এবং ডার্ক মোডের সঙ্গে স্বয়ংক্রিয়ভাবে মানিয়ে নেয়।

### ভিউর নিষ্ক্রিয় অবস্থা নিয়ন্ত্রণ

`disabled` ব্যবহার করা হয় ভিউ নিষ্ক্রিয় অবস্থায় আছে কি না তা নিয়ন্ত্রণ করতে:

```swift
.disabled(!canDecrease)
```

যখন `disabled` হলো `false`, ভিউ ট্যাপ করা যায়।

যখন `disabled` হলো `true`, ভিউ নিষ্ক্রিয় অবস্থায় থাকে, ট্যাপ করা যায় না।

এখানে `canDecrease` শর্ত হিসেবে ব্যবহার করলে কোড আরও সহজে বোঝা যায়।

`canDecrease` দেখলেই বোঝা যায় এটি “এখন আরও কমানো যাবে কি না” বোঝায়।

### অতিরিক্ত ব্যাখ্যা: কেন দুবার যাচাই?

`-` বোতামের মধ্যে:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

এখানে `.disabled(!canDecrease)` ব্যবহার করা হয়েছে, আবার বোতামের ভেতরে `if canDecrease`-ও ব্যবহার করা হয়েছে।

`.disabled(!canDecrease)` ইন্টারফেসে বোতাম নিষ্ক্রিয় করার দায়িত্ব নেয়, যাতে ব্যবহারকারী ট্যাপ করতে না পারে।

`if canDecrease` কোড চালানোর আগে আবার যাচাই করার দায়িত্ব নেয়; শুধু কমানো সম্ভব হলে `count -= 1` চালায়।

এটি দ্বিগুণ সুরক্ষা। বাস্তব ডেভেলপমেন্টে, বোতাম ইতিমধ্যেই নিষ্ক্রিয় থাকলে ভেতরের যাচাই বাদ দেওয়া যায়; কিন্তু শিক্ষণীয় উদাহরণে এটি রেখে দিলে `canDecrease`-এর ভূমিকা আরও পরিষ্কার হয়।

## সারাংশ

এই পাঠে আমরা মূলত কম্পিউটেড প্রপার্টি শিখেছি।

কম্পিউটেড প্রপার্টি সরাসরি মান সংরক্ষণ করে না; পড়ার সময় বিদ্যমান ডেটা অনুযায়ী একটি ফলাফল হিসাব করে।

উদাহরণ:

```swift
var c: Int {
    a + b
}
```

এখানে `c` আলাদা করে সংরক্ষণ করার দরকার নেই, কারণ এটি `a + b` দিয়ে হিসাব করা যায়।

কম্পিউটেড প্রপার্টি অবশ্যই `var` দিয়ে ঘোষণা করতে হয়, এবং ফেরত আসা টাইপ লিখে দিতে হয়।

```swift
var canDecrease: Bool {
    count > 1
}
```

কম্পিউটেড প্রপার্টি শুধু সংখ্যাগত মানই নয়, শর্তের ফলাফল, টেক্সট বিষয়বস্তু, এমনকি SwiftUI ভিউর বিষয়বস্তুও ফেরত দিতে পারে।

এই পাঠে আমরা `return`-ও শিখেছি।

`return` মানে একটি ফলাফল ফেরত দেওয়া:

```swift
var totalPriceText: String {
    let total = count * price
    return "মোট: \(total) $"
}
```

যদি কম্পিউটেড প্রপার্টিতে শুধু একটি সরাসরি ফলাফল তৈরি করা এক্সপ্রেশন থাকে, তাহলে `return` বাদ দেওয়া যায়।

```swift
var totalPrice: Int {
    count * price
}
```

এছাড়া আমরা `Color.primary` এবং `disabled` সম্পর্কেও জানলাম।

`Color.primary` হলো SwiftUI-র সিস্টেম semantic color, যা লাইট মোড এবং ডার্ক মোড অনুযায়ী দেখানোর ফলাফল স্বয়ংক্রিয়ভাবে সামঞ্জস্য করে।

```swift
.foregroundStyle(Color.primary)
```

`disabled` ব্যবহার করা হয় ভিউ নিষ্ক্রিয় কি না তা নিয়ন্ত্রণ করতে।

```swift
.disabled(true)
```

মানে নিষ্ক্রিয়, ট্যাপ করা যাবে না।

```swift
.disabled(false)
```

মানে উপলভ্য, ট্যাপ করা যাবে।

তাই SwiftUI-তে কম্পিউটেড প্রপার্টি খুবই সাধারণ। এটি আমাদের হিসাবের ফলাফল, শর্ত, এবং দেখানোর বিষয়বস্তু আরও পরিষ্কারভাবে সাজাতে সাহায্য করে।
