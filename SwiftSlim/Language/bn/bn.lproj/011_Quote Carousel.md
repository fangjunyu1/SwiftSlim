# উক্তি ক্যারোজেল

আজকের এই পাঠে, আমরা একটি "উক্তি ক্যারোজেল" (Quote Carousel) ফিচার তৈরি করব এবং সুইফট (Swift)-এর মৌলিক বিষয় যেমন অ্যারে (Array) এবং শর্তসাপেক্ষ স্টেটমেন্ট (if-else) সম্পর্কে গভীরভাবে শিখব।

আমরা দেখাবো কীভাবে একাধিক উক্তি সংরক্ষণ করতে হয় এবং বাটনের ইন্টারঅ্যাকশনের মাধ্যমে উক্তিগুলোর চক্রাকার স্লাইডশো বা ক্যারোজেল তৈরি করতে হয়।

![alt text](../../RESOURCE/011_word.png)

## উক্তি প্রদর্শন করা

প্রথমে, আমাদের SwiftUI-তে একটি উক্তি প্রদর্শন করতে হবে।

সবচেয়ে সহজ উপায় হলো Text ভিউ ব্যবহার করা:

```swift
Text("Slow progress is still progress.")
```

এই কোডটি শুধুমাত্র একটি নির্দিষ্ট উক্তি দেখাতে পারে। আমরা যদি একাধিক উক্তি দেখাতে চাই এবং সেগুলো পরিবর্তনের ফিচার তৈরি করতে চাই, তবে আমাদের উক্তিগুলো সংরক্ষণ করে রাখতে হবে।

তবে, সাধারণ স্ট্রিং ভেরিয়েবল কেবল একটি উক্তি সংরক্ষণ করতে পারে:

```swift
let sayings = "Slow progress is still progress."
```

যদি একাধিক উক্তি সংরক্ষণ করার প্রয়োজন হয়, তবে প্রতিটি উক্তির জন্য একটি করে ভেরিয়েবল ঘোষণা করতে হবে:

```swift
let saying1 = "Slow progress is still progress."
let saying2 = "Difficult roads lead to beautiful destinations."
let saying3 = "Awake minds are the strongest."
let saying4 = "Long-term thinking wins over short-term tricks."
let saying5 = "Master one thing, and do it well."
```

কিন্তু বাস্তব ডেভেলপমেন্টে, এই পদ্ধতিটি কেবল ঝামেলাপূর্ণই নয়, বরং প্রতিটি ভেরিয়েবল আলাদা হওয়ার কারণে আমরা নমনীয়ভাবে ক্যারোজেল পরিবর্তন করতে পারব না।

একাধিক উক্তি সহজে পরিচালনা করার জন্য, আমাদের এমন একটি ডেটা স্ট্রাকচার ব্যবহার করতে হবে যা সেগুলোকে একত্রে সংরক্ষণ করতে পারে, আর এটিই হলো অ্যারে (Array)।

অ্যারে ব্যবহার করার পর, উপরের কোডটি এভাবে সংরক্ষণ করা যেতে পারে:

```swift
let sayings = [
    "Slow progress is still progress.",
    "Difficult roads lead to beautiful destinations.",
    "Awake minds are the strongest.",
    "Long-term thinking wins over short-term tricks.",
    "Master one thing, and do it well."
    ]
```

**টিপস: প্রোগ্রামিংয়ের নিয়মে, একাধিক উপাদান ধারণকারী অ্যারে ভেরিয়েবলের নাম সাধারণত বহুবচনে লেখা হয়, যেমন sayings, যাতে এর সংগ্রহ বা কালেকশন হওয়ার বৈশিষ্ট্যটি প্রকাশ পায়।**

## অ্যারে (Array)

Swift-এ, অ্যারে হলো ক্রমানুসারে সাজানো উপাদানের একটি সংগ্রহ, যা স্কয়ার ব্র্যাকেট বা তৃতীয় বন্ধনী [] ব্যবহার করে প্রকাশ করা হয়।

```swift
[]
```

একটি অ্যারের ভেতরে একই ধরনের একাধিক উপাদান থাকতে পারে, এবং উপাদানগুলোকে ইংরেজি কমা , দিয়ে আলাদা করা হয়।

উদাহরণস্বরূপ:

```swift
[101, 102, 103, 104, 105]
```

আপনি অ্যারেকে খুব সহজেই একটি ট্রেনের সাথে তুলনা করতে পারেন:

![Array](../../RESOURCE/011_array1.png)

সম্পূর্ণ ট্রেনটি এই অ্যারে অবজেক্টকে উপস্থাপন করে, এবং প্রতিটি বগি ক্রমানুসারে সাজানো থাকে।

### ইনডেক্স এবং উপাদান অ্যাক্সেস করা

যেহেতু অ্যারে সুশৃঙ্খলভাবে সাজানো থাকে, তাই সিস্টেম নির্দিষ্ট ক্রমানুসারে অ্যারের নির্দিষ্ট উপাদানটি খুঁজে বের করতে পারে। এই খুঁজে বের করার পদ্ধতিকে ইনডেক্স (Index) বলা হয়।

Swift-এ (এবং বেশিরভাগ প্রোগ্রামিং ভাষায়), অ্যারের ইনডেক্স গণনা 0 থেকে শুরু হয়, 1 থেকে নয়। এর মানে হলো অ্যারের প্রথম উপাদানের ইনডেক্স হলো 0, দ্বিতীয় উপাদানের 1, এবং এভাবেই চলতে থাকে।

![Array](../../RESOURCE/011_array2.png)

অ্যারের কোনো নির্দিষ্ট উপাদান অ্যাক্সেস করতে চাইলে, অ্যারের নামের পর স্কয়ার ব্র্যাকেট যুক্ত করে তার ভেতরে নির্দিষ্ট উপাদানের ইনডেক্স মানটি বসিয়ে দিতে হবে।

উদাহরণস্বরূপ:

```swift
sayings[0]   // 101
sayings[1]   // 102
```

যদি আপনি এমন কোনো ইনডেক্স অ্যাক্সেস করার চেষ্টা করেন যা অ্যারের সীমানার বাইরে, তবে "ইনডেক্স আউট অফ রেঞ্জ (Index Out of Range)" সমস্যা দেখা দেবে। তাই, অ্যারে অ্যাক্সেস করার সময় সতর্ক থাকতে হবে যাতে ইনডেক্সটি সঠিক সীমানার মধ্যে থাকে।

**ইনডেক্স আউট অফ রেঞ্জ (Index Out of Range)**

উদাহরণস্বরূপ, যখন একটি অ্যারেতে মাত্র ৫টি উপাদান থাকে, তখন বৈধ ইনডেক্স সীমা হলো 0 থেকে 4। আমরা যদি sayings[5] অ্যাক্সেস করার চেষ্টা করি, তবে প্রোগ্রামটি এর সাথে মিল থাকা কোনো "বগি" খুঁজে পাবে না, যার ফলে "ইনডেক্স আউট অফ রেঞ্জ" এরর তৈরি হবে এবং অ্যাপটি ক্র্যাশ করবে।

### অ্যারে পরিচালনা করা (Operations)

অ্যারে শুধুমাত্র স্ট্যাটিক ভাবে সংজ্ঞায়িত করা যায় তা নয়, বরং উপাদান যোগ, মুছে ফেলা, পরিবর্তন করার মতো কাজও করা যায়, এমনকি অ্যারের দৈর্ঘ্য বা লেন্সও জানা যায়।

টিপস: অ্যারেতে কোনো পরিবর্তন করতে হলে অবশ্যই var দিয়ে ভেরিয়েবল ঘোষণা করতে হবে, let ধ্রুবক (constant) দিয়ে নয়।

**১. নতুন উপাদান যোগ করা**

append পদ্ধতি ব্যবহার করে অ্যারের শেষে নতুন উপাদান যোগ করা যায়:

```swift
var num = [0, 1, 2]
num.append(3)   // [0, 1, 2, 3]
```

**২. উপাদান মুছে ফেলা**

remove(at:) পদ্ধতি ব্যবহার করে অ্যারের নির্দিষ্ট উপাদানটি মুছে ফেলা যায়:

```swift
var num = [0, 1, 2]
num.remove(at: 0)   // [1, 2]
```

**৩. উপাদান পরিবর্তন করা**

সরাসরি ইনডেক্স ব্যবহার করে অ্যারের উপাদান পরিবর্তন করা যায়:

```swift
var num = [0, 1, 2]
num[1] = 4   // [0, 4, 2]
```

**৪. অ্যারের দৈর্ঘ্য গণনা করা**

count প্রপার্টি ব্যবহার করে অ্যারের উপাদানের সংখ্যা জানা যায়:

```swift
var num = [0, 1, 2]
num.count   // 3
```

### উক্তি প্রদর্শনের জন্য অ্যারে ব্যবহার করা

একাধিক উক্তি দেখানোর জন্য, আমরা উক্তিগুলোকে একটি অ্যারেতে সংরক্ষণ করতে পারি এবং তারপর ইনডেক্সের সাহায্যে অ্যাক্সেস করে সেগুলো প্রদর্শন করতে পারি।

প্রথমে, ContentView-এর মধ্যে উক্তি সংরক্ষণের জন্য একটি sayings অ্যারে তৈরি করুন, তারপর Text ভিউতে ইনডেক্স ব্যবহার করে নির্দিষ্ট উক্তিটি পড়ুন এবং প্রদর্শন করুন:

```swift
struct ContentView: View {
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[0])
        }
        .padding()
    }
}
```

এখানে, sayings[0] দ্বারা অ্যারের প্রথম উক্তিটিকে বোঝানো হয়েছে।

যদি ভিন্ন কোনো উক্তি দেখাতে চান, তবে কেবল স্কয়ার ব্র্যাকেটের ভেতরের ইনডেক্স মানটি পরিবর্তন করতে হবে:

```swift
Text(sayings[0]) // "Slow progress is still progress."
Text(sayings[1]) // "Difficult roads lead to beautiful destinations."
```

### উক্তির ইনডেক্স নির্ধারণ করা

উক্তিগুলো ডায়নামিকভাবে পরিবর্তনের জন্য, আমরা Text ভিউয়ের ভেতর ইনডেক্সকে একেবারে "স্থায়ী বা ফিক্সড" করে রাখতে পারি না।

আমাদের একটি ভেরিয়েবল তৈরি করতে হবে, যা বর্তমানে প্রদর্শিত ইনডেক্স মানটি আলাদাভাবে সংরক্ষণ করবে।

SwiftUI-তে, আমরা @State ব্যবহার করে একটি পরিবর্তনযোগ্য ইনডেক্স ঘোষণা করতে পারি:

```swift
@State private var index = 0
```

SwiftUI সবসময় @State দ্বারা মোড়ানো ভেরিয়েবলের দিকে নজর রাখে, যখন index-এর মান পরিবর্তন হয়, SwiftUI পুনরায় ভিউটি রেন্ডার করে এবং নতুন ইনডেক্স অনুযায়ী উক্তিটি প্রদর্শন করে।

এরপর, অ্যারে থেকে ডায়নামিকভাবে উক্তিটি নেওয়ার জন্য আমরা sayings[index] ব্যবহার করব:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[index])
        }
        .padding()
    }
}
```

যখন index এর মান পরিবর্তন হয়, তখন Text ভিন্ন ভিন্ন উক্তি প্রদর্শন করবে।

### ইনডেক্স নিয়ন্ত্রণ করতে বাটন ব্যবহার করা

উক্তি পরিবর্তন করার জন্য, আমরা index-এর মান পরিবর্তন করতে Button ব্যবহার করতে পারি। প্রতিবার বাটনে ক্লিক করলে, index স্বয়ংক্রিয়ভাবে 1 করে বৃদ্ধি পাবে:

```swift
Button("Next") {
    index += 1
}
```

যখন আমরা বাটনে ক্লিক করি, index-এর মান 0 থেকে 1 হয়ে যায়, যার ফলে ভিউ রিফ্রেশ হয় এবং Text(sayings[index]) পরবর্তী উক্তিটি পড়ে নেয়।

তবে, এখানে একটি বড় সমস্যা রয়েছে: যদি আমরা ক্রমাগত বাটনে ক্লিক করতে থাকি, index বৃদ্ধি পেয়ে অ্যারের সীমানার বাইরে চলে যাবে, যা ইনডেক্স আউট অফ রেঞ্জ এররের সৃষ্টি করবে। উদাহরণস্বরূপ, যখন index ৫ এ পৌঁছাবে (অ্যারের ইনডেক্স সীমা 0 থেকে 4), তখন প্রোগ্রামটি ক্র্যাশ করবে।

অ্যারের ইনডেক্স সীমার বাইরে যাওয়া রোধ করতে, আমাদের শর্ত বা কন্ডিশনাল কন্ট্রোল যুক্ত করতে হবে, যাতে নিশ্চিত হওয়া যায় যে index অ্যারের সীমানা ছাড়িয়ে যাবে না। index অ্যারের দৈর্ঘ্যের চেয়ে ছোট কি না, তা if-else স্টেটমেন্ট ব্যবহার করে যাচাই করা যেতে পারে।

## শর্তসাপেক্ষ নিয়ন্ত্রণ: if-else স্টেটমেন্ট

if-else স্টেটমেন্ট হলো Swift-এর সবচেয়ে বহুল ব্যবহৃত কন্ডিশনাল ব্রাঞ্চ স্টেটমেন্ট, যা কোনো শর্ত সত্য নাকি মিথ্যা তা যাচাই করতে ব্যবহৃত হয় এবং শর্তের সত্য বা মিথ্যার ওপর ভিত্তি করে ভিন্ন ভিন্ন কোড ব্লক এক্সিকিউট করে।

এর সাধারণ গঠন:

```swift
if condition {
    // condition সত্য (true) হলে এই অংশের কোড এক্সিকিউট হবে
} else {
    // condition মিথ্যা (false) হলে এই অংশের কোড এক্সিকিউট হবে
}
```

if স্টেটমেন্টে, condition বা শর্ত হলো একটি বুলিয়ান ভ্যালু (Bool টাইপ), যা true বা false হতে পারে। যখন শর্তটি true হয়, তখন if অংশের কোড এক্সিকিউট হয়, অন্যথায় else অংশের কোড এক্সিকিউট হয়।

উদাহরণস্বরূপ:

```swift
let age = 25
if age > 18 {
    print("Big Boy")
} else {
    print("Little Body")
}
```

এই উদাহরণে, age এর মান ২৫, if স্টেটমেন্ট যাচাই করে যে age > 18 শর্তটি সত্য কি না, শর্ত পূরণ হলে এটি "Big Boy" আউটপুট দেয়।

যদি else অংশের কোডের প্রয়োজন না হয়, তবে সেটি বাদ দেওয়া যেতে পারে:

```swift
if condition {
    // condition সত্য (true) হলে এই অংশের কোড এক্সিকিউট হবে
}
```

### ইনডেক্স সীমা নিয়ন্ত্রণে শর্ত ব্যবহার করা

অ্যারের ইনডেক্স আউট অফ রেঞ্জ সমস্যা প্রতিরোধ করতে, index যেন অ্যারের সীমানা অতিক্রম না করে তা নিশ্চিত করতে if স্টেটমেন্ট ব্যবহার করা যেতে পারে:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    }
}
```

লজিক বিশ্লেষণ: sayings.count এর মান হলো 5 (মোট ৫টি উক্তি রয়েছে), সুতরাং sayings.count - 1 এর মান হবে 4, এবং এটিই হলো অ্যারের সর্বশেষ বৈধ ইনডেক্স।

যখন index 4 এর চেয়ে ছোট থাকে, বাটনে ক্লিক করে এর মান 1 বৃদ্ধি করা নিরাপদ; কিন্তু যখন index 4 এ পৌঁছায়, তখন শর্তটি আর পূরণ হয় না, ফলে বাটনে ক্লিক করলে কোনো কিছুই ঘটবে না।

এখন, কোডটিতে সফলভাবে উক্তি পরিবর্তনের ফিচারটি বাস্তবায়িত হয়েছে:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack {
            Text(sayings[index])
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                }
            }
        }
        .padding()
    }
}
```

### উক্তিগুলোর চক্রাকার আবর্তন (লুপ)

যদি আমরা চাই যে শেষ উক্তিটি দেখানোর পর, বাটনে ক্লিক করলে আবার প্রথম উক্তিটি প্রদর্শিত হবে, যাতে উক্তিগুলোর একটি চক্রাকার আবর্তন তৈরি হয়, তবে আমরা else অংশ ব্যবহার করে সেটি করতে পারি:

```swift
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    } else {
        index = 0
    }
}
```

যখন বাটনে ক্লিক করা হয়, যদি index অ্যারের শেষ উপাদানে পৌঁছায়, তখন index পুনরায় 0 তে সেট হয়ে যায়, এবং প্রথম থেকে উক্তিগুলো দেখানো শুরু হয়।

## উক্তি ভিউটিকে উন্নত (Optimize) করা

এখন, আমাদের উক্তি ক্যারোজেলের লজিক বা চিন্তাধারা বাস্তবায়িত হয়েছে, তবে আমরা ইন্টারফেসটিকে আরও সুন্দর করার জন্য উন্নত করতে পারি।

সম্পূর্ণ কোড:

```swift
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack(spacing: 10) {
            Text(sayings[index])
                .padding(20)
                .background(.white)
                .cornerRadius(10)
                .opacity(0.8)
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                } else {
                    index = 0
                }
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .background(
            Image("001")
        )
    }
}
```

এই উদাহরণে, Text ভিউতে একটি সাদা আধা-স্বচ্ছ ব্যাকগ্রাউন্ড এবং গোলাকার কোণ যুক্ত করা হয়েছে, Button-এ .borderedProminent স্টাইল ব্যবহার করা হয়েছে, এবং VStack-এ একটি ব্যাকগ্রাউন্ড ছবি সেট করা হয়েছে।

অতিরিক্ত তথ্য: ইমেজের ব্যাকগ্রাউন্ড সেট করতে background() মডিফায়ার ব্যবহার করার সময়, এর ডিফল্ট কাজ হলো বর্তমান ভিউয়ের লেআউট এরিয়া যতটা সম্ভব পূর্ণ করা। অনেক ক্ষেত্রেই, এটি স্বাভাবিকভাবেই সেইফ এরিয়া (Safe Area) পর্যন্ত বিস্তৃত হয়ে কভার করে নেয়।

এখন, আমরা সফলভাবে উক্তি ক্যারোজেলের ভিউ তৈরি করে ফেলেছি।

## সারসংক্ষেপ

আজকের পাঠের মাধ্যমে, আমরা শিখেছি কীভাবে অ্যারে ব্যবহার করে একাধিক উক্তি সংরক্ষণ করতে হয় এবং if ও if-else স্টেটমেন্টের মাধ্যমে কীভাবে উক্তিগুলোর ক্যারোজেল তৈরি করতে হয়।

আমরা অ্যারের প্রাথমিক কাজগুলোও শিখেছি, যেমন নতুন উপাদান যোগ করা, মুছে ফেলা, পরিবর্তন করা এবং কীভাবে অ্যারের ইনডেক্স আউট অফ রেঞ্জ প্রতিরোধ করা যায়।

এই পাঠে কেবল উক্তি ক্যারোজেল ফিচারটি তৈরি করাই শেখানো হয়নি, বরং এর সাথে অ্যারে এবং কন্ডিশনাল স্টেটমেন্টের মৌলিক ব্যবহারগুলোও যুক্ত করা হয়েছে, যা আমাদের ডেটা পরিচালনা এবং প্রোগ্রামের প্রবাহ নিয়ন্ত্রণ করার দক্ষতা অর্জন করতে সাহায্য করেছে।

## অতিরিক্ত জ্ঞান - মাল্টিপল কন্ডিশনাল চেকিং: if-else if-else স্টেটমেন্ট

বাস্তব ডেভেলপমেন্টে, প্রায়ই একাধিক শর্ত নিয়ে কাজ করতে হয়। যেমন, কোনো একটি গেমে, স্কোর যখন ১ হয় তখন ইভেন্ট A ঘটে, ২ হলে ইভেন্ট B ঘটে, ৩ হলে ইভেন্ট C ঘটে, এবং এভাবেই চলতে থাকে।

দুইয়ের অধিক শর্তের ব্রাঞ্চ রয়েছে এমন পরিস্থিতিতে, আমাদের মাল্টিপল চেকিংয়ের জন্য if-else if-else স্টেটমেন্ট ব্যবহার করতে হয়।

সাধারণ গঠন:

```swift
if conditionA {
    // conditionA সত্য (true) হলে এই অংশের কোড এক্সিকিউট হবে
} else if conditionB {
    // conditionB সত্য (true) হলে এই অংশের কোড এক্সিকিউট হবে
} else if conditionC {
    // conditionC সত্য (true) হলে এই অংশের কোড এক্সিকিউট হবে
} else {
    // কোনো শর্তই পূরণ না হলে এই অংশের কোড এক্সিকিউট হবে
}
```

এই ধরনের পরিস্থিতিতে, প্রোগ্রামটি ক্রমানুসারে শর্তগুলো যাচাই করে এবং যে শর্তটি প্রথমে পূরণ হয় সেটি এক্সিকিউট করে। যদি কোনো শর্তই পূরণ না হয়, তবে else এর পরের কোডটি এক্সিকিউট হয়।

উক্তি ক্যারোজেলেও আমরা মাল্টিপল চেকিংয়ের জন্য if-else if-else স্টেটমেন্ট ব্যবহার করতে পারি:

```swift
Button("Next") {
    if index == 0 {
        index += 1
    } else if index == 1 {
        index += 1
    } else if index == 2 {
        index += 1
    } else if index == 3 {
        index += 1
    } else if index == 4 {
        index = 0
    } else {
        index = 0
    }
}
```

যখন index এর মান 0, 1, 2, 3 হয়, তখন বাটনে প্রতিবার ক্লিক করলে index += 1 এক্সিকিউট হয়, আর যখন index 4 (শেষ উপাদান) হয়, তখন index পুনরায় 0 তে সেট হয়ে যায়, যা চক্রটি সম্পূর্ণ করে।

else ব্রাঞ্চটি ফলব্যাক (fallback) হিসেবে কাজ করে, যাতে কোনো অবৈধ মান (যেমন, index ভুলবশত পরিবর্তন হয়ে গেলে) প্রতিরোধ করা যায়।

মনে রাখতে হবে যে, এখানে ব্যবহৃত == বোঝায় "সমান কি না" তা যাচাই করা, যদি if স্টেটমেন্ট দেখে যে index এবং নির্দিষ্ট কোনো সংখ্যা সমান, তবে এটি true রিটার্ন করে এবং পরবর্তী কোড ব্লক এক্সিকিউট করে। যদি সমান না হয়, তবে এটি পরবর্তী if স্টেটমেন্ট যাচাই করতে চলে যায়।

এই ধরনের মাল্টিপল চেকিং বিভিন্ন শর্তসাপেক্ষ পরিস্থিতিতে ভিন্ন ভিন্ন কোড এক্সিকিউট করতে দারুণ সহায়ক।
