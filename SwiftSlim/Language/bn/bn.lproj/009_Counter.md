# কাউন্টার

আগের দুটি ক্লাসে, আমরা ভেরিয়েবল, কনস্ট্যান্ট, বাটন এবং মেথড সম্পর্কে শিখেছি।

এই ক্লাসে, আমরা এই জ্ঞানগুলোকে একত্রিত করে একটি সাধারণ কাউন্টার অ্যাপ তৈরি করব।

এই উদাহরণে, আমরা শিখব কীভাবে ভেরিয়েবল সেভ করতে ও পরিবর্তন করতে হয় এবং @State ব্যবহার করে ভেরিয়েবল ম্যানেজ করতে হয়।

## কাউন্টার তৈরি করা

কাউন্টার সাধারণত এমন সংখ্যা রেকর্ড করতে ব্যবহৃত হয় যা ক্রমাগত বাড়ানোর প্রয়োজন হয়। উদাহরণস্বরূপ, দড়ি লাফানোর সংখ্যা, দৌড়ানোর চক্কর ইত্যাদি গণনার ক্ষেত্রে।

নিচে আমরা একটি সাধারণ কাউন্টার তৈরি করব: যেখানে একটি সংখ্যা এবং একটি বাটন প্রদর্শিত হবে, বাটনে ক্লিক করলে সংখ্যাটি বৃদ্ধি পাবে।

![Num](../../RESOURCE/009_num.png)

### সংখ্যা প্রদর্শন করা

প্রথমে, আমরা সংখ্যাটি প্রদর্শন করতে Text ব্যবহার করব।

```swift
Text("0")
```

তবে, সংখ্যাটি যদি পরিবর্তন করার প্রয়োজন হয়, তবে সরাসরি "0" লেখা যাবে না, বরং সংখ্যাটি সেভ করতে একটি ভেরিয়েবল ব্যবহার করতে হবে।

```swift
var num = 0
```

তারপর Text-এর মধ্যে ভেরিয়েবলটি প্রদর্শন করতে হবে:

```swift
Text("\(num)")
```

এখানে স্ট্রিং ইন্টারপোলেশন \() ব্যবহার করা হয়েছে, যা সংখ্যাটিকে স্ট্রিংয়ে রূপান্তর করে প্রদর্শন করে।

### বাটনের মাধ্যমে সংখ্যা বৃদ্ধি করা

এরপর, আমরা একটি বাটন যুক্ত করব।

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

যখন বাটনটিতে ক্লিক করা হবে, আমরা চাই সংখ্যাটি বৃদ্ধি পাক।

তাই বাটনের ভেতরে আমরা ভেরিয়েবলটি পরিবর্তন করতে পারি:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

এই কোডটি বোঝায়: যখন বাটনে ক্লিক করা হবে, তখন num + 1 হিসাব করা হবে এবং ফলাফলটি num-এ অ্যাসাইন করা হবে।

### আরও সহজবোধ্য লেখার নিয়ম

Swift আরও একটি সহজবোধ্য লেখার নিয়ম প্রদান করে:

```swift
num += 1
```

এটি এর সমতুল্য:

```swift
num = num + 1
```

এই ধরনের লেখার নিয়মকে "কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটর" বলা হয়।

Swift-এ ব্যবহৃত সাধারণ কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটরগুলো হলো:

```
+=   যোগ অ্যাসাইনমেন্ট
-=   বিয়োগ অ্যাসাইনমেন্ট
*=   গুণ অ্যাসাইনমেন্ট
/=   ভাগ অ্যাসাইনমেন্ট
%=   ভাগশেষ (মডুলাস) অ্যাসাইনমেন্ট
```

উদাহরণস্বরূপ:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### সম্পূর্ণ কোড

এখন, আমরা একটি সাধারণ কাউন্টার লিখতে পারি:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

তবে, এই মুহূর্তে কোডটি ঠিকমতো রান করবে না।

### এরর বা ত্রুটি দেখা দেওয়া

Xcode একটি সতর্কবার্তা দেখাবে:

```
Cannot assign to property: 'self' is immutable
```

এটি বোঝায় যে প্রপার্টি পরিবর্তন করা সম্ভব নয়, কারণ ভিউগুলো অপরিবর্তনযোগ্য।

এমনটা কেন হলো?

## SwiftUI-এর কাজের নীতি

SwiftUI-তে: **ইন্টারফেসটি ডেটা-ড্রিভেন (ডেটা দ্বারা নিয়ন্ত্রিত)**, যার মানে হলো ডেটা পরিবর্তিত হলে ইন্টারফেসটি স্বয়ংক্রিয়ভাবে রিফ্রেশ হবে।

তবে মনে রাখতে হবে, শুধুমাত্র যখন স্টেট (যেমন @State, @Binding) পরিবর্তিত হয়, তখনই SwiftUI পরিবর্তনটি পর্যবেক্ষণ করে এবং ভিউ রিফ্রেশ করে।

যদি সাধারণ ভেরিয়েবল ব্যবহার করা হয়, তবে SwiftUI শুধুমাত্র ভিউ তৈরির সময় ভেরিয়েবলের মান একবার পড়বে। এরপর ভেরিয়েবল পরিবর্তিত হলেও ইন্টারফেস আপডেট হবে না।

উদাহরণস্বরূপ:

```swift
var num = 0   // 0
num = 1   // 0
```

উপরের উদাহরণে, num একটি সাধারণ ভেরিয়েবল। যদিও এর মান পরিবর্তিত হয়েছে, তবুও SwiftUI ভিউ আপডেট করবে না এবং ইন্টারফেসে প্রাথমিক মান 0-ই প্রদর্শিত হবে।

![Num](../../RESOURCE/009_state.png)

তাই, শুধুমাত্র যখন ডেটাকে স্টেট হিসেবে চিহ্নিত করা হয় (যেমন @State, @Binding ইত্যাদি প্রপার্টি র‍্যাপার ব্যবহার করে), তখনই SwiftUI স্বয়ংক্রিয়ভাবে এর পরিবর্তন পর্যবেক্ষণ করবে এবং ভিউ আপডেট করবে।

## @State প্রপার্টি র‍্যাপার

SwiftUI-তে, যদি কোনো ভেরিয়েবল পরিবর্তন করতে হয় এবং ভিউ আপডেট করতে হয়, তবে অবশ্যই @State ব্যবহার করতে হবে।

```swift
@State var num = 0
```

@State ব্যবহার করলে SwiftUI এই ভেরিয়েবলটিকে নজরে রাখবে।

যখন ভেরিয়েবলটি পরিবর্তিত হবে, SwiftUI পুনরায় ভিউ হিসাব করবে এবং ভিউ আপডেট করবে।

সম্পূর্ণ কোড:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

এখন, যখন বাটনে ক্লিক করা হবে:

```swift
num += 1
```

SwiftUI শনাক্ত করবে যে num পরিবর্তিত হয়েছে এবং তারপর স্বয়ংক্রিয়ভাবে ভিউ আপডেট করবে।

![Num](../../RESOURCE/009_state1.png)

## @State ব্যবহারের নিয়ম

রিয়েল-লাইফ ডেভেলপমেন্টে, @State সাধারণত দুটি নিয়ম মেনে চলে।

### ১. @State সাধারণত private হিসেবে ব্যবহৃত হয়

@State ভেরিয়েবল সাধারণত শুধুমাত্র বর্তমান টাইপের (যেমন বর্তমান View স্ট্রাকচার) মধ্যেই ব্যবহৃত হয়, তাই এটি সাধারণত এভাবে লেখা হয়:

```swift
@State private var num = 0
```

এটি অন্যান্য ভিউকে সরাসরি এই ভেরিয়েবল অ্যাক্সেস বা পরিবর্তন করা থেকে বিরত রাখে।

আমরা এখনও private সম্পর্কে শিখিনি, তাই সহজভাবে এটি বোঝা যায়:

**private দিয়ে ডিক্লেয়ার করা জিনিসগুলো শুধুমাত্র বর্তমান ভিউতে ব্যবহার করা যাবে, বাইরের কোনো ভিউ এটি ব্যবহার করতে পারবে না।**

### ২. @State শুধুমাত্র বর্তমান ভিউয়ের স্টেটের জন্য ব্যবহৃত হয়

@State বর্তমান View-এর নিজস্ব পরিচালিত ডেটার জন্য প্রযোজ্য।

উদাহরণস্বরূপ, ইনপুট বক্সের টেক্সট, বা সুইচের অবস্থা।

```swift
@State private var isOn = false
@State private var text = ""
```

যদি একাধিক ভিউয়ের মধ্যে ডেটা শেয়ার করার প্রয়োজন হয়, তবে অন্যান্য স্টেট টাইপ ব্যবহার করতে হবে, যেমন @Binding বা @Observable।

এই বিষয়গুলো আমরা পরবর্তী ক্লাসগুলোতে শিখব।

### কমানোর বাটন

কাউন্টার শুধুমাত্র বাড়ানোর জন্যই নয়, কমানোর জন্যও ব্যবহার করা যেতে পারে।

আমরা একটি - বাটন যোগ করতে পারি।

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

তারপর HStack ব্যবহার করে বাটনগুলোকে অনুভূমিকভাবে সাজাতে পারি:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

ফলাফল:

![Num](../../RESOURCE/009_num1.png)

+ বাটনে ক্লিক করলে সংখ্যাটি ১ বৃদ্ধি পাবে; - বাটনে ক্লিক করলে সংখ্যাটি ১ কমে যাবে।

### রিসেট বাটন

সংখ্যাটিকে আবার 0 তে ফিরিয়ে আনতে আমরা একটি রিসেট বাটনও যুক্ত করতে পারি।

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

যখন রিসেট বাটনে ক্লিক করা হয়, তখন num এর মান 0 নির্ধারণ করা হয়।

ফলাফল:

![Num](../../RESOURCE/009_num2.png)

0 বাটনে ক্লিক করলে, সংখ্যাটি 0 হয়ে যায়।

### কাস্টম বাটন

বর্তমানে আমাদের বাটন শুধুমাত্র টেক্সট প্রদর্শন করতে পারে:

```swift
Button("+") {

}
```

এভাবে লেখা বাটনগুলো শুধুমাত্র টেক্সট কন্টেন্ট দেখাতে পারে, যেমন + বা -।

রিয়েল-লাইফ ডেভেলপমেন্টে, আমরা সাধারণত চাই বাটনের আউটলুক আরও সুন্দর হোক, যেমন আইকন বা অন্যান্য ভিউ ব্যবহার করা।

SwiftUI আমাদের বাটনের কন্টেন্ট কাস্টমাইজ করার সুযোগ দেয়, তখন আমরা বাটন লেখার আরেকটি নিয়ম ব্যবহার করতে পারি:

```swift
Button(action: {

}, label: {

})
```

এই নিয়মে, action বলতে বাটনে ক্লিক করলে যে কোডটি রান করবে তা বোঝায়, এবং label বলতে বাটনের ভিউ প্রদর্শন করাকে বোঝায়।

উদাহরণস্বরূপ, আমরা বাটনে একটি SF Symbols আইকন দেখাতে পারি:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

এভাবে বাটনে আর কোনো টেক্সট প্রদর্শিত হবে না, বরং একটি আইকন দেখা যাবে।

### কাউন্টার ইন্টারফেস অপ্টিমাইজ করা

কাউন্টারের ইন্টারফেস আরও পরিষ্কার করার জন্য, আমরা সংখ্যা প্রদর্শনের স্টাইলটি সামঞ্জস্য করতে পারি। উদাহরণস্বরূপ, সংখ্যাটি বড় করা:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

এরপর, আমরা বাটনের কন্টেন্ট পরিবর্তন করে আইকন যুক্ত করব এবং আইকনের আকার প্রয়োজনমতো বড় করব:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

একইভাবে, আমরা কমানোর বাটন এবং রিসেট বাটনকেও আইকনে রূপান্তর করতে পারি:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

এখন বাটনগুলো SF Symbols আইকন দেখাচ্ছে, এবং ইন্টারফেসটি আরও ব্যবহারকারীবান্ধব হয়ে উঠবে।

## সারাংশ

এই ক্লাসে একটি সাধারণ কাউন্টার তৈরি করার মাধ্যমে, আমরা ভেরিয়েবল, Text এবং Button-এর ব্যবহার সমন্বিতভাবে অনুশীলন করেছি। আমরা সংখ্যা সংরক্ষণ করতে ভেরিয়েবল ব্যবহার করেছি এবং বাটনের সাহায্যে সংখ্যা বাড়ানো বা কমানো নিয়ন্ত্রণ করেছি। পাশাপাশি, আমরা কম্পাউন্ড অ্যাসাইনমেন্ট অপারেটর শিখেছি, যেমন num += 1 এবং num -= 1।

এই তৈরির প্রক্রিয়ার মধ্যে, আমরা SwiftUI-এর কাজ করার পদ্ধতি বুঝতে পেরেছি: ইন্টারফেসটি ডেটা-ড্রিভেন। যখন ডেটা পরিবর্তিত হয়, তখনই ইন্টারফেস রিফ্রেশ হয়। তাই পরিবর্তনশীল স্টেট সংরক্ষণ করতে @State ব্যবহার করা প্রয়োজন, এবং যখন @State ভেরিয়েবল পরিবর্তিত হয়, তখন SwiftUI স্বয়ংক্রিয়ভাবে ভিউ আপডেট করে।

পরিশেষে, আমরা ইন্টারফেসটিকে কিছুটা অপ্টিমাইজ করেছি, Text-এর ফন্ট স্টাইল সামঞ্জস্য করেছি, এবং Button(action:label:) ব্যবহার করে বাটনের কন্টেন্ট কাস্টমাইজ করেছি। এর ফলে বাটনগুলো আইকন প্রদর্শন করতে পারছে, যা কাউন্টারের ইন্টারফেসকে আরও পরিষ্কার ও সুন্দর করে তুলেছে।

এখন, আমরা SwiftUI ভিউগুলোর বেসিক ব্যবহার সম্পর্কে জেনেছি। SwiftUI-তে, বেশিরভাগ ইন্টারফেসই ভেরিয়েবল দ্বারা পরিচালিত হয়। যখন ভেরিয়েবল পরিবর্তিত হয়, SwiftUI স্বয়ংক্রিয়ভাবে ভিউ আপডেট করে। তাই, SwiftUI অ্যাপ ডেভেলপ করার সময়, আমাদের সাধারণত প্রথমে কী কী ডেটা সংরক্ষণ করতে হবে তা ডিজাইন করে নিতে হয় এবং তারপর সেই ডেটা ব্যবহার করে ইন্টারফেস ও ইন্টার‍্যাকশন লজিক তৈরি করতে হয়।

## সম্পূর্ণ কোড

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
