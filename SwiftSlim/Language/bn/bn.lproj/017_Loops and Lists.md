# লুপ এবং তালিকা

এই পাঠে আমরা Swift-এর `for` লুপ এবং SwiftUI-এর `ForEach` শিখব।

দুটিই আমাদের অ্যারের উপাদানগুলোকে ক্রমানুসারে পড়তে সাহায্য করে। বাস্তব ডেভেলপমেন্টে আমাদের প্রায়ই একগুচ্ছ ডেটা একে একে বের করতে হয়, তারপর সেগুলো প্রক্রিয়া করতে বা দেখাতে হয়। আর এই কাজটি করার গুরুত্বপূর্ণ উপায়ই হলো লুপ।

যখন আমরা ওয়েব ব্রাউজ করি বা কোনো App ব্যবহার করি, তখন প্রায়ই অনেক তালিকা বা আইটেম দেখি। এই ধরনের কনটেন্ট সাধারণত আগে `অ্যারে`-তে সংরক্ষণ করা হয়, তারপর `লুপ` দিয়ে একে একে পড়া হয়, এবং শেষে ইন্টারফেসে দেখানো হয়।

উদাহরণস্বরূপ, Google-এর সার্চ রেজাল্ট তালিকা:

![Google](../../Resource/017_google.png)

সার্চ রেজাল্ট তালিকার প্রতিটি আইটেম সাধারণত একটি করে ডেটার সাথে সম্পর্কিত, এবং সেগুলো লুপের মাধ্যমে একে একে দেখানো হয়।

তাই `অ্যারে` এবং `লুপ` প্রায়ই একসাথে দেখা যায়। `অ্যারে` একগুচ্ছ ডেটা সংরক্ষণ করে, আর `লুপ` সেই ডেটা ক্রমানুসারে বের করে একই কাজ সম্পাদন করে।

এখন আমরা সবচেয়ে মৌলিক `for` লুপ থেকে শুরু করি।

## কাউন্টডাউন

ধরা যাক, আমরা একটি কাউন্টডাউন ফাংশন অনুকরণ করতে চাই, এবং ক্রমানুসারে `“5、4、3、2、1”` আউটপুট করতে চাই।

সবচেয়ে সরাসরি লেখাটি হলো:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

এই পদ্ধতিতে কাউন্টডাউন অনুকরণ করা যায়, কিন্তু যদি বিষয়বস্তু বেশি হয়, তাহলে এটি খুবই পুনরাবৃত্তিমূলক হয়ে যায়, এবং পরে পরিবর্তন করাও সুবিধাজনক থাকে না।

এই সময় আমরা আগে ডেটাগুলোকে একটি অ্যারেতে রাখতে পারি, তারপর `for` লুপ ব্যবহার করে একে একে আউটপুট করতে পারি:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

এই লেখাটি আরও সংক্ষিপ্ত, এবং বিস্তৃত করাও সহজ। কাউন্টডাউনের সংখ্যা অনেক হলেও, আমাদের `print` লাইন ধরে ধরে লিখতে হবে না, শুধু লুপকে অ্যারের প্রতিটি উপাদান স্বয়ংক্রিয়ভাবে প্রক্রিয়া করতে দিতে হবে।

আমরা এই কোডটি একটি বাটনের মধ্যে রেখে পরীক্ষা করতে পারি:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

বাটনে ট্যাপ করলে, `for` লুপটি `countDown`-এর প্রতিটি পূর্ণসংখ্যা ক্রমানুসারে বের করবে এবং কনসোলে প্রিন্ট করবে।

এরপর, আমরা দেখি `for` লুপ কীভাবে কাজ করে।

## for লুপ

`for` লুপ ব্যবহার করা হয় একগুচ্ছ ডেটা ক্রমানুসারে প্রক্রিয়া করতে। সবচেয়ে সাধারণ বিষয় হলো অ্যারে এবং রেঞ্জ।

মৌলিক লেখার ধরন:

```swift
for উপাদান in অ্যারে_অথবা_রেঞ্জ {
    কার্যকর_কোড
}
```

উদাহরণস্বরূপ, একটু আগের কাউন্টডাউনের কোড:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

এখানে `i` বর্তমান বের করা উপাদানকে বোঝায়, আর `in`-এর পরে রয়েছে যে ডেটার ওপর লুপ চালানো হচ্ছে, অর্থাৎ `countDown` অ্যারে।

এই কোডের অর্থ হলো: প্রোগ্রাম `countDown` অ্যারে থেকে প্রতিটি উপাদান ক্রমানুসারে বের করবে। প্রতিবার একটি উপাদান বের হলে, সেটিকে সাময়িকভাবে `i` দিয়ে প্রকাশ করবে, তারপর ব্রেসের ভেতরের কোড চালাবে।

কার্যপ্রক্রিয়াটি এভাবে বোঝা যেতে পারে:

* প্রথমে বের হওয়া উপাদান হলো `5`, তাই এই সময়ে `i`-এর মান `5`
* `print(i)` চালানো হয়, এবং `5` আউটপুট হয়
* তারপর পরের উপাদান `4` বের করা হয়
* আবার `print(i)` চালানো হয়, এবং `4` আউটপুট হয়

পরের `3`、`2`、`1`-ও একইভাবে একে একে চালানো হবে।

![for](../../Resource/017_for.png)

অ্যারের সব উপাদান প্রক্রিয়া হয়ে গেলে, `for` লুপ স্বয়ংক্রিয়ভাবে শেষ হয়ে যায়।

আমরা `for` লুপ ব্যবহার করে অ্যারের প্রতিটি উপাদান পেতে পারি, এবং এভাবে কাউন্টডাউনের মতো কাজ সম্পন্ন করতে পারি।

**টিপস**: `i` হলো `for` লুপে খুব সাধারণ একটি নামকরণ, যা বর্তমান বের করা উপাদানকে বোঝাতে ব্যবহৃত হয়। তবে এটি নির্দিষ্ট নয়, অন্য ইংরেজি নামও ব্যবহার করা যায়।

উদাহরণস্বরূপ, প্রতিটি বের করা উপাদান বোঝাতে `num` ব্যবহার করা যায়:

```swift
for num in countDown {
    print(num)
}
```

ব্রেসের ভেতরেও বর্তমান উপাদান বোঝাতে `num`-ই ব্যবহার করতে হবে।

### রেঞ্জে লুপ

`for` লুপ শুধু অ্যারে-তে নয়, রেঞ্জেও চলতে পারে।

উদাহরণস্বরূপ, যদি আমরা `50`-এর মধ্যে সব ধনাত্মক পূর্ণসংখ্যার যোগফল গণনা করতে চাই, তাহলে `1` থেকে `50` পর্যন্ত সব সংখ্যাকে একে একে যোগ করতে হবে।

সবচেয়ে সরাসরি লেখাটি এমন হতে পারে:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

এই লেখার সমস্যা হলো, প্রতিটি সংখ্যাই হাতে লিখতে হয়। এটি ঝামেলাপূর্ণ, এবং বাস্তব ডেভেলপমেন্টের জন্য উপযুক্ত নয়।

এই সময়, আমরা রেঞ্জ অপারেটরকে `for` লুপের সাথে ব্যবহার করতে পারি।

আগের “অপারেটর” অধ্যায়ে আমরা রেঞ্জ অপারেটর শিখেছি:

```swift
1...3   // 1, 2, 3
```

এখানে `...` হলো closed range, অর্থাৎ শুরু এবং শেষ—দুটো মানই এর মধ্যে অন্তর্ভুক্ত।

তাই এখানে `1` থেকে `50` পর্যন্ত পূর্ণসংখ্যার রেঞ্জে `for` লুপ ব্যবহার করা যায়:

```swift
for i in 1...50 {
    
}
```

আমাদের এই পূর্ণসংখ্যাগুলো যোগ করতে হবে, তাই মোট যোগফল রাখার জন্য একটি ভ্যারিয়েবলও দরকার:

```swift
var count = 0
```

তারপর প্রতিবার লুপ চললে, বর্তমান বের করা সংখ্যাটি `count`-এ যোগ করব:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

এই কোডের অর্থ হলো:

* `count` বর্তমান মোট যোগফল সংরক্ষণ করে
* `for` লুপ `1` থেকে `50` পর্যন্ত পূর্ণসংখ্যা একে একে বের করবে
* প্রতিবার একটি পূর্ণসংখ্যা বের হলে, `count += i` চালানো হবে
* লুপ শেষ হলে, `count`-এর মধ্যে চূড়ান্ত যোগফল সংরক্ষিত থাকবে

এভাবে আমরা `50`-এর মধ্যে সব ধনাত্মক পূর্ণসংখ্যার যোগফল পেতে পারি।

একইভাবে, যদি আমরা `1` থেকে `100` পর্যন্ত পূর্ণসংখ্যার যোগফল হিসাব করতে চাই, তাহলে শুধু রেঞ্জ পরিবর্তন করলেই হবে:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` লুপ শুধু অ্যারে-তেই নয়, একটি রেঞ্জেও চলতে পারে।

এইভাবে ক্রমানুসারে পুনরাবৃত্ত ডেটা প্রক্রিয়া করার ক্ষমতা ডেভেলপমেন্টে খুবই সাধারণ।

এটি আমাদের অনেক পুনরাবৃত্ত কাজ স্বয়ংক্রিয়ভাবে সম্পন্ন করতে সাহায্য করে, প্রতিটি সংখ্যা হাতে লিখে যোগ করার দরকার হয় না, বা এরকম অন্য পুনরাবৃত্ত কাজও হাতে করতে হয় না।

## SwiftUI-তে লুপ

আগের `for` লুপটি Swift ভাষার অংশ।

কিন্তু SwiftUI-তে আমরা ভিউ স্ট্রাকচারের ভিতরে সরাসরি `for` ব্যবহার করে ভিউ তৈরি করতে পারি না:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

কারণ `VStack`-এর ভেতরে SwiftUI ভিউ থাকতে হয়, কিন্তু সাধারণ `for` লুপ নিজে কোনো ভিউ নয়।

যদি আমরা SwiftUI-তে অ্যারের কনটেন্ট অনুযায়ী বারবার ইন্টারফেস তৈরি করতে চাই, তাহলে `ForEach` ব্যবহার করতে হবে।

উদাহরণস্বরূপ, যদি আমরা একটি দলের সব সদস্যের নাম দেখাতে চাই, তাহলে এভাবে লিখতে পারি:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

প্রদর্শনের ফলাফল:

![for1](../../Resource/017_for1.png)

এরপর আমরা প্রতিটি `Text`-এ একই modifier যোগ করি:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

এখন প্রতিটি `Text` আরও ছোট এবং মোটা ফন্টে দেখাবে।

প্রদর্শনের ফলাফল:

![](../../Resource/017_for2.png)

যদিও modifier-গুলো ইন্টারফেসকে আরও সুন্দর করে, এগুলো কোডকে আরও দীর্ঘ এবং পুনরাবৃত্তিমূলকও করে তোলে।

যদি নাম আরও বাড়তে থাকে, বা পরে একসাথে ফন্ট, রঙ ইত্যাদি স্টাইল পরিবর্তন করতে চাই, তাহলে আমাদের অনেক একই ধরনের কোড বারবার লিখতে ও পরিবর্তন করতে হবে, যা ঝামেলাপূর্ণ এবং রক্ষণাবেক্ষণের জন্যও সুবিধাজনক নয়।

এই সময় আমরা নামগুলো আগে একটি অ্যারেতে রাখতে পারি, তারপর SwiftUI-এর `ForEach` ব্যবহার করে অ্যারের ডেটা অনুযায়ী বারবার ভিউ তৈরি করতে পারি:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

এই কোডটি আরও সংক্ষিপ্ত। `ForEach` অ্যারের প্রতিটি উপাদান একে একে পড়বে, এবং সংশ্লিষ্ট কনটেন্ট ইন্টারফেসে দেখাবে।

পরে নতুন নাম যোগ করতে চাইলে, শুধু `names` অ্যারের কনটেন্ট বদলালেই হবে। প্রতিবার নতুন `Text` এবং একই modifier আবার লিখতে হবে না।

## ForEach

`ForEach` হলো SwiftUI-তে বারবার ভিউ তৈরি করার জন্য ব্যবহৃত একটি স্ট্রাকচার। একে SwiftUI-র মধ্যে `for` লুপের মতো একটি স্ট্রাকচার হিসেবে বোঝা যায়।

মৌলিক ব্যবহার:

```swift
ForEach(অ্যারে, id: \.self) { উপাদান in
    SwiftUI কোড
}
```

উদাহরণস্বরূপ, একটু আগের দলের নামগুলো:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

এই কোডের অর্থ হলো:

`names` হলো যে অ্যারেটির ওপর লুপ চলছে, অর্থাৎ যে ডেটা দেখানো হবে।

`id:\.self` বোঝায় যে অ্যারের উপাদানটিকেই প্রতিটি আইটেমকে আলাদা করার জন্য ব্যবহার করা হবে।

বর্তমান উদাহরণে, অ্যারেতে স্ট্রিং রয়েছে:

```swift
["Sam", "John", "Wathon", "Bob"]
```

তাই SwiftUI সরাসরি `"Sam"`、`"John"`、`"Wathon"`、`"Bob"`—এই স্ট্রিংগুলোকে আলাদা আইটেম হিসেবে শনাক্ত করতে ব্যবহার করবে।

শুরুর পর্যায়ের সাধারণ `String`、`Int` ধরনের সহজ অ্যারের জন্য আমরা সাধারণত সরাসরি লিখতে পারি:

```swift
id: \.self
```

এই মুহূর্তে আপাতত এই একটিই মনে রাখলেই যথেষ্ট, `id`-এর অন্য নিয়মগুলো নিয়ে এখন গভীরে যাওয়ার দরকার নেই।

`item in`-এ `item` বর্তমান বের করা উপাদানকে বোঝায়।

উদাহরণস্বরূপ, প্রথম লুপে `item` হলো `"Sam"` এবং দ্বিতীয় লুপে `item` হলো `"John"`।

তাই:

```swift
Text(item)
```

ক্রমে এটি হয়ে যায়:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

অতএব, এই কোডের কাজ হলো: `names` অ্যারে থেকে একে একে প্রতিটি নাম বের করা, এবং প্রতিটি নামের জন্য একটি `Text` ভিউ তৈরি করা।

### কার্যপ্রক্রিয়া

কার্যপ্রক্রিয়াটি এভাবে বোঝা যেতে পারে:

* `ForEach` আগে `names` অ্যারের প্রথম উপাদান `"Sam"` পড়ে
* `item` সাময়িকভাবে `"Sam"`-কে বোঝায়
* `Text(item)` `"Sam"` দেখায়
* তারপর পরের উপাদান `"John"` পড়ে
* এরপর একই ভিউ কোড আবার চালানো হয়

শেষ পর্যন্ত, অ্যারের সব নামই দেখানো হবে।

এভাবে লেখার সুবিধা হলো: নামের সংখ্যা বেড়ে গেলে, আমাদের অনেকগুলো `Text` বারবার লিখতে হয় না, শুধু অ্যারের কনটেন্ট বদলালেই হয়।

### ছবি লুপ করা

`ForEach` শুধু লেখা নয়, ছবি, রঙ, বা অন্য SwiftUI ভিউও দেখাতে পারে।

উদাহরণস্বরূপ, আমরা 4টি ছবি প্রস্তুত করি:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

এই চারটি ছবি Xcode-এর Assets ফোল্ডারে টেনে এনে রাখুন।

![image](../../Resource/017_for3.png)

যদি `ForEach` ব্যবহার না করি, তাহলে প্রতিটি ছবিই হাতে লিখতে হবে:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

প্রদর্শনের ফলাফল:

![image](../../Resource/017_for4.png)

এইভাবে কাজ করা যায়, কিন্তু কোড তুলনামূলকভাবে দীর্ঘ হয়ে যায়।

ছবি যত বাড়বে, ততই আরও বেশি `Image` কোড বারবার লিখতে হবে।

পরে যদি সব ছবির আকার একসাথে পরিবর্তন করতে চাই, তাহলে প্রতিটি আলাদা আলাদা করে বদলাতে হবে।

এই সময়, আমরা ছবির নামগুলোকে একটি অ্যারেতে রাখতে পারি, তারপর `ForEach` ব্যবহার করে বারবার ছবি ভিউ তৈরি করতে পারি:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

এখানে `images` হলো একটি স্ট্রিং অ্যারে, যার প্রতিটি উপাদানই একটি ছবির নাম।

`ForEach(images, id: \.self)` বোঝায়, `images` অ্যারের প্রতিটি উপাদান একে একে পড়া হবে, উপাদানটিকেই আলাদা করার জন্য ব্যবহার করা হবে, তারপর বর্তমান উপাদান অনুযায়ী সংশ্লিষ্ট ভিউ তৈরি করা হবে।

উদাহরণস্বরূপ, যদি প্রথমে `"430F9BEF"` পড়া হয়, তাহলে:

```swift
Image(item)
```

এটি সমতুল্য:

```swift
Image("430F9BEF")
```

পরের ছবিগুলোও একইভাবে একে একে দেখানো হবে।

![image](../../Resource/017_for4.png)

এখানেও আমরা এখনও ব্যবহার করছি:

```swift
id: \.self
```

কারণ আগের মতোই, `images` অ্যারের উপাদানগুলোও সহজ স্ট্রিং টাইপ, তাই শুরুর পর্যায়ে আমরা সরাসরি উপাদানটিকেই আলাদা করার জন্য ব্যবহার করতে পারি।

এটিকে একটি নির্দিষ্ট সূত্র হিসেবে সহজভাবে বোঝা যায়:

**যখন `ForEach` `String` বা `Int` ধরনের সহজ অ্যারে নিয়ে কাজ করে, তখন সাধারণত `id: \.self` লেখা হয়।**

### নতুন ছবি যোগ করা

পরে যদি নতুন ছবি যোগ করতে হয়, তাহলে শুধু অ্যারের কনটেন্ট বদলালেই হবে:

```swift
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

নতুন করে `Image` কোড বারবার লিখতে হবে না।

যদি ছবির সংখ্যা বেশি হয়, তাহলে `ScrollView`-এর সাথে ব্যবহার করা যায়, নাহলে স্ক্রিনের বাইরে চলে যাওয়া কনটেন্ট পুরো দেখা যাবে না।

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### ছবি পরিবর্তন

যখন আমাদের সব ছবির স্টাইল একসাথে পরিবর্তন করতে হবে, তখন শুধু `ForEach`-এর ভেতরের কোড বদলালেই হবে।

উদাহরণস্বরূপ, যদি প্রস্থ `80` করে দিই:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

তাহলে সব ছবিই একসাথে পরিবর্তিত হবে, একে একে বদলাতে হবে না।

এটাই `ForEach`-এর একটি গুরুত্বপূর্ণ কাজ:

**যখন অনেকগুলো ভিউয়ের গঠন একই, শুধু কনটেন্ট আলাদা, তখন আমরা একবারই ভিউ কোড লিখতে পারি, আর অ্যারের ডেটা ঠিক করে দেয় কী দেখানো হবে।**

## সারসংক্ষেপ

এই পাঠে আমরা `for` লুপ এবং `ForEach` শিখেছি।

`for` লুপ অ্যারে বা রেঞ্জের উপাদানগুলোকে ক্রমানুসারে প্রক্রিয়া করতে ব্যবহৃত হয়, এবং এটি মূলত ডেটা প্রক্রিয়াকরণের দিকে বেশি ঝোঁকে।

`ForEach` SwiftUI-তে অ্যারের কনটেন্ট অনুযায়ী বারবার ভিউ তৈরি করতে ব্যবহৃত হয়, এবং এটি ভিউ প্রদর্শনের দিকে বেশি মনোযোগী।

বর্তমান শুরুর পর্যায়ে `ForEach`-এর সবচেয়ে সাধারণ লেখাটি হলো:

```swift
ForEach(অ্যারে, id: \.self) { উপাদান in
    SwiftUI কোড
}
```

এখানে `id: \.self` বোঝায় প্রতিটি উপাদানকে আলাদা করতে উপাদানটিকেই ব্যবহার করা হচ্ছে।

যখন অ্যারের কনটেন্ট `String`、`Int`-এর মতো সহজ টাইপ হয়, তখন আমরা সাধারণত শুরুতে এই একটিই ব্যবহার করতে পারি।

ভবিষ্যতে, যখন আপনি ওয়েব ব্রাউজ করবেন বা App ব্যবহার করার সময় একগুচ্ছ পুনরাবৃত্ত লেখা, ছবি বা অন্য তালিকাভিত্তিক কনটেন্ট দেখবেন, তখন মনে রাখতে পারেন যে SwiftUI-তে এগুলো সাধারণত আগে অ্যারেতে রাখা হয়, তারপর `ForEach` দিয়ে একে একে দেখানো হয়।
