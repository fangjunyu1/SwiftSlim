# Swift এবং SwiftUI

অ্যাপল (Apple) প্ল্যাটফর্ম ডেভেলপমেন্ট শেখার সময়, অনেক টিউটোরিয়াল Swift এবং SwiftUI-কে আলাদাভাবে ব্যাখ্যা করার প্রবণতা দেখায়, যা নতুনদের মধ্যে বিভ্রান্তি সৃষ্টি করতে পারে এবং তারা ভুলবশত এগুলোকে দুটি সম্পূর্ণ স্বাধীন প্রযুক্তি হিসেবে ভাবতে পারে।

প্রকৃতপক্ষে, Swift এবং SwiftUI গভীরভাবে একত্রিত একটি সামগ্রিক ব্যবস্থা: Swift হলো প্রোগ্রামিং ভাষা, যা লজিক এবং ডেটা প্রদান করে; আর SwiftUI হলো Swift-এর ওপর ভিত্তি করে তৈরি করা একটি ইউজার ইন্টারফেস ফ্রেমওয়ার্ক, যা ভিউ রেন্ডারিং এবং ইন্টারঅ্যাকশনের জন্য দায়ী।

এই পাঠটি আপনাকে একটি স্পষ্ট ধারণাগত কাঠামো তৈরি করতে সাহায্য করবে: Swift কী, SwiftUI কী এবং কোডে এরা কীভাবে একে অপরের সাথে সহযোগিতা করে।

## Swift কী?

Swift হলো অ্যাপল কোম্পানি দ্বারা প্রবর্তিত একটি আধুনিক প্রোগ্রামিং ভাষা, যা iOS, macOS, watchOS এবং tvOS-এর মতো প্ল্যাটফর্মগুলোতে অ্যাপ্লিকেশন ডেভেলপমেন্টের জন্য ব্যবহৃত হয়।

![swift](../../RESOURCE/010_swift.png)

Swift একটি স্ট্রং-টাইপড এবং স্ট্যাটিক ভাষা, যার ডিজাইনে নিরাপত্তার ওপর বেশি জোর দেওয়া হয়েছে। এটি সাধারণ প্রোগ্রামিং ত্রুটিগুলো (যেমন নাল পয়েন্টার, আউট অব বাউন্ডস ইত্যাদি) অনেক কমিয়ে দেয় এবং C ও Objective-C এর সাথে নিরবচ্ছিন্ন ইন্টিগ্রেশন সমর্থন করে।

বাস্তব ডেভেলপমেন্টে, Swift সাধারণত অ্যাপ্লিকেশনের লজিক অংশ পরিচালনার জন্য দায়ী থাকে, যেমন ডেটা প্রসেসিং, নেটওয়ার্ক রিকোয়েস্ট, স্টোরেজ অপারেশন ইত্যাদি।

## SwiftUI কী?

SwiftUI হলো ২০১৯ সালে অ্যাপল দ্বারা চালু করা একটি নতুন ইন্টারফেস বিল্ডিং ফ্রেমওয়ার্ক। SwiftUI ডিক্লারেটিভ প্রোগ্রামিং (declarative programming) মডেল ব্যবহার করে, যা ডেভেলপারদের আরও সংক্ষিপ্ত কোডের মাধ্যমে ইন্টারফেসের গঠন এবং ইন্টারঅ্যাকশন বর্ণনা করতে দেয়।

![swiftui](../../RESOURCE/010_swiftui.png)

ডিক্লারেটিভ প্রোগ্রামিং এর অর্থ হলো ডেভেলপারকে কেবল সিস্টেমকে বলতে হবে সে কী দেখাতে চায়, এবং সিস্টেম ডেটার পরিবর্তনের সাথে সাথে ভিউ স্বয়ংক্রিয়ভাবে রিফ্রেশ করবে। ইন্টারফেসের স্টেট ম্যানুয়ালি আপডেট করার আর প্রয়োজন হয় না, যা UI ডেভেলপমেন্টের জটিলতাকে ব্যাপকভাবে কমিয়ে দেয়।

SwiftUI এর মূলে রয়েছে ভিউ কম্পোনেন্ট (যেমন Text, Image, Button ইত্যাদি) এবং লেআউট কন্টেইনার (যেমন VStack, HStack, ZStack)। এই কম্পোনেন্টগুলো ডেটা বাইন্ডিং এবং স্টেট ম্যানেজমেন্টের মাধ্যমে ইন্টারঅ্যাক্ট করে, যার ফলে ইন্টারফেস ডেটার পরিবর্তনে সাড়া দিতে এবং স্বয়ংক্রিয়ভাবে রিফ্রেশ হতে পারে।

## Swift এবং SwiftUI কীভাবে সহযোগিতা করে?

Swift এবং SwiftUI এর দায়িত্বগুলো নিচের পয়েন্টগুলোর মাধ্যমে সংক্ষেপে তুলে ধরা যেতে পারে:

**১. Swift: লজিক এবং ডেটা প্রসেস করে**

Swift মূলত ডেটা পরিচালনা, স্টেট সংরক্ষণ এবং লজিক কার্যকর করার জন্য ব্যবহৃত হয়। এই কোডগুলো সাধারণত সরাসরি ইন্টারফেস রেন্ডারিংয়ে প্রভাব ফেলে না, বরং ডেটা এবং আচরণের প্রক্রিয়াকরণের জন্য দায়ী থাকে।

উদাহরণস্বরূপ, Swift-এ ডেটা প্রসেসিং:

```swift
let markdown = try? String(contentsOf: url)
```

এই কোডগুলো ভেরিয়েবল ঘোষণা, কনস্ট্যান্ট সংজ্ঞায়িত করা, ফাংশন, কন্ট্রোল ফ্লো ইত্যাদির সাথে জড়িত, যা সরাসরি ভিউ প্রদর্শনে প্রভাব ফেলে না।

**২. SwiftUI: ইন্টারফেস ঘোষণা এবং রেন্ডার করে**

অন্যদিকে, SwiftUI ইউজার ইন্টারফেসের লেআউট এবং কন্টেন্ট তৈরি করতে ব্যবহৃত হয়, যা মূলত ইন্টারফেস রেন্ডারিং এবং ভিউ ইন্টারঅ্যাকশনের জন্য দায়ী।

উদাহরণস্বরূপ, SwiftUI ব্যবহার করে একটি টেক্সট ভিউ তৈরি করা:

```swift
Text("SwiftSlim")
```

এগুলো ইন্টারফেসের উপাদান তৈরি এবং নিয়ন্ত্রণ করার কোড, যা সরাসরি ইউজার ইন্টারফেসের সাথে ইন্টারঅ্যাক্ট করে।

### বেসিক উদাহরণ

নিচে একটি সাধারণ SwiftUI এর উদাহরণ দেওয়া হলো:

```swift
// ContentView.swift

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

এই উদাহরণে:

**SwiftUI কোড**: VStack, Image, Text, padding সবগুলোই SwiftUI এর অন্তর্ভুক্ত, যা ইন্টারফেসের প্রদর্শন এবং লেআউটের জন্য দায়ী।

**Swift কোড**: ফাইলের উপরের // কমেন্ট এবং import SwiftUI হলো Swift কোডের কাঠামো; নিচের #Preview হলো Xcode প্রিভিউয়ের জন্য ব্যবহৃত ম্যাক্রো, যা প্রকৃত ইন্টারফেস রেন্ডারিংয়ে অংশ নেয় না।

### অ্যাডভান্সড উদাহরণ

বাস্তব প্রজেক্টে, Swift এবং SwiftUI প্রায়শই তাদের নিজ নিজ কাজে পারদর্শী অংশগুলো সম্পন্ন করার জন্য মিশ্রিতভাবে ব্যবহৃত হয়:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    func printName() {
        print("name: \(name)")
    }

    var body: some View {
        VStack {
            Text(name)
            Button("Print") {
                printName()
            }
        }
        .padding()
    }
}
```

এই উদাহরণে:

**SwiftUI কোড**: @State হলো SwiftUI-এর নিজস্ব প্রোপার্টি র‍্যাপার, যা ভিউয়ের পরিবর্তনযোগ্য স্টেট ঘোষণা করতে ব্যবহৃত হয়; Button এবং Text হলো SwiftUI ভিউ কম্পোনেন্ট, যা ইন্টারফেস প্রদর্শন এবং ব্যবহারকারীর ইন্টারঅ্যাকশনের জন্য দায়ী।

**Swift কোড**: private var name এবং func printName() হলো Swift কোড, যা ডেটা সংরক্ষণ এবং লজিক কার্যকর করার জন্য ব্যবহৃত হয় এবং সরাসরি ভিউ রেন্ডারিংয়ে প্রভাব ফেলে না।

যখন ব্যবহারকারী বাটনে ক্লিক করেন, তখন SwiftUI বাটনের অপারেশন কোড ট্রিগার করে:

```swift
Button("Print") {
    printName()
}
```

এখানে, Button হলো SwiftUI কম্পোনেন্ট, এবং কার্যকরী printName() ফাংশনটি হলো Swift কোড, যা নির্দিষ্ট লজিক প্রক্রিয়াকরণের জন্য দায়ী।

এই সহযোগিতা Swift এবং SwiftUI-কে নির্বিঘ্নে একত্রিত হতে সাহায্য করে, যেখানে Swift ডেটা ও লজিক প্রক্রিয়াকরণের জন্য দায়ী এবং SwiftUI ইউজার ইন্টারফেস প্রদর্শনের জন্য দায়ী।

## Swift এবং SwiftUI কোড সাধারণত কোথায় লেখা হয়?

SwiftUI-এ, body প্রোপার্টি দ্বারা রিটার্ন করা ভিউয়ের মাধ্যমে ইন্টারফেস তৈরি করা হয়। তাই, ইন্টারফেস বর্ণনা করার জন্য ব্যবহৃত সমস্ত কোড সাধারণত body-এর মধ্যে লেখা হয়।

উদাহরণস্বরূপ:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

এই উদাহরণে, Text হলো SwiftUI ভিউ কম্পোনেন্ট, তাই এটিকে অবশ্যই body-এর মধ্যে লিখতে হবে, কারণ SwiftUI এই body-এর মাধ্যমেই ইন্টারফেস পড়ে এবং তৈরি করে।

ইন্টারফেসের সাথে সম্পর্কিত নয় এমন কোড, যেমন ভেরিয়েবল, ফাংশন বা ডেটা প্রক্রিয়াকরণের লজিক, সাধারণত body-এর বাইরে লেখা হয়। উদাহরণস্বরূপ:

```swift
struct ContentView: View {

    // Swift: ডেটা বা লজিক
    var name = "Fang"

    func printName() {
        print(name)
    }

    // SwiftUI: ইন্টারফেস
    var body: some View {
        Text(name)
    }
}
```

উল্লেখ্য যে, body-এর মধ্যেও Swift সিনট্যাক্স ব্যবহার করা যেতে পারে, যেমন if, for ইত্যাদির মতো কন্ট্রোল ফ্লো স্টেটমেন্ট, এগুলো শুধুমাত্র SwiftUI ভিউ তৈরি নিয়ন্ত্রণ করার জন্য ব্যবহৃত হয়:

```
if isLogin {
    Text("Welcome")
}
```

অতএব, SwiftUI ডেভেলপমেন্টে সহজেই এটি বোঝা যেতে পারে যে: ভিউ কোড (Text, Image, Button ইত্যাদি) সাধারণত body-তে লেখা হয়; এবং ডেটা ও লজিক কোড (ভেরিয়েবল, ফাংশন ইত্যাদি) সাধারণত body-এর বাইরে লেখা হয়।

## Swift ফাইল

শেখার গভীরতা বাড়ার সাথে সাথে, আমরা MVVM আর্কিটেকচারের সাথেও পরিচিত হব, যেখানে ViewModel এবং Model লেয়ারগুলো সাধারণত বিশুদ্ধ Swift কোড দ্বারা গঠিত হয় এবং ভিউ লেয়ার (SwiftUI) থেকে সম্পূর্ণ আলাদা থাকে।

উদাহরণস্বরূপ, অ্যাপ্লিকেশনের স্টেট পরিচালনার জন্য ব্যবহৃত ক্লাস:

```swift
class AppStorage: ObservableObject {
    static var shared = AppStorage()
    private var isLoading = false
}
```

অথবা ডেটা স্ট্রাকচার বর্ণনা করার জন্য ব্যবহৃত স্ট্রাকচার (struct):

```
struct SponsorStruct: Identifiable {
    var id:String
    var icon: String
}
```

এ ধরনের ফাইলের দায়িত্ব হলো ডেটা পরিচালনা এবং পারসিস্ট (persist) করা, যা সরাসরি ভিউতে প্রদর্শিত হয় না, তাই এগুলো সম্পূর্ণভাবে Swift-এর অন্তর্গত।

## ঐতিহাসিক পটভূমি

Swift এবং SwiftUI এর সম্পর্ক বুঝতে হলে অ্যাপল ডেভেলপমেন্ট প্রযুক্তির বিবর্তনের ইতিহাস জানাও প্রয়োজন। ২০২৬ সাল পর্যন্ত, এই টেকনোলজি স্ট্যাকটি বেশ কয়েকবার ইটারেশনের মধ্য দিয়ে গেছে।

### Swift এর ইতিহাস

Swift আসার আগে, Objective-C ছিল অ্যাপল প্ল্যাটফর্মের প্রধান প্রোগ্রামিং ভাষা, যা C ভাষার সাথে মিশ্রিত কোডিং সমর্থন করত। এর সিনট্যাক্স তুলনামূলকভাবে দীর্ঘ এবং নতুনদের জন্য বেশ কঠিন ছিল:

```Obj-c
// Objective-C এর লেখার পদ্ধতি
NSString *name = @"Fang";
NSLog(@"Hello，%@", name);
```

২০১৪ সালে, অ্যাপল WWDC-তে Swift ভাষা প্রকাশ করে। Swift এর সিনট্যাক্স অনেক বেশি আধুনিক, টাইপ সেফটি উচ্চতর, এবং এটি ধীরে ধীরে Objective-C এর জায়গা দখল করে মূল ডেভেলপমেন্ট ভাষা হয়ে ওঠে:

```swift
// Swift এর লেখার পদ্ধতি
let name = "Fang"
print("Hello, \(name)")
```

তবে, Objective-C ইতিহাসের পাতা থেকে মুছে যায়নি; এটি এখনও অনেক বিদ্যমান প্রজেক্ট এবং আন্ডারলাইং ফ্রেমওয়ার্কগুলোতে ব্যাপকভাবে ব্যবহৃত হয়। এর বেসিক সিনট্যাক্স জানা থাকলে তা পুরনো প্রজেক্ট রক্ষণাবেক্ষণ এবং সিস্টেমের আন্ডারলাইং লেয়ার বোঝার জন্য এখনও মূল্যবান।

### SwiftUI এর ইতিহাস

SwiftUI আসার আগে, iOS-এ **UIKit** এবং macOS-এ **AppKit** ব্যবহৃত হতো। এই দুটি ফ্রেমওয়ার্ক "ইম্পারেটিভ" (imperative) প্রোগ্রামিং পদ্ধতি ব্যবহার করত। ডেভেলপারদের স্টোরিবোর্ড (Storyboard) এর মাধ্যমে কন্ট্রোল টেনে বা ম্যানুয়ালি কোড লিখে ভিউ স্টেট নিয়ন্ত্রণ করতে হতো, যার ফলে কোডের পরিমাণ বেশি হতো এবং রক্ষণাবেক্ষণ খরচও বাড়ত, বিশেষ করে ইন্টারফেস লজিক জটিল হলে এটি বেশি চোখে পড়ত।

![storyboard](../../RESOURCE/010_xcode.png)

২০১৯ সালে, অ্যাপল WWDC-তে আনুষ্ঠানিকভাবে SwiftUI রিলিজ করে। SwiftUI "ডিক্লারেটিভ" প্রোগ্রামিং প্যারাডাইম চালু করেছে, যা UI ডেভেলপমেন্ট প্রক্রিয়াকে ব্যাপকভাবে সহজ করেছে।

![storyboard](../../RESOURCE/010_xcode1.png)

এটি লক্ষণীয় যে, SwiftUI সম্পূর্ণ স্বাধীন কোনো আন্ডারলাইং ইমপ্লিমেন্টেশন নয়। বিভিন্ন প্ল্যাটফর্মে, এটি মূলত UIKit (iOS) বা AppKit (macOS) এর সাথে ব্রিজ এবং কোলাবোরেশন করে কাজ করে, এবং এটি রান করার জন্য এখনও এই দুটি ফ্রেমওয়ার্কের উপর ভিত্তি করে নির্ভর করে।

### Swift এবং UIKit/AppKit এর সম্পর্ক

যদিও Swift একটি সাধারণ প্রোগ্রামিং ভাষা যা অ্যাপলের সমস্ত প্ল্যাটফর্মে চলতে পারে, তবে এটি সম্পূর্ণভাবে UIKit বা AppKit কে প্রতিস্থাপন করতে পারে না। কিছু জটিল ইন্টারফেসের প্রয়োজনীয়তার জন্য, অথবা যেসব ফিচার এখনও SwiftUI দ্বারা কভার করা হয়নি, সেগুলোর জন্য এখনও UIKit বা AppKit এর সাহায্য নেওয়া প্রয়োজন।

উদাহরণস্বরূপ, জটিল ভিউ কন্ট্রোলার ম্যানেজমেন্ট, অ্যানিমেশন ইফেক্ট, জেসচার রিকগনিশন ইত্যাদি পরিচালনায় UIKit ইতিমধ্যে বেশ পরিণত, এবং এর প্রোডাকশন লেভেলের ব্যাপক ব্যবহারিক প্রমাণ রয়েছে। অন্যদিকে, এই ক্ষেত্রগুলোতে SwiftUI এর সক্ষমতা ক্রমাগত বৃদ্ধি পেলেও, কিছু এজ কেস (edge case) পরিস্থিতিতে এর এখনও সীমাবদ্ধতা রয়েছে।

তাই, অনেক ডেভেলপার প্রজেক্টে SwiftUI এবং UIKit (বা AppKit) এর মিশ্র ব্যবহার করে থাকেন, যাতে উভয়ের নিজ নিজ সুবিধার পূর্ণ ব্যবহার করা যায়।

এই দৃষ্টিকোণ থেকে বুঝতে গেলে, SwiftUI কে UIKit / AppKit এর একটি হাই-লেভেল এনক্যাপসুলেশন (উন্নত মোড়ক) হিসেবে বিবেচনা করা যেতে পারে। SwiftUI শেখার পাশাপাশি UIKit এবং AppKit এর বেসিক ধারণাগুলো সম্পর্কে কিছুটা জানা থাকলে, তা পুরনো প্রজেক্টের রক্ষণাবেক্ষণ বা জটিল ফাংশন বাস্তবায়নের সময় আরও যৌক্তিক প্রযুক্তিগত সিদ্ধান্ত নিতে সাহায্য করে।

## সারসংক্ষেপ

**Swift**: মূলত লজিক লেখা, ডেটা প্রসেসিং, কন্ট্রোল ফ্লো ইত্যাদির জন্য ব্যবহৃত হয়, এর সাথে ভিউ লেআউটের কোনো সম্পর্ক নেই।

**SwiftUI**: ডিক্লারেটিভভাবে ইউজার ইন্টারফেস তৈরি করতে ব্যবহৃত হয়, ভিউয়ের কন্টেন্ট এবং লেআউটের কোডগুলো SwiftUI এর অন্তর্গত।

বাস্তব ডেভেলপমেন্টে, Swift এবং SwiftUI সাধারণত একত্রে ব্যবহৃত হয়, যেখানে Swift লজিক পরিচালনা করে এবং SwiftUI ইন্টারফেস পরিচালনা করে।

Objective-C এবং UIKit থেকে শুরু করে Swift এবং SwiftUI পর্যন্ত, অ্যাপলের ডেভেলপমেন্ট পরিবেশ ধীরে ধীরে আরও আধুনিক এবং সহজ প্রোগ্রামিং পদ্ধতির দিকে অগ্রসর হচ্ছে, তবে অনেক পুরোনো বা ঐতিহ্যবাহী প্রজেক্টে এখনও UIKit এবং AppKit কে উপেক্ষা করার সুযোগ নেই।

Swift এবং SwiftUI এর সম্পর্ক বোঝার মাধ্যমে, আমরা আরও দক্ষতার সাথে iOS/macOS ডেভেলপমেন্ট করতে পারি, এবং পুরনো প্রজেক্টগুলো রক্ষণাবেক্ষণ করার সময় আরও যৌক্তিক প্রযুক্তিগত পছন্দগুলোও বেছে নিতে পারি।
