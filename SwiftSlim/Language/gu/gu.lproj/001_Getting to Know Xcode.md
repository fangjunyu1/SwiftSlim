# Xcode ને ઓળખીએ

## Xcode શું છે

Xcode એ Apple દ્વારા આપવામાં આવેલ અધિકૃત Integrated Development Environment (IDE) છે, જે iOS, macOS, watchOS, tvOS જેવી Apple પ્લેટફોર્મ એપ્લિકેશન્સના વિકાસ માટે ઉપયોગમાં આવે છે.

![Swift](../../RESOURCE/001_xcode.png)

આ સમગ્ર કોર્સ દરમિયાન આપણે Xcode નો ઉપયોગ કરીને કોડ લખીશું અને પ્રોગ્રામ ચલાવીશું.

આ પાઠમાં આપણે Xcode કેવી રીતે ડાઉનલોડ કરવું, પ્રથમ iOS પ્રોજેક્ટ કેવી રીતે બનાવવો, અને Xcode ના ઈન્ટરફેસની પ્રાથમિક રચનાને કેવી રીતે ઓળખવી તે શીખીશું.

## Xcode ડાઉનલોડ કરવું

હાલમાં Xcode ઇન્સ્ટોલ કરવા માટે બે રીતો ઉપલબ્ધ છે:

1、[Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12) પરથી ડાઉનલોડ

2、[Apple Developer અધિકૃત વેબસાઇટ](https://developer.apple.com/xcode/) પરથી ડાઉનલોડ

બંને રીતોથી Xcode મેળવી શકાય છે. તફાવત એ છે કે Apple Developer વેબસાઇટ પરથી ફક્ત સત્તાવાર આવૃત્તિ જ નહીં, પણ [Beta](https://developer.apple.com/download/all/?q=Xcode) અથવા જૂની આવૃત્તિઓ પણ ડાઉનલોડ કરી શકાય છે.

જો તમે માત્ર શીખવા માટે વિકાસ કરી રહ્યા હો, તો સત્તાવાર આવૃત્તિનો ઉપયોગ કરવાની ભલામણ થાય છે. Beta આવૃત્તિઓ મુખ્યત્વે નવી સુવિધાઓનું પરીક્ષણ કરવા માટે હોય છે અને તેમાં અસ્થિરતા હોઈ શકે છે.

## પ્રથમ પ્રોજેક્ટ બનાવવો

Xcode ખોલ્યા પછી તમે સ્ટાર્ટ સ્ક્રીન જુઓ છો.

![Swift](../../RESOURCE/001_xcode1.png)

ડાબી બાજુએ ત્રણ સામાન્ય વિકલ્પો હોય છે:

- Create New Project —— નવો પ્રોજેક્ટ બનાવવો  
- Clone Git Repository —— Git રિપોઝિટરી ક્લોન કરવી  
- Open Existing Project —— પહેલેથી અસ્તિત્વમાં રહેલો પ્રોજેક્ટ ખોલવો

જમણી બાજુએ તાજેતરમાં ખોલાયેલા Xcode પ્રોજેક્ટ્સ દેખાશે.

અમે `"Create New Project"` વિકલ્પ પસંદ કરીએ છીએ.

### પ્રોજેક્ટ ટેમ્પલેટ પસંદ કરવું

![Swift](../../RESOURCE/001_xcode2.png)

હવે તમે પ્રોજેક્ટ ટેમ્પલેટ પસંદગી સ્ક્રીન પર આવો છો, જ્યાં વિવિધ પ્લેટફોર્મ (iOS, macOS વગેરે) અને વિવિધ પ્રકારનાં ટેમ્પલેટ્સ દેખાય છે.

સામાન્ય ટેમ્પલેટ્સનું વર્ણન:
- App —— સૌથી મૂળભૂત એપ પ્રકાર (ભલામણ કરાયેલ)
- Document App —— દસ્તાવેજ આધારિત એપ
- Game —— ગેમ વિકાસ માટે
- Framework —— પુનઃઉપયોગી મોડ્યુલ

શરૂઆતના તબક્કામાં આપણે ફક્ત `"iOS" - "App"` ટેમ્પલેટ પસંદ કરીશું, કારણ કે આ ઈન્ટરફેસ આધારિત એપ પ્રકાર છે.

પછી `"Next"` પર ક્લિક કરો.

### પ્રોજેક્ટ માહિતી ભરવી

હવે પ્રોજેક્ટ માહિતી ભરવાની સ્ક્રીન આવશે. અહીં દરેક ફીલ્ડ તેના પ્રકાર મુજબ ભરવાની રહેશે.

![Swift](../../RESOURCE/001_xcode3.png)

દરેક ફીલ્ડનો અર્થ:

- Product Name —— પ્રોજેક્ટનું નામ, ઉદાહરણ: `SwiftSlimTest`
- Team —— ડેવલપર ટીમ; જો તમારી પાસે પેઈડ Apple Developer એકાઉન્ટ ન હોય તો આ ખાલી રાખી શકો
- Organization Identifier —— સંસ્થાનો ઓળખચિહ્ન, સામાન્ય રીતે reverse domain format માં, ઉદાહરણ તરીકે:
  - `com.yourname`
  - `com.yourcompany`
  - જો તમારી પાસે ડોમેન ન હોય, તો તમે તમારી અંગ્રેજી નામનો ઉપયોગ પણ કરી શકો
- Bundle Identifier —— એપનું અનન્ય ઓળખચિહ્ન, જે `Organization Identifier` અને `Product Name` પરથી આપમેળે બને છે
- Interface —— ઈન્ટરફેસ ટેકનોલોજી, અહીં `SwiftUI` પસંદ કરો
- Language —— વિકાસ ભાષા, અહીં `Swift` પસંદ કરો
- Testing System —— મૂળભૂત રીતે Unit Tests માટે target બનાવે છે; શરૂઆતમાં તેને અવગણી શકાય
- Storage —— લોકલ ડેટા persistence framework, જેમાં SwiftData અથવા Core Data આપમેળે જોડાઈ શકે; શરૂઆતમાં તેને પણ અવગણો

ભરણાં પૂર્ણ થયા પછી `"Next"` પર ક્લિક કરો.

### પ્રોજેક્ટ સંગ્રહિત કરવો

![Swift](../../RESOURCE/001_xcode4.png)

પ્રોજેક્ટ સાચવવા માટે યોગ્ય ફોલ્ડર પસંદ કરો.

પછી `"Create"` બટન પર ક્લિક કરો; Xcode આપમેળે પ્રોજેક્ટની રચના બનાવી દેશે.

## Xcode પ્રોજેક્ટ સ્ટ્રક્ચર

Xcode પ્રોજેક્ટ સાચવ્યા પછી, Finder માં બનેલું પ્રોજેક્ટ ફોલ્ડર દેખાશે.

![Swift](../../RESOURCE/001_xcode5.png)

- `Assets.xcassets` —— ચિત્રો, App icons અને અન્ય સ્રોતો રાખવા માટેનું ફોલ્ડર

- `ContentView.swift` —— મૂળભૂત SwiftUI view file, જેમાં આપણે પછી ઈન્ટરફેસ કોડ લખીશું

- `SwiftSlimTestApp.swift` —— એપનું entry file (`App Entry Point`), જે એપ શરૂ કરવાનું કામ કરે છે; હાલમાં તેને વિગતે સમજવાની જરૂર નથી

- `SwiftSlimTest.xcodeproj` —— Xcode project file; આ ફાઇલ પર ડબલ-ક્લિક કરીને પ્રોજેક્ટ ખોલી શકાય છે

## Xcode ઇન્ટરફેસ ઓળખવું

`.xcodeproj` ફાઇલ પર ડબલ-ક્લિક કરીને પ્રોજેક્ટ ખોલો, પછી ડાબી બાજુની `Navigator` વિસ્તારમાં `ContentView.swift` પસંદ કરો.

Xcode નું ઇન્ટરફેસ મુખ્યત્વે પાંચ વિસ્તારોમાં વહેંચાયેલું છે:

- Navigator —— નેવિગેશન વિસ્તાર, ફાઇલ સ્ટ્રક્ચર જોવા અને શોધ કરવા માટે
- Editor —— એડિટિંગ વિસ્તાર, Swift અથવા SwiftUI કોડ લખવા માટે
- Canvas —— કેનવાસ વિસ્તાર, SwiftUI preview માટે
- Inspector —— પ્રોપર્ટી તપાસ વિસ્તાર, ફાઇલની વિગતો જોવા અને બદલવા માટે
- Debug Area —— ડિબગ વિસ્તાર, લોગ આઉટપુટ જોવા માટે

![Swift](../../RESOURCE/001_xcode6.png)

નોંધ: `Canvas` મુખ્યત્વે SwiftUI preview માટે ઉપયોગમાં આવે છે. Xcode ની અધિકૃત રચનાના દૃષ્ટિકોણથી, તે `Editor` નો સહાયક preview panel (`Preview Pane`) છે. તે ફક્ત SwiftUI view files ખોલેલી હોય ત્યારે જ ઉપલબ્ધ થાય છે; બીજા પ્રકારની ફાઇલોમાં preview સુવિધા દેખાતી નથી.

આગામી પાઠોમાં આપણે મુખ્યત્વે `Editor` વિસ્તારમાં Swift અને SwiftUI કોડ લખીશું.

પ્રથમ વખત Xcode પ્રોજેક્ટ ખોલતા `Inspector` અને `Debug Area` મૂળભૂત રીતે છુપાયેલા હોઈ શકે છે. તમે તેને ઉપર જમણે અને નીચે જમણે આવેલા બટનથી બતાવી શકો છો.

![Swift](../../RESOURCE/001_xcode7.png)

સૂચન: `Inspector` વિસ્તાર મુખ્યત્વે ફાઇલ પ્રોપર્ટીઝ જોવા અને બદલવા માટે છે. વાસ્તવિક વિકાસમાં, વધુ સંપાદન જગ્યા મેળવવા માટે તેને ઘણીવાર છુપાવવામાં આવે છે.

## સારાંશ

આ પાઠમાં આપણે Xcode કેવી રીતે ડાઉનલોડ કરવું, પ્રથમ Xcode પ્રોજેક્ટ કેવી રીતે બનાવવો, અને Xcode ના ઈન્ટરફેસનું મૂળભૂત ગોઠવણ કેવી રીતે ઓળખવું તે શીખ્યા.

હવે થોડો મુક્ત અભ્યાસ સમય:
- તમે `Navigator` વિસ્તારમાં ફાઇલ બનાવવાનો કે કાઢવાનો પ્રયાસ કરી શકો છો  
- `Editor` વિસ્તારમાં કોડ જુઓ અને તેની રચનાને ધ્યાનથી નિહાળો
- `Canvas` વિસ્તારમાં બટનો અજમાવો, preview ને zoom in / zoom out કરો

આગામી પાઠમાં આપણે થોડું સરળ કોડ લખવાનું શરૂ કરીશું, અને તે આપણને Xcode ને વધુ સારી રીતે સમજવામાં મદદ કરશે.
