# ફિલ લાઇટ એપ

આ પાઠમાં, આપણે એક ખૂબ જ રસપ્રદ ફિલ લાઇટ એપ બનાવશું. જ્યારે રાત પડે છે, ત્યારે આપણે ફોનની સ્ક્રીન પર અલગ અલગ રંગો દેખાડી શકીએ અને તેને એક સરળ ફિલ લાઇટ તરીકે વાપરી શકીએ.

આ ફિલ લાઇટ એપ સ્ક્રીન પર ટૅપ કરીને રંગો બદલી શકે છે, અને સ્લાઇડર વડે બ્રાઇટનેસ પણ સમાયોજિત કરી શકે છે.

આ ઉદાહરણમાં, આપણે `brightness` વડે વ્યૂની બ્રાઇટનેસ કેવી રીતે બદલવી, `onTapGesture` વડે વ્યૂમાં ટૅપ જેસ્ચર કેવી રીતે ઉમેરવું, અને `Slider` કંટ્રોલ વિશે શીખીશું.

## રંગ દર્શાવવો

સૌપ્રથમ, ચાલો વ્યૂમાં એક રંગ બતાવીએ.

SwiftUI માં, `Color` માત્ર રંગ દર્શાવતું નથી, તે એક વ્યૂ તરીકે પણ દેખાડી શકાય છે:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

અહીં `Color.red` લાલ રંગનું એક વ્યૂ દર્શાવે છે. `.ignoresSafeArea()` રંગના વ્યૂને આખી સ્ક્રીન પર ફેલાવી દે છે, જેથી તે એક સાચી ફિલ લાઇટ જેવી અસર આપે છે.

### રંગોની એરે અને ઇન્ડેક્સ

હાલમાં ફક્ત એક જ રંગ દેખાય છે. પરંતુ ફિલ લાઇટ સામાન્ય રીતે ફક્ત એક જ રંગની નથી હોતી. તે વાદળી, પીળી, જાંબલી, સફેદ અને બીજા રંગો પણ દેખાડી શકે છે.

અમે ઇચ્છીએ છીએ કે સ્ક્રીન પર ટૅપ કરતાં અલગ અલગ રંગોમાં ફેરફાર થાય. આ રંગોને આપણે એક એરેમાં મૂકી એકસરખી રીતે મેનેજ કરી શકીએ:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

એક એરે "સમાન પ્રકારના ડેટાના જૂથ"ને સંગ્રહ કરવા માટે યોગ્ય છે. અહીં એરેમાં રહેલો દરેક એલિમેન્ટ `Color` છે.

જો આપણે કોઈ ચોક્કસ રંગ બતાવવો હોય, તો આપણે ઇન્ડેક્સનો ઉપયોગ કરી શકીએ:

```swift
colors[0]
```

આનો અર્થ થાય છે એરેના `0` ઇન્ડેક્સ પર રહેલો રંગ વાંચવો, એટલે કે પહેલો રંગ.

હવે કોડ આ રીતે લખી શકાય:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

આ રીતે, સ્ક્રીન એરેમાં રહેલો પહેલો રંગ, એટલે કે લાલ, બતાવશે.

### ઇન્ડેક્સ વડે રંગ નિયંત્રિત કરવો

જો આપણે અલગ અલગ રંગોમાં ફેરફાર કરવો હોય, તો ઇન્ડેક્સને કોડમાં સ્થિર રાખવાને બદલે તેને મેનેજ કરવા માટે એક વેરિએબલ જોઈએ.

અમે ઇન્ડેક્સ સાચવવા માટે `@State` વડે એક વેરિએબલ જાહેર કરી શકીએ:

```swift
@State private var index = 0
```

અહીં `index` હાલના રંગનો ઇન્ડેક્સ દર્શાવે છે.

જ્યારે `index` બદલાય છે, ત્યારે SwiftUI ઇન્ટરફેસને ફરી ગણતરી કરે છે અને બતાવવામાં આવતું કન્ટેન્ટ અપડેટ કરે છે.

પછી આપણે મૂળ `colors[0]` ને બદલીને આ લખીએ:

```swift
colors[index]
```

આ રીતે, વ્યૂમાં દેખાતો રંગ `index` દ્વારા નક્કી થશે.

હવે કોડ આ રીતે બને છે:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

જ્યારે `index` બદલાશે, ત્યારે `colors[index]` પણ અલગ રંગ બતાવશે.

ઉદાહરણ તરીકે:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

એક બાબત ધ્યાનમાં રાખવાની છે કે `index` એરેના સૌથી મોટા ઇન્ડેક્સથી વધારે ન થવો જોઈએ, નહીં તો out-of-range error આવશે.

## ટૅપ જેસ્ચર

હવે આપણે `index` મુજબ અલગ અલગ રંગો બતાવી શકીએ છીએ, પરંતુ હજુ સુધી ટૅપ કરીને રંગ બદલી શકતા નથી.

પહેલા "Quote Carousel" પાઠમાં, અમે quote બદલીવા માટે `Button` નો ઉપયોગ કર્યો હતો.

પરંતુ આ વખતે, આપણે "આખા રંગના વિસ્તારમાં ટૅપ" કરીને રંગ બદલી શકીએ એવી ઇચ્છા છે, તેથી `onTapGesture` વધુ યોગ્ય છે.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

જ્યારે આપણે રંગના વ્યૂ પર ટૅપ કરીએ છીએ, ત્યારે આ કોડ ચલાવવામાં આવે છે:

```swift
index += 1
```

આનો અર્થ થાય છે `index` ને `1` થી વધારવો. ઇન્ડેક્સ વધ્યા પછી, `colors[index]` એરેમાં રહેલો આગળનો રંગ બતાવશે.

### `onTapGesture`

`onTapGesture` એક gesture modifier છે, જે વ્યૂમાં ટૅપ એક્શન ઉમેરે છે.

મૂળભૂત ઉપયોગ:

```swift
.onTapGesture {
    // code
}
```

ઉદાહરણ:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

જ્યારે આ લાલ વ્યૂ પર ટૅપ કરવામાં આવે છે, ત્યારે braces ની અંદરનો કોડ ચાલે છે અને console માં આ આઉટપુટ થાય છે:

```swift
Click color
```

`onTapGesture` દ્વારા આપણે નક્કી કરી શકીએ છીએ કે કોઈ વ્યૂ પર ટૅપ થયા પછી શું થવું જોઈએ.

### `Button` થી ફરક

અગાઉ આપણે `Button` વ્યૂ શીખ્યા હતા. `Button` અને `onTapGesture` બંને ટૅપ એક્શન હેન્ડલ કરી શકે છે, પરંતુ તેમની વાપરવાની પરિસ્થિતિઓ સંપૂર્ણપણે સમાન નથી.

`onTapGesture` "મોજૂદા વ્યૂમાં ટૅપ કરવાની કાર્યક્ષમતા ઉમેરવા" માટે વધુ યોગ્ય છે, જેમ કે `Color`, `Image`, `Text`, અથવા કોઈ અન્ય સામાન્ય વ્યૂ.

જ્યારે `Button` વધુ યોગ્ય છે જ્યારે આપણે "Confirm", "Submit", અથવા "Delete" જેવા સ્પષ્ટ બટનનું પ્રતિનિધિત્વ કરવું હોય.

આ ફિલ લાઇટ એપમાં, આપણે રંગ બદલવાની પ્રક્રિયા વધુ સરળ રાખવી છે. આખા રંગના વિસ્તારમાં ટૅપ કરીને રંગ બદલાય, તેથી અહીં `onTapGesture` સારો વિકલ્પ છે.

## ઇન્ડેક્સની સમસ્યા

હવે આપણે સ્ક્રીન પર ટૅપ કરીને અલગ અલગ રંગોમાં ફેરફાર કરી શકીએ છીએ.

પરંતુ અહીં એક મહત્વપૂર્ણ સમસ્યા છે: **ઇન્ડેક્સ એરેની મર્યાદા બહાર જઈ શકે છે**.

ઉદાહરણ:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

જો આપણે સ્ક્રીન પર વારંવાર ટૅપ કરતા રહીએ, તો `index` અંતે `4` બની જશે, અને ત્યારે "index out of range" error આવશે.

એનું કારણ એ છે કે `colors` એરેમાં `4` એલિમેન્ટ્સ છે, પરંતુ ઇન્ડેક્સ `0` થી શરૂ થાય છે, તેથી માન્ય ઇન્ડેક્સ રેન્જ `0 - 3` છે, `4` નહીં.

જો આપણે `colors[4]` ઍક્સેસ કરીએ, તો "index out of range" error આવશે.

હાલના કોડમાં, દરેક ટૅપ `index` ને `1` થી વધારશે. જો આપણે આ બાબત હેન્ડલ નહીં કરીએ, તો તે અંતે માન્ય મર્યાદા બહાર જશે.

તેથી સ્ક્રીન પર ટૅપ વખતે આપણને ઇન્ડેક્સ તપાસવો પડશે: જો તે છેલ્લો રંગ હોય, તો પહેલા રંગ પર પાછા જવું, નહીં તો `1` ઉમેરવાનું ચાલુ રાખવું.

આ આપણે `if` સ્ટેટમેન્ટ વડે કરી શકીએ:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

આ કોડમાં `colors.count` નો અર્થ એરેમાં રહેલા એલિમેન્ટ્સની સંખ્યા થાય છે.

હાલની એરેમાં 4 રંગો છે, એટલે:

```swift
colors.count // 4
```

પરંતુ સૌથી મોટો ઇન્ડેક્સ `4` નથી, તે `3` છે, કારણ કે ઇન્ડેક્સિંગ `0` થી શરૂ થાય છે.

તેથી છેલ્લો ઇન્ડેક્સ આ રીતે લખવો પડે:

```swift
colors.count - 1
```

અર્થાત્:

```swift
4 - 1 = 3
```

આ લોજિકનો અર્થ છે: જો હાલનો ઇન્ડેક્સ પહેલેથી જ છેલ્લો રંગ દર્શાવે છે, તો ઇન્ડેક્સને `0` પર રીસેટ કરો; નહીં તો તેને `1` થી વધારો.

આ રીતે, રંગો ચક્રાકારે બદલાઈ શકે છે.

### ઇન્ડેક્સ લોજિકને વધુ સંક્ષિપ્ત બનાવવી

જો આપણે કોડ વધુ ટૂંકો બનાવવો હોય, તો ternary operator પણ વાપરી શકીએ:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

આ કોડનો અર્થ છે: જો `index == colors.count - 1` સત્ય હોય, તો `0` રિટર્ન કરો. નહીં તો `index + 1` રિટર્ન કરો.

અંતે, આ પરિણામ પાછું `index` ને અસાઇન કરવામાં આવે છે.

હવે આપણે રંગ બદલવાનો ઇફેક્ટ અમલમાં મૂકી શકીએ છીએ.

સંપૂર્ણ કોડ:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

હવે આપણે સ્ક્રીન પર ટૅપ કરીને અલગ અલગ રંગોમાં ફેરફાર કરી શકીએ છીએ. એક મૂળભૂત ફિલ લાઇટ એપ તૈયાર થઈ ગઈ છે.

## રંગનું નામ બતાવવું

અમે રંગોને અનુરૂપ લખાણનો એક બીજો જૂથ ઉમેરી શકીએ, જેથી રંગ બદલાય ત્યારે હાલના રંગનું નામ પણ સ્ક્રીન પર દેખાય.

ઉદાહરણ તરીકે:

- લાલ માટે `Red`
- વાદળી માટે `Blue`
- પીળી માટે `Yellow`
- જાંબલી માટે `Purple`

અહીં પણ આપણે રંગોના નામ સાચવવા માટે એરેનો ઉપયોગ કરી શકીએ:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

આ એરેમાં રહેલા લખાણોનો ક્રમ રંગોની એરેમાં રહેલા રંગોના ક્રમ સાથે બિલકુલ મેળ ખાતો હોવો જોઈએ.

પછી આપણે હાલના ઇન્ડેક્સના રંગનું નામ બતાવવા `Text` નો ઉપયોગ કરી શકીએ:

```swift
Text(colorsName[index])
```

`Text` `index` મુજબ હાલના રંગનું નામ બતાવે છે.

`Text` નું દેખાવ સુધારવા modifiers વાપરો:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

આ `Text` ને સફેદ રંગમાં, title size સાથે અને bold font weight માં બતાવશે.

હવે અમારી પાસે full-screen `Color` વ્યૂ છે. જો આપણે `Text` ને `Color` વ્યૂની ઉપર બતાવવું હોય, તો બંનેને એક પર એક મૂકવા `ZStack` layout container વાપરવો પડશે.

```swift
ZStack {
    Color
    Text
}
```

તેથી કોડ આ રીતે બને છે:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

આ રીતે, પૃષ્ઠભૂમિમાં રંગનું વ્યૂ રહેશે, અને તેના ઉપર ટેક્સ્ટનું વ્યૂ મૂકવામાં આવશે.

પરિણામ:

![Color](../../Resource/018_color2.png)

એક બાબત ધ્યાનમાં રાખવાની છે કે `ZStack` માં પાછળ લખાયેલું વ્યૂ સામાન્ય રીતે આગળ દેખાય છે. જો `Text` ને `Color` પહેલાં લખવામાં આવે, તો પછીનું `Color` વ્યૂ `Text` ને ઢાંકી શકે છે.

## બ્રાઇટનેસ નિયંત્રિત કરવી

હવે આપણે અલગ અલગ રંગોમાં ફેરફાર કરી શકીએ છીએ, પરંતુ ફિલ લાઇટની બીજી એક મહત્વપૂર્ણ સુવિધા છે: **બ્રાઇટનેસ સમાયોજિત કરવી**.

SwiftUI માં, આપણે `brightness` modifier નો ઉપયોગ કરીને વ્યૂની બ્રાઇટનેસ સમાયોજિત કરી શકીએ છીએ.

ઉદાહરણ:

```swift
.brightness(1)
```

અમે આ રીતે લખી શકીએ:

```swift
colors[index]
    .brightness(0.5)
```

આથી હાલનો રંગ વધુ તેજસ્વી બને છે અને ફિલ લાઇટની અસર નજીક આવે છે.

બ્રાઇટનેસની રેન્જ `0 - 1` છે. `0` નો અર્થ મૂળ રંગ જાળવી રાખવો, અને કિંમત `1` ની જેટલી નજીક જશે, રંગ એટલો વધુ તેજસ્વી બનશે. `1` નો અર્થ સૌથી તેજસ્વી સફેદ પરિણામ.

જોકે આપણે કોડમાં `brightness` નિયંત્રિત કરી શકીએ છીએ, વપરાશકર્તા હજુ પણ તેને સીધો નિયંત્રિત કરી શકતો નથી.

તેથી આપણને ખેંચી શકાય એવો કંટ્રોલ ઉમેરવો પડશે: `Slider`.

## `Slider` વ્યૂ

SwiftUI માં `Slider` એક કંટ્રોલ છે જેનો ઉપયોગ નક્કી કરેલી રેન્જમાં કોઈ કિંમત પસંદ કરવા માટે થાય છે. Apple તેને "bounded linear range માંથી value પસંદ કરવા માટેનો control" તરીકે વર્ણવે છે.

મૂળભૂત ઉપયોગ:

```swift
Slider(value: $value, in: 0...1)
```

પેરામિટર્સની સમજણ:

1. `value: $value`: `Slider` ને એક વેરિએબલ સાથે bind કરવું પડે છે.

    જ્યારે સ્લાઇડર ખેંચાય છે, ત્યારે વેરિએબલની કિંમત પણ એ જ સમયે બદલાય છે. અને જો વેરિએબલ બદલાય, તો સ્લાઇડર પણ અપડેટ થાય છે.

    આ અગાઉ શીખેલા `TextField` જેવું જ છે. બંને "કંટ્રોલને વેરિએબલ સાથે bind કરે છે".

    binding દર્શાવવા માટે bind કરેલા વેરિએબલની આગળ `$` ચિહ્ન લખવું પડે છે.

2. `in: 0...1`: આ પેરામિટર સ્લાઇડરની value range બતાવે છે.

    અહીં `0...1` નો અર્થ છે કે minimum value `0` છે અને maximum value `1` છે.

    જ્યારે સ્લાઇડર એકદમ ડાબી બાજુ ખેંચાય, ત્યારે bound વેરિએબલ `0` ની નજીક હોય છે; જ્યારે એકદમ જમણી બાજુ જાય, ત્યારે તે `1` ની નજીક હોય છે.

ઉદાહરણ:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` `value` વેરિએબલ સાથે bind થયેલું છે. જ્યારે સ્લાઇડર ખેંચાય છે, ત્યારે `value` ની કિંમત પણ એ જ સમયે બદલાય છે.

દેખાવ:

![Slider](../../Resource/018_slider.png)

જ્યારે `Slider` ડાબી તરફ ખસેડાય છે, ત્યારે bound `value` `0` બને છે. જ્યારે જમણી તરફ ખસેડાય છે, ત્યારે તે `1` બને છે.

### value range

`Slider` ની value range સ્થિર નથી. તેને આ રીતે પણ લખી શકાય:

```swift
0...100
```

અથવા કોઈ બીજી range.

પરંતુ આ ફિલ લાઇટ એપમાં, આપણે બ્રાઇટનેસ નિયંત્રિત કરવી છે, તેથી `0...1` સૌથી યોગ્ય છે.

## `Slider` વડે બ્રાઇટનેસ નિયંત્રિત કરવી

હવે આપણે `Slider` ને `brightness` સાથે જોડવું છે.

સૌપ્રથમ, બ્રાઇટનેસની કિંમત સાચવવા માટે એક વેરિએબલ બનાવો:

```swift
@State private var slider = 0.0
```

અહીં `0.0` એ `Double` પ્રકારની value છે.

કારણ કે `Slider` સામાન્ય રીતે numeric type સાથે bind થાય છે, અને અહીં આપણે ઇચ્છીએ છીએ કે તેમાં સતત ફેરફાર થાય, તેથી `Double` વધુ યોગ્ય છે. ઉપરાંત `brightness` ફક્ત `Double` value જ સ્વીકારી શકે છે.

પછી આ value ને `brightness` માં આપો:

```swift
colors[index]
    .brightness(slider)
```

જ્યારે `slider == 0` હોય, ત્યારે રંગ default સ્થિતિમાં રહે છે. `slider` જેટલું `1` ની નજીક જશે, રંગ એટલો વધુ તેજસ્વી લાગશે.

### `Slider` કંટ્રોલ ઉમેરવો

પછી, આ વેરિએબલમાં ફેરફાર કરવા માટે `Slider` કંટ્રોલ ઉમેરો:

```swift
Slider(value: $slider, in: 0...1)
```

જ્યારે સ્લાઇડર બદલાય છે, ત્યારે `slider` ની value પણ બદલાય છે, અને `brightness(slider)` એ જ સમયે બ્રાઇટનેસ અપડેટ કરે છે.

આ SwiftUI માં "વેરિએબલ વ્યૂને ચલાવે છે" તેનો ખૂબ જ સામાન્ય ઉદાહરણ છે.

### `Slider` ના દેખાવને સમાયોજિત કરવો

ડિફોલ્ટ પ્રમાણે, `Slider` ઉપલબ્ધ પહોળાઈનો ઉપયોગ કરે છે.

અમે તેને એક નક્કી પહોળાઈ આપી શકીએ:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

પછી તેને વધુ સ્પષ્ટ બનાવવા માટે થોડા modifiers ઉમેરીએ:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

આ સ્લાઇડરને સફેદ પૃષ્ઠભૂમિ અને ગોળ ખૂણાં આપે છે, જેથી તે `Color` વ્યૂ પર વધુ સ્પષ્ટ દેખાય.

અંતમાં, તેને સ્ક્રીનના તળિયે મૂકો.

કારણ કે આપણે પહેલાથી જ `ZStack` વાપરી રહ્યા છીએ, તેથી તેમાં `VStack` મૂકી `Spacer()` વડે `Slider` ને નીચે ધકેલી શકીએ.

## સંપૂર્ણ કોડ

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## સારાંશ

અગાઉ શીખેલા જ્ઞાન સાથે અને રંગો અને એરે જેવા મૂળભૂત ખ્યાલોનો ઉપયોગ કરીને આપણે એક ખૂબ જ રસપ્રદ ફિલ લાઇટ એપ બનાવી.

આ ફિલ લાઇટ એપ દ્વારા, આપણે `brightness` વડે બ્રાઇટનેસ સમાયોજિત કરવી, `onTapGesture` વડે વ્યૂમાં ટૅપ એક્શન ઉમેરવી, અને `Slider` કંટ્રોલનો ઉપયોગ કરવો શીખ્યા.

અમે `Color` વ્યૂમાં `onTapGesture` ઉમેર્યું જેથી રંગો બદલી શકાય. અમે `Slider` કંટ્રોલ વડે `brightness` સંભાળતા વેરિએબલમાં ફેરફાર પણ કર્યો, જે "વેરિએબલ વ્યૂને ચલાવે છે" તેનો બીજો ઉદાહરણ છે.

અમે ternary operator પણ ફરી જોયો, વ્યૂઝને એક પર એક રાખવા `ZStack` નો ઉપયોગ કર્યો, અને સમાન પ્રકારના ડેટાના જૂથને મેનેજ કરવા એરેનો ઉપયોગ કર્યો. આથી એરે અને ઇન્ડેક્સ વિશેની સમજ વધુ ઊંડી બને છે, ખાસ કરીને out-of-range errors ટાળવાની વ્યવહારુ સમસ્યા અંગે.

જોકે આ ઉદાહરણ ખૂબ જટિલ નથી, તે અગાઉ શીખેલા ઘણા મૂળભૂત વિચારોને એકસાથે જોડે છે. જ્યારે આ બધું મળીને એક નાના વાસ્તવિક પ્રોજેક્ટમાં વપરાય છે, ત્યારે દરેક ખ્યાલ ક્યાં ઉપયોગી છે તે સમજવું વધુ સરળ બને છે.

### વાસ્તવિક ઉપયોગનો દૃશ્ય

કલ્પના કરો કે તમે એક જૂનો iPhone ટેબલ પર મૂકો અને પોતે બનાવેલી ફિલ લાઇટ એપ વડે પ્રકાશનો રંગ નિયંત્રિત કરો. તે ખૂબ સરસ અનુભવ હશે.

App Store માં ઘણી "ફિલ લાઇટ" એપ્સ છે, અને તે પણ ખાસ જટિલ લાગતી નથી.

![AppStore](../../Resource/018_appStore.PNG)

અમે સરળ એપ્સથી શરૂઆત કરી શકીએ અને તેને App Store પર પ્રકાશિત કરવાનો પ્રયાસ કરી શકીએ. આ ફક્ત ડેવલપમેન્ટ પ્રત્યેનો રસ જ નથી વધારતું, પણ આપણી વૃદ્ધિનો પણ રેકોર્ડ રાખે છે.

### પાઠ પછીનો અભ્યાસ

તમે આ ફિલ લાઇટ એપને વધુ કેવી રીતે વિસ્તારી શકાય તે અંગે વિચારવાનું ચાલુ રાખી શકો છો, જેમ કે:

- વધુ રંગો ઉમેરો
- હાલની બ્રાઇટનેસની value બતાવો
- નીચેના સ્લાઇડર વિસ્તારની ડિઝાઇન વધુ સારી બનાવો

જ્યારે તમે આ જ્ઞાનનો ખરેખર ઉપયોગ કરવાનું શરૂ કરશો, ત્યારે તમને સમજાશે કે આપણે શીખતા દરેક ખ્યાલ હકીકતમાં એપ્સ બનાવવાનું એક સાધન છે.

અમે જેટલા વધુ સાધનોમાં પારંગત થઈએ, એટલી વધુ સુવિધાઓ બનાવી શકીએ.
