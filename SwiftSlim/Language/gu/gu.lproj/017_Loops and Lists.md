# લૂપ અને યાદીઓ

આ પાઠમાં, આપણે Swift માં `for` લૂપ અને SwiftUI માં `ForEach` શીખીશું.

બંને અમને એરેમાં આવેલા તત્વોને ક્રમ પ્રમાણે એક પછી એક વાંચવામાં મદદ કરે છે. વાસ્તવિક વિકાસમાં, અમને ઘણી વાર ડેટાના સમૂહમાંથી દરેક આઇટમ એક પછી એક કાઢીને તેને પ્રોસેસ અથવા દર્શાવવાની જરૂર પડે છે, અને આ કામ માટે લૂપ ખૂબ જ મહત્વપૂર્ણ સાધન છે.

જ્યારે આપણે વેબસાઇટ બ્રાઉઝ કરીએ છીએ અથવા કોઈ એપનો ઉપયોગ કરીએ છીએ, ત્યારે આપણે ઘણી વાર ઘણી યાદીઓ અથવા આઇટમ્સ જોતા હોઈએ છીએ. આવી સામગ્રી સામાન્ય રીતે પહેલા `એરે` માં સંગ્રહિત થાય છે, પછી `લૂપ` દ્વારા એક પછી એક વાંચવામાં આવે છે, અને અંતે ઇન્ટરફેસ પર દર્શાવવામાં આવે છે.

ઉદાહરણ તરીકે, Google ની શોધ પરિણામોની યાદી:

![Google](../../Resource/017_google.png)

શોધ પરિણામોની યાદીમાંનો દરેક આઇટમ સામાન્ય રીતે એક ડેટાને અનુરૂપ હોય છે, અને તેને લૂપ દ્વારા એક પછી એક દર્શાવવામાં આવે છે.

તેથી, `એરે` અને `લૂપ` ઘણી વાર સાથે જોવા મળે છે. `એરે` ડેટાના સમૂહને સંગ્રહિત કરે છે, અને `લૂપ` તે ડેટાને ક્રમ પ્રમાણે બહાર કાઢીને તેના પર એકસરખું કાર્ય કરે છે.

હવે, ચાલો સૌથી મૂળભૂત `for` લૂપથી શરૂઆત કરીએ.

## કાઉન્ટડાઉન

ધારો કે આપણે એક કાઉન્ટડાઉન ફંક્શનનું અનુકરણ કરવું છે અને ક્રમ પ્રમાણે `“5, 4, 3, 2, 1”` આઉટપુટ કરવું છે.

તે લખવાની સૌથી સીધી રીત આ છે:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

આ રીતે કાઉન્ટડાઉન બતાવી શકાય છે, પરંતુ જો સામગ્રી વધુ હોય, તો આ રીત ખૂબ જ પુનરાવર્તિત લાગે છે અને પછી તેમાં ફેરફાર કરવો પણ સરળ નથી.

આ સમયે, આપણે પહેલા ડેટાને એરેમાં મૂકી શકીએ, અને પછી `for` લૂપ દ્વારા તેને એક પછી એક આઉટપુટ કરી શકીએ:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

આ રીતે લખવું વધુ સંક્ષિપ્ત છે અને તેને વિસ્તૃત કરવું પણ વધુ સરળ છે. ભલે કાઉન્ટડાઉનમાં ઘણા નંબર હોય, તો પણ અમને એક પછી એક `print` લખવાની જરૂર નથી, ફક્ત લૂપને એરેના દરેક તત્વને આપમેળે પ્રોસેસ કરવા દેવું પડે છે.

આ કોડને આપણે ટેસ્ટ કરવા માટે બટનમાં મૂકી શકીએ:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

જ્યારે બટન પર ટૅપ કરવામાં આવે છે, ત્યારે `for` લૂપ એરેના ક્રમ પ્રમાણે `countDown` ના દરેક પૂર્ણાંકને બહાર કાઢે છે અને તેને કન્સોલમાં પ્રિન્ટ કરે છે.

હવે, ચાલો જોઈએ કે `for` લૂપ કેવી રીતે કામ કરે છે.

## for લૂપ

`for` લૂપનો ઉપયોગ ડેટાના સમૂહને ક્રમ પ્રમાણે પ્રોસેસ કરવા માટે થાય છે. તેમાં સૌથી સામાન્ય વસ્તુઓ એરે અને રેન્જ છે.

મૂળભૂત લખવાની રીત:

```swift
for તત્વ in એરેઅથવારાેંજ {
    ચલાવવાનોકોડ
}
```

ઉદાહરણ તરીકે, હમણાં જ જોયેલો કાઉન્ટડાઉન કોડ:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

અહીં `i` વર્તમાનમાં બહાર કાઢવામાં આવેલા તત્વને દર્શાવે છે, અને `in` પછી લૂપ કરવાનું ડેટા આવે છે, એટલે કે `countDown` એરે.

આ કોડનો અર્થ એવો છે: પ્રોગ્રામ `countDown` એરેમાંથી દરેક તત્વને ક્રમ પ્રમાણે બહાર કાઢશે. જ્યારે પણ તે કોઈ તત્વ બહાર કાઢશે, તે તેને થોડા સમય માટે `i` વડે દર્શાવશે, અને પછી કર્લી બ્રેસિસની અંદરનો કોડ ચલાવશે.

તેની પ્રક્રિયા આ રીતે સમજી શકાય છે:

* પ્રથમ બહાર કાઢવામાં આવતું તત્વ `5` છે, એટલે આ સમયે `i` ની કિંમત `5` છે
* `print(i)` ચલાવવામાં આવે છે અને `5` આઉટપુટ થાય છે
* પછી આગળનું તત્વ `4` બહાર કાઢવામાં આવે છે
* ફરી `print(i)` ચલાવવામાં આવે છે અને `4` આઉટપુટ થાય છે

પછીના `3`, `2`, અને `1` પણ આ જ રીતે એક પછી એક ચલાવવામાં આવશે.

![for](../../Resource/017_for.png)

જ્યારે એરેના બધા તત્વો પ્રોસેસ થઈ જાય છે, ત્યારે `for` લૂપ આપમેળે સમાપ્ત થાય છે.

અમે `for` લૂપની મદદથી એરેના દરેક તત્વને મેળવી શકીએ છીએ અને આવા કાઉન્ટડાઉન જેવી સુવિધાઓ બનાવી શકીએ છીએ.

**સૂચન**: `i` એ for લૂપમાં ઘણું સામાન્ય નામ છે, જે વર્તમાનમાં બહાર કાઢાતા તત્વને દર્શાવે છે, પરંતુ તે નિશ્ચિત લખાણ નથી. તેના બદલે અન્ય અંગ્રેજી નામો પણ વાપરી શકાય છે.

ઉદાહરણ તરીકે, દરેક બહાર કાઢાતા તત્વને દર્શાવવા માટે `num` નો ઉપયોગ કરો:

```swift
for num in countDown {
    print(num)
}
```

કર્લી બ્રેસિસની અંદર પણ વર્તમાનમાં બહાર કાઢાતા તત્વને દર્શાવવા માટે `num` નો જ ઉપયોગ કરવો પડશે.

### રેન્જ પર લૂપ

`for` લૂપ ફક્ત એરે પર જ નહીં, પરંતુ રેન્જ પર પણ લૂપ કરી શકે છે.

ઉદાહરણ તરીકે, જો આપણે `50` સુધીના હકારાત્મક પૂર્ણાંકોનો સરવાળો કરવો હોય, તો `1` થી `50` વચ્ચેના બધા પૂર્ણાંકને એક પછી એક ઉમેરવા પડશે.

તે લખવાની સૌથી સીધી રીત કદાચ આ હોઈ શકે:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

આ રીતે લખવાની સમસ્યા એ છે કે દરેક નંબર હાથથી લખવો પડે. આ મુશ્કેલ છે અને વાસ્તવિક ડેવલપમેન્ટ માટે યોગ્ય નથી.

આ સમયે, આપણે રેન્જ ઓપરેટર અને `for` લૂપ સાથે ઉપયોગ કરીને આ કામ કરી શકીએ.

પાછલા “ઓપરેટર” પાઠમાં આપણે રેન્જ ઓપરેટર શીખ્યા હતા:

```swift
1...3   // 1, 2, 3
```

અહીં `...` નો અર્થ બંધ રેન્જ છે, એટલે કે તેમાં શરૂઆત અને અંત બન્ને સામેલ છે.

તેથી અહીં આપણે `1` થી `50` સુધીની પૂર્ણાંક રેન્જ પર `for` લૂપનો ઉપયોગ કરી શકીએ:

```swift
for i in 1...50 {
    
}
```

અપણે આ પૂર્ણાંકનો સરવાળો કરવો છે, એટલે કુલને સંગ્રહિત કરવા માટે એક વેરિયેબલની પણ જરૂર છે:

```swift
var count = 0
```

પછી દરેક લૂપ વખતે, હાલમાં બહાર કાઢેલા નંબરને `count` માં ઉમેરીએ:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

આ કોડનો અર્થ છે:

* `count` વર્તમાન કુલને સંગ્રહિત કરે છે
* `for` લૂપ `1` થી `50` સુધીના પૂર્ણાંકને એક પછી એક બહાર કાઢશે
* જ્યારે પણ કોઈ પૂર્ણાંક બહાર કાઢવામાં આવશે, ત્યારે `count += i` એક વાર ચાલશે
* લૂપ પૂર્ણ થયા પછી, `count` માં અંતિમ કુલ સંગ્રહિત રહેશે

આ રીતે, આપણે `50` સુધીના બધા હકારાત્મક પૂર્ણાંકોનો સરવાળો મેળવી શકીએ.

આ જ રીતે, જો આપણે `1` થી `100` સુધીના પૂર્ણાંકનો સરવાળો ગણવો હોય, તો ફક્ત રેન્જ બદલી દેવી પડશે:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` લૂપ ફક્ત એરે પર જ નહીં, પરંતુ રેન્જ પર પણ લૂપ કરી શકે છે. ડેટાને ક્રમ પ્રમાણે વારંવાર પ્રોસેસ કરવાની આ ક્ષમતા ડેવલપમેન્ટમાં ખૂબ સામાન્ય છે.

તે અમને ઘણા પુનરાવર્તિત કામોને આપમેળે સંભાળવામાં મદદ કરે છે, જેથી દરેક નંબરનો સરવાળો અથવા અન્ય સમાન પુનરાવર્તિત કાર્ય હાથથી લખવું ન પડે.

## SwiftUI માં લૂપ

ઉપરનો `for` લૂપ Swift ભાષાનો એક ભાગ છે.

પરંતુ SwiftUI માં, આપણે view structure ની અંદર સીધો `for` નો ઉપયોગ કરીને view બનાવી શકતા નથી:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

કારણ કે `VStack` ની અંદર SwiftUI views જોઈએ છે, જ્યારે સામાન્ય `for` લૂપ પોતે એક view નથી.

જો આપણે SwiftUI માં એરેની સામગ્રી અનુસાર પુનરાવર્તિત રીતે UI બનાવવી હોય, તો `ForEach` નો ઉપયોગ કરવો પડે.

ઉદાહરણ તરીકે, જો આપણે કોઈ જૂથના બધા સભ્યોના નામ દર્શાવવા હોય, તો આ રીતે લખી શકીએ:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

દર્શાવવાનો પરિણામ:

![for1](../../Resource/017_for1.png)

પછી, દરેક `Text` પર એકસરખા modifiers ઉમેરીએ:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

હવે દરેક `Text` નાની અને વધુ ગાઢ ફોન્ટમાં દેખાશે.

દર્શાવવાનો પરિણામ:

![](../../Resource/017_for2.png)

જ્યારે modifiers ઇન્ટરફેસને વધુ સુંદર બનાવે છે, ત્યારે તે કોડને વધુ લાંબો અને વધુ પુનરાવર્તિત પણ બનાવે છે.

જો નામો વધતા જાય, અથવા પછી આપણે ફોન્ટ, રંગ અને અન્ય સ્ટાઇલને એકસરખી રીતે બદલવી હોય, તો અમારે ઘણો સમાન કોડ વારંવાર લખવો અને બદલવો પડશે. આ કામ મુશ્કેલ પણ છે અને મેન્ટેન કરવા માટે પણ અનુકૂળ નથી.

આ સમયે, આપણે પહેલા આ નામોને એરેમાં મૂકી શકીએ, અને પછી SwiftUI માં `ForEach` નો ઉપયોગ કરીને એરેના ડેટા પરથી પુનરાવર્તિત રીતે views બનાવી શકીએ:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

આ કોડ વધુ સંક્ષિપ્ત છે. `ForEach` એરેના દરેક તત્વને એક પછી એક વાંચશે અને અનુરૂપ સામગ્રીને ઇન્ટરફેસ પર દર્શાવશે.

પછી જો આપણે નવા નામ ઉમેરીએ, તો ફક્ત `names` એરેમાં ફેરફાર કરવાની જરૂર પડશે. દરેક વખતે ફરીથી `Text` અને જટિલ modifiers ઉમેરવાની જરૂર નહીં પડે.

## ForEach

`ForEach` SwiftUI માં views ને પુનરાવર્તિત રીતે બનાવવા માટે ઉપયોગમાં લેવાતી રચના છે. તેને SwiftUI માં `for` લૂપ જેવી રચના તરીકે સમજી શકાય.

મૂળભૂત ઉપયોગ:

```swift
ForEach(એરે, id: \.self) { તત્વ in
    SwiftUI કોડ
}
```

ઉદાહરણ તરીકે, હમણાંના જૂથના નામ:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

આ કોડનો અર્થ છે:

`names` એ લૂપ થતો એરે છે, એટલે કે દર્શાવવાનો ડેટા.

`id:\.self` નો અર્થ એ છે કે એરેના તત્વને જ દરેક આઇટમને અલગ પાડવા માટે વાપરવું.

હાલના ઉદાહરણમાં, એરેમાં strings સંગ્રહિત છે:

```swift
["Sam", "John", "Wathon", "Bob"]
```

તેથી SwiftUI સીધા `"Sam"`, `"John"`, `"Wathon"`, અને `"Bob"` જેવી strings ને અલગ સામગ્રી તરીકે ઓળખશે.

શરૂઆતી તબક્કામાં સામાન્ય `String` અને `Int` જેવી સરળ એરે માટે, આપણે સામાન્ય રીતે સીધું આ રીતે લખી શકીએ:

```swift
id: \.self
```

હાલ માટે ફક્ત આ એક લખવાની રીત યાદ રાખવી પૂરતી છે. `id` ના વધુ નિયમોમાં ઊંડાણથી જવાની જરૂર નથી.

`item in` માંનો `item` વર્તમાનમાં બહાર કાઢાયેલ તત્વને દર્શાવે છે.

ઉદાહરણ તરીકે, પ્રથમ લૂપ વખતે `item` `"Sam"` છે, અને બીજા લૂપ વખતે `item` `"John"` છે.

તેથી:

```swift
Text(item)
```

એક પછી એક આ રીતે બની જશે:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

અતેએ, આ કોડનું કામ એ છે કે `names` એરેમાંથી દરેક નામને એક પછી એક બહાર કાઢવું અને દરેક નામ માટે `Text` view બનાવવું.

### કામગીરીની પ્રક્રિયા

કાર્યપ્રણાલી આ રીતે સમજી શકાય:

* `ForEach` પહેલા `names` એરેનું પ્રથમ તત્વ `"Sam"` વાંચે છે
* `item` થોડા સમય માટે `"Sam"` ને દર્શાવે છે
* `Text(item)` `"Sam"` દર્શાવે છે
* પછી તે આગળનું તત્વ `"John"` વાંચે છે
* અને ફરી એ જ view code ચલાવે છે

અંતે, એરેના બધા નામો દર્શાવવામાં આવશે.

આ રીતે લખવાનો લાભ એ છે કે જો નામો વધારે થાય, તો અમને ઘણાં `Text` વારંવાર લખવાની જરૂર નથી, ફક્ત એરેની સામગ્રીમાં ફેરફાર કરવો પડે છે.

### ફોટા પર લૂપ

`ForEach` ફક્ત લખાણ જ નહીં, પણ છબીઓ, રંગો અથવા અન્ય SwiftUI views પણ દર્શાવી શકે છે.

ઉદાહરણ તરીકે, આપણે 4 છબીઓ તૈયાર કરીએ:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

આ ચાર છબીઓને Xcode ના Assets folder માં ખેંચીને મૂકો.

![image](../../Resource/017_for3.png)

જો `ForEach` નો ઉપયોગ ન કરીએ, તો દરેક છબી હાથથી લખવી પડશે:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

દર્શાવવાનો પરિણામ:

![image](../../Resource/017_for4.png)

આ રીતે કાર્ય થઈ શકે છે, પરંતુ કોડ ઘણો લાંબો થઈ જાય છે.

જો છબીઓ વધતી જાય, તો વધુ અને વધુ `Image` કોડ ફરી ફરી લખવો પડશે.

જો પછી આપણે બધી છબીઓનું કદ એકસરખું બદલવું હોય, તો પણ એક પછી એક બધું બદલવું પડશે.

આ સમયે, આપણે છબીઓના નામને એરેમાં મૂકી શકીએ, અને પછી `ForEach` નો ઉપયોગ કરીને પુનરાવર્તિત રીતે image views બનાવી શકીએ:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

અહીં `images` એક string એરે છે, અને એરેનું દરેક તત્વ એક image name છે.

`ForEach(images, id: \.self)` નો અર્થ એ છે કે `images` એરેના દરેક તત્વને એક પછી એક વાંચવું, તત્વને જ દરેક આઇટમને અલગ પાડવા માટે વાપરવું, અને પછી વર્તમાન તત્વ અનુસાર સંબંધિત view બનાવવું.

ઉદાહરણ તરીકે, જો પ્રથમ વાંચાયેલ તત્વ `"430F9BEF"` હોય, તો:

```swift
Image(item)
```

એનો અર્થ આ જ છે:

```swift
Image("430F9BEF")
```

પછીની છબીઓ પણ આ જ રીતે એક પછી એક દર્શાવવામાં આવશે.

![image](../../Resource/017_for4.png)

અહીં પણ આપણે હજી આ જ વાપરીએ છીએ:

```swift
id: \.self
```

કારણ પહેલાં જેવું જ છે: `images` એરેના તત્વો પણ સરળ string પ્રકારના છે, તેથી શરૂઆતના તબક્કામાં આપણે સીધા તત્વને જ ઓળખ માટે વાપરી શકીએ.

તેને એક સરળ નિશ્ચિત સૂત્ર તરીકે સમજી શકાય:

**જ્યારે `ForEach` `String` અથવા `Int` જેવી સરળ એરે પર લૂપ કરે છે, ત્યારે સામાન્ય રીતે `id: \.self` લખવામાં આવે છે.**

### નવી છબીઓ ઉમેરવી

જો પછી અમને વધુ છબીઓ ઉમેરવી હોય, તો ફક્ત એરેની સામગ્રીમાં ફેરફાર કરવાની જરૂર છે:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

ફરીથી નવી `Image` code વારંવાર લખવાની જરૂર નથી.

જો છબીઓ ઘણી હોય, તો તેને `ScrollView` સાથે વાપરી શકાય. નહીં તો, સામગ્રી સ્ક્રીનથી બહાર નીકળી જશે અને સંપૂર્ણ preview શક્ય નહીં બને.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### છબીઓમાં ફેરફાર

જ્યારે અમને બધી છબીઓની સ્ટાઇલ એકસરખી રીતે બદલવી હોય, ત્યારે ફક્ત `ForEach` ની અંદરનો કોડ બદલવો પડે છે.

ઉદાહરણ તરીકે, પહોળાઈ `80` કરો:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

આ રીતે, બધી છબીઓ એકસાથે બદલાઈ જશે, અને એક પછી એક ફેરફાર કરવાની જરૂર નહીં પડે.

આ જ `ForEach` નું એક મહત્વપૂર્ણ કામ છે:

**જ્યારે ઘણા views ની રચના એકસરખી હોય અને ફક્ત સામગ્રી જ અલગ હોય, ત્યારે આપણે view code ફક્ત એક વાર લખી શકીએ અને એરેનો ડેટા નક્કી કરે કે શું દર્શાવવું.**

## સારાંશ

આ પાઠમાં, આપણે `for` લૂપ અને `ForEach` શીખ્યા.

`for` લૂપનો ઉપયોગ એરે અથવા રેન્જના તત્વોને ક્રમ પ્રમાણે પ્રોસેસ કરવા માટે થાય છે, અને તે વધુ ડેટા પ્રોસેસિંગ તરફ ઝુકે છે.

`ForEach` નો ઉપયોગ SwiftUI માં એરેની સામગ્રીના આધારે પુનરાવર્તિત રીતે views બનાવવા માટે થાય છે, અને તે વધુ view display તરફ ઝુકે છે.

હાલના શરૂઆતના તબક્કામાં, `ForEach` નું સૌથી સામાન્ય લખાણ આ છે:

```swift
ForEach(એરે, id: \.self) { તત્વ in
    SwiftUI કોડ
}
```

અહીં `id: \.self` નો અર્થ એ છે કે દરેક તત્વને અલગ પાડવા માટે તત્વને જ વાપરવું.

જ્યારે એરેની સામગ્રી `String` અને `Int` જેવી સરળ પ્રકારની હોય, ત્યારે આપણે સામાન્ય રીતે આ જ એક લખાણથી શરૂઆત કરી શકીએ.

આગળ જઈને, જ્યારે તમે વેબસાઇટ અથવા એપમાં પુનરાવર્તિત લખાણ, છબીઓ અથવા અન્ય યાદી સામગ્રી જુઓ, ત્યારે તમે વિચારી શકો કે SwiftUI માં આવી સામગ્રી સામાન્ય રીતે પહેલા એરેમાં મૂકાશે અને પછી `ForEach` દ્વારા એક પછી એક દર્શાવવામાં આવશે.
