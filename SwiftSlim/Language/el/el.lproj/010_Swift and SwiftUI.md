# Swift και SwiftUI

Όταν μαθαίνουμε ανάπτυξη για τις πλατφόρμες της Apple, πολλά tutorials τείνουν να εξηγούν τη Swift και το SwiftUI ξεχωριστά. Αυτό μπορεί εύκολα να δημιουργήσει στους αρχάριους μια αίσθηση αποσύνδεσης, σαν να πρόκειται για δύο εντελώς ανεξάρτητες τεχνολογίες.

Στην πραγματικότητα, η Swift και το SwiftUI είναι βαθιά συνδεδεμένα: η Swift είναι η γλώσσα προγραμματισμού και είναι υπεύθυνη για τη λογική και τα δεδομένα, ενώ το SwiftUI είναι ένα framework διεπαφής βασισμένο στη Swift, υπεύθυνο για το rendering και την αλληλεπίδραση των views.

Αυτό το μάθημα θα σε βοηθήσει να διαμορφώσεις ένα καθαρό νοητικό πλαίσιο: τι είναι η Swift, τι είναι το SwiftUI και πώς συνεργάζονται μέσα στον κώδικα.

## Τι είναι η Swift;

Η Swift είναι μια σύγχρονη γλώσσα προγραμματισμού που δημιούργησε η Apple και χρησιμοποιείται για την ανάπτυξη εφαρμογών σε πλατφόρμες όπως iOS, macOS, watchOS και tvOS.

![swift](../../RESOURCE/010_swift.png)

Η Swift είναι strongly typed και static language. Από πλευράς σχεδιασμού δίνει μεγαλύτερη έμφαση στην ασφάλεια και μειώνει πολλά συνηθισμένα λάθη προγραμματισμού, όπως null pointers ή out-of-range προβλήματα. Επίσης υποστηρίζει ομαλή συνεργασία με C και Objective-C.

Στην πραγματική ανάπτυξη, η Swift είναι κυρίως υπεύθυνη για το λογικό κομμάτι της εφαρμογής, όπως επεξεργασία δεδομένων, δικτυακά αιτήματα και λειτουργίες αποθήκευσης.

## Τι είναι το SwiftUI;

Το SwiftUI είναι ένα νέο framework δημιουργίας διεπαφών που παρουσίασε η Apple το 2019. Το SwiftUI υιοθετεί declarative programming και επιτρέπει στους προγραμματιστές να περιγράφουν τη δομή της διεπαφής και τη συμπεριφορά αλληλεπίδρασης με πιο απλό και σύντομο κώδικα.

![swiftui](../../RESOURCE/010_swiftui.png)

Το declarative programming σημαίνει ότι ο προγραμματιστής αρκεί να πει στο σύστημα τι θέλει να εμφανιστεί, και το σύστημα θα ανανεώσει αυτόματα τη view σύμφωνα με τις αλλαγές των δεδομένων. Δεν χρειάζεται πλέον χειροκίνητη ενημέρωση του UI state, κάτι που απλοποιεί σημαντικά την ανάπτυξη διεπαφών.

Ο πυρήνας του SwiftUI αποτελείται από view components όπως `Text`, `Image`, `Button`, καθώς και layout containers όπως `VStack`, `HStack` και `ZStack`. Αυτά συνεργάζονται μέσω data binding και state management, ώστε η διεπαφή να ανταποκρίνεται στις αλλαγές των δεδομένων και να ανανεώνεται αυτόματα.

## Πώς συνεργάζονται η Swift και το SwiftUI;

Οι ρόλοι της Swift και του SwiftUI μπορούν να συνοψιστούν ως εξής:

**1. Swift: επεξεργασία λογικής και δεδομένων**

Η Swift χρησιμοποιείται κυρίως για τη διαχείριση δεδομένων, την αποθήκευση κατάστασης και την εκτέλεση λογικής. Αυτός ο κώδικας συνήθως δεν επηρεάζει άμεσα το rendering της διεπαφής, αλλά είναι υπεύθυνος για τα δεδομένα και τη συμπεριφορά της εφαρμογής.

Για παράδειγμα, επεξεργασία δεδομένων στη Swift:

```swift
let markdown = try? String(contentsOf: url)
```

Αυτός ο κώδικας περιλαμβάνει δηλώσεις μεταβλητών, σταθερών, συναρτήσεων, control flow κτλ., αλλά δεν επηρεάζει άμεσα την εμφάνιση των views.

**2. SwiftUI: δήλωση και rendering της διεπαφής**

Το SwiftUI χρησιμοποιείται για τη δημιουργία του layout και του περιεχομένου της διεπαφής και είναι κυρίως υπεύθυνο για το rendering και την αλληλεπίδραση των views.

Για παράδειγμα, δημιουργία ενός text view με SwiftUI:

```swift
Text("SwiftSlim")
```

Αυτού του είδους ο κώδικας χρησιμοποιείται για να χτίζει και να ελέγχει τα στοιχεία της διεπαφής και σχετίζεται άμεσα με το UI.

### Βασικό παράδειγμα

Παρακάτω είναι ένα απλό παράδειγμα SwiftUI:

```swift
// ContentView.swift

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Σε αυτό το παράδειγμα:

**Κώδικας SwiftUI**: Τα `VStack`, `Image`, `Text` και `padding` ανήκουν στο SwiftUI και είναι υπεύθυνα για την εμφάνιση και το layout.

**Κώδικας Swift**: Τα `//` σχόλια στην κορυφή του αρχείου και το `import SwiftUI` ανήκουν στη δομή κώδικα της Swift. Το `#Preview` στο τέλος είναι macro για την προεπισκόπηση στο Xcode και δεν συμμετέχει στο πραγματικό rendering της διεπαφής.

### Πιο προχωρημένο παράδειγμα

Σε πραγματικά projects, η Swift και το SwiftUI συχνά χρησιμοποιούνται μαζί, ώστε η καθεμία να κάνει αυτό στο οποίο είναι καλύτερη:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    func printName() {
        print("name: \(name)")
    }

    var body: some View {
        VStack {
            Text(name)
            Button("Print") {
                printName()
            }
        }
        .padding()
    }
}
```

Σε αυτό το παράδειγμα:

**Κώδικας SwiftUI**: Το `@State` είναι property wrapper ειδικός του SwiftUI και χρησιμοποιείται για να δηλώσει mutable state μιας view. Το `Button` και το `Text` είναι components του SwiftUI και είναι υπεύθυνα για την εμφάνιση και την αλληλεπίδραση.

**Κώδικας Swift**: Το `private var name` και η `func printName()` είναι κώδικας Swift. Χρησιμοποιούνται για αποθήκευση δεδομένων και εκτέλεση λογικής και δεν επηρεάζουν άμεσα το rendering του view.

Όταν ο χρήστης πατήσει το button, το SwiftUI θα ενεργοποιήσει τον κώδικα της ενέργειας:

```swift
Button("Print") {
    printName()
}
```

Εδώ, το `Button` είναι component του SwiftUI, ενώ η συνάρτηση `printName()` που εκτελείται είναι κώδικας Swift υπεύθυνος για τη συγκεκριμένη λογική.

Αυτή η συνεργασία επιτρέπει στη Swift και στο SwiftUI να συνδυάζονται ομαλά: η Swift αναλαμβάνει δεδομένα και λογική, και το SwiftUI τη διεπαφή.

## Πού γράφεται συνήθως ο κώδικας Swift και SwiftUI;

Στο SwiftUI, η διεπαφή χτίζεται μέσα από τη view που επιστρέφει το property `body`. Επομένως, όλος ο κώδικας που περιγράφει τη διεπαφή συνήθως γράφεται μέσα στο `body`.

Για παράδειγμα:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

Σε αυτό το παράδειγμα, το `Text` είναι component του SwiftUI, άρα πρέπει να γράφεται μέσα στο `body`, επειδή το SwiftUI διαβάζει το `body` για να δημιουργήσει τη διεπαφή.

Κώδικας που δεν σχετίζεται άμεσα με τη διεπαφή, όπως μεταβλητές, συναρτήσεις ή λογική επεξεργασίας δεδομένων, συνήθως γράφεται έξω από το `body`. Για παράδειγμα:

```swift
struct ContentView: View {

    // Swift: δεδομένα ή λογική
    var name = "Fang"

    func printName() {
        print(name)
    }

    // SwiftUI: διεπαφή
    var body: some View {
        Text(name)
    }
}
```

Πρέπει να προσέξουμε ότι μέσα στο `body` μπορούμε ακόμη να χρησιμοποιούμε σύνταξη Swift, όπως `if` ή `for`, απλώς αυτά χρησιμοποιούνται για να ελέγχουν τη δημιουργία των SwiftUI views:

```swift
if isLogin {
    Text("Welcome")
}
```

Άρα, στην ανάπτυξη με SwiftUI, μπορούμε να το κατανοήσουμε απλά ως εξής: ο κώδικας των views (`Text`, `Image`, `Button` κτλ.) γράφεται συνήθως μέσα στο `body`, ενώ ο κώδικας δεδομένων και λογικής (μεταβλητές, συναρτήσεις κτλ.) γράφεται συνήθως έξω από το `body`.

## Swift αρχεία

Καθώς προχωράμε περισσότερο, θα έρθουμε επίσης σε επαφή με την αρχιτεκτονική MVVM, όπου τα layers `ViewModel` και `Model` συνήθως αποτελούνται από καθαρό κώδικα Swift και είναι πλήρως διαχωρισμένα από το layer της διεπαφής (SwiftUI).

Για παράδειγμα, μια κλάση που διαχειρίζεται την κατάσταση της εφαρμογής:

```swift
class AppStorage: ObservableObject {
    static var shared = AppStorage()
    private var isLoading = false
}
```

Ή ένα struct που περιγράφει δομή δεδομένων:

```swift
struct SponsorStruct: Identifiable {
    var id:String
    var icon: String
}
```

Η ευθύνη αυτών των αρχείων είναι η διαχείριση και η αποθήκευση δεδομένων, χωρίς να εμφανίζονται άμεσα στη διεπαφή. Γι’ αυτό ανήκουν εξ ολοκλήρου στη Swift.

## Ιστορικό πλαίσιο

Για να κατανοήσουμε πραγματικά τη σχέση ανάμεσα στη Swift και στο SwiftUI, βοηθά να γνωρίζουμε και την εξέλιξη των τεχνολογιών ανάπτυξης της Apple. Μέχρι το 2026, αυτό το τεχνολογικό stack έχει περάσει από αρκετές φάσεις εξέλιξης.

### Η ιστορία της Swift

Πριν εμφανιστεί η Swift, η βασική γλώσσα ανάπτυξης στις πλατφόρμες της Apple ήταν η Objective-C, η οποία μπορούσε να συνυπάρχει με C. Η σύνταξή της ήταν σχετικά πιο εκτενής και για αρχάριους είχε πιο υψηλό εμπόδιο κατανόησης:

```Obj-c
// Σύνταξη Objective-C
NSString *name = @"Fang";
NSLog(@"Hello，%@", name);
```

Το 2014, η Apple παρουσίασε τη Swift στο WWDC. Η σύνταξη της Swift ήταν πιο σύγχρονη και με μεγαλύτερη type safety, και σταδιακά αντικατέστησε την Objective-C ως κύρια γλώσσα ανάπτυξης:

```swift
// Σύνταξη Swift
let name = "Fang"
print("Hello, \(name)")
```

Ωστόσο, η Objective-C δεν εξαφανίστηκε. Παραμένει ευρέως χρησιμοποιούμενη σε πολλά παλαιότερα projects και σε χαμηλότερου επιπέδου frameworks. Η βασική γνώση της σύνταξής της εξακολουθεί να είναι χρήσιμη, τόσο για συντήρηση παλιότερων εφαρμογών όσο και για καλύτερη κατανόηση του συστήματος.

### Η ιστορία του SwiftUI

Πριν από το SwiftUI, στο iOS χρησιμοποιούνταν το **UIKit**, ενώ στο macOS το **AppKit**. Αυτά τα δύο frameworks βασίζονται σε imperative programming. Οι προγραμματιστές έπρεπε να ελέγχουν την κατάσταση των views είτε μέσω Storyboard με drag-and-drop controls είτε γράφοντας κώδικα χειροκίνητα. Αυτό οδηγούσε σε περισσότερο κώδικα και μεγαλύτερο κόστος συντήρησης, ειδικά όταν η λογική της διεπαφής γινόταν σύνθετη.

![storyboard](../../RESOURCE/010_xcode.png)

Το 2019, η Apple παρουσίασε επίσημα το SwiftUI στο WWDC. Το SwiftUI εισήγαγε το declarative programming paradigm και απλοποίησε σημαντικά την ανάπτυξη UI.

![storyboard](../../RESOURCE/010_xcode1.png)

Αξίζει να προσέξουμε ότι το SwiftUI δεν είναι πλήρως ανεξάρτητη low-level υλοποίηση. Στις διάφορες πλατφόρμες, στην ουσία συνεργάζεται και «γεφυρώνεται» με το UIKit (στο iOS) ή το AppKit (στο macOS), και εξακολουθεί να βασίζεται σε αυτά στο υπόβαθρο.

### Η σχέση της Swift με UIKit/AppKit

Παρότι η Swift είναι μια γενικής χρήσης γλώσσα προγραμματισμού που μπορεί να χρησιμοποιηθεί σε όλες τις πλατφόρμες της Apple, δεν μπορεί να αντικαταστήσει πλήρως το UIKit ή το AppKit. Για κάποιες πιο σύνθετες ανάγκες διεπαφής ή για λειτουργίες που το SwiftUI δεν καλύπτει ακόμη πλήρως, εξακολουθεί να χρειάζεται η βοήθεια του UIKit ή του AppKit.

Για παράδειγμα, το UIKit είναι ήδη πολύ ώριμο σε τομείς όπως η διαχείριση σύνθετων view controllers, τα animations και η αναγνώριση gestures, και έχει δοκιμαστεί εκτενώς στην πράξη. Το SwiftUI συνεχώς βελτιώνεται σε αυτά τα σημεία, αλλά σε ορισμένα edge cases εξακολουθούν να υπάρχουν περιορισμοί.

Γι’ αυτό πολλοί προγραμματιστές χρησιμοποιούν μικτό μοντέλο SwiftUI μαζί με UIKit ή AppKit, ώστε να αξιοποιούν τα πλεονεκτήματα και των δύο.

Από αυτή τη σκοπιά, μπορούμε να δούμε το SwiftUI ως ένα υψηλότερου επιπέδου abstraction πάνω από το UIKit / AppKit. Μαθαίνοντας SwiftUI, είναι χρήσιμο να έχουμε και μια βασική εικόνα για UIKit και AppKit, γιατί αυτό βοηθά στη λήψη καλύτερων τεχνικών αποφάσεων όταν συντηρούμε παλιότερα projects ή υλοποιούμε πιο σύνθετες λειτουργίες.

## Σύνοψη

**Swift**: χρησιμοποιείται κυρίως για λογική, επεξεργασία δεδομένων, control flow και γενικά ό,τι δεν αφορά το layout μιας view.

**SwiftUI**: χρησιμοποιείται για declarative κατασκευή της διεπαφής χρήστη. Ο κώδικας για το περιεχόμενο και το layout των views ανήκει στο SwiftUI.

Στην πράξη, η Swift και το SwiftUI χρησιμοποιούνται σχεδόν πάντα μαζί: η Swift επεξεργάζεται τη λογική, ενώ το SwiftUI διαχειρίζεται τη διεπαφή.

Από την Objective-C και το UIKit μέχρι τη Swift και το SwiftUI, το περιβάλλον ανάπτυξης της Apple έχει εξελιχθεί σταδιακά προς πιο σύγχρονους και πιο απλούς τρόπους προγραμματισμού. Παρ’ όλα αυτά, το UIKit και το AppKit παραμένουν σημαντικά σε πολλά παλιότερα projects.

Αν κατανοήσουμε σωστά τη σχέση της Swift με το SwiftUI, μπορούμε να αναπτύσσουμε πιο αποδοτικά εφαρμογές iOS και macOS και ταυτόχρονα να παίρνουμε πιο σωστές τεχνικές αποφάσεις όταν χρειάζεται να συντηρούμε παλαιότερα projects.
