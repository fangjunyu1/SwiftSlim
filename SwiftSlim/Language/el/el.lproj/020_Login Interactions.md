# Αλληλεπιδράσεις σύνδεσης

Στο προηγούμενο μάθημα δημιουργήσαμε μια προβολή σύνδεσης και, μέσω κουμπιών και λογικής ελέγχου, υλοποιήσαμε έναν απλό έλεγχο του περιεχομένου εισόδου.

Σε αυτό το μάθημα θα συνεχίσουμε με βάση τον κώδικα του προηγούμενου μαθήματος, θα μάθουμε το πεδίο εισαγωγής κωδικού `SecureField`, το πλαίσιο προειδοποίησης `Alert`, καθώς και πώς να ελέγχουμε την εμφάνιση διαφορετικών προβολών σύμφωνα με τις μεταβλητές.

Μέσα από αυτό το μάθημα μπορείς να κατανοήσεις καλύτερα τη βασική ροή αλληλεπίδρασης μιας προβολής σύνδεσης, κάτι που αποτελεί επίσης βασική λογική που χρησιμοποιείται σε πολλές εφαρμογές.

## Πεδίο εισαγωγής κωδικού

Στο προηγούμενο μάθημα χρησιμοποιήσαμε `TextField` για να εισαγάγουμε τον κωδικό.

Όμως σε μια πραγματική οθόνη σύνδεσης, ο κωδικός συνήθως δεν εμφανίζεται άμεσα, αλλά προβάλλεται ως τελείες ή με άλλο εφέ απόκρυψης, ώστε να προστατεύεται η ιδιωτικότητα του χρήστη.

Στο SwiftUI, το στοιχείο που χρησιμοποιείται ειδικά για την εισαγωγή κωδικού είναι το `SecureField`.

Για παράδειγμα:

```swift
SecureField("Password", text: $password)
```

Ο τρόπος γραφής του μοιάζει πολύ με το `TextField`, και επίσης χρειάζεται ένα κείμενο υπόδειξης και μια δεσμευμένη μεταβλητή.

Μπορούμε να το κατανοήσουμε απλά ως εξής: το `SecureField` είναι επίσης ένα πεδίο εισαγωγής, απλώς κρύβει το περιεχόμενο που πληκτρολογείται.

Μπορούμε να αλλάξουμε απευθείας το `TextField` που χρησιμοποιούσαμε αρχικά για την εισαγωγή του κωδικού σε `SecureField`:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Όταν ο χρήστης εισάγει τον κωδικό, το περιεχόμενο δεν θα εμφανίζεται άμεσα στη διεπαφή.

Τώρα μπορούμε να εισάγουμε όνομα χρήστη και κωδικό και, αφού πατήσουμε το κουμπί, να κρίνουμε αν η εισαγωγή είναι σωστή.

### Εμφάνιση μηνυμάτων ανατροφοδότησης

Πιο πριν χρησιμοποιούσαμε συνεχώς το `print` για να εμφανίζουμε αποτελέσματα.

Το περιεχόμενο του `print` εμφανίζεται μόνο στην κονσόλα. Ο προγραμματιστής μπορεί να το δει όταν εκτελεί το πρόγραμμα στο Xcode, αλλά οι συνηθισμένοι χρήστες στο κινητό δεν μπορούν να δουν αυτές τις ειδοποιήσεις.

Δηλαδή, παρόλο που το πρόγραμμα έχει ολοκληρώσει ήδη τον έλεγχο, ο χρήστης δεν γνωρίζει αν το περιεχόμενο που εισήγαγε είναι σωστό.

Σε αυτό το σημείο χρειαζόμαστε έναν τρόπο να εμφανίζουμε ειδοποιήσεις απευθείας στη διεπαφή.

## Πλαίσιο προειδοποίησης Alert

Στο SwiftUI μπορούμε να χρησιμοποιήσουμε το `Alert` για να εμφανίσουμε ένα αναδυόμενο πλαίσιο ειδοποίησης.

Βασική χρήση:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

Αυτό το κομμάτι κώδικα φαίνεται λίγο πιο σύνθετο από τα προηγούμενα στοιχεία, αλλά μπορούμε να το καταλάβουμε αν το διασπάσουμε.

Το `Alert` περιλαμβάνει κυρίως τα παρακάτω μέρη:

- `"Title"`: ο τίτλος του πλαισίου προειδοποίησης
- `isPresented`: μια δεσμευμένη boolean τιμή που χρησιμοποιείται για να ελέγχει αν θα εμφανιστεί το πλαίσιο προειδοποίησης
- `Button(...)`: το κουμπί του πλαισίου προειδοποίησης
- `message:`: το περιεχόμενο του μηνύματος που εμφανίζεται μέσα στο πλαίσιο προειδοποίησης

Μπορεί να κατανοηθεί απλά ως εξής: όταν η δεσμευμένη μεταβλητή γίνει `true`, το σύστημα θα εμφανίσει αυτό το πλαίσιο ειδοποίησης.

### Το Alert είναι τροποποιητής

Όπως και πολλοί από τους τροποποιητές που μάθαμε προηγουμένως, έτσι και το `Alert` προστίθεται πίσω από κάποια προβολή.

Για παράδειγμα, μπορεί να τοποθετηθεί πίσω από το κουμπί:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

Μπορεί επίσης να τοποθετηθεί πίσω από το εξωτερικό `VStack`:

```swift
VStack {
    // ...
}
.alert() {...}
```

Δηλαδή, το `Alert` δεν είναι μια ανεξάρτητη προβολή που τοποθετείται μόνη της στη διεπαφή, αλλά ένας τροποποιητής που συνδέεται με κάποια προβολή.

Μπορούμε να το κατανοήσουμε απλά έτσι: το `Alert` συνήθως γράφεται πίσω από το κουμπί ή το εξωτερικό layout, ώστε να εμφανίζει μια ειδοποίηση όταν ικανοποιείται κάποια συνθήκη.

### Εμφάνιση Alert

Αν θέλουμε να εμφανίσουμε ένα `Alert`, χρειαζόμαστε μια μεταβλητή για να το ελέγχει.

Για παράδειγμα:

```swift
@State private var showAlert = false
```

Αυτή η μεταβλητή είναι τύπου `Bool`, δηλαδή έχει μόνο δύο τιμές: `true` ή `false`.

Έπειτα τη δεσμεύουμε στο `isPresented` του `Alert`:

```swift
isPresented: $showAlert
```

Όταν το `showAlert` είναι `false`, το πλαίσιο προειδοποίησης δεν εμφανίζεται. Όταν το `showAlert` είναι `true`, εμφανίζεται το πλαίσιο προειδοποίησης.

### Εμφάνιση πλαισίου προειδοποίησης μέσω κουμπιού

Τώρα ας υλοποιήσουμε πρώτα το πιο απλό σενάριο:

Όταν ο χρήστης δεν έχει εισαγάγει τίποτα και πατήσει απευθείας το κουμπί `Sign in`, εμφανίζεται ένα πλαίσιο ειδοποίησης που του λέει: «Δεν έχεις εισαγάγει ακόμη κανένα περιεχόμενο».

Πρώτα, δημιούργησε μια μεταβλητή για να ελέγχει αυτό το πλαίσιο ειδοποίησης:

```swift
@State private var showEmpty = false
```

Έπειτα, πρόσθεσε ένα `Alert` πίσω από το `Button`:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

Στη συνέχεια, κάνε τον έλεγχο μέσα στον κώδικα του πατήματος του κουμπιού:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

Αν τόσο το `user` όσο και το `password` είναι κενά, τότε αλλάζουμε το `showEmpty` σε `true`.

Και επειδή το `Alert` είναι δεσμευμένο σε αυτή τη μεταβλητή, όταν αυτή γίνει `true`, το σύστημα θα εμφανίσει αυτόματα το πλαίσιο ειδοποίησης.

Αποτέλεσμα εμφάνισης:

![alert](../../Resource/020_alert1.png)

Έτσι, αφού ο χρήστης πατήσει το κουμπί, μπορεί αμέσως να δει την ειδοποίηση, χωρίς να χρειάζεται ο προγραμματιστής να ελέγξει το αποτέλεσμα του `print` στην κονσόλα.

### Βελτίωση του κουμπιού και των πλαισίων προειδοποίησης

Πριν από αυτό, είχαμε επεξεργαστεί μόνο την περίπτωση «δεν έχει συμπληρωθεί τίποτα».

Χρειάζεται επίσης να χειριστούμε τις παρακάτω περιπτώσεις:

- δεν έχει συμπληρωθεί το όνομα χρήστη
- δεν έχει συμπληρωθεί ο κωδικός
- έχουν συμπληρωθεί το όνομα χρήστη και ο κωδικός, αλλά η εισαγωγή είναι λανθασμένη

Επομένως, χρειάζεται να ετοιμάσουμε και τις αντίστοιχες ειδοποιήσεις για αυτές τις περιπτώσεις.

Πρώτα, πρόσθεσε τρεις νέες μεταβλητές:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

Έπειτα συνέχισε προσθέτοντας τα αντίστοιχα `Alert` πίσω από το κουμπί:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

Στη συνέχεια, ολοκλήρωσε τη λογική ελέγχου μέσα στο κουμπί:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

Όταν το κουμπί κρίνει ότι ο χρήστης δεν έχει συμπληρώσει καμία πληροφορία, έχει παραλείψει μία πληροφορία ή έχει αποτύχει η επαλήθευση της εισαγωγής, η αντίστοιχη boolean μεταβλητή αλλάζει σε `true` και στη συνέχεια εμφανίζεται το αντίστοιχο `Alert`.

Εδώ σου προτείνω να δοκιμάσεις μόνος σου μερικές περιπτώσεις:

- να μη συμπληρώσεις τίποτα
- να συμπληρώσεις μόνο το όνομα χρήστη
- να συμπληρώσεις μόνο τον κωδικό
- να συμπληρώσεις και όνομα χρήστη και κωδικό, αλλά με λανθασμένο περιεχόμενο
- να συμπληρώσεις και όνομα χρήστη και κωδικό, και το περιεχόμενο να είναι σωστό

Για να επιβεβαιώσουμε αν υπάρχει κάποιο πρόβλημα στον κώδικά μας. Αυτό είναι επίσης ένα πολύ σημαντικό στάδιο «δοκιμών» στην ανάπτυξη εφαρμογών.

Αν σε κάποιο σενάριο το πρόγραμμα δεν συμπεριφέρεται σωστά, αυτό σημαίνει ότι πιθανώς υπάρχει πρόβλημα στη λογική και έχει εμφανιστεί κάποιο `Bug` στο πρόγραμμα.

Αυτό που συχνά αποκαλούμε `Bug` στην ανάπτυξη σημαίνει λάθος, κενό ή συμπεριφορά που δεν ανταποκρίνεται στις προσδοκίες.

## Ολοκλήρωση της προβολής σύνδεσης

Τα `Alert` που αναφέραμε προηγουμένως αφορούσαν όλα το πώς να δίνουμε ειδοποιήσεις στον χρήστη όταν «η εισαγωγή είναι λανθασμένη».

Όταν ο χρήστης εισαγάγει σωστά το όνομα χρήστη και τον κωδικό, συνήθως δεν εμφανίζουμε «πλαίσιο προειδοποίησης», αλλά μπαίνουμε κατευθείαν στη διεπαφή μετά τη σύνδεση.

Εδώ εμπλέκεται ένα πολύ σημαντικό σημείο γνώσης: **εμφάνιση διαφορετικών προβολών ανάλογα με την τιμή μιας μεταβλητής.**

Για παράδειγμα:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

Εδώ χρησιμοποιούμε τον τύπο `Bool` για να διαχειριστούμε την εμφάνιση της προβολής:

- αν το `login` είναι `true`, σημαίνει ότι η σύνδεση έχει ήδη πετύχει και εμφανίζεται το περιεχόμενο μετά τη σύνδεση
- αν το `login` είναι `false`, σημαίνει ότι η σύνδεση δεν έχει πετύχει και συνεχίζει να εμφανίζεται η διεπαφή σύνδεσης.

Παρόλο που η εντολή `if` είναι μια συνθήκη ελέγχου της Swift, μέσα στο `body` του SwiftUI μπορεί να χρησιμοποιηθεί για να ελέγχει τη λογική εμφάνισης της προβολής.

Δηλαδή, το SwiftUI θα αποφασίσει, με βάση το αποτέλεσμα της συνθήκης, ποια προβολή πρέπει να εμφανιστεί αυτή τη στιγμή.

Μπορούμε να βάλουμε αυτή τη λογική εμφάνισης μέσα στο τρέχον `ContentView`:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Original login view code
		}
	}
}
```

Εδώ προσθέσαμε μια νέα μεταβλητή `login`, που χρησιμοποιείται για να δείχνει αν ο χρήστης έχει ήδη συνδεθεί επιτυχώς.

Έπειτα, τοποθετήσαμε τον κώδικα της προβολής σύνδεσης που είχαμε ήδη γράψει πίσω από την εντολή `else`.

Λογική εμφάνισης προβολής:

- όταν το `login` είναι `false`, σημαίνει ότι η σύνδεση δεν έχει πετύχει ακόμη, και συνεχίζει να εμφανίζεται η αρχική προβολή σύνδεσης.
- όταν το `login` είναι `true`, σημαίνει ότι η σύνδεση πέτυχε, και τότε δεν εμφανίζεται πλέον η προβολή σύνδεσης, αλλά εμφανίζεται το `Login successful`.

Δηλαδή, το `ContentView` θα εμφανίζει διαφορετικό περιεχόμενο προβολής ανάλογα με την τιμή του `login`.

Στη συνέχεια, ας τροποποιήσουμε τη λογική επιτυχούς σύνδεσης μέσα στο κουμπί:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

Όταν το όνομα χρήστη και ο κωδικός που εισήγαγε ο χρήστης είναι και τα δύο σωστά, αλλάζουμε το `login` σε `true`.

Μόλις αλλάξει η τιμή του `login`, θα αλλάξει και το αποτέλεσμα της κρίσης του `if` στο `ContentView`, και η διεπαφή θα μεταβεί από την αρχική προβολή σύνδεσης στο `Login successful`.

Το αποτέλεσμα εμφάνισης είναι το εξής:

![view](../../Resource/020_view.png)

Τώρα έχουμε πραγματικά εφαρμόσει τη γνώση «εμφάνιση διαφορετικών προβολών ανάλογα με τη συνθήκη» σε αυτήν την τρέχουσα σελίδα σύνδεσης.

Στις πραγματικές εφαρμογές, συνήθως χρησιμοποιείται ακριβώς αυτός ο τρόπος κρίσης: να εμφανίζονται διαφορετικές προβολές ανάλογα με κάποια τιμή.

Για παράδειγμα, όταν ο χρήστης δεν είναι συνδεδεμένος, εμφανίζεται η σελίδα σύνδεσης. Αφού ο χρήστης συνδεθεί επιτυχώς, εμφανίζεται μια άλλη διεπαφή.

Η βασική ιδέα είναι: **να ελέγχουμε τι περιεχόμενο εμφανίζεται στη διεπαφή μέσω των αλλαγών στις μεταβλητές.**

## Σύνοψη

Σε αυτό το μάθημα μάθαμε τρία σημαντικά πράγματα:

1. Να χρησιμοποιούμε `SecureField` για την εισαγωγή κωδικού, ώστε το περιεχόμενο του κωδικού να μην εμφανίζεται άμεσα στη διεπαφή.
2. Να χρησιμοποιούμε `Alert` για να εμφανίζουμε ειδοποιήσεις στον χρήστη, αντί να τις εκτυπώνουμε μόνο στην κονσόλα.
3. Να χρησιμοποιούμε μεταβλητές και ελέγχους `if` για να ελέγχουμε την εμφάνιση διαφορετικών προβολών και να υλοποιούμε την εναλλαγή μεταξύ της διεπαφής πριν και μετά τη σύνδεση.

Μέσα από τη μάθηση αυτών των γνώσεων, μπορούμε πλέον να ολοκληρώσουμε μια προβολή σύνδεσης με βασικά εφέ αλληλεπίδρασης και να κατανοήσουμε ακόμη καλύτερα τη λογική ανάπτυξης του SwiftUI: «η κατάσταση αλλάζει, η διεπαφή ενημερώνεται».

## Πλήρης κώδικας

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## Επεκταμένη γνώση - Alert με πολλά κουμπιά

Στο προηγούμενο αυτό μέρος του μαθήματος, χρησιμοποιήσαμε το πιο βασικό `Alert`, που έχει μόνο ένα κουμπί και χρησιμοποιείται κυρίως για να ειδοποιεί τον χρήστη.

Στην πραγματικότητα, το `Alert` μπορεί επίσης να περιλαμβάνει πολλά κουμπιά, ώστε ο χρήστης να κάνει διαφορετικές επιλογές.

Για παράδειγμα:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

Όταν πατηθεί το κουμπί, θα εμφανιστεί ένα πλαίσιο ειδοποίησης με τρεις επιλογές: `Ok`, `delete` και `cancel`.

Αποτέλεσμα εμφάνισης:

![alert](../../Resource/020_alert2.png)

Όταν πατηθούν διαφορετικά κουμπιά, θα εκτελεστεί ο αντίστοιχος κώδικας.

### Η ιδιότητα role του Button

Εδώ θα παρατηρήσεις ότι το `Button` μέσα στο `Alert` είναι λίγο διαφορετικό από τον συνηθισμένο τρόπο γραφής κουμπιών που μάθαμε προηγουμένως:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

Εδώ υπάρχει μία επιπλέον παράμετρος, η `role`.

Στο SwiftUI, η `role` χρησιμοποιείται για να ενημερώνει το σύστημα: σε ποιον τύπο ενέργειας ανήκει αυτό το κουμπί.

Μερικοί συνηθισμένοι ρόλοι είναι:

- `.cancel`: ενέργεια ακύρωσης
- `.destructive`: επικίνδυνη ενέργεια, για παράδειγμα διαγραφή
- `nil`: συνηθισμένη ενέργεια

Οι διαφορετικές τιμές `role` συνήθως φέρνουν και διαφορετικά στυλ εμφάνισης στη διεπαφή.

Για παράδειγμα, οι επικίνδυνες ενέργειες συχνά εμφανίζονται με πιο έντονο στυλ, για να υπενθυμίζουν στον χρήστη να πατήσει προσεκτικά.

Επιπλέον, η `role` παρέχει και επιπρόσθετες σημασιολογικές πληροφορίες στο κουμπί, ώστε οι αναγνώστες οθόνης (όπως το VoiceOver στις συσκευές Apple) να μπορούν πιο εύκολα να ενημερώνουν τον χρήστη για τον ρόλο του κουμπιού.

Επομένως, είναι πολύ καλή συνήθεια να προσθέτουμε `role` σε κουμπιά στις κατάλληλες περιπτώσεις.
