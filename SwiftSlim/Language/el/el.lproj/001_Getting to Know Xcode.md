# Γνωρίζοντας το Xcode

## Τι είναι το Xcode

Το Xcode είναι το επίσημο ολοκληρωμένο περιβάλλον ανάπτυξης (IDE, Integrated Development Environment) της Apple και χρησιμοποιείται για την ανάπτυξη εφαρμογών στις πλατφόρμες της Apple, όπως iOS, macOS, watchOS και tvOS.

![Swift](../../RESOURCE/001_xcode.png)

Σε όλη τη διάρκεια αυτού του tutorial θα χρησιμοποιούμε το Xcode για να γράφουμε κώδικα και να εκτελούμε προγράμματα.

Σε αυτό το μάθημα θα μάθουμε πώς να κατεβάζουμε το Xcode, πώς να δημιουργούμε την πρώτη μας εφαρμογή iOS και πώς να γνωρίσουμε τη βασική δομή της διεπαφής του Xcode.

## Λήψη του Xcode

Αυτή τη στιγμή υπάρχουν δύο τρόποι εγκατάστασης του Xcode:

1. Μέσω του [Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12)
2. Μέσω της [επίσημης ιστοσελίδας Apple Developer](https://developer.apple.com/xcode/)

Και οι δύο τρόποι επιτρέπουν τη λήψη του Xcode. Η διαφορά είναι ότι στην ιστοσελίδα Apple Developer, εκτός από τη σταθερή έκδοση, μπορείς να κατεβάσεις και [Beta](https://developer.apple.com/download/all/?q=Xcode) ή παλαιότερες εκδόσεις.

Αν απλώς μαθαίνεις ανάπτυξη εφαρμογών, προτείνεται να χρησιμοποιήσεις τη σταθερή έκδοση. Οι Beta εκδόσεις προορίζονται κυρίως για δοκιμή νέων δυνατοτήτων και μπορεί να είναι ασταθείς.

## Δημιουργία του πρώτου project

Αφού ανοίξεις το Xcode, θα δεις την αρχική οθόνη.

![Swift](../../RESOURCE/001_xcode1.png)

Στην αριστερή πλευρά υπάρχουν τρεις συνηθισμένες επιλογές:

- Create New Project — Δημιουργία νέου project
- Clone Git Repository — Κλωνοποίηση Git repository
- Open Existing Project — Άνοιγμα υπάρχοντος project

Στη δεξιά πλευρά εμφανίζονται τα πιο πρόσφατα projects του Xcode που έχουν ανοιχτεί.

Εμείς επιλέγουμε το "Create New Project".

### Επιλογή project template

![Swift](../../RESOURCE/001_xcode2.png)

Στη συνέχεια περνάς στην οθόνη επιλογής template, όπου εμφανίζονται διάφορες πλατφόρμες όπως iOS και macOS, καθώς και διαφορετικοί τύποι project templates.

Συνηθισμένα templates:

- App — Ο πιο βασικός τύπος εφαρμογής (προτείνεται)
- Document App — Εφαρμογή βασισμένη σε έγγραφα
- Game — Για ανάπτυξη παιχνιδιών
- Framework — Επαναχρησιμοποιήσιμο module

Στο στάδιο της εισαγωγής επιλέγουμε μόνο το template "iOS" - "App", επειδή είναι ο βασικός τύπος εφαρμογής με διεπαφή.

Έπειτα κάνε κλικ στο "Next".

### Συμπλήρωση πληροφοριών project

Τώρα μπαίνεις στην οθόνη συμπλήρωσης πληροφοριών του project, όπου πρέπει να συμπληρώσουμε τα πεδία ανάλογα με τον τύπο τους.

![Swift](../../RESOURCE/001_xcode3.png)

Η σημασία κάθε πεδίου:

- Product Name — Όνομα project, για παράδειγμα: SwiftSlimTest
- Team — Ομάδα προγραμματιστή. Αν δεν έχεις πληρωμένο λογαριασμό Apple Developer, μπορείς να το αφήσεις κενό
- Organization Identifier — Αναγνωριστικό οργανισμού, συνήθως σε αντίστροφη μορφή domain, για παράδειγμα:
  - com.yourname
  - com.yourcompany
  - Αν δεν έχεις domain, μπορείς να χρησιμοποιήσεις το αγγλικό σου όνομα
- Bundle Identifier — Το μοναδικό αναγνωριστικό της εφαρμογής, το οποίο δημιουργείται αυτόματα από το Organization Identifier και το Product Name
- Interface — Τεχνολογία διεπαφής, εδώ επιλέγουμε SwiftUI
- Language — Γλώσσα ανάπτυξης, εδώ επιλέγουμε Swift
- Testing System — Δημιουργεί από προεπιλογή test target (Unit Tests), το οποίο μπορούμε να αγνοήσουμε στην αρχή
- Storage — Framework για τοπική αποθήκευση δεδομένων, με δυνατότητα ενσωμάτωσης SwiftData ή Core Data, που επίσης αγνοούμε στο εισαγωγικό στάδιο

Όταν ολοκληρώσεις τη συμπλήρωση, κάνε κλικ στο "Next".

### Αποθήκευση project

![Swift](../../RESOURCE/001_xcode4.png)

Επίλεξε έναν κατάλληλο φάκελο για αποθήκευση.

Πάτησε το κουμπί "Create" και το Xcode θα δημιουργήσει αυτόματα τη δομή του project.

## Η δομή ενός Xcode project

Αφού αποθηκευτεί το project, στον Finder θα δεις τον φάκελο που δημιουργήθηκε.

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets — Φάκελος για εικόνες, εικονίδια εφαρμογής και άλλους πόρους
- ContentView.swift — Το προεπιλεγμένο αρχείο SwiftUI view, όπου αργότερα θα γράψουμε τον κώδικα της διεπαφής
- SwiftSlimTestApp.swift — Το αρχείο εισόδου της εφαρμογής (App Entry Point), υπεύθυνο για την εκκίνηση της εφαρμογής. Προς το παρόν δεν χρειάζεται να το κατανοήσουμε
- SwiftSlimTest.xcodeproj — Το αρχείο του Xcode project. Με διπλό κλικ ανοίγει το project

## Γνωρίζοντας τη διεπαφή του Xcode

Άνοιξε το project με διπλό κλικ στο αρχείο `.xcodeproj` και επίλεξε στην αριστερή περιοχή Navigator το αρχείο `ContentView.swift`.

Η διεπαφή του Xcode αποτελείται κυρίως από πέντε περιοχές:

- Navigator — Περιοχή πλοήγησης για προβολή δομής αρχείων και αναζήτηση
- Editor — Περιοχή επεξεργασίας για γραφή κώδικα Swift ή SwiftUI
- Canvas — Περιοχή καμβά για προεπισκόπηση SwiftUI
- Inspector — Περιοχή ιδιοτήτων για προβολή και τροποποίηση χαρακτηριστικών αρχείων
- Debug Area — Περιοχή αποσφαλμάτωσης για προβολή logs

![Swift](../../RESOURCE/001_xcode6.png)

Σημείωση: Το Canvas χρησιμοποιείται κυρίως για την προεπισκόπηση SwiftUI views. Από την πλευρά της αρχιτεκτονικής του Xcode, είναι ένα βοηθητικό panel προεπισκόπησης του Editor (Preview Pane). Μόνο όταν ανοίγεται αρχείο SwiftUI view μπορεί να ενεργοποιηθεί το Canvas· σε άλλους τύπους αρχείων δεν υπάρχει λειτουργία προεπισκόπησης.

Στα επόμενα μαθήματα θα γράφουμε κυρίως κώδικα Swift και SwiftUI στην περιοχή Editor.

Όταν ανοίγεις για πρώτη φορά ένα Xcode project, το Inspector και το Debug Area μπορεί να είναι κρυμμένα από προεπιλογή. Μπορείς να τα εμφανίσεις από τα κουμπιά πάνω δεξιά και κάτω δεξιά.

![Swift](../../RESOURCE/001_xcode7.png)

Συμβουλή: Η περιοχή Inspector χρησιμοποιείται κυρίως για προβολή και τροποποίηση ιδιοτήτων αρχείων. Στην πραγματική ανάπτυξη συνήθως παραμένει κρυφή, ώστε να υπάρχει περισσότερος χώρος για επεξεργασία κώδικα.

## Σύνοψη

Σε αυτό το μάθημα μάθαμε πώς να κατεβάζουμε το Xcode, πώς να δημιουργούμε το πρώτο μας Xcode project και πώς είναι οργανωμένη βασικά η διεπαφή του Xcode.

Τώρα είναι ώρα για λίγη ελεύθερη εξάσκηση:

- Δοκίμασε να δημιουργήσεις ή να διαγράψεις αρχεία στην περιοχή Navigator.
- Δες τον κώδικα στην περιοχή Editor και παρατήρησε τη δομή του.
- Δες τα κουμπιά στην περιοχή Canvas και δοκίμασε να μεγεθύνεις ή να μικρύνεις την προεπισκόπηση.

Στο επόμενο μάθημα θα αρχίσουμε να γράφουμε μερικές απλές γραμμές κώδικα και θα γνωρίσουμε ακόμη καλύτερα το Xcode.
