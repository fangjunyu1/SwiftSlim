# Ξεκινώντας από το ContentView

## Προετοιμασία

Σε αυτό το μάθημα θα ξεκινήσουμε από το αρχείο `ContentView.swift` και θα γνωρίσουμε συστηματικά τη βασική δομή του SwiftUI, συμπεριλαμβανομένων των εξής:

- Σχόλια
- Δομή View
- Layouts `VStack` / `HStack` / `ZStack`
- Εικονίδια SF Symbols
- View modifiers
- Κώδικας προεπισκόπησης `#Preview`

Αρχικά, βρες το Xcode project που είχες δημιουργήσει προηγουμένως και άνοιξέ το με διπλό κλικ στο αρχείο `.xcodeproj`.

Στην αριστερή περιοχή Navigator επίλεξε το αρχείο `ContentView.swift`.

Σημείωση: Κάθε φορά που ανοίγεις το project, το Canvas μπορεί να εμφανίζει το μήνυμα "Preview paused". Αν συμβεί αυτό, πάτησε το κουμπί ανανέωσης για να επανέλθει η προεπισκόπηση.

## Γνωρίζοντας το ContentView

Ο κώδικας του `ContentView`:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Παρόλο που ο κώδικας είναι λίγος, περιλαμβάνει ήδη τον πυρήνα της δομής του SwiftUI.

### 1. Σχόλια

Στην κορυφή του αρχείου:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

Αυτό είναι σχόλιο αρχείου, το οποίο χρησιμοποιείται για να καταγράφει πληροφορίες όπως το όνομα του αρχείου, το όνομα του project, τον δημιουργό και την ημερομηνία δημιουργίας.

Στη Swift χρησιμοποιούμε το `//` για σχόλια μίας γραμμής:

```swift
// Αυτό είναι σχόλιο
```

Τα σχόλια βελτιώνουν την αναγνωσιμότητα του κώδικα και βοηθούν τον προγραμματιστή να κατανοήσει τη λογική του.

Στην πράξη, ο κώδικας μπορεί εύκολα να γίνει δυσνόητος. Αν δεν γράφεις σχόλια, μπορεί να βρεθείς τρεις ημέρες μετά να μην καταλαβαίνεις ούτε τον δικό σου κώδικα.

Γι' αυτό, το να προσθέτεις ενεργά σχόλια καθώς γράφεις κώδικα είναι μια πολύ καλή συνήθεια. Τα σχόλια σε βοηθούν να καταγράφεις τη λογική και να διευκολύνεις τη μελλοντική συντήρηση του κώδικα.

**Προσωρινή απενεργοποίηση κώδικα**

Τα σχόλια μπορούν επίσης να χρησιμοποιηθούν για να απενεργοποιήσεις προσωρινά κώδικα και να εντοπίσεις προβλήματα.

Για παράδειγμα:

```text
A
B
C
```

Τα A, B και C είναι τρία κομμάτια κώδικα, και ένα από αυτά περιέχει σφάλμα. Μπορούμε λοιπόν να σχολιάσουμε προσωρινά κομμάτια κώδικα για να εντοπίσουμε το πρόβλημα.

Πρώτα σχολιάζουμε το A:

```text
// A
B
C
```

Αν μετά από αυτό ο κώδικας λειτουργεί κανονικά, τότε το πρόβλημα βρίσκεται στο A.

Αν το πρόβλημα παραμένει, μπορούμε να σχολιάσουμε και το B και να συνεχίσουμε έτσι μέχρι να βρούμε το προβληματικό σημείο.

Κατά την ανάπτυξη συναντάμε πολλά προβλήματα, και συχνά χρειάζεται να κάνουμε τέτοιου είδους έλεγχο σχολιάζοντας κώδικα. Αυτό μας βοηθά να εντοπίζουμε γρήγορα τα σφάλματα.

Στο Xcode μπορείς να χρησιμοποιήσεις τη συντόμευση:

```text
Command ⌘ + /
```

για να προσθέσεις ή να αφαιρέσεις σχόλια γρήγορα.

### 2. Εισαγωγή του SwiftUI framework

```swift
import SwiftUI
```

Αυτή η γραμμή σημαίνει ότι γίνεται εισαγωγή του framework SwiftUI.

Τύποι όπως `View`, `Text`, `Image` και `VStack` προέρχονται όλοι από αυτό το framework.

Αν δεν εισάγεις το SwiftUI, το Xcode θα εμφανίσει σφάλμα:

```text
Cannot find type 'View' in scope
```

Αυτό σημαίνει ότι ο compiler δεν μπορεί να αναγνωρίσει τον τύπο `View`.

### 3. Η δομή μιας View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Αν βλέπεις αυτή τη δομή View για πρώτη φορά, ίσως σου φανεί λίγο ξένη, επειδή περιέχει λέξεις-κλειδιά όπως `struct`, `View`, `var`, `body` και `some`.

Δεν έχουμε μάθει ακόμη αυτές τις λέξεις-κλειδιά. Προς το παρόν αρκεί να ξέρεις ότι αυτός ο κώδικας δημιουργεί μια View με όνομα `ContentView`.

Μπορείς να φανταστείς μια View σαν έναν καμβά πάνω στον οποίο ζωγραφίζουμε, και το εργαλείο ζωγραφικής είναι το SwiftUI.

Για παράδειγμα:

![Swift](../../RESOURCE/002_view.png)

Η παραπάνω εικόνα δείχνει τρεις σελίδες, αλλά στην πραγματικότητα πρόκειται απλώς για τρεις διαφορετικές Views.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

Όταν αναπτύσσουμε μια εφαρμογή με SwiftUI, κάθε σελίδα είναι μία View.

### 4. SwiftUI κώδικας

Μέσα στη View εμφανίζεται ο κώδικας SwiftUI:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Αυτός ο κώδικας SwiftUI σημαίνει ότι εμφανίζεται ένα κάθετο layout με ένα εικονίδιο και ένα κείμενο.

![Swift](../../RESOURCE/002_view1.png)

#### Layout με VStack

```swift
VStack { }  // Κάθετο layout
```

Το `VStack` είναι ένα container κάθετης διάταξης. Οι εσωτερικές Views τοποθετούνται από πάνω προς τα κάτω.

![Swift](../../RESOURCE/002_view8.png)

Τα τρία πιο συνηθισμένα layouts στο SwiftUI είναι:

- `VStack` — κάθετη διάταξη
- `HStack` — οριζόντια διάταξη
- `ZStack` — επικάλυψη (άξονας Z)

```swift
HStack { }  // Οριζόντια διάταξη
ZStack { }  // Επικάλυψη
```

Η σχηματική διάταξη των layouts είναι η εξής:

![Swift](../../RESOURCE/002_view2.png)

Για παράδειγμα, αν χρησιμοποιήσουμε `HStack` για οριζόντια διάταξη:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

τότε θα δεις ότι το εικονίδιο της γης και το κείμενο εμφανίζονται δίπλα δίπλα.

![Swift](../../RESOURCE/002_view3.png)

Όταν θέλουμε οριζόντια διάταξη, χρησιμοποιούμε `HStack`. Όταν θέλουμε επικάλυψη, χρησιμοποιούμε `ZStack`.

#### Image και SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Αυτός ο κώδικας εμφανίζει ένα εικονίδιο υδρογείου, το οποίο προέρχεται από το σύστημα εικονιδίων SF Symbols της Apple.

![Swift](../../RESOURCE/002_view9.png)

Το νόημα του κώδικα είναι: εμφάνισε ένα εικονίδιο υδρογείου, σε μεγάλο μέγεθος και με accent color.

Εκτός από το υδρόγειο, μπορούμε φυσικά να εμφανίσουμε και άλλα εικονίδια.

Για παράδειγμα, ένα σακίδιο:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**Πώς εμφανίζουμε άλλα εικονίδια;**

Χρειάζεται να χρησιμοποιήσουμε τη βιβλιοθήκη συστημικών εικονιδίων SF Symbols που παρέχει επίσημα η Apple.

Άνοιξε την ιστοσελίδα Apple Developer και κατέβασε το [SF Symbols](https://developer.apple.com/sf-symbols/).

![Swift](../../RESOURCE/002_sf.png)

Στη συνέχεια άνοιξε την εφαρμογή SF Symbols.

![Swift](../../RESOURCE/002_sf1.png)

Στα αριστερά εμφανίζονται οι κατηγορίες συμβόλων και στα δεξιά τα αντίστοιχα εικονίδια.

Αν κάνεις δεξί κλικ σε ένα εικονίδιο και επιλέξεις "Copy Name", θα πάρεις το όνομα του συμβόλου σε μορφή string.

Για παράδειγμα:

```text
"globe"
"backpack"
"heart"
```

Αν βάλεις αυτό το όνομα μέσα στο `Image(systemName:)`, μπορείς να εμφανίσεις διαφορετικά σύμβολα.

Σημείωση: Κάθε εικονίδιο SF Symbols έχει ελάχιστη υποστηριζόμενη έκδοση συστήματος. Αν η έκδοση του συστήματος είναι πολύ παλιά, το εικονίδιο ίσως να μην εμφανιστεί. Τις πληροφορίες συμβατότητας μπορείς να τις δεις μέσα στην εφαρμογή SF Symbols.

#### Modifiers

Στο SwiftUI, ένα modifier είναι μια μέθοδος που χρησιμοποιείται για να αλλάξει την εμφάνιση ή τη συμπεριφορά μιας View.

Μπορείς να φανταστείς τους modifiers σαν ρούχα: αν βάλεις διαφορετικά ρούχα, η εμφάνιση αλλάζει.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Τα `imageScale` και `foregroundStyle` είναι modifiers της `Image` view. Αλλάζουν την εμφάνιση της View χωρίς να αλλάζουν το ίδιο το περιεχόμενο της εικόνας.

**1. imageScale**

```swift
.imageScale(.large)
```

Με αυτό μπορείς να ελέγξεις το μέγεθος των εικονιδίων SF Symbols:

- `.small`
- `.medium`
- `.large`

![Swift](../../RESOURCE/002_view11.png)

Μπορείς να δοκιμάσεις διαφορετικές επιλογές για να εμφανίζονται τα SF Symbols σε διαφορετικά μεγέθη.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

Το `foregroundStyle` ελέγχει το χρώμα του προσκηνίου.

Το `.tint` σημαίνει το accent color του τρέχοντος περιβάλλοντος. Στο iOS είναι συνήθως μπλε από προεπιλογή.

Αν θέλουμε να αλλάξουμε το χρώμα σε κόκκινο:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Η Text view

Το `Text` είναι μια text view, που χρησιμοποιείται για την εμφάνιση strings.

```swift
Text("Hello, world!")
```

Για παράδειγμα, για να εμφανίσω το όνομά μου:

```swift
Text("FangJunyu")
```

Σημείωση: Τα strings πρέπει πάντα να είναι μέσα σε διπλά εισαγωγικά `""`.

Μπορείς να δοκιμάσεις να εμφανίσεις το όνομά σου, έναν αριθμό τηλεφώνου ή άλλο περιεχόμενο.

#### padding

Στο SwiftUI, το `padding` χρησιμοποιείται για να προσθέτει κενό ανάμεσα στο περιεχόμενο μιας view και στα όριά της. Ανήκει στην έννοια του εσωτερικού περιθωρίου (`padding / content inset`).

```swift
HStack {
    ...
}
.padding()
```

Ο παραπάνω κώδικας σημαίνει ότι στο `HStack` προστίθεται το προεπιλεγμένο system spacing.

**Τι είναι το padding;**

Το `padding` σημαίνει τον κενό χώρο ανάμεσα στο περιεχόμενο μιας view και στα όριά της.

Στην παρακάτω εικόνα, όταν το μπλε `HStack` αποκτήσει `padding`, η μπλε περιοχή «μαζεύεται» προς τα μέσα και φαίνεται σαν να μικραίνει λίγο.

![Swift](../../RESOURCE/002_view6.png)

**Προεπιλεγμένο περιθώριο**

Ο modifier `padding()` χρησιμοποιεί από προεπιλογή το standard spacing που προτείνει το σύστημα.

```swift
.padding()
```

Αυτή η τιμή μπορεί να διαφέρει ανάλογα με την πλατφόρμα και το περιβάλλον. Για παράδειγμα:

- Στο iOS είναι συνήθως περίπου 16 pt
- Στο macOS ή στο watchOS το standard spacing μπορεί να είναι διαφορετικό, μεγαλύτερο ή μικρότερο, ανάλογα με τις σχεδιαστικές οδηγίες της πλατφόρμας

**Προσαρμοσμένο padding**

Μπορούμε επίσης να ορίσουμε περιθώρια για συγκεκριμένες πλευρές.

1. Ορισμός μίας πλευράς

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

Επεξήγηση κατευθύνσεων:

- `.top`: πάνω περιθώριο
- `.bottom`: κάτω περιθώριο
- `.leading`: μπροστινή πλευρά
- `.trailing`: πίσω πλευρά

![Swift](../../RESOURCE/002_view12.png)

Σημείωση: Τα `leading` και `trailing` προσαρμόζονται αυτόματα ανάλογα με την κατεύθυνση της γλώσσας. Για παράδειγμα, σε περιβάλλον αραβικών (RTL) αντιστρέφονται αυτόματα.

2. Ορισμός πολλών πλευρών

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

Με τη χρήση array μπορούμε να ορίσουμε πολλές κατευθύνσεις μαζί. Τη συγκεκριμένη χρήση των arrays θα τη δούμε πιο αναλυτικά αργότερα· εδώ αρκεί να γνωρίζεις ότι υπάρχει αυτή η σύνταξη.

3. Ορισμός οριζόντιου ή κάθετου padding

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

Αυτό είναι ισοδύναμο με:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**Αφαίρεση padding**

Αν δεν θέλεις καθόλου padding, μπορείς να το αφαιρέσεις με `.padding(0)`:

```swift
.padding(0)
```

Ή να διαγράψεις εντελώς τον modifier `padding`:

```swift
// .padding()
```

### 6. Κώδικας προεπισκόπησης Preview

```swift

```

Αυτός ο κώδικας σημαίνει ότι δημιουργείται στο Canvas μια προεπισκόπηση της `ContentView`.

**Τι συμβαίνει αν σχολιάσουμε το Preview;**

```swift

// }
```

τότε το Canvas δεν θα εμφανίζει πλέον περιεχόμενο που μπορεί να γίνει render.

## Σύνοψη

Παρόλο που το αρχείο `ContentView.swift` δεν περιέχει πολύ κώδικα, στην πραγματικότητα καλύπτει αρκετές βασικές έννοιες του SwiftUI. Για έναν αρχάριο ίσως να φαίνεται ξένο, αλλά αν το αναλύσουμε βήμα-βήμα, μπορούμε να αποκτήσουμε μια πρώτη ξεκάθαρη εικόνα του SwiftUI.

Ανακεφαλαιώνοντας, σε αυτό το μάθημα γνωρίσαμε πρώτα τα σχόλια `//`, τα οποία μπορούμε να χρησιμοποιήσουμε είτε για να εξηγήσουμε τη λογική του κώδικα είτε για να απενεργοποιήσουμε προσωρινά κομμάτια του.

Στη συνέχεια μάθαμε ότι ένα αρχείο SwiftUI πρέπει να εισάγει το framework SwiftUI:

```swift
import SwiftUI
```

Χωρίς αυτή την εισαγωγή, ο compiler δεν μπορεί να αναγνωρίσει τύπους όπως το `View`.

Μετά γνωρίσαμε τη βασική δομή μιας SwiftUI view:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Όπου το `ContentView` είναι το όνομα της view.

Μάθαμε επίσης τρία βασικά layout containers: `VStack` (κάθετο), `HStack` (οριζόντιο) και `ZStack` (επικάλυψη).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Αυτός ο κώδικας σημαίνει ότι εμφανίζεται ένα κάθετο container, μέσα στο οποίο υπάρχει ένα εικονίδιο και ένα κείμενο.

Το `Image` μπορεί να εμφανίζει εικονίδια SF Symbols, ενώ με modifiers μπορούμε να ελέγξουμε το μέγεθος και το χρώμα τους.

Η `Text` view μπορεί να εμφανίζει περιεχόμενο κειμένου.

Το `padding` είναι περιθώριο και προσθέτει διαφανή χώρο γύρω από τη view.

Τέλος, είδαμε το `#Preview`, το οποίο επιτρέπει την εμφάνιση προεπισκόπησης στο Canvas.

### Ασκήσεις μετά το μάθημα

Για έναν αρχάριο το περιεχόμενο αυτού του μαθήματος μπορεί να φαίνεται αρκετά πυκνό. Μέσα από περισσότερη πρακτική, μπορείς να εμβαθύνεις καλύτερα όσα έμαθες:

- Άλλαξε το όνομα ενός εικονιδίου SF Symbols
- Άλλαξε το foreground color του εικονιδίου σε μαύρο
- Άλλαξε το `VStack` σε `HStack`
- Σχολίασε το `Image` ή το `Text` και παρατήρησε πώς αλλάζει η προεπισκόπηση

Για παράδειγμα, όταν αλλάζουμε τον modifier `imageScale`:

```swift
.imageScale(.)
```

το Xcode εμφανίζει τις διαθέσιμες επιλογές:

![Swift](../../RESOURCE/002_view10.png)

Αυτό είναι το σύστημα code completion. Βασίζεται στην type inference και στις προτάσεις enum members και βοηθά να γράφεις γρηγορότερα και με λιγότερα λάθη.

Στα επόμενα μαθήματα θα γνωρίσουμε επίσημα τα enums (`enum`). Προς το παρόν αρκεί να έχουμε μια πρώτη απλή εικόνα.
