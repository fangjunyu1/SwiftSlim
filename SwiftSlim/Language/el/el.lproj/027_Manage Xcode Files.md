# Διαχείριση αρχείων Xcode

Σε αυτό το μάθημα, θα μάθουμε τις συνηθισμένες λειτουργίες αρχείων στο Xcode.

Στα προηγούμενα μαθήματα, γράψαμε το μεγαλύτερο μέρος του κώδικα μέσα στο `ContentView`. Έτσι είναι πιο εύκολο να παρατηρούμε τον κώδικα και το αποτέλεσμα εμφάνισης.

Για παράδειγμα:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

Ωστόσο, σε μια πραγματική εφαρμογή, συνήθως δεν γράφουμε όλο τον κώδικα σε ένα μόνο αρχείο.

Όταν οι σελίδες, τα στοιχεία και οι λειτουργίες αυξάνονται, χρειάζεται να χωρίσουμε τις διαφορετικές προβολές σε διαφορετικά αρχεία SwiftUI.

Για παράδειγμα:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

Έτσι η δομή του κώδικα γίνεται πιο καθαρή και διευκολύνονται η συντήρηση και οι μελλοντικές αλλαγές.

## Γιατί χρειάζεται να διαχειριζόμαστε αρχεία

Στο SwiftUI, μια σελίδα ή ένα στοιχείο μπορεί συνήθως να γραφτεί ως ξεχωριστό αρχείο προβολής.

Για παράδειγμα:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

Το `CardView` εδώ είναι μια προβολή SwiftUI.

Αν όλες οι προβολές γραφτούν μέσα στο `ContentView`, ο κώδικας θα γίνεται όλο και μεγαλύτερος και όλο και πιο δύσκολος στην ανάγνωση.

Επομένως, χρειάζεται να μάθουμε να δημιουργούμε, να μετονομάζουμε και να διαγράφουμε αρχεία, καθώς και να χρησιμοποιούμε φακέλους για τη διαχείριση διαφορετικών αρχείων κώδικα.

## Δημιουργία αρχείου SwiftUI

Στο Xcode, υπάρχουν συνήθως δύο συνηθισμένοι τρόποι για τη δημιουργία αρχείου SwiftUI:

Ο ένας είναι να δημιουργήσουμε ένα κενό αρχείο και μετά να γράψουμε μόνοι μας τον κώδικα.

Ο άλλος είναι να δημιουργήσουμε ένα αρχείο προτύπου SwiftUI, ώστε το Xcode να δημιουργήσει αυτόματα τον βασικό κώδικα.

Για τους αρχάριους, συνιστάται περισσότερο η χρήση αρχείου προτύπου SwiftUI, επειδή δημιουργεί αυτόματα τη βασική δομή.

### Δημιουργία κενού αρχείου

Στην περιοχή Navigator στα αριστερά του Xcode, κάντε δεξί κλικ σε ένα κενό σημείο ή σε έναν φάκελο και επιλέξτε `New Empty File`.

Η θέση της ενέργειας είναι η εξής:

![xcode](../../../Resource/027_view1.png)

Μετά τη δημιουργία κενού αρχείου, το Xcode συνήθως δημιουργεί ένα ανώνυμο αρχείο Swift, όπως `Untitled.swift`.

Μπορούμε να το μετονομάσουμε σε `TestView.swift`.

Σε ένα έργο SwiftUI, αν αυτό το αρχείο χρησιμοποιείται για την εμφάνιση μιας προβολής, συνήθως προτείνεται να προσθέτουμε `View` στο τέλος του ονόματος.

Για παράδειγμα:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

Αυτό δεν είναι υποχρεωτικός κανόνας, αλλά μας επιτρέπει να καταλάβουμε με μια ματιά ότι πρόκειται για αρχείο προβολής.

Μετά τη δημιουργία κενού αρχείου, το αρχείο μπορεί να περιέχει μόνο προεπιλεγμένα σχόλια ή μπορεί να είναι κενό, ανάλογα με την έκδοση του Xcode και τον τρόπο δημιουργίας.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

Σε αυτό το σημείο, πρέπει να γράψουμε χειροκίνητα τον κώδικα SwiftUI:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

Έτσι, έχει δημιουργηθεί ένα απλό αρχείο προβολής SwiftUI.

### Μετονομασία αρχείου

Αν θέλετε να τροποποιήσετε το όνομα ενός αρχείου μετά τη δημιουργία του, μπορείτε να επιλέξετε το αρχείο στην περιοχή Navigator στα αριστερά του Xcode και μετά να πατήσετε το πλήκτρο Enter `⏎`.

![xcode](../../../Resource/027_view2.png)

Έτσι μπορείτε να αλλάξετε το όνομα του αρχείου.

Πρέπει να προσέξετε ότι η αλλαγή του ονόματος αρχείου δεν αλλάζει αυτόματα το όνομα της προβολής μέσα στον κώδικα.

Για παράδειγμα, αφού αλλάξετε το όνομα του αρχείου σε `ProfileView.swift`, μέσα στον κώδικα μπορεί να παραμένει ακόμα το `struct TestView: View`.

Αυτό δεν προκαλεί απαραίτητα σφάλμα, αλλά μπορεί εύκολα να δημιουργήσει σύγχυση.

Γι' αυτό, προτείνεται το όνομα του αρχείου και το όνομα της προβολής να παραμένουν συνεπή. Για παράδειγμα, αν το όνομα του αρχείου είναι `ProfileView.swift`, τότε ως όνομα προβολής χρησιμοποιείται επίσης το `ProfileView`.

## Δημιουργία αρχείου προτύπου SwiftUI

Εκτός από τη δημιουργία κενού αρχείου, μπορούμε επίσης να δημιουργήσουμε απευθείας ένα αρχείο προτύπου SwiftUI.

Στην περιοχή Navigator στα αριστερά του Xcode, κάντε δεξί κλικ σε ένα κενό σημείο ή σε έναν φάκελο και επιλέξτε `New File from Template...`.

Η θέση της ενέργειας είναι η εξής:

![xcode](../../../Resource/027_view3.png)

Στο παράθυρο που εμφανίζεται, μπορείτε να επιλέξετε διαφορετικές πλατφόρμες από την κορυφή, όπως `iOS`, `macOS` κ.λπ.

Εδώ αρκεί να διατηρήσουμε επιλεγμένη την πλατφόρμα που αντιστοιχεί στο τρέχον έργο. Για παράδειγμα, αν δημιουργούμε μια εφαρμογή iOS, επιλέγουμε `iOS`.

![xcode](../../../Resource/027_view4.png)

Στη συνέχεια, επιλέγουμε `SwiftUI View` στο κάτω μέρος.

![xcode](../../../Resource/027_view12.png)

Στη συνέχεια, κάντε κλικ στο `Next`, εισαγάγετε το όνομα αρχείου και αποθηκεύστε.

Για παράδειγμα, δημιουργούμε ένα `Test2View.swift`.

Το Xcode θα δημιουργήσει αυτόματα κώδικα παρόμοιο με τον παρακάτω:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

Σε σύγκριση με το κενό αρχείο, το αρχείο προτύπου περιλαμβάνει αυτόματα τα `import SwiftUI`, `struct`, `body` και `#Preview`.

Επομένως, για τους αρχάριους, η χρήση αρχείου προτύπου SwiftUI είναι πιο απλή και μειώνει την πιθανότητα να παραλειφθεί κώδικας.

**Συμβουλή**: Εκτός από τη δημιουργία αρχείου με δεξί κλικ, μπορείτε επίσης να κάνετε κλικ στο κουμπί συν στο κάτω μέρος της περιοχής Navigator στα αριστερά του Xcode για να δημιουργήσετε αρχείο ή φάκελο.

![xcode](../../../Resource/027_view5.png)

## Δημιουργία φακέλου

Όταν τα αρχεία γίνονται όλο και περισσότερα, μπορούμε να χρησιμοποιούμε φακέλους για τη διαχείρισή τους.

Στην περιοχή Navigator στα αριστερά του Xcode, κάντε δεξί κλικ σε ένα κενό σημείο ή σε έναν φάκελο και επιλέξτε `New Folder`.

Μετά τη δημιουργία του φακέλου, μπορείτε να σύρετε τα σχετικά αρχεία μέσα στον φάκελο.

![xcode](../../../Resource/027_view7.png)

Οι φάκελοι χρησιμοποιούνται κυρίως για την οργάνωση της δομής του έργου και δεν αλλάζουν τον τρόπο χρήσης της ίδιας της προβολής.

Εφόσον το όνομα της προβολής είναι σωστό, εξακολουθεί να μπορεί να χρησιμοποιείται σε άλλα αρχεία με τρόπους όπως `TestView()` και `Test2View()`.

## Διαγραφή αρχείου ή φακέλου

Αν θέλετε να διαγράψετε ένα αρχείο ή έναν φάκελο, μπορείτε να επιλέξετε το αντίστοιχο αρχείο στην περιοχή Navigator στα αριστερά του Xcode, έπειτα να κάνετε δεξί κλικ και να επιλέξετε `Delete`.

Μπορείτε επίσης να επιλέξετε το αρχείο και μετά να πατήσετε το πλήκτρο διαγραφής `⌫`.

Το Xcode θα εμφανίσει ένα παράθυρο επιβεβαίωσης διαγραφής.

![xcode](../../../Resource/027_view10.png)

Η επιλογή `Move to Trash` σημαίνει ότι το αρχείο θα μετακινηθεί στον Κάδο απορριμμάτων.

Αν δείτε `Remove Reference`, σημαίνει ότι αφαιρείται μόνο η αναφορά από το έργο Xcode, αλλά δεν διαγράφεται το αρχικό αρχείο από τον δίσκο.

Για τους αρχάριους, αν απλώς διαγράφετε ένα συνηθισμένο αρχείο που μόλις δημιουργήσατε, συνήθως αρκεί να επιλέξετε `Move to Trash`.

## Εμφάνιση πολλών προβολών SwiftUI

Αφού μάθουμε να δημιουργούμε αρχεία, μπορούμε να χωρίσουμε διαφορετικές προβολές σε διαφορετικά αρχεία.

Για παράδειγμα, δημιουργούμε δύο αρχεία SwiftUI:

```swift
AuthorView.swift
CardView.swift
```

Το `AuthorView` χρησιμοποιείται για την εμφάνιση του ονόματος του συγγραφέα:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

Το `CardView` χρησιμοποιείται για την εμφάνιση του φόντου της κάρτας:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

Έπειτα, μπορούμε να χρησιμοποιήσουμε αυτές τις δύο προβολές μέσα στο `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

Μετά την εκτέλεση, μπορούμε να δούμε ότι το `CardView` και το `AuthorView` εμφανίζονται μαζί ως συνδυασμός:

![xcode](../../../Resource/027_view11.png)

Σε αυτό το αποτέλεσμα, το `CardView` λειτουργεί ως φόντο και το `AuthorView` εμφανίζεται από πάνω, επειδή έχουν τοποθετηθεί μέσα σε ένα `ZStack`.

Στο `ContentView`, χρησιμοποιούμε αυτές τις δύο προβολές μέσω των `CardView()` και `AuthorView()`. Τα `()` μετά το όνομα σημαίνουν ότι δημιουργούμε και χρησιμοποιούμε αυτήν την προβολή.

Αυτή είναι η συνηθισμένη ένθεση προβολών στο SwiftUI: μέσα σε μία προβολή μπορούμε να συνεχίσουμε να χρησιμοποιούμε άλλες προβολές.

## Αρχείο εισόδου

Τέλος, ας επανεξετάσουμε από πού αρχίζει η εφαρμογή να εμφανίζει τη διεπαφή.

Σε ένα έργο SwiftUI, συνήθως υπάρχει ένα αρχείο εισόδου:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Το `@main` εδώ σημαίνει ότι αυτό είναι το σημείο εισόδου της εφαρμογής.

Μέσα στο `WindowGroup`, βλέπουμε:

```swift
ContentView()
```

Αυτό σημαίνει ότι μετά την εκκίνηση της εφαρμογής, εμφανίζεται πρώτα το `ContentView`.

Στη συνέχεια, μέσα στο `ContentView` εμφανίζεται επίσης:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Επομένως, η ιεραρχία των προβολών μπορεί να γίνει κατανοητή ως εξής:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

Αν αλλάξουμε την προβολή που εμφανίζεται στο αρχείο εισόδου σε `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

Τότε, μετά την εκκίνηση της εφαρμογής, θα εμφανίζεται μόνο το `CardView`.

Με άλλα λόγια, το `WindowGroup` στον κώδικα εισόδου καθορίζει ποια προβολή θα εμφανιστεί πρώτη μετά την εκκίνηση της εφαρμογής.

Και μέσα σε αυτήν την προβολή, μπορούν να συνεχίσουν να εμφανίζονται άλλες προβολές.

## Σύνοψη

Σε αυτό το μάθημα, μάθαμε τις συνηθισμένες λειτουργίες διαχείρισης αρχείων στο Xcode.

Μάθαμε πώς να δημιουργούμε αρχεία SwiftUI, πώς να μετονομάζουμε αρχεία, πώς να δημιουργούμε φακέλους και πώς να διαγράφουμε αρχεία.

Ταυτόχρονα, μάθαμε και μια σημαντική έννοια: οι προβολές SwiftUI μπορούν να χωριστούν σε διαφορετικά αρχεία και μετά να χρησιμοποιηθούν μέσα σε άλλες προβολές.

Για παράδειγμα:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Εδώ, τα `CardView()` και `AuthorView()` είναι η χρήση άλλων προβολών μέσα στο `ContentView`.

Τέλος, επανεξετάσαμε επίσης τη λογική εμφάνισης του αρχείου εισόδου.

Όταν ξεκινά η εφαρμογή, εισέρχεται πρώτα στον κώδικα εισόδου που έχει τη σήμανση `@main` και μετά εμφανίζει την προβολή μέσα στο `WindowGroup`.

Αυτό μας βοηθά να καταλάβουμε ότι τα διαφορετικά αρχεία SwiftUI δεν είναι απομονωμένα μεταξύ τους. Μπορούν να συνδυάζονται μεταξύ τους και τελικά να σχηματίζουν την πλήρη διεπαφή της εφαρμογής.
