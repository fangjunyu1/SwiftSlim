# Υπολογιζόμενες ιδιότητες

Σε αυτό το μάθημα θα μάθουμε κυρίως τις υπολογιζόμενες ιδιότητες.

Οι υπολογιζόμενες ιδιότητες χρησιμοποιούνται για να υπολογίζουν ένα νέο αποτέλεσμα με βάση υπάρχοντα δεδομένα. Μπορούν να υπολογίζουν αριθμητικές τιμές, αλλά και περιεχόμενο που πρέπει να εμφανίζεται σε SwiftUI views.

Για παράδειγμα:

```swift
let a = 10
let b = 20
let c = a + b
```

Εδώ το `c` αντιπροσωπεύει το αποτέλεσμα της πρόσθεσης του `a` και του `b`.

Σε κανονικό κώδικα, αυτός ο τρόπος γραφής είναι πολύ συνηθισμένος.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

Σημείωση: μέσα στο γεγονός πατήματος ενός `Button` μπορεί να εκτελεστεί κανονικός Swift κώδικας.

Όμως, αν γράψουμε παρόμοιο κώδικα απευθείας στις ιδιότητες ενός SwiftUI view, θα συναντήσουμε πρόβλημα.

Για παράδειγμα:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

Αυτός ο κώδικας θα εμφανίσει σφάλμα.

Φαίνεται ότι τα `a` και `b` έχουν ήδη γραφτεί πριν από το `c`, σαν να μπορεί να υπολογιστεί η τιμή του `c`.

Όμως μέσα στη δήλωση ιδιοτήτων μιας δομής, αυτή η γραφή δεν μπορεί να χρησιμοποιηθεί απευθείας.

## Γιατί το c δεν μπορεί να υπολογίσει απευθείας το a + b

Επειδή τα `a`, `b` και `c` δεν είναι προσωρινές σταθερές μέσα στο γεγονός πατήματος ενός κουμπιού, αλλά ιδιότητες του view `ContentView`.

Μέσα στο γεγονός πατήματος του κουμπιού, αυτός ο κώδικας μπορεί να εκτελεστεί κανονικά:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

Επειδή μετά το πάτημα του κουμπιού, ο κώδικας εκτελείται με τη σειρά. Πρώτα δημιουργείται το `a`, μετά δημιουργείται το `b`, και τέλος υπολογίζεται το `c` με `a + b`.

Όταν όμως δηλώνουμε ιδιότητες μέσα σε ένα view, η κατάσταση είναι διαφορετική:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

Εδώ τα `a`, `b` και `c` είναι όλα ιδιότητες του view `ContentView`.

Όταν δημιουργείται το view `ContentView`, η Swift πρέπει πρώτα να προετοιμάσει αυτές τις ιδιότητες. Για να διασφαλίσει ότι η διαδικασία δημιουργίας είναι ασφαλής, η Swift δεν επιτρέπει στην προεπιλεγμένη τιμή μιας αποθηκευμένης ιδιότητας στιγμιοτύπου να διαβάζει απευθείας άλλες ιδιότητες του ίδιου στιγμιοτύπου.

Γι' αυτό αυτή η γραμμή θα εμφανίσει σφάλμα:

```swift
let c = a + b
```

Μπορούμε να το καταλάβουμε απλά έτσι: **όταν δηλώνουμε ιδιότητες μέσα σε ένα view, δεν μπορούμε να χρησιμοποιήσουμε απευθείας μια κανονική ιδιότητα για να υπολογίσουμε μια άλλη κανονική ιδιότητα.**

Η ιδιότητα που αποθηκεύει απευθείας μια τιμή ονομάζεται «αποθηκευμένη ιδιότητα». Για ευκολότερη κατανόηση, μπορούμε προσωρινά να τη θεωρήσουμε κανονική ιδιότητα.

Για παράδειγμα:

```swift
let a = 10
```

Το `a` αποθηκεύει το `10`.

```swift
let b = 20
```

Το `b` αποθηκεύει το `20`.

Όμως:

```swift
let c = a + b
```

Εδώ το `c` δεν είναι μια τιμή γραμμένη απευθείας, αλλά θέλουμε να υπολογιστεί μέσω του `a + b`.

Σε μια τέτοια κατάσταση, όπου «παίρνουμε αποτέλεσμα με βάση υπάρχοντα δεδομένα», είναι πιο κατάλληλο να χρησιμοποιήσουμε υπολογιζόμενη ιδιότητα.

Μπορούμε να το αλλάξουμε έτσι:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

Εδώ το `c` είναι υπολογιζόμενη ιδιότητα.

```swift
var c: Int {
    return a + b
}
```

Σημαίνει: **όταν χρειάζεται να χρησιμοποιηθεί το `c`, τότε διαβάζονται τα `a` και `b` και επιστρέφεται το αποτέλεσμα του `a + b`.**

Για παράδειγμα:

```swift
Text("c: \(c)")
```

Όταν το `Text` εμφανίζει το `c`, τότε ενεργοποιείται ο υπολογισμός του `c`.

## Τι είναι μια υπολογιζόμενη ιδιότητα;

Μια υπολογιζόμενη ιδιότητα μοιάζει με μεταβλητή, αλλά η ίδια δεν αποθηκεύει δεδομένα.

Για παράδειγμα:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

Εδώ το `c` είναι υπολογιζόμενη ιδιότητα.

Δεν αποθηκεύει μια σταθερή τιμή όπως μια κανονική ιδιότητα, αλλά κάθε φορά που διαβάζεται το `c`, εκτελεί ξανά τον κώδικα μέσα στο `{}` και επιστρέφει το αποτέλεσμα του υπολογισμού.

Μπορούμε να το καταλάβουμε έτσι:

```swift
var c: Int {
    return a + b
}
```

Όταν χρειάζεται να χρησιμοποιήσουμε το `c`, υπολογίζεται το `a + b`.

Επομένως, οι υπολογιζόμενες ιδιότητες είναι κατάλληλες για αυτή την περίπτωση: **ένα αποτέλεσμα δεν χρειάζεται να αποθηκευτεί ξεχωριστά, αλλά μπορεί να υπολογιστεί από υπάρχοντα δεδομένα.**

## Βασική σύνταξη

Μια υπολογιζόμενη ιδιότητα συνήθως περιλαμβάνει τρία μέρη:

```swift
var c: Int {
    return a + b
}
```

### 1. Δήλωση με `var`

```swift
var c
```

Μια υπολογιζόμενη ιδιότητα πρέπει να δηλώνεται με `var`, δεν μπορεί να χρησιμοποιηθεί `let`.

Επειδή η υπολογιζόμενη ιδιότητα δεν είναι μια σταθερά αποθηκευμένη τιμή, αλλά ένα αποτέλεσμα που υπολογίζεται δυναμικά κάθε φορά που διαβάζεται.

### 2. Σήμανση του τύπου επιστροφής

```swift
var c: Int
```

Μια υπολογιζόμενη ιδιότητα πρέπει να δηλώνει τον τύπο που επιστρέφει.

Εδώ το `c` τελικά θα επιστρέψει έναν ακέραιο, άρα ο τύπος είναι `Int`.

### 3. Χρήση {} για τη λογική υπολογισμού

```swift
{
    return a + b
}
```

Μέσα στο `{}` γράφεται η λογική υπολογισμού· εδώ επιστρέφεται το `a + b`.

## Η λέξη-κλειδί return

Μια υπολογιζόμενη ιδιότητα πρέπει να επιστρέφει ένα αποτέλεσμα.

Για παράδειγμα:

```swift
var c: Int {
    return a + b
}
```

Εδώ το `return` σημαίνει: επέστρεψε προς τα έξω το αποτέλεσμα του υπολογισμού `a + b`.

Αν η υπολογιζόμενη ιδιότητα περιέχει μόνο μία έκφραση που παράγει άμεσα αποτέλεσμα, μπορούμε να παραλείψουμε το `return`:

```swift
var c: Int {
    a + b
}
```

Όμως, αν η υπολογιζόμενη ιδιότητα περιέχει πολλές γραμμές κώδικα, πρέπει να χρησιμοποιήσουμε `return` για να επιστρέψουμε καθαρά το αποτέλεσμα.

Για παράδειγμα:

```swift
var totalPriceText: String {
    let total = count * price
    return "Σύνολο: \(total) $"
}
```

Εδώ η διαδικασία υπολογισμού χωρίζεται σε δύο βήματα.

Πρώτο βήμα, πρώτα υπολογίζουμε τη συνολική τιμή:

```swift
let total = count * price
```

Δεύτερο βήμα, ενώνουμε τη συνολική τιμή σε κείμενο και την επιστρέφουμε:

```swift
return "Σύνολο: \(total) $"
```

Αν αφαιρέσουμε το `return`:

```swift
var totalPriceText: String {
    let total = count * price
    "Σύνολο: \(total) $"
}
```

Αυτός ο κώδικας θα εμφανίσει σφάλμα.

Ο λόγος είναι: **αυτή η υπολογιζόμενη ιδιότητα περιέχει ήδη πολλές γραμμές κώδικα, και η Swift δεν μπορεί πλέον να κρίνει αυτόματα ποια γραμμή είναι το τελικό αποτέλεσμα που πρέπει να επιστραφεί.**

Επομένως, όταν μια υπολογιζόμενη ιδιότητα έχει μόνο μία γραμμή αποτελέσματος, μπορούμε να παραλείψουμε το `return`.

```swift
var c: Int {
    a + b
}
```

Όταν μια υπολογιζόμενη ιδιότητα έχει πολλές γραμμές κώδικα, συνιστάται να γράφουμε καθαρά το `return`.

```swift
var totalPriceText: String {
    let total = count * price
    return "Σύνολο: \(total) $"
}
```

## Διαφορά ανάμεσα σε υπολογιζόμενη και κανονική ιδιότητα

Η κανονική ιδιότητα αποθηκεύει δεδομένα.

```swift
var c = 30
```

Εδώ το `c` αποθηκεύει μια συγκεκριμένη τιμή: `30`.

Η υπολογιζόμενη ιδιότητα δεν αποθηκεύει δεδομένα.

```swift
var c: Int {
    a + b
}
```

Εδώ το `c` δεν αποθηκεύει το `30`. Απλώς παρέχει έναν τρόπο υπολογισμού.

Όταν διαβάζεται το `c`, η Swift εκτελεί:

```swift
a + b
```

και στη συνέχεια επιστρέφει το αποτέλεσμα του υπολογισμού.

Επομένως, οι υπολογιζόμενες ιδιότητες είναι κατάλληλες για περιπτώσεις όπου το αποτέλεσμα υπολογίζεται από άλλα δεδομένα.

## Το body είναι επίσης υπολογιζόμενη ιδιότητα

Αφού μάθουμε τις υπολογιζόμενες ιδιότητες, μπορούμε να ξανακατανοήσουμε ένα από τα πιο συνηθισμένα κομμάτια κώδικα στο SwiftUI:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Εδώ το `body` είναι επίσης μια υπολογιζόμενη ιδιότητα.

Δηλώνεται με `var`:

```swift
var body
```

Έχει τύπο επιστροφής:

```swift
some View
```

Μέσα στο `{}` επιστρέφεται περιεχόμενο SwiftUI view:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

Επομένως, το `body` μπορεί να κατανοηθεί έτσι: όταν το SwiftUI χρειάζεται να εμφανίσει αυτό το view, διαβάζει το `body` και δημιουργεί τη διεπαφή σύμφωνα με το περιεχόμενο που επιστρέφει το `body`.

Αν γράψουμε `return`, μπορούμε να το καταλάβουμε έτσι:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Όμως στο SwiftUI συνήθως παραλείπουμε το `return` και γράφουμε απευθείας:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Όταν αλλάζουν τα δεδομένα `@State`, το SwiftUI διαβάζει ξανά το `body` και ενημερώνει τη διεπαφή σύμφωνα με τα νέα δεδομένα.

Για παράδειγμα:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("πλήθος: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

Μετά το πάτημα του κουμπιού, το `count` θα αλλάξει.

```swift
count += 1
```

Αφού αλλάξει το `count`, το SwiftUI θα υπολογίσει ξανά το `body`, οπότε και το κείμενο στη διεπαφή θα ενημερωθεί.

```swift
Text("πλήθος: \(count)")
```

Αυτός είναι και ο λόγος που στο SwiftUI η διεπαφή μπορεί να ανανεώνεται αυτόματα μετά από αλλαγές δεδομένων.

### Δεν συνιστάται να γράφουμε σύνθετους υπολογισμούς στο body

Εφόσον το `body` είναι υπολογιζόμενη ιδιότητα, μπορεί να διαβαστεί και να υπολογιστεί ξανά πολλές φορές.

Για παράδειγμα:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("αριθμός: \(num)")
            Text("όνομα: \(name)")
        }
    }
}
```

Όταν αλλάξει το `name`, το SwiftUI θα υπολογίσει ξανά το `body`.

Τότε θα εκτελεστεί ξανά και ο κώδικας μέσα στο `body`, και το `num` θα δημιουργηθεί επίσης ξανά.

```swift
let num = 10
```

Το `num` σε αυτό το παράδειγμα είναι πολύ απλό, επομένως η επίδραση δεν είναι μεγάλη.

Αν όμως μέσα στο `body` εκτελούνται σύνθετοι υπολογισμοί, όπως φιλτράρισμα μεγάλου όγκου δεδομένων, ταξινόμηση, επεξεργασία εικόνων κ.λπ., μπορεί να επηρεαστεί η ομαλότητα της διεπαφής.

Επομένως, στο SwiftUI το `body` πρέπει κυρίως να είναι υπεύθυνο για την περιγραφή της δομής της διεπαφής.

Απλά προσωρινά δεδομένα μπορούν να γραφτούν μέσα στο `body`.

Οι σύνθετοι υπολογισμοί μπορούν να τοποθετηθούν σε υπολογιζόμενες ιδιότητες έξω από το `body`, σε μεθόδους ή σε ξεχωριστή επεξεργασία δεδομένων.

## Παράδειγμα: ποσότητα και συνολική τιμή

Παρακάτω θα κατανοήσουμε τις υπολογιζόμενες ιδιότητες με ένα απλό παράδειγμα.

Ας υποθέσουμε ότι η τιμή μιας μονάδας καρότου είναι 2 $, ο χρήστης μπορεί να πατά κουμπιά για να αλλάζει την ποσότητα αγοράς και η διεπαφή πρέπει να εμφανίζει τη συνολική τιμή.

Ο τρόπος υπολογισμού της συνολικής τιμής:

```
ποσότητα * τιμή μονάδας
```

Αν χρησιμοποιήσουμε μια κανονική μεταβλητή για να αποθηκεύσουμε τη συνολική τιμή, θα είναι κάπως ενοχλητικό.

Επειδή κάθε φορά που αλλάζει η ποσότητα, πρέπει να ενημερώνουμε χειροκίνητα τη συνολική τιμή.

Καλύτερος τρόπος είναι να χρησιμοποιήσουμε υπολογιζόμενη ιδιότητα:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("τιμή: \(price) $")
                Text("ποσότητα: \(count)")
                Text("συνολική τιμή: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![view](../../Resource/024_view.png)

Σε αυτό το παράδειγμα:

```swift
@State private var count = 1
```

Το `count` αντιπροσωπεύει την ποσότητα των καρότων. Όταν πατηθεί το κουμπί, το `count` θα αλλάξει.

```swift
private let price = 2
```

Το `price` αντιπροσωπεύει την τιμή μονάδας του καρότου. Εδώ είναι σταθερή τιμή, άρα χρησιμοποιούμε `let`.

```swift
private var totalPrice: Int {
    count * price
}
```

Το `totalPrice` αντιπροσωπεύει τη συνολική τιμή.

Δεν χρειάζεται να αποθηκευτεί ξεχωριστά, επειδή η συνολική τιμή μπορεί πάντα να υπολογιστεί με `count * price`.

Όταν το `count` είναι `1`:

```swift
totalPrice = 1 * 2
```

Το αποτέλεσμα εμφάνισης είναι:

```swift
συνολική τιμή: 2 $
```

Μετά το πάτημα του κουμπιού `+`, το `count` γίνεται `2`.

Τότε, όταν διαβαστεί ξανά το `totalPrice`, θα υπολογιστεί εκ νέου:

```swift
totalPrice = 2 * 2
```

Το αποτέλεσμα εμφάνισης είναι:

```swift
συνολική τιμή: 4 $
```

Αυτός είναι ο ρόλος της υπολογιζόμενης ιδιότητας: να υπολογίζει δυναμικά νέο αποτέλεσμα με βάση υπάρχοντα δεδομένα.

## Οι υπολογιζόμενες ιδιότητες μπορούν να χρησιμοποιηθούν για έλεγχο

Οι υπολογιζόμενες ιδιότητες δεν μπορούν μόνο να υπολογίζουν αριθμητικές τιμές, αλλά μπορούν επίσης να επιστρέφουν αποτελέσματα ελέγχου.

Για παράδειγμα, όταν θέλουμε η ελάχιστη ποσότητα να είναι 1.

Όταν η ποσότητα είναι ήδη 1, το κουμπί `-` δεν πρέπει να συνεχίσει να μειώνει.

Μπορούμε να προσθέσουμε μια υπολογιζόμενη ιδιότητα:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Πλήρης κώδικας:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("τιμή: \(price) $")
                Text("ποσότητα: \(count)")
                Text("συνολική τιμή: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![view](../../Resource/024_view1.png)

Εδώ:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Αυτός ο κώδικας σημαίνει αν μπορεί να συνεχιστεί η μείωση αυτή τη στιγμή.

Όταν το `count` είναι μεγαλύτερο από `1`:

```swift
canDecrease == true
```

σημαίνει ότι μπορεί να γίνει μείωση.

Όταν το `count` είναι ίσο με `1`:

```swift
canDecrease == false
```

σημαίνει ότι δεν μπορεί να συνεχιστεί η μείωση.

### Έλεγχος συνθήκης μέσα στο κουμπί

Στο κουμπί χρησιμοποιείται:

```swift
if canDecrease {
    count -= 1
}
```

Μόνο όταν το `canDecrease` αξιολογείται ως `true`, μπορεί να μειωθεί το `count`.

### Έλεγχος του χρώματος προσκηνίου του view

Μπορούμε επίσης να χρησιμοποιήσουμε το `canDecrease` για να ελέγξουμε το χρώμα προσκηνίου του κουμπιού:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Εδώ χρησιμοποιείται ο τριαδικός τελεστής:

```swift
canDecrease ? Color.primary : Color.gray
```

Η σημασία αυτού του κώδικα είναι: αν το `canDecrease` είναι `true`, το χρώμα προσκηνίου χρησιμοποιεί `Color.primary`; αν το `canDecrease` είναι `false`, το χρώμα προσκηνίου χρησιμοποιεί `Color.gray`.

Το `Color.primary` είναι ένα συστημικό σημασιολογικό χρώμα που παρέχεται από το SwiftUI και αντιπροσωπεύει το κύριο χρώμα κειμένου στην τρέχουσα διεπαφή.

Σε φωτεινή λειτουργία, το `Color.primary` είναι συνήθως κοντά στο μαύρο· σε σκοτεινή λειτουργία, το `Color.primary` είναι συνήθως κοντά στο λευκό.

Επομένως, το πλεονέκτημα της χρήσης του `Color.primary` είναι ότι προσαρμόζεται αυτόματα στη φωτεινή και στη σκοτεινή λειτουργία.

### Έλεγχος της απενεργοποιημένης κατάστασης του view

Το `disabled` χρησιμοποιείται για να ελέγχει αν ένα view βρίσκεται σε απενεργοποιημένη κατάσταση:

```swift
.disabled(!canDecrease)
```

Όταν το `disabled` είναι `false`, το view μπορεί να πατηθεί.

Όταν το `disabled` είναι `true`, το view είναι απενεργοποιημένο και δεν μπορεί να πατηθεί.

Εδώ η χρήση του `canDecrease` ως συνθήκη κάνει τον κώδικα πιο εύκολο στην κατανόηση.

Όταν βλέπουμε `canDecrease`, ξέρουμε ότι σημαίνει «αν μπορεί να συνεχιστεί η μείωση αυτή τη στιγμή».

### Συμπληρωματική εξήγηση: γιατί υπάρχουν δύο έλεγχοι;

Στο κουμπί `-`:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Εδώ χρησιμοποιείται και το `.disabled(!canDecrease)` και το `if canDecrease` μέσα στο κουμπί.

Το `.disabled(!canDecrease)` είναι υπεύθυνο για την απενεργοποίηση του κουμπιού στη διεπαφή, ώστε ο χρήστης να μην μπορεί να το πατήσει.

Το `if canDecrease` είναι υπεύθυνο για έναν δεύτερο έλεγχο πριν από την εκτέλεση του κώδικα και εκτελεί το `count -= 1` μόνο όταν μπορεί να γίνει μείωση.

Αυτό είναι διπλή προστασία. Στην πραγματική ανάπτυξη, αν το κουμπί είναι ήδη απενεργοποιημένο, ο εσωτερικός έλεγχος μπορεί να παραλειφθεί· όμως σε ένα διδακτικό παράδειγμα τον κρατάμε, ώστε ο ρόλος του `canDecrease` να είναι πιο ξεκάθαρος.

## Σύνοψη

Σε αυτό το μάθημα μάθαμε κυρίως τις υπολογιζόμενες ιδιότητες.

Μια υπολογιζόμενη ιδιότητα δεν αποθηκεύει απευθείας τιμή, αλλά όταν διαβάζεται υπολογίζει ένα αποτέλεσμα με βάση υπάρχοντα δεδομένα.

Για παράδειγμα:

```swift
var c: Int {
    a + b
}
```

Εδώ το `c` δεν χρειάζεται να αποθηκευτεί ξεχωριστά, επειδή μπορεί να υπολογιστεί μέσω του `a + b`.

Η υπολογιζόμενη ιδιότητα πρέπει να δηλώνεται με `var` και πρέπει να δηλώνει τύπο επιστροφής.

```swift
var canDecrease: Bool {
    count > 1
}
```

Οι υπολογιζόμενες ιδιότητες μπορούν να επιστρέφουν όχι μόνο αριθμητικές τιμές, αλλά και αποτελέσματα ελέγχου, περιεχόμενο κειμένου, ακόμη και περιεχόμενο SwiftUI views.

Σε αυτό το μάθημα μάθαμε επίσης το `return`.

Το `return` σημαίνει επιστροφή ενός αποτελέσματος:

```swift
var totalPriceText: String {
    let total = count * price
    return "Σύνολο: \(total) $"
}
```

Αν μια υπολογιζόμενη ιδιότητα περιέχει μόνο μία έκφραση που παράγει άμεσα αποτέλεσμα, μπορούμε να παραλείψουμε το `return`.

```swift
var totalPrice: Int {
    count * price
}
```

Επιπλέον, γνωρίσαμε επίσης το `Color.primary` και το `disabled`.

Το `Color.primary` είναι συστημικό σημασιολογικό χρώμα του SwiftUI και προσαρμόζει αυτόματα το αποτέλεσμα εμφάνισης ανάλογα με τη φωτεινή και τη σκοτεινή λειτουργία.

```swift
.foregroundStyle(Color.primary)
```

Το `disabled` χρησιμοποιείται για να ελέγχει αν ένα view είναι απενεργοποιημένο.

```swift
.disabled(true)
```

Σημαίνει απενεργοποιημένο, δεν μπορεί να πατηθεί.

```swift
.disabled(false)
```

Σημαίνει διαθέσιμο, μπορεί να πατηθεί.

Επομένως, οι υπολογιζόμενες ιδιότητες είναι πολύ συνηθισμένες στο SwiftUI. Μας βοηθούν να οργανώνουμε πιο καθαρά τα αποτελέσματα υπολογισμών, τις συνθήκες ελέγχου και το περιεχόμενο εμφάνισης.
