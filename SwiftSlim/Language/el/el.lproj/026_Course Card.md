# Κάρτα μαθήματος

Αυτό το μάθημα είναι ένα μάθημα επανάληψης σταδίου. Θα ολοκληρώσουμε μια «κάρτα μαθήματος».

Μέσα από αυτή την άσκηση, μπορούμε να επαναλάβουμε όσα μάθαμε προηγουμένως, όπως `Text`, `Image`, `VStack`, `HStack`, `Spacer()`, `.padding()`, `.background()`, `.cornerRadius()` και άλλα, και να μάθουμε πώς να συνδυάζουμε πολλά μικρά views σε μια ολοκληρωμένη ενότητα προβολής.

Εμφάνιση της κάρτας μαθήματος:

![view](../../../Resource/026_view.png)

Αυτή η κάρτα μαθήματος περιλαμβάνει κυρίως τα εξής μέρη:

- Επίπεδο μαθήματος: `Αρχάριος`
- Περιεχόμενο μαθήματος: `20+ Μαθήματα`
- Τίτλος μαθήματος: `Εισαγωγικό μάθημα SwiftUI`
- Περιγραφή μαθήματος
- Κουμπί: `Ξεκινήστε τη μάθηση`
- Φόντο μαθήματος: εικονίδιο Swift και φόντο indigo

Μπορούμε να ολοκληρώσουμε αυτό το view μέσα στο `ContentView`.

## Επάνω περιοχή

Πρώτα, φτιάχνουμε την επάνω περιοχή της κάρτας μαθήματος.

![view](../../../Resource/026_view1.png)

Στην αριστερή πλευρά της επάνω περιοχής εμφανίζεται το επίπεδο του μαθήματος και στη δεξιά πλευρά εμφανίζεται το περιεχόμενο του μαθήματος.

Επειδή αυτά τα δύο στοιχεία είναι τοποθετημένα οριζόντια, μπορούμε να χρησιμοποιήσουμε `HStack`.

```swift
var topView: some View {
    HStack {
        Text("Αρχάριος")
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
            .padding(.vertical, 10)
            .padding(.horizontal, 16)
            .background(Color.white.opacity(0.15))
            .cornerRadius(20)

        Spacer()

        Text("20+ Μαθήματα")
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

Εδώ χωρίζουμε την επάνω περιοχή σε μια υπολογιζόμενη ιδιότητα:

```swift
var topView: some View
```

Το πλεονέκτημα αυτής της προσέγγισης είναι ότι κάνει το `body` πιο καθαρό.

Η επάνω περιοχή χρησιμοποιεί κυρίως τα εξής:

- `HStack`: κάνει τα δύο κείμενα να τοποθετούνται οριζόντια.
- `Text`: εμφανίζει περιεχόμενο κειμένου.
- `Spacer()`: σπρώχνει το αριστερό κείμενο προς τα αριστερά και το δεξί κείμενο προς τα δεξιά.
- `.font(.footnote)`: ορίζει ένα μικρότερο στυλ κειμένου.
- `.fontWeight(.bold)`: κάνει το κείμενο έντονο.
- `.foregroundStyle(Color.white)`: ορίζει το προσκήνιο σε λευκό.
- `.padding(.vertical, 10)`: ορίζει το εσωτερικό περιθώριο επάνω και κάτω.
- `.padding(.horizontal, 16)`: ορίζει το εσωτερικό περιθώριο αριστερά και δεξιά.
- `.background(Color.white.opacity(0.15))`: ορίζει ένα ημιδιαφανές λευκό φόντο.
- `.cornerRadius(20)`: ορίζει στρογγυλεμένες γωνίες.

Το `Αρχάριος` χρησιμοποιεί `.padding()` σε δύο κατευθύνσεις:

```swift
.padding(.vertical, 10)
.padding(.horizontal, 16)
```

Έτσι, το κείμενο έχει χώρο τόσο επάνω και κάτω όσο και αριστερά και δεξιά, και μοιάζει περισσότερο με ετικέτα.

`Color.white.opacity(0.15)` σημαίνει λευκό με αδιαφάνεια `15%`, δηλαδή ένα πολύ απαλό λευκό.

### Προσθήκη φόντου κάρτας

Πριν δημιουργήσαμε το `topView`, αλλά η απλή δημιουργία ενός view δεν το εμφανίζει αυτόματα.

Στο SwiftUI, το περιεχόμενο που εμφανίζεται πραγματικά στην οθόνη πρέπει να γραφτεί μέσα στο `body`.

Επομένως, μπορούμε πρώτα να βάλουμε το `topView` μέσα στο `body` για εμφάνιση:

```swift
struct ContentView: View {
    var body: some View {
        topView
    }
}
```

Τώρα θα δούμε ότι η επάνω περιοχή έχει ήδη εμφανιστεί:

![view](../../../Resource/026_view2.png)

Ωστόσο, επειδή το κείμενο στο `topView` είναι λευκό και το προεπιλεγμένο φόντο είναι επίσης ανοιχτόχρωμο, στη φωτεινή λειτουργία μπορεί να μην φαίνεται καθαρά.

Γι' αυτό μπορούμε να προσθέσουμε εξωτερικά ένα `VStack` και στη συνέχεια να ορίσουμε για ολόκληρη την περιοχή εσωτερικό περιθώριο, χρώμα φόντου και στρογγυλεμένες γωνίες:

```swift
var body: some View {
	VStack {
	    topView
	}
	.padding(20)
	.background(Color.indigo)
	.cornerRadius(20)
	.padding(30)
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view3.png)

Το `VStack` εδώ περιέχει προς το παρόν μόνο ένα `topView`, οπότε φαίνεται σαν να μην είναι απαραίτητο.

Αλλά στη συνέχεια θα προσθέσουμε τον τίτλο του μαθήματος, την περιγραφή του μαθήματος και την περιοχή κουμπιού. Όλα αυτά είναι περιεχόμενα που τοποθετούνται κάθετα, γι' αυτό εδώ χρησιμοποιούμε εκ των προτέρων `VStack`, ώστε να είναι πιο εύκολο να συνεχίσουμε να συνδυάζουμε views.

Εδώ υπάρχουν δύο `.padding()`, και οι ρόλοι τους είναι διαφορετικοί.

Το πρώτο `.padding(20)` γράφεται πριν από το `.background()`:

```swift
.padding(20)
.background(Color.indigo)
```

Ελέγχει το εσωτερικό περιθώριο της κάρτας, δηλαδή την απόσταση ανάμεσα στο περιεχόμενο και την άκρη του φόντου.

Το δεύτερο `.padding(30)` γράφεται μετά από το `.background()`:

```swift
.background(Color.indigo)
.cornerRadius(20)
.padding(30)
```

Ελέγχει το εξωτερικό περιθώριο της κάρτας, δηλαδή την απόσταση ανάμεσα σε ολόκληρη την κάρτα και την άκρη της οθόνης.

![view](../../../Resource/026_view4.png)

Άρα, ακόμη και το ίδιο `.padding()` μπορεί να έχει διαφορετικό αποτέλεσμα όταν γράφεται σε διαφορετική θέση.

## Περιοχή περιεχομένου

Στη συνέχεια φτιάχνουμε την περιοχή περιεχομένου της κάρτας μαθήματος.

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view5.png)

Η περιοχή περιεχομένου περιλαμβάνει τον τίτλο και την περιγραφή του μαθήματος. Είναι τοποθετημένα κάθετα, οπότε χρησιμοποιούμε `VStack`.

```swift
var contentView: some View {
    VStack(alignment: .leading, spacing: 10) {
        Text("Εισαγωγικό μάθημα SwiftUI")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)

        Text("Ξεκινήστε από το μηδέν με το SwiftUI και μάθετε συστηματικά τη δηλωτική ανάπτυξη UI της Apple. Μέσα από καθαρές εξηγήσεις και πρακτικά παραδείγματα, θα κατακτήσετε σταδιακά τη διάταξη, την αλληλεπίδραση και τη διαχείριση κατάστασης, ώστε να δημιουργείτε όμορφα και χρήσιμα περιβάλλοντα εφαρμογών.")
            .font(.footnote)
            .fontWeight(.medium)
            .foregroundStyle(Color.white)
            .lineLimit(3)
    }
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view6.png)

Εδώ στο `VStack` χρησιμοποιούμε δύο παραμέτρους:

```swift
VStack(alignment: .leading, spacing: 10)
```

`alignment: .leading` σημαίνει ότι τα views μέσα στο `VStack` ευθυγραμμίζονται αριστερά.

`spacing: 10` σημαίνει ότι τα views μέσα στο `VStack` κρατούν απόσταση `10 pt` μεταξύ τους.

Έτσι, ο τίτλος και η περιγραφή του μαθήματος θα ξεκινούν από την αριστερή πλευρά και θα διατηρούν μια ορισμένη απόσταση μεταξύ τους.

Η περιγραφή του μαθήματος είναι σχετικά μεγάλη. Αν εμφανιστεί ολόκληρη, η κάρτα θα γίνει πολύ ψηλή, οπότε εδώ χρησιμοποιούμε `.lineLimit()` για να περιορίσουμε τον αριθμό των γραμμών που εμφανίζονται:

```swift
.lineLimit(3)
```

Αυτό σημαίνει ότι εμφανίζονται έως `3` γραμμές. Όταν το περιεχόμενο ξεπερνά αυτό το όριο, το επιπλέον μέρος παραλείπεται.

### Αριστερή στοίχιση του view

Εδώ χρειάζεται να προσέξουμε ακόμη ένα σημείο.

Το `VStack(alignment: .leading)` μπορεί μόνο να ελέγχει τη στοίχιση των views μέσα στο `VStack`. Δεν ελέγχει άμεσα τη θέση του ίδιου του `VStack` μέσα στο εξωτερικό container.

Για παράδειγμα, αν η περιγραφή του μαθήματος είναι σχετικά σύντομη:

```swift
VStack(alignment: .leading, spacing: 10) {
    Text("Εισαγωγικό μάθημα SwiftUI")
        .font(.title3)
        .fontWeight(.bold)
        .foregroundStyle(Color.white)

    Text("Απλώς κάποιο περιεχόμενο.")
        .font(.footnote)
        .fontWeight(.medium)
        .foregroundStyle(Color.white)
        .lineLimit(3)
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view7.png)

Μπορούμε να δούμε ότι, παρόλο που ο τίτλος και η περιγραφή του μαθήματος είναι αριστερά στοιχισμένα μέσα στο `VStack`, ολόκληρο το `VStack` δεν έχει κολλήσει στην πιο αριστερή πλευρά της κάρτας.

Αυτό συμβαίνει επειδή, όταν το περιεχόμενο είναι σχετικά σύντομο, το πλάτος του `VStack` γίνεται επίσης σχετικά στενό. Κατά τη διάταξη, το εξωτερικό container μπορεί να τοποθετήσει αυτό το στενότερο `VStack` στο κέντρο.

Μπορούμε να το καταλάβουμε έτσι: το `VStack(alignment: .leading)` είναι υπεύθυνο για την εσωτερική αριστερή στοίχιση.
Αλλά δεν είναι υπεύθυνο για το να σπρώξει ολόκληρη την περιοχή περιεχομένου στην πιο αριστερή πλευρά.

Αν θέλουμε ολόκληρη η περιοχή περιεχομένου να βρίσκεται πραγματικά αριστερά, μπορούμε να την τυλίξουμε εξωτερικά με ένα ακόμη `HStack` και να προσθέσουμε `Spacer()`:

```swift
var contentView: some View {
    HStack {
        VStack(alignment: .leading, spacing: 10) {
            Text("Εισαγωγικό μάθημα SwiftUI")
                .font(.title3)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)

            Text("Απλώς κάποιο περιεχόμενο.")
                .font(.footnote)
                .fontWeight(.medium)
                .foregroundStyle(Color.white)
                .lineLimit(3)
        }

        Spacer()
    }
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view8.png)

Το `HStack` εδώ είναι οριζόντια διάταξη.

Το `Spacer()` θα καταλάβει τον υπόλοιπο χώρο στη δεξιά πλευρά, σπρώχνοντας έτσι το αριστερό `VStack` στην πιο αριστερή πλευρά.

Επομένως, το `HStack + Spacer()` μπορεί να χρησιμοποιηθεί για να ελέγξουμε τη θέση ολόκληρης της περιοχής περιεχομένου.

Στην πραγματική κάρτα μαθήματος, η περιγραφή του μαθήματος είναι σχετικά μεγάλη και συνήθως καταλαμβάνει περισσότερο πλάτος, οπότε αυτό το πρόβλημα δεν είναι απαραίτητα εμφανές. Όμως η κατανόηση αυτής της διαφοράς είναι σημαντική, επειδή όταν φτιάχνουμε διατάξεις με σύντομο κείμενο αργότερα, θα συναντάμε συχνά παρόμοιες περιπτώσεις.

### Έλεγχος της απόστασης ανάμεσα στα εσωτερικά views

Προηγουμένως γράψαμε `spacing: 10` μέσα στο `VStack`:

```swift
VStack(alignment: .leading, spacing: 10) {
    ...
}
```

Το `spacing` χρησιμοποιείται για τον έλεγχο της απόστασης ανάμεσα στα εσωτερικά views.

Αν δεν ορίσουμε `spacing`, το `VStack` θα έχει επίσης προεπιλεγμένη απόσταση, αλλά αυτή η προεπιλεγμένη απόσταση δεν ταιριάζει απαραίτητα με το αποτέλεσμα που θέλουμε.

Εδώ, αν ο τίτλος και η περιγραφή του μαθήματος είναι πολύ κοντά, η διάταξη θα φαίνεται στριμωγμένη, γι' αυτό χρησιμοποιούμε:

```swift
spacing: 10
```

ώστε να αφήσουμε λίγο χώρο μεταξύ τους.

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view9.png)

Παρομοίως, η επάνω περιοχή και η περιοχή περιεχομένου είναι επίσης τοποθετημένες κάθετα, οπότε μπορούμε επίσης να χρησιμοποιήσουμε `spacing` για να ελέγξουμε την απόσταση μεταξύ τους.

Τώρα βάζουμε το `topView` και το `contentView` μέσα στο `body`:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            topView
            contentView
        }
        .padding(20)
        .background(Color.indigo)
        .cornerRadius(20)
        .padding(30)
    }
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view10.png)

Το `VStack(spacing: 20)` εδώ ελέγχει την απόσταση ανάμεσα στο `topView` και το `contentView`.

Δηλαδή:

```swift
VStack(alignment: .leading, spacing: 10)
```

ελέγχει την απόσταση ανάμεσα στον τίτλο και την περιγραφή του μαθήματος.

```swift
VStack(spacing: 20)
```

ελέγχει την απόσταση ανάμεσα στην επάνω περιοχή και την περιοχή περιεχομένου.

Και τα δύο είναι `spacing`, αλλά εφαρμόζονται σε διαφορετικά `VStack`, οπότε επηρεάζουν διαφορετική περιοχή.

Έτσι ολοκληρώνεται το view της περιοχής περιεχομένου.

## Περιοχή κουμπιού

Στη συνέχεια φτιάχνουμε την κάτω περιοχή κουμπιού.

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view11.png)

Η περιοχή κουμπιού αποτελείται κυρίως από ένα εικονίδιο αναπαραγωγής και ένα κείμενο.

Μπορούμε να χρησιμοποιήσουμε `Image` για να εμφανίσουμε το εικονίδιο αναπαραγωγής και `Text` για να εμφανίσουμε το κείμενο του κουμπιού.

Εδώ φτιάχνουμε πρώτα την εμφάνιση ενός κουμπιού:

```swift
var buttonView: some View {
    HStack {
        Button {
            print("κλικ στο κουμπί")
        } label: {
            HStack {
                Image(systemName: "play.fill")

                Text("Ξεκινήστε τη μάθηση")
                    .font(.subheadline)
                    .fontWeight(.bold)
            }
            .foregroundStyle(Color.indigo)
            .padding(.vertical, 10)
            .padding(.horizontal, 16)
            .background(Color.white)
            .cornerRadius(16)
        }
        .buttonStyle(.plain)

        Spacer()
    }
}
```

Εμφάνιση αποτελέσματος:

![view](../../../Resource/026_view12.png)

Εδώ χρησιμοποιούνται δύο `HStack`:

Το εσωτερικό `HStack` χρησιμοποιείται για να τοποθετεί οριζόντια το εικονίδιο και το κείμενο:

```swift
HStack {
    Image(systemName: "play.fill")
    Text("Ξεκινήστε τη μάθηση")
}
```

Το εξωτερικό `HStack`, μαζί με το `Spacer()`, χρησιμοποιείται για να εμφανίζεται αυτή η εμφάνιση κουμπιού αριστερά:

```swift
HStack {
    ...
    Spacer()
}
```

Όταν πατάμε το κουμπί, η κονσόλα θα εμφανίσει `κλικ στο κουμπί`.

## Φόντο με εικονίδιο Swift

Τέλος, προσθέτουμε στην κάρτα ένα ημιδιαφανές φόντο με εικονίδιο Swift.

![view](../../../Resource/026_view.png)

Πριν χρησιμοποιήσαμε το `Color.indigo` ως χρώμα φόντου. Στην πραγματικότητα, το `.background()` εκτός από χρώμα μπορεί επίσης να προσθέσει ένα ολοκληρωμένο view.

Επομένως, μπορούμε πρώτα να δημιουργήσουμε ένα ειδικό view φόντου:

```swift
var backgroundView: some View {
    VStack {
        Spacer()

        HStack {
            Spacer()

            Image(systemName: "swift")
                .font(.system(size: 100))
                .foregroundStyle(Color.white.opacity(0.15))
        }
    }
    .padding(30)
}
```

Σε αυτό το view φόντου, χρησιμοποιούμε `VStack`, `HStack` και `Spacer()` για να σπρώξουμε το εικονίδιο Swift στην κάτω δεξιά γωνία.

Επειδή το εικονίδιο Swift χρησιμεύει μόνο ως διακοσμητικό φόντο, εδώ χρησιμοποιούμε ημιδιαφανές λευκό:

```swift
Color.white.opacity(0.15)
```

Έτσι το εικονίδιο δεν είναι υπερβολικά έντονο και δεν επηρεάζει το μπροστινό περιεχόμενο κειμένου.

Στη συνέχεια προσθέτουμε το `backgroundView` στην κάρτα:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            topView
            contentView
            buttonView
        }
        .padding(20)
        .background(backgroundView)
        .background(Color.indigo)
        .cornerRadius(20)
        .padding(30)
    }
}
```

Υλοποιημένο αποτέλεσμα:

![view](../../../Resource/026_view14.png)

Εδώ χρησιμοποιούνται διαδοχικά δύο `.background()`:

```swift
.background(backgroundView)
.background(Color.indigo)
```

Το πρώτο `.background()` χρησιμοποιείται για να προσθέσει το φόντο με το εικονίδιο Swift.
Το δεύτερο `.background()` χρησιμοποιείται για να προσθέσει το indigo φόντο.

Έτσι, το εικονίδιο Swift θα εμφανιστεί πάνω από το indigo φόντο και θα εμφανίζεται ως μέρος του φόντου της κάρτας.

Από εδώ μπορούμε να δούμε ότι το `.background()` εκτός από χρώμα μπορεί επίσης να προσθέσει προσαρμοσμένα views. Πολλαπλά `.background()` μπορούν επίσης να συνδυαστούν, ώστε να υλοποιηθούν πιο πλούσια εφέ φόντου.

Μέχρι εδώ, μια πλήρης κάρτα μαθήματος έχει ολοκληρωθεί.

## Σύνοψη

Σε αυτό το μάθημα, μέσα από μια κάρτα μαθήματος, επαναλάβαμε συνηθισμένους βασικούς τρόπους διάταξης στο SwiftUI.

Χρησιμοποιήσαμε `Text` για την εμφάνιση κειμένου, `Image` για την εμφάνιση εικονιδίων συστήματος, και `VStack`, `HStack` και `Spacer()` για τον έλεγχο της διάταξης των views.

Μάθαμε το `.lineLimit()`, το οποίο μπορεί να περιορίσει τον μέγιστο αριθμό γραμμών που εμφανίζει ένα κείμενο. Όταν το περιεχόμενο ξεπερνά το όριο, το επιπλέον μέρος παραλείπεται.

Παράλληλα, επαναλάβαμε συνηθισμένους modifiers όπως `.padding()`, `.background()`, `.cornerRadius()` και `.foregroundStyle()`.

Μετά την ολοκλήρωση αυτού του μαθήματος, μπορούμε ήδη να συνδυάζουμε πολλά μικρά views σε μια ολοκληρωμένη ενότητα κάρτας.

Αυτή είναι επίσης μια πολύ συνηθισμένη ιδέα στην ανάπτυξη με SwiftUI: πρώτα χωρίζουμε σε μικρά views και μετά τα συνδυάζουμε σε μια ολοκληρωμένη διεπαφή.

## Πλήρης κώδικας

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
```
