# Προβολή σύνδεσης

Σε αυτό το μάθημα, θα μάθουμε ένα πολύ πρακτικό σενάριο: την προβολή σύνδεσης.

Είτε πρόκειται για ιστότοπο είτε για εφαρμογή, κατά τη χρήση πολλές φορές χρειάζεται ο χρήστης να συνδεθεί στον λογαριασμό του και να εισαγάγει κωδικό πρόσβασης.

Για παράδειγμα, η σελίδα σύνδεσης του GitHub:

![WordPress](../../Resource/019_github.png)

Σε αυτό το μάθημα, θα δημιουργήσουμε μια παρόμοια προβολή σύνδεσης, ώστε ο χρήστης να εισάγει χειροκίνητα λογαριασμό και κωδικό πρόσβασης, και θα ελέγξουμε αν υπάρχει κάποιο πρόβλημα στο περιεχόμενο που εισήχθη.

## Επάνω διάταξη

Θα γράψουμε τον κώδικα της προβολής σύνδεσης στο αρχείο `ContentView`.

Αρχικά, ας γράψουμε την περιοχή σήμανσης στο επάνω μέρος της προβολής σύνδεσης. Μπορείς πρώτα να ετοιμάσεις μια κατάλληλη εικόνα εικονιδίου και να τη βάλεις στον φάκελο `Assets`.

![icon](../../Resource/019_icon1.png)

Στη συνέχεια, χρησιμοποίησε το `Image` και τροποποιητές για να εμφανίσεις την εικόνα:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

Αυτοί οι τροποποιητές σημαίνουν:

- `resizable()`: επιτρέπει στην εικόνα να αλλάζει μέγεθος.
- `scaledToFit()`: κάνει κλιμάκωση διατηρώντας την αρχική αναλογία της εικόνας.
- `frame(width: 100)`: ορίζει το πλάτος εμφάνισης της εικόνας σε `100`.

Στη συνέχεια, χρησιμοποίησε το `Text` και τροποποιητές για να εμφανίσεις τον τίτλο σύνδεσης:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

Έπειτα, χρησιμοποίησε το `VStack` για να τοποθετήσεις την εικόνα και το κείμενο κάθετα:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

Αποτέλεσμα:

![view](../../Resource/019_view.png)

Μέχρι εδώ, έχουμε ολοκληρώσει την περιοχή τίτλου στο επάνω μέρος της προβολής σύνδεσης.

Αυτή τη στιγμή, η εικόνα και ο τίτλος βρίσκονται από προεπιλογή στη μεσαία περιοχή της συνολικής διάταξης.

Αν θέλουμε να μοιάζουν περισσότερο με "σήμανση σύνδεσης στο επάνω μέρος της σελίδας", μπορούμε να χρησιμοποιήσουμε το `Spacer()` για να απλώσουμε τον υπόλοιπο χώρο και να εμφανιστεί το περιεχόμενο πιο κοντά στην κορυφή.

```swift

        .resizable()
        .scaledToFit()

    Text("Sign in to App")

        .fontWeight(.bold)

}
```

## Όνομα χρήστη και κωδικός πρόσβασης

Μια σελίδα σύνδεσης συνήθως απαιτεί από τον χρήστη να εισαγάγει όνομα χρήστη και κωδικό πρόσβασης.

Στο SwiftUI, μπορούμε να χρησιμοποιήσουμε το `TextField` για να δεχόμαστε το περιεχόμενο που εισάγει ο χρήστης.

Όμως πρέπει να προσέξουμε: το `TextField` από μόνο του δεν αποθηκεύει μακροπρόθεσμα τα δεδομένα εισόδου, είναι απλώς ένα στοιχείο εισαγωγής. Αυτό που αποθηκεύει πραγματικά αυτά τα δεδομένα είναι οι μεταβλητές που του κάνουμε bind.

Επομένως, πρώτα πρέπει να δημιουργήσουμε δύο μεταβλητές `@State` για να αποθηκεύσουμε το όνομα χρήστη και τον κωδικό πρόσβασης:

```swift
@State private var user = ""
@State private var password = ""
```

Όταν αλλάζει η τιμή μιας μεταβλητής `@State`, το SwiftUI ανανεώνει αυτόματα τις σχετικές προβολές.

Στη συνέχεια, χρησιμοποίησε το `TextField` για να κάνεις bind αυτές τις δύο μεταβλητές:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

Εδώ, τα `$user` και `$password` σημαίνουν "binding".

Δηλαδή, όταν ο χρήστης πληκτρολογεί κείμενο στο πεδίο εισαγωγής, η μεταβλητή αλλάζει ταυτόχρονα· και όταν αλλάζει η μεταβλητή, αλλάζει ταυτόχρονα και το περιεχόμενο που εμφανίζεται στο πεδίο εισαγωγής.

Αυτή η σχέση, όπου "η προβολή και τα δεδομένα συγχρονίζονται μεταξύ τους", είναι το binding.

Πρόσεξε ότι εδώ πρέπει να χρησιμοποιηθεί η γραφή με `$`:

```swift
$user
```

Επειδή το `TextField` δεν χρειάζεται ένα απλό string, αλλά μια τιμή binding που "μπορεί να τροποποιεί τα δεδομένα αμφίδρομα".

### Εμφάνιση των πεδίων εισαγωγής

Ας τα βάλουμε μέσα στο `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

Εφέ εμφάνισης:

![view](../../Resource/019_view3.png)

Επειδή τώρα τα `user` και `password` είναι και τα δύο κενά strings από προεπιλογή:

```swift
""
```

έτσι το πεδίο εισαγωγής θα εμφανίσει πρώτα κείμενο placeholder, για παράδειγμα:

```swift
input user
```

Αυτό το κείμενο απλώς υπενθυμίζει στον χρήστη "τι πρέπει να εισαχθεί εδώ" και δεν είναι το πραγματικό περιεχόμενο εισαγωγής.

### Βελτιστοποίηση των πεδίων εισαγωγής

Τώρα τα πεδία εισαγωγής μπορούν ήδη να χρησιμοποιηθούν, αλλά το προεπιλεγμένο στυλ είναι σχετικά απλό.

Για να γίνει το περιβάλλον πιο καθαρό, μπορούμε να προσθέσουμε έναν τίτλο μπροστά από το πεδίο εισαγωγής και να κάνουμε μια μικρή βελτιστοποίηση στυλ στο ίδιο το πεδίο.

Για παράδειγμα, πρόσθεσε πρώτα έναν τίτλο:

```swift
Text("Username")
    .fontWeight(.bold)
```

Έπειτα, χρησιμοποίησε το `HStack` για να βάλεις τον τίτλο και το πεδίο εισαγωγής στην ίδια γραμμή:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

Εδώ χρησιμοποιούμε `HStack`, επειδή θέλουμε "να εμφανίζεται ο τίτλος στα αριστερά και το πεδίο εισαγωγής στα δεξιά".

Πρέπει να σημειωθεί ότι το `TextField` από προεπιλογή καταλαμβάνει τον υπόλοιπο διαθέσιμο χώρο.

![color](../../Resource/019_view6.png)

Για να είναι πιο ομοιόμορφο το μέγεθος των δύο πεδίων εισαγωγής, μπορούμε να χρησιμοποιήσουμε το `frame(width:)` για να ορίσουμε πλάτος, ώστε τα πεδία να φαίνονται πιο τακτοποιημένα.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

Στη συνέχεια, πρόσθεσε και ένα περίγραμμα, ώστε το πεδίο εισαγωγής να φαίνεται πιο ξεκάθαρα:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Τώρα, ας προσθέσουμε τόσο το όνομα χρήστη όσο και τον κωδικό πρόσβασης στην προβολή `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

Εφέ εμφάνισης:

![view](../../Resource/019_view4.png)

Μέχρι εδώ, ο χρήστης μπορεί ήδη να εισαγάγει όνομα χρήστη και κωδικό πρόσβασης.

## Λογική σύνδεσης

Τώρα, θα προσθέσουμε στο κουμπί σύνδεσης τη βασικότερη λογική ελέγχου: **να ελέγξουμε αν ο χρήστης έχει εισαγάγει όνομα χρήστη και κωδικό πρόσβασης**.

Θέλουμε να υλοποιήσουμε τις παρακάτω καταστάσεις:

- Αν ο χρήστης δεν εισαγάγει τίποτα, να εμφανιστεί `Empty`
- Αν ο χρήστης εισαγάγει μόνο ένα από τα δύο, να δοθεί ειδοποίηση για το ποιο λείπει
- Αν έχουν εισαχθεί και όνομα χρήστη και κωδικός πρόσβασης, να εμφανιστεί `Success`

### Δεν εισήχθη καμία πληροφορία

Επειδή τα `user` και `password` είναι και τα δύο κενά strings από προεπιλογή:

```swift
@State private var user = ""
@State private var password = ""
```

όταν ο χρήστης δεν εισάγει τίποτα, παραμένουν κενά.

Στη Swift, τύποι όπως το `String` μπορούν να χρησιμοποιούν το `isEmpty` για να κρίνουν αν το περιεχόμενο είναι κενό.

### Ιδιότητα isEmpty

Το `isEmpty` χρησιμοποιείται συχνά για να ελέγχει αν strings, arrays και άλλα περιεχόμενα είναι κενά.

Για παράδειγμα:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

Αν το περιεχόμενο είναι κενό, το `isEmpty` επιστρέφει `true`· αν δεν είναι κενό, επιστρέφει `false`.

Επομένως, μπορούμε να το χρησιμοποιήσουμε για να κρίνουμε αν το όνομα χρήστη και ο κωδικός πρόσβασης έχουν περιεχόμενο.

### Χρήση του isEmpty για έλεγχο μεταβλητών

Αν ο χρήστης δεν έχει εισαγάγει καμία πληροφορία, τότε:

```swift
user.isEmpty // true
password.isEmpty    // true
```

Τότε μπορούμε να γράψουμε τον έλεγχο ως εξής:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

Αυτός ο κώδικας σημαίνει: αν το `user` είναι κενό και το `password` είναι επίσης κενό, τότε εμφάνισε:

```swift
Empty
```

Εδώ το `&&` είναι λογικός τελεστής και σημαίνει "και".

Δηλαδή, ολόκληρη η συνθήκη ισχύει μόνο όταν ισχύει η αριστερή συνθήκη και ισχύει επίσης η δεξιά συνθήκη.

Επομένως, αυτός ο κώδικας θα εκτελεστεί μόνο όταν και το όνομα χρήστη και ο κωδικός πρόσβασης είναι κενά.

## Ο χρήστης συμπλήρωσε μόνο ένα μέρος

Στη συνέχεια, ας εξετάσουμε μια άλλη κατάσταση: ο χρήστης συμπλήρωσε μόνο το όνομα χρήστη ή μόνο τον κωδικό πρόσβασης.

Για παράδειγμα:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

Αυτή τη στιγμή, το όνομα χρήστη δεν είναι κενό, αλλά ο κωδικός πρόσβασης είναι κενός.

Μπορούμε να συνεχίσουμε να χρησιμοποιούμε το `isEmpty` για να το ελέγξουμε:

### Κατάσταση όπου ο χρήστης έχει συμπληρώσει λογαριασμό ή κωδικό πρόσβασης

Αν ο χρήστης έχει συμπληρώσει μόνο το όνομα χρήστη ή μόνο τον κωδικό πρόσβασης, πρέπει να δώσουμε την αντίστοιχη ειδοποίηση για το τι λείπει.

Για παράδειγμα, αν ο χρήστης έχει συμπληρώσει μόνο το όνομα χρήστη:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Η λογική αυτού του κώδικα είναι: αν το όνομα χρήστη είναι κενό, ζήτησε να εισαχθεί όνομα χρήστη. Διαφορετικά, έλεγξε αν ο κωδικός πρόσβασης είναι κενός και, αν είναι, ζήτησε να εισαχθεί κωδικός πρόσβασης.

### Ο χρήστης συμπλήρωσε όλες τις πληροφορίες

Αν έχουν ήδη εισαχθεί και το όνομα χρήστη και ο κωδικός πρόσβασης, το `isEmpty` θα επιστρέψει `false` και για τα δύο.

Τότε μπορούμε να το γράψουμε έτσι:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

Εδώ το `!` σημαίνει "άρνηση".

Για παράδειγμα:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

Δηλαδή, το `user.isEmpty` χρησιμοποιείται για να κρίνει "αν το όνομα χρήστη είναι κενό", ενώ το `!user.isEmpty` χρησιμοποιείται για να κρίνει "αν το όνομα χρήστη δεν είναι κενό".

Ο κωδικός πρόσβασης ακολουθεί ακριβώς την ίδια λογική.

Άρα, η σημασία αυτού του κώδικα είναι: το όνομα χρήστη δεν είναι κενό και ο κωδικός πρόσβασης επίσης δεν είναι κενός.

Όταν ικανοποιούνται και οι δύο συνθήκες, το αποτέλεσμα θα είναι:

```swift
Success
```

## Πλήρης λογική ελέγχου

Τώρα, ας συνδυάσουμε αυτές τις τρεις καταστάσεις και ας τις γράψουμε μέσα στο `Button`:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

Η σειρά εκτέλεσης αυτού του κώδικα μπορεί να γίνει κατανοητή ως εξής:

Πρώτο βήμα, ελέγχουμε πρώτα:

```swift
if user.isEmpty && password.isEmpty
```

Αν και το όνομα χρήστη και ο κωδικός πρόσβασης είναι κενά, θα εμφανιστεί απευθείας `Empty`.

Αν δεν είναι και τα δύο κενά, συνεχίζουμε ελέγχοντας αυτή τη γραμμή:

```swift
else if user.isEmpty || password.isEmpty
```

Εδώ το `||` σημαίνει "ή".

Δηλαδή, αρκεί να ισχύει μία από τις συνθήκες, ώστε να ισχύει συνολικά η συνθήκη.

Επομένως, όσο το όνομα χρήστη είναι κενό ή ο κωδικός πρόσβασης είναι κενός, θα εκτελεστεί το μπλοκ κώδικα του `else if`.

Αφού μπούμε στο μπλοκ κώδικα, ελέγχουμε περαιτέρω ποιο πεδίο εισαγωγής ακριβώς δεν συμπληρώθηκε:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Αν καμία από τις δύο προηγούμενες συνθήκες δεν ισχύει, αυτό σημαίνει ότι το όνομα χρήστη δεν είναι κενό και ο κωδικός πρόσβασης επίσης δεν είναι κενός.

Τότε το πρόγραμμα θα μπει στο τελευταίο `else`:

```swift
print("Success")
```

Με αυτόν τον τρόπο, ολοκληρώσαμε τη βασικότερη λογική ελέγχου για την είσοδο σύνδεσης.

## Έλεγχος ονόματος χρήστη και κωδικού πρόσβασης

Η παραπάνω λογική ελέγχει μόνο "αν έχει εισαχθεί περιεχόμενο ή όχι".

Όμως σε μια πραγματική εφαρμογή δεν αρκεί μόνο η εισαγωγή περιεχομένου. Συνήθως χρειάζεται επίσης να σταλεί το όνομα χρήστη και ο κωδικός πρόσβασης στον διακομιστή για επαλήθευση.

Αν ο διακομιστής επιβεβαιώσει ότι το όνομα χρήστη υπάρχει και ότι ο κωδικός πρόσβασης είναι σωστός, θα επιτρέψει τη σύνδεση· διαφορετικά, θα ειδοποιήσει για αποτυχία σύνδεσης.

Για να εξασκηθούμε σε αυτή τη διαδικασία, μπορούμε πρώτα να ορίσουμε στον κώδικα έναν προσωρινό λογαριασμό και κωδικό πρόσβασης, ώστε να προσομοιώσουμε "σωστές πληροφορίες σύνδεσης":

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

Όταν και το όνομα χρήστη και ο κωδικός πρόσβασης δεν είναι κενά, συγκρίνουμε στη συνέχεια αν είναι σωστά:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

Εδώ το `==` σημαίνει "αν είναι ίσο".

Για παράδειγμα:

```swift
user == userID
```

Αυτό σημαίνει αν το όνομα χρήστη που εισήγαγε ο χρήστης είναι ακριβώς ίδιο με το σωστό όνομα χρήστη που έχουμε ορίσει· ο έλεγχος του κωδικού πρόσβασης ακολουθεί την ίδια λογική.

Επομένως, όταν το όνομα χρήστη και ο κωδικός πρόσβασης που εισήχθησαν είναι ίσα με τις καθορισμένες τιμές, το αποτέλεσμα θα είναι `Success`· αρκεί ένα από τα δύο να μην είναι ίσο, το αποτέλεσμα θα είναι `Input error`.

Έτσι, ολοκληρώσαμε μια απλή "προσομοιωμένη επαλήθευση σύνδεσης".

Αν και δεν είναι ακόμη πραγματική λειτουργία σύνδεσης συνδεδεμένη με διακομιστή, αρκεί ήδη για να μας βοηθήσει να καταλάβουμε: **αφού πατηθεί το κουμπί, μπορούν να εκτελεστούν διαφορετικοί κώδικες ανάλογα με διαφορετικές συνθήκες**.

## Σύνοψη

Σε αυτό το μάθημα, ολοκληρώσαμε μια βασική προβολή σύνδεσης και εστιάσαμε κυρίως στη "λογική αξιολόγησης του περιεχομένου εισόδου με βάση συνθήκες".

Ο βασικός πυρήνας αυτού του μαθήματος είναι: να χρησιμοποιούμε το `isEmpty` για να κρίνουμε αν ένα string είναι κενό και να χρησιμοποιούμε την εντολή `if` μαζί με λογικούς τελεστές για να χειριζόμαστε διαφορετικές καταστάσεις.

Ανάμεσά τους υπάρχουν δύο πολύ σημαντικοί λογικοί τελεστές:

- `&&`: σημαίνει "και", πρέπει να ικανοποιούνται και οι δύο συνθήκες
- `||`: σημαίνει "ή", αρκεί να ικανοποιείται μία από τις συνθήκες

Όταν αυτές οι συνθήκες τελικά δώσουν `true` ή `false`, η εντολή `if` θα εκτελέσει διαφορετικό κώδικα ανάλογα με το αποτέλεσμα.

Μέσα από αυτή την προβολή σύνδεσης, έχουμε ήδη αρχίσει να ερχόμαστε σε επαφή με έναν τρόπο ανάπτυξης που συνδυάζει "διεπαφή + δεδομένα + λογική αξιολόγηση".

## Πλήρης κώδικας

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Button("Sign in") {
                if user.isEmpty && password.isEmpty {
                    print("Empty")
                } else if user.isEmpty || password.isEmpty {
                    if user.isEmpty {
                        print("Please enter your username.")
                    } else if password.isEmpty {
                        print("Please enter your password.")
                    }
                } else {
                    if user == userID && password == passWord {
                        print("Success")
                    } else {
                        print("Input error")
                    }
                }
            }
            .buttonStyle(.borderedProminent)
            Spacer()
        }
        .padding(.top, 50)
    }
}

```
