# Τελεστές

Σε αυτό το μάθημα θα μάθουμε τους τελεστές της Swift (`Operators`).

Κάθε φορά που κάνουμε υπολογισμούς, συγκρίσεις ή λογικούς ελέγχους, χρειάζεται να χρησιμοποιούμε τελεστές. Αυτές οι λειτουργίες διατρέχουν κάθε στάδιο της εκτέλεσης ενός προγράμματος και αποτελούν μία από τις πιο βασικές δεξιότητες στον προγραμματισμό.

```swift
let a = 100 * 8
```

Κατά τη διαδικασία της εκμάθησης προγραμματισμού, πολλοί διαπιστώνουν ότι όσοι έχουν κάποια μαθηματική σκέψη κατανοούν πιο εύκολα τη λογική του προγράμματος. Αυτό συμβαίνει επειδή ο προγραμματισμός στην ουσία δεν μπορεί να υπάρξει χωρίς «υπολογισμό» και «έλεγχο κανόνων», και ακριβώς αυτές είναι οι βασικές λειτουργίες που αναλαμβάνουν οι τελεστές.

Στην πραγματική ανάπτυξη, οι τελεστές δεν χρησιμοποιούνται μόνο για απλές αριθμητικές πράξεις, αλλά εφαρμόζονται ευρέως και σε πιο συγκεκριμένα σενάρια.

Για παράδειγμα, σε ένα block game, όταν ο χρήστης σύρει ένα block σε συγκεκριμένη θέση, χρειάζεται να υπολογιστούν:

- οι συντεταγμένες του block
- οι θέσεις στις οποίες μπορεί να τοποθετηθεί
- αν η θέση-στόχος είναι διαθέσιμη

Αυτή η διαδικασία «υπολογισμού + ελέγχου» βασίζεται ουσιαστικά στους τελεστές.

![Block](../../Resource/013_block.png)

Η εκμάθηση τελεστών δεν σημαίνει μόνο εκμάθηση αριθμητικών πράξεων, αλλά και χτίσιμο βάσης για τα επόμενα μαθήματα, όπως layout διεπαφής, επεξεργασία δεδομένων και λογική αλληλεπίδρασης.

## Τρόπος εκτέλεσης υπολογισμών στο SwiftUI

Στο SwiftUI, συνήθως χρησιμοποιούμε ένα `Button` για να ενεργοποιήσουμε μια συνάρτηση και μέσα στη συνάρτηση εκτελούμε τη λογική των υπολογισμών.

Για παράδειγμα:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

Όταν πατήσουμε το button, στην Console θα εμφανιστεί:

```text
c: 20
```

Σε αυτό το μάθημα θα γράφουμε όλη τη λογική των υπολογισμών μέσα σε `func`, θα την εκτελούμε μέσω button και θα χρησιμοποιούμε `print` για να εμφανίζουμε τα αποτελέσματα.

Στα επόμενα παραδείγματα, για να απλοποιήσουμε την παρουσίαση του κώδικα, θα σημειώνουμε απευθείας τα αποτελέσματα μέσα στον κώδικα και δεν θα επαναλαμβάνουμε διαρκώς το `print`.

## Διαχείριση εξόδου στην Console

Επειδή κάθε πάτημα παράγει output, μπορούμε να πατήσουμε το κουμπί καθαρισμού κάτω δεξιά στην περιοχή της Console, ώστε να σβήνονται τα παλιά logs και να βλέπουμε πιο καθαρά τα τρέχοντα αποτελέσματα.

## Μια άλλη γραφή του Button

Όταν ένα button εκτελεί μόνο μία συνάρτηση, μπορούμε να χρησιμοποιήσουμε μια πιο σύντομη μορφή:

```swift
Button(action: ) {
    // View
}
```

Σε αυτή τη μορφή, το `action` καθορίζει τη συνάρτηση που θα εκτελείται όταν πατιέται το button (συνάρτηση χωρίς παραμέτρους), ενώ το block `{}` ορίζει το view του button.

Σε συνδυασμό με τη συνάρτηση `calculate`, μπορούμε να γράψουμε:

```swift
Button(action: calculate) {
    Text("Button")
}
```

Όταν πατήσουμε το button, θα εκτελεστεί απευθείας η συνάρτηση `calculate`.

Επομένως, όταν η λογική του button είναι απλή και περιορίζεται στην κλήση μίας μόνο συνάρτησης, αυτή η γραφή με `action` είναι πιο καθαρή και πιο σύμφωνη με τις coding conventions.

## Αριθμητικοί τελεστές

Οι αριθμητικοί τελεστές (`Arithmetic Operators`) χρησιμοποιούνται για αριθμητικούς υπολογισμούς:

- `+` πρόσθεση
- `-` αφαίρεση
- `*` πολλαπλασιασμός
- `/` διαίρεση
- `%` υπόλοιπο διαίρεσης

Παραδείγματα:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

Πρέπει να προσέξουμε ότι η Swift είναι strongly typed language και δεν αναμιγνύει αυτόματα διαφορετικούς τύπους δεδομένων:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

Όταν οι δύο τιμές έχουν διαφορετικό τύπο, πρέπει να γίνει χειροκίνητη μετατροπή.

## Συγκριτικοί τελεστές

Οι συγκριτικοί τελεστές (`Comparison Operators`) χρησιμοποιούνται για να ελέγχουν τη σχέση ανάμεσα σε δύο τιμές, και το αποτέλεσμα είναι `Bool`:

- `==` ίσο
- `!=` όχι ίσο
- `>` μεγαλύτερο από
- `<` μικρότερο από
- `>=` μεγαλύτερο ή ίσο
- `<=` μικρότερο ή ίσο

Παραδείγματα:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

Οι συγκριτικοί τελεστές χρησιμοποιούνται πολύ συχνά σε συνθήκες. Για παράδειγμα, στο «καρουζέλ αποφθεγμάτων» χρειαζόμασταν να βεβαιωθούμε ότι ο index δεν ξεπερνά τα όρια του array:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

Αν το `index` είναι μικρότερο από `sayings.count - 1`, το αποτέλεσμα της σύγκρισης είναι `true`, και ο κώδικας μέσα στο `if` θα εκτελεστεί. Αν είναι `false`, δεν θα εκτελεστεί ο επόμενος κώδικας.

## Λογικοί τελεστές

Οι λογικοί τελεστές (`Logical Operators`) χρησιμοποιούνται για να συνδυάζουν πολλές συνθήκες:

- `&&` και (`AND`)
- `||` ή (`OR`)
- `!` όχι (`NOT`)

### && και

Όταν πολλές συνθήκες πρέπει να ισχύουν ταυτόχρονα, χρησιμοποιούμε `&&`.

Για παράδειγμα:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

Σε αυτό το παράδειγμα υπάρχουν δύο συνθήκες: `age >= 18` και `age <= 100`.

Μόνο όταν και οι δύο είναι `true`, το `legalAge` θα γίνει `true`. Αν έστω μία δεν ισχύει, το αποτέλεσμα θα είναι `false`.

### || ή

Όταν αρκεί να ισχύει μία από πολλές συνθήκες, χρησιμοποιούμε `||`.

Για παράδειγμα:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

Αυτός ο κώδικας σημαίνει ότι αρκεί να ισχύει είτε `money >= 50` είτε `creditCard >= 50`, ώστε το `payAllowed` να γίνει `true`.

Μόνο όταν και οι δύο συνθήκες είναι ψευδείς, το αποτέλεσμα είναι `false`.

Για παράδειγμα, σε ένα σενάριο πληρωμής, μπορούμε να πληρώσουμε είτε με μετρητά είτε με πιστωτική κάρτα. Αν δεν είναι διαθέσιμος κανένας από τους δύο τρόπους, τότε η πληρωμή αποτυγχάνει.

### ! όχι

Το `!` είναι ο λογικός τελεστής άρνησης (`NOT`), που αντιστρέφει μια τιμή `Bool`.

Μπορούμε να το σκεφτούμε ως εξής: το `!` μετατρέπει το `true` σε `false`, και το `false` σε `true`.

Για παράδειγμα:

```swift
let allowed = true
let result = !allowed   // false
```

Εδώ το `!allowed` σημαίνει ότι αντιστρέφουμε την τιμή του `allowed`.

Πρέπει να προσέξουμε ότι τα `!` και `!=` είναι διαφορετικοί τελεστές: το `!` αντιστρέφει μια `Bool`, ενώ το `!=` είναι συγκριτικός τελεστής που επιστρέφει `Bool`.

## Τελεστές ανάθεσης

Οι τελεστές ανάθεσης (`Assignment Operators`) χρησιμοποιούνται για ανάθεση ή ενημέρωση μεταβλητών:

- `=` ανάθεση

Τόσο στις δηλώσεις μεταβλητών όσο και στις πράξεις υπολογισμού, εμφανίζεται διαρκώς η έννοια της ανάθεσης:

```swift
let a = 5
```

Αυτό σημαίνει ότι η αριθμητική τιμή `5` ανατίθεται στη μεταβλητή `a`.

### Σύνθετοι τελεστές ανάθεσης

Στην πράξη, εκτός από τον βασικό τελεστή ανάθεσης, χρησιμοποιούμε συχνά και τους σύνθετους τελεστές ανάθεσης, οι οποίοι μας επιτρέπουν να κάνουμε υπολογισμό πάνω στην υπάρχουσα τιμή και να την ενημερώνουμε απευθείας:

- `+=`
- `-=`
- `*=`
- `/=`

Στο προηγούμενο μάθημα του «μετρητή» είδαμε ήδη μια πρώτη εισαγωγή σε αυτούς.

Για παράδειγμα, όταν θέλουμε μια μεταβλητή να αυξάνεται αυτόματα με το πάτημα ενός button:

```swift
var num = 10
num += 5   // num = 15
```

Αυτό σημαίνει ότι γίνεται πρόσθεση του `num` με το `5`, και μετά το αποτέλεσμα ανατίθεται πάλι στο `num`.

Άρα είναι ισοδύναμο με:

```swift
var num = 10
num = num + 5   // num = 15
```

## Τριαδικός τελεστής

Στη Swift, ο τριαδικός τελεστής (`Ternary Operator`) ελέγχει μια συνθήκη και επιστρέφει μία από δύο πιθανές τιμές:

```swift
συνθήκη ? τιμή1 : τιμή2
```

Η λογική του είναι ότι, αν η συνθήκη είναι `true`, επιστρέφει την «τιμή1». Αν είναι `false`, επιστρέφει την «τιμή2».

Για παράδειγμα:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

Αυτός ο κώδικας ελέγχει την τιμή του `showColor`: αν είναι `true`, επιστρέφει `Color.blue`, και το background γίνεται μπλε· αν είναι `false`, επιστρέφει `Color.clear`, δηλαδή δεν εμφανίζεται background χρώμα.

Σημείωση: Το `Color.clear` σημαίνει διαφανές χρώμα.

### Διαφορά από το if-else

Ο τριαδικός τελεστής είναι λογικά ισοδύναμος με μια δήλωση `if-else`.

Για παράδειγμα, ο παραπάνω κώδικας μπορεί να γραφτεί και έτσι:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

Η λογική είναι ακριβώς η ίδια: όταν η συνθήκη είναι `true`, επιλέγεται το πρώτο αποτέλεσμα· όταν είναι `false`, επιλέγεται το δεύτερο.

Στην πράξη, η γραφή με τον τριαδικό τελεστή είναι πιο σύντομη και πιο κατάλληλη για απλές αποφάσεις «ένα από τα δύο», ειδικά όταν θέλουμε να επιστρέψουμε απευθείας μια τιμή.

Στην κατασκευή SwiftUI views, ο τριαδικός τελεστής είναι πολύ συχνός. Για παράδειγμα:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

Αυτός ο τρόπος μας επιτρέπει να αλλάζουμε δυναμικά το εφέ μιας view με βάση μια συνθήκη, μέσα σε μία μόνο γραμμή κώδικα.

## Τελεστές εύρους

Οι τελεστές εύρους (`Range Operators`) χρησιμοποιούνται για να εκφράζουν διαστήματα:

- `...` κλειστό διάστημα (περιλαμβάνει και τα δύο άκρα)
- `..<` ημι-ανοιχτό διάστημα (δεν περιλαμβάνει το δεξί άκρο)

Για παράδειγμα:

```text
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

Μπορούμε να το κατανοήσουμε ως εξής: το `1...3` σημαίνει από το 1 έως το 3 και περιλαμβάνει το 3· ενώ το `5..<8` σημαίνει από το 5 έως το 8 αλλά χωρίς να περιλαμβάνει το 8.

Επεκταμένες μορφές, μόνο για να τις γνωρίζεις:

```text
5...    // >= 5
...5    // <= 5
```

Αυτή η μορφή σημαίνει «μονόπλευρο εύρος» και χρησιμοποιείται συχνά σε condition checks ή pattern matching.

Οι τελεστές εύρους θα εμφανιστούν περισσότερο όταν μάθουμε `for` loops. Προς το παρόν αρκεί να τους καταλαβαίνουμε σαν ένα συνεχόμενο αριθμητικό διάστημα.

## Εξάσκηση με τελεστές

Παρακάτω, μέσω μερικών απλών παραδειγμάτων, θα εξασκηθούμε στη λειτουργία των τελεστών μέσα σε ένα view.

### 1. Υπολογισμός συνολικού αριθμού οχημάτων

Για παράδειγμα, σε μια συγκεκριμένη διασταύρωση περνούν καθημερινά 500 οχήματα. Θέλουμε να υπολογίσουμε πόσα οχήματα θα έχουν περάσει μέσα σε 30 ημέρες.

Μπορούμε να χρησιμοποιήσουμε αριθμητικούς τελεστές:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

Η λογική του υπολογισμού είναι: 500 οχήματα ανά ημέρα, επί 30 ημέρες, άρα συνολικά `500 × 30 = 15000`.

### 2. Έλεγχος συνδρομητικού μέλους

Μπορούμε να ελέγξουμε αν κάποιος είναι ενδο-αγοραστικό μέλος μέσα στην εφαρμογή. Αρκεί να έχει είτε lifetime membership είτε subscription membership, για να θεωρηθεί μέλος.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

Σε αυτό το παράδειγμα χρησιμοποιείται ο τελεστής `||` (ή): το `lifeTime = false` σημαίνει ότι δεν υπάρχει lifetime membership, ενώ το `subscription = true` σημαίνει ότι υπάρχει ενεργή συνδρομή.

Επειδή αρκεί να ισχύει μία από τις δύο συνθήκες, το `isMember` γίνεται `true`. Άρα το άτομο είναι συνδρομητικό μέλος.

## Σύνοψη

Στη Swift, οι τελεστές είναι η βάση τόσο των υπολογισμών όσο και της λογικής κρίσης. Όπως το `Text` εμφανίζει περιεχόμενο, έτσι οι τελεστές είναι το εργαλείο για την επεξεργασία των ίδιων των δεδομένων.

Από τους απλούς αριθμητικούς υπολογισμούς μέχρι τους σύνθετους λογικούς ελέγχους, αλλά και τον έλεγχο κατάστασης και τους υπολογισμούς layout μέσα σε views, παντού υπάρχει συμμετοχή τελεστών.

Μέσα από εξάσκηση σε διαφορετικά σενάρια, μπορούμε σταδιακά να εξοικειωθούμε και να κατακτήσουμε τις διάφορες κατηγορίες τελεστών που είδαμε σε αυτό το μάθημα. Αυτό αποτελεί πολύ σημαντική βάση για όσα ακολουθούν.

### Επέκταση - Bitwise operators

Επιπλέον, η Swift παρέχει και μια πιο χαμηλού επιπέδου κατηγορία τελεστών: τους bitwise operators:

- `&` (bitwise AND)
- `|` (bitwise OR)
- `^` (bitwise XOR)
- `~` (bitwise NOT)
- `>>` (δεξιά μετατόπιση)
- `<<` (αριστερή μετατόπιση)

Για παράδειγμα:

```swift
let a = 6  // binary 110
let b = 3  // binary 011

print(a & b) // 2 (binary 010)
print(a | b) // 7 (binary 111)
print(a ^ b) // 5 (binary 101)
print(~a)    // -7 (negation)
```

Στη Swift, οι ακέραιοι αναπαρίστανται εσωτερικά με δυαδικό συμπλήρωμα ως προς δύο, και γι’ αυτό η bitwise άρνηση οδηγεί σε αντίστοιχο αρνητικό αποτέλεσμα.

Οι bitwise operators δουλεύουν απευθείας πάνω στο δυαδικό επίπεδο και χρησιμοποιούνται κυρίως σε πιο χαμηλού επιπέδου επεξεργασία δεδομένων ή σε σενάρια βελτιστοποίησης απόδοσης.

Για τους αρχάριους, τα σενάρια χρήσης αυτών των τελεστών είναι σχετικά λίγα, και στην κλασική ανάπτυξη iOS ή SwiftUI δεν είναι συνηθισμένοι. Γι’ αυτό δεν θα τους αναλύσουμε βαθύτερα εδώ.

Αν αργότερα ασχοληθούμε με low-level λογική ή performance optimization, τότε αξίζει να τους ξαναδούμε.
