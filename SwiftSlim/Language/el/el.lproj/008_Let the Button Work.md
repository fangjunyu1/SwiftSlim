# Κάνοντας το Button να λειτουργεί

Σε αυτό το μάθημα θα μάθουμε κυρίως το `Button`, το `print` για έξοδο debug πληροφοριών και τις συναρτήσεις στη Swift.

Αυτά τα στοιχεία αποτελούν τη βάση του διαδραστικού προγραμματισμού στο SwiftUI.

## Button

Τα buttons είναι από τα πιο συνηθισμένα UI controls. Όταν θέλουμε να κατεβάσουμε, να ανοίξουμε ή να κλείσουμε κάτι, συνήθως πατάμε ένα button.

Για παράδειγμα, στο App Store πατάμε το κουμπί "Get" για να κατεβάσουμε μια εφαρμογή.

![Button](../../RESOURCE/008_button.png)

Στο SwiftUI, το button αναπαρίσταται με `Button`.

Βασική χρήση:

```swift
Button("") {
    
}
```

Η δομή μπορεί να χωριστεί σε δύο μέρη:

```swift
Button("Κείμενο κουμπιού") {
    Κώδικας που εκτελείται όταν πατηθεί το κουμπί
}
```

Για παράδειγμα:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

Όταν το τρέξουμε, θα εμφανιστεί ένα button με την ένδειξη `Start`.

![Button](../../RESOURCE/008_button1.png)

Όταν ο χρήστης πατήσει το button, ο κώδικας μέσα στις αγκύλες `{}` θα εκτελεστεί.

Πρέπει να προσέξουμε ότι το `Button` είναι στοιχείο διεπαφής του SwiftUI (`View`), επομένως πρέπει να γράφεται μέσα στο `body`.

### Ο modifier buttonStyle

Στο SwiftUI, το `Button` παρέχει ενσωματωμένα styles του συστήματος, τα οποία μπορούμε να χρησιμοποιήσουμε μέσω του modifier `buttonStyle`.

Για παράδειγμα:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

Το `buttonStyle` χρησιμοποιεί ένα style button που παρέχει το σύστημα.

Συνηθισμένες επιλογές είναι:

- `.automatic`
- `.plain`
- `.bordered`
- `.borderedProminent`
- `.borderless`

![Button](../../RESOURCE/008_button7.png)

Οι διαφορές ανάμεσα στα styles σχετίζονται κυρίως με το αν το button έχει περίγραμμα, background και πόσο έντονα τονίζεται οπτικά.

## Έξοδος με print

Κατά τον προγραμματισμό, συχνά χρειάζεται να ελέγξουμε αν το πρόγραμμα έφτασε σε ένα συγκεκριμένο σημείο κώδικα.

Η Swift παρέχει τη συνάρτηση `print` για έξοδο debug πληροφοριών.

Βασική μορφή:

```swift
print("Hello")
```

Αυτή η γραμμή θα εμφανίσει το κείμενο στην `Console`.

### Έλεγχος αν το button λειτουργεί

Μπορούμε να γράψουμε `print` μέσα στο button ώστε να ελέγξουμε αν πατήθηκε πραγματικά.

```swift
Button("Start") {
    print("Test 123")
}
```

Όταν πατηθεί το button, στην `Console` θα εμφανιστεί:

```text
Test 123
```

Αυτό δείχνει ότι το button ενεργοποιήθηκε επιτυχώς.

![Button](../../RESOURCE/008_button3.png)

Σημείωση: Το `print` είναι κώδικας Swift, όχι στοιχείο διεπαφής. Δεν μπορεί να τοποθετηθεί απευθείας στο `body`, γιατί τότε θα προκύψει σφάλμα.

Για παράδειγμα, ο παρακάτω κώδικας είναι λάθος:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // Σφάλμα: το body πρέπει να επιστρέφει ένα View
    }
}
```

Στο SwiftUI, το `body` χρησιμοποιείται για να περιγράψει τη δομή της διεπαφής, άρα μπορεί να περιέχει μόνο UI controls (`View`), όπως `Text`, `Button`, `Image` κτλ.

Κανονικός κώδικας, όπως λογική υπολογισμών ή `print`, πρέπει να βρίσκεται μέσα σε buttons, συναρτήσεις ή άλλους event handlers.

### Η Console

Το περιεχόμενο που τυπώνει το `print` εμφανίζεται στην `Console` του Xcode.

Αν δεν βλέπεις την `Console`, μπορείς να πατήσεις κάτω δεξιά το κουμπί `Show the Debug Area` για να ανοίξεις την περιοχή debugging και μετά το `Show the Console` για να εμφανιστεί η κονσόλα.

![](../../RESOURCE/008_button4.png)

Στο κάτω αριστερό μέρος της `Console` υπάρχουν δύο tabs: `Executable` και `Previews`.

![](../../RESOURCE/008_button5.png)

Αυτά τα δύο tabs αντιστοιχούν σε διαφορετικά περιβάλλοντα εκτέλεσης: όταν χρησιμοποιείς το Canvas preview, οι έξοδοι από `print` εμφανίζονται στο tab `Previews`. Αν η εφαρμογή τρέχει σε προσομοιωτή ή πραγματική συσκευή, οι έξοδοι εμφανίζονται στο tab `Executable`.

Άρα, αν παρατηρήσεις ότι το `print` δεν εμφανίζει τίποτα, πρώτα έλεγξε αν έχεις επιλέξει το σωστό tab.

## Παράδειγμα - ASCII art

Σε αρκετά code projects συναντάμε συχνά ASCII art.

Το ASCII art είναι σχέδια φτιαγμένα από απλούς χαρακτήρες, για παράδειγμα:

![ASCII](../../RESOURCE/008_ascii.png)

Μπορούμε να χρησιμοποιήσουμε ένα button μαζί με `print` για να εμφανίσουμε ένα ASCII art.

Παράδειγμα κώδικα:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Όταν πατηθεί το button, η `Console` θα εμφανίσει:

```text
 /\_/\
( o.o )
 > ^ <
```

Σημείωση: Το `\` μέσα σε string πρέπει να γραφτεί ως `\\`, επειδή το backslash είναι escape character μέσα στα strings.

## Συναρτήσεις

Μέσα σε ένα button μπορούμε να γράψουμε κώδικα απευθείας.

Για παράδειγμα:

```swift
Button("Start") {
    print("Hello")
}
```

Αν ο κώδικας είναι λίγος, αυτό δεν δημιουργεί πρόβλημα. Στην πραγματική ανάπτυξη όμως, η λογική που εκτελεί ένα button μπορεί να γίνει αρκετά μεγάλη.

Για παράδειγμα:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // Πιθανόν να υπάρχει πολύ περισσότερος κώδικας
}
```

Αν ο κώδικας μεγαλώνει όλο και περισσότερο, το να γράφεται απευθείας μέσα στο button κάνει τον κώδικα της διεπαφής ακατάστατο και δυσκολεύει την ανάγνωση και τη συντήρηση.

Γι’ αυτό συνήθως οργανώνουμε αυτόν τον κώδικα μέσα σε μια συνάρτηση και μετά καλούμε τη συνάρτηση από το button.

### Τι είναι μια συνάρτηση

Η συνάρτηση είναι ένα κομμάτι κώδικα που μπορεί να επαναχρησιμοποιηθεί.

Όταν θέλουμε να εκτελεστεί κάποιος κώδικας, αρκεί να καλέσουμε αυτή τη συνάρτηση και θα εκτελεστεί ο κώδικας που υπάρχει μέσα της.

Με αυτόν τον τρόπο η δομή του κώδικα γίνεται πιο καθαρή και η επαναχρησιμοποίηση πιο εύκολη.

### Βασική σύνταξη

Στη Swift, οι συναρτήσεις ορίζονται με τη λέξη-κλειδί `func`:

```swift
func randomInt() {
    // code
}
```

Αυτός ο κώδικας ορίζει μια συνάρτηση.

Το `randomInt` είναι το όνομα της συνάρτησης, δηλαδή η ταυτότητα αυτού του κομματιού κώδικα.

Τα `()` δηλώνουν τη θέση των παραμέτρων, δηλαδή των δεδομένων που μπορούν να περαστούν από έξω. Αν δεν χρειαζόμαστε δεδομένα, μπορούν να μείνουν κενά.

Μέσα στις αγκύλες `{}` βρίσκεται το σώμα της συνάρτησης, δηλαδή όλος ο κώδικας που θέλουμε να εκτελεστεί.

Για παράδειγμα:

```swift
func getName() {
    print("FangJunyu")
}
```

Αυτή η συνάρτηση έχει ως ρόλο να εμφανίζει ένα κείμενο.

### Κλήση συνάρτησης

Το να ορίσουμε μια συνάρτηση απλώς δημιουργεί αυτό το κομμάτι κώδικα. Αν θέλουμε να εκτελεστεί, πρέπει να την καλέσουμε.

Η κλήση γίνεται γράφοντας το όνομα της συνάρτησης ακολουθούμενο από `()`:

```swift
getName()
```

Όταν το πρόγραμμα φτάσει σε αυτή τη γραμμή, θα εκτελέσει τον κώδικα μέσα στη συνάρτηση `getName`.

Πλήρες παράδειγμα:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

Όταν τρέξει, η `Console` θα εμφανίσει:

```text
FangJunyu
```

### Χρήση συναρτήσεων στη SwiftUI

Στις SwiftUI views, οι συναρτήσεις συνήθως γράφονται έξω από το `body`.

Για παράδειγμα:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

Όταν πατηθεί το button, θα κληθεί η συνάρτηση `getName()`.

Έτσι ο κώδικας της διεπαφής παραμένει καθαρός, ενώ η πραγματική λογική οργανώνεται μέσα σε συναρτήσεις.

### Παράμετροι συναρτήσεων

Μερικές φορές, μια συνάρτηση χρειάζεται να εκτελεί διαφορετικές ενέργειες ανάλογα με τα δεδομένα που θα της δοθούν.

Για παράδειγμα, μπορεί να θέλουμε η συνάρτηση να εμφανίζει διαφορετικά ονόματα και όχι πάντα μόνο το σταθερό `"FangJunyu"`.

Σε αυτή την περίπτωση μπορούμε να χρησιμοποιήσουμε παραμέτρους. Μπορούμε να σκεφτούμε τις παραμέτρους σαν δεδομένα που περνάμε στη συνάρτηση τη στιγμή που την καλούμε.

Για παράδειγμα:

```swift
func getName(name: String) {
    print(name)
}
```

Αυτή η συνάρτηση έχει τώρα μια επιπλέον παράμετρο. Το `name` είναι το όνομα της παραμέτρου, και το `String` είναι ο τύπος της.

Αυτό σημαίνει ότι η συνάρτηση πρέπει να λάβει δεδομένα τύπου `String`.

Εφόσον η συνάρτηση χρειάζεται πλέον δεδομένα, όταν την καλούμε πρέπει να της τα δώσουμε.

```swift
getName(name: "Sam")
```

Όταν περάσουμε το `"Sam"` στην κλήση, η συνάρτηση θα χρησιμοποιήσει αυτή την τιμή για την έξοδο.

Αποτέλεσμα:

```text
Sam
```

## Παράδειγμα - εμφάνιση ASCII art μέσω συνάρτησης

Στο προηγούμενο παράδειγμα γράψαμε το `print` απευθείας μέσα στο button.

Αν όμως το ASCII art γίνει πιο σύνθετο, ο κώδικας θα γίνει υπερβολικά μεγάλος. Σε αυτή την περίπτωση, μπορούμε να βάλουμε τη λογική μέσα σε συνάρτηση ώστε ο κώδικας της διεπαφής να παραμένει πιο καθαρός.

Παράδειγμα:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Αφού τρέξει και πατήσεις το button, η `Console` θα εμφανίσει:

```text
 /\_/\
( o.o )
 > ^ <
```

Εδώ συμβαίνουν δύο πράγματα, μόλις ο χρήστης πατήσει το button:

1. Το button καλεί τη συνάρτηση `printCat()`
2. Εκτελείται ο κώδικας `print` που υπάρχει μέσα στη συνάρτηση

Με αυτόν τον τρόπο, ο κώδικας της διεπαφής είναι υπεύθυνος μόνο για το trigger του event, ενώ η πραγματική λογική μένει μέσα στη συνάρτηση.

Στην πράξη, οι περισσότερες ενέργειες buttons υλοποιούνται ακριβώς έτσι, μέσω συναρτήσεων.

## Σύνοψη

Σε αυτό το μάθημα μάθαμε τρία σημαντικά πράγματα:

**1. Button**

Το `Button` είναι το control του SwiftUI που χρησιμοποιείται για να εκκινεί ενέργειες.

```swift
Button("Start") {

}
```

Όταν ο χρήστης πατήσει το button, θα εκτελεστεί ο κώδικας μέσα στις αγκύλες.

**2. print**

Το `print` χρησιμοποιείται για να εμφανίζει πληροφορίες debugging στην `Console`.

```swift
print("Hello")
```

Οι προγραμματιστές μπορούν μέσω της `Console` να παρατηρούν την κατάσταση εκτέλεσης του προγράμματος.

**3. Συναρτήσεις**

Η συνάρτηση είναι ένα κομμάτι κώδικα που μπορεί να επαναχρησιμοποιηθεί.

```swift
func sayHello() {
    print("Hello")
}
```

Κλήση συνάρτησης:

```swift
sayHello()
```

Αν μια συνάρτηση χρειάζεται δεδομένα, μπορούμε να χρησιμοποιήσουμε παραμέτρους:

```swift
func sayHello(name: String) {
    print(name)
}
```

Στα επόμενα μαθήματα θα γνωρίσουμε περισσότερα SwiftUI controls και θα δούμε πώς αλληλεπιδρούν τα δεδομένα με τη διεπαφή.
