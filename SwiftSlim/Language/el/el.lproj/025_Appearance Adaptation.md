# Προσαρμογή εμφάνισης

Σε αυτό το μάθημα θα μάθουμε την προσαρμογή εμφάνισης για τη φωτεινή και τη σκοτεινή λειτουργία.

Σε iPhone, iPad και Mac, ο χρήστης μπορεί να επιλέξει φωτεινή λειτουργία ή σκοτεινή λειτουργία.

Όταν το σύστημα αλλάζει σε σκοτεινή λειτουργία, το φόντο, το κείμενο και τα χρώματα των εικονιδίων πολλών App αλλάζουν επίσης.

Για παράδειγμα, την ημέρα η διεπαφή μπορεί να έχει φωτεινό φόντο, ενώ τη νύχτα μπορεί να γίνει σκοτεινό φόντο.

![view](../../../Resource/025_view9.png)

Αυτή η δυνατότητα αυτόματης προσαρμογής της διεπαφής σύμφωνα με την εμφάνιση του συστήματος ονομάζεται προσαρμογή εμφάνισης.

## Προεπισκόπηση προσαρμογής εμφάνισης στο Xcode

Πριν μάθουμε την προσαρμογή εμφάνισης, ας δούμε πρώτα πώς μπορούμε να κάνουμε προεπισκόπηση φωτεινής και σκοτεινής λειτουργίας στο Xcode.

Στο κάτω μέρος της περιοχής `Canvas`, πατήστε το κουμπί `Device Settings` για να ανοίξετε το παράθυρο `Canvas Device Settings`.

![canvas](../../../Resource/025_view.png)

Στο παράθυρο μπορείτε να δείτε τη ρύθμιση `Color Scheme`.

![canvas1](../../../Resource/025_view1.png)

Συγκεκριμένα:

- Το `Light Appearance` σημαίνει φωτεινή λειτουργία.
- Το `Dark Appearance` σημαίνει σκοτεινή λειτουργία.

Μπορούμε να αλλάξουμε την εμφάνιση προεπισκόπησης από εδώ και να δούμε πώς εμφανίζεται η τρέχουσα διεπαφή σε φωτεινή και σκοτεινή λειτουργία.

## Παράλληλη εμφάνιση δύο εμφανίσεων

Αν θέλουμε να βλέπουμε ταυτόχρονα τη φωτεινή και τη σκοτεινή λειτουργία, μπορούμε να χρησιμοποιήσουμε τη δυνατότητα `Variants`.

Στο κάτω μέρος της περιοχής `Canvas`, πατήστε το κουμπί `Variants` και επιλέξτε `Color Scheme Variants`.

![canvas2](../../../Resource/025_view2.png)

Μετά την επιλογή, το `Canvas` θα εμφανίζει ταυτόχρονα τα αποτελέσματα προεπισκόπησης της φωτεινής και της σκοτεινής λειτουργίας.

![canvas3](../../../Resource/025_view3.png)

Έτσι μπορούμε να συγκρίνουμε πιο εύκολα τις διαφορές της διεπαφής ανάμεσα στις δύο εμφανίσεις.

Με απλά λόγια:

Αν θέλουμε μόνο να αλλάξουμε προσωρινά ανάμεσα σε φωτεινή/σκοτεινή λειτουργία, μπορούμε να χρησιμοποιήσουμε το `Canvas Device Settings`.

Αν θέλουμε να βλέπουμε ταυτόχρονα και τις δύο εμφανίσεις, μπορούμε να χρησιμοποιήσουμε το `Color Scheme Variants`.

## Οι προβολές προσαρμόζονται από προεπιλογή στην εμφάνιση

Στο SwiftUI, πολλές προβολές του συστήματος προσαρμόζονται αυτόματα από προεπιλογή στη φωτεινή και τη σκοτεινή λειτουργία.

Για παράδειγμα:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("Φως")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("κάποιο κείμενο")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![canvas](../../../Resource/025_view4.png)

Μπορούμε να δούμε ότι στη φωτεινή λειτουργία, το εικονίδιο και ο τίτλος συνήθως εμφανίζονται με σκούρο χρώμα.

Στη σκοτεινή λειτουργία, το εικονίδιο και ο τίτλος γίνονται αυτόματα ανοιχτόχρωμα.

Αυτό συμβαίνει επειδή προβολές SwiftUI όπως `Text` και `Image` προσαρμόζουν από προεπιλογή τα χρώματά τους σύμφωνα με την εμφάνιση του συστήματος.

Με άλλα λόγια, αν δεν ορίσουμε χειροκίνητα ένα σταθερό χρώμα, το SwiftUI θα χειριστεί για εμάς ένα μέρος της προσαρμογής εμφάνισης.

## Τα σταθερά χρώματα δεν εναλλάσσονται αυτόματα

Πρέπει να προσέξουμε ότι, αν ορίσουμε χειροκίνητα ένα σταθερό χρώμα, αυτό δεν θα εναλλάσσεται αυτόματα σύμφωνα με τη φωτεινή/σκοτεινή λειτουργία.

Για παράδειγμα:

```swift
Text("κάποιο κείμενο")
	.foregroundStyle(Color.gray)
```

Εδώ το κείμενο έχει οριστεί ως `Color.gray`, επομένως θα εμφανίζεται πάντα γκρι.

Άλλο παράδειγμα:

```swift
Text("Τίτλος")
    .foregroundStyle(Color.white)
```

Αυτός ο κώδικας θα εμφανίζει λευκό κείμενο τόσο στη φωτεινή όσο και στη σκοτεινή λειτουργία.

Αν και το φόντο είναι φωτεινό, το λευκό κείμενο μπορεί να μην είναι ευανάγνωστο.

Γι' αυτό, όταν κάνουμε προσαρμογή εμφάνισης, πρέπει να αποφεύγουμε όσο γίνεται τη βιαστική χρήση σταθερών χρωμάτων όπως `Color.white` και `Color.black`.

Σε πολλές περιπτώσεις, μπορούμε πρώτα να χρησιμοποιούμε σημασιολογικά στυλ του συστήματος, όπως:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

Συγκεκριμένα:

- Το `.primary` χρησιμοποιείται συνήθως για κύριο περιεχόμενο.
- Το `.secondary` χρησιμοποιείται συνήθως για δευτερεύον περιεχόμενο.

Αυτά προσαρμόζουν αυτόματα το αποτέλεσμα εμφάνισης σύμφωνα με τη φωτεινή/σκοτεινή λειτουργία.

## Εμφάνιση διαφορετικού περιεχομένου ανάλογα με την εμφάνιση

Μερικές φορές δεν θέλουμε μόνο να αλλάζει το χρώμα, αλλά και να εμφανίζεται διαφορετικό περιεχόμενο σε διαφορετικές εμφανίσεις.

Για παράδειγμα:

- Στη φωτεινή λειτουργία εμφανίζεται ένα απλό εικονίδιο λάμπας.
- Στη σκοτεινή λειτουργία εμφανίζεται ένα φωτεινό εικονίδιο λάμπας.
- Στη φωτεινή λειτουργία ο τίτλος εμφανίζει `Φως`.
- Στη σκοτεινή λειτουργία ο τίτλος εμφανίζει `Σκοτάδι`.

Σε αυτή την περίπτωση, χρειάζεται να ελέγξουμε αν το τρέχον σύστημα βρίσκεται σε φωτεινή ή σκοτεινή λειτουργία.

Στο SwiftUI, μπορούμε να πάρουμε την τρέχουσα λειτουργία εμφάνισης μέσω της περιβαλλοντικής τιμής `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Πλήρης κώδικας:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "Φως" : "Σκοτάδι"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("κάποιο κείμενο")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![view](../../../Resource/025_view5.png)

Από το αποτέλεσμα προεπισκόπησης μπορούμε να δούμε ότι η ίδια προβολή δεν εμφανίζει ακριβώς το ίδιο περιεχόμενο σε διαφορετικές λειτουργίες εμφάνισης.

Στη φωτεινή λειτουργία, το `Image` εμφανίζει το απλό εικονίδιο λάμπας και ο τίτλος εμφανίζεται ως `Φως`.

Στη σκοτεινή λειτουργία, το `Image` εμφανίζει το φωτεινό εικονίδιο λάμπας και ο τίτλος εμφανίζεται ως `Σκοτάδι`.

Δηλαδή, εδώ δεν αλλάζει μόνο το χρώμα, αλλά αλλάζουν και το όνομα του εικονιδίου και το ίδιο το κείμενο του τίτλου.

Αυτό συμβαίνει επειδή δεν γράψαμε απευθείας το εικονίδιο και τον τίτλο μέσα στο `body`, αλλά υπολογίσαμε διαφορετικό περιεχόμενο σύμφωνα με το `colorScheme`.

Σε αυτόν τον κώδικα, χρησιμοποιήσαμε δύο υπολογιζόμενες ιδιότητες:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "Φως" : "Σκοτάδι"
}
```

Όταν το `colorScheme` είναι `.light`, σημαίνει ότι αυτή τη στιγμή είναι φωτεινή λειτουργία.

Όταν το `colorScheme` είναι `.dark`, σημαίνει ότι αυτή τη στιγμή είναι σκοτεινή λειτουργία.

Έτσι μπορούμε, ανάλογα με τη διαφορετική λειτουργία εμφάνισης, να επιστρέφουμε διαφορετικό όνομα εικονιδίου και κείμενο τίτλου.

Αυτή είναι επίσης μια συνηθισμένη χρήση των υπολογιζόμενων ιδιοτήτων: υπολογίζουμε, σύμφωνα με την τρέχουσα κατάσταση, το περιεχόμενο που χρειάζεται να εμφανίσει η προβολή.

## Κατανόηση του @Environment

Αυτή είναι η πρώτη φορά που ερχόμαστε σε επαφή με το `@Environment`.

Το `@Environment` μπορούμε να το κατανοήσουμε ως εξής: διαβάζει μια τιμή από το περιβάλλον του SwiftUI.

Όταν εκτελείται το App, το σύστημα παρέχει πολλές πληροφορίες περιβάλλοντος, για παράδειγμα:

- Τρέχουσα γλώσσα
- Τρέχουσα λειτουργία εμφάνισης
- Τρέχουσα κατεύθυνση διάταξης
- Τρέχουσα ρύθμιση μεγέθους γραμματοσειράς

Όταν χρειάζεται να πάρουμε την τρέχουσα λειτουργία εμφάνισης, μπορούμε να διαβάσουμε το `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Όπου

```swift
\.colorScheme
```

σημαίνει ότι διαβάζουμε την τιμή `colorScheme` από το περιβάλλον.

```swift
private var colorScheme
```

σημαίνει ότι αποθηκεύουμε την τιμή που διαβάστηκε στη μεταβλητή `colorScheme`.

Το όνομα της μεταβλητής μπορεί να αποφασιστεί από εμάς, για παράδειγμα μπορεί επίσης να γραφτεί ως:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

Αρκεί το προηγούμενο `\.colorScheme` να μη αλλάξει, σημαίνει ότι διαβάζουμε την τρέχουσα λειτουργία εμφάνισης.

## Δύο συνηθισμένες τιμές του colorScheme

Το `colorScheme` έχει συνήθως δύο τιμές:

```swift
.light
.dark
```

Αυτές σημαίνουν:

- `.light`: φωτεινή λειτουργία
- `.dark`: σκοτεινή λειτουργία

Για παράδειγμα:

```swift
private var titleName: String {
    colorScheme == .light ? "Φως" : "Σκοτάδι"
}
```

Η σημασία αυτού του κώδικα είναι:

Αν αυτή τη στιγμή είναι φωτεινή λειτουργία, τότε το `colorScheme == .light` ισχύει και επιστρέφει `"Φως"`.

Διαφορετικά, επιστρέφει `"Σκοτάδι"`.

Επομένως, στη φωτεινή λειτουργία:

```swift
Text(titleName)
```

αυτό που εμφανίζεται είναι:

```swift
Φως
```

Στη σκοτεινή λειτουργία, αυτό που εμφανίζεται είναι:

```swift
Σκοτάδι
```

Με αυτόν τον τρόπο, μπορούμε να εμφανίζουμε διαφορετικό περιεχόμενο σύμφωνα με διαφορετικές εμφανίσεις.

## Παράδειγμα προσαρμογής χρώματος

Εκτός από το κείμενο και τα εικονίδια, μερικές φορές χρειάζεται να χειριστούμε μόνοι μας και την προσαρμογή χρώματος.

Για παράδειγμα:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Κείμενο")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![view](../../../Resource/025_view6.png)

Σε αυτόν τον κώδικα, το κείμενο είναι λευκό και το φόντο είναι μαύρο.

Στη φωτεινή λειτουργία, το μαύρο φόντο είναι σχετικά εμφανές.

Όμως στη σκοτεινή λειτουργία, αν το φόντο ολόκληρης της διεπαφής είναι επίσης μαύρο, αυτό το μαύρο φόντο θα ενσωματωθεί στο φόντο του συστήματος και δεν θα φαίνεται αρκετά καθαρά.

Τότε μπορούμε να αλλάξουμε το χρώμα του κειμένου και το χρώμα του φόντου σύμφωνα με τη λειτουργία εμφάνισης:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("Κείμενο")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![view](../../../Resource/025_view7.png)

Τώρα, στη φωτεινή λειτουργία έχουμε μαύρο φόντο και λευκό κείμενο.

Στη σκοτεινή λειτουργία έχουμε λευκό φόντο και μαύρο κείμενο.

Έτσι, ανεξάρτητα από την εμφάνιση, το κείμενο και το φόντο έχουν καθαρή αντίθεση και ο χρήστης μπορεί να βλέπει καθαρά το περιεχόμενο.

## Χρήση των Assets για προσαρμογή χρώματος

Εκτός από τη χειροκίνητη κρίση με το `colorScheme`, μπορούμε επίσης να χρησιμοποιήσουμε χρώματα πόρων `Assets` για να προσαρμόσουμε τη φωτεινή/σκοτεινή λειτουργία.

Αυτός ο τρόπος είναι πιο κατάλληλος για τη διαχείριση κοινών χρωμάτων μέσα στο App.

Στο έργο Xcode, βρείτε τον φάκελο πόρων `Assets`.

![assets](../../../Resource/025_color.png)

Ανοίξτε τον φάκελο `Assets`, κάντε δεξί κλικ σε μια κενή περιοχή και επιλέξτε `New Color Set` για να δημιουργήσετε έναν νέο πόρο χρώματος.

![assets](../../../Resource/025_color1.png)

Εδώ δημιουργούμε ένα χρώμα και το ονομάζουμε `redText`.

![assets](../../../Resource/025_color2.png)

Οι πόροι χρώματος μπορούν να ορίσουν ξεχωριστά χρώματα για τη φωτεινή και τη σκοτεινή λειτουργία.

![assets](../../../Resource/025_color3.png)

Αφού επιλέξουμε την αντίστοιχη περιοχή χρώματος, μπορούμε να τροποποιήσουμε το χρώμα στον επιθεωρητή ιδιοτήτων στα δεξιά.

Πατήστε το κουμπί `Hide or show the Inspectors` στην επάνω δεξιά γωνία για να ανοίξετε την περιοχή επιθεωρητών.

Στη συνέχεια επιλέξτε `Show the Attributes inspector`, βρείτε στο κάτω μέρος το `Show Color Panel` και ανοίξτε τον πίνακα χρωμάτων.

![assets](../../../Resource/025_color4.png)

Πατώντας ένα χρώμα στον πίνακα χρωμάτων, η αντίστοιχη περιοχή χρώματος του `redText` αλλάζει ταυτόχρονα.

Εδώ ρυθμίζουμε το `redText` ως εξής:

- Στη φωτεινή λειτουργία εμφανίζεται κόκκινο
- Στη σκοτεινή λειτουργία εμφανίζεται πράσινο

![assets](../../../Resource/025_color5.png)

Επιστρέφοντας στον κώδικα SwiftUI, μπορούμε να χρησιμοποιήσουμε αυτό το χρώμα ως εξής:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

Συγκεκριμένα:

```swift
Color("redText")
```

σημαίνει ότι διαβάζουμε από τα `Assets` τον πόρο χρώματος με όνομα `redText`.

Αποτέλεσμα εμφάνισης:

![assets](../../../Resource/025_view8.png)

Μπορούμε να δούμε ότι το ίδιο `Color("redText")` εμφανίζει διαφορετικό χρώμα στη φωτεινή και στη σκοτεινή λειτουργία.

Αυτό δείχνει ότι και τα ίδια τα χρώματα `Assets` υποστηρίζουν προσαρμογή εμφάνισης.

## Η διαφορά ανάμεσα σε colorScheme και Assets

Το `colorScheme` και τα `Assets` μπορούν και τα δύο να υλοποιήσουν προσαρμογή εμφάνισης, αλλά ταιριάζουν σε διαφορετικές περιπτώσεις.

Αν θέλουμε μόνο να αλλάζουμε περιεχόμενο κειμένου, όνομα εικόνας ή όνομα `SF Symbols` σύμφωνα με τη φωτεινή/σκοτεινή λειτουργία, μπορούμε να χρησιμοποιήσουμε το `colorScheme`.

Για παράδειγμα:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

Αν πρόκειται για χρώματα που χρησιμοποιούνται συχνά μέσα στο App, όπως χρώμα θέματος, χρώμα φόντου καρτών ή χρώμα κειμένου, προτείνεται περισσότερο η χρήση χρωμάτων `Assets`.

Επειδή τα χρώματα `Assets` μπορούν να επαναχρησιμοποιηθούν σε πολλές προβολές και είναι πιο βολικό να τροποποιηθούν αργότερα.

Για παράδειγμα:

```swift
Color("redText")
```

Έτσι χρειάζεται μόνο να τροποποιήσουμε το χρώμα στα `Assets`, και όλα τα σημεία που χρησιμοποιούν αυτό το χρώμα θα αλλάξουν μαζί.

## Σύνοψη

Σε αυτό το μάθημα μάθαμε την προσαρμογή εμφάνισης για τη φωτεινή και τη σκοτεινή λειτουργία.

Αρχικά μάθαμε πώς να κάνουμε προεπισκόπηση διαφορετικών εμφανίσεων στο `Canvas` του Xcode και πώς να χρησιμοποιούμε το `Color Scheme Variants` για να βλέπουμε ταυτόχρονα τη φωτεινή και τη σκοτεινή λειτουργία.

Στη συνέχεια μάθαμε ότι προβολές SwiftUI όπως `Text` και `Image` προσαρμόζουν αυτόματα από προεπιλογή τα χρώματά τους σύμφωνα με την εμφάνιση του συστήματος.

Όμως, αν ορίσουμε χειροκίνητα σταθερά χρώματα, όπως `Color.white` ή `Color.black`, αυτά τα χρώματα δεν θα εναλλάσσονται αυτόματα.

Έπειτα μάθαμε το `@Environment` και το `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Μέσω του `colorScheme`, μπορούμε να κρίνουμε αν αυτή τη στιγμή είναι φωτεινή ή σκοτεινή λειτουργία και να εμφανίζουμε διαφορετικά εικονίδια, κείμενο ή χρώματα σύμφωνα με τη διαφορετική εμφάνιση.

Τέλος, μάθαμε τα χρώματα `Assets`.

Τα χρώματα `Assets` μπορούν να ορίζουν ξεχωριστά χρώματα για τη φωτεινή και τη σκοτεινή λειτουργία και είναι κατάλληλα για τη διαχείριση κοινών χρωματικών πόρων μέσα στο App.

Η προσαρμογή εμφάνισης μπορεί να κάνει το App να έχει καλύτερο αποτέλεσμα εμφάνισης τόσο την ημέρα όσο και τη νύχτα, και μπορεί επίσης να αποτρέψει προβλήματα όπως δυσανάγνωστο κείμενο ή μη εμφανές φόντο.

Στην πραγματική ανάπτυξη, προτείνεται να δίνουμε προτεραιότητα στις προεπιλεγμένες δυνατότητες προσαρμογής του SwiftUI και στα χρώματα `Assets`.

Όταν σε διαφορετικές εμφανίσεις χρειάζεται να αλλάξουμε κείμενο, εικονίδια, εικόνες ή να χειριστούμε λίγα ειδικά οπτικά αποτελέσματα, τότε χρησιμοποιούμε το `colorScheme` για κρίση.

## Επεκταμένη γνώση: ορισμός μεγέθους γραμματοσειράς

Όταν μαθαίναμε προηγουμένως το `font`, χρησιμοποιήσαμε μερικά συστημικά στυλ γραμματοσειράς:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

Εκτός από τη χρήση συστημικών στυλ γραμματοσειράς όπως `.largeTitle` και `.callout`, μπορούμε επίσης να ορίσουμε χειροκίνητα το μέγεθος της γραμματοσειράς.

Για παράδειγμα:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

Εδώ το:

```swift
.font(.system(size: 100))
```

σημαίνει ότι ορίζουμε το μέγεθος της γραμματοσειράς σε `100 pt`.

Για τα εικονίδια `SF Symbols`, το `font` επηρεάζει επίσης το μέγεθος του εικονιδίου.

Επομένως, αν θέλουμε το εικονίδιο να εμφανίζεται μεγαλύτερο, μπορούμε να ορίσουμε το μέγεθος μέσω του `.font(.system(size:))`.

Για παράδειγμα:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

Πρέπει να προσέξουμε ότι, αν και ο χειροκίνητος ορισμός μεγέθους γραμματοσειράς είναι πιο ευέλικτος, δεν προτείνεται να χρησιμοποιούμε μαζικά σταθερά μεγέθη για κανονικό κείμενο.

Για κανονικό κείμενο είναι προτιμότερο να χρησιμοποιούμε πρώτα συστημικά στυλ γραμματοσειράς όπως `.title`, `.headline`, `.body` και `.caption`.

Έτσι μπορούμε να προσαρμοζόμαστε καλύτερα σε διαφορετικές συσκευές και στις ρυθμίσεις μεγέθους γραμματοσειράς του χρήστη.
