# Εφαρμογή Fill Light

Σε αυτό το μάθημα, θα δημιουργήσουμε μια πολύ ενδιαφέρουσα εφαρμογή fill light. Όταν πέφτει η νύχτα, μπορούμε να κάνουμε την οθόνη του τηλεφώνου να εμφανίζει διαφορετικά χρώματα και να τη χρησιμοποιούμε ως ένα απλό fill light.

Αυτή η εφαρμογή fill light μπορεί να αλλάζει χρώματα πατώντας την οθόνη και επίσης μπορεί να ρυθμίζει τη φωτεινότητα με ένα ρυθμιστικό.

Σε αυτό το παράδειγμα, θα μάθουμε πώς να χρησιμοποιούμε το `brightness` για να ρυθμίζουμε τη φωτεινότητα μιας προβολής, το `onTapGesture` για να προσθέτουμε μια χειρονομία πατήματος σε μια προβολή, και το στοιχείο ελέγχου `Slider`.

Αποτέλεσμα:

![Color](../../Resource/018_color.png)

## Εμφάνιση χρώματος

Πρώτα, ας κάνουμε την προβολή να εμφανίζει ένα χρώμα.

Στο SwiftUI, το `Color` δεν αντιπροσωπεύει μόνο ένα χρώμα, αλλά μπορεί επίσης να εμφανιστεί ως προβολή:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

Εδώ, το `Color.red` αντιπροσωπεύει μια κόκκινη προβολή. Το `.ignoresSafeArea()` κάνει την προβολή χρώματος να γεμίζει ολόκληρη την οθόνη, έτσι ώστε να μοιάζει περισσότερο με πραγματικό εφέ fill light.

Αποτέλεσμα:

![Color](../../Resource/018_color1.png)

### Πίνακας χρωμάτων και δείκτης

Αυτή τη στιγμή εμφανίζεται μόνο ένα χρώμα. Όμως ένα fill light συνήθως δεν έχει μόνο ένα χρώμα. Μπορεί επίσης να εμφανίζει μπλε, κίτρινο, μωβ, λευκό και άλλα χρώματα.

Θέλουμε να αλλάζουμε ανάμεσα σε διαφορετικά χρώματα όταν πατιέται η οθόνη. Μπορούμε να βάλουμε αυτά τα χρώματα σε έναν πίνακα για ενιαία διαχείριση:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

Ένας πίνακας είναι κατάλληλος για την αποθήκευση "μιας ομάδας δεδομένων του ίδιου τύπου". Εδώ, κάθε στοιχείο του πίνακα είναι ένα `Color`.

Αν θέλουμε να εμφανίσουμε ένα συγκεκριμένο χρώμα, μπορούμε να χρησιμοποιήσουμε έναν δείκτη:

```swift
colors[0]
```

Αυτό σημαίνει ότι διαβάζουμε το χρώμα στο δείκτη `0` του πίνακα, δηλαδή το πρώτο χρώμα.

Τώρα ο κώδικας μπορεί να γραφτεί έτσι:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

Με αυτόν τον τρόπο, η οθόνη εμφανίζει το πρώτο χρώμα του πίνακα, δηλαδή το κόκκινο.

### Χρήση του δείκτη για έλεγχο του χρώματος

Αν χρειάζεται να αλλάζουμε ανάμεσα σε διαφορετικά χρώματα, χρειαζόμαστε μια μεταβλητή για να διαχειριζόμαστε τον δείκτη αντί να γράφουμε τον δείκτη σταθερά.

Μπορούμε να χρησιμοποιήσουμε `@State` για να δηλώσουμε μια μεταβλητή που αποθηκεύει τον δείκτη:

```swift
@State private var index = 0
```

Εδώ, το `index` αντιπροσωπεύει τον δείκτη του τρέχοντος χρώματος.

Όταν αλλάζει το `index`, το SwiftUI υπολογίζει ξανά το περιβάλλον εργασίας και ενημερώνει το περιεχόμενο που εμφανίζεται.

Στη συνέχεια αλλάζουμε το αρχικό `colors[0]` σε:

```swift
colors[index]
```

Με αυτόν τον τρόπο, το χρώμα που εμφανίζεται στην προβολή καθορίζεται από το `index`.

Τώρα ο κώδικας γίνεται:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

Όταν αλλάζει το `index`, το `colors[index]` θα εμφανίζει επίσης διαφορετικό χρώμα.

Για παράδειγμα:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

Πρέπει να σημειωθεί ότι το `index` δεν μπορεί να ξεπεράσει τον μέγιστο δείκτη του πίνακα, διαφορετικά θα προκύψει σφάλμα δείκτη εκτός ορίων.

## Χειρονομία πατήματος

Τώρα μπορούμε να εμφανίζουμε διαφορετικά χρώματα ανάλογα με το `index`, αλλά ακόμα δεν μπορούμε να κάνουμε εναλλαγή πατώντας.

Στο προηγούμενο μάθημα "Quote Carousel", χρησιμοποιήσαμε `Button` για να ελέγξουμε την εναλλαγή των αποσπασμάτων.

Αυτή τη φορά όμως θέλουμε να "πατάμε ολόκληρη την περιοχή του χρώματος" για να αλλάζει το χρώμα, οπότε το `onTapGesture` είναι πιο κατάλληλο.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

Όταν πατάμε την προβολή χρώματος, εκτελείται αυτός ο κώδικας:

```swift
index += 1
```

Αυτό σημαίνει ότι αυξάνουμε το `index` κατά `1`. Αφού αυξηθεί ο δείκτης, το `colors[index]` θα εμφανίσει το επόμενο χρώμα του πίνακα.

### `onTapGesture`

Το `onTapGesture` είναι ένας τροποποιητής χειρονομίας που προσθέτει μια ενέργεια πατήματος σε μια προβολή.

Βασική χρήση:

```swift
.onTapGesture {
    // code
}
```

Για παράδειγμα:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

Όταν πατηθεί αυτή η κόκκινη προβολή, εκτελείται ο κώδικας μέσα στις αγκύλες και στην κονσόλα εμφανίζεται:

```swift
Click color
```

Μέσω του `onTapGesture`, μπορούμε να ορίσουμε τι θα συμβεί αφού πατηθεί μια προβολή.

### Διαφορά από το `Button`

Νωρίτερα μάθαμε για την προβολή `Button`. Τόσο το `Button` όσο και το `onTapGesture` μπορούν να χειριστούν ενέργειες πατήματος, αλλά τα σενάρια χρήσης τους δεν είναι ακριβώς τα ίδια.

Το `onTapGesture` είναι πιο κατάλληλο για "να προσθέτουμε δυνατότητα πατήματος σε μια υπάρχουσα προβολή", όπως `Color`, `Image`, `Text` ή άλλη συνηθισμένη προβολή.

Το `Button` είναι πιο κατάλληλο όταν θέλουμε να εκφράσουμε ένα σαφές κουμπί, όπως "Επιβεβαίωση", "Υποβολή" ή "Διαγραφή".

Σε αυτή την εφαρμογή fill light, θέλουμε η λειτουργία αλλαγής χρώματος να είναι πιο απλή. Το πάτημα ολόκληρης της χρωματικής περιοχής αλλάζει το χρώμα, επομένως το `onTapGesture` είναι καλή επιλογή εδώ.

## Το πρόβλημα του δείκτη

Τώρα μπορούμε να πατάμε την οθόνη για να αλλάζουμε διαφορετικά χρώματα.

Όμως εδώ υπάρχει ένα σημαντικό πρόβλημα: **ο δείκτης μπορεί να βγει έξω από το εύρος του πίνακα**.

Για παράδειγμα:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

Αν συνεχίσουμε να πατάμε την οθόνη, το `index` τελικά θα γίνει `4`, και τότε θα προκύψει σφάλμα "index out of range".

Αυτό συμβαίνει επειδή ο πίνακας `colors` έχει `4` στοιχεία, αλλά η αρίθμηση των δεικτών ξεκινά από το `0`, άρα το έγκυρο εύρος δεικτών είναι `0 - 3`, όχι `4`.

Αν προσπελάσουμε το `colors[4]`, θα προκύψει σφάλμα "index out of range".

Στον τρέχοντα κώδικα, κάθε πάτημα αυξάνει το `index` κατά `1`. Αν δεν το χειριστούμε, στο τέλος σίγουρα θα ξεπεράσει το έγκυρο εύρος.

Γι' αυτό πρέπει να ελέγχουμε τον δείκτη όταν πατιέται η οθόνη: αν είναι ήδη το τελευταίο χρώμα, να επιστρέφει στο πρώτο, διαφορετικά να συνεχίζει αυξάνοντας κατά `1`.

Μπορούμε να το υλοποιήσουμε με μια εντολή `if`:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

Σε αυτόν τον κώδικα, το `colors.count` σημαίνει τον αριθμό των στοιχείων στον πίνακα.

Στον τρέχοντα πίνακα υπάρχουν 4 χρώματα, άρα:

```swift
colors.count // 4
```

Αλλά ο μέγιστος δείκτης δεν είναι `4`, είναι `3`, επειδή η αρίθμηση ξεκινά από το `0`.

Άρα ο τελευταίος δείκτης πρέπει να γραφτεί ως:

```swift
colors.count - 1
```

Δηλαδή:

```swift
4 - 1 = 3
```

Αυτή η λογική σημαίνει: αν ο τρέχων δείκτης αντιστοιχεί ήδη στο τελευταίο χρώμα, επαναφέρετε τον δείκτη σε `0`, διαφορετικά αυξήστε τον κατά `1`.

Με αυτόν τον τρόπο, τα χρώματα μπορούν να αλλάζουν κυκλικά.

### Βελτιστοποίηση της λογικής του δείκτη

Αν θέλουμε ο κώδικας να είναι πιο σύντομος, μπορούμε επίσης να χρησιμοποιήσουμε τον τριαδικό τελεστή:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

Αυτός ο κώδικας σημαίνει: αν το `index == colors.count - 1` είναι αληθές, επέστρεψε `0`. Διαφορετικά, επέστρεψε `index + 1`.

Τέλος, αντιστοιχίζουμε ξανά το αποτέλεσμα στο `index`.

Τώρα μπορούμε να πετύχουμε το αποτέλεσμα της αλλαγής χρωμάτων.

Πλήρης κώδικας:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

Τώρα μπορούμε να πατάμε την οθόνη και να αλλάζουμε ανάμεσα σε διαφορετικά χρώματα. Μια βασική εφαρμογή fill light έχει ολοκληρωθεί.

## Εμφάνιση του ονόματος του χρώματος

Μπορούμε να προσθέσουμε μια ακόμη ομάδα κειμένων που αντιστοιχούν στα χρώματα, ώστε όταν αλλάζει το χρώμα, να εμφανίζεται και το όνομα του τρέχοντος χρώματος στην οθόνη.

Για παράδειγμα:

- Το κόκκινο εμφανίζει `Red`
- Το μπλε εμφανίζει `Blue`
- Το κίτρινο εμφανίζει `Yellow`
- Το μωβ εμφανίζει `Purple`

Και εδώ μπορούμε να χρησιμοποιήσουμε έναν πίνακα για να αποθηκεύσουμε τα ονόματα των χρωμάτων:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

Η σειρά των κειμένων σε αυτόν τον πίνακα πρέπει να αντιστοιχεί ακριβώς στη σειρά των χρωμάτων στον πίνακα χρωμάτων.

Στη συνέχεια μπορούμε να χρησιμοποιήσουμε `Text` για να εμφανίσουμε το όνομα του χρώματος για τον τρέχοντα δείκτη:

```swift
Text(colorsName[index])
```

Το `Text` εμφανίζει το τρέχον όνομα χρώματος σύμφωνα με το `index`.

Χρησιμοποιήστε τροποποιητές για να βελτιώσετε την εμφάνιση του `Text`:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

Αυτό κάνει το `Text` να εμφανίζεται σε λευκό χρώμα, με μέγεθος τίτλου και έντονη γραφή.

Τώρα έχουμε μια `Color` προβολή πλήρους οθόνης. Αν θέλουμε το `Text` να εμφανίζεται πάνω από την `Color` προβολή, πρέπει να χρησιμοποιήσουμε το δοχείο διάταξης `ZStack` για να τα τοποθετήσουμε το ένα πάνω στο άλλο.

```swift
ZStack {
    Color
    Text
}
```

Έτσι ο κώδικας γίνεται:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

Με αυτόν τον τρόπο, το φόντο είναι η προβολή χρώματος και από πάνω της επικαλύπτεται μια προβολή κειμένου.

Αποτέλεσμα:

![Color](../../Resource/018_color2.png)

Ένα σημαντικό σημείο είναι ότι σε ένα `ZStack`, οι προβολές που γράφονται αργότερα εμφανίζονται συνήθως μπροστά. Αν το `Text` γραφτεί πριν από το `Color`, η επόμενη `Color` προβολή μπορεί να καλύψει το `Text`.

## Έλεγχος φωτεινότητας

Τώρα μπορούμε να αλλάζουμε διαφορετικά χρώματα, αλλά ένα fill light έχει και ένα άλλο σημαντικό χαρακτηριστικό: **ρύθμιση φωτεινότητας**.

Στο SwiftUI, μπορούμε να χρησιμοποιήσουμε τον τροποποιητή `brightness` για να ρυθμίσουμε τη φωτεινότητα μιας προβολής.

Για παράδειγμα:

```swift
.brightness(1)
```

Μπορούμε να γράψουμε:

```swift
colors[index]
    .brightness(0.5)
```

Αυτό κάνει το τρέχον χρώμα πιο φωτεινό και πιο κοντά στο αποτέλεσμα ενός fill light.

Το εύρος φωτεινότητας είναι `0 - 1`. Το `0` σημαίνει ότι διατηρείται το αρχικό χρώμα, και όσο πιο κοντά είναι η τιμή στο `1`, τόσο πιο φωτεινό γίνεται το χρώμα. Το `1` σημαίνει το πιο φωτεινό λευκό αποτέλεσμα.

Παρόλο που μπορούμε να ελέγξουμε το `brightness` στον κώδικα, ο χρήστης δεν μπορεί ακόμη να το ρυθμίσει απευθείας.

Γι' αυτό πρέπει να προσθέσουμε ένα στοιχείο ελέγχου που σύρεται: `Slider`.

## Προβολή `Slider`

Στο SwiftUI, το `Slider` είναι ένα στοιχείο ελέγχου που χρησιμοποιείται για την επιλογή μιας τιμής μέσα σε ένα εύρος. Η Apple το περιγράφει ως "ένα στοιχείο ελέγχου για την επιλογή μιας τιμής από ένα οριοθετημένο γραμμικό εύρος".

Βασική χρήση:

```swift
Slider(value: $value, in: 0...1)
```

Επεξήγηση παραμέτρων:

1. `value: $value`: Το `Slider` πρέπει να είναι συνδεδεμένο με μια μεταβλητή.

    Όταν σύρεται το ρυθμιστικό, η τιμή της μεταβλητής αλλάζει ταυτόχρονα. Αντίστροφα, αν αλλάξει η μεταβλητή, ενημερώνεται και το ρυθμιστικό.

    Αυτό μοιάζει πολύ με το `TextField` που μάθαμε νωρίτερα. Και τα δύο "συνδέουν ένα στοιχείο ελέγχου με μια μεταβλητή".

    Μπροστά από τη συνδεδεμένη μεταβλητή χρειάζεται το σύμβολο `$` για να δηλωθεί το binding.

2. `in: 0...1`: Αυτή η παράμετρος αντιπροσωπεύει το εύρος τιμών του ρυθμιστικού.

    Εδώ, το `0...1` σημαίνει ότι η ελάχιστη τιμή είναι `0` και η μέγιστη τιμή είναι `1`.

    Όταν το ρυθμιστικό σύρεται εντελώς αριστερά, η συνδεδεμένη μεταβλητή είναι κοντά στο `0`. Όταν σύρεται εντελώς δεξιά, είναι κοντά στο `1`.

Για παράδειγμα:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

Το `Slider` είναι συνδεδεμένο με τη μεταβλητή `value`. Όταν σύρεται το ρυθμιστικό, η τιμή της `value` αλλάζει ταυτόχρονα.

Εμφάνιση:

![Slider](../../Resource/018_slider.png)

Όταν το `Slider` μετακινείται προς τα αριστερά, η συνδεδεμένη τιμή `value` γίνεται `0`. Όταν μετακινείται προς τα δεξιά, γίνεται `1`.

### Εύρος τιμών

Το εύρος τιμών του `Slider` δεν είναι σταθερό. Μπορεί επίσης να γραφτεί ως:

```swift
0...100
```

ή ως άλλο εύρος.

Αλλά σε αυτή την εφαρμογή fill light, πρέπει να ελέγχουμε τη φωτεινότητα, οπότε το `0...1` είναι η πιο κατάλληλη επιλογή.

## Χρήση του `Slider` για έλεγχο φωτεινότητας

Τώρα πρέπει να συνδέσουμε το `Slider` με το `brightness`.

Πρώτα, δημιουργήστε μια μεταβλητή για να αποθηκεύσετε την τιμή της φωτεινότητας:

```swift
@State private var slider = 0.0
```

Εδώ, το `0.0` είναι μια τιμή τύπου `Double`.

Επειδή το `Slider` συνήθως συνδέεται με έναν αριθμητικό τύπο, και εδώ θέλουμε να αλλάζει συνεχώς, το `Double` είναι πιο κατάλληλο. Επίσης, το `brightness` μπορεί να δεχτεί μόνο μια τιμή τύπου `Double`.

Έπειτα περάστε αυτή την τιμή στο `brightness`:

```swift
colors[index]
    .brightness(slider)
```

Όταν `slider == 0`, το χρώμα παραμένει στην προεπιλεγμένη του κατάσταση. Όσο πιο κοντά είναι το `slider` στο `1`, τόσο πιο φωτεινό φαίνεται το χρώμα.

### Προσθήκη του στοιχείου ελέγχου `Slider`

Στη συνέχεια, προσθέστε ένα στοιχείο ελέγχου `Slider` για να τροποποιείτε αυτή τη μεταβλητή:

```swift
Slider(value: $slider, in: 0...1)
```

Όταν αλλάζει το ρυθμιστικό, αλλάζει και η τιμή του `slider`, και το `brightness(slider)` ενημερώνει τη φωτεινότητα ταυτόχρονα.

Αυτό είναι ένα πολύ τυπικό παράδειγμα του ότι "οι μεταβλητές οδηγούν την προβολή" στο SwiftUI.

### Ρύθμιση της εμφάνισης του `Slider`

Από προεπιλογή, το `Slider` χρησιμοποιεί το διαθέσιμο πλάτος.

Μπορούμε να του δώσουμε σταθερό πλάτος:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

Στη συνέχεια, προσθέτουμε μερικούς ακόμη τροποποιητές για να φαίνεται πιο καθαρά:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

Αυτό δίνει στο ρυθμιστικό λευκό φόντο και στρογγυλεμένες γωνίες, ώστε να ξεχωρίζει πιο καθαρά πάνω στην προβολή `Color`.

Τέλος, τοποθετήστε το στο κάτω μέρος της οθόνης.

Εφόσον χρησιμοποιήσαμε ήδη `ZStack`, μπορούμε να τοποθετήσουμε μέσα του ένα `VStack` και να χρησιμοποιήσουμε `Spacer()` για να σπρώξουμε το `Slider` προς τα κάτω.

## Πλήρης κώδικας

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## Σύνοψη

Χρησιμοποιώντας τις γνώσεις που μάθαμε προηγουμένως μαζί με βασικές έννοιες όπως τα χρώματα και οι πίνακες, δημιουργήσαμε μια πολύ ενδιαφέρουσα εφαρμογή fill light.

Μέσα από αυτή την εφαρμογή, μάθαμε πώς να χρησιμοποιούμε το `brightness` για ρύθμιση φωτεινότητας, το `onTapGesture` για προσθήκη ενεργειών πατήματος σε προβολές και το στοιχείο ελέγχου `Slider`.

Προσθέσαμε το `onTapGesture` στην προβολή `Color` για να αλλάζουμε χρώματα. Χρησιμοποιήσαμε επίσης το στοιχείο ελέγχου `Slider` για να αλλάζουμε μια μεταβλητή που διαχειρίζεται το `brightness`, κάτι που είναι άλλο ένα παράδειγμα του ότι "οι μεταβλητές οδηγούν την προβολή".

Επίσης επαναλάβαμε τον τριαδικό τελεστή, χρησιμοποιήσαμε `ZStack` για να στοιβάζουμε προβολές και χρησιμοποιήσαμε πίνακες για να διαχειριζόμαστε μια ομάδα δεδομένων του ίδιου τύπου. Αυτό βοηθά να εμβαθύνουμε την κατανόησή μας για τους πίνακες και τους δείκτες, συμπεριλαμβανομένου του πρακτικού ζητήματος της αποφυγής σφαλμάτων δείκτη εκτός ορίων.

Παρόλο που αυτό το παράδειγμα δεν είναι περίπλοκο, συνδέει πολλές από τις βασικές ιδέες που μάθαμε προηγουμένως. Όταν συνδυάζονται σε ένα μικρό πραγματικό έργο, γίνεται πιο εύκολο να καταλάβουμε σε τι χρησιμεύει κάθε έννοια.

### Σενάριο πραγματικής χρήσης

Φανταστείτε να τοποθετείτε ένα παλιό iPhone στο γραφείο και να χρησιμοποιείτε την εφαρμογή fill light που αναπτύξατε εσείς οι ίδιοι για να ελέγχετε το χρώμα του φωτός. Αυτή θα ήταν μια πολύ όμορφη εμπειρία.

Στο App Store υπάρχουν πολλές εφαρμογές "fill light", και ούτε αυτές φαίνονται πολύ περίπλοκες.

![AppStore](../../Resource/018_appStore.PNG)

Μπορούμε να ξεκινήσουμε αναπτύσσοντας απλές εφαρμογές και να προσπαθήσουμε να τις δημοσιεύσουμε στο App Store. Αυτό όχι μόνο αυξάνει το ενδιαφέρον μας για την ανάπτυξη, αλλά καταγράφει και την πορεία της εξέλιξής μας.

### Άσκηση μετά το μάθημα

Μπορείτε να συνεχίσετε να σκέφτεστε τρόπους για να επεκτείνετε αυτή την εφαρμογή fill light, όπως:

- Προσθήκη περισσότερων χρωμάτων
- Εμφάνιση της τρέχουσας τιμής φωτεινότητας
- Βελτίωση του σχεδιασμού της κάτω περιοχής του ρυθμιστικού

Όταν αρχίσετε πραγματικά να χρησιμοποιείτε αυτές τις γνώσεις, θα ανακαλύψετε ότι κάθε έννοια που μαθαίνουμε είναι στην πραγματικότητα ένα εργαλείο για τη δημιουργία εφαρμογών.

Όσο περισσότερα εργαλεία κατακτούμε, τόσο περισσότερες λειτουργίες μπορούμε να δημιουργήσουμε.
