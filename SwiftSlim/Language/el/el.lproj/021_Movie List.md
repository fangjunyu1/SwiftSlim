# Λίστα Ταινιών

Σε αυτό το μάθημα, θα δημιουργήσουμε μια λίστα ταινιών.

![movie](../../Resource/021_movie.png)

Στη λίστα θα εμφανίζονται η αφίσα, το όνομα, ο σκηνοθέτης και η βαθμολογία κάθε ταινίας.

Σε αυτό το παράδειγμα, θα μάθουμε ένα πολύ σημαντικό σημείο γνώσης: τη δομή `struct`. Μπορεί να μας βοηθήσει να συνδυάσουμε πολλαπλές πληροφορίες μιας ταινίας σε ένα ενιαίο σύνολο. Εκτός από αυτό, θα γνωρίσουμε επίσης τα `UUID()`, `ForEach`, τη διαχωριστική γραμμή `Divider`, καθώς και το πώς να χρησιμοποιούμε προσαρμοσμένα αντικείμενα για τη διαχείριση δεδομένων.

Αυτές οι γνώσεις είναι πολύ συνηθισμένες στη μετέπειτα ανάπτυξη με SwiftUI. Αφού ολοκληρώσεις αυτό το μάθημα, όχι μόνο θα μπορείς να φτιάξεις μια λίστα ταινιών, αλλά θα αρχίσεις επίσης να κατανοείς «πώς να εμφανίζεις ένα σύνολο δεδομένων ως διεπαφή».

## Μία μόνο ταινία

Μπορούμε πρώτα να ξεκινήσουμε από τη διεπαφή μιας μεμονωμένης ταινίας.

![movie](../../Resource/021_movie1.png)

Η διάταξη αυτής της διεπαφής αποτελείται κυρίως από δύο μέρη: στα αριστερά είναι η αφίσα της ταινίας και στα δεξιά είναι η παρουσίαση της ταινίας.

### Αφίσα ταινίας

Στα αριστερά εμφανίζεται η αφίσα της ταινίας και μπορούμε να χρησιμοποιήσουμε το `Image` για να εμφανίσουμε την εικόνα.

Για παράδειγμα:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

Αυτός ο κώδικας σημαίνει ότι εμφανίζεται μια εικόνα με όνομα `"The Shawshank Redemption"`.

Εδώ πρέπει να προσέξουμε ότι το όνομα της εικόνας πρέπει να υπάρχει πραγματικά στον φάκελο πόρων `Assets`, διαφορετικά η διεπαφή δεν θα μπορέσει να εμφανίσει σωστά αυτή την εικόνα.

![movie](../../Resource/021_movie2.png)

Οι λειτουργίες αυτών των τροποποιητών είναι αντίστοιχα:

- `resizable()` σημαίνει ότι το μέγεθος της εικόνας μπορεί να αλλάξει.
- `scaledToFit()` σημαίνει ότι κατά την κλιμάκωση διατηρείται η αρχική αναλογία, ώστε η εικόνα να μην τεντώνεται και παραμορφώνεται.
- `frame(height: 180)` σημαίνει ότι το ύψος της εικόνας ορίζεται στο 180.
- `cornerRadius(10)` σημαίνει ότι ορίζεται ακτίνα γωνίας 10 στην εικόνα.

Έτσι, μπορούμε να αποκτήσουμε μια εικόνα αφίσας ταινίας με κατάλληλο μέγεθος και στρογγυλεμένες γωνίες.

![movie](../../Resource/021_movie3.png)

### Παρουσίαση ταινίας

Στα δεξιά εμφανίζεται η παρουσίαση της ταινίας, συμπεριλαμβανομένων του ονόματος της ταινίας, του σκηνοθέτη και της βαθμολογίας.

![movie](../../Resource/021_movie4.png)

Μπορούμε να χρησιμοποιήσουμε το `Text` για να εμφανίσουμε αυτό το μέρος:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Σε αυτόν τον κώδικα, το `.font` δηλώνει το μέγεθος της γραμματοσειράς, ενώ το `.fontWeight` δηλώνει το πάχος της γραμματοσειράς.

Για να υπάρχει πιο καθαρό κενό ανάμεσα στο «όνομα της ταινίας», τις «πληροφορίες σκηνοθέτη» και τις «πληροφορίες βαθμολογίας», εδώ χρησιμοποιούμε `VStack` στο εξωτερικό επίπεδο και ορίζουμε:

```swift
spacing: 10
```

Αυτό σημαίνει ότι η απόσταση μεταξύ κάθε ομάδας περιεχομένου μέσα σε αυτό το `VStack` είναι 10.

Ο λόγος που ο σκηνοθέτης και η βαθμολογία τυλίγονται ξανά ο καθένας σε ένα ξεχωριστό `VStack` είναι ότι και τα δύο ανήκουν σε μια δομή «τίτλος + περιεχόμενο». Έτσι, τα επίπεδα της διεπαφής γίνονται πιο καθαρά και είναι πιο εύκολο να συνεχίσουμε αργότερα να προσαρμόζουμε το στυλ.

Επιπλέον, το `VStack` είναι από προεπιλογή κεντραρισμένο. Για να ευθυγραμμίσουμε όλο το κείμενο προς τα αριστερά, ορίζουμε:

```swift
alignment: .leading
```

Έτσι, το μέρος της παρουσίασης της ταινίας φαίνεται πιο τακτοποιημένο.

### Διαχωριστική γραμμή

Τώρα, παρόλο που το περιεχόμενο στην παρουσίαση της ταινίας έχει απομακρυνθεί με το `spacing`, το όριο ανάμεσα στα διαφορετικά περιεχόμενα εξακολουθεί να μην είναι αρκετά σαφές.

Σε αυτή την περίπτωση, μπορούμε να προσθέσουμε μια διαχωριστική γραμμή:

```swift
Divider()
```

Για παράδειγμα:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Έτσι, το μέρος της παρουσίασης της ταινίας θα αποκτήσει ένα πιο εμφανές διαχωριστικό αποτέλεσμα.

![movie](../../Resource/021_movie4.png)

Το `Divider` είναι μια πολύ απλή αλλά πολύ συχνά χρησιμοποιούμενη προβολή, της οποίας η λειτουργία είναι να χωρίζει διαφορετικά περιεχόμενα.

Μέσα σε `VStack`, το `Divider()` εμφανίζεται ως οριζόντια γραμμή.

Μέσα σε `HStack`, το `Divider()` εμφανίζεται ως κατακόρυφη γραμμή.

Επιπλέον, μπορούμε επίσης να τροποποιήσουμε το στυλ του `Divider` μέσω των `.frame`, `.background`, `.padding` και άλλων τρόπων.

Για παράδειγμα:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

Αυτό θα δημιουργήσει μια μπλε διαχωριστική γραμμή με πάχος 2 και εσωτερικό περιθώριο αριστερά και δεξιά.

![divider](../../Resource/021_divider.png)

### Πλήρης διεπαφή

Τέλος, μπορούμε να χρησιμοποιήσουμε το `HStack` για να τοποθετήσουμε την αφίσα της ταινίας και την παρουσίαση της ταινίας αριστερά και δεξιά, δίπλα-δίπλα.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

Εδώ, το `HStack(spacing: 20)` σημαίνει ότι διατηρείται απόσταση 20 ανάμεσα στο αριστερό και το δεξί μέρος.

Πρέπει να προσέξουμε ότι το `Divider()`, όπως και τα `TextField` και `Slider` που είδαμε προηγουμένως, από προεπιλογή προσπαθεί να καταλάβει όσο το δυνατόν περισσότερο διαθέσιμο χώρο.

Γι' αυτό ορίζουμε ένα σταθερό πλάτος στο εξωτερικό `VStack` του δεξιού μέρους της παρουσίασης:

```swift
.frame(width: 200)
```

Έτσι μπορούμε να κρατήσουμε το πλάτος της διαχωριστικής γραμμής και της περιοχής κειμένου ίδιο, ώστε το σύνολο να φαίνεται πιο τακτοποιημένο.

Μέχρι εδώ, έχουμε ολοκληρώσει τη διεπαφή εμφάνισης για «μία μόνο ταινία».

![movie](../../Resource/021_movie1.png)

## Αποθήκευση ταινιών σε πίνακα

Αν θέλουμε να εμφανίσουμε πολλές ταινίες διαδοχικά με το ίδιο στυλ, αυτό σημαίνει ότι πρέπει να γράψουμε παρόμοιο κώδικα για κάθε ταινία.

Για παράδειγμα:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

Αν και ο κώδικας που γράφεται με αυτόν τον τρόπο μπορεί να πετύχει το αποτέλεσμα, θα γίνεται όλο και περισσότερος και η συντήρησή του θα είναι πολύ δύσκολη.

Για παράδειγμα, αν χρειαστεί να αλλάξουμε την απόσταση όλων των ταινιών από `20` σε `15`, ή να αλλάξουμε το πλάτος της δεξιάς πλευράς από `200` σε
`220`, τότε θα πρέπει να αλλάξουμε χειροκίνητα κάθε επαναλαμβανόμενο κομμάτι κώδικα.

Αυτό προφανώς δεν είναι μια αποδοτική μέθοδος.

Προηγουμένως μάθαμε τους πίνακες και μάθαμε επίσης να χρησιμοποιούμε το `ForEach` για να εμφανίζουμε επαναλαμβανόμενα προβολές με βάση έναν πίνακα.

Για παράδειγμα:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Αυτός ο τρόπος είναι πολύ κατάλληλος για την περίπτωση όπου υπάρχει «μόνο ένα δεδομένο», όπως μια ομάδα από ονόματα εικόνων.

Όμως, μια ταινία δεν έχει μόνο μία τιμή. Περιλαμβάνει τουλάχιστον:

1. αφίσα ταινίας
2. όνομα ταινίας
3. σκηνοθέτη
4. βαθμολογία

Δηλαδή, μια ταινία είναι στην πραγματικότητα ένα σύνολο σχετικών δεδομένων και όχι απλώς ένα μεμονωμένο string.

Αν χρησιμοποιήσουμε μόνο πίνακες, τότε μπορούμε να αποθηκεύσουμε αυτές τις πληροφορίες μόνο χωριστά:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

Κατά την εμφάνιση, θα πρέπει να βασιστούμε στον ίδιο δείκτη για να τα αντιστοιχίσουμε ένα προς ένα:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

Το πρόβλημα με αυτή τη γραφή είναι ότι το κόστος συντήρησης είναι πολύ υψηλό.

Επειδή τα δεδομένα μιας ταινίας διασκορπίζονται σε πολλούς πίνακες, αρκεί ένας πίνακας να έχει ένα στοιχείο παραπάνω, ένα λιγότερο ή διαφορετική σειρά, και το αποτέλεσμα της εμφάνισης θα είναι λάθος.

Ειδικά όταν προσθέτεις ή διαγράφεις ταινίες, πρέπει να τροποποιείς πολλούς πίνακες ταυτόχρονα και είναι πολύ εύκολο να παραλείψεις κάτι.

Τότε, πώς μπορούμε να συνδυάσουμε πληροφορίες όπως «όνομα ταινίας, σκηνοθέτης, βαθμολογία» σε ένα ενιαίο σύνολο;

Για αυτό χρειάζεται να χρησιμοποιήσουμε το `struct`.

## Ορισμός της δομής struct

Στη Swift, το `struct` είναι μια «δομή».

Μπορείς να το κατανοήσεις ως έναν «προσαρμοσμένο τύπο δεδομένων», που μπορεί να ενώσει πολλά σχετικά πεδία και να τα μετατρέψει σε ένα σύνολο.

Βασική χρήση:

```swift
struct StructName {
	let name: String
}
```

Σε αυτόν τον κώδικα, το `struct` είναι η λέξη-κλειδί που δείχνει ότι ορίζουμε μια δομή. Το `StructName` είναι το όνομα της δομής.

Το περιεχόμενο μέσα στις αγκύλες είναι τα πεδία που περιέχει αυτή η δομή και κάθε πεδίο πρέπει να έχει καθαρά γραμμένο το όνομα και τον τύπο του.

Συνήθως, το όνομα μιας `struct` ξεκινά με κεφαλαίο γράμμα, όπως `Movie`, `Student`, `UserInfo`. Αυτή είναι μια συνηθισμένη συνήθεια ονοματοδοσίας στη Swift.

Μπορείς να καταλάβεις απλά το `struct` ως ένα άδειο κουτί συσκευασίας, όπου κάθε πεδίο είναι σαν μια κενή θέση που έχει προβλεφθεί μέσα στο κουτί.

Όταν αυτές οι θέσεις δεν έχουν ακόμα γεμίσει, είναι απλώς ένα άδειο κουτί. Μόνο όταν όλα τα πεδία συμπληρωθούν με κατάλληλες τιμές, γίνεται ένα πλήρες κουτί δώρου.

### Στιγμιότυπο struct

Πριν ορίσαμε μόνο την ίδια τη δομή, κάτι που ισοδυναμεί με την προετοιμασία του σχήματος ενός κουτιού συσκευασίας.

Χρειάζεται ακόμη να βάλουμε μέσα το συγκεκριμένο περιεχόμενο, ώστε να αποκτήσουμε ένα πραγματικό «στιγμιότυπο» που μπορεί να χρησιμοποιηθεί.

Κατά τη δημιουργία στιγμιότυπου, συνήθως προσθέτουμε `()` μετά το όνομα της δομής:

```swift
StructName(...)
```

Αυτό που γράφεται μέσα στις παρενθέσεις είναι το περιεχόμενο των πεδίων που χρειάζεται αυτή η δομή.

Για παράδειγμα:

```swift
StructName(name: "Fang Junyu")
```

Αυτός ο κώδικας σημαίνει: σύμφωνα με τη μορφή της δομής `StructName`, δημιουργούμε ένα νέο στιγμιότυπο και δίνουμε στο πεδίο `name` την τιμή `"Fang Junyu"`.

Όταν συμπληρώσουμε όλο το απαραίτητο περιεχόμενο των πεδίων, είναι σαν να γεμίζουμε πλήρως αυτό το κουτί συσκευασίας.

Τότε αποκτούμε ένα πλήρες στιγμιότυπο δομής.

### Πρόσβαση σε ιδιότητες struct

Αφού δημιουργήσουμε ένα στιγμιότυπο, μπορούμε να χρησιμοποιήσουμε τη «σύνταξη με τελεία» για να αποκτήσουμε πρόσβαση στις εσωτερικές ιδιότητές του.

Βασική γραφή:

```swift
στιγμιότυπο.όνομαΙδιότητας
```

Για παράδειγμα:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

Εδώ το `st` είναι ένα στιγμιότυπο δομής και το `st.name` σημαίνει ότι διαβάζουμε την τιμή `name` αυτού του στιγμιότυπου.

Μπορείς να καταλάβεις αυτόν τον τρόπο ανάγνωσης έτσι: όταν γράφουμε `st.name`, είναι σαν να διαβάζουμε το περιεχόμενο που είναι αποθηκευμένο σε μια συγκεκριμένη θέση μέσα σε αυτό το κουτί.

Αυτός ο τρόπος γραφής θα είναι πολύ συνηθισμένος στις μεταγενέστερες διεπαφές SwiftUI.

Για παράδειγμα, αν έχουμε μια δομή μαθητή που περιέχει όνομα, ηλικία και τάξη, τότε κατά την εμφάνιση στη διεπαφή μπορούμε να διαβάζουμε ξεχωριστά:

```swift
student.name
student.age
student.className
```

Το πλεονέκτημα είναι ότι τα δεδομένα γίνονται πιο καθαρά και επίσης πιο εύκολα στη διαχείριση.

### Θέση του struct

Για τους αρχάριους, συνήθως μπορούμε να γράψουμε τη δομή έξω από το `ContentView`.

Για παράδειγμα:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

Έτσι η δομή γίνεται πιο καθαρή και επίσης πιο εύκολη στην ανάγνωση.

Θα παρατηρήσεις ότι το ίδιο το `ContentView` είναι στην πραγματικότητα επίσης ένα `struct`.

### Δομή Movie

Με βάση όσα είδαμε πριν, μπορούμε να ορίσουμε μια δομή ταινίας:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

Αυτή είναι μια δομή με όνομα `Movie`, που αποτελείται από τα τρία πεδία `name`, `director` και `rating`, τα οποία αντιπροσωπεύουν αντίστοιχα το όνομα της ταινίας, τον σκηνοθέτη και τη βαθμολογία.

Έτσι, οι πληροφορίες που σχετίζονται με μια ταινία δεν χρειάζεται πλέον να χωρίζονται και να αποθηκεύονται σε πολλούς πίνακες, αλλά μπορούν να συνδυαστούν απευθείας σε ένα ενιαίο σύνολο.

Για παράδειγμα, μπορούμε να δημιουργήσουμε ένα τέτοιο στιγμιότυπο ταινίας:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

Αυτό το στιγμιότυπο αντιπροσωπεύει «ένα πλήρες σύνολο δεδομένων ταινίας».

### Αποθήκευση δομών σε πίνακα

Προηγουμένως είπαμε ότι ο πίνακας μπορεί να αποθηκεύει μόνο τον ίδιο τύπο.

Τώρα έχουμε ήδη τη δομή `Movie`, άρα ο πίνακας μπορεί να αποθηκεύει πολλά `Movie`.

```swift
let lists: [Movie] = []
```

Για παράδειγμα:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Σε αυτόν τον πίνακα `lists`, δεν αποθηκεύονται strings, αλλά πολλά στιγμιότυπα `Movie`.

Δηλαδή, κάθε στοιχείο του πίνακα είναι μια πλήρης ταινία.

Έτσι, όταν χρειάζεται να εμφανίσουμε τη λίστα ταινιών, μπορούμε μέσω του `ForEach` να διαβάζουμε κάθε ταινία μία προς μία.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

Εδώ το `movie in` σημαίνει ότι σε κάθε επανάληψη θα παίρνεται μία ταινία από τον πίνακα και θα ονομάζεται προσωρινά `movie`.

Έπειτα, μπορούμε να χρησιμοποιήσουμε:

```swift
movie.name
movie.director
movie.rating
```

για να διαβάσουμε αντίστοιχα το όνομα, τον σκηνοθέτη και τη βαθμολογία αυτής της ταινίας και να τα εμφανίσουμε στη διεπαφή.

## Σφάλμα ForEach

Μέχρι εδώ, έχουμε ήδη ολοκληρώσει τη βασική γραφή του πίνακα ταινιών και του `ForEach`.

Ωστόσο, αν τρέξεις απευθείας τον παρακάτω κώδικα:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

θα εμφανιστεί σφάλμα:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

Το νόημα αυτού του σφάλματος είναι ότι όταν γράφεις `id: \.self`, το SwiftUI χρειάζεται να αντιμετωπίσει κάθε στοιχείο του πίνακα το ίδιο ως «μοναδικό αναγνωριστικό» για να το ξεχωρίζει.

Αυτή η γραφή είναι κατάλληλη για απλούς τύπους όπως `String` και `Int`, επειδή αυτοί οι τύποι διακρίνονται πιο εύκολα από μόνοι τους.

Αλλά το `Movie` είναι μια προσαρμοσμένη δομή που ορίσαμε εμείς, και το `ForEach` δεν ξέρει πώς να χρησιμοποιήσει την ίδια τη δομή ως μοναδικό αναγνωριστικό, γι’ αυτό προκύπτει το σφάλμα.

Ο πιο συνηθισμένος τρόπος για να λύσουμε αυτό το πρόβλημα είναι να κάνουμε το `Movie` να ακολουθεί το πρωτόκολλο `Identifiable`.

## Πρωτόκολλο

Στη Swift, ένα πρωτόκολλο μπορεί να κατανοηθεί ως ένα είδος «κανόνα» ή «απαίτησης».

Μπορείς να το καταλάβεις σαν μια συμφωνία σε έναν διεθνή οργανισμό: αν μια χώρα θέλει να ενταχθεί σε έναν συγκεκριμένο διεθνή οργανισμό, συνήθως πρέπει πρώτα να ικανοποιήσει κάποιους κανόνες που θέτει ο οργανισμός, να δημοσιοποιήσει κάποια δεδομένα και να τηρήσει κάποιους κοινούς κανόνες. Μόνο αφού πληροί αυτές τις προϋποθέσεις μπορεί να ενταχθεί ή να συμμετάσχει σε ορισμένες υποθέσεις.

Τα πρωτόκολλα λειτουργούν με παρόμοιο τρόπο.

Όταν ένας τύπος θέλει να ακολουθεί ένα συγκεκριμένο πρωτόκολλο, πρέπει να ικανοποιεί το περιεχόμενο που απαιτεί αυτό το πρωτόκολλο. Μόνο αφού καλυφθούν αυτές οι απαιτήσεις, ο τύπος αυτός μπορεί να χρησιμοποιήσει την αντίστοιχη λειτουργικότητα.

Βασική γραφή:

```swift
struct Movie: Identifiable {
    // ...
}
```

Εδώ, το `: Identifiable` σημαίνει ότι το `Movie` ακολουθεί το πρωτόκολλο `Identifiable`.

Αν ένας τύπος ακολουθεί πολλά πρωτόκολλα, μπορούμε να τα χωρίσουμε με κόμμα:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

Για το `Identifiable`, η πιο σημαντική απαίτηση είναι ότι ο τύπος πρέπει να έχει ένα `id` με το οποίο να μπορεί να αναγνωρίζεται.

Για παράδειγμα:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

Εδώ το `id` είναι σαν αριθμός ταυτότητας και χρησιμοποιείται για να ξεχωρίζει κάθε ταινία.

Όσο το `id` κάθε ταινίας είναι διαφορετικό, το SwiftUI μπορεί να διακρίνει σωστά κάθε στοιχείο στον πίνακα.

Για παράδειγμα:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Και μετά χρησιμοποιούμε στο `ForEach`:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

Έτσι δεν θα εμφανιστεί πλέον σφάλμα.

Επειδή το `Movie` ήδη ακολουθεί το `Identifiable`, ένας πιο συνηθισμένος τρόπος γραφής είναι να παραλείψουμε εντελώς το `id:`:

```swift
ForEach(lists) { movie in
    // ...
}
```

Επειδή το SwiftUI γνωρίζει ήδη ότι πρέπει να χρησιμοποιήσει το `movie.id` για να ξεχωρίζει κάθε στοιχείο.

## Το αναγνωριστικό διάκρισης του ForEach

Παρακάτω, θα κατανοήσουμε πιο βαθιά τον ρόλο του `id` στο `ForEach`.

Για παράδειγμα:

```swift
ForEach(lists, id: \.self)
```

Εδώ σημαίνει: χρήση του ίδιου του στοιχείου ως αναγνωριστικού διάκρισης.

Ενώ:

```swift
ForEach(lists, id: \.id)
```

σημαίνει: χρήση του πεδίου `id` του στοιχείου ως αναγνωριστικού διάκρισης.

Αν κάποιο πεδίο μπορεί από μόνο του να εγγυηθεί μοναδικότητα, μπορεί επίσης να χρησιμοποιηθεί προσωρινά.

Για παράδειγμα, αν το όνομα κάθε ταινίας είναι διαφορετικό, τότε και η παρακάτω γραφή μπορεί να λειτουργήσει κανονικά με τα τρέχοντα δεδομένα:

```swift
ForEach(lists, id: \.name)
```

Αλλά εδώ υπάρχει ένα κρυφό πρόβλημα: το `name` δεν είναι απαραίτητα πάντα μοναδικό.

Αν υποθέσουμε ότι αργότερα προσθέσεις μια άλλη ταινία με το ίδιο όνομα, τότε το `name` δεν θα μπορεί πλέον να ξεχωρίζει με ακρίβεια κάθε στοιχείο.

Τότε, παρόλο που ο κώδικας μπορεί ακόμη να κάνει compile, κατά την ανανέωση, ενημέρωση, εισαγωγή ή διαγραφή προβολών, το SwiftUI μπορεί να αναγνωρίσει λάθος τα στοιχεία, προκαλώντας ανωμαλίες στην εμφάνιση.

Γι’ αυτό, όταν χρειάζεται να ξεχωρίζουμε σταθερά τα δεδομένα, είναι καλύτερο να χρησιμοποιούμε το πραγματικά μοναδικό πεδίο, δηλαδή το `id`.

## UUID

Αν και η χειροκίνητη γραφή όπως `id: 1`, `id: 2` μπορεί να λύσει το πρόβλημα, υπάρχει ακόμη ένας κίνδυνος: μπορεί κατά λάθος να γραφτεί ένα επαναλαμβανόμενο `id`, και τότε το μοναδικό αναγνωριστικό θα χάσει την εγκυρότητά του.

Για παράδειγμα:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Έτσι θα εμφανιστούν δύο ίδια `id`, και το μοναδικό αναγνωριστικό θα χάσει την ισχύ του.

Για να αποφύγουμε χειροκίνητα λάθη, συνήθως χρησιμοποιούμε το `UUID()`.

Στη Swift, το `UUID()` δημιουργεί τυχαία ένα αναγνωριστικό 128 bit. Συνήθως εμφανίζεται ως μια μεγάλη ακολουθία από γράμματα και αριθμούς.

Για παράδειγμα:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

Εδώ το `UUID()` στην πραγματικότητα δημιουργεί ένα στιγμιότυπο του τύπου `UUID`· μπορείς να το καταλάβεις ως έναν αυτόματα παραγόμενο «μοναδικό αριθμό».

Επομένως, οι τιμές που δημιουργούνται κάθε φορά σχεδόν δεν επαναλαμβάνονται ποτέ, γι’ αυτό είναι πολύ κατάλληλες για χρήση ως `id`.

Μπορούμε να τροποποιήσουμε το `Movie` ως εξής:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Αυτός ο κώδικας σημαίνει ότι το πεδίο `id` στη δομή `Movie` θα λαμβάνει από προεπιλογή ένα νέο στιγμιότυπο `UUID`.

Δηλαδή, κάθε φορά που δημιουργούμε ένα νέο `Movie`, το σύστημα θα μας δημιουργεί αυτόματα πρώτα ένα μοναδικό `id`.

Το `id` έχει ήδη προεπιλεγμένη τιμή, οπότε αργότερα, όταν δημιουργούμε στιγμιότυπα `Movie`, δεν χρειάζεται πια να συμπληρώνουμε χειροκίνητα το `id`.

Για παράδειγμα:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Παρόλο που εδώ δεν γράφεται το `id`, στην πραγματικότητα αυτό το στιγμιότυπο ταινίας εξακολουθεί να έχει το δικό του `id`, απλώς αυτή η τιμή έχει ήδη παραχθεί αυτόματα από το `UUID()`.

Μετά τη χρήση αυτής της μεθόδου, δεν χρειάζεται πλέον να συμπληρώνουμε χειροκίνητα `id` για κάθε ταινία, κάτι που όχι μόνο μειώνει τον όγκο του κώδικα, αλλά αποφεύγει και τα σφάλματα λόγω επαναλαμβανόμενων τιμών id.

Τέλος, αρκεί να προσθέσουμε μια εικόνα `Banner` στην κορυφή και να χρησιμοποιήσουμε `ScrollView` ώστε το περιεχόμενο να μπορεί να γίνεται κύλιση, και έτσι ολοκληρώνεται ολόκληρη η προβολή λίστας ταινιών.

## Σύνοψη

Σε αυτό το μάθημα, μάθαμε ένα πολύ σημαντικό σημείο γνώσης: `struct`.

Μέσω του `struct`, μπορούμε να συνδυάσουμε πολλά σχετικά πεδία μιας ταινίας σε ένα σύνολο, αντί να διασπάμε το όνομα, τον σκηνοθέτη και τη βαθμολογία σε πολλούς πίνακες.

Αφού συνδυαστούν τα δεδομένα, μπορούμε να χρησιμοποιήσουμε το `ForEach` για να εμφανίζουμε κάθε ταινία του πίνακα μία-μία μέσα στη view.

Ταυτόχρονα, κατανοήσαμε επίσης τον ρόλο του «αναγνωριστικού διάκρισης» στο `ForEach`: το SwiftUI πρέπει να γνωρίζει πώς να διακρίνει κάθε στοιχείο του πίνακα, ώστε να μπορεί να εμφανίζει και να ενημερώνει σωστά τη view.

Γι’ αυτό κάναμε το `Movie` να ακολουθεί το πρωτόκολλο `Identifiable` και του δώσαμε ένα μοναδικό `id`.

Για να αποφύγουμε λάθη κατά τη χειροκίνητη συμπλήρωση του `id`, μάθαμε επιπλέον και το `UUID()`, ώστε το σύστημα να δημιουργεί αυτόματα το μοναδικό αναγνωριστικό.

Αφού ολοκληρώσεις αυτό το μάθημα, όχι μόνο θα έχεις φτιάξει τη λίστα ταινιών, αλλά θα αρχίσεις επίσης να έρχεσαι σε επαφή με μια σημαντική ιδέα του SwiftUI: **πρώτα οργάνωσε τα δεδομένα και μετά δημιούργησε τη view με βάση τα δεδομένα.**

## Πλήρης κώδικας

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
