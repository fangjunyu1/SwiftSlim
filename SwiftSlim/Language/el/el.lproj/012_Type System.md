# Σύστημα τύπων

Στα προηγούμενα μαθήματα μάθαμε ήδη τις μεταβλητές και γνωρίζουμε ότι μπορούν να αποθηκεύουν δεδομένα διαφορετικών τύπων.

Για παράδειγμα:

```swift
var hello = "Hello, World"
```

Εδώ η `hello` αποθηκεύει ένα κομμάτι κειμένου, άρα ο τύπος της είναι `String`.

Στη συνέχεια θα μελετήσουμε συστηματικά την έννοια του τύπου (`Type`), καθώς και τη ρητή δήλωση τύπου και την type inference, ώστε να κατανοήσουμε βαθύτερα τη φύση των μεταβλητών.

## Τι είναι ο τύπος

Στη Swift, κάθε τιμή έχει έναν σαφή τύπο. Ο τύπος καθορίζει τι είναι αυτή η τιμή και τι μπορεί να κάνει.

Για παράδειγμα:

- Το `42` είναι `Int` (ακέραιος)
- Το `"Hello"` είναι `String` (string)
- Το `true` είναι `Bool` (boolean)

Αυτά ανήκουν σε διαφορετικούς τύπους και άρα έχουν διαφορετικές χρήσεις.

## Συνηθισμένοι βασικοί τύποι στη Swift

Στο αρχικό στάδιο, οι πιο συνηθισμένοι τύποι είναι οι εξής:

- `String`: string (κείμενο)
- `Int`: ακέραιος
- `Double`: floating-point αριθμός (με δεκαδικά και μεγαλύτερη ακρίβεια)
- `Bool`: boolean (`true`/`false`)

Εκτός από αυτούς, υπάρχει και ο `Float` (floating-point, αλλά με μικρότερη ακρίβεια από το `Double`, και συνήθως χρησιμοποιείται λιγότερο), καθώς και ο `Character` (μονός χαρακτήρας), όπως `"A"` ή `"!"`.

### Δήλωση τύπου (ρητός τύπος)

Στη Swift μπορούμε, κατά τη δήλωση μιας μεταβλητής, να καθορίσουμε χειροκίνητα τον τύπο της:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

Προσθέτουμε δηλαδή `: Τύπος` μετά το όνομα της μεταβλητής.

Η σύνταξη για arrays είναι `: [Τύπος]`:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

Σε αυτό το παράδειγμα, το array `scores` μπορεί να αποθηκεύει μόνο στοιχεία τύπου `Int`, ενώ το `tags` μόνο στοιχεία τύπου `String`.

Η ρητή δήλωση τύπου κάνει την πρόθεση του κώδικα πιο καθαρή και σε ορισμένες περιπτώσεις βοηθά να αποφευχθούν λάθη στην type inference.

## Type inference

Σε πολλές περιπτώσεις, δεν χρειάζεται να δηλώσουμε ρητά τον τύπο:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"     // String
var isLogin = false  // Bool
var scores = [90, 85, 78]    // [Int]
var tags = ["Swift", "iOS"]  // [String]
```

Αυτό συμβαίνει επειδή ο compiler της Swift μπορεί να συμπεράνει αυτόματα τον τύπο από την «αρχική τιμή».

## Ρητή δήλωση και type inference

Στα προηγούμενα μαθήματα δεν είχαμε εισαγάγει ρητή δήλωση τύπου, όπως:

```swift
var age: Int = 18
```

αλλά προτιμούσαμε την type inference:

```swift
var age = 18
```

Σε αυτό το παράδειγμα, οι δύο γραφές είναι ισοδύναμες και στις δύο περιπτώσεις το `age` καθορίζεται ως `Int`. Σε σύγκριση με τη ρητή δήλωση, η type inference είναι πιο σύντομη.

Ο λόγος που στην αρχή δεν δώσαμε έμφαση στη ρητή δήλωση είναι ότι η type inference μειώνει τις επιπλέον πληροφορίες μέσα στον κώδικα και έτσι χαμηλώνει τη δυσκολία κατανόησης για τον αρχάριο.

## Γιατί χρειαζόμαστε τύπους

Η Swift είναι strongly typed language.

Αυτό σημαίνει ότι, μόλις καθοριστεί ο τύπος μιας μεταβλητής, δεν μπορεί να αλλάξει αυθαίρετα.

Για παράδειγμα:

```swift
var age: Int = 18

age = 20    // Success
age = "20"  // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

Το πρώτο `20` είναι τύπου `Int`, άρα μπορεί να ανατεθεί στο `age`. Το δεύτερο `"20"` είναι τύπου `String`, και επειδή δεν ταιριάζει με το `Int`, προκύπτει σφάλμα.

Το `nums` είναι array τύπου `[Int]`, οπότε μπορεί να αποθηκεύει μόνο ακέραιους και όχι strings.

Οι τύποι επιβάλλουν περιορισμούς στη χρήση των δεδομένων και βοηθούν τον compiler να εντοπίζει type mismatches ήδη στη φάση της μεταγλώττισης, μειώνοντας bugs και βελτιώνοντας τη σταθερότητα και τη συντηρησιμότητα του κώδικα.

## Πότε πρέπει οπωσδήποτε να δηλώσουμε ρητά τύπο

Παρόλο που η Swift μπορεί να συμπεράνει αυτόματα τύπους, υπάρχουν περιπτώσεις όπου ο compiler δεν μπορεί να τους συμπεράνει ή τους συμπεραίνει λανθασμένα. Τότε χρειάζεται ρητή δήλωση.

**1. Κενά arrays**

Όταν δημιουργούμε κενό array, πρέπει να δηλώσουμε ρητά τον τύπο:

```swift
var nums: [Int] = []
```

Αν δεν γράψουμε τύπο:

```swift
var nums = []   // Error, Empty collection literal requires an explicit type
```

ο compiler δεν μπορεί να ξέρει από ένα κενό array τι τύπο στοιχείων θα περιέχει, και έτσι εμφανίζεται compile error.

**2. Αποφυγή λανθασμένης inference**

```swift
var value = 10   // Int
```

Σε αυτό το παράδειγμα, παρότι το `10` θα μπορούσε θεωρητικά να είναι και floating-point αριθμός (`Double`), επειδή δεν έχει δεκαδικό σημείο, ο compiler θα το θεωρήσει από προεπιλογή `Int`.

Αν θέλουμε το `value` να είναι floating-point, τότε πρέπει να δηλώσουμε ρητά τον τύπο:

```swift
var value: Double = 10   // Double
```

Όταν το αποτέλεσμα της type inference δεν είναι αυτό που περιμένουμε, πρέπει να χρησιμοποιούμε ρητή δήλωση τύπου ή να προσαρμόζουμε τη literal μορφή της τιμής.

**3. Προσαρμοσμένοι τύποι**

Αργότερα θα μάθουμε και custom types. Όταν χρησιμοποιούμε custom types, συχνά χρειάζεται επίσης ρητή δήλωση:

```swift
var info: GameInfo
var users: [User] = []
```

Ο λόγος είναι ο ίδιος όπως και στα κενά arrays: ο compiler δεν μπορεί να συμπεράνει αυτόματα τον τύπο από μια κενή τιμή.

## Μετατροπή τύπων

Στη Swift, διαφορετικοί τύποι δεν μετατρέπονται αυτόματα μεταξύ τους· πρέπει να τους μετατρέψουμε χειροκίνητα.

Για παράδειγμα, αν θέλουμε να προσθέσουμε δύο αριθμούς διαφορετικών τύπων:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b // Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

Εδώ το `a` είναι τύπου `Int` και το `b` τύπου `Double`. Εφόσον οι τύποι είναι διαφορετικοί, δεν μπορούν να χρησιμοποιηθούν απευθείας στην ίδια πράξη.

Πριν κάνουμε την πράξη, πρέπει πρώτα να μετατρέψουμε μία από τις τιμές στον ίδιο τύπο:

```swift
var result = Double(a) + b
```

Αυτό σημαίνει ότι μετατρέπουμε το `a` από `Int` σε `Double`, και μετά το προσθέτουμε στο `b`. Το αποτέλεσμα αποθηκεύεται στο `result`.

Πρέπει να προσέξουμε ότι η μετατροπή τύπου δεν αλλάζει τον αρχικό τύπο της μεταβλητής. Για παράδειγμα, το `Double(a)` δεν αλλάζει το `a` σε `Double`, αλλά δημιουργεί μια νέα τιμή τύπου `Double` μόνο για να συμμετάσχει στην πράξη.

Και άλλοι τύποι μπορούν να μετατραπούν με παρόμοιο τρόπο, δηλαδή γράφοντας τον τύπο-στόχο και έπειτα `()`, περνώντας μέσα την τιμή που θέλουμε να μετατρέψουμε. Για παράδειγμα:

```text
Int()、Double()、String()
```

Παραδείγματα:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int, 9
```

Πρέπει να προσέξουμε ότι δεν επιτυγχάνουν όλες οι μετατροπές με τον τρόπο που ίσως περιμένουμε. Για παράδειγμα, όταν μετατρέπουμε `Double` σε `Int`, το δεκαδικό μέρος απλώς κόβεται και δεν γίνεται στρογγυλοποίηση.

## Typealias

Στη Swift μπορούμε επίσης να δώσουμε ένα άλλο όνομα σε έναν υπάρχοντα τύπο, ώστε η σημασία του κώδικα να γίνεται πιο καθαρή:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

Εδώ το `UserID` στην ουσία παραμένει `Int`. Ο τύπος δεν αλλάζει, απλώς το όνομα γίνεται πιο περιγραφικό.

Σε projects με πολύ κώδικα και πιο σύνθετους τύπους, τα type aliases κάνουν τον κώδικα πιο ευανάγνωστο.

## Παράδειγμα με τύπους

Ας δούμε τώρα ένα απλό παράδειγμα για το πώς λειτουργούν οι τύποι στην πράξη.

### Δημιουργία μεταβλητής

Αρχικά, δημιουργούμε μια μεταβλητή `age`:

```swift
var age = 20
```

Επειδή θέλουμε να εμφανίζεται και να αλλάζει μέσα στο SwiftUI, πρέπει να τη δηλώσουμε με `@State`:

```swift
@State private var age = 20
```

Το `private` το έχουμε ήδη αναφέρει, και σημαίνει ότι αυτή η μεταβλητή μπορεί να χρησιμοποιηθεί μόνο μέσα στην τρέχουσα view. Θα το δούμε πιο αναλυτικά αργότερα.

### Εμφάνιση δεδομένων στη SwiftUI

Εμφάνιση της μεταβλητής `age` μέσα στο SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

Τώρα η view θα εμφανίζει:

```text
age: 20
```

### Τροποποίηση δεδομένων

Στη συνέχεια, προσθέτουμε δύο buttons για να αλλάξουμε το `age`:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### Αποτελέσματα εκτέλεσης

Όταν πατήσουμε το `changeAge1`, το `age` θα πάρει την τιμή `18`:

```swift
age = 18
```

Επειδή το `18` είναι τύπου `Int`, δηλαδή του ίδιου τύπου με το `age`, η ανάθεση είναι έγκυρη.

Στο `changeAge2` όμως θα υπάρξει σφάλμα:

```swift
age = "18"
```

Μήνυμα σφάλματος:

```text
Cannot assign value of type 'String' to type 'Int'
```

Αυτό συμβαίνει επειδή το `"18"` είναι τύπου `String` (τα strings γράφονται μέσα σε `""`), ενώ το `age` είναι τύπου `Int`. Οι τύποι δεν ταιριάζουν, άρα η ανάθεση δεν επιτρέπεται.

### Γιατί εμφανίζεται σφάλμα;

Το κλειδί βρίσκεται σε αυτή τη γραμμή:

```swift
@State private var age = 20   // Int
```

Παρόλο που δεν υπάρχει ρητή δήλωση τύπου, επειδή η αρχική τιμή `20` είναι τύπου `Int`, ο compiler συμπεραίνει αυτόματα ότι το `age` είναι `Int`.

Μόλις καθοριστεί ο τύπος, δεν μπορεί να του ανατεθεί τιμή άλλου τύπου.

Επιπλέον, το button `changeAge2` δεν αποτυγχάνει «κατά την εκτέλεση», αλλά προκαλεί σφάλμα ήδη στο compile time, οπότε ο κώδικας δεν μπορεί καν να μεταγλωττιστεί.

Αυτό είναι ένα από τα πλεονεκτήματα της strong typing στη Swift: μπορεί να εντοπίζει type problems ήδη την ώρα που γράφουμε κώδικα.

### Σωστή γραφή

Αν αφαιρέσουμε τον λανθασμένο κώδικα:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

τότε, όταν πατήσουμε το button `changeAge1`, η view θα ενημερωθεί σε:

```text
age: 18
```

### Ρητή δήλωση τύπου (προαιρετικά)

Μπορούμε επίσης να ορίσουμε ρητά τον τύπο κατά τον ορισμό:

```swift
@State private var age: Int = 20
```

Αυτή η γραφή κάνει τον τύπο πιο ξεκάθαρο, αλλά σε απλά σενάρια δεν είναι απαραίτητη.

## Σύνοψη

Αυτό το μάθημα επικεντρώθηκε κυρίως στο σύστημα τύπων της Swift και στον ρόλο του στην πραγματική ανάπτυξη.

Στη Swift, κάθε τιμή έχει σαφή τύπο. Ο τύπος καθορίζει τη σημασία των δεδομένων και ποιες πράξεις μπορούν να γίνουν πάνω τους.

Η Swift είναι strongly typed. Μόλις καθοριστεί ο τύπος μιας μεταβλητής, δεν μπορεί να της ανατεθεί τιμή άλλου τύπου. Αυτός ο περιορισμός επιτρέπει στον compiler να εντοπίζει σφάλματα από νωρίς, μειώνοντας τα runtime προβλήματα και αυξάνοντας την ασφάλεια του κώδικα.

Ο τύπος μιας μεταβλητής μπορεί να συμπεραστεί αυτόματα από την αρχική τιμή ή να δηλωθεί ρητά. Σε ορισμένες περιπτώσεις, όπως κενά arrays, λανθασμένη type inference ή custom types, ο compiler δεν μπορεί να προσδιορίσει σωστά τον τύπο. Τότε η ρητή δήλωση τύπου είναι απαραίτητη.

Διαφορετικοί τύποι δεν μετατρέπονται αυτόματα μεταξύ τους, αλλά πρέπει να μετατραπούν ρητά με `Int()`, `Double()`, `String()` και παρόμοιους τρόπους.

Οι τύποι υποστηρίζουν επίσης aliases. Ιδιαίτερα σε πιο σύνθετα projects, τα type aliases μπορούν να βελτιώσουν σημαντικά την αναγνωσιμότητα του κώδικα.

Ο τύπος δεν είναι απλώς ένα «label» για τα δεδομένα, αλλά και ένας μηχανισμός περιορισμού. Έτσι διασφαλίζεται ότι τα δεδομένα χρησιμοποιούνται με ασφάλεια, σαφήνεια και έλεγχο.

## Επέκταση - Optional types

Σε ορισμένες περιπτώσεις μετατροπής τύπων, η μετατροπή μπορεί να αποτύχει.

Για παράδειγμα, αν προσπαθήσουμε να μετατρέψουμε ένα `String` σε `Double`:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

Εδώ, το `Double(input)` δεν επιστρέφει `Double`, αλλά:

```text
Double?
```

Αυτός είναι ένας optional type (`Optional`).

### Γιατί επιστρέφεται Optional;

Επειδή το περιεχόμενο ενός string είναι αβέβαιο. Το `"3.14"` μπορεί να μετατραπεί σε `Double`, αλλά το `"Hello"` όχι.

Δηλαδή, αυτή η πράξη μπορεί είτε να πετύχει είτε να αποτύχει.

Γι’ αυτό η Swift χρησιμοποιεί optional types για να αναπαριστά αυτή την αβεβαιότητα.

### Τι είναι ένας optional type;

Ένας optional type σημαίνει ότι μια τιμή μπορεί να υπάρχει, αλλά μπορεί και να μην υπάρχει.

Για παράδειγμα:

```text
Double?
```

Αυτός ο τύπος σημαίνει ότι αν υπάρχει τιμή, όταν τυπώνεται θα εμφανίζεται σαν `Optional(...)`, ενώ αν δεν υπάρχει τιμή, θα επιστρέφεται `nil`.

Πρέπει να προσέξουμε ότι το `nil` σημαίνει «δεν υπάρχει τιμή» και όχι `0` ή κενό string.

### Παράδειγμα

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Έξοδος:

```text
pi:Optional(3.14)
```

Αυτό σημαίνει ότι το string μετατράπηκε επιτυχώς σε `Double` και η τιμή είναι `3.14`, αλλά επειδή ο τύπος είναι `Double?`, η τιμή τυλίγεται μέσα σε `Optional`.

Αν η μετατροπή αποτύχει:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

Έξοδος:

```text
pi: nil
```

Επειδή το `"Hello"` δεν μπορεί να μετατραπεί σε `Double`, η μετατροπή αποτυγχάνει και επιστρέφεται `nil`.

### Τι είναι το Optional(...);

Όταν τυπώνουμε απευθείας έναν optional type:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

η Swift εμφανίζει τη λεγόμενη "debug description":

```text
Optional(3.14)
```

Αυτό δεν είναι μέρος της πραγματικής τιμής, αλλά μια ένδειξη ότι η τιμή είναι optional και περιέχει εσωτερικά ένα κανονικό value.

### Unwrapping optional types

Στην πράξη, συνήθως θέλουμε να πάρουμε την πραγματική τιμή από μέσα από ένα optional. Αυτή η διαδικασία λέγεται unwrapping.

Ένας συνηθισμένος τρόπος είναι να χρησιμοποιήσουμε το `??` για να δώσουμε default value:

```text
??
```

Αυτό ονομάζεται Nil-Coalescing Operator.

Για παράδειγμα:

```swift
name ?? "FangJunyu"
```

Αυτό σημαίνει ότι αν το `name` έχει τιμή, επιστρέφει αυτή την τιμή. Αν είναι `nil`, χρησιμοποιεί την προεπιλεγμένη τιμή `"FangJunyu"`.

Πρέπει να προσέξουμε ότι το default value πρέπει να έχει τον ίδιο τύπο με το optional.

### Χρήση του ?? για default value

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")    // 30
```

Έξοδος:

```text
pi: 30
```

Όταν το `input` μετατρέπεται επιτυχώς σε `Double`, εμφανίζεται η πραγματική τιμή. Αν η μετατροπή αποτύχει, εμφανίζεται το default value που δίνεται μετά το `??`.

Αν η μετατροπή αποτύχει:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

Έξοδος:

```text
pi: 10
```

Εδώ, το `input` αποτυγχάνει να μετατραπεί σε `Double` και επιστρέφεται `nil`, οπότε το `??` δίνει ένα default value.

Οι optional types χρησιμοποιούνται για να αναπαριστούν περιπτώσεις όπου μια τιμή μπορεί να υπάρχει, αλλά μπορεί και να μην υπάρχει.

Σε πράξεις που μπορεί να αποτύχουν, όπως η μετατροπή τύπων, η Swift επιστρέφει optional type για λόγους ασφάλειας.

Όταν θέλουμε να χρησιμοποιήσουμε την εσωτερική τιμή, μπορούμε να δώσουμε ένα default μέσω του `??`, έτσι ώστε ακόμη και στην περίπτωση του `nil` να έχουμε ένα λογικό αποτέλεσμα.
