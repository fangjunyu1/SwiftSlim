# Βρόχοι και λίστες

Σε αυτό το μάθημα, θα μάθουμε τον βρόχο `for` στη Swift, καθώς και το `ForEach` στο SwiftUI.

Και τα δύο μπορούν να μας βοηθήσουν να διαβάζουμε τα στοιχεία ενός πίνακα με τη σειρά. Στην πραγματική ανάπτυξη εφαρμογών, συχνά χρειάζεται να παίρνουμε μια ομάδα δεδομένων ένα προς ένα και μετά να τα επεξεργαζόμαστε ή να τα εμφανίζουμε. Ο βρόχος είναι ακριβώς το σημαντικό εργαλείο που ολοκληρώνει αυτή τη δουλειά.

Όταν περιηγούμαστε στον ιστό ή χρησιμοποιούμε μια εφαρμογή, συχνά βλέπουμε πολλές λίστες ή καταχωρίσεις. Τέτοιο περιεχόμενο συνήθως αποθηκεύεται πρώτα σε έναν `πίνακα`, έπειτα διαβάζεται διαδοχικά μέσω ενός `βρόχου` και στο τέλος εμφανίζεται στη διεπαφή.

Για παράδειγμα, η λίστα αποτελεσμάτων αναζήτησης της Google:

![Google](../../Resource/017_google.png)

Κάθε καταχώριση στη λίστα αποτελεσμάτων αναζήτησης αντιστοιχεί συνήθως σε ένα κομμάτι δεδομένων και εμφανίζεται μία προς μία μέσω βρόχου.

Γι’ αυτό, οι `πίνακες` και οι `βρόχοι` εμφανίζονται συχνά μαζί. Ο `πίνακας` είναι υπεύθυνος για την αποθήκευση μιας ομάδας δεδομένων, ενώ ο `βρόχος` είναι υπεύθυνος για το να παίρνει αυτά τα δεδομένα με τη σειρά και να εκτελεί την ίδια ενέργεια.

Στη συνέχεια, ας ξεκινήσουμε πρώτα από τον πιο βασικό βρόχο `for`.

## Αντίστροφη μέτρηση

Ας υποθέσουμε ότι θέλουμε να προσομοιώσουμε μια λειτουργία αντίστροφης μέτρησης και να εκτυπώσουμε διαδοχικά `“5, 4, 3, 2, 1”`.

Ο πιο άμεσος τρόπος γραφής είναι:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

Αυτός ο τρόπος μπορεί να προσομοιώσει την αντίστροφη μέτρηση, αλλά αν το περιεχόμενο είναι πολύ, θα γίνει πολύ επαναλαμβανόμενο, και αργότερα δεν θα είναι βολικό να το τροποποιήσουμε.

Σε αυτή την περίπτωση, μπορούμε πρώτα να βάλουμε τα δεδομένα σε έναν πίνακα και μετά να χρησιμοποιήσουμε έναν βρόχο `for` για να τα εκτυπώσουμε ένα προς ένα:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Αυτός ο τρόπος γραφής είναι πιο σύντομος και πιο εύκολος στην επέκταση. Ακόμα κι αν υπάρχουν πολλοί αριθμοί στην αντίστροφη μέτρηση, δεν χρειάζεται να γράφουμε `print` γραμμή προς γραμμή. Αρκεί να αφήσουμε τον βρόχο να επεξεργαστεί αυτόματα κάθε στοιχείο του πίνακα.

Μπορούμε να βάλουμε αυτόν τον κώδικα μέσα σε ένα κουμπί για να τον δοκιμάσουμε:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

Όταν πατηθεί το κουμπί, ο βρόχος `for` θα πάρει κάθε ακέραιο μέσα στο `countDown` με τη σειρά και θα τον εκτυπώσει στην κονσόλα.

Στη συνέχεια, ας δούμε πώς λειτουργεί ο βρόχος `for`.

## Βρόχος for

Ο βρόχος `for` χρησιμοποιείται για την επεξεργασία μιας ομάδας δεδομένων με τη σειρά. Τα πιο συνηθισμένα αντικείμενα είναι οι πίνακες και τα εύρη.

Βασική σύνταξη:

```swift
for στοιχείο in πίνακας_ή_εύρος {
    κώδικας_προς_εκτέλεση
}
```

Για παράδειγμα, ο κώδικας αντίστροφης μέτρησης που είδαμε πριν:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Εδώ, το `i` αντιπροσωπεύει το τρέχον στοιχείο που βγαίνει, και μετά το `in` ακολουθούν τα δεδομένα πάνω στα οποία γίνεται η επανάληψη, δηλαδή ο πίνακας `countDown`.

Η σημασία αυτού του κώδικα είναι η εξής: το πρόγραμμα θα παίρνει από τον πίνακα `countDown` κάθε στοιχείο με τη σειρά. Κάθε φορά που παίρνει ένα στοιχείο, το εκφράζει προσωρινά με το `i`, και μετά εκτελεί τον κώδικα μέσα στις αγκύλες.

Η διαδικασία εκτέλεσης μπορεί να γίνει κατανοητή ως εξής:

* Το πρώτο στοιχείο που βγαίνει είναι το `5`, οπότε αυτή τη στιγμή η τιμή του `i` είναι `5`
* Εκτελείται το `print(i)` και εμφανίζεται το `5`
* Έπειτα συνεχίζει και παίρνει το επόμενο στοιχείο `4`
* Εκτελεί ξανά το `print(i)` και εμφανίζει το `4`

Τα επόμενα `3`, `2` και `1` θα εκτελεστούν επίσης με τον ίδιο τρόπο, διαδοχικά.

![for](../../Resource/017_for.png)

Όταν ολοκληρωθεί η επεξεργασία όλων των στοιχείων του πίνακα, ο βρόχος `for` τερματίζεται αυτόματα.

Μπορούμε να χρησιμοποιήσουμε τον βρόχο `for` για να πάρουμε κάθε στοιχείο του πίνακα και να ολοκληρώσουμε λειτουργίες όπως αυτή η αντίστροφη μέτρηση.

**Συμβουλή**: Το `i` είναι ένας πολύ συνηθισμένος τρόπος ονομασίας μέσα σε έναν βρόχο `for` και χρησιμοποιείται για να εκφράζει το τρέχον στοιχείο που βγαίνει, αλλά δεν είναι υποχρεωτικό. Μπορείς επίσης να χρησιμοποιήσεις άλλα αγγλικά ονόματα.

Για παράδειγμα, μπορούμε να χρησιμοποιήσουμε το `num` για να εκφράσουμε κάθε στοιχείο που βγαίνει:

```swift
for num in countDown {
    print(num)
}
```

Μέσα στις αγκύλες, πρέπει επίσης να χρησιμοποιήσεις το `num` για να εκφράσεις το τρέχον στοιχείο.

### Επανάληψη σε εύρος

Ο βρόχος `for` δεν μπορεί να επαναλαμβάνει μόνο πάνω σε πίνακες, αλλά και πάνω σε εύρη.

Για παράδειγμα, όταν θέλουμε να υπολογίσουμε το άθροισμα των θετικών ακεραίων μέχρι το `50`, πρέπει να προσθέσουμε όλους τους ακέραιους από το `1` έως το `50`, έναν προς έναν.

Ο πιο άμεσος τρόπος γραφής ίσως να ήταν:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

Το πρόβλημα με αυτόν τον τρόπο είναι ότι πρέπει να γράψουμε κάθε αριθμό χειροκίνητα. Αυτό είναι κουραστικό και ακατάλληλο για πραγματικά σενάρια ανάπτυξης.

Σε αυτή την περίπτωση, μπορούμε να χρησιμοποιήσουμε τον τελεστή εύρους μαζί με τον βρόχο `for`.

Στο προηγούμενο κεφάλαιο για τους “τελεστές”, έχουμε ήδη μάθει τον τελεστή εύρους:

```swift
1...3   // 1, 2, 3
```

Εδώ, το `...` σημαίνει κλειστό εύρος, δηλαδή περιλαμβάνει και την αρχική και την τελική τιμή.

Γι’ αυτό εδώ μπορούμε να χρησιμοποιήσουμε έναν βρόχο `for` για το εύρος ακεραίων από το `1` έως το `50`:

```swift
for i in 1...50 {
    
}
```

Πρέπει να προσθέσουμε αυτούς τους ακέραιους, οπότε χρειαζόμαστε επίσης μια μεταβλητή για να αποθηκεύσουμε το άθροισμα:

```swift
var count = 0
```

Έπειτα, σε κάθε επανάληψη του βρόχου, προσθέτουμε τον τρέχοντα αριθμό που βγαίνει στο `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

Αυτός ο κώδικας σημαίνει:

* Το `count` χρησιμοποιείται για να αποθηκεύει το τρέχον άθροισμα
* Ο βρόχος `for` θα παίρνει τους ακέραιους από το `1` έως το `50`, έναν προς έναν
* Κάθε φορά που βγαίνει ένας ακέραιος, εκτελείται το `count += i`
* Όταν τελειώσει ο βρόχος, μέσα στο `count` θα είναι αποθηκευμένο το τελικό άθροισμα

Έτσι μπορούμε να πάρουμε το άθροισμα όλων των θετικών ακεραίων μέχρι το `50`.

Με τον ίδιο τρόπο, αν θέλουμε να υπολογίσουμε το άθροισμα των ακεραίων από το `1` έως το `100`, αρκεί να αλλάξουμε το εύρος:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

Ο βρόχος `for` δεν μπορεί να επαναλαμβάνει μόνο πάνω σε πίνακες, αλλά και πάνω σε ένα εύρος.

Αυτή η ικανότητα να επεξεργάζεται επαναλαμβανόμενα δεδομένα με τη σειρά είναι πολύ συνηθισμένη στην ανάπτυξη εφαρμογών.

Μπορεί να μας βοηθήσει να επεξεργαστούμε αυτόματα πολλές επαναλαμβανόμενες εργασίες, χωρίς να χρειάζεται να γράφουμε χειροκίνητα την πρόσθεση κάθε αριθμού ή άλλες παρόμοιες επαναλαμβανόμενες λειτουργίες.

## Επανάληψη στο SwiftUI

Ο προηγούμενος βρόχος `for` ανήκει στη γλώσσα Swift.

Όμως στο SwiftUI δεν μπορούμε να χρησιμοποιήσουμε απευθείας το `for` μέσα στη δομή μιας προβολής για να δημιουργήσουμε προβολές:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

Αυτό συμβαίνει επειδή μέσα στο `VStack` πρέπει να υπάρχουν προβολές SwiftUI, ενώ ένας συνηθισμένος βρόχος `for` από μόνος του δεν είναι προβολή.

Αν θέλουμε στο SwiftUI να δημιουργούμε επαναλαμβανόμενα διεπαφή με βάση το περιεχόμενο ενός πίνακα, πρέπει να χρησιμοποιήσουμε το `ForEach`.

Για παράδειγμα, αν θέλουμε να εμφανίσουμε τα ονόματα όλων των μελών μιας ομάδας, μπορούμε να το γράψουμε έτσι:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

Αποτέλεσμα εμφάνισης:

![for1](../../Resource/017_for1.png)

Στη συνέχεια, προσθέτουμε τους ίδιους τροποποιητές σε κάθε `Text`:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Τώρα, κάθε `Text` θα εμφανίζεται με μικρότερη και πιο έντονη γραμματοσειρά.

Αποτέλεσμα εμφάνισης:

![](../../Resource/017_for2.png)

Παρόλο που οι τροποποιητές κάνουν τη διεπαφή πιο όμορφη, κάνουν επίσης τον κώδικα πιο μεγάλο και πιο επαναλαμβανόμενο.

Αν τα ονόματα συνεχίσουν να αυξάνονται ή αργότερα θελήσουμε να αλλάξουμε ενιαία τη γραμματοσειρά, το χρώμα ή άλλα στυλ, θα χρειαστεί να γράψουμε και να τροποποιήσουμε πολλές παρόμοιες γραμμές κώδικα ξανά και ξανά, κάτι που θα είναι κουραστικό και δύσκολο στη συντήρηση.

Σε αυτή την περίπτωση, μπορούμε πρώτα να βάλουμε αυτά τα ονόματα σε έναν πίνακα και μετά να χρησιμοποιήσουμε το `ForEach` του SwiftUI για να δημιουργήσουμε προβολές επαναλαμβανόμενα σύμφωνα με τα δεδομένα του πίνακα:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

Αυτό το σύνολο κώδικα είναι πιο σύντομο. Το `ForEach` θα διαβάζει διαδοχικά κάθε στοιχείο του πίνακα και θα εμφανίζει το αντίστοιχο περιεχόμενο στη διεπαφή.

Όταν αργότερα προσθέσουμε νέα ονόματα, αρκεί να τροποποιήσουμε το περιεχόμενο του πίνακα `names`, χωρίς να χρειάζεται κάθε φορά να προσθέτουμε ξανά `Text` και τους ίδιους πολλούς τροποποιητές.

## ForEach

Το `ForEach` είναι μια δομή στο SwiftUI που χρησιμοποιείται για τη δημιουργία προβολών επαναλαμβανόμενα. Μπορείς να το καταλάβεις ως μια δομή στο SwiftUI παρόμοια με βρόχο `for`.

Βασική χρήση:

```swift
ForEach(πίνακας, id: \.self) { στοιχείο in
    Κώδικας SwiftUI
}
```

Για παράδειγμα, τα ονόματα της ομάδας από πριν:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Αυτός ο κώδικας σημαίνει:

Το `names` είναι ο πίνακας πάνω στον οποίο γίνεται η επανάληψη, δηλαδή τα δεδομένα που θέλουμε να εμφανίσουμε.

Το `id:\.self` σημαίνει ότι χρησιμοποιείται το ίδιο το στοιχείο του πίνακα για να διακρίνεται κάθε καταχώριση.

Στο τρέχον παράδειγμα, ο πίνακας περιέχει συμβολοσειρές:

```swift
["Sam", "John", "Wathon", "Bob"]
```

Έτσι, το SwiftUI θα χρησιμοποιήσει απευθείας τις ίδιες τις συμβολοσειρές `"Sam"`, `"John"`, `"Wathon"` και `"Bob"` για να διακρίνει διαφορετικά περιεχόμενα.

Για απλούς πίνακες που συναντάμε συχνά στο αρχικό στάδιο, όπως `String` και `Int`, συνήθως μπορούμε απλώς να γράψουμε:

```swift
id: \.self
```

Προς το παρόν, αρκεί να θυμάσαι μόνο αυτόν τον τρόπο γραφής. Δεν χρειάζεται ακόμα να μπεις βαθύτερα σε περισσότερους κανόνες για το `id`.

Στο `item in`, το `item` αντιπροσωπεύει το τρέχον στοιχείο που βγαίνει.

Για παράδειγμα, στην πρώτη επανάληψη το `item` είναι `"Sam"`, και στη δεύτερη επανάληψη είναι `"John"`.

Άρα:

```swift
Text(item)
```

θα μετατραπεί διαδοχικά σε:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

Επομένως, η λειτουργία αυτού του κώδικα είναι: να παίρνει διαδοχικά κάθε όνομα από τον πίνακα `names` και να δημιουργεί μια προβολή `Text` για κάθε όνομα.

### Διαδικασία εκτέλεσης

Η διαδικασία εκτέλεσης μπορεί να γίνει κατανοητή ως εξής:

* Το `ForEach` διαβάζει πρώτα το πρώτο στοιχείο `"Sam"` από τον πίνακα `names`
* Το `item` αντιπροσωπεύει προσωρινά το `"Sam"`
* Το `Text(item)` εμφανίζει το `"Sam"`
* Έπειτα συνεχίζει διαβάζοντας το επόμενο στοιχείο `"John"`
* Και εκτελεί ξανά τον ίδιο κώδικα προβολής

Στο τέλος, όλα τα ονόματα του πίνακα θα εμφανιστούν.

Το πλεονέκτημα αυτού του τρόπου γραφής είναι ότι, όταν τα ονόματα αυξάνονται, δεν χρειάζεται να γράφουμε πολλά `Text` ξανά και ξανά. Αρκεί να αλλάζουμε το περιεχόμενο του πίνακα.

### Επανάληψη σε φωτογραφίες

Το `ForEach` δεν μπορεί να εμφανίζει μόνο κείμενο, αλλά και εικόνες, χρώματα ή άλλες προβολές SwiftUI.

Για παράδειγμα, ας ετοιμάσουμε 4 εικόνες:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

Σύρε αυτές τις τέσσερις εικόνες στον φάκελο Assets του Xcode.

![image](../../Resource/017_for3.png)

Αν δεν χρησιμοποιήσουμε το `ForEach`, θα πρέπει να γράψουμε κάθε εικόνα χειροκίνητα:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

Αποτέλεσμα εμφάνισης:

![image](../../Resource/017_for4.png)

Αυτός ο τρόπος γραφής μπορεί να υλοποιήσει τη λειτουργία, αλλά ο κώδικας θα είναι σχετικά μακρύς.

Αν οι εικόνες αυξηθούν, θα χρειαστεί να γράφουμε όλο και περισσότερο επαναλαμβανόμενο κώδικα `Image`.

Αργότερα, αν θέλουμε να αλλάξουμε ενιαία το μέγεθος των εικόνων, θα πρέπει να τις τροποποιούμε μία προς μία.

Σε αυτή την περίπτωση, μπορούμε να βάλουμε τα ονόματα των εικόνων σε έναν πίνακα και μετά να χρησιμοποιήσουμε το `ForEach` για να δημιουργούμε επαναλαμβανόμενα προβολές εικόνων:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Εδώ, το `images` είναι ένας πίνακας συμβολοσειρών, και κάθε στοιχείο του είναι το όνομα μιας εικόνας.

Το `ForEach(images, id: \.self)` σημαίνει ότι διαβάζεται διαδοχικά κάθε στοιχείο του πίνακα `images`, χρησιμοποιείται το ίδιο το στοιχείο για να διακρίνεται κάθε καταχώριση, και μετά δημιουργείται η αντίστοιχη προβολή σύμφωνα με το τρέχον στοιχείο.

Για παράδειγμα, αν το πρώτο στοιχείο που διαβάζεται είναι το `"430F9BEF"`, τότε:

```swift
Image(item)
```

ισοδυναμεί με:

```swift
Image("430F9BEF")
```

Οι επόμενες εικόνες θα εμφανιστούν επίσης διαδοχικά με τον ίδιο τρόπο.

![image](../../Resource/017_for4.png)

Εδώ εξακολουθούμε να χρησιμοποιούμε:

```swift
id: \.self
```

Ο λόγος είναι ο ίδιος όπως πριν, επειδή τα στοιχεία του πίνακα `images` είναι επίσης απλές συμβολοσειρές, οπότε στο αρχικό στάδιο μπορούμε να χρησιμοποιήσουμε απευθείας το ίδιο το στοιχείο ως τρόπο διάκρισης.

Μπορείς να το καταλάβεις απλά σαν έναν σταθερό τύπο:

**Όταν το `ForEach` επαναλαμβάνει πάνω σε έναν απλό πίνακα `String` ή `Int`, συνήθως γράφουμε `id: \.self`.**

### Προσθήκη εικόνων

Αν αργότερα χρειαστεί να προσθέσουμε νέες εικόνες, αρκεί να τροποποιήσουμε το περιεχόμενο του πίνακα:

```swift
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

Δεν χρειάζεται να γράψουμε ξανά νέο κώδικα `Image`.

Αν υπάρχουν πολλές εικόνες, μπορούμε να το χρησιμοποιήσουμε μαζί με `ScrollView`, αλλιώς όταν το περιεχόμενο ξεπερνά την οθόνη δεν θα μπορεί να γίνει πλήρης προεπισκόπηση.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
```

### Τροποποίηση εικόνων

Όταν χρειάζεται να τροποποιήσουμε ενιαία το στυλ των εικόνων, αρκεί να αλλάξουμε μόνο τον κώδικα μέσα στο `ForEach`.

Για παράδειγμα, να αλλάξουμε το πλάτος σε `80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

Έτσι, όλες οι εικόνες θα αλλάξουν μαζί, χωρίς να χρειάζεται να τις τροποποιούμε μία προς μία.

Αυτός είναι ένας σημαντικός ρόλος του `ForEach`:

**Όταν πολλές προβολές έχουν την ίδια δομή και διαφέρει μόνο το περιεχόμενο, μπορούμε να γράψουμε τον κώδικα της προβολής μόνο μία φορά και μετά να αφήσουμε τα δεδομένα του πίνακα να καθορίσουν τι θα εμφανιστεί.**

## Σύνοψη

Σε αυτό το μάθημα, μάθαμε τον βρόχο `for` και το `ForEach`.

Ο βρόχος `for` χρησιμοποιείται για να επεξεργάζεται με τη σειρά τα στοιχεία ενός πίνακα ή ενός εύρους και τείνει περισσότερο προς την επεξεργασία δεδομένων.

Το `ForEach` χρησιμοποιείται στο SwiftUI για να δημιουργεί επαναλαμβανόμενα προβολές σύμφωνα με το περιεχόμενο ενός πίνακα και τείνει περισσότερο προς την εμφάνιση της διεπαφής.

Στο τρέχον αρχικό στάδιο, ο πιο συνηθισμένος τρόπος γραφής του `ForEach` είναι:

```swift
ForEach(πίνακας, id: \.self) { στοιχείο in
    Κώδικας SwiftUI
}
```

Εδώ, το `id: \.self` σημαίνει ότι χρησιμοποιείται το ίδιο το στοιχείο για να διακρίνεται κάθε καταχώριση.

Όταν το περιεχόμενο του πίνακα είναι απλοί τύποι όπως `String` και `Int`, συνήθως μπορούμε αρχικά να χρησιμοποιούμε μόνο αυτόν τον τρόπο γραφής.

Αργότερα, όταν περιηγείσαι στον ιστό ή χρησιμοποιείς εφαρμογές και βλέπεις μια ομάδα από επαναλαμβανόμενα κείμενα, εικόνες ή άλλο περιεχόμενο λίστας, μπορείς να θυμάσαι ότι στο SwiftUI αυτό το περιεχόμενο συνήθως μπαίνει πρώτα σε έναν πίνακα και έπειτα εμφανίζεται διαδοχικά μέσω του `ForEach`.
