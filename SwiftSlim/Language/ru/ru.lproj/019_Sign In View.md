# Экран входа

В этом уроке мы изучим очень практичный сценарий — экран входа.

Будь то сайт или приложение, во многих случаях во время использования пользователю нужно войти в аккаунт и ввести пароль.

Например, страница входа GitHub:

![WordPress](../../Resource/019_github.png)

В этом уроке мы создадим похожий экран входа, чтобы пользователь мог вручную ввести аккаунт и пароль и проверить, нет ли проблем во введенном содержимом.

## Верхняя часть макета

Мы будем писать код экрана входа в файле `ContentView`.

Сначала создадим область идентификации в верхней части экрана входа. Вы можете заранее подготовить подходящее изображение значка и поместить его в папку `Assets`.

![icon](../../Resource/019_icon1.png)

Затем используйте `Image` и модификаторы, чтобы показать изображение:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

Эти модификаторы означают:

- `resizable()`: позволяет изменять размер изображения.
- `scaledToFit()`: масштабирует изображение с сохранением исходных пропорций.
- `frame(width: 100)`: задает ширину отображаемого изображения `100`.

Далее используйте `Text` и модификаторы, чтобы показать заголовок входа:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

Затем используйте `VStack`, чтобы расположить изображение и текст вертикально:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

Результат:

![view](../../Resource/019_view.png)

На этом мы завершили область заголовка в верхней части экрана входа.

### Оптимизация макета

Сейчас изображение и заголовок по умолчанию находятся в центральной части всего макета.

Если мы хотим, чтобы они больше походили на «знак входа в верхней части страницы», можно использовать `Spacer()`, чтобы растянуть оставшееся пространство и сдвинуть содержимое ближе к верху.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

Результат:

![view](../../Resource/019_view1.png)

Но теперь `Image` и `Text` находятся слишком близко к верхней части экрана и выглядят немного тесно.

В этот момент можно использовать `padding`, чтобы добавить верхний отступ для всего `VStack`.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 60)
    Text("Sign in to App")
        .font(.title)
        .fontWeight(.bold)
    Spacer()
}
.padding(.top, 50)
```

Результат:

![view](../../Resource/019_view2.png)

Так область заголовка в верхней части страницы выглядит более уместно.

## Имя пользователя и пароль

На странице входа обычно требуется, чтобы пользователь ввел имя пользователя и пароль.

В SwiftUI мы можем использовать `TextField`, чтобы принимать содержимое, вводимое пользователем.

Но обратите внимание: `TextField` сам по себе не хранит введенные данные долгое время, это всего лишь элемент ввода. По-настоящему сохраняют эти данные переменные, которые мы к нему привязываем.

Поэтому сначала нам нужно создать две переменные `@State` для хранения имени пользователя и пароля:

```swift
@State private var user = ""
@State private var password = ""
```

Когда значение переменной `@State` меняется, SwiftUI автоматически обновляет связанное представление.

Далее используйте `TextField`, чтобы привязать эти две переменные:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

Здесь `$user` и `$password` означают «привязку».

То есть, когда пользователь вводит текст в поле ввода, переменная синхронно изменяется; а когда изменяется переменная, синхронно меняется и содержимое, отображаемое в поле ввода.

Такая связь, при которой «представление и данные синхронизируются друг с другом», и есть привязка.

Обратите внимание, что здесь нужно использовать запись с `$`:

```swift
$user
```

Потому что `TextField` нужен не обычный строковый тип, а значение привязки, которое «может двусторонне изменять данные».

### Отображение полей ввода

Поместим их в `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

Эффект отображения:

![view](../../Resource/019_view3.png)

Поскольку сейчас `user` и `password` по умолчанию являются пустыми строками:

```swift
""
```

Поле ввода сначала покажет текст-заполнитель, например:

```swift
input user
```

Этот текст лишь подсказывает пользователю, «что здесь нужно ввести», и не является реальным введенным содержимым.

### Оптимизация полей ввода

Теперь поля ввода уже можно использовать, но стиль по умолчанию довольно простой.

Чтобы интерфейс был понятнее, мы можем добавить заголовок перед полем ввода и немного улучшить стиль самого поля.

Например, сначала добавим заголовок:

```swift
Text("Username")
    .fontWeight(.bold)
```

Затем используем `HStack`, чтобы разместить заголовок и поле ввода в одной строке:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

Здесь используется `HStack`, потому что мы хотим, чтобы «слева был заголовок, а справа поле ввода».

Нужно обратить внимание, что `TextField` по умолчанию занимает все оставшееся доступное пространство.

![color](../../Resource/019_view6.png)

Чтобы размеры двух полей ввода были более одинаковыми, можно использовать `frame(width:)`, чтобы задать им ширину и сделать их более аккуратными.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

Затем добавьте рамку, чтобы поле ввода стало заметнее:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Теперь добавим и имя пользователя, и пароль в представление `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

Эффект отображения:

![view](../../Resource/019_view4.png)

На этом этапе пользователь уже может ввести имя пользователя и пароль.

## Логика входа

Теперь добавим кнопке входа самую базовую логику проверки: **определить, ввел ли пользователь имя пользователя и пароль**.

Мы хотим реализовать следующие ситуации:

- Если пользователь ничего не ввел, вывести `Empty`
- Если пользователь ввел только одно из двух полей, подсказать, чего не хватает
- Если и имя пользователя, и пароль введены, вывести `Success`

### Не введено никакой информации

Поскольку `user` и `password` по умолчанию являются пустыми строками:

```swift
@State private var user = ""
@State private var password = ""
```

Когда пользователь ничего не вводит, они по-прежнему остаются пустыми.

В Swift такие типы, как `String`, могут использовать `isEmpty`, чтобы проверять, пусто ли содержимое.

### Свойство isEmpty

`isEmpty` часто используется для проверки, пусты ли строки, массивы и другое содержимое.

Например:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

Если содержимое пусто, `isEmpty` возвращает `true`; если содержимое не пусто, `isEmpty` возвращает `false`.

Поэтому мы можем использовать его, чтобы определить, введены ли имя пользователя и пароль.

### Использование isEmpty для проверки переменных

Если пользователь не ввел никакой информации, то:

```swift
user.isEmpty // true
password.isEmpty    // true
```

В этом случае можно написать условие так:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

Этот код означает: если `user` пустой и `password` тоже пустой, вывести:

```swift
Empty
```

Здесь `&&` — это логический оператор, означающий «и».

То есть, только когда условие слева истинно и условие справа тоже истинно, все условие считается истинным.

Поэтому этот код выполнится только тогда, когда и имя пользователя, и пароль пусты.

## Пользователь заполнил только часть

Теперь рассмотрим другую ситуацию: пользователь ввел только имя пользователя или только пароль.

Например:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

В этом случае имя пользователя не пустое, но пароль пустой.

Мы можем продолжить использовать `isEmpty` для проверки:

### Ситуация, когда пользователь ввел учетную запись или пароль

Если пользователь ввел только имя пользователя или только пароль, нам нужно выдать соответствующую подсказку о том, чего не хватает.

Например, пользователь ввел только имя пользователя:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Логика этого кода такова: если имя пользователя пустое, попросить ввести имя пользователя. Иначе проверить, пуста ли пароль; если пароль пустой, попросить ввести пароль.

### Пользователь ввел всю информацию

Если и имя пользователя, и пароль уже введены, тогда `isEmpty` для обоих вернет `false`.

В этом случае можно написать так:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

Здесь `!` означает «отрицание».

Например:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

То есть `user.isEmpty` используется для проверки, «пусто ли имя пользователя», а `!user.isEmpty` — для проверки, «не пусто ли имя пользователя».

Для пароля логика такая же.

Поэтому смысл этого кода таков: имя пользователя не пустое, и пароль тоже не пустой.

Когда оба этих условия выполняются, выводится:

```swift
Success
```

## Полная логика проверки

Теперь объединим эти три ситуации и запишем их в `Button`:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

Порядок выполнения этого кода можно понять так:

Первый шаг — сначала проверить:

```swift
if user.isEmpty && password.isEmpty
```

Если и имя пользователя, и пароль пусты, сразу выводится `Empty`.

Если оба не пусты, программа продолжит проверять следующую строку:

```swift
else if user.isEmpty || password.isEmpty
```

Здесь `||` означает «или».

То есть, если выполняется хотя бы одно из условий, все выражение считается истинным.

Поэтому, пока имя пользователя пустое или пароль пустой, будет выполняться блок кода `else if`.

После входа в этот блок кода мы дополнительно проверяем, какое именно поле не заполнено:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Если два предыдущих условия не выполняются, это означает, что имя пользователя не пустое и пароль тоже не пустой.

Тогда программа перейдет в последний `else`:

```swift
print("Success")
```

Так мы завершили самую базовую логику проверки ввода для входа.

## Проверка имени пользователя и пароля

Приведенная выше логика лишь проверяет, «было ли что-то введено».

Но в реальном приложении просто ввести данные недостаточно. Обычно также нужно отправить имя пользователя и пароль на сервер для проверки.

Если сервер подтвердит, что имя пользователя существует и пароль верный, вход будет разрешен; в противном случае будет показано сообщение о неудачном входе.

Чтобы потренироваться в этом процессе, мы можем сначала задать в коде временный аккаунт и пароль, имитируя «правильные данные для входа»:

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

Когда и имя пользователя, и пароль не пусты, мы затем сравниваем их, чтобы проверить, правильны ли они:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

Здесь `==` означает «равны ли значения».

Например:

```swift
user == userID
```

То есть имя пользователя, введенное пользователем, полностью совпадает с правильным именем пользователя, которое мы задали; для пароля логика такая же.

Поэтому, когда введенные имя пользователя и пароль равны заданным значениям, выводится `Success`; если хотя бы одно из них не равно, выводится `Input error`.

Так мы завершили простую «имитацию проверки входа».

Хотя это еще не настоящая функция входа, подключенная к серверу, этого уже достаточно, чтобы помочь нам понять: **после нажатия кнопки можно выполнять разный код в зависимости от разных условий**.

## Итоги

В этом уроке мы завершили базовый экран входа и сосредоточились на изучении «логики определения введенного содержимого на основе условий».

Основное содержание этого урока — использование `isEmpty` для проверки, пуста ли строка, а также использование инструкции `if` вместе с логическими операторами для обработки разных ситуаций.

Среди них два очень важных логических оператора:

- `&&`: означает «и», оба условия должны быть выполнены
- `||`: означает «или», достаточно выполнения хотя бы одного условия

Когда эти условия в итоге дают `true` или `false`, инструкция `if` выполняет разный код в зависимости от результата.

Благодаря этому экрану входа мы уже начали знакомиться с подходом к разработке, который объединяет «интерфейс + данные + логические суждения».
