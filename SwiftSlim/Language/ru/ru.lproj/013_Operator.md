# Операторы

На этом уроке мы изучим операторы Swift (`Operators`).

Когда мы выполняем вычисления, сравнения или логические проверки, нам необходимы операторы. Эти действия проходят через все этапы выполнения программы и являются одной из самых базовых способностей в программировании.

```swift id="4m2qra"
let a = 100 * 8
```

В процессе изучения программирования многие замечают: людям с определённым математическим мышлением обычно легче понимать программную логику. Это связано с тем, что программирование по своей сути неотделимо от «вычислений» и «проверки правил», а именно это и составляет основную роль операторов.

В реальной разработке операторы используются не только для простых числовых вычислений, но и очень широко применяются в самых разных практических сценариях.

Например, в игре с блоками, когда пользователь перетаскивает блок в заданную позицию, нужно вычислить:

* координаты блока
* позиции, которые блок может заполнить
* доступна ли целевая позиция

Этот процесс «вычисление + проверка» по своей сути тоже опирается на операторы.

![Block](../../Resource/013_block.png)

Изучение операторов — это не просто изучение числовых вычислений, а подготовка основы для дальнейшего изучения построения интерфейсов, обработки данных и логики взаимодействия.

## Как выполняются вычисления в SwiftUI

В SwiftUI вычислительная логика обычно запускается через `Button`, а сами вычисления выполняются внутри функции.

Например:

```swift id="3l9h2a"
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

При нажатии на кнопку в `Console` выводится:

```swift id="t8kd2n"
c: 20
```

В этом уроке мы будем единообразно помещать вычислительную логику в `func`, запускать её через кнопку и использовать `print` для вывода результата.

В следующих примерах, чтобы упростить отображение кода, мы будем сразу подписывать результаты вычислений в коде и больше не будем каждый раз отдельно использовать `print`.

## Управление выводом в Console

Поскольку каждый клик создаёт новую строку вывода, можно нажать кнопку очистки в правом нижнем углу области `Console`, чтобы удалить историю логов и удобнее просматривать текущий результат.

## Другой способ записи `Button`

Когда кнопка выполняет только одну функцию, можно использовать более компактную форму записи:

```swift id="9wb9qx"
Button(action: ) {
    // View
}
```

В этой записи `action` указывает функцию, которая должна выполниться при нажатии кнопки (функция без параметров), а фигурные скобки `{}` используются для определения представления кнопки.

В сочетании с функцией `calculate`, показанной выше, это можно записать так:

```swift id="n7pp9d"
Button(action: calculate) {
    Text("Button")
}
```

При нажатии на кнопку функция `calculate` будет вызвана напрямую.

Поэтому, когда логика кнопки простая и она лишь вызывает одну функцию, запись через `action` выглядит более ясно и лучше соответствует стилю кода.

## Арифметические операторы

Арифметические операторы (`Arithmetic Operators`) используются для числовых вычислений:

* `+` сложение
* `-` вычитание
* `*` умножение
* `/` деление
* `%` остаток от деления

Пример:

```swift id="mhv0m7"
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

Нужно обратить внимание, что Swift — язык со строгой типизацией и не смешивает автоматически разные типы данных:

```swift id="6rdk5z"
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

Когда типы различаются, необходимо вручную преобразовать тип.

## Операторы сравнения

Операторы сравнения (`Comparison Operators`) используются для проверки отношения между двумя значениями, а результат имеет тип `Bool`:

* `==` равно
* `!=` не равно
* `>` больше
* `<` меньше
* `>=` больше или равно
* `<=` меньше или равно

Пример:

```swift id="hscwk9"
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

Операторы сравнения часто используются в условных выражениях. Например, в примере с «каруселью цитат» нам нужно контролировать, чтобы индекс не выходил за пределы массива:

```swift id="cv7u8e"
if index < sayings.count - 1 {
    index += 1
}
```

Если `index` меньше, чем `sayings.count - 1`, результат сравнения будет `true`, и код внутри `if` выполнится; если результат будет `false`, следующий код выполняться не будет.

## Логические операторы

Логические операторы (`Logical Operators`) используются для объединения нескольких условий:

* `&&` и (`AND`)
* `||` или (`OR`)
* `!` не (`NOT`)

### `&&` И

Когда нужно, чтобы одновременно выполнялись несколько условий, используется `&&`.

Например:

```swift id="7h3mbn"
let age = 16
let legalAge = age >= 18 && age <= 100
```

В этом коде есть два условия: `age >= 18` и `age <= 100`.

Только если оба этих условия равны `true`, `legalAge` тоже будет `true`; если хотя бы одно условие не выполнено, результат будет `false`.

### `||` Или

Когда достаточно выполнения хотя бы одного из нескольких условий, используется `||`.

Например:

```swift id="r0d8vh"
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

Этот фрагмент означает: если выполняется `money >= 50` или `creditCard >= 50`, то `payAllowed` будет `true`.

Только если оба условия не выполняются, результат будет `false`.

Например, в сценарии оплаты можно использовать наличные или кредитную карту. Если ни один из этих способов не подходит, оплата не пройдёт.

### `!` Не

`!` — это оператор логического отрицания (`NOT`), который инвертирует значение `Bool`.

Его можно понять так: `!` превращает `true` в `false`, а `false` — в `true`.

Например:

```swift id="5u8szs"
let allowed = true
let result = !allowed   // false
```

Здесь `!allowed` означает инверсию значения `allowed`.

Нужно помнить, что `!` и `!=` — это разные операторы: `!` инвертирует значение `Bool`, а `!=` является оператором сравнения и возвращает `Bool`.

## Операторы присваивания

Операторы присваивания (`Assignment Operators`) используются для присваивания или обновления значения переменной:

* `=` присваивание

При объявлении переменных и в процессе вычислений всегда используется операция присваивания:

```swift id="4lyc1i"
let a = 5
```

Этот код означает: присвоить числовое значение `5` переменной `a`.

### Составные операторы присваивания

В реальной разработке, помимо базового оператора присваивания, часто используются и **составные операторы присваивания**, которые позволяют сразу выполнить вычисление на основе текущего значения и обновить его:

* `+=`
* `-=`
* `*=`
* `/=`

На предыдущем уроке про «счётчик» мы уже кратко касались составных операторов присваивания.

Например, если нужно, чтобы при нажатии кнопки значение автоматически увеличивалось:

```swift id="ki7hoq"
var num = 10
num += 5   // num = 15
```

Этот код означает: выполнить сложение `num` и `5`, а затем снова присвоить результат переменной `num`.

То есть это эквивалентно следующему коду:

```swift id="ebv6v3"
var num = 10
num = num + 5   // num = 15
```

## Тернарный оператор

Тернарный оператор (`Ternary Operator`) в Swift выполняет проверку условия и возвращает одно из двух значений:

```swift id="g0xz4p"
условие ? значение1 : значение2
```

Логика выполнения такова: если условие равно `true`, возвращается «значение1»; если условие равно `false`, возвращается «значение2».

Например:

```swift id="7j0p7f"
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

Этот код проверяет значение `showColor`: если оно равно `true`, возвращается `Color.blue`, и фон будет синим; если `false`, возвращается `Color.clear`, то есть фон отображаться не будет.

Подсказка: `Color.clear` означает прозрачный цвет (ничего не отображается).

### Отличие от `if-else`

По логике тернарный оператор эквивалентен конструкции `if-else`.

Например, предыдущий код можно записать так:

```swift id="z0t7rk"
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

Логика выполнения у обоих вариантов полностью одинакова: если условие равно `true`, используется первый результат; если `false` — второй.

На практике тернарный оператор короче и лучше подходит для простых ситуаций «одно из двух», особенно когда нужно сразу вернуть значение.

При построении `View` в SwiftUI тернарный оператор встречается очень часто. Например:

```swift id="8vfe3r"
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

Такая запись позволяет в одной строке кода динамически менять внешний вид представления в зависимости от условия.

## Операторы диапазона

Операторы диапазона (`Range Operators`) используются для обозначения интервала:

* `...` закрытый диапазон (включает оба конца)
* `..<` полуоткрытый диапазон (не включает правую границу)

Например:

```swift id="2otjtw"
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

Это можно понимать так: `1...3` означает от `1` до `3`, включая `3`; `5..<8` означает от `5` до `8`, но без включения `8`.

Расширенные формы (только для ознакомления):

```swift id="7v7jv8"
5...    // >= 5
...5    // <= 5
```

Такая запись означает «односторонний диапазон» и часто используется в условных проверках или сопоставлении с образцом.

Операторы диапазона будут использоваться позже при изучении цикла `for`; пока достаточно понимать их как обозначение непрерывного числового промежутка.

## Практика с операторами

Теперь с помощью нескольких простых примеров посмотрим, как работают операторы в представлении.

### 1. Подсчёт общего количества автомобилей

Например, если через перекрёсток каждый день проезжает 500 машин, можно вычислить общее количество машин за 30 дней.

Для этого используем арифметический оператор:

```swift id="jlwmru"
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

Здесь логика вычисления такая: 500 машин в день, за 30 дней общее количество равно `500 × 30`, в итоге получаем 15000 машин.

### 2. Проверка статуса подписки на встроенные покупки

Можно определить, является ли человек участником платного членства: если у него есть либо пожизненный доступ, либо подписка, он считается участником.

```swift id="ummk29"
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

В этом примере используется оператор `||` (или): `lifeTime = false` означает, что пожизненного членства нет; `subscription = true` означает, что подписка активна.

Поскольку достаточно выполнения одного из условий, `isMember` будет `true`. Следовательно, этот человек является участником платного членства.

## Итог

В Swift операторы являются основой вычислений данных и логических проверок. Так же как `Text` отвечает за отображение содержимого, операторы отвечают за обработку самих данных.

От самых простых числовых вычислений до сложных логических проверок, а также управления состоянием представления и расчётов интерфейса — всё это невозможно без операторов.

Постепенно тренируясь на разных сценариях, можно шаг за шагом освоить и уверенно использовать различные типы операторов, представленные в этом уроке. Это важная основа для дальнейшего обучения.

### Дополнительные знания — побитовые операторы

Кроме того, Swift предоставляет ещё одну, более низкоуровневую группу операторов — **побитовые операторы** (`Bitwise Operators`):

* `&` (побитовое И)
* `|` (побитовое ИЛИ)
* `^` (побитовое исключающее ИЛИ)
* `~` (побитовое НЕ)
* `>>` (сдвиг вправо)
* `<<` (сдвиг влево)

Например:

```swift id="lgb2ng"
let a = 6  // двоичное 110
let b = 3  // двоичное 011

print(a & b) // 2 (двоичное 010)
print(a | b) // 7 (двоичное 111)
print(a ^ b) // 5 (двоичное 101)
print(~a)    // -7 (инверсия)
```

В Swift целые числа представлены в двоичном дополнительном коде, поэтому после побитового отрицания получается соответствующее отрицательное число.

Побитовые операторы напрямую работают с двоичным представлением данных и обычно используются при низкоуровневой обработке данных и в сценариях оптимизации производительности.

Для начинающих такие операторы встречаются редко; в обычной разработке под iOS или SwiftUI они тоже используются нечасто, поэтому здесь мы не будем разбирать их подробно.

Если позже вы столкнётесь с низкоуровневой логикой или задачами оптимизации, тогда можно будет изучить их глубже.
