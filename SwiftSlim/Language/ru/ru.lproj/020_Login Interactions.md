# Взаимодействие при входе

На прошлом уроке мы создали view входа и с помощью кнопок и условной логики реализовали простую проверку введённого содержимого.

На этом уроке мы продолжим работу на основе кода из прошлого урока и изучим поле ввода пароля `SecureField`, окно предупреждения `Alert`, а также то, как управлять отображением разных view в зависимости от переменных.

С помощью этого урока вы сможете лучше понять базовый поток взаимодействия в view входа. Это также базовая логика, которая часто используется во многих приложениях.

## Поле ввода пароля

На прошлом уроке мы использовали `TextField` для ввода пароля.

Однако в реальном интерфейсе входа пароль обычно не показывается напрямую. Вместо этого он отображается в виде точек или с другим скрывающим эффектом, что помогает защитить конфиденциальность пользователя.

В SwiftUI элемент управления, специально предназначенный для ввода пароля, называется `SecureField`.

Например:

```swift
SecureField("Password", text: $password)
```

Его синтаксис очень похож на `TextField`. Ему также нужны текст-подсказка и привязанная переменная.

Это можно просто понять так: `SecureField` тоже является полем ввода, только оно скрывает вводимое содержимое.

Мы можем напрямую заменить `TextField`, который изначально использовался для ввода пароля, на `SecureField`:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Когда пользователь вводит пароль, его содержимое больше не будет показываться напрямую в интерфейсе.

Теперь мы можем ввести имя пользователя и пароль, а затем нажать кнопку, чтобы определить, верны ли введённые данные.

### Отображение сообщений обратной связи

Ранее мы постоянно использовали `print` для вывода результатов.

Содержимое `print` отображается только в консоли. Разработчики могут видеть его при запуске программы в Xcode, но обычные пользователи на телефоне не видят эти сообщения.

Это значит, что, хотя программа уже завершила проверку, пользователь всё равно не знает, правильно ли введено содержимое.

В этот момент нам нужен способ отображать сообщения прямо в интерфейсе.

## Окно предупреждения Alert

В SwiftUI мы можем использовать `Alert`, чтобы вывести всплывающее окно сообщения.

Базовое использование:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

Этот код выглядит немного сложнее, чем предыдущие элементы управления, но мы можем понять его, если разберём по частям.

`Alert` в основном содержит следующие части:

- `"Title"`: заголовок окна предупреждения
- `isPresented`: привязанное логическое значение, используемое для управления тем, показывать ли окно предупреждения
- `Button(...)`: кнопка в окне предупреждения
- `message:`: текст сообщения, отображаемый в окне предупреждения

Это можно просто понять так: когда привязанная переменная становится `true`, система показывает это окно предупреждения.

### Alert является модификатором

Как и многие модификаторы, которые мы изучали раньше, `Alert` тоже добавляется после определённого view.

Например, он может быть прикреплён после кнопки:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

Он также может быть прикреплён после внешнего `VStack`:

```swift
VStack {
    // ...
}
.alert() {...}
```

Это означает, что `Alert` не является самостоятельным view, размещённым отдельно в интерфейсе. Вместо этого это модификатор, прикреплённый к определённому view.

Это можно просто понять так: `Alert` обычно пишется после кнопки или после внешнего layout, чтобы показывать сообщение при выполнении определённого условия.

### Показ Alert

Если мы хотим показать `Alert`, нам нужна переменная для управления им.

Например:

```swift
@State private var showAlert = false
```

Эта переменная имеет тип `Bool`, то есть у неё только два значения: `true` или `false`.

Затем привяжите её к `isPresented` у `Alert`:

```swift
isPresented: $showAlert
```

Когда `showAlert` равно `false`, окно предупреждения не будет показано. Когда `showAlert` равно `true`, окно предупреждения появится.

### Показ окна предупреждения с помощью кнопки

Теперь сначала реализуем самый простой сценарий:

Когда пользователь вообще ничего не вводит и сразу нажимает кнопку `Sign in`, появляется окно сообщения, которое говорит ему: «Вы ещё не ввели никакое содержимое».

Сначала создайте переменную для управления этим окном сообщения:

```swift
@State private var showEmpty = false
```

Затем добавьте `Alert` после `Button`:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

Далее выполните проверку в коде нажатия кнопки:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

Если и `user`, и `password` пусты, установите `showEmpty` в `true`.

А поскольку `Alert` привязан к этой переменной, когда она становится `true`, система автоматически показывает окно предупреждения.

Эффект отображения:

![alert](../../Resource/020_alert1.png)

Таким образом, после нажатия кнопки пользователь может сразу увидеть сообщение, и разработчику не нужно проверять вывод `print` в консоли.

### Улучшение кнопки и окон предупреждения

Ранее мы обработали только случай, когда «ничего не было заполнено».

Нам также нужно обработать следующие ситуации:

- имя пользователя не введено
- пароль не введён
- имя пользователя и пароль введены, но данные неверны

Поэтому нам также нужно подготовить соответствующие сообщения для этих случаев.

Сначала добавьте три новые переменные:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

Затем продолжайте добавлять соответствующие `Alert` после кнопки:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

Далее завершите логику принятия решений внутри кнопки:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

Когда кнопка определяет, что пользователь не заполнил никакую информацию, пропустил одну часть информации или не прошёл проверку введённых данных, соответствующая логическая переменная становится `true`, а затем появляется соответствующий `Alert`.

Здесь я рекомендую вам самостоятельно протестировать несколько ситуаций:

- ничего не вводить
- ввести только имя пользователя
- ввести только пароль
- ввести и имя пользователя, и пароль, но с неверным содержимым
- ввести и имя пользователя, и пароль, и содержимое верное

Это помогает проверить, есть ли проблемы в нашем коде. Это также очень важный этап «тестирования» в разработке приложений.

Если в каком-то сценарии программа ведёт себя неправильно, это означает, что в логике может быть проблема и в программе появился `Bug`.

То, что разработчики обычно называют `Bug`, означает ошибку, дефект или поведение, которое не соответствует ожиданиям.

## Завершение view входа

Примеры `Alert`, о которых мы говорили ранее, были посвящены тому, как давать пользователю обратную связь, когда «ввод неправильный».

Когда пользователь вводит правильные имя пользователя и пароль, мы обычно не показываем «окно предупреждения». Вместо этого мы сразу переходим в интерфейс после входа.

Здесь затрагивается очень важная концепция: **отображение разных view в зависимости от значения переменной.**

Например:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

Здесь мы используем тип `Bool`, чтобы управлять тем, какое view показывается:

- если `login` равно `true`, это означает, что вход уже успешен, и показывается содержимое после входа
- если `login` равно `false`, это означает, что вход не удался, и интерфейс входа продолжает отображаться.

Хотя оператор `if` является условным оператором в Swift, внутри `body` в SwiftUI он может использоваться для управления логикой отображения view.

Это означает, что SwiftUI определяет, какое view должно быть показано в данный момент, на основе результата условия.

Мы можем поместить эту логику отображения в текущий `ContentView`:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Код исходного view входа
		}
	}
}
```

Здесь мы добавили новую переменную `login`, чтобы указать, успешно ли пользователь уже вошёл в систему.

Затем мы поместили код view входа, который уже написали, после оператора `else`.

Логика отображения view:

- когда `login` равно `false`, это означает, что вход ещё не был успешным, поэтому продолжается показ исходного view входа.
- когда `login` равно `true`, это означает, что вход успешен. В этот момент view входа больше не показывается, а вместо него показывается `Login successful`.

Это означает, что `ContentView` будет показывать разное содержимое view в зависимости от значения `login`.

Далее давайте изменим логику успешного входа внутри кнопки:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

Когда имя пользователя и пароль, введённые пользователем, оба правильные, мы устанавливаем `login` в `true`.

Как только значение `login` изменяется, меняется и результат проверки `if` в `ContentView`, и интерфейс переключается с исходного view входа на `Login successful`.

Эффект отображения выглядит следующим образом:

![view](../../Resource/020_view.png)

Теперь мы действительно применили идею «отображения разных view в зависимости от условий» на этой текущей странице входа.

В реальных приложениях обычно всё работает именно так: разные view отображаются в зависимости от определённого значения.

Например, когда пользователь не вошёл в систему, показывается страница входа. После успешного входа пользователя показывается другой интерфейс.

Ключевая идея такова: **использовать изменения переменных для управления тем, какое содержимое показывает интерфейс.**

## Итог

На этом уроке мы изучили три важные вещи:

1. Использовать `SecureField` для ввода паролей, чтобы содержимое пароля не отображалось напрямую в интерфейсе.
2. Использовать `Alert` для отображения сообщений пользователю, а не выводить их только в консоль.
3. Использовать переменные и условия `if` для управления отображением разных view, что позволяет переключаться между интерфейсами до и после входа.

Изучив эти знания, мы уже можем завершить view входа с базовыми эффектами взаимодействия и лучше понять идею разработки в SwiftUI: «когда состояние меняется, интерфейс обновляется».

## Полный код

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## Дополнительные знания - Alert с несколькими кнопками

В предыдущей части этого урока мы использовали самый базовый `Alert`, у которого только одна кнопка и который в основном используется для уведомления пользователя.

На самом деле `Alert` также может содержать несколько кнопок, чтобы пользователь мог сделать разные выборы.

Например:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

После нажатия кнопки появится окно сообщения с тремя вариантами: `Ok`, `delete` и `cancel`.

Эффект отображения:

![alert](../../Resource/020_alert2.png)

При нажатии разных кнопок будет выполняться соответствующий код.

### Свойство `role` у Button

Здесь вы заметите, что `Button` внутри `Alert` немного отличается от обычного синтаксиса кнопки, который мы изучали ранее:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

Здесь есть дополнительный параметр `role`.

В SwiftUI `role` используется для того, чтобы сообщить системе, какой тип действия представляет эта кнопка.

Некоторые распространённые роли:

- `.cancel`: действие отмены
- `.destructive`: опасное действие, например удаление
- `nil`: обычное действие

Разные значения `role` обычно также приводят к различиям во внешнем стиле интерфейса.

Например, опасные действия часто отображаются в более заметном стиле, чтобы напомнить пользователям о необходимости нажимать осторожно.

Кроме того, `role` также предоставляет кнопке дополнительную семантическую информацию, благодаря чему экранные дикторы, например VoiceOver на устройствах Apple, могут проще объяснить пользователю назначение кнопки.

Поэтому в подходящих ситуациях добавлять `role` к кнопке — очень хорошая привычка.
