# Как заставить кнопку работать

В этом уроке мы в основном изучим кнопку Button, вывод отладочной информации через print и функции в Swift.

Это базовые элементы интерактивного программирования в SwiftUI.

## Кнопка

Кнопка — один из самых распространённых элементов UI. Когда нам нужно что-то скачать, открыть или закрыть, мы обычно нажимаем кнопку.

Например, в App Store, чтобы скачать приложение, нужно нажать кнопку «Получить».

![Button](../../RESOURCE/008_button.png)

В SwiftUI кнопка обозначается с помощью Button.

Базовое использование:

```swift
Button("") {
    
}
```

Эту структуру можно разделить на две части:

```swift
Button("Текст кнопки") {
    Код, который выполняется при нажатии кнопки
}
```

Например:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

После запуска на экране появится кнопка Start.

![Button](../../RESOURCE/008_button1.png)

Когда пользователь нажимает кнопку, код внутри {} выполняется.

Обратите внимание: Button относится к элементам интерфейса SwiftUI (View), поэтому его нужно писать внутри body.

### Модификатор buttonStyle

В SwiftUI Button предоставляет встроенные системные стили кнопок, которые можно использовать через модификатор buttonStyle.

Например:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

buttonStyle применяет системный стиль кнопки.

Распространённые варианты:

* .automatic
* .plain
* .bordered
* .borderedProminent
* .borderless

![Button](../../RESOURCE/008_button7.png)

Различия между стилями в основном заключаются в наличии рамки, фона и степени визуального акцента.

## Вывод через print

В программировании часто нужно проверить, дошла ли программа до определённого участка кода.

В Swift для вывода отладочной информации используется функция print.

Базовая запись:

```swift
print("Hello")
```

Эта строка выведет текст в Console (консоль).

### Проверка работоспособности кнопки

Мы можем поместить print внутрь кнопки, чтобы проверить, была ли она нажата.

```swift
Button("Start") {
    print("Test 123")
}
```

Когда вы нажмёте кнопку, в Console появится:

```swift
Test 123
```

Это означает, что кнопка успешно сработала.

![Button](../../RESOURCE/008_button3.png)

Обратите внимание: print — это код Swift, а не элемент интерфейса. Его нельзя напрямую помещать в body, иначе возникнет ошибка.

Например, такой код будет неверным:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // Ошибка: body должен возвращать View
    }
}
```

В SwiftUI body используется для описания структуры интерфейса, поэтому внутри него можно размещать только элементы интерфейса (View), например Text, Button, Image и так далее.

Обычный код (вычисления, print и т. д.) нужно помещать в кнопку, функцию или другой обработчик событий.

### Консоль Console

Содержимое, выводимое через print, отображается в Console (консоли) Xcode.

Если Console не отображается, можно нажать кнопку Show the Debug Area в правом нижнем углу, чтобы открыть область отладки, а затем нажать Show the Console, чтобы показать консоль.

![](../../RESOURCE/008_button4.png)

В левом нижнем углу Console есть две вкладки: Executable и Previews.

![](../../RESOURCE/008_button5.png)

Эти две вкладки соответствуют разным средам запуска: если вы используете Canvas Preview, вывод print будет отображаться во вкладке Previews; если приложение запущено в симуляторе или на реальном устройстве, вывод будет отображаться во вкладке Executable.

Поэтому, если вы не видите вывод print, сначала проверьте, выбрана ли правильная вкладка.

## Пример — ASCII-арт из символов

Во многих кодовых проектах часто можно увидеть ASCII-арт.

ASCII-арт — это рисунки, составленные из обычных символов. Например:

![ASCII](../../RESOURCE/008_ascii.png)

Мы можем использовать кнопку + print, чтобы вывести ASCII-арт.

Пример кода:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Когда вы нажмёте кнопку, Console выведет:

```swift
 /\_/\
( o.o )
 > ^ <
```

Обратите внимание: символ \ в строке нужно писать как \, потому что обратная косая черта в строках является символом экранирования.

## Функции

Внутри кнопки мы можем писать код напрямую.

Например:

```swift
Button("Start") {
    print("Hello")
}
```

Если кода немного, такой вариант вполне подходит. Но в реальной разработке логика, срабатывающая по нажатию кнопки, может быть довольно большой.

Например:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // Здесь может быть ещё много кода
}
```

Если кода становится всё больше, размещать его прямо внутри кнопки неудобно: код интерфейса становится громоздким, его трудно читать и поддерживать.

Поэтому обычно такой код выносят в функцию, а затем вызывают эту функцию внутри кнопки.

### Что такое функция

Функция — это блок кода, который можно использовать повторно.

Когда нужно выполнить определённый код, достаточно вызвать эту функцию, и будет выполнен весь код внутри неё.

Это делает структуру программы более понятной и упрощает повторное использование кода.

### Базовая запись

В Swift функция определяется с помощью ключевого слова **func**:

```swift
func randomInt() {
    // code
}
```

Этот код определяет функцию.

randomInt — это имя функции, оно используется для обозначения этого блока кода.

() обозначает место для параметров. Здесь можно принимать данные, переданные извне. Если данные не нужны, скобки можно оставить пустыми.

{} — это область кода функции. Всё, что должно выполняться, пишется внутри.

Например:

```swift
func getName() {
    print("FangJunyu")
}
```

Задача этой функции — вывести текст.

### Вызов функции

Определение функции только создаёт этот блок кода. Чтобы он выполнился, функцию нужно вызвать.

Способ вызова — написать имя функции и добавить после него ():

```swift
getName()
```

Когда программа дойдёт до этой строки, выполнится код внутри функции getName.

Полный пример:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

После запуска Console выведет:

```swift
FangJunyu
```

### Использование функций в SwiftUI

В представлении SwiftUI функции обычно пишут вне body.

Например:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

Когда пользователь нажимает кнопку, вызывается функция getName().

Это позволяет сохранить код интерфейса чистым, а конкретную логику вынести в функцию.

### Параметры функции

Иногда функция должна выполнять разные действия в зависимости от переданных данных.

Например, мы хотим, чтобы функция могла выводить разные имена, а не только фиксированное "FangJunyu".

Для этого используются параметры. Параметры можно понимать как данные, передаваемые при вызове функции.

Например:

```swift
func getName(name: String) {
    print(name)
}
```

У этой функции появился параметр: name — это имя параметра, String — его тип.

Это означает, что функция должна получить данные типа String.

Поскольку теперь функции нужны данные, при вызове эти данные нужно обязательно передать.

```swift
getName(name: "Sam")
```

Когда при вызове передаётся "Sam", функция использует это значение для вывода.

Результат выполнения:

```swift
Sam
```

## Пример — вывод ASCII-арта через функцию

В предыдущем примере мы писали print прямо внутри кнопки.

Но если ASCII-арт становится более сложным, код становится слишком длинным. В таком случае можно вынести логику в функцию, чтобы код интерфейса оставался более понятным.

Пример кода:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

После запуска и нажатия кнопки Console выведет:

```swift
 /\_/\
( o.o )
 > ^ <
```

Здесь происходят две вещи, когда пользователь нажимает кнопку:

1. Кнопка вызывает функцию printCat()
2. Выполняется код print внутри функции

Это позволяет коду интерфейса отвечать только за запуск события, а конкретную логику обрабатывать внутри функции.

В реальной разработке большинство действий кнопок реализуется именно через функции.

## Итог

В этом уроке мы изучили три важных элемента:

**1. Кнопка Button**

Button — это элемент SwiftUI, который используется для запуска действий.

```swift
Button("Start") {

}
```

Когда пользователь нажимает кнопку, выполняется код внутри фигурных скобок.

**2. Вывод через print**

print используется для вывода отладочной информации в Console.

```swift
print("Hello")
```

Разработчик может использовать Console, чтобы отслеживать состояние выполнения программы.

**3. Функции**

Функция — это блок кода, который можно использовать повторно.

```swift
func sayHello() {
    print("Hello")
}
```

Вызов функции:

```swift
sayHello()
```

Если функции нужны данные, можно использовать параметры:

```swift
func sayHello(name: String) {
    print(name)
}
```

В следующих уроках мы продолжим изучать другие элементы SwiftUI и то, как данные взаимодействуют с интерфейсом.
