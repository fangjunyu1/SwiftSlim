# Приложение Fill Light

В этом уроке мы создадим очень интересное приложение fill light. Когда наступает ночь, мы можем заставить экран телефона показывать разные цвета и использовать его как простой источник заполняющего света.

Это приложение fill light может переключать цвета по нажатию на экран, а также регулировать яркость с помощью ползунка.

В этом примере мы изучим, как использовать `brightness` для регулировки яркости представления, `onTapGesture` для добавления жеста нажатия к представлению, а также элемент управления `Slider`.

## Отображение цвета

Сначала давайте сделаем так, чтобы представление показывало один цвет.

В SwiftUI `Color` не только обозначает цвет, но и может отображаться как представление:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

Здесь `Color.red` означает красное представление. `.ignoresSafeArea()` заставляет цветное представление заполнить весь экран, поэтому оно больше похоже на настоящий эффект fill light.

Результат:

![Color](../../Resource/018_color1.png)

### Массив цветов и индекс

Сейчас отображается только один цвет. Но fill light обычно не ограничивается одним цветом. Он также может показывать синий, жёлтый, фиолетовый, белый и другие цвета.

Мы хотим переключаться между разными цветами при нажатии на экран. Для этого можно поместить эти цвета в массив и управлять ими вместе:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

Массив подходит для хранения «группы данных одного типа». Здесь каждый элемент массива — это `Color`.

Если мы хотим показать определённый цвет, можно использовать индекс:

```swift
colors[0]
```

Это означает чтение цвета по индексу `0` в массиве, то есть первого цвета.

Теперь код можно записать так:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

Таким образом, на экране будет отображаться первый цвет в массиве, то есть красный.

### Управление цветом через индекс

Если нам нужно переключаться между разными цветами, нам нужна переменная для управления индексом, а не жёстко заданное значение индекса.

Мы можем использовать `@State`, чтобы объявить переменную, которая хранит индекс:

```swift
@State private var index = 0
```

Здесь `index` представляет индекс текущего цвета.

Когда `index` изменяется, SwiftUI заново вычисляет интерфейс и обновляет отображаемое содержимое.

Затем мы меняем прежнее `colors[0]` на:

```swift
colors[index]
```

Таким образом, цвет, который отображается в представлении, определяется значением `index`.

Теперь код становится таким:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

Когда `index` меняется, `colors[index]` тоже будет показывать другой цвет.

Например:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

Важно помнить, что `index` не может превышать максимальный индекс массива, иначе возникнет ошибка выхода индекса за пределы массива.

## Жест нажатия

Теперь мы можем отображать разные цвета в зависимости от `index`, но пока ещё не можем нажатием переключать их.

В предыдущем уроке «Карусель цитат» мы использовали `Button`, чтобы управлять переключением цитат.

Но на этот раз мы хотим «нажимать на всю цветную область», чтобы менять цвет, поэтому `onTapGesture` подходит лучше.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

Когда мы нажимаем на цветное представление, выполняется этот код:

```swift
index += 1
```

Это означает увеличить `index` на `1`. После увеличения индекса `colors[index]` покажет следующий цвет в массиве.

### `onTapGesture`

`onTapGesture` — это модификатор жеста, который добавляет действие нажатия к представлению.

Базовое использование:

```swift
.onTapGesture {
    // code
}
```

Например:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

Когда на это красное представление нажимают, код внутри фигурных скобок выполняется, а в консоль выводится:

```swift
Click color
```

С помощью `onTapGesture` мы можем задать, что произойдёт после нажатия на представление.

### Отличие от `Button`

Ранее мы изучали представление `Button`. И `Button`, и `onTapGesture` могут обрабатывать нажатия, но сценарии их использования не совсем одинаковы.

`onTapGesture` больше подходит для «добавления функции нажатия к уже существующему представлению», например `Color`, `Image`, `Text` или другому обычному представлению.

`Button` больше подходит тогда, когда нужно представить явную кнопку, например «Подтвердить», «Отправить» или «Удалить».

В этом приложении fill light мы хотим сделать переключение цветов более простым. Нажатие на всю цветную область меняет цвет, поэтому здесь `onTapGesture` — хороший выбор.

## Проблема индекса

Теперь мы можем нажимать на экран и переключаться между разными цветами.

Но здесь есть важная проблема: **индекс может выйти за пределы массива**.

Например:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

Если продолжать нажимать на экран, `index` в итоге станет `4`, и тогда возникнет ошибка «index out of range».

Это происходит потому, что в массиве `colors` есть `4` элемента, но индексация начинается с `0`, поэтому допустимый диапазон индексов — `0 - 3`, а не `4`.

Если обратиться к `colors[4]`, возникнет ошибка «index out of range».

В текущем коде каждое нажатие автоматически увеличивает `index` на `1`. Если мы это не обработаем, он в конце концов обязательно выйдет за допустимый диапазон.

Поэтому при нажатии на экран нам нужно проверять индекс: если это уже последний цвет, возвращаемся к первому; иначе продолжаем прибавлять `1`.

Это можно реализовать с помощью `if`:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

В этом коде `colors.count` означает количество элементов в массиве.

В текущем массиве 4 цвета, поэтому:

```swift
colors.count // 4
```

Но максимальный индекс — не `4`, а `3`, потому что индексация начинается с `0`.

Поэтому последний индекс нужно писать так:

```swift
colors.count - 1
```

То есть:

```swift
4 - 1 = 3
```

Эта логика означает: если текущий индекс уже соответствует последнему цвету, сбросьте его на `0`; иначе увеличьте его на `1`.

Таким образом, цвета будут переключаться по кругу.

### Упрощение логики индекса

Если мы хотим сделать код короче, можно использовать тернарный оператор:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

Этот код означает: если `index == colors.count - 1` истинно, верни `0`. Иначе верни `index + 1`.

В конце результат снова присваивается переменной `index`.

Теперь мы можем реализовать эффект смены цвета.

Полный код:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

Теперь мы можем нажимать на экран и переключаться между разными цветами. Базовое приложение fill light готово.

## Отображение названия цвета

Мы можем добавить ещё одну группу текста, соответствующую цветам, чтобы при смене цвета на экране также отображалось название текущего цвета.

Например:

- Красный показывает `Red`
- Синий показывает `Blue`
- Жёлтый показывает `Yellow`
- Фиолетовый показывает `Purple`

Здесь мы также можем использовать массив для хранения названий цветов:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

Порядок текста в этом массиве должен точно соответствовать порядку цветов в массиве цветов.

Затем мы можем использовать `Text`, чтобы показать название цвета для текущего индекса:

```swift
Text(colorsName[index])
```

`Text` отображает текущее название цвета в соответствии с `index`.

Используем модификаторы, чтобы улучшить вид `Text`:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

Это сделает `Text` белым, с размером заголовка и полужирным шрифтом.

Теперь у нас есть полноэкранное представление `Color`. Если мы хотим, чтобы `Text` отображался поверх представления `Color`, нам нужно использовать контейнер компоновки `ZStack`, чтобы наложить их друг на друга.

```swift
ZStack {
    Color
    Text
}
```

Тогда код станет таким:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

Таким образом, фоном будет цветное представление, а поверх него будет наложено текстовое представление.

Результат:

![Color](../../Resource/018_color2.png)

Важно отметить, что в `ZStack` представления, написанные позже, обычно показываются впереди. Если `Text` написать перед `Color`, более позднее представление `Color` может закрыть `Text`.

## Управление яркостью

Теперь мы можем менять разные цвета, но у fill light есть ещё одна важная функция: **регулировка яркости**.

В SwiftUI мы можем использовать модификатор `brightness`, чтобы регулировать яркость представления.

Например:

```swift
.brightness(1)
```

Мы можем написать:

```swift
colors[index]
    .brightness(0.5)
```

Это делает текущий цвет ярче и ближе к эффекту fill light.

Диапазон яркости — `0 - 1`. `0` означает сохранить исходный цвет, а чем ближе значение к `1`, тем ярче становится цвет. `1` означает самый яркий белый результат.

Хотя мы можем управлять `brightness` в коде, пользователь всё ещё не может регулировать его напрямую сам.

Поэтому нужно добавить элемент управления, который можно перетаскивать: `Slider`.

## Представление `Slider`

В SwiftUI `Slider` — это элемент управления, который используется для выбора значения в определённом диапазоне. Apple описывает его как «элемент управления для выбора значения из ограниченного линейного диапазона».

Базовое использование:

```swift
Slider(value: $value, in: 0...1)
```

Объяснение параметров:

1. `value: $value`: `Slider` должен быть связан с переменной.

    Когда ползунок перемещается, значение переменной изменяется одновременно. И наоборот, если переменная меняется, ползунок тоже обновляется.

    Это очень похоже на `TextField`, который мы изучали ранее. Оба «связывают элемент управления с переменной».

    Перед связанной переменной должен стоять символ `$`, чтобы показать binding.

2. `in: 0...1`: этот параметр представляет диапазон значений ползунка.

    Здесь `0...1` означает, что минимальное значение — `0`, а максимальное — `1`.

    Когда ползунок перемещён полностью влево, связанная переменная близка к `0`; когда он перемещён полностью вправо, она близка к `1`.

Например:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` связан с переменной `value`. Когда ползунок перемещается, значение переменной `value` тоже меняется одновременно.

Внешний вид:

![Slider](../../Resource/018_slider.png)

Когда `Slider` перемещается влево, связанное значение `value` становится `0`. Когда он перемещается вправо, оно становится `1`.

### Диапазон значений

Диапазон значений `Slider` не фиксирован. Его также можно записать так:

```swift
0...100
```

или как другой диапазон.

Но в этом приложении fill light нам нужно управлять яркостью, поэтому `0...1` — самый подходящий выбор.

## Использование `Slider` для управления яркостью

Теперь нам нужно связать `Slider` с `brightness`.

Сначала создадим переменную для хранения значения яркости:

```swift
@State private var slider = 0.0
```

Здесь `0.0` — это значение типа `Double`.

Поскольку `Slider` обычно связан с числовым типом, а здесь мы хотим, чтобы он менялся плавно, `Double` подходит лучше. Кроме того, `brightness` принимает только значение типа `Double`.

Затем передадим это значение в `brightness`:

```swift
colors[index]
    .brightness(slider)
```

Когда `slider == 0`, цвет остаётся в своём обычном состоянии. Чем ближе `slider` к `1`, тем ярче выглядит цвет.

### Добавление элемента управления `Slider`

Затем добавим элемент управления `Slider`, чтобы изменять эту переменную:

```swift
Slider(value: $slider, in: 0...1)
```

Когда ползунок меняется, значение `slider` тоже меняется, а `brightness(slider)` одновременно обновляет яркость.

Это очень типичный пример того, как в SwiftUI «переменные управляют представлением».

### Настройка внешнего вида `Slider`

По умолчанию `Slider` использует доступную ширину.

Мы можем задать ему фиксированную ширину:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

Затем добавим ещё несколько модификаторов, чтобы он был заметнее:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

Это даёт ползунку белый фон и скруглённые углы, благодаря чему он лучше выделяется поверх представления `Color`.

Наконец, поместим его в нижнюю часть экрана.

Поскольку мы уже используем `ZStack`, мы можем поместить внутри него `VStack` и с помощью `Spacer()` сдвинуть `Slider` вниз.

## Полный код

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## Итоги

Используя знания, которые мы изучили ранее, вместе с базовыми понятиями вроде цветов и массивов, мы создали очень интересное приложение fill light.

Благодаря этому приложению fill light мы научились использовать `brightness` для регулировки яркости, `onTapGesture` для добавления действий нажатия к представлениям и элемент управления `Slider`.

Мы добавили `onTapGesture` к представлению `Color`, чтобы менять цвета. Мы также использовали элемент управления `Slider`, чтобы изменять переменную, которая управляет `brightness`, и это ещё один пример того, что «переменные управляют представлением».

Мы также повторили тернарный оператор, использовали `ZStack` для наложения представлений и массивы для управления группой данных одного типа. Это помогает углубить понимание массивов и индексов, включая практическую задачу избегания ошибок выхода индекса за пределы массива.

Хотя этот пример несложный, он связывает между собой многие базовые идеи, которые мы изучили раньше. Когда они объединяются в маленьком реальном проекте, становится легче понять, для чего нужна каждая из этих концепций.
