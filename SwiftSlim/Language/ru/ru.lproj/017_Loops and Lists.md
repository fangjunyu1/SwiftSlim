# Циклы и списки

В этом уроке мы изучим цикл `for` в Swift и `ForEach` в SwiftUI.

Они оба помогают по порядку читать элементы массива. В реальной разработке нам часто нужно по одному доставать группу данных, а затем обрабатывать или отображать их, и цикл как раз является важным инструментом для выполнения такой задачи.

Когда мы просматриваем веб-страницы или используем приложение, мы часто видим множество списков или элементов. Такой контент обычно сначала хранится в `массиве`, затем по одному считывается с помощью `цикла`, а в конце отображается в интерфейсе.

Например, список результатов поиска Google:

![Google](../../Resource/017_google.png)

Каждый элемент в списке результатов поиска обычно соответствует одной единице данных и поочерёдно отображается с помощью цикла.

Поэтому `массив` и `цикл` часто появляются вместе. `Массив` отвечает за хранение группы данных, а `цикл` — за то, чтобы по порядку доставать эти данные и выполнять одно и то же действие.

Ниже мы начнём с самого базового цикла `for`.

## Обратный отсчёт

Предположим, мы хотим смоделировать функцию обратного отсчёта и по очереди вывести `«5、4、3、2、1»`.

Самый прямой способ записи такой:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

Такой способ действительно может имитировать обратный отсчёт, но если содержимого много, код будет выглядеть очень повторяющимся, а в дальнейшем его будет неудобно изменять.

В этом случае можно сначала поместить данные в массив, а затем использовать цикл `for`, чтобы выводить их по очереди:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Такая запись более компактна и её легче расширять. Даже если чисел в обратном отсчёте будет много, нам не нужно будет построчно повторять `print`, достаточно позволить циклу автоматически обработать каждый элемент массива.

Мы можем поместить этот код в кнопку и протестировать:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

Когда мы нажимаем кнопку, цикл `for` в порядке, заданном в массиве, поочерёдно достаёт каждое целое число из `countDown` и выводит его в консоль.

Далее посмотрим, как именно работает цикл `for`.

## Цикл for

Цикл `for` используется для последовательной обработки группы данных. Чаще всего это массивы и диапазоны.

Базовая форма записи:

```swift
for элемент in массив_или_диапазон {
    выполняемый код
}
```

Например, код обратного отсчёта, который был выше:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Здесь `i` обозначает текущий извлечённый элемент, а после `in` идёт набор данных, по которому выполняется цикл, то есть массив `countDown`.

Смысл этого кода такой: программа будет по порядку брать каждый элемент из массива `countDown`. Каждый раз, когда берётся один элемент, он временно обозначается как `i`, а затем выполняется код внутри фигурных скобок.

Процесс выполнения можно понять так:

* Сначала извлекается `5`, поэтому в этот момент значение `i` равно `5`
* Выполняется `print(i)`, и выводится `5`
* Затем берётся следующий элемент `4`
* Снова выполняется `print(i)`, и выводится `4`

Следующие `3`、`2`、`1` тоже будут выполняться по той же схеме по очереди.

![for](../../Resource/017_for.png)

Когда все элементы массива будут обработаны, цикл `for` автоматически завершится.

С помощью цикла `for` мы можем получить каждый элемент массива и реализовать такие функции, как обратный отсчёт.

**Подсказка**: `i` — это очень распространённый способ именования в цикле for, который используется для обозначения текущего извлечённого элемента, но это не фиксированное правило. Можно использовать и другие английские названия.

Например, использовать `num` для обозначения каждого извлечённого элемента:

```swift
for num in countDown {
    print(num)
}
```

Внутри фигурных скобок тоже нужно использовать `num` для обозначения текущего извлечённого элемента.

### Диапазон в цикле

Цикл `for` может проходить не только по массиву, но и по диапазону.

Например, когда нам нужно вычислить сумму положительных целых чисел до `50`, нужно поочерёдно сложить все целые числа от `1` до `50`.

Самая прямая запись могла бы выглядеть так:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

Проблема такой записи в том, что нам пришлось бы вручную выписывать каждое число. Это неудобно и не подходит для реальных сценариев разработки.

Здесь как раз можно использовать оператор диапазона вместе с циклом `for`.

В предыдущей главе про «операторы» мы уже изучали оператор диапазона:

```swift
1...3   // 1, 2, 3
```

Здесь `...` обозначает закрытый диапазон, то есть включает и начальную, и конечную точки.

Поэтому здесь можно использовать цикл `for` для диапазона целых чисел от `1` до `50`:

```swift
for i in 1...50 {
    
}
```

Нам нужно складывать эти целые числа, поэтому также нужна переменная для хранения общей суммы:

```swift
var count = 0
```

Затем при каждом проходе цикла мы добавляем текущее извлечённое число к `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

Этот код означает:

* `count` используется для хранения текущей суммы
* Цикл `for` будет по очереди брать целые числа от `1` до `50`
* Каждый раз, когда извлекается одно целое число, выполняется `count += i`
* После завершения цикла в `count` будет храниться итоговая сумма

Так мы можем получить сумму всех положительных целых чисел до `50`.

Точно так же, если нам нужно вычислить сумму целых чисел от `1` до `100`, нужно просто изменить диапазон:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

Цикл `for` может проходить не только по массиву, но и по диапазону. Такая способность последовательно и повторяющимся образом обрабатывать данные очень часто встречается в разработке.

Она помогает нам автоматически выполнять много повторяющейся работы, не заставляя вручную прописывать каждое сложение числа или другие подобные повторяющиеся операции.

## Циклы в SwiftUI

Ранее рассмотренный цикл `for` относится к самому языку Swift.

Но в SwiftUI мы не можем напрямую использовать `for` внутри структуры представления для генерации view:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

Это связано с тем, что внутри `VStack` должны находиться SwiftUI view, а обычный цикл `for` сам по себе view не является.

Если мы хотим в SwiftUI повторно генерировать интерфейс на основе содержимого массива, нужно использовать `ForEach`.

Например, если мы хотим отобразить имена всех участников группы, можно написать так:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

Эффект отображения:

![for1](../../Resource/017_for1.png)

Затем добавим одинаковые модификаторы для каждого `Text`:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Теперь каждый `Text` отображается более мелким и более жирным шрифтом.

Эффект отображения:

![](../../Resource/017_for2.png)

Хотя модификаторы делают интерфейс более красивым, они также делают код длиннее и более повторяющимся.

Если имён станет больше или позже нам потребуется единообразно изменить шрифт, цвет и другие стили, придётся снова и снова писать и изменять много одинакового кода. Это неудобно и плохо сказывается на поддержке.

В этом случае можно сначала поместить имена в массив, а затем использовать `ForEach` в SwiftUI, чтобы многократно генерировать view на основе данных массива:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

Этот набор кода более краткий. `ForEach` по порядку читает каждый элемент массива и отображает соответствующее содержимое в интерфейсе.

Когда позже мы захотим добавить новые имена, нам нужно будет только изменить имена в массиве `names`, а не каждый раз повторно добавлять `Text` и сложные модификаторы.

## ForEach

`ForEach` — это структура в SwiftUI, которая используется для многократной генерации view. Её можно понимать как конструкцию SwiftUI, похожую на цикл `for`.

Базовое использование:

```swift
ForEach(массив, id: \.self) { элемент in
    код SwiftUI
}
```

Например, с именами группы из предыдущего примера:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Этот код означает:

`names` — это массив, по которому выполняется цикл, то есть данные, которые будут отображаться.

`id:\.self` означает использование самого элемента массива для различения каждого элемента.

В текущем примере массив хранит строки:

```swift
["Sam", "John", "Wathon", "Bob"]
```

Поэтому SwiftUI будет напрямую использовать сами строки "Sam"、"John"、"Wathon"、"Bob" для различения разного содержимого.

Для простых массивов, которые часто встречаются на начальном этапе, например `String` и `Int`, обычно можно сначала просто писать:

```swift
id: \.self
```

Пока что достаточно запомнить только такую форму записи, без необходимости углубляться в другие правила id.

`item in` означает текущий извлечённый элемент.

Например, на первой итерации `item` — это `"Sam"`, а на второй итерации `item` — это `"John"`.

Следовательно:

```swift
Text(item)
```

по очереди превращается в:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

То есть задача этого кода — поочерёдно брать каждое имя из массива `names` и создавать для каждого имени view `Text`.

### Процесс выполнения

Процесс выполнения можно понять так:

* `ForEach` сначала читает первый элемент `"Sam"` из массива `names`
* `item` временно обозначает `"Sam"`
* `Text(item)` отображает `"Sam"`
* Затем читается следующий элемент `"John"`
* После этого снова выполняется тот же код view

В итоге все имена из массива будут отображены.

Преимущество такого подхода в том, что когда имён становится больше, нам не нужно повторять множество `Text`, достаточно изменить содержимое массива.

### Перебор фотографий

`ForEach` может отображать не только текст, но и изображения, цвета или другие SwiftUI view.

Например, подготовим 4 изображения:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

Перетащите эти четыре изображения в папку Assets в Xcode.

![image](../../Resource/017_for3.png)

Если не использовать `ForEach`, нам придётся вручную писать каждое изображение:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

Эффект отображения:

![image](../../Resource/017_for4.png)

Такой способ работает, но код становится довольно длинным.

Если изображений будет всё больше, нам придётся повторять всё больше и больше кода `Image`.

Если позже мы захотим единообразно изменить размер изображений, нам тоже придётся менять их по одному.

В этом случае можно поместить названия изображений в массив, а затем использовать `ForEach`, чтобы многократно генерировать image view:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Здесь `images` — это массив строк, и каждый элемент массива является именем изображения.

`ForEach(images, id: \.self)` означает, что элементы массива `images` поочерёдно считываются, сам элемент используется для различения каждого пункта, а затем на основе текущего элемента создаётся соответствующее view.

Например, если первым считывается `"430F9BEF"`, тогда:

```swift
Image(item)
```

равносильно:

```swift
Image("430F9BEF")
```

Следующие изображения тоже будут поочерёдно отображаться таким же образом.

![image](../../Resource/017_for4.png)

Здесь мы по-прежнему используем:

```swift
id: \.self
```

Причина та же, что и раньше: элементы массива `images` тоже являются простыми строками, поэтому на начальном этапе можно напрямую использовать сам элемент как способ различения.

Это можно просто запомнить как устойчивую формулу:

**Когда `ForEach` проходит по простому массиву вроде `String` или `Int`, обычно пишется `id: \.self`.**

### Добавление новых изображений

Если позже нам потребуется добавить новые изображения, нужно просто изменить содержимое массива:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

Нам не нужно будет снова повторно писать новый код `Image`.

Если изображений много, это можно использовать вместе с `ScrollView`, иначе, когда содержимое выйдет за пределы экрана, невозможно будет полностью просмотреть его.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### Изменение изображений

Когда нам нужно единообразно изменить стиль изображений, достаточно изменить код внутри `ForEach`.

Например, изменить ширину на `80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

Таким образом все изображения будут изменены одновременно, и не придётся изменять их по одному.

В этом и заключается одна из важных ролей `ForEach`:

**Когда несколько view имеют одинаковую структуру и различаются только содержимым, можно написать один фрагмент кода view, а данные массива будут определять, что именно отображать.**

## Итог

В этом уроке мы изучили цикл `for` и `ForEach`.

Цикл `for` используется для последовательной обработки элементов массива или диапазона и больше ориентирован на обработку данных.

`ForEach` используется в SwiftUI для многократной генерации view на основе содержимого массива и больше ориентирован на отображение интерфейса.

На текущем начальном этапе самая распространённая форма записи `ForEach` такая:

```swift
ForEach(массив, id: \.self) { элемент in
    код SwiftUI
}
```

Здесь `id: \.self` означает использование самого элемента для различения каждого пункта.

Когда содержимое массива состоит из простых типов вроде `String` и `Int`, обычно можно сначала использовать именно такую форму.

В дальнейшем, когда вы будете просматривать веб-страницы или приложения и увидите группу повторяющегося текста, изображений или другого списочного контента, можно вспомнить, что в SwiftUI такой контент обычно сначала помещается в массив, а затем поочерёдно отображается с помощью `ForEach`.
