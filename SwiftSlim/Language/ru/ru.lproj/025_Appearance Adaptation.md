# Адаптация внешнего вида

В этом уроке мы изучим адаптацию внешнего вида для светлого и тёмного режима.

На iPhone, iPad и Mac пользователь может выбрать светлый режим или тёмный режим.

Когда система переключается в тёмный режим, фон, текст и цвета значков во многих App также меняются.

Например, днём интерфейс может иметь светлый фон, а ночью — тёмный фон.

![view](../../../Resource/025_view9.png)

Способность интерфейса автоматически подстраивать отображение под внешний вид системы называется адаптацией внешнего вида.

## Предпросмотр адаптации внешнего вида в Xcode

Перед изучением адаптации внешнего вида сначала посмотрим, как в Xcode предварительно просматривать светлый и тёмный режим.

В нижней части области `Canvas` нажмите кнопку `Device Settings`, чтобы открыть всплывающее окно `Canvas Device Settings`.

![canvas](../../../Resource/025_view.png)

Во всплывающем окне можно увидеть настройку `Color Scheme`.

![canvas1](../../../Resource/025_view1.png)

Здесь:

- `Light Appearance` означает светлый режим.
- `Dark Appearance` означает тёмный режим.

Здесь можно переключать внешний вид предпросмотра и смотреть, как текущий интерфейс отображается в светлом и тёмном режиме.

## Параллельное отображение двух вариантов внешнего вида

Если нужно одновременно посмотреть светлый и тёмный режим, можно использовать функцию `Variants`.

В нижней части области `Canvas` нажмите кнопку `Variants` и выберите `Color Scheme Variants`.

![canvas2](../../../Resource/025_view2.png)

После выбора `Canvas` одновременно покажет предпросмотр светлого и тёмного режима.

![canvas3](../../../Resource/025_view3.png)

Так удобнее сравнивать различия интерфейса в двух вариантах внешнего вида.

Проще говоря:

Если нужно только временно переключить светлый/тёмный режим, можно использовать `Canvas Device Settings`.

Если нужно одновременно увидеть оба варианта внешнего вида, можно использовать `Color Scheme Variants`.

## Представления по умолчанию адаптируются к внешнему виду

В SwiftUI многие системные представления по умолчанию автоматически адаптируются к светлому и тёмному режиму.

Например:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("Светлый")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("какой-то текст")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Результат отображения:

![canvas](../../../Resource/025_view4.png)

Можно увидеть, что в светлом режиме значок и заголовок обычно отображаются тёмным цветом.

В тёмном режиме значок и заголовок автоматически становятся светлыми.

Это происходит потому, что представления SwiftUI, такие как `Text` и `Image`, по умолчанию настраивают цвета в соответствии с внешним видом системы.

То есть, если мы не указываем фиксированный цвет вручную, SwiftUI берёт на себя часть адаптации внешнего вида.

## Фиксированные цвета не меняются автоматически

Важно учитывать: если мы вручную указали фиксированный цвет, он не будет автоматически меняться между светлым и тёмным режимом.

Например:

```swift
Text("какой-то текст")
	.foregroundStyle(Color.gray)
```

Здесь для текста указан `Color.gray`, поэтому он всегда будет отображаться серым цветом.

Ещё пример:

```swift
Text("Заголовок")
    .foregroundStyle(Color.white)
```

Этот код и в светлом, и в тёмном режиме будет отображать белый текст.

Если фон тоже светлый, белый текст может быть плохо виден.

Поэтому при адаптации внешнего вида лучше избегать случайного жёсткого указания таких фиксированных цветов, как `Color.white` и `Color.black`.

Во многих случаях лучше сначала использовать системные семантические стили, например:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

Здесь:

- `.primary` обычно используется для основного содержимого.
- `.secondary` обычно используется для второстепенного содержимого.

Они автоматически настраивают отображение в зависимости от светлого или тёмного режима.

## Отображение разного содержимого в зависимости от внешнего вида

Иногда мы хотим, чтобы менялся не только цвет, но и содержимое в разных вариантах внешнего вида.

Например:

- В светлом режиме показывать обычный значок лампочки.
- В тёмном режиме показывать светящийся значок лампочки.
- В светлом режиме заголовок показывает `Светлый`.
- В тёмном режиме заголовок показывает `Тёмный`.

В этом случае нужно определить, находится ли текущая система в светлом или тёмном режиме.

В SwiftUI можно получить текущий режим внешнего вида через значение окружения `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Полный код:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "Светлый" : "Тёмный"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("какой-то текст")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Результат отображения:

![view](../../../Resource/025_view5.png)

По результату предпросмотра видно, что одно и то же представление в разных режимах внешнего вида отображает не полностью одинаковое содержимое.

В светлом режиме `Image` показывает обычный значок лампочки, а заголовок показывает `Светлый`.

В тёмном режиме `Image` показывает светящийся значок лампочки, а заголовок показывает `Тёмный`.

То есть здесь меняется не только цвет, но и сами имя значка и текст заголовка.

Это потому, что мы не записали значок и заголовок напрямую в `body`, а вычислили разное содержимое на основе `colorScheme`.

В этом коде мы использовали два вычисляемых свойства:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "Светлый" : "Тёмный"
}
```

Когда `colorScheme` равен `.light`, это означает, что сейчас светлый режим.

Когда `colorScheme` равен `.dark`, это означает, что сейчас тёмный режим.

Поэтому мы можем возвращать разные имена значков и тексты заголовков в зависимости от режима внешнего вида.

Это также одно из распространённых применений вычисляемых свойств: вычислять содержимое, которое должно отображать представление, на основе текущего состояния.

## Понимание @Environment

Здесь мы впервые знакомимся с `@Environment`.

`@Environment` можно понимать так: чтение значения из окружения SwiftUI.

Во время работы App система предоставляет много информации окружения, например:

- Текущий язык
- Текущий режим внешнего вида
- Текущее направление макета
- Текущую настройку размера шрифта

Когда нам нужно получить текущий режим внешнего вида, можно прочитать `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Здесь

```swift
\.colorScheme
```

означает чтение значения `colorScheme` из окружения.

```swift
private var colorScheme
```

означает сохранить прочитанное значение в переменной `colorScheme`.

Имя переменной можно выбрать самостоятельно, например также можно написать так:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

Пока первая часть `\.colorScheme` не меняется, это означает чтение текущего режима внешнего вида.

## Два распространённых значения colorScheme

У `colorScheme` часто встречаются два значения:

```swift
.light
.dark
```

Они означают:

- `.light`: светлый режим
- `.dark`: тёмный режим

Например:

```swift
private var titleName: String {
    colorScheme == .light ? "Светлый" : "Тёмный"
}
```

Смысл этого кода:

Если сейчас светлый режим, условие `colorScheme == .light` истинно, поэтому возвращается `"Светлый"`.

Иначе возвращается `"Тёмный"`.

Поэтому в светлом режиме:

```swift
Text(titleName)
```

отображается:

```swift
Светлый
```

В тёмном режиме отображается:

```swift
Тёмный
```

Таким способом мы можем отображать разное содержимое в зависимости от внешнего вида.

## Пример адаптации цвета

Кроме текста и значков, иногда нам нужно самостоятельно обрабатывать адаптацию цвета.

Например:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Текст")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

Результат отображения:

![view](../../../Resource/025_view6.png)

В этом коде текст белый, а фон чёрный.

В светлом режиме чёрный фон достаточно заметен.

Но в тёмном режиме, если фон всего интерфейса тоже чёрный, этот чёрный фон сольётся с системным фоном и будет выглядеть недостаточно ясно.

В этом случае можно переключать цвет текста и цвет фона в зависимости от режима внешнего вида:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("Текст")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

Результат отображения:

![view](../../../Resource/025_view7.png)

Теперь в светлом режиме используется чёрный фон и белый текст.

В тёмном режиме используется белый фон и чёрный текст.

Так при любом варианте внешнего вида текст и фон имеют заметный контраст, и пользователь может ясно видеть содержимое.

## Использование Assets для адаптации цвета

Кроме ручной проверки через `colorScheme`, мы также можем использовать цвета ресурсов `Assets` для адаптации к светлому и тёмному режиму.

Этот способ лучше подходит для управления общими цветами в App.

В проекте Xcode найдите папку ресурсов `Assets`.

![assets](../../../Resource/025_color.png)

Откройте папку `Assets`, щёлкните правой кнопкой по пустой области и выберите `New Color Set`, чтобы создать новый цветовой ресурс.

![assets](../../../Resource/025_color1.png)

Здесь мы создаём цвет и называем его `redText`.

![assets](../../../Resource/025_color2.png)

Для цветового ресурса можно отдельно настроить цвета для светлого и тёмного режима.

![assets](../../../Resource/025_color3.png)

После выбора соответствующей области цвета можно изменить цвет в инспекторе свойств справа.

Нажмите кнопку `Hide or show the Inspectors` в правом верхнем углу, чтобы открыть область инспектора.

Затем выберите `Show the Attributes inspector`, внизу найдите `Show Color Panel` и откройте панель цветов.

![assets](../../../Resource/025_color4.png)

При выборе цвета на панели цветов соответствующая область цвета `redText` изменится синхронно.

Здесь мы задаём для `redText` следующее:

- В светлом режиме отображается красный цвет
- В тёмном режиме отображается зелёный цвет

![assets](../../../Resource/025_color5.png)

Вернувшись к коду SwiftUI, этот цвет можно использовать так:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

Здесь:

```swift
Color("redText")
```

означает чтение цветового ресурса с именем `redText` из `Assets`.

Результат отображения:

![assets](../../../Resource/025_view8.png)

Можно увидеть, что один и тот же `Color("redText")` в светлом и тёмном режиме отображает разные цвета.

Это показывает, что сами цвета `Assets` также поддерживают адаптацию внешнего вида.

## Разница между colorScheme и Assets

И `colorScheme`, и `Assets` могут реализовать адаптацию внешнего вида, но они подходят для разных сценариев.

Если нужно только переключать текст, имя изображения или имя `SF Symbols` в зависимости от светлого/тёмного режима, можно использовать `colorScheme`.

Например:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

Если это часто используемый цвет в App, например цвет темы, фон карточки или цвет текста, лучше использовать цвета `Assets`.

Потому что цвета `Assets` можно переиспользовать в нескольких представлениях, а позже их удобнее изменять.

Например:

```swift
Color("redText")
```

Так нужно изменить цвет только в `Assets`, и все места, где используется этот цвет, изменятся вместе.

## Итоги

В этом уроке мы изучили адаптацию внешнего вида для светлого и тёмного режима.

Сначала мы изучили, как предварительно просматривать разные варианты внешнего вида в `Canvas` Xcode, а также как использовать `Color Scheme Variants` для одновременного просмотра светлого и тёмного режима.

Затем мы узнали, что представления SwiftUI, такие как `Text` и `Image`, по умолчанию автоматически адаптируют цвета к внешнему виду системы.

Но если вручную указать фиксированный цвет, например `Color.white` или `Color.black`, эти цвета не будут автоматически меняться.

Далее мы изучили `@Environment` и `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Через `colorScheme` мы можем определить, является ли текущий режим светлым или тёмным, и отображать разные значки, текст или цвета в зависимости от внешнего вида.

В конце мы изучили цвета `Assets`.

Цвета `Assets` могут отдельно задавать цвета для светлого и тёмного режима, поэтому они подходят для управления общими цветовыми ресурсами в App.

Адаптация внешнего вида позволяет App лучше отображаться и днём, и ночью, а также помогает избежать проблем вроде плохо читаемого текста или недостаточно заметного фона.

В реальной разработке рекомендуется сначала использовать встроенные возможности адаптации SwiftUI и цвета `Assets`.

Когда в разных режимах внешнего вида нужно переключать текст, значки, изображения или обрабатывать небольшое количество специальных эффектов отображения, тогда используйте `colorScheme` для проверки.

## Дополнительные знания: указание размера шрифта

Когда мы раньше изучали `font`, мы использовали некоторые системные стили шрифта:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

Кроме использования системных стилей шрифта вроде `.largeTitle` и `.callout`, можно также вручную указать размер шрифта.

Например:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

Здесь:

```swift
.font(.system(size: 100))
```

означает установить размер шрифта `100 pt`.

Для значков `SF Symbols` параметр `font` также влияет на размер значка.

Поэтому, если нужно сделать значок крупнее, можно указать размер через `.font(.system(size:))`.

Например:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

Важно учитывать, что ручное указание размера шрифта более гибкое, но для обычного текста не рекомендуется массово использовать фиксированные размеры.

Для обычного текста лучше сначала использовать системные стили шрифта, такие как `.title`, `.headline`, `.body`, `.caption`.

Так можно лучше адаптироваться к разным устройствам и пользовательским настройкам размера шрифта.
