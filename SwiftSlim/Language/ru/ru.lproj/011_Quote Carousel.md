# Карусель цитат

В этом уроке мы реализуем функцию «карусели цитат» и подробнее изучим базовые знания Swift, такие как массивы (Array) и условные операторы (if-else).

Мы покажем, как сохранять несколько цитат и с помощью взаимодействия с кнопкой реализовать их циклическое переключение.

![alt text](../../RESOURCE/011_word.png)

## Отображение цитаты

Сначала нам нужно отобразить одну цитату в SwiftUI.

Самый простой способ — использовать представление Text:

```swift
Text("Slow progress is still progress.")
```

Этот код может отображать только одну фиксированную цитату. Если мы хотим показывать несколько цитат и реализовать переключение, нам нужно сохранить эти цитаты.

Однако обычная строковая переменная может хранить только одну цитату:

```swift id="c2k7d1"
let sayings = "Slow progress is still progress."
```

Если нужно сохранить несколько цитат, придётся объявлять отдельную переменную для каждой из них:

```swift id="n5v8p4"
let saying1 = "Slow progress is still progress."
let saying2 = "Difficult roads lead to beautiful destinations."
let saying3 = "Awake minds are the strongest."
let saying4 = "Long-term thinking wins over short-term tricks."
let saying5 = "Master one thing, and do it well."
```

Но в реальной разработке такой способ не только громоздкий, но и неудобный: поскольку каждая переменная независима, мы не сможем гибко реализовать карусельное переключение.

Чтобы удобно управлять несколькими цитатами, нам нужна структура данных, которая позволит хранить их вместе. Это и есть массив (Array).

После использования массива приведённый выше код можно записать так:

```swift id="i9u3w6"
let sayings = [
    "Slow progress is still progress.",
    "Difficult roads lead to beautiful destinations.",
    "Awake minds are the strongest.",
    "Long-term thinking wins over short-term tricks.",
    "Master one thing, and do it well."
    ]
```

**Подсказка: в соглашениях по программированию имена переменных массивов, которые хранят несколько элементов, обычно используют во множественном числе, например sayings, чтобы подчеркнуть их природу как коллекции.**

## Массив

В Swift массив — это коллекция упорядоченных элементов, которая обозначается квадратными скобками [].

```swift id="w4a7m2"
[]
```

Внутри массива может содержаться несколько элементов одного и того же типа, а сами элементы разделяются английской запятой ,.

Например:

```swift id="b7q1f8"
[101, 102, 103, 104, 105]
```

Массив можно упрощённо представить как поезд:

![Array](../../RESOURCE/011_array1.png)

Весь поезд представляет сам объект массива, а каждый вагон расположен в определённом порядке.

### Индекс и доступ к элементам

Поскольку массив упорядочен, система может находить конкретный элемент массива по его позиции. Такой механизм называется индексом (Index).

В Swift, как и в большинстве языков программирования, индексация массива начинается с 0, а не с 1. Это означает, что первый элемент массива имеет индекс 0, второй — 1 и так далее.

![Array](../../RESOURCE/011_array2.png)

Если нужно получить доступ к определённому элементу массива, достаточно после имени массива поставить квадратные скобки и указать внутри индекс нужного элемента.

Например:

```swift id="r8n5k0"
sayings[0]   // 101
sayings[1]   // 102
```

Если попытаться обратиться к индексу, который выходит за допустимые пределы массива, возникнет проблема «выхода за границы индекса» (Index Out of Range). Поэтому при работе с массивом нужно быть внимательным и проверять, что индекс находится в допустимом диапазоне.

**Выход за границы индекса**

Например, если массив содержит только 5 элементов, допустимый диапазон индексов — от 0 до 4. Если мы попытаемся обратиться к sayings[5], программа не сможет найти соответствующий «вагон», из-за чего возникнет ошибка «выход за границы индекса», и приложение завершится с ошибкой.

![Array](../../RESOURCE/011_array3.png)

### Операции с массивом

Массив поддерживает не только статическое определение, но и операции добавления, удаления, изменения элементов, а также получение длины массива.

Подсказка: если массив нужно изменять, его необходимо объявлять через var, а не через let.

**1. Добавление элемента**

Можно использовать метод append, чтобы добавить элемент в конец массива:

```swift id="h2m9z7"
var num = [0, 1, 2]
num.append(3)   // [0, 1, 2, 3]
```

**2. Удаление элемента**

С помощью метода remove(at:) можно удалить указанный элемент массива:

```swift id="q6d4t3"
var num = [0, 1, 2]
num.remove(at: 0)   // [1, 2]
```

**3. Изменение элемента**

Можно напрямую изменить элемент массива по его индексу:

```swift id="p3x8v1"
var num = [0, 1, 2]
num[1] = 4   // [0, 4, 2]
```

**4. Подсчёт длины массива**

С помощью свойства count можно получить количество элементов массива:

```swift id="u1k7y5"
var num = [0, 1, 2]
num.count   // 3
```

### Использование массива для отображения цитат

Чтобы отобразить несколько цитат, мы можем сохранить их в массив и затем получать и показывать нужную цитату по индексу.

Сначала в ContentView создадим массив sayings для хранения цитат, а затем в представлении Text будем читать и отображать соответствующую цитату по индексу:

```swift id="z9c2r6"
struct ContentView: View {
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[0])
        }
        .padding()
    }
}
```

Здесь sayings[0] означает первую цитату в массиве.

Если нужно показать другую цитату, достаточно изменить значение индекса в квадратных скобках:

```swift id="m4t8n0"
Text(sayings[0]) // "Slow progress is still progress."
Text(sayings[1]) // "Difficult roads lead to beautiful destinations."
```

### Определение индекса цитаты

Чтобы реализовать динамическое переключение цитат, мы не можем «жёстко прописать» индекс прямо внутри Text.

Нам нужно создать переменную, которая отдельно будет хранить текущее значение индекса.

В SwiftUI мы можем использовать @State для объявления изменяемого индекса:

```swift id="e7v3p9"
@State private var index = 0
```

SwiftUI будет отслеживать переменную, обёрнутую в @State. Когда значение index изменится, SwiftUI заново отрисует представление и покажет соответствующую цитату.

Затем мы будем динамически получать цитату из массива через sayings[index]:

```swift id="k5n1w4"
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    
    var body: some View {
        VStack {
            Text(sayings[index])
        }
        .padding()
    }
}
```

Когда значение index изменяется, Text показывает другую цитату.

### Использование кнопки для управления индексом

Чтобы управлять переключением цитат, мы можем использовать Button для изменения значения index. При каждом нажатии кнопки index автоматически увеличивается на 1:

```swift id="t2f9q8"
Button("Next") {
    index += 1
}
```

Когда мы нажимаем кнопку, значение index изменяется с 0 на 1, это вызывает обновление представления, и Text(sayings[index]) считывает следующую цитату.

Однако здесь есть потенциальная проблема: если нажимать кнопку много раз подряд, index может увеличиться до значения, выходящего за пределы массива, что вызовет ошибку выхода за границы индекса. Например, когда index станет равен 5, а допустимый диапазон индексов массива — от 0 до 4, программа завершится с ошибкой.

Чтобы предотвратить выход за границы массива, нужно ввести условное управление и убедиться, что index не превышает допустимый диапазон. Для этого можно использовать оператор if-else и проверить, меньше ли index длины массива.

## Условное управление: оператор if-else

Оператор if-else — одно из самых распространённых средств условного ветвления в Swift. Он используется для проверки, выполняется ли условие, и в зависимости от результата выполняет разные блоки кода.

Базовая структура:

```swift id="j8m2c7"
if condition {
    // Код, который выполняется, когда condition равно true
} else {
    // Код, который выполняется, когда condition равно false
}
```

В операторе if условие condition представляет собой булево значение (тип Bool), которое может быть true или false. Если условие равно true, выполняется код из блока if, иначе выполняется блок else.

Например:

```swift id="f6w4r1"
let age = 25
if age > 18 {
    print("Big Boy")
} else {
    print("Little Body")
}
```

В этом примере значение age равно 25. Оператор if проверяет, выполняется ли условие age > 18. Поскольку условие истинно, выводится "Big Boy".

Если код для else не нужен, его можно опустить:

```swift id="y3n8u5"
if condition {
    // Код, который выполняется, когда condition равно true
}
```

### Использование условия для ограничения индекса

Чтобы предотвратить выход индекса за пределы массива, можно использовать if и убедиться, что index не превышает допустимый диапазон:

```swift id="s7q1x9"
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    }
}
```

Логика такова: значение sayings.count равно 5, потому что всего есть 5 цитат, значит sayings.count - 1 равно 4 — это и есть последний допустимый индекс массива.

Когда index меньше 4, нажатие кнопки и увеличение на 1 безопасны. Как только index достигает 4, условие перестаёт выполняться, и нажатие кнопки больше ничего не делает.

Теперь код уже реализует переключение цитат:

```swift id="g4v6b2"
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack {
            Text(sayings[index])
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                }
            }
        }
        .padding()
    }
}
```

### Циклические цитаты

Если мы хотим, чтобы после показа последней цитаты при следующем нажатии снова отображалась первая цитата, то есть чтобы цитаты переключались по кругу, это можно реализовать через блок else:

```swift id="v1p5n8"
Button("Next") {
    if index < sayings.count - 1 {
        index += 1
    } else {
        index = 0
    }
}
```

Когда кнопка нажимается и index доходит до последнего элемента массива, значение index сбрасывается в 0, и цитаты начинают показываться по кругу.

## Улучшение интерфейса цитат

Теперь логика карусели цитат уже реализована, но мы можем дополнительно улучшить интерфейс, чтобы он выглядел красивее.

Полный код:

```swift id="d8u2m4"
struct ContentView: View {
    @State private var index = 0
    let sayings = [
        "Slow progress is still progress.",
        "Difficult roads lead to beautiful destinations.",
        "Awake minds are the strongest.",
        "Long-term thinking wins over short-term tricks.",
        "Master one thing, and do it well."
    ]
    var body: some View {
        VStack(spacing: 10) {
            Text(sayings[index])
                .padding(20)
                .background(.white)
                .cornerRadius(10)
                .opacity(0.8)
            Button("Next") {
                if index < sayings.count - 1 {
                    index += 1
                } else {
                    index = 0
                }
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .background(
            Image("001")
        )
    }
}
```

В этом примере для Text добавлены белый полупрозрачный фон и скруглённые углы, для кнопки Button используется стиль .borderedProminent, а для VStack задано фоновое изображение.

Дополнение к знанию: при использовании модификатора background() для фонового изображения его поведение по умолчанию заключается в том, чтобы по возможности заполнить область компоновки текущего представления. Во многих случаях оно естественным образом растягивается и покрывает безопасную область (Safe Area).

## Дополнительные знания — множественные условия: оператор if-else if-else

В реальной разработке часто требуется обрабатывать несколько условий. Например, в какой-либо игре при количестве очков 1 срабатывает событие A, при 2 — событие B, при 3 — событие C и так далее.

Когда сценарий содержит больше двух ветвей, нам нужен оператор if-else if-else для множественной проверки условий.

Базовый синтаксис:

```swift id="a5r9t3"
if conditionA {
    // Код, который выполняется, когда conditionA равно true
} else if conditionB {
    // Код, который выполняется, когда conditionB равно true
} else if conditionC {
    // Код, который выполняется, когда conditionC равно true
} else {
    // Код, который выполняется, когда ни одно условие не выполнено
}
```

В этом случае программа последовательно проверяет условия и выполняет код первого условия, которое оказалось истинным. Если ни одно условие не подошло, выполняется код после else.

В карусели цитат мы тоже можем использовать if-else if-else для множественной проверки:

```swift id="l2x7m1"
Button("Next") {
    if index == 0 {
        index += 1
    } else if index == 1 {
        index += 1
    } else if index == 2 {
        index += 1
    } else if index == 3 {
        index += 1
    } else if index == 4 {
        index = 0
    } else {
        index = 0
    }
}
```

Когда index равен 0, 1, 2 или 3, каждое нажатие кнопки выполняет index += 1. Когда index равен 4, то есть последнему элементу, значение index сбрасывается в 0, реализуя циклическое переключение.

Ветвь else используется как страховка на случай некорректного значения, например если index был ошибочно изменён.

Следует обратить внимание, что здесь используется ==, которое означает проверку «равно ли». Если в операторе if index равен какому-либо числу, выражение возвращает true и выполняется соответствующий блок кода. Если не равен, проверка переходит к следующему if.

Такое множественное ветвление полезно тем, что позволяет выполнять разный код в зависимости от разных условий.
