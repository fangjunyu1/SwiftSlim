# Переменные и константы

В этом уроке мы в основном изучим переменные и константы в Swift, а также познакомимся с некоторыми распространёнными типами данных и базовыми операторами.

Это самые фундаментальные знания в программировании, и они также являются важной основой для дальнейшего изучения SwiftUI.

## Переменные и константы в повседневной жизни

Мы можем понять переменные и константы через примеры из повседневной жизни.

Переменные в жизни:

- Содержание каждой программы по телевизору разное
- Погода каждый день разная
- Часы меняются каждую секунду

Общая особенность этих вещей в том, что они изменяются.

Если бы на телевизоре всегда была только одна картинка, погода всегда была солнечной, а стрелки часов никогда не двигались, тогда эти вещи были бы константами.

Одна вещь может изменяться, а другая — нет.

## Понимание переменных и констант

При разработке App обычно нужно, чтобы пользователь вводил или сохранял какую-либо информацию.

Например:

- Имя аккаунта
- Дата рождения
- Контактные данные
- Адрес

Эта информация будет сохранена и отображена.

Например, пользователь вводит в App имя:

```

FangJunyu

```

Нам нужно сохранить это имя, чтобы затем показать его в App.

Процесс сохранения можно условно представить так: мы кладём вещь в ящик.

Когда мы сохраняем имя, это похоже на то, как будто мы кладём имя в ящик.

Сохраняемых вещей может быть много, и ящиков тоже может быть много. Чтобы понимать, что лежит в каждом ящике, нам нужно дать ящику имя.

Например:

```

name

```

В этом примере name — это имя, а FangJunyu — сохранённая информация.

![Var](../../RESOURCE/007_var.png)

**В Swift, если нужно сохранить данные, необходимо объявить переменную (var) или константу (let).**

Поскольку имя обычно можно изменять, нам следует использовать переменную.

```swift
var name = "FangJunyu"
```

Здесь объявлена переменная с именем name, её тип — String, а значение — "FangJunyu".

### Разница между переменной и константой

Переменная объявляется с помощью var:

```swift
var
```

Константа объявляется с помощью let:

```swift
let
```

Например:

```swift
var name = "FangJunyu"
let id = 123456
```

Если значение объявлено как переменная, его можно изменять; если как константа, изменить его уже нельзя.

Поэтому главное различие между переменной и константой заключается в том, можно ли менять значение.

### Изменение константы

Если в коде попытаться изменить значение константы:

```swift
let name = "Sam"
name = "Bob"
```

Swift сообщит, что выполнить присваивание нельзя, потому что это константа.

```
Cannot assign to property: 'name' is a 'let' constant
```

Такой механизм помогает разработчику избежать случайного изменения важных данных.

### Отображение в SwiftUI

Откройте файл ContentView.swift и объявите переменную внутри View:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

Здесь объявлена переменная с именем name, и через Text она отображается на экране.

Если мы изменим содержимое переменной:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

Текст, отображаемый в Text, тоже изменится.

Таким способом мы можем использовать переменную для управления содержимым, которое отображается в интерфейсе, не изменяя каждый раз текст напрямую внутри Text.

Если эта информация нужна в другом месте, мы также можем передать туда переменную — как будто передаём содержимое «ящика» в другое место для использования.

### Расположение переменных и констант в SwiftUI

В SwiftUI переменные и константы обычно пишутся вне body:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

Конечно, переменные и константы можно писать и внутри body:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

Однако между этими двумя вариантами есть важное различие:

* Если переменные и константы написаны внутри body, то при каждом перерасчёте представления эта переменная будет создаваться заново.
* Если они написаны вне body, то переменная будет существовать как свойство структуры представления, и код будет выглядеть более понятно.

Поэтому в реальной разработке переменные и константы обычно пишут вне body.

## Типы данных

Переменные могут хранить много разных видов данных. Эти виды называются типами данных.

Например, ранее мы использовали Text для отображения текста:

```swift
Text("Hello, World")
```

Здесь "Hello, World" относится к типу строки (String).

Строка используется для представления текстового содержимого, и в Swift она должна быть заключена в двойные кавычки "".

Например:

```swift
var hello = "Hello, World"
```

Помимо строк, в Swift существует много других типов данных.

На начальном этапе чаще всего используются четыре типа:

* String
* Int
* Double
* Bool

Этих четырёх типов достаточно для выполнения большей части базовой обработки данных.

**String**

String представляет текстовое содержимое, например:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

Строки обычно используются для представления: имени пользователя, заголовка, текстового содержимого.

**Int**

Int представляет целые числа, то есть числа без дробной части.

Например:

```swift
var age = 26
var count = 100
```

Целые числа обычно используются для представления: возраста, количества, счётчиков.

**Double**

Double представляет числа с дробной частью.

Например:

```swift
var weight = 74.5
var height = 185.0
```

В Swift числа с дробной частью по умолчанию обычно относятся к типу Double.

Есть ещё похожий тип под названием Float, но диапазон значений у него меньше, поэтому в реальной разработке чаще используется Double.

**Bool**

Тип Bool используется для представления двух состояний: true и false.

Например:

```swift
var isShowAlert = true
var isLogin = false
```

Тип Bool часто используется в условных проверках. Например, для определения, нужно ли показывать окно-подсказку.

Для ситуаций, где есть только два возможных результата, очень удобно использовать Bool.

### Отображение в SwiftUI

Когда мы отображаем значение типа String, можно напрямую использовать Text:

```swift
Text(name)
```

Но такие типы, как Int и Double, не являются строками, поэтому их нельзя напрямую использовать как часть текстового содержимого.

Если нужно отобразить такие данные в Text, можно использовать интерполяцию строк (String Interpolation):

```swift
\()
```

Интерполяция строк должна записываться внутри строки и использовать \() для обёртывания переменной или константы.

Например:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

В коде выше "" обозначает строку, а \() используется для вставки переменной или константы внутрь строки.

Например:

```swift
"DoubleNum: \(num * 2)"
```

Отображаемое содержимое будет таким:

```swift
DoubleNum: 2
```

Таким способом мы можем вставлять переменные или константы в строку, а затем отображать их через Text.

Подсказка: \() можно использовать только внутри строки "".

## Операторы

При объявлении переменной или константы мы можем увидеть такую запись:

```swift
var num = 1
```

Здесь = называется оператором присваивания.

Его задача — присвоить значение справа переменной слева.

В этом примере значение 1 присваивается переменной num.

Кроме оператора присваивания, есть ещё несколько распространённых арифметических операторов:

* `+`
* `-`
* `*`
* `/`

Когда мы выполняем вычисления с числами, например с данными типа Int или Double, используются именно эти операторы.

Например:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

После выполнения вычисления результат сохраняется в переменной.

## Итог

Переменные, константы и операторы — это самые базовые понятия в программировании.

С помощью переменных и констант мы можем сохранять разные данные в программе; с помощью типов данных можно определить вид этих данных; с помощью операторов можно выполнять вычисления и обработку данных.

Эти знания подобны базовым инструментам мира программирования. Освоив их, можно заложить прочную основу для дальнейшего изучения Swift и SwiftUI.
