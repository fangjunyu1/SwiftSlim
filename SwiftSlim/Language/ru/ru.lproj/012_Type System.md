# Система типов

На предыдущих уроках мы уже изучили переменные и знаем, что переменные могут хранить данные разных типов.

Например:

```swift
var hello = "Hello, World"
```

Здесь `hello` хранит текст, поэтому его тип — `String`.

Далее мы системно изучим понятие типа (`Type`), а также явное объявление типа и вывод типа, чтобы глубже понять сущность переменных.

## Что такое тип

В Swift каждое значение имеет чётко определённый тип. Тип определяет, чем является это значение и что с ним можно делать.

Например:

* `42` — это `Int` (целое число)
* `"Hello"` — это `String` (строка)
* `true` — это `Bool` (булево значение)

Они относятся к разным типам и используются по-разному.

## Распространённые базовые типы в Swift

На начальном этапе чаще всего используются следующие типы:

* `String`: строка (текст)
* `Int`: целое число
* `Double`: число с плавающей точкой (с десятичной частью, более высокая точность)
* `Bool`: булево значение (`true` / `false`)

Кроме того, есть `Float` (число с плавающей точкой), но его точность ниже, чем у `Double`, поэтому он используется реже; а также `Character` (один символ), например `"A"`, `"!"` и другие одиночные символы.

### Объявление типа (явный тип)

В Swift можно вручную указать тип при объявлении переменной:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

После имени переменной добавляется `: Тип`, чтобы объявить её тип.

Тип массива записывается как `: [Тип]`:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

В этом фрагменте кода массив `scores` может хранить только элементы типа `Int`, а `tags` — только `String`.

Явное объявление типа делает намерение кода более понятным и в некоторых случаях помогает избежать ошибок вывода типа.

## Вывод типа

Во многих случаях тип можно не указывать явно:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"     // String
var isLogin = false  // Bool
var scores = [90, 85, 78]   // [Int]
var tags = ["Swift", "iOS"] // [String]
```

Это возможно потому, что компилятор Swift автоматически выводит тип по **начальному значению**.

## Явное объявление и вывод типа

В предыдущих уроках мы не вводили явное объявление типа, например:

```swift
var age: Int = 18
```

А вместо этого сначала использовали вывод типа:

```swift
var age = 18
```

В этом примере обе записи эквивалентны: `age` в обоих случаях будет определён как тип `Int`. При этом запись с выводом типа короче.

Мы не делали акцент на явном объявлении в самом начале, потому что вывод типа позволяет сократить количество лишней информации в коде и тем самым уменьшить сложность понимания для новичков.

## Зачем нужен тип

Swift — это язык со строгой типизацией (`Strongly Typed Language`).

Это означает: как только тип переменной определён, его нельзя произвольно изменить.

Например:

```swift
var age: Int = 18

age = 20      // Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

Первое значение `20` имеет тип `Int`, поэтому его можно присвоить `age`; второе значение `"20"` имеет тип `String`, который не совпадает с `Int`, поэтому возникнет ошибка.

`nums` — это массив типа `[Int]`, он может хранить только целые числа и не может содержать строку.

Типы ограничивают использование данных и позволяют обнаруживать несоответствия типов на этапе компиляции, тем самым уменьшая количество ошибок, вызванных путаницей типов, и повышая стабильность и сопровождаемость кода.

## Когда обязательно нужно явно объявлять тип

Хотя Swift может автоматически выводить тип, в некоторых случаях компилятор не может его определить или делает это неправильно. Тогда тип нужно объявить вручную.

**1. Пустой массив**

При создании пустого массива нужно обязательно явно указать тип:

```swift
var nums: [Int] = []
```

Если не указать тип:

```swift
var nums = []   // Error, Empty collection literal requires an explicit type
```

Компилятор не может определить тип элементов пустого массива, поэтому возникает ошибка компиляции.

**2. Предотвращение неправильного вывода типа**

```swift
var value = 10   // Int
```

В этом примере, хотя `10` можно рассматривать и как число с плавающей точкой (`Double`), из-за отсутствия десятичной части компилятор по умолчанию выведет тип `Int`.

Если нужно, чтобы `value` был числом с плавающей точкой, тип нужно указать явно:

```swift
var value: Double = 10   // Double
```

Если результат вывода типа не соответствует ожиданиям, следует использовать явное объявление типа или изменить форму литерала, чтобы гарантировать правильный тип.

**3. Пользовательские типы**

Позже мы будем изучать пользовательские типы. При их использовании тоже часто требуется явное объявление:

```swift
var info: GameInfo
var users: [User] = []
```

Причина та же, что и с пустым массивом: компилятор не может автоматически вывести тип из пустого значения.

## Преобразование типов

В Swift разные типы не преобразуются автоматически — это нужно делать вручную.

Например, сложение двух чисел разных типов:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b   // Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

В этом фрагменте `a` имеет тип `Int`, а `b` — `Double`. Поскольку типы различаются, выполнить операцию напрямую нельзя.

Перед вычислением нужно сначала преобразовать одно из значений к тому же типу:

```swift
var result = Double(a) + b
```

Эта запись означает: преобразовать `a` типа `Int` в `Double`, а затем сложить его с `b`; результат присваивается переменной `result`.

Важно понимать, что преобразование типа не меняет тип исходной переменной. Например, `Double(a)` не изменяет тип `a` с `Int` на `Double`, а лишь создаёт новое значение типа `Double`, которое участвует в вычислении.

Другие типы можно преобразовывать аналогичным образом: используется форма `ЦелевойТип()`, в которую передаётся значение для преобразования. Например:

```swift
Int()、Double()、String()
```

Пример:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

Нужно учитывать, что не все преобразования ведут себя одинаково. Например, при преобразовании `Double → Int` дробная часть просто отбрасывается, а не округляется.

## Псевдонимы типов

В Swift также можно дать существующему типу другое имя, чтобы код стал понятнее по смыслу:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

Здесь `UserID` по сути всё равно остаётся `Int`; сам тип не изменился, изменилось только более осмысленное название.

В больших проектах с более сложными типами псевдонимы делают код легче для чтения.

## Пример с типами

Далее на простом примере покажем, какую роль типы играют в реальной разработке.

### Создание переменной

Сначала создадим переменную `age`:

```swift
var age = 20
```

Поскольку нам нужно отображать и изменять эту переменную в SwiftUI, её следует объявить через `@State`:

```swift
@State private var age = 20
```

`private`, как упоминалось ранее, означает, что эта переменная может использоваться только внутри текущего представления; позже мы изучим это подробнее.

### Отображение данных в SwiftUI

Покажем переменную `age` в SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

Теперь в интерфейсе будет отображаться:

```swift
age: 20
```

### Изменение данных

Теперь добавим две кнопки для изменения `age`:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### Результат выполнения

При нажатии `changeAge1` переменной `age` будет присвоено значение `18`:

```swift
age = 18
```

Поскольку `18` имеет тип `Int`, совпадающий с типом `age`, присваивание выполнится успешно.

`changeAge2` вызовет ошибку:

```swift
age = "18"
```

Сообщение об ошибке:

```swift
Cannot assign value of type 'String' to type 'Int'
```

Это происходит потому, что `"18"` имеет тип `String` (строки заключаются в `""`), а `age` имеет тип `Int`; типы не совпадают, поэтому присваивание невозможно.

### Почему возникает ошибка?

Ключевая строка здесь:

```swift
@State private var age = 20   // Int
```

Хотя тип не указан явно, из-за того что начальное значение `20` имеет тип `Int`, компилятор автоматически выводит для `age` тип `Int`.

Как только тип определён, присвоить значение другого типа уже нельзя.

Причём кнопка `changeAge2` не приводит к сбою **во время выполнения** — ошибка возникает уже **на этапе компиляции**, и код не сможет собраться.

В этом и заключается одно из преимуществ строгой типизации Swift: проблемы с типами обнаруживаются уже в момент написания кода.

### Правильный вариант

После удаления ошибочного кода:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

При нажатии кнопки `changeAge1` интерфейс обновится до:

```swift
age: 18
```

### Явное объявление типа (необязательно)

Также можно явно указать тип при определении:

```swift
@State private var age: Int = 20
```

Такая запись делает тип более очевидным, но в простых ситуациях это не обязательно.

## Итог

На этом уроке мы рассмотрели систему типов Swift и её роль в реальной разработке.

В Swift каждое значение имеет чёткий тип. Тип определяет смысл данных и то, в каких операциях они могут участвовать.

Swift — язык со строгой типизацией: как только тип переменной определён, присвоить ей значение другого типа уже нельзя. Это ограничение позволяет обнаруживать ошибки на этапе компиляции, уменьшать количество проблем во время выполнения и повышать безопасность кода.

Тип переменной можно вывести автоматически из начального значения, а можно указать явно. В некоторых случаях (например, пустой массив, неверный вывод типа, пользовательские типы) компилятор не может корректно вывести тип, и тогда явное объявление обязательно.

Между разными типами нет автоматического преобразования — для этого нужно явно использовать `Int()`、`Double()`、`String()` и подобные конструкции.

Также типы поддерживают псевдонимы. Особенно в более сложных проектах они помогают сделать код читабельнее.

Тип — это не просто метка данных, но и механизм ограничений, который обеспечивает безопасное, понятное и контролируемое использование данных.

## Дополнительные знания — опциональный тип

При преобразовании типов некоторые операции могут завершиться неудачей.

Например, преобразование `String` в `Double`:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

Здесь `Double(input)` возвращает не `Double`, а:

```swift
Double?
```

Это и есть опциональный тип (`Optional`).

### Почему возвращается опциональный тип?

Потому что содержимое строки заранее неизвестно: `"3.14"` можно преобразовать в `Double`, а `"Hello"` — нельзя.

То есть такая операция может как завершиться успешно, так и неудачно.

Поэтому Swift использует опциональный тип, чтобы обозначить эту неопределённость.

### Что такое опциональный тип?

Опциональный тип означает: значение может существовать, а может и отсутствовать.

Например:

```swift
Double?
```

Этот тип означает: если значение есть, при печати будет показано `Optional(...)`; если значения нет, вернётся `nil`.

Важно понимать, что `nil` означает «значения нет», а не `0` и не пустую строку.

### Пример

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Вывод:

```swift
pi:Optional(3.14)
```

Это означает, что строка успешно преобразовалась в `Double`, получилось `3.14`, но так как тип — `Double?`, значение обёрнуто в `Optional`.

Если преобразование не удалось:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

Вывод:

```swift
pi: nil
```

Поскольку `"Hello"` нельзя преобразовать в `Double`, преобразование завершается неудачей и возвращает `nil`.

### Что такое `Optional(...)`?

Когда мы напрямую печатаем опциональный тип:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift показывает его «отладочное представление»:

```swift
Optional(3.14)
```

Это не часть самих данных, а подсказка о том, что текущее значение является **опциональным типом** и внутри него есть значение.

### Распаковка опционального типа

В реальной разработке нам обычно нужно извлечь настоящее значение из опционального типа. Этот процесс называется распаковкой (`Unwrapping`).

Один из распространённых способов — использовать `??`, чтобы задать значение по умолчанию:

```swift
??
```

Это называется: оператор объединения с `nil` (`Nil-Coalescing Operator`).

Например:

```swift
name ?? "FangJunyu"
```

Эта запись означает: если у `name` есть значение, вернуть его; если там `nil`, использовать значение по умолчанию `"FangJunyu"`.

Важно, чтобы тип значения по умолчанию совпадал с типом опционального значения.

### Использование `??` для задания значения по умолчанию

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")   // 30
```

Вывод:

```swift
pi: 30
```

При преобразовании `input` в `Double` преобразование проходит успешно и выводится полученное значение. Если бы преобразование не удалось, вывелось бы значение по умолчанию, указанное после `??`.

Если преобразование не удалось:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

Вывод:

```swift
pi: 10
```

Здесь при попытке преобразовать `input` в `Double` операция завершается неудачей и возвращает `nil`, а `??` подставляет значение по умолчанию.

Опциональный тип используется для обозначения ситуации, когда «значение может быть, а может и не быть».

В операциях, которые могут завершиться неудачей, например при преобразовании типов, Swift возвращает опциональный тип, чтобы обеспечить безопасность программы.

Когда нам нужно использовать внутреннее значение, можно воспользоваться `??` и задать значение по умолчанию, чтобы даже при появлении `nil` программа всё равно получила разумный результат.
