# Счётчик

В предыдущих двух уроках мы изучили переменные, константы, кнопки и функции.

В этом уроке мы объединим эти знания и создадим простое приложение-счётчик.

В этом примере мы изучим, как сохранять и изменять переменные, а также как использовать @State для управления переменными.

## Реализация счётчика

Счётчик обычно используется для записи количества, которое нужно постоянно увеличивать. Например, количество прыжков на скакалке, кругов при беге и другие подобные сценарии.

Теперь давайте создадим простой счётчик: отобразим число и кнопку, при нажатии на которую число будет увеличиваться.

![Num](../../RESOURCE/009_num.png)

### Отображение числа

Сначала мы используем Text для отображения числа.

```swift
Text("0")
````

Но если число должно изменяться, нельзя просто жёстко прописать "0", нужно использовать переменную для хранения числа.

```swift
var num = 0
```

Затем отобразим переменную в Text:

```swift
Text("\(num)")
```

Здесь используется интерполяция строк \(), чтобы преобразовать число в строку и отобразить его.

### Увеличение числа кнопкой

Далее добавим кнопку.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

Когда кнопка нажата, мы хотим увеличить число.

Поэтому можно изменить переменную внутри кнопки:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

Этот код означает: при нажатии кнопки вычисляется num + 1, и результат присваивается переменной num.

### Более краткая запись

Swift предоставляет более краткую форму записи:

```swift
num += 1
```

Она эквивалентна:

```swift
num = num + 1
```

Такая запись называется «составной оператор присваивания».

В Swift часто используются следующие составные операторы присваивания:

```
+=   присваивание с сложением
-=   присваивание с вычитанием
*=   присваивание с умножением
/=   присваивание с делением
%=   присваивание с остатком от деления
```

Например:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### Полный код

Теперь мы можем написать простой счётчик:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

Но в таком виде код не будет работать корректно.

### Возникновение ошибки

Xcode покажет сообщение:

```
Cannot assign to property: 'self' is immutable
```

Это означает, что свойство нельзя изменить, потому что представление является неизменяемым.

Почему так происходит?

## Принцип работы SwiftUI

В SwiftUI: **интерфейс управляется данными**, это означает, что когда данные изменяются, интерфейс автоматически обновляется.

Но важно обратить внимание: только когда изменяется состояние (например, @State, @Binding), SwiftUI отслеживает эти изменения и обновляет представление.

Если использовать обычную переменную, SwiftUI прочитает её значение только один раз при создании представления. Даже если потом переменная изменится, интерфейс не обновится.

Например:

```swift
var num = 0   // 0
num = 1   // 0
```

В этом примере num — обычная переменная. Хотя её значение изменилось, SwiftUI не обновит представление, и интерфейс по-прежнему будет показывать исходное 0.

![Num](../../RESOURCE/009_state.png)

Поэтому только когда данные помечены как состояние, например с использованием обёрток свойств @State, @Binding и других, SwiftUI автоматически отслеживает их изменения и обновляет представление.

## Обёртка свойства @State

В SwiftUI, если нужно изменять переменную и обновлять представление, необходимо использовать @State.

```swift
@State var num = 0
```

@State заставляет SwiftUI отслеживать эту переменную.

Когда переменная изменяется, SwiftUI заново вычисляет представление и обновляет его.

Полный код:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

Теперь, когда кнопка нажата:

```swift
num += 1
```

SwiftUI обнаружит, что num изменился, и автоматически обновит представление.

![Num](../../RESOURCE/009_state1.png)

## Правила использования @State

В реальной разработке @State обычно следует двум правилам.

### 1. @State обычно используется с private

Переменная @State обычно используется только внутри текущего типа, например внутри текущей структуры View, поэтому её обычно пишут так:

```swift
@State private var num = 0
```

Это позволяет избежать прямого доступа или изменения этой переменной из других представлений.

Пока мы ещё не изучали private, можно просто понимать это так:

**Содержимое, помеченное private, можно использовать только в текущем представлении, а внешние представления не могут его использовать.**

### 2. @State используется только для состояния текущего представления

@State подходит для данных, которыми управляет сам текущий View.

Например, содержимое текстового поля или состояние переключателя.

```swift
@State private var isOn = false
@State private var text = ""
```

Если данные нужно разделять между несколькими представлениями, тогда нужно использовать другие типы состояния, например @Binding или @Observable.

Эти темы будут изучаться в следующих уроках.

### Кнопка уменьшения

Счётчик можно не только увеличивать, но и уменьшать.

Мы можем добавить кнопку -.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

Затем используем HStack, чтобы расположить кнопки по горизонтали:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

Эффект:

![Num](../../RESOURCE/009_num1.png)

При нажатии кнопки + число увеличивается на 1; при нажатии кнопки - число уменьшается на 1.

### Кнопка сброса

Мы также можем добавить кнопку сброса, чтобы вернуть число к 0.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

Когда кнопка сброса нажата, num получает значение 0.

Эффект:

![Num](../../RESOURCE/009_num2.png)

При нажатии кнопки 0 число становится равным 0.

### Пользовательская кнопка

Сейчас наши кнопки могут отображать только текст:

```swift
Button("+") {

}
```

Кнопка в таком виде может показывать только текстовое содержимое, например + или -.

В реальной разработке мы обычно хотим, чтобы внешний вид кнопки был богаче, например с иконкой или другим представлением.

SwiftUI позволяет настраивать содержимое кнопки. Для этого можно использовать другую форму записи кнопки:

```swift
Button(action: {

}, label: {

})
```

В этой записи action обозначает код, который выполняется при нажатии кнопки, а label определяет представление, отображаемое на кнопке.

Например, мы можем сделать так, чтобы кнопка показывала иконку SF Symbols:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

Теперь кнопка будет отображать не текст, а иконку.

### Улучшение интерфейса счётчика

Чтобы интерфейс счётчика выглядел понятнее, мы также можем изменить стиль отображения числа. Например, увеличить размер шрифта:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

Теперь давайте тоже заменим содержимое кнопок на иконки и немного увеличим их размер:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

Аналогично можно заменить кнопку уменьшения и кнопку сброса на иконки:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

Теперь кнопки отображают иконки SF Symbols, и интерфейс выглядит более наглядно.

## Итог

В этом уроке, реализуя простой счётчик, мы комплексно потренировались использовать переменные, Text и Button. Мы использовали переменную для хранения числа и управляли увеличением и уменьшением числа с помощью кнопок, а также изучили составные операторы присваивания, например num += 1 и num -= 1.

В процессе реализации мы поняли, как работает SwiftUI: интерфейс управляется данными. Когда данные изменяются, интерфейс обновляется. Поэтому для хранения изменяемого состояния нужно использовать @State. Когда переменная @State изменяется, SwiftUI автоматически обновляет представление.

В конце мы также немного улучшили интерфейс: изменили стиль шрифта у Text и использовали Button(action:label:) для настройки содержимого кнопок, чтобы они могли отображать иконки. Это сделало интерфейс счётчика более понятным и аккуратным.

Теперь мы уже познакомились с базовым способом построения представлений в SwiftUI. В SwiftUI большая часть интерфейса управляется переменными. Когда переменные изменяются, SwiftUI автоматически обновляет представление. Поэтому при разработке приложений на SwiftUI нам обычно нужно сначала продумать, какие данные нужно сохранять, а затем уже строить интерфейс и логику взаимодействия на основе этих данных.

## Полный код

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
