## Стандартный механизм компоновки в SwiftUI

При предварительном просмотре ContentView мы заметим, что иконка и текст отображаются по центру, а не выстраиваются от верхнего края.

![Swift](../../RESOURCE/003_view.png)

По умолчанию выравнивание контейнера Stack — это .center, поэтому дочерние представления обычно отображаются по центру.

### Alignment — выравнивание

Очевидно, что выравнивание по центру — это только один из вариантов. Если мы хотим выровнять содержимое по левому или правому краю, нужно использовать alignment для управления выравниванием представления.

```swift
alignment
```

В SwiftUI выравнивание обычно встречается в двух сценариях:

**1. Параметр выравнивания контейнера Stack**

Например, если мы хотим выровнять иконку и текст в ContentView по левому краю:

```swift
VStack(alignment: .leading) {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
```

![Swift](../../RESOURCE/003_alignment.png)

alignment у VStack управляет выравниванием по горизонтали.

Варианты выравнивания:

```swift
VStack(alignment: .leading)
VStack(alignment: .center)
VStack(alignment: .trailing)
```

![Swift](../../RESOURCE/003_alignment1.png)

HStack выполняет горизонтальную компоновку, поэтому alignment управляет выравниванием по вертикали:

```swift
HStack(alignment: .top)
HStack(alignment: .center)
HStack(alignment: .bottom)
```

![Swift](../../RESOURCE/003_alignment2.png)

ZStack выполняет наложение слоёв, и alignment может управлять выравниванием как по горизонтали, так и по вертикали:

```swift
ZStack(alignment: .leading)
ZStack(alignment: .trailing)
ZStack(alignment: .top)
ZStack(alignment: .topLeading)
ZStack(alignment: .topTrailing)
ZStack(alignment: .center)
ZStack(alignment: .bottom)
ZStack(alignment: .bottomLeading)
ZStack(alignment: .bottomTrailing)
```

Если alignment не указан явно, VStack, HStack и ZStack по умолчанию используют .center.

**2. alignment внутри frame**

```swift
Text("Hello")
    .frame(width: 200, alignment: .leading)
```

Когда размер, заданный через frame, больше собственного размера представления, alignment определяет положение представления внутри frame. Подробное использование frame мы разберём позже, а здесь пока просто познакомимся с этим понятием.

### Spacer и механизм распределения пространства

alignment позволяет выстраивать представления по горизонтали или вертикали. Но когда нам нужно расположить текст и изображение по разным краям, одного выравнивания уже недостаточно.

Например, если мы хотим сделать верхнюю панель, похожую на верхнюю часть сайта [NHK](https://www3.nhk.or.jp/nhkworld/zh/news/): слева расположен логотип NHK, а справа — иконка меню сайта.

![Swift](../../RESOURCE/003_alignment3.png)

Если использовать только alignment, и логотип NHK, и иконка меню окажутся с одной стороны. Разнести два элемента по левому и правому краю таким способом нельзя, поэтому нужен Spacer для распределения оставшегося пространства.

Spacer — это гибкое представление для компоновки, которое автоматически заполняет оставшееся свободное пространство.

Использование:

```swift
Spacer()
```

Например:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Spacer()
    Text("Hello, world!")
}
```

Когда мы добавляем Spacer между Image и Text, он заполняет оставшееся пространство и раздвигает Image и Text к верхнему и нижнему краю.

![Swift](../../RESOURCE/003_view1.png)

Если Spacer несколько:

```swift
VStack {
    Text("Hello, world!")
    Spacer()
    Text("FangJunyu")
    Spacer()
    Text("fangjunyu.com")
    Spacer()
}
```

оставшееся пространство будет распределено между Spacer поровну.

![Swift](../../RESOURCE/003_spacer.png)

## Отображение Image и управление размером

Представление Image в основном используется для отображения изображений. Иконки SF Symbols, которые мы изучали на прошлом уроке, — это только один из вариантов использования Image.

Использование:

```swift
Image("imageName")
```

Внутри двойных кавычек указывается имя изображения без расширения файла.

### Отображение изображения

Сначала подготовим изображение.

![Swift](../../RESOURCE/003_img.jpg)

В Xcode выберите папку ресурсов Assets и перетащите изображение в Assets.

![Swift](../../RESOURCE/003_img1.png)

В ContentView используйте Image для отображения изображения:

```swift
struct ContentView: View {
    var body: some View {
        Image("004_img")
    }
}
```

![Swift](../../RESOURCE/003_img2.png)

Обратите внимание: Image в SwiftUI не поддерживает воспроизведение GIF-анимации, может отображаться только статический кадр.

### Управление размером изображения

В SwiftUI по умолчанию Image отображается в исходном размере. Если нужно изменить отображаемый размер изображения, сначала следует использовать resizable, чтобы сделать содержимое масштабируемым, а затем через frame задать размеры компоновки.

```swift
Image("image")
    .resizable()
    .frame(width: 200, height: 200)
```

### Модификатор resizable

Модификатор resizable позволяет изображению участвовать в масштабировании внутри макета, а не использовать всегда фиксированный исходный размер.

```swift
.resizable()
```

Только после добавления resizable() frame действительно сможет изменить размер отображаемого изображения.

Если не использовать resizable:

```swift
Image("image")
    .frame(width: 50, height: 50)
```

то frame только выделит пространство для изображения, но само изображение не изменит размер.

### Модификатор frame

frame(width:height) используется для задания ширины и высоты представления.

Базовое использование:

```swift
.frame(width: 10,height: 10)
```

Например, зададим изображению прямоугольную форму с шириной 300 и высотой 100:

```swift
Image("image")
    .resizable()
    .frame(width: 300, height: 100)
```

![Swift](../../RESOURCE/003_img3.png)

Можно задавать только ширину или только высоту:

```swift
.frame(width: 200)
.frame(height: 100)
```

Комбинация resizable + frame позволяет гибко управлять размером изображения в интерфейсе, сохраняя возможность масштабирования.

### Масштабирование: scaledToFit и scaledToFill

Когда мы задаём через frame ширину и высоту с другим соотношением сторон, изображение может растягиваться и искажаться.

Если мы хотим сохранить пропорции изображения и при этом вписать его в доступное пространство, можно использовать scaledToFit или scaledToFill.

**scaledToFit**

scaledToFit сохраняет исходное соотношение сторон изображения и масштабирует его так, чтобы оно полностью помещалось в доступное пространство, без обрезки:

```swift
.scaledToFit()
```

или

```swift
.aspectRatio(contentMode: .fit)
```

Этот способ подходит для случаев, когда изображение должно отображаться целиком и без искажений.

Если для всех изображений задавать одинаковую ширину и высоту, неизбежно появятся случаи растяжения.

Например:

```swift
Image("image")
    .resizable()
    .frame(width: 300, height: 200)
```

Без настройки режима масштабирования изображение не сможет сохранить исходные пропорции.

![Swift](../../RESOURCE/003_img4.png)

Если добавить scaledToFit, изображение сохранит исходные пропорции.

```swift
Image("image")
    .resizable()
    .scaledToFit()
    .frame(width: 300, height: 200)
```

![Swift](../../RESOURCE/003_img5.png)

**scaledToFill**

scaledToFill тоже сохраняет пропорции изображения, но заполняет всё доступное пространство. Если пропорции не совпадают, лишняя часть изображения будет обрезана:

```swift
.scaledToFill()
```

или

```swift
.aspectRatio(contentMode: .fill)
```

Этот способ подходит для сценариев, где изображение должно полностью покрывать область, например для фоновых изображений или banner. Особенно часто он используется именно для фона.

## Текст

В SwiftUI Text используется для отображения текста.

Базовое использование:

```swift
Text("FangJunyu")
```

Мы уже изучали Text на прошлом уроке. В этом уроке мы продолжим и разберём, как управлять размером и толщиной шрифта, чтобы текст выглядел выразительнее в интерфейсе.

### Размер шрифта

Для управления размером текста используется модификатор font:

```swift
.font(.title)
```

Например:

```swift
Text("FangJunyu")
    .font(.largeTitle)
Text("FangJunyu")
    .font(.callout)
```

![Swift](../../RESOURCE/003_txt.png)

Часто используемые размеры шрифта (от большего к меньшему):

```
.largeTitle
.title
.title2
.title3
.headline
.body
.subheadline
.callout
.footnote
.caption
.caption2
```

### Толщина шрифта

Если нужно сделать текст более жирным, можно использовать модификатор fontWeight:

```swift
.fontWeight(.bold)
```

Например:

```swift
Text("FangJunyu")
    .font(.title)
    .fontWeight(.bold)
Text("FangJunyu")
    .font(.title)
```

![Swift](../../RESOURCE/003_txt1.png)

Часто используемые значения толщины шрифта (от тонкого к жирному):

```
.ultraLight
.thin
.light
.regular
.medium
.semibold
.bold
.heavy
.black
```

font управляет размером шрифта, fontWeight — толщиной шрифта. Их можно комбинировать, чтобы сделать текст более выразительным.

## Итоги и практика

К этому моменту мы уже изучили стандартную компоновку SwiftUI, Spacer, Image и Text. Этих знаний уже достаточно, чтобы создавать простые представления.

Например: [Google](https://www.google.com/)

![Swift](../../RESOURCE/003_google.png)

Интерфейс Google очень чистый и простой. Он включает изображения и текст, и мы можем попробовать разобрать его структуру с точки зрения SwiftUI:

1. Вся структура делится на три части: логотип Google, строка поиска и подсказочный текст. Для этого можно использовать вертикальную компоновку VStack.
2. Логотип Google — это изображение, его можно показать через Image.
3. Строка поиска содержит поле ввода и иконку. Если пока не учитывать само поле ввода, иконку поиска можно отобразить через Image.
4. Подсказочный текст отображается с помощью Text. Для горизонтального расположения текста можно использовать HStack, а цвет шрифта задать через foregroundStyle.

Через такую практику можно собрать несколько простых представлений и глубже понять использование Image, Text и их модификаторов.
