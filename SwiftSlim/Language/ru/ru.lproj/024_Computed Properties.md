# Вычисляемые свойства

В этом уроке мы в основном изучим вычисляемые свойства.

Вычисляемое свойство используется для вычисления нового результата на основе уже существующих данных. Оно может вычислять числовые значения, а также содержимое, которое нужно показать в представлении SwiftUI.

Например:

```swift
let a = 10
let b = 20
let c = a + b
```

Здесь `c` означает результат сложения `a` и `b`.

В обычном коде такой способ записи встречается очень часто.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

Подсказка: внутри события нажатия `Button` можно выполнять обычный Swift-код.

Но если написать похожий код напрямую в свойствах представления SwiftUI, возникнет проблема.

Например:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

Этот код вызовет ошибку.

На первый взгляд кажется, что `a` и `b` уже написаны перед `c`, поэтому значение `c` вроде бы можно вычислить.

Но при объявлении свойств в структуре такой способ записи нельзя использовать напрямую.

## Почему c нельзя напрямую вычислить как a + b?

Потому что `a`, `b` и `c` — это не временные константы внутри события нажатия кнопки, а свойства представления `ContentView`.

Внутри события нажатия кнопки этот код может выполняться нормально:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

Потому что после нажатия кнопки код выполняется по порядку. Сначала создается `a`, затем создается `b`, а в конце с помощью `a + b` вычисляется `c`.

Но при объявлении свойств внутри представления ситуация другая:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

Здесь `a`, `b` и `c` — это все свойства представления `ContentView`.

Когда создается представление `ContentView`, Swift сначала должен подготовить эти свойства. Чтобы процесс создания был безопасным, Swift не разрешает значению по умолчанию одного хранимого свойства экземпляра напрямую читать другие свойства того же экземпляра.

Поэтому эта строка вызовет ошибку:

```swift
let c = a + b
```

Проще говоря: **при объявлении свойств внутри представления нельзя напрямую использовать одно обычное свойство для вычисления другого обычного свойства.**

Такое свойство, которое напрямую хранит значение, называется «хранимым свойством». Для простоты понимания его временно можно считать обычным свойством.

Например:

```swift
let a = 10
```

`a` хранит `10`.

```swift
let b = 20
```

`b` хранит `20`.

Но:

```swift
let c = a + b
```

Здесь `c` — это не фиксированное значение, написанное напрямую. Оно должно быть вычислено через `a + b`.

В такой ситуации, когда «результат получается на основе уже существующих данных», больше подходит вычисляемое свойство.

Можно изменить код так:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

Здесь `c` — это вычисляемое свойство.

```swift
var c: Int {
    return a + b
}
```

Его смысл такой: **когда нужно использовать `c`, тогда считываются `a` и `b`, а результат `a + b` возвращается наружу.**

Например:

```swift
Text("c: \(c)")
```

Когда `Text` показывает `c`, только тогда запускается вычисление `c`.

## Что такое вычисляемое свойство?

Вычисляемое свойство выглядит как переменная, но само не хранит данные.

Например:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

Здесь `c` — это вычисляемое свойство.

Оно не хранит фиксированное значение, как обычное свойство. Каждый раз, когда `c` читается, код внутри `{}` выполняется заново и возвращает вычисленный результат.

Это можно понять так:

```swift
var c: Int {
    return a + b
}
```

Когда нужно `c`, вычисляется `a + b`.

Поэтому вычисляемое свойство подходит для такой ситуации: **результат не нужно хранить отдельно, потому что его можно вычислить на основе уже существующих данных.**

## Основная запись

Вычисляемое свойство обычно состоит из трех частей:

```swift
var c: Int {
    return a + b
}
```

### 1. Объявление с помощью `var`

```swift
var c
```

Вычисляемое свойство обязательно объявляется с помощью `var`; использовать `let` нельзя.

Потому что вычисляемое свойство — это не фиксированное сохраненное значение, а результат, который динамически вычисляется каждый раз при чтении.

### 2. Указание возвращаемого типа

```swift
var c: Int
```

Вычисляемое свойство должно указывать возвращаемый тип.

Здесь `c` в итоге возвращает целое число, поэтому тип — `Int`.

### 3. Запись логики вычисления внутри {}

```swift
{
    return a + b
}
```

Внутри `{}` пишется логика вычисления. Здесь возвращается `a + b`.

## Ключевое слово return

Вычисляемое свойство должно возвращать результат.

Например:

```swift
var c: Int {
    return a + b
}
```

Здесь `return` означает: вернуть результат вычисления `a + b`.

Если внутри вычисляемого свойства есть только одно выражение, которое напрямую создает результат, `return` можно опустить:

```swift
var c: Int {
    a + b
}
```

Но если внутри вычисляемого свойства несколько строк кода, нужно использовать `return`, чтобы явно указать возвращаемый результат.

Например:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Здесь процесс вычисления разделен на два шага.

Первый шаг — сначала вычислить общую цену:

```swift
let total = count * price
```

Второй шаг — соединить общую цену с текстом и вернуть результат:

```swift
return "Total：\(total) $"
```

Если убрать `return`:

```swift
var totalPriceText: String {
    let total = count * price
    "Total：\(total) $"
}
```

Этот код вызовет ошибку.

Причина: **в этом вычисляемом свойстве уже есть несколько строк кода, поэтому Swift не может автоматически определить, какая строка является итоговым возвращаемым результатом.**

Поэтому, если в вычисляемом свойстве есть только одно выражение, создающее результат, `return` можно опустить.

```swift
var c: Int {
    a + b
}
```

Если в вычисляемом свойстве несколько строк кода, лучше явно писать `return`.

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

## Разница между вычисляемым свойством и обычным свойством

Обычное свойство хранит данные.

```swift
var c = 30
```

Здесь `c` хранит конкретное значение: `30`.

Вычисляемое свойство не хранит данные.

```swift
var c: Int {
    a + b
}
```

Здесь `c` не хранит `30`. Оно только предоставляет способ вычисления.

Когда `c` читается, Swift выполняет:

```swift
a + b
```

Затем возвращает вычисленный результат.

Поэтому вычисляемое свойство подходит для ситуаций, когда результат нужно вычислить на основе других данных.

## body тоже является вычисляемым свойством

После изучения вычисляемых свойств мы можем заново понять самый часто встречающийся код в SwiftUI:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Здесь `body` тоже является вычисляемым свойством.

Оно объявлено с помощью `var`:

```swift
var body
```

У него есть возвращаемый тип:

```swift
some View
```

Внутри `{}` возвращается содержимое SwiftUI-представления:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

Поэтому `body` можно понять так: когда SwiftUI нужно показать это представление, он читает `body` и на основе содержимого, которое возвращает `body`, создает интерфейс.

Если записать `return`, это можно понять так:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Но в SwiftUI мы обычно опускаем `return` и пишем напрямую:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Когда данные `@State` изменяются, SwiftUI заново читает `body` и обновляет интерфейс на основе новых данных.

Например:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("count: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

После нажатия кнопки `count` изменяется.

```swift
count += 1
```

После изменения `count` SwiftUI заново вычисляет `body`, поэтому текст в интерфейсе тоже обновляется.

```swift
Text("count: \(count)")
```

Это также объясняет, почему в SwiftUI интерфейс может автоматически обновляться после изменения данных.

### Не рекомендуется писать сложные вычисления внутри body

Поскольку `body` является вычисляемым свойством, оно может читаться и пересчитываться много раз.

Например:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("num: \(num)")
            Text("name: \(name)")
        }
    }
}
```

Когда `name` изменяется, SwiftUI заново вычисляет `body`.

В этот момент код внутри `body` тоже выполняется заново, и `num` также создается заново.

```swift
let num = 10
```

В этом примере `num` очень простой, поэтому влияние небольшое.

Но если внутри `body` выполнять сложные вычисления, например фильтрацию большого количества данных, сортировку, обработку изображений и так далее, это может повлиять на плавность интерфейса.

Поэтому в SwiftUI `body` должно в основном отвечать за описание структуры интерфейса.

Простые временные данные можно писать внутри `body`.

Сложные вычисления лучше выносить за пределы `body`: в вычисляемое свойство, метод или отдельную часть обработки данных.

## Пример: количество и общая цена

Теперь с помощью простого примера разберем вычисляемое свойство.

Предположим, цена одной моркови — 2 $. Пользователь может нажимать кнопки, чтобы изменить количество покупаемых морковей, а в интерфейсе нужно показать общую цену.

Способ вычисления общей цены:

```text
Количество * Цена за единицу
```

Если хранить общую цену в обычной переменной, это будет неудобно.

Потому что каждый раз при изменении количества нужно вручную обновлять общую цену.

Лучший способ — использовать вычисляемое свойство:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Результат отображения:

![view](../../Resource/024_view.png)

В этом примере:

```swift
@State private var count = 1
```

`count` означает количество морковей. При нажатии кнопки `count` изменяется.

```swift
private let price = 2
```

`price` означает цену одной моркови. Здесь это фиксированное значение, поэтому используется `let`.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` означает общую цену.

Ее не нужно хранить отдельно, потому что общую цену всегда можно вычислить через `count * price`.

Когда `count` равно `1`:

```swift
totalPrice = 1 * 2
```

Отображаемый результат:

```swift
totalPrice: 2 $
```

После нажатия кнопки `+` значение `count` становится `2`.

В этот момент при повторном чтении `totalPrice` оно вычисляется заново:

```swift
totalPrice = 2 * 2
```

Отображаемый результат:

```swift
totalPrice: 4 $
```

В этом и заключается роль вычисляемого свойства: динамически вычислять новый результат на основе уже существующих данных.

## Вычисляемое свойство можно использовать для проверки условий

Вычисляемое свойство может не только вычислять числа, но и возвращать результат проверки.

Например, когда мы хотим, чтобы минимальное количество было равно 1.

Когда количество уже равно 1, кнопка `-` не должна продолжать уменьшать значение.

Можно добавить вычисляемое свойство:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Полный код:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Результат отображения:

![view](../../Resource/024_view1.png)

Здесь:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Этот код означает, можно ли сейчас продолжать уменьшать количество.

Когда `count` больше `1`:

```swift
canDecrease == true
```

Это означает, что уменьшать можно.

Когда `count` равно `1`:

```swift
canDecrease == false
```

Это означает, что дальше уменьшать нельзя.

### Проверка условия внутри кнопки

В кнопке используется:

```swift
if canDecrease {
    count -= 1
}
```

Только когда `canDecrease` равно `true`, можно уменьшить `count`.

### Управление цветом переднего плана представления

Также можно использовать `canDecrease`, чтобы управлять цветом переднего плана кнопки:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Здесь используется тернарный оператор:

```swift
canDecrease ? Color.primary : Color.gray
```

Смысл этого кода: если `canDecrease` равно `true`, цвет переднего плана будет `Color.primary`; если `canDecrease` равно `false`, цвет переднего плана будет `Color.gray`.

`Color.primary` — это системный семантический цвет, предоставляемый SwiftUI. Он означает основной цвет текста в текущем интерфейсе.

В светлом режиме `Color.primary` обычно близок к черному; в темном режиме обычно близок к белому.

Поэтому преимущество использования `Color.primary` в том, что он автоматически адаптируется к светлому и темному режимам.

### Управление отключенным состоянием представления

`disabled` используется для управления тем, находится ли представление в отключенном состоянии:

```swift
.disabled(!canDecrease)
```

Когда `disabled` равно `false`, по представлению можно нажать.

Когда `disabled` равно `true`, представление находится в отключенном состоянии, и по нему нельзя нажать.

Здесь `canDecrease` используется как условие, поэтому код легче понять.

Увидев `canDecrease`, можно сразу понять, что оно означает «можно ли сейчас продолжать уменьшать».

### Дополнительное пояснение: почему проверка выполняется два раза?

В кнопке `-`:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Здесь используется и `.disabled(!canDecrease)`, и `if canDecrease` внутри кнопки.

`.disabled(!canDecrease)` отвечает за отключение кнопки на уровне интерфейса, чтобы пользователь не мог нажать на нее.

`if canDecrease` еще раз проверяет условие перед выполнением кода. Только когда уменьшать можно, выполняется `count -= 1`.

Это двойная защита. В реальной разработке, если кнопка уже отключена, внутреннюю проверку можно опустить. Но в учебном примере ее можно оставить, чтобы роль `canDecrease` была понятнее.

## Итоги

В этом уроке мы в основном изучили вычисляемые свойства.

Вычисляемое свойство не хранит значение напрямую. Когда оно читается, оно вычисляет результат на основе уже существующих данных.

Например:

```swift
var c: Int {
    a + b
}
```

Здесь `c` не нужно хранить отдельно, потому что его можно вычислить через `a + b`.

Вычисляемое свойство обязательно объявляется с помощью `var`, и для него нужно указывать возвращаемый тип.

```swift
var canDecrease: Bool {
    count > 1
}
```

Вычисляемое свойство может возвращать не только числовое значение, но и результат проверки условия, текстовое содержимое и даже содержимое SwiftUI-представления.

В этом уроке мы также изучили `return`.

`return` означает возвращение результата:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Если в вычисляемом свойстве есть только одно выражение, которое напрямую создает результат, `return` можно опустить.

```swift
var totalPrice: Int {
    count * price
}
```

Кроме того, мы также узнали о `Color.primary` и `disabled`.

`Color.primary` — это системный семантический цвет SwiftUI. Он автоматически адаптирует отображение к светлому и темному режимам.

```swift
.foregroundStyle(Color.primary)
```

`disabled` используется для управления тем, отключено ли представление.

```swift
.disabled(true)
```

```swift
.disabled(false)
```

Это означает, что представление доступно, и по нему можно нажать.

Поэтому вычисляемые свойства очень часто используются в SwiftUI. Они помогают более ясно организовать результаты вычислений, условия и отображаемое содержимое.
