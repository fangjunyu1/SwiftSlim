# Список фильмов

В этом уроке мы создадим список фильмов.

![movie](../../Resource/021_movie.png)

В списке будут отображаться постер, название, режиссер и рейтинг каждого фильма.

В этом примере мы изучим очень важную тему: структуру `struct`. Она помогает объединить несколько сведений об одном фильме в единое целое. Кроме того, мы познакомимся с `UUID()`, `ForEach`, разделительной линией `Divider`, а также с тем, как использовать пользовательские объекты для управления данными.

Эти знания очень часто используются в дальнейшей разработке на SwiftUI. После этого урока вы не только сможете сделать список фильмов, но и начнете понимать, «как отображать группу данных в виде интерфейса».

## Один фильм

Сначала мы можем начать с интерфейса одного фильма.

![movie](../../Resource/021_movie1.png)

Макет этого интерфейса в основном состоит из двух частей: слева постер фильма, справа описание фильма.

### Постер фильма

Слева отображается постер фильма, и для показа изображения мы можем использовать `Image`.

Например:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

Этот код означает, что отображается изображение с именем `"The Shawshank Redemption"`.

Здесь нужно обратить внимание на то, что имя изображения должно действительно существовать в папке ресурсов `Assets`, иначе интерфейс не сможет правильно его показать.

![movie](../../Resource/021_movie2.png)

Назначение этих модификаторов следующее:

- `resizable()` означает, что размер изображения можно изменять.
- `scaledToFit()` означает, что при масштабировании сохраняются исходные пропорции, чтобы изображение не растягивалось и не деформировалось.
- `frame(height: 180)` означает, что высота изображения устанавливается на 180.
- `cornerRadius(10)` означает, что изображению задаются скругленные углы радиусом 10.

Таким образом, мы получаем постер фильма подходящего размера и со скругленными углами.

![movie](../../Resource/021_movie3.png)

### Описание фильма

Справа отображается описание фильма, включая название фильма, режиссера и рейтинг.

![movie](../../Resource/021_movie4.png)

Для отображения этой части мы можем использовать `Text`:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

В этом коде `.font` обозначает размер шрифта, а `.fontWeight` обозначает толщину шрифта.

Чтобы сделать расстояние между «названием фильма», «информацией о режиссере» и «информацией о рейтинге» более понятным, здесь на самом внешнем уровне используется `VStack`, и задается:

```swift
spacing: 10
```

Это означает, что расстояние между каждой группой содержимого внутри этого `VStack` равно 10.

Режиссер и рейтинг дополнительно оборачиваются каждый в свой `VStack`, потому что оба относятся к структуре вида «заголовок + содержимое». Поэтому иерархия интерфейса становится более понятной, а в дальнейшем нам будет удобнее настраивать стиль.

Кроме того, `VStack` по умолчанию выравнивается по центру. Чтобы весь текст был выровнен по левому краю, мы задаем:

```swift
alignment: .leading
```

Так часть с описанием фильма выглядит аккуратнее.

### Разделительная линия

Теперь, хотя содержимое описания фильма уже разнесено с помощью `spacing`, границы между разными частями все еще недостаточно заметны.

В этот момент мы можем добавить разделительную линию:

```swift
Divider()
```

Например:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Таким образом, часть с описанием фильма получит более заметный эффект разделения.

![movie](../../Resource/021_movie4.png)

`Divider` — это очень простой, но очень часто используемый view, задача которого — разделять разное содержимое.

В `VStack` `Divider()` отображается как горизонтальная линия.

В `HStack` `Divider()` отображается как вертикальная линия.

Кроме того, стиль `Divider` можно изменять с помощью `.frame`, `.background`, `.padding` и других способов.

Например:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

Это создаст синюю разделительную линию толщиной 2 с внутренними отступами слева и справа.

![divider](../../Resource/021_divider.png)

### Полный интерфейс

Наконец, мы можем использовать `HStack`, чтобы расположить постер фильма и описание фильма слева и справа.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

Здесь `HStack(spacing: 20)` означает, что между левой и правой частями сохраняется расстояние 20.

Важно отметить, что `Divider()`, как и упомянутые ранее `TextField` и `Slider`, по умолчанию старается занять как можно больше доступного пространства.

Поэтому для внешнего `VStack` в правой части описания мы задаем фиксированную ширину:

```swift
.frame(width: 200)
```

Так ширина разделительной линии и текстовой области будет одинаковой, и общий вид станет аккуратнее.

На этом этапе мы завершили интерфейс отображения «одного фильма».

![movie](../../Resource/021_movie1.png)

## Сохранение фильмов в массиве

Если мы хотим отображать несколько фильмов один за другим в одном и том же стиле, это означает, что для каждого фильма нужно будет писать похожий код.

Например:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

Хотя такой код может дать нужный результат, его будет становиться все больше, и поддерживать его будет очень неудобно.

Например, если нужно изменить расстояние между всеми фильмами с `20` на `15`, или изменить ширину справа с `200` на
`220`, придется вручную менять каждый повторяющийся фрагмент кода.

Очевидно, что это неэффективный способ.

Ранее мы уже изучали массивы, а также то, как использовать `ForEach` для повторного отображения view на основе массива.

Например:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Такой способ очень хорошо подходит для ситуаций, когда есть «только один вид данных», например группа названий изображений.

Однако фильм — это не только одно значение. Он как минимум включает:

1. Постер фильма
2. Название фильма
3. Режиссера
4. Рейтинг

То есть один фильм на самом деле представляет собой группу связанных данных, а не одну отдельную строку.

Если использовать только массивы, то можно лишь хранить эту информацию раздельно:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

При отображении пришлось бы полагаться на одинаковый индекс, чтобы сопоставить их один к одному:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

Проблема этого способа записи в том, что стоимость поддержки очень высока.

Поскольку данные одного фильма разбросаны по нескольким массивам, достаточно, чтобы в одном массиве было на один элемент больше, на один меньше или порядок отличался, и результат отображения будет ошибочным.

Особенно при добавлении или удалении фильмов приходится одновременно менять несколько массивов, и очень легко что-то упустить.

Как же объединить такие данные, как «название фильма, режиссер, рейтинг», в одно целое?

Для этого нужно использовать `struct`.

## Определение структуры `struct`

В Swift `struct` — это «структура».

Вы можете понимать ее как «пользовательский тип данных», который объединяет несколько связанных полей в единое целое.

Базовое использование:

```swift
struct StructName {
	let name: String
}
```

В этом коде `struct` — это ключевое слово, которое показывает, что мы определяем структуру. `StructName` — это имя структуры.

Содержимое в фигурных скобках — это поля, которые содержит данная структура, и для каждого поля обязательно нужно четко указать имя и тип.

Обычно имя `struct` начинается с заглавной буквы, например `Movie`, `Student`, `UserInfo`. Это распространенная конвенция именования в Swift.

Можно просто представить `struct` как пустую коробку, в которой каждое поле — это как зарезервированное пустое место.

Пока эти места не заполнены содержимым, это просто пустая коробка. Только когда все поля заполнены подходящими значениями, она становится полноценной подарочной коробкой.

### Экземпляр `struct`

До этого мы только определили саму структуру, то есть подготовили форму коробки.

Нам все еще нужно заполнить ее конкретным содержимым, чтобы получить настоящий «экземпляр», который можно использовать.

При создании экземпляра обычно после имени структуры ставят `()`:

```swift
StructName(...)
```

В круглых скобках указывается содержимое полей, которое требуется этой структуре.

Например:

```swift
StructName(name: "Fang Junyu")
```

Этот код означает: создать новый экземпляр по формату структуры `StructName` и присвоить полю `name` значение `"Fang Junyu"`.

Когда мы заполняем все нужные поля, это равносильно тому, что мы наполняем коробку.

В этот момент мы получаем полноценный экземпляр структуры.

### Доступ к свойствам `struct`

После создания экземпляра мы можем использовать «точечный синтаксис» для доступа к его внутренним свойствам.

Базовая запись:

```swift
instance.propertyName
```

Например:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

Здесь `st` — это экземпляр структуры, а `st.name` означает чтение значения `name` у этого экземпляра.

Вы можете понимать такой способ чтения так: когда мы пишем `st.name`, это похоже на чтение содержимого, которое лежит в определенном месте коробки.

Такой способ записи будет очень часто встречаться в последующих интерфейсах SwiftUI.

Например, если у нас есть структура студента, содержащая имя, возраст и класс, то при отображении в интерфейсе можно отдельно читать:

```swift
student.name
student.age
student.className
```

Преимущество такого подхода в том, что данные становятся более понятными, и ими удобнее управлять.

### Где размещать `struct`

Для начинающих обычно удобно писать структуру вне `ContentView`.

Например:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

Так структура будет понятнее и ее будет легче читать.

Вы заметите, что сам `ContentView` на самом деле тоже является `struct`.

### Структура `Movie`

Имея приведенную выше основу, мы можем определить структуру фильма:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

Это структура с именем `Movie`, которая состоит из трех полей: `name`, `director` и `rating`, обозначающих соответственно название фильма, режиссера и рейтинг.

Таким образом, информация об одном фильме больше не должна храниться в нескольких массивах по отдельности, а может быть напрямую объединена в единое целое.

Например, мы можем создать такой экземпляр фильма:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

Этот экземпляр представляет «один полный набор данных фильма».

### Хранение структур в массиве

Ранее мы говорили, что массив может хранить только один и тот же тип.

Теперь у нас уже есть структура `Movie`, а значит массив может хранить несколько `Movie`.

```swift
let lists: [Movie] = []
```

Например:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

В этом массиве `lists` хранятся не строки, а несколько экземпляров `Movie`.

То есть каждый элемент массива — это один полноценный фильм.

Таким образом, когда нам нужно отобразить список фильмов, мы можем с помощью `ForEach` последовательно читать каждый фильм.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

Здесь `movie in` означает, что при каждом проходе цикла из массива берется один фильм и временно получает имя `movie`.

Затем мы можем использовать:

```swift
movie.name
movie.director
movie.rating
```

чтобы по отдельности прочитать название, режиссера и рейтинг этого фильма и отобразить их в интерфейсе.

## Ошибка `ForEach`

На этом этапе мы уже завершили базовую запись массива фильмов и `ForEach`.

Но если напрямую запустить следующий код:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

появится ошибка:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

Смысл этой ошибки в том, что когда вы пишете `id: \.self`, SwiftUI нужно использовать каждый элемент массива как «уникальный идентификатор», чтобы различать их.

Такой способ подходит для простых типов вроде `String` и `Int`, потому что их самих по себе относительно легко различить.

Но `Movie` — это определенная нами пользовательская структура, и `ForEach` не знает, как использовать ее саму в качестве уникального идентификатора, поэтому и возникает ошибка.

Чтобы решить эту проблему, самый распространенный способ — заставить `Movie` соответствовать протоколу `Identifiable`.

## Протокол

В Swift протокол можно понимать как своего рода «правило» или «требование».

Его можно сравнить с соглашением в международной организации: если какая-то страна хочет вступить в международную организацию, обычно ей нужно сначала соответствовать ряду правил этой организации, раскрыть определенные данные и соблюдать общие нормы. Только после выполнения этих условий она может вступить или участвовать в определенных делах.

Протокол работает по похожей логике.

Когда какой-то тип хочет соответствовать определенному протоколу, ему нужно удовлетворять содержанию, которое этот протокол требует. Только после выполнения этих требований тип сможет использовать соответствующую функциональность.

Базовая запись:

```swift
struct Movie: Identifiable {
    // ...
}
```

Здесь `: Identifiable` означает, что `Movie` соответствует протоколу `Identifiable`.

Если тип соответствует нескольким протоколам, их можно разделять запятыми:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

Для `Identifiable` самое важное требование состоит в том, что у типа должен быть `id`, с помощью которого он идентифицирует сам себя.

Например:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

Здесь `id` похож на номер удостоверения личности, который используется для различения каждого фильма.

Пока `id` каждого фильма отличается, SwiftUI сможет правильно различать каждый элемент массива.

Например:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

А затем использовать в `ForEach`:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

Тогда ошибки не будет.

Поскольку `Movie` уже соответствует `Identifiable`, еще более распространенный способ записи — просто опустить `id:`:

```swift
ForEach(lists) { movie in
    // ...
}
```

Потому что SwiftUI уже знает, что для различения каждого элемента нужно использовать `movie.id`.

## Идентификатор различения в `ForEach`

Ниже мы глубже поймем роль `id` в `ForEach`.

Например:

```swift
ForEach(lists, id: \.self)
```

Это означает: использовать сам элемент как идентификатор различения.

А:

```swift
ForEach(lists, id: \.id)
```

означает: использовать поле `id` элемента как идентификатор различения.

Если какое-то поле само по себе может гарантировать уникальность, его тоже можно временно использовать.

Например, если название каждого фильма отличается, то следующая запись может нормально работать и на текущих данных:

```swift
ForEach(lists, id: \.name)
```

Но здесь есть скрытая проблема: `name` не обязательно всегда уникально.

Предположим, что позже вы добавите еще один фильм с таким же названием — тогда `name` уже не сможет точно различать каждый элемент.

В этот момент код все еще может компилироваться, но при обновлении view, вставке, удалении и изменении SwiftUI может ошибочно распознавать элементы, что приведет к неправильному отображению.

Поэтому, когда нужно стабильно различать данные, лучше использовать действительно уникальное поле, то есть `id`.

## UUID

Хотя ручная запись `id: 1`, `id: 2` может решить проблему, все еще остается риск: можно случайно написать повторяющийся `id`, и уникальный идентификатор перестанет работать.

Например:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

В таком случае появятся два одинаковых `id`, и уникальная идентификация потеряет силу.

Чтобы избежать ручных ошибок, мы обычно используем `UUID()`.

В Swift `UUID()` случайным образом создает 128-битный идентификатор. Обычно он отображается как длинная строка из букв и цифр.

Например:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

Здесь `UUID()` фактически создает экземпляр типа `UUID`; вы можете понимать его как автоматически сгенерированный «уникальный номер».

Поэтому создаваемое каждый раз значение почти никогда не повторяется, и оно очень хорошо подходит для использования в качестве `id`.

Мы можем изменить `Movie` так:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Этот код означает, что полю `id` в структуре `Movie` по умолчанию присваивается новый экземпляр `UUID`.

То есть каждый раз, когда мы создаем новый `Movie`, система автоматически сначала генерирует для нас уникальный `id`.

У `id` уже есть значение по умолчанию, поэтому при последующем создании экземпляров `Movie` нам больше не нужно вручную заполнять `id`.

Например:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Хотя здесь `id` не написан, на самом деле этот экземпляр фильма все равно имеет собственный `id`; просто это значение уже автоматически сгенерировано через `UUID()`.

После использования этого способа нам больше не нужно вручную заполнять `id` для каждого фильма, что не только уменьшает объем кода, но и позволяет избежать ошибок из-за повторяющихся id.

В конце нам нужно лишь добавить сверху изображение `Banner` и использовать `ScrollView`, чтобы содержимое можно было прокручивать — и весь вид списка фильмов будет завершен.

## Итог

В этом уроке мы изучили очень важную тему: `struct`.

С помощью `struct` мы можем объединить несколько связанных полей одного фильма в одно целое, вместо того чтобы разносить название, режиссера и рейтинг по нескольким массивам.

После объединения данных мы можем использовать `ForEach`, чтобы поочередно отображать каждый фильм из массива во view.

Одновременно мы поняли и роль «идентификатора различения» в `ForEach`. SwiftUI должно знать, как различать каждый элемент массива, чтобы правильно отображать и обновлять view.

Поэтому мы заставили `Movie` соответствовать протоколу `Identifiable` и предоставили ему уникальный `id`.

Чтобы избежать ошибок при ручном заполнении `id`, мы дополнительно изучили `UUID()`, позволяющий системе автоматически генерировать уникальный идентификатор.

После прохождения этого урока вы не только завершили список фильмов, но и начали знакомиться с важным подходом в SwiftUI: **сначала упорядочить данные, а затем на их основе генерировать view.**

## Полный код

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
