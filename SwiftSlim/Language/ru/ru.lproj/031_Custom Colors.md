# Пользовательские цвета

В этом уроке мы узнаем, как создавать пользовательские цвета в SwiftUI.

В предыдущих уроках мы уже использовали стандартные цвета, предоставляемые SwiftUI, например:

```swift
Color.blue
Color.red
Color.green
```

Эти цвета очень удобны в использовании, но в реальной разработке приложений стандартных цветов обычно недостаточно для точного соответствия дизайну.

Например, в макете дизайна могут использоваться такие цвета:

```text
#2c54c2
#4875ed
#213e8d
```

Такие цвета называются Hex-цветами.

В этом уроке мы сначала добавим в SwiftUI поддержку Hex-цветов, а затем с помощью `static` упорядочим часто используемые цвета.

В конце мы применим пользовательские цвета к представлению энциклопедии животных и дополнительно используем градиентный фон, чтобы кнопки выглядели более объемными.

## Зачем нужны пользовательские цвета?

В SwiftUI мы можем напрямую использовать системные цвета.

Например:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

Здесь `.blue` означает синий цвет. На самом деле это сокращенная запись `Color.blue`.

Преимущество стандартных цветов в том, что они простые и удобные, но выбор цветов довольно ограничен.

Например:

```swift
Color.blue
```

Может обозначать только стандартный синий цвет, предоставляемый SwiftUI.

![Color.blue](../../../Resource/031_color6.png)

Но в реальной разработке нам часто нужны более конкретные цвета.

Например, один и тот же синий может иметь разные варианты: светло-синий, темно-синий, серо-синий, ярко-синий и другие эффекты.

![More Blue](../../../Resource/031_color5.png)

В такой ситуации, если использовать только `Color.blue`, будет трудно точно воспроизвести дизайнерский эффект.

Поэтому нам нужно сделать так, чтобы SwiftUI поддерживал пользовательские цвета.

## Что такое Hex-цвет?

Цвета на экране обычно состоят из трех каналов: красного, зеленого и синего, то есть RGB.

RGB означает:

```text
Red     // красный
Green   // зеленый
Blue    // синий
```

Hex-цвет — это один из способов записи RGB-цвета.

Например:

```swift
#5479FF
```

Это значение цвета можно условно разделить на три части:

```text
54  // обозначает красный канал
79  // обозначает зеленый канал
FF  // обозначает синий канал
```

В этом уроке нам не нужно вычислять эти значения и глубоко разбираться в правилах шестнадцатеричной записи.

Сейчас достаточно знать: `#5479FF` обозначает конкретный цвет.

Позже, когда мы увидим записи вроде `#2c54c2` или `#4875ed`, мы сначала можем понимать их как значения цвета.

В таких инструментах дизайна, как Sketch, Figma и Photoshop, подобные значения цветов тоже встречаются очень часто.

![color](../../../Resource/031_color.png)

Но по умолчанию в SwiftUI нельзя напрямую написать так:

```swift
Color(hex: "#5479FF")
```

Поэтому нам нужно самостоятельно расширить тип `Color`, чтобы он поддерживал создание цвета через Hex-строку.

## Создание файла Color+Hex.swift

Сначала мы создаем новый Swift-файл.

Имя файла можно написать так:

```text
Color.swift
```

Также можно написать более понятно:

```text
Color+Hex.swift
```

Здесь более рекомендуется использовать:

```text
Color+Hex.swift
```

В Swift-проектах имена файлов вроде `Color+Hex.swift` встречаются очень часто.

Они означают, что этот файл является файлом расширения, который добавляет `Color` возможность работы с Hex.

Имя файла напрямую не влияет на выполнение кода, оно лишь помогает нам легче понять назначение этого файла.

## Добавление кода Color(hex:)

В файле `Color+Hex.swift` введите следующий код:

```swift
import SwiftUI

extension Color {
    init(hex: String) {
        let scanner = Scanner(string: hex)
        
        if hex.hasPrefix("#") {
            scanner.currentIndex = hex.index(after: hex.startIndex)
        }
        
        var rgbValue: UInt64 = 0
        scanner.scanHexInt64(&rgbValue)
        
        let red = Double((rgbValue & 0xFF0000) >> 16) / 255.0
        let green = Double((rgbValue & 0x00FF00) >> 8) / 255.0
        let blue = Double(rgbValue & 0x0000FF) / 255.0
        
        self.init(red: red, green: green, blue: blue)
    }
}
```

Этот код использует `extension`, чтобы расширить тип `Color`, и добавляет в `Color` новый инициализатор:

```swift
init(hex: String)
```

После появления этого инициализатора мы можем передавать Hex-строку в `Color` и тем самым создавать пользовательский цвет:

```swift
Color(hex: "#5479FF")
```

Внутри этот код расширения преобразует Hex-строку в RGB-цвет, который SwiftUI может распознать.

На этом этапе не нужно глубоко понимать каждую строку логики преобразования. Достаточно знать, что добавленный метод `Color(hex:)` позволяет создавать пользовательские цвета через значения Hex-цветов.

## Использование пользовательских цветов

Теперь мы можем протестировать пользовательские цвета в `ContentView`.

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Color.blue")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color.blue)
            
            Text("#2c54c2")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#2c54c2"))
            
            Text("#4875ed")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#4875ed"))
            
            Text("#213e8d")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#213e8d"))
        }
        .padding()
    }
}
```

Результат отображения:

![color](../../../Resource/031_color1.png)

В этом примере первая строка использует системный цвет:

```swift
.foregroundStyle(Color.blue)
```

Следующие три строки используют пользовательские Hex-цвета:

```swift
.foregroundStyle(Color(hex: "#2c54c2"))
.foregroundStyle(Color(hex: "#4875ed"))
.foregroundStyle(Color(hex: "#213e8d"))
```

Сравнив их, можно заметить, что системный синий цвет имеет только один стандартный эффект.

А Hex-цвета могут выражать более тонкие изменения синего.

В этом и состоит ценность пользовательских цветов: они позволяют цветам интерфейса быть ближе к реальному дизайну и дают нам более удобный контроль над визуальным стилем приложения.

## Использование static для упорядочивания цветов

Теперь мы уже можем создавать цвета через Hex-строку:

```swift
Color(hex: "#2c54c2")
```

Такая запись работает нормально, но если один и тот же цвет будет повторяться в нескольких местах, дальнейшее обслуживание станет не слишком удобным.

Если это значение цвета написано в 10 местах, то при желании изменить этот синий цвет придется изменять его по одному в каждом месте.

В такой ситуации мы можем использовать `static`, чтобы единообразно упорядочить часто используемые цвета.

Внизу файла `Color+Hex.swift` продолжите добавлять следующий код:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Здесь мы добавили в `Color` три статических свойства: `animalBlue`, `animalLightBlue` и `animalDarkBlue`. Они обозначают разные оттенки синего.

Поскольку эти свойства используют `static`, они принадлежат самому типу `Color`.

При использовании к ним можно обращаться напрямую через `Color.`:

```swift
Color.animalBlue
```

Такая запись понятнее, чем прямое написание Hex-строки.

Когда мы видим `Color.animalBlue`, мы можем понять, что он обозначает синий цвет, используемый в энциклопедии животных.

А когда мы видим `Color(hex: "#2c54c2")`, мы можем лишь понять, что это значение цвета, но не так легко увидеть его конкретное назначение.

Единое управление цветами имеет еще одно преимущество: последующие изменения становятся удобнее.

Если нужно настроить основной цвет энциклопедии животных, достаточно изменить место определения:

```swift
static let animalBlue = Color(hex: "#1f5cff")
```

Все места, где используется этот цвет, обновятся вместе.

В этом и заключается смысл использования `static` для упорядочивания цветов: сделать названия цветов понятнее, а последующее обслуживание удобнее.

## Применение к энциклопедии животных

Теперь мы можем применить пользовательские цвета к предыдущему представлению энциклопедии животных.

Раньше фон кнопки животного был белым:

```swift
.background(Color.white)
```

![single white button](../../../Resource/031_color7.png)

Теперь мы можем заменить его на пользовательский цвет:

```swift
.background(Color.animalBlue)
```

![single blue button](../../../Resource/031_color8.png)

Здесь `Color.animalBlue` — это статический цвет, который мы только что определили в `Color+Hex.swift`.

Текст кнопки использует белый цвет:

```swift
.foregroundStyle(Color.white)
```

Фон emoji животного использует полупрозрачный белый цвет:

```swift
.background(Color.white.opacity(0.15))
```

Так кнопка сформирует единый синий визуальный стиль.

Главная цель этого шага не в добавлении сложного кода, а в том, чтобы действительно применить изученные ранее пользовательские цвета в интерфейсе.

## Использование градиентного фона

Помимо использования одного цвета, мы также можем объединить несколько цветов в градиентный эффект.

Например, ранее мы определили такие цвета:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Эти цвета можно использовать не только по отдельности, но и объединять в градиентный фон.

В SwiftUI можно использовать `LinearGradient` для создания линейного градиента.

Например:

```swift
LinearGradient(
    colors: [Color.animalBlue, Color.animalLightBlue],
    startPoint: .leading,
    endPoint: .trailing
)
```

Этот код создает градиентный фон слева направо, где цвет постепенно переходит от `Color.animalBlue` к `Color.animalLightBlue`.

Здесь `colors` используется для настройки цветов, участвующих в градиенте, а `startPoint` и `endPoint` используются для управления направлением градиента.

### Тестирование градиентного эффекта

Мы можем протестировать градиентный фон с помощью простого `Text`.

```swift
Text("Linear Gradient")
    .font(.title)
    .fontWeight(.bold)
    .padding()
    .foregroundStyle(Color.white)
    .background {
        LinearGradient(
            colors: [Color.animalBlue, Color.animalLightBlue],
            startPoint: .leading,
            endPoint: .trailing
        )
    }
    .cornerRadius(12)
```

Результат отображения:

![color](../../../Resource/031_color3.png)

В этом примере фон `Text` больше не является одним сплошным цветом, а представляет собой градиент, постепенно изменяющийся слева направо.

По сравнению с обычным цветом фона, градиентный фон выглядит более объемным и легче создает визуальный акцент в интерфейсе.

## Использование градиентного фона в энциклопедии животных

Теперь мы можем заменить цвет фона кнопки животного:

```swift
.background(Color.animalBlue)
```

На градиентный фон:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .leading,
        endPoint: .trailing
    )
}
```

Полный код выглядит так:

```swift
import SwiftUI

struct ContentView: View {
    @State private var selectedAnimal: Animal? = nil
    
    let animals = Animal.animals
    
    var body: some View {
        VStack(alignment: .leading, spacing: 20) {
            ForEach(animals) { animal in
                Button(action: {
                    selectedAnimal = animal
                }, label: {
                    HStack(spacing: 10) {
                        Text(animal.avatarEmoji)
                            .font(.title2)
                            .frame(width: 40, height: 40)
                            .background(Color.white.opacity(0.15))
                            .cornerRadius(20)
                        
                        Text(animal.name)
                            .fontWeight(.bold)
                            .foregroundStyle(Color.white)
                        
                        Spacer()
                    }
                    .padding(15)
                    .frame(width: 200, height: 60)
                    .background {
                        LinearGradient(
                            colors: [Color.animalBlue, Color.animalLightBlue],
                            startPoint: .leading,
                            endPoint: .trailing
                        )
                    }
                    .cornerRadius(12)
                    .shadow(radius: 5)
                })
                .buttonStyle(.plain)
            }
        }
        .padding()
        .sheet(item: $selectedAnimal) { animal in
            AnimalDetailView(animal: animal)
        }
    }
}
```

Результат отображения:

![color](../../../Resource/031_color4.png)

Теперь кнопки животных больше не имеют только однотонный фон, а получили градиентный эффект слева направо.

По сравнению с однотонным фоном, градиентный фон делает интерфейс более объемным и ближе к визуальному дизайну реального приложения.

## Разница между однотонным и градиентным фоном

Однотонный фон использует только один цвет.

Например:

```swift
.background(Color.animalBlue)
```

Такая запись простая и понятная, она подходит для большинства базовых интерфейсов.

Градиентный фон использует несколько цветов.

Например:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .topLeading,
        endPoint: .bottomTrailing
    )
}
```

Такая запись может сделать интерфейс более объемным, но также легче увеличивает визуальную сложность.

Поэтому в реальной разработке это можно понимать так:

Однотонный фон подходит для обычного текста, обычных кнопок, обычных карточек и обычного фона страницы.

Градиентный фон подходит для ключевых кнопок, верхних областей, обложек-карточек, входов в функции и других мест, которые нужно выделить.

Так код станет понятнее, а стиль интерфейса будет легче унифицировать.
