## Знакомство с ContentView

Код ContentView:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Хотя кода здесь немного, он уже содержит ключевую структуру SwiftUI.

### 1. Комментарии

В верхней части файла:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

Это комментарий файла, который используется для записи информации о файле, включая имя файла, имя проекта, автора и время создания.

В Swift для однострочного комментария используется //:

```swift
// Это комментарий
```

Комментарии повышают читаемость кода и помогают разработчику понимать его логику.

В реальной разработке код иногда бывает трудно понять. Если не писать комментарии, можно столкнуться с ситуацией, когда уже через три дня не понимаешь собственный код.

Поэтому привычка активно добавлять комментарии во время написания кода — это хороший стиль разработки. С помощью комментариев удобно фиксировать логику кода, чтобы позже было легче поддерживать проект.

**Временное отключение кода**

Комментарии также позволяют временно отключать код, что помогает при поиске ошибок.

Например:

```
A
B
C
```

A, B и C — это три фрагмента кода, и один из них содержит ошибку. Мы можем временно закомментировать код и поочерёдно проверить каждый фрагмент.

Сначала добавим комментарий к A:

```
// A
B
C
```

Если после отключения A код начинает работать нормально, значит, проблема находится в коде A.

Если после отключения A проблема остаётся, можно затем отключить B и так далее, пока не будет найден проблемный участок.

Во время разработки возникает много проблем, и чаще всего мы используем комментирование кода для поиска причины. Это помогает точно определить проблемный участок и найти BUG.

В Xcode можно использовать сочетание клавиш:

```
Command ⌘ + /
```

чтобы быстро добавить или удалить комментарий.

### 2. Импорт фреймворка SwiftUI

```swift
import SwiftUI
```

Этот код означает импорт фреймворка SwiftUI.

Такие типы SwiftUI, как View, Text, Image, VStack и другие, приходят именно из этого фреймворка.

Если не импортировать SwiftUI, Xcode покажет ошибку:

```
Cannot find type 'View' in scope
```

Это означает, что компилятор не может распознать тип View.

### 3. Структура View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Когда вы впервые видите такую структуру View, она может показаться незнакомой, потому что в ней используются ключевые слова struct, View, var, body, some.

Сейчас мы ещё не изучали эти ключевые слова. Пока достаточно понимать, что этот код создаёт представление View с именем ContentView.

Можно представить View как холст, на котором мы рисуем, а инструментом для рисования выступает SwiftUI.

Например:

![Swift](../../RESOURCE/002_view.png)

На изображении выше показаны три страницы, и по сути это три разных View.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}

    var body: some View {

```

Когда мы разрабатываем приложение на SwiftUI, каждая страница представляет собой отдельный View.

### 4. Код SwiftUI

Внутри представления View отображается такой код SwiftUI:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Этот код SwiftUI означает, что перед нами вертикальный контейнер, в котором отображаются иконка и текст.

![Swift](../../RESOURCE/002_view1.png)

#### Макет VStack

```swift
VStack { }  // вертикальный макет
```

VStack обозначает контейнер вертикального расположения. Внутренние представления размещаются сверху вниз.

![Swift](../../RESOURCE/002_view8.png)

В SwiftUI есть три наиболее распространённых типа макетов:

* VStack —— вертикальное расположение
* HStack —— горизонтальное расположение
* ZStack —— наложение по слоям (ось Z)

```swift
HStack { }  // горизонтальное расположение
ZStack { }  // наложение
```

Схема расположения элементов в разных макетах:

![Swift](../../RESOURCE/002_view2.png)

Например, использование HStack для горизонтального расположения:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Можно увидеть, что иконка глобуса и текст отображаются по горизонтали.

![Swift](../../RESOURCE/002_view3.png)

Когда нам нужно расположить элементы по горизонтали, мы используем HStack. Когда требуется наложение, используется ZStack.

#### Image и SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Этот код отображает иконку глобуса. Иконка глобуса взята из системы иконок Apple под названием SF Symbols.

![Swift](../../RESOURCE/002_view9.png)

Смысл этого кода — показать иконку глобуса большого размера с цветом акцента.

Кроме глобуса, мы можем отображать и другие иконки.

Например, показать рюкзак:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**Как показывать другие иконки?**

Для этого нужно использовать официальный системный каталог иконок Apple — SF Symbols.

Откройте официальный сайт Apple Developer и скачайте [SF Symbols](https://developer.apple.com/sf-symbols/).

![Swift](../../RESOURCE/002_sf.png)

Откройте приложение SF Symbols.

![Swift](../../RESOURCE/002_sf1.png)

Слева находятся категории символов, справа — соответствующие иконки.

Щёлкните правой кнопкой мыши по иконке и выберите "Скопировать 1 имя"; имя будет строкой, соответствующей символу.

Например:

```
"globe"
"backpack"
"heart"
```

Если вставить скопированное имя иконки в Image(systemName:), можно отобразить другой символ.

Обратите внимание: у каждой иконки SF Symbols есть минимальная поддерживаемая версия системы. Если версия системы слишком старая, иконка может не отображаться. Информацию о совместимости можно посмотреть в приложении SF Symbols.

#### Модификаторы

В SwiftUI модификатор (modifier) — это способ изменить внешний вид или поведение представления.

Модификаторы можно представить как одежду: одна и та же вещь, но в разной одежде она выглядит по-разному.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Здесь imageScale и foregroundStyle — это модификаторы представления Image, которые изменяют его внешний вид, не меняя само содержимое Image.

**1. imageScale**

```swift
.imageScale(.large)
```

Этот модификатор управляет размером иконки SF Symbols:

* .small
* .medium
* .large

![Swift](../../RESOURCE/002_view11.png)

Мы можем выбирать разные варианты, чтобы менять размер символа SF Symbols.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

foregroundStyle управляет цветом переднего плана.

.tint означает текущий акцентный цвет среды (accent color); по умолчанию в iOS это синий цвет.

Если мы хотим изменить цвет переднего плана на красный:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Представление Text

Text — это текстовое представление, которое используется для отображения строк.

```swift
Text("Hello, world!")
```

Например, можно показать моё имя:

```swift
Text("FangJunyu")
```

Обратите внимание: строка должна быть заключена в двойные кавычки "".

Вы можете попробовать вывести своё имя, номер телефона и другие данные.

#### Отступ padding

В SwiftUI padding используется для добавления пустого пространства между содержимым представления и его границей. Это относится к внутреннему отступу (padding / content inset).

```swift
HStack {
    ...
}
.padding()
```

Этот код означает, что к представлению HStack добавляется системный стандартный отступ.

**Что такое padding?**

padding означает "область пустого пространства между содержимым представления и его границей".

На изображении ниже после установки padding для синего HStack синяя область как бы сжимается внутрь и визуально становится "чуть меньше".

![Swift](../../RESOURCE/002_view6.png)

**Отступ по умолчанию**

Модификатор padding() по умолчанию использует стандартный рекомендуемый системой отступ.

```swift
.padding()
```

На разных платформах и в разных контекстах это значение может отличаться. Например:

* в iOS обычно это около 16 pt
* в macOS или watchOS стандартный отступ может быть больше или меньше в зависимости от правил дизайна конкретной платформы

**Настраиваемый отступ**

Также можно задавать отступ отдельно для разных сторон.

1. Установка одного направления

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

Обозначения направлений:

* .top: верхний отступ
* .bottom: нижний отступ
* .leading: передний край
* .trailing: задний край

![Swift](../../RESOURCE/002_view12.png)

Обратите внимание: leading и trailing автоматически адаптируются к направлению письма. Например, в арабской среде (RTL) они автоматически меняются местами.

2. Установка нескольких направлений

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

Можно использовать массив, чтобы одновременно указать несколько направлений. Подробно массивы будут рассматриваться в следующих уроках, а пока достаточно просто знать такой способ записи.

3. Установка горизонтального или вертикального направления

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

Это эквивалентно:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**Удаление отступа**

Если отступ совсем не нужен, можно использовать .padding(0):

```swift
.padding(0)
```

Или просто удалить модификатор padding:

```swift
// .padding()
```

### 6. Код предпросмотра Preview

```swift
#Preview {
    ContentView()
}
```

Этот код означает, что в Canvas будет создан предпросмотр представления ContentView.

Обратите внимание: #Preview — это новый синтаксис, введённый в Swift 5.9 / Xcode 15. До этого использовалась структура PreviewProvider.

**Что произойдёт, если закомментировать Preview?**

Если закомментировать Preview:

```swift
// #Preview {
//    ContentView()
// }
```

Canvas больше не будет показывать визуализируемое содержимое.

![Swift](../../RESOURCE/002_xcode.png)

Именно #Preview управляет отображением предварительного просмотра в Canvas.

Когда нам нужно просматривать SwiftUI-представление в Xcode, мы добавляем код предпросмотра #Preview. Если предпросмотр не нужен, этот код можно закомментировать или удалить.

## Итоги

Хотя в файле ContentView.swift немного кода, на самом деле он охватывает сразу несколько ключевых понятий SwiftUI. Для новичков этот код может показаться непривычным, но если разобрать его по частям, можно сформировать первое целостное представление о SwiftUI.

Подведём итог тому, что мы изучили в этом уроке. Сначала мы разобрали комментарии //, которые можно использовать для пояснения логики кода или для временного отключения фрагментов кода.

Затем мы узнали, что файл SwiftUI должен импортировать фреймворк SwiftUI:

```swift
import SwiftUI
```

Без импорта этого фреймворка компилятор не сможет распознать такие типы, как View.

После этого мы познакомились с базовой структурой SwiftUI-представления:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Здесь ContentView — это имя представления.

Мы также изучили три распространённых контейнера компоновки: VStack (вертикальное расположение), HStack (горизонтальное расположение) и ZStack (наложение).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Этот код означает, что отображается контейнер с вертикальным расположением, а внутри него находятся иконка и текст.

Image может показывать иконки SF Symbols, а с помощью модификаторов можно управлять размером и цветом иконки.

Представление Text может отображать текстовое содержимое.

padding — это отступ, который добавляет прозрачное пространство вокруг представления.

И наконец, #Preview отвечает за предварительный просмотр, который отображается в Canvas.

### Практика после урока

Для начинающих материал этого урока может показаться довольно насыщенным. Чтобы лучше закрепить знания, можно выполнить несколько дополнительных упражнений:

* изменить имя иконки SF Symbols
* изменить цвет иконки на чёрный
* заменить VStack на HStack
* закомментировать Image или Text и посмотреть, как изменится предпросмотр

### Бонус: автодополнение кода (Code Completion)

Во время ввода кода вы, возможно, уже заметили, что Xcode автоматически показывает список доступных вариантов.

Например, когда мы меняем модификатор imageScale:

```swift
.imageScale(.)
```

Xcode показывает доступные варианты:

![Swift](../../RESOURCE/002_view10.png)

Это относится к механизму автодополнения кода (Code Completion). Он основан на выводе типов и подсказках членов перечислений, помогает ускорить ввод и уменьшить количество ошибок.

В последующих уроках мы официально познакомимся с перечислениями (enum), а сейчас достаточно просто иметь общее представление.
