# Сохранение суммы в копилке

На прошлом уроке мы реализовали простую «копилку». На этом уроке мы изучим, как сделать так, чтобы сумма в «копилке» сохранялась постоянно.

Основная тема этого урока — `UserDefaults` и property wrapper `@AppStorage`. Также мы дополнительно разберём назначение `onAppear` и использование опционального типа (`nil`).

Нужно обратить внимание: в этом уроке мы продолжаем использовать код «копилки», созданный на прошлом занятии.

## Проблема постоянного хранения данных

Сейчас интерфейс «копилки» очень простой и удобный в использовании.

![Piggy Bank](../../Resource/014_view.png)

Но есть одна очевидная проблема: каждый раз при обновлении представления или закрытии приложения общая сумма в «копилке» снова становится `0`, а прежняя сумма полностью теряется.

Это означает, что данные на самом деле не сохраняются. Иными словами, текущая «копилка» умеет хранить только временные данные.

### Почему данные не сохраняются постоянно?

Это связано с тем, что мы используем переменную, объявленную через `@State`:

```swift id="4j8dza"
@State private var amount = 0
```

Переменная, объявленная через `@State`, полностью зависит по времени жизни от представления.

Когда представление создаётся, `amount` инициализируется значением `0`; когда представление уничтожается, `amount` тоже уничтожается.

Данные в `amount` существуют только в памяти и не сохраняются на устройстве.

Если мы не хотим, чтобы `amount` был привязан к жизненному циклу представления, нам нужно реализовать постоянное хранение данных. То есть сохранить данные на устройстве.

### Что такое «постоянное хранение данных»

Постоянное хранение данных можно понимать как сохранение данных из «временной памяти» в «хранилище устройства».

Когда представление закрывается или приложение завершается, данные не теряются.

В разработке на Swift для простого постоянного хранения можно использовать `UserDefaults`, а для более сложных данных — `SwiftData` или `CoreData`.

На этом уроке мы сначала изучим самый простой вариант — `UserDefaults`.

## UserDefaults

`UserDefaults` используется для хранения небольших данных в виде пар ключ-значение. Обычно он применяется для `String`, `Int`, `Double`, `Bool` и других базовых типов.

### Сохранение данных

`UserDefaults` использует метод `set` для сохранения данных:

```swift id="m2g3fh"
UserDefaults.standard.set("FangJunyu", forKey: "name")
UserDefaults.standard.set(28, forKey: "age")
UserDefaults.standard.set(true, forKey: "isMember")
```

Первый параметр `UserDefaults` — это сохраняемое значение. Здесь передаются данные типа `String`, `Int` и т. д.

Второй параметр `forKey` похож на имя переменной и используется для обозначения данных, сохранённых в `UserDefaults`.

### Чтение данных

`UserDefaults` использует соответствующие методы для чтения:

```swift id="r7n2pk"
let name = UserDefaults.standard.string(forKey: "name")	// FangJunyu
let age = UserDefaults.standard.integer(forKey: "age")	// 28
let isMember = UserDefaults.standard.bool(forKey: "isMember")	// true
```

При чтении необходимо использовать метод соответствующего типа, иначе можно получить ошибку или некорректные данные.

### На что обратить внимание при работе с UserDefaults

#### 1. Возврат опционального типа

При чтении данных из `UserDefaults` некоторые методы возвращают опциональный тип (`Optional`).

Например:

```swift id="2s6vla"
let name = UserDefaults.standard.string(forKey: "name")
```

Тип `name` будет таким:

```swift id="z4e8tw"
String?
```

Это означает, что значение может быть, а может и отсутствовать.

**Почему возвращается опциональный тип?**

Потому что у некоторого `Key` в `UserDefaults` не обязательно уже есть соответствующие данные.

Например:

```swift id="q9h3mb"
let name = UserDefaults.standard.string(forKey: "City") // nil
```

Если ключ `"City"` никогда раньше не сохранялся, то результат чтения будет:

```swift id="b5x7rn"
nil
```

Нужно подчеркнуть, что `nil` означает отсутствие данных, а не пустую строку.

В разделе «Система типов» мы уже сталкивались с похожей ситуацией: когда преобразование типа завершается неудачно, возвращается опциональный тип.

В таком случае нам нужно использовать `??`, чтобы задать значение по умолчанию и выполнить распаковку:

```swift id="k1u9cy"
let name = UserDefaults.standard.string(forKey: "City")	?? "Rizhao"
```

Если `UserDefaults` считал данные, используется считанное значение; если данные отсутствуют (`nil`), используется значение по умолчанию.

#### 2. Возврат не опционального типа

При чтении `Int`, `Double`, `Bool` из `UserDefaults` возвращаются не опциональные типы.

Например:

```swift id="p6w2jk"
let num = UserDefaults.standard.integer(forKey: "num")	// 0
let height = UserDefaults.standard.double(forKey: "height")	// 0.0
let isBuyCar = UserDefaults.standard.bool(forKey: "isBuyCar")	// false
```

Даже если для этих ключей данные никогда не сохранялись, возвращаемое значение не будет `nil`.

Вместо этого будут возвращены значения по умолчанию:

```swift id="n8f1de"
Int → 0
Double → 0.0
Bool → false
```

#### 3. Ограничение по объёму данных

`UserDefaults` подходит только для хранения небольшого количества данных, например настроек приложения или простых состояний.

На практике он не подходит для хранения больших объёмов данных или данных, которые часто читаются и записываются.

## Сохранение суммы «копилки»

Теперь мы можем использовать `UserDefaults`, чтобы реализовать сохранение общей суммы «копилки».

Когда пользователь нажимает кнопку, введённая сумма прибавляется к общей сумме, после чего общая сумма сохраняется в `UserDefaults`.

```swift id="v3m7oq"
Button("Save") {
    amount += number
    number = 0
	UserDefaults.standard.set(amount, forKey: "amount")	// UserDefaults
}
.buttonStyle(.borderedProminent)
```

Теперь мы реализовали логику записи данных.

## Пояснение логики постоянного хранения

Хотя мы уже реализовали логику записи данных, при реальном запуске можно заметить: при каждом обновлении представления или повторном открытии приложения переменная `amount` всё равно остаётся `0`.

Причина в следующем:

```swift id="y6k9rv"
@State private var amount = 0
```

В SwiftUI переменная `@State` при создании представления инициализируется заново.

Даже если значение `amount` уже сохранено в `UserDefaults`, при повторной загрузке представления `amount` всё равно снова становится `0`.

Это показывает, что пока что данные только записываются в `UserDefaults`, но не восстанавливаются обратно в представление.

Поэтому при загрузке представления нам нужно вручную считать данные из `UserDefaults` и заново присвоить их переменной `amount`. Только так можно завершить полную логику постоянного хранения.

### Аналогия для понимания логики

Этот процесс можно представить как «классную доску в аудитории»:

Во время урока учитель пишет материал на доске. Эти записи похожи на текущее состояние представления (`@State`).

Когда урок заканчивается, доску стирают, чтобы подготовить её к следующему занятию. Это похоже на уничтожение представления, при котором данные в `@State` тоже очищаются.

Чтобы материал не потерялся, учитель заранее сохраняет его в конспекте. Этот конспект аналогичен данным, сохранённым в `UserDefaults`.

Когда занятие начинается снова, доска вначале пустая (`@State` инициализируется заново), и учителю нужно, опираясь на конспект, снова записать прежний материал на доску (прочитать `UserDefaults`).

Важно понимать: доска сама по себе не восстанавливает прежнее содержимое автоматически. Его нужно заново записать, «считав конспект» (то есть прочитав `UserDefaults`).

## Использование onAppear для чтения данных

Когда представление появляется на экране, нужно вручную считать значение из `UserDefaults` и присвоить его переменной `amount`, объявленной через `@State`. Так и реализуется полноценное постоянное хранение данных.

В SwiftUI для выполнения инициализирующей логики при появлении представления можно использовать `onAppear`:

```swift id="f2z4lw"
.onAppear {}
```

Код чтения `UserDefaults` можно поместить в `onAppear`:

```swift id="t7e5mx"
.onAppear {
    amount = UserDefaults.standard.integer(forKey: "amount")
}
```

Когда представление создаётся, `amount` сначала инициализируется нулём; когда представление появляется, срабатывает `onAppear`, и значение считывается из `UserDefaults` и присваивается обратно. Благодаря этому представление может показать последнюю сохранённую сумму.

Теперь вся логика постоянного хранения данных для «копилки» завершена.

## Полный код

```swift id="c8r1vn"
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
                UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
        .onAppear {
            amount = UserDefaults.standard.integer(forKey: "amount")
        }
    }
}
```

## Property wrapper @AppStorage

SwiftUI предоставляет property wrapper `@AppStorage`, который позволяет упростить логику сохранения через `UserDefaults`.

```swift id="g5n8hj"
@AppStorage("amount") private var amount = 0
```

`@AppStorage` — это обёртка над `UserDefaults`, а `"amount"` соответствует ключу `UserDefaults`.

Когда представление создаётся, данные считываются из `UserDefaults`; когда значение переменной изменяется, оно автоматически записывается обратно в `UserDefaults`.

Он также поддерживает механизм обновления состояний SwiftUI и, как `@State`, будет обновлять представление при изменении значения.

Иными словами, `@AppStorage` сам реализует чтение и сохранение данных, поэтому больше не нужно вручную вызывать `UserDefaults`, а также не требуется использовать `onAppear` для ручного чтения данных.

Поэтому мы можем заменить код с `UserDefaults` на `@AppStorage`:

```swift id="u4b6pf"
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

## Итог

На этом уроке мы изучили следующие основные моменты:

Во-первых, `UserDefaults` используется для хранения лёгких данных. Мы использовали `UserDefaults`, чтобы сохранить общую сумму на устройстве. `UserDefaults` предоставляет методы чтения и записи, и при работе с ним также нужно учитывать опциональные типы и другие особенности.

Во-вторых, переменная `@State` предназначена только для управления временным состоянием внутри представления, и её жизненный цикл совпадает с жизненным циклом представления. Когда представление создаётся заново, `@State` тоже инициализируется заново.

При восстановлении данных из `UserDefaults` мы изучили, как использовать `onAppear` для чтения `UserDefaults` в момент появления представления и восстановления данных.

Наконец, мы заменили `UserDefaults` на `@AppStorage`, чтобы автоматически считывать данные при загрузке представления и автоматически записывать их при изменении значения. Это позволяет отказаться от ручного вызова `UserDefaults` и `onAppear`.

Благодаря этим знаниям мы можем реализовать базовое хранение данных в приложении, чтобы данные сохранялись постоянно.

## Дополнительные знания — проблема значения по умолчанию в поле ввода

Сейчас мы можем ещё немного улучшить взаимодействие с полем ввода, чтобы ввод суммы стал удобнее.

![textfield](../../Resource/015_view.png)

В текущей реализации значение поля ввода по умолчанию равно `0`. Каждый раз при вводе суммы пользователю приходится сначала удалять `0`, а затем вводить новое значение, и такой сценарий неудобен.

Правильнее сделать так, чтобы поле ввода изначально было пустым, а не показывало `0`.

Причина в том, что `TextField` и `number` связаны двусторонней привязкой:

```swift id="w1q3mu"
@State private var number = 0
TextField("Amount", value: $number, format: .number)
```

Когда `number` инициализируется значением `0`, поле ввода тоже показывает `0`.

Поэтому нам нужно изменить `number` на пустое значение, то есть `nil`.

```swift id="k7z2es"
@State private var number = nil
```

В этот момент возникнет ошибка:

```swift id="p9v4ra"
Generic parameter 'Value' could not be inferred
```

Это именно та ситуация, о которой мы говорили в разделе «Система типов»: когда компилятор не может автоматически вывести тип переменной, его нужно указать явно.

Поскольку мы присваиваем `number` значение `nil`, а само `nil` не содержит никакой информации о типе, компилятор не может понять, должна ли эта переменная быть `String`, `Int` или чем-то ещё, поэтому появляется ошибка.

Следовательно, нам нужно явно объявить тип `number`:

```swift id="x5m8cn"
@State private var number:Int? = nil
```

Здесь `Int?` означает опциональный тип (`Optional`), то есть значение `number` может быть либо `Int`, либо `nil`.

В таком случае, если при инициализации задать `nil`, поле ввода будет пустым, а не `0`.

### Распаковка опционального типа

После того как мы сделали `number` опциональным типом, исходный код вычисления начнёт выдавать ошибку:

```swift id="r2t6qo"
amount += number
```

Подсказка об ошибке будет такой:

```swift id="m4k9yb"
Value of optional type 'Int?' must be unwrapped to a value of type 'Int'
```

Причина в том, что `number` теперь может быть `nil`. Если напрямую использовать его в вычислении, получится:

```swift id="s8n5dl"
amount += nil
```

Очевидно, что `nil` нельзя складывать с `amount`, такая операция недопустима.

Поэтому, когда опциональный тип участвует в вычислении, его сначала нужно распаковать.

Здесь можно использовать `??`, чтобы задать значение по умолчанию:

```swift id="h3u7xe"
amount += number ?? 0
```

Когда `number` равно `nil`, используется значение по умолчанию `0`; когда у `number` есть значение, используется фактически введённое пользователем число.

Так мы одновременно сохраняем корректность вычислений и добиваемся того, чтобы поле ввода по умолчанию было пустым, а пользователю больше не приходилось вручную удалять `0`.

## Расширенный код

```swift id="n6b1gw"
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number:Int? = nil
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number ?? 0
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```
