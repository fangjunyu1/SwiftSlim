# Яркие цвета

В этом уроке мы изучим распространённые визуальные модификаторы в SwiftUI, включая:

- цвет
- цвет переднего плана
- цвет фона
- смещение
- прозрачность
- размытие

Также мы познакомимся с Safe Area (безопасной областью).

Эти модификаторы используются для управления внешним видом представлений, делая интерфейс более чётким и многослойным.

## Цвет

В SwiftUI можно задавать цвет текста.

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

.blue означает синий цвет. На самом деле это сокращённая форма записи Color.blue благодаря выводу типа.

К распространённым цветам относятся:

```id="q7v2kp"
.black
.green
.yellow
.pink
.gray
...
```

Все они являются статическими свойствами Color.

![Color](../../RESOURCE/006_color.png)

Color можно понимать как тип цвета, а .blue, .red и другие — как его конкретные значения.

### Представление Color

В SwiftUI Color также может использоваться как самостоятельное представление.

```swift
Color.red
    .frame(width: 100, height:100)
    .cornerRadius(10)
```

![Color](../../RESOURCE/006_color5.png)

Этот код создаёт красный квадрат размером 100×100.

Также можно сделать так, чтобы весь интерфейс отображался в одном цвете:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
    }
}
```

![Color](../../RESOURCE/006_color1.png)

После запуска видно, что красный цвет не заполняет весь экран: верхняя и нижняя области iPhone по-прежнему остаются белыми. Здесь как раз проявляется существование Safe Area (безопасной области).

## Safe Area (безопасная область)

Safe Area — это область, которую система резервирует, чтобы содержимое не перекрывалось важными элементами интерфейса, включая:

1. верхнюю строку состояния (время, батарея)

2. нижнюю полоску Home

3. область выреза (notch) или Dynamic Island

![Color](../../RESOURCE/006_color3.png)

По умолчанию SwiftUI ограничивает содержимое безопасной областью, поэтому представления не растягиваются до самых краёв экрана.

### Игнорирование Safe Area (безопасной области)

Если вы хотите, чтобы цвет заполнял весь экран, можно использовать ignoresSafeArea:

```swift
Color.red
    .ignoresSafeArea()
```

Или использовать edgesIgnoringSafeArea:

```swift
Color.red
    .edgesIgnoringSafeArea(.all)
```

![Color](../../RESOURCE/006_color4.png)

Тогда представление растянется на весь экран.

Нужно учитывать, что edgesIgnoringSafeArea — это более старый способ записи. Начиная с iOS 14, рекомендуется использовать ignoresSafeArea.

## Цвет переднего плана

### Модификатор foregroundStyle

В предыдущих уроках мы уже изучали, как задавать цвет с помощью foregroundStyle.

Например:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

foregroundStyle — это новая система стилей, которая поддерживает цвета, градиенты, материалы и другое.

![Color](../../RESOURCE/006_color2.png)

### Модификатор foregroundColor

Цвет также можно задавать через foregroundColor:

```swift
Text("Fang Junyu")
    .foregroundColor(.blue)
```

Используется он почти так же, как foregroundStyle.

В последних версиях Xcode появляется подсказка, что foregroundColor может быть помечен как устаревающий в будущих версиях iOS, поэтому рекомендуется в первую очередь использовать foregroundStyle.

## Фон

Если вы хотите добавить представлению цвет фона, можно использовать background:

```swift
background(.red)
```

Например, добавим фон тексту:

```swift
Text("FangJunyu.com")
    .background(.orange)
```

![Color](../../RESOURCE/006_color6.png)

Похожий эффект фоновой подсветки можно увидеть, когда мы выделяем текст в Word, Chrome и других приложениях.

![Color](../../RESOURCE/006_color16.png)

Если нужно увеличить область фона, следует использовать его вместе с padding:

```swift
Text("FangJunyu.com")
    .padding(10)
    .background(Color.orange)
```

![Color](../../RESOURCE/006_color7.png)

Здесь важно помнить одно правило:

в SwiftUI модификаторы строят представление сверху вниз. Модификатор, написанный ниже, применяется к результату предыдущих.

Поэтому:

```swift id="21yz4v"
.padding()
.background()
```

Это означает, что фон будет оборачивать представление уже после добавления внутренних отступов.

Если поменять порядок:

```swift
Text("FangJunyu.com")
    .background(Color.orange)
    .padding(10)
```

![Color](../../RESOURCE/006_color6.png)

фон не увеличится, потому что он не оборачивает последующий padding.

## Пример — квадратная гайка с круглым отверстием

Теперь создадим простое представление, напоминающее гайку.

![Color](../../RESOURCE/006_color8.png)

Сначала создадим белый квадрат размером 50 × 50:

```swift
Color.white
    .frame(width: 50, height: 50)
```

Если нужно превратить его в круг, можно использовать cornerRadius:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
```

Когда радиус скругления равен половине ширины и высоты, фигура становится кругом.

Теперь добавим синий фон:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
    .padding(20)
    .background(.blue)
```

![Color](../../RESOURCE/006_color8.png)

Добавленный padding увеличивает внешнюю область, а background рисует синий цвет по этой внешней области.

Так мы получаем эффект простой гайки.

### Другой способ

Помимо использования фонового цвета, такой эффект можно реализовать и через ZStack.

Ранее мы изучали, что ZStack позволяет накладывать представления друг на друга. Эту гайку также можно понимать как наложение круга на прямоугольник.

```swift
ZStack {
    Color.blue
        .frame(width: 70, height: 70)
    Color.white
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color8.png)

ZStack накладывает представления по порядку, и те, что добавлены позже, отображаются сверху.

## Пример — два перекрывающихся круга

Многие иконки состоят из наложения простых фигур, например двух перекрывающихся кругов.

![Color](../../RESOURCE/006_color14.png)

Сначала создадим два круга:

```swift
Color.red
    .frame(width: 50, height:50)
    .cornerRadius(25)
Color.orange
    .frame(width: 50, height:50)
    .cornerRadius(25)
```

![Color](../../RESOURCE/006_color11.png)

Поскольку круги должны отображаться с наложением, используем макет ZStack:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color12.png)

Сейчас два круга одинакового размера полностью накладываются друг на друга.

Нам нужно, чтобы круги перекрывались лишь частично, а не совпадали полностью. Для этого можно использовать offset и создать эффект смещения.

## Смещение

offset изменяет только положение отрисовки представления, но не влияет на расчёт компоновки родительского контейнера.

Использование:

```swift id="ghm8te"
.offset(x:y:)
```

x — это смещение по горизонтали, y — смещение по вертикали.

Положительные значения означают смещение вправо / вниз, отрицательные — влево / вверх.

Используем offset, чтобы сделать круги частично перекрывающимися:

```swift
ZStack {
    Color.red
        .frame(width: 50, height: 50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color13.png)

Положение красного круга в компоновке не изменилось, но его отрисовка сместилась влево на 25 пунктов. Поэтому два круга визуально стали частично перекрываться.

## Прозрачность

В SwiftUI opacity используется для задания прозрачности представления.

Базовое использование:

```swift
.opacity(0.5)
```

Параметр opacity находится в диапазоне от 0.0 до 1.0, где:

* 0 означает полную прозрачность
* 1 означает полную непрозрачность

Можно использовать opacity, чтобы задать прозрачность оранжевому кругу:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
```

![Color](../../RESOURCE/006_color14.png)

После установки opacity в 0.8 для оранжевого круга его непрозрачность становится 80%, и при наложении двух кругов в области пересечения появляется эффект смешивания цветов.

## Размытие

В SwiftUI для создания эффекта размытия можно использовать blur:

```swift
.blur(radius:10)
```

radius задаёт радиус размытия. Чем больше значение, тем сильнее эффект.

Мы можем добавить размытие двум кругам:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x:-35)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
.blur(radius:10)
```

![Color](../../RESOURCE/006_color15.png)

В итоге мы увидим два сильно размытых круга.
