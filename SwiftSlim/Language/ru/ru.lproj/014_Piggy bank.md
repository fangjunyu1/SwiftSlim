# Копилка

В этом уроке мы разработаем простой проект «Копилка». Он несложный по функциям, но включает в себя полную логику взаимодействия, поэтому очень хорошо подходит для начинающих.

С помощью этого проекта мы изучим `TextField` (поле ввода) и рамку `border`, а также разберём привязку данных (`$`).

Итоговая цель — реализовать «копилку», в которой можно ввести сумму, нажать кнопку сохранения и накапливать общий баланс.

Результат будет выглядеть так:

![Piggy Bank](../../RESOURCE/014_view.png)

## Отображение общей суммы

Сначала нам нужно показать общую сумму в «копилке».

В `ContentView` объявим переменную, которая будет хранить общую сумму:

```swift id="1u6x8y"
@State private var amount = 0
```

Затем с помощью `Text` отобразим это значение:

```swift id="b8u7cq"
Text("\(amount)")
	.font(.largeTitle)
	.fontWeight(.bold)
```

Здесь с помощью `.font` и `.fontWeight` задаются размер и жирность шрифта.

```swift id="r9n4dp"
struct ContentView: View {
    
    @State private var amount = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
			.font(.largeTitle)
			.fontWeight(.bold)
		}
    }
}
```

Когда `amount` изменяется, `Text` автоматически обновляет отображаемую общую сумму.

## Ввод суммы

«Копилка» должна не только показывать общую сумму, но и позволять пользователю **вводить** сумму для пополнения. Для этого используется элемент интерфейса SwiftUI `TextField`.

### `TextField`

`TextField` — это элемент SwiftUI для ввода данных, обычно используемый для однострочного ввода.

`TextField` в основном поддерживает два формата привязки:

**1. Привязка строкового типа (`String`)**

```swift id="cv4m2h"
TextField("Placeholder", text: $text)
```

Подходит для ввода имени, заголовка и другого текстового содержимого.

**2. Привязка числового типа (`Int` / `Double`)**

```swift id="j5zq1k"
TextField("Amount", value: $number, format: .number)
```

Подходит для возраста, зарплаты, суммы и других числовых значений.

`format: .number` означает, что это поле ввода будет разбирать и отображать содержимое в **числовом формате**.

#### Текст-заполнитель

Первый параметр `TextField` — это заполнитель (`Placeholder`), который подсказывает пользователю, что нужно ввести:

```swift id="m3w0vn"
TextField("input your name", text: $text)
```

Когда поле ввода пустое, в нём показывается серая подсказка.

![textfield](../../RESOURCE/014_textfield.png)

#### Привязка данных

`TextField` не отвечает за постоянное хранение введённого содержимого, поэтому для управления данными нужна привязанная переменная.

```swift id="v2o8kd"
@State private var text = ""
TextField("Placeholder", text: $text)
```

Введённое пользователем содержимое напрямую сохраняется в переменной `text`, а сам `TextField` отвечает только за интерфейс ввода.

`$` означает привязку (`Binding`):

```swift id="x4f9ua"
$text
```

Это уже не обычная переменная, а тип `Binding`.

Его задача — установить связь между представлением и данными:

когда пользователь вводит текст в `TextField`, переменная `text` автоматически обновляется; когда `text` меняется, `TextField` тоже синхронно обновляется.

Например:

```swift id="w7n1js"
struct ContentView: View {
    @State private var text = ""
    
    var body: some View {
        VStack {
			Text("text:\(text)")
        	TextField("input your name", text: $text)
            	.frame(width: 150)
		}
    }
}
```

Когда вы вводите содержимое в `TextField`, верхний `Text` будет синхронно отображать его в реальном времени.

![textField](../../RESOURCE/014_textfield1.png)

### Привязка числового типа

Когда нужно вводить число, следует привязать переменную соответствующего типа:

```swift id="q8e6tr"
@State private var number = 0

TextField("Amount", value: $number, format: .number)
```

Когда пользователь вводит сумму, `TextField` будет разбирать введённое содержимое и автоматически обновлять значение `number`.

## Добавление поля ввода

Разобрав базовое использование `TextField`, применим его в проекте «Копилка».

В «Копилке» нужно вводить **сумму**, поэтому мы используем вариант с **привязкой числового типа (`Int` / `Double`)** и добавим новую переменную `number`, которая будет хранить введённую пользователем сумму:

```swift id="k6r5py"
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
            	.font(.largeTitle)
            	.fontWeight(.bold)
        	TextField("Amount", value: $number, format: .number)
		}
    }
}
```

Теперь `TextField` по умолчанию показывает значение `number` (по умолчанию это `0`).

Когда содержимое очищается, отображается заполнитель `"Amount"`.

Это ещё раз показывает, что `TextField` отображает значение привязанной переменной, а не хранит содержимое сам по себе.

### Проблема с размером поля ввода

В текущем представлении можно заметить, что `TextField` не имеет рамки, показывает только `0`, а содержимое выглядит выровненным по левому краю.

Результат выглядит так:

![textfield](../../RESOURCE/014_textfield2.png)

Это происходит потому, что по умолчанию `TextField` занимает всю ширину родительского представления.

Мы можем проверить это, добавив цвет фона:

```swift id="s4t8mb"
TextField("Amount", value: $number, format: .number)
    .background(Color.red)
```

Результат будет таким:

![textfield](../../RESOURCE/014_textfield3.png)

Можно увидеть, что `TextField` на самом деле растягивается на всю ширину строки.

### Установка фиксированной ширины

Если мы хотим сделать поле ввода компактнее, можно ограничить его ширину с помощью `frame`:

```swift id="h1x2vl"
TextField("Amount", value: $number, format: .number)
	.frame(width: 100)
```

Результат:

![textfield](../../RESOURCE/014_textfield4.png)

### Добавление рамки и внутренних отступов

Поскольку у `TextField` по умолчанию нет рамки, мы можем вручную добавить оформление:

```swift id="g3q9ze"
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Теперь это поле ввода выглядит больше похоже на «стандартное поле ввода».

![textfield](../../RESOURCE/014_textfield5.png)

## Рамка

В SwiftUI `border` используется для добавления рамки к представлению.

Базовое использование:

```swift id="n2c5ru"
border(Color.black, width: 1)
```

Здесь `Color.black` обозначает цвет рамки, а `width` — её толщину.

В `TextField` мы используем:

```swift id="u9m4oh"
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Это означает, что полю ввода добавляется чёрная рамка толщиной `1 pt`.

Также можно попробовать другие цвета, например `Color.blue`, `Color.green`, или сделать рамку толще с помощью `width: 2`.

## Кнопка сохранения

Теперь у нас есть переменная `amount` для отображения общей суммы, `TextField` для ввода суммы и переменная `number`, привязанная к полю ввода.

Текущий код выглядит так:

```swift id="e5d8kt"
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
			Text("\(amount)")
            	.font(.largeTitle)
            	.fontWeight(.bold)
        	TextField("Amount", value: $number, format: .number)
            	.frame(width: 100)
            	.padding(5)
            	.border(Color.black, width: 1)
		}
    }
}
```

### Добавление кнопки

Далее нам нужно добавить кнопку, которая будет запускать действие сохранения денег.

```swift id="f6v2na"
Button("Save") {
    
}
```

### Реализация логики сохранения

Мы хотим, чтобы пользователь вводил сумму, нажимал кнопку, и эта сумма автоматически прибавлялась к общей сумме.

```swift id="z1l4jc"
Button("Save") {
    amount += number
}
```

Здесь используется **составной оператор присваивания**, чтобы прибавить введённую пользователем сумму `number` к общей сумме `amount`.

### Сброс введённой суммы

Теперь возникает проблема: после нажатия кнопки сумма в поле ввода не очищается.

Если пользователь введёт `10`, нажмёт сохранить, то в поле ввода всё равно останется `10`, и это повлияет на следующий ввод.

Причина в том, что `TextField` привязан к `number`, а при нажатии кнопки мы изменили только `amount`, тогда как `number` не изменился. Поэтому поле ввода продолжает показывать старое значение.

Нужно добавить логику очистки: после нажатия кнопки сначала прибавить введённую сумму к общей, а затем сбросить введённое значение:

```swift id="p7y3mh"
Button("Save") {
    amount += number
	number = 0
}
```

Теперь, когда пользователь вводит сумму и нажимает кнопку, введённое значение прибавляется к общей сумме. Затем `number` сбрасывается в `0`, чтобы пользователь мог ввести следующую сумму.

Так мы реализовали всю основную логику «копилки».

## Полный код

```swift id="a2k7vf"
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

### Отладочный вывод

Чтобы дополнительно проверить правильность логики, можно добавить в кнопку отладочный вывод и посмотреть, корректно ли обрабатывается введённая сумма:

```swift id="m9u3yb"
Button("Save") {
    print("---Saving---")
    amount += number
    number = 0
    print("amount:\(amount)")
    print("number:\(number)")
}
```

Каждый раз, когда вы сохраняете сумму и нажимаете кнопку, можно увидеть соответствующий отладочный вывод и проверить, нет ли ошибок в логике кода.

```swift id="v4e1ql"
---Saving---
amount:11
number:0
```
