# struct и class

На прошлом уроке мы только начали знакомиться с `struct`, определили одну структуру и заодно расширили знания о протоколах, `UUID` и других темах.

Теперь у нас уже есть начальное понимание ключевого слова `struct`, но пока еще нельзя сказать, что мы действительно его поняли.

На этом уроке мы продолжим делать `struct` основной темой и заодно познакомимся с `class`. Благодаря этому уроку тебе станет понятнее: что такое экземпляр, что такое конструктор и почему мы можем использовать `struct` и `class` для организации данных.

## Система управления студентами

Например, сейчас мы хотим разработать систему управления студентами. У каждого студента есть имя, пол, телефон, возраст, домашний адрес и другая информация.

В этот момент мы можем объединить эту информацию и определить ее как структуру `Student`:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}
```

Здесь мы используем `struct`, чтобы определить тип студента.

Эта структура `Student` содержит несколько свойств:

- `id`: уникальный идентификатор студента
- `name`: имя студента
- `sex`: пол студента
- `phone`: телефон студента
- `age`: возраст студента
- `address`: домашний адрес

Таким образом, несколько разрозненных данных объединяются в одного полноценного «студента».

Здесь `Student` также соответствует протоколу `Identifiable` и определяет поле `id`, чтобы в дальнейшем было удобно отображать данные вместе с циклом `ForEach`.

Затем мы можем создать экземпляр студента:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

Этот код означает, что на основе структуры `Student` можно создать конкретные данные о студенте.

Здесь `Student` — это тип, а `Student(...)` обозначает созданный экземпляр.

В SwiftUI мы можем присвоить этот экземпляр переменной, а затем отобразить его.

Например:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}

struct ContentView: View {
    let student = Student(name: "Fang Junyu",
                          sex: "man",
                          phone: "123456",
                          age: 18,
                          address: "China, Shandong, Rizhao")
    
    var body: some View {
        Text(student.name)
    }
}
```

Так мы сможем увидеть в представлении имя этого студента: `Fang Junyu`.

Здесь мы сначала создаем экземпляр `Student`, затем присваиваем его переменной `student`, а в конце читаем одно из его свойств через `student.name`.

## Что такое экземпляр

Когда мы только начинаем изучать `struct`, слово «экземпляр» может казаться незнакомым.

Но на самом деле мы используем экземпляры с самого начала, просто раньше не обращали на это особого внимания.

Например:

```swift
let name = "Fang Junyu"
```

Здесь `"Fang Junyu"` — это строковое значение, и по своей сути оно является экземпляром типа `String`.

Мы также можем написать так:

```swift
let name = String("Fang Junyu")
```

Обе записи выражают одно и то же: создать экземпляр типа `String` и присвоить его переменной `name`.

То же самое относится и к `Int`:

```swift
let num = 3
```

Здесь `3` по своей сути является значением типа `Int`, и его тоже можно понимать как экземпляр `Int`.

Поэтому экземпляр — это не какой-то особый концепт, который появляется только в `struct`.

Будь то `String`, `Int`, `Double`, `Bool` или созданный нами `Student`, все они могут создавать экземпляры.

Поэтому, когда мы пишем:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

По сути это то же самое, что и создание экземпляров `String` или `Int`, просто `Student` — это тип, который мы определили сами.

## Конструктор

Когда мы создаем экземпляр:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

На самом деле мы вызываем метод инициализации `Student`, то есть `init`.

Конструктор можно понимать как: **точку входа, которая отвечает за присваивание значений свойствам при создании экземпляра.**

Иначе говоря, прежде чем экземпляр будет действительно создан, обычно он сначала проходит через `init`.

Например, когда раньше мы изучали жизненный цикл представлений SwiftUI, мы писали такой код:

```swift
struct ContentView: View {
    init() {
        print("Create ContentView")
    }
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
    }
}
```

Здесь `init()` выполняется при создании экземпляра `ContentView`, а `onAppear` выполняется только тогда, когда представление действительно показывается на экране. Иными словами, создание представления и его отображение происходят не в один и тот же момент.

Аналогично, когда мы создаем `Student(...)`, мы вызываем конструктор `Student`.

### Конструктор, автоматически сгенерированный системой

Когда мы определяли `Student`, мы не писали `init` вручную:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}
```

Но при этом мы все равно можем напрямую создать экземпляр:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

Почему можно создать экземпляр без конструктора?

Потому что для `struct`, если мы не пишем конструктор вручную, система обычно автоматически генерирует для нас подходящий конструктор.

На начальном этапе это можно просто понимать так: Swift автоматически дописывает за нас код инициализации.

Приблизительно можно считать, что система дописывает за нас такой код:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
    
    init(name: String, sex: String, phone: String, age: Int, address: String) {
        self.name = name
        self.sex = sex
        self.phone = phone
        self.age = age
        self.address = address
    }
}
```

Здесь нужно обратить внимание на одну деталь:

```swift
let id = UUID()
```

У `id` уже есть значение по умолчанию, поэтому при создании экземпляра нам не нужно вручную передавать `id`.

То есть свойства без значения по умолчанию обычно нужно передавать в конструктор, а свойства, у которых значение по умолчанию уже есть, обычно могут использовать его напрямую.

Поэтому, когда мы создаем экземпляр `Student`, нам нужно передать только `name`, `sex`, `phone`, `age` и `address`.

### Написание собственного конструктора

Хотя система может автоматически сгенерировать конструктор, иногда такой способ инициализации по умолчанию не полностью соответствует нашим потребностям.

Например, у нас есть структура `Movie`:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Согласно автоматически сгенерированному конструктору, при создании экземпляра мы должны одновременно передавать все три параметра:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Но если сейчас нам нужно ввести группу старых фильмов и мы знаем только название фильма, а режиссера и рейтинг не знаем, это будет неудобно.

Потому что нам остается писать только так:

```swift
Movie(name: "The Night of the Hunter", director: "--", rating: "--")
Movie(name: "Picnic at Hanging Rock", director: "--", rating: "--")
Movie(name: "Days of Heaven", director: "--", rating: "--")
```

Хотя это работает, каждый раз повторять `--` неудобно.

В этот момент мы можем написать конструктор:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Тогда при создании старых фильмов нам нужно будет передавать только `name`:

```swift
Movie(name: "The Night of the Hunter")
Movie(name: "Picnic at Hanging Rock")
Movie(name: "Days of Heaven")
```

В этом и заключается один из смыслов написания собственного конструктора: **сделать способ создания экземпляра более соответствующим реальным потребностям.**

### Разбор пользовательского конструктора

Давайте посмотрим на этот вручную написанный конструктор:

```swift
init(name: String) {
    self.name = name
    self.director = "--"
    self.rating = "--"
}
```

Смысл этого кода в том, что при создании экземпляра `Movie` требуется передать только один параметр `name`.

Затем внутри конструктора переданный `name` присваивается собственному свойству экземпляра `name`, а `director` и `rating` получают значение по умолчанию `--`.

`self` здесь означает «сам текущий экземпляр».

Например:

```swift
self.name = name
```

Левая часть `self.name` обозначает свойство экземпляра, а правая часть `name` — переданный нами параметр.

Это можно просто понимать так: внешнее значение сохраняется в собственное свойство экземпляра.

Когда всем свойствам присвоены значения, создание экземпляра завершено.

### Механизм работы конструктора

Когда мы явно объявляем конструктор, происходит еще одно изменение:

Тот конструктор, который система автоматически сгенерировала изначально, обычно уже нельзя использовать напрямую.

Например:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Теперь, если мы снова напишем так:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Компилятор выдаст ошибку:

```swift
Extra arguments at positions #2, #3 in call
```

Это означает: ты передал лишние аргументы.

Почему возникает эта ошибка?

Потому что в текущей структуре `Movie` есть только один вручную объявленный конструктор:

```swift
init(name: String)
```

Он принимает только один параметр `name` и не принимает `director` и `rating`.

Иначе говоря, после того как ты добавил собственный конструктор, тот способ инициализации, который автоматически генерировался раньше, уже нельзя использовать напрямую.

Если мы хотим поддерживать и вариант «передать только название», и вариант «передать полную информацию», нам нужно самостоятельно дописать еще один полный конструктор:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String, director: String, rating: String) {
        self.name = name
        self.director = director
        self.rating = rating
    }

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Так у нас будет два разных конструктора.

При создании старых фильмов можно писать так:

```swift
Movie(name: "The Night of the Hunter")
```

При создании новых фильмов с полной информацией можно писать так:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Такой способ записи также показывает: у одного и того же типа может быть несколько конструкторов, если формы их параметров различаются.

## Что такое перегрузка

Здесь дополним еще один необходимый момент.

Только что мы написали для `Movie` два `init`:

```swift
init(name: String, director: String, rating: String)
init(name: String)
```

Такой способ записи, когда «имя функции одно и то же, но параметры разные», называется «перегрузкой».

Здесь обе функции называются `init`, но они принимают разные параметры, поэтому Swift может различить, какую функцию ты вызываешь.

Например:

```swift
Movie(name: "Days of Heaven")
```

Вызовет:

```swift
init(name: String)
```

А:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Вызовет другую, полную версию метода инициализации.

Поэтому роль перегрузки состоит в том, чтобы один и тот же тип мог предоставлять разные способы создания для разных сценариев.

## Знакомство с class

Далее давай кратко познакомимся с `class`.

Помимо привычного в Swift `struct`, во многих языках программирования для создания экземпляров используется `class`, например в Java, C#, Kotlin и других.

В Swift синтаксис `class` очень близок к `struct`.

Например:

```swift
class Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String

    init(id: UUID = UUID(), name: String, director: String, rating: String) {
        self.id = id
        self.name = name
        self.director = director
        self.rating = rating
    }
}
```

Ты заметишь, что, кроме смены ключевого слова с `struct` на `class`, все остальное выглядит почти одинаково.

Способ создания экземпляра тоже очень похож:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Поэтому на уровне синтаксиса `struct` и `class` действительно очень похожи.

### Разница между Struct и Class

Хотя `struct` и `class` выглядят очень похоже, они не совсем одинаковы.

**1. Разный способ инициализации**

Для `struct`, если не писать конструктор вручную, система обычно автоматически генерирует подходящий метод инициализации.

Но для `class`, если есть свойства без значений по умолчанию, обычно нужно вручную написать `init`, иначе возникнет ошибка.

Например:

```swift
class Movie {
    let name: String
    let director: String
    let rating: String
}
```

У этого `class` нет конструктора, поэтому свойства остаются неинициализированными и возникает ошибка.

Компилятор покажет сообщение примерно такого вида:

```swift
Class 'Movie' has no initializers
```

Поэтому в этом примере для `class` нужно вручную написать конструктор.

**2. `struct` — это тип значения, `class` — ссылочный тип**

`struct` — это тип значения, а `class` — ссылочный тип.

На начальном этапе не нужно полностью разбираться в этих двух понятиях, достаточно иметь о них простое представление.

Например, `struct` больше похоже на «сделать новую копию значения».

```swift
struct User {
    var name: String
}

var user1 = User(name: "Tom")
var user2 = user1

user2.name = "Jack"

print(user1.name) // Tom
print(user2.name) // Jack
```

После присваивания `user1` в `user2` здесь как будто создается новая копия данных, поэтому изменение `user2` не влияет на `user1`.

А `class` больше похоже на «совместно ссылаться на один и тот же экземпляр».

```swift
class User {
    var name: String
    
    init(name: String) {
        self.name = name
    }
}

var user1 = User(name: "Tom")
var user2 = user1

user2.name = "Jack"

print(user1.name) // Jack
print(user2.name) // Jack
```

Здесь `user1` и `user2` указывают на один и тот же экземпляр.

Сейчас нам не нужно полностью понимать внутренний механизм типов значения и ссылочных типов, достаточно знать, что `struct` больше подходит для хранения независимых данных, а `class` — для случаев, когда в нескольких местах нужен общий экземпляр.

## Как выбирать на начальном этапе

На текущем этапе для обычных моделей данных в первую очередь стоит рассматривать `struct`, а для данных, которые нужно совместно использовать и синхронно изменять, уже рассматривать `class`.

Это не абсолютное правило, но для новичков это достаточно практичный способ принятия решения.

Особенно в SwiftUI ты часто будешь замечать, что `struct` уже может выполнить большую часть работы.

Кроме того, во многих материалах по программированию содержимое, созданное через `class`, также называют «объектом». На начальном этапе можешь пока понимать это просто как экземпляр и не делать специального различия.

## Итог

На этом уроке мы подробно изучили `struct` и заодно познакомились с `class`; оба могут использоваться для определения типов.

Типы могут создавать экземпляры. При создании экземпляра вызывается конструктор, то есть `init`.

Для `struct`, если конструктор не написан вручную, система обычно автоматически генерирует подходящий конструктор. Если конструктор по умолчанию не удовлетворяет требованиям, мы можем написать свой вручную.

Кроме того, мы также предварительно познакомились с различием между `struct` и `class`: `struct` — это тип значения, а `class` — ссылочный тип. В SwiftUI на начальном этапе чаще всего мы сталкиваемся именно со `struct`.

Позже, когда мы будем изучать передачу значений во view, модели данных и управление состоянием, эти знания будут продолжать использоваться.
