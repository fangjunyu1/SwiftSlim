# Знакомство с Xcode

## Что такое Xcode

Xcode — это официальная интегрированная среда разработки Apple (IDE, Integrated Development Environment), предназначенная для создания приложений для платформ Apple, таких как iOS, macOS, watchOS и tvOS.

![Swift](../../RESOURCE/001_xcode.png)

На протяжении всего курса мы будем использовать Xcode для написания кода и запуска программ.

В этом уроке мы изучим, как скачать Xcode, создать своё первое iOS-приложение и в общих чертах познакомиться со структурой интерфейса Xcode.

## Загрузка Xcode

Сейчас существует два способа установить Xcode:

1. Скачать через [Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12)

2. Скачать через [официальный сайт Apple Developer](https://developer.apple.com/xcode/)

Оба способа позволяют скачать Xcode. Разница в том, что на официальном сайте Apple Developer, помимо стабильной версии, можно скачать также [Beta](https://developer.apple.com/download/all/?q=Xcode)-версии и старые версии.

Если вы только начинаете учиться разработке, рекомендуется использовать стабильную версию. Beta-версии в основном предназначены для тестирования новых функций и могут работать нестабильно.

## Создание первого проекта

После открытия Xcode вы увидите стартовый экран.

![Swift](../../RESOURCE/001_xcode1.png)

Слева находятся три часто используемые опции:

- Create New Project —— создать новый проект  
- Clone Git Repository —— клонировать Git-репозиторий  
- Open Existing Project —— открыть существующий проект

Справа отображаются недавно открытые проекты Xcode.

Мы выбираем опцию "Create New Project".

### Выбор шаблона проекта

![Swift](../../RESOURCE/001_xcode2.png)

После этого вы попадёте на экран выбора шаблона проекта, где отображаются разные платформы (iOS, macOS и т.д.) и различные типы шаблонов проектов.

Описание распространённых шаблонов:
- App —— самый базовый тип приложения (рекомендуется)
- Document App —— приложение, основанное на работе с документами
- Game —— для разработки игр
- Framework —— переиспользуемый модуль

На начальном этапе мы выбираем только шаблон "iOS" - "App", потому что это тип приложения с пользовательским интерфейсом.

Нажмите "Next".

### Заполнение информации о проекте

После этого откроется экран заполнения информации о проекте, где нужно ввести данные в соответствии с типами полей.

![Swift](../../RESOURCE/001_xcode3.png)

Значение каждого поля:

- Product Name —— имя проекта, например: SwiftSlimTest
- Team —— команда разработчика; если у вас нет платной учётной записи Apple Developer, можно не выбирать
- Organization Identifier —— идентификатор организации, обычно используется формат обратного доменного имени, например:
  - com.yourname
  - com.yourcompany
  - если у вас нет домена, можно использовать своё английское имя как идентификатор
- Bundle Identifier —— уникальный идентификатор приложения; он автоматически создаётся на основе Organization Identifier и Product Name
- Interface —— технология интерфейса, выберите SwiftUI
- Language —— язык разработки, выберите Swift
- Testing System —— по умолчанию будет создана цель для тестирования (Unit Tests), на начальном этапе это можно игнорировать
- Storage —— фреймворк для локального хранения данных; может автоматически интегрировать SwiftData или Core Data, на начальном этапе это тоже можно игнорировать

После заполнения нажмите "Next".

### Сохранение проекта

![Swift](../../RESOURCE/001_xcode4.png)

Выберите подходящую папку для сохранения.

Нажмите кнопку "Create", и Xcode автоматически сгенерирует структуру проекта.

## Структура проекта Xcode

После сохранения проекта Xcode в Finder можно увидеть созданную папку проекта.

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets —— папка для хранения изображений, иконок приложения и других ресурсов

- ContentView.swift —— файл SwiftUI-представления, создаваемый по умолчанию; далее именно здесь мы будем писать код интерфейса

- SwiftSlimTestApp.swift —— входной файл приложения (App Entry Point), отвечающий за запуск приложения; пока что нам не нужно его подробно разбирать

- SwiftSlimTest.xcodeproj —— файл проекта Xcode; двойной щелчок по нему открывает проект

## Знакомство с интерфейсом Xcode

Дважды щёлкните по файлу .xcodeproj, чтобы открыть проект, затем в левой области Navigator выберите файл ContentView.swift.

Интерфейс Xcode в основном состоит из пяти областей:

- Navigator —— область навигации, используется для просмотра структуры файлов и поиска
- Editor —— область редактора, используется для написания кода Swift или SwiftUI
- Canvas —— область холста, используется для предварительного просмотра SwiftUI
- Inspector —— область инспектора свойств, используется для просмотра и изменения свойств файла
- Debug Area —— область отладки, используется для просмотра вывода журналов

![Swift](../../RESOURCE/001_xcode6.png)

Обратите внимание: Canvas в основном используется для предварительного просмотра SwiftUI-представлений. С точки зрения официальной архитектуры Xcode, это вспомогательная панель предварительного просмотра Editor (Preview Pane). Canvas можно включить только при открытии файла SwiftUI-представления; для других типов файлов функция предпросмотра недоступна.

В последующих уроках мы в основном будем писать код Swift и SwiftUI в области Editor.

При первом открытии проекта Xcode области Inspector и Debug Area по умолчанию могут быть скрыты; их можно отобразить, нажав кнопки в правом верхнем и правом нижнем углу.

![Swift](../../RESOURCE/001_xcode7.png)

Подсказка: область Inspector в основном используется для просмотра и изменения свойств файла. В реальной разработке её обычно скрывают, чтобы получить больше пространства для редактирования.
