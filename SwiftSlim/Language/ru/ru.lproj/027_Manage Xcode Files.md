# Управление файлами Xcode

В этом уроке мы изучим распространённые операции с файлами в Xcode.

В предыдущих уроках большую часть кода мы писали в `ContentView`. Так удобнее наблюдать за кодом и результатом отображения.

Например:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

Но в реальном App обычно не пишут весь код в одном файле.

Когда страниц, компонентов и функций становится всё больше, нам нужно разделять разные представления по разным файлам SwiftUI.

Например:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

Так структура кода становится понятнее, а последующее обслуживание и изменение — удобнее.

## Зачем управлять файлами

В SwiftUI одну страницу или один компонент обычно можно написать как отдельный файл представления.

Например:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

Здесь `CardView` — это представление SwiftUI.

Если все представления писать в `ContentView`, код будет становиться всё длиннее, и читать его будет всё сложнее.

Поэтому нам нужно научиться создавать, переименовывать и удалять файлы, а также использовать папки для управления разными файлами кода.

## Создание файла SwiftUI

В Xcode обычно есть два распространённых способа создать файл SwiftUI:

первый — создать пустой файл, а затем самостоятельно написать код.

второй — создать файл по шаблону SwiftUI, чтобы Xcode автоматически сгенерировал базовый код.

Для начинающих больше рекомендуется использовать файл шаблона SwiftUI, потому что он автоматически создаёт базовую структуру.

### Создание пустого файла

В области Navigator слева в Xcode щёлкните правой кнопкой по пустому месту или папке и выберите `New Empty File`.

Место действия показано ниже:

![xcode](../../../Resource/027_view1.png)

После создания пустого файла Xcode обычно создаёт безымянный Swift-файл, например `Untitled.swift`.

Мы можем переименовать его в `TestView.swift`.

В проекте SwiftUI, если этот файл используется для отображения представления, обычно рекомендуется добавлять `View` в конце имени.

Например:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

Это не обязательное правило, но оно позволяет сразу понять, что это файл представления.

После создания пустого файла в нём могут быть только стандартные комментарии, а может быть и пустое содержимое. Это зависит от версии Xcode и способа создания.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

В этот момент нам нужно вручную написать код SwiftUI:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

Так создаётся простой файл представления SwiftUI.

### Переименование файла

Если после создания файла нужно изменить его имя, можно выбрать файл в области Navigator слева в Xcode, а затем нажать клавишу Return `⏎`.

![xcode](../../../Resource/027_view2.png)

Так можно изменить имя файла.

Важно учитывать, что изменение имени файла не меняет автоматически имя представления в коде.

Например, после изменения имени файла на `ProfileView.swift` в коде всё ещё может оставаться `struct TestView: View`.

Хотя это не обязательно приведёт к ошибке, но легко может вызвать путаницу.

Поэтому рекомендуется сохранять согласованность между именем файла и именем представления. Например, если файл называется `ProfileView.swift`, имя представления тоже должно быть `ProfileView`.

## Создание файла шаблона SwiftUI

Помимо создания пустого файла, мы также можем напрямую создать файл шаблона SwiftUI.

В области Navigator слева в Xcode щёлкните правой кнопкой по пустому месту или папке и выберите `New File from Template...`.

Место действия показано ниже:

![xcode](../../../Resource/027_view3.png)

В появившемся окне сверху можно выбрать разные платформы, например `iOS`, `macOS` и другие.

Здесь достаточно оставить платформу, соответствующую текущему проекту. Например, если мы создаём iOS App, выбираем `iOS`.

![xcode](../../../Resource/027_view4.png)

Затем внизу выбираем `SwiftUI View`.

![xcode](../../../Resource/027_view12.png)

Затем нажимаем `Next`, вводим имя файла и сохраняем.

Например, мы создаём `Test2View.swift`.

Xcode автоматически сгенерирует код, похожий на приведённый ниже:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

По сравнению с пустым файлом, файл шаблона автоматически содержит `import SwiftUI`, `struct`, `body` и `#Preview`.

Поэтому для начинающих использование файла шаблона SwiftUI проще и снижает вероятность пропустить нужный код.

**Подсказка**: кроме создания файла через щелчок правой кнопкой, можно также нажать кнопку с плюсом внизу области Navigator слева в Xcode, чтобы создать файл или папку.

![xcode](../../../Resource/027_view5.png)

## Создание папки

Когда файлов становится всё больше, мы можем использовать папки для управления ими.

В области Navigator слева в Xcode щёлкните правой кнопкой по пустому месту или папке и выберите `New Folder`.

После создания папки можно перетащить в неё связанные файлы.

![xcode](../../../Resource/027_view7.png)

Папки в основном используются для упорядочивания структуры проекта и не меняют способ использования самого представления.

Если имя представления указано правильно, в других файлах его по-прежнему можно использовать таким образом: `TestView()`, `Test2View()`.

## Удаление файла или папки

Если нужно удалить файл или папку, можно выбрать соответствующий файл в области Navigator слева в Xcode, затем щёлкнуть правой кнопкой и выбрать `Delete`.

Также можно выбрать файл и нажать клавишу удаления `⌫`.

Xcode покажет окно подтверждения удаления.

![xcode](../../../Resource/027_view10.png)

Выбор `Move to Trash` означает перемещение файла в корзину.

Если вы видите `Remove Reference`, это означает, что из проекта Xcode будет удалена только ссылка, но исходный файл на диске не будет удалён.

Для начинающих, если вы просто удаляете обычный файл, который только что создали сами, обычно можно выбрать `Move to Trash`.

## Отображение нескольких представлений SwiftUI

После того как мы научились создавать файлы, мы можем разделять разные представления по разным файлам.

Например, мы создаём два файла SwiftUI:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` используется для отображения имени автора:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` используется для отображения фона карточки:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

Затем мы можем использовать эти два представления в `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

После запуска можно увидеть, что `CardView` и `AuthorView` отображаются вместе:

![xcode](../../../Resource/027_view11.png)

В этом результате `CardView` служит фоном, а `AuthorView` отображается сверху, потому что они помещены в `ZStack`.

В `ContentView` мы используем эти два представления через `CardView()` и `AuthorView()`. Скобки `()` после имени означают создание и использование этого представления.

Это и есть распространённое вложение представлений в SwiftUI: внутри одного представления можно продолжать использовать другие представления.

## Входной файл

В конце ещё раз вспомним, откуда App начинает отображать интерфейс.

В проекте SwiftUI обычно есть входной файл:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Здесь `@main` означает, что это входная точка App.

В `WindowGroup` мы видим:

```swift
ContentView()
```

Это означает, что после запуска App сначала отображает `ContentView`.

Затем внутри `ContentView` также отображается:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Поэтому иерархию представлений можно понимать так:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

Если мы изменим представление, отображаемое во входном файле, на `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

Тогда после запуска App будет отображать только `CardView`.

Иначе говоря, `WindowGroup` во входном коде определяет, какое представление первым отображается после запуска App.

А внутри этого представления можно продолжать отображать другие представления.

## Итоги

В этом уроке мы изучили распространённые операции управления файлами в Xcode.

Мы узнали, как создавать файлы SwiftUI, как переименовывать файлы, как создавать папки и как удалять файлы.

Также мы изучили важную концепцию: представления SwiftUI можно разделять по разным файлам, а затем использовать в других представлениях.

Например:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Здесь `CardView()` и `AuthorView()` — это использование других представлений внутри `ContentView`.

В конце мы также вспомнили логику отображения входного файла.

При запуске App сначала входит во входной код, помеченный `@main`, а затем отображает представление внутри `WindowGroup`.

Это помогает нам понять: разные файлы SwiftUI не изолированы друг от друга. Они могут комбинироваться между собой и в итоге формировать полноценный интерфейс App.
