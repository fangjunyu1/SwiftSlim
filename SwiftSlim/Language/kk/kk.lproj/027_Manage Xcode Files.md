# Xcode файлдарын басқару

Бұл сабақта біз Xcode ішіндегі жиі қолданылатын файл операцияларын үйренеміз.

Алдыңғы сабақтарда біз кодтың көп бөлігін `ContentView` ішінде жаздық. Бұл кодты және көрсету нәтижесін бақылауға ыңғайлы.

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

Бірақ нақты App ішінде барлық кодты бір файлға жазу әдетте дұрыс емес.

Беттер, компоненттер және функциялар көбейген сайын, әртүрлі көріністерді әртүрлі SwiftUI файлдарына бөлуіміз керек.

Мысалы:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

Бұл код құрылымын айқынырақ етеді және кейінгі қызмет көрсету мен өзгертуді жеңілдетеді.

## Неліктен файлдарды басқару керек

SwiftUI ішінде бір бет немесе бір компонент әдетте жеке көрініс файлы ретінде жазыла алады.

Мысалы:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

Мұндағы `CardView` — SwiftUI көрінісі.

Егер барлық көріністер `ContentView` ішінде жазылса, код барған сайын ұзарып, оқуға қиындай түседі.

Сондықтан біз файл жасау, атын өзгерту, жою және әртүрлі код файлдарын басқару үшін қалталарды пайдалануды үйренуіміз керек.

## SwiftUI файлын жасау

Xcode ішінде SwiftUI файлын жасаудың жиі қолданылатын екі тәсілі бар:

Біріншісі — бос файл жасау, содан кейін кодты өзіміз жазу.

Екіншісі — SwiftUI үлгі файлын жасау, мұнда Xcode негізгі кодты автоматты түрде жасайды.

Жаңадан бастаушыларға SwiftUI үлгі файлын қолдану көбірек ұсынылады, өйткені ол негізгі құрылымды автоматты түрде жасайды.

### Бос файл жасау

Xcode сол жағындағы Navigator аймағында бос орынға немесе қалтаға тінтуірдің оң жақ батырмасымен басып, `New Empty File` таңдаңыз.

Операция орны төмендегідей:

![xcode](../../../Resource/027_view1.png)

Бос файл жасалғаннан кейін, Xcode әдетте атаусыз Swift файлын жасайды, мысалы `Untitled.swift`.

Біз оның атын `TestView.swift` деп өзгерте аламыз.

SwiftUI жобасында, егер бұл файл бір көріністі көрсету үшін қолданылса, атаудың соңына әдетте `View` қосу ұсынылады.

Мысалы:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

Бұл міндетті ереже емес, бірақ файлдың көрініс файлы екенін бірден түсінуге көмектеседі.

Бос файл жасалғаннан кейін, файлда тек әдепкі түсініктеме болуы мүмкін немесе ол бос болуы мүмкін. Бұл Xcode нұсқасына және жасау тәсіліне байланысты.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

Бұл кезде SwiftUI кодын қолмен жазуымыз керек:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

Осылайша, қарапайым SwiftUI көрініс файлы жасалып бітеді.

### Файл атын өзгерту

Егер файлды жасағаннан кейін оның атын өзгерткіңіз келсе, Xcode сол жағындағы Navigator аймағынан файлды таңдап, `⏎` Enter пернесін басуға болады.

![xcode](../../../Resource/027_view2.png)

Осылай файл атын өзгертуге болады.

Назар аудару керек: файл атын өзгерту код ішіндегі көрініс атауын автоматты түрде өзгертпейді.

Мысалы, файл атауы `ProfileView.swift` болып өзгергеннен кейін, код ішінде әлі де `struct TestView: View` болуы мүмкін.

Бұл міндетті түрде қате тудырмауы мүмкін, бірақ шатастыруы оңай.

Сондықтан файл атауы мен көрініс атауын бірдей ұстау ұсынылады. Мысалы, файл атауы `ProfileView.swift` болса, көрініс атауы да `ProfileView` болсын.

## SwiftUI үлгі файлын жасау

Бос файл жасаудан бөлек, біз SwiftUI үлгі файлын тікелей жасай аламыз.

Xcode сол жағындағы Navigator аймағында бос орынға немесе қалтаға тінтуірдің оң жақ батырмасымен басып, `New File from Template...` таңдаңыз.

Операция орны төмендегідей:

![xcode](../../../Resource/027_view3.png)

Ашылған терезенің жоғарғы жағында `iOS`, `macOS` сияқты әртүрлі платформаларды таңдауға болады.

Мұнда қазіргі жобаға сәйкес платформаны таңдалған күйде қалдырсаңыз жеткілікті. Мысалы, біз iOS App жасап жатсақ, `iOS` таңдаймыз.

![xcode](../../../Resource/027_view4.png)

Содан кейін төменгі жақтан `SwiftUI View` таңдаңыз.

![xcode](../../../Resource/027_view12.png)

Содан кейін `Next` басып, файл атауын енгізіп сақтаңыз.

Мысалы, біз `Test2View.swift` файлын жасаймыз.

Xcode төмендегіге ұқсас кодты автоматты түрде жасайды:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

Бос файлмен салыстырғанда, үлгі файл автоматты түрде `import SwiftUI`, `struct`, `body` және `#Preview` қамтиды.

Сондықтан жаңадан бастаушылар үшін SwiftUI үлгі файлын пайдалану оңайырақ және кодты ұмытып кету ықтималдығы да төмен.

**Кеңес**: файлды тінтуірдің оң жақ батырмасымен жасаудан бөлек, Xcode сол жағындағы Navigator төменгі бөлігіндегі қосу батырмасын басып, файл немесе қалта жасауға болады.

![xcode](../../../Resource/027_view5.png)

## Қалта жасау

Файлдар көбейген кезде, оларды басқару үшін қалталарды қолдануға болады.

Xcode сол жағындағы Navigator аймағында бос орынға немесе қалтаға тінтуірдің оң жақ батырмасымен басып, `New Folder` таңдаңыз.

Қалта жасалғаннан кейін, байланысты файлдарды қалтаға сүйреп апаруға болады.

![xcode](../../../Resource/027_view7.png)

Қалталар негізінен жоба құрылымын реттеу үшін қолданылады, олар көріністің өзін пайдалану тәсілін өзгертпейді.

Көрініс атауы дұрыс болса, басқа файлдарда оны әлі де `TestView()` және `Test2View()` сияқты тәсілмен қолдануға болады.

## Файлды немесе қалтаны жою

Файлды немесе қалтаны жойғыңыз келсе, Xcode сол жағындағы Navigator аймағында тиісті файлды таңдап, тінтуірдің оң жақ батырмасымен басып `Delete` таңдаңыз.

Файлды таңдағаннан кейін, жою пернесін `⌫` басуға да болады.

Xcode жоюды растау терезесін шығарады.

![xcode](../../../Resource/027_view10.png)

`Move to Trash` таңдау файлды себетке жылжытуды білдіреді.

Егер `Remove Reference` көрсеңіз, бұл тек Xcode жобасынан сілтемені алып тастайды, бірақ дискідегі бастапқы файлды жоймайды.

Жаңадан бастаушылар үшін, егер өзіңіз жаңа ғана жасаған кәдімгі файлды жойып жатсаңыз, әдетте `Move to Trash` таңдау жеткілікті.

## Бірнеше SwiftUI көрінісін көрсету

Файл жасауды үйренгеннен кейін, әртүрлі көріністерді әртүрлі файлдарға бөле аламыз.

Мысалы, біз екі SwiftUI файлын жасаймыз:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` автор атын көрсету үшін қолданылады:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` карточка фонын көрсету үшін қолданылады:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

Содан кейін біз осы екі көріністі `ContentView` ішінде қолдана аламыз:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

Іске қосқаннан кейін, `CardView` және `AuthorView` бірге біріктіріліп көрсетілгенін көруге болады:

![xcode](../../../Resource/027_view11.png)

Бұл нәтижеде `CardView` фон ретінде, ал `AuthorView` оның үстінде көрсетіледі, өйткені олар `ZStack` ішінде орналастырылған.

`ContentView` ішінде біз осы екі көріністі `CardView()` және `AuthorView()` арқылы қолданамыз. Атаудан кейінгі `()` осы көріністі жасау және қолдануды білдіреді.

Бұл SwiftUI ішіндегі жиі кездесетін көрініс кірістіруі: бір көрініс ішінде басқа көріністерді қолдануға болады.

## Кіру файлы

Соңында, App интерфейсті қай жерден бастап көрсететінін қайта қарап шығайық.

SwiftUI жобасында әдетте бір кіру файлы болады:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Мұндағы `@main` бұл App кіру орны екенін білдіреді.

`WindowGroup` ішінде біз мынаны көреміз:

```swift
ContentView()
```

Бұл App іске қосылғаннан кейін алдымен `ContentView` көрсетілетінін білдіреді.

Содан кейін `ContentView` ішінде тағы мыналар көрсетіледі:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Сондықтан көрініс деңгейін былай түсінуге болады:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

Егер кіру файлында көрсетілетін көріністі `CardView` етіп өзгертсек:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

Онда App іске қосылғаннан кейін тек `CardView` көрсетіледі.

Яғни, кіру кодындағы `WindowGroup` App іске қосылғаннан кейін алдымен қай көрініс көрсетілетінін анықтайды.

Ал осы көріністің ішінде басқа көріністерді әрі қарай көрсетуге болады.

## Қорытынды

Бұл сабақта біз Xcode ішіндегі жиі қолданылатын файл басқару операцияларын үйрендік.

Біз SwiftUI файлын қалай жасау, файл атын қалай өзгерту, қалтаны қалай жасау және файлды қалай жою керегін түсіндік.

Сонымен қатар, біз маңызды бір ұғымды үйрендік: SwiftUI көріністерін әртүрлі файлдарға бөліп, кейін басқа көріністер ішінде қолдануға болады.

Мысалы:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Мұндағы `CardView()` және `AuthorView()` — `ContentView` ішінде басқа көріністерді пайдалану.

Соңында, біз кіру файлының көрсету логикасын да қайта қарап шықтық.

App іске қосылғанда, алдымен `@main` белгісі бар кіру кодына кіреді, содан кейін `WindowGroup` ішіндегі көріністі көрсетеді.

Бұл бізге мынаны түсінуге көмектеседі: әртүрлі SwiftUI файлдары оқшау емес, олар бір-бірімен бірігіп, соңында толық App интерфейсін құрай алады.
