# Фильмдер тізімі

Бұл сабақта біз фильмдер тізімін жасаймыз.

![movie](../../Resource/021_movie.png)

Тізімде әр фильмнің постері, атауы, режиссері және бағасы көрсетіледі.

Бұл мысалда біз өте маңызды бір ұғымды үйренеміз: `struct` құрылымы. Ол бір фильмге қатысты бірнеше ақпаратты біртұтас етіп біріктіруге көмектеседі. Бұдан бөлек, біз `UUID()`, `ForEach`, `Divider` бөлгіш сызығымен және деректерді басқару үшін арнайы объектілерді қалай қолдануға болатынымен де танысамыз.

Бұл білімдер кейінгі SwiftUI әзірлеуінде өте жиі кездеседі. Осы сабақты аяқтағаннан кейін, сен тек фильмдер тізімін жасап қана қоймай, «деректер тобын интерфейс түрінде қалай көрсетуге болатынын» да түсіне бастайсың.

## Бір фильм

Алдымен бір фильмнің интерфейсінен бастап көрейік.

![movie](../../Resource/021_movie1.png)

Бұл интерфейстің орналасуы негізінен екі бөліктен тұрады: сол жақта фильмнің постері, ал оң жақта фильм туралы таныстырылым.

### Фильм постері

Сол жақта фильмнің постері көрсетіледі, ал суретті көрсету үшін `Image` пайдалануға болады.

Мысалы:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

Бұл код `"The Shawshank Redemption"` деп аталатын суретті көрсететінін білдіреді.

Мұнда назар аударатын нәрсе: сурет атауы `Assets` ресурстар қалтасында шын мәнінде болуы керек, әйтпесе интерфейс бұл суретті дұрыс көрсете алмайды.

![movie](../../Resource/021_movie2.png)

Бұл модификаторлардың қызметі мыналар:

- `resizable()` суреттің өлшемін өзгертуге болатынын білдіреді.
- `scaledToFit()` масштабтау кезінде бастапқы пропорцияны сақтайды, осылайша сурет созылып, бұрмаланбайды.
- `frame(height: 180)` суреттің биіктігін 180 етіп орнатады.
- `cornerRadius(10)` суретке 10 радиусты дөңгелек бұрыш береді.

Осылайша біз өлшемі қолайлы, бұрыштары дөңгеленген фильм постерін аламыз.

![movie](../../Resource/021_movie3.png)

### Фильм туралы таныстырылым

Оң жақта фильм туралы таныстырылым көрсетіледі, оған фильм атауы, режиссері және бағасы кіреді.

![movie](../../Resource/021_movie4.png)

Бұл бөлікті көрсету үшін `Text` қолдануға болады:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Бұл кодта `.font` қаріп өлшемін, ал `.fontWeight` қаріптің қалыңдығын білдіреді.

«Фильм атауы», «режиссер туралы ақпарат» және «баға туралы ақпарат» арасындағы арақашықтықты анығырақ ету үшін, мұнда ең сыртқы қабатта `VStack` қолданылып, мынадай параметр беріледі:

```swift
spacing: 10
```

Бұл осы `VStack` ішіндегі әрбір мазмұн тобының арасындағы аралық 10 дегенді білдіреді.

Режиссер мен бағаның әрқайсысы бөлек тағы бір `VStack` ішіне оралғанының себебі, екеуі де «тақырып + мазмұн» құрылымына жатады. Сондықтан интерфейстің қабаттары анығырақ болады және кейін стильді баптау да оңайырақ болады.

Сонымен қатар, `VStack` әдепкіде ортаға тураланады. Барлық мәтінді сол жаққа туралау үшін біз мынаны қоямыз:

```swift
alignment: .leading
```

Осылайша фильм туралы таныстырылым бөлігі ұқыптырақ көрінеді.

### Бөлгіш сызық

Қазір фильм туралы таныстырылымдағы мазмұн `spacing` арқылы бір-бірінен алыстатылғанымен, әртүрлі мазмұн арасындағы шекара әлі де жеткілікті айқын емес.

Осы кезде біз бөлгіш сызық қоса аламыз:

```swift
Divider()
```

Мысалы:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Осылайша фильм туралы таныстырылым бөлігінде айқынырақ бөліну әсері пайда болады.

![movie](../../Resource/021_movie4.png)

`Divider` өте қарапайым, бірақ өте жиі қолданылатын view, оның қызметі әртүрлі мазмұнды бөліп көрсету.

`VStack` ішінде `Divider()` көлденең сызық ретінде көрінеді.

`HStack` ішінде `Divider()` тік сызық ретінде көрінеді.

Бұдан бөлек, `.frame`, `.background`, `.padding` және басқа тәсілдер арқылы `Divider` стилін де өзгертуге болады.

Мысалы:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

Бұл қалыңдығы 2 болатын, көк түсті және екі жағында көлденең ішкі аралығы бар бөлгіш сызық жасайды.

![divider](../../Resource/021_divider.png)

### Толық интерфейс

Соңында, `HStack` арқылы фильм постері мен фильм туралы таныстырылымды қатарластырып орналастыра аламыз.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

Мұндағы `HStack(spacing: 20)` сол және оң жақ бөліктердің арасында 20 аралық қалатынын білдіреді.

Айта кету керек, `Divider()` алдыңғы айтылған `TextField` және `Slider` сияқты әдепкіде қолжетімді кеңістікті мүмкіндігінше көбірек алуға тырысады.

Сондықтан оң жақтағы таныстырылым бөлігінің сыртқы `VStack`-іне тұрақты ен береміз:

```swift
.frame(width: 200)
```

Осылайша бөлгіш сызық пен мәтін аймағының ені бірдей болып, жалпы көрініс ұқыптырақ болады.

Осы жерге дейін біз «бір фильмнің» көрсетілім интерфейсін аяқтадық.

![movie](../../Resource/021_movie1.png)

## Фильмдерді массивте сақтау

Егер біз бірнеше фильмді бірдей стильмен бірінен кейін бірін көрсеткіміз келсе, бұл әр фильм үшін ұқсас кодты қайтадан жазу керек дегенді білдіреді.

Мысалы:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

Мұндай жазылған код қажетті нәтижеге жеткізе алғанымен, код көлемі барған сайын көбейеді және оны қолдап отыру өте қиын болады.

Мысалы, барлық фильмдердің арақашықтығын `20`-дан `15`-ке өзгерту керек болса немесе оң жақ бөліктің енін `200`-ден
`220`-ға өзгерту керек болса, әрбір қайталанатын код бөлігін қолмен өзгертуге тура келеді.

Бұл, әрине, тиімді әдіс емес.

Бұған дейін біз массивтерді үйрендік, сондай-ақ `ForEach` арқылы массивке сүйеніп view-ларды қайталап көрсету тәсілін де үйрендік.

Мысалы:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Бұл тәсіл «тек бір ғана дерек» болатын жағдайларға өте қолайлы, мысалы, сурет атауларының тобы.

Бірақ бір фильмде тек бір мән ғана болмайды. Ол кем дегенде мыналарды қамтиды:

1. фильм постері
2. фильм атауы
3. режиссер
4. баға

Яғни, бір фильм шын мәнінде өзара байланысты деректер жиыны, жеке бір string емес.

Егер тек массивтерді қолдансақ, бұл ақпаратты тек бөлек-бөлек сақтай аламыз:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

Көрсету кезінде оларды бір-бірімен сәйкестендіру үшін бірдей индекске сүйенуіміз керек:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

Бұл тәсілдің мәселесі - оны қолдау құны өте жоғары.

Себебі бір фильмнің деректері бірнеше массивке бөлініп кетеді. Сол массивтердің біреуінде артық бір элемент, кем бір элемент немесе басқа реттілік болса, көрсетілім нәтижесі қате болады.

Әсіресе, фильм қосқанда немесе өшіргенде, бірнеше массивті бір уақытта өзгерту керек болады, сондықтан бір нәрсені ұмытып кету өте оңай.

Ендеше, «фильм атауы, режиссер және баға» сияқты ақпараттарды біртұтас етіп қалай біріктіруге болады?

Бұл үшін `struct` керек.

## struct құрылымын анықтау

Swift тілінде `struct` - «құрылым».

Оны бірнеше байланысты өрістерді біріктіріп, біртұтас ететін «арнайы дерек түрі» деп түсінуге болады.

Негізгі қолдану:

```swift
struct StructName {
	let name: String
}
```

Бұл кодта `struct` - біз құрылым анықтап жатқанымызды білдіретін кілтсөз. `StructName` - құрылымның атауы.

Қисық жақшаның ішіндегі мазмұн - осы құрылымда болатын өрістер, және әр өрістің атауы мен түрі анық жазылуы тиіс.

Әдетте `struct` атауы бас әріптен басталады, мысалы `Movie`, `Student`, `UserInfo`. Бұл Swift тіліндегі кең таралған атау беру тәсілі.

`struct`-ты қарапайым түрде ішіндегі әр өрісі алдын ала бөлінген бос орын сияқты болатын бос қорап деп елестетуге болады.

Бұл орындар әлі толтырылмаған кезде ол жай бос қорап қана болады. Барлық өрістер тиісті мәндермен толтырылғанда ғана ол толық сыйлық қорабына айналады.

### struct данасы

Бұған дейін біз тек құрылымның өзін анықтадық, бұл қораптың пішінін дайындаумен бірдей.

Енді оның ішіне нақты мазмұнды толтыруымыз керек, сонда ғана қолдануға болатын шынайы «дана» аламыз.

Дана жасағанда әдетте құрылым атауынан кейін `()` қойылады:

```swift
StructName(...)
```

Жақшаның ішіне осы құрылымға қажет өріс мәндері жазылады.

Мысалы:

```swift
StructName(name: "Fang Junyu")
```

Бұл код `StructName` құрылымының пішіні бойынша жаңа дана жасап, `name` өрісіне `"Fang Junyu"` мәнін беруді білдіреді.

Қажетті өрістердің барлығын толтырған кезде, бұл сол қорапты толық толтырғанмен бірдей болады.

Осы кезде біз толық struct данасын аламыз.

### struct қасиеттеріне қол жеткізу

Дана жасалғаннан кейін, оның ішкі қасиеттеріне «нүктелік синтаксис» арқылы қол жеткізе аламыз.

Негізгі жазылуы:

```swift
дана.қасиетАтауы
```

Мысалы:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

Мұнда `st` - struct данасы, ал `st.name` осы дананың `name` мәнін оқуды білдіреді.

Мұны былай түсінуге болады: `st.name` деп жазғанда, біз сол қораптың белгілі бір орнында сақталған мазмұнды оқып тұрғандай боламыз.

Мұндай жазу тәсілі кейінгі SwiftUI интерфейстерінде өте жиі кездеседі.

Мысалы, егер бізде аты, жасы және сыныбы бар student құрылымы болса, оны интерфейсте көрсету кезінде бөлек-бөлек оқи аламыз:

```swift
student.name
student.age
student.className
```

Мұның артықшылығы - деректер анық әрі басқаруға ыңғайлы болады.

### struct орны

Жаңадан үйренушілер үшін құрылымды көбіне `ContentView`-ден тыс жерде жазуға болады.

Мысалы:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

Осылайша құрылым анығырақ болады және оқу да жеңілдейді.

Сен `ContentView`-дің өзі де шын мәнінде `struct` екенін байқайсың.

### Movie құрылымы

Алдыңғы негіз арқылы енді фильм құрылымын анықтай аламыз:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

Бұл `Movie` деп аталатын құрылым, ол `name`, `director` және `rating` атты үш өрістен тұрады, олар сәйкесінше фильм атауын, режиссерді және бағаны білдіреді.

Осылайша бір фильмге қатысты ақпаратты бірнеше массивке бөліп сақтаудың қажеті болмайды, оларды бір бүтінге біріктіруге болады.

Мысалы, мынадай фильм данасын жасай аламыз:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

Бұл дана «бір фильмге арналған толық деректерді» білдіреді.

### Құрылымдарды массивте сақтау

Бұған дейін массив тек бір типті ғана сақтай алатынын айтқанбыз.

Енді бізде `Movie` құрылымы бар, сондықтан массив бірнеше `Movie` сақтай алады.

```swift
let lists: [Movie] = []
```

Мысалы:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Бұл `lists` массивінде string емес, бірнеше `Movie` данасы сақталады.

Яғни, массивтегі әрбір элемент - толық бір фильм.

Осылайша фильмдер тізімін көрсету керек болғанда, `ForEach` арқылы фильмдерді бір-бірлеп оқи аламыз.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

Мұндағы `movie in` әрбір айналымда массивтен бір фильм алынып, уақытша `movie` деп аталатынын білдіреді.

Содан кейін біз:

```swift
movie.name
movie.director
movie.rating
```

арқылы осы фильмнің атауын, режиссерін және бағасын оқып, интерфейсте көрсете аламыз.

## ForEach қатесі

Осы жерге дейін біз фильмдер массиві мен `ForEach`-тің негізгі жазылуын аяқтадық.

Алайда, төмендегі кодты тікелей іске қоссақ:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

қате шығады:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

Бұл қатенің мағынасы: `id: \.self` деп жазғанда, SwiftUI массивтің әр элементінің өзін «бірегей идентификатор» ретінде қолдануы керек.

Бұл жазылу `String` және `Int` сияқты қарапайым типтерге қолайлы, өйткені оларды бір-бірінен ажырату оңайырақ.

Бірақ `Movie` - біз өзіміз анықтаған арнайы құрылым, ал `ForEach` оның өзін бірегей идентификатор ретінде қалай қолдану керегін білмейді, сондықтан қате пайда болады.

Бұл мәселені шешудің ең кең таралған жолы - `Movie` құрылымын `Identifiable` протоколына сәйкестендіру.

## Протокол

Swift тілінде протоколды «ереже» немесе «талап» ретінде түсінуге болады.

Оны халықаралық ұйымдағы келісім сияқты елестетуге болады: егер бір ел белгілі бір халықаралық ұйымға қосылғысы келсе, әдетте алдымен сол ұйым қойған кейбір ережелерді орындап, кейбір деректерді жариялап, ортақ ережелерді сақтау керек. Сол талаптар орындалғаннан кейін ғана ол ұйымға қосыла алады немесе кейбір істерге қатыса алады.

Протоколдар да осыған ұқсас қағидамен жұмыс істейді.

Қандай да бір тип белгілі бір протоколға сай болғысы келсе, ол сол протокол талап ететін мазмұнды орындауы керек. Осы талаптар орындалғаннан кейін ғана ол тип тиісті функцияларды пайдалана алады.

Негізгі жазылуы:

```swift
struct Movie: Identifiable {
    // ...
}
```

Мұндағы `: Identifiable` `Movie` құрылымы `Identifiable` протоколына сәйкес келетінін білдіреді.

Егер бір тип бірнеше протоколға сай келсе, оларды үтірмен бөлуге болады:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

`Identifiable` үшін ең маңызды талап - типтің өзін анықтай алатын `id` болуы.

Мысалы:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

Мұндағы `id` жеке куәлік нөмірі сияқты, ол әр фильмді ажырату үшін пайдаланылады.

Әр фильмнің `id`-і әртүрлі болғанда, SwiftUI массивтегі әр элементті дұрыс ажырата алады.

Мысалы:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Содан кейін `ForEach` ішінде мынаны қолданамыз:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

Осылайша қате болмайды.

`Movie` бұрыннан `Identifiable` протоколына сәйкес келгендіктен, жиірек қолданылатын жазылу - `id:` бөлігін мүлде алып тастау:

```swift
ForEach(lists) { movie in
    // ...
}
```

Өйткені SwiftUI әр элементті ажырату үшін `movie.id` қолданылатынын онсыз да біледі.

## ForEach ішіндегі ажыратқыш идентификатор

Енді біз `ForEach` ішіндегі `id` рөлін тереңірек түсінеміз.

Мысалы:

```swift
ForEach(lists, id: \.self)
```

Бұл элементтің өзін ажыратқыш идентификатор ретінде қолдануды білдіреді.

Ал:

```swift
ForEach(lists, id: \.id)
```

элементтің `id` өрісін ажыратқыш идентификатор ретінде қолдануды білдіреді.

Егер қандай да бір өріс өзі-ақ бірегейлікті қамтамасыз ете алса, оны уақытша пайдалануға болады.

Мысалы, егер әр фильмнің атауы бөлек болса, онда төмендегі жазылу қазіргі деректерде де дұрыс жұмыс істеуі мүмкін:

```swift
ForEach(lists, id: \.name)
```

Бірақ мұнда жасырын бір мәселе бар: `name` әрқашан бірегей бола бермейді.

Егер кейін дәл сондай атауы бар тағы бір фильм қоссаң, `name` әр элементті нақты ажырата алмай қалады.

Ондай жағдайда код компиляциядан өткенмен, view жаңартылғанда, енгізілгенде немесе өшірілгенде SwiftUI элементтерді қате танып, көрсетілімде ауытқулар тудыруы мүмкін.

Сондықтан деректерді тұрақты түрде ажырату қажет болғанда, шын мәнінде бірегей өрісті, яғни `id`-ті қолданған дұрыс.

## UUID

`id: 1`, `id: 2` деп қолмен жазу мәселені шеше алады, бірақ әлі де бір қауіп бар: байқаусызда бірдей `id` жазып қоюың мүмкін, сонда бірегей идентификатор жарамсыз болып қалады.

Мысалы:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Осылайша екі бірдей `id` пайда болып, бірегей идентификатор күшін жоғалтады.

Қолмен жіберілетін қателерді болдырмау үшін біз әдетте `UUID()` қолданамыз.

Swift тілінде `UUID()` кездейсоқ 128 биттік идентификатор жасайды. Ол әдетте әріптер мен сандардан тұратын ұзын жол түрінде көрінеді.

Мысалы:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

Мұндағы `UUID()` шын мәнінде `UUID` типінің данасын жасайды; оны автоматты түрде жасалатын «бірегей нөмір» ретінде түсінуге болады.

Сондықтан әр жасалған мән іс жүзінде қайталанбайды деуге болады, сол себепті оны `id` ретінде қолдану өте ыңғайлы.

`Movie` құрылымын мынадай етіп өзгерте аламыз:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Бұл код `Movie` құрылымындағы `id` өрісіне әдепкіде жаңа `UUID` данасы берілетінін білдіреді.

Яғни, әр жаңа `Movie` құрғанда, жүйе бізге автоматты түрде бірегей `id` жасап береді.

`id`-де әдепкі мән бар болғандықтан, кейін `Movie` даналарын жасағанда `id`-ті қолмен толтырудың қажеті болмайды.

Мысалы:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Мұнда `id` жазылмағанымен, іс жүзінде бұл фильм данасының өз `id`-і бар; тек бұл мәнді `UUID()` автоматты түрде жасап қойған.

Осы әдісті қолданғаннан кейін әр фильм үшін `id`-ті қолмен толтырудың қажеті болмайды. Бұл код көлемін азайтып қана қоймай, қайталанатын id салдарынан болатын қателердің алдын алады.

Соңында, жоғары жағына `Banner` суретін қосып, мазмұнды айналдырып көрсету үшін `ScrollView` қолдансақ, бүкіл фильмдер тізімі view-ы дайын болады.

## Қорытынды

Бұл сабақта біз өте маңызды бір ұғымды үйрендік: `struct`.

`struct` арқылы бір фильмге қатысты бірнеше байланысты өрісті бір бүтінге біріктіре аламыз, яғни атау, режиссер және бағаны бірнеше массивке бөлек сақтамаймыз.

Деректер біріктірілгеннен кейін, `ForEach` арқылы массивтегі әр фильмді бір-бірлеп view ішінде көрсете аламыз.

Сонымен бірге, `ForEach` ішіндегі «ажыратқыш идентификатордың» рөлін де түсіндік: SwiftUI view-ды дұрыс көрсетуі және жаңартуы үшін массивтегі әр элементті қалай ажырату керегін білуі керек.

Сондықтан `Movie` құрылымын `Identifiable` протоколына сәйкестендіріп, оған бірегей `id` бердік.

`id`-ті қолмен толтырған кездегі қателерден аулақ болу үшін, `UUID()`-ты да үйрендік, сонда жүйе бірегей идентификаторды автоматты түрде жасайды.

Осы сабақты аяқтағаннан кейін сен тек фильмдер тізімін жасап қана қоймай, SwiftUI-дегі маңызды бір ойлау тәсіліне де жақындайсың: **алдымен деректерді ретте, содан кейін view-ды сол деректерге сүйеніп құр.**

## Толық код

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
