# ContentView-ден бастау

## Сабаққа дейінгі дайындық

Бұл сабақта біз `ContentView.swift` файлынан бастап, SwiftUI-дің базалық құрылымын жүйелі түрде танысамыз. Оған мыналар кіреді:

- түсіндірме комментарийлер
- View құрылымы
- VStack / HStack / ZStack орналасуы
- SF Symbols белгішелері
- view modifier-лері
- `#Preview` алдын ала қарау коды

Алдымен бұрын жасаған Xcode жобаңызды тауып, `.xcodeproj` файлын екі рет шертіңіз.

Содан кейін сол жақтағы Navigator аймағынан `ContentView.swift` файлын таңдаңыз.

Ескерту: жобаны әр ашқан сайын Canvas ішінде `"Preview paused"` көрінуі мүмкін. Алдын ала қарауды қалпына келтіру үшін жаңарту батырмасын басыңыз.

![Swift](../../RESOURCE/002_view7.png)

## ContentView-пен танысу

ContentView коды:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Код көп емес, бірақ ол SwiftUI-дің негізгі құрылымын қамтиды.

### 1. Түсіндірме комментарий

Файлдың жоғарғы бөлігі:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

Бұл — файл туралы ақпаратты жазатын түсіндірме комментарий. Онда файл атауы, жоба атауы, автор және жасалған уақыты көрсетілген.

Swift-те бір жолдық комментарий `//` арқылы жазылады:

```swift
// Бұл комментарий
```

Комментарий кодтың оқылымдылығын арттырып, әзірлеушіге логиканы түсінуге көмектеседі.

Нақты әзірлеуде код кейде күрделі болып көрінуі мүмкін. Егер комментарий жазылмаса, үш күннен кейін өз кодыңызды өзіңіз түсінбей қалуыңыз мүмкін.

Сондықтан код жазу барысында түсіндірме комментарийлерді белсенді түрде қосу — жақсы әдет. Олар код логикасын жазып қалдыруға және кейін техникалық қолдауды жеңілдетуге көмектеседі.

**Кодты уақытша өшіру**

Комментарий кодты уақытша өшіру үшін де пайдаланылады, бұл мәселені іздеуге көмектеседі.

Мысалы:

```
A
B
C
```

A, B, C — үш бөлек код бөлігі. Соның біреуі қате беріп тұр делік. Онда комментарий арқылы кезекпен тексеруге болады.

Алдымен A-ны комментарийге айналдырамыз:

```
// A
B
C
```

Егер A-ны сөндіргеннен кейін код дұрыс жұмыс істесе, мәселе A бөлімінде деген сөз.

Егер мәселе сақталса, кейін B-ні де комментарийге айналдырып тексереміз, осылай жалғастырып, ақаулы кодты табамыз.

Әзірлеу барысында түрлі мәселе жиі кездеседі, және олардың көбін кодты уақытша комментарийлеу арқылы талдаймыз. Бұл мәселе орналасқан кодты тауып, BUG-ті анықтауға көмектеседі.

Xcode ішінде мына жылдам пернені пайдалануға болады:

```
Command ⌘ + /
```

Ол комментарийді тез қосуға немесе өшіруге мүмкіндік береді.

### 2. SwiftUI фреймворкін импорттау

```swift
import SwiftUI
```

Бұл код SwiftUI фреймворкін импорттайтынын білдіреді.

SwiftUI ішіндегі `View`, `Text`, `Image`, `VStack` сияқты типтердің барлығы осы фреймворктен келеді.

Егер SwiftUI фреймворкін импорттамасаңыз, Xcode қате көрсетеді:

```
Cannot find type 'View' in scope
```

Бұл компилятор `View` типін тани алмай тұрғанын білдіреді.

### 3. View құрылымы

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Мұндай View құрылымын алғаш көргенде тосын болып көрінуі мүмкін, өйткені мұнда `struct`, `View`, `var`, `body`, `some` сияқты кілт сөздер бар.

Біз бұл кілт сөздерді әлі толық оқыған жоқпыз. Қазір тек мына мағынаны түсінсек жеткілікті: бұл код `ContentView` деп аталатын View жасайды.

View-ді сурет салатын кенеп ретінде елестетуге болады, ал сол кенепке сурет салу құралы — SwiftUI.

Мысалы:

![Swift](../../RESOURCE/002_view.png)

Жоғарыдағы суретте үш бет көрсетілген, негізінде бұлардың әрқайсысы — жеке View.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

SwiftUI арқылы App жасағанда, әрбір бет жеке View болады.

### 4. SwiftUI коды

View ішінде SwiftUI коды көрсетіледі:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Бұл SwiftUI коды тік орналасқан контейнер ішінде белгіше мен мәтінді көрсететінін білдіреді.

![Swift](../../RESOURCE/002_view1.png)

#### VStack орналасуы

```swift
VStack { }  // тік орналасу
```

`VStack` — тік орналасу контейнері. Оның ішіндегі view-лер жоғарыдан төмен қарай реттеледі.

![Swift](../../RESOURCE/002_view8.png)

SwiftUI-де жиі қолданылатын үш орналасу түрі:

- `VStack` —— тік реттеу
- `HStack` —— көлденең реттеу
- `ZStack` —— қабаттап реттеу (Z осі)

```swift
HStack { }  // көлденең реттеу
ZStack { }  // қабаттап реттеу
```

Орналасу түрлерінің сызбасы:

![Swift](../../RESOURCE/002_view2.png)

Мысалы, `HStack` арқылы көлденең реттеу:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Мұнда жер шары белгішесі мен мәтін көлденең орналасады.

![Swift](../../RESOURCE/002_view3.png)

Көлденең реттеу керек болса `HStack`, қабаттап орналастыру қажет болса `ZStack` қолданылады.

#### Image және SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Бұл код жер шары белгішесін көрсетеді, ал белгіше Apple-дің SF Symbols жүйесінен алынады.

![Swift](../../RESOURCE/002_view9.png)

Кодтың мағынасы: үлкен өлшемдегі, акцент түсті жер шары белгішесін көрсету.

Біз тек жер шарын ғана емес, басқа да белгішелерді көрсете аламыз.

Мысалы, рюкзак белгішесін көрсету:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**Басқа белгішелерді қалай көрсетуге болады?**

Ол үшін Apple ұсынатын жүйелік белгіше кітапханасы SF Symbols қажет.

Apple Developer ресми сайтынан [SF Symbols](https://developer.apple.com/sf-symbols/) жүктеп алыңыз.

![Swift](../../RESOURCE/002_sf.png)

SF Symbols қолданбасын ашыңыз.

![Swift](../../RESOURCE/002_sf1.png)

Сол жақта — белгіше санаттары, оң жақта — соған сәйкес белгішелер.

Белгішені оң жақ батырмамен басып, `"Copy 1 Name"` таңдаңыз. Атау мәтіндік жол түрінде көшіріледі.

Мысалы:

```
"globe"
"backpack"
"heart"
```

Көшірілген белгіше атауын `Image(systemName:)` ішіне қойсаңыз, басқа белгішені көрсете аласыз.

Ескерту: әрбір SF Symbols белгішесінің қолдайтын ең төменгі жүйе нұсқасы болады. Егер жүйе нұсқасы тым ескі болса, белгіше көрсетілмеуі мүмкін. Оның үйлесімділік ақпаратын SF Symbols қолданбасынан көруге болады.

#### Modifier

SwiftUI-де modifier — view-дің сыртқы көрінісін немесе мінез-құлқын өзгертетін әдіс.

Modifier-ді киімге ұқсатуға болады: әртүрлі киім киген сайын, сыртқы түрі де өзгереді.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Мұндағы `imageScale` пен `foregroundStyle` — `Image` view-інің modifier-лері. Олар `Image` мазмұнын өзгертпей, оның көрінісін безендіреді.

**1. imageScale**

```swift
.imageScale(.large)
```

Ол SF Symbols белгішесінің өлшемін басқара алады:

- `.small`
- `.medium`
- `.large`

![Swift](../../RESOURCE/002_view11.png)

Әртүрлі нұсқаны таңдап, SF Symbols белгішесінің өлшемі қалай өзгеретінін байқауға болады.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

`foregroundStyle` алдыңғы түсті басқарады.

`.tint` ағымдағы ортаның акцент түсін білдіреді, әдепкі бойынша iOS-та ол көк түсті болады.

Егер алдыңғы түсті қызылға өзгерткіміз келсе:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Text view

`Text` — мәтінді көрсетуге арналған view.

```swift
Text("Hello, world!")
```

Мысалы, өз атымды көрсету:

```swift
Text("FangJunyu")
```

Ескерту: мәтіндік жол міндетті түрде `""` қос тырнақшаның ішінде жазылуы керек.

Сіз өз атыңызды, телефон нөміріңізді немесе басқа мәтінді көрсетіп көре аласыз.

#### padding шегінісі

SwiftUI-де `padding` view мазмұны мен шекарасының арасындағы бос кеңістікті қосу үшін пайдаланылады. Ол "ішкі шегініс (padding / content inset)" болып саналады.

```swift
HStack {
    ...
}
.padding()
```

Жоғарыдағы код `HStack` view-іне жүйелік әдепкі шегініс қосатынын білдіреді.

**padding деген не?**

`padding` — "view мазмұны мен оның шекарасы арасындағы бос аймақ".

Төмендегі суретте көк түсті `HStack`-ке `padding` берілгеннен кейін, көк аймақ ішке қарай жиырылып, бір шеңберге кішірейгендей көрінеді.

![Swift](../../RESOURCE/002_view6.png)

**Әдепкі шегініс**

`padding()` modifier-і әдепкі түрде жүйе ұсынған стандарт арақашықтықты пайдаланады.

```swift
.padding()
```

Бұл мән әртүрлі платформа мен контексте сәл өзгеше болуы мүмкін, мысалы:

- iOS-та әдетте шамамен 16 pt
- macOS немесе watchOS-та жүйелік стандарт арақашықтық басқа болуы мүмкін, ол әр платформаның дизайн ережесіне байланысты

**Теңшелетін шегініс**

View үшін жеке шегініс те орнатуға болады.

1. Бір бағытты орнату

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

Бағыттардың мағынасы:

- `.top`: жоғарғы шегініс
- `.bottom`: төменгі шегініс
- `.leading`: алдыңғы жақ шегініс
- `.trailing`: артқы жақ шегініс

![Swift](../../RESOURCE/002_view12.png)

Ескерту: `leading` пен `trailing` тіл бағытына қарай автоматты түрде бейімделеді. Мысалы, араб тілі (RTL) ортасында олар автоматты түрде ауысады.

2. Бірнеше бағытты орнату

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

Мұнда массив арқылы бірнеше бағытты бірден көрсетуге болады. Массивтің нақты қолданысын кейінгі сабақтарда толығырақ қарастырамыз, әзірге осы жазылу формасын біліп жүру жеткілікті.

3. Көлденең немесе тік бағытты орнату

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

Мынаған тең:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**Шегіністі алып тастау**

Егер ешқандай шегініс керек болмаса, `.padding(0)` арқылы алып тастауға болады:

```swift
.padding(0)
```

Немесе `padding` modifier-ін мүлде өшіруге болады:

```swift
// .padding()
```

## Қорытынды

`ContentView.swift` файлының коды көп болмаса да, іс жүзінде SwiftUI-дің бірнеше негізгі ұғымын қамтиды. Жаңадан бастаушылар үшін бұл код бастапқыда тосын көрінуі мүмкін, бірақ оны бөліп қарастырсақ, SwiftUI туралы алғашқы түсінік қалыптастыруға болады.

Осы сабақта біз алдымен `//` комментарийін үйрендік. Ол код логикасын түсіндіруге немесе кодты уақытша өшіруге көмектеседі.

Кейін SwiftUI файлы міндетті түрде SwiftUI фреймворкін импорттауы керек екенін көрдік:

```swift
import SwiftUI
```

Егер фреймворк импортталмаса, компилятор `View` сияқты типтерді тани алмайды.

Содан кейін SwiftUI view-інің негізгі құрылымымен таныстық:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Мұнда `ContentView` — view атауы.

Сонымен қатар үш жиі қолданылатын орналасу контейнерін үйрендік: `VStack` (тік реттеу), `HStack` (көлденең реттеу) және `ZStack` (қабаттап реттеу).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Бұл код тік орналасқан контейнер ішінде белгіше мен мәтінді көрсетеді.

`Image` SF Symbols белгішесін көрсете алады, ал modifier-лер арқылы оның өлшемі мен түсін өзгертуге болады.

`Text` view-і мәтінді көрсетеді.

`padding` — view айналасына мөлдір бос кеңістік қосатын шегініс.

Соңында `#Preview` preview view-ін қарастырдық, ол Canvas ішінде алдын ала қарауды көрсетеді.

### Сабақтан кейінгі жаттығу

Жаңадан бастаушылар үшін бұл сабақтағы материал сәл күрделі болып көрінуі мүмкін. Оны жақсырақ түсіну үшін қосымша жаттығу жасап көріңіз:

- SF Symbols белгішесінің атауын өзгерту
- Белгішенің алдыңғы түсін қара түске өзгерту
- `VStack`-ті `HStack`-ке ауыстыру
- `Image` немесе `Text`-ті комментарийге айналдырып, preview-дің қалай өзгеретінін бақылау

### Бонус: кодты толықтыру (Code Completion)

Код жазу кезінде сіз Xcode қолжетімді нұсқалардың тізімін автоматты түрде көрсететінін байқаған шығарсыз.

Мысалы, `imageScale` modifier-ін өзгерткен кезде:

```swift
.imageScale(.)
```

Xcode қолжетімді опцияларды көрсетеді:

![Swift](../../RESOURCE/002_view10.png)

Бұл — code completion механизмі. Ол типті болжау мен enum мүшелерін ұсыну негізінде жұмыс істейді, соның арқасында енгізу жылдамдығын арттырып, қателерді азайтады.

Кейінгі сабақтарда enum-ды ресми түрде таныстырамыз, ал қазірше оны жай ғана шолып өтеміз.
