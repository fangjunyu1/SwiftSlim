# Жарқын түстер

Бұл сабақта біз SwiftUI-дегі жиі қолданылатын визуал modifier-лерді үйренеміз. Оларға мыналар кіреді:

- түс
- алдыңғы түс
- фон түсі
- ығыстыру
- мөлдірлік
- бұлдырлық

Сонымен бірге Safe Area (қауіпсіз аймақ) туралы да үйренеміз.

Бұл modifier-лер view-дің сыртқы көрінісін басқаруға қолданылады және интерфейсті айқынырақ, әрі қабаттырақ етеді.

## Түс

SwiftUI-де мәтіннің түсін орнатуға болады.

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

`.blue` көк түсті білдіреді, шын мәнінде бұл `Color.blue`-дың қысқартылған жазылуы (type inference).

Жиі қолданылатын түстер:

```
.black
.green
.yellow
.pink
.gray
...
```

Бұлардың барлығы `Color`-дың статикалық қасиеттері.

![Color](../../RESOURCE/006_color.png)

`Color`-ды түс типі ретінде елестетуге болады, ал `.blue`, `.red` соның нақты түстері.

### Color view

SwiftUI-де `Color` өзі де view ретінде көрсетіле алады.

```swift
Color.red
    .frame(width: 100, height:100)
    .cornerRadius(10)
```

![Color](../../RESOURCE/006_color5.png)

Бұл код 100×100 өлшемді қызыл түсті шаршы жасайды.

Тіпті бүкіл интерфейсті белгілі бір түспен көрсетуге болады:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
    }
}
```

![Color](../../RESOURCE/006_color1.png)

Іске қосқаннан кейін, қызыл түс бүкіл экранды түгел жаппағанын көреміз: iPhone-ның жоғары және төменгі бөліктері ақ күйінде қалады. Бұл жерде Safe Area (қауіпсіз аймақ) ұғымы бар.

## Safe Area (қауіпсіз аймақ)

Safe Area — мазмұн жабылып қалмауы үшін жүйе алдын ала бос қалдыратын аймақ. Оған мыналар кіреді:

1. Жоғарғы күй жолағы (уақыт, батарея)

2. Төменгі Home көрсеткіші

3. Ойық (notch) немесе Dynamic Island аймағы

![Color](../../RESOURCE/006_color3.png)

SwiftUI әдепкі бойынша мазмұнды қауіпсіз аймақтың ішінде шектейді, сондықтан view экран жиегіне дейін созылмайды.

### Safe Area-ны елемеу

Егер түстің бүкіл экранды жабуын қаласаңыз, `ignoresSafeArea` қолдануға болады:

```swift
Color.red
    .ignoresSafeArea()
```

Немесе `edgesIgnoringSafeArea` қолдануға болады:

```swift
Color.red
    .edgesIgnoringSafeArea(.all)
```

![Color](../../RESOURCE/006_color4.png)

Осылайша view бүкіл экранға созылады.

Маңыздысы, `edgesIgnoringSafeArea` — ескілеу жазылу формасы. iOS 14-тен бастап `ignoresSafeArea` қолдану ұсынылады.

## Алдыңғы түс

### foregroundStyle modifier-і

Алдыңғы сабақтарда түсті орнату үшін `foregroundStyle` қолданғанбыз.

Мысалы:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

`foregroundStyle` — жаңа стиль жүйесі, ол түсті, градиентті, материалды және т.б. қолдайды.

![Color](../../RESOURCE/006_color2.png)

### foregroundColor modifier-і

`foregroundColor` арқылы да түс орнатуға болады:

```swift
Text("Fang Junyu")
    .foregroundColor(.blue)
```

Оның қолданылуы `foregroundStyle` сияқты.

Соңғы Xcode нұсқаларында Xcode `foregroundColor` болашақ iOS нұсқаларында ескіруі мүмкін екенін ескертеді, сондықтан басым түрде `foregroundStyle` қолдану ұсынылады.

## Фон

Егер view-ге фон түсін бергіңіз келсе, `background` қолданыңыз:

```swift
background(.red)
```

Мысалы, мәтінге фон түсін қосу:

```swift
Text("FangJunyu.com")
    .background(.orange)
```

![Color](../../RESOURCE/006_color6.png)

Word, Chrome сияқты App-терде мәтінді таңдағанда, осындай фон түсіне ұқсас әсерді көруге болады.

![Color](../../RESOURCE/006_color16.png)

Егер фонды үлкейткіңіз келсе, `padding`-пен бірге қолдану керек:

```swift
Text("FangJunyu.com")
    .padding(10)
    .background(Color.orange)
```

![Color](../../RESOURCE/006_color7.png)

Мұнда бір маңызды ережені ескеру керек:

SwiftUI modifier-лері view-ді жоғарыдан төмен қарай құрастырады. Кейін жазылған modifier алдыңғы нәтиженің үстіне әсер етеді.

Сондықтан:

```swift
.padding()
.background()
```

Бұл фон ішкі шегініс қосылғаннан кейінгі view-ді орап тұратынын білдіреді.

Егер ретін ауыстырсақ:

```swift
Text("FangJunyu.com")
    .background(Color.orange)
    .padding(10)
```

![Color](../../RESOURCE/006_color6.png)

Фон үлкеймейді, өйткені фон кейінгі `padding`-ті орап тұрған жоқ.

## Мысал - төрт бұрышты гайка

Енді қарапайым төрт бұрышты гайкаға ұқсас view жасаймыз.

![Color](../../RESOURCE/006_color8.png)

Алдымен 50 × 50 өлшемді ақ шаршы жасаймыз:

```swift
Color.white
    .frame(width: 50, height: 50)
```

Егер оны дөңгелекке айналдырғымыз келсе, `cornerRadius` қолданамыз:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
```

Дөңгелек бұрыш радиусы ен мен биіктіктің жартысына тең болғанда, ол шеңберге айналады.

Енді көк түсті фон қосайық:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
    .padding(20)
    .background(.blue)
```

![Color](../../RESOURCE/006_color8.png)

Жаңадан қосылған `padding` сыртқы аймақты үлкейтеді, ал `background` сол сыртқы аймаққа көк түсті сызады.

Осылайша төрт бұрышты гайка әсерін жасаймыз.

### Тағы бір тәсіл

Фон түсін қолданудан бөлек, төрт бұрышты гайканы `ZStack` арқылы да жасауға болады.

Бұған дейін `ZStack` қабаттап реттеу жасайтынын үйрендік. Төрт бұрышты гайканы да бір шеңбер мен бір тікбұрыштың қабаттасуы деп қарастыруға болады.

```swift
ZStack {
    Color.blue
        .frame(width: 70, height: 70)
    Color.white
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color8.png)

`ZStack` view-лерді ретімен қабаттайды, кейін қосылған view жоғарғы қабатта көрінеді.

## Мысал - қабаттасқан екі шеңбер

Көптеген белгішелер қарапайым фигуралардың қабаттасуынан тұрады, мысалы, екі қабаттасқан шеңбер.

![Color](../../RESOURCE/006_color14.png)

Алдымен екі шеңбер жасаймыз:

```swift
Color.red
    .frame(width: 50, height:50)
    .cornerRadius(25)
Color.orange
    .frame(width: 50, height:50)
    .cornerRadius(25)
```

![Color](../../RESOURCE/006_color11.png)

Екі шеңбер қабаттасып көрінуі керек болғандықтан, `ZStack` layout-ын қолданамыз:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color12.png)

Қазір өлшемдері бірдей екі шеңбер толықтай бірінің үстіне бірі түсіп тұр.

Бізге екі шеңбердің ішінара ғана қабаттасуы керек, толық жабылуы емес. Ол үшін `offset` арқылы ығыстыру әсерін жасауға болады.

## Ығыстыру

`offset` тек view-дің сызылатын орнын өзгертеді, бірақ ата-аналық view-дің layout есебіне әсер етпейді.

Қолданылуы:

```swift
.offset(x:y:)
```

`x` — көлденең ығысу, `y` — тік ығысу.

Оң мән — оңға/төменге, теріс мән — солға/жоғарыға ығыстыруды білдіреді.

`offset` арқылы екі шеңберді жартылай қабаттастыру:

```swift
ZStack {
    Color.red
        .frame(width: 50, height: 50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color13.png)

Қызыл шеңбердің layout ішіндегі орны өзгерген жоқ, бірақ оның сызылатын орны солға қарай 25 нүктеге жылжыды. Сондықтан екі шеңбер арасында жартылай қабаттасқан көрініс пайда болды.

## Мөлдірлік

SwiftUI-де `opacity` view мөлдірлігін орнатады.

Негізгі қолданылуы:

```swift
.opacity(0.5)
```

`opacity` параметрі 0.0 мен 1.0 аралығында болады, мұнда:

- 0 — толық мөлдір
- 1 — мүлде мөлдір емес

`opacity` арқылы қызғылт сары шеңбердің мөлдірлігін орнатуға болады:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
```

![Color](../../RESOURCE/006_color14.png)

Қызғылт сары шеңбердің `opacity` мәні 0.8 болғанда, оның мөлдір еместігі 80% болады. Екі шеңбер қабаттасқанда, ортақ аймақта түстер араласып көрінеді.

## Бұлдырлық

SwiftUI-де `blur` арқылы бұлдырлық әсерін беруге болады:

```swift
.blur(radius:10)
```

`radius` бұлдырлық радиусын анықтайды: мән неғұрлым үлкен болса, әсер соғұрлым айқын болады.

Екі шеңберге бұлдырлық әсерін қосып көрейік:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x:-35)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
.blur(radius:10)
```

![Color](../../RESOURCE/006_color15.png)

Нәтижесінде, өте бұлдыр екі шеңберді көреміз.

## Қорытынды

Бұл сабақ SwiftUI-дегі жиі қолданылатын визуал modifier-лерге арналды. Біз modifier-лер арқылы view түсін, орнын және көрнекі әсерлерін қалай басқаруға болатынын үйрендік.

Нақты мысалдар арқылы әртүрлі визуал modifier-лердің интерфейстегі нақты әсерін көрдік және қауіпсіз аймақ туралы білім алдық.

Бұлардың барлығы өте базалық modifier-лер. Оларды көбірек қайталап, жиі қолдану нақты әзірлеуде интерфейс әсерін анық әрі сенімді басқаруға көмектеседі.

### Сабақтан кейінгі жаттығу

- Бір суретке мөлдірлік пен бұлдырлық әсерін қосу
- Мөлдірлігі әртүрлі үш шеңберді қабаттастыру
- Бүкіл экранды жабатын фон суретін жасап, қауіпсіз аймақты елемеу
- `offset` арқылы бірнеше view орнын реттеу

Жаттығудың мақсаты — API-ді жаттап алу емес, визуал өзгеріс пен layout әрекетінің арасындағы байланысты бақылау.
