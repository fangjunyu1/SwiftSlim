# Swift және SwiftUI

Apple платформаларында әзірлеуді үйренгенде, көптеген оқулықтар Swift пен SwiftUI-ді бөлек-бөлек түсіндіруге бейім келеді. Бұл жаңадан бастаушыларға екеуі бір-бірінен мүлде бөлек екі технология сияқты көрініп, түсінікті үзік етуі мүмкін.

Шын мәнінде, Swift пен SwiftUI — өзара терең біріктірілген тұтас жүйе: Swift — бағдарламалау тілі, ол логика мен деректерді қамтамасыз етеді; ал SwiftUI — Swift негізінде құрылған пайдаланушы интерфейсі фреймворкі, ол view-ді бейнелеу мен интеракцияға жауап береді.

Бұл сабақ сізге анық түсінік қалыптастыруға көмектеседі: Swift деген не, SwiftUI деген не және олар код ішінде қалай бірлесіп жұмыс істейді.

## Swift деген не?

Swift — Apple компаниясы ұсынған заманауи бағдарламалау тілі, ол iOS, macOS, watchOS және tvOS сияқты платформаларға қолданба әзірлеу үшін пайдаланылады.

![swift](../../RESOURCE/010_swift.png)

Swift — қатаң типтелген, статикалық тіл. Ол қауіпсіздікке көбірек көңіл бөліп жасалған, сол арқылы көптеген жиі кездесетін бағдарламалау қателерін (мысалы, бос көрсеткіш, шектен шығып кету және т.б.) азайтады, әрі C және Objective-C-пен бірлесіп жұмыс істеуді қолдайды.

Нақты әзірлеуде Swift көбіне қолданбаның логикалық бөлігіне жауап береді, мысалы деректерді өңдеу, желілік сұраулар, сақтау операциялары және т.б.

## SwiftUI деген не?

SwiftUI — Apple компаниясы 2019 жылы шығарған жаңа интерфейс құру фреймворкі. SwiftUI декларативті бағдарламалау үлгісін қолданады, соның арқасында әзірлеуші интерфейс құрылымы мен интерактив мінез-құлықты әлдеқайда ықшам кодпен сипаттай алады.

![swiftui](../../RESOURCE/010_swiftui.png)

Декларативті бағдарламалау дегеніміз — әзірлеуші жүйеге тек нені көрсету керегін айтады, ал жүйе деректің өзгеруіне қарай view-ді автоматты түрде жаңартады. Яғни интерфейс күйін қолмен жаңартып отырудың қажеті жоқ, бұл UI әзірлеуді айтарлықтай жеңілдетеді.

SwiftUI-дің өзегі — view компоненттері (`Text`, `Image`, `Button` және т.б.) және layout контейнерлері (`VStack`, `HStack`, `ZStack` және т.б.). Бұл компоненттер дерек байланысы мен күйді басқару арқылы өзара әрекеттеседі, соның арқасында интерфейс дерек өзгерісіне жауап беріп, автоматты түрде жаңарады.

## Swift пен SwiftUI қалай бірлесіп жұмыс істейді?

Swift пен SwiftUI-дің міндеттерін былайша қорытындылауға болады:

**1. Swift: логика мен деректерді өңдеу**

Swift негізінен деректерді басқару, күйді сақтау және логиканы орындау үшін қолданылады. Мұндай код көбіне интерфейс рендерингіне тікелей әсер етпейді, оның орнына дерек пен әрекетті өңдеуге жауап береді.

Мысалы, Swift ішінде дерек өңдеу:

```swift
let markdown = try? String(contentsOf: url)
```

Мұнда айнымалы жариялау, тұрақты анықтау, функциялар, басқару ағыны сияқты код бар, бірақ ол view көрсетілімін тікелей басқармайды.

**2. SwiftUI: интерфейсті жариялау және рендерлеу**

Ал SwiftUI пайдаланушы интерфейсінің құрылымы мен мазмұнын құру үшін қолданылады және негізінен интерфейсті бейнелеу мен view интеракциясына жауап береді.

Мысалы, SwiftUI арқылы мәтіндік view жасау:

```swift
Text("SwiftSlim")
```

Мұндай код интерфейс элементтерін құру мен басқаруға арналған, яғни тікелей пайдаланушы интерфейсімен байланысты.

### Базалық мысал

Төменде қарапайым SwiftUI мысалы берілген:

```swift
// ContentView.swift

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Бұл мысалда:

**SwiftUI коды**: `VStack`, `Image`, `Text`, `padding` — бұлардың барлығы SwiftUI-ге жатады және интерфейсті көрсету мен layout үшін жауап береді.

**Swift коды**: файл жоғарғы бөлігіндегі `//` комментарийлер мен `import SwiftUI` — Swift код құрылымына жатады; ал төмендегі `#Preview` — Xcode preview үшін қолданылатын макрос, ол нақты интерфейс рендерингіне қатыспайды.

### Кеңейтілген мысал

Нақты жобада Swift пен SwiftUI жиі аралас қолданылып, әрқайсысы өзіне ыңғайлы міндетті орындайды:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    func printName() {
        print("name: \(name)")
    }

    var body: some View {
        VStack {
            Text(name)
            Button("Print") {
                printName()
            }
        }
        .padding()
    }
}
```

Бұл мысалда:

**SwiftUI коды**: `@State` — SwiftUI-ге тән property wrapper, ол view-дің өзгеретін күйін жариялау үшін қолданылады; `Button` мен `Text` — SwiftUI view компоненттері, олар интерфейсті көрсету мен пайдаланушымен әрекеттесуге жауап береді.

**Swift коды**: `private var name` және `func printName()` — Swift коды, ол деректерді сақтау мен логиканы орындауға арналған, бірақ view рендерингіне тікелей жауап бермейді.

Пайдаланушы батырманы басқанда, SwiftUI батырма ішіндегі әрекет кодын іске қосады:

```swift
Button("Print") {
    printName()
}
```

Мұнда `Button` — SwiftUI компоненті, ал орындалатын `printName()` функциясы — нақты логиканы өңдейтін Swift коды.

Осындай бірлескен жұмыс Swift пен SwiftUI-ді табиғи түрде біріктіреді: Swift дерек пен логикаға, SwiftUI интерфейсті көрсетуге жауап береді.

## Swift пен SwiftUI коды әдетте қайда жазылады?

SwiftUI-де интерфейс `body` қасиеті қайтаратын view арқылы құрылады. Сондықтан интерфейсті сипаттайтын барлық код әдетте `body` ішінде жазылады.

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

Бұл мысалда `Text` — SwiftUI view компоненті, сондықтан ол міндетті түрде `body` ішінде жазылады, себебі SwiftUI интерфейсті дәл осы `body` арқылы құрады.

Интерфейске тікелей қатысы жоқ кодтар, мысалы айнымалылар, функциялар немесе дерек өңдеу логикасы, әдетте `body` сыртында жазылады. Мысалы:

```swift
struct ContentView: View {

    // Swift：дерек немесе логика
    var name = "Fang"

    func printName() {
        print(name)
    }

    // SwiftUI：интерфейс
    var body: some View {
        Text(name)
    }
}
```

Маңыздысы, `body` ішінде де Swift синтаксисін қолдануға болады, мысалы `if`, `for` сияқты басқару ағындары. Бірақ олар тек SwiftUI view-лерінің жасалуын басқару үшін қолданылады:

```
if isLogin {
    Text("Welcome")
}
```

Сондықтан SwiftUI әзірлеуде былайша қарапайым түсінуге болады: view коды (`Text`, `Image`, `Button` және т.б.) әдетте `body` ішінде жазылады; ал дерек пен логика коды (айнымалылар, функциялар және т.б.) көбіне `body` сыртында жазылады.

## Swift файлдары

Оқу тереңдеген сайын біз MVVM архитектурасымен де танысамыз. Онда `ViewModel` мен `Model` қабаттары көбіне таза Swift кодынан тұрады және view қабатынан (SwiftUI) толықтай бөлінеді.

Мысалы, қолданба күйін басқаратын класс:

```swift
class AppStorage: ObservableObject {
    static var shared = AppStorage()
    private var isLoading = false
}
```

Немесе дерек құрылымын сипаттайтын құрылым:

```
struct SponsorStruct: Identifiable {
    var id:String
    var icon: String
}
```

Мұндай файлдардың міндеті — деректерді басқару мен сақтау, олар view ішінде тікелей көрсетілмейді, сондықтан олар толықтай Swift-ке жатады.

## Тарихи негіз

Swift пен SwiftUI арасындағы байланысты толық түсіну үшін, Apple әзірлеу технологияларының қалай дамығанын да білу керек. 2026 жылға қарай бұл технологиялық стек бірнеше рет жаңарудан өтті.

### Swift тарихы

Swift пайда болғанға дейін, Apple платформаларындағы негізгі бағдарламалау тілі Objective-C болды, ол C тілімен аралас кодты қолдайтын. Оның синтаксисі салыстырмалы түрде ұзақ әрі күрделі, сондықтан жаңадан бастаушылар үшін кіру шегі жоғары болды:

```Obj-c
// Objective-C жазылуы
NSString *name = @"Fang";
NSLog(@"Hello，%@", name);
```

2014 жылы Apple WWDC-те Swift тілін ұсынды. Swift синтаксисі анағұрлым заманауи, тип қауіпсіздігі жоғары және біртіндеп Objective-C-ті негізгі әзірлеу тілі ретінде алмастыра бастады:

```swift
// Swift жазылуы
let name = "Fang"
print("Hello, \(name)")
```

Дегенмен Objective-C тарих сахнасынан мүлде кеткен жоқ. Ол әлі де көптеген бұрыннан бар жобаларда және төменгі деңгейлі фреймворктерде кеңінен қолданылады. Оның базалық синтаксисін білу ескі жобаларды қолдау мен жүйе негізін түсіну үшін пайдалы.

### SwiftUI тарихы

SwiftUI пайда болғанға дейін, iOS жүйесінде **UIKit**, ал macOS жүйесінде **AppKit** қолданылатын. Бұл екі фреймворк "императивті" бағдарламалау тәсіліне сүйенеді. Әзірлеуші view күйін Storyboard арқылы drag-and-drop жасап немесе кодты қолмен жазып басқаруы керек болды. Код көлемі көп, ал қолдау құны жоғары болатын, әсіресе интерфейс логикасы күрделі болғанда.

![storyboard](../../RESOURCE/010_xcode.png)

2019 жылы Apple WWDC-те SwiftUI-ді ресми түрде шығарды. SwiftUI "декларативті" бағдарламалау парадигмасын енгізіп, UI әзірлеу процесін айтарлықтай жеңілдетті.

![storyboard](../../RESOURCE/010_xcode1.png)

Мұнда бір маңызды нәрсе бар: SwiftUI мүлде тәуелсіз төменгі деңгейлі іске асыру емес. Әртүрлі платформаларда ол шын мәнінде UIKit (iOS) немесе AppKit (macOS) арқылы байланысып, солармен бірге жұмыс істейді, яғни түпкілікті деңгейде осы екі фреймворкке сүйенеді.

### Swift пен UIKit/AppKit арасындағы байланыс

Swift жалпы мақсаттағы бағдарламалау тілі болғанымен және Apple-дің барлық платформаларында жұмыс істей алғанымен, ол UIKit не AppKit-ті толық алмастыра алмайды. Кейбір күрделі интерфейс талаптары немесе SwiftUI әлі толық жаппаған функциялар үшін UIKit немесе AppKit көмегіне жүгінуге тура келеді.

Мысалы, UIKit күрделі view controller басқару, анимация әсерлері, gesture тану сияқты мүмкіндіктерде өте жетілген және өндірістік тәжірибеде көп тексерілген. SwiftUI бұл бағыттарда үнемі күшейіп келеді, бірақ кейбір шеткі сценарийлерде әлі де шектеулері бар.

Сондықтан көптеген әзірлеушілер жобада SwiftUI мен UIKit-ті (немесе AppKit-ті) аралас қолданады және әрқайсысының артықшылығын тиімді пайдаланады.

Осы тұрғыдан қарағанда, SwiftUI-ді UIKit / AppKit-тің жоғары деңгейдегі қаптамасы ретінде түсінуге болады. SwiftUI-ді оқып жүргенде UIKit пен AppKit-тің базалық ұғымдарын да аздап білу, ескі жобаларды қолдағанда немесе күрделі функционал жасағанда, технологиялық шешімді дұрысырақ таңдауға көмектеседі.

## Қорытынды

**Swift**：негізінен логика, дерек өңдеу, басқару ағыны сияқты view layout-қа қатысы жоқ кодты жазу үшін қолданылады.

**SwiftUI**：пайдаланушы интерфейсін декларативті түрде құру үшін қолданылады, view мазмұны мен layout коды осыған жатады.

Нақты әзірлеуде Swift пен SwiftUI әдетте бірге қолданылады: Swift логиканы өңдейді, SwiftUI интерфейсті өңдейді.

Objective-C пен UIKit-тен Swift пен SwiftUI-ге дейін, Apple әзірлеу ортасы біртіндеп анағұрлым заманауи әрі ықшам тәсілге ауысты. Бірақ UIKit пен AppKit көптеген дәстүрлі жобаларда әлі де маңызды болып қала береді.

Swift пен SwiftUI арасындағы байланысты түсіну бізге iOS/macOS әзірлеуді тиімдірек жүргізуге көмектеседі, сондай-ақ ескі жобаларды қолдағанда технологиялық таңдауды анағұрлым орынды жасауға мүмкіндік береді.
