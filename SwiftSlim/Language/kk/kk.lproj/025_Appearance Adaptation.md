# Сыртқы көрініске бейімдеу

Бұл сабақта жарық режим мен қараңғы режимге арналған сыртқы көрініске бейімдеуді үйренеміз.

iPhone, iPad және Mac құрылғыларында пайдаланушы жарық режимді де, қараңғы режимді де таңдай алады.

Жүйе қараңғы режимге ауысқанда, көптеген App қолданбаларының фоны, мәтіні және белгіше түстері де бірге өзгереді.

Мысалы, күндіз интерфейс ашық түсті фон болуы мүмкін, ал түнде ол қараңғы фонға айналуы мүмкін.

![view](../../../Resource/025_view9.png)

Жүйе сыртқы көрінісіне қарай интерфейсті автоматты түрде реттеп көрсету қабілеті сыртқы көрініске бейімдеу деп аталады.

## Xcode ішінде сыртқы көріністі алдын ала қарау

Сыртқы көрініске бейімдеуді үйренбес бұрын, Xcode ішінде жарық режим мен қараңғы режимді қалай алдын ала көруге болатынын қарастырайық.

`Canvas` аймағының төменгі жағында `Device Settings` түймесін бассаңыз, `Canvas Device Settings` қалқымалы терезесі ашылады.

![canvas](../../../Resource/025_view.png)

Қалқымалы терезеде `Color Scheme` параметрін көруге болады.

![canvas1](../../../Resource/025_view1.png)

Мұнда:

- `Light Appearance` жарық режимді білдіреді.
- `Dark Appearance` қараңғы режимді білдіреді.

Осы жерден алдын ала қарау көрінісін ауыстырып, қазіргі интерфейстің жарық режим мен қараңғы режимдегі көрсетілу әсерін көре аламыз.

## Екі көріністі қатар көрсету

Егер жарық режим мен қараңғы режимді бір уақытта көргіңіз келсе, `Variants` мүмкіндігін пайдалануға болады.

`Canvas` аймағының төменгі жағында `Variants` түймесін басып, `Color Scheme Variants` таңдаңыз.

![canvas2](../../../Resource/025_view2.png)

Таңдағаннан кейін, `Canvas` жарық режим мен қараңғы режимнің алдын ала қарау әсерлерін бір уақытта көрсетеді.

![canvas3](../../../Resource/025_view3.png)

Осылайша екі түрлі сыртқы көріністегі интерфейс айырмашылықтарын салыстыру ыңғайлырақ болады.

Қарапайым айтқанда:

Егер жарық/қараңғы режимді уақытша ғана ауыстырғыңыз келсе, `Canvas Device Settings` қолдануға болады.

Егер екі көріністі бір уақытта көргіңіз келсе, `Color Scheme Variants` қолдануға болады.

## Көріністер әдепкі бойынша сыртқы көрініске бейімделеді

SwiftUI ішінде көптеген жүйелік көріністер әдепкі бойынша жарық режим мен қараңғы режимге автоматты түрде бейімделеді.

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("Жарық")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("мысал мәтін")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Көрсетілу әсері:

![canvas](../../../Resource/025_view4.png)

Көріп тұрғанымыздай, жарық режимде белгіше мен тақырып әдетте қою түспен көрсетіледі.

Қараңғы режимде белгіше мен тақырып автоматты түрде ашық түске айналады.

Себебі `Text`, `Image` сияқты SwiftUI көріністері әдепкі бойынша түстерін жүйенің сыртқы көрінісіне қарай реттейді.

Яғни, біз тұрақты түсті қолмен көрсетпесек, SwiftUI сыртқы көрініске бейімдеудің бір бөлігін өзі өңдейді.

## Тұрақты түстер автоматты түрде ауыспайды

Назар аудару керек: егер тұрақты түсті қолмен көрсетсек, ол жарық/қараңғы режимге қарай автоматты түрде ауыспайды.

Мысалы:

```swift
Text("мысал мәтін")
	.foregroundStyle(Color.gray)
```

Мұндағы мәтін `Color.gray` деп көрсетілген, сондықтан ол үнемі сұр түспен көрсетіледі.

Тағы бір мысал:

```swift
Text("Тақырып")
    .foregroundStyle(Color.white)
```

Бұл код жарық режимде де, қараңғы режимде де ақ мәтін көрсетеді.

Егер фон да ашық түсті болса, ақ мәтінді анық көру қиын болуы мүмкін.

Сондықтан сыртқы көрініске бейімдеу кезінде `Color.white`, `Color.black` сияқты тұрақты түстерді ойланбай жазудан мүмкіндігінше аулақ болу керек.

Көп жағдайда жүйелік семантикалық стильдерді алдымен қолдануға болады, мысалы:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

Мұнда:

- `.primary` әдетте негізгі мазмұн үшін қолданылады.
- `.secondary` әдетте қосымша мазмұн үшін қолданылады.

Олар жарық/қараңғы режимге қарай көрсетілу әсерін автоматты түрде реттейді.

## Сыртқы көрініске қарай әртүрлі мазмұн көрсету

Кейде біз тек түстің өзгеруін ғана емес, әртүрлі сыртқы көріністе әртүрлі мазмұн көрсетуді де қалаймыз.

Мысалы:

- Жарық режимде кәдімгі шам белгішесі көрсетіледі.
- Қараңғы режимде жарық шашатын шам белгішесі көрсетіледі.
- Жарық режимде тақырып `Light` болып көрсетіледі.
- Қараңғы режимде тақырып `Dark` болып көрсетіледі.

Мұндай кезде қазіргі жүйенің жарық режимде ме, әлде қараңғы режимде ме екенін анықтау қажет.

SwiftUI ішінде қазіргі сыртқы көрініс режимін `colorScheme` орта мәні арқылы алуға болады:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Толық код:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "Жарық" : "Қараңғы"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("мысал мәтін")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

Көрсетілу әсері:

![view](../../../Resource/025_view5.png)

Алдын ала қарау нәтижесінен бір көріністің әртүрлі сыртқы көрініс режимдерінде толықтай бірдей мазмұн көрсетпейтінін көруге болады.

Жарық режимде `Image` кәдімгі шам белгішесін көрсетеді, ал тақырып `Light` болып көрсетіледі.

Қараңғы режимде `Image` жарық шашатын шам белгішесін көрсетеді, ал тақырып `Dark` болып көрсетіледі.

Яғни мұндағы өзгеріс тек түстің өзгеруі ғана емес, белгіше атауы мен тақырып мәтінінің өзі де өзгереді.

Себебі біз белгіше мен тақырыпты `body` ішінде тікелей жазып қойған жоқпыз, оларды `colorScheme` арқылы әртүрлі мазмұн етіп есептеп шығардық.

Бұл кодта біз екі есептелетін қасиет қолдандық:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "Жарық" : "Қараңғы"
}
```

`colorScheme` мәні `.light` болса, қазіргі режим жарық режим екенін білдіреді.

`colorScheme` мәні `.dark` болса, қазіргі режим қараңғы режим екенін білдіреді.

Сондықтан біз әртүрлі сыртқы көрініс режиміне қарай әртүрлі белгіше атауы мен тақырып мәтінін қайтара аламыз.

Бұл да есептелетін қасиеттердің жиі қолданылатын тәсілі: қазіргі күйге қарай көрініс көрсетуі керек мазмұнды есептеу.

## @Environment түсіну

Бұл біздің `@Environment` ұғымымен бірінші рет танысуымыз.

`@Environment` дегенді былай түсінуге болады: SwiftUI ортасынан бір мәнді оқу.

App жұмыс істеп тұрғанда, жүйе көптеген орта ақпараттарын береді, мысалы:

- Қазіргі тіл
- Қазіргі сыртқы көрініс режимі
- Қазіргі орналасу бағыты
- Қазіргі қаріп өлшемі параметрі

Қазіргі сыртқы көрініс режимін алу қажет болғанда, `colorScheme` оқуға болады:

```swift
@Environment(\.colorScheme) private var colorScheme
```

Мұнда

```swift
\.colorScheme
```

ортадан `colorScheme` деген мәнді оқуды білдіреді.

```swift
private var colorScheme
```

оқылған мәнді `colorScheme` айнымалысына сақтауды білдіреді.

Айнымалы атауын өзіңіз таңдай аласыз, мысалы былай да жазуға болады:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

Алдыңғы `\.colorScheme` өзгермесе, бұл қазіргі сыртқы көрініс режимін оқып жатқанын білдіреді.

## colorScheme мәнінің екі жиі түрі

`colorScheme` жиі кездесетін екі мәнге ие:

```swift
.light
.dark
```

Олар мынаны білдіреді:

- `.light`: жарық режим
- `.dark`: қараңғы режим

Мысалы:

```swift
private var titleName: String {
    colorScheme == .light ? "Жарық" : "Қараңғы"
}
```

Бұл кодтың мағынасы:

Егер қазіргі режим жарық режим болса, `colorScheme == .light` шарты орындалады және `"Жарық"` қайтарылады.

Әйтпесе, `"Қараңғы"` қайтарылады.

Сондықтан жарық режимде:

```swift
Text(titleName)
```

көрсетілетіні:

```swift
Жарық
```

Қараңғы режимде көрсетілетіні:

```swift
Қараңғы
```

Осы тәсіл арқылы біз әртүрлі сыртқы көрініске қарай әртүрлі мазмұн көрсете аламыз.

## Түсті бейімдеу мысалы

Мәтін мен белгішеден бөлек, кейде түсті өзіміз өңдеуіміз қажет болады.

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Мәтін")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

Көрсетілу әсері:

![view](../../../Resource/025_view6.png)

Бұл кодта мәтін ақ түсті, фон қара түсті.

Жарық режимде қара фон анық көрінеді.

Бірақ қараңғы режимде, егер бүкіл интерфейстің фоны да қара болса, бұл қара фон жүйе фонымен бірігіп кетіп, жеткілікті анық көрінбеуі мүмкін.

Мұндай кезде сыртқы көрініс режиміне қарай мәтін түсі мен фон түсін ауыстыра аламыз:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("Мәтін")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

Көрсетілу әсері:

![view](../../../Resource/025_view7.png)

Енді жарық режимде қара фон және ақ мәтін көрсетіледі.

Қараңғы режимде ақ фон және қара мәтін көрсетіледі.

Осылайша кез келген сыртқы көріністе мәтін мен фон арасында айқын контраст болады, пайдаланушы мазмұнды анық көре алады.

## Assets арқылы түстерді бейімдеу

`colorScheme` арқылы қолмен анықтаудан бөлек, жарық/қараңғы режимге бейімдеу үшін `Assets` ресурс түстерін де қолдануға болады.

Бұл тәсіл App ішіндегі жалпы түстерді басқаруға көбірек сәйкес келеді.

![assets](../../../Resource/025_color1.png)

Мұнда біз бір түс жасап, оны `redText` деп атаймыз.

![assets](../../../Resource/025_color2.png)

Түс ресурсы жарық режим мен қараңғы режимдегі түстерді бөлек орната алады.

![assets](../../../Resource/025_color3.png)

Сәйкес түс аймағын таңдағаннан кейін, оң жақтағы атрибуттар инспекторында түсті өзгертуге болады.

Жоғарғы оң жақтағы `Hide or show the Inspectors` түймесін басып, инспектор аймағын ашыңыз.

Мұнда біз `redText` мәнін былай орнатамыз:

- Жарық режимде қызыл түс көрсетіледі

Мұнда:

```swift
Color("redText")
```

Бұл `Assets` түсінің өзі де сыртқы көрініске бейімдеуді қолдайтынын білдіреді.

## colorScheme және Assets айырмашылығы

`colorScheme` және `Assets` екеуі де сыртқы көрініске бейімдеуді жүзеге асыра алады, бірақ олар қолданылатын жағдайлар әртүрлі.

Егер тек жарық/қараңғы режимге қарай мәтін мазмұнын, сурет атауын немесе `SF Symbols` атауын ауыстыру керек болса, `colorScheme` қолдануға болады.

Мысалы:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

Егер App ішінде жиі қолданылатын түстер болса, мысалы тақырып түсі, карта фоны, мәтін түсі, онда `Assets` түстерін қолдану көбірек ұсынылады.

Себебі `Assets` түстерін бірнеше көріністе қайта қолдануға болады, әрі кейін өзгерту де ыңғайлырақ.

Мысалы:

```swift
Color("redText")
```

Осылайша тек `Assets` ішінде түсті өзгерту жеткілікті, бұл түсті қолданған барлық жер бірге өзгереді.

## Қорытынды

Бұл сабақта жарық режим мен қараңғы режимге арналған сыртқы көрініске бейімдеуді үйрендік.

Алдымен Xcode ішіндегі `Canvas` арқылы әртүрлі сыртқы көріністі алдын ала көруді үйрендік, сондай-ақ `Color Scheme Variants` арқылы жарық режим мен қараңғы режимді бір уақытта көруді үйрендік.

Содан кейін SwiftUI ішіндегі `Text`, `Image` сияқты көріністер жүйе сыртқы көрінісіне қарай түстерді әдепкі бойынша автоматты түрде бейімдейтінін түсіндік.

Бірақ егер `Color.white` немесе `Color.black` сияқты тұрақты түстерді қолмен көрсетсек, бұл түстер автоматты түрде ауыспайды.

Одан кейін `@Environment` және `colorScheme` үйрендік:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` арқылы қазіргі режим жарық режим бе, әлде қараңғы режим бе екенін анықтап, әртүрлі сыртқы көрініске қарай әртүрлі белгіше, мәтін немесе түс көрсете аламыз.

Соңында `Assets` түстерін үйрендік.

`Assets` түстері жарық режим мен қараңғы режимдегі түстерді бөлек орната алады, бұл App ішіндегі жалпы түс ресурстарын басқаруға қолайлы.

Сыртқы көрініске бейімдеу App-тың күндіз де, түнде де жақсы көрінуіне көмектеседі, сондай-ақ мәтін анық көрінбеуі немесе фон айқын болмауы сияқты мәселелердің алдын алады.

Нақты әзірлеу кезінде алдымен SwiftUI-дің әдепкі бейімдеу қабілетін және `Assets` түстерін қолданған жөн.

Әртүрлі сыртқы көріністе мәтінді, белгішені, суретті ауыстыру немесе аздаған арнайы көрсету әсерлерін өңдеу қажет болғанда ғана `colorScheme` арқылы анықтау жасаңыз.

## Қосымша білім: қаріп өлшемін көрсету

Бұрын `font` үйренгенде, біз кейбір жүйелік қаріп стильдерін қолданғанбыз:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` сияқты жүйелік қаріп стильдерін қолданудан бөлек, қаріп өлшемін қолмен де көрсетуге болады.

Мысалы:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

Мұндағы:

```swift
.font(.system(size: 100))
```

қаріп өлшемін `100 pt` етіп орнатуды білдіреді.

`SF Symbols` белгішелері үшін `font` белгіше өлшеміне де әсер етеді.

Сондықтан белгішені үлкенірек көрсету қажет болса, `.font(.system(size:))` арқылы өлшемді көрсетуге болады.

Мысалы:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

Ескеретін жайт: қаріп өлшемін қолмен көрсету икемдірек болғанымен, кәдімгі мәтінде тұрақты қаріп өлшемдерін көп қолдану ұсынылмайды.

Кәдімгі мәтінде алдымен `.title`, `.headline`, `.body`, `.caption` сияқты жүйелік қаріп стильдерін қолданған дұрыс.

Осылайша әртүрлі құрылғылар мен пайдаланушының қаріп параметрлеріне жақсырақ бейімделуге болады.
