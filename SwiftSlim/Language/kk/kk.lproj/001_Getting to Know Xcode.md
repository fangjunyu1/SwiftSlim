# Xcode-пен танысу

## Xcode деген не?

Xcode — Apple ұсынатын ресми интеграцияланған әзірлеу ортасы (IDE, Integrated Development Environment), ол iOS, macOS, watchOS, tvOS сияқты Apple платформаларына қолданба жасау үшін пайдаланылады.

![Swift](../../RESOURCE/001_xcode.png)

Осы оқу процесінің барлығында біз код жазу және бағдарламаны іске қосу үшін Xcode қолданамыз.

Бұл сабақта біз Xcode-ты қалай жүктеп алу керегін, алғашқы iOS қолданбасын қалай құру керегін және Xcode интерфейсінің негізгі құрылымымен қалай танысуға болатынын үйренеміз.

## Xcode жүктеп алу

Қазіргі уақытта Xcode-ты орнатудың екі тәсілі бар:

1、[Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12) арқылы жүктеп алу

2、[Apple Developer ресми сайты](https://developer.apple.com/xcode/) арқылы жүктеп алу

Екі әдіспен де Xcode жүктеп алуға болады. Айырмашылығы мынада: Apple Developer ресми сайтында ресми нұсқадан бөлек, [Beta](https://developer.apple.com/download/all/?q=Xcode) нұсқасын немесе ескі нұсқаларды да жүктеуге болады.

Егер тек оқу үшін қолдансаңыз, ресми нұсқаны пайдаланған дұрыс. Beta нұсқасы көбіне жаңа мүмкіндіктерді сынауға арналған және тұрақсыз болуы мүмкін.

## Алғашқы жобаны құру

Xcode-ты ашқаннан кейін, бастапқы экранды көресіз.

![Swift](../../RESOURCE/001_xcode1.png)

Сол жақта жиі қолданылатын үш опция бар:

- Create New Project —— жаңа жоба құру  
- Clone Git Repository —— Git репозиторийін клондау  
- Open Existing Project —— бар жобаны ашу

Оң жақта жақында ашылған Xcode жобалары көрсетіледі.

Біз `"Create New Project"` опциясын таңдаймыз.

### Жоба шаблонын таңдау

![Swift](../../RESOURCE/001_xcode2.png)

Жоба шаблонын таңдау экранына кіргенде, мұнда бірнеше платформа (iOS, macOS және т.б.) және әртүрлі жоба түрлері көрсетіледі.

Жиі кездесетін шаблондар:
- App —— ең базалық қолданба түрі (ұсынылады)
- Document App —— құжатқа негізделген қолданба
- Game —— ойын әзірлеуге арналған
- Framework —— қайта пайдалануға болатын модуль

Бастапқы кезеңде біз тек `"iOS" - "App"` шаблонын таңдаймыз, өйткені бұл интерфейсі бар қолданба түрі.

`"Next"` түймесін басыңыз.

### Жоба ақпаратын толтыру

Жоба туралы ақпаратты толтыру экранына өткенде, өрістерді олардың мағынасына қарай толтыруымыз керек.

![Swift](../../RESOURCE/001_xcode3.png)

Әр өрістің мағынасы:

- Product Name —— жоба атауы, мысалы: SwiftSlimTest
- Team —— әзірлеуші тобы. Егер ақылы Apple Developer аккаунтыңыз болмаса, таңдамасаңыз болады
- Organization Identifier —— ұйым идентификаторы, әдетте кері домен форматы қолданылады, мысалы:
  - com.yourname
  - com.yourcompany
  - Егер доменіңіз болмаса, өзіңіздің ағылшынша атыңызды идентификатор ретінде қолдануға болады
- Bundle Identifier —— қолданбаның бірегей идентификаторы, ол Organization Identifier мен Product Name негізінде автоматты түрде жасалады
- Interface —— интерфейс технологиясы, SwiftUI таңдаңыз
- Language —— әзірлеу тілі, Swift таңдаңыз
- Testing System —— әдепкі бойынша тест нысанын (Unit Tests) жасайды, бастапқы кезеңде елемеуге болады
- Storage —— жергілікті деректерді сақтау фреймворкі, SwiftData немесе Core Data-ны автоматты түрде қоса алады, бастапқы кезеңде елемеуге болады

Толтырып болған соң, `"Next"` түймесін басыңыз.

## Xcode жоба құрылымы

Xcode жобасын сақтағаннан кейін, Finder ішінде жасалған жоба қалтасын көруге болады.

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets —— суреттерді, App белгішелерін және басқа ресурстарды сақтайтын қалта

- ContentView.swift —— әдепкі түрде жасалатын SwiftUI view файлы, кейін интерфейс кодын осында жазамыз

- SwiftSlimTestApp.swift —— қолданбаның кіру файлы (App Entry Point), қолданбаны іске қосуға жауап береді, әзірше оны терең түсінудің қажеті жоқ

- SwiftSlimTest.xcodeproj —— Xcode жоба файлы, осы файлды екі рет шерту арқылы жобаны ашуға болады

## Xcode интерфейсімен танысу

`.xcodeproj` файлын екі рет шертіп, жобаны ашыңыз, содан кейін сол жақтағы Navigator аймағынан `ContentView.swift` файлын таңдаңыз.

Xcode интерфейсі негізінен бес аймақтан тұрады:

- Navigator —— навигация аймағы, файл құрылымын көру және іздеу үшін
- Editor —— өңдеу аймағы, Swift немесе SwiftUI кодын жазу үшін
- Canvas —— кенеп аймағы, SwiftUI алдын ала қарауы үшін
- Inspector —— қасиеттерді тексеру аймағы, файл қасиеттерін көру және өзгерту үшін
- Debug Area —— жөндеу аймағы, лог шығуын қарау үшін

![Swift](../../RESOURCE/001_xcode6.png)

Ескерту: Canvas негізінен SwiftUI view-лерді алдын ала қарау үшін пайдаланылады. Xcode-тың ресми архитектурасы бойынша, ол Editor-дің көмекші алдын ала қарау панелі (Preview Pane) болып саналады. Canvas тек SwiftUI view файлы ашылғанда ғана қолжетімді болады, басқа файл түрлерінде алдын ала қарау көрсетілмейді.

Келесі сабақтарда біз негізінен Editor аймағында Swift пен SwiftUI кодын жазамыз.

Xcode жобасын алғаш ашқанда, Inspector мен Debug Area әдепкі бойынша жасырын болуы мүмкін. Оларды оң жақ жоғарғы және төменгі бұрыштағы батырмалар арқылы көрсетуге болады.

![Swift](../../RESOURCE/001_xcode7.png)

Кеңес: Inspector аймағы негізінен файл қасиеттерін көру және өзгерту үшін қолданылады. Нақты әзірлеуде, редакциялау кеңістігін кеңейту үшін бұл аймақты көбіне жасырады.

## Қорытынды

Бұл сабақта біз Xcode-ты қалай жүктеу керегін, алғашқы Xcode жобасын қалай құру керегін және Xcode интерфейсінің негізгі орналасуын үйрендік.

Төменде еркін тәжірибе жасау уақыты:
- Navigator аймағында файл жасап көруге және жоюға тырысыңыз.  
- Editor аймағындағы кодты қарап, құрылымын бақылаңыз.
- Canvas аймағындағы батырмаларды қарап, алдын ала қарауды үлкейтіп-кішірейтіп көріңіз.

Келесі сабақта біз қарапайым код жаза бастаймыз және Xcode-ты одан әрі жақсырақ түсінуге кірісеміз。
