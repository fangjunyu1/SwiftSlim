# Fill Light қолданбасы

Бұл сабақта біз өте қызықты fill light қолданбасын жасаймыз. Түн түскенде, телефон экранында әртүрлі түстерді көрсетіп, оны қарапайым fill light ретінде пайдалана аламыз.

Бұл fill light қолданбасы экранды түрту арқылы түстерді ауыстыра алады, сонымен қатар жүгірткі арқылы жарықтылықты реттей алады.

Осы мысалда біз `brightness` арқылы көріністің жарықтығын реттеуді, `onTapGesture` арқылы көрініске түрту қимылын қосуды және `Slider` басқару элементін үйренеміз.

![Color](../../Resource/018_color.png)

## Түсті көрсету

Алдымен көріністе бір түсті көрсетейік.

SwiftUI-де `Color` тек түсті білдіріп қана қоймайды, оны көрініс ретінде де көрсетуге болады:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

Мұнда `Color.red` қызыл түсті көріністі білдіреді. `.ignoresSafeArea()` түсті көріністі бүкіл экранға жайып, оны нағыз fill light әсеріне көбірек ұқсатады.

Нәтиже:

![Color](../../Resource/018_color1.png)

### Түстер массиві және индекс

Қазір тек бір ғана түс көрсетіліп тұр. Бірақ fill light әдетте бір ғана түспен шектелмейді. Ол көк, сары, күлгін, ақ және басқа түстерді де көрсете алады.

Біз экранды түрткенде әртүрлі түстер арасында ауысқымыз келеді. Бұл түстерді бір массивке салып, бірге басқара аламыз:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

Массив "бір типтегі деректер тобын" сақтауға қолайлы. Мұнда массивтегі әрбір элемент `Color` болып табылады.

Егер белгілі бір түсті көрсеткіміз келсе, индекс қолдана аламыз:

```swift
colors[0]
```

Бұл массивтің `0` индексіндегі түсті, яғни бірінші түсті алуды білдіреді.

Енді кодты былай жазуға болады:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

Осылайша экран массивтегі бірінші түсті, яғни қызыл түсті көрсетеді.

### Индекс арқылы түсті басқару

Егер біз әртүрлі түстер арасында ауысқымыз келсе, индексті тұрақты етіп жазбай, оны басқаратын айнымалы қажет болады.

Индексті сақтау үшін `@State` арқылы айнымалы жариялауға болады:

```swift
@State private var index = 0
```

Мұнда `index` ағымдағы түстің индексін білдіреді.

`index` өзгерген кезде, SwiftUI интерфейсті қайта есептеп, көрсетілетін мазмұнды жаңартады.

Содан кейін бастапқы `colors[0]` орнына мынаны жазамыз:

```swift
colors[index]
```

Осылайша көріністе көрсетілетін түс `index` арқылы анықталады.

Енді код былай өзгереді:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

`index` өзгергенде, `colors[index]` те басқа түсті көрсетеді.

Мысалы:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

Маңыздысы, `index` массивтегі ең үлкен индекстен аспауы керек, әйтпесе индекс шегінен шығу қатесі пайда болады.

## Түрту қимылы

Енді біз `index` бойынша әртүрлі түстерді көрсете аламыз, бірақ әлі түрту арқылы ауыстыра алмаймыз.

Бұған дейінгі «Дәйексөз каруселі» сабағында біз дәйексөздерді ауыстыру үшін `Button` қолдандық.

Ал бұл жолы біз «бүкіл түсті аймақты түрту» арқылы түсті ауыстырғымыз келеді, сондықтан `onTapGesture` анағұрлым қолайлы.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

Түсті көріністі түрткенде, мына код орындалады:

```swift
index += 1
```

Бұл `index` мәнін `1`-ге арттыруды білдіреді. Индекс артқаннан кейін `colors[index]` массивтегі келесі түсті көрсетеді.

### `onTapGesture`

`onTapGesture` — көрініске түрту әрекетін қосатын gesture modifier.

Негізгі қолданылуы:

```swift
.onTapGesture {
    // code
}
```

Мысалы:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

Осы қызыл көрініс түртілгенде, жақша ішіндегі код орындалып, консольге мынаны шығарады:

```swift
Click color
```

`onTapGesture` арқылы біз көрініс түртілгеннен кейін не болатынын анықтай аламыз.

### `Button`-нан айырмашылығы

Бұған дейін біз `Button` көрінісін оқыдық. `Button` мен `onTapGesture` екеуі де түрту әрекеттерін өңдей алады, бірақ олардың қолданылу жағдайлары бірдей емес.

`onTapGesture` «бар көрініске түрту мүмкіндігін қосу» үшін ыңғайлырақ, мысалы `Color`, `Image`, `Text` немесе басқа қарапайым көріністерге.

Ал `Button` «Растау», «Жіберу» немесе «Өшіру» сияқты нақты батырманы көрсетуге ыңғайлырақ.

Бұл fill light қолданбасында біз түстерді ауыстыруды қарапайым еткіміз келеді. Бүкіл түсті аймақты түрту арқылы түс ауысатындықтан, мұнда `onTapGesture` жақсы таңдау болады.

## Индекс мәселесі

Енді экранды түрту арқылы әртүрлі түстер арасында ауыса аламыз.

Бірақ мұнда маңызды бір мәселе бар: **индекс массив шегінен шығып кетуі мүмкін**.

Мысалы:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

Егер экранды қайта-қайта түрте берсек, `index` соңында `4` болады да, «index out of range» қатесі шығады.

Себебі `colors` массивінде `4` элемент бар, бірақ индекстеу `0`-ден басталады, сондықтан жарамды индекс аралығы `0 - 3`, ал `4` емес.

Егер `colors[4]` мәніне жүгінсек, «index out of range» қатесі пайда болады.

Қазіргі кодта экранды әр түрткен сайын `index` автоматты түрде `1`-ге өседі. Егер біз мұны өңдемесек, ол міндетті түрде шектен шығып кетеді.

Сондықтан экран түртілген кезде индексті тексеру керек: егер ол соңғы түске жетсе, бірінші түске қайтарамыз, ал болмаса `1` қоса береміз.

Мұны `if` операторы арқылы жасауға болады:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

Бұл кодта `colors.count` массивтегі элементтердің санын білдіреді.

Қазіргі массивте 4 түс бар, сондықтан:

```swift
colors.count // 4
```

Бірақ ең үлкен индекс `4` емес, `3`, себебі индекстеу `0`-ден басталады.

Сондықтан соңғы индекс былай жазылады:

```swift
colors.count - 1
```

Яғни:

```swift
4 - 1 = 3
```

Бұл логика былай дейді: егер ағымдағы индекс соңғы түске сәйкес келсе, индексті `0`-ге қайтар; әйтпесе оны `1`-ге арттыр.

Осылайша түстер шеңбер бойымен ауысып отыра алады.

### Индекс логикасын ықшамдау

Егер кодты қысқартқымыз келсе, үштік операторды да қолдануға болады:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

Бұл кодтың мағынасы: егер `index == colors.count - 1` ақиқат болса, `0` қайтар; ал олай болмаса `index + 1` қайтар.

Соңында осы нәтижені қайтадан `index` айнымалысына меншіктейміз.

Енді біз түстерді ауыстыру әсерін іске асыра аламыз.

Толық код:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

Енді экранды түрту арқылы әртүрлі түстер арасында ауыса аламыз. Негізгі fill light қолданбасы дайын болды.

## Түстің атауын көрсету

Біз түстерге сәйкес келетін тағы бір мәтін тобын қоса аламыз, сонда түс өзгерген кезде экранда ағымдағы түстің атауы да бірге көрсетіледі.

Мысалы:

- Қызыл `Red` көрсетеді
- Көк `Blue` көрсетеді
- Сары `Yellow` көрсетеді
- Күлгін `Purple` көрсетеді

Мұнда да түстердің атауларын сақтау үшін массив қолдана аламыз:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

Осы массивтегі мәтіндердің реті түстер массивіндегі түстердің ретімен дәл сәйкес келуі керек.

Содан кейін ағымдағы индекс бойынша түстің атауын көрсету үшін `Text` қолдана аламыз:

```swift
Text(colorsName[index])
```

`Text` `index` бойынша ағымдағы түстің атауын көрсетеді.

`Text` көрінісін жақсарту үшін modifier-ларды қолданайық:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

Осылайша `Text` ақ түспен, тақырып өлшемімен және қалың қаріппен көрсетіледі.

Қазір бізде толық экранды `Color` көрінісі бар. Егер `Text`-ті `Color` көрінісінің үстіне көрсеткіміз келсе, оларды қабаттау үшін `ZStack` layout контейнерін пайдалануымыз керек.

```swift
ZStack {
    Color
    Text
}
```

Сонда код былай болады:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

Осылайша артқы фонда түс көрінісі болады, ал оның үстіне мәтін көрінісі қабатталып шығады.

Нәтиже:

![Color](../../Resource/018_color2.png)

Бір ескеретін жайт, `ZStack` ішінде кейінірек жазылған көріністер әдетте алдыңғы қатарда көрінеді. Егер `Text` `Color` алдында жазылса, кейін жазылған `Color` көрінісі `Text`-ті жауып қалуы мүмкін.

## Жарықтылықты басқару

Енді біз әртүрлі түстерді ауыстыра аламыз, бірақ fill light-тың тағы бір маңызды қызметі бар: **жарықтылықты реттеу**.

SwiftUI-де біз `brightness` modifier-ын пайдаланып көріністің жарықтылығын реттей аламыз.

Мысалы:

```swift
.brightness(1)
```

Біз былай жаза аламыз:

```swift
colors[index]
    .brightness(0.5)
```

Бұл ағымдағы түсті жарығырақ етіп, оны fill light әсеріне жақындатады.

Жарықтылық диапазоны `0 - 1`. `0` бастапқы түсті сақтауды білдіреді, ал мән `1`-ге жақындаған сайын түс жарығырақ болады. `1` ең жарық ақ нәтижені білдіреді.

Біз `brightness` мәнін код арқылы басқара алсақ та, пайдаланушы оны әзірге өзі тікелей реттей алмайды.

Сондықтан бізге сүйреп реттеуге болатын басқару элементін қосу керек: `Slider`.

## `Slider` көрінісі

SwiftUI-де `Slider` — белгілі бір диапазон ішінен мән таңдауға арналған басқару элементі. Apple оны «шектелген сызықтық диапазоннан мән таңдауға арналған басқару элементі» деп сипаттайды.

Негізгі қолданылуы:

```swift
Slider(value: $value, in: 0...1)
```

Параметрлердің түсіндірмесі:

1. `value: $value`: `Slider` айнымалыға байланыстырылуы керек.

    Жүгірткіні сүйреген кезде, айнымалының мәні де бір уақытта өзгереді. Керісінше, айнымалы өзгерсе, жүгірткі де жаңарады.

    Бұл біз бұрын үйренген `TextField`-ке өте ұқсас. Екеуі де «басқару элементін айнымалыға байлайды».

    Байланысты білдіру үшін айнымалының алдына `$` таңбасы қойылады.

2. `in: 0...1`: бұл параметр жүгірткінің мәндер аралығын білдіреді.

    Мұнда `0...1` ең кіші мән `0`, ал ең үлкен мән `1` екенін көрсетеді.

    Жүгірткі сол жақ шетке дейін жылжытылса, байланыстырылған айнымалы `0`-ге жақын болады; оң жаққа дейін жылжытылса, `1`-ге жақын болады.

Мысалы:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` `value` айнымалысына байланысқан. Жүгірткіні жылжытқанда, `value` айнымалысының мәні де бірге өзгереді.

Көрінісі:

![Slider](../../Resource/018_slider.png)

`Slider` сол жаққа жылжытылса, байланыстырылған `value` мәні `0` болады. Оң жаққа жылжытылса, ол `1` болады.

### Мәндер диапазоны

`Slider`-дің мәндер диапазоны тұрақты емес. Оны былай да жазуға болады:

```swift
0...100
```

немесе басқа бір диапазон түрінде.

Бірақ осы fill light қолданбасында біз жарықтылықты басқаруымыз керек, сондықтан `0...1` ең қолайлы таңдау.

## `Slider` арқылы жарықтылықты басқару

Енді `Slider`-ді `brightness`-пен байланыстыруымыз керек.

Алдымен жарықтылық мәнін сақтау үшін айнымалы жасаймыз:

```swift
@State private var slider = 0.0
```

Мұнда `0.0` — `Double` типіндегі мән.

Себебі `Slider` әдетте сандық типке байланысады, ал мұнда оның үздіксіз өзгергенін қалаймыз, сондықтан `Double` ыңғайлырақ. Бұдан бөлек, `brightness` тек `Double` мәнін ғана қабылдайды.

Содан кейін бұл мәнді `brightness`-ке береміз:

```swift
colors[index]
    .brightness(slider)
```

`slider == 0` болғанда, түс әдепкі күйінде қалады. `slider` мәні `1`-ге жақындаған сайын, түс жарығырақ көрінеді.

### `Slider` басқару элементін қосу

Келесіде осы айнымалыны өзгерту үшін `Slider` басқару элементін қосайық:

```swift
Slider(value: $slider, in: 0...1)
```

Жүгірткі өзгерген кезде, `slider` мәні де өзгереді, ал `brightness(slider)` бір уақытта жарықтылықты жаңартады.

Бұл SwiftUI-дегі «айнымалы көріністі басқарады» деген өте типтік мысал.

### `Slider` көрінісін реттеу

Әдепкі бойынша, `Slider` қолжетімді енді пайдаланады.

Біз оған тұрақты ен бере аламыз:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

Содан кейін оны айқынырақ ету үшін тағы бірнеше modifier қосайық:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

Бұл жүгірткіге ақ фон мен жұмыр бұрыштар береді, сондықтан ол `Color` көрінісінің үстінде анық көрінеді.

Соңында оны экранның төменгі жағына орналастырамыз.

Біз `ZStack` қолданғандықтан, оның ішіне `VStack` қойып, `Spacer()` арқылы `Slider`-ді төменге итере аламыз.

## Толық код

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## Қорытынды

Бұған дейін үйренген білімімізді, сондай-ақ түстер мен массивтер сияқты негізгі ұғымдарды пайдаланып, біз өте қызықты fill light қолданбасын жасадық.

Осы fill light қолданбасы арқылы біз `brightness` арқылы жарықтылықты реттеуді, `onTapGesture` арқылы көріністерге түрту әрекеттерін қосуды және `Slider` басқару элементін қолдануды үйрендік.

Біз түстерді ауыстыру үшін `Color` көрінісіне `onTapGesture` қостық. Сондай-ақ `brightness` мәнін басқаратын айнымалыны өзгерту үшін `Slider` басқару элементін қолдандық, бұл «айнымалы көріністі басқарады» дегеннің тағы бір мысалы.

Сонымен қатар біз үштік операторды қайталадық, көріністерді қабаттау үшін `ZStack` қолдандық және бір типтегі деректер тобын басқару үшін массивтерді қолдандық. Бұл массивтер мен индекстер туралы түсінігімізді тереңдетеді, соның ішінде индекс шегінен шығу қателерінен сақтану сияқты тәжірибелік мәселелерді де.

Бұл мысал күрделі болмаса да, бұрын үйренген көптеген негізгі ұғымдарды біріктіреді. Олар шағын нақты жобаға біріктірілгенде, әрбір ұғымның не үшін қолданылатынын түсіну әлдеқайда жеңіл болады.

### Нақты қолдану сценарийі

Ескі iPhone-ды үстелге қойып, өзіңіз жасаған fill light қолданбасы арқылы жарықтың түсін басқарып отырғаныңызды елестетіңіз. Бұл өте жақсы тәжірибе болар еді.

App Store-да көптеген «fill light» қолданбалары бар, олардың өзі де аса күрделі көрінбейді.

![AppStore](../../Resource/018_appStore.PNG)

Біз қарапайым қолданбалардан бастап дамыта аламыз және оларды App Store-ға жариялап көруге тырыса аламыз. Бұл тек әзірлеуге деген қызығушылығымызды арттырып қана қоймай, өсу жолымызды да тіркейді.

### Сабақтан кейінгі жаттығу

Сіз бұл fill light қолданбасын қалай кеңейтуге болатыны туралы әрі қарай ойлана аласыз, мысалы:

- Көбірек түстер қосу
- Ағымдағы жарықтылық мәнін көрсету
- Төменгі жүгірткі аймағының дизайнын жақсарту

Осы білімді шынымен қолдана бастағанда, біз үйренетін әрбір ұғымның қолданба жасаудың құралы екенін байқайсыз.

Қаншалықты көп құралды меңгерсек, соншалықты көп мүмкіндіктер жасай аламыз.
