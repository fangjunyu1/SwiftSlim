# Кіру көрінісі

Бұл сабақта біз өте пайдалы бір жағдайды үйренеміз, яғни кіру көрінісін.

Мейлі ол веб-сайт болсын, мейлі қолданба болсын, пайдалану барысында пайдаланушыға өз аккаунтына кіріп, құпиясөз енгізу жиі қажет болады.

Мысалы, GitHub-тың кіру беті:

![WordPress](../../Resource/019_github.png)

Осы сабақта біз пайдаланушы аккаунт пен құпиясөзді қолмен енгізе алатын, ал біз енгізілген мазмұнда мәселе бар-жоғын тексере алатын соған ұқсас кіру көрінісін жасаймыз.

## Жоғарғы орналасу

Кіру көрінісінің кодын `ContentView` файлында жазамыз.

Алдымен, кіру көрінісінің жоғарғы бөлігіндегі белгі аймағын жазайық. Алдын ала қолайлы белгіше суретін дайындап, оны `Assets` қалтасына сала аласыз.

![icon](../../Resource/019_icon1.png)

Содан кейін суретті көрсету үшін `Image` және модификаторларды пайдаланыңыз:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

Бұл модификаторлардың мағынасы:

- `resizable()`: суреттің өлшемін өзгертуге мүмкіндік береді.
- `scaledToFit()`: суреттің бастапқы пропорциясын сақтай отырып масштабтайды.
- `frame(width: 100)`: суреттің көрсетілетін енін `100` етіп орнатады.

Келесіде, кіру тақырыбын көрсету үшін `Text` және модификаторларды пайдаланыңыз:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

Содан кейін сурет пен мәтінді тігінен орналастыру үшін `VStack` пайдаланыңыз:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

### Орналасуды оңтайландыру

Қазір сурет пен тақырып әдепкі бойынша жалпы орналасудың ортаңғы аймағында тұр.

Егер олардың «беттің жоғарғы бөлігіндегі кіру белгісі» сияқты көрінгенін қаласақ, онда `Spacer()` арқылы қалған орынды созып, мазмұнды жоғарғы жаққа жақынырақ көрсете аламыз.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

Осы кезде бүкіл `VStack` үшін жоғарғы арақашықтық қосу үшін `padding` пайдалануға болады.

## Пайдаланушы аты және құпиясөз

Кіру беті әдетте пайдаланушыдан пайдаланушы аты мен құпиясөзді енгізуді талап етеді.

SwiftUI-де пайдаланушы енгізген мазмұнды қабылдау үшін `TextField` пайдалана аламыз.

Бірақ мынаны ескеру керек: `TextField` өзі енгізілген деректерді ұзақ уақыт сақтамайды, ол тек енгізу элементі ғана. Бұл енгізілген мазмұнды шын мәнінде сақтайтын нәрсе — біз оған байлаған айнымалылар.

Сондықтан алдымен пайдаланушы аты мен құпиясөзді сақтау үшін екі `@State` айнымалысын жасауымыз керек:

```swift
@State private var user = ""
@State private var password = ""
```

`@State` айнымалысының мәні өзгерген кезде, SwiftUI байланысты көріністерді автоматты түрде жаңартады.

Содан кейін осы екі айнымалыны байлау үшін `TextField` пайдаланыңыз:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

Мұндағы `$user` және `$password` «binding»-ті білдіреді.

Яғни пайдаланушы енгізу жолағына мәтін жазғанда, айнымалының мәні де сол сәтте өзгереді; ал айнымалы өзгергенде, енгізу жолағында көрсетілетін мазмұн да сол сәтте өзгереді.

«Көрініс пен деректердің бір-бірімен синхрондалуы» деген осы қатынас binding деп аталады.

Мұнда `$` таңбасымен жазылатын нұсқаны пайдалану керектігін ескеріңіз:

```swift
$user
```

Себебі `TextField` кәдімгі жолды емес, «деректерді екі бағытта өзгерте алатын» binding мәнін қажет етеді.

### Енгізу өрістерін көрсету

Оларды `ContentView` ішіне орналастырайық:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

```swift

```

енгізу өрісінде алдымен placeholder мәтіні көрсетіледі, мысалы:


Бұл мәтін пайдаланушыға тек «мұнда нені енгізу керек» екенін ғана көрсетеді, ол нақты енгізілген мазмұн емес.

### Енгізу өрістерін оңтайландыру

Енді енгізу өрістерін қолдануға болады, бірақ әдепкі стилі салыстырмалы түрде қарапайым.

Интерфейсті түсініктірек ету үшін енгізу өрісінің алдына тақырып қосып, өрістің өзіне де аздап қарапайым стильдік оңтайландыру жасай аламыз.

Мысалы, алдымен тақырып қосайық:

```swift
Text("Username")
    .fontWeight(.bold)
```

Содан кейін тақырып пен енгізу өрісін бір қатарға қою үшін `HStack` пайдаланыңыз:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

Мұнда біз `HStack` пайдаланамыз, себебі «сол жақта тақырып, оң жақта енгізу өрісі» көрсетілгенін қалаймыз.

Мынаны ескеру керек: `TextField` әдепкі бойынша қалған бос орынды алып қояды.

![color](../../Resource/019_view6.png)

Екі енгізу өрісінің өлшемі біркелкілеу болуы үшін, `frame(width:)` арқылы ені беруге болады, сонда енгізу өрістері ұқыптырақ көрінеді.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

Содан кейін енгізу өрісі анық көрінуі үшін жақтау да қосыңыз:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Енді пайдаланушы аты мен құпиясөздің екеуін де `ContentView` көрінісіне қосайық:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```


## Кіру логикасы

Енді кіру батырмасына ең негізгі тексеру логикасын қосамыз: **пайдаланушының пайдаланушы аты мен құпиясөз енгізгенін тексеру**.

Біз төмендегі жағдайларды жүзеге асырғымыз келеді:

- Егер пайдаланушы ештеңе енгізбесе, `Empty` шығару
- Егер пайдаланушы екеуінің біреуін ғана енгізсе, қайсысы жетіспейтінін көрсету
- Егер пайдаланушы аты мен құпиясөздің екеуі де енгізілсе, `Success` шығару

### Ешқандай ақпарат енгізілмеген

`user` мен `password` әдепкі бойынша екеуі де бос жолдар болғандықтан:

```swift
@State private var user = ""
@State private var password = ""
```

пайдаланушы ештеңе енгізбесе, олар әлі де бос күйінде қалады.

Swift-те `String` сияқты типтер мазмұнның бос екенін анықтау үшін `isEmpty` қасиетін пайдалана алады.

### isEmpty қасиеті

`isEmpty` көбінесе жолдар, массивтер және басқа мазмұндардың бос екенін тексеру үшін қолданылады.

Мысалы:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

Егер мазмұн бос болса, `isEmpty` `true` қайтарады; егер мазмұн бос болмаса, `false` қайтарады.

Сондықтан оны пайдаланушы аты мен құпиясөзде енгізілген мазмұн бар-жоғын анықтау үшін пайдалана аламыз.

### isEmpty арқылы айнымалыларды тексеру

Егер пайдаланушы ешқандай ақпарат енгізбесе, онда:

```swift
user.isEmpty // true
password.isEmpty    // true
```

Осы кезде шартты былай жаза аламыз:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

Бұл кодтың мағынасы: егер `user` бос болса және `password` та бос болса, онда мынаны шығару:

```swift
Empty
```

Мұндағы `&&` — логикалық оператор және «және» дегенді білдіреді.

Яғни бүкіл шарт сол жақтағы шарт шын болғанда және оң жақтағы шарт та шын болғанда ғана орындалады.

Сондықтан бұл код тек пайдаланушы аты мен құпиясөз екеуі де бос болғанда ғана орындалады.

## Пайдаланушы тек бір бөлігін ғана толтырды

Келесіде басқа бір жағдайды қарастырайық: пайдаланушы тек пайдаланушы атын ғана немесе тек құпиясөзді ғана енгізді.

Мысалы:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

Осы кезде пайдаланушы аты бос емес, бірақ құпиясөз бос.

Мұны тексеру үшін `isEmpty` қолдануды жалғастыра аламыз:

### Пайдаланушы аккаунтты немесе құпиясөзді толтырған жағдай

Егер пайдаланушы тек пайдаланушы атын немесе тек құпиясөзді енгізсе, онда не жетіспейтінін көрсететін сәйкес ескертуді беруіміз керек.

Мысалы, егер пайдаланушы тек пайдаланушы атын ғана енгізсе:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Бұл кодтың логикасы: егер пайдаланушы аты бос болса, пайдаланушы атын енгізуді сұра. Әйтпесе, құпиясөздің бос екенін тексер, егер бос болса, құпиясөзді енгізуді сұра.

### Пайдаланушы барлық ақпаратты енгізді

Егер пайдаланушы аты мен құпиясөз екеуі де енгізіліп қойылса, онда `isEmpty` екеуіне де `false` қайтарады.

Осы кезде оны былай жаза аламыз:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

Мұндағы `!` «терістеу» дегенді білдіреді.

Мысалы:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

Яғни `user.isEmpty` «пайдаланушы аты бос па» екенін анықтау үшін қолданылады, ал `!user.isEmpty` «пайдаланушы аты бос емес пе» екенін анықтау үшін қолданылады.

Құпиясөз үшін де дәл сол логика қолданылады.

Сонымен, бұл кодтың мағынасы: пайдаланушы аты бос емес, әрі құпиясөз де бос емес.

Осы екі шарттың екеуі де шын болғанда, нәтиже мынадай болады:

```swift
Success
```

## Толық тексеру логикасы

Енді осы үш жағдайды біріктіріп, `Button` ішіне жазайық:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

Бұл кодтың орындалу ретін былай түсінуге болады:

Бірінші қадамда алдымен мынаны тексереміз:

```swift
if user.isEmpty && password.isEmpty
```

Егер пайдаланушы аты мен құпиясөздің екеуі де бос болса, бірден `Empty` шығады.

Егер екеуі бірдей бос болмаса, онда келесі жолды тексеруді жалғастырамыз:

```swift
else if user.isEmpty || password.isEmpty
```

Мұндағы `||` «немесе» дегенді білдіреді.

Яғни шарттардың біреуі орындалса, бүкіл шарт орындалады.

Сондықтан пайдаланушы аты бос болса немесе құпиясөз бос болса, `else if` код блогы орындалады.

Код блогына кіргеннен кейін, нақты қай енгізу өрісі толтырылмағанын тереңірек тексереміз:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Егер алдыңғы екі шарттың ешқайсысы шын болмаса, бұл пайдаланушы аты бос емес және құпиясөз де бос емес дегенді білдіреді.

Сонда бағдарлама соңғы `else` бөлігіне кіреді:

```swift
print("Success")
```

Осылайша біз кіру енгізуінің ең негізгі тексеру логикасын аяқтаймыз.

## Пайдаланушы аты мен құпиясөзді тексеру

Жоғарыдағы логика тек «бірдеңе енгізілді ме, жоқ па» дегенді тексереді.

Бірақ шынайы қолданбада жай ғана мазмұн енгізу жеткіліксіз. Әдетте пайдаланушы аты мен құпиясөзді серверге жіберіп, тексеру де қажет болады.

Егер сервер пайдаланушы аты бар екенін және құпиясөз дұрыс екенін растаса, онда кіруге рұқсат береді; әйтпесе, кіру сәтсіз болғанын хабарлайды.

Бұл процесті жаттықтыру үшін, кодта уақытша аккаунт пен құпиясөз орнатып, «дұрыс кіру мәліметтерін» модельдей аламыз:

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

Пайдаланушы аты мен құпиясөз екеуі де бос болмаған кезде, олардың дұрыс екенін салыстырамыз:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

Мұндағы `==` «тең бе» дегенді білдіреді.

Мысалы:

```swift
user == userID
```

Бұл пайдаланушы енгізген пайдаланушы аты біз орнатқан дұрыс пайдаланушы атымен дәл бірдей ме дегенді білдіреді. Құпиясөзді тексеру де дәл осы логикамен жүреді.

Сондықтан енгізілген пайдаланушы аты мен құпиясөз орнатылған мәндерге тең болған кезде `Success` шығады; ал олардың біреуі тең болмаса, `Input error` шығады.

Осылайша біз қарапайым «жалған кіру тексеруін» аяқтадық.

Бұл әлі серверге қосылған шынайы кіру мүмкіндігі болмаса да, бізге мынаны түсінуге жеткілікті: **батырма түртілгеннен кейін әртүрлі шарттарға байланысты әртүрлі код орындала алады**.

## Қорытынды

Осы сабақта біз негізгі кіру көрінісін аяқтадық және әсіресе «енгізілген мазмұнды шарттарға қарай бағалау логикасын» үйренуге назар аудардық.

Бұл сабақтың негізгі мазмұны: жолдың бос екенін анықтау үшін `isEmpty` пайдалану және әртүрлі жағдайларды өңдеу үшін `if` операторларын логикалық операторлармен бірге қолдану.

Олардың ішінде екі өте маңызды логикалық оператор бар:

- `&&`: «және» дегенді білдіреді, екі шарт та орындалуы керек
- `||`: «немесе» дегенді білдіреді, бір шарттың орындалуы жеткілікті

Бұл шарттар соңында `true` немесе `false` бергенде, `if` операторы нәтижеге қарай әртүрлі код орындайды.

Осы кіру көрінісі арқылы біз «интерфейс + деректер + логикалық бағалау» үйлесетін даму тәсілімен таныса бастадық.

## Толық код

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Button("Sign in") {
                if user.isEmpty && password.isEmpty {
                    print("Empty")
                } else if user.isEmpty || password.isEmpty {
                    if user.isEmpty {
                        print("Please enter your username.")
                    } else if password.isEmpty {
                        print("Please enter your password.")
                    }
                } else {
                    if user == userID && password == passWord {
                        print("Success")
                    } else {
                        print("Input error")
                    }
                }
            }
            .buttonStyle(.borderedProminent)
            Spacer()
        }
        .padding(.top, 50)
    }
}

```
