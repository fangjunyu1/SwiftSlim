# Циклдер мен тізімдер

Бұл сабақта біз Swift тіліндегі `for` циклін және SwiftUI ішіндегі `ForEach` құрылымын үйренеміз.

Екеуі де массивтегі элементтерді ретімен, бір-бірден оқуға көмектеседі. Нақты әзірлеу барысында бізге көбіне бір топ деректерді бір-бірден алып, кейін оларды өңдеу немесе көрсету қажет болады, ал циклдер — осы жұмысты орындауға арналған маңызды құрал.

Біз веб-беттерді қарағанда немесе қолданбаны пайдаланғанда, көп тізімдер мен элементтерді жиі көреміз. Мұндай мазмұн әдетте алдымен `массивте` сақталады, содан кейін `цикл` арқылы бір-бірден оқылады, соңында интерфейсте көрсетіледі.

Мысалы, Google іздеу нәтижелерінің тізімі:

![Google](../../Resource/017_google.png)

Іздеу нәтижелері тізіміндегі әрбір элемент әдетте бір дерекке сәйкес келеді және цикл арқылы бір-бірден көрсетіледі.

Сондықтан `массив` пен `цикл` жиі бірге қолданылады. `Массив` бір топ деректі сақтауға жауап береді, ал `цикл` сол деректерді ретімен алып шығып, оларға бірдей әрекетті орындауға жауап береді.

Енді ең негізгі `for` циклінен бастайық.

## Кері санау

Біз кері санау функциясын модельдеп, `“5, 4, 3, 2, 1”` мәндерін ретімен шығарғымыз келеді деп елестетейік.

Оны жазудың ең тікелей жолы мынадай:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

Бұл тәсіл кері санауды көрсетуге мүмкіндік береді, бірақ мазмұн көп болса, код өте қайталанатын болып көрінеді және кейін өзгерту де ыңғайсыз болады.

Осындай кезде алдымен деректерді массивке салып, содан кейін `for` циклі арқылы оларды бір-бірден шығара аламыз:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Бұл жазылу түрі ықшамырақ және кеңейтуге де жеңілірек. Кері санаудағы сандар көп болса да, `print` жолдарын қайта-қайта жазудың қажеті жоқ, массивтегі әрбір элементті циклдің өзі автоматты түрде өңдейді.

Бұл кодты тексеру үшін оны батырмаға да орналастыруға болады:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

Батырманы басқанда, `for` циклі массивтегі тәртіп бойынша `countDown` ішіндегі әрбір бүтін санды бір-бірден алып, консольге шығарады.

Енді `for` циклінің қалай жұмыс істейтінін қарайық.

## for циклі

`for` циклі бір топ деректерді ретімен өңдеу үшін қолданылады. Ең жиі қолданылатын нысандар — массивтер мен диапазондар.

Негізгі жазылу түрі:

```swift
for элемент in массивНемесеДиапазон {
    орындалатынКод
}
```

Мысалы, жаңағы кері санау коды:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Мұнда `i` — дәл қазір алынған элементті білдіреді, ал `in` сөзінен кейін цикл жүретін дерек, яғни `countDown` массиві жазылады.

Бұл кодтың мағынасы: бағдарлама `countDown` массивінен әрбір элементті ретімен алып шығады. Әрбір элемент алынған сайын, оны уақытша `i` арқылы белгілейді де, фигуралы жақшаның ішіндегі кодты орындайды.

Орындалу процесін былай түсінуге болады:

* Алғашқы алынған элемент `5`, сондықтан осы сәтте `i` мәні `5`
* `print(i)` орындалады да, `5` шығады
* Кейін келесі элемент `4` алынады
* `print(i)` қайта орындалып, `4` шығады

Кейінгі `3`, `2` және `1` де дәл осылай бір-бірден орындалады.

![for](../../Resource/017_for.png)

Массивтегі барлық элементтер өңделіп болғаннан кейін, `for` циклі автоматты түрде аяқталады.

Біз `for` циклі арқылы массивтегі әрбір элементті ала аламыз және осындай кері санау сияқты функцияларды орындай аламыз.

**Ескерту**: `i` — for циклінде өте жиі қолданылатын атау, ол ағымдағы алынған элементті білдіреді, бірақ бұл міндетті емес. Оның орнына басқа ағылшынша атауларды да қолдануға болады.

Мысалы, алынған әрбір элементті белгілеу үшін `num` атауын қолдануға болады:

```swift
for num in countDown {
    print(num)
}
```

Фигуралы жақшаның ішінде де ағымдағы алынған элементті белгілеу үшін `num` қолданылуы керек.

### Диапазонды қайталау

`for` циклі тек массивтерде ғана емес, диапазондарда да қолданылады.

Мысалы, егер `50`-ге дейінгі оң бүтін сандардың қосындысын есептеу керек болса, онда `1` мен `50` арасындағы барлық бүтін сандарды бір-бірден қосу қажет.

Оны жазудың ең тікелей жолы мынадай болуы мүмкін:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

Бұл тәсілдің мәселесі — әрбір санды қолмен жазу керек. Бұл ыңғайсыз және нақты әзірлеу жағдайларына сәйкес келмейді.

Осындай кезде диапазон операторын `for` циклімен бірге қолдануға болады.

Алдыңғы “Операторлар” бөлімінде біз диапазон операторын үйрендік:

```swift
1...3   // 1, 2, 3
```

Мұндағы `...` — жабық диапазон, яғни басталу нүктесі мен аяқталу нүктесінің екеуін де қамтиды.

Сондықтан мұнда `1`-ден `50`-ге дейінгі бүтін сандар диапазонына `for` циклін қолдануға болады:

```swift
for i in 1...50 {
    
}
```

Бұл сандарды қосу керек, сондықтан қосындыны сақтайтын айнымалы да қажет:

```swift
var count = 0
```

Содан кейін циклдің әрбір қадамында қазіргі алынған санды `count` айнымалысына қосамыз:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

Бұл кодтың мағынасы:

* `count` ағымдағы қосындыны сақтау үшін қолданылады
* `for` циклі `1`-ден `50`-ге дейінгі бүтін сандарды бір-бірден алады
* Әрбір бүтін сан алынған сайын, `count += i` бір рет орындалады
* Цикл аяқталғаннан кейін, `count` ішінде соңғы қосынды сақталады

Осылайша біз `50`-ге дейінгі барлық оң бүтін сандардың қосындысын ала аламыз.

Сол сияқты, егер `1`-ден `100`-ге дейінгі бүтін сандардың қосындысын есептеу керек болса, тек диапазонды өзгерту жеткілікті:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` циклі тек массивтермен ғана емес, диапазонмен де жұмыс істей алады. Деректерді ретімен қайталап өңдеу мүмкіндігі әзірлеуде өте жиі кездеседі.

Бұл бізге көптеген қайталанатын жұмыстарды автоматты түрде орындауға көмектеседі, яғни әрбір санды қосу сияқты әрекеттерді қолмен жазудың қажеті болмайды.

## SwiftUI ішіндегі циклдер

Жоғарыдағы `for` циклі — Swift тілінің өз бөлігі.

Бірақ SwiftUI ішінде view құрылымының ішінде `for` арқылы тікелей view құра алмаймыз:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

Себебі `VStack` ішінде SwiftUI view-лары болуы керек, ал кәдімгі `for` циклі өздігінен view емес.

Егер SwiftUI ішінде массив мазмұнына байланысты интерфейсті қайталап құрғымыз келсе, онда `ForEach` қолдануымыз керек.

Мысалы, егер топтағы барлық мүшелердің аттарын көрсеткіміз келсе, былай жаза аламыз:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

Көрсетілу нәтижесі:

![for1](../../Resource/017_for1.png)

Одан кейін әрбір `Text` үшін бірдей модификаторларды қосамыз:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Енді әрбір `Text` кішірек әрі қалың қаріппен көрсетіледі.

Көрсетілу нәтижесі:

![](../../Resource/017_for2.png)

Модификаторлар интерфейсті әдемірек етеді, бірақ кодты ұзартып, қайталанатын етеді.

Егер аттар көбейе берсе немесе кейін қаріп, түс және басқа стильдерді бірдей өзгерту қажет болса, онда бірдей кодты қайта-қайта жазып, өзгерту керек болады. Бұл ыңғайсыз әрі қолдауға қиын.

Осындай кезде алдымен бұл аттарды массивке салып, кейін SwiftUI ішіндегі `ForEach` арқылы массив деректеріне сүйене отырып view-ларды қайталап жасай аламыз:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

Бұл код әлдеқайда ықшам. `ForEach` массивтегі әрбір элементті бір-бірден оқып, сәйкес мазмұнды интерфейсте көрсетеді.

Кейін жаңа аттар қосылатын болса, тек `names` массивінің ішіндегі мәндерді өзгерту жеткілікті. Әр жолы қайтадан `Text` пен күрделі модификаторларды қосудың қажеті жоқ.

## ForEach

`ForEach` — SwiftUI ішіндегі view-ларды қайталап құруға арналған құрылым. Оны SwiftUI ішіндегі `for` цикліне ұқсас құрылым ретінде түсінуге болады.

Негізгі қолданылуы:

```swift
ForEach(массив, id: \.self) { элемент in
    SwiftUI коды
}
```

Мысалы, жаңағы топ атаулары:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Бұл кодтың мағынасы:

`names` — қайталанып шығатын массив, яғни көрсетілетін дерек.

`id:\.self` — массивтің элементінің өзін әрбір элементті ажырату үшін қолдану дегенді білдіреді.

Қазіргі мысалда массивте жолдар сақталған:

```swift
["Sam", "John", "Wathon", "Bob"]
```

Сондықтан SwiftUI әртүрлі мазмұнды ажырату үшін `"Sam"`, `"John"`, `"Wathon"` және `"Bob"` жолдарының өзін тікелей қолданады.

Бастапқы кезеңде жиі кездесетін `String` және `Int` сияқты қарапайым массивтер үшін әдетте былай жазуға болады:

```swift
id: \.self
```

Қазіргі кезде осы бір жазылу түрін есте сақтау жеткілікті. `id` ережелерінің басқа түрлерін тереңірек түсінудің қажеті жоқ.

`item in` ішіндегі `item` — дәл қазір алынған элементті білдіреді.

Мысалы, бірінші қайталауда `item` — `"Sam"`, ал екінші қайталауда `item` — `"John"`.

Сондықтан:

```swift
Text(item)
```

біртіндеп мынаған айналады:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

Яғни бұл кодтың міндеті — `names` массивінен әрбір атты бір-бірден алып, әрбір ат үшін `Text` view құру.

### Орындалу процесі

Орындалу процесін былай түсінуге болады:

* `ForEach` алдымен `names` массивіндегі бірінші элемент `"Sam"` мәнін оқиды
* `item` уақытша `"Sam"` мәнін білдіреді
* `Text(item)` `"Sam"` мәнін көрсетеді
* Кейін келесі `"John"` элементін оқиды
* Сосын дәл сол view кодын қайта орындайды

Соңында массивтегі барлық аттар көрсетіледі.

Бұл тәсілдің артықшылығы — аттар көбейген кезде көптеген `Text` жолдарын қайта-қайта жазудың қажеті жоқ, тек массив мазмұнын өзгерту жеткілікті.

### Суреттерді қайталау

`ForEach` тек мәтінді ғана емес, суреттерді, түстерді немесе басқа SwiftUI view-ларын да көрсете алады.

Мысалы, біз 4 сурет дайындаймыз:

```text
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

Осы төрт суретті Xcode ішіндегі Assets қалтасына сүйреп апарыңыз.

![image](../../Resource/017_for3.png)

Егер `ForEach` қолданбасақ, онда әрбір суретті қолмен жазу керек болады:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

Көрсетілу нәтижесі:

![image](../../Resource/017_for4.png)

Бұл тәсілмен функцияны орындауға болады, бірақ код салыстырмалы түрде ұзын болады.

Егер суреттер саны көбейсе, онда `Image` кодын да қайта-қайта көбірек жазуға тура келеді.

Егер кейін барлық суреттердің өлшемін бірдей өзгерту керек болса, оларды бір-бірден өзгерту қажет болады.

Осындай кезде сурет атауларын массивке салып, кейін `ForEach` арқылы сурет view-ларын қайталап жасай аламыз:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Мұндағы `images` — жолдық массив, ал массивтегі әрбір элемент — сурет атауы.

`ForEach(images, id: \.self)` — `images` массивіндегі әрбір элементті бір-бірден оқу, элементтің өзін әр тармақты ажырату үшін пайдалану және ағымдағы элементке сәйкес view құру дегенді білдіреді.

Мысалы, егер бірінші оқылған элемент `"430F9BEF"` болса, онда:

```swift
Image(item)
```

мынамен бірдей болады:

```swift
Image("430F9BEF")
```

Келесі суреттер де дәл осылай бір-бірден көрсетіледі.

![image](../../Resource/017_for4.png)

Мұнда да біз әлі де мына жазуды қолданамыз:

```swift
id: \.self
```

Себебі бұрынғыдай: `images` массивінің элементтері де қарапайым жолдық тип болғандықтан, бастапқы кезеңде элементтің өзін ажырату белгісі ретінде қолдануға болады.

Мұны қарапайым тұрақты ереже ретінде түсінуге болады:

**`ForEach` `String` немесе `Int` сияқты қарапайым массивті қайталағанда, әдетте `id: \.self` деп жазылады.**

### Сурет қосу

Кейін жаңа сурет қосу керек болса, тек массивтің мазмұнын өзгерту жеткілікті:

```text
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

Жаңа `Image` кодын қайта-қайта жазу қажет болмайды.

Егер суреттер көп болса, оны `ScrollView`-пен бірге қолдануға болады. Әйтпесе мазмұн экраннан асып кетсе, барлығын толық көру мүмкін болмайды.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### Суреттерді өзгерту

Барлық суреттердің стилін бірдей өзгерту керек болса, тек `ForEach` ішіндегі кодты өзгерту жеткілікті.

Мысалы, енін `80` етіп өзгертейік:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

Осылайша, барлық суреттер бір уақытта өзгереді де, оларды бір-бірден өзгертудің қажеті болмайды.

Бұл — `ForEach` құрылымының маңызды қызметтерінің бірі:

**Егер бірнеше view құрылымы бірдей болып, тек мазмұны ғана өзгеше болса, view кодын бір рет қана жазып, не көрсетілетінін массивтегі деректердің өзіне шештіруге болады.**

## Қорытынды

Бұл сабақта біз `for` циклін және `ForEach` құрылымын үйрендік.

`for` циклі массив немесе диапазон элементтерін ретімен өңдеу үшін қолданылады және көбірек деректерді өңдеуге бағытталған.

`ForEach` SwiftUI ішінде массив мазмұнына сүйеніп view-ларды қайталап құру үшін қолданылады және көбірек интерфейсті көрсетуге бағытталған.

Қазіргі бастапқы кезеңде `ForEach`-тің ең жиі қолданылатын түрі мынау:

```swift
ForEach(массив, id: \.self) { элемент in
    SwiftUI коды
}
```

Мұндағы `id: \.self` — әрбір элементті ажырату үшін элементтің өзін қолдану деген сөз.

Егер массив мазмұны `String` және `Int` сияқты қарапайым типтерден тұрса, әдетте осы бір жазылу түрінен бастауға болады.

Кейін веб-беттерді немесе қолданбаларды қарағанда, қайталанып келетін мәтіндер, суреттер немесе басқа тізім мазмұнын көрсеңіз, SwiftUI ішінде мұндай мазмұн әдетте алдымен массивке салынып, содан кейін `ForEach` арқылы бір-бірден көрсетіледі деп ойлай аласыз.
