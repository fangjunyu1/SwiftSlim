# SwiftUI view өмірлік циклі

Алдыңғы сабақта біз `@State` айнымалысының view өмірлік цикліне тәуелді екенін түсіндік және `onAppear`-дің базалық қолданысын да үйрендік.

Бұл сабақта біз SwiftUI view өмірлік циклін тереңірек түсінеміз: view қалай жасалатынын, қалай экранда көрсетілетінін, view ішіндегі айнымалылардың қалай жасалып, қайта жасалатынын қарастырамыз. Сонымен қатар `onAppear`, `onDisappear` және `init` конструкторын үйренеміз.

Осы білімдердің негізінде біз SwiftUI view құру процесі туралы тұтас түсінік қалыптастырамыз.

## Қолданбаның іске қосылу реті

iOS жобасын жасағанда, Xcode әдепкі түрде екі файл жасайды:

```
ContentView.swift
ЖобаАтауы + App.swift
```

Мұндағы “ЖобаАтауы + App.swift” — бүкіл қолданбаның кіру файлы.

Мысалы:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

### Кодтың орындалу ағымы

Біз App-ті іске қосқанда (симуляторда немесе нақты құрылғыда), жүйе алдымен `@main` кілт сөзін іздейді:

```swift
@main
struct SwiftSlimTestApp: App {...}
```

Кіру файлы расталғаннан кейін, код орындала бастайды.

Алдымен `App` құрылымына кіреді, `body` ішіндегі кодты орындайды, содан кейін `WindowGroup` жасалып, оның ішіндегі `ContentView` жүктеледі.

### WindowGroup не үшін керек

`WindowGroup` терезелерді басқару үшін қолданылады:

```swift
WindowGroup {
    ContentView()
}
```

iPad / macOS жүйесінде бірнеше терезе қолдауы бар, ал iPhone әдетте тек бір терезені қолданады.

Сондықтан iPhone ішінде `WindowGroup` негізінен экранда көрсетілетін алғашқы интерфейсті басқарады.

### View жүктелу процесі

Жүйе `ContentView`-ді тапқаннан кейін:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
    }
}
```

SwiftUI `body` ішіндегі кодты орындайды, содан кейін қайтарылған view құрылымына (`VStack`, `Text` және т.б.) сүйеніп интерфейсті құрады және экранда көрсетеді.

Осы қадамдар аяқталған соң, біз view ішінде `ContentView`-ді көреміз.

![ContentView](../../Resource/016_view1.png)

Маңыздысы, `body`-дің қызметі — view құру, ал view-ді сақтау емес.

Бұл дегеніміз, view жаңарған сайын SwiftUI `body`-ді қайта есептеп, жаңа view құрылымын қайта жасайды.

### Preview view логикасы

Xcode preview (`Canvas`) және нақты орындалу — екі бөлек механизм.

Мысалы, кіру файлына бір жөндеу шығуын қосайық:

```swift
@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .onAppear {
                    print("Go to ContentView")
                }
        }
    }
}
```

Xcode ішінде `ContentView`-ді preview жасағанда, орындалу `@main` кіру нүктесінен басталмайды, сондықтан кіру файлындағы `onAppear` шақырылмайды.

Бірақ егер қолданбаны симуляторда немесе нақты құрылғыда іске қоссақ, жүйе `@main` арқылы бүкіл App-ті толық процеспен орындайды, сол себепті кіру нүктесіндегі `onAppear` та шақырылып, жөндеу ақпараты шығады.

Мұндағы негізгі айырмашылық: Xcode preview — тек “жергілікті рендер”, ол тек ағымдағы view-ді көрсету үшін қолданылады; ал симулятор мен нақты құрылғы — “толық орындалу ортасы”, сондықтан бүкіл App орындалады.

Сондықтан App функциясын тексергенде, негізгі өлшем симулятор не нақты құрылғы болуы керек, өйткені Xcode preview толық орындалу ортасын бермейді.

## View өмірлік циклі

Қазіргі кезеңде біздің бүкіл кодымыз `ContentView` ішінде шоғырланған. Бірақ нақты қолданбада бір App ішінде әдетте бірнеше view болады және олар әртүрлі сценарийге қарай ауысып тұрады.

Мысалы, “ақша жинайтын құты” жобасында:

![PiggyBank](../../Resource/016_view.png)

Егер пайдаланушы “ақша салу” әрекетін таңдаса, `SaveView` view-і көрсетіледі; ақша салу аяқталған соң немесе view жабылса, `SaveView` алынып тасталады.

Осы процесс — view өмірлік циклі: view жасалады, экранда көрсетіледі, содан кейін жойылады.

### Қолданбаны жабу

App жабылған кезде, барлық view жойылады және оларға қатысты күй деректері де жоғалады.

Сондықтан `ContentView` және басқа view-лер жадтан алынып тасталады, ал бүкіл App-тің жұмыс күйі тазаланады.

## View ішіндегі айнымалылардың өмірлік циклі

SwiftUI-де айнымалының өмірлік циклі әдетте view-мен байланысты.

Мысалы:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"
    var body: some View {
        var num = 10
        VStack {
            Text("num:\(num)")
            Text("name:\(name)")
        }
    }
}
```

### @State айнымалысы

`@State` арқылы жарияланған айнымалы view өмірлік цикліне байланысады.

```swift
@State private var name = "FangJunyu"
```

View жасалғанда `name` де жасалады; view жойылғанда `name` де жойылады. 

Міне, сондықтан `UserDefaults` сияқты тәсілдер арқылы деректі тұрақты сақтау қажет болады.

### body ішіндегі айнымалы

`body` ішінде анықталған айнымалы:

```swift
var num = 10
```

Оның өмірлік циклі `body` орындалу процесіне байланған.

SwiftUI күйі өзгергенде, мысалы:

```swift
@State private var name = "FangJunyu"
name = "Hu"
```

`name` өзгерген кезде, `@State` деректің өзгергенін байқап, SwiftUI-ге view-ді қайта есептеу туралы хабар береді, сонда `body` қайта есептеледі.

`body` қайта есептелгенде, оның ішіндегі барлық код қайта орындалады, демек `body` ішіндегі айнымалылар да (`num` сияқты) қайта жасалады.

Сондықтан `body` ішінде күрделі айнымалылар анықтау ұсынылмайды.

Өйткені view әр жаңарған сайын `body` ішіндегі айнымалылар қайта жасалып, есептеу шығынын арттырады және өнімділікке әсер етеді.

SwiftUI-де әртүрлі типтегі дерек әртүрлі тәсілмен басқарылуы керек: view өмірлік цикліне ілесетін деректерді `@State` арқылы сақтау керек, ал уақытша есептеу деректерін `body` ішінде жазуға болады.

## onAppear және onDisappear

Алдыңғы сабақта біз `onAppear` туралы үйрендік — view экранда көрсетілгенде ол шақырылады.

```swift
.onAppear {}
```

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
        .onAppear {
            print("Show ContentView")
        }
    }
}
```

Xcode preview кезінде де, нақты іске қосқанда да `onAppear` ішіндегі жөндеу шығуын көруге болады.

### onDisappear

`onAppear`-ге сәйкес келетін екінші modifier — `onDisappear`:

```swift
.onDisappear {}
```

View жабылған кезде, `onDisappear` шақырылады.

Мысалы:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

View экраннан алынғанда, `onDisapper` ішіндегі код орындалады.

Ескерту: қазіргі кезеңде қолданып жүрген `ContentView` — қолданбаның түбірлік view-і, сондықтан оны тікелей жабуға немесе экраннан алып тастауға болмайды. Сол себепті бұл кезеңде `onDisappear` әсерін тікелей бақылау мүмкін емес.

Кейінірек бет ауысуын немесе жаңа view ашуды үйренгенде, `onDisappear` қашан шақырылатынын байқай аламыз.

## Жасалу және көрсетілу логикасы

Маңыздысы, view-дің жасалуы мен экранда көрсетілуі — екі бөлек кезең.

View жасалған кезде, `struct` конструкторы шақырылады:

```swift
init() {}
```

SwiftUI view — `struct` құрылымы болғандықтан:

```swift
struct ContentView: View { ... }
```

View экземпляры жасалған сәтте `init` конструкторы орындалады, ал view экранда көрсетілген кезде ғана `onAppear` шақырылады.

Мысалы:

```swift
struct ContentView: View {
    init() {
        print("Create ContentView")
    }
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
        .onDisappear {
            print("Close ContentView")
        }
    }
}
```

Xcode ішінде preview жасағанда, мынадай жөндеу шығуын көруге болады:

```swift
Create ContentView
Show ContentView
```

Бұл алдымен `init` орындалып, view жасалатынын, содан кейін `body` ішіндегі код есептелетінін, ең соңында view экранда көрсетіліп, `onAppear` шақырылатынын білдіреді.

Сондықтан view-дің жасалу кезеңі мен көрсетілу кезеңі бір-бірінен бөлек екенін түсіну қажет.

### init конструкторы

`init` — Swift-тің базалық синтаксисі, оны `struct` пен `class` екеуінде де анықтауға болады және объектіні инициализациялау үшін қолданылады.

```swift
init() {}
```

SwiftUI-де view жасалған кезде, жүйе сол view-дің `init` әдісін шақырады. Бұл параметр қабылдау, деректі инициализациялау сияқты жұмыстар үшін пайдалы.

Егер `init`-ті өзіңіз қолмен жазбасаңыз, Swift `struct` үшін әдепкі конструкторды автоматты түрде жасап береді.

`ContentView` сияқты SwiftUI view-лер үшін, view жасалғанда `init`, ал view экранда көрсетілгенде `onAppear` орындалады.

Сондықтан `init` — view жасалған сәтте орындалатын конструктор, әрі кейін параметр беру немесе деректі инициализациялау кезінде маңызды рөл атқарады.

## Қорытынды

Бұл сабақта біз қолданбаның іске қосылу ретін үйрендік: кіру файлынан бастап `ContentView` файлына дейінгі орындалу ағымын қарастырдық.

Сонымен бірге SwiftUI view өмірлік циклін түсіндік: view жасалғанда `init` орындалады, экранда көрсетілгенде `onAppear`, ал view алынып тасталғанда немесе жабылғанда `onDisappear` орындалады.

Бұған қоса, біз view жаңарту механизмін де үйрендік: view күй арқылы басқарылады, ал `@State` сияқты күй өзгергенде, SwiftUI view-ді жаңартып, `body`-ді қайта есептейді. Сол кезде `body` ішіндегі айнымалылар да қайта жасалады.

Айнымалының өмірлік циклі view өмірлік цикліне байланысады, ал `body` ішіндегі уақытша айнымалылар әр жаңарту сайын қайтадан жасалады.

Осы өмірлік цикл мен айнымалы мінез-құлқын түсіну кодты жақсырақ ұйымдастыруға және қолданба логикасын анығырақ құруға көмектеседі.
