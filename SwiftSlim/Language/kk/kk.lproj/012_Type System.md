# Түр жүйесі

Алдыңғы сабақтарда біз айнымалыларды үйрендік және айнымалы әртүрлі типтегі деректерді сақтай алатынын білдік.

Мысалы:

```swift
var hello = "Hello, World"
```

Мұндағы `hello` мәтін сақтап тұр, сондықтан оның типі — `String`.

Келесі қадамда біз тип (Type) ұғымын, айқын тип жариялауды және типті автоматты түрде анықтауды жүйелі түрде үйренеміз, сонда айнымалының табиғатын тереңірек түсіне аламыз.

## Тип деген не

Swift-те әрбір мәннің нақты типі болады. Тип бұл мәннің не екенін және ол не істей алатынын анықтайды.

Мысалы:

- `42` — `Int` (бүтін сан)
- `"Hello"` — `String` (жол)
- `true` — `Bool` (логикалық мән)

Бұлар әртүрлі типтерге жатады және қолданылуы да әртүрлі.

## Swift-тегі жиі кездесетін базалық типтер

Бастапқы кезеңде ең жиі кездесетін типтер мыналар:

- `String`：жол (мәтін)
- `Int`：бүтін сан
- `Double`：қалқымалы нүктелі сан (бөлшекпен, дәлдігі жоғары)
- `Bool`：логикалық мән (`true/false`)

Бұдан бөлек `Float` та бар (қалқымалы нүктелі сан), бірақ оның дәлдігі `Double`-дан төмен, сондықтан сирек қолданылады; сондай-ақ `Character` бар, ол бір таңбаны білдіреді, мысалы `"A"`, `"!"` және т.б.

### Типті жариялау (айқын тип)

Swift-те айнымалы жарияланған кезде типті қолмен көрсетуге болады:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

Айнымалы атауынан кейін `: Тип` жазу арқылы тип жарияланады.

Массив типі `: [Тип]` түрінде жазылады:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

Бұл кодта `scores` массиві тек `Int` типті элементтерді, ал `tags` тек `String` типті элементтерді сақтай алады.

Айқын тип жариялау кодтың не істейтінін анық көрсетеді және кейбір жағдайларда типті қате анықтаудан сақтайды.

## Типті автоматты анықтау

Көп жағдайда типті арнайы көрсетпей де жазуға болады:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"    // String
var isLogin = false  // Bool
var scores= [90, 85, 78]	// [Int]
var tags = ["Swift", "iOS"]	// [String]
```

Себебі Swift компиляторы бастапқы мәнге қарай типті автоматты түрде шығарады.

## Айқын жариялау және автоматты анықтау

Алдыңғы сабақтарда біз:

```swift
var age: Int = 18
```

сияқты айқын жариялауды емес, алдымен типті автоматты анықтауды қолдандық:

```swift
var age = 18
```

Бұл мысалда екі жазылу бірдей мағына береді, `age` екі жағдайда да `Int` болып анықталады. Соның ішінде автоматты анықтау жазылуы қысқа әрі ықшам.

Алғашқы кезеңде айқын жариялауға ерекше мән берілмеуінің себебі — автоматты анықтау кодтағы артық ақпаратты азайтады, сол арқылы жаңадан бастаған кезде түсіну шығынын төмендетеді.

## Неге тип қажет

Swift — қатаң типтелген тіл (`Strongly Typed Language`).

Бұл дегеніміз: айнымалының типі анықталғаннан кейін, оны өз бетінше басқа типке ауыстыруға болмайды.

Мысалы:

```swift
var age: Int = 18

age = 20	// Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

Бірінші `20` — `Int` типі, сондықтан оны `age`-ке бере аламыз; ал екінші `"20"` — `String`, ол `Int`-пен сәйкес келмейді, сондықтан қате шығады.

`nums` — `[Int]` типіндегі массив, демек ол тек бүтін сандарды сақтай алады және жолды бірге араластыра алмайды.

Тип деректі қолдануды шектеп, компиляция кезеңінде тип сәйкес келмеу мәселесін табуға мүмкіндік береді. Соның арқасында тип араласып кетуден туатын BUG азайып, кодтың тұрақтылығы мен қолдауға ыңғайлылығы артады.

## Қашан типті міндетті түрде айқын жариялау керек

Swift көбіне типті өзі анықтай алады, бірақ кей жағдайларда компилятор оны шығара алмайды немесе қате шығарады. Мұндайда типті қолмен жариялау керек.

**1. Бос массив**

Бос массив жасағанда, типті міндетті түрде айқын көрсету керек:

```swift
var nums: [Int] = []
```

Егер тип жазылмаса:

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

Компилятор бос массивтен оның элемент типін шығара алмайды, сондықтан компиляция қатесі болады.

**2. Қате тип анықтаудан сақтану**

```swift
var value = 10   // Int
```

Бұл мысалда `10` бөлшек сан (`Double`) ретінде де қарастырылуы мүмкін, бірақ нүкте болмағандықтан компилятор оны әдепкі түрде `Int` деп анықтайды.

Егер `value` бөлшек сан болсын десек, типті айқын көрсету қажет:

```swift
var value: Double = 10   // Double
```

Егер автоматты анықтау нәтижесі күткендей болмаса, типті айқын жариялау немесе литерал формасын өзгерту арқылы дұрыс типті қамтамасыз ету керек.

**3. Қолданушы анықтаған тип**

Кейін біз өзіміздің типтерімізді де үйренеміз. Ондай кезде типті айқын жариялау жиі қажет болады:

```swift
var info: GameInfo
var users: [User] = []
```

Себебі бос мәннен компилятор типті автоматты түрде шығара алмайды.

## Типті түрлендіру

Swift-те әртүрлі типтер арасында автоматты түрлендіру болмайды, оны қолмен жасау керек.

Мысалы, екі түрлі типтегі санды қосу:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b	// Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

Бұл кодта `a` — `Int`, `b` — `Double`, яғни типтері әртүрлі, сондықтан оларды тікелей есептеуге болмайды.

Есептеу алдында олардың бірін екіншісінің типіне келтіру қажет:

```swift
var result = Double(a) + b
```

Бұл жерде `a` (`Int`) алдымен `Double` типіне айналдырылып, содан кейін `b`-мен қосылады, ал нәтиже `result`-қа беріледі.

Маңыздысы, типті түрлендіру бастапқы айнымалының типін өзгертпейді. Мысалы, `Double(a)` `a`-ның `Int` типін өзгертпейді, тек есептеуге қатысатын жаңа `Double` мәнін жасайды.

Басқа типтерді де осылай түрлендіруге болады, яғни мақсатты тип + `()` жазылып, оның ішіне түрленетін мән беріледі. Мысалы:

```
Int()、Double()、String()
```

Мысал:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

Маңыздысы, барлық тип түрлендіру бірдей нәтиже бермейді. Мысалы, `Double → Int` түрленгенде, бөлшек бөлік тікелей кесіліп тасталады, дөңгелектелмейді.

## Тип бүркеншік аты

Swift-те бұрыннан бар типке жаңа бүркеншік ат беруге болады, бұл код мағынасын анығырақ етеді:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

Мұндағы `UserID` негізінде бәрібір `Int`, яғни типтің өзі өзгерген жоқ, тек атауы мағыналырақ болды.

Код көлемі үлкен, типтер күрделі жобаларда typealias кодтың оқылымдылығын жақсартады.

## Типке мысал

Төменде типтің нақты әзірлеудегі рөлін қарапайым мысал арқылы көрсетеміз.

### Айнымалы жасау

Алдымен `age` айнымалысын жасаймыз:

```swift
var age = 20
```

Оны SwiftUI ішінде көрсету және өзгерту қажет болғандықтан, `@State` арқылы жариялаймыз:

```swift
@State private var age = 20
```

`private` туралы бұған дейін айттық, ол бұл айнымалы тек ағымдағы view ішінде ғана қолданылатынын білдіреді. Кейінірек оны тереңірек үйренеміз.

### SwiftUI ішінде дерек көрсету

SwiftUI ішінде `age` айнымалысын көрсету:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

Қазір view ішінде мынадай жазу көрінеді:

```swift
age: 20
```

### Деректі өзгерту

Енді `age` мәнін өзгерту үшін екі батырма қосамыз:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### Іске қосу нәтижесі

`changeAge1` батырмасын басқанда, `age` мәні 18 болады:

```swift
age = 18
```

Себебі `18` — `Int` типі, ал `age` де `Int`, сондықтан меншіктеу сәтті орындалады.

`changeAge2` қате береді:

```swift
age = "18"
```

Қате туралы ақпарат:

```
Cannot assign value of type 'String' to type 'Int'
```

Себебі `"18"` — `String` типі (жол әрқашан `""` ішінде жазылады), ал `age` — `Int`, демек типтері сәйкес емес және мән беру мүмкін емес.

### Неге қате шығады?

Мұндағы негізгі жол:

```swift
@State private var age = 20   // Int
```

Мұнда тип айқын көрсетілмегенімен, бастапқы мән `20` — `Int`, сондықтан компилятор `age`-ті автоматты түрде `Int` деп анықтайды.

Тип анықталғаннан кейін, оған басқа типтің мәнін беруге болмайды.

Оның үстіне, `changeAge2` батырмасы “орындалу кезінде” емес, компиляция кезеңінде-ақ қате береді, сондықтан код мүлде компиляциядан өтпейді.

Бұл да Swift-тің қатаң тип жүйесінің артықшылықтарының бірі: тип қателерін код жазу кезінде-ақ ерте табуға болады.

### Дұрыс жазылуы

Қате кодты өшіргеннен кейін:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

`changeAge1` батырмасын басқанда, view мынадай болып жаңарады:

```
age: 18
```

### Типті айқын жариялау (міндетті емес)

Анықтау кезінде типті нақты көрсетіп жазуға да болады:

```swift
@State private var age: Int = 20
```

Мұндай жазылу типті анықырақ етеді, бірақ қарапайым сценарийлерде ол міндетті емес.

## Қорытынды

Бұл сабақта негізінен Swift-тің тип жүйесі және оның нақты әзірлеудегі рөлі қарастырылды.

Swift-те әрбір мәннің нақты типі болады, ал тип деректің мағынасын және оған қандай әрекеттер жасауға болатынын анықтайды.

Swift — қатаң типтелген тіл, яғни айнымалының типі анықталғаннан кейін, оған басқа типтің мәнін беруге болмайды. Бұл шектеу қателерді компиляция кезеңінде анықтауға мүмкіндік береді, сол арқылы орындалу кезіндегі проблемаларды азайтып, код қауіпсіздігін арттырады.

Айнымалы типі бастапқы мән арқылы автоматты түрде анықталуы мүмкін немесе айқын жариялануы мүмкін. Кейбір жағдайларда (бос массив, қате тип анықталуы, қолданушы анықтаған тип) компилятор типті дұрыс шығара алмайды, сондықтан ондайда типті міндетті түрде айқын көрсету керек.

Әртүрлі типтер арасында автоматты түрлендіру болмайды, оны `Int()`, `Double()`, `String()` сияқты тәсілмен қолмен жасау қажет.

Типтер бүркеншік атты да қолдайды, әсіресе күрделі жобаларда typealias оқылымдылықты жақсартады.

Тип — тек дерек белгісі ғана емес, ол бір шектеу механизмі де. Ол деректің қолданылуын қауіпсіз, нақты және басқарылатын етеді.

## Қосымша білім - Optional тип

Тип түрлендіру кезінде кейбір операциялар сәтсіз аяқталуы мүмкін.

Мысалы, `String`-ті `Double`-ға айналдыру:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

Мұндағы `Double(input)` қайтарған мән жай `Double` емес, ол:

```
Double?
```

Бұл — Optional тип.

### Неге Optional тип қайтарылады?

Себебі жол мазмұны әрдайым белгілі емес: `"3.14"` `Double`-ға айналады, ал `"Hello"` `Double`-ға айналмайды.

Яғни бұл операция сәтті болуы да, сәтсіз болуы да мүмкін.

Сондықтан Swift осындай белгісіздікті көрсету үшін Optional типті қолданады.

### Optional тип деген не?

Optional типтің мағынасы: мән болуы да мүмкін, болмауы да мүмкін.

Мысалы:

```
Double?
```

Бұл тип егер мән болса, басып шығарғанда `Optional(...)` деп көрінетінін, ал мән болмаса `nil` қайтаратынын білдіреді.

Маңыздысы, `nil` — "мән жоқ" дегенді білдіреді, ол 0 немесе бос жол емес.

### Мысал

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Шығатын нәтиже:

```
pi:Optional(3.14)
```

Бұл жолдың `Double`-ға сәтті айналғанын, нәтиже `3.14` екенін көрсетеді. Бірақ типі `Double?` болғандықтан, ол `Optional` ішінде оралған.

Егер түрлендіру сәтсіз болса:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

Шығатын нәтиже:

```
pi: nil
```

Себебі `"Hello"` `Double` типіне айнала алмайды, сондықтан түрлендіру сәтсіз аяқталып, `nil` қайтарады.

### Optional(...) деген не?

Optional типті тікелей басып шығарғанда:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift оның “debug description”-ын көрсетеді:

```
Optional(3.14)
```

Бұл нақты деректің бір бөлігі емес, тек ағымдағы мән Optional тип екенін және ішінде нақты мән бар екенін білдіретін ескерту.

### Optional-ды ашу

Нақты әзірлеуде бізге Optional ішінен шын мәнді алу қажет болады. Бұл процесс unwrapping деп аталады.

Жиі қолданылатын тәсілдердің бірі — `??` арқылы әдепкі мән беру:

```
??
```

Бұл — Nil-Coalescing Operator.

Мысалы:

```swift
name ?? "FangJunyu"
```

Бұл кодтың мағынасы: егер `name` мәнге ие болса, сол мәнді қайтарады; егер `nil` болса, әдепкі `"FangJunyu"` мәнін қолданады.

Маңыздысы, әдепкі мәннің типі Optional ішіндегі негізгі типпен бірдей болуы керек.

### ?? арқылы әдепкі мән беру

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")	// 30
```

Шығатын нәтиже:

```
pi: 30
```

`input` `Double` типіне сәтті айналғанда, нақты мән шығады. Егер айналдыру сәтсіз болса, `??` кейінгі әдепкі мәнді береді.

Егер айналдыру сәтсіз болса:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

Шығатын нәтиже:

```
pi: 10
```

Мұнда `input` `Double` типіне айналғанда сәтсіздікке ұшырап, `nil` қайтарады, ал `??` осы `nil` үшін әдепкі мәнді береді.

Optional тип — “мән болуы да, болмауы да мүмкін” деген жағдайды білдіру үшін қолданылады.

Типті түрлендіру сияқты сәтсіз болуы мүмкін операцияларда Swift Optional қайтарады, сол арқылы бағдарламаның қауіпсіздігін қамтамасыз етеді.

Ал бізге ондағы мәнді қолдану керек болғанда, `??` арқылы әдепкі мән беріп, `nil` жағдайында да ақылға қонымды нәтиже ала аламыз.
