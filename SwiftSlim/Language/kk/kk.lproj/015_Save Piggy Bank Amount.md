# Ақша жинайтын құтыдағы соманы сақтау

Алдыңғы сабақта біз қарапайым “ақша жинайтын құтыны” жасадық. Бұл сабақта сол құтыдағы соманы тұрақты түрде сақтауды үйренеміз.

Бұл сабақтың негізгі тақырыптары — `UserDefaults` және `@AppStorage` property wrapper-і. Сонымен қатар `onAppear` рөлін және Optional типті (`nil`) қолдану тәсілін де кеңейте түсіндіреміз.

Маңыздысы, бұл сабақта алдыңғы сабақта жасалған “ақша жинайтын құты” кодын жалғастырып қолдану қажет.

## Тұрақты сақтау мәселесі

Қазір “ақша жинайтын құтының” интерфейсі өте қарапайым және қолдануға ыңғайлы.

![Piggy Bank](../../Resource/014_view.png)

Бірақ бір айқын мәселе бар: view жаңарған сайын немесе қолданба жабылған сайын “ақша жинайтын құтыдағы” жалпы сома қайтадан 0 болып кетеді, ал алдыңғы жинақтың бәрі жоғалады.

Бұл дерек шын мәнінде сақталмағанын білдіреді. Басқаша айтқанда, қазіргі “ақша жинайтын құты” тек уақытша деректерді ғана сақтай алады.

### Неге тұрақты сақталмайды?

Себебі біз `@State` арқылы жарияланған айнымалыны қолданып отырмыз:

```swift
@State private var amount = 0
```

`@State` арқылы жарияланған айнымалының өмірлік циклі толығымен view-ге тәуелді.

View құрылған кезде `amount` 0 болып инициализацияланады, ал view жойылғанда `amount` та жойылады.

Мұндағы `amount` сақтап тұрған дерек тек жедел жадта ғана өмір сүреді, құрылғының өзіне сақталмайды.

Біз `amount` өмірлік циклі view-мен бірге шектелгенін қаламаймыз, сондықтан деректі тұрақты сақтау механизмін іске асыру керек. Яғни деректі құрылғыға сақтау қажет.

### “Деректі тұрақты сақтау” деген не

Деректі тұрақты сақтау дегенді “уақытша жадтағы” деректі “құрылғы жадына” сақтау деп түсінуге болады.

Сонда view жабылса да, қолданба жабылса да, дерек жоғалмайды.

Swift әзірлеуде қарапайым “дерек тұрақты сақтау” үшін `UserDefaults` қолдануға болады, ал күрделі деректер үшін `SwiftData` немесе `CoreData` қажет.

Бұл сабақта біз алдымен ең қарапайым `UserDefaults`-ті үйренеміз.

## UserDefaults

`UserDefaults` — жеңіл салмақты key-value деректерді сақтау үшін қолданылады, және көбіне `String`, `Int`, `Double`, `Bool` сияқты негізгі типтерді сақтау үшін пайдаланылады.

### Дерек сақтау

`UserDefaults` деректі `set` әдісі арқылы сақтайды:

```swift
UserDefaults.standard.set("FangJunyu", forKey: "name")
UserDefaults.standard.set(28, forKey: "age")
UserDefaults.standard.set(true, forKey: "isMember")
```

`UserDefaults`-тің бірінші параметрі — сақталатын дерек. Мұнда `String`, `Int` сияқты мәндер беріліп тұр.

Екінші параметр `forKey` айнымалы атауына ұқсайды және `UserDefaults` ішіндегі деректі белгілеу үшін қолданылады.

### Деректі оқу

`UserDefaults` деректі тиісті әдіспен оқиды:

```swift
let name = UserDefaults.standard.string(forKey: "name")	// FangJunyu
let age = UserDefaults.standard.integer(forKey: "age")	// 28
let isMember = UserDefaults.standard.bool(forKey: "isMember")	// true
```

Оқығанда дұрыс типке сәйкес әдісті пайдалану керек, әйтпесе қате шығуы немесе дерек дұрыс оқылмауы мүмкін.

### UserDefaults туралы ескертулер

#### 1. Optional тип қайтару

`UserDefaults` арқылы дерек оқығанда, кейбір әдістер Optional тип қайтарады.

Мысалы:

```swift
let name = UserDefaults.standard.string(forKey: "name")
```

Қайтарылған `name` типі:

```swift
String?
```

Бұл мән болуы да, болмауы да мүмкін дегенді білдіреді.

**Неге Optional тип қайтарады?**

Себебі `UserDefaults` ішіндегі белгілі бір `Key`-ге міндетті түрде дерек сақталған болмауы мүмкін.

Мысалы:

```swift
let name = UserDefaults.standard.string(forKey: "City") // nil
```

Егер бұрын `"City"` деген `Key` ешқашан сақталмаған болса, нәтиже:

```swift
nil
```

Маңыздысы, `nil` — бұл “дерек жоқ” деген сөз, ол бос жол емес.

“Түр жүйесі” сабағында ұқсас жағдайды көрдік: егер тип түрлендіру сәтсіз болса, Optional тип қайтарылады.

Мұндайда `??` арқылы әдепкі мән беріп, Optional-ды ашу керек:

```swift
let name = UserDefaults.standard.string(forKey: "City")	?? "Rizhao"
```

Егер `UserDefaults` деректі оқыса, сол мән қолданылады; ал егер оқи алмаса (`nil` болса), әдепкі мән пайдаланылады.

#### 2. Optional емес тип қайтару

`UserDefaults` `Int`, `Double`, `Bool` типтерін оқығанда Optional емес тип қайтарады.

Мысалы:

```swift
let num = UserDefaults.standard.integer(forKey: "num")	// 0
let height = UserDefaults.standard.double(forKey: "height")	// 0.0
let isBuyCar = UserDefaults.standard.bool(forKey: "isBuyCar")	// false
```

Бұл `Key`-лерге ешқашан дерек сақталмаған болса да, қайтарылған мән `nil` болмайды.

Оның орнына мынадай әдепкі мәндер қайтарылады:

```swift
Int → 0
Double → 0.0
Bool → false
```

#### 3. Дерек көлемінің шектеуі

`UserDefaults` тек аз көлемдегі дерекке жарайды, мысалы қолданба баптаулары, қарапайым күй деректері және т.б.

Нақты жобада үлкен көлемді дерек немесе өте жиі оқылып-жазылатын дерек үшін ол қолайлы емес.

## “Ақша жинайтын құты” сомасын сақтау

Енді `UserDefaults` арқылы “ақша жинайтын құтыдағы” жалпы соманы сақтайтын код логикасын жаза аламыз.

Пайдаланушы батырманы басқанда, енгізілген сома жалпы сомаға қосылады да, кейін сол жалпы сома `UserDefaults`-ке сақталады.

```swift
Button("Save") {
    amount += number
    number = 0
	UserDefaults.standard.set(amount, forKey: "amount")	// UserDefaults
}
.buttonStyle(.borderedProminent)
```

Осылайша, деректі жазу логикасы іске асырылды.

## Деректі тұрақты сақтау логикасын түсіндіру

Біз деректі тұрақты сақтау үшін жазу логикасын аяқтағанмен, нақты іске қосып көрсек, әр view жаңарған сайын немесе қолданба қайта ашылған сайын `amount` айнымалысы бәрібір 0 болып қалады.

Себебі:

```swift
@State private var amount = 0
```

SwiftUI-де `@State` айнымалысы view құрылған кезде қайта инициализацияланады.

`UserDefaults` ішінде `amount` сақталып тұрса да, view қайта жүктелгенде `amount` қайтадан 0 болады.

Бұл қазір біз тек деректі `UserDefaults` ішіне жазғанымызды, бірақ view ішінде оны қайта оқып, қалпына келтірмегенімізді білдіреді.

Сондықтан view жүктелген кезде `UserDefaults` ішінен деректі қолмен оқып, оны `amount`-қа қайта беру керек. Сонда ғана деректі тұрақты сақтау логикасы толық жұмыс істейді.

### Логикаға ұқсастық

Осы процесті “сыныптағы тақта” деп елестетуге болады:

Сабақ кезінде мұғалім мазмұнды тақтаға жазады, ал ол мазмұн ағымдағы view ішіндегі күйге (`@State`) ұқсайды.

Сабақ аяқталған соң тақта сүртіледі, келесі сабаққа дайындалады. Бұл view жойылып, `@State` ішіндегі деректің де жойылуына ұқсайды.

Ақпарат жоғалып кетпес үшін, мұғалім оны алдын ала оқу материалына сақтап қояды. Бұл оқу материалы — `UserDefaults` ішінде сақталған дерекке ұқсайды.

Келесі сабақ басталғанда, тақта бастапқыда бос болады (`@State` қайта инициализацияланған), сондықтан мұғалім оқу материалындағы ақпаратқа сүйеніп, бұрынғы мазмұнды қайтадан тақтаға жазады (`UserDefaults`-тен оқу).

Маңыздысы — тақта бұрынғы мазмұнды автоматты түрде қалпына келтірмейді, оны тек “материалды оқу” (`UserDefaults`-ті оқу) арқылы қайта жазуға болады.

## onAppear арқылы деректі оқу

View экранда көрсетілген кезде, `UserDefaults` ішіндегі мәнді қолмен оқып, оны `@State` ішіндегі `amount` айнымалысына беру керек, сонда нақты тұрақты сақтау іске асады.

SwiftUI-де view экранға шыққанда инициализация логикасын орындау үшін `onAppear` қолдануға болады:

```swift
.onAppear {}
```

`UserDefaults` оқу кодын `onAppear` ішіне орналастырамыз:

```swift
.onAppear {
    amount = UserDefaults.standard.integer(forKey: "amount")
}
```

View құрылған кезде `amount` 0 болады; view экранда пайда болғанда `onAppear` шақырылып, `UserDefaults` ішіндегі мән оқылып, `amount`-қа беріледі. Осылайша view алдыңғы сақталған жалпы соманы көрсете алады.

Енді “ақша жинайтын құтының” бүкіл тұрақты сақтау логикасы дайын болды.

## Толық код

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
                UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
        .onAppear {
            amount = UserDefaults.standard.integer(forKey: "amount")
        }
    }
}
```

## @AppStorage property wrapper-і

SwiftUI `@AppStorage` property wrapper-ін береді, ол `UserDefaults` сақтау логикасын жеңілдетеді.

```
@AppStorage("amount") private var amount = 0
```

`@AppStorage` — бұл `UserDefaults` үшін жасалған орама, ал `"amount"` — `UserDefaults` ішіндегі `Key`.

View құрылған кезде, ол `UserDefaults` ішінен деректі оқиды; ал айнымалы өзгерген кезде, автоматты түрде `UserDefaults` ішіне жазады.

Ол SwiftUI-дің күй жаңарту механизмімен үйлесімді, яғни `@State` сияқты мән өзгергенде SwiftUI view-і де жаңарады.

Басқаша айтқанда, `@AppStorage` деректі оқу мен сақтауды бірден іске асырады, сондықтан `UserDefaults`-ті қолмен шақырудың да, `onAppear` арқылы қолмен оқудың да қажеті жоқ.

Сондықтан кодтағы `UserDefaults` орнына `@AppStorage` қолдануға болады:

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

## Қорытынды

Бұл сабақта біз негізінен мына білімдерді үйрендік:

Біріншіден, `UserDefaults` жеңіл салмақты деректерді сақтау үшін қолданылады, және біз оны жалпы соманы құрылғыға сақтау үшін пайдаландық. `UserDefaults` оқу мен жазу әдістерін ұсынады, сонымен бірге Optional тип сияқты жағдайларды да ескеруді талап етеді.

Екіншіден, `@State` айнымалысы view ішіндегі уақытша күйді ғана басқарады, ал оның өмірлік циклі view-мен бірдей. View қайта жасалған кезде, `@State` те қайта инициализацияланады.

`UserDefaults` ішіндегі деректі қалпына келтіргенде, біз `onAppear` арқылы view экранға шыққанда `UserDefaults`-ті оқуды және деректі қалпына келтіруді үйрендік.

Соңында `@AppStorage` арқылы `UserDefaults`-ті ауыстырып, view жүктелгенде деректі автоматты түрде оқу мен дерек өзгергенде автоматты жазуды іске асырдық. Осылайша қолмен `UserDefaults` пен `onAppear` логикасын жазудың қажеті қалмайды.

Осы білімдердің арқасында біз App ішіндегі базалық дерек сақтау функциясын жасап, деректі тұрақты сақтай аламыз.

## Сабақтан кейінгі жаттығу

Қазір “ақша жинайтын құты” соманы сақтай алады, бірақ функциясы әлі де толық емес.

1. Егер пайдаланушы ақша алғысы келсе, оған сәйкес ақша шығару логикасын да жобалау керек.
2. Егер пайдаланушы соманы нөлге түсіргісі келсе, оған қайта орнату логикасын да жасау керек.
3. Егер ақша жинайтын құтыға атау бергіңіз келсе, онда `String` типіндегі деректі сақтау механизмін де қосу керек.

Осы бар негіздің үстіне осы функцияларды толықтырып, “ақша жинайтын құтыны” одан әрі толық жобаға айналдырып көріңіз.

## Қосымша білім - Енгізу өрісінің әдепкі мәні мәселесі

Енді енгізу өрісінің интеракциясын тағы да жақсартып, енгізу әрекетін ыңғайлы ете аламыз.

![textfield](../../Resource/015_view.png)

Қазіргі іске асыруда енгізу өрісінің әдепкі мәні — 0. Пайдаланушы әр жолы жаңа сан енгізу үшін алдымен 0-ді өшіруі керек, ал бұл аса ыңғайлы тәжірибе емес.

Енгізу өрісінің бастапқы мәні 0 емес, бос болуы керек.

Себебі `TextField` пен `number` екіжақты байланысқан:

```swift
@State private var number = 0
TextField("Amount", value: $number, format: .number)
```

`number` 0 болып инициализацияланса, енгізу өрісі де 0 көрсетеді.

Сондықтан `number`-ды бос мәнге, яғни `nil`-ге өзгерту керек.

```swift
@State private var number = nil
```

Бұл кезде қате пайда болады:

```
Generic parameter 'Value' could not be inferred
```

Бұл — “түр жүйесі” сабағында қарастырған жағдай: айнымалының типін автоматты түрде шығару мүмкін болмаса, онда типті айқын жариялау керек.

Себебі `number`-ды `nil` етіп орнатқанда, `nil` өзінде ешқандай тип ақпараты болмайды, сондықтан компилятор бұл айнымалы `String` пе, `Int` пе, әлде басқа тип пе — соны анықтай алмайды.

Сондықтан `number` үшін типті айқын көрсетуіміз керек:

```swift
@State private var number:Int? = nil
```

Мұндағы `Int?` — Optional тип, яғни `number` мәні `Int` те болуы мүмкін, `nil` де болуы мүмкін.

Осылайша оны бастапқыда `nil` етіп орнатсақ, енгізу өрісі 0 емес, бос болып көрінеді.

### Optional типті ашу

`number`-ды Optional типке ауыстырғаннан кейін, бұрынғы есептеу коды қате береді:

```swift
amount += number
```

Көрсетілетін қате:

```swift
Value of optional type 'Int?' must be unwrapped to a value of type 'Int'
```

Себебі қазір `number` мәні `nil` болуы мүмкін. Егер оны тікелей есептеуге қоссақ, жағдай мынадай болады:

```swift
amount += nil
```

Әрине, `nil` пен `amount` қосылмайды, сондықтан бұл жарамсыз операция.

Сондықтан Optional типті есептеуге қоспас бұрын, оны алдымен ашу керек.

Бұл жерде `??` арқылы әдепкі мән беруге болады:

```swift
amount += number ?? 0
```

Егер `number` — `nil` болса, әдепкі 0 қолданылады; ал егер оның мәні болса, пайдаланушы енгізген нақты сан пайдаланылады.

Осылайша есептеудің дұрыстығы да сақталады, әрі енгізу өрісі бос күйде басталады, яғни пайдаланушыға 0-ді қолмен өшірудің қажеті болмайды.

## Қосымша код

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number:Int? = nil
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number ?? 0
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```
