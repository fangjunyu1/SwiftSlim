# Есептелетін қасиет

Бұл сабақта негізінен есептелетін қасиетті үйренеміз.

Есептелетін қасиет бар деректерге сүйеніп жаңа нәтиже есептеу үшін қолданылады. Ол сандық мәндерді де, SwiftUI көрінісінде көрсетілетін мазмұнды да есептей алады.

Мысалы:

```swift
let a = 10
let b = 20
let c = a + b
````

Мұндағы `c` — `a` мен `b` қосылғаннан кейінгі нәтиже.

Кәдімгі кодта мұндай жазу тәсілі өте жиі кездеседі.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

Ескерту: `Button` басылған кезде кәдімгі Swift кодын орындауға болады.

Бірақ ұқсас кодты SwiftUI көрінісінің қасиеттері ішінде тікелей жазсақ, мәселе пайда болады.

Мысалы:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

Бұл код қате береді.

Бір қарағанда, `a` мен `b` `c`-ден бұрын жазылған сияқты, сондықтан `c` мәнін есептеуге болатын сияқты көрінеді.

Бірақ құрылымның қасиеттерін жариялау кезінде мұндай жазу тәсілін тікелей қолдануға болмайды.

## Неліктен c тікелей a + b деп есептеле алмайды?

Себебі `a`, `b`, `c` батырма басылған кездегі уақытша тұрақтылар емес, олар `ContentView` көрінісінің қасиеттері.

Батырма басылған кездегі код қалыпты орындалады:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

Өйткені батырма басылғаннан кейін код ретімен орындалады. Алдымен `a` жасалады, содан кейін `b` жасалады, соңында `a + b` арқылы `c` есептеледі.

Бірақ көрініс ішінде қасиеттерді жариялағанда жағдай басқаша:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

Мұндағы `a`, `b`, `c` — барлығы `ContentView` көрінісінің қасиеттері.

`ContentView` көрінісі жасалған кезде Swift алдымен осы қасиеттерді дайындауы керек. Жасалу процесінің қауіпсіз болуын қамтамасыз ету үшін Swift бір экземплярдағы сақталатын қасиеттің бастапқы мәнінде сол экземплярдың басқа экземпляр қасиеттерін тікелей оқуға рұқсат бермейді.

Сондықтан мына жол қате береді:

```swift
let c = a + b
```

Қарапайым түрде былай түсінуге болады: **көрініс ішінде қасиеттерді жариялағанда, бір кәдімгі қасиетті екінші кәдімгі қасиетті есептеу үшін тікелей қолдануға болмайды.**

Мұндай мәнді тікелей сақтайтын қасиет “сақталатын қасиет” деп аталады. Түсінуді жеңілдету үшін оны уақытша кәдімгі қасиет деп қабылдауға болады.

Мысалы:

```swift
let a = 10
```

`a` ішінде `10` сақталады.

```swift
let b = 20
```

`b` ішінде `20` сақталады.

Бірақ:

```swift
let c = a + b
```

Мұндағы `c` тікелей жазылған тұрақты мән емес, ол `a + b` арқылы есептелгісі келеді.

Осындай “бар деректерден нәтиже алу” жағдайында есептелетін қасиетті қолдану қолайлырақ.

Оны былай өзгертуге болады:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

Мұндағы `c` — есептелетін қасиет.

```swift
var c: Int {
    return a + b
}
```

Оның мағынасы: **`c` қажет болған кезде ғана `a` мен `b` оқылады да, `a + b` нәтижесі қайтарылады.**

Мысалы:

```swift
Text("c: \(c)")
```

`Text` ішінде `c` көрсетілген кезде ғана `c` есептеледі.

## Есептелетін қасиет деген не?

Есептелетін қасиет айнымалыға ұқсайды, бірақ өзі дерек сақтамайды.

Мысалы:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

Мұндағы `c` — есептелетін қасиет.

Ол кәдімгі қасиет сияқты тұрақты бір мәнді сақтамайды. Әр жолы `c` оқылған кезде `{}` ішіндегі код қайта орындалып, есептелген нәтиже қайтарылады.

Оны былай түсінуге болады:

```swift
var c: Int {
    return a + b
}
```

`c` керек болған кезде барып `a + b` есептеледі.

Сондықтан есептелетін қасиет мына жағдайға сай келеді: **нәтижені бөлек сақтаудың қажеті жоқ, оны бар деректерден есептеп алуға болады.**

## Негізгі жазылуы

Есептелетін қасиет әдетте үш бөліктен тұрады:

```swift
var c: Int {
    return a + b
}
```

### 1. `var` арқылы жариялау

```swift
var c
```

Есептелетін қасиет міндетті түрде `var` арқылы жарияланады, `let` қолданылмайды.

Себебі есептелетін қасиет тұрақты сақталған мән емес, ол әр оқылған сайын динамикалық түрде есептелетін нәтиже.

### 2. Қайтарылатын типті көрсету

```swift
var c: Int
```

Есептелетін қасиет қайтарылатын типті көрсетуі керек.

Мұнда `c` соңында бүтін сан қайтарады, сондықтан типі `Int`.

### 3. {} ішінде есептеу логикасын жазу

```swift
{
    return a + b
}
```

`{}` ішінде есептеу логикасы жазылады. Мұнда қайтарылатын нәтиже — `a + b`.

## return кілтсөзі

Есептелетін қасиет бір нәтиже қайтаруы керек.

Мысалы:

```swift
var c: Int {
    return a + b
}
```

Мұндағы `return` — `a + b` есептеу нәтижесін сыртқа қайтару дегенді білдіреді.

Егер есептелетін қасиет ішінде нәтиже беретін бір ғана өрнек болса, `return` сөзін жазбауға болады:

```swift
var c: Int {
    a + b
}
```

Бірақ есептелетін қасиет ішінде бірнеше жол код болса, нәтижені нақты қайтару үшін `return` жазу керек.

Мысалы:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Мұнда есептеу екі қадамға бөлінген.

Бірінші қадам — жалпы бағаны есептеу:

```swift
let total = count * price
```

Екінші қадам — жалпы бағаны мәтінге біріктіріп, қайтару:

```swift
return "Total：\(total) $"
```

Егер `return` алып тасталса:

```swift
var totalPriceText: String {
    let total = count * price
    "Total：\(total) $"
}
```

Бұл код қате береді.

Себебі: **бұл есептелетін қасиет ішінде бірнеше жол код бар, Swift қай жол соңғы нәтиже екенін автоматты түрде анықтай алмайды.**

Сондықтан есептелетін қасиетте бір ғана нәтиже беретін жол болса, `return` жазбауға болады.

```swift
var c: Int {
    a + b
}
```

Ал есептелетін қасиет ішінде бірнеше жол код болса, `return` нақты жазылғаны дұрыс.

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

## Есептелетін қасиет пен кәдімгі қасиеттің айырмашылығы

Кәдімгі қасиет дерек сақтайды.

```swift
var c = 30
```

Мұндағы `c` нақты бір мәнді сақтайды: `30`.

Есептелетін қасиет дерек сақтамайды.

```swift
var c: Int {
    a + b
}
```

Мұндағы `c` `30` мәнін сақтамайды. Ол тек есептеу тәсілін береді.

`c` оқылған кезде Swift мына кодты орындайды:

```swift
a + b
```

Содан кейін есептеу нәтижесін қайтарады.

Сондықтан есептелетін қасиет басқа деректер арқылы нәтиже алатын жағдайларға қолайлы.

## body де есептелетін қасиет

Есептелетін қасиетті үйренгеннен кейін, SwiftUI ішінде ең жиі кездесетін мына кодты қайта түсінуге болады:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Мұндағы `body` де есептелетін қасиет.

Ол `var` арқылы жарияланған:

```swift
var body
```

Оның қайтарылатын типі бар:

```swift
some View
```

Оның `{}` ішінде қайтарылатын нәрсе — SwiftUI көрінісінің мазмұны:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

Сондықтан `body` былай түсініледі: SwiftUI осы көріністі көрсетуі керек болған кезде `body` оқылады да, `body` қайтарған мазмұн арқылы интерфейс жасалады.

Егер `return` жазсақ, оны былай түсінуге болады:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Бірақ SwiftUI ішінде әдетте `return` жазбаймыз, тікелей былай жазамыз:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

`@State` дерегі өзгерген кезде SwiftUI `body` мәнін қайта оқып, жаңа деректерге сәйкес интерфейсті жаңартады.

Мысалы:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("count: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

Батырма басылғаннан кейін `count` өзгереді.

```swift
count += 1
```

`count` өзгерген соң SwiftUI `body` мәнін қайта есептейді, сондықтан интерфейстегі мәтін де жаңарады.

```swift
Text("count: \(count)")
```

SwiftUI ішінде дерек өзгерген кезде интерфейстің автоматты жаңаруының себебі де осы.

### body ішінде күрделі есептеулер жазу ұсынылмайды

`body` есептелетін қасиет болғандықтан, ол бірнеше рет оқылып, қайта есептелуі мүмкін.

Мысалы:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("num: \(num)")
            Text("name: \(name)")
        }
    }
}
```

`name` өзгерген кезде SwiftUI `body` мәнін қайта есептейді.

Бұл кезде `body` ішіндегі код та қайта орындалады, `num` да қайта жасалады.

```swift
let num = 10
```

Бұл мысалдағы `num` өте қарапайым, сондықтан әсері үлкен емес.

Бірақ `body` ішінде күрделі есептеулер жасалса, мысалы көп деректі сүзу, сұрыптау, сурет өңдеу сияқты операциялар орындалса, интерфейстің бірқалыпты жұмыс істеуіне әсер етуі мүмкін.

Сондықтан SwiftUI ішінде `body` негізінен интерфейс құрылымын сипаттауға жауап беруі керек.

Қарапайым уақытша деректерді `body` ішінде жазуға болады.

Ал күрделі есептеулерді `body` сыртындағы есептелетін қасиетке, әдіске немесе бөлек дерек өңдеу бөлігіне шығарған дұрыс.

## Мысал: сан және жалпы баға

Енді қарапайым мысал арқылы есептелетін қасиетті түсінейік.

Бір сәбіздің бағасы 2 $ делік. Пайдаланушы батырма басу арқылы сатып алатын санын өзгерте алады, ал интерфейсте жалпы баға көрсетілуі керек.

Жалпы бағаның есептелуі:

```
Саны * Бірлік бағасы
```

Егер жалпы бағаны кәдімгі айнымалыда сақтасақ, бұл біраз қолайсыз болады.

Өйткені саны өзгерген сайын жалпы бағаны қолмен жаңартып отыру керек.

Одан жақсы тәсіл — есептелетін қасиетті қолдану:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Көрсетілу нәтижесі:

![view](../../Resource/024_view.png)

Бұл мысалда:

```swift
@State private var count = 1
```

`count` сәбіз санын білдіреді. Батырма басылған кезде `count` өзгереді.

```swift
private let price = 2
```

`price` сәбіздің бірлік бағасын білдіреді. Мұнда ол тұрақты мән, сондықтан `let` қолданылады.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` жалпы бағаны білдіреді.

Оны бөлек сақтаудың қажеті жоқ, себебі жалпы баға әрқашан `count * price` арқылы есептеледі.

`count` `1` болған кезде:

```swift
totalPrice = 1 * 2
```

Көрсетілетін нәтиже:

```swift
totalPrice: 2 $
```

`+` батырмасы басылғаннан кейін `count` `2` болады.

Бұл кезде `totalPrice` қайта оқылғанда, ол қайта есептеледі:

```swift
totalPrice = 2 * 2
```

Көрсетілетін нәтиже:

```swift
totalPrice: 4 $
```

Есептелетін қасиеттің қызметі осы: бар деректерге сүйеніп жаңа нәтижені динамикалық түрде есептеу.

## Есептелетін қасиетті шарт тексеру үшін де қолдануға болады

Есептелетін қасиет тек сандық мәнді ғана емес, шарт нәтижесін де қайтара алады.

Мысалы, біз санның ең төменгі мәні 1 болуын қалаймыз.

Сан 1 болып тұрғанда, `-` батырмасы одан әрі азайтпауы керек.

Ол үшін мынадай есептелетін қасиет қосуға болады:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Толық код:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("price: \(price) $")
                Text("count: \(count)")
                Text("totalPrice: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Көрсетілу нәтижесі:

![view](../../Resource/024_view1.png)

Мұндағы:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Бұл код қазіргі санда одан әрі азайтуға бола ма дегенді білдіреді.

`count` `1`-ден үлкен болған кезде:

```swift
canDecrease == true
```

Бұл азайтуға болады дегенді білдіреді.

`count` `1`-ге тең болған кезде:

```swift
canDecrease == false
```

Бұл одан әрі азайтуға болмайды дегенді білдіреді.

### Батырма ішіндегі шарт тексеру

Батырма ішінде мына код қолданылады:

```swift
if canDecrease {
    count -= 1
}
```

Тек `canDecrease` мәні `true` болған кезде ғана `count` азаяды.

### Көріністің алдыңғы түсін басқару

`canDecrease` арқылы батырманың алдыңғы түсін де басқаруға болады:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Мұнда үштік оператор қолданылған:

```swift
canDecrease ? Color.primary : Color.gray
```

Бұл кодтың мағынасы: егер `canDecrease` `true` болса, алдыңғы түс `Color.primary` болады; егер `canDecrease` `false` болса, алдыңғы түс `Color.gray` болады.

`Color.primary` — SwiftUI ұсынатын жүйелік семантикалық түс. Ол қазіргі интерфейстегі негізгі мәтін түсін білдіреді.

Ашық режимде `Color.primary` әдетте қара түске жақын болады; қараңғы режимде әдетте ақ түске жақын болады.

Сондықтан `Color.primary` қолданудың артықшылығы: ол ашық режим мен қараңғы режимге автоматты түрде бейімделеді.

### Көріністің өшірулі күйін басқару

`disabled` көріністің өшірулі күйде екенін басқару үшін қолданылады:

```swift
.disabled(!canDecrease)
```

`disabled` мәні `false` болса, көріністі басуға болады.

`disabled` мәні `true` болса, көрініс өшірулі күйде болады және оны басуға болмайды.

Мұнда `canDecrease` шарт ретінде қолданылғандықтан, кодты түсіну жеңілдейді.

`canDecrease` атауын көргенде, оның “қазіргі кезде одан әрі азайтуға бола ма” дегенді білдіретінін бірден түсінуге болады.

### Қосымша түсіндіру: неліктен екі рет тексеріледі?

`-` батырмасында:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Мұнда `.disabled(!canDecrease)` та, батырма ішінде `if canDecrease` та қолданылған.

`.disabled(!canDecrease)` интерфейс деңгейінде батырманы өшіріп, пайдаланушыға оны басуға мүмкіндік бермейді.

`if canDecrease` код орындалмас бұрын тағы бір рет тексеріп, тек азайтуға болатын жағдайда ғана `count -= 1` орындайды.

Бұл қос қорғанысқа жатады. Нақты әзірлеуде батырма өшірілген болса, ішкі шартты алып тастауға болады. Бірақ оқыту мысалында оны қалдыру `canDecrease` қызметін айқынырақ көрсетеді.

## Қорытынды

Бұл сабақта негізінен есептелетін қасиетті үйрендік.

Есептелетін қасиет мәнді тікелей сақтамайды. Ол оқылған кезде бар деректерге сүйеніп нәтиже есептейді.

Мысалы:

```swift
var c: Int {
    a + b
}
```

Мұндағы `c` бөлек сақталмайды, себебі оны `a + b` арқылы есептеуге болады.

Есептелетін қасиет міндетті түрде `var` арқылы жарияланады және қайтарылатын типі көрсетілуі керек.

```swift
var canDecrease: Bool {
    count > 1
}
```

Есептелетін қасиет тек сандық мәнді ғана емес, шарт нәтижесін, мәтін мазмұнын, тіпті SwiftUI көрінісінің мазмұнын да қайтара алады.

Бұл сабақта біз `return` сөзін де үйрендік.

`return` нәтижені қайтаруды білдіреді:

```swift
var totalPriceText: String {
    let total = count * price
    return "Total：\(total) $"
}
```

Егер есептелетін қасиет ішінде бір ғана нәтиже беретін өрнек болса, `return` жазбауға болады.

```swift
var totalPrice: Int {
    count * price
}
```

Сонымен қатар, біз `Color.primary` және `disabled` туралы да білдік.

`Color.primary` — SwiftUI жүйелік семантикалық түсі. Ол ашық режим мен қараңғы режимге қарай автоматты түрде өзгеріп отырады.

```swift
.foregroundStyle(Color.primary)
```

`disabled` көріністі өшіру немесе қосу үшін қолданылады.

```swift
.disabled(true)
```

Бұл өшірулі, басуға болмайды дегенді білдіреді.

```swift
.disabled(false)
```

Бұл қолжетімді, басуға болады дегенді білдіреді.

Сондықтан есептелетін қасиет SwiftUI ішінде өте жиі қолданылады. Ол есептеу нәтижелерін, шарттарды және көрсетілетін мазмұнды анағұрлым анық ұйымдастыруға көмектеседі.
