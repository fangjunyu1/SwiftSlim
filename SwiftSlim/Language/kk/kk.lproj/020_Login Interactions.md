# Кіру интеракциялары

Алдыңғы сабақта біз кіру көрінісін жасап, батырмалар мен тексеру логикасы арқылы енгізілген мазмұнды қарапайым тексеруді іске асырдық.

Осы сабақта біз алдыңғы сабақтың коды негізінде `SecureField` құпиясөз енгізу өрісін, `Alert` ескерту терезесін, сондай-ақ айнымалыларға қарай әртүрлі көріністердің көрсетілуін қалай басқаруға болатынын үйренеміз.

Осы сабақ арқылы сен кіру көрінісінің негізгі интеракция ағынын тереңірек түсіне аласың, бұл көптеген қолданбаларда қолданылатын негізгі логика да болып табылады.

## Құпиясөз енгізу өрісі

Өткен сабақта біз құпиясөз енгізу үшін `TextField` қолдандық.

Бірақ шынайы кіру интерфейсінде құпиясөз әдетте тікелей көрсетілмейді, оның орнына нүктелер немесе басқа жасырылған әсер түрінде көрінеді. Бұл пайдаланушының құпиялылығын қорғау үшін қажет.

SwiftUI-де құпиясөз енгізуге арнайы арналған басқару элементі `SecureField` болып табылады.

Мысалы:

```swift
SecureField("Password", text: $password)
```

Оның жазылу тәсілі `TextField`-ке өте ұқсас, дәл сол сияқты оған да көмекші мәтін және байланыстырылған айнымалы қажет.

Мұны қарапайым түрде былай түсінуге болады: `SecureField` те енгізу өрісі, тек ол енгізілген мазмұнды жасырып көрсетеді.

Біз бұрын құпиясөз енгізу үшін қолданған `TextField`-ті тікелей `SecureField`-ке ауыстыра аламыз:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Пайдаланушы құпиясөз енгізген кезде, оның мазмұны интерфейсте тікелей көрінбейді.

Көрсету әсері:

![password](../../Resource/020_password.png)

Енді біз пайдаланушы атын және құпиясөзді енгізіп, батырманы басқаннан кейін енгізудің дұрыс не бұрыс екенін анықтай аламыз.

### Кері байланыс хабарларын көрсету

Бұған дейін біз нәтижелерді шығару үшін `print`-ті үнемі қолданып келдік.

`print` мазмұны тек консольде ғана көрсетіледі. Әзірлеуші оны Xcode-та бағдарламаны іске қосқанда көре алады, бірақ қарапайым пайдаланушы бұл хабарларды телефонда көре алмайды.

Яғни, бағдарлама тексеруді аяқтағанымен, пайдаланушы енгізілген мәліметтің дұрыс екенін білмейді.

Осындай жағдайда бізге хабарларды тікелей интерфейсте көрсетудің бір жолы қажет.

## Alert ескерту терезесі

SwiftUI-де біз `Alert` арқылы қалқымалы хабарлама терезесін көрсете аламыз.

Негізгі қолданылуы:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

Бұл код алдыңғы басқару элементтеріне қарағанда сәл күрделілеу көрінеді, бірақ оны бөліп қарастырсақ, түсіну оңай.

`Alert` негізінен мына бөліктерден тұрады:

- `"Title"`：ескерту терезесінің тақырыбы
- `isPresented`：ескерту терезесі көрсетіле ме, жоқ па соны басқаратын байланыстырылған логикалық мән
- `Button(...)`：ескерту терезесіндегі батырма
- `message:`：ескерту терезесінде көрсетілетін хабар мәтіні

Мұны қарапайым түрде былай түсінуге болады: байланыстырылған айнымалы `true` болған кезде, жүйе осы ескерту терезесін көрсетеді.

### Alert — бұл модификатор

Бұрын үйренген көптеген модификаторлар сияқты, `Alert` те белгілі бір көріністің артына қосылады.

Мысалы, оны батырманың артына тіркеуге болады:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

Оны сыртқы `VStack`-тің артына да тіркеуге болады:

```swift
VStack {
    // ...
}
.alert() {...}
```

Яғни, `Alert` интерфейсте жеке тұратын дербес көрініс емес, белгілі бір көрініске тіркелетін модификатор.

Мұны қарапайым түрде былай түсінуге болады: `Alert` әдетте батырманың немесе сыртқы layout-тың артына жазылады және белгілі бір шарт орындалғанда хабарды қалқымалы түрде көрсетеді.

### Alert көрсету

Егер біз `Alert` көрсеткіміз келсе, оны басқаратын айнымалы қажет.

Мысалы:

```swift
@State private var showAlert = false
```

Бұл айнымалы `Bool` түрінде, яғни оның тек екі ғана мәні бар: `true` немесе `false`.

Содан кейін оны `Alert`-тің `isPresented` мәніне байланыстырамыз:

```swift
isPresented: $showAlert
```

`showAlert` мәні `false` болғанда, ескерту терезесі көрсетілмейді; `showAlert` мәні `true` болғанда, ескерту терезесі шығады.

### Батырма арқылы ескерту терезесін көрсету

Енді ең қарапайым сценарийді іске асырайық:

Пайдаланушы ештеңе енгізбей, тікелей `Sign in` батырмасын басқан кезде, оған: «Сен әлі ештеңе енгізген жоқсың» деп хабарлайтын хабарлама терезесі шығады.

Алдымен осы хабарлама терезесін басқару үшін айнымалы құрайық:

```swift
@State private var showEmpty = false
```

Содан кейін `Button` артынан `Alert` қосамыз:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

Одан кейін батырманы басу кодында шартты тексереміз:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

Егер `user` мен `password` екеуі де бос болса, онда `showEmpty` мәнін `true` қыламыз.

Ал `Alert` осы айнымалымен байланыстырылғандықтан, ол `true` болған кезде жүйе ескерту терезесін автоматты түрде көрсетеді.

Көрсету әсері:

![alert](../../Resource/020_alert1.png)

Осылайша, пайдаланушы батырманы басқаннан кейін хабарламаны бірден көре алады, ал әзірлеушіге `print` нәтижесін көру үшін консольге қараудың қажеті болмайды.

### Батырма мен ескерту терезесін толықтыру

Бұған дейін біз тек «ештеңе толтырылмаған» жағдайды ғана өңдедік.

Бізге тағы келесі жағдайларды да өңдеу керек:

- пайдаланушы аты толтырылмаған
- құпиясөз толтырылмаған
- пайдаланушы аты мен құпиясөз екеуі де толтырылған, бірақ енгізу қате

Сондықтан осы жағдайларға сәйкес хабарламаларды да дайындауымыз керек.

Алдымен үш жаңа айнымалы қосамыз:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

Одан кейін батырманың артына сәйкес `Alert`-терді қоса береміз:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

Содан кейін батырманың ішіндегі шешім логикасын толықтырамыз:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

Батырма пайдаланушы ешқандай ақпарат толтырмағанын, бір ақпаратты өткізіп жібергенін немесе енгізуді тексеру сәтсіз болғанын анықтағанда, сәйкес логикалық айнымалыны `true` мәніне өзгертеді, содан кейін сәйкес `Alert` шығады.

Мұнда саған бірнеше жағдайды өзің тексеріп көруге кеңес беремін:

- ештеңе толтырмау
- тек пайдаланушы атын толтыру
- тек құпиясөзді толтыру
- пайдаланушы аты мен құпиясөзді толтыру, бірақ қате мазмұнмен
- пайдаланушы аты мен құпиясөзді толтыру, әрі мазмұны дұрыс болу

Осылайша біз кодымызда мәселе бар-жоғын тексере аламыз. Бұл сондай-ақ қолданба әзірлеудегі өте маңызды «тестілеу» кезеңі.

Егер қандай да бір сценарийде бағдарлама дұрыс жұмыс істемесе, бұл логикада мәселе болуы мүмкін екенін және бағдарламада `Bug` пайда болғанын білдіреді.

Әзірлеуде жиі айтылатын `Bug` дегеніміз — қате, ақау немесе күтілгенге сәйкес келмейтін әрекет.

## Кіру көрінісін аяқтау

Жоғарыда айтылған `Alert`-тердің барлығы «енгізу қате болғанда» пайдаланушыға қалай хабар беру керектігі туралы еді.

Пайдаланушы дұрыс пайдаланушы атын және құпиясөзді енгізген кезде, біз әдетте «ескерту терезесін» шығармаймыз, оның орнына бірден кіруден кейінгі интерфейске өтеміз.

Мұнда өте маңызды бір ұғым бар: **айнымалы мәніне қарай әртүрлі көріністерді көрсету.**

Мысалы:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

Мұнда біз көріністің көрсетілуін басқару үшін `Bool` түрін қолданамыз:

- егер `login` `true` болса, бұл кірудің сәтті болғанын білдіреді және кіруден кейінгі мазмұн көрсетіледі
- егер `login` `false` болса, бұл кіру сәтсіз болғанын білдіреді және кіру интерфейсі көрсетіле береді.

`if` операторы Swift тіліндегі шартты оператор болғанымен, SwiftUI-дың `body` ішінде оны көріністердің көрсетілу логикасын басқару үшін пайдалануға болады.

Яғни, SwiftUI шарт нәтижесіне қарай қазіргі сәтте қай көріністі көрсету керектігін анықтайды.

Осы көрсету логикасын ағымдағы `ContentView` ішіне орналастыра аламыз:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Original login view code
		}
	}
}
```

Мұнда біз пайдаланушының сәтті кіргенін көрсету үшін жаңа `login` айнымалысын қостық.

Содан кейін бұрын жазылған кіру көрінісінің кодын `else` операторынан кейін орналастырдық.

Көріністі көрсету логикасы:

- `login` `false` болғанда, бұл әлі кіру сәтті болмағанын білдіреді, сондықтан бастапқы кіру көрінісі көрсетіле береді.
- `login` `true` болғанда, бұл кірудің сәтті болғанын білдіреді; осы сәтте кіру көрінісі енді көрсетілмейді, оның орнына `Login successful` көрсетіледі.

Яғни, `ContentView` `login` мәніне байланысты әртүрлі көрініс мазмұнын көрсетеді.

Келесіде батырманың ішіндегі сәтті кіру логикасын өзгертейік:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

Пайдаланушы енгізген пайдаланушы аты мен құпиясөз екеуі де дұрыс болғанда, біз `login` мәнін `true` қыламыз.

`login` мәні өзгерген бойда, `ContentView` ішіндегі `if` тексерісінің нәтижесі де өзгереді, және интерфейс бастапқы кіру көрінісінен `Login successful` көрінісіне ауысады.

Көрсету әсері төмендегідей:

![view](../../Resource/020_view.png)

Енді біз «шартқа байланысты әртүрлі көріністерді көрсету» білімін дәл осы ағымдағы кіру бетінде шын мәнінде қолдандық.

Нақты қолданбаларда да әдетте дәл осылай істеледі: белгілі бір мәнге сәйкес әртүрлі көріністер көрсетіледі.

Мысалы, пайдаланушы жүйеге кірмеген кезде кіру беті көрсетіледі; ал пайдаланушы сәтті кіргеннен кейін басқа интерфейс көрсетіледі.

Негізгі идея мынау: **айнымалылардың өзгеруі арқылы интерфейсте қандай мазмұн көрсетілетінін басқару.**

## Қорытынды

Осы сабақта біз үш маңызды нәрсені үйрендік:

1. Құпиясөзді енгізу үшін `SecureField` пайдалану, осылайша құпиясөз мазмұны интерфейсте тікелей көрсетілмейді.
2. Хабарларды тек консольге шығарып қана қоймай, пайдаланушыға көрсету үшін `Alert` пайдалану.
3. Әртүрлі көріністердің көрсетілуін басқару және кіруге дейінгі және кейінгі интерфейстер арасындағы ауысуды іске асыру үшін айнымалылар мен `if` шарттарын пайдалану.

Осы білімдерді меңгеру арқылы біз енді негізгі интеракция әсерлері бар кіру көрінісін аяқтай аламыз және SwiftUI-дағы «күй өзгереді, интерфейс жаңарады» деген әзірлеу идеясын тереңірек түсіне аламыз.

## Толық код

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## Кеңейтілген білім - бірнеше батырмасы бар Alert

Осы сабақтың алдыңғы бөлігінде біз ең негізгі `Alert`-ті қолдандық, онда тек бір ғана батырма болды және ол негізінен пайдаланушыға ескерту көрсету үшін қолданылды.

Шын мәнінде, `Alert` бірнеше батырманы да қамти алады, сондықтан пайдаланушы әртүрлі таңдаулар жасай алады.

Мысалы:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

Батырманы басқаннан кейін үш нұсқасы бар хабарлама терезесі шығады: `Ok`, `delete`, және `cancel`.

Көрсету әсері:

![alert](../../Resource/020_alert2.png)

Әртүрлі батырмаларды басқанда, сәйкес код орындалады.

### Button-ның role қасиеті

Мұнда сен `Alert` ішіндегі `Button`-ның біз бұрын үйренген кәдімгі батырма жазылу тәсілінен сәл өзгеше екенін байқайсың:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

Мұнда қосымша `role` параметрі бар.

SwiftUI-де `role` жүйеге осы батырманың қандай әрекет түріне жататынын айту үшін қолданылады.

Кең таралған бірнеше рөл мыналар:

- `.cancel`：бас тарту әрекеті
- `.destructive`：қауіпті әрекет, мысалы өшіру
- `nil`：қалыпты әрекет

Әртүрлі `role` мәндері әдетте интерфейстегі көрнекі стильдің де әртүрлі болуына әкеледі.

Мысалы, қауіпті әрекеттер пайдаланушыны абайлап басуға ескерту үшін көбіне көзге түсетін стильмен көрсетіледі.

Сонымен қатар, `role` батырмаға қосымша семантикалық ақпарат береді, бұл экран оқырмандарына (мысалы, Apple құрылғыларындағы VoiceOver) пайдаланушыға батырманың қызметін түсіндіруді жеңілдетеді.

Сондықтан, тиісті жағдайларда батырмаға `role` қосу өте жақсы әдет болып табылады.
