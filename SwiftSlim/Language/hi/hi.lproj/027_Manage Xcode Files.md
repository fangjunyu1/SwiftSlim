# Xcode फ़ाइलों का प्रबंधन

इस पाठ में, हम Xcode में आम फ़ाइल संचालन सीखेंगे।

पिछले पाठों में, हमने ज़्यादातर कोड `ContentView` में लिखा था। इससे कोड और प्रदर्शन प्रभाव देखना आसान होता है।

उदाहरण के लिए:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

लेकिन वास्तविक App में, आम तौर पर सारा कोड एक ही फ़ाइल में नहीं लिखा जाता।

जब पेज, कॉम्पोनेंट और फ़ंक्शन बढ़ते जाते हैं, तो हमें अलग-अलग View को अलग-अलग SwiftUI फ़ाइलों में बाँटना पड़ता है।

उदाहरण के लिए:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

इससे कोड संरचना अधिक स्पष्ट होती है, और बाद में रखरखाव व संशोधन भी आसान हो जाते हैं।

## फ़ाइलों का प्रबंधन क्यों करें

SwiftUI में, एक पेज या एक कॉम्पोनेंट को आम तौर पर एक अलग View फ़ाइल के रूप में लिखा जा सकता है।

उदाहरण के लिए:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

यहाँ `CardView` एक SwiftUI View है।

यदि सभी View `ContentView` में लिखे जाएँ, तो कोड धीरे-धीरे लंबा होता जाएगा और पढ़ना भी कठिन होता जाएगा।

इसलिए, हमें फ़ाइल बनाना, फ़ाइल का नाम बदलना, फ़ाइल हटाना, और अलग-अलग कोड फ़ाइलों को प्रबंधित करने के लिए फ़ोल्डर का उपयोग करना सीखना चाहिए।

## SwiftUI फ़ाइल बनाना

Xcode में SwiftUI फ़ाइल बनाने के लिए आम तौर पर दो सामान्य तरीके होते हैं:

एक तरीका है खाली फ़ाइल बनाना, फिर स्वयं कोड लिखना।

दूसरा तरीका है SwiftUI टेम्पलेट फ़ाइल बनाना, जिसमें Xcode मूल कोड अपने आप बनाता है।

शुरुआती लोगों के लिए SwiftUI टेम्पलेट फ़ाइल का उपयोग करना अधिक अनुशंसित है, क्योंकि यह मूल संरचना अपने आप बना देती है।

### खाली फ़ाइल बनाना

Xcode के बाएँ तरफ़ Navigator क्षेत्र में, खाली स्थान या फ़ोल्डर पर राइट-क्लिक करें, फिर `New Empty File` चुनें।

कार्य स्थान इस प्रकार है:

![xcode](../../../Resource/027_view1.png)

खाली फ़ाइल बनाने के बाद, Xcode आम तौर पर एक बिना नाम वाली Swift फ़ाइल बनाता है, जैसे `Untitled.swift`।

हम इसे `TestView.swift` नाम दे सकते हैं।

SwiftUI प्रोजेक्ट में, यदि यह फ़ाइल किसी View को दिखाने के लिए है, तो आम तौर पर नाम के अंत में `View` जोड़ने की सलाह दी जाती है।

उदाहरण के लिए:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

यह अनिवार्य नियम नहीं है, लेकिन इससे हमें एक नज़र में पता चल जाता है कि यह एक View फ़ाइल है।

खाली फ़ाइल बनाने के बाद, फ़ाइल में केवल डिफ़ॉल्ट टिप्पणियाँ हो सकती हैं, या वह खाली भी हो सकती है। यह Xcode संस्करण और बनाने के तरीके पर निर्भर करता है।

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

इस समय, हमें SwiftUI कोड हाथ से लिखना होगा:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

इस तरह, एक सरल SwiftUI View फ़ाइल बन जाती है।

### फ़ाइल का नाम बदलना

यदि फ़ाइल बनाने के बाद उसका नाम बदलना हो, तो Xcode के बाएँ तरफ़ Navigator क्षेत्र में फ़ाइल चुनें, फिर Enter कुंजी `⏎` दबाएँ।

![xcode](../../../Resource/027_view2.png)

इससे फ़ाइल का नाम बदला जा सकता है।

ध्यान देने की बात है कि फ़ाइल का नाम बदलने से कोड में View का नाम अपने आप नहीं बदलता।

उदाहरण के लिए, फ़ाइल का नाम `ProfileView.swift` करने के बाद भी कोड में `struct TestView: View` रह सकता है।

हालाँकि इससे आवश्यक रूप से त्रुटि नहीं होगी, लेकिन इससे भ्रम पैदा हो सकता है।

इसलिए, फ़ाइल नाम और View नाम को एक जैसा रखने की सलाह दी जाती है। उदाहरण के लिए, यदि फ़ाइल नाम `ProfileView.swift` है, तो View नाम भी `ProfileView` उपयोग करें।

## SwiftUI टेम्पलेट फ़ाइल बनाना

खाली फ़ाइल बनाने के अलावा, हम सीधे SwiftUI टेम्पलेट फ़ाइल भी बना सकते हैं।

Xcode के बाएँ तरफ़ Navigator क्षेत्र में, खाली स्थान या फ़ोल्डर पर राइट-क्लिक करें, फिर `New File from Template...` चुनें।

कार्य स्थान इस प्रकार है:

![xcode](../../../Resource/027_view3.png)

खुलने वाली विंडो में, ऊपर अलग-अलग प्लेटफ़ॉर्म चुने जा सकते हैं, जैसे `iOS`, `macOS` आदि।

यहाँ वर्तमान प्रोजेक्ट से मेल खाने वाला प्लेटफ़ॉर्म चुनना ही ठीक है। उदाहरण के लिए, यदि हम iOS App बना रहे हैं, तो `iOS` चुनें।

![xcode](../../../Resource/027_view4.png)

फिर नीचे `SwiftUI View` चुनें।

![xcode](../../../Resource/027_view12.png)

फिर `Next` पर क्लिक करें, फ़ाइल नाम दर्ज करें और सेव करें।

उदाहरण के लिए, हम `Test2View.swift` बनाते हैं।

Xcode अपने आप नीचे जैसे कोड बनाएगा:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

खाली फ़ाइल की तुलना में, टेम्पलेट फ़ाइल अपने आप `import SwiftUI`, `struct`, `body` और `#Preview` शामिल करती है।

इसलिए शुरुआती लोगों के लिए SwiftUI टेम्पलेट फ़ाइल का उपयोग करना अधिक सरल है, और कोड छूटने की संभावना भी कम होती है।

**सुझाव**: राइट-क्लिक करके फ़ाइल बनाने के अलावा, आप Xcode के बाएँ तरफ़ Navigator के नीचे मौजूद प्लस बटन पर क्लिक करके भी फ़ाइल या फ़ोल्डर बना सकते हैं।

![xcode](../../../Resource/027_view5.png)

## फ़ोल्डर बनाना

जब फ़ाइलें अधिक हो जाएँ, तो हम उन्हें प्रबंधित करने के लिए फ़ोल्डर का उपयोग कर सकते हैं।

Xcode के बाएँ तरफ़ Navigator क्षेत्र में, खाली स्थान या फ़ोल्डर पर राइट-क्लिक करें, फिर `New Folder` चुनें।

फ़ोल्डर बनाने के बाद, संबंधित फ़ाइलों को उसमें खींचकर डाला जा सकता है।

![xcode](../../../Resource/027_view7.png)

फ़ोल्डर मुख्य रूप से प्रोजेक्ट संरचना को व्यवस्थित करने के लिए होता है, यह View के उपयोग के तरीके को नहीं बदलता।

जब तक View नाम सही है, अन्य फ़ाइलों में अब भी `TestView()` और `Test2View()` जैसे तरीकों से उसका उपयोग किया जा सकता है।

## फ़ाइल या फ़ोल्डर हटाना

यदि फ़ाइल या फ़ोल्डर हटाना हो, तो Xcode के बाएँ तरफ़ Navigator क्षेत्र में संबंधित फ़ाइल चुनें, फिर राइट-क्लिक करके `Delete` चुनें।

फ़ाइल चुनने के बाद Delete कुंजी `⌫` भी दबाई जा सकती है।

Xcode हटाने की पुष्टि वाली विंडो दिखाएगा।

![xcode](../../../Resource/027_view10.png)

`Move to Trash` चुनने का अर्थ है फ़ाइल को Trash में ले जाना।

यदि `Remove Reference` दिखाई दे, तो इसका अर्थ है कि केवल Xcode प्रोजेक्ट से संदर्भ हटाया जाएगा, लेकिन डिस्क पर मौजूद मूल फ़ाइल नहीं हटेगी।

शुरुआती लोगों के लिए, यदि केवल स्वयं अभी बनाई गई सामान्य फ़ाइल हटानी है, तो आम तौर पर `Move to Trash` चुनना पर्याप्त है।

## कई SwiftUI View दिखाना

फ़ाइल बनाना सीखने के बाद, हम अलग-अलग View को अलग-अलग फ़ाइलों में बाँट सकते हैं।

उदाहरण के लिए, हम दो SwiftUI फ़ाइलें बनाते हैं:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` लेखक का नाम दिखाने के लिए उपयोग किया जाता है:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` कार्ड की पृष्ठभूमि दिखाने के लिए उपयोग किया जाता है:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

फिर, हम `ContentView` में इन दोनों View का उपयोग कर सकते हैं:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

रन करने के बाद, हम देख सकते हैं कि `CardView` और `AuthorView` एक साथ संयोजित होकर दिख रहे हैं:

![xcode](../../../Resource/027_view11.png)

इस प्रभाव में, `CardView` पृष्ठभूमि के रूप में है, और `AuthorView` ऊपर दिखता है। ऐसा इसलिए है क्योंकि उन्हें `ZStack` में रखा गया है।

`ContentView` में, हम `CardView()` और `AuthorView()` के माध्यम से इन दोनों View का उपयोग करते हैं। नाम के बाद `()` इस View को बनाने और उपयोग करने का संकेत देता है।

यही SwiftUI में आम View nesting है: एक View के भीतर दूसरे View का उपयोग जारी रखा जा सकता है।

## प्रवेश फ़ाइल

अंत में, हम फिर से देखें कि App इंटरफ़ेस दिखाना कहाँ से शुरू करता है।

SwiftUI प्रोजेक्ट में, आम तौर पर एक प्रवेश फ़ाइल होती है:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

यहाँ `@main` दर्शाता है कि यह App का प्रवेश स्थान है।

`WindowGroup` में, हम देखते हैं:

```swift
ContentView()
```

इसका अर्थ है कि App शुरू होने के बाद सबसे पहले `ContentView` दिखाता है।

फिर `ContentView` के भीतर फिर दिखाया गया है:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

इसलिए View hierarchy को इस तरह समझा जा सकता है:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

यदि हम प्रवेश फ़ाइल में दिखाए जाने वाले View को `CardView` में बदल दें:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

तो App शुरू होने के बाद केवल `CardView` दिखाई देगा।

अर्थात, प्रवेश कोड में `WindowGroup` यह तय करता है कि App शुरू होने के बाद सबसे पहले कौन-सा View दिखेगा।

और उस View के भीतर, अन्य View भी आगे दिखाए जा सकते हैं।

## सारांश

इस पाठ में, हमने Xcode में आम फ़ाइल प्रबंधन संचालन सीखे।

हमने जाना कि SwiftUI फ़ाइल कैसे बनाते हैं, फ़ाइल का नाम कैसे बदलते हैं, फ़ोल्डर कैसे बनाते हैं, और फ़ाइल कैसे हटाते हैं।

साथ ही, हमने एक महत्वपूर्ण अवधारणा भी सीखी: SwiftUI View को अलग-अलग फ़ाइलों में बाँटा जा सकता है, फिर उन्हें अन्य View में उपयोग किया जा सकता है।

उदाहरण के लिए:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

यहाँ `CardView()` और `AuthorView()` का अर्थ है कि `ContentView` के भीतर अन्य View का उपयोग किया जा रहा है।

अंत में, हमने प्रवेश फ़ाइल की प्रदर्शन logic भी दोहराई।

App शुरू होने पर, वह पहले `@main` चिह्नित प्रवेश कोड में प्रवेश करता है, फिर `WindowGroup` में मौजूद View दिखाता है।

इससे हमें समझने में मदद मिलती है कि अलग-अलग SwiftUI फ़ाइलें अलग-थलग नहीं हैं। वे एक-दूसरे के साथ संयोजित होकर अंततः पूरा App इंटरफ़ेस बनाती हैं।
