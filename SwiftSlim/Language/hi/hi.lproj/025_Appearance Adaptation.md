# रूप-रंग अनुकूलन

इस पाठ में, हम हल्के मोड और गहरे मोड के लिए इंटरफ़ेस के रूप-रंग अनुकूलन को सीखेंगे।

iPhone, iPad और Mac पर, उपयोगकर्ता हल्का मोड चुन सकते हैं, और गहरा मोड भी चुन सकते हैं।

जब सिस्टम गहरे मोड में बदलता है, तो कई App के बैकग्राउंड, टेक्स्ट और आइकन के रंग भी उसके अनुसार बदल जाते हैं।

उदाहरण के लिए, दिन में इंटरफ़ेस हल्के बैकग्राउंड का उपयोग कर सकता है, और रात में गहरे बैकग्राउंड में बदल सकता है।

![view](../../../Resource/025_view9.png)

सिस्टम के रूप-रंग के अनुसार इंटरफ़ेस डिस्प्ले को अपने-आप समायोजित करने की इस क्षमता को रूप-रंग अनुकूलन कहा जाता है।

## Xcode में रूप-रंग अनुकूलन का प्रीव्यू

रूप-रंग अनुकूलन सीखने से पहले, हम पहले समझते हैं कि Xcode में हल्के मोड और गहरे मोड का प्रीव्यू कैसे देखा जाता है।

`Canvas` क्षेत्र के नीचे, `Device Settings` बटन पर क्लिक करें। इससे `Canvas Device Settings` पॉपअप खुल सकता है।

![canvas](../../../Resource/025_view.png)

पॉपअप में आप `Color Scheme` सेटिंग देख सकते हैं।

![canvas1](../../../Resource/025_view1.png)

इनका अर्थ है:

- `Light Appearance` हल्का मोड दर्शाता है।
- `Dark Appearance` गहरा मोड दर्शाता है।

हम यहाँ से प्रीव्यू रूप-रंग बदल सकते हैं, और देख सकते हैं कि वर्तमान इंटरफ़ेस हल्के मोड और गहरे मोड में कैसा दिखता है।

## दो रूप-रंग साथ-साथ दिखाना

यदि आप हल्के मोड और गहरे मोड को एक साथ देखना चाहते हैं, तो `Variants` फ़ंक्शन का उपयोग कर सकते हैं।

`Canvas` क्षेत्र के नीचे, `Variants` बटन पर क्लिक करें, और `Color Scheme Variants` चुनें।

![canvas2](../../../Resource/025_view2.png)

चुनने के बाद, `Canvas` हल्के मोड और गहरे मोड के प्रीव्यू प्रभाव को एक साथ दिखाएगा।

![canvas3](../../../Resource/025_view3.png)

इससे दोनों रूप-रंगों में इंटरफ़ेस के अंतर की तुलना करना अधिक सुविधाजनक हो जाता है।

सरल रूप में:

यदि आप केवल अस्थायी रूप से हल्के/गहरे मोड के बीच बदलना चाहते हैं, तो `Canvas Device Settings` का उपयोग कर सकते हैं।

यदि आप दोनों रूप-रंगों को एक साथ देखना चाहते हैं, तो `Color Scheme Variants` का उपयोग कर सकते हैं।

## व्यू डिफ़ॉल्ट रूप से रूप-रंग के अनुकूल होते हैं

SwiftUI में, कई सिस्टम व्यू डिफ़ॉल्ट रूप से हल्के मोड और गहरे मोड के अनुकूल अपने-आप हो जाते हैं।

उदाहरण के लिए:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("प्रकाश")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("कुछ कुछ कुछ")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

दिखने का परिणाम:

![canvas](../../../Resource/025_view4.png)

आप देख सकते हैं कि हल्के मोड में, आइकन और शीर्षक आम तौर पर गहरे रंग में दिखते हैं।

गहरे मोड में, आइकन और शीर्षक अपने-आप हल्के रंग में बदल जाते हैं।

ऐसा इसलिए है क्योंकि `Text`, `Image` जैसे SwiftUI व्यू डिफ़ॉल्ट रूप से सिस्टम रूप-रंग के अनुसार रंग समायोजित करते हैं।

दूसरे शब्दों में, यदि हम हाथ से कोई स्थिर रंग निर्दिष्ट नहीं करते, तो SwiftUI हमारे लिए रूप-रंग अनुकूलन का एक हिस्सा संभालता है।

## स्थिर रंग अपने-आप अदला-बदली नहीं करते

ध्यान देने की बात है कि यदि हम हाथ से कोई स्थिर रंग निर्दिष्ट करते हैं, तो वह हल्के/गहरे मोड के अनुसार अपने-आप नहीं बदलता।

उदाहरण के लिए:

```swift
Text("कुछ कुछ कुछ")
	.foregroundStyle(Color.gray)
```

यहाँ टेक्स्ट को `Color.gray` के रूप में निर्दिष्ट किया गया है, इसलिए यह हमेशा ग्रे रंग में दिखेगा।

एक और उदाहरण:

```swift
Text("शीर्षक")
    .foregroundStyle(Color.white)
```

यह कोड हल्के मोड और गहरे मोड दोनों में सफेद टेक्स्ट दिखाएगा।

यदि बैकग्राउंड भी हल्का है, तो सफेद टेक्स्ट साफ़ दिखाई नहीं दे सकता।

इसलिए रूप-रंग अनुकूलन करते समय, `Color.white`, `Color.black` जैसे स्थिर रंगों को मनमाने ढंग से हार्ड-कोड करने से यथासंभव बचना चाहिए।

कई स्थितियों में, आप पहले सिस्टम सेमांटिक शैली का उपयोग कर सकते हैं, जैसे:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

इनका अर्थ है:

- `.primary` आम तौर पर मुख्य सामग्री के लिए उपयोग होता है।
- `.secondary` आम तौर पर द्वितीयक सामग्री के लिए उपयोग होता है।

वे हल्के/गहरे मोड के अनुसार अपने डिस्प्ले प्रभाव को अपने-आप समायोजित करते हैं।

## रूप-रंग के अनुसार अलग सामग्री दिखाना

कभी-कभी, हम केवल रंग बदलना नहीं चाहते, बल्कि अलग-अलग रूप-रंगों में अलग सामग्री भी दिखाना चाहते हैं।

उदाहरण के लिए:

- हल्के मोड में सामान्य बल्ब आइकन दिखाएँ।
- गहरे मोड में चमकता हुआ बल्ब आइकन दिखाएँ।
- हल्के मोड में शीर्षक `प्रकाश` दिखाएँ।
- गहरे मोड में शीर्षक `अंधकार` दिखाएँ।

इस समय, हमें यहजांचना होगा कि वर्तमान सिस्टम हल्के मोड में है या गहरे मोड में।

SwiftUI में, वर्तमान रूप-रंग मोड पाने के लिए `colorScheme` पर्यावरण मान का उपयोग किया जा सकता है:

```swift
@Environment(\.colorScheme) private var colorScheme
```

पूरा कोड:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "प्रकाश" : "अंधकार"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("कुछ कुछ कुछ")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

दिखने का परिणाम:

![view](../../../Resource/025_view5.png)

प्रीव्यू परिणाम से देखा जा सकता है कि एक ही व्यू अलग-अलग रूप-रंग मोड में बिल्कुल समान सामग्री नहीं दिखाता।

हल्के मोड में, `Image` सामान्य बल्ब आइकन दिखाता है, और शीर्षक `प्रकाश` दिखाता है।

गहरे मोड में, `Image` चमकता हुआ बल्ब आइकन दिखाता है, और शीर्षक `अंधकार` दिखाता है।

दूसरे शब्दों में, यहाँ बदलाव केवल रंग का बदलाव नहीं है, बल्कि आइकन का नाम और शीर्षक टेक्स्ट स्वयं भी बदल गए हैं।

ऐसा इसलिए है क्योंकि हमने आइकन और शीर्षक को सीधे `body` में हार्ड-कोड नहीं किया, बल्कि `colorScheme` के आधार पर अलग सामग्री की गणना की है।

इस कोड में, हमने दो computed properties का उपयोग किया है:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "प्रकाश" : "अंधकार"
}
```

जब `colorScheme` `.light` होता है, तो इसका अर्थ है कि वर्तमान मोड हल्का मोड है।

जब `colorScheme` `.dark` होता है, तो इसका अर्थ है कि वर्तमान मोड गहरा मोड है।

इसलिए हम अलग-अलग रूप-रंग मोड के अनुसार अलग आइकन नाम और शीर्षक टेक्स्ट लौटा सकते हैं।

यह computed properties का एक सामान्य उपयोग भी है: वर्तमान स्थिति के आधार पर व्यू को दिखाने के लिए आवश्यक सामग्री की गणना करना।

## @Environment को समझना

यह हमारा `@Environment` से पहला परिचय है।

`@Environment` को इस तरह समझा जा सकता है: SwiftUI के पर्यावरण से कोई मान पढ़ना।

App चलने पर, सिस्टम कई तरह की पर्यावरण जानकारी प्रदान करता है, उदाहरण के लिए:

- वर्तमान भाषा
- वर्तमान रूप-रंग मोड
- वर्तमान लेआउट दिशा
- वर्तमान फ़ॉन्ट आकार सेटिंग

जब हमें वर्तमान रूप-रंग मोड प्राप्त करना हो, तो हम `colorScheme` पढ़ सकते हैं:

```swift
@Environment(\.colorScheme) private var colorScheme
```

यहाँ

```swift
\.colorScheme
```

का अर्थ है पर्यावरण से `colorScheme` नाम का मान पढ़ना।

```swift
private var colorScheme
```

का अर्थ है पढ़े गए मान को `colorScheme` नाम के वेरिएबल में सहेजना।

वेरिएबल का नाम आप स्वयं तय कर सकते हैं। उदाहरण के लिए, इसे इस तरह भी लिखा जा सकता है:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

जब तक आगे वाला `\.colorScheme` नहीं बदलता, इसका अर्थ है कि पढ़ा जा रहा मान वर्तमान रूप-रंग मोड ही है।

## colorScheme के दो सामान्य मान

`colorScheme` के आम तौर पर दो मान होते हैं:

```swift
.light
.dark
```

इनका अर्थ है:

- `.light`: हल्का मोड
- `.dark`: गहरा मोड

उदाहरण के लिए:

```swift
private var titleName: String {
    colorScheme == .light ? "प्रकाश" : "अंधकार"
}
```

इस कोड का अर्थ है:

यदि वर्तमान मोड हल्का मोड है, तो `colorScheme == .light` सत्य है, इसलिए यह `"प्रकाश"` लौटाता है।

अन्यथा, यह `"अंधकार"` लौटाता है।

इसलिए, हल्के मोड में:

```swift
Text(titleName)
```

दिखने वाला टेक्स्ट है:

```swift
प्रकाश
```

गहरे मोड में, दिखने वाला टेक्स्ट है:

```swift
अंधकार
```

इस तरीके से, हम अलग-अलग रूप-रंगों के अनुसार अलग सामग्री दिखा सकते हैं।

## रंग अनुकूलन उदाहरण

टेक्स्ट और आइकन के अलावा, कभी-कभी हमें रंग अनुकूलन स्वयं भी संभालना पड़ता है।

उदाहरण के लिए:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("पाठ")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

दिखने का परिणाम:

![view](../../../Resource/025_view6.png)

इस कोड में, टेक्स्ट सफेद है और बैकग्राउंड काला है।

हल्के मोड में, काला बैकग्राउंड काफी स्पष्ट होता है।

लेकिन गहरे मोड में, यदि पूरा इंटरफ़ेस बैकग्राउंड भी काला है, तो यह काला बैकग्राउंड सिस्टम बैकग्राउंड में घुल जाएगा, और पर्याप्त स्पष्ट नहीं लगेगा।

इस समय, हम रूप-रंग मोड के अनुसार टेक्स्ट रंग और बैकग्राउंड रंग बदल सकते हैं:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("पाठ")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

दिखने का परिणाम:

![view](../../../Resource/025_view7.png)

अब, हल्के मोड में काला बैकग्राउंड और सफेद टेक्स्ट है।

गहरे मोड में सफेद बैकग्राउंड और काला टेक्स्ट है।

इस तरह, चाहे कोई भी रूप-रंग हो, टेक्स्ट और बैकग्राउंड के बीच स्पष्ट कंट्रास्ट रहता है, और उपयोगकर्ता सामग्री को साफ़ देख सकते हैं।

## रंग अनुकूलन के लिए Assets का उपयोग

`colorScheme` से हाथ से जांच करने के अलावा, हम हल्के/गहरे मोड के अनुकूल होने के लिए `Assets` संसाधन रंगों का उपयोग भी कर सकते हैं।

यह तरीका App में सामान्य रंगों को प्रबंधित करने के लिए अधिक उपयुक्त है।

Xcode प्रोजेक्ट में, `Assets` संसाधन फ़ोल्डर ढूँढ़ें।

![assets](../../../Resource/025_color.png)

`Assets` फ़ोल्डर खोलें, खाली क्षेत्र पर राइट-क्लिक करें, और नया रंग संसाधन बनाने के लिए `New Color Set` चुनें।

![assets](../../../Resource/025_color1.png)

यहाँ हम एक रंग बनाते हैं और उसका नाम `redText` रखते हैं।

![assets](../../../Resource/025_color2.png)

रंग संसाधन में हल्के मोड और गहरे मोड के रंग अलग-अलग सेट किए जा सकते हैं।

![assets](../../../Resource/025_color3.png)

संबंधित रंग क्षेत्र चुनने के बाद, दाईं ओर के attributes inspector में रंग बदला जा सकता है।

ऊपरी-दाएँ कोने के `Hide or show the Inspectors` बटन पर क्लिक करके inspector क्षेत्र खोलें।

फिर `Show the Attributes inspector` चुनें, नीचे `Show Color Panel` ढूँढ़ें, और रंग पैनल खोलें।

![assets](../../../Resource/025_color4.png)

रंग पैनल में किसी रंग पर क्लिक करने पर, `redText` से संबंधित रंग क्षेत्र भी साथ-साथ बदल जाएगा।

यहाँ हम `redText` को इस प्रकार सेट करते हैं:

- हल्के मोड में लाल दिखे
- गहरे मोड में हरा दिखे

![assets](../../../Resource/025_color5.png)

SwiftUI कोड में वापस आकर, आप इस रंग को इस तरह उपयोग कर सकते हैं:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

यहाँ:

```swift
Color("redText")
```

का अर्थ है `Assets` से `redText` नाम का रंग संसाधन पढ़ना।

दिखने का परिणाम:

![assets](../../../Resource/025_view8.png)

आप देख सकते हैं कि वही `Color("redText")`, हल्के मोड और गहरे मोड में अलग-अलग रंग दिखाता है।

इससे पता चलता है कि `Assets` रंग स्वयं भी रूप-रंग अनुकूलन का समर्थन करते हैं।

## colorScheme और Assets का अंतर

`colorScheme` और `Assets` दोनों रूप-रंग अनुकूलन लागू कर सकते हैं, लेकिन वे अलग-अलग स्थितियों के लिए उपयुक्त हैं।

यदि आपको केवल हल्के/गहरे मोड के अनुसार टेक्स्ट सामग्री, चित्र नाम या `SF Symbols` नाम बदलना हो, तो `colorScheme` का उपयोग कर सकते हैं।

उदाहरण के लिए:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

यदि वह रंग App में अक्सर उपयोग होता है, जैसे थीम रंग, कार्ड बैकग्राउंड रंग या टेक्स्ट रंग, तो `Assets` रंगों का उपयोग करना अधिक अनुशंसित है।

क्योंकि `Assets` रंगों को कई व्यू में पुन: उपयोग किया जा सकता है, और बाद में संशोधित करना भी अधिक सुविधाजनक होता है।

उदाहरण के लिए:

```swift
Color("redText")
```

इस तरह, केवल `Assets` में रंग बदलना होगा, और जहाँ-जहाँ यह रंग उपयोग हुआ है, वे सभी साथ में बदल जाएँगे।

## सारांश

इस पाठ में, हमने हल्के मोड और गहरे मोड के रूप-रंग अनुकूलन को सीखा।

सबसे पहले, हमने सीखा कि Xcode के `Canvas` में अलग-अलग रूप-रंगों का प्रीव्यू कैसे देखा जाता है, और `Color Scheme Variants` का उपयोग करके हल्के मोड और गहरे मोड को एक साथ कैसे देखा जाता है।

फिर, हमने जाना कि SwiftUI में `Text`, `Image` जैसे व्यू डिफ़ॉल्ट रूप से सिस्टम रूप-रंग के अनुसार रंगों को अपने-आप अनुकूलित करते हैं।

लेकिन यदि हाथ से स्थिर रंग निर्दिष्ट किए जाते हैं, जैसे `Color.white` या `Color.black`, तो ये रंग अपने-आप अदला-बदली नहीं करेंगे।

इसके बाद, हमने `@Environment` और `colorScheme` सीखा:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` के माध्यम से, हम यहजांच सकते हैं कि वर्तमान मोड हल्का है या गहरा, और अलग-अलग रूप-रंगों के अनुसार अलग आइकन, टेक्स्ट या रंग दिखा सकते हैं।

अंत में, हमने `Assets` रंग सीखे।

`Assets` रंग हल्के मोड और गहरे मोड के रंगों को अलग-अलग सेट कर सकते हैं, इसलिए वे App में सामान्य रंग संसाधनों के प्रबंधन के लिए उपयुक्त हैं।

रूप-रंग अनुकूलन App को दिन और रात दोनों समय बेहतर दिखा सकता है, और टेक्स्ट साफ़ न दिखने या बैकग्राउंड अस्पष्ट होने जैसी समस्याओं से भी बचा सकता है।

वास्तविक विकास में, SwiftUI की डिफ़ॉल्ट अनुकूलन क्षमता और `Assets` रंगों को प्राथमिकता देने की सलाह दी जाती है।

जब अलग-अलग रूप-रंगों में टेक्स्ट, आइकन, चित्र बदलने हों, या थोड़े विशेष डिस्प्ले प्रभाव संभालने हों, तब `colorScheme` से जांच करें।

## अतिरिक्त ज्ञान: फ़ॉन्ट आकार निर्दिष्ट करना

पहले `font` सीखते समय, हमने कुछ सिस्टम फ़ॉन्ट शैलियों का उपयोग किया था:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` जैसी सिस्टम फ़ॉन्ट शैलियों का उपयोग करने के अलावा, आप फ़ॉन्ट आकार को हाथ से भी निर्दिष्ट कर सकते हैं।

उदाहरण के लिए:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

यहाँ:

```swift
.font(.system(size: 100))
```

का अर्थ है फ़ॉन्ट आकार को `100 pt` पर सेट करना।

`SF Symbols` आइकन के लिए, `font` आइकन के आकार को भी प्रभावित करता है।

इसलिए, यदि आप आइकन को बड़ा दिखाना चाहते हैं, तो `.font(.system(size:))` के माध्यम से आकार निर्दिष्ट कर सकते हैं।

उदाहरण के लिए:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

ध्यान देने की बात है कि हाथ से फ़ॉन्ट आकार निर्दिष्ट करना अधिक लचीला है, लेकिन सामान्य टेक्स्ट में बड़ी मात्रा में स्थिर फ़ॉन्ट आकार का उपयोग करने की सलाह नहीं दी जाती।

सामान्य टेक्स्ट के लिए `.title`, `.headline`, `.body`, `.caption` जैसी सिस्टम फ़ॉन्ट शैलियों को प्राथमिकता दें।

इससे अलग-अलग उपकरणों और उपयोगकर्ता की फ़ॉन्ट सेटिंग्स के अनुकूल होना बेहतर होता है।
