# రూపానికి అనుగుణంగా మార్చడం

ఈ పాఠంలో, మనం లైట్ మోడ్ మరియు డార్క్ మోడ్ కోసం రూప అనుకూలీకరణను నేర్చుకుంటాము.

iPhone, iPad మరియు Mac లో, వినియోగదారులు లైట్ మోడ్‌ను లేదా డార్క్ మోడ్‌ను ఎంచుకోవచ్చు.

సిస్టమ్ డార్క్ మోడ్‌కు మారినప్పుడు, అనేక App లలో నేపథ్యం, వచనం మరియు ఐకాన్ రంగులు కూడా దానితో పాటు మారుతాయి.

ఉదాహరణకు, పగటిపూట ఇంటర్‌ఫేస్ లైట్ నేపథ్యంగా ఉండవచ్చు; రాత్రివేళ అది డార్క్ నేపథ్యంగా మారవచ్చు.

![view](../../../Resource/025_view9.png)

సిస్టమ్ రూపాన్ని బట్టి ఇంటర్‌ఫేస్ ప్రదర్శనను ఆటోమేటిక్‌గా సర్దుబాటు చేసే ఈ సామర్థ్యాన్ని రూప అనుకూలీకరణ అంటారు.

## Xcode లో రూప అనుకూలీకరణను ప్రివ్యూ చేయడం

రూప అనుకూలీకరణను నేర్చుకునే ముందు, Xcode లో లైట్ మోడ్ మరియు డార్క్ మోడ్‌ను ఎలా ప్రివ్యూ చేయాలో ముందుగా తెలుసుకుందాం.

`Canvas` ప్రాంతం దిగువన ఉన్న `Device Settings` బటన్‌ను క్లిక్ చేస్తే, `Canvas Device Settings` పాప్‌అప్ తెరుచుకుంటుంది.

![canvas](../../../Resource/025_view.png)

పాప్‌అప్‌లో `Color Scheme` సెట్టింగ్‌ను చూడవచ్చు.

![canvas1](../../../Resource/025_view1.png)

ఇందులో:

- `Light Appearance` అంటే లైట్ మోడ్.
- `Dark Appearance` అంటే డార్క్ మోడ్.

ఇక్కడి ద్వారా ప్రివ్యూ రూపాన్ని మార్చి, ప్రస్తుత ఇంటర్‌ఫేస్ లైట్ మోడ్ మరియు డార్క్ మోడ్‌లలో ఎలా కనిపిస్తుందో చూడవచ్చు.

## రెండు రూపాలను పక్కపక్కన చూపించడం

లైట్ మోడ్ మరియు డార్క్ మోడ్‌ను ఒకేసారి చూడాలనుకుంటే, `Variants` ఫీచర్‌ను ఉపయోగించవచ్చు.

`Canvas` ప్రాంతం దిగువన ఉన్న `Variants` బటన్‌ను క్లిక్ చేసి, `Color Scheme Variants` ఎంచుకోండి.

![canvas2](../../../Resource/025_view2.png)

ఎంచుకున్న తర్వాత, `Canvas` లైట్ మోడ్ మరియు డార్క్ మోడ్ ప్రివ్యూ ఫలితాలను ఒకేసారి చూపిస్తుంది.

![canvas3](../../../Resource/025_view3.png)

ఇలా చేస్తే రెండు రూపాల్లోని ఇంటర్‌ఫేస్ తేడాలను పోల్చడం మరింత సులభం అవుతుంది.

సంక్షిప్తంగా:

లైట్/డార్క్ మోడ్‌ను తాత్కాలికంగా మాత్రమే మార్చాలనుకుంటే, `Canvas Device Settings` ఉపయోగించవచ్చు.

రెండు రూపాలను ఒకేసారి చూడాలనుకుంటే, `Color Scheme Variants` ఉపయోగించవచ్చు.

## View డిఫాల్ట్‌గా రూపానికి అనుగుణంగా మారుతుంది

SwiftUI లో, అనేక సిస్టమ్ view లు డిఫాల్ట్‌గా లైట్ మోడ్ మరియు డార్క్ మోడ్‌కు ఆటోమేటిక్‌గా అనుగుణంగా మారుతాయి.

ఉదాహరణకు:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("వెలుగు")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("కొంత వచనం కొంత వచనం")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ప్రదర్శన ఫలితం:

![canvas](../../../Resource/025_view4.png)

లైట్ మోడ్‌లో, ఐకాన్ మరియు శీర్షిక సాధారణంగా ముదురు రంగులో కనిపిస్తాయి.

డార్క్ మోడ్‌లో, ఐకాన్ మరియు శీర్షిక ఆటోమేటిక్‌గా లైట్ రంగులోకి మారుతాయి.

దీనికి కారణం `Text`, `Image` వంటి SwiftUI view లు డిఫాల్ట్‌గా సిస్టమ్ రూపాన్ని బట్టి రంగులను సర్దుబాటు చేస్తాయి.

అంటే, మనం స్థిరమైన రంగును చేతితో నిర్దేశించకపోతే, రూప అనుకూలీకరణలో కొంత భాగాన్ని SwiftUI మన కోసం నిర్వహిస్తుంది.

## స్థిర రంగులు ఆటోమేటిక్‌గా పరస్పరం మారవు

గమనించాల్సిన విషయం ఏమిటంటే, ఒక స్థిర రంగును మనం చేతితో నిర్దేశిస్తే, అది లైట్/డార్క్ మోడ్‌లను బట్టి ఆటోమేటిక్‌గా మారదు.

ఉదాహరణకు:

```swift
Text("కొంత వచనం కొంత వచనం")
	.foregroundStyle(Color.gray)
```

ఇక్కడ వచనం `Color.gray` గా నిర్దేశించబడింది, కాబట్టి అది ఎప్పుడూ బూడిద రంగులోనే కనిపిస్తుంది.

మరొక ఉదాహరణ:

```swift
Text("శీర్షిక")
    .foregroundStyle(Color.white)
```

ఈ కోడ్ లైట్ మోడ్‌లో అయినా డార్క్ మోడ్‌లో అయినా వచనాన్ని తెలుపు రంగులోనే చూపిస్తుంది.

నేపథ్యం కూడా లైట్ రంగులో ఉంటే, తెలుపు వచనం స్పష్టంగా కనిపించకపోవచ్చు.

కాబట్టి రూప అనుకూలీకరణ చేస్తూ ఉన్నప్పుడు, `Color.white`, `Color.black` వంటి స్థిర రంగులను యాదృచ్ఛికంగా hardcode చేయకుండా ఉండటం మంచిది.

చాలా సందర్భాల్లో, ముందుగా సిస్టమ్ semantic style లను ఉపయోగించవచ్చు. ఉదాహరణకు:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

ఇందులో:

- `.primary` సాధారణంగా ప్రధాన కంటెంట్‌కు ఉపయోగిస్తారు.
- `.secondary` సాధారణంగా ద్వితీయ కంటెంట్‌కు ఉపయోగిస్తారు.

వీటి ప్రదర్శన ప్రభావం లైట్/డార్క్ మోడ్‌లను బట్టి ఆటోమేటిక్‌గా సర్దుబాటు అవుతుంది.

## రూపాన్ని బట్టి వేర్వేరు కంటెంట్ చూపించడం

కొన్నిసార్లు, మనం రంగు మారడమే కాకుండా, వేర్వేరు రూపాల్లో వేర్వేరు కంటెంట్ చూపించాలని కోరుకుంటాము.

ఉదాహరణకు:

- లైట్ మోడ్‌లో సాధారణ బల్బ్ ఐకాన్‌ను చూపడం.
- డార్క్ మోడ్‌లో వెలిగే బల్బ్ ఐకాన్‌ను చూపడం.
- లైట్ మోడ్‌లో శీర్షికను `వెలుగు` గా చూపడం.
- డార్క్ మోడ్‌లో శీర్షికను `చీకటి` గా చూపడం.

ఈ సమయంలో, ప్రస్తుత సిస్టమ్ లైట్ మోడ్‌లో ఉందా లేదా డార్క్ మోడ్‌లో ఉందా అని నిర్ణయించాలి.

SwiftUI లో, ప్రస్తుత రూప మోడ్‌ను `colorScheme` environment value ద్వారా పొందవచ్చు:

```swift
@Environment(\.colorScheme) private var colorScheme
```

పూర్తి కోడ్:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "వెలుగు" : "చీకటి"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("కొంత వచనం కొంత వచనం")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ప్రదర్శన ఫలితం:

![view](../../../Resource/025_view5.png)

ప్రివ్యూ ఫలితాల నుండి, ఒకే view వేర్వేరు రూప మోడ్‌లలో చూపించే కంటెంట్ పూర్తిగా ఒకేలా ఉండదని చూడవచ్చు.

లైట్ మోడ్‌లో, `Image` సాధారణ బల్బ్ ఐకాన్‌ను చూపుతుంది, శీర్షిక `వెలుగు` గా కనిపిస్తుంది.

డార్క్ మోడ్‌లో, `Image` వెలిగే బల్బ్ ఐకాన్‌ను చూపుతుంది, శీర్షిక `చీకటి` గా కనిపిస్తుంది.

అంటే, ఇక్కడ మార్పు రంగుల మార్పుతో మాత్రమే పరిమితం కాదు; ఐకాన్ పేరు మరియు శీర్షిక వచనం కూడా మారుతున్నాయి.

ఎందుకంటే మనం ఐకాన్ మరియు శీర్షికను నేరుగా `body` లో hardcode చేయలేదు; `colorScheme` ఆధారంగా వేర్వేరు కంటెంట్‌ను లెక్కించాము.

ఈ కోడ్‌లో, మనం రెండు computed property లను ఉపయోగించాము:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "వెలుగు" : "చీకటి"
}
```

`colorScheme` `.light` అయితే, ప్రస్తుత మోడ్ లైట్ మోడ్ అని అర్థం.

`colorScheme` `.dark` అయితే, ప్రస్తుత మోడ్ డార్క్ మోడ్ అని అర్థం.

కాబట్టి వేర్వేరు రూప మోడ్‌లను బట్టి, మనం వేర్వేరు ఐకాన్ పేర్లు మరియు శీర్షిక వచనాన్ని తిరిగి ఇవ్వవచ్చు.

ప్రస్తుత స్థితిని బట్టి view చూపాల్సిన కంటెంట్‌ను లెక్కించడం computed property యొక్క సాధారణ ఉపయోగాల్లో ఒకటి.

## @Environment ను అర్థం చేసుకోవడం

`@Environment` ను మనం మొదటిసారి చూస్తున్నాం.

`@Environment` ను ఇలా అర్థం చేసుకోవచ్చు: SwiftUI environment నుండి ఒక value ను చదవడం.

App నడుస్తున్నప్పుడు, సిస్టమ్ అనేక environment సమాచారాన్ని అందిస్తుంది. ఉదాహరణకు:

- ప్రస్తుత భాష
- ప్రస్తుత రూప మోడ్
- ప్రస్తుత layout దిశ
- ప్రస్తుత font size సెట్టింగ్

ప్రస్తుత రూప మోడ్‌ను పొందాల్సినప్పుడు, `colorScheme` ను చదవవచ్చు:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ఇందులో

```swift
\.colorScheme
```

environment నుండి `colorScheme` అనే value ను చదవడాన్ని సూచిస్తుంది.

```swift
private var colorScheme
```

చదివిన value ను `colorScheme` అనే variable లో నిల్వ చేయడాన్ని సూచిస్తుంది.

variable పేరు మీరు స్వయంగా నిర్ణయించవచ్చు. ఉదాహరణకు, ఇలా కూడా వ్రాయవచ్చు:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

ముందున్న `\.colorScheme` మారకుండా ఉన్నంత వరకు, చదివేది ప్రస్తుత రూప మోడ్‌నే అని అర్థం.

## colorScheme యొక్క రెండు సాధారణ values

`colorScheme` కు సాధారణంగా రెండు values ఉంటాయి:

```swift
.light
.dark
```

వాటి అర్థం:

- `.light`: లైట్ మోడ్
- `.dark`: డార్క్ మోడ్

ఉదాహరణకు:

```swift
private var titleName: String {
    colorScheme == .light ? "వెలుగు" : "చీకటి"
}
```

ఈ కోడ్ అర్థం:

ప్రస్తుతది లైట్ మోడ్ అయితే, `colorScheme == .light` నిజమవుతుంది, అప్పుడు `"వెలుగు"` తిరిగి వస్తుంది.

లేకపోతే, `"చీకటి"` తిరిగి వస్తుంది.

అందువల్ల, లైట్ మోడ్‌లో:

```swift
Text(titleName)
```

చూపేది:

```swift
వెలుగు
```

డార్క్ మోడ్‌లో చూపేది:

```swift
చీకటి
```

ఈ విధంగా, వేర్వేరు రూపాల ఆధారంగా వేర్వేరు కంటెంట్‌ను చూపించవచ్చు.

## రంగు అనుకూలీకరణ ఉదాహరణ

వచనం మరియు ఐకాన్‌లతో పాటు, కొన్నిసార్లు మనమే రంగు అనుకూలీకరణను నిర్వహించాలి.

ఉదాహరణకు:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("వచనం")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

ప్రదర్శన ఫలితం:

![view](../../../Resource/025_view6.png)

ఈ కోడ్‌లో, వచనం తెలుపు రంగులో ఉంది, నేపథ్యం నలుపు రంగులో ఉంది.

లైట్ మోడ్‌లో, నలుపు నేపథ్యం స్పష్టంగా కనిపిస్తుంది.

కానీ డార్క్ మోడ్‌లో, మొత్తం ఇంటర్‌ఫేస్ నేపథ్యమూ నలుపు అయితే, ఈ నలుపు నేపథ్యం సిస్టమ్ నేపథ్యంతో కలిసిపోయి తక్కువ స్పష్టంగా కనిపించవచ్చు.

ఈ సమయంలో, రూప మోడ్‌ను బట్టి వచన రంగు మరియు నేపథ్య రంగును మార్చవచ్చు:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("వచనం")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

ప్రదర్శన ఫలితం:

![view](../../../Resource/025_view7.png)

ఇప్పుడు, లైట్ మోడ్‌లో నలుపు నేపథ్యం మరియు తెలుపు వచనం ఉంటాయి.

డార్క్ మోడ్‌లో తెలుపు నేపథ్యం మరియు నలుపు వచనం ఉంటాయి.

ఇలా ఏ రూపంలోనైనా వచనం మరియు నేపథ్యం మధ్య స్పష్టమైన వ్యత్యాసం ఉంటుంది; వినియోగదారులు కంటెంట్‌ను స్పష్టంగా చూడగలరు.

## Assets ఉపయోగించి రంగులను అనుకూలీకరించడం

`colorScheme` ను చేతితో తనిఖీ చేయడమే కాకుండా, లైట్/డార్క్ మోడ్‌లకు అనుకూలంగా `Assets` resource color లను కూడా ఉపయోగించవచ్చు.

ఈ విధానం App లో సాధారణంగా ఉపయోగించే రంగులను నిర్వహించడానికి మరింత అనుకూలం.

Xcode ప్రాజెక్ట్‌లో, `Assets` resource folder ను కనుగొనండి.

![assets](../../../Resource/025_color.png)

`Assets` folder ను తెరిచి, ఖాళీ ప్రాంతంపై right-click చేసి, `New Color Set` ఎంచుకుని ఒక కొత్త color resource సృష్టించండి.

![assets](../../../Resource/025_color1.png)

ఇక్కడ మనం ఒక రంగును సృష్టించి, దానికి `redText` అని పేరు పెడుతున్నాము.

![assets](../../../Resource/025_color2.png)

Color resource లో లైట్ మోడ్ మరియు డార్క్ మోడ్‌లకు వేర్వేరు రంగులను సెట్ చేయవచ్చు.

![assets](../../../Resource/025_color3.png)

సంబంధిత రంగు ప్రాంతాన్ని ఎంచుకున్న తర్వాత, కుడివైపు attributes inspector లో రంగును మార్చవచ్చు.

కుడి పై మూలలోని `Hide or show the Inspectors` బటన్‌ను క్లిక్ చేసి inspector ప్రాంతాన్ని తెరవండి.

ఆ తర్వాత `Show the Attributes inspector` ఎంచుకుని, దిగువన `Show Color Panel` కనుగొని color panel ను తెరవండి.

![assets](../../../Resource/025_color4.png)

color panel లోని రంగుపై క్లిక్ చేస్తే, `redText` కు సంబంధించిన రంగు ప్రాంతం సమకాలికంగా మారుతుంది.

ఇక్కడ మనం `redText` ను ఇలా సెట్ చేస్తున్నాము:

- లైట్ మోడ్‌లో ఎరుపు రంగు చూపుతుంది
- డార్క్ మోడ్‌లో ఆకుపచ్చ రంగు చూపుతుంది

![assets](../../../Resource/025_color5.png)

SwiftUI కోడ్‌కు తిరిగి వచ్చాక, ఈ రంగును ఇలా ఉపయోగించవచ్చు:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

ఇందులో:

```swift
Color("redText")
```

`Assets` నుండి `redText` అనే పేరున్న color resource ను చదవడాన్ని సూచిస్తుంది.

ప్రదర్శన ఫలితం:

![assets](../../../Resource/025_view8.png)

ఒకే `Color("redText")` లైట్ మోడ్ మరియు డార్క్ మోడ్‌లలో వేర్వేరు రంగులుగా కనిపిస్తుందని చూడవచ్చు.

దీని అర్థం `Assets` రంగులు స్వయంగా కూడా రూప అనుకూలీకరణకు మద్దతు ఇస్తాయి.

## colorScheme మరియు Assets మధ్య తేడా

`colorScheme` మరియు `Assets` రెండూ రూప అనుకూలీకరణను సాధించగలవు, కానీ వాటికి సరిపోయే సందర్భాలు వేర్వేరు.

లైట్/డార్క్ మోడ్‌ను బట్టి వచన కంటెంట్, image పేరు లేదా `SF Symbols` పేరు మార్చడమే అయితే, `colorScheme` ఉపయోగించవచ్చు.

ఉదాహరణకు:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

App లో తరచుగా ఉపయోగించే రంగులు, ఉదాహరణకు theme color, card background color, text color వంటివి అయితే, `Assets` రంగులను ఉపయోగించడం మరింత సిఫారసు చేయబడుతుంది.

ఎందుకంటే `Assets` రంగులను అనేక view లలో మళ్లీ ఉపయోగించవచ్చు, తరువాత మార్చడమూ సులభంగా ఉంటుంది.

ఉదాహరణకు:

```swift
Color("redText")
```

ఇలా చేస్తే `Assets` లో రంగును మాత్రమే మార్చాలి; ఈ రంగును ఉపయోగించే అన్ని చోట్లూ కలిసి మారుతాయి.

## సారాంశం

ఈ పాఠంలో, మనం లైట్ మోడ్ మరియు డార్క్ మోడ్ కోసం రూప అనుకూలీకరణను నేర్చుకున్నాము.

మొదట, Xcode లోని `Canvas` లో వేర్వేరు రూపాలను ఎలా ప్రివ్యూ చేయాలో నేర్చుకున్నాము; అలాగే `Color Scheme Variants` ఉపయోగించి లైట్ మోడ్ మరియు డార్క్ మోడ్‌ను ఒకేసారి ఎలా చూడాలో కూడా నేర్చుకున్నాము.

తర్వాత, SwiftUI లోని `Text`, `Image` వంటి view లు డిఫాల్ట్‌గా సిస్టమ్ రూపాన్ని బట్టి రంగులను ఆటోమేటిక్‌గా అనుకూలీకరిస్తాయని తెలుసుకున్నాము.

కానీ `Color.white` లేదా `Color.black` వంటి స్థిర రంగులను చేతితో నిర్దేశిస్తే, అవి ఆటోమేటిక్‌గా పరస్పరం మారవు.

తరువాత, మనం `@Environment` మరియు `colorScheme` నేర్చుకున్నాము:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` ద్వారా, ప్రస్తుతది లైట్ మోడ్ లేదా డార్క్ మోడ్ అని నిర్ణయించి, వేర్వేరు రూపాలకు అనుగుణంగా వేర్వేరు ఐకాన్‌లు, వచనం లేదా రంగులను చూపించవచ్చు.

చివరిగా, మనం `Assets` రంగులను నేర్చుకున్నాము.

`Assets` రంగులు లైట్ మోడ్ మరియు డార్క్ మోడ్‌లకు వేర్వేరు రంగులను సెట్ చేయగలవు; App లో సాధారణంగా ఉపయోగించే రంగు resources ను నిర్వహించడానికి ఇవి సరిపోతాయి.

రూప అనుకూలీకరణ App ను పగలు మరియు రాత్రి రెండింటిలోనూ మెరుగ్గా కనిపించేలా చేస్తుంది; అలాగే వచనం కనిపించకపోవడం, నేపథ్యం స్పష్టంగా లేకపోవడం వంటి సమస్యలను నివారించగలదు.

ప్రాయోగిక అభివృద్ధిలో, SwiftUI డిఫాల్ట్ అనుకూలీకరణ సామర్థ్యాన్ని మరియు `Assets` రంగులను ముందుగా ఉపయోగించడం మంచిది.

వేర్వేరు రూపాల్లో వచనం, ఐకాన్‌లు, చిత్రాలు మార్చాల్సినప్పుడు, లేదా కొద్దిపాటి ప్రత్యేక ప్రదర్శన ప్రభావాలను నిర్వహించాల్సినప్పుడు, ఆ తర్వాత `colorScheme` ద్వారా నిర్ణయించండి.

## విస్తృత జ్ఞానం: font size ను నిర్దేశించడం

ముందు `font` నేర్చుకున్నప్పుడు, కొన్ని system font style లను ఉపయోగించాము:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` వంటి system font style లను ఉపయోగించడమే కాకుండా, font size ను చేతితో కూడా నిర్దేశించవచ్చు.

ఉదాహరణకు:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

ఇక్కడి:

```swift
.font(.system(size: 100))
```

font size ను `100 pt` గా సెట్ చేయడాన్ని సూచిస్తుంది.

`SF Symbols` ఐకాన్‌లకు కూడా, `font` ఐకాన్ పరిమాణాన్ని ప్రభావితం చేస్తుంది.

కాబట్టి ఐకాన్‌ను పెద్దగా చూపించాలని ఉంటే, `.font(.system(size:))` ద్వారా పరిమాణాన్ని నిర్దేశించవచ్చు.

ఉదాహరణకు:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

గమనించాల్సిన విషయం ఏమిటంటే, font size ను చేతితో నిర్దేశించడం మరింత వశ్యత ఇస్తుంది; కానీ సాధారణ వచనంలో స్థిర font sizeలను పెద్ద మొత్తంలో ఉపయోగించడం సిఫారసు చేయబడదు.

సాధారణ వచనానికి ముందుగా `.title`, `.headline`, `.body`, `.caption` వంటి system font style లను ఉపయోగించండి.

ఇలా చేస్తే వేర్వేరు devices మరియు వినియోగదారుల font settings కు మెరుగ్గా అనుకూలించవచ్చు.
