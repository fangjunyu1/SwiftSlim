# లాగిన్ పరస్పర చర్యలు

మునుపటి పాఠంలో, మేము ఒక లాగిన్ వీక్షణను రూపొందించాము మరియు బటన్‌లు, షరతు ఆధారిత లాజిక్‌ను ఉపయోగించి ఇన్‌పుట్ చేసిన విషయంపై సరళమైన ధృవీకరణను అమలు చేశాము.

ఈ పాఠంలో, మునుపటి పాఠంలోని కోడ్‌ను ఆధారంగా తీసుకుని `SecureField` పాస్‌వర్డ్ ఇన్‌పుట్ ఫీల్డ్, `Alert` హెచ్చరిక పెట్టె, అలాగే వేరియబుల్స్ ఆధారంగా వేర్వేరు వీక్షణల ప్రదర్శనను ఎలా నియంత్రించాలో నేర్చుకుంటాము.

ఈ పాఠం ద్వారా, లాగిన్ వీక్షణ యొక్క ప్రాథమిక పరస్పర చర్యల ప్రవాహాన్ని మీరు మరింత బాగా అర్థం చేసుకోవచ్చు. ఇది అనేక App‌లలో సాధారణంగా ఉపయోగించే ప్రాథమిక లాజిక్ కూడా.

## పాస్‌వర్డ్ ఇన్‌పుట్ ఫీల్డ్

మునుపటి పాఠంలో, పాస్‌వర్డ్‌ను నమోదు చేయడానికి మేము `TextField` ఉపయోగించాము.

కానీ నిజమైన లాగిన్ ఇంటర్‌ఫేస్‌లో, పాస్‌వర్డ్ సాధారణంగా నేరుగా చూపించబడదు. బదులుగా అది బొట్లు లేదా మరో దాచిన ప్రభావం రూపంలో చూపబడుతుంది; ఇది వినియోగదారుడి గోప్యతను రక్షించడంలో సహాయపడుతుంది.

SwiftUI లో, పాస్‌వర్డ్‌లను నమోదు చేయడానికి ప్రత్యేకంగా ఉపయోగించే నియంత్రణ `SecureField`.

ఉదాహరణకు:

```swift
SecureField("Password", text: $password)
```

దాని రాత `TextField` కు చాలా దగ్గరగా ఉంటుంది. దీనికీ ఒక సూచన వచనం మరియు ఒక బౌండ్ వేరియబుల్ అవసరం.

దీనిని సులభంగా ఇలా అర్థం చేసుకోవచ్చు: `SecureField` కూడా ఒక ఇన్‌పుట్ ఫీల్డ్‌నే, కానీ ఇందులో నమోదు చేసిన విషయం దాచబడుతుంది.

ఇంతకుముందు పాస్‌వర్డ్‌ను నమోదు చేయడానికి ఉపయోగించిన `TextField` ను నేరుగా `SecureField` గా మార్చవచ్చు:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

వినియోగదారు పాస్‌వర్డ్ నమోదు చేసినప్పుడు, పాస్‌వర్డ్ విషయం ఇకపై ఇంటర్‌ఫేస్‌లో నేరుగా కనిపించదు.

ప్రదర్శన ప్రభావం:

![password](../../Resource/020_password.png)

ఇప్పుడు మనం వినియోగదారు పేరు మరియు పాస్‌వర్డ్‌ను నమోదు చేసి, ఆపై బటన్‌ను నొక్కి ఇన్‌పుట్ సరైనదా కాదా అని నిర్ణయించవచ్చు.

### ఫీడ్‌బ్యాక్ సందేశాలను చూపించడం

ఇంతకుముందు మేము ఫలితాలను ప్రదర్శించడానికి ఎప్పుడూ `print` ఉపయోగించేవాళ్లం.

`print` లోని విషయం Console లో మాత్రమే కనిపిస్తుంది. డెవలపర్లు Xcode లో ప్రోగ్రామ్ నడిపేటప్పుడు దాన్ని చూడగలరు, కానీ ఫోన్‌లోని సాధారణ వినియోగదారులు ఈ సందేశాలను చూడలేరు.

అంటే, ప్రోగ్రామ్ ఇప్పటికే ధృవీకరణ పూర్తి చేసినప్పటికీ, ఇన్‌పుట్ చేసిన విషయం సరైనదా కాదా అనేది వినియోగదారుడికి తెలియదు.

ఈ సమయంలో, సందేశాలను నేరుగా ఇంటర్‌ఫేస్‌లో చూపించే ఒక విధానం అవసరం.

## Alert హెచ్చరిక పెట్టె

SwiftUI లో, ఒక సందేశ పెట్టెను పాప్-అప్ చేయడానికి `Alert` ఉపయోగించవచ్చు.

ప్రాథమిక వినియోగం:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

ఈ కోడ్ ముందటి నియంత్రణల కంటే కొంచెం క్లిష్టంగా కనిపిస్తుంది, కానీ దాన్ని భాగాలుగా విభజించి అర్థం చేసుకోవచ్చు.

`Alert` ప్రధానంగా ఈ క్రింది భాగాలను కలిగి ఉంటుంది:

- `"Title"`: హెచ్చరిక పెట్టె యొక్క శీర్షిక
- `isPresented`: హెచ్చరిక పెట్టెను చూపించాలా వద్దా అని నియంత్రించడానికి ఉపయోగించే బౌండ్ Boolean విలువ
- `Button(...)`: హెచ్చరిక పెట్టెలోని బటన్
- `message:`: హెచ్చరిక పెట్టెలో చూపబడే సందేశ విషయం

దీనిని సులభంగా ఇలా అర్థం చేసుకోవచ్చు: బౌండ్ వేరియబుల్ `true` గా మారినప్పుడు, సిస్టమ్ ఈ హెచ్చరిక పెట్టెను చూపిస్తుంది.

### Alert ఒక Modifier

మనం ముందు నేర్చుకున్న అనేక Modifier‌ల మాదిరిగానే, `Alert` కూడా ఏదో ఒక వీక్షణకు తర్వాత జోడించబడుతుంది.

ఉదాహరణకు, ఇది ఒక బటన్ తర్వాత జోడించవచ్చు:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

అలాగే బయట ఉన్న `VStack` తర్వాత కూడా జోడించవచ్చు:

```swift
VStack {
    // ...
}
.alert() {...}
```

అంటే, `Alert` అనేది ఇంటర్‌ఫేస్‌లో విడిగా ఉంచబడే స్వతంత్ర వీక్షణ కాదు. దాని బదులుగా, అది ఒక నిర్దిష్ట వీక్షణకు జతచేయబడే Modifier.

దీనిని సులభంగా ఇలా అర్థం చేసుకోవచ్చు: సాధారణంగా `Alert` ను బటన్ లేదా బాహ్య లేఅవుట్ తర్వాత రాసి, ఒక నిర్దిష్ట షరతు నెరవేరినప్పుడు సందేశాన్ని చూపించడానికి ఉపయోగిస్తారు.

### Alert ను చూపించడం

ఒక `Alert` చూపించాలని అనుకుంటే, దాన్ని నియంత్రించడానికి ఒక వేరియబుల్ అవసరం.

ఉదాహరణకు:

```swift
@State private var showAlert = false
```

ఈ వేరియబుల్ `Bool` రకానికి చెందింది, అంటే దీనికి రెండు విలువలు మాత్రమే ఉంటాయి: `true` లేదా `false`.

తర్వాత దాన్ని `Alert` యొక్క `isPresented` కు బైండ్ చేయండి:

```swift
isPresented: $showAlert
```

`showAlert` `false` గా ఉన్నప్పుడు హెచ్చరిక పెట్టె కనిపించదు; `showAlert` `true` గా ఉన్నప్పుడు హెచ్చరిక పెట్టె కనిపిస్తుంది.

### బటన్ ద్వారా హెచ్చరిక పెట్టెను చూపించడం

ఇప్పుడు ముందు సరళమైన పరిస్థితిని అమలు చేద్దాం:

వినియోగదారు ఏమీ నమోదు చేయకుండా నేరుగా `Sign in` బటన్‌ను నొక్కినప్పుడు, ఇంకా ఎలాంటి విషయం నమోదు చేయలేదని చెప్పే ఒక సందేశ పెట్టెను చూపించాలి.

మొదట, ఈ సందేశ పెట్టెను నియంత్రించడానికి ఒక వేరియబుల్ సృష్టించండి:

```swift
@State private var showEmpty = false
```

తర్వాత, `Button` తర్వాత ఒక `Alert` జోడించండి:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

తర్వాత, బటన్ క్లిక్ కోడ్‌లో తనిఖీ చేయండి:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

`user` మరియు `password` రెండూ ఖాళీగా ఉంటే, `showEmpty` ను `true` గా మార్చండి.

మరియు `Alert` ఈ వేరియబుల్‌కు బైండ్ అయి ఉండటంతో, అది `true` గా మారినప్పుడు సిస్టమ్ ఆటోమేటిక్‌గా హెచ్చరిక పెట్టెను చూపిస్తుంది.

ప్రదర్శన ప్రభావం:

![alert](../../Resource/020_alert1.png)

ఈ విధంగా, వినియోగదారు బటన్ నొక్కిన వెంటనే సందేశాన్ని చూడగలడు, మరియు డెవలపర్‌కు Console లోని `print` అవుట్‌పుట్‌ను తనిఖీ చేయాల్సిన అవసరం ఉండదు.

### బటన్ మరియు హెచ్చరిక పెట్టెలను మరింత మెరుగుపరచడం

ఇంతకుముందు, మేము “ఏమీ పూరించని” పరిస్థితినే మాత్రమే నిర్వహించాము.

ఇంకా ఈ క్రింది పరిస్థితులను కూడా నిర్వహించాలి:

- వినియోగదారు పేరు నమోదు కాలేదు
- పాస్‌వర్డ్ నమోదు కాలేదు
- వినియోగదారు పేరు మరియు పాస్‌వర్డ్ రెండూ నమోదు చేశారు, కానీ ఇన్‌పుట్ తప్పుగా ఉంది

కాబట్టి, ఈ పరిస్థితులకు అనుగుణమైన సందేశాలను కూడా సిద్ధం చేయాలి.

ముందుగా, మూడు కొత్త వేరియబుల్‌లను జోడించండి:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

తర్వాత, బటన్ తర్వాత సంబంధిత `Alert` లను జోడిస్తూ కొనసాగండి:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

ఆపై, బటన్‌లోని నిర్ణయ లాజిక్‌ను పూర్తి చేయండి:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

వినియోగదారు ఎలాంటి సమాచారం నమోదు చేయలేదా, ఒక సమాచారాన్ని వదిలేశాడా, లేదా ఇన్‌పుట్ ధృవీకరణలో విఫలమైందా అని బటన్ నిర్ణయించినప్పుడు, సంబంధిత Boolean వేరియబుల్‌ను `true` గా మార్చుతుంది; ఆ తర్వాత సంబంధిత `Alert` కనిపిస్తుంది.

ఇక్కడ మీరు కొన్ని పరిస్థితులను మీరే పరీక్షించాలని నేను సూచిస్తున్నాను:

- ఏమీ పూరించవద్దు
- వినియోగదారు పేరును మాత్రమే పూరించండి
- పాస్‌వర్డ్‌ను మాత్రమే పూరించండి
- వినియోగదారు పేరు మరియు పాస్‌వర్డ్ రెండూ పూరించండి, కానీ విషయం తప్పుగా ఉండాలి
- వినియోగదారు పేరు మరియు పాస్‌వర్డ్ రెండూ పూరించండి, విషయం సరిగ్గా ఉండాలి

ఇది మన కోడ్‌లో సమస్యలేమైనా ఉన్నాయా అని నిర్ధారించడానికి సహాయపడుతుంది. ఇది App అభివృద్ధిలో చాలా ముఖ్యమైన “పరీక్ష” దశ కూడా.

ఏదైనా ఒక పరిస్థితిలో ప్రోగ్రామ్ తప్పుగా ప్రవర్తిస్తే, లాజిక్‌లో సమస్య ఉండవచ్చని, అలాగే ప్రోగ్రామ్‌లో ఒక `Bug` కనిపించిందని అర్థం.

అభివృద్ధిలో సాధారణంగా `Bug` అని చెప్పేది లోపం, బలహీనత లేదా ఆశించిన ప్రవర్తనకు సరిపోని ప్రవర్తనను సూచిస్తుంది.

## లాగిన్ వీక్షణను పూర్తి చేయడం

మునుపు మనం మాట్లాడుకున్న `Alert` ఉదాహరణలు అన్నీ “ఇన్‌పుట్ తప్పుగా ఉన్నప్పుడు” వినియోగదారునికి ఎలా ఫీడ్‌బ్యాక్ ఇవ్వాలనే విషయంపై ఉండేవి.

వినియోగదారు సరైన వినియోగదారు పేరు మరియు పాస్‌వర్డ్‌ను నమోదు చేసినప్పుడు, సాధారణంగా మేము “హెచ్చరిక పెట్టె”ను చూపించము. బదులుగా, లాగిన్ తర్వాతి ఇంటర్‌ఫేస్‌లోకి నేరుగా ప్రవేశిస్తాము.

ఇది చాలా ముఖ్యమైన భావనను కలిగి ఉంది: **ఒక వేరియబుల్ విలువ ఆధారంగా వేర్వేరు వీక్షణలను చూపించడం.**

ఉదాహరణకు:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

ఇక్కడ, ఏ వీక్షణ చూపించబడాలో నిర్వహించడానికి `Bool` రకాన్ని ఉపయోగిస్తున్నాము:

- `login` `true` అయితే, లాగిన్ ఇప్పటికే విజయవంతమైందని, లాగిన్ తర్వాతి విషయం చూపబడుతుందని అర్థం
- `login` `false` అయితే, లాగిన్ విజయవంతం కాలేదని, లాగిన్ ఇంటర్‌ఫేస్ ఇంకా చూపబడుతూనే ఉంటుందని అర్థం.

`if` స్టేట్‌మెంట్ Swift లో ఒక షరతు ప్రకటన అయినప్పటికీ, SwiftUI యొక్క `body` లో ఇది వీక్షణల ప్రదర్శన లాజిక్‌ను నియంత్రించడానికి ఉపయోగించవచ్చు.

అంటే, షరతు ఫలితం ఆధారంగా ప్రస్తుతం ఏ వీక్షణ చూపించాలో SwiftUI నిర్ణయిస్తుంది.

ఈ ప్రదర్శన లాజిక్‌ను ప్రస్తుత `ContentView` లో ఉంచవచ్చు:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// అసలు లాగిన్ వీక్షణ కోడ్
		}
	}
}
```

ఇక్కడ, వినియోగదారు ఇప్పటికే విజయవంతంగా లాగిన్ అయ్యాడో లేదో తెలియజేయడానికి `login` అనే కొత్త వేరియబుల్‌ను జోడించాము.

తర్వాత, మేము ముందే రాసిన లాగిన్ వీక్షణ కోడ్‌ను `else` స్టేట్‌మెంట్ తర్వాత ఉంచాము.

వీక్షణ ప్రదర్శన లాజిక్:

- `login` `false` గా ఉన్నప్పుడు, లాగిన్ ఇంకా విజయవంతం కాలేదని అర్థం; కాబట్టి అసలు లాగిన్ వీక్షణ చూపబడుతూనే ఉంటుంది.
- `login` `true` గా ఉన్నప్పుడు, లాగిన్ విజయవంతమైందని అర్థం. ఈ సమయంలో లాగిన్ వీక్షణ ఇకపై చూపబడదు; బదులుగా `Login successful` చూపబడుతుంది.

అంటే, `ContentView` `login` విలువ ఆధారంగా వేర్వేరు వీక్షణ విషయాలను చూపిస్తుంది.

తర్వాత, బటన్‌లోని విజయవంతమైన లాగిన్ లాజిక్‌ను మార్చుదాం:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

వినియోగదారు నమోదు చేసిన వినియోగదారు పేరు మరియు పాస్‌వర్డ్ రెండూ సరైనవైతే, మేము `login` ను `true` గా మార్చుతాము.

`login` విలువ మారిన వెంటనే, `ContentView` లోని `if` తనిఖీ ఫలితం కూడా మారుతుంది, మరియు ఇంటర్‌ఫేస్ అసలు లాగిన్ వీక్షణ నుండి `Login successful` కు మారుతుంది.

ప్రదర్శన ప్రభావం ఈ విధంగా ఉంటుంది:

![view](../../Resource/020_view.png)

ఇప్పుడు “షరతుల ఆధారంగా వేర్వేరు వీక్షణలను చూపించడం” అనే భావనను ఈ ప్రస్తుత లాగిన్ పేజీలో నిజంగా ఉపయోగించాము.

నిజమైన App‌లలో కూడా సాధారణంగా ఇదే విధంగా పనిచేస్తుంది: ఒక నిర్దిష్ట విలువ ఆధారంగా వేర్వేరు వీక్షణలు చూపబడతాయి.

ఉదాహరణకు, వినియోగదారు లాగిన్ కాలేదంటే లాగిన్ పేజీ చూపబడుతుంది; వినియోగదారు విజయవంతంగా లాగిన్ అయిన తర్వాత, మరో ఇంటర్‌ఫేస్ చూపబడుతుంది.

మూల ఆలోచన ఏమిటంటే: **ఇంటర్‌ఫేస్ ఏ విషయాన్ని చూపించాలో, వేరియబుల్స్‌లో మార్పుల ద్వారా నియంత్రించడం.**

## సారాంశం

ఈ పాఠంలో, మేము మూడు ముఖ్యమైన విషయాలు నేర్చుకున్నాము:

1. పాస్‌వర్డ్ విషయం ఇంటర్‌ఫేస్‌లో నేరుగా కనిపించకుండా ఉండేందుకు పాస్‌వర్డ్‌లను నమోదు చేయడానికి `SecureField` ఉపయోగించడం.
2. సందేశాలను కేవలం Console లో మాత్రమే ప్రదర్శించకుండా వినియోగదారునికి చూపించడానికి `Alert` ఉపయోగించడం.
3. లాగిన్ ముందు మరియు లాగిన్ తర్వాతి ఇంటర్‌ఫేస్‌ల మధ్య మార్పు చేయడం సాధ్యమయ్యేలా వేర్వేరు వీక్షణల ప్రదర్శనను నియంత్రించడానికి వేరియబుల్స్ మరియు `if` షరతులను ఉపయోగించడం.

ఈ జ్ఞానంతో, ఇప్పుడు మేము ప్రాథమిక పరస్పర ప్రభావాలతో కూడిన లాగిన్ వీక్షణను పూర్తి చేయగలుగుతున్నాము, మరియు SwiftUI లోని “స్థితి మారితే ఇంటర్‌ఫేస్ నవీకరించబడుతుంది” అనే అభివృద్ధి ఆలోచనను మరింత బాగా అర్థం చేసుకోగలుగుతున్నాము.

## పూర్తి కోడ్

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## విస్తృత జ్ఞానం - అనేక బటన్‌లతో Alert

ఈ పాఠంలోని మునుపటి భాగంలో, మేము ఒకే బటన్ కలిగిన, ముఖ్యంగా వినియోగదారునికి తెలియజేయడానికి ఉపయోగించే అత్యంత ప్రాథమిక `Alert` ను ఉపయోగించాము.

నిజానికి, `Alert` లో అనేక బటన్‌లు కూడా ఉండవచ్చు, తద్వారా వినియోగదారు వేర్వేరు ఎంపికలు చేసుకోవచ్చు.

ఉదాహరణకు:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

బటన్‌ను క్లిక్ చేసిన తర్వాత, `Ok`, `delete`, మరియు `cancel` అనే మూడు ఎంపికలతో ఒక సందేశ పెట్టె కనిపిస్తుంది.

ప్రదర్శన ప్రభావం:

![alert](../../Resource/020_alert2.png)

వేర్వేరు బటన్‌లను క్లిక్ చేసినప్పుడు, సంబంధిత కోడ్ అమలవుతుంది.

### Button యొక్క `role` లక్షణం

ఇక్కడ మీరు గమనించవచ్చు, `Alert` లోపల ఉన్న `Button`, మేము ముందుగా నేర్చుకున్న సాధారణ బటన్ రాతకన్నా కొద్దిగా భిన్నంగా ఉంటుంది:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

ఇక్కడ అదనంగా ఒక `role` పరామితి ఉంది.

SwiftUI లో, ఈ బటన్ ఏ రకమైన చర్యను సూచిస్తుందో సిస్టమ్‌కు తెలియజేయడానికి `role` ఉపయోగించబడుతుంది.

సాధారణంగా కనిపించే కొన్ని పాత్రలు ఇవి:

- `.cancel`: రద్దు చర్య
- `.destructive`: తొలగింపు వంటి ప్రమాదకర చర్య
- `nil`: సాధారణ చర్య

వేర్వేరు `role` విలువలు సాధారణంగా ఇంటర్‌ఫేస్‌లో వేర్వేరు దృశ్య శైలులకు కూడా దారితీస్తాయి.

ఉదాహరణకు, ప్రమాదకర చర్యలు వినియోగదారులు జాగ్రత్తగా నొక్కాలని గుర్తుచేయడానికి తరచుగా మరింత స్పష్టమైన శైలిలో చూపబడతాయి.
