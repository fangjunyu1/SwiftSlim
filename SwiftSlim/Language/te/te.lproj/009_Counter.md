# కౌంటర్

గత రెండు పాఠాల్లో, మనం వేరియబుల్స్, కాన్స్టెంట్స్, బటన్లు మరియు మెథడ్స్ గురించి నేర్చుకున్నాం.

ఈ పాఠంలో, ఈ జ్ఞానాన్ని కలిపి, ఒక సరళమైన కౌంటర్ యాప్‌ను తయారు చేస్తాము.

ఈ ఉదాహరణలో, వేరియబుల్‌ను ఎలా నిల్వ చేయాలి, ఎలా మార్చాలి, అలాగే వేరియబుల్‌ను నిర్వహించడానికి @State ను ఎలా ఉపయోగించాలి అనే విషయాలను నేర్చుకుంటాము.

## కౌంటర్‌ను అమలు చేయడం

కౌంటర్ సాధారణంగా వరుసగా పెరుగుతూ ఉండే సంఖ్యను నమోదు చేయడానికి ఉపయోగిస్తారు. ఉదాహరణకు, జంప్ రోప్ సంఖ్య, పరుగుల ల్యాప్‌లు వంటి లెక్కింపు సందర్భాలు.

ఇప్పుడు ఒక సరళమైన కౌంటర్‌ను తయారు చేద్దాం: ఒక సంఖ్యను చూపించి, బటన్‌ను నొక్కినప్పుడు ఆ సంఖ్య పెరగాలి.

![Num](../../RESOURCE/009_num.png)

### సంఖ్యను చూపించడం

మొదట, Text ను ఉపయోగించి సంఖ్యను చూపిస్తాము.

```swift
Text("0")
```

కానీ, సంఖ్య మారాల్సి ఉంటే, "0" ను నేరుగా రాయకూడదు; బదులుగా సంఖ్యను నిల్వ చేయడానికి ఒక వేరియబుల్‌ను ఉపయోగించాలి.

```swift
var num = 0
```

తర్వాత Text లో వేరియబుల్‌ను చూపించండి:

```swift
Text("\(num)")
```

ఇక్కడ స్ట్రింగ్ ఇంటర్‌పొలేషన్ \() ను ఉపయోగించి, సంఖ్యను స్ట్రింగ్‌గా మార్చి చూపిస్తున్నాం.

### బటన్ ద్వారా సంఖ్యను పెంచడం

తర్వాత, ఒక బటన్‌ను జోడిద్దాం.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

బటన్ నొక్కబడినప్పుడు, సంఖ్య పెరగాలని మనం కోరుకుంటున్నాం.

కాబట్టి బటన్‌లో వేరియబుల్‌ను మార్చవచ్చు:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

ఈ కోడ్ అంటే: బటన్‌ను నొక్కినప్పుడు, num + 1 ను లెక్కించి, ఫలితాన్ని num కు మళ్లీ కేటాయిస్తుంది.

### ఇంకా సంక్షిప్తమైన రీతిలో

Swift మరింత సంక్షిప్తమైన రీతిని అందిస్తుంది:

```swift
num += 1
```

ఇది క్రింది కోడ్‌కు సమానం:

```swift
num = num + 1
```

ఈ రీతిని “కాంపౌండ్ అసైన్‌మెంట్ ఆపరేటర్” అని అంటారు.

Swift లో సాధారణంగా ఉపయోగించే కాంపౌండ్ అసైన్‌మెంట్ ఆపరేటర్లు ఇవి:

```
+=   జోడింపు కేటాయింపు
-=   తీసివేత కేటాయింపు
*=   గుణకారం కేటాయింపు
/=   భాగాకారం కేటాయింపు
%=   మిగులు కేటాయింపు
```

ఉదాహరణకు:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### పూర్తి కోడ్

ఇప్పుడు, మనం ఒక సరళమైన కౌంటర్‌ను వ్రాయవచ్చు:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

కానీ, ఈ సమయంలో కోడ్ సరిగ్గా పని చేయదు.

### లోపం కనిపించడం

Xcode ఈ క్రింది లోపాన్ని చూపిస్తుంది:

```
Cannot assign to property: 'self' is immutable
```

అంటే, view మార్చలేనిది కనుక, దాని ప్రాపర్టీని మార్చలేమని అర్థం.

ఇలా ఎందుకు జరుగుతోంది?

## SwiftUI పనిచేసే విధానం

SwiftUI లో: **ఇంటర్‌ఫేస్ డేటా ద్వారా నడపబడుతుంది**, అంటే డేటా మారినప్పుడు, ఇంటర్‌ఫేస్ స్వయంచాలకంగా రీఫ్రెష్ అవుతుంది.

కానీ గమనించాల్సిన విషయం ఏమిటంటే, స్థితి (ఉదాహరణకు @State, @Binding) మారినప్పుడే SwiftUI ఆ మార్పును గమనించి view ను రీఫ్రెష్ చేస్తుంది.

సాధారణ వేరియబుల్‌ను ఉపయోగిస్తే, view సృష్టించిన సమయంలో SwiftUI ఆ విలువను ఒక్కసారి మాత్రమే చదవుతుంది. తర్వాత వేరియబుల్ మారినా, ఇంటర్‌ఫేస్ అప్డేట్ కాదు.

ఉదాహరణకు:

```swift
var num = 0   // 0
num = 1   // 0
```

పై ఉదాహరణలో, num ఒక సాధారణ వేరియబుల్. దాని విలువ మారినా, SwiftUI view ను అప్డేట్ చేయదు. అందువల్ల ఇంటర్‌ఫేస్ మొదటి విలువ అయిన 0 నే చూపిస్తుంది.

![Num](../../RESOURCE/009_state.png)

అందువల్ల, డేటాను స్థితిగా గుర్తించినప్పుడే (ఉదాహరణకు @State, @Binding వంటి property wrappers ద్వారా) SwiftUI దాని మార్పును స్వయంచాలకంగా గమనించి view ను అప్డేట్ చేస్తుంది.

## @State ప్రాపర్టీ ర్యాపర్

SwiftUI లో, వేరియబుల్‌ను మార్చి view ను అప్డేట్ చేయాలంటే తప్పనిసరిగా @State ను ఉపయోగించాలి.

```swift
@State var num = 0
```

@State వాడితే, SwiftUI ఈ వేరియబుల్‌ను గమనిస్తుంది.

వేరియబుల్ మారినప్పుడు, SwiftUI మళ్లీ view ను లెక్కించి అప్డేట్ చేస్తుంది.

పూర్తి కోడ్:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

ఇప్పుడు, బటన్‌ను నొక్కినప్పుడు:

```swift
num += 1
```

SwiftUI, num మారిందని గుర్తించి, view ను స్వయంచాలకంగా అప్డేట్ చేస్తుంది.

![Num](../../RESOURCE/009_state1.png)

## @State ఉపయోగించే నియమాలు

వ్యవహారిక అభివృద్ధిలో, @State సాధారణంగా రెండు నియమాలను అనుసరిస్తుంది.

### 1. @State ను సాధారణంగా private తో వాడుతారు

@State వేరియబుల్ సాధారణంగా ప్రస్తుత టైప్‌లోనే (ఉదాహరణకు ప్రస్తుత View struct లోనే) ఉపయోగించబడుతుంది. అందుకే సాధారణంగా ఇలా వ్రాస్తారు:

```swift
@State private var num = 0
```

ఇలా వ్రాయడం వల్ల ఇతర views ఈ వేరియబుల్‌ను నేరుగా యాక్సెస్ చేయడం లేదా మార్చడం నివారించబడుతుంది.

ఇప్పటికీ private గురించి మనం నేర్చుకోలేదు. సులభంగా అర్థం చేసుకోవాలంటే:

**private తో గుర్తించిన విషయం ప్రస్తుత view లో మాత్రమే ఉపయోగించవచ్చు; బయట ఉన్న views దాన్ని ఉపయోగించలేవు.**

### 2. @State ప్రస్తుత view యొక్క స్థితికే ఉపయోగించాలి

@State అనేది ప్రస్తుత View తానే నిర్వహించే డేటాకు సరిపోతుంది.

ఉదాహరణకు, TextField లోని విషయం, switch స్థితి.

```swift
@State private var isOn = false
@State private var text = ""
```

డేటా అనేక views మధ్య భాగస్వామ్యం కావాల్సి ఉంటే, @Binding లేదా @Observable వంటి ఇతర స్థితి రకాల్ని ఉపయోగించాలి.

ఈ విషయాలను తర్వాతి పాఠాల్లో నేర్చుకుంటాము.

### తగ్గించే బటన్

కౌంటర్‌లో పెంచడం మాత్రమే కాకుండా, తగ్గించే అవకాశం కూడా ఉండాలి.

దానికి ఒక - బటన్ జోడించవచ్చు.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

తర్వాత బటన్లను అడ్డంగా అమర్చేందుకు HStack ను ఉపయోగించండి:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

ఫలితం:

![Num](../../RESOURCE/009_num1.png)

* బటన్‌ను నొక్కితే సంఖ్య 1 పెరుగుతుంది; - బటన్‌ను నొక్కితే సంఖ్య 1 తగ్గుతుంది.

### రీసెట్ బటన్

మనం ఇంకొక రీసెట్ బటన్‌ను కూడా జోడించవచ్చు, ఇది సంఖ్యను 0 గా మార్చుతుంది.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

రీసెట్ బటన్‌ను నొక్కినప్పుడు, num కు 0 కేటాయించబడుతుంది.

ఫలితం:

![Num](../../RESOURCE/009_num2.png)

0 బటన్‌ను నొక్కినప్పుడు, సంఖ్య 0 గా మారుతుంది.

### కస్టమ్ బటన్

ప్రస్తుతం మన బటన్లు కేవలం టెక్స్ట్‌ను మాత్రమే చూపిస్తున్నాయి:

```swift
Button("+") {

}
```

ఈ రీతిలో వ్రాసిన బటన్ కేవలం + లేదా - వంటి టెక్స్ట్‌ను మాత్రమే చూపించగలదు.

వ్యవహారిక అభివృద్ధిలో, బటన్ రూపం మరింత సమృద్ధిగా ఉండాలని మనం కోరుకుంటాము. ఉదాహరణకు, ఐకాన్ లేదా ఇతర view ను ఉపయోగించడం.

SwiftUI లో బటన్ యొక్క చూపించబడే కంటెంట్‌ను కస్టమైజ్ చేయవచ్చు. అప్పుడు బటన్‌ను ఇలా వ్రాయాలి:

```swift
Button(action: {

}, label: {

})
```

ఈ రీతిలో, action అంటే బటన్‌ను నొక్కినప్పుడు అమలయ్యే కోడ్, label అంటే బటన్‌లో చూపించబడే view.

ఉదాహరణకు, బటన్‌లో ఒక SF Symbols ఐకాన్‌ను చూపించవచ్చు:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

ఈ విధంగా బటన్ ఇక టెక్స్ట్ కాకుండా ఒక ఐకాన్‌ను చూపిస్తుంది.

### కౌంటర్ ఇంటర్‌ఫేస్‌ను మెరుగుపరచడం

కౌంటర్ ఇంటర్‌ఫేస్ మరింత స్పష్టంగా కనిపించేందుకు, సంఖ్య యొక్క ప్రదర్శన శైలిని కూడా మార్చవచ్చు. ఉదాహరణకు, సంఖ్యను పెద్దగా చూపించవచ్చు:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

తర్వాత, బటన్లలో చూపే కంటెంట్‌ను కూడా ఐకాన్‌లుగా మార్చి, వాటి పరిమాణాన్ని కొద్దిగా పెంచుదాం:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

అదే విధంగా, తగ్గించే బటన్ మరియు రీసెట్ బటన్‌లను కూడా ఐకాన్ రూపంలో మార్చవచ్చు:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

ఇప్పుడు బటన్లు SF Symbols ఐకాన్‌లను చూపిస్తాయి. అందువల్ల ఇంటర్‌ఫేస్ మరింత స్పష్టంగా మరియు సహజంగా కనిపిస్తుంది.

## సారాంశం

ఈ పాఠంలో, ఒక సరళమైన కౌంటర్‌ను అమలు చేసి, వేరియబుల్స్, Text మరియు Button వినియోగాన్ని సమగ్రంగా అభ్యాసం చేశాము. సంఖ్యను నిల్వ చేయడానికి వేరియబుల్‌ను ఉపయోగించాము, అలాగే బటన్ల ద్వారా సంఖ్యను పెంచడం, తగ్గించడం చేశాము. ఇదే సమయంలో num += 1 మరియు num -= 1 వంటి కాంపౌండ్ అసైన్‌మెంట్ ఆపరేటర్లను కూడా నేర్చుకున్నాము.

అమలు చేసే ప్రక్రియలో, SwiftUI పని చేసే విధానాన్ని అర్థం చేసుకున్నాము: ఇంటర్‌ఫేస్ డేటా ఆధారంగా నడుస్తుంది. డేటా మారినప్పుడే ఇంటర్‌ఫేస్ రీఫ్రెష్ అవుతుంది. అందుకే మారే స్థితిని నిల్వ చేయడానికి @State ను ఉపయోగించాలి. @State వేరియబుల్ మారినప్పుడు, SwiftUI స్వయంగా view ను అప్డేట్ చేస్తుంది.

చివరగా, ఇంటర్‌ఫేస్‌పై చిన్న మెరుగులు చేశాము. Text యొక్క ఫాంట్ శైలిని మార్చాము, అలాగే Button(action:label:) ద్వారా బటన్ కంటెంట్‌ను కస్టమైజ్ చేసి, బటన్‌లో ఐకాన్‌లను చూపించాము. దీనివల్ల కౌంటర్ ఇంటర్‌ఫేస్ మరింత స్పష్టంగా మరియు అందంగా మారింది.

ఇప్పుడు, మనం SwiftUI views యొక్క ప్రాథమిక వినియోగాన్ని ఇప్పటికే తాకాము. SwiftUI లో, ఎక్కువ భాగం ఇంటర్‌ఫేస్ వేరియబుల్స్ ద్వారా నడపబడుతుంది. వేరియబుల్స్ మారినప్పుడు, SwiftUI స్వయంగా view ను అప్డేట్ చేస్తుంది. అందువల్ల SwiftUI యాప్‌లను అభివృద్ధి చేసే సమయంలో, సాధారణంగా ముందుగా ఏ డేటాను నిల్వ చేయాలో ఆలోచించి, తర్వాత ఆ డేటా ఆధారంగా ఇంటర్‌ఫేస్ మరియు ఇంటరాక్షన్ లాజిక్‌ను నిర్మిస్తాము.

## పూర్తి కోడ్

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
