# Xcode ఫైళ్ల నిర్వహణ

ఈ పాఠంలో, Xcode లో సాధారణంగా ఉపయోగించే ఫైల్ ఆపరేషన్లను నేర్చుకుంటాము.

ముందటి పాఠాల్లో, మేము ఎక్కువ భాగం కోడ్‌ను `ContentView` లోనే రాశాము. ఇలా చేయడం వల్ల కోడ్ మరియు చూపించే ఫలితాన్ని గమనించడం సులభం.

ఉదాహరణకు:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

కానీ నిజమైన App లో, సాధారణంగా అన్ని కోడ్‌ను ఒకే ఫైల్‌లో రాయము.

పేజీలు, భాగాలు మరియు ఫీచర్లు పెరుగుతున్నప్పుడు, విభిన్న వీక్షణలను వేర్వేరు SwiftUI ఫైళ్లుగా విభజించాలి.

ఉదాహరణకు:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

ఇలా చేస్తే కోడ్ నిర్మాణం మరింత స్పష్టంగా ఉంటుంది, తర్వాత నిర్వహణ మరియు మార్పులు చేయడం కూడా సులభం అవుతుంది.

## ఫైళ్లను ఎందుకు నిర్వహించాలి

SwiftUI లో, ఒక పేజీ లేదా ఒక భాగాన్ని సాధారణంగా ప్రత్యేక View ఫైల్‌గా రాయవచ్చు.

ఉదాహరణకు:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ఇక్కడ `CardView` ఒక SwiftUI వీక్షణ.

అన్ని వీక్షణలను `ContentView` లోనే రాస్తే, కోడ్ క్రమంగా పొడవుగా మారుతుంది, చదవడం కూడా కష్టమవుతుంది.

కాబట్టి, ఫైళ్లను సృష్టించడం, పేరు మార్చడం, తొలగించడం, అలాగే వేర్వేరు కోడ్ ఫైళ్లను నిర్వహించడానికి ఫోల్డర్లను ఉపయోగించడం నేర్చుకోవాలి.

## SwiftUI ఫైల్ సృష్టించడం

Xcode లో SwiftUI ఫైల్ సృష్టించడానికి సాధారణంగా రెండు విధానాలు ఉన్నాయి:

ఒకటి ఖాళీ ఫైల్ సృష్టించి, తర్వాత మనమే కోడ్ రాయడం.

మరొకటి SwiftUI టెంప్లేట్ ఫైల్ సృష్టించడం, దీని ద్వారా Xcode ప్రాథమిక కోడ్‌ను స్వయంచాలకంగా రూపొందిస్తుంది.

ప్రారంభికులకు SwiftUI టెంప్లేట్ ఫైల్ ఉపయోగించడం మరింత మంచిది, ఎందుకంటే అది ప్రాథమిక నిర్మాణాన్ని స్వయంచాలకంగా రూపొందిస్తుంది.

### ఖాళీ ఫైల్ సృష్టించడం

Xcode ఎడమ వైపున ఉన్న Navigator ప్రాంతంలో, ఖాళీ ప్రదేశం లేదా ఫోల్డర్‌పై రైట్ క్లిక్ చేసి, `New Empty File` ఎంచుకోండి.

ఆపరేషన్ స్థానం ఇలా ఉంటుంది:

![xcode](../../../Resource/027_view1.png)

ఖాళీ ఫైల్ సృష్టించిన తర్వాత, Xcode సాధారణంగా `Untitled.swift` వంటి పేరు లేని Swift ఫైల్‌ను రూపొందిస్తుంది.

దాన్ని `TestView.swift` గా పేరు మార్చవచ్చు.

SwiftUI ప్రాజెక్ట్‌లో, ఈ ఫైల్ ఒక View చూపించడానికి ఉపయోగిస్తే, పేరుకి చివర `View` జోడించడం సాధారణంగా మంచిది.

ఉదాహరణకు:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

ఇది తప్పనిసరి నియమం కాదు, కానీ ఇది ఒక View ఫైల్ అని ఒక్క చూపులోనే తెలుసుకోవడానికి సహాయపడుతుంది.

ఖాళీ ఫైల్ సృష్టించిన తర్వాత, ఆ ఫైల్‌లో డిఫాల్ట్ వ్యాఖ్యలు మాత్రమే ఉండవచ్చు లేదా పూర్తిగా ఖాళీగా ఉండవచ్చు. ఇది Xcode వెర్షన్ మరియు సృష్టించే విధానంపై ఆధారపడి ఉంటుంది.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ఈ సమయంలో, SwiftUI కోడ్‌ను మనమే చేతితో రాయాలి:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

ఇలా ఒక సరళమైన SwiftUI View ఫైల్ సృష్టించబడుతుంది.

### ఫైల్ పేరు మార్చడం

ఫైల్ సృష్టించిన తర్వాత ఫైల్ పేరు మార్చాలనుకుంటే, Xcode ఎడమ వైపున ఉన్న Navigator ప్రాంతంలో ఫైల్‌ను ఎంచుకుని, తర్వాత రిటర్న్ కీ `⏎` నొక్కండి.

![xcode](../../../Resource/027_view2.png)

ఇలా ఫైల్ పేరును మార్చవచ్చు.

గమనించాల్సిందేమిటంటే, ఫైల్ పేరు మార్చడం వల్ల కోడ్‌లోని View పేరు స్వయంచాలకంగా మారదు.

ఉదాహరణకు, ఫైల్ పేరు `ProfileView.swift` గా మార్చిన తర్వాత కూడా, కోడ్‌లో `struct TestView: View` అలాగే ఉండవచ్చు.

ఇది తప్పనిసరిగా లోపానికి దారితీయకపోయినా, సులభంగా గందరగోళం కలిగిస్తుంది.

కాబట్టి, ఫైల్ పేరు మరియు View పేరు ఒకే విధంగా ఉండేలా ఉంచడం మంచిది. ఉదాహరణకు, ఫైల్ పేరు `ProfileView.swift` అయితే, View పేరుగా కూడా `ProfileView` ఉపయోగించాలి.

## SwiftUI టెంప్లేట్ ఫైల్ సృష్టించడం

ఖాళీ ఫైల్ సృష్టించడం కాకుండా, SwiftUI టెంప్లేట్ ఫైల్‌ను కూడా నేరుగా సృష్టించవచ్చు.

Xcode ఎడమ వైపున ఉన్న Navigator ప్రాంతంలో, ఖాళీ ప్రదేశం లేదా ఫోల్డర్‌పై రైట్ క్లిక్ చేసి, `New File from Template...` ఎంచుకోండి.

ఆపరేషన్ స్థానం ఇలా ఉంటుంది:

![xcode](../../../Resource/027_view3.png)

తెరుచుకునే విండోలో, పైభాగంలో `iOS`, `macOS` వంటి వేర్వేరు ప్లాట్‌ఫారమ్‌లను ఎంచుకోవచ్చు.

ఇక్కడ ప్రస్తుత ప్రాజెక్ట్‌కు సరిపోయే ప్లాట్‌ఫారమ్‌ను అలాగే ఉంచండి. ఉదాహరణకు, మనం iOS App సృష్టిస్తున్నట్లయితే, `iOS` ఎంచుకోండి.

![xcode](../../../Resource/027_view4.png)

తర్వాత కింద ఉన్న `SwiftUI View` ఎంచుకోండి.

![xcode](../../../Resource/027_view12.png)

తర్వాత `Next` క్లిక్ చేసి, ఫైల్ పేరు నమోదు చేసి సేవ్ చేయండి.

ఉదాహరణకు, మనం `Test2View.swift` అనే ఫైల్ సృష్టిద్దాం.

Xcode కింది విధమైన కోడ్‌ను స్వయంచాలకంగా రూపొందిస్తుంది:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

ఖాళీ ఫైల్‌తో పోలిస్తే, టెంప్లేట్ ఫైల్‌లో `import SwiftUI`, `struct`, `body` మరియు `#Preview` స్వయంచాలకంగా ఉంటాయి.

అందువల్ల ప్రారంభికుల కోసం, SwiftUI టెంప్లేట్ ఫైల్ ఉపయోగించడం సులభం, అలాగే కోడ్ మిస్ అయ్యే అవకాశం కూడా తక్కువ.

**సూచన**: రైట్ క్లిక్ చేసి ఫైల్ సృష్టించడం కాకుండా, Xcode ఎడమ వైపు Navigator దిగువన ఉన్న ప్లస్ బటన్‌ను క్లిక్ చేసి కూడా ఫైల్ లేదా ఫోల్డర్ సృష్టించవచ్చు.

![xcode](../../../Resource/027_view5.png)

## ఫోల్డర్ సృష్టించడం

ఫైళ్లు ఎక్కువయ్యే సమయంలో, వాటిని నిర్వహించడానికి ఫోల్డర్లను ఉపయోగించవచ్చు.

Xcode ఎడమ వైపున ఉన్న Navigator ప్రాంతంలో, ఖాళీ ప్రదేశం లేదా ఫోల్డర్‌పై రైట్ క్లిక్ చేసి, `New Folder` ఎంచుకోండి.

ఫోల్డర్ సృష్టించిన తర్వాత, సంబంధిత ఫైళ్లను ఆ ఫోల్డర్‌లోకి లాగవచ్చు.

![xcode](../../../Resource/027_view7.png)

ఫోల్డర్లు ప్రధానంగా ప్రాజెక్ట్ నిర్మాణాన్ని క్రమబద్ధీకరించడానికి ఉపయోగిస్తారు. అవి View స్వయంగా ఉపయోగించే విధానాన్ని మార్చవు.

View పేరు సరిగా ఉన్నంత వరకు, ఇతర ఫైళ్లలో కూడా `TestView()`、`Test2View()` వంటి విధంగా ఉపయోగించవచ్చు.

## ఫైల్ లేదా ఫోల్డర్ తొలగించడం

ఫైల్ లేదా ఫోల్డర్ తొలగించాలనుకుంటే, Xcode ఎడమ వైపున ఉన్న Navigator ప్రాంతంలో సంబంధిత ఫైల్‌ను ఎంచుకుని, రైట్ క్లిక్ చేసి `Delete` ఎంచుకోండి.

లేదా ఫైల్‌ను ఎంచుకున్న తర్వాత, డిలీట్ కీ `⌫` నొక్కవచ్చు.

Xcode తొలగింపు నిర్ధారణ విండోను చూపిస్తుంది.

![xcode](../../../Resource/027_view10.png)

`Move to Trash` ఎంచుకోవడం అంటే ఫైల్‌ను Trash లోకి తరలించడం.

`Remove Reference` కనిపిస్తే, అది Xcode ప్రాజెక్ట్ నుండి రిఫరెన్స్‌ను మాత్రమే తొలగిస్తుంది, కానీ డిస్క్‌లోని అసలు ఫైల్‌ను తొలగించదు.

ప్రారంభికుల కోసం, మీరు కేవలం ఇప్పుడే సృష్టించిన సాధారణ ఫైల్‌ను తొలగిస్తున్నట్లయితే, సాధారణంగా `Move to Trash` ఎంచుకోవడం సరిపోతుంది.

## అనేక SwiftUI వీక్షణలను చూపించడం

ఫైళ్లను సృష్టించడం నేర్చుకున్న తర్వాత, విభిన్న వీక్షణలను వేర్వేరు ఫైళ్లుగా విభజించవచ్చు.

ఉదాహరణకు, మనం రెండు SwiftUI ఫైళ్లను సృష్టిద్దాం:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` రచయిత పేరును చూపించడానికి ఉపయోగిస్తారు:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` కార్డ్ నేపథ్యాన్ని చూపించడానికి ఉపయోగిస్తారు:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

తర్వాత, ఈ రెండు వీక్షణలను `ContentView` లో ఉపయోగించవచ్చు:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

రన్ చేసిన తర్వాత, `CardView` మరియు `AuthorView` కలిపి చూపబడినట్లు చూడవచ్చు:

![xcode](../../../Resource/027_view11.png)

ఈ ఫలితంలో, `CardView` నేపథ్యంగా ఉంటుంది, `AuthorView` దాని పైభాగంలో కనిపిస్తుంది. కారణం అవి `ZStack` లో ఉంచబడ్డాయి.

`ContentView` లో, `CardView()` మరియు `AuthorView()` ద్వారా ఈ రెండు వీక్షణలను ఉపయోగిస్తున్నాం. పేరుకి తర్వాత ఉన్న `()` ఆ View ను సృష్టించి ఉపయోగిస్తున్నామని సూచిస్తుంది.

ఇదే SwiftUI లో సాధారణంగా కనిపించే View nesting: ఒక View లో, మరిన్ని ఇతర Views ను ఉపయోగించవచ్చు.

## ప్రవేశ ఫైల్

చివరిగా, App ఇంటర్‌ఫేస్ ఎక్కడి నుంచి ప్రారంభమవుతుందో మళ్లీ చూద్దాం.

SwiftUI ప్రాజెక్ట్‌లో సాధారణంగా ఒక ప్రవేశ ఫైల్ ఉంటుంది:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ఇక్కడ `@main` ఇది App యొక్క ప్రవేశ స్థానం అని సూచిస్తుంది.

`WindowGroup` లో మనం ఇది చూస్తాము:

```swift
ContentView()
```

దీని అర్థం App ప్రారంభమైన తర్వాత ముందుగా `ContentView` చూపబడుతుంది.

తర్వాత `ContentView` లో మళ్లీ ఇది చూపబడుతుంది:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

కాబట్టి View స్థాయులను ఇలా అర్థం చేసుకోవచ్చు:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

ప్రవేశ ఫైల్‌లో చూపించే View ను `CardView` గా మార్చితే:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

అప్పుడు App ప్రారంభమైన తర్వాత `CardView` మాత్రమే చూపబడుతుంది.

అంటే, ప్రవేశ కోడ్‌లోని `WindowGroup` App ప్రారంభమైన తర్వాత ముందుగా ఏ View చూపించాలో నిర్ణయిస్తుంది.

ఆ View లోపల, మళ్లీ ఇతర Views ను కొనసాగించి చూపించవచ్చు.

## సారాంశం

ఈ పాఠంలో, Xcode లో సాధారణంగా ఉపయోగించే ఫైల్ నిర్వహణ ఆపరేషన్లను నేర్చుకున్నాము.

SwiftUI ఫైళ్లను ఎలా సృష్టించాలి, ఫైళ్ల పేర్లు ఎలా మార్చాలి, ఫోల్డర్లు ఎలా సృష్టించాలి, అలాగే ఫైళ్లను ఎలా తొలగించాలి అనే విషయాలను తెలుసుకున్నాము.

అదే సమయంలో, ఒక ముఖ్యమైన భావనను కూడా నేర్చుకున్నాము: SwiftUI Views ను వేర్వేరు ఫైళ్లుగా విభజించి, తర్వాత ఇతర Views లో ఉపయోగించవచ్చు.

ఉదాహరణకు:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ఇక్కడ `CardView()` మరియు `AuthorView()` అంటే `ContentView` లో ఇతర Views ను ఉపయోగిస్తున్నామన్న అర్థం.

చివరగా, ప్రవేశ ఫైల్ యొక్క ప్రదర్శన లాజిక్‌ను కూడా మళ్లీ చూశాము.

App ప్రారంభమైనప్పుడు, ముందుగా `@main` గుర్తుతో ఉన్న ప్రవేశ కోడ్‌లోకి ప్రవేశిస్తుంది, తర్వాత `WindowGroup` లోని View ను చూపిస్తుంది.

ఇది మనకు అర్థం చేసుకోవడంలో సహాయపడుతుంది: వేర్వేరు SwiftUI ఫైళ్లు ఒంటరిగా ఉండవు. అవి పరస్పరం కలిపి, చివరికి పూర్తి App ఇంటర్‌ఫేస్‌ను రూపొందించవచ్చు.
