# ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ

ਇਸ ਪਾਠ ਵਿੱਚ ਅਸੀਂ ਮੁੱਖ ਤੌਰ 'ਤੇ Swift ਵਿੱਚ ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਸਿੱਖਾਂਗੇ, ਅਤੇ ਕੁਝ ਆਮ data types ਅਤੇ ਬੁਨਿਆਦੀ operators ਨਾਲ ਜਾਣ-ਪਛਾਣ ਕਰਾਂਗੇ।

ਇਹ ਸਮੱਗਰੀ programming ਦੀ ਸਭ ਤੋਂ ਬੁਨਿਆਦੀ ਜਾਣਕਾਰੀ ਹੈ, ਅਤੇ ਅੱਗੇ SwiftUI ਸਿੱਖਣ ਲਈ ਵੀ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਨੀਂਹ ਹੈ।

## ਰੋਜ਼ਾਨਾ ਜੀਵਨ ਵਿੱਚ ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ

ਅਸੀਂ ਰੋਜ਼ਮਰਾ ਦੀਆਂ ਚੀਜ਼ਾਂ ਰਾਹੀਂ ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਨੂੰ ਸਮਝ ਸਕਦੇ ਹਾਂ।

ਰੋਜ਼ਾਨਾ ਜੀਵਨ ਵਿੱਚ ਵੇਰੀਏਬਲ:

- ਟੀਵੀ ਵਿੱਚ ਹਰ ਪ੍ਰੋਗਰਾਮ ਦੀ ਸਮੱਗਰੀ ਵੱਖਰੀ ਹੁੰਦੀ ਹੈ
- ਹਰ ਦਿਨ ਮੌਸਮ ਇੱਕੋ ਜਿਹਾ ਨਹੀਂ ਹੁੰਦਾ
- ਘੜੀ ਹਰ ਸਕਿੰਟ ਬਦਲਦੀ ਰਹਿੰਦੀ ਹੈ

ਇਨ੍ਹਾਂ ਚੀਜ਼ਾਂ ਦੀ ਸਾਂਝੀ ਵਿਸ਼ੇਸ਼ਤਾ ਇਹ ਹੈ ਕਿ: ਇਹ ਬਦਲਦੀਆਂ ਰਹਿੰਦੀਆਂ ਹਨ।

ਜੇ ਟੀਵੀ 'ਤੇ ਹਮੇਸ਼ਾਂ ਇੱਕੋ ਹੀ ਤਸਵੀਰ ਹੋਵੇ, ਮੌਸਮ ਹਮੇਸ਼ਾਂ ਧੁੱਪ ਵਾਲਾ ਹੀ ਰਹੇ, ਅਤੇ ਘੜੀ ਦੀਆਂ ਸੂਈਆਂ ਕਦੇ ਨਾ ਹਿਲਣ, ਤਾਂ ਇਹ ਚੀਜ਼ਾਂ ਕਾਂਸਟੈਂਟ ਹੋਣਗੀਆਂ।

ਇੱਕ ਚੀਜ਼ ਬਦਲ ਸਕਦੀ ਹੈ, ਦੂਜੀ ਨਹੀਂ ਬਦਲਦੀ।

## ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਨੂੰ ਸਮਝਣਾ

App development ਵਿੱਚ, ਆਮ ਤੌਰ 'ਤੇ ਯੂਜ਼ਰ ਨੂੰ ਕੁਝ ਜਾਣਕਾਰੀ ਭਰਨੀ ਜਾਂ ਸੰਭਾਲਣੀ ਪੈਂਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

- ਖਾਤੇ ਦਾ ਨਾਮ
- ਜਨਮਦਿਨ
- ਸੰਪਰਕ ਜਾਣਕਾਰੀ
- ਪਤਾ

ਇਹ ਜਾਣਕਾਰੀਆਂ save ਕੀਤੀਆਂ ਜਾਂਦੀਆਂ ਹਨ ਅਤੇ ਫਿਰ ਦਿਖਾਈਆਂ ਜਾਂਦੀਆਂ ਹਨ।

ਉਦਾਹਰਨ ਵਜੋਂ, ਜਦੋਂ ਯੂਜ਼ਰ App ਵਿੱਚ ਇੱਕ ਨਾਮ ਦਰਜ ਕਰਦਾ ਹੈ:

```text
FangJunyu
````

ਸਾਨੂੰ ਇਹ ਨਾਮ save ਕਰਨਾ ਪੈਂਦਾ ਹੈ, ਤਾਂ ਜੋ ਇਸਨੂੰ App ਵਿੱਚ ਦਿਖਾਇਆ ਜਾ ਸਕੇ।

ਅਸੀਂ save ਕਰਨ ਦੀ ਪ੍ਰਕਿਰਿਆ ਨੂੰ ਆਸਾਨ ਤਰੀਕੇ ਨਾਲ ਇਸ ਤਰ੍ਹਾਂ ਸਮਝ ਸਕਦੇ ਹਾਂ: ਜਿਵੇਂ ਕਿਸੇ ਦਰਾਜ਼ ਵਿੱਚ ਚੀਜ਼ ਰੱਖਣਾ।

ਜਦੋਂ ਅਸੀਂ ਨਾਮ save ਕਰਦੇ ਹਾਂ, ਤਾਂ ਇਹ ਇਸ ਤਰ੍ਹਾਂ ਹੈ ਜਿਵੇਂ ਨਾਮ ਨੂੰ ਇੱਕ ਦਰਾਜ਼ ਵਿੱਚ ਰੱਖ ਦਿੱਤਾ ਹੋਵੇ।

ਜਿਹੜੀਆਂ ਚੀਜ਼ਾਂ save ਕਰਨੀ ਹੁੰਦੀਆਂ ਹਨ, ਉਹ ਕਾਫ਼ੀ ਹੋ ਸਕਦੀਆਂ ਹਨ, ਅਤੇ ਦਰਾਜ਼ਾਂ ਵੀ ਕਈ ਹੋ ਸਕਦੀਆਂ ਹਨ। ਇਹ ਜਾਣਨ ਲਈ ਕਿ ਕਿਸ ਦਰਾਜ਼ ਵਿੱਚ ਕੀ ਪਿਆ ਹੈ, ਸਾਨੂੰ ਦਰਾਜ਼ ਦਾ ਇੱਕ ਨਾਮ ਰੱਖਣਾ ਪੈਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```text
name
```

ਇਸ ਉਦਾਹਰਨ ਵਿੱਚ, `name` ਨਾਮ ਹੈ, ਅਤੇ `FangJunyu` save ਕੀਤੀ ਗਈ ਜਾਣਕਾਰੀ ਹੈ।

![Var](../../RESOURCE/007_var.png)

**Swift ਵਿੱਚ, ਜੇ ਡਾਟਾ save ਕਰਨਾ ਹੋਵੇ, ਤਾਂ ਉਸਨੂੰ ਵੇਰੀਏਬਲ (`var`) ਜਾਂ ਕਾਂਸਟੈਂਟ (`let`) ਰਾਹੀਂ declare ਕਰਨਾ ਲਾਜ਼ਮੀ ਹੈ।**

ਕਿਉਂਕਿ ਨਾਮ ਆਮ ਤੌਰ 'ਤੇ ਬਦਲਿਆ ਜਾ ਸਕਦਾ ਹੈ, ਇਸ ਲਈ ਅਸੀਂ ਵੇਰੀਏਬਲ ਵਰਤਣਾ ਚਾਹੀਦਾ ਹੈ।

```swift
var name = "FangJunyu"
```

ਇੱਥੇ `name` ਨਾਮ ਦਾ ਇੱਕ ਵੇਰੀਏਬਲ declare ਕੀਤਾ ਗਿਆ ਹੈ। ਇਸ ਦੀ type `String` ਹੈ, ਅਤੇ ਇਸ ਦੀ value `"FangJunyu"` ਹੈ।

### ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਵਿੱਚ ਫਰਕ

ਵੇਰੀਏਬਲ ਨੂੰ `var` ਨਾਲ declare ਕੀਤਾ ਜਾਂਦਾ ਹੈ:

```swift
var
```

ਕਾਂਸਟੈਂਟ ਨੂੰ `let` ਨਾਲ declare ਕੀਤਾ ਜਾਂਦਾ ਹੈ:

```swift
let
```

ਉਦਾਹਰਨ ਲਈ:

```swift
var name = "FangJunyu"
let id = 123456
```

ਜੇ ਕਿਸੇ value ਨੂੰ ਵੇਰੀਏਬਲ ਵਜੋਂ declare ਕੀਤਾ ਗਿਆ ਹੈ, ਤਾਂ ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ value ਬਦਲ ਸਕਦੀ ਹੈ; ਜੇ ਇਸਨੂੰ ਕਾਂਸਟੈਂਟ ਵਜੋਂ declare ਕੀਤਾ ਗਿਆ ਹੈ, ਤਾਂ ਇਸਨੂੰ ਦੁਬਾਰਾ ਬਦਲਾ ਨਹੀਂ ਜਾ ਸਕਦਾ।

ਇਸ ਲਈ, ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਵਿਚਲਾ ਮੁੱਖ ਫਰਕ ਇਹ ਹੈ: value ਨੂੰ ਬਦਲਣ ਦੀ ਇਜਾਜ਼ਤ ਹੈ ਜਾਂ ਨਹੀਂ।

### ਕਾਂਸਟੈਂਟ ਨੂੰ ਬਦਲਣਾ

ਜੇ ਕੋਡ ਕਾਂਸਟੈਂਟ ਦੀ value ਨੂੰ ਬਦਲਣ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰਦਾ ਹੈ:

```swift
let name = "Sam"
name = "Bob"
```

Swift ਇਹ ਗਲਤੀ ਦਿਖਾਏਗਾ ਕਿ assignment ਨਹੀਂ ਹੋ ਸਕਦਾ, ਕਿਉਂਕਿ ਇਹ ਇੱਕ constant ਹੈ।

```text
Cannot assign to property: 'name' is a 'let' constant
```

ਇਹ ਮਕੈਨਿਜ਼ਮ developers ਦੀ ਮਦਦ ਕਰਦਾ ਹੈ ਤਾਂ ਜੋ ਕੁਝ ਮਹੱਤਵਪੂਰਨ ਡਾਟਾ ਗਲਤੀ ਨਾਲ ਨਾ ਬਦਲ ਜਾਵੇ।

### SwiftUI ਵਿੱਚ ਦਿਖਾਉਣਾ

`ContentView.swift` ਫਾਈਲ ਖੋਲ੍ਹੋ, ਅਤੇ View ਦੇ ਅੰਦਰ ਵੇਰੀਏਬਲ declare ਕਰੋ:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

ਇੱਥੇ `name` ਨਾਮ ਦਾ ਇੱਕ ਵੇਰੀਏਬਲ declare ਕੀਤਾ ਗਿਆ ਹੈ, ਅਤੇ `Text` ਰਾਹੀਂ ਇਸਨੂੰ interface 'ਤੇ ਦਿਖਾਇਆ ਗਿਆ ਹੈ।

ਜੇ ਅਸੀਂ ਵੇਰੀਏਬਲ ਦੀ value ਬਦਲ ਦੇਈਏ:

```swift
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

ਤਾਂ `Text` ਵਿੱਚ ਦਿਖਾਈ ਦੇਣ ਵਾਲੀ ਲਿਖਤ ਵੀ ਬਦਲ ਜਾਵੇਗੀ।

ਇਸ ਤਰੀਕੇ ਨਾਲ, ਅਸੀਂ interface ਵਿੱਚ ਦਿਖਾਈ ਜਾਣ ਵਾਲੀ ਸਮੱਗਰੀ ਨੂੰ ਵੇਰੀਏਬਲ ਰਾਹੀਂ ਨਿਯੰਤਰਿਤ ਕਰ ਸਕਦੇ ਹਾਂ, ਅਤੇ ਹਰ ਵਾਰ `Text` ਦੇ ਅੰਦਰ ਦੀ ਲਿਖਤ ਨੂੰ ਸਿੱਧੇ ਤੌਰ 'ਤੇ ਬਦਲਣ ਦੀ ਲੋੜ ਨਹੀਂ ਪੈਂਦੀ।

ਜੇ ਹੋਰ ਕਿਸੇ ਜਗ੍ਹਾ ਨੂੰ ਵੀ ਇਹ ਜਾਣਕਾਰੀ ਚਾਹੀਦੀ ਹੋਵੇ, ਤਾਂ ਅਸੀਂ ਇਹ ਵੇਰੀਏਬਲ ਉੱਥੇ pass ਕਰ ਸਕਦੇ ਹਾਂ। ਇਸਨੂੰ ਤੁਸੀਂ ਇਹ ਵੀ ਸਮਝ ਸਕਦੇ ਹੋ ਕਿ “ਦਰਾਜ਼” ਦੇ ਅੰਦਰ ਵਾਲੀ ਚੀਜ਼ ਕਿਸੇ ਹੋਰ ਜਗ੍ਹਾ ਵਰਤੋਂ ਲਈ ਦੇ ਦਿੱਤੀ ਜਾ ਰਹੀ ਹੈ।

### SwiftUI ਵਿੱਚ ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਦੀ ਪੋਜ਼ੀਸ਼ਨ

SwiftUI ਵਿੱਚ, ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਆਮ ਤੌਰ 'ਤੇ `body` ਤੋਂ ਬਾਹਰ ਲਿਖੇ ਜਾਂਦੇ ਹਨ:

```swift
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

ਜ਼ਾਹਿਰ ਹੈ, ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ `body` ਦੇ ਅੰਦਰ ਵੀ ਲਿਖੇ ਜਾ ਸਕਦੇ ਹਨ:

```swift
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

ਪਰ ਦੋਨਾਂ ਵਿੱਚ ਇੱਕ ਮਹੱਤਵਪੂਰਨ ਫਰਕ ਹੈ:

* ਜੇ ਵੇਰੀਏਬਲ ਜਾਂ ਕਾਂਸਟੈਂਟ `body` ਦੇ ਅੰਦਰ ਲਿਖੇ ਜਾਣ, ਤਾਂ ਹਰ ਵਾਰ view ਮੁੜ calculate ਹੋਣ 'ਤੇ ਇਹ ਵੇਰੀਏਬਲ ਮੁੜ ਬਣਦਾ ਹੈ।
* ਜੇ ਇਹ `body` ਤੋਂ ਬਾਹਰ ਲਿਖੇ ਜਾਣ, ਤਾਂ ਇਹ view struct ਦੀ property ਵਜੋਂ ਮੌਜੂਦ ਰਹਿੰਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਕੋਡ ਦੀ ਬਣਤਰ ਹੋਰ ਸਾਫ਼ ਹੁੰਦੀ ਹੈ।

ਇਸ ਲਈ, ਅਸਲ development ਵਿੱਚ, ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਆਮ ਤੌਰ 'ਤੇ `body` ਤੋਂ ਬਾਹਰ ਲਿਖੇ ਜਾਂਦੇ ਹਨ।

## ਡਾਟਾ ਟਾਈਪ

ਵੇਰੀਏਬਲ ਕਈ ਵੱਖ-ਵੱਖ ਕਿਸਮਾਂ ਦਾ ਡਾਟਾ save ਕਰ ਸਕਦੇ ਹਨ। ਇਨ੍ਹਾਂ ਕਿਸਮਾਂ ਨੂੰ data types ਕਿਹਾ ਜਾਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਅਸੀਂ ਪਹਿਲਾਂ `Text` ਰਾਹੀਂ ਲਿਖਤ ਦਿਖਾਈ ਸੀ:

```swift
Text("Hello, World")
```

ਇੱਥੇ `"Hello, World"` string (`String`) type ਦੀ value ਹੈ।

String ਟੈਕਸਟ ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ। Swift ਵਿੱਚ string ਨੂੰ double quotes `""` ਵਿੱਚ ਲਿਖਿਆ ਜਾਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
var hello = "Hello, World"
```

String ਤੋਂ ਇਲਾਵਾ ਵੀ Swift ਵਿੱਚ ਕਈ ਹੋਰ data types ਹਨ।

ਸ਼ੁਰੂਆਤੀ ਪੜਾਅ ਵਿੱਚ ਸਭ ਤੋਂ ਆਮ ਚਾਰ types ਇਹ ਹਨ:

* String
* Int
* Double
* Bool

ਇਹ ਚਾਰ types ਬਹੁਤ ਸਾਰੇ ਬੁਨਿਆਦੀ data processing ਕੰਮ ਪੂਰੇ ਕਰ ਸਕਦੇ ਹਨ।

**String**

`String` ਟੈਕਸਟ ਸਮੱਗਰੀ ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ। ਉਦਾਹਰਨ ਲਈ:

```swift
var name = "FangJunyu"
var city = "Rizhao"
```

String ਆਮ ਤੌਰ 'ਤੇ ਵਰਤੀ ਜਾਂਦੀ ਹੈ: user name, ਸਿਰਲੇਖ, ਅਤੇ ਲਿਖਤ ਸਮੱਗਰੀ ਦਿਖਾਉਣ ਲਈ।

**Int**

`Int` integer ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ, ਅਰਥਾਤ ਬਿਨਾਂ ਦਸ਼ਮਲਵ ਵਾਲੀ ਸੰਖਿਆ।

ਉਦਾਹਰਨ ਲਈ:

```swift
var age = 26
var count = 100
```

Integer ਆਮ ਤੌਰ 'ਤੇ ਵਰਤੀ ਜਾਂਦੀ ਹੈ: ਉਮਰ, ਗਿਣਤੀ, ਅਤੇ count ਵਰਗੀਆਂ values ਲਈ।

**Double**

`Double` ਦਸ਼ਮਲਵ ਵਾਲੀਆਂ ਸੰਖਿਆਵਾਂ ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
var weight = 74.5
var height = 185.0
```

Swift ਵਿੱਚ decimal numbers ਆਮ ਤੌਰ 'ਤੇ default ਤੌਰ 'ਤੇ `Double` type ਹੁੰਦੇ ਹਨ।

ਇੱਕ ਇਸ ਵਰਗੀ ਹੋਰ type `Float` ਵੀ ਹੈ, ਪਰ ਇਹ ਘੱਟ range ਵਾਲੀਆਂ values ਦਿਖਾ ਸਕਦੀ ਹੈ, ਇਸ ਲਈ ਅਸਲ development ਵਿੱਚ `Double` ਹੋਰ ਵੱਧ ਵਰਤੀ ਜਾਂਦੀ ਹੈ।

**Bool**

`Bool` type ਦੋ ਹਾਲਤਾਂ ਦਰਸਾਉਂਦੀ ਹੈ: `true` ਅਤੇ `false`।

ਉਦਾਹਰਨ ਲਈ:

```swift
var isShowAlert = true
var isLogin = false
```

`Bool` ਆਮ ਤੌਰ 'ਤੇ conditional judgment ਲਈ ਵਰਤੀ ਜਾਂਦੀ ਹੈ। ਉਦਾਹਰਨ ਵਜੋਂ, alert ਦਿਖਾਉਣਾ ਹੈ ਜਾਂ ਨਹੀਂ।

ਜਿੱਥੇ ਸਿਰਫ਼ ਦੋ ਹੀ ਸੰਭਾਵੀ ਨਤੀਜੇ ਹੁੰਦੇ ਹਨ, ਉੱਥੇ `Bool` type ਬਹੁਤ ਉਚਿਤ ਹੈ।

### SwiftUI ਵਿੱਚ ਦਿਖਾਉਣਾ

ਜਦੋਂ ਅਸੀਂ `String` type ਦੀ value ਦਿਖਾਉਂਦੇ ਹਾਂ, ਤਾਂ ਸਿੱਧਾ `Text` ਵਰਤ ਸਕਦੇ ਹਾਂ:

```swift
Text(name)
```

ਪਰ `Int`, `Double` ਆਦਿ strings ਨਹੀਂ ਹੁੰਦੇ, ਇਸ ਲਈ ਇਹਨਾਂ ਨੂੰ ਸਿੱਧੇ text ਦੇ ਹਿੱਸੇ ਵਜੋਂ ਨਹੀਂ ਦਿਖਾਇਆ ਜਾ ਸਕਦਾ।

ਜੇ ਇਹ ਡਾਟਾ `Text` ਵਿੱਚ ਦਿਖਾਉਣਾ ਹੋਵੇ, ਤਾਂ string interpolation ਵਰਤ ਸਕਦੇ ਹਾਂ:

```swift
\()
```

String interpolation ਨੂੰ string ਦੇ ਅੰਦਰ ਲਿਖਿਆ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਵੇਰੀਏਬਲ ਜਾਂ ਕਾਂਸਟੈਂਟ ਨੂੰ `\()` ਵਿੱਚ ਲਪੇਟਿਆ ਜਾਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

ਉੱਪਰ ਦਿੱਤੇ ਕੋਡ ਵਿੱਚ, `""` string ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ, ਅਤੇ `\()` ਰਾਹੀਂ ਵੇਰੀਏਬਲ ਜਾਂ ਕਾਂਸਟੈਂਟ string ਵਿੱਚ ਸ਼ਾਮਲ ਕੀਤੇ ਜਾਂਦੇ ਹਨ।

ਉਦਾਹਰਨ ਲਈ:

```swift
"DoubleNum: \(num * 2)"
```

ਦਿਖਾਈ ਜਾਣ ਵਾਲੀ ਸਮੱਗਰੀ ਇਹ ਹੋਵੇਗੀ:

```text
DoubleNum: 2
```

ਇਸ ਤਰ੍ਹਾਂ, ਅਸੀਂ string ਦੇ ਅੰਦਰ ਵੇਰੀਏਬਲ ਜਾਂ ਕਾਂਸਟੈਂਟ ਸ਼ਾਮਲ ਕਰ ਸਕਦੇ ਹਾਂ, ਅਤੇ ਫਿਰ `Text` ਰਾਹੀਂ ਉਹਨਾਂ ਨੂੰ interface 'ਤੇ ਦਿਖਾ ਸਕਦੇ ਹਾਂ।

ਧਿਆਨ ਦਿਓ: `\()` ਸਿਰਫ਼ string `""` ਦੇ ਅੰਦਰ ਹੀ ਵਰਤੀ ਜਾ ਸਕਦੀ ਹੈ।

## ਓਪਰੇਟਰ

ਜਦੋਂ ਅਸੀਂ ਵੇਰੀਏਬਲ ਜਾਂ ਕਾਂਸਟੈਂਟ declare ਕਰਦੇ ਹਾਂ, ਤਾਂ ਅਕਸਰ ਅਸੀਂ ਇਹ ਲਿਖਤ ਵੇਖਦੇ ਹਾਂ:

```swift
var num = 1
```

ਇੱਥੇ `=` ਨੂੰ assignment operator ਕਿਹਾ ਜਾਂਦਾ ਹੈ।

ਇਸ ਦਾ ਕੰਮ ਹੈ: ਸੱਜੇ ਪਾਸੇ ਵਾਲੀ value ਨੂੰ ਖੱਬੇ ਪਾਸੇ ਵਾਲੇ ਵੇਰੀਏਬਲ ਨੂੰ ਦੇਣਾ।

ਇਸ ਉਦਾਹਰਨ ਵਿੱਚ, `1` value ਵੇਰੀਏਬਲ `num` ਨੂੰ assign ਕੀਤੀ ਗਈ ਹੈ।

Assignment operator ਤੋਂ ਇਲਾਵਾ, ਕੁਝ ਆਮ arithmetic operators ਵੀ ਹੁੰਦੇ ਹਨ:

* `+`
* `-`
* `*`
* `/`

ਜਦੋਂ ਅਸੀਂ ਸੰਖਿਆਵਾਂ 'ਤੇ ਹਿਸਾਬ ਕਰਦੇ ਹਾਂ, ਜਿਵੇਂ `Int` ਜਾਂ `Double` type ਦਾ ਡਾਟਾ, ਤਾਂ ਇਹ operators ਵਰਤੇ ਜਾਂਦੇ ਹਨ।

ਉਦਾਹਰਨ ਲਈ:

```swift
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

ਹਿਸਾਬ ਪੂਰਾ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਨਤੀਜਾ ਵੇਰੀਏਬਲ ਵਿੱਚ save ਕਰ ਦਿੱਤਾ ਜਾਂਦਾ ਹੈ।

## ਸਾਰ

ਵੇਰੀਏਬਲ, ਕਾਂਸਟੈਂਟ ਅਤੇ operators programming ਦੇ ਸਭ ਤੋਂ ਬੁਨਿਆਦੀ ਸੰਕਲਪ ਹਨ।

ਵੇਰੀਏਬਲ ਅਤੇ ਕਾਂਸਟੈਂਟ ਦੀ ਮਦਦ ਨਾਲ ਅਸੀਂ ਪ੍ਰੋਗਰਾਮ ਵਿੱਚ ਵੱਖ-ਵੱਖ ਕਿਸਮਾਂ ਦਾ ਡਾਟਾ save ਕਰ ਸਕਦੇ ਹਾਂ; data types ਦੀ ਮਦਦ ਨਾਲ ਅਸੀਂ ਡਾਟਾ ਦੀ ਕਿਸਮ ਸਪੱਸ਼ਟ ਕਰ ਸਕਦੇ ਹਾਂ; ਅਤੇ operators ਰਾਹੀਂ ਅਸੀਂ ਡਾਟਾ 'ਤੇ ਹਿਸਾਬ ਅਤੇ ਸੰਭਾਲ ਕਰ ਸਕਦੇ ਹਾਂ।

ਇਹ ਜਾਣਕਾਰੀਆਂ programming ਦੀ ਦੁਨੀਆ ਦੇ ਬੁਨਿਆਦੀ tools ਵਾਂਗ ਹਨ। ਜੇ ਇਹਨਾਂ ਨੂੰ ਚੰਗੀ ਤਰ੍ਹਾਂ ਸਮਝ ਲਿਆ ਜਾਵੇ, ਤਾਂ ਅੱਗੇ Swift ਅਤੇ SwiftUI ਸਿੱਖਣ ਲਈ ਮਜ਼ਬੂਤ ਬੁਨਿਆਦ ਤਿਆਰ ਹੋ ਜਾਂਦੀ ਹੈ।
