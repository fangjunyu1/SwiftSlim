# ਲੂਪ ਅਤੇ ਲਿਸਟ

ਇਸ ਪਾਠ ਵਿੱਚ ਅਸੀਂ Swift ਵਿੱਚ `for` ਲੂਪ ਅਤੇ SwiftUI ਵਿੱਚ `ForEach` ਸਿੱਖਾਂਗੇ।

ਇਹ ਦੋਵੇਂ ਸਾਨੂੰ ਐਰੇ ਦੇ ਐਲੀਮੈਂਟ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਕ੍ਰਮ ਵਿੱਚ ਪੜ੍ਹਨ ਵਿੱਚ ਮਦਦ ਕਰਦੇ ਹਨ। ਅਸਲ ਡਿਵੈਲਪਮੈਂਟ ਵਿੱਚ, ਸਾਨੂੰ ਅਕਸਰ ਇੱਕ ਸਮੂਹ ਡਾਟਾ ਨੂੰ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਕੱਢਣਾ ਹੁੰਦਾ ਹੈ, ਫਿਰ ਉਸਨੂੰ ਪ੍ਰੋਸੈਸ ਜਾਂ ਦਿਖਾਉਣਾ ਹੁੰਦਾ ਹੈ, ਅਤੇ ਇਹ ਕੰਮ ਕਰਨ ਲਈ ਲੂਪ ਇੱਕ ਮਹੱਤਵਪੂਰਨ ਸਾਧਨ ਹੈ।

ਜਦੋਂ ਅਸੀਂ ਵੈੱਬਸਾਈਟਾਂ ਬ੍ਰਾਊਜ਼ ਕਰਦੇ ਹਾਂ ਜਾਂ ਕੋਈ App ਵਰਤਦੇ ਹਾਂ, ਅਸੀਂ ਅਕਸਰ ਬਹੁਤ ਸਾਰੀਆਂ ਲਿਸਟਾਂ ਜਾਂ ਆਈਟਮ ਵੇਖਦੇ ਹਾਂ। ਇਸ ਤਰ੍ਹਾਂ ਦੀ ਸਮੱਗਰੀ ਆਮ ਤੌਰ 'ਤੇ ਪਹਿਲਾਂ `ਐਰੇ` ਵਿੱਚ ਸੰਭਾਲੀ ਜਾਂਦੀ ਹੈ, ਫਿਰ `ਲੂਪ` ਰਾਹੀਂ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਪੜ੍ਹੀ ਜਾਂਦੀ ਹੈ, ਅਤੇ ਆਖ਼ਿਰਕਾਰ ਇੰਟਰਫੇਸ ਵਿੱਚ ਦਿਖਾਈ ਜਾਂਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, Google ਦੀ search results list:

![Google](../../Resource/017_google.png)

ਸਰਚ ਨਤੀਜਿਆਂ ਦੀ ਲਿਸਟ ਵਿੱਚ ਹਰ ਇੱਕ ਆਈਟਮ ਆਮ ਤੌਰ 'ਤੇ ਇੱਕ ਡਾਟਾ ਨਾਲ ਸੰਬੰਧਿਤ ਹੁੰਦੀ ਹੈ, ਅਤੇ ਲੂਪ ਰਾਹੀਂ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਦਿਖਾਈ ਜਾਂਦੀ ਹੈ।

ਇਸ ਲਈ `ਐਰੇ` ਅਤੇ `ਲੂਪ` ਅਕਸਰ ਇਕੱਠੇ ਆਉਂਦੇ ਹਨ। `ਐਰੇ` ਇੱਕ ਸਮੂਹ ਡਾਟਾ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ, ਅਤੇ `ਲੂਪ` ਉਸ ਡਾਟਾ ਨੂੰ ਕ੍ਰਮਵਾਰ ਕੱਢ ਕੇ ਇੱਕੋ ਜਿਹੀ ਕਾਰਵਾਈ ਕਰਦਾ ਹੈ।

ਹੁਣ ਅਸੀਂ ਸਭ ਤੋਂ ਬੁਨਿਆਦੀ `for` ਲੂਪ ਤੋਂ ਸ਼ੁਰੂ ਕਰਦੇ ਹਾਂ।

## ਕਾਊਂਟਡਾਊਨ

ਮੰਨੋ ਅਸੀਂ ਇੱਕ countdown feature ਬਣਾਉਣਾ ਚਾਹੁੰਦੇ ਹਾਂ, ਅਤੇ ਕ੍ਰਮਵਾਰ `“5、4、3、2、1”` ਆਉਟਪੁੱਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹਾਂ।

ਸਭ ਤੋਂ ਸਿੱਧਾ ਲਿਖਣ ਦਾ ਤਰੀਕਾ ਇਹ ਹੈ:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

ਇਹ ਤਰੀਕਾ countdown effect ਤਾਂ ਦਿਖਾ ਸਕਦਾ ਹੈ, ਪਰ ਜੇ ਸਮੱਗਰੀ ਜ਼ਿਆਦਾ ਹੋਵੇ ਤਾਂ ਇਹ ਬਹੁਤ ਦੁਹਰਾਵਾਂ ਵਾਲਾ ਲੱਗਦਾ ਹੈ, ਅਤੇ ਬਾਅਦ ਵਿੱਚ ਇਸਨੂੰ ਸੋਧਣਾ ਵੀ ਆਸਾਨ ਨਹੀਂ ਰਹਿੰਦਾ।

ਇਸ ਵੇਲੇ ਅਸੀਂ ਪਹਿਲਾਂ ਡਾਟਾ ਨੂੰ ਐਰੇ ਵਿੱਚ ਰੱਖ ਸਕਦੇ ਹਾਂ, ਫਿਰ `for` ਲੂਪ ਨਾਲ ਉਸਨੂੰ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਆਉਟਪੁੱਟ ਕਰ ਸਕਦੇ ਹਾਂ:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ਇਹ ਲਿਖਤ ਹੋਰ ਸਧਾਰਣ ਹੈ, ਅਤੇ ਇਸਨੂੰ ਵਧਾਉਣਾ ਵੀ ਆਸਾਨ ਹੈ। ਭਾਵੇਂ countdown ਵਿੱਚ ਕਿੰਨੇ ਵੀ ਅੰਕ ਹੋਣ, ਸਾਨੂੰ `print` ਇੱਕ-ਇੱਕ ਲਾਈਨ ਵਿੱਚ ਵਾਰ-ਵਾਰ ਨਹੀਂ ਲਿਖਣਾ ਪੈਂਦਾ; ਸਾਨੂੰ ਸਿਰਫ਼ ਲੂਪ ਨੂੰ ਐਰੇ ਦੇ ਹਰ ਐਲੀਮੈਂਟ ਨੂੰ ਆਪੇ ਸੰਭਾਲਣ ਦੇਣਾ ਹੁੰਦਾ ਹੈ।

ਅਸੀਂ ਇਸ ਕੋਡ ਨੂੰ ਬਟਨ ਵਿੱਚ ਰੱਖ ਕੇ ਟੈਸਟ ਕਰ ਸਕਦੇ ਹਾਂ:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

ਜਦੋਂ ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕੀਤਾ ਜਾਂਦਾ ਹੈ, `for` ਲੂਪ ਐਰੇ ਦੇ ਕ੍ਰਮ ਅਨੁਸਾਰ `countDown` ਵਿੱਚੋਂ ਹਰ ਇੱਕ integer ਕੱਢਦਾ ਹੈ ਅਤੇ ਉਹਨਾਂ ਨੂੰ console ਵਿੱਚ print ਕਰਦਾ ਹੈ।

ਹੁਣ ਆਓ ਦੇਖੀਏ ਕਿ `for` ਲੂਪ ਕਿਵੇਂ ਕੰਮ ਕਰਦਾ ਹੈ।

## for ਲੂਪ

`for` ਲੂਪ ਦਾ ਉਪਯੋਗ ਇੱਕ ਸਮੂਹ ਡਾਟਾ ਨੂੰ ਕ੍ਰਮਵਾਰ ਪ੍ਰੋਸੈਸ ਕਰਨ ਲਈ ਹੁੰਦਾ ਹੈ। ਸਭ ਤੋਂ ਆਮ objects ਐਰੇ ਅਤੇ range ਹੁੰਦੇ ਹਨ।

ਮੂਲ ਲਿਖਤ:

```swift
for ਐਲੀਮੈਂਟ in ਐਰੇ ਜਾਂ ਰੇਂਜ {
    ਚਲਾਇਆ ਜਾਣ ਵਾਲਾ ਕੋਡ
}
```

ਉਦਾਹਰਨ ਲਈ, ਹੁਣੇ ਵਾਲਾ countdown code:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

ਇੱਥੇ `i` ਮੌਜੂਦਾ ਕੱਢੇ ਗਏ ਐਲੀਮੈਂਟ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ, ਅਤੇ `in` ਤੋਂ ਬਾਅਦ ਉਹ ਡਾਟਾ ਆਉਂਦਾ ਹੈ ਜਿਸ 'ਤੇ ਲੂਪ ਚੱਲੇਗਾ, ਯਾਨੀ `countDown` ਐਰੇ।

ਇਸ ਕੋਡ ਦਾ ਅਰਥ ਹੈ: ਪ੍ਰੋਗਰਾਮ `countDown` ਐਰੇ ਵਿੱਚੋਂ ਕ੍ਰਮਵਾਰ ਹਰ ਇੱਕ ਐਲੀਮੈਂਟ ਕੱਢੇਗਾ। ਹਰ ਵਾਰੀ ਇੱਕ ਐਲੀਮੈਂਟ ਕੱਢਿਆ ਜਾਂਦਾ ਹੈ, ਉਸਨੂੰ ਅਸਥਾਈ ਤੌਰ 'ਤੇ `i` ਨਾਲ ਦਰਸਾਇਆ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਫਿਰ curly braces ਦੇ ਅੰਦਰ ਵਾਲਾ ਕੋਡ ਚਲਾਇਆ ਜਾਂਦਾ ਹੈ।

ਇਸਦੀ execution process ਨੂੰ ਅਸੀਂ ਇਸ ਤਰ੍ਹਾਂ ਸਮਝ ਸਕਦੇ ਹਾਂ:

* ਪਹਿਲੀ ਵਾਰੀ `5` ਕੱਢਿਆ ਜਾਂਦਾ ਹੈ, ਇਸ ਲਈ ਇਸ ਵੇਲੇ `i` ਦੀ value `5` ਹੈ
* `print(i)` ਚਲਦਾ ਹੈ, ਅਤੇ `5` ਆਉਟਪੁੱਟ ਹੁੰਦਾ ਹੈ
* ਫਿਰ ਅਗਲਾ ਐਲੀਮੈਂਟ `4` ਕੱਢਿਆ ਜਾਂਦਾ ਹੈ
* ਫਿਰ ਦੁਬਾਰਾ `print(i)` ਚਲਦਾ ਹੈ, ਅਤੇ `4` ਆਉਟਪੁੱਟ ਹੁੰਦਾ ਹੈ

ਬਾਕੀ `3`、`2`、`1` ਵੀ ਇਸੇ ਤਰੀਕੇ ਨਾਲ ਕ੍ਰਮਵਾਰ ਚਲਦੇ ਹਨ।

![for](../../Resource/017_for.png)

ਜਦੋਂ ਐਰੇ ਦੇ ਸਾਰੇ ਐਲੀਮੈਂਟ ਪ੍ਰੋਸੈਸ ਹੋ ਜਾਂਦੇ ਹਨ, `for` ਲੂਪ ਆਪਣੇ ਆਪ ਖਤਮ ਹੋ ਜਾਂਦਾ ਹੈ।

ਅਸੀਂ `for` ਲੂਪ ਦੀ ਮਦਦ ਨਾਲ ਐਰੇ ਦੇ ਹਰ ਐਲੀਮੈਂਟ ਨੂੰ ਲੈ ਸਕਦੇ ਹਾਂ ਅਤੇ countdown ਵਰਗੀਆਂ functionalities ਪੂਰੀਆਂ ਕਰ ਸਕਦੇ ਹਾਂ।

**ਸੁਝਾਅ**: `i` for ਲੂਪ ਵਿੱਚ ਬਹੁਤ ਆਮ ਨਾਮ ਹੈ, ਜਿਸਦਾ ਉਪਯੋਗ ਮੌਜੂਦਾ ਕੱਢੇ ਗਏ ਐਲੀਮੈਂਟ ਲਈ ਹੁੰਦਾ ਹੈ, ਪਰ ਇਹ ਕੋਈ ਫਿਕਸ ਤਰੀਕਾ ਨਹੀਂ; ਤੁਸੀਂ ਹੋਰ English ਨਾਮ ਵੀ ਵਰਤ ਸਕਦੇ ਹੋ।

ਉਦਾਹਰਨ ਲਈ, ਹਰ ਕੱਢੇ ਗਏ ਐਲੀਮੈਂਟ ਲਈ `num` ਵਰਤਣਾ:

```swift
for num in countDown {
    print(num)
}
```

Curly braces ਦੇ ਅੰਦਰ ਵੀ ਮੌਜੂਦਾ ਕੱਢੇ ਗਏ ਐਲੀਮੈਂਟ ਲਈ `num` ਹੀ ਵਰਤਣਾ ਪਵੇਗਾ।

### ਰੇਂਜ 'ਤੇ ਲੂਪ

`for` ਲੂਪ ਸਿਰਫ਼ ਐਰੇ 'ਤੇ ਹੀ ਨਹੀਂ, ਰੇਂਜ 'ਤੇ ਵੀ ਚੱਲ ਸਕਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਜਦੋਂ ਸਾਨੂੰ `50` ਦੇ ਅੰਦਰਲੇ positive integers ਦਾ sum ਕੈਲਕੂਲੇਟ ਕਰਨਾ ਹੋਵੇ, ਤਾਂ ਸਾਨੂੰ `1` ਤੋਂ `50` ਤੱਕ ਦੇ ਸਾਰੇ integers ਇੱਕ-ਇੱਕ ਕਰਕੇ ਜੋੜਣੇ ਪੈਂਦੇ ਹਨ।

ਸਭ ਤੋਂ ਸਿੱਧੀ ਲਿਖਤ ਇਹ ਹੋ ਸਕਦੀ ਹੈ:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

ਇਸ ਤਰੀਕੇ ਦੀ ਸਮੱਸਿਆ ਇਹ ਹੈ ਕਿ ਸਾਨੂੰ ਹਰ ਇੱਕ ਨੰਬਰ ਹੱਥੋਂ ਲਿਖਣਾ ਪੈਂਦਾ ਹੈ। ਇਹ ਔਖਾ ਵੀ ਹੈ ਅਤੇ ਅਸਲ ਡਿਵੈਲਪਮੈਂਟ ਸਥਿਤੀਆਂ ਲਈ ਵੀ ਢੰਗ ਦਾ ਨਹੀਂ।

ਇਸ ਵੇਲੇ ਅਸੀਂ range operator ਨੂੰ `for` ਲੂਪ ਨਾਲ ਮਿਲਾਕੇ ਇਹ ਕੰਮ ਕਰ ਸਕਦੇ ਹਾਂ।

ਪਿਛਲੇ “operators” ਅਧਿਆਇ ਵਿੱਚ ਅਸੀਂ range operator ਸਿੱਖਿਆ ਸੀ:

```swift
1...3   // 1, 2, 3
```

ਇੱਥੇ `...` ਇੱਕ closed range ਦੱਸਦਾ ਹੈ, ਯਾਨੀ ਸ਼ੁਰੂਆਤੀ ਅਤੇ ਅੰਤਿਮ ਦੋਵੇਂ values ਸ਼ਾਮਲ ਹਨ।

ਇਸ ਲਈ ਇੱਥੇ `1` ਤੋਂ `50` ਤੱਕ ਦੇ integer range ਲਈ `for` ਲੂਪ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ:

```swift
for i in 1...50 {
    
}
```

ਸਾਨੂੰ ਇਹਨਾਂ integers ਨੂੰ ਜੋੜਨਾ ਹੈ, ਇਸ ਲਈ ਇੱਕ variable ਵੀ ਚਾਹੀਦਾ ਹੈ ਜੋ total sum ਸੰਭਾਲ ਸਕੇ:

```swift
var count = 0
```

ਫਿਰ ਹਰ iteration ਵਿੱਚ ਮੌਜੂਦਾ ਕੱਢੇ ਗਏ ਨੰਬਰ ਨੂੰ `count` ਵਿੱਚ ਜੋੜ ਦਿੰਦੇ ਹਾਂ:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

ਇਸ ਕੋਡ ਦਾ ਅਰਥ ਹੈ:

* `count` ਮੌਜੂਦਾ total sum ਸੰਭਾਲਣ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ
* `for` ਲੂਪ `1` ਤੋਂ `50` ਤੱਕ ਦੇ integers ਕ੍ਰਮਵਾਰ ਕੱਢਦਾ ਹੈ
* ਹਰ ਵਾਰੀ ਇੱਕ integer ਕੱਢਿਆ ਜਾਂਦਾ ਹੈ, `count += i` ਇੱਕ ਵਾਰ ਚੱਲਦਾ ਹੈ
* ਲੂਪ ਖਤਮ ਹੋਣ ਤੋਂ ਬਾਅਦ, `count` ਵਿੱਚ final sum ਸੰਭਾਲਿਆ ਹੋਇਆ ਹੁੰਦਾ ਹੈ

ਇਸ ਤਰ੍ਹਾਂ, ਅਸੀਂ `50` ਦੇ ਅੰਦਰਲੇ ਸਾਰੇ positive integers ਦਾ ਜੋੜ ਲੱਭ ਸਕਦੇ ਹਾਂ।

ਇਸੇ ਤਰ੍ਹਾਂ, ਜੇ ਸਾਨੂੰ `1` ਤੋਂ `100` ਤੱਕ ਦੇ integers ਦਾ sum ਕੈਲਕੂਲੇਟ ਕਰਨਾ ਹੋਵੇ, ਤਾਂ ਸਾਨੂੰ ਸਿਰਫ਼ range ਬਦਲਣੀ ਪਵੇਗੀ:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` ਲੂਪ ਸਿਰਫ਼ ਐਰੇ 'ਤੇ ਹੀ ਨਹੀਂ, ਇੱਕ range 'ਤੇ ਵੀ ਚੱਲ ਸਕਦਾ ਹੈ। ਡਾਟਾ ਨੂੰ ਕ੍ਰਮਵਾਰ ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਪ੍ਰੋਸੈਸ ਕਰਨ ਦੀ ਇਹ ਸਮਰੱਥਾ ਡਿਵੈਲਪਮੈਂਟ ਵਿੱਚ ਬਹੁਤ ਆਮ ਹੈ।

ਇਹ ਸਾਨੂੰ ਬਹੁਤ ਸਾਰਾ repetitive ਕੰਮ ਆਪਣੇ ਆਪ ਕਰਵਾਉਣ ਵਿੱਚ ਮਦਦ ਕਰਦੀ ਹੈ, ਅਤੇ ਸਾਨੂੰ ਹਰ ਇੱਕ ਨੰਬਰ ਦੇ sum ਜਾਂ ਹੋਰ ਇਸ ਤਰ੍ਹਾਂ ਦੇ repetitive operations ਹੱਥੋਂ ਨਹੀਂ ਲਿਖਣੇ ਪੈਂਦੇ।

## SwiftUI ਵਿੱਚ ਲੂਪ

ਪਹਿਲਾਂ ਵਾਲਾ `for` ਲੂਪ Swift ਭਾਸ਼ਾ ਦਾ ਹਿੱਸਾ ਹੈ।

ਪਰ SwiftUI ਵਿੱਚ ਅਸੀਂ view structure ਦੇ ਅੰਦਰ ਸਿੱਧਾ `for` ਵਰਤ ਕੇ views generate ਨਹੀਂ ਕਰ ਸਕਦੇ:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

ਇਸਦਾ ਕਾਰਨ ਇਹ ਹੈ ਕਿ `VStack` ਦੇ ਅੰਦਰ SwiftUI views ਚਾਹੀਦੀਆਂ ਹੁੰਦੀਆਂ ਹਨ, ਅਤੇ ਆਮ `for` ਲੂਪ ਖੁਦ ਕੋਈ view ਨਹੀਂ ਹੁੰਦਾ।

ਜੇ ਅਸੀਂ SwiftUI ਵਿੱਚ ਐਰੇ ਦੀ ਸਮੱਗਰੀ ਦੇ ਆਧਾਰ 'ਤੇ interface ਨੂੰ ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਬਣਾਉਣਾ ਚਾਹੀਏ, ਤਾਂ ਸਾਨੂੰ `ForEach` ਵਰਤਣਾ ਪੈਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਜੇ ਅਸੀਂ ਕਿਸੇ ਸਮੂਹ ਦੇ ਸਾਰੇ ਮੈਂਬਰਾਂ ਦੇ ਨਾਮ ਦਿਖਾਉਣੇ ਹੋਣ, ਤਾਂ ਇਹ ਇਸ ਤਰ੍ਹਾਂ ਲਿਖਿਆ ਜਾ ਸਕਦਾ ਹੈ:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

ਦਿਖਾਈ ਦੇਣ ਵਾਲਾ effect:

![for1](../../Resource/017_for1.png)

ਫਿਰ ਅਸੀਂ ਹਰ `Text` ਨੂੰ ਇੱਕੋ ਜਿਹੇ modifiers ਦੇ ਸਕਦੇ ਹਾਂ:

```swift

    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)

```

ਹੁਣ ਹਰ `Text` ਹੋਰ ਛੋਟੇ ਅਤੇ bold ਫੌਂਟ ਵਿੱਚ ਦਿਖਾਈ ਦੇਵੇਗਾ।

ਦਿਖਾਈ ਦੇਣ ਵਾਲਾ effect:

![](../../Resource/017_for2.png)

ਹਾਲਾਂਕਿ modifiers interface ਨੂੰ ਹੋਰ ਸੁੰਦਰ ਬਣਾਉਂਦੇ ਹਨ, ਪਰ ਇਹ ਕੋਡ ਨੂੰ ਹੋਰ ਲੰਮਾ ਅਤੇ repetitive ਵੀ ਬਣਾ ਦਿੰਦੇ ਹਨ।

ਜੇ ਨਾਮ ਹੋਰ ਵੱਧ ਜਾਣ, ਜਾਂ ਬਾਅਦ ਵਿੱਚ ਸਾਨੂੰ ਇਕੱਠੇ ਫੌਂਟ, ਰੰਗ ਆਦਿ style ਬਦਲਣੀ ਹੋਵੇ, ਤਾਂ ਬਹੁਤ ਸਾਰਾ ਇੱਕੋ ਜਿਹਾ ਕੋਡ ਮੁੜ-ਮੁੜ ਲਿਖਣਾ ਅਤੇ ਬਦਲਣਾ ਪਵੇਗਾ। ਇਸ ਨਾਲ ਕੰਮ ਔਖਾ ਹੋ ਜਾਂਦਾ ਹੈ ਅਤੇ maintenance ਲਈ ਵੀ ਇਹ ਚੰਗਾ ਨਹੀਂ ਹੁੰਦਾ।

ਇਸ ਵੇਲੇ ਅਸੀਂ ਪਹਿਲਾਂ ਇਹਨਾਂ ਨਾਮਾਂ ਨੂੰ ਐਰੇ ਵਿੱਚ ਰੱਖ ਸਕਦੇ ਹਾਂ, ਫਿਰ SwiftUI ਦੇ `ForEach` ਦੀ ਮਦਦ ਨਾਲ ਐਰੇ ਦੇ ਡਾਟਾ ਦੇ ਆਧਾਰ 'ਤੇ views ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਜਨਰੇਟ ਕਰ ਸਕਦੇ ਹਾਂ:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

ਇਹ ਕੋਡ ਹੋਰ ਸੰਖੇਪ ਹੈ। `ForEach` ਐਰੇ ਦੇ ਹਰ ਐਲੀਮੈਂਟ ਨੂੰ ਕ੍ਰਮਵਾਰ ਪੜ੍ਹਦਾ ਹੈ, ਅਤੇ ਉਸਦੇ ਅਨੁਸਾਰ ਸਮੱਗਰੀ ਨੂੰ interface ਵਿੱਚ ਦਿਖਾਉਂਦਾ ਹੈ।

ਜਦੋਂ ਅਸੀਂ ਬਾਅਦ ਵਿੱਚ ਨਵੇਂ ਨਾਮ ਜੋੜਦੇ ਹਾਂ, ਤਾਂ ਸਾਨੂੰ ਸਿਰਫ਼ `names` ਐਰੇ ਦੇ ਅੰਦਰਲੇ ਨਾਮ ਬਦਲਣੇ ਪੈਂਦੇ ਹਨ; ਸਾਨੂੰ ਹਰ ਵਾਰ `Text` ਅਤੇ complex modifiers ਮੁੜ ਨਹੀਂ ਲਿਖਣੇ ਪੈਂਦੇ।

## ForEach

`ForEach` SwiftUI ਵਿੱਚ views ਨੂੰ ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਜਨਰੇਟ ਕਰਨ ਲਈ ਵਰਤੀ ਜਾਣ ਵਾਲੀ structure ਹੈ। ਤੁਸੀਂ ਇਸਨੂੰ SwiftUI ਵਿੱਚ `for` ਲੂਪ ਵਰਗੀ ਇੱਕ structure ਵਜੋਂ ਸਮਝ ਸਕਦੇ ਹੋ।

ਮੂਲ ਵਰਤੋਂ:

```swift
ForEach(ਐਰੇ, id: \.self) { ਐਲੀਮੈਂਟ in
    SwiftUI ਕੋਡ
}
```

ਉਦਾਹਰਨ ਲਈ, ਹੁਣੇ ਵਾਲੇ group names:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

ਇਸ ਕੋਡ ਦਾ ਅਰਥ ਹੈ:

`names` ਉਹ ਐਰੇ ਹੈ ਜਿਸ 'ਤੇ ਲੂਪ ਚੱਲ ਰਿਹਾ ਹੈ, ਯਾਨੀ ਉਹ ਡਾਟਾ ਜੋ ਦਿਖਾਇਆ ਜਾਣਾ ਹੈ।

`id:\.self` ਦਾ ਅਰਥ ਹੈ ਕਿ ਐਰੇ ਦੇ ਐਲੀਮੈਂਟ ਨੂੰ ਆਪ ਹੀ ਹਰ ਇੱਕ ਆਈਟਮ ਨੂੰ ਵੱਖ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾ ਰਿਹਾ ਹੈ।

ਮੌਜੂਦਾ ਉਦਾਹਰਨ ਵਿੱਚ, ਐਰੇ ਵਿੱਚ strings ਹਨ:

```swift
["Sam", "John", "Wathon", "Bob"]
```

ਇਸ ਲਈ SwiftUI ਸਿੱਧਾ "Sam"、"John"、"Wathon"、"Bob" ਇਹਨਾਂ strings ਨੂੰ ਆਪ ਹੀ ਵੱਖ-ਵੱਖ ਸਮੱਗਰੀ ਦੀ ਪਹਿਚਾਣ ਲਈ ਵਰਤਦਾ ਹੈ।

ਸ਼ੁਰੂਆਤੀ ਪੱਧਰ 'ਤੇ ਆਮ `String` ਅਤੇ `Int` ਵਰਗੇ simple arrays ਲਈ, ਅਸੀਂ ਆਮ ਤੌਰ 'ਤੇ ਪਹਿਲਾਂ ਇਸ ਤਰ੍ਹਾਂ ਲਿਖ ਸਕਦੇ ਹਾਂ:

```swift
id: \.self
```

ਫਿਲਹਾਲ ਤੁਹਾਨੂੰ ਸਿਰਫ਼ ਇਹ ਇੱਕ ਲਿਖਤ ਯਾਦ ਰੱਖਣੀ ਹੈ; id ਦੇ ਹੋਰ ਨਿਯਮਾਂ ਨੂੰ ਗਹਿਰਾਈ ਨਾਲ ਸਮਝਣ ਦੀ ਲੋੜ ਨਹੀਂ।

`item in` ਵਿੱਚ `item` ਮੌਜੂਦਾ ਕੱਢੇ ਗਏ ਐਲੀਮੈਂਟ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਪਹਿਲੀ iteration ਵਿੱਚ `item` `"Sam"` ਹੈ, ਅਤੇ ਦੂਜੀ iteration ਵਿੱਚ `item` `"John"` ਹੈ।

ਇਸ ਲਈ:

```swift
Text(item)
```

ਕ੍ਰਮਵਾਰ ਇਹ ਬਣ ਜਾਂਦਾ ਹੈ:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

ਇਸ ਲਈ, ਇਸ ਕੋਡ ਦਾ ਕੰਮ ਇਹ ਹੈ: `names` ਐਰੇ ਵਿੱਚੋਂ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਹਰ ਨਾਮ ਕੱਢਣਾ, ਅਤੇ ਹਰ ਨਾਮ ਲਈ ਇੱਕ `Text` view ਜਨਰੇਟ ਕਰਨਾ।

### ਚਲਣ ਦੀ ਪ੍ਰਕਿਰਿਆ

ਚਲਣ ਦੀ ਪ੍ਰਕਿਰਿਆ ਨੂੰ ਅਸੀਂ ਇਸ ਤਰ੍ਹਾਂ ਸਮਝ ਸਕਦੇ ਹਾਂ:

* `ForEach` ਪਹਿਲਾਂ `names` ਐਰੇ ਦਾ ਪਹਿਲਾ ਐਲੀਮੈਂਟ `"Sam"` ਪੜ੍ਹਦਾ ਹੈ
* `item` ਅਸਥਾਈ ਤੌਰ 'ਤੇ `"Sam"` ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ
* `Text(item)` `"Sam"` ਦਿਖਾਉਂਦਾ ਹੈ
* ਫਿਰ ਅਗਲਾ ਐਲੀਮੈਂਟ `"John"` ਪੜ੍ਹਿਆ ਜਾਂਦਾ ਹੈ
* ਫਿਰ ਉਹੀ view code ਦੁਬਾਰਾ ਚਲਦਾ ਹੈ

ਆਖ਼ਿਰਕਾਰ ਐਰੇ ਦੇ ਸਾਰੇ ਨਾਮ ਦਿਖਾਈ ਦੇ ਜਾਣਗੇ।

ਇਸ ਤਰ੍ਹਾਂ ਲਿਖਣ ਦਾ ਫਾਇਦਾ ਇਹ ਹੈ: ਜਦੋਂ ਨਾਮ ਵੱਧ ਜਾਂਦੇ ਹਨ, ਸਾਨੂੰ ਬਹੁਤ ਸਾਰੇ `Text` ਮੁੜ-ਮੁੜ ਨਹੀਂ ਲਿਖਣੇ ਪੈਂਦੇ; ਸਾਨੂੰ ਸਿਰਫ਼ ਐਰੇ ਦੀ ਸਮੱਗਰੀ ਬਦਲਣੀ ਪੈਂਦੀ ਹੈ।

### ਫੋਟੋਆਂ 'ਤੇ ਲੂਪ

`ForEach` ਸਿਰਫ਼ ਟੈਕਸਟ ਹੀ ਨਹੀਂ, ਸਗੋਂ ਤਸਵੀਰਾਂ, ਰੰਗਾਂ ਜਾਂ ਹੋਰ SwiftUI views ਵੀ ਦਿਖਾ ਸਕਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਮੰਨੋ ਅਸੀਂ 4 ਤਸਵੀਰਾਂ ਤਿਆਰ ਕੀਤੀਆਂ ਹਨ:


![image](../../Resource/017_for3.png)

ਜੇ `ForEach` ਨਾ ਵਰਤਿਆ ਜਾਵੇ, ਤਾਂ ਸਾਨੂੰ ਹਰ ਤਸਵੀਰ ਹੱਥੋਂ ਲਿਖਣੀ ਪਵੇਗੀ:

```swift
struct ContentView: View {
```

ਇਹ ਤਰੀਕਾ ਕੰਮ ਤਾਂ ਕਰ ਦਿੰਦਾ ਹੈ, ਪਰ ਕੋਡ ਲੰਮਾ ਹੋ ਜਾਂਦਾ ਹੈ।

ਜੇ ਤਸਵੀਰਾਂ ਹੋਰ ਵਧਣ ਲੱਗਣ, ਤਾਂ ਸਾਨੂੰ ਹੋਰ ਵੀ ਵੱਧ `Image` ਕੋਡ ਮੁੜ-ਮੁੜ ਲਿਖਣਾ ਪਵੇਗਾ।

ਬਾਅਦ ਵਿੱਚ ਜੇ ਸਾਨੂੰ ਸਾਰੀਆਂ ਤਸਵੀਰਾਂ ਦਾ size ਇਕੱਠੇ ਬਦਲਣਾ ਹੋਵੇ, ਤਾਂ ਹਰ ਇੱਕ ਨੂੰ ਵੱਖ-ਵੱਖ ਸੋਧਣਾ ਪਵੇਗਾ।

ਇਸ ਵੇਲੇ ਅਸੀਂ ਤਸਵੀਰਾਂ ਦੇ ਨਾਮ ਐਰੇ ਵਿੱਚ ਰੱਖ ਸਕਦੇ ਹਾਂ, ਫਿਰ `ForEach` ਦੀ ਮਦਦ ਨਾਲ image views ਨੂੰ ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਜਨਰੇਟ ਕਰ ਸਕਦੇ ਹਾਂ:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

ਇੱਥੇ `images` ਇੱਕ string array ਹੈ, ਅਤੇ ਐਰੇ ਦਾ ਹਰ ਐਲੀਮੈਂਟ ਇੱਕ ਤਸਵੀਰ ਦਾ ਨਾਮ ਹੈ।

`ForEach(images, id: \.self)` ਦਾ ਅਰਥ ਹੈ ਕਿ `images` ਐਰੇ ਦਾ ਹਰ ਐਲੀਮੈਂਟ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਪੜ੍ਹਿਆ ਜਾਵੇਗਾ, ਅਤੇ ਐਲੀਮੈਂਟ ਨੂੰ ਆਪ ਹੀ ਹਰ ਇੱਕ item ਨੂੰ ਵੱਖ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਵੇਗਾ, ਫਿਰ ਮੌਜੂਦਾ ਐਲੀਮੈਂਟ ਦੇ ਆਧਾਰ 'ਤੇ ਸੰਬੰਧਿਤ view ਬਣਾਈ ਜਾਵੇਗੀ।

ਉਦਾਹਰਨ ਲਈ, ਜੇ ਪਹਿਲੀ ਵਾਰੀ `"430F9BEF"` ਪੜ੍ਹਿਆ ਗਿਆ, ਤਾਂ:

```swift
Image(item)
```

ਇਹ ਇਸਦੇ ਬਰਾਬਰ ਹੈ:

```swift
Image("430F9BEF")
```

ਬਾਕੀ ਤਸਵੀਰਾਂ ਵੀ ਇਸੇ ਤਰੀਕੇ ਨਾਲ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਦਿਖਾਈ ਜਾਣਗੀਆਂ।

![image](../../Resource/017_for4.png)

ਇੱਥੇ ਵੀ ਅਸੀਂ ਇਹ ਹੀ ਵਰਤਦੇ ਹਾਂ:

```swift
id: \.self
```

ਕਾਰਨ ਪਹਿਲਾਂ ਵਰਗਾ ਹੀ ਹੈ: `images` ਐਰੇ ਦੇ ਐਲੀਮੈਂਟ ਵੀ simple string type ਹਨ, ਇਸ ਲਈ ਸ਼ੁਰੂਆਤੀ ਪੜਾਅ ਵਿੱਚ ਅਸੀਂ ਸਿੱਧਾ ਐਲੀਮੈਂਟ ਨੂੰ ਹੀ ਪਹਿਚਾਣ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹਾਂ।

ਇਸਨੂੰ ਇੱਕ fixed formula ਵਾਂਗ ਆਸਾਨੀ ਨਾਲ ਸਮਝਿਆ ਜਾ ਸਕਦਾ ਹੈ:

**ਜਦੋਂ `ForEach` `String` ਜਾਂ `Int` ਵਰਗੇ simple arrays 'ਤੇ ਲੂਪ ਕਰਦਾ ਹੈ, ਤਾਂ ਅਸੀਂ ਆਮ ਤੌਰ 'ਤੇ `id: \.self` ਲਿਖਦੇ ਹਾਂ।**

### ਨਵੀਆਂ ਤਸਵੀਰਾਂ ਜੋੜਣਾ

ਜੇ ਬਾਅਦ ਵਿੱਚ ਹੋਰ ਤਸਵੀਰਾਂ ਜੋੜਣੀਆਂ ਹੋਣ, ਤਾਂ ਸਾਨੂੰ ਸਿਰਫ਼ ਐਰੇ ਦੀ ਸਮੱਗਰੀ ਬਦਲਣੀ ਪਵੇਗੀ:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

ਸਾਨੂੰ ਨਵਾਂ `Image` ਕੋਡ ਦੁਬਾਰਾ ਨਹੀਂ ਲਿਖਣਾ ਪਵੇਗਾ।

ਜੇ ਤਸਵੀਰਾਂ ਜ਼ਿਆਦਾ ਹੋਣ, ਤਾਂ ਇਸਨੂੰ `ScrollView` ਨਾਲ ਮਿਲਾਕੇ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ, ਨਹੀਂ ਤਾਂ ਸਮੱਗਰੀ ਸਕ੍ਰੀਨ ਤੋਂ ਬਾਹਰ ਨਿਕਲ ਜਾਣ 'ਤੇ ਪੂਰੀ ਤਰ੍ਹਾਂ preview ਨਹੀਂ ਹੋਵੇਗੀ।

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### ਤਸਵੀਰਾਂ ਸੋਧਣਾ

ਜਦੋਂ ਸਾਨੂੰ ਤਸਵੀਰਾਂ ਦੀ style ਇਕੱਠੇ ਬਦਲਣੀ ਹੋਵੇ, ਤਾਂ ਸਾਨੂੰ ਸਿਰਫ਼ `ForEach` ਦੇ ਅੰਦਰਲਾ ਕੋਡ ਬਦਲਣਾ ਹੁੰਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, width ਨੂੰ `80` ਕਰਨਾ:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

ਇਸ ਤਰ੍ਹਾਂ ਸਾਰੀਆਂ ਤਸਵੀਰਾਂ ਇਕੱਠੇ ਬਦਲ ਜਾਣਗੀਆਂ; ਸਾਨੂੰ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਸੋਧਣ ਦੀ ਲੋੜ ਨਹੀਂ ਪਵੇਗੀ।

ਇਹ `ForEach` ਦਾ ਇੱਕ ਮਹੱਤਵਪੂਰਨ ਕੰਮ ਹੈ:

**ਜਦੋਂ ਕਈ views ਦੀ structure ਇੱਕੋ ਜਿਹੀ ਹੋਵੇ ਅਤੇ ਕੇਵਲ ਸਮੱਗਰੀ ਵੱਖ ਹੋਵੇ, ਤਦ ਅਸੀਂ ਇੱਕੋ view code ਲਿਖ ਸਕਦੇ ਹਾਂ, ਅਤੇ ਐਰੇ ਦਾ ਡਾਟਾ ਇਹ ਨਿਰਧਾਰਤ ਕਰੇਗਾ ਕਿ ਕੀ ਦਿਖਾਉਣਾ ਹੈ।**

## ਸਾਰ

ਇਸ ਪਾਠ ਵਿੱਚ ਅਸੀਂ `for` ਲੂਪ ਅਤੇ `ForEach` ਸਿੱਖੇ।

`for` ਲੂਪ ਐਰੇ ਜਾਂ range ਦੇ ਐਲੀਮੈਂਟ ਨੂੰ ਕ੍ਰਮਵਾਰ ਪ੍ਰੋਸੈਸ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਇਹ ਡਾਟਾ ਪ੍ਰੋਸੈਸਿੰਗ ਵੱਲ ਜ਼ਿਆਦਾ ਝੁਕਿਆ ਹੁੰਦਾ ਹੈ।

`ForEach` SwiftUI ਵਿੱਚ ਐਰੇ ਦੀ ਸਮੱਗਰੀ ਦੇ ਆਧਾਰ 'ਤੇ views ਨੂੰ ਦੁਹਰਾਉਂਦੇ ਹੋਏ ਜਨਰੇਟ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਇਹ view display ਵੱਲ ਜ਼ਿਆਦਾ ਝੁਕਿਆ ਹੁੰਦਾ ਹੈ।

ਮੌਜੂਦਾ ਸ਼ੁਰੂਆਤੀ ਪੜਾਅ ਵਿੱਚ, `ForEach` ਦੀ ਸਭ ਤੋਂ ਆਮ ਲਿਖਤ ਇਹ ਹੈ:

```swift
ForEach(ਐਰੇ, id: \.self) { ਐਲੀਮੈਂਟ in
    SwiftUI ਕੋਡ
}
```

ਇੱਥੇ `id: \.self` ਦਾ ਅਰਥ ਹੈ ਕਿ ਐਲੀਮੈਂਟ ਨੂੰ ਆਪ ਹੀ ਹਰ ਇੱਕ item ਨੂੰ ਵੱਖ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾ ਰਿਹਾ ਹੈ।

ਜਦੋਂ ਐਰੇ ਦੀ ਸਮੱਗਰੀ `String` ਜਾਂ `Int` ਵਰਗੇ simple types ਹੋਣ, ਤਾਂ ਅਸੀਂ ਆਮ ਤੌਰ 'ਤੇ ਪਹਿਲਾਂ ਇਹੀ ਇੱਕ ਲਿਖਤ ਵਰਤ ਸਕਦੇ ਹਾਂ।

ਆਉਣ ਵਾਲੇ ਸਮੇਂ ਵਿੱਚ, ਜਦੋਂ ਤੁਸੀਂ ਵੈੱਬਸਾਈਟਾਂ ਜਾਂ Apps ਬ੍ਰਾਊਜ਼ ਕਰਦੇ ਹੋਏ ਕਿਸੇ ਸਮੂਹ ਵਿੱਚ ਦੁਹਰਾਇਆ ਹੋਇਆ ਟੈਕਸਟ, ਤਸਵੀਰਾਂ ਜਾਂ ਹੋਰ ਲਿਸਟ ਸਮੱਗਰੀ ਵੇਖੋ, ਤਾਂ ਯਾਦ ਰੱਖ ਸਕਦੇ ਹੋ ਕਿ SwiftUI ਵਿੱਚ ਇਹ ਸਮੱਗਰੀ ਆਮ ਤੌਰ 'ਤੇ ਪਹਿਲਾਂ ਐਰੇ ਵਿੱਚ ਰੱਖੀ ਜਾਂਦੀ ਹੈ, ਅਤੇ ਫਿਰ `ForEach` ਰਾਹੀਂ ਇੱਕ-ਇੱਕ ਕਰਕੇ ਦਿਖਾਈ ਜਾਂਦੀ ਹੈ।
