# Xcode ਫਾਈਲਾਂ ਦਾ ਪ੍ਰਬੰਧਨ

ਇਸ ਪਾਠ ਵਿੱਚ, ਅਸੀਂ Xcode ਵਿੱਚ ਆਮ ਫਾਈਲ ਕਾਰਵਾਈਆਂ ਸਿੱਖਾਂਗੇ।

ਪਿਛਲੇ ਪਾਠਾਂ ਵਿੱਚ, ਅਸੀਂ ਆਪਣੀ ਜ਼ਿਆਦਾਤਰ ਸ਼ਿਫ਼ਰ `ContentView` ਵਿੱਚ ਲਿਖੀ ਸੀ। ਇਸ ਨਾਲ ਸ਼ਿਫ਼ਰ ਅਤੇ ਦਿਖਾਈ ਦੇਣ ਵਾਲੇ ਪ੍ਰਭਾਵ ਨੂੰ ਵੇਖਣਾ ਆਸਾਨ ਹੁੰਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

ਪਰ ਅਸਲੀ App ਵਿੱਚ, ਆਮ ਤੌਰ 'ਤੇ ਸਾਰੀ ਸ਼ਿਫ਼ਰ ਇੱਕ ਹੀ ਫਾਈਲ ਵਿੱਚ ਨਹੀਂ ਲਿਖੀ ਜਾਂਦੀ।

ਜਦੋਂ ਪੰਨੇ, ਕੰਪੋਨੈਂਟ ਅਤੇ ਫੰਕਸ਼ਨ ਵਧਦੇ ਜਾਂਦੇ ਹਨ, ਤਾਂ ਸਾਨੂੰ ਵੱਖ-ਵੱਖ ਵਿਊਜ਼ ਨੂੰ ਵੱਖ-ਵੱਖ SwiftUI ਫਾਈਲਾਂ ਵਿੱਚ ਵੰਡਣ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

ਇਸ ਨਾਲ ਸ਼ਿਫ਼ਰ ਦੀ ਬਣਤਰ ਹੋਰ ਸਪਸ਼ਟ ਹੁੰਦੀ ਹੈ, ਅਤੇ ਬਾਅਦ ਦੀ ਸੰਭਾਲ ਅਤੇ ਸੋਧ ਵੀ ਆਸਾਨ ਹੁੰਦੀ ਹੈ।

## ਫਾਈਲਾਂ ਦਾ ਪ੍ਰਬੰਧਨ ਕਿਉਂ ਕਰਨਾ ਹੈ

SwiftUI ਵਿੱਚ, ਇੱਕ ਪੰਨਾ ਜਾਂ ਇੱਕ ਕੰਪੋਨੈਂਟ ਆਮ ਤੌਰ 'ਤੇ ਵੱਖਰੇ ਵਿਊ ਫਾਈਲ ਵਜੋਂ ਲਿਖਿਆ ਜਾ ਸਕਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ਇੱਥੇ `CardView` ਇੱਕ SwiftUI ਵਿਊ ਹੈ।

ਜੇ ਸਾਰੇ ਵਿਊਜ਼ `ContentView` ਵਿੱਚ ਲਿਖੇ ਜਾਣ, ਤਾਂ ਸ਼ਿਫ਼ਰ ਧੀਰੇ-ਧੀਰੇ ਲੰਬੀ ਅਤੇ ਪੜ੍ਹਨ ਵਿੱਚ ਔਖੀ ਹੋ ਜਾਵੇਗੀ।

ਇਸ ਲਈ, ਸਾਨੂੰ ਫਾਈਲਾਂ ਬਣਾਉਣ, ਨਾਮ ਬਦਲਣ, ਮਿਟਾਉਣ ਅਤੇ ਵੱਖ-ਵੱਖ ਸ਼ਿਫ਼ਰ ਫਾਈਲਾਂ ਨੂੰ ਪ੍ਰਬੰਧਿਤ ਕਰਨ ਲਈ ਫੋਲਡਰ ਵਰਤਣਾ ਸਿੱਖਣਾ ਚਾਹੀਦਾ ਹੈ।

## SwiftUI ਫਾਈਲ ਬਣਾਉਣਾ

Xcode ਵਿੱਚ, SwiftUI ਫਾਈਲ ਬਣਾਉਣ ਦੇ ਆਮ ਤੌਰ 'ਤੇ ਦੋ ਤਰੀਕੇ ਹਨ:

ਇੱਕ ਤਰੀਕਾ ਖਾਲੀ ਫਾਈਲ ਬਣਾਉਣਾ ਹੈ, ਫਿਰ ਸ਼ਿਫ਼ਰ ਆਪਣੇ ਆਪ ਲਿਖਣੀ ਹੈ।

ਦੂਜਾ ਤਰੀਕਾ SwiftUI ਟੈਂਪਲੇਟ ਫਾਈਲ ਬਣਾਉਣਾ ਹੈ, ਜਿਸ ਵਿੱਚ Xcode ਆਪਣੇ ਆਪ ਬੁਨਿਆਦੀ ਸ਼ਿਫ਼ਰ ਤਿਆਰ ਕਰਦਾ ਹੈ।

ਸ਼ੁਰੂਆਤੀ ਸਿੱਖਣ ਵਾਲਿਆਂ ਲਈ, SwiftUI ਟੈਂਪਲੇਟ ਫਾਈਲ ਵਰਤਣ ਦੀ ਜ਼ਿਆਦਾ ਸਿਫ਼ਾਰਸ਼ ਕੀਤੀ ਜਾਂਦੀ ਹੈ, ਕਿਉਂਕਿ ਇਹ ਆਪਣੇ ਆਪ ਬੁਨਿਆਦੀ ਬਣਤਰ ਤਿਆਰ ਕਰਦੀ ਹੈ।

### ਖਾਲੀ ਫਾਈਲ ਬਣਾਉਣਾ

Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਖੇਤਰ ਵਿੱਚ, ਖਾਲੀ ਥਾਂ ਜਾਂ ਫੋਲਡਰ 'ਤੇ ਰਾਈਟ-ਕਲਿੱਕ ਕਰੋ, ਫਿਰ `New Empty File` ਚੁਣੋ।

ਕਾਰਵਾਈ ਦੀ ਥਾਂ ਹੇਠਾਂ ਦਿੱਤੀ ਹੈ:

![xcode](../../../Resource/027_view1.png)

ਖਾਲੀ ਫਾਈਲ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, Xcode ਆਮ ਤੌਰ 'ਤੇ ਇੱਕ ਬਿਨਾਂ ਨਾਮ ਵਾਲੀ Swift ਫਾਈਲ ਬਣਾਂਦਾ ਹੈ, ਜਿਵੇਂ `Untitled.swift`।

ਅਸੀਂ ਇਸਦਾ ਨਾਮ `TestView.swift` ਰੱਖ ਸਕਦੇ ਹਾਂ।

SwiftUI ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ, ਜੇ ਇਹ ਫਾਈਲ ਇੱਕ ਵਿਊ ਦਿਖਾਉਣ ਲਈ ਵਰਤੀ ਜਾਂਦੀ ਹੈ, ਤਾਂ ਆਮ ਤੌਰ 'ਤੇ ਨਾਮ ਦੇ ਅੰਤ ਵਿੱਚ `View` ਜੋੜਨ ਦੀ ਸਿਫ਼ਾਰਸ਼ ਕੀਤੀ ਜਾਂਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

ਇਹ ਕੋਈ ਲਾਜ਼ਮੀ ਨਿਯਮ ਨਹੀਂ ਹੈ, ਪਰ ਇਸ ਨਾਲ ਅਸੀਂ ਇੱਕ ਨਜ਼ਰ ਵਿੱਚ ਸਮਝ ਸਕਦੇ ਹਾਂ ਕਿ ਇਹ ਇੱਕ ਵਿਊ ਫਾਈਲ ਹੈ।

ਖਾਲੀ ਫਾਈਲ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, ਫਾਈਲ ਵਿੱਚ ਸਿਰਫ਼ ਡਿਫਾਲਟ ਟਿੱਪਣੀਆਂ ਹੋ ਸਕਦੀਆਂ ਹਨ, ਜਾਂ ਇਹ ਖਾਲੀ ਵੀ ਹੋ ਸਕਦੀ ਹੈ। ਇਹ Xcode ਦੇ ਸੰਸਕਰਣ ਅਤੇ ਬਣਾਉਣ ਦੇ ਤਰੀਕੇ 'ਤੇ ਨਿਰਭਰ ਕਰਦਾ ਹੈ।

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ਇਸ ਵੇਲੇ, ਸਾਨੂੰ SwiftUI ਸ਼ਿਫ਼ਰ ਹੱਥੋਂ ਲਿਖਣੀ ਪੈਂਦੀ ਹੈ:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

ਇਸ ਤਰ੍ਹਾਂ, ਇੱਕ ਸਧਾਰਨ SwiftUI ਵਿਊ ਫਾਈਲ ਬਣ ਜਾਂਦੀ ਹੈ।

### ਫਾਈਲ ਦਾ ਨਾਮ ਬਦਲਣਾ

ਜੇ ਫਾਈਲ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ ਤੁਸੀਂ ਫਾਈਲ ਦਾ ਨਾਮ ਬਦਲਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਖੇਤਰ ਵਿੱਚ ਫਾਈਲ ਚੁਣੋ, ਫਿਰ ਰਿਟਰਨ ਕੁੰਜੀ `⏎` ਦਬਾਓ।

![xcode](../../../Resource/027_view2.png)

ਇਸ ਤਰ੍ਹਾਂ ਫਾਈਲ ਦਾ ਨਾਮ ਬਦਲਿਆ ਜਾ ਸਕਦਾ ਹੈ।

ਧਿਆਨ ਦੇਣ ਵਾਲੀ ਗੱਲ ਇਹ ਹੈ ਕਿ ਫਾਈਲ ਦਾ ਨਾਮ ਬਦਲਣ ਨਾਲ ਸ਼ਿਫ਼ਰ ਵਿੱਚ ਵਿਊ ਦਾ ਨਾਮ ਆਪਣੇ ਆਪ ਨਹੀਂ ਬਦਲਦਾ।

ਉਦਾਹਰਨ ਲਈ, ਫਾਈਲ ਦਾ ਨਾਮ `ProfileView.swift` ਕਰਨ ਤੋਂ ਬਾਅਦ, ਸ਼ਿਫ਼ਰ ਵਿੱਚ ਅਜੇ ਵੀ `struct TestView: View` ਹੋ ਸਕਦਾ ਹੈ।

ਇਹ ਲਾਜ਼ਮੀ ਤੌਰ 'ਤੇ ਗਲਤੀ ਨਹੀਂ ਕਰਦਾ, ਪਰ ਲੋਕਾਂ ਨੂੰ ਗੁੰਝਲ ਵਿੱਚ ਪਾ ਸਕਦਾ ਹੈ।

ਇਸ ਲਈ, ਫਾਈਲ ਦੇ ਨਾਮ ਅਤੇ ਵਿਊ ਦੇ ਨਾਮ ਨੂੰ ਇੱਕੋ ਜਿਹਾ ਰੱਖਣ ਦੀ ਸਿਫ਼ਾਰਸ਼ ਕੀਤੀ ਜਾਂਦੀ ਹੈ। ਉਦਾਹਰਨ ਲਈ, ਜੇ ਫਾਈਲ ਦਾ ਨਾਮ `ProfileView.swift` ਹੈ, ਤਾਂ ਵਿਊ ਦਾ ਨਾਮ ਵੀ `ProfileView` ਵਰਤੋ।

## SwiftUI ਟੈਂਪਲੇਟ ਫਾਈਲ ਬਣਾਉਣਾ

ਖਾਲੀ ਫਾਈਲ ਬਣਾਉਣ ਤੋਂ ਇਲਾਵਾ, ਅਸੀਂ ਸਿੱਧੇ SwiftUI ਟੈਂਪਲੇਟ ਫਾਈਲ ਵੀ ਬਣਾ ਸਕਦੇ ਹਾਂ।

Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਖੇਤਰ ਵਿੱਚ, ਖਾਲੀ ਥਾਂ ਜਾਂ ਫੋਲਡਰ 'ਤੇ ਰਾਈਟ-ਕਲਿੱਕ ਕਰੋ, ਫਿਰ `New File from Template...` ਚੁਣੋ।

ਕਾਰਵਾਈ ਦੀ ਥਾਂ ਹੇਠਾਂ ਦਿੱਤੀ ਹੈ:

![xcode](../../../Resource/027_view3.png)

ਖੁੱਲ੍ਹਣ ਵਾਲੀ ਵਿੰਡੋ ਵਿੱਚ, ਉੱਪਰ ਵੱਖ-ਵੱਖ ਪਲੇਟਫਾਰਮ ਚੁਣੇ ਜਾ ਸਕਦੇ ਹਨ, ਜਿਵੇਂ `iOS`、`macOS` ਆਦਿ।

ਇੱਥੇ ਮੌਜੂਦਾ ਪ੍ਰੋਜੈਕਟ ਨਾਲ ਸੰਬੰਧਿਤ ਪਲੇਟਫਾਰਮ ਹੀ ਚੁਣਿਆ ਰੱਖੋ। ਉਦਾਹਰਨ ਲਈ, ਜੇ ਅਸੀਂ iOS App ਬਣਾ ਰਹੇ ਹਾਂ, ਤਾਂ `iOS` ਚੁਣੋ।

![xcode](../../../Resource/027_view4.png)

ਫਿਰ ਹੇਠਾਂ `SwiftUI View` ਚੁਣੋ।

![xcode](../../../Resource/027_view12.png)

ਫਿਰ `Next` 'ਤੇ ਕਲਿੱਕ ਕਰੋ, ਫਾਈਲ ਨਾਮ ਦਰਜ ਕਰੋ ਅਤੇ ਸੰਭਾਲੋ।

ਉਦਾਹਰਨ ਲਈ, ਅਸੀਂ `Test2View.swift` ਬਣਾਂਦੇ ਹਾਂ।

Xcode ਆਪਣੇ ਆਪ ਹੇਠਾਂ ਦਿੱਤੀ ਸ਼ਿਫ਼ਰ ਵਰਗੀ ਸ਼ਿਫ਼ਰ ਤਿਆਰ ਕਰੇਗਾ:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

ਖਾਲੀ ਫਾਈਲ ਨਾਲ ਤੁਲਨਾ ਕਰੀਏ ਤਾਂ, ਟੈਂਪਲੇਟ ਫਾਈਲ ਵਿੱਚ ਆਪਣੇ ਆਪ `import SwiftUI`、`struct`、`body` ਅਤੇ `#Preview` ਸ਼ਾਮਲ ਹੁੰਦੇ ਹਨ।

ਇਸ ਲਈ ਸ਼ੁਰੂਆਤੀ ਸਿੱਖਣ ਵਾਲਿਆਂ ਲਈ, SwiftUI ਟੈਂਪਲੇਟ ਫਾਈਲ ਵਰਤਣਾ ਹੋਰ ਆਸਾਨ ਹੈ, ਅਤੇ ਸ਼ਿਫ਼ਰ ਛੱਡ ਜਾਣ ਦੀ ਸੰਭਾਵਨਾ ਵੀ ਘੱਟ ਹੁੰਦੀ ਹੈ।

**ਸੁਝਾਅ**: ਰਾਈਟ-ਕਲਿੱਕ ਕਰਕੇ ਫਾਈਲ ਬਣਾਉਣ ਤੋਂ ਇਲਾਵਾ, ਤੁਸੀਂ Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਦੇ ਹੇਠਾਂ ਪਲੱਸ ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰਕੇ ਵੀ ਫਾਈਲ ਜਾਂ ਫੋਲਡਰ ਬਣਾ ਸਕਦੇ ਹੋ।

![xcode](../../../Resource/027_view5.png)

## ਫੋਲਡਰ ਬਣਾਉਣਾ

ਜਦੋਂ ਫਾਈਲਾਂ ਵਧ ਜਾਂਦੀਆਂ ਹਨ, ਤਾਂ ਅਸੀਂ ਪ੍ਰਬੰਧਨ ਲਈ ਫੋਲਡਰ ਵਰਤ ਸਕਦੇ ਹਾਂ।

Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਖੇਤਰ ਵਿੱਚ, ਖਾਲੀ ਥਾਂ ਜਾਂ ਫੋਲਡਰ 'ਤੇ ਰਾਈਟ-ਕਲਿੱਕ ਕਰੋ, ਫਿਰ `New Folder` ਚੁਣੋ।

ਫੋਲਡਰ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, ਸੰਬੰਧਿਤ ਫਾਈਲਾਂ ਨੂੰ ਫੋਲਡਰ ਵਿੱਚ ਖਿੱਚ ਕੇ ਰੱਖਿਆ ਜਾ ਸਕਦਾ ਹੈ।

![xcode](../../../Resource/027_view7.png)

ਫੋਲਡਰ ਮੁੱਖ ਤੌਰ 'ਤੇ ਪ੍ਰੋਜੈਕਟ ਦੀ ਬਣਤਰ ਸੰਗਠਿਤ ਕਰਨ ਲਈ ਵਰਤੇ ਜਾਂਦੇ ਹਨ, ਅਤੇ ਇਹ ਵਿਊ ਦੇ ਆਪਣੇ ਵਰਤੋਂ ਦੇ ਤਰੀਕੇ ਨੂੰ ਨਹੀਂ ਬਦਲਦੇ।

ਜਦੋਂ ਤੱਕ ਵਿਊ ਦਾ ਨਾਮ ਸਹੀ ਹੈ, ਹੋਰ ਫਾਈਲਾਂ ਵਿੱਚ ਇਹ ਅਜੇ ਵੀ `TestView()`、`Test2View()` ਵਰਗੇ ਤਰੀਕੇ ਨਾਲ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ।

## ਫਾਈਲ ਜਾਂ ਫੋਲਡਰ ਮਿਟਾਉਣਾ

ਜੇ ਤੁਸੀਂ ਫਾਈਲ ਜਾਂ ਫੋਲਡਰ ਮਿਟਾਉਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ Xcode ਦੇ ਖੱਬੇ ਪਾਸੇ Navigator ਖੇਤਰ ਵਿੱਚ ਸੰਬੰਧਿਤ ਫਾਈਲ ਚੁਣੋ, ਫਿਰ ਰਾਈਟ-ਕਲਿੱਕ ਕਰਕੇ `Delete` ਚੁਣੋ।

ਤੁਸੀਂ ਫਾਈਲ ਚੁਣ ਕੇ ਡਿਲੀਟ ਕੁੰਜੀ `⌫` ਵੀ ਦਬਾ ਸਕਦੇ ਹੋ।

Xcode ਮਿਟਾਉਣ ਦੀ ਪੁਸ਼ਟੀ ਵਾਲੀ ਵਿੰਡੋ ਦਿਖਾਏਗਾ।

![xcode](../../../Resource/027_view10.png)

`Move to Trash` ਚੁਣਨ ਦਾ ਮਤਲਬ ਹੈ ਫਾਈਲ ਨੂੰ Trash ਵਿੱਚ ਭੇਜਣਾ।

ਜੇ ਤੁਸੀਂ `Remove Reference` ਵੇਖਦੇ ਹੋ, ਤਾਂ ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਸਿਰਫ਼ Xcode ਪ੍ਰੋਜੈਕਟ ਵਿੱਚੋਂ ਰੈਫ਼ਰੈਂਸ ਹਟਾਇਆ ਜਾਵੇਗਾ, ਪਰ ਡਿਸਕ ਉੱਤੇ ਮੂਲ ਫਾਈਲ ਨਹੀਂ ਮਿਟੇਗੀ।

ਸ਼ੁਰੂਆਤੀ ਸਿੱਖਣ ਵਾਲਿਆਂ ਲਈ, ਜੇ ਤੁਸੀਂ ਸਿਰਫ਼ ਆਪਣੀ ਹੁਣੇ ਬਣਾਈ ਆਮ ਫਾਈਲ ਮਿਟਾ ਰਹੇ ਹੋ, ਤਾਂ ਆਮ ਤੌਰ 'ਤੇ `Move to Trash` ਚੁਣਨਾ ਕਾਫ਼ੀ ਹੈ।

## ਕਈ SwiftUI ਵਿਊਜ਼ ਦਿਖਾਉਣਾ

ਫਾਈਲ ਬਣਾਉਣਾ ਸਿੱਖਣ ਤੋਂ ਬਾਅਦ, ਅਸੀਂ ਵੱਖ-ਵੱਖ ਵਿਊਜ਼ ਨੂੰ ਵੱਖ-ਵੱਖ ਫਾਈਲਾਂ ਵਿੱਚ ਵੰਡ ਸਕਦੇ ਹਾਂ।

ਉਦਾਹਰਨ ਲਈ, ਅਸੀਂ ਦੋ SwiftUI ਫਾਈਲਾਂ ਬਣਾਂਦੇ ਹਾਂ:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` ਲੇਖਕ ਦਾ ਨਾਮ ਦਿਖਾਉਣ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` ਕਾਰਡ ਦਾ ਪਿਛੋਕੜ ਦਿਖਾਉਣ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

ਫਿਰ, ਅਸੀਂ `ContentView` ਵਿੱਚ ਇਹ ਦੋ ਵਿਊਜ਼ ਵਰਤ ਸਕਦੇ ਹਾਂ:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

ਰਨ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਵੇਖ ਸਕਦੇ ਹੋ ਕਿ `CardView` ਅਤੇ `AuthorView` ਇਕੱਠੇ ਮਿਲਾ ਕੇ ਦਿਖਾਏ ਗਏ ਹਨ:

![xcode](../../../Resource/027_view11.png)

ਇਸ ਪ੍ਰਭਾਵ ਵਿੱਚ, `CardView` ਪਿਛੋਕੜ ਵਜੋਂ ਕੰਮ ਕਰਦਾ ਹੈ, ਅਤੇ `AuthorView` ਉੱਪਰ ਦਿਖਾਈ ਦਿੰਦਾ ਹੈ, ਕਿਉਂਕਿ ਉਹਨਾਂ ਨੂੰ `ZStack` ਵਿੱਚ ਰੱਖਿਆ ਗਿਆ ਹੈ।

`ContentView` ਵਿੱਚ, ਅਸੀਂ `CardView()` ਅਤੇ `AuthorView()` ਰਾਹੀਂ ਇਹ ਦੋ ਵਿਊਜ਼ ਵਰਤਦੇ ਹਾਂ। ਨਾਮ ਦੇ ਪਿੱਛੇ `()` ਦਾ ਅਰਥ ਹੈ ਇਸ ਵਿਊ ਨੂੰ ਬਣਾਉਣਾ ਅਤੇ ਵਰਤਣਾ।

ਇਹ SwiftUI ਵਿੱਚ ਆਮ ਵਿਊ ਨੇਸਟਿੰਗ ਹੈ: ਇੱਕ ਵਿਊ ਦੇ ਅੰਦਰ ਹੋਰ ਵਿਊਜ਼ ਵਰਤੇ ਜਾ ਸਕਦੇ ਹਨ।

## ਐਂਟਰੀ ਫਾਈਲ

ਅੰਤ ਵਿੱਚ, ਅਸੀਂ ਮੁੜ ਵੇਖਦੇ ਹਾਂ ਕਿ App ਇੰਟਰਫੇਸ ਦਿਖਾਉਣਾ ਕਿੱਥੋਂ ਸ਼ੁਰੂ ਕਰਦਾ ਹੈ।

SwiftUI ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ, ਆਮ ਤੌਰ 'ਤੇ ਇੱਕ ਐਂਟਰੀ ਫਾਈਲ ਹੁੰਦੀ ਹੈ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ਇੱਥੇ `@main` ਦਰਸਾਂਦਾ ਹੈ ਕਿ ਇਹ App ਦੀ ਐਂਟਰੀ ਥਾਂ ਹੈ।

`WindowGroup` ਵਿੱਚ, ਅਸੀਂ ਵੇਖਦੇ ਹਾਂ:

```swift
ContentView()
```

ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ App ਸ਼ੁਰੂ ਹੋਣ ਤੋਂ ਬਾਅਦ ਸਭ ਤੋਂ ਪਹਿਲਾਂ `ContentView` ਦਿਖਾਇਆ ਜਾਂਦਾ ਹੈ।

ਫਿਰ `ContentView` ਦੇ ਅੰਦਰ ਇਹ ਵੀ ਦਿਖਾਇਆ ਜਾਂਦਾ ਹੈ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ਇਸ ਲਈ ਵਿਊ ਲੇਅਰ ਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਸਮਝਿਆ ਜਾ ਸਕਦਾ ਹੈ:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

ਜੇ ਅਸੀਂ ਐਂਟਰੀ ਫਾਈਲ ਵਿੱਚ ਦਿਖਾਏ ਜਾਣ ਵਾਲੇ ਵਿਊ ਨੂੰ `CardView` ਵਿੱਚ ਬਦਲ ਦਈਏ:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

ਤਾਂ App ਸ਼ੁਰੂ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਸਿਰਫ਼ `CardView` ਹੀ ਦਿਖਾਈ ਦੇਵੇਗਾ।

ਅਰਥਾਤ, ਐਂਟਰੀ ਸ਼ਿਫ਼ਰ ਵਿੱਚ `WindowGroup` ਇਹ ਨਿਰਧਾਰਤ ਕਰਦਾ ਹੈ ਕਿ App ਸ਼ੁਰੂ ਹੋਣ ਤੋਂ ਬਾਅਦ ਸਭ ਤੋਂ ਪਹਿਲਾਂ ਕਿਹੜਾ ਵਿਊ ਦਿਖੇਗਾ।

ਅਤੇ ਇਸ ਵਿਊ ਦੇ ਅੰਦਰ, ਹੋਰ ਵਿਊਜ਼ ਵੀ ਅੱਗੇ ਦਿਖਾਏ ਜਾ ਸਕਦੇ ਹਨ।

## ਸੰਖੇਪ

ਇਸ ਪਾਠ ਵਿੱਚ, ਅਸੀਂ Xcode ਵਿੱਚ ਆਮ ਫਾਈਲ ਪ੍ਰਬੰਧਨ ਕਾਰਵਾਈਆਂ ਸਿੱਖੀਆਂ।

ਅਸੀਂ ਜਾਣਿਆ ਕਿ SwiftUI ਫਾਈਲਾਂ ਕਿਵੇਂ ਬਣਾਉਣੀਆਂ ਹਨ, ਫਾਈਲਾਂ ਦੇ ਨਾਮ ਕਿਵੇਂ ਬਦਲਣੇ ਹਨ, ਫੋਲਡਰ ਕਿਵੇਂ ਬਣਾਉਣੇ ਹਨ, ਅਤੇ ਫਾਈਲਾਂ ਕਿਵੇਂ ਮਿਟਾਉਣੀਆਂ ਹਨ।

ਇਸਦੇ ਨਾਲ ਹੀ, ਅਸੀਂ ਇੱਕ ਮਹੱਤਵਪੂਰਨ ਧਾਰਣਾ ਵੀ ਸਿੱਖੀ: SwiftUI ਵਿਊਜ਼ ਨੂੰ ਵੱਖ-ਵੱਖ ਫਾਈਲਾਂ ਵਿੱਚ ਵੰਡਿਆ ਜਾ ਸਕਦਾ ਹੈ, ਫਿਰ ਹੋਰ ਵਿਊਜ਼ ਵਿੱਚ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ।

ਉਦਾਹਰਨ ਲਈ:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ਇੱਥੇ `CardView()` ਅਤੇ `AuthorView()` ਦਾ ਮਤਲਬ ਹੈ ਕਿ `ContentView` ਵਿੱਚ ਹੋਰ ਵਿਊਜ਼ ਵਰਤੇ ਜਾ ਰਹੇ ਹਨ।

ਅੰਤ ਵਿੱਚ, ਅਸੀਂ ਐਂਟਰੀ ਫਾਈਲ ਦੀ ਦਿਖਾਵਟ ਲਾਜ਼ਿਕ ਵੀ ਮੁੜ ਵੇਖੀ।

App ਸ਼ੁਰੂ ਹੋਣ ਵੇਲੇ, ਇਹ ਪਹਿਲਾਂ `@main` ਨਿਸ਼ਾਨ ਵਾਲੀ ਐਂਟਰੀ ਸ਼ਿਫ਼ਰ ਵਿੱਚ ਦਾਖ਼ਲ ਹੁੰਦਾ ਹੈ, ਫਿਰ `WindowGroup` ਵਿੱਚ ਦਿੱਤੇ ਵਿਊ ਨੂੰ ਦਿਖਾਉਂਦਾ ਹੈ।

ਇਹ ਸਾਨੂੰ ਸਮਝਣ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ: ਵੱਖ-ਵੱਖ SwiftUI ਫਾਈਲਾਂ ਇਕੱਲੀਆਂ ਨਹੀਂ ਹੁੰਦੀਆਂ। ਉਹ ਇਕ-ਦੂਜੇ ਨਾਲ ਜੋੜੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ, ਅਤੇ ਆਖ਼ਰਕਾਰ ਇੱਕ ਪੂਰਾ App ਇੰਟਰਫੇਸ ਬਣਾਉਂਦੀਆਂ ਹਨ।
