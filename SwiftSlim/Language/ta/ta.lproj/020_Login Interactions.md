# உள்நுழைவு தொடர்புகள்

முந்தைய பாடத்தில், நாம் ஒரு உள்நுழைவு காட்சியை உருவாக்கி, பொத்தான்கள் மற்றும் நிபந்தனை தர்க்கத்தின் மூலம் உள்ளிடப்பட்ட உள்ளடக்கத்திற்கான எளிய சரிபார்ப்பை செயல்படுத்தினோம்.

இந்தப் பாடத்தில், முந்தைய பாடத்தின் குறியீட்டை அடிப்படையாகக் கொண்டு `SecureField` கடவுச்சொல் உள்ளீட்டு புலம், `Alert` எச்சரிக்கை பெட்டி, மேலும் மாறிகளின் அடிப்படையில் வெவ்வேறு காட்சிகளின் காட்சிப்படுத்தலை எவ்வாறு கட்டுப்படுத்துவது என்பதை கற்போம்.

இந்தப் பாடத்தின் மூலம், ஒரு உள்நுழைவு காட்சியின் அடிப்படை தொடர்பு ஓட்டத்தை நீங்கள் மேலும் நன்றாகப் புரிந்துகொள்ள முடியும். இது பல App-களில் பொதுவாக பயன்படுத்தப்படும் அடிப்படை தர்க்கமாகவும் உள்ளது.

## கடவுச்சொல் உள்ளீட்டு புலம்

முந்தைய பாடத்தில், கடவுச்சொல்லை உள்ளிட `TextField` பயன்படுத்தினோம்.

ஆனால் உண்மையான உள்நுழைவு இடைமுகத்தில், கடவுச்சொல் பொதுவாக நேரடியாக காட்டப்படாது. அதற்கு பதிலாக அது புள்ளிகளாக அல்லது வேறு மறைவு விளைவாக காட்டப்படும்; இது பயனரின் தனியுரிமையை பாதுகாக்க உதவுகிறது.

SwiftUI-இல், கடவுச்சொற்களை உள்ளிடுவதற்கு குறிப்பாக பயன்படுத்தப்படும் கட்டுப்பாடு `SecureField` ஆகும்.

உதாரணமாக:

```swift
SecureField("Password", text: $password)
```

இதன் எழுத்துமுறை `TextField`-ஐ மிகவும் ஒத்திருக்கும். இதற்கும் ஒரு குறிப்புத் தொகுப்பு உரையும், இணைக்கப்பட்ட ஒரு மாறியும் தேவைப்படும்.

இதை எளிமையாக இவ்வாறு புரிந்துகொள்ளலாம்: `SecureField` கூட ஒரு உள்ளீட்டு புலம்தான், ஆனால் அது உள்ளிடப்படும் உள்ளடக்கத்தை மறைத்து விடும்.

கடவுச்சொல்லை உள்ளிட முன்பு பயன்படுத்தப்பட்ட `TextField`-ஐ நேரடியாக `SecureField` ஆக மாற்றலாம்:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

பயனர் கடவுச்சொல்லை உள்ளிடும்போது, அதன் உள்ளடக்கம் இனி இடைமுகத்தில் நேரடியாக காட்டப்படாது.

காட்சித் தாக்கம்:

![password](../../Resource/020_password.png)

இப்போது நாம் பயனர்பெயர் மற்றும் கடவுச்சொல்லை உள்ளிட்டு, பின்னர் பொத்தானை அழுத்தி உள்ளீடு சரியானதா என்பதைத் தீர்மானிக்கலாம்.

### பின்னூட்டச் செய்திகள் காட்டுதல்

முன்பு, முடிவுகளை வெளியிட நாம் தொடர்ந்து `print`-ஐ பயன்படுத்தினோம்.

`print`-இன் உள்ளடக்கம் Console-இல் மட்டுமே தோன்றும். டெவலப்பர்கள் Xcode-இல் நிரலை இயக்கும்போது அதை காண முடியும், ஆனால் கைப்பேசியில் இருக்கும் சாதாரண பயனர்கள் இந்தச் செய்திகளைப் பார்க்க முடியாது.

அதாவது, நிரல் ஏற்கனவே சரிபார்ப்பை முடித்திருந்தாலும், உள்ளிடப்பட்ட உள்ளடக்கம் சரியா என்பதை பயனருக்கு இன்னும் தெரியாது.

இந்த நேரத்தில், செய்திகளை நேரடியாக இடைமுகத்தில் காட்டும் ஒரு வழி நமக்குத் தேவை.

## Alert எச்சரிக்கை பெட்டி

SwiftUI-இல், ஒரு செய்தி பெட்டியை பாப்-அப் செய்ய `Alert` பயன்படுத்தலாம்.

அடிப்படை பயன்பாடு:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

இந்தக் குறியீடு முந்தைய கட்டுப்பாடுகளை விட சற்றுக் கடினமாகத் தெரிந்தாலும், அதை பகுத்து பார்த்தால் எளிதாகப் புரிந்துகொள்ளலாம்.

`Alert` முக்கியமாக பின்வரும் பகுதிகளை கொண்டுள்ளது:

- `"Title"`: எச்சரிக்கை பெட்டியின் தலைப்பு
- `isPresented`: எச்சரிக்கை பெட்டி காட்டப்படுமா என்பதை கட்டுப்படுத்தும் இணைக்கப்பட்ட Boolean மதிப்பு
- `Button(...)`: எச்சரிக்கை பெட்டியில் உள்ள பொத்தான்
- `message:`: எச்சரிக்கை பெட்டியில் காட்டப்படும் செய்தி உள்ளடக்கம்

இதை எளிமையாக இவ்வாறு புரிந்துகொள்ளலாம்: இணைக்கப்பட்ட மாறி `true` ஆகும் போது, அமைப்பு இந்த எச்சரிக்கை பெட்டியை காட்டும்.

### Alert ஒரு மாற்றியமைப்பான்

முன்பு கற்ற பல மாற்றியமைப்பான்களைப் போலவே, `Alert`-மும் ஒரு குறிப்பிட்ட காட்சியின் பின்னால் சேர்க்கப்படுகிறது.

உதாரணமாக, இது ஒரு பொத்தானுக்குப் பிறகு சேர்க்கப்படலாம்:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

இது வெளிப்புற `VStack`-க்குப் பிறகும் சேர்க்கப்படலாம்:

```swift
VStack {
    // ...
}
.alert() {...}
```

அதாவது, `Alert` என்பது இடைமுகத்தில் தனித்து வைக்கப்படும் ஒரு சுயாதீன காட்சி அல்ல. அதற்கு பதிலாக, அது ஒரு குறிப்பிட்ட காட்சியுடன் இணைக்கப்பட்ட மாற்றியமைப்பான் ஆகும்.

இதை எளிதாக இவ்வாறு புரிந்துகொள்ளலாம்: `Alert` பொதுவாக ஒரு பொத்தானுக்குப் பிறகோ அல்லது வெளிப்புற அடுக்கமைப்புக்குப் பிறகோ எழுதப்படும்; ஒரு குறிப்பிட்ட நிபந்தனை நிறைவேறும்போது ஒரு செய்தியை காட்ட பயன்படும்.

### Alert-ஐ காட்டுதல்

ஒரு `Alert`-ஐ காட்ட வேண்டுமெனில், அதை கட்டுப்படுத்த ஒரு மாறி தேவை.

உதாரணமாக:

```swift
@State private var showAlert = false
```

இந்த மாறி `Bool` வகையைச் சேர்ந்தது; அதாவது இதற்கு `true` அல்லது `false` என்ற இரண்டு மதிப்புகள் மட்டுமே உள்ளன.

பின், அதை `Alert`-இன் `isPresented`-க்கு இணைக்கவும்:

```swift
isPresented: $showAlert
```

`showAlert` என்பது `false` ஆக இருந்தால், எச்சரிக்கை பெட்டி காட்டப்படாது; `showAlert` என்பது `true` ஆக இருந்தால், எச்சரிக்கை பெட்டி தோன்றும்.

### பொத்தானின் மூலம் எச்சரிக்கை பெட்டியை காட்டுதல்

இப்போது முதலில் மிக எளிய நிலையை செயல்படுத்தலாம்:

பயனர் எதையும் உள்ளிடாமல் நேரடியாக `Sign in` பொத்தானை அழுத்தினால், இன்னும் எந்த உள்ளடக்கமும் உள்ளிடப்படவில்லை என்று தெரிவிக்கும் ஒரு செய்தி பெட்டியை காட்டுங்கள்.

முதலில், இந்தச் செய்தி பெட்டியை கட்டுப்படுத்த ஒரு மாறியை உருவாக்குங்கள்:

```swift
@State private var showEmpty = false
```

பிறகு, `Button`-க்குப் பிறகு ஒரு `Alert` சேர்க்கவும்:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

அடுத்து, பொத்தானை அழுத்தும் குறியீட்டில் சரிபார்ப்பைச் செய்யுங்கள்:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

`user` மற்றும் `password` இரண்டும் காலியாக இருந்தால், `showEmpty`-ஐ `true` ஆக அமைக்கவும்.

மேலும் `Alert` இந்த மாறியுடன் இணைக்கப்பட்டிருப்பதால், அது `true` ஆகும் போது, அமைப்பு தானாகவே எச்சரிக்கை பெட்டியை காட்டும்.

காட்சித் தாக்கம்:

![alert](../../Resource/020_alert1.png)

இவ்வாறு, பயனர் பொத்தானை அழுத்திய உடனே செய்தியைப் பார்க்க முடியும்; டெவலப்பருக்கு Console-இல் உள்ள `print` வெளியீட்டை பார்க்க வேண்டியதில்லை.

### பொத்தானையும் எச்சரிக்கை பெட்டிகளையும் மேம்படுத்துதல்

முன்பு, “எதுவும் நிரப்பப்படாத” நிலையை மட்டுமே கையாள்ந்தோம்.

இன்னும் பின்வரும் நிலைகளை கையாள வேண்டும்:

- பயனர்பெயர் இல்லை
- கடவுச்சொல் இல்லை
- பயனர்பெயரும் கடவுச்சொலும் நிரப்பப்பட்டுள்ளன, ஆனால் உள்ளீடு தவறானது

எனவே, இந்த நிலைகளுக்குத் தொடர்புடைய செய்திகளையும் தயார் செய்ய வேண்டும்.

முதலில், மூன்று புதிய மாறிகளைச் சேர்க்கவும்:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

பின்னர் பொத்தானுக்குப் பிறகு தொடர்புடைய `Alert`-களை தொடர்ந்து சேர்க்கவும்:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

அடுத்து, பொத்தானின் உள்ளேயுள்ள முடிவு தர்க்கத்தை முழுமையாக்கவும்:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

பயனர் எந்தத் தகவலையும் நிரப்பவில்லை, ஒரு தகவலை விடுத்துவிட்டார், அல்லது உள்ளீட்டு சரிபார்ப்பு தோல்வியடைந்தது என்பதை பொத்தான் தீர்மானிக்கும் போது, அதனுடன் தொடர்புடைய Boolean மாறி `true` ஆக மாறும்; பின்னர் அதற்கான `Alert` தோன்றும்.

இங்கே, நீங்கள் சில நிலைகளை நீங்களே சோதிக்க பரிந்துரைக்கிறேன்:

- எதையும் நிரப்ப வேண்டாம்
- பயனர்பெயரை மட்டும் நிரப்பவும்
- கடவுச்சொல்லை மட்டும் நிரப்பவும்
- பயனர்பெயரும் கடவுச்சொலும் நிரப்பவும், ஆனால் உள்ளடக்கம் தவறாக இருக்கட்டும்
- பயனர்பெயரும் கடவுச்சொலும் நிரப்பவும், உள்ளடக்கம் சரியாக இருக்கட்டும்

இது நமது குறியீட்டில் ஏதேனும் சிக்கல்கள் உள்ளனவா என்பதை சரிபார்க்க உதவுகிறது. இது App உருவாக்கத்தில் மிகவும் முக்கியமான “சோதனை” கட்டமாகவும் உள்ளது.

ஒரு குறிப்பிட்ட நிலையில் நிரல் தவறாக நடந்து கொண்டால், தர்க்கத்தில் சிக்கல் இருக்கலாம், மேலும் நிரலில் ஒரு `Bug` தோன்றியுள்ளது என்பதைக் குறிக்கும்.

உருவாக்கத்தில் பொதுவாக `Bug` என்று கூறப்படுவது, ஒரு பிழை, குறைபாடு அல்லது எதிர்பார்ப்புக்கு பொருந்தாத நடத்தை என்பதைக் குறிக்கும்.

## உள்நுழைவு காட்சியை முடித்தல்

முன்பு நாம் பேசிய `Alert` உதாரணங்கள் அனைத்தும் “உள்ளீடு தவறாக இருக்கும்போது” பயனருக்கு எப்படி பின்னூட்டம் வழங்குவது என்பதைக் குறித்தவையே.

பயனர் சரியான பயனர்பெயரும் கடவுச்சொல்லையும் உள்ளிடும்போது, நாம் பொதுவாக ஒரு “எச்சரிக்கை பெட்டி”யை காட்ட மாட்டோம். அதற்கு பதிலாக, உள்நுழைவுக்குப் பிந்தைய இடைமுகத்திற்கே நேரடியாக செல்வோம்.

இது ஒரு மிக முக்கியமான கருத்தை உள்ளடக்குகிறது: **மாறியின் மதிப்பின் அடிப்படையில் வெவ்வேறு காட்சிகளை காட்டுவது.**

உதாரணமாக:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

இங்கே, எந்த காட்சி காட்டப்பட வேண்டும் என்பதை நிர்வகிக்க `Bool` வகையைப் பயன்படுத்துகிறோம்:

- `login` என்பது `true` ஆக இருந்தால், உள்நுழைவு ஏற்கனவே வெற்றியடைந்தது என்று பொருள்; உள்நுழைவுக்குப் பிந்தைய உள்ளடக்கம் காட்டப்படும்
- `login` என்பது `false` ஆக இருந்தால், உள்நுழைவு வெற்றியடையவில்லை என்று பொருள்; உள்நுழைவு இடைமுகம் தொடர்ந்து காட்டப்படும்.

`if` கூற்று Swift-இல் ஒரு நிபந்தனை கூற்றாக இருந்தாலும், SwiftUI-யின் `body`-க்குள் அது காட்சிகளின் காட்சித்தர்க்கத்தை கட்டுப்படுத்த பயன்படுத்தப்படலாம்.

அதாவது, நிபந்தனையின் முடிவின் அடிப்படையில் தற்போது எந்த காட்சி காட்டப்பட வேண்டும் என்பதை SwiftUI தீர்மானிக்கிறது.

இந்தக் காட்சித் தர்க்கத்தை தற்போதைய `ContentView`-இல் வைக்கலாம்:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// அசல் உள்நுழைவு காட்சியின் குறியீடு
		}
	}
}
```

இங்கே, பயனர் ஏற்கனவே வெற்றிகரமாக உள்நுழைந்தாரா என்பதைச் சுட்டிக்காட்ட `login` என்ற புதிய மாறியைச் சேர்த்துள்ளோம்.

பின்னர், முன்பே எழுதியிருந்த உள்நுழைவு காட்சிக் குறியீட்டை `else`-க்குப் பிறகு வைத்தோம்.

காட்சி காட்சிப்படுத்தல் தர்க்கம்:

- `login` என்பது `false` ஆக இருந்தால், உள்நுழைவு இன்னும் வெற்றியடையவில்லை என்று பொருள்; ஆகவே அசல் உள்நுழைவு காட்சி தொடர்ந்து காட்டப்படும்.
- `login` என்பது `true` ஆக இருந்தால், உள்நுழைவு வெற்றியடைந்தது என்று பொருள். அப்போது உள்நுழைவு காட்சி இனி காட்டப்படாது; அதன் பதிலாக `Login successful` காட்டப்படும்.

அதாவது, `ContentView` என்பது `login`-ன் மதிப்பைப் பொறுத்து வெவ்வேறு காட்சி உள்ளடக்கங்களை காட்டும்.

அடுத்து, பொத்தானின் உள்ளேயுள்ள வெற்றிகரமான உள்நுழைவு தர்க்கத்தை மாற்றுவோம்:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

பயனர் உள்ளிட்ட பயனர்பெயரும் கடவுச்சொலும் இரண்டும் சரியாக இருக்கும்போது, `login`-ஐ `true` ஆக அமைக்கிறோம்.

`login`-ன் மதிப்பு மாறும் உடனே, `ContentView`-இல் உள்ள `if` சரிபார்ப்பின் முடிவும் மாறும்; இடைமுகம் அசல் உள்நுழைவு காட்சியிலிருந்து `Login successful`-க்கு மாறும்.

காட்சித் தாக்கம் பின்வருமாறு:

![view](../../Resource/020_view.png)

இப்போது “நிபந்தனைகளின் அடிப்படையில் வெவ்வேறு காட்சிகளை காட்டுவது” என்ற கருத்தை இந்த உள்நுழைவு பக்கத்தில் நாங்கள் உண்மையாகப் பயன்படுத்தியுள்ளோம்.

உண்மையான App-களிலும் இது பொதுவாக இப்படித்தான் செயல்படும்: ஒரு குறிப்பிட்ட மதிப்பின் அடிப்படையில் வெவ்வேறு காட்சிகள் காட்டப்படும்.

உதாரணமாக, பயனர் உள்நுழையாதபோது உள்நுழைவு பக்கம் காட்டப்படும்; பயனர் வெற்றிகரமாக உள்நுழைந்த பிறகு வேறு ஒரு இடைமுகம் காட்டப்படும்.

முக்கியக் கருத்து இதுதான்: **இடைமுகம் எந்த உள்ளடக்கத்தை காட்ட வேண்டும் என்பதை மாறிகளின் மாற்றங்கள் மூலம் கட்டுப்படுத்துவது.**

## சுருக்கம்

இந்தப் பாடத்தில், நாம் மூன்று முக்கியமான விஷயங்களை கற்றுக்கொண்டோம்:

1. கடவுச்சொற்களை உள்ளிட `SecureField` பயன்படுத்துவது; இதனால் கடவுச்சொல் உள்ளடக்கம் இடைமுகத்தில் நேரடியாக காட்டப்படாது.
2. பயனருக்கு அறிவுறுத்தல் செய்திகளை Console-இல் மட்டும் காட்டாமல் `Alert` மூலம் காட்டுவது.
3. வெவ்வேறு காட்சிகளின் காட்சிப்படுத்தலை கட்டுப்படுத்த மாறிகள் மற்றும் `if` நிபந்தனைகளைப் பயன்படுத்துவது; இதன் மூலம் உள்நுழைவுக்கு முன்பும் பின்பும் இருக்கும் இடைமுகங்களுக்கு இடையில் மாற முடியும்.

இந்த அறிவுகளின் மூலம், அடிப்படை தொடர்பு விளைவுகளுடன் கூடிய ஒரு உள்நுழைவு காட்சியை இப்போது நாமே முடிக்க முடிகிறது; மேலும் SwiftUI-யின் “நிலை மாறும்போது, இடைமுகம் புதுப்பிக்கப்படுகிறது” என்ற அபிவிருத்தி சிந்தனையையும் நன்றாகப் புரிந்துகொள்ள முடிகிறது.

## முழு குறியீடு

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## விரிவாக்க அறிவு - பல பொத்தான்கள் கொண்ட Alert

இந்தப் பாடத்தின் முந்தைய பகுதியில், ஒரு பொத்தானை மட்டுமே கொண்ட மிக அடிப்படையான `Alert`-ஐ பயன்படுத்தினோம்; இது முக்கியமாக பயனருக்கு அறிவுறுத்துவதற்காக பயன்படுத்தப்படுகிறது.

உண்மையில், `Alert` பல பொத்தான்களையும் கொண்டிருக்க முடியும்; இதனால் பயனர் வெவ்வேறு தேர்வுகளைச் செய்யலாம்.

உதாரணமாக:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

பொத்தானை அழுத்திய பிறகு, `Ok`, `delete`, மற்றும் `cancel` என்ற மூன்று தேர்வுகளைக் கொண்ட ஒரு அறிவுறுத்தல் பெட்டி தோன்றும்.

காட்சித் தாக்கம்:

![alert](../../Resource/020_alert2.png)

வெவ்வேறு பொத்தான்களை அழுத்தும் போது, அதற்கான குறியீடு செயல்படும்.

### Button-ன் `role` பண்பு

இங்கே, `Alert` உள்ளேயுள்ள `Button`, நாம் முன்பு கற்ற சாதாரண பொத்தான் எழுத்துமுறையை விட சற்று வேறுபட்டிருப்பதை நீங்கள் கவனிப்பீர்கள்:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

இங்கே ஒரு கூடுதல் `role` அளவுரு உள்ளது.

SwiftUI-இல், இந்த பொத்தான் எந்த வகை செயலைக் குறிக்கிறது என்பதை அமைப்பிற்கு தெரிவிக்க `role` பயன்படுத்தப்படுகிறது.

பொதுவான சில பங்குகள்:

- `.cancel`: ரத்து செயல்
- `.destructive`: நீக்குதல் போன்ற ஆபத்தான செயல்
- `nil`: சாதாரண செயல்

வெவ்வேறு `role` மதிப்புகள் பொதுவாக இடைமுகத்தில் வெவ்வேறு காட்சி வடிவங்களையும் உருவாக்கும்.

உதாரணமாக, ஆபத்தான செயல்கள் பொதுவாக அதிக கவனம் ஈர்க்கும் பாணியில் காட்டப்படும்; இது பயனர்கள் கவனமாக அழுத்த வேண்டும் என்பதை நினைவூட்டும்.

மேலும், `role` பொத்தானுக்குக் கூடுதல் அர்த்தத் தகவல்களையும் வழங்குகிறது; இதனால் Apple சாதனங்களில் உள்ள VoiceOver போன்ற திரைவாசிப்பிகள், பொத்தானின் நோக்கத்தை பயனருக்கு எளிதாக விளக்க முடியும்.

எனவே, பொருத்தமான சூழல்களில் பொத்தானுக்கு `role` சேர்ப்பது ஒரு நல்ல பழக்கமாகும்.
