# உள்நுழைவு காட்சி

இந்தப் பாடத்தில் நாம் மிகவும் பயனுள்ள ஒரு சூழலைக் கற்போம் — அதாவது உள்நுழைவு காட்சி.

இது ஒரு இணையதளம் ஆகட்டும் அல்லது App ஆகட்டும், பயன்பாட்டின் போது பல சமயங்களில் பயனர் கணக்கும் கடவுச்சொல்லும் உள்ளிட்டு உள்நுழைய வேண்டி இருக்கும்.

உதாரணமாக, GitHub உள்நுழைவு பக்கம்:

![WordPress](../../Resource/019_github.png)

இந்தப் பாடத்தில், பயனர் கையால் கணக்கும் கடவுச்சொல்லும் உள்ளிட்டு, உள்ளீட்டில் ஏதேனும் பிரச்சனை உள்ளதா என்பதைச் சரிபார்க்க முடியும் வகையில், இதற்கு ஒத்த ஒரு உள்நுழைவு காட்சியை உருவாக்கப் போகிறோம்.

## மேல் அமைப்பு

நாம் உள்நுழைவு காட்சிக்கான குறியீட்டை `ContentView` கோப்பில் எழுதுவோம்.

முதலில், உள்நுழைவு காட்சியின் மேல்பகுதியில் உள்ள அடையாளப் பகுதியை உருவாக்குவோம். முதலில் பொருத்தமான ஒரு ஐகான் படத்தைத் தயாரித்து, அதை `Assets` கோப்புறையில் வைக்கலாம்.

![icon](../../Resource/019_icon1.png)

பிறகு, படத்தை காட்ட `Image` மற்றும் modifier-களைப் பயன்படுத்துங்கள்:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

இங்கே உள்ள modifier-களின் அர்த்தம்:

- `resizable()`: படத்தின் அளவை மாற்ற அனுமதிக்கும்.
- `scaledToFit()`: படத்தின் அசல் விகிதத்தை வைத்தபடி அளவாற்றம் செய்கிறது.
- `frame(width: 100)`: படத்தின் காட்டப்படும் அகலத்தை `100` ஆக அமைக்கிறது.

அடுத்து, உள்நுழைவு தலைப்பைக் காட்ட `Text` மற்றும் modifier-களைப் பயன்படுத்துங்கள்:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

பிறகு, படம் மற்றும் உரையை மேல்-கீழ் அடுக்க `VStack`-ஐப் பயன்படுத்துங்கள்:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

காணும் விளைவு:

![view](../../Resource/019_view.png)

இங்கே வரையில், உள்நுழைவு காட்சியின் மேலிருக்கும் தலைப்பு பகுதியை நாங்கள் முடித்துவிட்டோம்.

### அமைப்பை மேம்படுத்துதல்

இப்போது படம் மற்றும் தலைப்பு முழு அமைப்பின் நடுப்பகுதியில் இயல்பாக இருக்கின்றன.

இவை “பக்கத்தின் மேல் பகுதியில் உள்ள உள்நுழைவு அடையாளம்” போலத் தோன்ற வேண்டும் எனில், மீதமுள்ள இடத்தை நிரப்ப `Spacer()`-ஐப் பயன்படுத்தி, உள்ளடக்கத்தை மேற்பகுதிக்குக் கொண்டு வரலாம்.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

காணும் விளைவு:

![view](../../Resource/019_view1.png)

ஆனால் இப்போது `Image` மற்றும் `Text` திரையின் மேல்பகுதிக்குப் மிக அருகில் இருப்பதால் கொஞ்சம் நெருக்கமாகத் தெரிகின்றன.

இந்த நேரத்தில், முழு `VStack`-க்கும் மேல்புற இடைவெளி சேர்க்க `padding`-ஐப் பயன்படுத்தலாம்.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 60)
    Text("Sign in to App")
        .font(.title)
        .fontWeight(.bold)
    Spacer()
}
.padding(.top, 50)
```

காணும் விளைவு:

![view](../../Resource/019_view2.png)

இவ்வாறு, பக்கத்தின் மேலிருக்கும் தலைப்பு பகுதி இன்னும் பொருத்தமாகத் தெரிகிறது.

## பயனர்பெயர் மற்றும் கடவுச்சொல்

உள்நுழைவு பக்கத்தில் பொதுவாக பயனர் பயனர்பெயரும் கடவுச்சொல்லும் உள்ளிட வேண்டும்.

SwiftUI-இல், பயனர் உள்ளிடும் உள்ளடக்கத்தைப் பெற `TextField`-ஐப் பயன்படுத்தலாம்.

ஆனால் கவனிக்க வேண்டியது: `TextField` தானாகவே நீண்டநேரம் உள்ளீட்டு தரவைச் சேமிக்காது; அது ஒரு உள்ளீட்டு கட்டுப்பாடு மட்டுமே. உண்மையில் இந்த உள்ளடக்கத்தைச் சேமிப்பது, அதனுடன் நாம் bind செய்யும் மாறிகள்தான்.

எனவே, முதலில் பயனர்பெயர் மற்றும் கடவுச்சொல்லைச் சேமிக்க இரண்டு `@State` மாறிகளை உருவாக்க வேண்டும்:

```swift
@State private var user = ""
@State private var password = ""
```

`@State` மாறியின் மதிப்பு மாறும்போது, SwiftUI தொடர்புடைய காட்சியை தானாகப் புதுப்பிக்கும்.

அடுத்து, இந்த இரண்டு மாறிகளையும் bind செய்ய `TextField`-ஐப் பயன்படுத்துங்கள்:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

இங்கே `$user` மற்றும் `$password` என்பவை “binding” என்பதைக் குறிக்கின்றன.

அதாவது, பயனர் உள்ளீட்டு பெட்டியில் உரையைத் টাইப் செய்யும் போது மாறியும் அதே சமயம் மாறும்; மாறி மாறும் போது பெட்டியில் காட்டப்படும் உள்ளடக்கமும் அதே சமயம் மாறும்.

“காட்சி மற்றும் தரவு ஒருவருக்கொருவர் ஒத்திசைவாக மாறுவது” என்ற இந்த உறவுதான் binding.

இங்கே `$` உடன் வரும் எழுதும் முறையை பயன்படுத்த வேண்டும் என்பதை கவனிக்கவும்:

```swift
$user
```

ஏனெனில் `TextField`-க்கு சாதாரண string அல்ல, “இரு திசைகளிலும் தரவை மாற்றக்கூடிய” binding value தேவை.

### உள்ளீட்டு பெட்டிகளை காட்டுதல்

அவற்றை `ContentView`-இல் வைப்போம்:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

காணும் விளைவு:

![view](../../Resource/019_view3.png)

ஏனெனில் இப்போது `user` மற்றும் `password` இரண்டும் இயல்பாக காலி string-களாக உள்ளன:

```swift
""
```

எனவே உள்ளீட்டு பெட்டி முதலில் placeholder உரையை காட்டும், எடுத்துக்காட்டாக:

```swift
input user
```

இந்த உரை பயனருக்கு “இங்கே என்ன உள்ளிட வேண்டும்” என்று மட்டும் சுட்டிக்காட்டும்; அது உண்மையான உள்ளீட்டு உள்ளடக்கம் அல்ல.

### உள்ளீட்டு பெட்டிகளை மேம்படுத்துதல்

இப்போது உள்ளீட்டு பெட்டிகளைப் பயன்படுத்தலாம், ஆனால் இயல்புநிலை style கொஞ்சம் எளிமையானதாக உள்ளது.

இடைமுகம் இன்னும் தெளிவாக இருக்க, உள்ளீட்டு பெட்டிக்கு முன் ஒரு தலைப்பைச் சேர்த்து, பெட்டிக்கும் சிறிய style மேம்பாடு செய்யலாம்.

எடுத்துக்காட்டாக, முதலில் ஒரு தலைப்பைச் சேர்க்கலாம்:

```swift
Text("Username")
    .fontWeight(.bold)
```

பிறகு, தலைப்பையும் உள்ளீட்டு பெட்டியையும் ஒரே வரியில் வைக்க `HStack`-ஐப் பயன்படுத்துங்கள்:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

இங்கே `HStack` பயன்படுத்தப்படுவதன் காரணம், “இடப்புறத்தில் தலைப்பு, வலப்புறத்தில் உள்ளீட்டு பெட்டி” என்பதைக் காட்ட வேண்டும் என்பதுதான்.

கவனிக்க வேண்டியது, `TextField` இயல்பாக மீதமுள்ள கிடைக்கும் இடத்தைப் பயன்படுத்தும்.

![color](../../Resource/019_view6.png)

இரண்டு உள்ளீட்டு பெட்டிகளின் அளவும் ஒரே மாதிரியாக இருக்க, `frame(width:)` பயன்படுத்தி அகலத்தை அமைக்கலாம்; இதனால் பெட்டிகள் இன்னும் ஒழுங்காகத் தோன்றும்.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

பிறகு, உள்ளீட்டு பெட்டி இன்னும் தெளிவாகத் தெரிய ஒரு border-ஐச் சேர்க்கலாம்:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

இப்போது பயனர்பெயரும் கடவுச்சொல்லும் இரண்டையும் `ContentView` காட்சியில் சேர்ப்போம்:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

காணும் விளைவு:

![view](../../Resource/019_view4.png)

இந்த நிலையில், பயனர் பயனர்பெயரும் கடவுச்சொல்லும் உள்ளிட முடியும்.

## உள்நுழைவு பொத்தான்

அடுத்ததாக, உள்ளீட்டு பெட்டிகளின் கீழே ஒரு உள்நுழைவு பொத்தானைச் சேர்ப்போம்.

```swift
Button("Sign in") {

}
```

பிறகு, பொத்தானுக்கு தெளிவான system style அளிக்க `buttonStyle`-ஐப் பயன்படுத்துங்கள்:

```swift
Button("Sign in") {

}
.buttonStyle(.borderedProminent)
```

காணும் விளைவு:

![view](../../Resource/019_view5.png)

பொத்தான் உண்மையில் click-ஐச் சமாளிக்கிறதா என்பதை உறுதிப்படுத்த, முதலில் பொத்தானுக்குள் `print` எழுதி சோதிக்கலாம்:

```swift
Button("Sign in") {
    print("click Sign in")
}
.buttonStyle(.borderedProminent)
```

பொத்தானை அழுத்தும் போது, console இதைப் பதிப்பிக்கும்:

```swift
click Sign in
```

இதனால், பொத்தான் click-க்கு வெற்றிகரமாகப் பதிலளிப்பது உறுதியாகிறது.

## உள்நுழைவு தர்க்கம்

இப்போது, உள்நுழைவு பொத்தானில் மிகவும் அடிப்படைமான சரிபார்ப்பு தர்க்கத்தைச் சேர்ப்போம்: **பயனர் பயனர்பெயரும் கடவுச்சொல்லும் உள்ளிட்டாரா என்பதைச் சரிபார்ப்பது**.

நாம் பின்வரும் நிலைகளை அடைய விரும்புகிறோம்:

- பயனர் எதையும் உள்ளிடவில்லை என்றால், `Empty` என்பதை வெளியிடவும்
- பயனர் இதில் ஒன்றை மட்டும் உள்ளிட்டிருந்தால், எது குறைவாக உள்ளது என்று தெரிவிக்கவும்
- பயனர்பெயரும் கடவுச்சொல்லும் இரண்டும் உள்ளிடப்பட்டிருந்தால், `Success` என்பதை வெளியிடவும்

### எந்த தகவலும் உள்ளிடப்படவில்லை

ஏனெனில் `user` மற்றும் `password` இரண்டும் இயல்பாக காலி string-கள்:

```swift
@State private var user = ""
@State private var password = ""
```

ஆகையால் பயனர் எதையும் உள்ளிடவில்லை என்றால், அவை காலியாகவே இருக்கும்.

Swift-இல், `String` போன்ற வகைகள் `isEmpty`-ஐப் பயன்படுத்தி உள்ளடக்கம் காலியா என்பதைச் சரிபார்க்கலாம்.

### isEmpty பண்பு

`isEmpty` பொதுவாக string, array போன்றவற்றின் உள்ளடக்கம் காலியா என்பதைச் சரிபார்க்கப் பயன்படுகிறது.

உதாரணமாக:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

உள்ளடக்கம் காலியாக இருந்தால் `isEmpty` `true`-ஐத் தரும்; காலியாக இல்லையெனில் `false`-ஐத் தரும்.

எனவே, பயனர்பெயர் மற்றும் கடவுச்சொல் உள்ளீடு உள்ளதா என்பதை அறிய இதைப் பயன்படுத்தலாம்.

### மாறிகளைச் சரிபார்க்க isEmpty பயன்படுத்துதல்

பயனர் எந்த தகவலும் உள்ளிடவில்லை என்றால்:

```swift
user.isEmpty // true
password.isEmpty    // true
```

அப்போது, நிபந்தனையை இவ்வாறு எழுதலாம்:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

இந்தக் குறியீட்டின் அர்த்தம்: `user` காலியாகவும் `password` கூட காலியாகவும் இருந்தால், இதை வெளியிடு:

```swift
Empty
```

இங்கே `&&` என்பது “மற்றும்” எனப் பொருள்படும் logical operator.

அதாவது, இடப்புற நிபந்தனையும் வலப்புற நிபந்தனையும் இரண்டும் உண்மையாக இருந்தால் மட்டுமே முழு நிபந்தனை உண்மையாகும்.

எனவே, பயனர்பெயரும் கடவுச்சொல்லும் இரண்டும் காலியாக இருக்கும்போது மட்டுமே இந்தக் குறியீடு இயக்கப்படும்.

## பயனர் ஒரு பகுதியை மட்டும் நிரப்பியுள்ளார்

அடுத்து, இன்னொரு நிலையைப் பார்ப்போம்: பயனர் பயனர்பெயரை மட்டும் அல்லது கடவுச்சொல்லை மட்டும் உள்ளிட்டுள்ளார்.

உதாரணமாக:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

இந்நேரத்தில் பயனர்பெயர் காலியாக இல்லை, ஆனால் கடவுச்சொல் காலியாக உள்ளது.

இதையும் `isEmpty`-ஐப் பயன்படுத்தி தொடர்ந்து சரிபார்க்கலாம்:

### பயனர் கணக்கு அல்லது கடவுச்சொல் உள்ளிட்ட நிலை

பயனர் பயனர்பெயரை மட்டும் அல்லது கடவுச்சொல்லை மட்டும் உள்ளிட்டிருந்தால், எது குறைவாக உள்ளது என்பதற்கான பொருத்தமான அறிவிப்பை அளிக்க வேண்டும்.

உதாரணமாக, பயனர் பயனர்பெயரை மட்டும் உள்ளிட்டிருந்தால்:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

இந்தக் குறியீட்டின் தர்க்கம்: பயனர்பெயர் காலியாக இருந்தால், பயனர்பெயரை உள்ளிடச் சொல்லுங்கள். இல்லையெனில், கடவுச்சொல் காலியா என்று சரிபார்க்கவும்; காலியாக இருந்தால், கடவுச்சொல்லை உள்ளிடச் சொல்லுங்கள்.

### பயனர் எல்லா தகவலையும் உள்ளிட்டுள்ளார்

பயனர்பெயரும் கடவுச்சொல்லும் இரண்டும் உள்ளிடப்பட்டிருந்தால், அவற்றின் `isEmpty` இரண்டுமே `false`-ஐத் தரும்.

அப்போது, இதை இவ்வாறு எழுதலாம்:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

இங்கே `!` என்பது “மறுப்பு” என்பதைக் குறிக்கும்.

உதாரணமாக:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

அதாவது, `user.isEmpty` என்பது “பயனர்பெயர் காலியா” என்பதைச் சரிபார்க்கப் பயன்படும்; `!user.isEmpty` என்பது “பயனர்பெயர் காலியாக இல்லையா” என்பதைச் சரிபார்க்கப் பயன்படும்.

கடவுச்சொல்லுக்கும் அதே தர்க்கமே பொருந்தும்.

எனவே, இந்தக் குறியீட்டின் பொருள்: பயனர்பெயர் காலியாக இல்லை, கடவுச்சொல்லும் காலியாக இல்லை.

இந்த இரண்டு நிபந்தனைகளும் பூர்த்தியானால், இது வெளியிடப்படும்:

```swift
Success
```

## முழுமையான சரிபார்ப்பு தர்க்கம்

இப்போது இந்த மூன்று நிலைகளையும் ஒன்றுசேர்த்து `Button`-இல் எழுதுவோம்:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

இந்தக் குறியீட்டின் செயலாக்க வரிசையை இவ்வாறு புரிந்துகொள்ளலாம்:

முதல் படி, முதலில் இதைப் பார்க்க வேண்டும்:

```swift
if user.isEmpty && password.isEmpty
```

பயனர்பெயரும் கடவுச்சொல்லும் இரண்டும் காலியாக இருந்தால், நேரடியாக `Empty` வெளியிடப்படும்.

இரண்டும் காலியாக இல்லையெனில், அடுத்த இந்த வரி சரிபார்க்கப்படும்:

```swift
else if user.isEmpty || password.isEmpty
```

இங்கே `||` என்பது “அல்லது” என்பதைக் குறிக்கிறது.

அதாவது, ஒரு நிபந்தனை உண்மையாக இருந்தாலே முழு நிபந்தனையும் உண்மையாகும்.

எனவே, பயனர்பெயர் காலியாக இருந்தாலும் அல்லது கடவுச்சொல் காலியாக இருந்தாலும் `else if`-இன் குறியீட்டு தொகுதி இயக்கப்படும்.

குறியீட்டு தொகுதிக்குள் சென்ற பிறகு, எந்த உள்ளீட்டு பெட்டி நிரப்பப்படவில்லை என்பதை மேலும் சரிபார்க்கிறோம்:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

முன்னைய இரண்டு நிபந்தனைகளும் பொருந்தவில்லை என்றால், பயனர்பெயர் காலியாக இல்லை, கடவுச்சொல்லும் காலியாக இல்லை என்பதைக் குறிக்கும்.

அப்போது program கடைசி `else`-க்குள் செல்கிறது:

```swift
print("Success")
```

இவ்வாறு, உள்நுழைவு உள்ளீட்டிற்கான மிக அடிப்படைமான சரிபார்ப்பு தர்க்கத்தை நாங்கள் முடித்துவிட்டோம்.

## பயனர்பெயர் மற்றும் கடவுச்சொல் சரிபார்ப்பு

மேலுள்ள தர்க்கம் “எதாவது உள்ளீடு செய்யப்பட்டதா” என்பதை மட்டும் சரிபார்க்கிறது.

ஆனால் உண்மையான App-இல், ஏதாவது உள்ளிடுவது மட்டுமே போதாது. பொதுவாக, பயனர்பெயர் மற்றும் கடவுச்சொல்லை server-க்கு அனுப்பி சரிபார்க்கவும் வேண்டும்.

Server பயனர்பெயர் இருப்பதையும் கடவுச்சொல் சரியானதையும் உறுதிப்படுத்தினால், உள்நுழைவு அனுமதிக்கப்படும்; இல்லையெனில் உள்நுழைவு தோல்வி என்று அறிவிக்கப்படும்.

இந்த செயல்முறையைப் பயிற்சி செய்ய, முதலில் குறியீட்டில் ஒரு தற்காலிக கணக்கும் கடவுச்சொல்லும் அமைத்து, “சரியான உள்நுழைவு தகவலை” மாதிரியாக்கலாம்:

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

பயனர்பெயரும் கடவுச்சொல்லும் இரண்டும் காலியாக இல்லாதபோது, அவை சரியா என்று அடுத்து ஒப்பிடலாம்:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

இங்கே `==` என்பது “சமமா” என்பதைக் குறிக்கிறது.

உதாரணமாக:

```swift
user == userID
```

அதாவது, பயனர் உள்ளிட்ட பயனர்பெயர், நாங்கள் அமைத்த சரியான பயனர்பெயருடன் முற்றிலும் பொருந்துகிறதா என்பதைச் சொல்கிறது; கடவுச்சொல்லுக்கும் இதே தர்க்கம் பொருந்தும்.

எனவே, உள்ளிட்ட பயனர்பெயரும் கடவுச்சொல்லும் அமைக்கப்பட்ட மதிப்புகளுடன் சமமாக இருந்தால் `Success` வெளியிடப்படும்; அவற்றில் ஏதேனும் ஒன்று சமமாக இல்லையெனில் `Input error` வெளியிடப்படும்.

இவ்வாறு, ஒரு எளிய “மாதிரி உள்நுழைவு சரிபார்ப்பை” நாங்கள் முடித்துவிட்டோம்.

இது இன்னும் server-ஓடு இணைக்கப்பட்ட உண்மையான உள்நுழைவு செயல்பாடு அல்ல என்றாலும், இது ஒரு முக்கியமான புரிதலைத் தருகிறது: **பொத்தானை அழுத்திய பின், வெவ்வேறு நிபந்தனைகளின் அடிப்படையில் வெவ்வேறு குறியீடுகள் இயக்கப்படலாம்**.

## சுருக்கம்

இந்தப் பாடத்தில், ஒரு அடிப்படை உள்நுழைவு காட்சியை நாம் முடித்தோம், மேலும் “நிபந்தனைகளின் அடிப்படையில் உள்ளீட்டைக் கணிக்கும் தர்க்கம்” பற்றி முக்கியமாகக் கற்றுக்கொண்டோம்.

இந்தப் பாடத்தின் மையக் கருத்து: string காலியா என்பதை அறிய `isEmpty`-ஐப் பயன்படுத்துவது, மேலும் வெவ்வேறு நிலைகளைக் கையாள `if` statement-ஐ logical operator-களுடன் சேர்த்து பயன்படுத்துவது.

இதில் மிகவும் முக்கியமான இரண்டு logical operator-கள்:

- `&&`: “மற்றும்” என்பதைக் குறிக்கும்; இரண்டு நிபந்தனைகளும் பூர்த்தியாக வேண்டும்
- `||`: “அல்லது” என்பதைக் குறிக்கும்; ஒரு நிபந்தனை பூர்த்தியானாலே போதும்

இந்த நிபந்தனைகள் இறுதியில் `true` அல்லது `false` என்ற மதிப்பைத் தரும் போது, `if` statement முடிவின் அடிப்படையில் வெவ்வேறு குறியீடுகளை இயக்கும்.

இந்த உள்நுழைவு காட்சியின் மூலம், “interface + data + logical judgment” ஆகிய மூன்றையும் இணைக்கும் ஒரு அபிவிருத்தி முறையை நாங்கள் ஏற்கனவே தொடக்க நிலையில் அறிந்து கொண்டோம்.

struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Button("Sign in") {
                if user.isEmpty && password.isEmpty {
                    print("Empty")
                } else if user.isEmpty || password.isEmpty {
                    if user.isEmpty {
                        print("Please enter your username.")
                    } else if password.isEmpty {
                        print("Please enter your password.")
                    }
                } else {
                    if user == userID && password == passWord {
                        print("Success")
                    } else {
                        print("Input error")
                    }
                }
            }
            .buttonStyle(.borderedProminent)
            Spacer()
        }
        .padding(.top, 50)
    }
}
