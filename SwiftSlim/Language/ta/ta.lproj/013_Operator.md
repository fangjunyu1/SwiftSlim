# ஆபரேட்டர்கள்

இந்தப் பாடத்தில், நாம் Swift ஆபரேட்டர்கள் (Operators) பற்றி கற்போம்.

கணக்கிடுதல், ஒப்பிடுதல், அல்லது தர்க்கரீதியான முடிவெடுத்தல் போன்ற செயல்களை செய்யும்போது, ஆபரேட்டர்கள் தேவைப்படும். இத்தகைய செயல்கள் நிரல் இயங்கும் முழு செயல்முறையிலும் இடம்பெறும்; ஆகவே, இவை நிரலாக்கத்தின் மிக அடிப்படையான திறன்களில் ஒன்றாகும்.

```swift
let a = 100 * 8
```

நிரலாக்கத்தை கற்றுக்கொள்ளும் போது, கணித சிந்தனைத் திறன் உள்ளவர்கள் நிரல் தர்க்கத்தை எளிதாகப் புரிந்துகொள்வார்கள் என்று பலர் உணர்கிறார்கள். காரணம், நிரலாக்கத்தின் அடிப்படை “கணக்கீடு” மற்றும் “விதிமுறை அடிப்படையிலான தீர்மானம்” ஆகும்; அவற்றைச் செயல்படுத்துவதில் ஆபரேட்டர்களே முக்கிய பங்கை வகிக்கின்றன.

நடைமுறை அபிவிருத்தியில், ஆபரேட்டர்கள் வெறும் எளிய எண் கணக்கீடுகளுக்கே மட்டும் பயன்படுத்தப்படுவதில்லை; பல குறிப்பிட்ட சூழல்களிலும் அவை பரவலாக பயன்படுகின்றன.

உதாரணமாக, ஒரு block game-இல், பயனர் ஒரு block-ஐ குறிப்பிட்ட இடத்துக்கு இழுத்துச் செல்லும்போது, கணக்கிட வேண்டியது:

* block-இன் coordinate இடம்
* block நிரப்பக்கூடிய இடங்கள்
* குறிக்கப்பட்ட இடம் பயன்படுத்தக்கூடியதா என்பதைத் தீர்மானித்தல்

இந்த “கணக்கீடு + தீர்மானம்” ஆகிய செயல்முறைகள் அனைத்தும் அடிப்படையில் ஆபரேட்டர்களைச் சார்ந்தவையே.

![Block](../../Resource/013_block.png)

ஆபரேட்டர்களைப் படிப்பது என்பது வெறும் எண்களை கணக்கிடுவது மட்டுமல்ல; பின்னர் வரும் layout அமைப்பு, data processing, மற்றும் interaction logic ஆகியவற்றிற்கும் இது அடித்தளமாக அமைகிறது.

## SwiftUI-இல் கணக்கீடு செயல்படும் முறை

SwiftUI-இல், பொதுவாக Button மூலம் ஒரு function-ஐத் தொடங்கி, அந்த function-க்குள் கணக்கீட்டு logic எழுதப்படும்.

உதாரணமாக:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

Button-ஐ அழுத்தும் போது, Console-இல் வெளியீடு:

```
c: 20
```

இந்தப் பாடத்தில், கணக்கீட்டு logic அனைத்தையும் ஒரே மாதிரியாக `func`-க்குள் எழுதுவோம்; பின்னர் Button மூலம் அதை இயக்கி, `print` பயன்படுத்தி முடிவை வெளியிடுவோம்.

பின்னர் வரும் எடுத்துக்காட்டுகளில், code-ஐ எளிமையாகக் காட்டுவதற்காக, மீண்டும் மீண்டும் `print` பயன்படுத்தாமல், கணக்கீட்டு முடிவுகளை நேரடியாக code-இல் குறிப்பிட்டு விடுவோம்.

## Console வெளியீட்டை மேலாண்மை செய்வது

ஒவ்வொரு click-க்கும் புதிய output உருவாகும். அதனால், Console பகுதியில் வலது கீழ் மூலையில் உள்ள clear button-ஐ அழுத்தி, பழைய logs-ஐ அகற்றி, தற்போதைய முடிவுகளை தெளிவாகப் பார்க்கலாம்.

![Console](../../Resource/013_console.png)

## Button-இன் மற்றொரு எழுதும் முறை

Button ஒரு function மட்டும் இயக்க வேண்டிய சூழலில், இன்னும் சுருக்கமான வடிவில் எழுதலாம்:

```swift
Button(action: ) {
    // View
}
```

இந்த வடிவில், `action` என்பது Button click ஆனபோது இயங்க வேண்டிய function-ஐ (parameter இல்லாத function) குறிக்கிறது; `{}` உள்ள பகுதி Button-இன் view-ஐ வரையறுக்கிறது.

முன்னர் பார்த்த `calculate` function-ஐ கொண்டு இதை இவ்வாறு எழுதலாம்:

```swift
Button(action: calculate) {
    Text("Button")
}
```

Button-ஐ அழுத்தும் போது, `calculate` function நேரடியாக இயக்கப்படும்.

அதனால், Button logic எளிமையாக இருந்து, ஒரு function-ஐ மட்டும் அழைக்கும் போது, `action` வடிவை பயன்படுத்துவது இன்னும் தெளிவாகவும், code நடைமுறைக்கும் ஏற்றதாகவும் இருக்கும்.

## கணித ஆபரேட்டர்கள்

கணித ஆபரேட்டர்கள் (Arithmetic Operators) எண் கணக்கீட்டிற்கு பயன்படுத்தப்படுகின்றன:

* `+` கூட்டல்
* `-` கழித்தல்
* `*` பெருக்கல்
* `/` வகுத்தல்
* `%` மீதி (remainder)

உதாரணம்:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

கவனிக்க வேண்டியது என்னவென்றால், Swift ஒரு strong type language; வேறு வேறு data type-களை தானாக கலந்து கணக்கிடாது:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

இரண்டு type-கள் வேறுபட்டால், type conversion-ஐ கையால் செய்ய வேண்டும்.

## ஒப்பீட்டு ஆபரேட்டர்கள்

ஒப்பீட்டு ஆபரேட்டர்கள் (Comparison Operators) இரண்டு மதிப்புகளுக்கிடையிலான உறவைச் சோதிக்கின்றன; அதன் முடிவு `Bool` ஆக இருக்கும்:

* `==` சமம்
* `!=` சமமல்ல
* `>` விட பெரியது
* `<` விட சிறியது
* `>=` பெரியது அல்லது சமம்
* `<=` சிறியது அல்லது சமம்

உதாரணம்:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

ஒப்பீட்டு ஆபரேட்டர்கள் பொதுவாக conditional statements-இல் பயன்படுத்தப்படுகின்றன. உதாரணமாக, “quote carousel” பகுதியில், index array வரம்பை மீறாமல் இருக்க நாம் கட்டுப்படுத்த வேண்டும்:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

`index < sayings.count - 1` என்ற ஒப்பீட்டின் முடிவு `true` என்றால், `if` உள்ள code இயங்கும்; `false` என்றால், அடுத்த code இயங்காது.

## தர்க்க ஆபரேட்டர்கள்

தர்க்க ஆபரேட்டர்கள் (Logical Operators) பல நிபந்தனைகளை ஒன்றாக இணைக்க பயன்படுகின்றன:

* `&&` மற்றும் (AND)
* `||` அல்லது (OR)
* `!` இல்லை / மறுப்பு (NOT)

### && மற்றும்

பல நிபந்தனைகள் அனைத்தும் ஒன்றாக பூர்த்தி செய்யப்பட வேண்டுமெனில், `&&` பயன்படுத்தப்படுகிறது.

உதாரணமாக:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

இந்த code-இல் இரண்டு நிபந்தனைகள் உள்ளன: `age >= 18` மற்றும் `age <= 100`।

இந்த இரண்டு நிபந்தனைகளும் `true` ஆக இருந்தால் மட்டுமே `legalAge` என்பது `true` ஆகும்; அவற்றில் ஏதேனும் ஒன்று தவறானால், முடிவு `false` ஆகும்.

### || அல்லது

பல நிபந்தனைகளில் ஏதேனும் ஒன்று மட்டுமே பூர்த்தி செய்யப்பட்டாலே போதுமான சூழலில், `||` பயன்படுத்தப்படுகிறது.

உதாரணமாக:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

இந்த code-ன் பொருள்: `money >= 50` அல்லது `creditCard >= 50` இதில் ஏதேனும் ஒன்று சரியாக இருந்தாலே, `payAllowed` என்பது `true` ஆகும்.

இரண்டு நிபந்தனைகளும் சரியில்லை என்றால் மட்டுமே முடிவு `false` ஆகும்.

உதாரணமாக, payment சூழலில், cash அல்லது credit card இதில் ஏதேனும் ஒன்றைப் பயன்படுத்தி கட்டணம் செலுத்தலாம். இரண்டிலும் முடியாவிட்டால், payment தோல்வியடையும்.

### ! இல்லை / மறுப்பு

`!` என்பது logical NOT operator; இது `Bool` மதிப்பைத் திருப்பிப் போட பயன்படுகிறது.

அதாவது, `true`-ஐ `false` ஆகவும், `false`-ஐ `true` ஆகவும் மாற்றும்.

உதாரணமாக:

```swift
let allowed = true
let result = !allowed   // false
```

இங்கு `!allowed` என்பது `allowed`-இன் மதிப்பை மறுப்பதைக் குறிக்கிறது.

கவனிக்க வேண்டியது: `!` மற்றும் `!=` இவை இரண்டு வேறு ஆபரேட்டர்கள். `!` என்பது `Bool` மதிப்பைத் திருப்பும்; `!=` என்பது ஒப்பீட்டு ஆபரேட்டர், அதன் முடிவும் `Bool` தான்.

## ஒதுக்கீட்டு ஆபரேட்டர்கள்

ஒதுக்கீட்டு ஆபரேட்டர்கள் (Assignment Operators) மதிப்பை ஒதுக்க அல்லது variable-ஐப் புதுப்பிக்கப் பயன்படுத்தப்படுகின்றன:

* `=` ஒதுக்கீடு

Variable declaration அல்லது கணக்கீட்டு செயல்முறைகளில் எல்லாம் assignment இடம்பெறும்:

```swift
let a = 5
```

இந்த code-ன் பொருள், `5` என்ற மதிப்பு `a` என்ற variable-க்கு ஒதுக்கப்படுகிறது.

### கூட்டு ஒதுக்கீட்டு ஆபரேட்டர்கள்

நடைமுறை அபிவிருத்தியில், அடிப்படை assignment operator மட்டும் அல்லாமல், “compound assignment operators” என்பவையும் மிகவும் பயன்படுத்தப்படுகின்றன. இவை, ஏற்கனவே உள்ள மதிப்பை அடிப்படையாகக் கொண்டு நேரடியாகக் கணக்கிட்டு, அதையே புதுப்பிக்கும்:

* `+=`
* `-=`
* `*=`
* `/=`

முன்னர் “counter” பாடத்தில், compound assignment operators குறித்து சுருக்கமாகக் கற்றிருந்தோம்.

உதாரணமாக, Button அழுத்தும் போது, ஒரு variable தானாக அதிகரிக்க வேண்டுமெனில்:

```swift
var num = 10
num += 5   // num = 15
```

இந்த code-ன் பொருள், `num` மற்றும் `5` கூட்டப்பட்டு, அதன் முடிவு மீண்டும் `num`-க்கு ஒதுக்கப்படுகிறது.

ஆகவே, இது பின்வருவதற்கு சமம்:

```swift
var num = 10
num = num + 5   // num = 15
```

## மும்மை ஆபரேட்டர்

Swift-இல் உள்ள ternary operator ஒரு condition-ஐச் சோதித்து, இரண்டு முடிவுகளில் ஒன்றைத் திருப்பித் தரும்:

```swift
நிபந்தனை ? மதிப்பு 1 : மதிப்பு 2
```

அதன் செயல்முறை: condition `true` என்றால் “值1” திரும்பும்; condition `false` என்றால் “值2” திரும்பும்.

உதாரணமாக:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

இந்த code, `showColor`-இன் மதிப்பைச் சோதிக்கிறது: அது `true` என்றால் `Color.blue` திரும்பும், பின்னணி நீலமாக இருக்கும்; `false` என்றால் `Color.clear` திரும்பும், அதாவது பின்னணி நிறம் காட்டப்படாது.

குறிப்பு: `Color.clear` என்பது transparent color-ஐக் குறிக்கிறது (எதுவும் தெரியாது).

### if-else உடன் உள்ள வித்தியாசம்

தர்க்க ரீதியாகப் பார்த்தால், ternary operator என்பது `if-else` statement-க்கு சமமானது.

உதாரணமாக, மேலுள்ள code-ஐ பின்வருமாறு எழுதலாம்:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

இரண்டின் செயல்முறையும் முழுமையாக ஒரே மாதிரியானது. condition `true` என்றால் முதல் முடிவு; `false` என்றால் இரண்டாம் முடிவு.

நடைமுறைப் பயன்பாட்டில், ternary operator இன்னும் சுருக்கமாக இருப்பதால், “இரண்டில் ஒன்றைத் தேர்வு செய்” என்ற எளிய சூழல்களில் இது மிகவும் வசதியாக இருக்கும். இது பொதுவாக நேரடியாக ஒரு மதிப்பைத் திருப்ப வேண்டிய இடங்களில் பயன்படுத்தப்படும்.

SwiftUI-இல் View கட்டமைப்பின்போது, ternary operator மிகவும் பொதுவாகக் காணப்படும். உதாரணமாக:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

இந்த வடிவில், ஒரு வரி code-இல் condition-ஐ அடிப்படையாகக் கொண்டு, view effect-ஐ dynamic-ஆக மாற்ற முடியும்.

## வரம்பு ஆபரேட்டர்கள்

வரம்பு ஆபரேட்டர்கள் (Range Operators) ஒரு interval அல்லது range-ஐக் குறிக்கப் பயன்படுத்தப்படுகின்றன:

* `...` மூடிய வரம்பு (இரு முனைகளும் உட்படும்)
* `..<` அரைத் திறந்த வரம்பு (வலது பக்கம் உட்படாது)

உதாரணமாக:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

இதனை இவ்வாறு புரிந்துகொள்ளலாம்: `1...3` என்பது 1 முதல் 3 வரை, 3-ஐச் சேர்த்துக் காட்டும்; `5..<8` என்பது 5 முதல் 8 வரை, ஆனால் 8 சேராது.

விரிவாக்க வடிவுகள் (இப்போது தெரிந்தால் போதும்):

```
5...    // >= 5
...5    // <= 5
```

இந்த வடிவுகள் “ஒரு பக்க வரம்பு” (one-sided range) என்பதைக் குறிக்கும்; இவை பொதுவாக condition check அல்லது pattern matching-இல் பயன்படும்.

பின்னர் `for` loop கற்றுக்கொள்ளும் போது range operators அதிகமாக வரும். தற்போது, இது தொடர்ச்சியான ஒரு எண் வரம்பைக் குறிக்கும் எனப் புரிந்துகொண்டால் போதும்.

## ஆபரேட்டர்களைப் பயிற்சி செய்வோம்

இப்போது, சில எளிய எடுத்துக்காட்டுகள் மூலம், view-இல் ஆபரேட்டர்களின் பங்கைப் பயிற்சி செய்து பார்ப்போம்.

### 1. வாகனங்களின் மொத்த எண்ணிக்கையை கணக்கிடுதல்

உதாரணமாக, ஒரு சந்திப்பில் தினமும் 500 வாகனங்கள் செல்கின்றன எனக் கொள்ளலாம். 30 நாட்களில் சென்ற மொத்த வாகன எண்ணிக்கையை கணக்கிடலாம்.

இதற்கு கணித ஆபரேட்டர்களைப் பயன்படுத்தலாம்:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

இங்கு கணக்கீட்டு நடைமுறை: தினமும் 500 வாகனங்கள், 30 நாட்களில் மொத்தம் `500 × 30`, அதாவது 15000 வாகனங்கள்.

### 2. In-App Purchase உறுப்பினர் என்பதை நிரூபித்தல்

ஒருவர் in-app purchase member ஆவாரா என்பதை நாம் சோதிக்கலாம். அவர் lifetime membership அல்லது subscription membership இவற்றில் ஏதேனும் ஒன்றை வைத்திருந்தால், அவரை member எனக் கருதலாம்.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

இந்த code-இல் `||` (அல்லது) operator பயன்படுத்தப்பட்டுள்ளது: `lifeTime = false` என்பது lifetime membership இல்லை என்பதைக் குறிக்கும்; `subscription = true` என்பது subscription membership உள்ளது என்பதைக் குறிக்கும்.

இரண்டு நிபந்தனைகளில் ஏதேனும் ஒன்று பூர்த்தியானாலே போதுமானதால், `isMember` என்பது `true` ஆகும். அதனால், இந்த நபர் in-app purchase member ஆவார்.

## சுருக்கம்

Swift-இல், ஆபரேட்டர்கள் என்பது data calculation மற்றும் logical judgment ஆகியவற்றின் அடிப்படை. `Text` உள்ளடக்கத்தைப் காட்டுவது போல, ஆபரேட்டர்கள் data-ஐச் செயலாக்குவதற்குப் பயன்படுகின்றன.

எளிய எண் கணக்கீடு முதல் சிக்கலான logical judgment வரை, மேலும் view-இல் state control மற்றும் layout calculation வரை, ஆபரேட்டர்கள் அனைத்திலும் பங்கேற்கின்றன.

பல்வேறு சூழல் அடிப்படையிலான பயிற்சிகள் மூலம், இந்தப் பாடத்தில் அறிமுகப்படுத்தப்பட்ட ஆபரேட்டர்களை மெதுவாகப் பழகி, நன்கு கையாள முடியும். இது அடுத்தடுத்த கற்றலுக்கான முக்கிய அடித்தளமாகும்.

### விரிவாக்க அறிவு - Bitwise ஆபரேட்டர்கள்

மேலும், Swift ஒரு கீழ்மட்டத் தன்மை கொண்ட இன்னொரு வகை ஆபரேட்டர்களையும் வழங்குகிறது — bitwise operators:

* `&`（bitwise AND）
* `|`（bitwise OR）
* `^`（bitwise XOR）
* `~`（bitwise NOT）
* `>>`（வலப்புற shift）
* `<<`（இடப்புற shift）

உதாரணமாக:

```swift
let a = 6  // இரும குறியீடு 110
let b = 3  // இரும குறியீடு 011

print(a & b) // 2 (இரும குறியீடு 010)
print(a | b) // 7 (இரும குறியீடு 111)
print(a ^ b) // 5 (இரும குறியீடு 101)
print(~a)    // -7 (மாற்றுப்படுத்தல்)
```

Swift-இல், integers இரும எண் complement வடிவில் சேமிக்கப்படுகின்றன; அதனால் bitwise NOT செய்தபின், அதற்கான negative result கிடைக்கும்.

Bitwise operators நேரடியாக binary bits மீது செயல்படுகின்றன; பொதுவாக இவை கீழ்மட்ட data processing மற்றும் performance optimization போன்ற சூழல்களில் பயன்படுத்தப்படுகின்றன.

ஆரம்பநிலை மாணவர்களுக்கு, இந்த வகை operators பயன்படும் சூழல்கள் மிகக் குறைவு. பொதுவான iOS அல்லது SwiftUI அபிவிருத்தியிலும் இவை பெரும்பாலும் அரிதாகவே பயன்படுத்தப்படுகின்றன; ஆகவே, இங்கு இதை ஆழமாகப் பார்க்கவில்லை.

பின்னர் கீழ்மட்ட logic அல்லது performance optimization போன்ற விஷயங்கள் வந்தால், அப்போது இதைப் பற்றி மேலும் விரிவாகக் கற்றுக்கொள்ளலாம்.
