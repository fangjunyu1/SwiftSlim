# Xcode கோப்புகளை நிர்வகித்தல்

இந்தப் பாடத்தில், Xcode இல் பொதுவாக பயன்படுத்தப்படும் கோப்பு செயல்பாடுகளை கற்போம்.

முந்தைய பாடங்களில், நமது பெரும்பாலான குறியீடு `ContentView` உள்ளே எழுதப்பட்டது. இது குறியீட்டையும் காட்சி விளைவையும் கவனிக்க வசதியாக இருக்கும்.

எடுத்துக்காட்டு:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

ஆனால் உண்மையான App ஒன்றில், பொதுவாக எல்லா குறியீட்டையும் ஒரே கோப்பில் எழுத மாட்டோம்.

பக்கங்கள், கூறுகள், செயல்பாடுகள் அதிகரிக்கும்போது, வெவ்வேறு view-களை வெவ்வேறு SwiftUI கோப்புகளாகப் பிரிக்க வேண்டும்.

எடுத்துக்காட்டு:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

இது குறியீட்டு அமைப்பை தெளிவாக்கும்; பின்னர் பராமரிக்கவும் திருத்தவும் வசதியாக இருக்கும்.

## கோப்புகளை ஏன் நிர்வகிக்க வேண்டும்

SwiftUI இல், ஒரு பக்கம் அல்லது ஒரு கூறு பொதுவாக தனி view கோப்பாக எழுதப்படலாம்.

எடுத்துக்காட்டு:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

இங்குள்ள `CardView` என்பது ஒரு SwiftUI view ஆகும்.

அனைத்து view-களையும் `ContentView` உள்ளே எழுதினால், குறியீடு மேலும் மேலும் நீளமாகி, படிப்பதும் கடினமாகும்.

எனவே, கோப்புகளை உருவாக்குவது, மறுபெயரிடுவது, நீக்குவது, மேலும் வெவ்வேறு குறியீட்டு கோப்புகளை நிர்வகிக்க folder-களைப் பயன்படுத்துவது ஆகியவற்றைக் கற்றுக்கொள்ள வேண்டும்.

## SwiftUI கோப்பை உருவாக்குதல்

Xcode இல், SwiftUI கோப்பை உருவாக்க பொதுவாக இரண்டு வழிகள் உள்ளன:

ஒன்று வெற்று கோப்பை உருவாக்கி, பின்னர் குறியீட்டை நாமே எழுதுவது.

மற்றொன்று SwiftUI template கோப்பை உருவாக்குவது; இதனால் Xcode அடிப்படை குறியீட்டை தானாக உருவாக்கும்.

தொடக்கநிலையர்களுக்கு SwiftUI template கோப்பைப் பயன்படுத்துவது அதிகம் பரிந்துரைக்கப்படுகிறது, ஏனெனில் அது அடிப்படை அமைப்பை தானாக உருவாக்கும்.

### வெற்று கோப்பை உருவாக்குதல்

Xcode இடதுபுற Navigator பகுதியில், காலியான இடத்தில் அல்லது folder-இல் வலது கிளிக் செய்து, `New Empty File` என்பதைத் தேர்வு செய்யவும்.

செயல்பாட்டு இடம் பின்வருமாறு:

![xcode](../../../Resource/027_view1.png)

வெற்று கோப்பை உருவாக்கிய பின், Xcode பொதுவாக பெயரிடப்படாத Swift கோப்பை உருவாக்கும், எடுத்துக்காட்டாக `Untitled.swift`.

அதை `TestView.swift` என மறுபெயரிடலாம்.

SwiftUI திட்டத்தில், இந்த கோப்பு ஒரு view-ஐ காட்டப் பயன்படுத்தப்படுமானால், பெயரின் இறுதியில் பொதுவாக `View` சேர்க்க பரிந்துரைக்கப்படுகிறது.

எடுத்துக்காட்டு:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

இது கட்டாய விதி அல்ல, ஆனால் இது ஒரு view கோப்பு என்பதை ஒரே பார்வையில் அறிய உதவும்.

வெற்று கோப்பை உருவாக்கிய பின், கோப்பில் இயல்புநிலை comment-கள் மட்டும் இருக்கலாம், அல்லது முழுவதும் காலியாக இருக்கலாம்; இது Xcode பதிப்பு மற்றும் உருவாக்கும் முறையைப் பொறுத்தது.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

இந்த நேரத்தில், SwiftUI குறியீட்டை கையால் எழுத வேண்டும்:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

இதனால், ஒரு எளிய SwiftUI view கோப்பு உருவாக்கப்பட்டுவிடும்.

### கோப்பை மறுபெயரிடுதல்

கோப்பை உருவாக்கிய பின் அதன் பெயரை மாற்ற விரும்பினால், Xcode இடதுபுற Navigator பகுதியில் கோப்பைத் தேர்ந்தெடுத்து, பின்னர் `⏎` Enter விசையை அழுத்தலாம்.

![xcode](../../../Resource/027_view2.png)

இதனால் கோப்பின் பெயரை மாற்றலாம்.

கவனிக்க வேண்டியது: கோப்பின் பெயரை மாற்றுவது, குறியீட்டில் உள்ள view பெயரை தானாக மாற்றாது.

எடுத்துக்காட்டாக, கோப்பின் பெயரை `ProfileView.swift` என மாற்றிய பின்பும், குறியீட்டில் இன்னும் `struct TestView: View` இருக்கலாம்.

இது அவசியமாக பிழையை ஏற்படுத்தாது, ஆனால் குழப்பத்தை ஏற்படுத்தலாம்.

எனவே, கோப்பின் பெயரும் view பெயரும் ஒரேபோல இருக்க பரிந்துரைக்கப்படுகிறது. எடுத்துக்காட்டாக, கோப்பின் பெயர் `ProfileView.swift` என்றால், view பெயரும் `ProfileView` ஆக இருக்க வேண்டும்.

## SwiftUI template கோப்பை உருவாக்குதல்

வெற்று கோப்பை உருவாக்குவதற்கு கூடுதலாக, SwiftUI template கோப்பையும் நேரடியாக உருவாக்கலாம்.

Xcode இடதுபுற Navigator பகுதியில், காலியான இடத்தில் அல்லது folder-இல் வலது கிளிக் செய்து, `New File from Template...` என்பதைத் தேர்வு செய்யவும்.

செயல்பாட்டு இடம் பின்வருமாறு:

![xcode](../../../Resource/027_view3.png)

திறக்கும் சாளரத்தின் மேற்பகுதியில், `iOS`, `macOS` போன்ற வெவ்வேறு platform-களைத் தேர்வு செய்யலாம்.

இங்கே தற்போதைய திட்டத்திற்குப் பொருந்தும் platform-ஐத் தேர்ந்தெடுத்தே வைத்திருக்கலாம். எடுத்துக்காட்டாக, நாம் iOS App உருவாக்குகிறோம் என்றால், `iOS` என்பதைத் தேர்வு செய்யவும்.

![xcode](../../../Resource/027_view4.png)

பின்னர் கீழ்பகுதியில் `SwiftUI View` என்பதைத் தேர்வு செய்யவும்.

![xcode](../../../Resource/027_view12.png)

பின்னர் `Next` கிளிக் செய்து, கோப்புப் பெயரை உள்ளிட்டு சேமிக்கவும்.

எடுத்துக்காட்டாக, நாம் `Test2View.swift` என்ற கோப்பை உருவாக்குகிறோம்.

Xcode தானாக கீழேயுள்ளதைப் போன்ற குறியீட்டை உருவாக்கும்:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

வெற்று கோப்புடன் ஒப்பிடும்போது, template கோப்பில் `import SwiftUI`, `struct`, `body`, `#Preview` ஆகியவை தானாகவே சேர்க்கப்படும்.

எனவே தொடக்கநிலையர்களுக்கு, SwiftUI template கோப்பைப் பயன்படுத்துவது எளிதானது; குறியீட்டை எழுத மறக்கும் வாய்ப்பும் குறையும்.

**குறிப்பு**: வலது கிளிக் செய்து கோப்பை உருவாக்குவதற்கு கூடுதலாக, Xcode இடதுபுற Navigator பகுதியின் கீழே உள்ள plus பொத்தானையும் கிளிக் செய்து கோப்பு அல்லது folder உருவாக்கலாம்.

![xcode](../../../Resource/027_view5.png)

## folder உருவாக்குதல்

கோப்புகள் அதிகரிக்கும்போது, அவற்றை நிர்வகிக்க folder-களைப் பயன்படுத்தலாம்.

Xcode இடதுபுற Navigator பகுதியில், காலியான இடத்தில் அல்லது folder-இல் வலது கிளிக் செய்து, `New Folder` என்பதைத் தேர்வு செய்யவும்.

folder உருவாக்கிய பின், தொடர்புடைய கோப்புகளை அந்த folder-க்குள் இழுத்துச் செல்லலாம்.

![xcode](../../../Resource/027_view7.png)

folder-கள் முக்கியமாக திட்ட அமைப்பை ஒழுங்குபடுத்தப் பயன்படுத்தப்படுகின்றன; view தானாகப் பயன்படுத்தப்படும் முறையை மாற்றாது.

view பெயர் சரியாக இருக்கும் வரை, பிற கோப்புகளிலும் அதை `TestView()`, `Test2View()` போன்ற முறையில் பயன்படுத்தலாம்.

## கோப்பு அல்லது folder-ஐ நீக்குதல்

கோப்பு அல்லது folder-ஐ நீக்க விரும்பினால், Xcode இடதுபுற Navigator பகுதியில் அதற்குரிய கோப்பைத் தேர்ந்தெடுத்து, வலது கிளிக் செய்து `Delete` என்பதைத் தேர்வு செய்யலாம்.

கோப்பைத் தேர்ந்தெடுத்து, delete விசை `⌫` அழுத்தியும் நீக்கலாம்.

Xcode நீக்குதல் உறுதிப்படுத்தும் சாளரத்தை காட்டும்.

![xcode](../../../Resource/027_view10.png)

`Move to Trash` என்பதைத் தேர்வு செய்தால், கோப்பு Trash-க்கு நகர்த்தப்படும்.

`Remove Reference` என்பதைப் பார்த்தால், அது Xcode திட்டத்திலிருந்து reference-ஐ மட்டும் நீக்கும்; ஆனால் disk-இல் உள்ள மூல கோப்பை நீக்காது.

தொடக்கநிலையர்களுக்கு, தாங்களே புதிதாக உருவாக்கிய சாதாரண கோப்பை மட்டும் நீக்குகிறீர்கள் என்றால், பொதுவாக `Move to Trash` தேர்வு செய்தால் போதும்.

## பல SwiftUI view-களை காட்டுதல்

கோப்புகளை உருவாக்க கற்றுக்கொண்ட பிறகு, வெவ்வேறு view-களை வெவ்வேறு கோப்புகளாகப் பிரிக்கலாம்.

எடுத்துக்காட்டாக, இரண்டு SwiftUI கோப்புகளை உருவாக்குகிறோம்:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` ஆசிரியரின் பெயரை காட்டப் பயன்படும்:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` கார்டின் பின்னணியை காட்டப் பயன்படும்:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

பின்னர், இந்த இரண்டு view-களையும் `ContentView` உள்ளே பயன்படுத்தலாம்:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

இயக்கிய பின், `CardView` மற்றும் `AuthorView` சேர்த்து காட்டப்படுவதைக் காணலாம்:

![xcode](../../../Resource/027_view11.png)

இந்த விளைவில், `CardView` பின்னணியாக செயல்படுகிறது, `AuthorView` மேல் பகுதியில் காட்டப்படுகிறது; இதற்குக் காரணம் அவை `ZStack` உள்ளே வைக்கப்பட்டுள்ளன.

`ContentView` உள்ளே, `CardView()` மற்றும் `AuthorView()` மூலம் இந்த இரண்டு view-களையும் பயன்படுத்துகிறோம். பெயருக்குப் பின் வரும் `()` அந்த view-ஐ உருவாக்கி பயன்படுத்துவதை குறிக்கும்.

இதுவே SwiftUI இல் பொதுவாகக் காணப்படும் view nesting: ஒரு view-க்குள் மற்ற view-களைத் தொடர்ந்து பயன்படுத்தலாம்.

## நுழைவு கோப்பு

இறுதியாக, App எந்த இடத்திலிருந்து interface-ஐ காட்டத் தொடங்குகிறது என்பதை மீண்டும் பார்ப்போம்.

SwiftUI திட்டத்தில், பொதுவாக ஒரு நுழைவு கோப்பு இருக்கும்:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

இங்குள்ள `@main` இதுவே App-இன் நுழைவு இடம் என்பதை குறிக்கிறது.

`WindowGroup` உள்ளே நாம் காண்பது:

```swift
ContentView()
```

இதன் பொருள், App தொடங்கிய பின் முதலில் `ContentView` காட்டப்படும்.

பின்னர் `ContentView` உள்ளே மீண்டும் காட்டப்படுவது:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

எனவே view hierarchy-ஐ பின்வருமாறு புரிந்துகொள்ளலாம்:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

நுழைவு கோப்பில் காட்டப்படும் view-ஐ `CardView` ஆக மாற்றினால்:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

அப்படியானால், App தொடங்கிய பின் `CardView` மட்டும் காட்டப்படும்.

அதாவது, நுழைவு குறியீட்டில் உள்ள `WindowGroup`, App தொடங்கிய பின் முதலில் எந்த view காட்டப்பட வேண்டும் என்பதை நிர்ணயிக்கிறது.

அந்த view-இன் உள்ளேயும், மற்ற view-களை தொடர்ந்து காட்டலாம்.

## சுருக்கம்

இந்தப் பாடத்தில், Xcode இல் பொதுவாக பயன்படுத்தப்படும் கோப்பு நிர்வாக செயல்பாடுகளை கற்றோம்.

SwiftUI கோப்புகளை உருவாக்குவது, கோப்புகளை மறுபெயரிடுவது, folder-களை உருவாக்குவது, மற்றும் கோப்புகளை நீக்குவது எப்படி என்பதை அறிந்தோம்.

அதே நேரத்தில், ஒரு முக்கியமான கருத்தையும் கற்றோம்: SwiftUI view-களை வெவ்வேறு கோப்புகளாகப் பிரித்து, பின்னர் பிற view-களில் பயன்படுத்தலாம்.

எடுத்துக்காட்டு:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

இங்குள்ள `CardView()` மற்றும் `AuthorView()` என்பது `ContentView` உள்ளே பிற view-களைப் பயன்படுத்துவதாகும்.

இறுதியாக, நுழைவு கோப்பின் காட்சி தர்க்கத்தையும் மீண்டும் பார்த்தோம்.

App தொடங்கும்போது, முதலில் `@main` குறியீடு கொண்ட நுழைவு குறியீட்டுக்குள் செல்லும்; பின்னர் `WindowGroup` உள்ளே உள்ள view-ஐ காட்டும்.

இது வெவ்வேறு SwiftUI கோப்புகள் தனித்தனியாகப் பிரிந்தவை அல்ல; அவை ஒன்றோடொன்று சேர்ந்து, இறுதியில் முழுமையான App interface-ஐ உருவாக்க முடியும் என்பதைப் புரிந்துகொள்ள உதவும்.
