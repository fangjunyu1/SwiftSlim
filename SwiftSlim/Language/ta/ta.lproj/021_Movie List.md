# திரைப்படப் பட்டியல்

இந்தப் பாடத்தில், நாம் ஒரு திரைப்படப் பட்டியலை உருவாக்கப் போகிறோம்.

![movie](../../Resource/021_movie.png)

இந்தப் பட்டியலில் ஒவ்வொரு திரைப்படத்தின் போஸ்டர், பெயர், இயக்குநர் மற்றும் மதிப்பீடு காட்டப்படும்.

இந்த எடுத்துக்காட்டில், நாம் ஒரு மிகவும் முக்கியமான கருத்தைக் கற்கப் போகிறோம்: `struct` கட்டமைப்பு. இது ஒரு திரைப்படத்தின் பல தகவல்களை ஒன்றாக இணைத்து ஒரு முழுமையான தொகுப்பாக மாற்ற உதவும். இதற்கு அப்பால், `UUID()`, `ForEach`, `Divider` பிரிப்பு கோடு, மேலும் தரவை நிர்வகிக்க தனிப்பயன் பொருட்களை எப்படி பயன்படுத்துவது என்பதையும் அறிந்துகொள்வோம்.

இந்த அறிவுகள் பிந்தைய SwiftUI மேம்பாட்டில் மிகவும் பொதுவாக வருகின்றன. இந்தப் பாடத்தை முடித்த பிறகு, நீங்கள் ஒரு திரைப்படப் பட்டியலை உருவாக்குவதோடு மட்டுமல்லாமல், “ஒரு தரவுக் குழுவை எப்படி இடைமுகமாக காட்டுவது” என்பதையும் புரிந்துகொள்ளத் தொடங்குவீர்கள்.

## ஒற்றை திரைப்படம்

முதலில் ஒரு திரைப்படத்திற்கான இடைமுகத்திலிருந்து தொடங்கலாம்.

![movie](../../Resource/021_movie1.png)

இந்த இடைமுகத்தின் அமைப்பு முக்கியமாக இரண்டு பகுதிகளால் ஆனது: இடப்புறம் திரைப்படப் போஸ்டர், வலப்புறம் திரைப்பட அறிமுகம்.

### திரைப்படப் போஸ்டர்

இடப்புறத்தில் திரைப்படப் போஸ்டர் காட்டப்படுகிறது; படத்தை காட்ட `Image` ஐ பயன்படுத்தலாம்.

உதாரணமாக:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

இந்தக் குறியீடு `"The Shawshank Redemption"` என்ற பெயருடைய ஒரு படத்தை காட்டுகிறது என்பதைக் குறிக்கிறது.

இங்கே கவனிக்க வேண்டிய ஒன்று: படத்தின் பெயர் `Assets` வளக் கோப்புறையில் உண்மையாக இருக்க வேண்டும்; இல்லையெனில் இடைமுகம் அந்தப் படத்தை சரியாக காட்ட முடியாது.

![movie](../../Resource/021_movie2.png)

இந்த modifiers-களின் பணி பின்வருமாறு:

- `resizable()` படம் அளவை மாற்றலாம் என்பதைக் குறிக்கிறது.
- `scaledToFit()` அளவை மாற்றும் போது படத்தின் அசல் விகிதம் பாதுகாக்கப்படும் என்பதைக் குறிக்கிறது; இதனால் படம் நீள்ந்து சிதையாது.
- `frame(height: 180)` படத்தின் உயரத்தை 180 ஆக அமைக்கிறது.
- `cornerRadius(10)` படத்துக்கு 10 அளவிலான வட்டமான மூலைகளை வழங்குகிறது.

இவ்வாறு, சரியான அளவிலும் வட்டமான மூலைகளுடனும் ஒரு திரைப்படப் போஸ்டரைப் பெறலாம்.

![movie](../../Resource/021_movie3.png)

### திரைப்பட அறிமுகம்

வலப்புறத்தில் திரைப்பட அறிமுகம் காட்டப்படுகிறது; அதில் திரைப்படத்தின் பெயர், இயக்குநர் மற்றும் மதிப்பீடு அடங்கும்.

![movie](../../Resource/021_movie4.png)

இந்தப் பகுதியைக் காட்ட `Text` ஐ பயன்படுத்தலாம்:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

இந்தக் குறியீட்டில் `.font` என்பது எழுத்துரு அளவைக் குறிக்கும்; `.fontWeight` என்பது எழுத்துரு எடையைக் குறிக்கும்.

“திரைப்பட பெயர்”, “இயக்குநர் தகவல்”, “மதிப்பீட்டு தகவல்” இவற்றுக்கு இடையில் தெளிவான இடைவெளி இருக்க, இங்கே வெளிப்புறத்தில் `VStack` பயன்படுத்தி இதைப் பொருத்துகிறோம்:

```swift
spacing: 10
```

இதன் அர்த்தம் இந்த `VStack` உள்ளேயுள்ள ஒவ்வொரு உள்ளடக்கக் குழுவிற்கும் இடைவேளையாக 10 இருக்கும் என்பதாகும்.

இயக்குநர் மற்றும் மதிப்பீடு தலா ஒரு தனி `VStack` மூலம் மறுபடியும் சுற்றப்படுவதற்கான காரணம், அவை இரண்டும் “தலைப்பு + உள்ளடக்கம்” என்ற அமைப்பைச் சேர்ந்தவை. இதனால் இடைமுகத்தின் அடுக்குகள் தெளிவாகும்; பின்னர் பாணியை மாற்றுவதும் எளிதாகும்.

மேலும், `VStack` இயல்பாக நடுவில் align செய்யப்படும். முழு உரையும் இடப்புறமாக align ஆக வேண்டும் என்பதற்காக நாம் இதைப் பொருத்துகிறோம்:

```swift
alignment: .leading
```

இவ்வாறு திரைப்பட அறிமுகப் பகுதி மேலும் சீராகத் தெரியும்.

### பிரிப்பு கோடு

இப்போது, திரைப்பட அறிமுகத்தின் உள்ளடக்கம் `spacing` மூலம் பிரிக்கப்பட்டிருந்தாலும், வெவ்வேறு உள்ளடக்கங்களுக்கிடையிலான எல்லை இன்னும் போதுமான அளவு தெளிவாக இல்லை.

இந்த நேரத்தில், நாம் ஒரு பிரிப்பு கோட்டைச் சேர்க்கலாம்:

```swift
Divider()
```

உதாரணமாக:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

இவ்வாறு திரைப்பட அறிமுகப் பகுதியில் மேலும் தெளிவான பிரிப்பு விளைவு கிடைக்கும்.

![movie](../../Resource/021_movie4.png)

`Divider` என்பது மிகவும் எளிய, ஆனால் மிகவும் பொதுவாகப் பயன்படுத்தப்படும் ஒரு view. இதன் பணி வெவ்வேறு உள்ளடக்கங்களைப் பிரிப்பதாகும்.

`VStack` இல் `Divider()` ஒரு கிடைமட்ட கோடாகக் காட்டப்படும்.

`HStack` இல் `Divider()` ஒரு நெடுவரிசை கோடாகக் காட்டப்படும்.

மேலும், `.frame`, `.background`, `.padding` போன்றவற்றின் மூலம் `Divider` இன் பாணியை மாற்றலாம்.

உதாரணமாக:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

இது நீல நிறத்தில், 2 தடிமனுடைய, இடப்புறம் மற்றும் வலப்புறம் padding உடைய ஒரு பிரிப்பு கோட்டைக் கொடுக்கும்.

![divider](../../Resource/021_divider.png)

### முழுமையான இடைமுகம்

இறுதியாக, திரைப்படப் போஸ்டரும் திரைப்பட அறிமுகமும் இடமும் வலமும் பக்கப்பக்கமாக வர `HStack` ஐப் பயன்படுத்தலாம்.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

இங்கே `HStack(spacing: 20)` என்பது இடது மற்றும் வலது இரண்டு பகுதிகளுக்கிடையே 20 அளவிலான இடைவெளி இருக்கும் என்பதைக் குறிக்கிறது.

கவனிக்க வேண்டிய ஒன்று என்னவென்றால், `Divider()` மற்றும் முன்னர் பார்த்த `TextField`, `Slider` போல, இயல்பாகக் கிடைக்கும் இடத்தை முடிந்தவரை அதிகமாகப் பயன்படுத்த முயலும்.

ஆகவே, வலப்புற அறிமுகப் பகுதியின் வெளிப்புற `VStack` க்கு ஒரு நிலையான அகலத்தை அமைக்கிறோம்:

```swift
.frame(width: 200)
```

இவ்வாறு பிரிப்பு கோடும் உரைப் பகுதியின் அகலமும் ஒரே மாதிரியாக இருக்கும்; மொத்த தோற்றமும் சீராக இருக்கும்.

இதுவரை, “ஒற்றை திரைப்படம்” காண்பிக்கும் இடைமுகத்தை நாம் முடித்துவிட்டோம்.

![movie](../../Resource/021_movie1.png)

## array-இல் திரைப்படங்களைச் சேமித்தல்

ஒரே பாணியில் பல திரைப்படங்களை ஒன்றன் பின் ஒன்றாகக் காட்ட வேண்டும் என்றால், ஒவ்வொரு திரைப்படத்திற்கும் இதே போன்ற குறியீட்டை மீண்டும் மீண்டும் எழுத வேண்டும் என்பதைக் குறிக்கும்.

உதாரணமாக:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

இவ்வாறு எழுதப்பட்ட குறியீடு விளைவைக் கொடுக்கலாம்; ஆனால் குறியீடு தொடர்ந்து பெருகும், பராமரிப்பதும் சிரமமாகும்.

உதாரணமாக, அனைத்து திரைப்படங்களின் இடைவெளியை `20` இலிருந்து `15` ஆக மாற்றவேண்டுமென்றாலும், அல்லது வலப்புற அகலத்தை `200` இலிருந்து
`220` ஆக மாற்றவேண்டுமென்றாலும், ஒவ்வொரு மீளும் குறியீட்டையும் கைமுறையாக மாற்ற வேண்டி வரும்.

இது தெளிவாகவே ஒரு திறமையான முறை அல்ல.

நாம் முன்பு array-களைப் பற்றி கற்றோம்; மேலும் array-ஐ அடிப்படையாகக் கொண்டு view-களை மீண்டும் மீண்டும் காட்ட `ForEach` பயன்படுத்துவதையும் கற்றோம்.

உதாரணமாக:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

இந்த முறை “ஒரே வகை தரவு மட்டும்” இருக்கும் சூழல்களுக்கு மிகவும் பொருத்தமானது, உதாரணமாக படங்களின் பெயர்களின் ஒரு தொகுப்பு.

ஆனால், ஒரு திரைப்படம் என்பது ஒரு மதிப்பு மட்டும் அல்ல. அதில் குறைந்தது இவை அடங்கும்:

1. திரைப்படப் போஸ்டர்
2. திரைப்படப் பெயர்
3. இயக்குநர்
4. மதிப்பீடு

அதாவது, ஒரு திரைப்படம் உண்மையில் ஒன்றோடொன்று தொடர்புடைய தரவுகளின் தொகுப்பு; அது ஒரு தனிப்பட்ட string அல்ல.

நாம் array மட்டும் பயன்படுத்தினால், இந்தத் தகவல்களைத் தனித்தனியாக மட்டுமே சேமிக்க முடியும்:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

காட்டும் போது, ஒரே index-ஐ நம்பி அவற்றை ஒன்றுக்கு ஒன்று பொருத்த வேண்டி வரும்:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

இந்த எழுத்து முறையின் பிரச்சனை என்னவென்றால், பராமரிப்பு செலவு மிகவும் அதிகமாகும்.

ஏனெனில், ஒரு திரைப்படத்தின் தரவு பல array-களில் சிதறிக்கிடக்கும். ஒரு array-இல் ஒரு item கூட அதிகமாகவோ, குறையாகவோ, அல்லது வரிசை வேறுபட்டாலோ, காட்சித் தோற்றம் தவறாகிவிடும்.

குறிப்பாக, புதிய திரைப்படங்களைச் சேர்க்கும்போதோ அல்லது நீக்கும்போதோ, பல array-களை ஒரே நேரத்தில் மாற்ற வேண்டும்; ஏதாவது ஒன்று தவறவிடப்படுவது மிகவும் எளிது.

அப்படியானால், “திரைப்படப் பெயர், இயக்குநர், மதிப்பீடு” போன்ற தகவல்களை ஒரே தொகுப்பாக எப்படிச் சேர்ப்பது?

இதற்காக `struct` பயன்படுத்த வேண்டும்.

## `struct` கட்டமைப்பை வரையறுத்தல்

Swift இல், `struct` என்பது “கட்டமைப்பு” ஆகும்.

இதனை நீங்கள் ஒரு “தனிப்பயன் தரவுவகை” என்று புரிந்துகொள்ளலாம்; இது பல தொடர்புடைய fields-ஐ ஒரே முழுமையாக இணைக்க முடியும்.

அடிப்படை பயன்பாடு:

```swift
struct StructName {
	let name: String
}
```

இந்தக் குறியீட்டில், `struct` என்பது நாம் ஒரு கட்டமைப்பை வரையறுக்கிறோம் என்பதைச் சொல்லும் keyword. `StructName` என்பது கட்டமைப்பின் பெயர்.

வளைந்த அடைப்புக்குறிக்குள் உள்ளவை இந்தக் கட்டமைப்பில் உள்ள fields ஆகும்; ஒவ்வொரு field-க்கும் பெயரும் வகையும் தெளிவாக எழுதப்பட வேண்டும்.

பொதுவாக, `struct` இன் பெயர் பெரிய எழுத்தில் தொடங்கும்; உதாரணமாக `Movie`, `Student`, `UserInfo`. இது Swift-இல் பொதுவான பெயரிடும் வழக்கம்.

`struct` ஐ ஒரு காலியான பொதியுப் பெட்டியாக எளிதாகக் கருதலாம்; அதில் உள்ள ஒவ்வொரு field-மும் அந்தப் பெட்டிக்குள் முன்பே ஒதுக்கப்பட்ட ஒரு காலியிடமாக இருக்கும்.

இந்த இடங்கள் இன்னும் நிரப்பப்படாதபோது, அது வெறும் காலிப் பெட்டிதான். எல்லா fields-லும் சரியான மதிப்புகள் நிரப்பப்பட்ட பிறகுதான் அது ஒரு முழுமையான பரிசுப் பெட்டியாக மாறும்.

### `struct` instance

இதற்கு முன் நாம் கட்டமைப்பை மட்டுமே வரையறுத்தோம்; அதாவது ஒரு பொதியுப் பெட்டியின் வடிவமைப்பைத் தயார் செய்தோம்.

இப்போது அதில் குறிப்பிட்ட உள்ளடக்கத்தை நிரப்ப வேண்டும்; அப்போதுதான் பயன்படுத்தக்கூடிய உண்மையான “instance” கிடைக்கும்.

ஒரு instance உருவாக்கும்போது, பொதுவாக கட்டமைப்பின் பெயருக்கு பின் `()` சேர்க்கப்படும்:

```swift
StructName(...)
```

இந்தக் கோடுகளில் நிரப்பப்படுவது, அந்தக் கட்டமைப்புக்குத் தேவையான field values ஆகும்.

உதாரணமாக:

```swift
StructName(name: "Fang Junyu")
```

இந்தக் குறியீடு `StructName` என்ற கட்டமைப்பின் வடிவத்தின்படி ஒரு புதிய instance-ஐ உருவாக்கி, `name` field-க்கு `"Fang Junyu"` என்ற மதிப்பை அளிக்கிறது என்பதைக் குறிக்கிறது.

தேவையான அனைத்து field values-யும் நிரப்பப்பட்ட பிறகு, அது அந்தப் பெட்டியை முழுமையாக நிரப்பியதற்கு சமமாகும்.

அப்போது நமக்கு ஒரு முழுமையான `struct` instance கிடைக்கும்.

### `struct` பண்புகளுக்கான அணுகல்

ஒரு instance உருவாக்கப்பட்ட பிறகு, அதன் உள்ளக பண்புகளுக்குச் செல்ல “dot syntax” ஐ பயன்படுத்தலாம்.

அடிப்படை எழுதும் முறை:

```swift
instance.propertyName
```

உதாரணமாக:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

இங்கே `st` என்பது ஒரு `struct` instance; `st.name` என்பது அந்த instance-இல் உள்ள `name` மதிப்பைப் படிப்பதைக் குறிக்கும்.

இந்தப் படிக்கும் முறையை இப்படிப் புரிந்துகொள்ளலாம்: நாம் `st.name` என்று எழுதும்போது, அந்தப் பெட்டிக்குள் ஒரு குறிப்பிட்ட இடத்தில் வைக்கப்பட்டுள்ள உள்ளடக்கத்தைப் படிப்பதைப் போல இருக்கும்.

இந்த எழுத்து முறை பின்னர் SwiftUI இடைமுகங்களில் மிகவும் பொதுவாக வரும்.

உதாரணமாக, பெயர், வயது, வகுப்பு ஆகியவை கொண்ட ஒரு student கட்டமைப்பு இருந்தால், இடைமுகத்தில் காட்டும் போது நாம் தனித்தனியாகப் படிக்கலாம்:

```swift
student.name
student.age
student.className
```

இதன் நன்மை என்னவென்றால், தரவு மேலும் தெளிவாக இருக்கும்; நிர்வகிக்கவும் எளிதாக இருக்கும்.

### `struct` அமைக்கப்படும் இடம்

தொடக்கநிலையிலுள்ளவர்களுக்கு, கட்டமைப்பை `ContentView` க்கு வெளியே எழுதுவது பொதுவாக ஏற்றதாக இருக்கும்.

உதாரணமாக:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

இவ்வாறு செய்தால், கட்டமைப்பு மேலும் தெளிவாக இருக்கும்; படிக்கவும் எளிதாக இருக்கும்.

`ContentView` என்பதும் உண்மையில் ஒரு `struct` என்பதையும் நீங்கள் கவனிப்பீர்கள்.

### `Movie` கட்டமைப்பு

மேலே கண்ட அடிப்படையை வைத்து, நாம் ஒரு திரைப்படக் கட்டமைப்பை வரையறுக்கலாம்:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

இது `Movie` என்ற பெயருடைய ஒரு கட்டமைப்பு; இதில் `name`, `director`, `rating` என்ற மூன்று fields உள்ளன. இவை முறையே திரைப்படப் பெயர், இயக்குநர், மதிப்பீடு என்பவற்றைக் குறிக்கின்றன.

இவ்வாறு, ஒரு திரைப்படத்துடன் தொடர்புடைய தகவல்களை பல array-களாகப் பிரித்துச் சேமிக்க வேண்டிய அவசியமில்லை; அவற்றை நேரடியாக ஒரே தொகுப்பாகச் சேர்க்கலாம்.

உதாரணமாக, இவ்வாறு ஒரு திரைப்பட instance-ஐ உருவாக்கலாம்:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

இந்த instance “ஒரு முழுமையான திரைப்படத் தரவு” என்பதைக் குறிக்கிறது.

### கட்டமைப்புகளை array-இல் சேமித்தல்

முன்பு நாம் array ஒரே வகை தரவையே சேமிக்க முடியும் என்று சொன்னோம்.

இப்போது நமக்கு `Movie` கட்டமைப்பு உள்ளது; ஆகவே array பல `Movie` values-ஐ சேமிக்க முடியும்.

```swift
let lists: [Movie] = []
```

உதாரணமாக:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

இந்த `lists` array-இல் strings சேமிக்கப்படவில்லை; பல `Movie` instances சேமிக்கப்பட்டுள்ளன.

அதாவது, array-இன் ஒவ்வொரு element-மும் ஒரு முழுமையான திரைப்படமாகும்.

இவ்வாறு, திரைப்படப் பட்டியலைக் காட்ட வேண்டியபோது, `ForEach` மூலம் ஒவ்வொரு திரைப்படத்தையும் ஒன்றன்பின் ஒன்றாகப் படிக்கலாம்.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

இங்கே `movie in` என்பது ஒவ்வொரு loop-இலும் array-இலிருந்து ஒரு திரைப்படம் எடுக்கப்பட்டு, தற்காலிகமாக `movie` என்று பெயரிடப்படுவது என்பதைக் குறிக்கும்.

பிறகு, நாம் இதைப் பயன்படுத்தலாம்:

```swift
movie.name
movie.director
movie.rating
```

அந்த திரைப்படத்தின் பெயர், இயக்குநர் மற்றும் மதிப்பீட்டை தனித்தனியாகப் படித்து, இடைமுகத்தில் காட்டுவதற்காக.

## `ForEach` பிழை

இதுவரை, திரைப்பட array மற்றும் `ForEach` இன் அடிப்படை எழுதும் முறையை முடித்துவிட்டோம்.

ஆனால் கீழ்க்கண்ட குறியீட்டை நேரடியாக இயக்கினால்:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

ஒரு பிழை வரும்:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

இந்தப் பிழையின் அர்த்தம் என்னவென்றால், நீங்கள் `id: \.self` என்று எழுதும்போது, SwiftUI array-இன் ஒவ்வொரு element-ஐயும் “தனித்துவமான அடையாளம்” ஆகப் பயன்படுத்த வேண்டும்.

இந்த எழுதும் முறை `String`, `Int` போன்ற எளிய வகைகளுக்கு ஏற்றது; ஏனெனில் அவற்றைத் தனித்தனியாக வேறுபடுத்துவது எளிது.

ஆனால் `Movie` என்பது நாம் வரையறுத்த custom கட்டமைப்பு; அதைத் தானாகவே தனித்துவமான அடையாளமாக `ForEach` எப்படி பயன்படுத்த வேண்டும் என்பதை அது அறியாது. அதனால் இந்தப் பிழை ஏற்படுகிறது.

இந்தப் பிரச்சினையைத் தீர்க்க மிகச் சாதாரணமான வழி: `Movie` ஐ `Identifiable` protocol-ஐப் பின்பற்றச் செய்வது.

## Protocol

Swift இல் protocol என்பது ஒரு வகையான “விதி” அல்லது “தேவை” என்று புரிந்துகொள்ளலாம்.

இதனை ஒரு சர்வதேச அமைப்பின் உடன்படிக்கையாகக் கருதலாம்: ஒரு நாடு ஏதேனும் சர்வதேச அமைப்பில் சேர விரும்பினால், அந்த அமைப்பு வைத்துள்ள சில விதிகளைக் கடைபிடிக்க வேண்டும், சில தரவுகளை வெளியிட வேண்டும், மேலும் சில பொதுவான விதிகளைப் பின்பற்ற வேண்டும். இந்த நிபந்தனைகள் பூர்த்தியான பிறகே அது அந்த அமைப்பில் சேர முடியும் அல்லது சில செயல்பாடுகளில் பங்கேற்க முடியும்.

Protocol-மும் இதேபோன்ற தர்க்கத்தில் செயல்படுகிறது.

ஒரு type ஏதேனும் protocol-ஐப் பின்பற்ற விரும்பினால், அந்த protocol கேட்டுள்ள உள்ளடக்கங்களை அது பூர்த்தி செய்ய வேண்டும். அவை நிறைவேறிய பிறகே, அந்த type அதற்கான functionality-ஐப் பயன்படுத்த முடியும்.

அடிப்படை எழுதும் முறை:

```swift
struct Movie: Identifiable {
    // ...
}
```

இங்கே `: Identifiable` என்பது `Movie` `Identifiable` protocol-ஐப் பின்பற்றுகிறது என்பதைக் குறிக்கிறது.

ஒரு type பல protocol-களைப் பின்பற்றினால், அவற்றை comma கொண்டு பிரிக்கலாம்:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

`Identifiable`-க்கு மிக முக்கியமான தேவை என்னவென்றால்: type-இல் தன்னைத்தானே அடையாளப்படுத்த ஒரு `id` இருக்க வேண்டும்.

உதாரணமாக:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

இங்கே `id` என்பது அடையாள அட்டை எண்ணைப் போன்றது; ஒவ்வொரு திரைப்படத்தையும் வேறுபடுத்த இதைப் பயன்படுத்துகிறோம்.

ஒவ்வொரு திரைப்படத்திற்கும் வேறுபட்ட `id` இருந்தால், SwiftUI array-இன் ஒவ்வொரு element-ஐயும் சரியாகப் பிரித்தறிய முடியும்.

உதாரணமாக:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

பிறகு `ForEach` இல் இதைப் பயன்படுத்தலாம்:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

இவ்வாறு செய்தால் பிழை இருக்காது.

ஏனெனில் `Movie` ஏற்கனவே `Identifiable` ஐப் பின்பற்றுகிறது; ஆகவே `id:` ஐ நேரடியாக விடுவது இன்னும் பொதுவான முறை:

```swift
ForEach(lists) { movie in
    // ...
}
```

ஏனெனில் SwiftUI ஏற்கனவே ஒவ்வொரு item-ஐ வேறுபடுத்த `movie.id` ஐப் பயன்படுத்த வேண்டும் என்பதை அறிவது.

## `ForEach` இன் வேறுபடுத்தும் அடையாளம்

இப்போது `ForEach` இல் `id` என்ன பங்கு வகிக்கிறது என்பதை ஆழமாகப் புரிந்துகொள்வோம்.

உதாரணமாக:

```swift
ForEach(lists, id: \.self)
```

இதன் பொருள்: element-ஐயே வேறுபடுத்தும் அடையாளமாகப் பயன்படுத்து.

அதேபோல்:

```swift
ForEach(lists, id: \.id)
```

என்பது: element-இன் `id` field-ஐ வேறுபடுத்தும் அடையாளமாகப் பயன்படுத்து.

ஏதேனும் ஒரு field தானாகவே uniqueness-ஐ உறுதி செய்ய முடிந்தால், தற்காலிகமாக அதையும் பயன்படுத்தலாம்.

உதாரணமாக, ஒவ்வொரு திரைப்படத்திற்கும் பெயர் வேறுபட்டிருந்தால், தற்போது உள்ள தரவுகளுடன் கீழ்க்கண்ட எழுதும் முறை சரியாகவே செயல்படலாம்:

```swift
ForEach(lists, id: \.name)
```

ஆனால் இங்கே ஒரு மறைமுகமான பிரச்சினை இருக்கிறது: `name` எப்போதும் தனித்துவமாக இருக்க வேண்டிய அவசியமில்லை.

பின்னர் நீங்கள் அதே பெயருடைய இன்னொரு திரைப்படத்தைச் சேர்த்துவிட்டீர்கள் என்று வைத்துக்கொள்ளுங்கள்; அப்போது `name` ஒவ்வொரு element-ஐயும் துல்லியமாக வேறுபடுத்த முடியாது.

அந்த நிலையில் code compile ஆகலாம்; ஆனால் view refresh, update, insert, delete போன்ற சமயங்களில் SwiftUI தவறாக அடையாளம் காணலாம்; இதனால் காட்சியில் பிரச்சினை தோன்றும்.

எனவே, தரவை நிலையாகவே வேறுபடுத்த வேண்டிய சூழலில், உண்மையிலேயே தனித்துவமான field-ஆன `id` ஐப் பயன்படுத்துவது சிறந்தது.

## UUID

`id: 1`, `id: 2` என்று கையால் எழுதுவது பிரச்சினையைத் தீர்க்கலாம்; ஆனால் இன்னும் ஒரு அபாயம் இருக்கும்: தவறுதலாக ஒரே `id` மீண்டும் எழுதப்படலாம்; அப்போது தனித்துவ அடையாளம் செயலிழக்கும்.

உதாரணமாக:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

இவ்வாறு இரண்டு ஒரே மாதிரியான `id` கள் தோன்றும்; தனித்துவ அடையாளம் செயலிழக்கும்.

கைமுறைப் பிழைகளைத் தவிர்க்க, நாம் பொதுவாக `UUID()` ஐப் பயன்படுத்துகிறோம்.

Swift இல், `UUID()` என்பது 128-bit அடையாளத்தை சீரற்ற முறையில் உருவாக்கும். இது பொதுவாக எழுத்துக்களும் எண்களும் சேர்ந்த நீண்ட string ஆகக் காட்டப்படும்.

உதாரணமாக:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

இங்கே `UUID()` உண்மையில் `UUID` வகையின் ஒரு instance-ஐ உருவாக்குகிறது; இதனை தானாக உருவாகும் “தனித்துவ எண்” எனக் கருதலாம்.

அதனால், ஒவ்வொரு முறை உருவாகும் value மீண்டும் திரும்புவது மிகவும் அரிது; எனவே அது `id` ஆகப் பயன்படுத்த மிகவும் ஏற்றது.

நாம் `Movie` ஐ இவ்வாறு மாற்றலாம்:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

இந்தக் குறியீடு `Movie` கட்டமைப்பில் உள்ள `id` field-க்கு default ஆக ஒரு புதிய `UUID` instance value வழங்கப்படும் என்பதைக் குறிக்கிறது.

அதாவது, ஒவ்வொரு முறையும் ஒரு புதிய `Movie` உருவாக்கும்போது, system தானாகவே அதற்கென ஒரு தனித்துவமான `id` ஐ உருவாக்கும்.

`id` ஏற்கனவே default value-ஐக் கொண்டதால், பின்னர் `Movie` instances உருவாக்கும்போது `id` ஐ கைமுறையாக நிரப்பத் தேவையில்லை.

உதாரணமாக:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

இங்கே `id` எழுதப்படாவிட்டாலும், இந்த திரைப்பட instance-க்கு உண்மையில் அதன் சொந்த `id` இருக்கும்; அந்த value ஏற்கனவே `UUID()` மூலம் தானாக உருவாக்கப்பட்டுள்ளது.

இந்த முறையைப் பயன்படுத்திய பிறகு, ஒவ்வொரு திரைப்படத்திற்கும் `id` ஐ கைமுறையாக நிரப்ப வேண்டியதில்லை; இதனால் code அளவு குறையும்; மேலும் duplicate id காரணமாக வரும் பிழைகளும் தவிர்க்கப்படுகின்றன.

இறுதியாக, மேலே ஒரு `Banner` படத்தைச் சேர்த்து, உள்ளடக்கம் scroll ஆக `ScrollView` பயன்படுத்தினால், முழு திரைப்படப் பட்டியல் view தயார் ஆகிவிடும்.

## சுருக்கம்

இந்தப் பாடத்தில், நாம் ஒரு மிக முக்கியமான கருத்தைக் கற்றுக்கொண்டோம்: `struct`.

`struct` மூலம், ஒரு திரைப்படத்தின் பல தொடர்புடைய fields-ஐ ஒரே தொகுப்பாக இணைக்க முடியும்; பெயர், இயக்குநர், மதிப்பீடு ஆகியவற்றை பல array-களில் பிரிக்க வேண்டியதில்லை.

தரவு ஒன்றாக ஒருங்கிணைக்கப்பட்ட பிறகு, array-இல் உள்ள ஒவ்வொரு திரைப்படத்தையும் `ForEach` மூலம் ஒன்றன்பின் ஒன்றாக view-இல் காட்டலாம்.

அதே சமயம், `ForEach` இல் உள்ள “வேறுபடுத்தும் அடையாளம்” என்னும் கருத்தின் பங்கையும் புரிந்துகொண்டோம். SwiftUI க்கு array-இன் ஒவ்வொரு element-ஐ எப்படிப் வேறுபடுத்துவது என்பது தெரிந்திருக்க வேண்டும்; அப்போதுதான் view சரியாகக் காட்சியளிக்கவும் புதுப்பிக்கவும் முடியும்.

அதனால், `Movie` ஐ `Identifiable` protocol-ஐப் பின்பற்றச் செய்து, அதற்கு தனித்துவமான `id` ஐ வழங்கினோம்.

`id` ஐ கைமுறையாக நிரப்பும்போது பிழைகள் வராமல் இருக்க, system தானாகவே தனித்துவ அடையாளத்தை உருவாக்கும் `UUID()` பற்றியும் மேலும் கற்றுக்கொண்டோம்.

இந்தப் பாடத்தை முடித்த பிறகு, நீங்கள் ஒரு திரைப்படப் பட்டியலை முடித்ததோடு மட்டுமல்லாமல், SwiftUI-இல் உள்ள ஒரு முக்கியமான சிந்தனை முறையையும் அறியத் தொடங்கியுள்ளீர்கள்: **முதலில் தரவை ஒழுங்குபடுத்து, பிறகு அந்தத் தரவை அடிப்படையாகக் கொண்டு view-ஐ உருவாக்கு.**
