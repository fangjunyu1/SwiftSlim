# தோற்றத் தழுவல்

இந்த பாடத்தில், வெளிர் முறை மற்றும் அடர் முறைக்கான தோற்றத் தழுவலைக் கற்போம்.

iPhone, iPad மற்றும் Mac-இல், பயனர் வெளிர் முறையையோ அடர் முறையையோ தேர்வு செய்யலாம்.

சிஸ்டம் அடர் முறைக்கு மாறும்போது, பல App-களின் பின்னணி, எழுத்து மற்றும் ஐகான் நிறங்களும் அதற்கேற்ப மாறும்.

எடுத்துக்காட்டாக, பகலில் இடைமுகம் வெளிர் பின்னணியாக இருக்கலாம்; இரவில் அது அடர் பின்னணியாக மாறலாம்.

![view](../../../Resource/025_view9.png)

சிஸ்டத்தின் தோற்றத்தின் அடிப்படையில் இடைமுகக் காட்சியை தானாகச் சரிசெய்யும் இந்த திறனையே தோற்றத் தழுவல் என்று அழைக்கிறோம்.

## Xcode-இல் தோற்றத் தழுவலை முன்னோட்டமாகப் பார்ப்பது

தோற்றத் தழுவலைக் கற்கும் முன், Xcode-இல் வெளிர் முறை மற்றும் அடர் முறையை எப்படி முன்னோட்டமாகப் பார்ப்பது என்பதை முதலில் அறிந்துகொள்வோம்.

`Canvas` பகுதியின் அடிப்பகுதியில் உள்ள `Device Settings` பொத்தானைக் கிளிக் செய்தால், `Canvas Device Settings` பாப்அப் சாளரத்தைத் திறக்கலாம்.

![canvas](../../../Resource/025_view.png)

பாப்அப் சாளரத்தில் `Color Scheme` அமைப்பைக் காணலாம்.

![canvas1](../../../Resource/025_view1.png)

அவற்றில்:

- `Light Appearance` என்பது வெளிர் முறையை குறிக்கும்.
- `Dark Appearance` என்பது அடர் முறையை குறிக்கும்.

இங்கிருந்து முன்னோட்டத் தோற்றத்தை மாற்றி, தற்போதைய இடைமுகம் வெளிர் முறை மற்றும் அடர் முறையில் எப்படி காட்டுகிறது என்பதைப் பார்க்கலாம்.

## இரண்டு தோற்றங்களையும் பக்கப்பக்கமாகக் காட்டுதல்

வெளிர் முறை மற்றும் அடர் முறையை ஒரே நேரத்தில் பார்க்க விரும்பினால், `Variants` செயல்பாட்டைப் பயன்படுத்தலாம்.

`Canvas` பகுதியின் அடிப்பகுதியில் உள்ள `Variants` பொத்தானைக் கிளிக் செய்து, `Color Scheme Variants` என்பதைத் தேர்வு செய்யுங்கள்.

![canvas2](../../../Resource/025_view2.png)

தேர்வு செய்த பிறகு, `Canvas` வெளிர் முறை மற்றும் அடர் முறையின் முன்னோட்ட விளைவுகளை ஒரே நேரத்தில் காட்டும்.

![canvas3](../../../Resource/025_view3.png)

இதனால் இரண்டு தோற்றங்களின் இடைமுக வேறுபாடுகளை ஒப்பிடுவது இன்னும் வசதியாக இருக்கும்.

எளிமையாகச் சொன்னால்:

வெளிர்/அடர் முறையை தற்காலிகமாக மட்டும் மாற்றிப் பார்க்க வேண்டும் என்றால், `Canvas Device Settings` பயன்படுத்தலாம்.

இரண்டு தோற்றங்களையும் ஒரே நேரத்தில் பார்க்க வேண்டும் என்றால், `Color Scheme Variants` பயன்படுத்தலாம்.

## View இயல்பாகவே தோற்றத்துக்கு ஏற்ப தழுவும்

SwiftUI-இல், பல சிஸ்டம் view-கள் இயல்பாகவே வெளிர் முறை மற்றும் அடர் முறைக்கு தானாகத் தழுவும்.

எடுத்துக்காட்டு:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("ஒளி")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("சில உரை சில உரை")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

காட்சி விளைவு:

![canvas](../../../Resource/025_view4.png)

வெளிர் முறையில், ஐகான் மற்றும் தலைப்பு பொதுவாக அடர் நிறத்தில் காட்டப்படுவதை பார்க்கலாம்.

அடர் முறையில், ஐகான் மற்றும் தலைப்பு தானாக வெளிர் நிறமாக மாறும்.

இதற்குக் காரணம், `Text`, `Image` போன்ற SwiftUI view-கள் இயல்பாகவே சிஸ்டம் தோற்றத்தின் அடிப்படையில் நிறத்தை மாற்றுகின்றன.

அதாவது, நாம் நிலையான நிறத்தை கைமுறையாக குறிப்பிடவில்லை என்றால், தோற்றத் தழுவலின் ஒரு பகுதியை SwiftUI நமக்காக கையாளும்.

## நிலையான நிறங்கள் தானாக மாற்றமடையாது

கவனிக்க வேண்டியது: ஒரு நிறத்தை நாம் கைமுறையாக நிலையான நிறமாகக் குறிப்பிட்டால், அது வெளிர்/அடர் முறைக்கு ஏற்ப தானாக மாற்றமடையாது.

எடுத்துக்காட்டு:

```swift
Text("சில உரை சில உரை")
	.foregroundStyle(Color.gray)
```

இங்கே உரை `Color.gray` எனக் குறிப்பிடப்பட்டுள்ளது. ஆகவே அது எப்போதும் சாம்பல் நிறமாகவே காட்டப்படும்.

மற்றொரு எடுத்துக்காட்டு:

```swift
Text("தலைப்பு")
    .foregroundStyle(Color.white)
```

இந்தக் குறியீடு வெளிர் முறையிலோ அடர் முறையிலோ இருந்தாலும், உரையை வெள்ளை நிறமாகவே காட்டும்.

பின்னணியும் வெளிர் நிறமாக இருந்தால், வெள்ளை உரை தெளிவாகத் தெரியாமல் போகலாம்.

எனவே தோற்றத் தழுவலைச் செய்யும்போது, `Color.white`, `Color.black` போன்ற நிலையான நிறங்களை சீரற்ற முறையில் hardcode செய்வதை முடிந்தவரைத் தவிர்க்க வேண்டும்.

பல சூழல்களில், சிஸ்டம் வழங்கும் semantic style-களை முன்னுரிமையாகப் பயன்படுத்தலாம். உதாரணமாக:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

அவற்றில்:

- `.primary` பொதுவாக முக்கிய உள்ளடக்கத்திற்குப் பயன்படுத்தப்படுகிறது.
- `.secondary` பொதுவாக துணை உள்ளடக்கத்திற்குப் பயன்படுத்தப்படுகிறது.

இவை வெளிர்/அடர் முறைக்கு ஏற்ப காட்சி விளைவை தானாகச் சரிசெய்யும்.

## தோற்றத்தின் அடிப்படையில் வேறுபட்ட உள்ளடக்கத்தைக் காட்டுதல்

சில நேரங்களில், நிறம் மாறுவதையே மட்டும் நாம் விரும்பமாட்டோம்; வேறு தோற்றங்களில் வேறு உள்ளடக்கங்களையும் காட்ட விரும்புவோம்.

எடுத்துக்காட்டாக:

- வெளிர் முறையில் சாதாரண விளக்கு ஐகானைக் காட்டுதல்.
- அடர் முறையில் ஒளிரும் விளக்கு ஐகானைக் காட்டுதல்.
- வெளிர் முறையில் தலைப்பு `ஒளி` எனக் காட்டுதல்.
- அடர் முறையில் தலைப்பு `இருள்` எனக் காட்டுதல்.

இந்த நேரத்தில், தற்போதைய சிஸ்டம் வெளிர் முறையிலா அல்லது அடர் முறையிலா உள்ளது என்பதைதீர்மானிக்க வேண்டும்.

SwiftUI-இல், தற்போதைய தோற்ற முறையை `colorScheme` environment value மூலம் பெறலாம்:

```swift
@Environment(\.colorScheme) private var colorScheme
```

முழு குறியீடு:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "ஒளி" : "இருள்"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("சில உரை சில உரை")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

காட்சி விளைவு:

![view](../../../Resource/025_view5.png)

முன்னோட்ட விளைவிலிருந்து பார்க்கும்போது, ஒரே view வெவ்வேறு தோற்ற முறைகளில் முற்றிலும் ஒரே மாதிரியாகக் காட்டப்படாது.

வெளிர் முறையில், `Image` சாதாரண விளக்கு ஐகானைக் காட்டும்; தலைப்பு `ஒளி` எனக் காட்டப்படும்.

அடர் முறையில், `Image` ஒளிரும் விளக்கு ஐகானைக் காட்டும்; தலைப்பு `இருள்` எனக் காட்டப்படும்.

அதாவது, இங்குள்ள மாற்றம் நிறம் மட்டும் மாறுவது அல்ல; ஐகான் பெயரும் தலைப்பு உரையும் தாமே மாறுகின்றன.

இதற்குக் காரணம், நாம் ஐகான் மற்றும் தலைப்பை நேரடியாக `body`-க்குள் hardcode செய்யவில்லை; அதற்கு பதிலாக `colorScheme` அடிப்படையில் வேறு உள்ளடக்கத்தை கணக்கிடுகிறோம்.

இந்தக் குறியீட்டில், நாம் இரண்டு computed property-களைப் பயன்படுத்தியுள்ளோம்:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "ஒளி" : "இருள்"
}
```

`colorScheme` `.light` ஆக இருந்தால், தற்போதைய தோற்றம் வெளிர் முறை என்பதை குறிக்கும்.

`colorScheme` `.dark` ஆக இருந்தால், தற்போதைய தோற்றம் அடர் முறை என்பதை குறிக்கும்.

எனவே வேறு தோற்ற முறைகளின் அடிப்படையில், வேறு ஐகான் பெயரும் தலைப்பு உரையும் திருப்பிக் கொடுக்கலாம்.

இதுவும் computed property-களின் ஒரு பொதுவான பயன்பாடு: தற்போதைய நிலையின் அடிப்படையில், view காட்ட வேண்டிய உள்ளடக்கத்தை கணக்கிடுவது.

## @Environment-ஐப் புரிந்துகொள்வது

`@Environment`-ஐ நாம் முதன்முறையாக இங்கு சந்திக்கிறோம்.

`@Environment` என்பதை இவ்வாறு புரிந்துகொள்ளலாம்: SwiftUI-ன் environment-இலிருந்து ஒரு மதிப்பை வாசிப்பது.

App இயங்கும்போது, சிஸ்டம் பல environment தகவல்களை வழங்கும். உதாரணமாக:

- தற்போதைய மொழி
- தற்போதைய தோற்ற முறை
- தற்போதைய layout திசை
- தற்போதைய எழுத்துரு அளவு அமைப்பு

தற்போதைய தோற்ற முறையைப் பெற வேண்டும்போது, `colorScheme`-ஐ வாசிக்கலாம்:

```swift
@Environment(\.colorScheme) private var colorScheme
```

அவற்றில்

```swift
\.colorScheme
```

environment-இலிருந்து `colorScheme` என்ற மதிப்பை வாசிப்பதைக் குறிக்கும்.

```swift
private var colorScheme
```

வாசிக்கப்பட்ட மதிப்பை `colorScheme` என்ற variable-இல் சேமிப்பதைக் குறிக்கும்.

variable பெயரை நீங்களே தீர்மானிக்கலாம். உதாரணமாக இவ்வாறு எழுதவும் முடியும்:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

முன்பகுதியான `\.colorScheme` மாறாமல் இருந்தால், வாசிக்கப்படுவது தற்போதைய தோற்ற முறை என்பதையே குறிக்கும்.

## colorScheme-இன் இரண்டு பொதுவான மதிப்புகள்

`colorScheme`-க்கு பொதுவாக இரண்டு மதிப்புகள் உள்ளன:

```swift
.light
.dark
```

அவை குறிக்குவது:

- `.light`: வெளிர் முறை
- `.dark`: அடர் முறை

எடுத்துக்காட்டு:

```swift
private var titleName: String {
    colorScheme == .light ? "ஒளி" : "இருள்"
}
```

இந்தக் குறியீட்டின் பொருள்:

தற்போது வெளிர் முறை என்றால், `colorScheme == .light`உண்மையாகும்; அப்போது `"ஒளி"` திருப்பிக் கொடுக்கப்படும்.

இல்லையெனில், `"இருள்"` திருப்பிக் கொடுக்கப்படும்.

அதனால், வெளிர் முறையில்:

```swift
Text(titleName)
```

காட்டப்படுவது:

```swift
ஒளி
```

அடர் முறையில், காட்டப்படுவது:

```swift
இருள்
```

இந்த முறையின் மூலம், வேறு தோற்றங்களின் அடிப்படையில் வேறு உள்ளடக்கத்தைக் காட்டலாம்.

## நிறத் தழுவல் எடுத்துக்காட்டு

உரை மற்றும் ஐகான்களுக்கு அப்பாற்பட்டு, சில சமயங்களில் நிறத் தழுவலை நாமே கையாள வேண்டியிருக்கும்.

எடுத்துக்காட்டு:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("உரை")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

காட்சி விளைவு:

![view](../../../Resource/025_view6.png)

இந்தக் குறியீட்டில், உரை வெள்ளை நிறம்; பின்னணி கருப்பு நிறம்.

வெளிர் முறையில், கருப்பு பின்னணி தெளிவாகத் தெரியும்.

ஆனால் அடர் முறையில், முழு இடைமுகப் பின்னணியும் கருப்பாக இருந்தால், இந்த கருப்பு பின்னணி சிஸ்டம் பின்னணியுடன் கலந்து, தெளிவாகத் தெரியாமல் இருக்கலாம்.

இந்த நேரத்தில், தோற்ற முறையின் அடிப்படையில் உரை நிறத்தையும் பின்னணி நிறத்தையும் மாற்றலாம்:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("உரை")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

காட்சி விளைவு:

![view](../../../Resource/025_view7.png)

இப்போது, வெளிர் முறையில் கருப்பு பின்னணி மற்றும் வெள்ளை உரை இருக்கும்.

அடர் முறையில் வெள்ளை பின்னணி மற்றும் கருப்பு உரை இருக்கும்.

இதனால் எந்த தோற்றத்திலும், உரை மற்றும் பின்னணிக்கிடையே தெளிவான contrast இருக்கும்; பயனர் உள்ளடக்கத்தை தெளிவாகப் பார்க்க முடியும்.

## Assets பயன்படுத்தி நிறத்தைத் தழுவுதல்

## colorScheme மற்றும் Assets இடையிலான வேறுபாடு

`colorScheme` மற்றும் `Assets` இரண்டும் தோற்றத் தழுவலைச் செய்ய முடியும். ஆனால் அவற்றுக்கு ஏற்ற பயன்பாட்டு சூழல்கள் வேறுபடும்.

வெளிர்/அடர் முறையின் அடிப்படையில் உரை உள்ளடக்கம், படப் பெயர் அல்லது `SF Symbols` பெயரை மட்டும் மாற்ற வேண்டுமானால், `colorScheme` பயன்படுத்தலாம்.

எடுத்துக்காட்டு:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

App-இல் அடிக்கடி பயன்படுத்தப்படும் நிறங்கள், உதாரணமாக theme color, card background color, text color போன்றவை என்றால், `Assets` நிறத்தைப் பயன்படுத்துவது அதிகம் பரிந்துரைக்கப்படுகிறது.

ஏனெனில் `Assets` நிறங்களை பல view-களில் மீண்டும் பயன்படுத்தலாம்; பின்னர் மாற்றுவதும் எளிதாக இருக்கும்.

எடுத்துக்காட்டு:

```swift
Color("redText")
```

இவ்வாறு செய்தால், `Assets`-இல் நிறத்தை மட்டும் மாற்றினால் போதும்; இந்த நிறத்தைப் பயன்படுத்தும் எல்லா இடங்களும் ஒன்றாக மாறும்.

## சுருக்கம்

இந்த பாடத்தில், வெளிர் முறை மற்றும் அடர் முறைக்கான தோற்றத் தழுவலைக் கற்றோம்.

முதலில், Xcode-இன் `Canvas`-இல் வேறு தோற்றங்களை எப்படி முன்னோட்டமாகப் பார்ப்பது என்பதை கற்றோம்; மேலும் `Color Scheme Variants` பயன்படுத்தி வெளிர் முறை மற்றும் அடர் முறையை ஒரே நேரத்தில் பார்க்கும் முறையையும் கற்றோம்.

பிறகு, SwiftUI-இல் உள்ள `Text`, `Image` போன்ற view-கள் இயல்பாகவே சிஸ்டம் தோற்றத்தின் அடிப்படையில் நிறத்தை தானாகத் தழுவும் என்பதை அறிந்தோம்.

ஆனால், `Color.white` அல்லது `Color.black` போன்ற நிலையான நிறங்களை கைமுறையாகக் குறிப்பிட்டால், அவை தானாக மாற்றமடையாது.

அடுத்து, `@Environment` மற்றும் `colorScheme` பற்றி கற்றோம்:

```swift
@Environment(\.colorScheme) private var colorScheme
```

`colorScheme` மூலம், தற்போதைய தோற்றம் வெளிர் முறையா அடர் முறையா என்பதைதீர்மானித்து, வேறு தோற்றங்களின் அடிப்படையில் வேறு ஐகான், உரை அல்லது நிறத்தை காட்டலாம்.

இறுதியாக, `Assets` நிறங்களை கற்றோம்.

`Assets` நிறங்களில் வெளிர் முறை மற்றும் அடர் முறைக்கான நிறங்களை தனித்தனியாக அமைக்கலாம்; App-இல் பொதுவாகப் பயன்படுத்தப்படும் color resource-களை நிர்வகிக்க இது ஏற்றது.

தோற்றத் தழுவல் App-ஐ பகலிலும் இரவிலும் சிறப்பாகக் காட்ட உதவும்; உரை தெளிவாக தெரியாதது, பின்னணி தெளிவாக இல்லாதது போன்ற பிரச்சினைகளையும் தவிர்க்கலாம்.

உண்மையான development-இல், SwiftUI-ன் இயல்பான தழுவல் திறன் மற்றும் `Assets` நிறங்களை முன்னுரிமையாகப் பயன்படுத்த பரிந்துரைக்கப்படுகிறது.

வேறு தோற்றங்களில் உரை, ஐகான், படம் மாற்ற வேண்டியிருந்தால், அல்லது சில சிறிய சிறப்பு காட்சி விளைவுகளை கையாள வேண்டியிருந்தால், பின்னர் `colorScheme` மூலம்தீர்மானிக்கலாம்.

## கூடுதல் அறிவு: எழுத்துரு அளவை குறிப்பிடுதல்

முன்பு `font` கற்றபோது, நாம் சில சிஸ்டம் font style-களைப் பயன்படுத்தியுள்ளோம்:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

`.largeTitle`, `.callout` போன்ற சிஸ்டம் font style-களைப் பயன்படுத்துவதற்கு அப்பாற்பட்டு, எழுத்துரு அளவை கைமுறையாகவும் குறிப்பிடலாம்.

எடுத்துக்காட்டு:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

இங்குள்ள:

```swift
.font(.system(size: 100))
```

எழுத்துரு அளவை `100 pt` ஆக அமைப்பதைக் குறிக்கும்.

`SF Symbols` ஐகான்களுக்கும், `font` ஐகானின் அளவை பாதிக்கும்.

எனவே, ஐகானை பெரிதாகக் காட்ட விரும்பினால், `.font(.system(size:))` மூலம் அளவை குறிப்பிடலாம்.

எடுத்துக்காட்டு:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

கவனிக்க வேண்டியது: எழுத்துரு அளவை கைமுறையாகக் குறிப்பிடுவது அதிக நெகிழ்வைத் தரினாலும், சாதாரண உரைக்கு fixed font size-ஐ அதிகமாகப் பயன்படுத்த பரிந்துரைக்கப்படாது.

சாதாரண உரைக்கு `.title`, `.headline`, `.body`, `.caption` போன்ற சிஸ்டம் font style-களை முன்னுரிமையாகப் பயன்படுத்த வேண்டும்.

இதனால் வேறு சாதனங்களுக்கும் பயனரின் எழுத்துரு அமைப்புகளுக்கும் இன்னும் சிறப்பாகத் தழுவ முடியும்.
