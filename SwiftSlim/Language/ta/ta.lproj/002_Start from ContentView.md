# ContentView-இலிருந்து தொடங்கலாம்

## பாடத்திற்கு முன் தயாராகுதல்

இந்தப் பாடத்தில், நாம் ContentView.swift கோப்பிலிருந்து தொடங்கி, SwiftUI-யின் அடிப்படை அமைப்பை முறையாக அறிந்து கொள்வோம். இதில் உள்ளவை:

* குறிப்புரைகள்
* View அமைப்பு
* VStack / HStack / ZStack அமைப்புகள்

முதலில், முன்பு உருவாக்கிய Xcode திட்டத்தைத் தேடி, .xcodeproj கோப்பை இருமுறை சொடுக்கவும்.

இடப்புற Navigator வழிசெலுத்தல் பகுதியில் ContentView.swift கோப்பைத் தேர்வு செய்யவும்.

கவனம்: ஒவ்வொரு முறையும் திட்டத்தைத் திறக்கும் போது, Canvas-ல் "Preview paused" எனக் காட்டப்படலாம். Refresh பொத்தானை அழுத்தினால் preview மீண்டும் செயல்படும்.

## ContentView-ஐ அறிதல்

ContentView குறியீடு:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

குறியீடு குறைவாக இருந்தாலும், இதில் SwiftUI-யின் மைய அமைப்பு அடங்கியுள்ளது.

### 1. குறிப்புரை

கோப்பின் மேல் பகுதி:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

இது கோப்பு குறிப்புரை. இது கோப்பு பெயர், திட்டப் பெயர், உருவாக்கியவர், உருவாக்கப்பட்ட தேதி போன்ற தகவல்களை பதிவு செய்யப் பயன்படுகிறது.

Swift-இல், `//` பயன்படுத்தி ஒற்றை வரி குறிப்புரை எழுதப்படுகிறது:

```swift
// இது ஒரு குறிப்புரை
```

குறிப்புரைகள், குறியீட்டின் வாசிப்புத் தெளிவை மேம்படுத்துகின்றன; மேலும், டெவலப்பர்கள் குறியீட்டு தர்க்கத்தைப் புரிந்துகொள்ள உதவுகின்றன.

நடைமுறை அபிவிருத்தியில், குறியீடு சில சமயங்களில் சிக்கலாக இருக்கும். குறிப்புரை எழுதாமல் விட்டால், மூன்று நாட்கள் கழித்து தானே எழுதிய குறியீட்டையும் புரிந்துகொள்ள முடியாத நிலை ஏற்படலாம்.

ஆகையால், குறியீடு எழுதும் போதே குறிப்புரைகளைச் சேர்ப்பது ஒரு நல்ல அபிவிருத்தி பழக்கமாகும். குறிப்புரைகள் மூலம் குறியீட்டு தர்க்கத்தை பதிவு செய்தால், பின்னர் பராமரிப்பு செய்வது எளிதாகும்.

**குறியீட்டை தற்காலிகமாக முடக்குதல்**

குறிப்புரைகள் மூலம் குறியீட்டை தற்காலிகமாக முடக்கியும், பிரச்சினையை கண்டுபிடிக்கலாம்.

உதாரணமாக:

```
A
B
C
```

A, B, C ஆகியவை மூன்று தனித்த குறியீட்டு பகுதிகள். இதில் ஒன்றில் பிழை உள்ளது எனக் கொள்ளலாம். அப்போது, அவற்றை தற்காலிகமாக குறிப்புரையாக்கி பிரச்சினையைத் தேடலாம்.

முதலில் A-ஐ குறிப்புரையாக்கலாம்:

```
// A
B
C
```

A-ஐ குறிப்புரையாக்கிய பிறகு குறியீடு சரியாக இயங்கினால், A பகுதியில் பிரச்சினை இருப்பது தெரியும்.

A-ஐ குறிப்புரையாக்கியும் பிரச்சினை நீங்காவிட்டால், B-ஐ குறிப்புரையாக்கலாம். இப்படியாகத் தொடர்ந்து, பிழை உள்ள குறியீட்டை கண்டுபிடிக்கலாம்.

அபிவிருத்தி செயல்முறையில் பல பிரச்சினைகளை நாம் சந்திப்போம். பெரும்பாலும், குறியீட்டை குறிப்புரையாக்கி பிரச்சினையின் காரணத்தைப் பின்தொடர்வோம். இது BUG ஏற்பட்டுள்ள பகுதியைத் துல்லியமாகக் கண்டறிய உதவும்.

Xcode-இல், கீழ்க்கண்ட விசைப்பலகைச் சுருக்கத்தை பயன்படுத்தலாம்:

```
Command ⌘ + /
```

இதன் மூலம் குறிப்புரையை விரைவாகச் சேர்க்கவும் நீக்கவும் முடியும்.

### 2. SwiftUI framework-ஐ import செய்தல்

```swift
import SwiftUI
```

இந்தக் குறியீடு SwiftUI framework-ஐ import செய்கிறது என்பதை குறிக்கிறது.

SwiftUI-யிலுள்ள `View`, `Text`, `Image`, `VStack` போன்ற வகைகள் அனைத்தும் இந்த framework-இலிருந்து வருகின்றன.

SwiftUI framework-ஐ import செய்யவில்லை என்றால், Xcode பிழை காட்டும்:

```
Cannot find type 'View' in scope
```

அதாவது, compiler-க்கு `View` வகையை அடையாளம் காண முடியவில்லை.

### 3. View அமைப்பு

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

முதல்முறையாக இந்த View அமைப்பைப் பார்க்கும் போது, இது புதிதாகத் தோன்றலாம். ஏனெனில் இதில் `struct`, `View`, `var`, `body`, `some` போன்ற முக்கிய சொற்கள் உள்ளன.

இப்போது நாமிதைப் பற்றிய எல்லா முக்கிய சொற்களையும் இன்னும் கற்றுக்கொள்ளவில்லை. ஆகவே இப்போது அறிந்திருக்க வேண்டியது, இந்தக் குறியீடு `ContentView` என்ற பெயருடைய ஒரு View-ஐ உருவாக்குகிறது என்பதே.

நீங்கள் View-ஐ ஒரு ஓவிய பலகையாகக் கற்பனை செய்யலாம். அந்தப் பலகையில் நாம் வரைபடம் போடலாம்; அந்த வரையும் கருவி SwiftUI ஆகும்.

உதாரணமாக:

![Swift](../../RESOURCE/002_view.png)

மேலுள்ள படத்தில் மூன்று பக்கங்கள் காணப்படுகின்றன; உண்மையில் அவை மூன்று View-கள்.

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

SwiftUI-யைப் பயன்படுத்தி App உருவாக்கும் போது, ஒவ்வொரு பக்கமும் ஒரு View ஆகும்.

### 4. SwiftUI குறியீடு

View-க்குள் காட்டப்படும் SwiftUI குறியீடு:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

இந்த SwiftUI குறியீடு, இது ஒரு செங்குத்து அமைப்பு என்றும், அதில் ஒரு சின்னமும் ஒரு உரையும் காட்டப்படுகின்றன என்றும் பொருள் தருகிறது.

![Swift](../../RESOURCE/002_view1.png)

#### VStack அமைப்பு

```swift
VStack { }  // செங்குத்து அமைப்பு
```

`VStack` என்பது ஒரு செங்குத்து layout container ஆகும். இதன் உள்ளேயுள்ள view-கள் மேலிருந்து கீழாக ஒழுங்குபடுத்தப்படும்.

![Swift](../../RESOURCE/002_view8.png)

SwiftUI-யில் பொதுவாகப் பயன்படுத்தப்படும் மூன்று layout-கள்:

* VStack —— செங்குத்து வரிசைப்படுத்தல்
* HStack —— கிடைமட்ட வரிசைப்படுத்தல்
* ZStack —— அடுக்கி வரிசைப்படுத்தல் (Z அச்சு)

```swift
HStack { }  // கிடைமட்ட வரிசைப்படுத்தல்
ZStack { }  // அடுக்கி வரிசைப்படுத்தல்
```

ஒவ்வொரு layout-இன் ஒழுங்கு விளக்கப்படம்:

![Swift](../../RESOURCE/002_view2.png)

உதாரணமாக, `HStack` பயன்படுத்தி கிடைமட்ட வரிசைப்படுத்தல் செய்வோம்:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

இப்போது, பூமி சின்னமும் உரையும் குறுக்காகக் காட்டப்படுகின்றன என்பதைப் பார்க்கலாம்.

![Swift](../../RESOURCE/002_view3.png)

நாம் view-களை கிடைமட்டமாக ஒழுங்குபடுத்த விரும்பினால் `HStack` பயன்படுத்த வேண்டும். அடுக்கி வைக்க வேண்டுமெனில் `ZStack` பயன்படுத்த வேண்டும்.

#### Image மற்றும் SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

இந்தக் குறியீடு ஒரு பூமி சின்னத்தை காட்டுகிறது. இந்தச் சின்னம் Apple-இன் SF Symbols icon system-இலிருந்து வருகிறது.

![Swift](../../RESOURCE/002_view9.png)

இந்தக் குறியீட்டின் பொருள்: ஒரு பூமி சின்னத்தை, பெரிய அளவில், accent color-இல் காட்டவும்.

பூமி சின்னத்தை மட்டும் அல்லாமல், வேறு சின்னங்களையும் காட்டலாம்.

உதாரணமாக, ஒரு backpack சின்னத்தை காட்டலாம்:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**மற்ற சின்னங்களை எப்படி காட்டுவது?**

அதற்காக, Apple வழங்கும் system icon library ஆன SF Symbols-ஐ பயன்படுத்த வேண்டும்.

Apple Developer அதிகாரப்பூர்வ இணையதளத்தைத் திறந்து, [SF Symbols](https://developer.apple.com/sf-symbols/) ஐப் பதிவிறக்கம் செய்யவும்.

![Swift](../../RESOURCE/002_sf.png)

பிறகு SF Symbols App-ஐத் திறக்கவும்.

![Swift](../../RESOURCE/002_sf1.png)

இடப்புறம் symbol category-கள் இருக்கும்; வலப்புறம் அதற்குரிய சின்னங்கள் காட்டப்படும்.

சின்னத்தின் மீது right click செய்து "拷贝1个名称" என்பதைத் தேர்வு செய்யவும். அந்தப் பெயரே குறியீட்டில் பயன்படுத்த வேண்டிய string ஆகும்.

உதாரணமாக:

```
"globe"
"backpack"
"heart"
```

நகலெடுத்த சின்னப் பெயரை `Image(systemName:)` இல் வைத்து விட்டால், வேறு சின்னங்களை காட்ட முடியும்.

கவனம்: ஒவ்வொரு SF Symbols சின்னத்திற்கும் குறைந்தபட்ச system version தேவைப்படும். system version மிகவும் பழையதாக இருந்தால், சின்னம் காட்டப்படாமல் இருக்கலாம். அதன் compatibility தகவலை SF Symbols App-இல் பார்க்க வேண்டும்.

#### Modifier-கள்

SwiftUI-யில், modifier என்பது ஒரு view-இன் தோற்றத்தையோ நடத்தையோ மாற்ற பயன்படுத்தப்படும் ஒரு முறை.

Modifier-ஐ உடையாகக் கற்பனை செய்யலாம். வேறு வேறு உடைகள் அணிந்தால், தோற்றமும் மாறும்.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

இங்கே `imageScale` மற்றும் `foregroundStyle` இரண்டும் `Image` view-இன் modifier-கள். `Image`-இன் உள்ளடக்கத்தை மாற்றாமல், அதன் தோற்றத்தை மாற்றுகின்றன.

**1. imageScale**

```swift
.imageScale(.large)
```

இது SF Symbols சின்னத்தின் அளவை கட்டுப்படுத்தும்:

* .small
* .medium
* .large

![Swift](../../RESOURCE/002_view11.png)

வேறு வேறு விருப்பங்களை முயற்சி செய்து, SF Symbols சின்னம் வெவ்வேறு அளவுகளில் காட்டப்படுவதைப் பார்க்கலாம்.

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

`foregroundStyle` முன்புற நிறத்தை கட்டுப்படுத்துகிறது.

`.tint` என்பது தற்போதைய சூழலின் accent color-ஐ குறிக்கும். இயல்பாக iOS-இல் அது நீல நிறமாக இருக்கும்.

நாம் முன்புற நிறத்தை சிவப்பாக மாற்ற விரும்பினால்:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Text view

`Text` என்பது உரையை காட்டும் view ஆகும்.

```swift
Text("Hello, world!")
```

உதாரணமாக, என் பெயரை காட்டலாம்:

```swift
Text("FangJunyu")
```

கவனம்: string கட்டாயமாக `""` double quote-களுக்குள் இருக்க வேண்டும்.

நீங்கள் உங்கள் பெயர், தொலைபேசி எண் போன்றவற்றைக் காட்டிப் பார்த்து முயற்சி செய்யலாம்.

#### padding ஓரம் இடைவெளி

SwiftUI-யில், `padding` என்பது view-இன் உள்ளடக்கம் மற்றும் அதன் எல்லை ஆகியவற்றுக்கிடையில் கூடுதல் காலி இடத்தைச் சேர்க்கப் பயன்படுகிறது. இது “உள் ஓரம் இடைவெளி (padding / content inset)” ஆகும்.

```swift
HStack {
    ...
}
.padding()
```

மேலுள்ள குறியீடு, `HStack` view-க்கு system-இன் இயல்புநிலை padding-ஐச் சேர்க்கிறது.

**padding என்பது என்ன?**

padding என்பது “view-இன் உள்ளடக்கம் மற்றும் அதன் எல்லைக்கிடையிலான காலி இடம்” என்று பொருள்.

கீழே உள்ள படத்தில், நீல நிற `HStack`-க்கு padding அமைத்த பிறகு, நீல நிற பகுதி உள்ளே சுருங்கியதுபோல் தெரியும்; அதாவது அது “சற்றுக் குறைந்தது” போலத் தோன்றும்.

![Swift](../../RESOURCE/002_view6.png)

**இயல்புநிலை இடைவெளி**

`padding()` modifier இயல்பாக system பரிந்துரைக்கும் standard spacing-ஐப் பயன்படுத்துகிறது.

```swift
.padding()
```

வேறு வேறு தளங்களிலும் சூழல்களிலும் இந்த மதிப்பு மாறக்கூடும். உதாரணமாக:

* iOS-இல் இது பொதுவாக சுமார் 16 pt இருக்கும்.
* macOS அல்லது watchOS-இல் system standard spacing மாறக்கூடும்; அது ஒவ்வொரு தளத்தின் design விதிமுறைகளைப் பொறுத்தது.

**தனிப்பயன் padding**

view-க்கு தனித்தனியான padding-களையும் அமைக்கலாம்.

1. ஒரு திசைக்கு மட்டும் அமைத்தல்

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

திசை விளக்கம்:

* .top: மேல் ஓரம்
* .bottom: கீழ் ஓரம்
* .leading: முன்புற ஓரம்
* .trailing: பின்புற ஓரம்

![Swift](../../RESOURCE/002_view12.png)

கவனம்: `leading` மற்றும் `trailing` மொழியின் எழுத்து திசைக்கு ஏற்ப தானாகப் பொருந்தும். உதாரணமாக, அரபு மொழி போன்ற RTL சூழலில் அவை தானாக மாற்றப்படும்.

2. பல திசைகளை அமைத்தல்

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

array பயன்படுத்தி ஒரே நேரத்தில் பல திசைகளை குறிப்பிடலாம். array-யின் முழு பயன்பாட்டை பின்னர் விரிவாகப் பார்க்கலாம்; இப்போது இந்த எழுதும் முறையை அறிந்தால் போதும்.

3. கிடைமட்டம் அல்லது செங்குத்து திசை அமைத்தல்

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

இது கீழே உள்ளவற்றுக்கு சமம்:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**padding-ஐ அகற்றுதல்**

எந்த padding-மும் வேண்டாம் என்றால், `.padding(0)` பயன்படுத்தி அதை நீக்கலாம்:

```swift
.padding(0)
```

அல்லது `padding` modifier-ஐ நேரடியாக நீக்கலாம்:

```swift
// .padding()
```

### 6. Preview முன்தோற்றக் குறியீடு

```swift
#Preview {
    ContentView()
}
```

இந்தக் குறியீடு, Canvas-இல் `ContentView`-க்கான preview-ஐ உருவாக்குகிறது.

கவனம்: `#Preview` என்பது Swift 5.9 / Xcode 15-இல் அறிமுகப்படுத்தப்பட்ட புதிய syntax ஆகும். அதற்கு முன் `PreviewProvider` structure பயன்படுத்தப்பட்டது.

**Preview-ஐ குறிப்புரையாக்கினால் என்ன ஆகும்?**

`Preview`-ஐ குறிப்புரையாக்கினால்:

```swift
// #Preview {
//    ContentView()
// }
```

Canvas இனி render செய்யக்கூடிய உள்ளடக்கத்தை காட்டாது.

![Swift](../../RESOURCE/002_xcode.png)

இதன் மூலம், Canvas-இல் preview view-ஐ காட்டுவதைக் `#Preview` கட்டுப்படுத்துகிறது என்பதை அறியலாம்.

Xcode-இல் SwiftUI view-ஐ preview செய்ய வேண்டுமெனில், `#Preview` குறியீட்டைச் சேர்க்க வேண்டும். preview தேவையில்லையெனில், `#Preview` குறியீட்டை குறிப்புரையாக்கவோ நீக்கவோலாம்.

## சுருக்கம்

`ContentView.swift` கோப்பில் குறியீடு அதிகம் இல்லாவிட்டாலும், அது SwiftUI-யின் பல முக்கியக் கருத்துகளை உள்ளடக்கியுள்ளது. தொடக்கநிலையினருக்கு இந்தக் குறியீடு புதிதாகவும் சற்று சிக்கலாகவும் தோன்றலாம்; ஆனால் அதைத் துண்டித்து புரிந்துகொண்டால், SwiftUI பற்றிய ஆரம்பகட்ட புரிதலை உருவாக்க முடியும்.

இந்தப் பாடத்தில் நாம் கற்றதைக் திரும்பப் பார்க்கலாம். முதலில் `//` குறிப்புரையைப் பற்றி கற்றோம். குறிப்புரைகளைப் பயன்படுத்தி குறியீட்டு தர்க்கத்தை விளக்கலாம் அல்லது குறியீட்டை தற்காலிகமாக முடக்கலாம்.

அடுத்து, SwiftUI கோப்புகளில் கட்டாயமாக SwiftUI framework-ஐ import செய்ய வேண்டும் என்பதை அறிந்தோம்:

```swift
import SwiftUI
```

இந்த framework-ஐ import செய்யவில்லை என்றால், compiler-க்கு `View` போன்ற வகைகளை அடையாளம் காண முடியாது.

பிறகு, SwiftUI view-இன் அடிப்படை அமைப்பை அறிந்தோம்:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

இதில் `ContentView` என்பது view-இன் பெயர்.

மேலும், மூன்று பொதுவான layout container-களையும் கற்றோம்: `VStack` (செங்குத்து வரிசைப்படுத்தல்), `HStack` (கிடைமட்ட வரிசைப்படுத்தல்), `ZStack` (அடுக்கி வரிசைப்படுத்தல்).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

இந்தக் குறியீட்டில், ஒரு செங்குத்து வரிசைப்படுத்தப்பட்ட container காட்டப்படுகிறது; அதன் உள்ளே ஒரு சின்னமும் ஒரு உரையும் உள்ளன.

`Image` மூலம் SF Symbols சின்னங்களை காட்டலாம்; modifier-கள் மூலம் அதன் அளவும் நிறமும் கட்டுப்படுத்தலாம்.

`Text` view மூலம் உரை உள்ளடக்கத்தை காட்டலாம்.

`padding` என்பது view-ஐச் சுற்றி ஒரு வெளிப்படையான இடைவெளியைச் சேர்க்கும் ஓரம் இடைவெளியாகும்.

இறுதியாக, `#Preview` முன்தோற்ற view-ஐ பார்த்தோம்; இது Canvas-இல் preview-ஐ காட்டுகிறது.

### பாடத்திற்குப் பிறகான பயிற்சி

தொடக்கநிலையினருக்கு இந்தப் பாடத்தின் உள்ளடக்கம் சற்றுக் கடினமாக இருக்கலாம். ஆகவே, கீழ்கண்ட பயிற்சிகள் மூலம் இங்கு கற்றவற்றை மேலும் ஆழமாகப் புரிந்துகொள்ளலாம்:

* SF Symbols சின்னத்தின் பெயரை மாற்றிப் பார்க்கவும்
* சின்னத்தின் foreground color-ஐ கருப்பாக மாற்றவும்
* `VStack`-ஐ `HStack`-ஆக மாற்றவும்
* `Image` அல்லது `Text`-ஐ குறிப்புரையாக்கி preview-இல் ஏற்படும் மாற்றத்தைப் பார்க்கவும்

### சிறு கூடுதல்: Code Completion

குறியீடு எழுதும் போது, கவனமாக இருந்தால் Xcode தானாகவே கிடைக்கக்கூடிய option பட்டியலைக் காட்டுவதை நீங்கள் ஏற்கனவே கவனித்திருக்கலாம்.

உதாரணமாக, `imageScale` modifier-ஐ மாற்றும்போது:

```swift
.imageScale(.)
```

Xcode கிடைக்கக்கூடிய option-களைச் காட்டும்:

![Swift](../../RESOURCE/002_view10.png)

இது code completion mechanism ஆகும். இது type inference மற்றும் enum member suggestion அடிப்படையில் இயங்குகிறது; இதனால் typing வேகம் அதிகரிக்கும், மேலும் பிழைகள் குறையும்.

பின்னர் வரும் பாடங்களில், நாம் `enum` பற்றி முறையாகக் கற்போம். இப்போது இதைப் பற்றி ஒரு ஆரம்பநிலை அறிமுகம் மட்டும் தெரிந்தால் போதும்.
