# Xcode-ஐ அறிதல்

## Xcode என்பது என்ன

Xcode என்பது Apple அதிகாரப்பூர்வமாக வழங்கும் ஒருங்கிணைந்த மேம்பாட்டு சூழல் (IDE, Integrated Development Environment) ஆகும். இது iOS, macOS, watchOS, tvOS போன்ற Apple தள பயன்பாடுகளை உருவாக்கப் பயன்படுத்தப்படுகிறது.

![Swift](../../RESOURCE/001_xcode.png)

முழு கற்றல் செயல்முறையிலும், நாங்கள் Xcode-ஐப் பயன்படுத்தி குறியீடு எழுதவும், நிரலை இயக்கவும் போகிறோம்.

இந்தப் பாடத்தில், Xcode-ஐ எவ்வாறு பதிவிறக்கம் செய்வது, முதல் iOS பயன்பாட்டை உருவாக்குவது, மேலும் Xcode இடைமுக அமைப்பை ஆரம்பநிலையாக அறிதல் ஆகியவற்றைப் படிப்போம்.

## Xcode-ஐ பதிவிறக்கம் செய்தல்

Xcode-ஐ தற்போது இரண்டு முறைகளில் நிறுவலாம்:

1、[Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12) மூலம் பதிவிறக்கம் செய்தல்

2、[Apple Developer அதிகாரப்பூர்வ இணையதளம்](https://developer.apple.com/xcode/) மூலம் பதிவிறக்கம் செய்தல்

இரண்டு முறைகளிலும் Xcode-ஐ பதிவிறக்கம் செய்யலாம். வித்தியாசம் என்னவென்றால், Apple Developer அதிகாரப்பூர்வ தளத்தில் நிலையான பதிப்புக்கு கூடுதலாக [Beta](https://developer.apple.com/download/all/?q=Xcode) பதிப்பு அல்லது பழைய பதிப்புகளையும் பதிவிறக்கம் செய்யலாம்.

வளர்ச்சியை கற்றுக்கொள்வதற்காக மட்டுமே பயன்படுத்துகிறீர்கள் என்றால், நிலையான பதிப்பைப் பயன்படுத்த பரிந்துரைக்கப்படுகிறது. Beta பதிப்பு புதிய அம்சங்களைச் சோதிக்கப் பயன்படுகிறது; அதனால் சில நிலைத்தன்மை சிக்கல்கள் இருக்கலாம்.

## முதல் திட்டத்தை உருவாக்குதல்

Xcode-ஐ திறந்தவுடன், துவக்கத் திரை காணப்படும்.

![Swift](../../RESOURCE/001_xcode1.png)

இடப்பக்கத்தில் மூன்று பொதுவான விருப்பங்கள் உள்ளன:

* Create New Project —— புதிய திட்டத்தை உருவாக்குதல்
* Clone Git Repository —— Git களஞ்சியத்தை clone செய்வது
* Open Existing Project —— ஏற்கனவே உள்ள திட்டத்தைத் திறப்பது

வலப்பக்கத்தில் சமீபத்தில் திறக்கப்பட்ட Xcode திட்டங்கள் காட்டப்படும்.

நாம் "Create New Project" விருப்பத்தைத் தேர்ந்தெடுக்கிறோம்.

### திட்ட வார்ப்புருவைத் தேர்வு செய்தல்

![Swift](../../RESOURCE/001_xcode2.png)

திட்ட வார்ப்புரு தேர்வு திரைக்குள் சென்ற பிறகு, பல தளங்கள் (iOS, macOS போன்றவை) மற்றும் வெவ்வேறு வகை திட்ட வார்ப்புருக்கள் காணப்படும்.

பொதுவான வார்ப்புருக்களின் விளக்கம்:

* App —— மிகவும் அடிப்படையான பயன்பாட்டு வகை (பரிந்துரை செய்யப்படுகிறது)
* Document App —— ஆவண முறை அடிப்படையிலான பயன்பாடு
* Game —— விளையாட்டு மேம்பாட்டிற்கானது
* Framework —— மீண்டும் பயன்படுத்தக்கூடிய தொகுதி

ஆரம்பநிலையில், நாம் "iOS" - "App" வார்ப்புருவை மட்டும் தேர்வு செய்கிறோம், ஏனெனில் இது இடைமுக பயன்பாட்டு வகையாகும்.

"Next" ஐ அழுத்தவும்.

### திட்டத் தகவலை நிரப்புதல்

திட்டத் தகவலை நிரப்பும் திரைக்கு சென்ற பிறகு, ஒவ்வொரு புலத்திற்கும் ஏற்ப தகவலை நிரப்ப வேண்டும்.

![Swift](../../RESOURCE/001_xcode3.png)

ஒவ்வொரு புலத்தின் அர்த்தம்:

* Product Name —— திட்டத்தின் பெயர், உதாரணமாக: SwiftSlimTest
* Team —— அபிவிருத்தி குழு; கட்டண Apple Developer கணக்கு இல்லையெனில் இதைத் தேர்வு செய்யாமல் விடலாம்
* Organization Identifier —— நிறுவனம் அல்லது அடையாளக் குறி; பொதுவாக reverse domain name வடிவில் பயன்படுத்தப்படுகிறது, உதாரணமாக:

  * com.yourname
  * com.yourcompany
  * உங்கள் சொந்த domain இல்லையெனில், உங்கள் ஆங்கிலப் பெயரை அடையாளமாக பயன்படுத்தலாம்
* Bundle Identifier —— பயன்பாட்டின் தனிப்பட்ட அடையாளம்; இது Organization Identifier மற்றும் Product Name அடிப்படையில் தானாக உருவாகும்
* Interface —— இடைமுக தொழில்நுட்பம்; SwiftUI தேர்வு செய்யவும்
* Language —— அபிவிருத்தி மொழி; Swift தேர்வு செய்யவும்
* Testing System —— இயல்பாக சோதனை இலக்கு (Unit Tests) உருவாக்கப்படும்; ஆரம்பநிலையில் இதை புறக்கணிக்கலாம்
* Storage —— உள்ளூர் தரவு நிலைத்த சேமிப்பு framework; இது SwiftData அல்லது Core Data-ஐ தானாக ஒருங்கிணைக்க முடியும்; ஆரம்பநிலையில் இதையும் புறக்கணிக்கலாம்

எல்லாவற்றையும் நிரப்பிய பிறகு, "Next" ஐ அழுத்தவும்.

### திட்டத்தைச் சேமித்தல்

![Swift](../../RESOURCE/001_xcode4.png)

சேமிக்க ஏற்ற ஒரு கோப்புறையைத் தேர்வு செய்யவும்.

"Create" பொத்தானை அழுத்தியவுடன், Xcode திட்ட அமைப்பை தானாக உருவாக்கும்.

## Xcode திட்ட அமைப்பு

Xcode திட்டத்தைச் சேமித்த பிறகு, Finder-இல் உருவாக்கப்பட்ட திட்ட கோப்புறையைப் பார்க்கலாம்.

![Swift](../../RESOURCE/001_xcode5.png)

* Assets.xcassets —— படங்கள், App சின்னங்கள் போன்ற வளங்களைச் சேமிக்கப் பயன்படும் கோப்புறை

* ContentView.swift —— இயல்பாக உருவாகும் SwiftUI view கோப்பு; பின்னர் இதில் நாங்கள் இடைமுகக் குறியீட்டை எழுதுவோம்

* SwiftSlimTestApp.swift —— பயன்பாட்டின் தொடக்கக் கோப்பு (App Entry Point); இது பயன்பாட்டைத் தொடங்க பொறுப்பானது. இதை இப்போது புரிந்துகொள்ள தேவையில்லை

* SwiftSlimTest.xcodeproj —— Xcode திட்டக் கோப்பு; இந்தக் கோப்பை இருமுறை சொடுக்கினால் திட்டத்தைத் திறக்கலாம்

## Xcode இடைமுகத்தை அறிதல்

.xcodeproj கோப்பை இருமுறை சொடுக்கி திட்டத்தைத் திறந்த பிறகு, இடப்பக்க Navigator பகுதியில் ContentView.swift கோப்பைத் தேர்வு செய்யவும்.

Xcode இடைமுகம் முக்கியமாக ஐந்து பகுதிகளாகப் பிரிக்கப்பட்டுள்ளது:

* Navigator —— வழிசெலுத்தல் பகுதி; கோப்பு அமைப்பு, தேடல் போன்றவற்றைப் பார்க்கப் பயன்படும்
* Editor —— தொகுப்புப் பகுதி; Swift அல்லது SwiftUI குறியீட்டை எழுதப் பயன்படும்
* Canvas —— கேன்வாஸ் பகுதி; SwiftUI முன்தோற்றத்திற்குப் பயன்படும்
* Inspector —— பண்புகள் ஆய்வு பகுதி; கோப்பின் பண்புகளைப் பார்க்கவும் மாற்றவும் பயன்படும்
* Debug Area —— பிழைத்திருத்த பகுதி; பதிவுகள் மற்றும் வெளியீடுகளைப் பார்க்கப் பயன்படும்

![Swift](../../RESOURCE/001_xcode6.png)

கவனம்: Canvas முக்கியமாக SwiftUI view-களின் preview-க்காக பயன்படுத்தப்படுகிறது. Xcode அதிகாரப்பூர்வ கட்டமைப்பு பார்வையில், இது Editor-க்கான உதவி preview panel (Preview Pane) ஆகும். SwiftUI view கோப்பைத் திறந்திருக்கும்போது மட்டுமே Canvas-ஐ செயல்படுத்த முடியும்; மற்ற வகை கோப்புகளில் preview அம்சம் காணப்படாது.

பின்னர் வரும் பாடங்களில், நாம் முக்கியமாக Editor பகுதியில் Swift மற்றும் SwiftUI குறியீட்டை எழுதுவோம்.

முதல் முறை Xcode திட்டத்தைத் திறக்கும் போது, Inspector மற்றும் Debug Area இயல்பாக மறைக்கப்பட்டிருக்கலாம்; அவற்றை காட்ட மேல் வலது மூலை மற்றும் கீழ் வலது மூலையில் உள்ள பொத்தான்களை அழுத்தலாம்.

![Swift](../../RESOURCE/001_xcode7.png)

குறிப்பு: Inspector பகுதி முக்கியமாக கோப்பின் பண்புகளைப் பார்க்கவும் மாற்றவும் பயன்படுகிறது. நடைமுறை அபிவிருத்தியில், பெரிய தொகுப்பு இடத்தைப் பெறுவதற்காக இந்தப் பகுதி பொதுவாக மறைக்கப்பட்டே இருக்கும்.

## சுருக்கம்

இந்தப் பாடத்தில், Xcode-ஐ எவ்வாறு பதிவிறக்கம் செய்வது, முதல் Xcode திட்டத்தை உருவாக்குவது, மேலும் Xcode-இன் அடிப்படை இடைமுக அமைப்பை கற்றுக்கொண்டோம்.

கீழே சுயமாக முயற்சி செய்யும் நேரம்:

* Navigator பகுதியில் கோப்புகளை உருவாக்கவும், நீக்கவும் முயற்சி செய்யலாம்.
* Editor பகுதியில் உள்ள குறியீட்டை பார்த்து, அதன் அமைப்பைக் கவனிக்கலாம்.
* Canvas பகுதியில் உள்ள பொத்தான்களைப் பார்த்து, preview-வை பெரிதாக்கவும் சிறிதாக்கவும் முயற்சி செய்யலாம்.

அடுத்தப் பாடத்தில், சில எளிய குறியீடுகளை எழுதத் தொடங்குவோம்; அதன்மூலம் Xcode-ஐ இன்னும் நன்றாக அறிந்துகொள்ள உதவும்.
