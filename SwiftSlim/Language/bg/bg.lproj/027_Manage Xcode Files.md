# Управление на файлове в Xcode

В този урок ще научим често използвани операции с файлове в Xcode.

В предишните уроци писахме по-голямата част от кода в `ContentView`. Това улеснява наблюдаването на кода и резултата от показването.

Например:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

Но в истинско App обикновено не поставяме целия код в един файл.

Когато страниците, компонентите и функциите стават все повече, трябва да разделим различните изгледи в различни SwiftUI файлове.

Например:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

Това прави структурата на кода по-ясна и улеснява последващата поддръжка и промени.

## Защо трябва да управляваме файлове

В SwiftUI една страница или един компонент обикновено може да бъде написан като отделен файл за изглед.

Например:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

Тук `CardView` е SwiftUI изглед.

Ако всички изгледи са написани в `ContentView`, кодът ще става все по-дълъг и все по-труден за четене.

Затова трябва да се научим да създаваме, преименуваме и изтриваме файлове, както и да използваме папки за управление на различни файлове с код.

## Създаване на SwiftUI файл

В Xcode има два често използвани начина за създаване на SwiftUI файл:

Единият е да създадем празен файл и след това сами да напишем кода.

Другият е да създадем SwiftUI шаблонен файл, при който Xcode автоматично генерира основния код.

За начинаещи е по-препоръчително да се използва SwiftUI шаблонен файл, защото той автоматично генерира основната структура.

### Създаване на празен файл

В лявата зона Navigator на Xcode щракнете с десния бутон върху празно място или папка и изберете `New Empty File`.

Мястото на операцията е показано по-долу:

![xcode](../../../Resource/027_view1.png)

След създаване на празен файл Xcode обикновено ще генерира неименуван Swift файл, например `Untitled.swift`.

Можем да го преименуваме на `TestView.swift`.

В SwiftUI проект, ако този файл се използва за показване на изглед, обикновено се препоръчва в края на името да се добави `View`.

Например:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

Това не е задължително правило, но ни позволява с един поглед да разберем, че това е файл за изглед.

След създаване на празен файл в него може да има само стандартни коментари или да е празен. Това зависи от версията на Xcode и начина на създаване.

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

В този момент трябва ръчно да напишем SwiftUI код:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

Така един прост файл за SwiftUI изглед е създаден.

### Преименуване на файл

Ако след създаване на файл искате да промените името му, можете да изберете файла в лявата зона Navigator на Xcode и след това да натиснете клавиша Enter `⏎`.

![xcode](../../../Resource/027_view2.png)

Така можете да промените името на файла.

Трябва да се има предвид, че промяната на името на файла няма автоматично да промени името на изгледа в кода.

Например, след като името на файла бъде променено на `ProfileView.swift`, в кода все още може да остане `struct TestView: View`.

Това невинаги води до грешка, но лесно може да създаде объркване.

Затова се препоръчва името на файла и името на изгледа да бъдат еднакви. Например, ако името на файла е `ProfileView.swift`, името на изгледа също трябва да бъде `ProfileView`.

## Създаване на SwiftUI шаблонен файл

Освен да създадем празен файл, можем също директно да създадем SwiftUI шаблонен файл.

В лявата зона Navigator на Xcode щракнете с десния бутон върху празно място или папка и изберете `New File from Template...`.

Мястото на операцията е показано по-долу:

![xcode](../../../Resource/027_view3.png)

В изскачащия прозорец отгоре може да изберете различни платформи, например `iOS`, `macOS` и други.

Тук е достатъчно да оставим избрана платформата, която съответства на текущия проект. Например, ако създаваме iOS App, избираме `iOS`.

![xcode](../../../Resource/027_view4.png)

След това изберете `SwiftUI View` в долната част.

![xcode](../../../Resource/027_view12.png)

След това щракнете върху `Next`, въведете име на файла и го запазете.

Например създаваме `Test2View.swift`.

Xcode автоматично ще генерира код, подобен на следния:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

В сравнение с празния файл, шаблонният файл автоматично съдържа `import SwiftUI`, `struct`, `body` и `#Preview`.

Затова за начинаещите използването на SwiftUI шаблонен файл е по-просто и е по-малко вероятно да се пропусне код.

**Съвет**: Освен чрез щракване с десния бутон за създаване на файл, можете също да щракнете върху бутона плюс в долната част на лявата зона Navigator на Xcode, за да създадете файл или папка.

![xcode](../../../Resource/027_view5.png)

## Създаване на папка

Когато файловете станат все повече, можем да използваме папки за управление.

В лявата зона Navigator на Xcode щракнете с десния бутон върху празно място или папка и изберете `New Folder`.

След създаване на папка можете да плъзнете свързаните файлове в нея.

![xcode](../../../Resource/027_view7.png)

Папките се използват основно за организиране на структурата на проекта и няма да променят начина на използване на самия изглед.

Стига името на изгледа да е правилно, той все още може да се използва в други файлове чрез `TestView()`, `Test2View()` и подобни начини.

## Изтриване на файл или папка

Ако искате да изтриете файл или папка, можете да изберете съответния файл в лявата зона Navigator на Xcode, след това да щракнете с десния бутон и да изберете `Delete`.

Можете също да изберете файла и след това да натиснете клавиша за изтриване `⌫`.

Xcode ще покаже прозорец за потвърждение на изтриването.

![xcode](../../../Resource/027_view10.png)

Изборът на `Move to Trash` означава файлът да бъде преместен в кошчето.

Ако видите `Remove Reference`, това означава само премахване на препратката от Xcode проекта, без да се изтрива оригиналният файл от диска.

За начинаещи, ако просто изтривате обикновен файл, който току-що сте създали, обикновено е достатъчно да изберете `Move to Trash`.

## Показване на няколко SwiftUI изгледа

След като се научим да създаваме файлове, можем да разделим различните изгледи в различни файлове.

Например създаваме два SwiftUI файла:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` се използва за показване на името на автора:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` се използва за показване на фона на картата:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

След това можем да използваме тези два изгледа в `ContentView`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

След стартиране можем да видим, че `CardView` и `AuthorView` са комбинирани и показани заедно:

![xcode](../../../Resource/027_view11.png)

В този резултат `CardView` служи като фон, а `AuthorView` се показва отгоре, защото са поставени в `ZStack`.

В `ContentView` използваме тези два изгледа чрез `CardView()` и `AuthorView()`. `()` след името означава създаване и използване на този изглед.

Това е често срещаното влагане на изгледи в SwiftUI: в един изглед можем да продължим да използваме други изгледи.

## Входен файл

Накрая ще преговорим откъде App започва да показва интерфейса.

В SwiftUI проект обикновено има входен файл:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Тук `@main` означава, че това е входната точка на App.

В `WindowGroup` виждаме:

```swift
ContentView()
```

Това означава, че след стартиране App първо показва `ContentView`.

След това в `ContentView` се показва още:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Затова йерархията на изгледите може да се разбира така:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

Ако променим изгледа, показван във входния файл, на `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

Тогава след стартиране App ще показва само `CardView`.

Тоест `WindowGroup` във входния код определя кой изглед се показва първи след стартиране на App.

А вътре в този изглед може да продължат да се показват други изгледи.

## Обобщение

В този урок научихме често използвани операции за управление на файлове в Xcode.

Разбрахме как да създаваме SwiftUI файлове, как да преименуваме файлове, как да създаваме папки и как да изтриваме файлове.

Същевременно научихме и важна концепция: SwiftUI изгледите могат да бъдат разделени в различни файлове и след това да се използват в други изгледи.

Например:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

Тук `CardView()` и `AuthorView()` означават използване на други изгледи в `ContentView`.

Накрая преговорихме и логиката на показване във входния файл.

При стартиране App първо влиза във входния код, маркиран с `@main`, след което показва изгледа в `WindowGroup`.

Това ни помага да разберем: различните SwiftUI файлове не са изолирани един от друг. Те могат да се комбинират взаимно и в крайна сметка да образуват цялостен интерфейс на App.
