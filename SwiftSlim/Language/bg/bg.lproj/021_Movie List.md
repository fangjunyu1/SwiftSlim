# Списък с филми

В този урок ще създадем списък с филми.

![movie](../../Resource/021_movie.png)

В списъка ще се показват плакатът, името, режисьорът и оценката на всеки филм.

В този пример ще научим една много важна тема: структурата `struct`. Тя може да ни помогне да обединим няколко части от информацията за един филм в едно цяло. Освен това ще се докоснем до `UUID()`, `ForEach`, разделителната линия `Divider` и до това как да използваме персонализирани обекти за управление на данни.

Тези знания са много често срещани в по-нататъшната работа със SwiftUI. След този урок не само ще можете да направите списък с филми, но и ще започнете да разбирате „как да превърнем група данни в интерфейс“.

## Един филм

Можем първо да започнем с интерфейса за един-единствен филм.

![movie](../../Resource/021_movie1.png)

Оформлението на този интерфейс се състои основно от две части: отляво е плакатът на филма, а отдясно е представянето на филма.

### Плакат на филма

Отляво се показва плакатът на филма, а за показването на изображението можем да използваме `Image`.

Например:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

Този код означава да се покаже изображение с име `"The Shawshank Redemption"`.

Тук трябва да обърнем внимание, че името на изображението трябва действително да съществува в папката с ресурси `Assets`, иначе интерфейсът няма да може да покаже правилно това изображение.

![movie](../../Resource/021_movie2.png)

Функциите на тези модификатори са съответно:

- `resizable()` означава, че размерът на изображението може да се променя.
- `scaledToFit()` означава, че при мащабиране се запазват оригиналните пропорции, за да не се разтяга и деформира изображението.
- `frame(height: 180)` означава, че височината на изображението се задава на 180.
- `cornerRadius(10)` означава, че на изображението се задава радиус на заобляне 10.

Така можем да получим плакат на филм с подходящ размер и заоблени ъгли.

![movie](../../Resource/021_movie3.png)

### Представяне на филма

Отдясно се показва представянето на филма, включително името на филма, режисьора и оценката.

![movie](../../Resource/021_movie4.png)

Можем да използваме `Text`, за да покажем тази част:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

В този код `.font` означава размера на шрифта, а `.fontWeight` означава дебелината на шрифта.

За да има по-ясно разстояние между „името на филма“, „информацията за режисьора“ и „информацията за оценката“, тук използваме `VStack` на най-външното ниво и задаваме:

```swift
spacing: 10
```

Това означава, че разстоянието между всяка група съдържание вътре в този `VStack` е 10.

Причината режисьорът и оценката да бъдат обвити поотделно в още един `VStack` е, че и двете са структура от типа „заглавие + съдържание“. Така йерархията на интерфейса ще бъде по-ясна и ще бъде по-удобно по-късно да продължим да настройваме стила.

Освен това `VStack` по подразбиране е централно подравнен. За да подравним целия текст вляво, задаваме:

```swift
alignment: .leading
```

Така частта с представянето на филма изглежда по-подредена.

### Разделителна линия

Сега, макар че съдържанието в представянето на филма е раздалечено чрез `spacing`, границата между различните части все още не е достатъчно ясна.

В този момент можем да добавим разделителна линия:

```swift
Divider()
```

Например:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

Така в частта с представянето на филма ще се появи по-ясен ефект на разделяне.

![movie](../../Resource/021_movie4.png)

`Divider` е много прост, но много често използван изглед, чиято роля е да разделя различно съдържание.

В `VStack` `Divider()` се показва като хоризонтална линия.

В `HStack` `Divider()` се показва като вертикална линия.

Освен това можем да променяме стила на `Divider` чрез `.frame`, `.background`, `.padding` и други начини.

Например:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

Това ще създаде синя разделителна линия с дебелина 2 и вътрешни отстояния отляво и отдясно.

![divider](../../Resource/021_divider.png)

### Пълен интерфейс

Накрая можем да използваме `HStack`, за да подредим плаката на филма и представянето на филма един до друг.

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

Тук `HStack(spacing: 20)` означава да се остави разстояние 20 между лявата и дясната част.

Трябва да се отбележи, че `Divider()`, както и разглежданите по-рано `TextField` и `Slider`, по подразбиране се стреми да заеме възможно най-много от наличното пространство.

Затова задаваме фиксирана ширина на външния `VStack` в дясната описателна част:

```swift
.frame(width: 200)
```

Така ширината на разделителната линия и текстовата област може да остане еднаква и цялостният вид ще е по-подреден.

Дотук завършихме интерфейса за показване на „един филм“.

![movie](../../Resource/021_movie1.png)

## Масив за съхраняване на филми

Ако искаме да показваме няколко филма един след друг в един и същ стил, това означава, че за всеки филм трябва да напишем подобен код.

Например:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

Макар че написаният по този начин код може да постигне желания ефект, кодът ще става все повече и повече, а поддръжката му ще бъде много трудна.

Например, ако трябва да променим разстоянието за всички филми от `20` на `15` или да променим ширината на дясната част от `200` на
`220`, ще трябва ръчно да редактираме всеки повтарящ се блок код.

Това очевидно не е ефективен подход.

По-рано вече научихме за масивите, а също така научихме как да използваме `ForEach`, за да показваме повторно изгледи според масив.

Например:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Този подход е много подходящ за случаи, в които има „само една стойност данни“, например група имена на изображения.

Но един филм не съдържа само една стойност. Той съдържа поне:

1. плакат на филма
2. име на филма
3. режисьор
4. оценка

Тоест един филм всъщност е група свързани данни, а не отделен низ.

Ако използваме само масиви, можем единствено да съхраняваме тази информация разделена:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

При показване ще трябва да разчитаме на един и същ индекс, за да съответстват елементите един на един:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

Проблемът при този начин на писане е, че поддръжката е много скъпа.

Защото данните за един филм са разпръснати в няколко масива и ако само в един масив има един елемент повече, един по-малко или редът не съвпада, резултатът при показване ще бъде грешен.

Особено при добавяне или изтриване на филм трябва едновременно да променяте няколко масива и е много лесно да пропуснете нещо.

И така, как можем да обединим информация като „име на филма, режисьор, оценка“ в едно цяло?

Тук трябва да използваме `struct`.

## Дефиниране на структура struct

В Swift `struct` означава „структура“.

Можете да я разбирате като „персонализиран тип данни“, който може да събере няколко свързани полета заедно и да ги превърне в едно цяло.

Основна употреба:

```swift
struct StructName {
	let name: String
}
```

В този код `struct` е ключова дума, която показва, че дефинираме структура. `StructName` е името на структурата.

Съдържанието в къдравите скоби са полетата, които тази структура съдържа, като за всяко поле трябва ясно да се посочат име и тип.

Обикновено името на `struct` започва с главна буква, например `Movie`, `Student`, `UserInfo`. Това е често срещан стил на именуване в Swift.

Можете просто да разбирате `struct` като празна кутия, а всяко поле е като запазено празно място в тази кутия.

Когато тези места все още не са попълнени, това е просто празна кутия. Едва след като всички полета бъдат попълнени с подходящи стойности, тя се превръща в завършена подаръчна кутия.

### Екземпляр на struct

По-рано само дефинирахме самата структура, което е равносилно на това да подготвим формата на опаковъчната кутия.

Трябва още да попълним конкретното съдържание, за да получим истински „екземпляр“, който може да се използва.

Когато създаваме екземпляр, обикновено добавяме `()` след името на структурата:

```swift
StructName(...)
```

Съдържанието, което се попълва в скобите, са именно стойностите на полетата, които тази структура изисква.

Например:

```swift
StructName(name: "Fang Junyu")
```

Този код означава: според формата на структурата `StructName` създаваме нов екземпляр и присвояваме на полето `name` стойността `"Fang Junyu"`.

Когато попълним всички необходими стойности на полетата, това е все едно да напълним тази кутия.

Тогава получаваме завършен екземпляр на структурата.

### Достъп до свойства на struct

След като създадем екземпляр, можем да използваме „dot syntax“, за да достъпваме свойствата му.

Основен запис:

```swift
екземпляр.имеНаСвойство
```

Например:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

Тук `st` е екземпляр на структура, а `st.name` означава да прочетем стойността на `name` в този екземпляр.

Можете да разбирате този начин на четене така: когато пишем `st.name`, това е все едно да прочетем съдържанието, съхранено на определено място в тази кутия.

Този начин на писане ще бъде много често срещан в интерфейсите на SwiftUI по-нататък.

Например, ако имаме структура за ученик, която съдържа име, възраст и клас, тогава при показване в интерфейса можем поотделно да четем:

```swift
student.name
student.age
student.className
```

Предимството на това е, че данните стават по-ясни и по-лесни за управление.

### Позиция на struct

За начинаещи обикновено е удобно структурата да се пише извън `ContentView`.

Например:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

Така структурата става по-ясна и се чете по-лесно.

Ще забележите, че самият `ContentView` всъщност също е `struct`.

### Структурата Movie

С тази основа вече можем да дефинираме структура за филм:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

Това е структура с име `Movie`, която се състои от трите полета `name`, `director` и `rating`, съответно представящи името на филма, режисьора и оценката.

Така информацията, свързана с един филм, вече не трябва да се разделя и пази в няколко масива, а може директно да се комбинира в едно цяло.

Например можем да създадем такъв екземпляр на филм:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

Този екземпляр представлява „пълни данни за един филм“.

### Масив за съхраняване на структури

По-рано казахме, че масивът може да съхранява само един и същ тип.

Сега вече имаме структурата `Movie`, така че масивът може да съхранява множество `Movie`.

```swift
let lists: [Movie] = []
```

Например:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

В този масив `lists` не се съхраняват низове, а множество екземпляри на `Movie`.

Тоест всеки елемент в масива е един завършен филм.

Така, когато трябва да покажем списъка с филми, можем чрез `ForEach` да прочетем всеки филм един по един.

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

Тук `movie in` означава, че при всяка итерация ще се взема по един филм от масива и временно ще се нарича `movie`.

След това можем да използваме:

```swift
movie.name
movie.director
movie.rating
```

за да прочетем името, режисьора и оценката на този филм и да ги покажем в интерфейса.

## Грешка на ForEach

Дотук вече завършихме основния начин на писане на масив с филми и `ForEach`.

Но ако директно стартирате следния код:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

ще се появи съобщение за грешка:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

Значението на тази грешка е, че когато напишете `id: \.self`, SwiftUI трябва да използва самия всеки елемент в масива като „уникален идентификатор“, за да ги различава.

Този начин е подходящ за прости типове като `String` и `Int`, защото те по природа лесно се различават.

Но `Movie` е персонализирана структура, която ние сме дефинирали, и `ForEach` не знае как да използва самата нея като уникален идентификатор, затова възниква грешка.

Най-често срещаният начин да решим този проблем е: да накараме `Movie` да следва протокола `Identifiable`.

## Протокол

В Swift протоколът може да се разбира като вид „правило“ или „изискване“.

Можете да го разбирате като международно споразумение: ако една държава иска да се присъедини към определена международна организация, обикновено първо трябва да изпълни някои правила, поставени от тази организация, да разкрие някои данни и да спазва някои общи правила. Едва след като отговори на тези условия, тя може да се присъедини или да участва в определени дейности.

Протоколите действат по подобен начин.

Когато даден тип иска да следва определен протокол, той трябва да изпълни съдържанието, изисквано от този протокол. Едва след като отговори на тези изисквания, този тип може да използва съответната функционалност.

Основен запис:

```swift
struct Movie: Identifiable {
    // ...
}
```

Тук `: Identifiable` означава, че `Movie` следва протокола `Identifiable`.

Ако един тип следва няколко протокола, можем да ги разделим със запетая:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

За `Identifiable` най-важното изискване е: типът да има `id`, с който да може да се идентифицира.

Например:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

Тук `id` е като номер на лична карта и се използва, за да различава всеки филм.

Докато `id` на всеки филм е различен, SwiftUI може правилно да различава всеки елемент в масива.

Например:

```swift
let lists: [Movie] = [
```

След това използваме в `ForEach`:

Така няма да има грешка.

Тъй като `Movie` вече следва `Identifiable`, по-често срещаният начин е директно да пропуснем `id:`:

```swift
ForEach(lists) { movie in
    // ...
}
```

Защото SwiftUI вече знае, че трябва да използва `movie.id`, за да различава всеки елемент.

## Идентификаторът за разграничаване на ForEach

По-долу ще разберем по-задълбочено ролята на `id` в `ForEach`.

Например:

```swift
ForEach(lists, id: \.self)
```

Това означава: използвай самия елемент като идентификатор за разграничаване.

А:

```swift
ForEach(lists, id: \.id)
```

означава: използвай полето `id` на елемента като идентификатор за разграничаване.

Ако дадено поле само по себе си може да гарантира уникалност, то също може временно да се използва.

Например, ако имената на всеки филм са различни, тогава следният запис също може да работи нормално с текущите данни:

```swift
ForEach(lists, id: \.name)
```

Но тук има един скрит проблем: `name` не е задължително винаги уникално.

Да предположим, че по-късно добавите още един филм със същото име, тогава `name` вече няма да може точно да различава всеки елемент.

В този момент, макар че кодът все още може да се компилира, при опресняване, обновяване, вмъкване или изтриване на изгледи SwiftUI може да разпознае елементите неправилно и това да доведе до аномалии в показването.

Затова, когато ни е нужно стабилно разграничаване на данните, най-добре е да използваме наистина уникалното поле, а именно `id`.

## UUID

Макар че ръчното писане на `id: 1`, `id: 2` може да реши проблема, все още има един риск: възможно е неволно да напишете повтарящо се `id`, при което уникалният идентификатор ще стане невалиден.

Например:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Така ще се появят две еднакви `id` и уникалният идентификатор ще стане невалиден.

За да избегнем ръчни грешки, обикновено използваме `UUID()`.

В Swift `UUID()` генерира на случаен принцип 128-битов идентификатор. Обикновено той се показва като дълъг низ от букви и цифри.

Например:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

Тук `UUID()` всъщност създава екземпляр от типа `UUID` и можете да го разбирате като автоматично генериран „уникален номер“.

Следователно стойностите, които се създават всеки път, почти никога не се повтарят, затова са много подходящи за използване като `id`.

Можем да променим `Movie` така:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Този код означава, че полето `id` в структурата `Movie` по подразбиране ще получи нов екземпляр на `UUID`.

Тоест всеки път, когато създаваме нов `Movie`, системата автоматично ще ни генерира уникално `id`.

`id` вече има стойност по подразбиране, затова при по-късно създаване на екземпляри на `Movie` повече няма нужда ръчно да попълваме `id`.

Например:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

Макар че тук не сме написали `id`, всъщност този екземпляр на филм пак има свое собствено `id`, просто тази стойност вече е генерирана автоматично от `UUID()`.

След като използваме този начин, вече няма да е нужно ръчно да попълваме `id` за всеки филм, което не само намалява кода, но и избягва грешки, породени от повтарящи се id стойности.

Накрая е нужно само да добавим една `Banner` картинка най-отгоре и да използваме `ScrollView`, за да може съдържанието да се превърта, и така ще завършим целия изглед със списъка с филми.

## Обобщение

В този урок научихме една много важна тема: `struct`.

Чрез `struct` можем да обединим няколко свързани полета на един филм в едно цяло, вместо да разделяме името, режисьора и оценката в няколко отделни масива.

След като данните бъдат комбинирани, можем да използваме `ForEach`, за да показваме всеки филм от масива един по един в изгледа.

Същевременно разбрахме и ролята на „идентификатора за разграничаване“ в `ForEach`: SwiftUI трябва да знае как да различава всеки елемент в масива, за да може правилно да показва и обновява изгледа.

Затова накарахме `Movie` да следва протокола `Identifiable` и му предоставихме уникално `id`.

За да избегнем грешки при ръчно попълване на `id`, допълнително научихме и `UUID()`, така че системата автоматично да генерира уникалния идентификатор.

След като завършите този урок, не само ще сте направили списък с филми, но и ще сте започнали да се докосвате до една важна идея в SwiftUI: **първо подреди данните, после генерирай изгледа според данните.**

## Пълен код

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
