# Нека бутонът започне да работи

Този урок е посветен основно на бутона `Button`, на `print` за дебъг изход и на функциите в Swift.

Тези теми са основата на интерактивното програмиране в SwiftUI.

## Бутон

Бутоните са едни от най-често срещаните UI контроли. Когато искаме да изтеглим, отворим, излезем или извършим друго действие, обикновено натискаме бутон.

Например в App Store натискаме бутона `Get`, за да изтеглим приложение.

![Button](../../RESOURCE/008_button.png)

В SwiftUI бутоните се представят чрез `Button`.

Основна употреба:

```swift
Button("") {
    
}
```

Структурата му може да бъде разделена на две части:

```swift
Button("Текст на бутона") {
    Кодът, който се изпълнява при натискане на бутона
}
```

Например:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

След стартиране ще се покаже бутон `Start`.

![Button](../../RESOURCE/008_button1.png)

Когато потребителят натисне бутона, кодът вътре в `{}` ще се изпълни.

Важно е да се отбележи, че `Button` е SwiftUI интерфейсен контрол, тоест е `View`, затова трябва да бъде написан вътре в `body`.

### Модификаторът `buttonStyle`

В SwiftUI `Button` предоставя вградени системни стилове за бутони, които могат да се използват чрез модификатора `buttonStyle`.

Например:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

`buttonStyle` прилага стил на бутон, предоставен от системата.

Често използвани опции са:

- `.automatic`
- `.plain`
- `.bordered`
- `.borderedProminent`
- `.borderless`

![Button](../../RESOURCE/008_button7.png)

Разликите между стиловете са главно в това дали бутонът има рамка, фон и колко силно е визуално подчертан.

## Изход с `print`

В програмирането често имаме нужда да проверим дали програмата е стигнала до определена част от кода.

Swift предоставя функцията `print` за извеждане на дебъг информация.

Основен синтаксис:

```swift
print("Hello")
```

Този ред извежда текста в `Console`.

### Тестване дали бутонът работи

Можем да поставим `print` вътре в бутона, за да проверим дали бутонът е натиснат.

```swift
Button("Start") {
    print("Test 123")
}
```

Когато бутонът бъде натиснат, `Console` ще изведе:

```text
Test 123
```

Това означава, че бутонът е задействан успешно.

![Button](../../RESOURCE/008_button3.png)

Забележка: `print` е Swift код, а не UI контрол. Той не може да бъде поставян директно в `body`, иначе ще доведе до грешка.

Например следният запис е грешен:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // Грешка: body трябва да връща View
    }
}
```

В SwiftUI `body` се използва, за да описва структурата на интерфейса, затова в него могат да стоят само `View` елементи като `Text`, `Button` и `Image`.

Обикновен код, като изчисления или `print`, трябва да бъде поставен в бутони, функции или други обработчици на събития.

### Console

Съдържанието, изведено чрез `print`, се появява в `Console` на Xcode.

Ако не виждаш `Console`, можеш да натиснеш бутона `Show the Debug Area` в долния десен ъгъл, за да отвориш дебъг зоната, а след това да натиснеш `Show the Console`, за да покажеш `Console`.

![](../../RESOURCE/008_button4.png)

В долния ляв ъгъл на `Console` има два таба: `Executable` и `Previews`.

![](../../RESOURCE/008_button5.png)

Тези два таба съответстват на различни среди на изпълнение: когато правиш преглед в `Canvas`, изходът от `print` се показва в таба `Previews`; ако приложението работи в симулатора или на реално устройство, изходът се показва в таба `Executable`.

Затова ако ти се струва, че `print` не показва нищо, първо провери дали гледаш правилния таб.

## Пример - ASCII art

В някои кодови проекти често можем да видим ASCII art.

ASCII art е рисунка, направена от обикновени символи, например:

![ASCII](../../RESOURCE/008_ascii.png)

Можем да използваме бутон плюс `print`, за да изведем ASCII рисунка.

Примерен код:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

Когато бутонът бъде натиснат, `Console` ще изведе:

```text
 /\_/\
( o.o )
 > ^ <
```

Забележка: в низ символът `\` трябва да бъде escape-нат, защото обратната наклонена черта е escape символ в низовете.

## Функции

Вътре в бутон можем директно да пишем код.

Например:

```swift
Button("Start") {
    print("Hello")
}
```

Ако кодът е малко, това е напълно нормално. Но в реалната разработка логиката, която се изпълнява при натискане на бутон, може да стане доста по-дълга.

Например:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // Може да има още много код
}
```

Ако кодът продължи да нараства, писането му директно вътре в бутона ще направи интерфейсния код разхвърлян и труден за четене или поддръжка.

Затова обикновено организираме тази логика във функция и след това извикваме функцията от бутона.

### Какво е функция?

Функцията е блок от код, който може да бъде използван многократно.

Когато даден код трябва да се изпълни, е достатъчно просто да извикаме функцията и кодът вътре в нея ще се изпълни.

Това прави структурата на кода по-ясна и улеснява повторната му употреба.

### Основен синтаксис

В Swift функциите се дефинират с ключовата дума `func`:

```swift
func randomInt() {
    // code
}
```

Този код дефинира функция.

`randomInt` е името на функцията, което служи за разпознаване на този блок код.

`()` е мястото за параметри, откъдето функцията може да получава данни, подадени отвън. Ако не са нужни данни, може да остане празно.

Вътре в `{}` се намира тялото на функцията, където се пише целият код, който трябва да се изпълни.

Например:

```swift
func getName() {
    print("FangJunyu")
}
```

Задачата на тази функция е да изведе един текст.

### Извикване на функция

Дефинирането на функция само я създава. Ако искаме тя действително да се изпълни, трябва да я извикаме.

Начинът за извикване на функция е да добавим `()` след името ѝ:

```swift
getName()
```

Когато програмата стигне до този ред, тя ще изпълни кода вътре в `getName`.

Пълен пример:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

След стартиране `Console` ще изведе:

```text
FangJunyu
```

### Използване на функции в SwiftUI

В SwiftUI изглед функциите обикновено се пишат извън `body`.

Например:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

Когато бутонът бъде натиснат, той извиква функцията `getName()`.

Така кодът на интерфейса остава изчистен, а конкретната логика се обработва във функцията.

### Параметри на функция

Понякога една функция трябва да извършва различни действия според различни данни.

Например може да искаме функцията да извежда различни имена, вместо винаги да извежда фиксирания низ `"FangJunyu"`.

В такъв случай можем да използваме параметри. Параметърът може да бъде разбран като данни, които се подават при извикване на функцията.

Например:

```swift
func getName(name: String) {
    print(name)
}
```

Тази функция добавя един параметър. `name` е името на параметъра, а `String` е типът му.

Това означава, че функцията трябва да получи данни от тип `String`.

Тъй като функцията вече има нужда от данни, при извикването ѝ трябва задължително да подадем такива.

```swift
getName(name: "Sam")
```

Когато подадем `"Sam"` при извикването, функцията ще използва тази стойност за извеждане.

Резултатът от изпълнението е:

```text
Sam
```

## Пример - използване на функция за ASCII art

В предишния пример написахме `print` директно вътре в бутона.

Но ако ASCII рисунката е по-сложна, кодът ще стане доста дълъг. В такъв случай можем да преместим логиката във функция, така че интерфейсният код да остане по-ясен.

Примерен код:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

След стартиране, когато бутонът бъде натиснат, `Console` ще изведе:

```text
 /\_/\
( o.o )
 > ^ <
```

Тук се случват две неща, когато потребителят натисне бутона:

1. бутонът извиква функцията `printCat()`
2. кодът с `print` вътре във функцията се изпълнява

Така интерфейсният код отговаря само за задействането на събитието, а конкретната логика се обработва във функция.

В реалната разработка повечето действия на бутоните се реализират именно чрез функции.

## Обобщение

В този урок научихме три важни неща:

**1. Бутон `Button`**

`Button` е контролът в SwiftUI, който се използва за задействане на действия.

```swift
Button("Start") {

}
```

Когато потребителят натисне бутона, кодът в скобите се изпълнява.

**2. Изход с `print`**

`print` се използва за извеждане на дебъг информация в `Console`.

```swift
print("Hello")
```

Разработчикът може да следи състоянието на програмата чрез `Console`.

**3. Функции**

Функцията е блок код, който може да бъде използван многократно.

```swift
func sayHello() {
    print("Hello")
}
```

Извикване на функция:

```swift
sayHello()
```

Ако функцията има нужда от данни, можем да използваме параметри:

```swift
func sayHello(name: String) {
    print(name)
}
```

В следващите уроци ще продължим да учим още SwiftUI контроли, както и как данните взаимодействат с интерфейса.
