# Изглед за вход

В този урок ще научим един много практичен сценарий: изглед за вход.

Независимо дали става дума за уебсайт или приложение, по време на използване често се налага потребителят да влезе в профила си и да въведе парола.

Например страницата за вход в GitHub:

![WordPress](../../Resource/019_github.png)

В този урок ще създадем подобен изглед за вход, така че потребителят да въвежда ръчно потребителско име и парола, а ние да проверяваме дали има проблем с въведеното съдържание.

## Горна подредба

Ще напишем кода за изгледа за вход във файла `ContentView`.

Първо, нека създадем зоната с логото в горната част на изгледа за вход. Можеш предварително да подготвиш подходяща икона и да я поставиш в папката `Assets`.

![icon](../../Resource/019_icon1.png)

След това използвай `Image` и модификатори, за да покажеш изображението:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

Тези няколко модификатора означават:

- `resizable()`: позволява размерът на изображението да се променя.
- `scaledToFit()`: мащабира изображението, като запазва оригиналните му пропорции.
- `frame(width: 100)`: задава ширината на показваното изображение на `100`.

След това използвай `Text` и модификатори, за да покажеш заглавието за вход:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

После използвай `VStack`, за да подредиш изображението и текста вертикално:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

Резултат:

![view](../../Resource/019_view.png)

До тук завършихме заглавната зона в горната част на изгледа за вход.

### Оптимизиране на подредбата

В момента изображението и заглавието по подразбиране се намират в средната част на цялата подредба.

Ако искаме да изглеждат повече като "знак за вход в горната част на страницата", можем да използваме `Spacer()`, за да разтеглим оставащото пространство и да приближим съдържанието до върха.

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

Резултат:

![view](../../Resource/019_view1.png)

Сега обаче `Image` и `Text` са твърде близо до горната част на екрана и изглеждат малко претрупани.

Тук можем да използваме `padding`, за да добавим горен отстъп към целия `VStack`.

```swift

        .scaledToFit()
```

## Потребителско име и парола

Страницата за вход обикновено изисква потребителят да въведе потребителско име и парола.

В SwiftUI можем да използваме `TextField`, за да приемаме съдържанието, въведено от потребителя.

Но трябва да обърнем внимание: `TextField` сам по себе си не пази въведените данни дългосрочно, той е само контрола за въвеждане. Това, което наистина съхранява въведеното, са променливите, които сме свързали с него.

Затова първо трябва да създадем две променливи `@State`, които да пазят потребителското име и паролата:

```swift
@State private var user = ""
@State private var password = ""
```

Когато стойността на променлива `@State` се промени, SwiftUI автоматично обновява свързаните изгледи.

След това използвай `TextField`, за да свържеш тези две променливи:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

Тук `$user` и `$password` означават "binding".

Тоест, когато потребителят въведе текст в полето, стойността на променливата се променя едновременно; а когато стойността на променливата се промени, съдържанието, показано в полето, също се променя синхронно.

Тази връзка, при която "изгледът и данните се синхронизират взаимно", се нарича binding.

Обърни внимание, че тук трябва да се използва записът с `$`:

```swift
$user
```

Защото `TextField` не се нуждае от обикновен низ, а от стойност за binding, която "може да променя данните двупосочно".

### Оптимизиране на полетата за въвеждане

Сега полетата за въвеждане вече могат да се използват, но стилът по подразбиране е сравнително прост.

За да бъде интерфейсът по-ясен, можем да добавим заглавие пред полето за въвеждане и да направим малка стилистична оптимизация на самото поле.

Например първо добави заглавие:

```swift
Text("Username")
    .fontWeight(.bold)
```

След това използвай `HStack`, за да поставиш заглавието и полето за въвеждане на един и същи ред:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

Тук използваме `HStack`, защото искаме "вляво да се показва заглавието, а вдясно полето за въвеждане".

Трябва да се отбележи, че `TextField` по подразбиране заема оставащото налично пространство.

![color](../../Resource/019_view6.png)

За да бъдат размерите на двете полета за въвеждане по-еднакви, можем да използваме `frame(width:)`, за да зададем ширина и полетата да изглеждат по-подредени.

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

После добави и рамка, за да изпъква полето повече:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Сега нека добавим и потребителското име, и паролата към изгледа `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

## Логика за вход

Сега ще добавим към бутона за вход най-основната логика за проверка: **да се провери дали потребителят е въвел потребителско име и парола**.

Искаме да реализираме следните случаи:

- Ако потребителят не е въвел нищо, да се изведе `Empty`
- Ако потребителят е въвел само едно от двете, да се подскаже кое липсва
- Ако и потребителското име, и паролата са въведени, да се изведе `Success`

### Няма въведена никаква информация

Тъй като `user` и `password` по подразбиране са празни низове:

```swift
@State private var user = ""
@State private var password = ""
```

ако потребителят не въведе нищо, те ще останат празни.

В Swift типове като `String` могат да използват `isEmpty`, за да се прецени дали съдържанието е празно.

### Свойството isEmpty

`isEmpty` често се използва за проверка дали низове, масиви и друго съдържание са празни.

Например:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

Ако съдържанието е празно, `isEmpty` връща `true`; ако съдържанието не е празно, `isEmpty` връща `false`.

Затова можем да го използваме, за да проверим дали са въведени потребителско име и парола.

### Използване на isEmpty за проверка на променливи

Ако потребителят не е въвел никаква информация, тогава:

```swift
user.isEmpty // true
password.isEmpty    // true
```

Тогава можем да напишем проверката така:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

Този код означава: ако `user` е празен и `password` също е празен, изведи:

```swift
Empty
```

`&&` тук е логически оператор и означава "и".

Тоест цялата проверка е изпълнена само когато условието отляво е вярно и условието отдясно също е вярно.

Следователно този код ще се изпълни само когато и потребителското име, и паролата са празни.

## Потребителят е попълнил само част

След това разглеждаме друга ситуация: потребителят е попълнил само потребителското име или само паролата.

Например:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

В този момент потребителското име не е празно, но паролата е празна.

Можем да продължим да използваме `isEmpty`, за да проверим:

### Случай, в който потребителят е попълнил акаунт или парола

Ако потребителят е попълнил само потребителското име или само паролата, трябва да дадем съответното съобщение за липсващото поле.

Например, ако потребителят е попълнил само потребителското име:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Логиката на този код е: ако потребителското име е празно, подскажи да се въведе потребителско име. В противен случай провери дали паролата е празна и ако е празна, подскажи да се въведе парола.

### Потребителят е попълнил цялата информация

Ако и потребителското име, и паролата вече са въведени, `isEmpty` и за двете ще върне `false`.

Тогава можем да го напишем така:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

`!` тук означава "отрицание".

Например:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

Тоест `user.isEmpty` се използва, за да се провери "дали потребителското име е празно", а `!user.isEmpty` се използва, за да се провери "дали потребителското име не е празно".

Същата логика важи и за паролата.

Следователно смисълът на този код е: потребителското име не е празно и паролата също не е празна.

Когато и двете условия са изпълнени, се извежда:

```swift
Success
```

## Пълна логика за проверка

Сега ще комбинираме тези три случая и ще ги запишем в `Button`:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

Последователността на изпълнение на този код може да се разбере така:

Първата стъпка е да се провери:

```swift
if user.isEmpty && password.isEmpty
```

Ако и потребителското име, и паролата са празни, директно се извежда `Empty`.

Ако и двете не са празни, продължаваме с преценката на следващия ред:

```swift
else if user.isEmpty || password.isEmpty
```

`||` тук означава "или".

Тоест стига едно от условията да е изпълнено, цялото условие е изпълнено.

Следователно, щом потребителското име е празно или паролата е празна, ще се изпълни блокът код на `else if`.

След като влезем в блока, правим допълнителна проверка кое точно поле не е попълнено:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

Ако нито едно от предишните две условия не е изпълнено, това означава, че потребителското име не е празно и паролата също не е празна.

Тогава програмата ще влезе в последния `else`:

```swift
print("Success")
```

Така завършихме най-основната логика за проверка на входните данни при вход.

## Проверка на потребителско име и парола

Горната логика проверява само "дали има въведено съдържание".

Но в реално приложение не е достатъчно само да има въведено съдържание. Обикновено трябва също да изпратим потребителското име и паролата към сървъра за проверка.

Ако сървърът потвърди, че потребителското име съществува и паролата е правилна, ще позволи вход; в противен случай ще покаже, че входът е неуспешен.

За да упражним този процес, можем първо да зададем временен акаунт и парола в кода, за да симулираме "правилни данни за вход":

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

Когато и потребителското име, и паролата не са празни, след това ги сравняваме дали са правилни:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

`==` тук означава "дали е равно".

Например:

```swift
user == userID
```

Това означава: дали потребителското име, въведено от потребителя, е напълно еднакво с правилното потребителско име, което сме задали. Проверката за паролата следва същата логика.

Следователно, когато въведените потребителско име и парола са равни на зададените стойности, се извежда `Success`; стига едното от тях да не е равно, се извежда `Input error`.

Така завършихме една проста "симулация на проверка при вход".

Макар че това все още не е истинска функция за вход, свързана със сървър, то вече е напълно достатъчно, за да ни помогне да разберем: **след натискане на бутона може да се изпълнява различен код според различни условия**.

## Обобщение

В този урок завършихме базов изглед за вход и се съсредоточихме върху изучаването на "логиката за проверка на въведеното според условията".

Основното съдържание в този урок е: използване на `isEmpty`, за да се провери дали низът е празен, и използване на `if` заедно с логически оператори, за да се обработват различни ситуации.

Сред тях два много важни логически оператора са:

- `&&`: означава "и", и двете условия трябва да са изпълнени
- `||`: означава "или", достатъчно е едно условие да е изпълнено

Когато тези условия накрая дадат `true` или `false`, операторът `if` ще изпълни различен код според резултата.

Чрез този изглед за вход вече започнахме да се докосваме до начина на разработка, който съчетава "интерфейс + данни + логически проверки".

```swift
                    .frame(width: 200)

            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)

                        print("Please enter your password.")
                    }
                } else {

                        print("Success")
                    } else {

                    }

    }
}

```
