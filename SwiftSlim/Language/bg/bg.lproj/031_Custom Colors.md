# Персонализирани цветове

В този урок ще научим как да създаваме персонализирани цветове в SwiftUI.

В предишните уроци вече използвахме стандартните цветове, предоставени от SwiftUI, например:

```swift
Color.blue
Color.red
Color.green
```

Тези цветове са удобни за използване, но в реалната разработка на App стандартните цветове често не са достатъчно точни.

Например в дизайн макет може да се използват такива цветове:

```text
#2c54c2
#4875ed
#213e8d
```

Този тип цвят се нарича Hex цвят.

В този урок първо ще направим така, че SwiftUI да поддържа Hex цветове, а след това ще използваме `static`, за да подредим често използваните цветове.

Накрая ще приложим персонализираните цветове към изгледа на енциклопедията за животни и допълнително ще използваме градиентен фон, за да изглеждат бутоните по-слоести.

## Защо са нужни персонализирани цветове?

В SwiftUI можем директно да използваме системни цветове.

Например:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

Тук `.blue` означава син цвят. Всъщност това е съкратен запис на `Color.blue`.

Предимството на стандартните цветове е, че са прости и удобни, но изборът на цветове е сравнително ограничен.

Например:

```swift
Color.blue
```

Той може да представя само стандартното синьо, предоставено от SwiftUI.

![Color.blue](../../../Resource/031_color6.png)

Но в реалната разработка често имаме нужда от по-конкретни цветове.

Например, дори всички те да са сини, може да има различни ефекти като светлосиньо, тъмносиньо, сиво-синьо и яркосиньо.

![More Blue](../../../Resource/031_color5.png)

В този момент, ако използваме само `Color.blue`, ще бъде трудно да възпроизведем ефекта от дизайна.

Затова трябва да направим така, че SwiftUI да поддържа персонализирани цветове.

## Какво е Hex цвят?

Цветовете на екрана обикновено се състоят от три канала: червен, зелен и син, тоест RGB.

RGB съответно означава:

```text
Red     // червено
Green   // зелено
Blue    // синьо
```

Hex цветът е един от начините за представяне на RGB цвят.

Например:

```swift
#5479FF
```

Тази стойност на цвят може да се разбере просто като три части:

```text
54  // представлява червения канал
79  // представлява зеления канал
FF  // представлява синия канал
```

В този урок не е нужно да изчисляваме тези стойности и не е нужно да разбираме в дълбочина правилата на шестнадесетичната бройна система.

Сега е достатъчно да знаем: `#5479FF` представлява конкретен цвят.

В бъдеще, когато видим записи като `#2c54c2` или `#4875ed`, можем първо да ги разбираме като стойност на цвят.

В дизайн инструменти като Sketch, Figma и Photoshop също често могат да се видят подобни стойности на цветове.

![color](../../../Resource/031_color.png)

Но по подразбиране SwiftUI не може директно да се пише така:

```swift
Color(hex: "#5479FF")
```

Затова трябва сами да разширим типа `Color`, за да поддържа създаване на цветове чрез Hex низ.

## Създаване на файла Color+Hex.swift

Първо създаваме нов Swift файл.

Името на файла може да бъде:

```text
Color.swift
```

Може да бъде и по-ясното:

```text
Color+Hex.swift
```

Тук е по-препоръчително да използваме:

```text
Color+Hex.swift
```

В Swift проект имена на файлове като `Color+Hex.swift` са много често срещани.

Това означава, че този файл е разширение, което добавя Hex възможност към `Color`.

Името на файла няма директно да повлияе на изпълнението на кода; то само ни помага по-лесно да разберем предназначението на файла.

## Добавяне на кода Color(hex:)

Във файла `Color+Hex.swift` въведете следния код:

```swift
import SwiftUI

extension Color {
    init(hex: String) {
        let scanner = Scanner(string: hex)
        
        if hex.hasPrefix("#") {
            scanner.currentIndex = hex.index(after: hex.startIndex)
        }
        
        var rgbValue: UInt64 = 0
        scanner.scanHexInt64(&rgbValue)
        
        let red = Double((rgbValue & 0xFF0000) >> 16) / 255.0
        let green = Double((rgbValue & 0x00FF00) >> 8) / 255.0
        let blue = Double(rgbValue & 0x0000FF) / 255.0
        
        self.init(red: red, green: green, blue: blue)
    }
}
```

Този код използва `extension`, за да разшири типа `Color`, и добавя към `Color` нов метод за инициализация:

```swift
init(hex: String)
```

След като имаме този метод за инициализация, можем да подадем Hex низ към `Color` и така да създадем персонализиран цвят:

```swift
Color(hex: "#5479FF")
```

Вътрешно този код на разширението преобразува Hex низа в RGB цвят, който SwiftUI може да разпознае.

На този етап не е нужно да разбираме в дълбочина логиката на преобразуване във всеки ред. Достатъчно е да знаем, че добавеният метод `Color(hex:)` ни позволява да създаваме персонализирани цветове чрез Hex стойности.

## Използване на персонализирани цветове

Сега можем да тестваме персонализирани цветове в `ContentView`.

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("Color.blue")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color.blue)
            
            Text("#2c54c2")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#2c54c2"))
            
            Text("#4875ed")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#4875ed"))
            
            Text("#213e8d")
                .font(.largeTitle)
                .fontWeight(.bold)
                .foregroundStyle(Color(hex: "#213e8d"))
        }
        .padding()
    }
}
```

Ефект на показване:

![color](../../../Resource/031_color1.png)

В този пример първият ред използва системен цвят:

```swift
.foregroundStyle(Color.blue)
```

Следващите три реда използват персонализирани Hex цветове:

```swift
.foregroundStyle(Color(hex: "#2c54c2"))
.foregroundStyle(Color(hex: "#4875ed"))
.foregroundStyle(Color(hex: "#213e8d"))
```

Чрез сравнение можем да видим, че системното синьо има само един стандартен ефект.

А Hex цветът може да представя по-фини промени в синьото.

Това е стойността на персонализираните цветове: те могат да направят цветовете на интерфейса по-близки до реалния дизайн и също така да ни помогнат по-удобно да контролираме визуалния стил на App.

## Използване на static за подреждане на цветове

Сега вече можем да създаваме цветове чрез Hex низ:

```swift
Color(hex: "#2c54c2")
```

Този запис може да се използва нормално, но ако един и същ цвят се появява многократно на различни места, поддръжката по-късно няма да е много удобна.

Ако напишем тази цветова стойност на 10 места, когато по-късно искаме да променим това синьо, ще трябва да я променяме едно по едно.

В този момент можем да използваме `static`, за да подредим често използваните цветове на едно място.

В долната част на файла `Color+Hex.swift` продължете да добавяте следния код:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Тук добавихме към `Color` три статични свойства: `animalBlue`, `animalLightBlue` и `animalDarkBlue`. Те съответно представят различни нюанси на синьото.

Тъй като тези свойства използват `static`, те принадлежат на самия тип `Color`.

При използване можем директно да ги достъпваме чрез `Color.`:

```swift
Color.animalBlue
```

Този запис е по-ясен от директното писане на Hex низ.

Когато видим `Color.animalBlue`, можем да знаем, че той представлява синьото, използвано в енциклопедията за животни.

А когато видим `Color(hex: "#2c54c2")`, можем само да знаем, че това е цветова стойност, но не е лесно да видим конкретното ѝ предназначение.

Единното управление на цветовете има още едно предимство: по-късните промени са по-удобни.

Ако искаме да коригираме основния цвят на енциклопедията за животни, трябва да променим само мястото на дефиницията:

```swift
static let animalBlue = Color(hex: "#1f5cff")
```

Всички места, които използват този цвят, ще се обновят заедно.

Това е смисълът на използването на `static` за подреждане на цветове: имената на цветовете стават по-ясни, а по-късната поддръжка става по-удобна.

## Прилагане към енциклопедията за животни

Сега можем да приложим персонализираните цветове към предишния изглед на енциклопедията за животни.

Преди фонът на бутона за животно беше бял:

```swift
.background(Color.white)
```

![single white button](../../../Resource/031_color7.png)

Сега можем да го променим на персонализиран цвят:

```swift
.background(Color.animalBlue)
```

![single blue button](../../../Resource/031_color8.png)

Тук `Color.animalBlue` е статичният цвят, който току-що дефинирахме в `Color+Hex.swift`.

Текстът на бутона използва бяло:

```swift
.foregroundStyle(Color.white)
```

Фонът на animal emoji използва полупрозрачно бяло:

```swift
.background(Color.white.opacity(0.15))
```

Така бутонът ще оформи единен син визуален стил.

Фокусът на тази стъпка не е да добавяме сложен код, а наистина да приложим персонализираните цветове, които научихме по-рано, в интерфейса.

## Използване на градиентен фон

Освен да използваме един цвят, можем също да комбинираме няколко цвята в градиентен ефект.

Например по-рано дефинирахме тези цветове:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
    static let animalLightBlue = Color(hex: "#4875ed")
    static let animalDarkBlue = Color(hex: "#213e8d")
}
```

Тези цветове могат не само да се използват самостоятелно, но и да се комбинират като градиентен фон.

В SwiftUI можем да използваме `LinearGradient`, за да създадем линеен градиент.

Например:

```swift
LinearGradient(
    colors: [Color.animalBlue, Color.animalLightBlue],
    startPoint: .leading,
    endPoint: .trailing
)
```

Този код ще създаде градиентен фон отляво надясно, като цветът постепенно преминава от `Color.animalBlue` към `Color.animalLightBlue`.

В него `colors` се използва за задаване на цветовете, участващи в градиента, а `startPoint` и `endPoint` се използват за контрол на посоката на градиента.

### Тестване на градиентния ефект

Можем да използваме прост Text, за да тестваме градиентния фон.

```swift
Text("Linear Gradient")
    .font(.title)
    .fontWeight(.bold)
    .padding()
    .foregroundStyle(Color.white)
    .background {
        LinearGradient(
            colors: [Color.animalBlue, Color.animalLightBlue],
            startPoint: .leading,
            endPoint: .trailing
        )
    }
    .cornerRadius(12)
```

Ефект на показване:

![color](../../../Resource/031_color3.png)

В този пример фонът на `Text` вече не е единичен цвят, а градиентен цвят, който постепенно се променя отляво надясно.

В сравнение с обикновения фонов цвят, градиентният фон има повече слоеве и по-лесно създава визуален акцент в интерфейса.

## Използване на градиентен фон в енциклопедията за животни

Сега можем да вземем фоновия цвят на бутона за животно:

```swift
.background(Color.animalBlue)
```

И да го променим на градиентен фон:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .leading,
        endPoint: .trailing
    )
}
```

Пълният код е следният:

```swift
import SwiftUI

struct ContentView: View {
    @State private var selectedAnimal: Animal? = nil
    
    let animals = Animal.animals
    
    var body: some View {
        VStack(alignment: .leading, spacing: 20) {
            ForEach(animals) { animal in
                Button(action: {
                    selectedAnimal = animal
                }, label: {
                    HStack(spacing: 10) {
                        Text(animal.avatarEmoji)
                            .font(.title2)
                            .frame(width: 40, height: 40)
                            .background(Color.white.opacity(0.15))
                            .cornerRadius(20)
                        
                        Text(animal.name)
                            .fontWeight(.bold)
                            .foregroundStyle(Color.white)
                        
                        Spacer()
                    }
                    .padding(15)
                    .frame(width: 200, height: 60)
                    .background {
                        LinearGradient(
                            colors: [Color.animalBlue, Color.animalLightBlue],
                            startPoint: .leading,
                            endPoint: .trailing
                        )
                    }
                    .cornerRadius(12)
                    .shadow(radius: 5)
                })
                .buttonStyle(.plain)
            }
        }
        .padding()
        .sheet(item: $selectedAnimal) { animal in
            AnimalDetailView(animal: animal)
        }
    }
}
```

Ефект на показване:

![color](../../../Resource/031_color4.png)

Сега бутонът за животно вече не е само с единичен фонов цвят, а има градиентен ефект отляво надясно.

В сравнение с едноцветния фон, градиентният фон може да направи интерфейса по-слоест и по-близък до визуалния дизайн в реални App.

## Разлика между едноцветен фон и градиентен фон

Едноцветният фон използва само един цвят.

Например:

```swift
.background(Color.animalBlue)
```

Този запис е прост и ясен и е подходящ за повечето базови интерфейси.

Градиентният фон използва няколко цвята.

Например:

```swift
.background {
    LinearGradient(
        colors: [Color.animalBlue, Color.animalLightBlue],
        startPoint: .topLeading,
        endPoint: .bottomTrailing
    )
}
```

Този запис може да направи интерфейса по-слоест, но също така по-лесно увеличава визуалната сложност.

Затова в реалната разработка можем да го разбираме така:

Едноцветният фон е подходящ за обикновен текст, обикновени бутони, обикновени карти и обикновен фон на страница.

Градиентният фон е подходящ за важни бутони, горни области, обложки на карти, входове към функции и други позиции, които трябва да бъдат подчертани.

## Обобщение

В този урок научихме как да използваме персонализирани цветове в SwiftUI.

Първо разбрахме Hex цветовете.

Например:

```text
#2c54c2
```

Той представлява конкретен цвят.

След това използвахме `extension Color`, за да разширим типа `Color`.

Така SwiftUI може да създава цветове по следния начин:

```swift
Color(hex: "#2c54c2")
```

След това използвахме `static`, за да подредим често използваните цветове:

```swift
extension Color {
    static let animalBlue = Color(hex: "#2c54c2")
}
```

Така в други изгледи можем директно да използваме:

```swift
Color.animalBlue
```

В сравнение с писането на Hex низ всеки път, този начин е по-ясен и също така по-удобен за единна промяна на цветовете по-късно.

Накрая научихме `LinearGradient` и комбинирахме персонализираните цветове в градиентен фон:

```swift
LinearGradient(
    colors: [Color.animalBlue, Color.animalLightBlue],
    startPoint: .leading,
    endPoint: .trailing
)
```

Чрез този урок завършихме процеса от „създаване на персонализирани цветове“ до „използване на цветове в реален интерфейс“.

В бъдеще, когато един цвят трябва да се използва многократно на различни места, можем първо да помислим да го подредим в разширение на `Color`.

Така кодът ще бъде по-ясен, а стилът на интерфейса ще бъде по-лесен за уеднаквяване.
