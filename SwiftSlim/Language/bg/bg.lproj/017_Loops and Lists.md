# Цикли и списъци

В този урок ще научим `for` цикъла в Swift, както и `ForEach` в SwiftUI.

И двете могат да ни помогнат да прочитаме елементите в масив по ред. В реалната разработка често се налага да вземаме група от данни един по един елемент, а след това да ги обработваме или показваме. Именно цикълът е важният инструмент, с който това се прави.

Когато разглеждаме уеб страници или използваме приложение, често виждаме много списъци или отделни елементи. Такова съдържание обикновено първо се съхранява в `масив`, след това се прочита последователно чрез `цикъл`, и накрая се показва в интерфейса.

Например, списъкът с резултати от търсене в Google:

![Google](../../Resource/017_google.png)

Всеки елемент в списъка с резултатите от търсене обикновено съответства на една част от данните и се показва един по един чрез цикъл.

Затова `масивите` и `циклите` често се появяват заедно. `Масивът` отговаря за съхраняването на група от данни, а `цикълът` отговаря за това да извлича тези данни последователно и да изпълнява една и съща операция.

Нека първо започнем с най-основния `for` цикъл.

## Обратно броене

Да предположим, че искаме да симулираме функция за обратно броене и последователно да изведем `„5, 4, 3, 2, 1“`.

Най-директният начин е:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

Този начин може да симулира обратно броене, но ако съдържанието е повече, той ще изглежда много повтарящ се, а и по-късно няма да е удобен за промяна.

Тогава можем първо да поставим данните в масив, а след това да използваме `for` цикъл, за да ги изведем една по една:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Този запис е по-кратък и по-лесен за разширяване. Дори ако числата в обратното броене са много, няма нужда да пишем `print` ред по ред. Нужно е само да оставим цикъла автоматично да обработи всеки елемент в масива.

Можем да поставим този код в бутон, за да го изпробваме:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

Когато натиснем бутона, `for` цикълът ще вземе всяко цяло число в `countDown` по ред и ще го отпечата в конзолата.

След това нека разгледаме как работи `for` цикълът.

## for цикъл

`for` цикълът се използва за последователна обработка на група от данни. Най-често това са масиви и диапазони.

Основен запис:

```swift
for елемент in масивИлиДиапазон {
    кодЗаИзпълнение
}
```

Например, кодът за обратното броене отпреди малко:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

Тук `i` представлява текущия извлечен елемент, а след `in` стои данните, върху които се прави цикълът, тоест масивът `countDown`.

Смисълът на този код е следният: програмата ще вземе всеки елемент от масива `countDown` по ред. Всеки път, когато вземе един елемент, временно ще го представи с `i`, а след това ще изпълни кода в къдравите скоби.

Процесът на изпълнение може да се разбере така:

* Първият извлечен елемент е `5`, така че в този момент стойността на `i` е `5`
* Изпълнява се `print(i)` и се извежда `5`
* След това се извлича следващият елемент `4`
* Отново се изпълнява `print(i)` и се извежда `4`

Следващите `3`, `2` и `1` също ще бъдат изпълнени по същия начин.

![for](../../Resource/017_for.png)

Когато всички елементи в масива бъдат обработени, `for` цикълът автоматично приключва.

Чрез `for` цикъла можем да вземем всеки елемент от масива и да реализираме функции като това обратно броене.

**Подсказка**: `i` е много често използвано име във `for` цикъл, с което се представя текущият извлечен елемент, но то не е задължително. Можете да използвате и други английски имена.

Например, може да използваме `num`, за да представим всеки извлечен елемент:

```swift
for num in countDown {
    print(num)
}
```

Вътре в къдравите скоби също трябва да използваме `num`, за да представяме текущия елемент.

### Цикъл по диапазон

`for` цикълът може да обхожда не само масиви, но и диапазони.

Например, когато искаме да изчислим сумата на положителните цели числа до `50`, трябва да съберем всички цели числа между `1` и `50` едно по едно.

Най-директният запис може да изглежда така:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

Проблемът при този запис е, че трябва ръчно да изпишем всяко число. Това е досадно и не е подходящо за реални сценарии в разработката.

Тогава можем да използваме оператора за диапазон заедно с `for` цикъл, за да постигнем това.

В предишния урок за „Оператори“ вече научихме оператора за диапазон:

```swift
1...3   // 1, 2, 3
```

Тук `...` означава затворен диапазон, тоест включва началната и крайната стойност.

Затова тук можем да използваме `for` цикъл за диапазона от цели числа от `1` до `50`:

```swift
for i in 1...50 {
    
}
```

Трябва да съберем тези цели числа, затова ни е нужна и променлива, в която да пазим сумата:

```swift
var count = 0
```

След това при всяка итерация на цикъла добавяме текущото извлечено число към `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

Този код означава:

* `count` се използва за съхраняване на текущата сума
* `for` цикълът ще взема целите числа от `1` до `50` едно по едно
* Всеки път, когато се вземе едно цяло число, се изпълнява `count += i`
* След като цикълът приключи, в `count` ще бъде запазена крайната сума

По този начин можем да получим сумата на всички положителни цели числа до `50`.

По същия начин, ако искаме да изчислим сумата на целите числа от `1` до `100`, е нужно само да променим диапазона:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

`for` цикълът може да обхожда не само масив, но и диапазон.

Тази способност да обработва повтарящи се данни по ред е много често срещана в разработката.

Тя може да ни помогне автоматично да обработваме много повтаряща се работа, без да се налага ръчно да пишем събирането на всяко число или други подобни повтарящи се операции.

## Цикли в SwiftUI

Предишният `for` цикъл принадлежи на езика Swift.

Но в SwiftUI не можем директно да използваме `for` вътре в структурата на изгледа, за да създаваме изгледи:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

Това е така, защото вътре във `VStack` трябва да има SwiftUI изгледи, а обикновеният `for` цикъл сам по себе си не е изглед.

Ако искаме в SwiftUI да създаваме интерфейсни елементи многократно според съдържанието на масив, трябва да използваме `ForEach`.

Например, ако искаме да покажем имената на всички членове на една група, можем да го напишем така:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

Резултатът от показването:

![for1](../../Resource/017_for1.png)

След това добавяме еднакви модификатори към всеки `Text`:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Сега всеки `Text` се показва с по-малък и по-плътен шрифт.

Резултатът от показването:

![](../../Resource/017_for2.png)

Въпреки че модификаторите правят интерфейса по-красив, те също така правят кода по-дълъг и по-повтарящ се.

Ако имената продължат да се увеличават или по-късно искаме едновременно да променим шрифта, цвета и други стилове, ще трябва многократно да пишем и променяме много сходен код. Това е неудобно и не е лесно за поддръжка.

Тогава можем първо да поставим тези имена в масив, а след това да използваме `ForEach` в SwiftUI, за да създаваме изгледите многократно според данните в масива:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

Тази група код е по-кратка. `ForEach` ще прочете всеки елемент от масива по ред и ще покаже съответното съдържание на интерфейса.

Когато по-късно добавяме нови имена, е нужно само да променим имената в масива `names`, без всеки път да добавяме отново `Text` и всички повтарящи се модификатори.

## ForEach

`ForEach` е структура в SwiftUI, която се използва за многократно създаване на изгледи. Можете да я разбирате като структура в SwiftUI, подобна на `for` цикъл.

Основен запис:

```swift
ForEach(масив, id: \.self) { елемент in
    SwiftUI код
}
```

Например, имената на групата отпреди малко:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

Този код означава:

`names` е масивът, по който се прави обхождането, тоест това са данните, които ще се показват.

`id:\.self` означава, че самият елемент от масива се използва, за да се различава всяка отделна стойност.

В текущия пример масивът съдържа низове:

```swift
["Sam", "John", "Wathon", "Bob"]
```

Затова SwiftUI директно ще използва самите низове `"Sam"`, `"John"`, `"Wathon"` и `"Bob"`, за да различава различното съдържание.

За често срещаните прости масиви в началния етап, като `String` и `Int`, обикновено можем просто да пишем:

```swift
id: \.self
```

На този етап е достатъчно само да запомните този запис. Няма нужда засега да навлизате по-дълбоко в другите правила за `id`.

`item in` означава, че `item` е текущият извлечен елемент.

Например, при първото обхождане `item` е `"Sam"`, а при второто е `"John"`.

Затова:

```swift
Text(item)
```

последователно ще се превърне в:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

Следователно ролята на този код е: да вземе всяко име от масива `names` едно по едно и да създаде `Text` изглед за всяко име.

### Процес на изпълнение

Процесът на изпълнение може да се разбере така:

* `ForEach` първо прочита първия елемент `"Sam"` от масива `names`
* `item` временно представя `"Sam"`
* `Text(item)` показва `"Sam"`
* След това се прочита следващият елемент `"John"`
* И отново се изпълнява същият код за изгледа

Накрая всички имена от масива ще бъдат показани.

Предимството на този запис е, че когато имената станат повече, няма нужда да пишем многократно много `Text`, а е достатъчно само да променим съдържанието на масива.

### Обхождане на снимки

`ForEach` не може да показва само текст, а и снимки, цветове или други SwiftUI изгледи.

Например, нека подготвим 4 изображения:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

Плъзнете тези четири изображения в папката Assets в Xcode.

![image](../../Resource/017_for3.png)

Ако не използваме `ForEach`, ще трябва ръчно да напишем всяко изображение:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

Резултатът от показването:

![image](../../Resource/017_for4.png)

Този запис може да реализира функцията, но кодът става сравнително дълъг.

Ако изображенията стават повече, ще трябва да пишем все повече повтарящ се код за `Image`.

По-късно, ако искаме да променим размера на всички изображения едновременно, ще трябва да ги променяме едно по едно.

Тогава можем да поставим имената на изображенията в масив, а след това да използваме `ForEach`, за да създаваме многократно изгледи за изображения:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

Тук `images` е масив от низове, като всеки елемент в него е име на изображение.

`ForEach(images, id: \.self)` означава последователно да прочетем всеки елемент от масива `images`, да използваме самия елемент, за да различим всяка позиция, и след това да създадем съответния изглед според текущия елемент.

Например, ако първият прочетен елемент е `"430F9BEF"`, тогава:

```swift
Image(item)
```

е еквивалентно на:

```swift
Image("430F9BEF")
```

Следващите изображения също ще се покажат по същия начин, едно след друго.

![image](../../Resource/017_for4.png)

Тук все още използваме:

```swift
id: \.self
```

Причината е същата като преди, защото елементите в масива `images` също са прости низове, така че на началния етап можем директно да използваме самия елемент като начин за разграничаване.

Можете да го разбирате като едно фиксирано правило:

**Когато `ForEach` обхожда прост масив от тип `String` или `Int`, обикновено пишем `id: \.self`.**

### Добавяне на изображения

Ако по-късно трябва да добавим нови изображения, достатъчно е само да променим съдържанието на масива:

```swift
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

Не е нужно отново да пишем нов код за `Image`.

Ако изображенията са много, можем да използваме `ScrollView` заедно с него, иначе след като съдържанието излезе извън екрана, няма да може да се прегледа напълно.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### Промяна на изображенията

Когато трябва да променим стила на изображенията едновременно, е достатъчно само да променим кода вътре във `ForEach`.

Например, да променим ширината на `80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

Така всички изображения ще бъдат променени едновременно и няма да се налага да ги редактираме едно по едно.

Това е една от важните роли на `ForEach`:

**Когато няколко изгледа имат една и съща структура и се различават само по съдържанието, можем да напишем кода за изгледа само веднъж, а след това данните в масива да определят какво да се показва.**

## Обобщение

В този урок научихме `for` цикъла и `ForEach`.

`for` цикълът се използва за последователна обработка на елементите в масив или диапазон и е по-насочен към обработката на данни.

`ForEach` се използва в SwiftUI за многократно създаване на изгледи според съдържанието на масива и е по-насочен към визуалното показване.

На текущия начален етап най-често използваният запис за `ForEach` е:

```swift
ForEach(масив, id: \.self) { елемент in
    SwiftUI код
}
```

Тук `id: \.self` означава, че самият елемент се използва, за да различава всяка позиция.

Когато съдържанието на масива е от прости типове като `String` и `Int`, обикновено в началото можем да използваме точно този запис.

По-късно, когато разглеждате уеб страници или използвате приложения и виждате група от повтарящи се текстове, изображения или друго съдържание в списък, можете да си спомните, че в SwiftUI такова съдържание обикновено първо се поставя в масив, а след това се показва последователно чрез `ForEach`.
