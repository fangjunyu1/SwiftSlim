# Започваме от ContentView

## Подготовка преди урока

В този урок ще започнем от файла `ContentView.swift` и систематично ще се запознаем с основната структура на SwiftUI, включително:

- коментари
- структурата `View`
- подредбите `VStack` / `HStack` / `ZStack`
- икони от SF Symbols
- модификатори на изгледи (`modifier`)
- кода за преглед `#Preview`

Първо намери Xcode проекта, който създаде по-рано, и щракни два пъти върху файла `.xcodeproj`.

В лявата навигационна област `Navigator` избери файла `ContentView.swift`.

Забележка: всеки път, когато отваряш проекта, `Canvas` може да показва `Preview paused`. Натисни бутона за обновяване, за да възстановиш прегледа.

## Запознаване с ContentView

Кодът на `ContentView`:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
```

Въпреки че кодът е малко, той вече съдържа основната структура на SwiftUI.

### 1. Коментари

В началото на файла:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

Това са коментари на файла, които служат за запис на информация за файла, включително името на файла, името на проекта, автора и датата на създаване.

В Swift за едноредови коментари се използва `//`:

```swift
// Това е коментар
```

Коментарите подобряват четимостта на кода и помагат на разработчиците да разберат логиката му.

В реалната разработка кодът понякога става труден за разбиране. Ако не пишеш коментари, много лесно може след три дни вече да не разбираш собствения си код.

Затова е добър навик по време на писане на код активно да добавяш коментари. Така можеш да записваш логиката на кода и да улесняваш последващата му поддръжка.

**Временно изключване на код**

Коментарите могат също временно да изключват код, за да ни помогнат при откриване на проблеми.

Например:

```text
A
B
C
```

`A`, `B` и `C` са три части код, като една от тях съдържа грешка. Можем временно да коментираме кода, за да проверим къде е проблемът.

Първо добавяме коментар към `A`:

```text
// A
B
C
```

Ако след коментирането на `A` кодът започне да работи нормално, значи проблемът вероятно е в `A`.

Ако проблемът остане, можем да коментираме `B` и така нататък, докато намерим проблемния код.

По време на разработката се сблъскваме с много проблеми и в повечето случаи използваме коментиране на код, за да открием причината. Това ни помага да локализираме проблемния код и да намерим `BUG`.

В Xcode можеш да използваш клавишната комбинация:

```text
Command ⌘ + /
```

за бързо добавяне или премахване на коментар.

### 2. Импортиране на SwiftUI framework

```swift
import SwiftUI
```

Този ред означава, че импортираме framework-а `SwiftUI`.

Типове като `View`, `Text`, `Image` и `VStack` идват именно от него.

Ако не импортираш `SwiftUI`, Xcode ще даде грешка:

```text
Cannot find type 'View' in scope
```

Това означава, че компилаторът не може да разпознае типа `View`.

### 3. Структурата на View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Когато за първи път видиш тази `View` структура, тя може да ти се стори непозната, защото съдържа ключови думи като `struct`, `View`, `var`, `body` и `some`.

Все още не сме учили тези ключови думи. Засега е достатъчно да знаеш, че този код създава изглед `View` с име `ContentView`.

Можеш да мислиш за `View` като за платно, върху което рисуваме, а инструментът за рисуване е SwiftUI.

Например:

![Swift](../../RESOURCE/002_view.png)

Картинката по-горе показва три страници, които всъщност са три различни `View`.

```swift
struct View1: View {
    var body: some View {

    }
}
struct View2: View {
    var body: some View {
```

Когато разработваме приложение със SwiftUI, всяка страница е един `View`.

### 4. SwiftUI код

Вътре в `View` виждаме SwiftUI код:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Този SwiftUI код означава, че имаме вертикална подредба, в която се показват една икона и един текст.

![Swift](../../RESOURCE/002_view1.png)

#### Подредба `VStack`

```swift
VStack { }  // Вертикална подредба
```

`VStack` е контейнер за вертикална подредба. Изгледите в него се подреждат отгоре надолу.

![Swift](../../RESOURCE/002_view8.png)

Трите най-често срещани подредби в SwiftUI са:

- `VStack` — вертикално подреждане
- `HStack` — хоризонтално подреждане
- `ZStack` — наслагване по оста Z

```swift
HStack { }  // Хоризонтално подреждане
ZStack { }  // Наслагване
```

Схема на различните видове подредба:

![Swift](../../RESOURCE/002_view2.png)

Например можем да използваме `HStack` за хоризонтално подреждане:

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Така иконата на глобус и текстът ще бъдат показани един до друг.

![Swift](../../RESOURCE/002_view3.png)

Когато искаме хоризонтално подреждане, използваме `HStack`. Когато искаме наслагване, използваме `ZStack`.

#### `Image` и SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

Този код показва икона на глобус. Тя идва от системата за икони SF Symbols на Apple.

![Swift](../../RESOURCE/002_view9.png)

Значението на кода е: покажи икона на глобус, с голям размер и с акцентния цвят.

Освен глобус можем да показваме и други икони.

Например, за да покажем раница:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**Как да показваме други икони?**

Трябва да използваме официалната библиотека със системни икони на Apple, SF Symbols.

Отвори сайта на Apple Developer и изтегли [SF Symbols](../../RESOURCE/https://developer.apple.com/sf-symbols/).

![Swift](../../RESOURCE/002_sf.png)

Отвори приложението SF Symbols.

![Swift](../../RESOURCE/002_sf1.png)

Отляво ще видиш категории символи, а отдясно съответните икони.

Щракни с десен бутон върху иконата и избери „Copy Name“, за да копираш името, което съответства на низа.

Например:

```text
"globe"
"backpack"
"heart"
```

Постави копираното име в `Image(systemName:)`, за да покажеш различна икона.

Забележка: всеки SF Symbols символ има минимална поддържана версия на системата. Ако версията на системата е твърде стара, иконата може да не се показва. Провери информацията за съвместимост в приложението SF Symbols.

#### Модификатори

В SwiftUI модификаторът (`modifier`) е метод, който се използва за промяна на външния вид или поведението на даден изглед.

Можеш да мислиш за модификаторите като за дрехи: когато носиш различни дрехи, изглеждаш по различен начин.

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

`imageScale` и `foregroundStyle` са модификатори на изгледа `Image`. Те променят изгледа на `Image`, без да променят самото съдържание.

**1. `imageScale`**

```swift
.imageScale(.large)
```

Той контролира размера на иконите от SF Symbols:

- `.small`
- `.medium`
- `.large`

![Swift](../../RESOURCE/002_view11.png)

Можем да променяме различните опции и така SF Symbols ще се показват с различни размери.

**2. `foregroundStyle`**

```swift
.foregroundStyle(.tint)
```

`foregroundStyle` контролира предния цвят.

`.tint` означава акцентния цвят на текущата среда, който по подразбиране в iOS е син.

Ако искаме да променим предния цвят на червен:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### Изгледът `Text`

`Text` е текстов изглед, който се използва за показване на низове.

```swift
Text("Hello, world!")
```

Например, за да покажем моето име:

```swift
Text("FangJunyu")
```

Забележка: низовете задължително трябва да бъдат оградени с двойни кавички `""`.

Можеш да опиташ да покажеш своето име, телефонен номер или друго съдържание.

#### `padding` отстояние

В SwiftUI `padding` се използва за добавяне на празно пространство между съдържанието на изгледа и неговата граница. То принадлежи към „вътрешните отстояния“ (`padding / content inset`).

```swift
HStack {
    ...
}
.padding()
```

Кодът по-горе означава, че към изгледа `HStack` се добавя системното стандартно отстояние.

**Какво е `padding`?**

`padding` означава „празно пространство между съдържанието на изгледа и неговата граница“.

На илюстрацията по-долу, след като към синия `HStack` бъде приложен `padding`, синята област ще се свие навътре и ще изглежда сякаш става „малко по-малка“.

![Swift](../../RESOURCE/002_view6.png)

**Стандартно отстояние**

Модификаторът `padding()` по подразбиране използва стандартното препоръчано системно разстояние.

```swift
.padding()
```

На различни платформи и в различни контексти тази стойност може да е различна. Например:

- в iOS обикновено е около `16 pt`
- в macOS или watchOS стандартното разстояние може да е по-голямо или по-малко според дизайнерските правила на съответната платформа

**Персонализирано отстояние**

Можем също така да задаваме отделни отстояния за различни посоки.

1. Задаване на една посока

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

Обяснение на посоките:

- `.top`: горно отстояние
- `.bottom`: долно отстояние
- `.leading`: предно/ляво отстояние
- `.trailing`: задно/дясно отстояние

![Swift](../../RESOURCE/002_view12.png)

Забележка: `leading` и `trailing` автоматично се адаптират според посоката на езика. Например в арабска среда (`RTL`) те автоматично ще се обърнат.

2. Задаване на няколко посоки

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

Можем да зададем няколко посоки едновременно чрез масив. По-подробно ще учим масивите в следващите уроци, а тук е достатъчно само да познаваш този запис.

3. Задаване на хоризонтална или вертикална посока

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

Това е еквивалентно на:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**Премахване на отстояние**

Ако не искаме никакво отстояние, можем да използваме `.padding(0)`:

```swift
.padding(0)
```

или просто да премахнем модификатора `padding`:

```swift
// .padding()
```

### 6. Кодът за преглед `Preview`

```swift
#Preview {
    ContentView()
}
```

Този код означава, че в `Canvas` се генерира преглед на изгледа `ContentView`.

Забележка: `#Preview` е нов синтаксис, въведен в Swift 5.9 / Xcode 15. Преди това се използваше структурата `PreviewProvider`.

**Какво става, ако коментираме `Preview`?**

Ако коментираме `Preview`:

```swift
// #Preview {
//    ContentView()
// }
```

`Canvas` вече няма да показва съдържание за визуализация.

![Swift](../../RESOURCE/002_xcode.png)

Това е така, защото `#Preview` управлява показването на предварителния изглед в `Canvas`.

Когато искаме да преглеждаме SwiftUI изглед в Xcode, добавяме `#Preview`. Ако не искаме преглед, можем да коментираме или изтрием кода на `#Preview`.

## Обобщение

Въпреки че файлът `ContentView.swift` не съдържа много код, той всъщност обхваща няколко основни концепции на SwiftUI. За начинаещите този код може да изглежда непознат, но като го разделим на части, можем да изградим първоначално разбиране за SwiftUI.

Нека обобщим съдържанието на този урок. Първо научихме за коментарите `//`, които могат да се използват за обяснение на логиката на кода или за временно изключване на код.

След това разбрахме, че SwiftUI файловете задължително трябва да импортират framework-а `SwiftUI`:

```swift
import SwiftUI
```

Ако не импортираме framework-а, компилаторът няма да може да разпознае типове като `View`.

След това се запознахме с основната структура на SwiftUI изгледа:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

Тук `ContentView` е името на изгледа.

Научихме също трите най-често използвани контейнера за подредба: `VStack` (вертикално), `HStack` (хоризонтално) и `ZStack` (наслагване).

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

Този код означава, че се показва контейнер с вертикална подредба, в който има една икона и един текст.

`Image` може да показва икони от SF Symbols, а чрез модификатори можем да управляваме размера и цвета им.

Изгледът `Text` може да показва текстово съдържание.

`padding` е отстояние, което добавя прозрачно пространство около изгледа.

Накрая имаме `#Preview`, който показва предварителния изглед в `Canvas`.

```swift
.imageScale(.)
```

Xcode ще покаже наличните опции:

![Swift](../../RESOURCE/002_view10.png)

В следващите уроци официално ще въведем `enum`, а засега е достатъчно просто да знаеш, че такъв механизъм съществува.
