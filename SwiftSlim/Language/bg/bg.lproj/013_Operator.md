# Оператори

В този урок ще изучим операторите (`Operators`) в Swift.

Когато извършваме изчисления, сравнения или логически проверки, винаги използваме оператори. Тези операции присъстват във всяка част от изпълнението на програмата и са една от най-базовите способности в програмирането.

```swift
let a = 100 * 8
```

Докато учат програмиране, много хора откриват, че онези, които имат известно математическо мислене, често разбират по-лесно програмната логика. Това е така, защото програмирането по същество не може без „изчисления“ и „проверка на правила“, а именно операторите изпълняват тази ключова роля.

В реалната разработка операторите не се използват само за прости числови изчисления, а и в много конкретни сценарии.

Например в игра с блокчета, когато потребителят плъзне блок до определена позиция, трябва да се изчисли:

- координатната позиция на блока
- позициите, които блокът може да запълни
- дали целевата позиция е валидна

Тези процеси на „изчисление + проверка“ по същество разчитат на операторите.

![Block](../../Resource/013_block.png)

Изучаването на операторите не означава само учене на числови изчисления, а полагане на основа за бъдещи теми като подредба на интерфейса, обработка на данни и логика на взаимодействие.

## Как се изпълняват изчисленията в SwiftUI

В SwiftUI обикновено използваме `Button`, за да задействаме функция, а самата логика на изчислението се изпълнява във функцията.

Например:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

Когато бутонът се натисне, `Console` ще изведе:

```text
c: 20
```

В този урок ще пишем логиката на операциите във `func`, ще я задействаме чрез бутон и ще използваме `print`, за да показваме резултатите.

В следващите примери, за да опростим показването на кода, ще отбелязваме резултатите директно в кода, без да използваме многократно `print`.

## Управление на изхода в Console

Понеже всяко натискане ще създава нов изход, можеш да използваш бутона за изчистване в долния десен ъгъл на `Console`, за да изчистиш старите логове и по-лесно да виждаш текущия резултат.

## Друг запис за `Button`

Когато бутонът изпълнява само една функция, можем да използваме по-кратък синтаксис:

```swift
Button(action: ) {
    // View
}
```

В този запис `action` задава функцията, която трябва да се изпълни при натискане на бутона (функция без параметри), а блокът `{}` дефинира изгледа на бутона.

Комбинирано с функцията `calculate` отгоре, можем да напишем:

```swift
Button(action: calculate) {
    Text("Button")
}
```

Когато бутонът се натисне, функцията `calculate` ще бъде изпълнена директно.

Затова, когато логиката на бутона е проста и се състои само от едно извикване на функция, записът с `action` е по-ясен и по-добре отговаря на добрите практики.

## Аритметични оператори

Аритметичните оператори (`Arithmetic Operators`) се използват за числови изчисления:

- `+` събиране
- `-` изваждане
- `*` умножение
- `/` деление
- `%` остатък от деление

Примери:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

Трябва да обърнем внимание, че Swift е строго типизиран език и не смесва автоматично различни типове данни:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

Когато двата типа са различни, трябва ръчно да преобразуваме един от тях.

## Оператори за сравнение

Операторите за сравнение (`Comparison Operators`) се използват за проверка на отношението между две стойности, а резултатът е `Bool`:

- `==` равно
- `!=` различно
- `>` по-голямо
- `<` по-малко
- `>=` по-голямо или равно
- `<=` по-малко или равно

Примери:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

Операторите за сравнение често се използват в условни конструкции. Например във „въртележката с цитати“ трябва да гарантираме, че индексът не надхвърля диапазона на масива:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

Ако `index` е по-малък от `sayings.count - 1`, резултатът от сравнението е `true` и кодът в `if` ще се изпълни. Ако е `false`, той няма да се изпълни.

## Логически оператори

Логическите оператори (`Logical Operators`) се използват, за да комбинираме няколко условия:

- `&&` и (`AND`)
- `||` или (`OR`)
- `!` не (`NOT`)

### `&&` и

Когато няколко условия трябва да бъдат изпълнени едновременно, използваме `&&`.

Например:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

В този код има две условия: `age >= 18` и `age <= 100`.

Само когато и двете условия са `true`, `legalAge` ще бъде `true`; ако дори едно от тях не е изпълнено, резултатът ще бъде `false`.

### `||` или

Когато е достатъчно да е изпълнено поне едно от няколко условия, използваме `||`.

Например:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

Този код означава, че е достатъчно да е изпълнено или `money >= 50`, или `creditCard >= 50`, за да бъде `payAllowed` равно на `true`.

Само ако и двете условия не са изпълнени, резултатът ще бъде `false`.

Например в сценарий на плащане можем да платим или в брой, или с кредитна карта. Ако и двата начина не са възможни, плащането ще бъде неуспешно.

### `!` не

`!` е логическият оператор за отрицание (`NOT`), който обръща булева стойност.

Можем да го разбираме така: `!` превръща `true` във `false`, а `false` в `true`.

Например:

```swift
let allowed = true
let result = !allowed   // false
```

Тук `!allowed` означава да обърнем стойността на `allowed`.

Важно е да отбележим, че `!` и `!=` са два различни оператора: `!` обръща булева стойност, докато `!=` е оператор за сравнение, който връща `Bool`.

## Оператори за присвояване

Операторите за присвояване (`Assignment Operators`) се използват за задаване или обновяване на стойност в променлива:

- `=` присвояване

Както при деклариране на променлива, така и в процеса на изчисления често имаме операции на присвояване:

```swift
let a = 5
```

Този код означава, че стойността `5` се присвоява на променливата `a`.

### Оператори за съкратено присвояване

В реалната разработка освен основния оператор за присвояване много често използваме и „оператори за съкратено присвояване“, които ни позволяват директно да извършим изчисление върху текущата стойност и да я обновим:

- `+=`
- `-=`
- `*=`
- `/=`

В урока за „брояч“ вече накратко се запознахме с тях.

Например, когато искаме променливата да се увеличи при натискане на бутон:

```swift
var num = 10
num += 5   // num = 15
```

Този код означава, че `num` и `5` ще бъдат събрани, а резултатът ще бъде присвоен обратно на `num`.

Следователно той е еквивалентен на:

```swift
var num = 10
num = num + 5   // num = 15
```

## Тернарен оператор

Тернарният оператор (`Ternary Operator`) в Swift извършва проверка на условие и връща една от две възможни стойности:

```swift
условие ? стойност1 : стойност2
```

Логиката му е следната: ако условието е `true`, се връща „стойност1“; ако условието е `false`, се връща „стойност2“.

Например:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

Този код проверява стойността на `showColor`: ако е `true`, връща `Color.blue` и фонът ще е син; ако е `false`, връща `Color.clear`, което означава, че няма да има видим фонов цвят.

Подсказка: `Color.clear` означава прозрачен цвят.

### Разликата спрямо `if-else`

Тернарният оператор е логически еквивалентен на `if-else`.

Например горният код може да бъде написан и така:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

И в двата случая логиката е една и съща: ако условието е `true`, използваме първия резултат; ако е `false`, използваме втория.

На практика тернарният оператор е по-кратък и е много удобен за изразяване на проста логика „или едното, или другото“, особено когато искаме директно да върнем стойност.

В SwiftUI при изграждане на `View` тернарният оператор се среща много често. Например:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

Този запис ни позволява само в един ред динамично да променяме ефекта на изгледа според условие.

## Оператори за диапазон

Операторите за диапазон (`Range Operators`) се използват за представяне на интервал:

- `...` затворен диапазон (включва двата края)
- `..<` полуотворен диапазон (не включва десния край)

Например:

```text
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

Можем да ги разбираме така: `1...3` означава от `1` до `3`, включително `3`; `5..<8` означава от `5` до `8`, но без самото `8`.

Разширена форма (само за обща представа):

```text
5...    // >= 5
...5    // <= 5
```

Този запис означава „едностранен диапазон“ и често се използва в условни проверки или pattern matching.

По-нататък ще използваме операторите за диапазон, когато учим `for` цикли. Засега е достатъчно да ги разбираш като непрекъснат интервал от стойности.

## Упражняване на операторите

Сега чрез няколко прости примера ще упражним как работят операторите в изглед.

### 1. Изчисляване на общ брой автомобили

Например, ако през дадено кръстовище минават по 500 автомобила на ден, какъв е общият им брой за 30 дни?

Можем да използваме аритметични оператори:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

Логиката тук е: по 500 автомобила на ден, за 30 дни общият брой е `500 × 30`, което дава `15000`.

### 2. Проверка за членство в приложение

Можем да проверим дали някой е платен член в приложението. Ако има или доживотно членство, или абонамент, ще го считаме за член.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

В този код използваме оператора `||` („или“): `lifeTime = false` означава, че няма доживотно членство; `subscription = true` означава, че има активен абонамент.

Понеже е достатъчно да е изпълнено поне едно от условията, `isMember` е `true`. Следователно този човек е платен член.

## Обобщение

В Swift операторите са основата на обработката на данни и логическите проверки. Точно както `Text` показва съдържание, така операторите обработват самите данни.

От най-простите числови изчисления до по-сложните логически проверки, както и до управлението на състоянието в изгледите и изчисленията за подредба, навсякъде участват оператори.

Чрез упражнения в различни сценарии можем постепенно да свикнем и да овладеем различните видове оператори, представени в този урок. Това е важна основа за следващите теми.

### Допълнителни знания - побитови оператори

Освен това Swift предлага и по-ниско ниво оператори, наречени побитови оператори (`Bitwise Operators`):

- `&` (побитово И)
- `|` (побитово ИЛИ)
- `^` (побитово XOR)
- `~` (побитово НЕ)
- `>>` (изместване надясно)
- `<<` (изместване наляво)

Например:

```swift
let a = 6  // двоично 110
let b = 3  // двоично 011

print(a & b) // 2 (двоично 010)
print(a | b) // 7 (двоично 111)
print(a ^ b) // 5 (двоично 101)
print(~a)    // -7 (обръщане)
```

В Swift целите числа се представят в двоичен допълнителен код, затова след побитово обръщане резултатът е съответното отрицателно число.

Побитовите оператори работят директно върху двоичното представяне и обикновено се използват в по-ниско ниво обработка на данни и при оптимизация на производителността.

За начинаещите тези оператори се използват сравнително рядко, а и в типична iOS или SwiftUI разработка не са често срещани, затова тук няма да ги разглеждаме подробно.

Ако по-нататък се наложи да се занимаваме с по-ниско ниво логика или оптимизация, тогава можем да ги изучим по-задълбочено.
