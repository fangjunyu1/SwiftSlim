# struct и class

В предишния урок тъкмо започнахме да се запознаваме с `struct`, дефинирахме една структура и допълнително разгледахме знания като протоколи, `UUID` и други.

Сега вече имаме първоначална представа за ключовата дума `struct`, но това все още не може да се нарече истинско разбиране.

В този урок ще продължим да поставяме `struct` в центъра на вниманието и междувременно ще се запознаем и с `class`. Чрез този урок ще разберете по-ясно: какво е инстанция, какво е конструктор и защо можем да използваме `struct` и `class`, за да организираме данни.

## Система за управление на ученици

Например, сега искаме да разработим система за управление на ученици. Всеки ученик има информация като име, пол, телефон, възраст, домашен адрес и т.н.

Тогава можем да съберем тази информация заедно и да я дефинираме като структура `Student`:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}
```

Тук използваме `struct`, за да дефинираме тип за ученик.

Тази структура `Student` съдържа множество свойства:

- `id`: уникалният идентификатор на ученика
- `name`: името на ученика
- `sex`: полът на ученика
- `phone`: телефонът на ученика
- `age`: възрастта на ученика
- `address`: домашният адрес

По този начин няколко първоначално разпръснати данни се комбинират в един цялостен „ученик“.

Тук `Student` също така следва протокола `Identifiable` и дефинира поле `id`, за да е удобно по-късно данните да се показват чрез цикъл `ForEach`.

След това можем да създадем инстанция на ученик:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

Този код показва, че по структурата `Student` може да се създаде конкретен запис за ученик.

Тук `Student` е типът, а `Student(...)` представлява създадена инстанция.

В SwiftUI можем да присвоим тази инстанция на променлива и след това да я покажем.

Например:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}

struct ContentView: View {
    let student = Student(name: "Fang Junyu",
                          sex: "man",
                          phone: "123456",
                          age: 18,
                          address: "China, Shandong, Rizhao")
    
    var body: some View {
        Text(student.name)
    }
}
```

Така можем да видим името на този ученик в изгледа: `Fang Junyu`.

Тук първо създаваме инстанция на `Student`, после я присвояваме на променливата `student` и накрая четем свойство от нея чрез `student.name`.

## Какво е инстанция

Когато тъкмо започнем да учим `struct`, думата „инстанция“ може да ни се стори много непозната.

Но всъщност ние използваме инстанции още от самото начало, просто преди не сме им обръщали специално внимание.

Например:

```swift
let name = "Fang Junyu"
```

Тук `"Fang Junyu"` е низова стойност и по същество е инстанция от тип `String`.

Можем също да го напишем така:

```swift
let name = String("Fang Junyu")
```

И двата начина изразяват едно и също: създаване на инстанция от тип `String` и присвояването ѝ на променливата `name`.

По същия начин стои въпросът и при `Int`:

```swift
let num = 3
```

Тук `3` по същество е стойност от тип `Int`, която също може да се разбира като инстанция на `Int`.

Следователно инстанцията не е някакво специално понятие, което се появява само в `struct`.

Независимо дали става дума за `String`, `Int`, `Double`, `Bool` или за персонализирания `Student`, всички те могат да създават инстанции.

Затова, когато напишем:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

по същество това е същото като създаване на инстанция на `String` или `Int`, само че `Student` е тип, който сме дефинирали сами.

## Конструктор

Когато създаваме инстанция:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

всъщност извикваме инициализиращия метод на `Student`, тоест `init`.

Можете да разбирате конструктора като: **входа, който отговаря за присвояването на стойности на свойствата при създаване на инстанция.**

Тоест, преди една инстанция наистина да бъде завършена, тя обикновено първо трябва да премине през `init`.

Например, когато по-рано учихме жизнения цикъл на SwiftUI изгледите, написахме код като този:

```swift
struct ContentView: View {
    init() {
        print("Create ContentView")
    }
    var body: some View {
        VStack {
            Image(systemName: "globe")
            Text("Hello, world!")
        }
        .onAppear {
            print("Show ContentView")
        }
    }
}
```

Тук `init()` се изпълнява, когато бъде създадена инстанцията `ContentView`, а `onAppear` се изпълнява чак когато изгледът действително се покаже на екрана. Тоест създаването и показването на изгледа не са един и същ момент.

По същата логика, когато създаваме `Student(...)`, ние извикваме конструктора на `Student`.

### Автоматично генериран от системата конструктор

Когато дефинирахме `Student`, не написахме ръчно `init`:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
}
```

Но въпреки това все пак можем директно да създадем инстанция:

```swift
Student(name: "Fang Junyu",
        sex: "man",
        phone: "123456",
        age: 18,
        address: "China, Shandong, Rizhao")
```

Защо може да се създаде инстанция, когато няма написан конструктор?

Защото при `struct`, ако не напишем ръчно конструктор, системата обикновено автоматично ни генерира подходящ конструктор.

В началния етап на обучение можете просто да го разбирате така: Swift автоматично ни дописва кода за инициализация.

Приблизително може да се разбира, че системата ни дописва следния код:

```swift
struct Student: Identifiable {
    let id = UUID()
    let name: String
    let sex: String
    let phone: String
    let age: Int
    let address: String
    
    init(name: String, sex: String, phone: String, age: Int, address: String) {
        self.name = name
        self.sex = sex
        self.phone = phone
        self.age = age
        self.address = address
    }
}
```

Тук трябва да обърнем внимание на една подробност:

```swift
let id = UUID()
```

`id` вече има стойност по подразбиране, затова при създаване на инстанция не е нужно ние отново ръчно да подаваме `id`.

Тоест свойствата без стойност по подразбиране обикновено трябва да се подадат в конструктора; а свойствата, които вече имат стойност по подразбиране, обикновено могат директно да използват първоначалната стойност.

Затова, когато създаваме инстанция на `Student`, трябва да подадем само `name`, `sex`, `phone`, `age` и `address`.

### Да напишем конструктор сами

Макар че системата може автоматично да генерира конструктор, понякога този начин на инициализация по подразбиране не отговаря напълно на нуждите ни.

Например, имаме структура `Movie`:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

Според автоматично генерирания от системата конструктор, когато създаваме инстанция, трябва едновременно да подадем и трите параметъра:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Но ако сега искаме да въведем група стари филми и знаем само имената им, без да знаем режисьора и оценката, това ще е доста неудобно.

Защото можем да пишем само така:

```swift
Movie(name: "The Night of the Hunter", director: "--", rating: "--")
Movie(name: "Picnic at Hanging Rock", director: "--", rating: "--")
Movie(name: "Days of Heaven", director: "--", rating: "--")
```

Макар че това работи, не е удобно всеки път да повтаряме `--`.

Тогава можем да напишем собствен конструктор:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Така при създаване на стари филми ще е нужно да подадем само `name`:

```swift
Movie(name: "The Night of the Hunter")
Movie(name: "Picnic at Hanging Rock")
Movie(name: "Days of Heaven")
```

Това е едно от значенията на това сами да пишем конструктор: **да направим начина на създаване на инстанции по-подходящ за реалните нужди.**

### Разбор на персонализиран конструктор

Нека разгледаме този ръчно написан конструктор отпреди малко:

```swift
init(name: String) {
    self.name = name
    self.director = "--"
    self.rating = "--"
}
```

Смисълът на този код е, че при създаване на инстанция на `Movie` се изисква да подадем само един параметър `name`.

След това вътре в конструктора подаденото `name` ще бъде присвоено на собственото `name` на инстанцията, а на `director` и `rating` ще бъде зададена стойност по подразбиране `--`.

Тук `self` означава „самата текуща инстанция“.

Например:

```swift
self.name = name
```

Лявата страна `self.name` означава свойството в инстанцията; дясната страна `name` означава параметъра, който сме подали.

Може просто да се разбира така: стойността, подадена отвън, се записва в собственото свойство на инстанцията.

След като всички свойства получат стойности, създаването на тази инстанция е завършено.

### Механизъм на конструктора

Когато изрично декларираме конструктор, настъпва и още една промяна:

първоначално автоматично генерираният от системата конструктор обикновено вече не може да се използва директно.

Например:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Сега ако отново напишем така:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

компилаторът ще даде грешка:

```swift
Extra arguments at positions #2, #3 in call
```

Смисълът е: подали сте излишни параметри.

Защо се появява тази грешка?

Защото в текущата структура `Movie` има само този един вид конструктор, който сте декларирали ръчно:

```swift
init(name: String)
```

Той приема само един параметър `name` и не приема `director` и `rating`.

Тоест, когато ръчно добавите собствен конструктор, първоначалният автоматично генериран начин на инициализация вече не може директно да се използва.

Ако искаме едновременно да поддържаме „само име“ и „пълна информация“, трябва сами да допишем още един пълен конструктор:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String

    init(name: String, director: String, rating: String) {
        self.name = name
        self.director = director
        self.rating = rating
    }

    init(name: String) {
        self.name = name
        self.director = "--"
        self.rating = "--"
    }
}
```

Така вече имаме два различни конструктора.

При създаване на стар филм можем да пишем така:

```swift
Movie(name: "The Night of the Hunter")
```

А при създаване на нов филм с пълна информация можем да пишем така:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Този начин на писане също показва, че един и същ тип може да има няколко конструктора, стига формите на параметрите им да са различни.

## Какво е претоварване

Тук нека допълним още една необходима точка.

Току-що написахме два `init` за `Movie`:

```swift
init(name: String, director: String, rating: String)
init(name: String)
```

Този начин на писане, „едно и също име на функция, но различни параметри“, се нарича „претоварване“.

Тук и двете функции се казват `init`, но параметрите, които приемат, са различни, затова Swift може да различи коя функция извиквате.

Например:

```swift
Movie(name: "Days of Heaven")
```

ще извика:

```swift
init(name: String)
```

А:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

ще извика другата пълна версия на метода за инициализация.

Следователно ролята на претоварването е: да позволи на един и същ тип да предоставя различни начини за създаване според различни ситуации.

## Запознаване с class

След това нека накратко се запознаем и с `class`.

Освен често срещания в Swift `struct`, много езици за програмиране използват `class`, за да създават инстанции, например Java, C#, Kotlin и други.

В Swift начинът на писане на `class` е много близък до `struct`.

Например:

```swift
class Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String

    init(id: UUID = UUID(), name: String, director: String, rating: String) {
        self.id = id
        self.name = name
        self.director = director
        self.rating = rating
    }
}
```

Ще забележите, че освен смяната на ключовата дума от `struct` на `class`, останалите части изглеждат почти еднакво.

Начинът за създаване на инстанция също е много сходен:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10")
```

Следователно на ниво синтаксис `struct` и `class` наистина си приличат много.

### Разликата между Struct и Class

Макар че `struct` и `class` изглеждат много сходно, те не са напълно еднакви.

**1. Начинът на инициализация е различен**

При `struct`, ако не напишем ръчно конструктор, системата обикновено автоматично генерира подходящ метод за инициализация.

Но при `class`, ако има свойства без стойност по подразбиране, обикновено трябва ръчно да напишем `init`, иначе ще има грешка.

Например:

```swift
class Movie {
    let name: String
    let director: String
    let rating: String
}
```

Този `class` ще даде грешка, защото няма конструктор и свойствата не са инициализирани.

Компилаторът ще покаже подсказка от рода на:

```swift
Class 'Movie' has no initializers
```

Следователно в този пример `class` задължително трябва да има ръчно написан конструктор.

**2. `struct` е value type, а `class` е reference type**

`struct` е value type, а `class` е reference type.

В началния етап не е нужно да овладявате напълно тези две понятия, достатъчно е просто да имате елементарна представа.

Например `struct` е по-близо до идеята „да копираш нова стойност“.

```swift
struct User {
    var name: String
}

var user1 = User(name: "Tom")
var user2 = user1

user2.name = "Jack"

print(user1.name) // Tom
print(user2.name) // Jack
```

След като тук присвоим `user1` на `user2`, това е все едно сме копирали нови данни, така че промяната на `user2` няма да повлияе на `user1`.

А `class` е по-близо до идеята „няколко имена сочат към една и съща инстанция“.

```swift
class User {
    var name: String
    
    init(name: String) {
        self.name = name
    }
}

var user1 = User(name: "Tom")
var user2 = user1

user2.name = "Jack"

print(user1.name) // Jack
print(user2.name) // Jack
```

Тук `user1` и `user2` сочат към една и съща инстанция.

Сега не е нужно напълно да разбираме механизма в дълбочина на value type и reference type, достатъчно е само да знаем, че `struct` е по-подходящ за съхраняване на независими данни, а `class` е по-подходящ, когато на няколко места трябва да се споделя една и съща инстанция.

## Как да избираме в началния етап

На текущия етап за обикновените модели на данни първо обмисляйте `struct`, а когато са нужни данни, които да се споделят и синхронно променят, тогава обмисляйте `class`.

Това не е абсолютно правило, но за начинаещите е достатъчно практичен начин за преценка.

Особено в SwiftUI често ще откриете, че `struct` вече може да свърши по-голямата част от работата.

Освен това в много програмни материали съдържанието, създадено чрез `class`, се нарича и „обект“. В началния етап можете първо просто да го разбирате като инстанция, без умишлено да правите разлика.

## Обобщение

В този урок задълбочено изучихме `struct`, а междувременно се запознахме и с `class` - и двете могат да се използват за дефиниране на типове.

Типовете могат да създават инстанции. При създаване на инстанция се извиква конструкторът, тоест `init`.

При `struct`, ако не е написан ръчно конструктор, системата обикновено автоматично генерира подходящ конструктор. Ако конструкторът по подразбиране не отговаря на нуждите, можем и сами да напишем конструктор.

Освен това получихме и първоначална представа за разликата между `struct` и `class`: `struct` е value type, а `class` е reference type. В SwiftUI в началния етап по-често ще се срещате именно със `struct`.

По-нататък, когато учим предаване на стойности между изгледи, модели на данни и управление на състоянието, всички тези неща ще продължат да се използват.
