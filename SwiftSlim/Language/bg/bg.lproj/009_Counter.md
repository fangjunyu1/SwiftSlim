# Брояч

В предишните два урока научихме за променливите, константите, бутоните и функциите.

В този урок ще комбинираме тези знания, за да създадем просто приложение брояч.

В този пример ще научим как да съхраняваме и променяме променлива, както и как да я управляваме чрез `@State`.

## Създаване на брояч

Броячът често се използва за следене на количество, което трябва постоянно да нараства, например брой скокове на въже или обиколки при бягане.

Сега нека направим прост брояч: показваме число и бутон, а всеки път когато бутонът бъде натиснат, числото се увеличава.

![Num](../../RESOURCE/009_num.png)

### Показване на число

Първо използваме `Text`, за да покажем число.

```swift
Text("0")
```

Но ако числото трябва да се променя, не бива да записваме `"0"` директно в кода. Вместо това трябва да съхраняваме числото в променлива.

```swift
var num = 0
```

След това показваме променливата в `Text`:

```swift
Text("\(num)")
```

Тук използваме интерполация на низове `\()`, за да превърнем числото в низ и да го покажем.

### Увеличаване на числото с бутон

След това добавяме бутон.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

Когато бутонът бъде натиснат, искаме числото да се увеличи.

Затова можем да променим променливата вътре в бутона:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

Този код означава: когато бутонът бъде натиснат, изчисли `num + 1` и после присвои резултата обратно на `num`.

### По-кратък запис

Swift предоставя и по-кратък начин да запишем това:

```swift
num += 1
```

Това е еквивалентно на:

```swift
num = num + 1
```

Този запис се нарича оператор за съкратено присвояване.

Често използвани оператори за съкратено присвояване в Swift са:

```text
+=   присвояване със събиране
-=   присвояване с изваждане
*=   присвояване с умножение
/=   присвояване с деление
%=   присвояване с остатък
```

Например:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### Пълен код

Сега вече можем да напишем прост брояч:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
            Text("\(num)")
            Button("+") {
                num += 1
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
```

Но този код няма да работи правилно.

### Поява на грешка

Xcode ще покаже:

```text
Cannot assign to property: 'self' is immutable
```

Това означава, че свойството не може да бъде променяно, защото изгледът е неизменим.

Защо се получава така?

## Как работи SwiftUI

В SwiftUI **интерфейсът се управлява от данни**, което означава, че когато данните се променят, интерфейсът се обновява автоматично.

Но има една важна подробност: SwiftUI следи промените и обновява изгледа само когато се промени състояние, като `@State` или `@Binding`.

Ако използваме обикновена променлива, SwiftUI ще прочете стойността ѝ само веднъж при създаването на изгледа. След това, дори променливата да се промени, интерфейсът няма да се обнови.

Например:

```swift
var num = 0   // 0
num = 1   // 0
```

В примера по-горе `num` е обикновена променлива. Макар че стойността ѝ се променя, SwiftUI не обновява изгледа и интерфейсът продължава да показва началната стойност `0`.

![Num](../../RESOURCE/009_state.png)

Затова само когато данните са маркирани като състояние, например чрез property wrapper-и като `@State` или `@Binding`, SwiftUI автоматично ще следи промените и ще обновява изгледа.

## Property wrapper-ът `@State`

В SwiftUI, ако искаме да променяме променлива и заедно с това да обновяваме изгледа, трябва да използваме `@State`.

```swift
@State var num = 0
```

`@State` казва на SwiftUI да следи тази променлива.

Когато стойността ѝ се промени, SwiftUI преизчислява изгледа и го обновява.

Пълен код:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
            Text("\(num)")
            Button("+") {
                num += 1
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
```

Сега, когато бутонът бъде натиснат:

```swift
num += 1
```

SwiftUI ще открие, че `num` се е променил, и автоматично ще обнови изгледа.

![Num](../../RESOURCE/009_state1.png)

## Правила за използване на `@State`

В реалната разработка `@State` обикновено следва две правила.

### 1. `@State` обикновено използва `private`

Променливите с `@State` обикновено се използват само вътре в текущия тип, например в текущата `View` структура, затова често се пишат така:

```swift
@State private var num = 0
```

Това предотвратява директния достъп или промяната на тази променлива от други изгледи.

Все още не сме учили официално `private`, така че засега можеш просто да го разбираш така:

**Съдържанието, маркирано с `private`, може да се използва само в текущия изглед. Изгледите отвън не могат да го използват.**

### 2. `@State` е само за собственото състояние на текущия изглед

`@State` е подходящ за данни, които се управляват от текущия `View`.

Например съдържанието на текстово поле или състоянието на switch:

```swift
@State private var isOn = false
@State private var text = ""
```

Ако данните трябва да бъдат споделяни между няколко изгледа, тогава трябва да използваме други типове състояние, като `@Binding` или `@Observable`.

Ще научим това по-късно.

### Бутон за намаляване

Броячът не само увеличава. Той може и да намалява.

Можем да добавим бутон `-`.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

След това използваме `HStack`, за да подредим бутоните хоризонтално:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

Резултат:

![Num](../../RESOURCE/009_num1.png)

Когато бутонът `+` бъде натиснат, числото се увеличава с `1`. Когато бутонът `-` бъде натиснат, числото намалява с `1`.

### Бутон за нулиране

Можем също да добавим бутон за нулиране, който да върне числото обратно на `0`.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

Когато бутонът за нулиране бъде натиснат, на `num` се присвоява стойността `0`.

Резултат:

![Num](../../RESOURCE/009_num2.png)

Когато бутонът `0` бъде натиснат, числото става `0`.

### Персонализирани бутони

В момента нашите бутони могат да показват само текст:

```swift
Button("+") {

}
```

Тази форма на бутон може да показва само текстово съдържание, като `+` или `-`.

В реалната разработка обикновено искаме бутоните да изглеждат по-богато, например чрез икони или други изгледи.

SwiftUI ни позволява да персонализираме това, което бутонът показва. В такъв случай можем да използваме друга форма на `Button`:

```swift
Button(action: {

}, label: {

})
```

В този синтаксис `action` е кодът, който се изпълнява при натискане на бутона, а `label` е изгледът, който бутонът показва.

Например можем да направим така, че бутонът да показва SF Symbols икона:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

Така бутонът вече няма да показва текст, а икона.

### Подобряване на интерфейса на брояча

За да направим интерфейса на брояча по-ясен, можем също да променим начина, по който се показва числото. Например да го увеличим:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

След това можем да променим и съдържанието на бутоните така, че вместо текст да показват икони, както и леко да увеличим размера на иконите:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

По същия начин можем да превърнем и бутона за намаляване, и бутона за нулиране в икони:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

Сега бутоните показват SF Symbols икони и интерфейсът изглежда по-интуитивен.

![Num](../../RESOURCE/009_num3.png)

## Обобщение

В този урок чрез създаването на прост брояч комбинирахме упражнения с променливи, `Text` и `Button`. Използвахме променлива, за да запазим числото, а чрез бутоните го увеличавахме и намалявахме, като също така научихме съкратените оператори за присвояване, като `num += 1` и `num -= 1`.

По време на реализацията разбрахме начина, по който SwiftUI работи: интерфейсът се управлява от данни. Когато данните се променят, интерфейсът се обновява. Затова трябва да използваме `@State`, за да съхраняваме променящото се състояние, и когато променливата с `@State` се промени, SwiftUI автоматично обновява изгледа.

Накрая направихме и проста оптимизация на интерфейса, като променихме стила на шрифта в `Text` и използвахме `Button(action:label:)`, за да персонализираме съдържанието на бутоните, така че те да могат да показват икони и интерфейсът на брояча да стане по-ясен и по-красив.

Сега вече сме се запознали с основния начин на използване на SwiftUI изгледите. В SwiftUI по-голямата част от интерфейса се управлява чрез променливи. Когато променливите се променят, SwiftUI автоматично обновява изгледа. Затова при разработка на SwiftUI приложения обикновено първо трябва да планираме какви данни трябва да съхраняваме и след това да изграждаме интерфейса и логиката за взаимодействие върху тези данни.

## Пълен код

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
