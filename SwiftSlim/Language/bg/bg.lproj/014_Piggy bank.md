# Касичка

В този урок ще разработим прост проект „касичка“. Функцията е проста, но съдържа завършена логика на взаимодействие и е много подходяща за начинаещи.

Чрез този проект ще научим за `TextField` (поле за въвеждане), `border` (рамка) и обвързване на данни (`$`).

Крайната цел е да направим проект „касичка“, в който можем да въведем сума, да натиснем бутон за запазване и да натрупваме общата сума.

## Показване на общата сума

Първо трябва да покажем общата сума в „касичката“.

В `ContentView` декларираме променлива, която да съхранява общата сума:

```swift
@State private var amount = 0
```

След това използваме `Text`, за да покажем тази сума:

```swift
Text("\(amount)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

Тук използваме `.font` и `.fontWeight`, за да зададем размера и дебелината на текста.

```swift
struct ContentView: View {
    
    @State private var amount = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
        }
    }
}
```

Когато `amount` се промени, `Text` автоматично ще обнови показаната сума.

## Въвеждане на сума

Освен да показва общата сума, „касичката“ трябва да позволява на потребителя да въвежда сума за внасяне. За това ни е нужен контролът `TextField` от SwiftUI.

### TextField

`TextField` е SwiftUI контрол за въвеждане на съдържание, който обикновено се използва за едноредов вход.

`TextField` основно предлага два вида обвързване:

**1. Обвързване към низ (`String`)**

```swift
TextField("Placeholder", text: $text)
```

Подходящо за въвеждане на име, заглавие и друго текстово съдържание.

**2. Обвързване към числова стойност (`Int/Double`)**

```swift
TextField("Amount", value: $number, format: .number)
```

Подходящо за възраст, заплата, сума и други числови входове.

`format: .number` означава, че това поле ще анализира и показва стойността като число.

#### Placeholder текст

Първият параметър на `TextField` е `Placeholder`, който подсказва на потребителя какво трябва да въведе:

```swift
TextField("input your name", text: $text)
```

Когато полето е празно, ще се показва сив подсказващ текст.

![textfield](../../RESOURCE/014_textfield.png)

#### Data Binding

`TextField` сам по себе си не отговаря за постоянното съхраняване на съдържанието, а използва обвързана променлива, за да управлява данните.

```swift
@State private var text = ""
TextField("Placeholder", text: $text)
```

Въведеното от потребителя съдържание директно се запазва в променливата `text`, а `TextField` отговаря само за интерфейса на въвеждане.

`$` означава обвързване (`Binding`):

```swift
$text
```

Това не е обикновена променлива, а стойност от тип `Binding`.

Нейната роля е да изгради връзка между изгледа и данните:

когато потребителят въвежда текст в `TextField`, `text` автоматично се обновява; когато `text` се промени, `TextField` също ще се обнови.

Например:

```swift
struct ContentView: View {
    @State private var text = ""
    
    var body: some View {
        VStack {
            Text("text:\(text)")
            TextField("input your name", text: $text)
                .frame(width: 150)
        }
    }
}
```

Когато въведеш съдържание в `TextField`, `Text` отгоре ще го показва в реално време.

![textField](../../RESOURCE/014_textfield1.png)

### Обвързване на числов тип

Когато трябва да въвеждаме числа, трябва да обвържем променлива от съответния тип:

```swift
@State private var number = 0

TextField("Amount", value: $number, format: .number)
```

Когато потребителят въведе сума, `TextField` ще анализира входа и автоматично ще обнови стойността на `number`.

## Добавяне на поле за въвеждане

След като вече разбираме основната употреба на `TextField`, можем да го приложим към проекта „касичка“.

„Касичката“ трябва да въвежда „сума“, затова ще използваме начина за обвързване към числов тип (`Int/Double`) и ще добавим нова променлива `number`, която ще съхранява въведената сума:

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
        }
    }
}
```

Сега `TextField` по подразбиране показва стойността на `number` (която по подразбиране е `0`).

Когато съдържанието бъде изтрито, ще се покаже placeholder текстът `"Amount"`.

Това още веднъж показва, че `TextField` показва стойността на обвързаната променлива, а не пази сам съдържанието.

### Проблем с размера на полето

В текущия изглед ще забележим, че `TextField` няма рамка, показва само едно `0`, а съдържанието изглежда ляво подравнено.

Ефектът е следният:

![textfield](../../RESOURCE/014_textfield2.png)

Причината е, че по подразбиране `TextField` заема цялата ширина на родителския изглед.

Можем да потвърдим това, като добавим фонов цвят:

```swift
TextField("Amount", value: $number, format: .number)
    .background(Color.red)
```

Ефектът е:

![textfield](../../RESOURCE/014_textfield3.png)

Ясно се вижда, че `TextField` всъщност запълва цялата ширина на реда.

### Задаване на фиксирана ширина

Ако искаме полето да е по-компактно, можем да използваме `frame`, за да ограничим ширината му:

```swift
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
```

Ефектът е:

![textfield](../../RESOURCE/014_textfield4.png)

### Добавяне на рамка и вътрешно отстояние

Тъй като `TextField` по подразбиране няма рамка, можем ръчно да му добавим стил:

```swift
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Сега това поле изглежда много повече като „стандартно поле за въвеждане“.

## Border

В SwiftUI `border` се използва за добавяне на рамка към изглед.

Основна употреба:

```swift
border(Color.black, width: 1)
```

Тук `Color.black` означава цвета на рамката, а `width` е дебелината ѝ.

В `TextField` използваме:

```swift
TextField("Amount", value: $number, format: .number)
    .frame(width: 100)
    .padding(5)
    .border(Color.black, width: 1)
```

Това означава, че към полето за въвеждане се добавя черна рамка с дебелина `1 pt`.

Можеш също да опиташ други цветове, например `Color.blue`, `Color.green`, или по-дебела рамка като `width: 2`.

## Бутон за спестяване

Сега вече имаме променлива `amount`, която показва общата сума, `TextField`, в който въвеждаме сума, и променливата `number`, обвързана с полето.

Текущият код е:

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
        }
    }
}
```

### Добавяне на бутон

Следващата стъпка е да добавим бутон, който да задейства действието по „спестяване“.

```swift
Button("Save") {
    
}
```

### Реализиране на логиката

Искаме да реализираме следното: потребителят въвежда сума, натиска бутона и въведената сума се добавя към общата сума.

```swift
Button("Save") {
    amount += number
}
```

Тук използваме оператор за съкратено присвояване, за да добавим въведената сума `number` към общата сума `amount`.

### Нулиране на въведената сума

Но тук има един проблем: след натискане на бутона стойността в полето не се изчиства.

Ако потребителят въведе `10`, натисне `Save`, в полето ще остане `10`, което ще повлияе на следващото въвеждане.

Причината е, че `TextField` е обвързан с `number`. При натискане на бутона ние променяме само `amount`, но `number` остава непроменен, затова в полето продължава да се показва старата стойност.

Трябва да добавим логика за изчистване: след натискане на бутона първо добавяме сумата към общата стойност, а после нулираме въведената стойност:

```swift
Button("Save") {
    amount += number
    number = 0
}
```

Сега, когато потребителят въведе сума и натисне бутона, сумата ще бъде добавена към общата, а `number` ще се върне на `0`, за да е готова за следващия вход.

Така вече реализирахме цялата логика за „спестяване“ в нашата касичка.

## Подобряване на изгледа

След като функционалността е готова, можем леко да подобрим интерфейса, като добавим фонова снимка и стил на бутона.

## Пълен код

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

### Дебъг изход

За да проверим дали логиката е правилна, можем да добавим и дебъг логика в бутона, за да валидираме дали въведената сума се обработва правилно:

```swift
Button("Save") {
    print("---Saving---")
    amount += number
    number = 0
    print("amount:\(amount)")
    print("number:\(number)")
}
```

Всеки път, когато запазваме сума и натиснем бутона, ще виждаме съответния дебъг изход и така ще можем да проверим дали кодът работи правилно.

```text
---Saving---
amount:11
number:0
```
