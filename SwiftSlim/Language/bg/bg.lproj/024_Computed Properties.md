# Изчисляеми свойства

В този урок ще научим основно за изчисляемите свойства.

Изчисляемите свойства се използват, за да изчисляват нов резултат според вече съществуващи данни. Те могат да изчисляват числови стойности, а също и съдържание, което трябва да се показва в SwiftUI изгледи.

Например:

```swift
let a = 10
let b = 20
let c = a + b
```

Тук `c` представлява резултата от събирането на `a` и `b`.

В обикновен код този начин на писане е много често срещан.

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c:\(c)")
}
```

Съвет: в събитието за натискане на `Button` може да се изпълнява обикновен Swift код.

Но ако напишем подобен код директно в свойствата на SwiftUI изглед, ще срещнем проблем.

Например:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
    var body: some View {
        VStack {
            Text("c:\(c)")
        }
        .padding(20)
    }
}
```

Този код ще даде грешка.

Изглежда, че `a` и `b` вече са написани преди `c`, сякаш стойността на `c` може да се изчисли.

Но в декларациите на свойства в структура този начин на писане не може да се използва директно.

## Защо c не може директно да изчисли a + b

Защото `a`, `b` и `c` не са временни константи в събитие за натискане на бутон, а свойства в изгледа `ContentView`.

В събитието за натискане на бутон този код може да се изпълни нормално:

```swift
Button("a + b") {
    let a = 10
    let b = 20
    let c = a + b
    print("c: \(c)")
}
```

Защото след натискането на бутона кодът се изпълнява по ред. Първо се създава `a`, после се създава `b`, а накрая чрез `a + b` се изчислява `c`.

Но когато декларираме свойства вътре в изглед, ситуацията е различна:

```swift
struct ContentView: View {
    let a = 10
    let b = 20
    let c = a + b
}
```

Тук `a`, `b` и `c` са свойства в изгледа `ContentView`.

Когато се създава изгледът `ContentView`, Swift първо трябва да подготви тези свойства. За да гарантира безопасността на процеса на създаване, Swift не позволява стойността по подразбиране на едно съхранявано свойство на инстанция да чете директно други свойства на същата инстанция.

Затова този ред ще даде грешка:

```swift
let c = a + b
```

Можем да го разберем просто така: **когато декларираме свойства вътре в изглед, не можем директно да използваме едно обикновено свойство, за да изчислим друго обикновено свойство.**

Свойство, което директно съхранява стойност, се нарича „съхранявано свойство“. За по-лесно разбиране временно можем да го приемем и като обикновено свойство.

Например:

```swift
let a = 10
```

`a` съхранява `10`.

```swift
let b = 20
```

`b` съхранява `20`.

Но:

```swift
let c = a + b
```

Тук `c` не е директно записана стойност, а искаме тя да бъде изчислена чрез `a + b`.

За такава ситуация, в която „получаваме резултат според вече съществуващи данни“, по-подходящо е да използваме изчисляемо свойство.

Можем да го променим така:

```swift
struct ContentView: View {
    let a = 10
    let b = 20

    var c: Int {
        return a + b
    }

    var body: some View {
        VStack {
            Text("c: \(c)")
        }
        .padding(20)
    }
}
```

Тук `c` е изчисляемо свойство.

```swift
var c: Int {
    return a + b
}
```

То означава: **когато трябва да използваме `c`, тогава прочети `a` и `b` и върни резултата от `a + b`.**

Например:

```swift
Text("c: \(c)")
```

Когато `Text` показва `c`, едва тогава се задейства изчисляването на `c`.

## Какво е изчисляемо свойство?

Изчисляемото свойство изглежда като променлива, но само по себе си не съхранява данни.

Например:

```swift
let a = 10
let b = 20
var c: Int {
    return a + b
}
```

Тук `c` е изчисляемо свойство.

То не съхранява фиксирана стойност като обикновено свойство, а при всяко прочитане на `c` изпълнява отново кода в `{}` и връща резултата от изчислението.

Можем да го разберем така:

```swift
var c: Int {
    return a + b
}
```

Когато трябва да използваме `c`, отиваме да изчислим `a + b`.

Затова изчисляемите свойства са подходящи за такава ситуация: **резултатът не трябва да се съхранява отделно, а може да се изчисли от вече съществуващи данни.**

## Основен синтаксис

Изчисляемото свойство обикновено съдържа три части:

```swift
var c: Int {
    return a + b
}
```

### 1. Деклариране с `var`

```swift
var c
```

Изчисляемото свойство трябва да се декларира с `var`; не може да се използва `let`.

Защото изчисляемото свойство не е фиксирана съхранявана стойност, а резултат, който се изчислява динамично при всяко прочитане.

### 2. Посочване на върнатия тип

```swift
var c: Int
```

Изчисляемото свойство трябва да посочи върнат тип.

Тук `c` накрая ще върне цяло число, затова типът е `Int`.

### 3. Използване на {} за изчислителната логика

```swift
{
    return a + b
}
```

В `{}` се пише изчислителната логика; тук се връща `a + b`.

## Ключовата дума return

Изчисляемото свойство трябва да върне резултат.

Например:

```swift
var c: Int {
    return a + b
}
```

Тук `return` означава: върни навън резултата от изчислението `a + b`.

Ако изчисляемото свойство съдържа само един израз, който директно създава резултата, `return` може да се пропусне:

```swift
var c: Int {
    a + b
}
```

Но ако изчисляемото свойство съдържа няколко реда код, трябва да използваме `return`, за да посочим ясно върнатия резултат.

Например:

```swift
var totalPriceText: String {
    let total = count * price
    return "Общо: \(total) $"
}
```

Тук процесът на изчисление е разделен на две стъпки.

Първа стъпка, първо изчисляваме общата цена:

```swift
let total = count * price
```

Втора стъпка, съединяваме общата цена в текст и я връщаме:

```swift
return "Общо: \(total) $"
```

Ако премахнем `return`:

```swift
var totalPriceText: String {
    let total = count * price
    "Общо: \(total) $"
}
```

Този код ще даде грешка.

Причината е: **това изчисляемо свойство вече съдържа няколко реда код и Swift вече не може автоматично да прецени кой ред е крайният резултат, който трябва да се върне.**

Затова, когато изчисляемото свойство има само един ред с резултат, може да пропуснем `return`.

```swift
var c: Int {
    a + b
}
```

Когато изчисляемото свойство съдържа няколко реда код, е препоръчително ясно да напишем `return`.

```swift
var totalPriceText: String {
    let total = count * price
    return "Общо: \(total) $"
}
```

## Разлика между изчисляемо и обикновено свойство

Обикновеното свойство съхранява данни.

```swift
var c = 30
```

Тук `c` съхранява конкретна стойност: `30`.

Изчисляемото свойство не съхранява данни.

```swift
var c: Int {
    a + b
}
```

Тук `c` не съхранява `30`. То само предоставя начин за изчисление.

Когато прочетем `c`, Swift ще изпълни:

```swift
a + b
```

и след това ще върне резултата от изчислението.

Затова изчисляемите свойства са подходящи за случаи, в които резултатът се изчислява от други данни.

## body също е изчисляемо свойство

След като научихме за изчисляемите свойства, можем отново да разберем най-често срещания код в SwiftUI:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Тук `body` също е изчисляемо свойство.

То се декларира с `var`:

```swift
var body
```

Има върнат тип:

```swift
some View
```

В `{}` се връща съдържание на SwiftUI изглед:

```swift
VStack(spacing: 20) {
    Image(systemName: "carrot")
        .font(.largeTitle)
}
```

Затова `body` може да се разбере така: когато SwiftUI трябва да покаже този изглед, той прочита `body` и според съдържанието, което `body` връща, създава интерфейса.

Ако напишем `return`, може да го разберем така:

```swift
var body: some View {
    return VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Но в SwiftUI обикновено пропускаме `return` и пишем директно:

```swift
var body: some View {
    VStack(spacing: 20) {
        Image(systemName: "carrot")
            .font(.largeTitle)
    }
}
```

Когато данните `@State` се променят, SwiftUI отново ще прочете `body` и ще обнови интерфейса според новите данни.

Например:

```swift
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("брой: \(count)")

            Button("+") {
                count += 1
            }
        }
    }
}
```

След натискане на бутона `count` ще се промени.

```swift
count += 1
```

След като `count` се промени, SwiftUI ще преизчисли `body`, затова текстът в интерфейса също ще се обнови.

```swift
Text("брой: \(count)")
```

Това е и причината в SwiftUI интерфейсът да може автоматично да се обновява след промяна на данните.

### Не се препоръчва да пишем сложни изчисления в body

Тъй като `body` е изчисляемо свойство, то може да бъде прочитано и преизчислявано много пъти.

Например:

```swift
struct ContentView: View {
    @State private var name = "FangJunyu"

    var body: some View {
        let num = 10

        VStack {
            Text("число: \(num)")
            Text("име: \(name)")
        }
    }
}
```

Когато `name` се промени, SwiftUI ще преизчисли `body`.

Тогава кодът вътре в `body` също ще се изпълни отново и `num` също ще бъде създаден наново.

```swift
let num = 10
```

`num` в този пример е много прост, затова влиянието не е голямо.

Но ако в `body` се изпълняват сложни изчисления, например филтриране или сортиране на голямо количество данни, обработка на изображения и т.н., това може да повлияе на плавността на интерфейса.

Затова в SwiftUI `body` трябва основно да отговаря за описването на структурата на интерфейса.

Прости временни данни могат да се пишат в `body`.

Сложните изчисления могат да се поставят в изчисляеми свойства извън `body`, в методи или в отделна обработка на данни.

## Пример: количество и обща цена

По-долу ще разберем изчисляемите свойства чрез прост пример.

Да предположим, че единичната цена на един морков е 2 $, потребителят може да натиска бутони, за да променя купуваното количество, а интерфейсът трябва да показва общата цена.

Начинът за изчисляване на общата цена:

```
количество * единична цена
```

Ако използваме обикновена променлива, за да съхраняваме общата цена, ще стане сравнително неудобно.

Защото всеки път, когато количеството се промени, трябва ръчно да обновяваме общата цена.

По-добрият начин е да използваме изчисляемо свойство:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if count > 1 {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("цена: \(price) $")
                Text("количество: \(count)")
                Text("обща цена: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Ефект на показване:

![view](../../Resource/024_view.png)

В този пример:

```swift
@State private var count = 1
```

`count` представлява количеството моркови. Когато натиснем бутона, `count` ще се промени.

```swift
private let price = 2
```

`price` представлява единичната цена на моркова. Тук стойността е фиксирана, затова използваме `let`.

```swift
private var totalPrice: Int {
    count * price
}
```

`totalPrice` представлява общата цена.

Не е нужно тя да се съхранява отделно, защото общата цена винаги може да се изчисли чрез `count * price`.

Когато `count` е `1`:

```swift
totalPrice = 1 * 2
```

Показаният резултат е:

```swift
обща цена: 2 $
```

След натискане на бутона `+`, `count` става `2`.

Когато тогава отново прочетем `totalPrice`, тя ще се преизчисли:

```swift
totalPrice = 2 * 2
```

Показаният резултат е:

```swift
обща цена: 4 $
```

Това е ролята на изчисляемото свойство: според вече съществуващи данни динамично да изчислява нов резултат.

## Изчисляемите свойства могат да се използват за проверка

Изчисляемите свойства могат не само да изчисляват числови стойности, но и да връщат резултат от проверка.

Например, когато искаме минималното количество да бъде 1.

Когато количеството вече е 1, бутонът `-` не трябва да продължава да намалява.

Можем да добавим изчисляемо свойство:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Пълният код:

```swift
struct ContentView: View {
    @State private var count = 1
    private let price = 2
    
    private var totalPrice: Int {
        count * price
    }
    
    private var canDecrease: Bool {
        count > 1
    }
    
    var body: some View {
        VStack(spacing: 20) {
            HStack(spacing: 20) {
                Button("-") {
                    if canDecrease {
                        count -= 1
                    }
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
                .foregroundStyle(canDecrease ? Color.primary : Color.gray)
                .disabled(!canDecrease)

                Image(systemName: "carrot")
                    .font(.largeTitle)

                Button("+") {
                    count += 1
                }
                .font(.largeTitle)
                .buttonStyle(.plain)
            }

            VStack {
                Text("цена: \(price) $")
                Text("количество: \(count)")
                Text("обща цена: \(totalPrice) $")
            }
            .foregroundStyle(.gray)
        }
    }
}
```

Ефект на показване:

![view](../../Resource/024_view1.png)

Тук:

```swift
private var canDecrease: Bool {
    count > 1
}
```

Този код означава дали в момента може да се намалява.

Когато `count` е по-голямо от `1`:

```swift
canDecrease == true
```

това означава, че може да се намалява.

Когато `count` е равно на `1`:

```swift
canDecrease == false
```

това означава, че не може да се продължи намаляването.

### Условна проверка в бутона

В бутона се използва:

```swift
if canDecrease {
    count -= 1
}
```

Само когато `canDecrease` се оценява като `true`, може да се намали `count`.

### Управление на цвета на предния план на изгледа

Можем също да използваме `canDecrease`, за да управляваме цвета на предния план на бутона:

```swift
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Тук е използван тернарният оператор:

```swift
canDecrease ? Color.primary : Color.gray
```

Смисълът на този код е: ако `canDecrease` е `true`, цветът на предния план използва `Color.primary`; ако `canDecrease` е `false`, цветът на предния план използва `Color.gray`.

`Color.primary` е системен семантичен цвят, предоставен от SwiftUI, и представлява основния цвят на текста в текущия интерфейс.

В светъл режим `Color.primary` обикновено е близо до черно; в тъмен режим `Color.primary` обикновено е близо до бяло.

Затова предимството на използването на `Color.primary` е, че той автоматично се адаптира към светъл и тъмен режим.

### Управление на забраненото състояние на изгледа

`disabled` се използва, за да управлява дали изгледът е в забранено състояние:

```swift
.disabled(!canDecrease)
```

Когато `disabled` е `false`, изгледът може да се натиска.

Когато `disabled` е `true`, изгледът е забранен и не може да се натиска.

Тук използването на `canDecrease` като условие прави кода по-лесен за разбиране.

Когато видим `canDecrease`, можем да разберем, че то означава „дали в момента може да се продължи намаляването“.

### Допълнение: защо има две проверки?

В бутона `-`:

```swift
Button("-") {
    if canDecrease {
        count -= 1
    }
}
.font(.largeTitle)
.buttonStyle(.plain)
.foregroundStyle(canDecrease ? Color.primary : Color.gray)
.disabled(!canDecrease)
```

Тук използваме както `.disabled(!canDecrease)`, така и `if canDecrease` вътре в бутона.

`.disabled(!canDecrease)` отговаря за забраняването на бутона от интерфейса, така че потребителят да не може да го натисне.

`if canDecrease` отговаря за повторна проверка преди изпълнение на кода; само когато може да се намалява, се изпълнява `count -= 1`.

Това е двойна защита. В реална разработка, ако бутонът вече е забранен, вътрешната проверка може да се пропусне; но в учебен пример я запазваме, за да стане ролята на `canDecrease` по-ясна.

## Обобщение

В този урок основно научихме за изчисляемите свойства.

Изчисляемото свойство не съхранява стойността директно, а при прочитане изчислява резултат според вече съществуващи данни.

Например:

```swift
var c: Int {
    a + b
}
```

Тук `c` не трябва да се съхранява отделно, защото може да се изчисли чрез `a + b`.

Изчисляемото свойство трябва да се декларира с `var` и трябва да посочи върнат тип.

```swift
var canDecrease: Bool {
    count > 1
}
```

Изчисляемите свойства могат да връщат не само числови стойности, но и резултати от проверки, текстово съдържание и дори съдържание на SwiftUI изгледи.

В този урок научихме и за `return`.

`return` означава връщане на резултат:

```swift
var totalPriceText: String {
    let total = count * price
    return "Общо: \(total) $"
}
```

Ако изчисляемото свойство съдържа само един израз, който директно създава резултата, `return` може да се пропусне.

```swift
var totalPrice: Int {
    count * price
}
```

Освен това разгледахме `Color.primary` и `disabled`.

`Color.primary` е системен семантичен цвят в SwiftUI и автоматично настройва начина на показване според светъл и тъмен режим.

```swift
.foregroundStyle(Color.primary)
```

`disabled` се използва, за да управлява дали изгледът е забранен.

```swift
.disabled(true)
```

Означава забранено, не може да се натиска.

```swift
.disabled(false)
```

Означава достъпно, може да се натиска.

Затова изчисляемите свойства са много често срещани в SwiftUI. Те могат да ни помогнат да подредим по-ясно резултатите от изчисления, условията за проверки и съдържанието за показване.
