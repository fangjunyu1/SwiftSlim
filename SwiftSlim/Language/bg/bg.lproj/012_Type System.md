# Система на типовете

В предишните уроци вече научихме за променливите и знаем, че те могат да съхраняват данни от различни типове.

Например:

```swift
var hello = "Hello, World"
```

Тук `hello` съхранява текст, затова неговият тип е `String`.

Сега ще изучим систематично понятието тип (`Type`), както и явното деклариране на типове и извеждането на типа, за да разберем по-дълбоко същността на променливите.

## Какво е тип

В Swift всяка стойност има ясен тип. Типът определя какво представлява тази стойност и какво може да прави.

Например:

- `42` е `Int` (цяло число)
- `"Hello"` е `String` (низ)
- `true` е `Bool` (булева стойност)

Те принадлежат към различни типове и имат различно предназначение.

## Често използвани основни типове в Swift

На начално ниво най-често срещаните типове са:

- `String`: низ (текст)
- `Int`: цяло число
- `Double`: число с плаваща запетая (с десетични знаци, с по-висока точност)
- `Bool`: булева стойност (`true/false`)

Освен това има и `Float` (число с плаваща запетая), но той има по-ниска точност от `Double` и затова се използва по-рядко; има и `Character` (един символ), например `"A"` или `"!"`.

### Деклариране на тип (явен тип)

В Swift можем ръчно да зададем тип при деклариране на променлива:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

Добавяме `: Тип` след името на променливата, за да декларираме нейния тип.

Записът за тип на масив е `: [Тип]`:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

В този код масивът `scores` може да пази само елементи от тип `Int`, а `tags` само елементи от тип `String`.

Явното деклариране на типа прави намерението на кода по-ясно и в някои случаи предотвратява грешки при извеждането на типа.

## Извеждане на типа

В много случаи можем изобщо да не декларираме типа изрично:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"    // String
var isLogin = false  // Bool
var scores= [90, 85, 78]    // [Int]
var tags = ["Swift", "iOS"] // [String]
```

Това е така, защото компилаторът на Swift може автоматично да изведе типа според „началната стойност“.

## Явно деклариране и извеждане на типа

В предишните уроци не въвеждахме явното деклариране, например:

```swift
var age: Int = 18
```

а използвахме приоритетно извеждане на типа:

```swift
var age = 18
```

В този пример двата записа са еквивалентни и `age` ще бъде определен като `Int`. В сравнение с това, записът с извеждане на типа е по-кратък.

Причината в началото да не наблягаме на явното деклариране е, че извеждането на типа намалява допълнителната информация в кода и по този начин прави разбирането по-лесно за начинаещите.

## Защо са нужни типовете

Swift е строго типизиран език (`Strongly Typed Language`).

Това означава: след като типът на една променлива бъде определен, не можем произволно да го променяме.

Например:

```swift
var age: Int = 18

age = 20    // Success
age = "20"  // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

Първото `20` е от тип `Int`, затова може да бъде присвоено на `age`; второто `"20"` е от тип `String` и не съвпада с `Int`, затова ще има грешка.

`nums` е масив от тип `[Int]`, който може да съхранява само цели числа, а не и низове.

Типовете ограничават начина, по който се използват данните, и ни позволяват още по време на компилация да откриваме несъответствия в типовете. Това намалява `BUG`-ове, повишава стабилността и подобрява поддръжката на кода.

## Кога е задължително да декларираме типа явно

Макар че Swift може автоматично да извежда типовете, има ситуации, в които компилаторът не може да го направи или го прави погрешно. Тогава трябва да декларираме типа ръчно.

**1. Празен масив**

При създаване на празен масив е задължително да посочим типа:

```swift
var nums: [Int] = []
```

Ако не запишем типа:

```swift
var nums = []   // Error, Empty collection literal requires an explicit type
```

компилаторът няма как да разбере какъв трябва да е типът на елементите в празния масив и ще върне грешка.

**2. Избягване на грешно извеждане**

```swift
var value = 10   // Int
```

В този пример, макар `10` да може да представлява и десетично число (`Double`), понеже няма десетична точка, компилаторът по подразбиране ще го изведе като `Int`.

Ако искаме `value` да бъде десетично число, трябва да декларираме типа явно:

```swift
var value: Double = 10   // Double
```

Когато резултатът от извеждането на типа не съвпада с очакването, трябва да използваме явна декларация на типа или да променим записа на литерала.

**3. Потребителски типове**

По-нататък ще учим и потребителски типове. При тях също често се налага явно деклариране:

```swift
var info: GameInfo
var users: [User] = []
```

Причината е същата като при празния масив: компилаторът не може автоматично да изведе типа от празна стойност.

## Преобразуване на типове

В Swift различните типове не се преобразуват автоматично един към друг, а трябва да ги преобразуваме ръчно.

Например, ако съберем две числа от различен тип:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b    // Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

В този код `a` е от тип `Int`, а `b` е от тип `Double`. Те са от различни типове и не могат директно да участват в една и съща аритметична операция.

Преди да извършим операцията, трябва да преобразуваме едната стойност в същия тип:

```swift
var result = Double(a) + b
```

Това означава да преобразуваме `a` от `Int` в `Double`, след което да го съберем с `b`.

Важно е да се отбележи, че преобразуването не променя типа на оригиналната променлива. Например `Double(a)` не променя `a` в `Double`, а просто създава нова стойност от тип `Double`, която участва в изчислението.

И други типове могат да бъдат преобразувани по подобен начин, като използваме името на целевия тип + `()`, в което подаваме стойността.

Например:

```text
Int()、Double()、String()
```

Примери:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int, 9
```

Трябва да внимаваме, защото не всяко преобразуване дава очаквания резултат. Например при `Double → Int` десетичната част се отрязва директно, а не се закръгля.

## Псевдоними на типове

В Swift можем да дадем псевдоним на вече съществуващ тип, така че смисълът в кода да стане по-ясен:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

Тук `UserID` по същество си остава `Int`, типът не се променя, просто името става по-смислено.

В по-големи проекти, където типовете са по-сложни, псевдонимите правят кода по-четим.

## Пример с типове

Сега нека използваме един прост пример, за да покажем ролята на типовете в реалната разработка.

### Създаване на променлива

Първо създаваме променлива `age`:

```swift
var age = 20
```

Тъй като трябва да показваме и променяме тази променлива в SwiftUI, трябва да я декларираме с `@State`:

```swift
@State private var age = 20
```

За `private` вече споменахме по-рано, че означава променливата да може да се използва само в текущия изглед. По-нататък ще го изучим по-подробно.

### Показване на данни в SwiftUI

Показване на променливата `age` в SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

Сега изгледът ще показва:

```swift
age: 20
```

### Промяна на данните

След това добавяме два бутона, за да променяме `age`:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### Резултат от изпълнението

Когато натиснем `changeAge1`, на `age` ще бъде присвоена стойността `18`:

```swift
age = 18
```

Понеже `18` е от тип `Int` и съответства на типа на `age`, присвояването е успешно.

`changeAge2` ще доведе до грешка:

```swift
age = "18"
```

Съобщението за грешка е:

```text
Cannot assign value of type 'String' to type 'Int'
```

Причината е, че `"18"` е `String` (низовете трябва да бъдат оградени с `""`), а `age` е `Int`, така че типовете не съвпадат.

### Защо има грешка?

Ключът е в следния ред:

```swift
@State private var age = 20   // Int
```

Макар че типът не е деклариран изрично, понеже началната стойност `20` е `Int`, компилаторът автоматично извежда `age` като `Int`.

След като типът бъде определен, вече не можем да присвояваме стойности от друг тип.

Освен това бутонът `changeAge2` не „се проваля по време на изпълнение“, а грешката се появява още на етап компилация и кодът изобщо не може да се компилира.

Това е едно от предимствата на строгата типизация в Swift: проблемите с типовете се откриват предварително, още при писане на кода.

### Правилен запис

След като премахнем грешния код:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

Когато натиснем бутона `changeAge1`, изгледът ще се обнови на:

```text
age: 18
```

### Явно деклариране на типа (по избор)

Можем също така още при дефинирането да посочим типа изрично:

```swift
@State private var age: Int = 20
```

Този запис прави типа по-ясен, но при по-прости случаи не е задължителен.

## Обобщение

Този урок беше посветен основно на системата на типовете в Swift и на ролята ѝ в реалната разработка.

В Swift всяка стойност има ясен тип, който определя значението на данните и какви операции могат да се извършват с тях.

Swift е строго типизиран език. След като типът на една променлива бъде определен, не можем повече да ѝ присвояваме стойности от друг тип. Това ограничение ни помага да откриваме грешки още на етап компилация, което намалява проблемите по време на изпълнение и повишава безопасността на кода.

Типът на променливата може да бъде изведен автоматично от началната стойност или да бъде деклариран явно. В някои ситуации, като празни масиви, грешно извеждане на типа или потребителски типове, компилаторът не може правилно да изведе типа и тогава трябва задължително да го запишем явно.

Различните типове не могат да се преобразуват автоматично. Трябва да използваме `Int()`, `Double()`, `String()` и подобни конструкции за явно преобразуване.

Типовете поддържат и псевдоними. Особено в по-сложни проекти те могат да направят кода по-четим.

Типът не е просто етикет на данните, а механизъм за ограничения, който гарантира, че данните се използват по безопасен, ясен и контролиран начин.

## Допълнителни знания - Optional тип

При преобразуване на типове има операции, които могат да се провалят.

Например преобразуване на `String` към `Double`:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

Тук `Double(input)` не връща `Double`, а:

```text
Double?
```

Това е `Optional`.

### Защо се връща Optional?

Защото съдържанието на низа е несигурно. `"3.14"` може да бъде преобразуван в `Double`, но `"Hello"` не може.

Тоест тази операция може да успее, но може и да се провали.

Затова Swift използва `Optional`, за да представи тази неопределеност.

### Какво е Optional?

Optional означава: една стойност може да съществува, но може и да не съществува.

Например:

```text
Double?
```

Този тип означава, че ако има стойност, при отпечатване ще се покаже `Optional(...)`, а ако няма стойност, ще върне `nil`.

Трябва да обърнем внимание, че `nil` означава „няма стойност“, а не `0` или празен низ.

### Пример

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Изход:

```text
pi:Optional(3.14)
```

Това означава, че низът е бил успешно преобразуван в `Double` и е получена стойността `3.14`, но понеже типът е `Double?`, тя е обвита в `Optional`.

Ако преобразуването се провали:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

Изход:

```text
pi: nil
```

Понеже `"Hello"` не може да бъде преобразуван в `Double`, преобразуването се проваля и връща `nil`.

### Какво е Optional(...)?

Когато отпечатаме Optional директно:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift ще покаже неговото „описание за дебъг“:

```text
Optional(3.14)
```

Това не е част от реалните данни, а подсказка, че текущата стойност е `Optional` и вътре в него има стойност.

### Разопаковане на Optional

В реалната разработка обикновено искаме да извадим истинската стойност от Optional. Този процес се нарича разопаковане (`Unwrapping`).

Един често срещан начин е да използваме `??`, за да дадем стойност по подразбиране:

```text
??
```

Това се нарича оператор за сливане с `nil` (`Nil-Coalescing Operator`).

Например:

```swift
name ?? "FangJunyu"
```

Този код означава: ако `name` има стойност, върни нея; ако е `nil`, използвай стойността по подразбиране `"FangJunyu"`.

Трябва да отбележим, че стойността по подразбиране трябва да е от същия тип като Optional-а.

### Използване на `??`, за да зададем стойност по подразбиране

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")    // 30
```

Изход:

```text
pi: 30
```

Когато `input` се преобразува успешно в `Double`, се извежда получената стойност. Ако преобразуването се провали, ще се използва стойността по подразбиране, подадена след `??`.

Ако преобразуването се провали:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

Изход:

```text
pi: 10
```

Тук, когато `input` не може да бъде преобразуван в `Double`, резултатът е `nil`, а `??` предоставя стойност по подразбиране.

Optional типът се използва, за да представя ситуации, в които „може да има стойност, но може и да няма“.

При операции, които могат да се провалят, като преобразуването на типове, Swift връща Optional, за да запази безопасността на програмата.

Когато ни трябва съдържащата се стойност, можем да използваме `??`, за да зададем стойност по подразбиране, така че дори при `nil` да получим разумен резултат.
