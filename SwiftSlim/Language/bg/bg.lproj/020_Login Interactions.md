# Взаимодействия при вход

В предишния урок създадохме изглед за вход и чрез бутони и логика за проверка реализирахме проста валидация на въведеното съдържание.

В този урок ще продължим на базата на кода от предишния урок и ще научим за полето за парола `SecureField`, предупредителния прозорец `Alert`, както и как да управляваме показването на различни изгледи според променливи.

Чрез този урок можеш да разбереш по-добре основния процес на взаимодействие в изгледа за вход, а това е и базова логика, която се използва в много приложения.

## Поле за въвеждане на парола

В предишния урок използвахме `TextField`, за да въвеждаме парола.

Но в реален екран за вход паролата обикновено не се показва директно, а се визуализира като точки или друг скрит ефект, за да се защити поверителността на потребителя.

В SwiftUI контролата, специално предназначена за въвеждане на парола, е `SecureField`.

Например:

```swift
SecureField("Password", text: $password)
```

Начинът на писане е много подобен на `TextField` и също изисква подсказващ текст и обвързана променлива.

Можем просто да разберем това така: `SecureField` също е поле за въвеждане, но скрива съдържанието, което се въвежда.

Можем директно да заменим `TextField`, който първоначално използвахме за въвеждане на парола, със `SecureField`:

```swift
SecureField("input password", text: $password)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

Когато потребителят въвежда парола, съдържанието няма да се показва директно на екрана.

Ефект на показване:

![password](../../Resource/020_password.png)

Сега можем да въведем потребителско име и парола и след натискане на бутона да проверим дали въведеното е правилно.

### Показване на обратна връзка

Досега постоянно използвахме `print`, за да извеждаме резултати.

Съдържанието на `print` се показва само в конзолата и разработчикът може да го види, когато стартира програмата в Xcode, но обикновеният потребител на телефона не може да види тези подсказки.

Тоест, макар програмата вече да е извършила проверката, потребителят не знае дали въведеното съдържание е правилно.

Тогава ни е нужен начин за подсказка, която може да се показва директно в интерфейса.

## Предупредителен прозорец Alert

В SwiftUI можем да използваме `Alert`, за да показваме изскачащ прозорец с подсказка.

Основна употреба:

```swift
.alert("Title", isPresented: $isPresented) {
    Button("Button", role: .cancel) {}
} message: {
    Text("Information")
}
```

Този код изглежда малко по-сложен от предишните контроли, но можем да го разберем, ако го разделим.

`Alert` основно съдържа следните части:

- `"Title"`: заглавието на предупредителния прозорец
- `isPresented`: обвързана булева стойност, която управлява дали прозорецът да се показва
- `Button(...)`: бутонът на предупредителния прозорец
- `message:`: съдържанието на подсказката, което се показва в предупредителния прозорец

Може просто да се разбере така: когато обвързаната променлива стане `true`, системата ще покаже този прозорец.

### Alert е модификатор

Подобно на много модификатори, които научихме по-рано, `Alert` също се добавя след даден изглед.

Например може да се постави след бутон:

```swift
Button("Sign in") {
	// ...
}
.alert() {...}
```

Може също да се постави след външния `VStack`:

```swift
VStack {
    // ...
}
.alert() {...}
```

Тоест `Alert` не е самостоятелен изглед, който се поставя отделно в интерфейса, а модификатор, прикрепен към даден изглед.

Може просто да се разбере така: `Alert` обикновено се пише след бутон или след външния layout, за да показва подсказка, когато е изпълнено определено условие.

### Показване на Alert

Ако искаме да покажем `Alert`, ни е нужна променлива, която да го управлява.

Например:

```swift
@State private var showAlert = false
```

Тази променлива е от тип `Bool`, което означава, че има само две стойности: `true` или `false`.

След това я обвързваме с `isPresented` на `Alert`:

```swift
isPresented: $showAlert
```

Когато `showAlert` е `false`, предупредителният прозорец няма да се показва; когато `showAlert` е `true`, прозорецът ще изскочи.

### Показване на предупредителен прозорец чрез бутон

Сега нека първо реализираме най-простия сценарий:

Когато потребителят не е въвел нищо и директно натисне бутона `Sign in`, да се покаже прозорец, който му казва „Все още не сте въвели нищо“.

Първо създай променлива, която управлява този прозорец:

```swift
@State private var showEmpty = false
```

След това добави `Alert` след `Button`:

```swift
Button("Sign in") {
    // ...
}
.alert("Alert", isPresented: $showEmpty) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("No information")
}
```

След това направи проверка в кода при натискане на бутона:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } 
    // ...
}
```

Ако и `user`, и `password` са празни, ще променим `showEmpty` на `true`.

А `Alert` е обвързан с тази променлива, така че когато тя стане `true`, системата автоматично ще покаже предупредителния прозорец.

Ефект на показване:

![alert](../../Resource/020_alert1.png)

Така след натискане на бутона потребителят веднага ще види подсказката и няма да е нужно разработчикът да проверява изхода от `print` в конзолата.

### Подобряване на бутона и предупредителните прозорци

По-рано обработихме само случая „нищо не е попълнено“.

Все още трябва да обработим и следните случаи:

- липсва потребителско име
- липсва парола
- потребителското име и паролата са попълнени, но въведеното е грешно

Затова трябва да подготвим съответни подсказки и за тези случаи.

Първо добави три нови променливи:

```swift
@State private var showMissUser = false
@State private var showMissPassword = false
@State private var showWrongInfo = false
```

След това продължи, като добавиш съответните `Alert` след бутона:

```swift
Button("Sign in") {
	// ...
}
.alert("Alert", isPresented: $showMissUser) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your username.")
}
.alert("Alert", isPresented: $showMissPassword) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Please enter your password.")
}
.alert("Alert", isPresented: $showWrongInfo) {
    Button("Ok", role: .cancel) {}
} message: {
    Text("Username or password verification failed.")
}
```

След това подобри логиката за проверка в бутона:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
        showEmpty = true
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
            showMissUser = true
        } else if password.isEmpty {
            print("Please enter your password.")
            showMissPassword = true
        }
    } else {
        if user == userID && password == passWord {
            print("Success")
        } else {
            print("Input error")
            showWrongInfo = true
        }
    }
}
```

Когато бутонът установи, че потребителят не е попълнил никаква информация, пропуснал е едно поле или проверката на въведеното е неуспешна, съответната булева променлива ще се промени на `true`, а след това ще се покаже съответният `Alert`.

Тук ти препоръчвам сам да тестваш няколко ситуации:

- не попълвай нищо
- попълни само потребителско име
- попълни само парола
- попълни и потребителско име, и парола, но със сгрешено съдържание
- попълни и потребителско име, и парола, и съдържанието е правилно

Така ще проверим дали има проблем в кода ни, а това също е много важен етап от разработката на приложения, наречен „тестване“.

Ако в някой сценарий програмата се държи неправилно, това означава, че вероятно има проблем в логиката и в програмата се е появил `Bug`.

Това, което в разработката често наричаме `Bug`, означава грешка, дефект или поведение, което не отговаря на очакванията.

## Завършване на изгледа за вход

`Alert`, за който говорихме по-рано, се използваше за това как да дадем подсказка на потребителя при „грешно въвеждане“.

Когато потребителят въведе правилно потребителско име и парола, обикновено не показваме „предупредителен прозорец“, а директно преминаваме към интерфейса след вход.

Тук се засяга една много важна идея: **показване на различни изгледи според стойността на променлива.**

Например:

```swift
@State private var login = false

if login {
	Text("Personal View")
} else {
	Text("Login View")
}
```

Тук използваме типа `Bool`, за да управляваме показването на изгледа:

- ако `login` е `true`, това означава, че входът вече е успешен и се показва съдържанието след вход
- ако `login` е `false`, това означава, че входът не е успешен и интерфейсът за вход продължава да се показва.

Въпреки че `if` е условна конструкция в Swift, в `body` на SwiftUI тя може да се използва за управление на логиката за показване на изгледа.

Тоест SwiftUI ще определи кой изглед трябва да се показва в момента според резултата от условието.

Можем да поставим тази логика за показване в текущия `ContentView`:

```swift
struct ContentView: View {
	@State private var login = false
	var body: some View {
    	if login {
    	    Text("Login successful")
    	} else {
			// Original login view code
		}
	}
}
```

Тук добавихме нова променлива `login`, която показва дали потребителят вече е влязъл успешно.

След това поставихме вече написания код за изгледа за вход след `else`.

Логика за показване на изгледа:

- когато `login` е `false`, това означава, че входът още не е успешен и продължаваме да показваме първоначалния изглед за вход.
- когато `login` е `true`, това означава, че входът е успешен и тогава вече не се показва изгледът за вход, а се показва `Login successful`.

Тоест `ContentView` ще показва различно съдържание на изгледа според различната стойност на `login`.

След това нека променим логиката за успешен вход в бутона:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
		// code
    } else if user.isEmpty || password.isEmpty {
		// code
    } else {
        if user == userID && password == passWord {
            print("Success")
            login = true
        } else {
			// code
        }
    }
}
```

Когато въведените от потребителя потребителско име и парола са правилни, променяме `login` на `true`.

Щом стойността на `login` се промени, резултатът от проверката `if` в `ContentView` също ще се промени и интерфейсът ще се превключи от първоначалния изглед за вход към `Login successful`.

Ефектът на показване е следният:

![view](../../Resource/020_view.png)

Сега вече наистина използвахме знанието „показване на различни изгледи според условие“ в текущата страница за вход.

В реалните приложения това обикновено е същият начин на преценка: според определена стойност се показват различни изгледи.

Например, когато потребителят не е влязъл, се показва страницата за вход; след успешен вход се показва друг интерфейс.

Основната идея е: **чрез промяна на променливи да се управлява какво съдържание показва интерфейсът.**

## Обобщение

В този урок научихме три важни неща:

1. Да използваме `SecureField` за въвеждане на парола, така че съдържанието ѝ да не се показва директно на екрана.
2. Да използваме `Alert`, за да показваме подсказки на потребителя, вместо само да извеждаме информация в конзолата.
3. Да използваме променливи и проверка с `if`, за да управляваме показването на различни изгледи и да реализираме превключването между интерфейса преди и след вход.

Чрез усвояването на тези знания вече можем да завършим изглед за вход с базови интерактивни ефекти и още по-добре да разберем идеята в SwiftUI за разработка: „състоянието се променя, интерфейсът се обновява“.

## Пълен код

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    @State private var showEmpty = false
    @State private var showMissUser = false
    @State private var showMissPassword = false
    @State private var showWrongInfo = false
    @State private var login = false
    let userID = "FangJunyu"
    let passWord = "123456"
    var body: some View {
        if login {
            Text("Login successful")
        } else {
            VStack {
                Image("019_icon")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 60)
                Text("Sign in to App")
                    .font(.title)
                    .fontWeight(.bold)
                HStack {
                    Text("Username")
                        .fontWeight(.bold)
                    TextField("input user", text: $user)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                HStack {
                    Text("Password")
                        .fontWeight(.bold)
                    SecureField("input password", text: $password)
                        .frame(width: 200)
                        .border(Color.gray,width: 1)
                }
                Button("Sign in") {
                    if user.isEmpty && password.isEmpty {
                        print("Empty")
                        showEmpty = true
                    } else if user.isEmpty || password.isEmpty {
                        if user.isEmpty {
                            print("Please enter your username.")
                            showMissUser = true
                        } else if password.isEmpty {
                            print("Please enter your password.")
                            showMissPassword = true
                        }
                    } else {
                        if user == userID && password == passWord {
                            print("Success")
                            login = true
                        } else {
                            print("Input error")
                            showWrongInfo = true
                        }
                    }
                }
                .buttonStyle(.borderedProminent)
                .alert("Alert", isPresented: $showEmpty) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("No information")
                }
                .alert("Alert", isPresented: $showMissUser) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your username.")
                }
                .alert("Alert", isPresented: $showMissPassword) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Please enter your password.")
                }
                .alert("Alert", isPresented: $showWrongInfo) {
                    Button("Ok", role: .cancel) {}
                } message: {
                    Text("Username or password verification failed.")
                }
                Spacer()
            }
            .padding(.top, 50)
        }
    }
}
```

## Разширени знания - Alert с няколко бутона

В предишната част на този урок използвахме най-базовия `Alert`, който има само един бутон и се използва основно за подсказка към потребителя.

Всъщност `Alert` може да съдържа и няколко бутона, за да позволи на потребителя да направи различен избор.

Например:

```swift
Button("Sign in") {
	showAlert = true
}
.alert("Alert", isPresented: $showAlert) {
    Button("Ok") {
        print("click Ok")
    }
    Button("delete", role: .destructive) {
        print("click delete")
    }
    Button("cancel", role: .cancel) {
        print("click cancel")
    }
} message: {
    Text("Delete the file?")
}
```

След натискане на бутона ще се покаже прозорец с три опции: `Ok`, `delete` и `cancel`.

Ефект на показване:

![alert](../../Resource/020_alert2.png)

При натискане на различни бутони ще се изпълни съответният код.

### Свойството role на Button

Тук ще забележиш, че `Button` в `Alert` е малко по-различен от обикновения бутон, който научихме по-рано:

```swift
Button("delete", role: .destructive) {
    print("click delete")
}
```

Тук има още един параметър `role`.

В SwiftUI `role` се използва, за да каже на системата какъв тип действие представлява този бутон.

Няколко често срещани роли са:

- `.cancel`: действие за отказ
- `.destructive`: опасно действие, например изтриване
- `nil`: обикновено действие

Различните `role` обикновено водят и до различен визуален стил в интерфейса.

Например опасните действия често се показват с по-забележим стил, за да напомнят на потребителя да натиска внимателно.

Освен това `role` предоставя допълнителна семантична информация за бутона, което помага на екранните четци (например VoiceOver на устройствата на Apple) да подсказват предназначението му на потребителя.

Затова е много добър навик да добавяме `role` на бутоните в подходящите ситуации.
