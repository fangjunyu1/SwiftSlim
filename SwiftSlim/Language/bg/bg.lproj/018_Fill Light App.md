# Приложение за запълваща светлина

В този урок ще създадем едно много интересно приложение за запълваща светлина. Когато настъпи вечерта, можем да накараме екрана на телефона да показва различни цветове и да го използваме като проста запълваща светлина.

Това приложение може да сменя цветовете при докосване на екрана, а също така може да регулира яркостта с плъзгач.

В този пример ще научим как да използваме `brightness` за регулиране на яркостта на изгледа, `onTapGesture` за добавяне на жест при докосване към изглед и контролата `Slider`.

![Color](../../Resource/018_color.png)

## Показване на цвят

Първо, нека накараме изгледа да показва един цвят.

В SwiftUI `Color` не само представлява цвят, но може да се показва и като изглед:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
            .ignoresSafeArea()
    }
}
```

Тук `Color.red` представлява червен изглед. `.ignoresSafeArea()` кара цветния изглед да запълни целия екран, така че да изглежда повече като истински ефект на запълваща светлина.

Резултат:

![Color](../../Resource/018_color1.png)

### Масив от цветове и индекс

В момента се показва само един цвят. Но запълващата светлина обикновено не е само в един цвят. Тя може също да показва синьо, жълто, лилаво, бяло и други цветове.

Искаме да превключваме между различни цветове, когато екранът бъде докоснат. Можем да сложим тези цветове в масив за общо управление:

```swift
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
```

Масивът е подходящ за съхраняване на "група данни от един и същи тип". Тук всеки елемент в масива е `Color`.

Ако искаме да покажем определен цвят, можем да използваме индекс:

```swift
colors[0]
```

Това означава да вземем цвета на индекс `0` в масива, тоест първия цвят.

Сега кодът може да бъде написан така:

```swift
struct ContentView: View {
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[0]
            .ignoresSafeArea()
    }
}
```

По този начин екранът показва първия цвят в масива, който е червен.

### Използване на индекса за управление на цвета

Ако трябва да превключваме между различни цветове, ни трябва променлива, която да управлява индекса, вместо индексът да е фиксиран.

Можем да използваме `@State`, за да декларираме променлива, която пази индекса:

```swift
@State private var index = 0
```

Тук `index` представлява индекса на текущия цвят.

Когато `index` се промени, SwiftUI преизчислява интерфейса и обновява показваното съдържание.

След това променяме първоначалното `colors[0]` на:

```swift
colors[index]
```

По този начин цветът, показван в изгледа, се определя от `index`.

Сега кодът става:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        colors[index]
            .ignoresSafeArea()
    }
}
```

Когато `index` се промени, `colors[index]` също ще показва различен цвят.

Например:

```swift
@State private var index = 0
colors[index]   // Color.red

@State private var index = 1
colors[index]   // Color.blue
```

Важно е да се отбележи, че `index` не може да надвишава най-големия индекс в масива, иначе ще възникне грешка за излизане извън диапазона.

## Жест при докосване

Сега можем да показваме различни цветове според `index`, но все още не можем да ги сменяме с докосване.

В предишния урок "Въртележка с цитати" използвахме `Button`, за да управляваме смяната на цитатите.

Този път обаче искаме "да докоснем цялата цветна област", за да сменим цвета, така че `onTapGesture` е по-подходящ.

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index += 1
                }
        }
    }
}
```

Когато докоснем цветния изглед, се изпълнява този код:

```swift
index += 1
```

Това означава да увеличим `index` с `1`. След като индексът се увеличи, `colors[index]` ще покаже следващия цвят в масива.

### `onTapGesture`

`onTapGesture` е модификатор за жест, който добавя действие при докосване към даден изглед.

Основна употреба:

```swift
.onTapGesture {
    // code
}
```

Например:

```swift
Color.red
    .onTapGesture {
        print("Click color")
    }
```

Когато този червен изглед бъде докоснат, кодът в скобите се изпълнява и в конзолата се извежда:

```swift
Click color
```

Чрез `onTapGesture` можем да определим какво да се случва след докосване на даден изглед.

### Разлика спрямо `Button`

По-рано научихме за изгледа `Button`. И `Button`, и `onTapGesture` могат да обработват действия при докосване, но техните сценарии на употреба не са напълно еднакви.

`onTapGesture` е по-подходящ за "добавяне на функция за докосване към вече съществуващ изглед", като `Color`, `Image`, `Text` или друг обикновен изглед.

`Button` е по-подходящ, когато представяме ясен бутон като "Потвърди", "Изпрати" или "Изтрий".

В това приложение за запълваща светлина искаме смяната на цветовете да е по-проста. Докосването на цялата цветна област сменя цвета, така че `onTapGesture` е добър избор тук.

## Проблемът с индекса

Сега можем да докосваме екрана, за да сменяме различни цветове.

Но тук има един важен проблем: **индексът може да излезе извън диапазона на масива**.

Например:

```swift
@State private var index = 4
let colors = [Color.red, Color.blue, Color.yellow, Color.purple]

colors[index]   // Error
```

Ако продължаваме да докосваме екрана, `index` накрая ще стане `4`, и тогава ще възникне грешка "index out of range".

Това е така, защото масивът `colors` има `4` елемента, но индексирането започва от `0`, така че валидният диапазон на индексите е `0 - 3`, а не `4`.

Ако достъпим `colors[4]`, ще възникне грешка "index out of range".

В текущия код всяко докосване увеличава `index` с `1`. Ако не го обработим, той неизбежно ще излезе извън допустимия диапазон.

Затова трябва да проверим индекса при докосване на екрана: ако вече е последният цвят, връщаме се към първия; иначе продължаваме да добавяме `1`.

Можем да направим това с `if` израз:

```swift
.onTapGesture {
    if index == colors.count - 1 {
        index = 0
    } else {
        index += 1
    }
}
```

В този код `colors.count` означава броя на елементите в масива.

В текущия масив има 4 цвята, така че:

```swift
colors.count // 4
```

Но най-големият индекс не е `4`, а `3`, защото индексирането започва от `0`.

Затова последният индекс трябва да бъде написан така:

```swift
colors.count - 1
```

Тоест:

```swift
4 - 1 = 3
```

Тази логика означава: ако текущият индекс вече съответства на последния цвят, нулирай индекса на `0`; в противен случай го увеличи с `1`.

По този начин цветовете могат да се сменят циклично.

### Оптимизиране на логиката за индекса

Ако искаме кодът да е по-кратък, можем също да използваме тернарния оператор:

```swift
.onTapGesture {
    index = index == colors.count - 1 ? 0 : index + 1
}
```

Този код означава: ако `index == colors.count - 1` е вярно, върни `0`. В противен случай върни `index + 1`.

Накрая присвояваме резултата обратно на `index`.

Сега можем да постигнем ефекта на смяна на цветовете.

Пълен код:

```swift
struct ContentView: View {
    @State private var index = 0
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        VStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
        }
    }
}
```

Сега можем да докосваме екрана и да сменяме различни цветове. Основното приложение за запълваща светлина е готово.

## Показване на името на цвета

Можем да добавим още една група текстове, съответстващи на цветовете, така че когато цветът се промени, на екрана да се показва и името на текущия цвят.

Например:

- Червеното показва `Red`
- Синьото показва `Blue`
- Жълтото показва `Yellow`
- Лилавото показва `Purple`

Тук също можем да използваме масив, за да съхраняваме имената на цветовете:

```swift
let colorsName = ["Red", "Blue", "Yellow", "Purple"]
```

Редът на текста в този масив трябва точно да съответства на реда на цветовете в цветния масив.

След това можем да използваме `Text`, за да покажем името на цвета за текущия индекс:

```swift
Text(colorsName[index])
```

`Text` показва текущото име на цвета според `index`.

Използвайте модификатори, за да подобрите външния вид на `Text`:

```swift
Text(colorsName[index])
    .font(.title)
    .fontWeight(.bold)
    .foregroundStyle(Color.white)
```

Това прави `Text` да се показва в бяло, с размер на заглавие и удебелен шрифт.

Сега имаме `Color` изглед на цял екран. Ако искаме `Text` да се появи върху `Color` изгледа, трябва да използваме контейнера за подреждане `ZStack`, за да ги насложим.

```swift
ZStack {
    Color
    Text
}
```

Така кодът става:

```swift
struct ContentView: View {
    @State private var index = 0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
        }
    }
}
```

По този начин фонът е цветният изглед, а текстовият изглед е наслоен отгоре.

Резултат:

![Color](../../Resource/018_color2.png)

Важно е да се отбележи, че в `ZStack` изгледите, написани по-късно, обикновено се появяват отпред. Ако `Text` е написан преди `Color`, по-късният `Color` изглед може да покрие `Text`.

## Управление на яркостта

Сега можем да превключваме между различни цветове, но запълващата светлина има още една важна функция: **регулиране на яркостта**.

В SwiftUI можем да използваме модификатора `brightness`, за да регулираме яркостта на даден изглед.

Например:

```swift
.brightness(1)
```

Можем да напишем:

```swift
colors[index]
    .brightness(0.5)
```

Това прави текущия цвят по-ярък и по-близък до ефекта на запълваща светлина.

Диапазонът на яркостта е `0 - 1`. `0` означава запазване на оригиналния цвят, а колкото по-близка е стойността до `1`, толкова по-ярък става цветът. `1` означава най-яркия бял резултат.

Въпреки че можем да контролираме `brightness` в кода, потребителят все още не може да го настройва директно.

Затова трябва да добавим контролен елемент, който може да се плъзга: `Slider`.

## Изглед `Slider`

В SwiftUI `Slider` е контрола, използвана за избор на стойност в даден диапазон. Apple я описва като "контрола за избор на стойност от ограничен линеен диапазон".

Основна употреба:

```swift
Slider(value: $value, in: 0...1)
```

Обяснение на параметрите:

1. `value: $value`: `Slider` трябва да бъде обвързан с променлива.

    Когато плъзгачът се мести, стойността на променливата се променя едновременно. Обратно, ако променливата се промени, плъзгачът също се обновява.

    Това е много подобно на `TextField`, който научихме по-рано. И двете "обвързват контрола с променлива".

    Пред обвързаната променлива трябва да има символ `$`, за да се обозначи обвързване.

2. `in: 0...1`: Този параметър представлява диапазона от стойности на плъзгача.

    Тук `0...1` означава, че минималната стойност е `0`, а максималната стойност е `1`.

    Когато плъзгачът е преместен най-вляво, обвързаната променлива е близо до `0`; когато е преместен най-вдясно, тя е близо до `1`.

Например:

```swift
@State private var value = 0.0

HStack(spacing:20) {
    Text("0")
        .foregroundStyle(.gray)
    Slider(value: $value, in: 0...1)
        .frame(width: 100)
    Text("1")
}
```

`Slider` е обвързан с променливата `value`. Когато плъзгачът се мести, стойността на `value` се променя едновременно.

Външен вид:

![Slider](../../Resource/018_slider.png)

Когато `Slider` бъде преместен наляво, обвързаната стойност `value` става `0`. Когато бъде преместен надясно, става `1`.

### Диапазон на стойностите

Диапазонът на стойностите на `Slider` не е фиксиран. Може също да бъде записан така:

```swift
0...100
```

или като друг диапазон.

Но в това приложение за запълваща светлина трябва да управляваме яркостта, така че `0...1` е най-подходящият избор.

## Използване на `Slider` за управление на яркостта

Сега трябва да свържем `Slider` с `brightness`.

Първо, създайте променлива за съхраняване на стойността на яркостта:

```swift
@State private var slider = 0.0
```

Тук `0.0` е стойност от тип `Double`.

Тъй като `Slider` обикновено е обвързан с числов тип, а тук искаме той да се променя непрекъснато, `Double` е по-подходящ. Освен това `brightness` може да приема само стойност от тип `Double`.

След това подайте тази стойност към `brightness`:

```swift
colors[index]
    .brightness(slider)
```

Когато `slider == 0`, цветът остава в състоянието си по подразбиране. Колкото по-близо е `slider` до `1`, толкова по-ярък изглежда цветът.

### Добавяне на контролата `Slider`

След това добавете контрола `Slider`, за да променяте тази променлива:

```swift
Slider(value: $slider, in: 0...1)
```

Когато плъзгачът се промени, стойността на `slider` също се променя, а `brightness(slider)` обновява яркостта едновременно.

Това е много типичен пример за "променливите управляват изгледа" в SwiftUI.

### Настройване на външния вид на `Slider`

По подразбиране `Slider` използва наличната ширина.

Можем да му зададем фиксирана ширина:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
```

След това добавяме още няколко модификатора, за да бъде по-лесно видим:

```swift
Slider(value: $slider, in: 0...1)
    .frame(width: 200)
    .padding(20)
    .background(Color.white)
    .cornerRadius(10)
```

Това дава на плъзгача бял фон и заоблени ъгли, което го кара да изпъква по-ясно върху изгледа `Color`.

Накрая го поставете в долната част на екрана.

Тъй като вече използвахме `ZStack`, можем да поставим `VStack` вътре в него и да използваме `Spacer()`, за да избутаме `Slider` надолу.

## Пълен код

```swift
struct ContentView: View {
    @State private var index = 0
    @State private var slider = 0.0
    let colorsName = ["Red", "Blue", "Yellow", "Purple"]
    let colors = [Color.red, Color.blue, Color.yellow, Color.purple]
    var body: some View {
        ZStack {
            colors[index]
                .brightness(slider)
                .ignoresSafeArea()
                .onTapGesture {
                    index = index == colors.count - 1 ? 0 : index + 1
                }
            Text(colorsName[index])
                .font(.title)
                .fontWeight(.bold)
                .foregroundStyle(Color.white)
            VStack {
                Spacer()
                Slider(value: $slider, in: 0...1)
                    .frame(width: 200)
                    .padding(20)
                    .background(Color.white)
                    .cornerRadius(10)
            }
        }
    }
}
```

## Обобщение

Използвайки знанията, които научихме по-рано, заедно с основни понятия като цветове и масиви, създадохме едно много интересно приложение за запълваща светлина.

Чрез това приложение научихме как да използваме `brightness` за регулиране на яркостта, `onTapGesture` за добавяне на действия при докосване към изгледи и контролата `Slider`.

Добавихме `onTapGesture` към изгледа `Color`, за да сменяме цветовете. Използвахме и контролата `Slider`, за да променяме променлива, която управлява `brightness`, което е още един пример за това как "променливите управляват изгледа".

Също така преговорихме тернарния оператор, използвахме `ZStack`, за да наслагваме изгледи, и използвахме масиви, за да управляваме група данни от един и същи тип. Това помага да задълбочим разбирането си за масиви и индекси, включително практичния проблем с избягването на грешки при излизане на индекса извън диапазона.

Въпреки че този пример не е сложен, той свързва много от основните идеи, които научихме преди. Когато те се комбинират в малък реален проект, става по-лесно да разберем за какво се използва всяка концепция.
