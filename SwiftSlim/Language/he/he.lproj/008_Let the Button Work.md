# לגרום לכפתור לעבוד

בשיעור הזה נלמד על כפתורי `Button`, על `print` לצורך הדפסת מידע לדיבוג, ועל פונקציות ב-Swift.

כל אלה הם יסודות חשובים של תכנות אינטראקטיבי ב-SwiftUI.

## כפתורים

כפתור הוא אחד מרכיבי ה-UI הנפוצים ביותר. כאשר אנחנו רוצים להוריד, לפתוח או לצאת, בדרך כלל נלחץ על כפתור.

לדוגמה, ב-App Store לוחצים על כפתור “Get” כדי להוריד אפליקציה.

![Button](../../RESOURCE/008_button.png)

ב-SwiftUI, כפתור מיוצג באמצעות `Button`.

שימוש בסיסי:

```swift
Button("") {
    
}
```

אפשר לחלק את המבנה לשני חלקים:

```swift
Button("טקסט הכפתור") {
    קוד שמתבצע בלחיצה על הכפתור
}
```

למשל:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

לאחר ההרצה יופיע כפתור בשם `Start`.

![Button](../../RESOURCE/008_button1.png)

כאשר המשתמש לוחץ על הכפתור, הקוד שבתוך `{}` יתבצע.

חשוב לשים לב: `Button` הוא רכיב ממשק של SwiftUI (`View`), ולכן חייבים לכתוב אותו בתוך ה-`body`.

### modifier בשם `buttonStyle`

ב-SwiftUI, `Button` מספק כמה סגנונות מובנים של המערכת, וניתן להשתמש בהם דרך modifier בשם `buttonStyle`.

למשל:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

`buttonStyle` משתמש בסגנון הכפתור שמספקת המערכת.

אפשרויות נפוצות כוללות:

- `.automatic`
- `.plain`
- `.bordered`
- `.borderedProminent`
- `.borderless`

![Button](../../RESOURCE/008_button7.png)

ההבדלים בין הסגנונות באים לידי ביטוי בעיקר בכך שלכפתור יש או אין מסגרת, רקע, וכמה דגש חזותי הוא מקבל.

## פלט של `print`

בתכנות, לעיתים קרובות צריך לבדוק האם התוכנית הגיעה לקטע קוד מסוים.

Swift מספקת את הפונקציה `print` כדי להדפיס מידע לדיבוג.

כתיבה בסיסית:

```swift
print("Hello")
```

השורה הזאת תדפיס טקסט אל ה-`Console`.

### בדיקת תקינות הכפתור

אפשר לכתוב `print` בתוך הכפתור כדי לבדוק האם לחיצה עליו אכן עובדת.

```swift
Button("Start") {
    print("Test 123")
}
```

כאשר לוחצים על הכפתור, יופיע ב-Console:

```
Test 123
```

זה אומר שהכפתור הופעל בהצלחה.

![Button](../../RESOURCE/008_button3.png)

שימו לב: `print` הוא קוד Swift רגיל, ולא רכיב ממשק. אי אפשר לשים אותו ישירות בתוך ה-`body`, אחרת תופיע שגיאה.

למשל, הכתיבה הבאה שגויה:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // שגיאה: body חייב להחזיר View
    }
}
```

ב-SwiftUI, `body` מתאר את מבנה הממשק, ולכן רק רכיבי ממשק (`View`) יכולים להופיע בו, כמו `Text`、`Button`、`Image` ועוד.

קוד רגיל (חישובים, `print` וכן הלאה) חייב להופיע בתוך כפתור, פונקציה, או מטפל אירועים אחר.

### קונסול

הפלט של `print` מופיע ב-`Console` של Xcode.

אם אינכם רואים את ה-`Console`, אפשר ללחוץ על הכפתור `Show the Debug Area` בפינה הימנית התחתונה כדי לפתוח את אזור הדיבוג, ואז ללחוץ על `Show the Console` כדי להציג את הקונסול.

![](../../RESOURCE/008_button4.png)

בפינה השמאלית התחתונה של ה-Console יש שתי לשוניות: `Executable` ו-`Previews`.

![](../../RESOURCE/008_button5.png)

שתי הלשוניות האלה מתאימות לסביבות הרצה שונות: כאשר עובדים ב-Canvas preview, הפלט של `print` יופיע תחת `Previews`; ואם מריצים את האפליקציה על סימולטור או על מכשיר אמיתי, הפלט יופיע תחת `Executable`.

לכן, אם נראה לכם ש-`print` לא מדפיס דבר, כדאי קודם לבדוק שבחרתם את הלשונית הנכונה.

## דוגמה - ציור ASCII

בכמה פרויקטים אפשר לראות לעיתים קרובות ציורים שנבנים מתווים רגילים, כלומר ASCII Art.

ציור ASCII הוא תמונה שבנויה מתווים רגילים, למשל:

![ASCII](../../RESOURCE/008_ascii.png)

אנחנו יכולים להשתמש בשילוב של כפתור ו-`print` כדי להדפיס ציור ASCII.

קוד לדוגמה:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

כאשר לוחצים על הכפתור, הקונסול יציג:

```
 /\_/\
( o.o )
 > ^ <
```

שימו לב: בתוך מחרוזת, התו `\` צריך להיכתב בתור `\\`, משום ש-backslash הוא תו escape במחרוזות.

## פונקציות

בתוך כפתור אפשר כמובן לכתוב קוד ישירות.

למשל:

```swift
Button("Start") {
    print("Hello")
}
```

אם הקוד קצר, זה בסדר גמור. אבל בפיתוח אמיתי, הלוגיקה שמופעלת על ידי כפתור עלולה להיות ארוכה הרבה יותר.

למשל:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // ייתכן שעוד הרבה קוד
}
```

ככל שהקוד מתארך, כתיבה ישירה בתוך הכפתור הופכת את קוד הממשק למבולגן וקשה יותר לקריאה ותחזוקה.

לכן בדרך כלל נרכז את הקוד הזה בתוך פונקציה, ונקרא לפונקציה מתוך הכפתור.

### מהי פונקציה

פונקציה היא קטע קוד שאפשר להשתמש בו שוב ושוב.

כאשר צריך לבצע קוד מסוים, פשוט קוראים לפונקציה, ואז הקוד שבתוכה רץ.

כך המבנה של הקוד נעשה ברור יותר, וגם קל יותר לעשות בו שימוש חוזר.

### כתיבה בסיסית

ב-Swift מגדירים פונקציה בעזרת מילת המפתח **`func`**:

```swift
func randomInt() {
    // code
}
```

קטע הקוד הזה מגדיר פונקציה.

`randomInt` הוא שם הפונקציה, והוא משמש לזיהוי קטע הקוד הזה.

`()` הוא המקום לפרמטרים. כאן אפשר לקבל נתונים שמגיעים מבחוץ. אם לא צריך נתונים, אפשר להשאיר ריק.

`{}` הוא האזור שבו נמצא קוד הפונקציה, וכל מה שצריך להתבצע נכתב בפנים.

למשל:

```swift
func getName() {
    print("FangJunyu")
}
```

התפקיד של הפונקציה הזו הוא להדפיס טקסט.

### קריאה לפונקציה

הגדרת פונקציה רק יוצרת את הקוד, אבל אם רוצים להפעיל אותו, צריך גם לקרוא לפונקציה.

הדרך לקרוא לה היא לכתוב את שם הפונקציה ואחריו `()`:

```swift
getName()
```

כאשר התוכנית מגיעה לשורה הזו, הקוד שבתוך `getName` יתבצע.

דוגמה מלאה:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

לאחר ההרצה, יופיע ב-Console:

```
FangJunyu
```

### שימוש בפונקציות ב-SwiftUI

בתצוגת SwiftUI, פונקציות בדרך כלל נכתבות מחוץ ל-`body`.

למשל:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

כאשר לוחצים על הכפתור, הפונקציה `getName()` תיקרא.

כך קוד הממשק נשאר נקי, והלוגיקה עצמה מטופלת בתוך פונקציה.

### פרמטרים של פונקציה

לפעמים פונקציה צריכה לבצע פעולה שונה בהתאם לנתונים שונים.

למשל, נניח שאנחנו רוצים שהפונקציה תוכל להדפיס שמות שונים, ולא רק `"FangJunyu"` הקבוע.

כאן נכנסים הפרמטרים לתמונה. אפשר להבין פרמטר כמידע שמעבירים לפונקציה בזמן הקריאה.

למשל:

```swift
func getName(name: String) {
    print(name)
}
```

כעת לפונקציה יש פרמטר חדש בשם `name`, ו-`String` הוא הסוג שלו.

זה אומר שהפונקציה צריכה לקבל נתון מסוג `String`.

מכיוון שהפונקציה עכשיו זקוקה לנתון, חובה לספק אותו בעת הקריאה:

```swift
getName(name: "Sam")
```

כאשר מעבירים `"Sam"` בזמן הקריאה, הפונקציה תשתמש בו לצורך ההדפסה.

תוצאת ההרצה:

```
Sam
```

## דוגמה - שימוש בפונקציה כדי להדפיס ציור ASCII

בדוגמה הקודמת כתבנו את ה-`print` ישירות בתוך הכפתור.

אבל אם ציור ה-ASCII מורכב יותר, הקוד יכול להיות ארוך מדי. במצב כזה אפשר להעביר את הלוגיקה לפונקציה, וכך קוד הממשק נשאר ברור יותר.

קוד לדוגמה:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

לאחר ההרצה, לחיצה על הכפתור תציג בקונסול:

```
 /\_/\
( o.o )
 > ^ <
```

כאן קורים למעשה שני דברים כאשר המשתמש לוחץ על הכפתור:

1. הכפתור קורא לפונקציה `printCat()`
2. קוד ה-`print` שבתוך הפונקציה מתבצע

כך קוד הממשק אחראי רק על הפעלת האירוע, ואילו הלוגיקה עצמה נשמרת בתוך פונקציה.

בפיתוח אמיתי, רוב הפעולות של כפתורים ממומשות באמצעות פונקציות.

## סיכום

בשיעור הזה למדנו שלושה נושאים חשובים:

**1. כפתור `Button`**

`Button` הוא רכיב ב-SwiftUI שמשמש להפעלת פעולה.

```swift
Button("Start") {

}
```

כאשר המשתמש לוחץ על הכפתור, הקוד שבין הסוגריים המסולסלים מתבצע.

**2. פלט של `print`**

`print` משמש להדפסת מידע לדיבוג אל ה-Console.

```swift
print("Hello")
```

כך מפתחים יכולים לבדוק את מצב הריצה של התוכנית.

**3. פונקציות**

פונקציה היא קטע קוד שאפשר להשתמש בו שוב ושוב.

```swift
func sayHello() {
    print("Hello")
}
```

קריאה לפונקציה:

```swift
sayHello()
```

אם הפונקציה צריכה לקבל נתונים, אפשר להשתמש בפרמטרים:

```swift
func sayHello(name: String) {
    print(name)
}
```

בשיעורים הבאים נמשיך ללמוד רכיבי SwiftUI נוספים, וגם איך נתונים מתקשרים עם הממשק.

## תרגול

נסו להשלים את התרגילים הבאים:

1. צרו כפתור שהטקסט עליו הוא `"Hello"`
2. כאשר לוחצים על הכפתור, הדפיסו ל-Console את `Hello Swift`
3. העבירו את קוד ההדפסה לפונקציה, ואז קראו לפונקציה מתוך הכפתור
