# לולאות ורשימות

בשיעור הזה נלמד על לולאת `for` ב-Swift, וגם על `ForEach` ב-SwiftUI.

שתיהן יכולות לעזור לנו לקרוא את האיברים שבמערך לפי הסדר, אחד אחרי השני. בפיתוח מעשי, לעיתים קרובות אנחנו צריכים להוציא קבוצת נתונים פריט אחרי פריט, ואז לעבד או להציג אותם, ולולאות הן בדיוק כלי חשוב לביצוע המשימה הזאת.

כאשר אנחנו גולשים באתרי אינטרנט או משתמשים באפליקציה, אנחנו רואים לעיתים קרובות הרבה רשימות או פריטים. תוכן כזה בדרך כלל נשמר קודם בתוך `מערך`, אחר כך נקרא אחד אחרי השני באמצעות `לולאה`, ולבסוף מוצג בממשק.

לדוגמה, רשימת תוצאות החיפוש של Google:

![Google](../../Resource/017_google.png)

כל פריט ברשימת תוצאות החיפוש בדרך כלל מתאים לפריט נתונים אחד, והוא מוצג אחד אחרי השני באמצעות לולאה.

לכן, `מערכים` ו-`לולאות` מופיעים לעיתים קרובות יחד. ה-`מערך` אחראי לשמירה של קבוצת נתונים, וה-`לולאה` אחראית להוציא את הנתונים האלה לפי הסדר ולבצע עליהם את אותה פעולה.

עכשיו נתחיל מהלולאה הבסיסית ביותר, `for`.

## ספירה לאחור

נניח שאנחנו רוצים לדמות פונקציית ספירה לאחור, ולהדפיס לפי הסדר את `“5, 4, 3, 2, 1”`.

הדרך הישירה ביותר לכתוב זאת היא:

```swift
print(5)
print(4)
print(3)
print(2)
print(1)
```

אמנם בדרך הזאת אפשר לדמות ספירה לאחור, אבל אם יש הרבה תוכן, זה ייראה מאוד חוזר על עצמו, וגם יהיה לא נוח לשנות אחר כך.

בשלב הזה, אפשר קודם לשים את הנתונים בתוך מערך, ואז להשתמש בלולאת `for` כדי להדפיס אותם אחד אחרי השני:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

הכתיבה הזאת יותר תמציתית, וגם יותר קלה להרחבה. אפילו אם יש הרבה מספרים בספירה לאחור, אנחנו לא צריכים לכתוב `print` שורה אחרי שורה, אלא רק לתת ללולאה לטפל אוטומטית בכל איבר במערך.

אנחנו יכולים לשים את קטע הקוד הזה בתוך כפתור כדי לבדוק אותו:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Button("for") {
                let countDown = [5, 4, 3, 2, 1]

                for i in countDown {
                    print(i)
                }
            }
        }
    }
}
```

כאשר לוחצים על הכפתור, לולאת `for` תעבור לפי הסדר על כל מספר שלם בתוך `countDown`, ותדפיס אותו לקונסול.

עכשיו נסתכל איך לולאת `for` עובדת.

## לולאת for

לולאת `for` משמשת לעיבוד קבוצת נתונים לפי הסדר. האובייקטים הנפוצים ביותר הם מערכים וטווחים.

כתיבה בסיסית:

```swift
for איבר in מערךאווחום {
    קודשמורץ
}
```

לדוגמה, קוד הספירה לאחור שראינו קודם:

```swift
let countDown = [5, 4, 3, 2, 1]

for i in countDown {
    print(i)
}
```

כאן, `i` מייצג את האיבר שנשלף כרגע, ואחרי `in` מופיעים הנתונים שעליהם עושים לולאה, כלומר המערך `countDown`.

המשמעות של הקוד הזה היא: התוכנית תשלוף לפי הסדר כל איבר מתוך המערך `countDown`. בכל פעם שהיא שולפת איבר, היא תשתמש זמנית ב-`i` כדי לייצג אותו, ואז תבצע את הקוד שבתוך הסוגריים המסולסלים.

אפשר להבין את תהליך הביצוע כך:

* האיבר הראשון שנשלף הוא `5`, ולכן באותו רגע הערך של `i` הוא `5`
* מתבצעת `print(i)` ומודפס `5`
* לאחר מכן נשלף האיבר הבא `4`
* שוב מתבצעת `print(i)` ומודפס `4`

גם `3`, `2` ו-`1` שאחריהם יתבצעו באותה דרך, אחד אחרי השני.

![for](../../Resource/017_for.png)

כאשר כל האיברים במערך עובדו, לולאת `for` תסתיים אוטומטית.

אנחנו יכולים להשתמש בלולאת `for` כדי לקבל כל איבר במערך, ולהשלים פונקציות כמו ספירה לאחור.

**טיפ**: `i` הוא שם נפוץ מאוד בתוך לולאת for, והוא משמש לייצוג האיבר שנשלף כרגע, אבל זו לא כתיבה קבועה. אפשר גם להשתמש בשמות אחרים באנגלית.

לדוגמה, להשתמש ב-`num` כדי לייצג כל איבר שנשלף:

```swift
for num in countDown {
    print(num)
}
```

בתוך הסוגריים המסולסלים צריך גם להשתמש ב-`num` כדי לייצג את האיבר שנשלף כרגע.

### לולאה על טווח

לולאת `for` יכולה לעבור לא רק על מערכים, אלא גם על טווחים.

לדוגמה, כאשר אנחנו צריכים לחשב את הסכום של המספרים השלמים החיוביים עד `50`, אנחנו צריכים לחבר אחד אחרי השני את כל המספרים השלמים בין `1` ל-`50`.

הדרך הישירה ביותר אולי תהיה:

```swift
let count = 1 + 2 + 3 + ... + 49 + 50
print(count)
```

הבעיה בדרך הזאת היא שאנחנו צריכים לכתוב ידנית כל מספר. זה מסורבל, וגם לא מתאים לתרחישי פיתוח אמיתיים.

בשלב הזה, אפשר להשתמש באופרטור הטווח יחד עם לולאת `for` כדי לעשות זאת.

בשיעור הקודם על “אופרטורים”, כבר למדנו את אופרטור הטווח:

```swift
1...3   // 1, 2, 3
```

כאן, `...` מייצג טווח סגור, כלומר הוא כולל גם את נקודת ההתחלה וגם את נקודת הסיום.

לכן כאן אפשר להשתמש בלולאת `for` על טווח המספרים השלמים מ-`1` עד `50`:

```swift
for i in 1...50 {
    
}
```

אנחנו צריכים לחבר את כל המספרים האלה, ולכן אנחנו צריכים גם משתנה שישמור את הסכום:

```swift
var count = 0
```

ואז בכל פעם שהלולאה רצה, נוסיף את המספר שנשלף כרגע אל `count`:

```swift
var count = 0
for i in 1...50 {
    count += i
}
print(count)	// 1275
```

המשמעות של הקוד הזה היא:

* `count` משמש לשמירת הסכום הנוכחי
* לולאת `for` תשלוף אחד אחרי השני את המספרים השלמים מ-`1` עד `50`
* בכל פעם שנשלף מספר שלם, מתבצעת פעם אחת `count += i`
* כאשר הלולאה מסתיימת, `count` שומר את הסכום הסופי

כך אפשר לקבל את הסכום של כל המספרים השלמים החיוביים עד `50`.

באותה דרך, אם אנחנו צריכים לחשב את הסכום של המספרים השלמים מ-`1` עד `100`, צריך רק לשנות את הטווח:

```swift
var count = 0
for i in 1...100 {
    count += i
}
print(count)	// 5050
```

לולאת `for` יכולה לעבור לא רק על מערכים, אלא גם על טווח. היכולת הזאת לעבד נתונים שוב ושוב לפי הסדר היא מאוד נפוצה בפיתוח.

היא יכולה לעזור לנו לטפל אוטומטית בהרבה עבודות שחוזרות על עצמן, בלי שנצטרך לכתוב ידנית את החיבור של כל מספר, או פעולות חוזרות דומות אחרות.

## לולאות ב-SwiftUI

לולאת `for` הקודמת שייכת לשפת Swift.

אבל ב-SwiftUI, אי אפשר להשתמש ישירות ב-`for` בתוך מבנה של תצוגה כדי לייצר תצוגות:

```swift
VStack {
	for i in 1...100 {	// Error
	    count += i
	}
}
```

הסיבה לכך היא שבתוך `VStack` צריך להיות תוכן שהוא תצוגות SwiftUI, בעוד שלולאת `for` רגילה אינה תצוגה בפני עצמה.

אם אנחנו רוצים ב-SwiftUI ליצור ממשק באופן חוזר לפי התוכן של מערך, אנחנו צריכים להשתמש ב-`ForEach`.

לדוגמה, אם אנחנו רוצים להציג את כל השמות של חברי קבוצה, אפשר לכתוב כך:

```swift
VStack {
    Text("Sam")
    Text("John")
    Text("Wathon")
    Text("Bob")
}
```

תוצאת התצוגה:

![for1](../../Resource/017_for1.png)

אחר כך נוסיף לכל `Text` את אותם modifiers:

```swift
VStack {
    Text("Sam")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("John")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Wathon")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
    Text("Bob")
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

עכשיו כל `Text` יוצג בגופן קטן ועבה יותר.

תוצאת התצוגה:

![](../../Resource/017_for2.png)

אמנם modifiers הופכים את הממשק ליפה יותר, אבל הם גם הופכים את הקוד לארוך יותר ולחוזר יותר.

אם יתווספו עוד שמות, או אם אחר כך נרצה לשנות באופן אחיד את הגופן, הצבע וסגנונות אחרים, נצטרך לכתוב ולשנות הרבה קוד זהה שוב ושוב. זה יהיה מסורבל וגם לא נוח לתחזוקה.

בשלב הזה, אפשר קודם לשים את השמות האלה במערך, ואז להשתמש ב-`ForEach` ב-SwiftUI כדי ליצור תצוגות באופן חוזר לפי הנתונים שבמערך:

```swift
VStack {
    let names = ["Sam", "John", "Wathon", "Bob"]
    ForEach(names, id:\.self) { item in
        Text(item)
            .font(.footnote)
            .fontWeight(.bold)
            .foregroundStyle(Color.black)
    }
}
```

קבוצת הקוד הזאת תמציתית יותר. `ForEach` יקרא אחד אחרי השני כל איבר במערך, ויציג בממשק את התוכן המתאים.

כאשר בהמשך נוסיף שמות חדשים, נצטרך רק לשנות את השמות בתוך המערך `names`, ולא יהיה צורך להוסיף כל פעם שוב `Text` ו-modifiers מורכבים.

## ForEach

`ForEach` הוא מבנה ב-SwiftUI שמשמש ליצירה חוזרת של תצוגות. אפשר להבין אותו כמבנה ב-SwiftUI שדומה ללולאת `for`.

שימוש בסיסי:

```swift
ForEach(מערך, id: \.self) { איבר in
    קודSwiftUI
}
```

לדוגמה, שמות הקבוצה שראינו קודם:

```swift
let names = ["Sam", "John", "Wathon", "Bob"]
ForEach(names, id:\.self) { item in
    Text(item)
        .font(.footnote)
        .fontWeight(.bold)
        .foregroundStyle(Color.black)
}
```

המשמעות של הקוד הזה היא:

`names` הוא המערך שעליו עושים לולאה, כלומר הנתונים שצריכים להיות מוצגים.

`id:\.self` פירושו להשתמש באיבר עצמו שבמערך כדי להבדיל בין כל איבר.

בדוגמה הנוכחית, המערך שומר מחרוזות:

```swift
["Sam", "John", "Wathon", "Bob"]
```

לכן SwiftUI ישתמש ישירות במחרוזות עצמן, כמו "Sam", "John", "Wathon" ו-"Bob", כדי להבדיל בין תכנים שונים.

בשלב ההתחלתי, עבור מערכים פשוטים נפוצים כמו `String` ו-`Int`, בדרך כלל אפשר פשוט לכתוב:

```swift
id: \.self
```

כרגע מספיק רק לזכור את צורת הכתיבה הזאת. אין צורך להבין לעומק חוקים נוספים של `id`.

ה-`item` שב-`item in` מייצג את האיבר שנשלף כרגע.

לדוגמה, בלולאה הראשונה `item` הוא `"Sam"`, ובלולאה השנייה `item` הוא `"John"`.

לכן:

```swift
Text(item)
```

יהפוך אחד אחרי השני ל:

```swift
Text("Sam")
Text("John")
Text("Wathon")
Text("Bob")
```

לכן, התפקיד של הקוד הזה הוא: לשלוף אחד אחרי השני כל שם מתוך המערך `names`, וליצור עבור כל שם תצוגת `Text`.

### תהליך הביצוע

אפשר להבין את תהליך הביצוע כך:

* `ForEach` קורא קודם את האיבר הראשון `"Sam"` מתוך המערך `names`
* `item` מייצג זמנית את `"Sam"`
* `Text(item)` מציג את `"Sam"`
* אחר כך הוא ממשיך לקרוא את האיבר הבא `"John"`
* ואז חוזר ומבצע את אותו קוד תצוגה

בסוף, כל השמות שבמערך יוצגו.

היתרון של כתיבה כזאת הוא שכאשר יש יותר שמות, אין צורך לכתוב שוב ושוב הרבה `Text`, אלא רק לשנות את התוכן של המערך.

### לולאה על תמונות

`ForEach` לא יכול להציג רק טקסט, אלא גם תמונות, צבעים או תצוגות אחרות של SwiftUI.

לדוגמה, נכין 4 תמונות:

```
430F9BEF.jpeg
193E0C8C.jpeg
FEE2AB8D.jpeg
35AA4D15.jpeg
```

גררו את ארבע התמונות האלה לתיקיית Assets ב-Xcode.

![image](../../Resource/017_for3.png)

אם לא נשתמש ב-`ForEach`, נצטרך לכתוב כל תמונה ידנית:

```swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image("430F9BEF")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("193E0C8C")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("FEE2AB8D")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
            Image("35AA4D15")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
        }
    }
}
```

תוצאת התצוגה:

![image](../../Resource/017_for4.png)

אמנם בדרך הזאת אפשר להשיג את הפונקציה, אבל הקוד יהיה ארוך יחסית.

אם יהיו יותר ויותר תמונות, נצטרך לכתוב שוב ושוב יותר ויותר קוד `Image`.

אם אחר כך נרצה לשנות את גודל התמונות באופן אחיד, נצטרך גם לשנות אותן אחת אחת.

בשלב הזה, אפשר לשים את שמות התמונות בתוך מערך, ואז להשתמש ב-`ForEach` כדי ליצור תצוגות תמונה באופן חוזר:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

כאן, `images` הוא מערך של מחרוזות, וכל איבר במערך הוא שם של תמונה.

`ForEach(images, id: \.self)` פירושו לקרוא אחד אחרי השני כל איבר במערך `images`, להשתמש באיבר עצמו כדי להבדיל בין כל פריט, ואז ליצור את התצוגה המתאימה לפי האיבר הנוכחי.

לדוגמה, אם האיבר הראשון שנקרא הוא `"430F9BEF"`, אז:

```swift
Image(item)
```

שווה ערך ל:

```swift
Image("430F9BEF")
```

גם התמונות שאחריו יוצגו אחת אחרי השנייה באותה דרך.

![image](../../Resource/017_for4.png)

כאן אנחנו עדיין משתמשים ב:

```swift
id: \.self
```

הסיבה זהה למה שהיה קודם: מכיוון שהאיברים שבמערך `images` הם גם מסוג מחרוזת פשוט, בשלב ההתחלתי אפשר להשתמש ישירות באיבר עצמו כדרך להבדיל בין כל פריט.

אפשר להבין זאת בפשטות כנוסחה קבועה:

**כאשר `ForEach` עושה לולאה על מערך פשוט כמו `String` או `Int`, בדרך כלל כותבים `id: \.self`.**

### הוספת תמונות

אם אחר כך נצטרך להוסיף תמונות, מספיק רק לשנות את התוכן של המערך:

```
let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15","430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
```

אין צורך לכתוב שוב ושוב קוד `Image` חדש.

אם יש הרבה תמונות, אפשר להשתמש בזה יחד עם `ScrollView`. אחרת, כשהתוכן יחרוג מגבולות המסך, לא יהיה אפשר לצפות בכולו.

```swift
ScrollView {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]
    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

### שינוי תמונות

כאשר אנחנו צריכים לשנות את סגנון התמונות באופן אחיד, מספיק רק לשנות את הקוד שבתוך `ForEach`.

לדוגמה, לשנות את הרוחב ל-`80`:

```swift
ForEach(images, id: \.self) {  item in
    Image(item)
        .resizable()
        .scaledToFit()
        .frame(width: 80)
}
```

כך כל התמונות ישתנו יחד, ולא יהיה צורך לשנות אותן אחת אחת.

זהו אחד התפקידים החשובים של `ForEach`:

**כאשר לכמה תצוגות יש אותו מבנה ורק התוכן שונה, אפשר לכתוב את קוד התצוגה פעם אחת בלבד, ואז לתת לנתונים שבמערך להחליט מה יוצג.**

## סיכום

בשיעור הזה למדנו על לולאת `for` ועל `ForEach`.

לולאת `for` משמשת לעיבוד איברים במערך או בטווח לפי הסדר, והיא נוטה יותר לעיבוד נתונים.

`ForEach` משמש ב-SwiftUI ליצירה חוזרת של תצוגות לפי התוכן של מערך, והוא נוטה יותר להצגת ממשק.

בשלב ההתחלתי הנוכחי, צורת הכתיבה הנפוצה ביותר של `ForEach` היא:

```swift
ForEach(מערך, id: \.self) { איבר in
    קודSwiftUI
}
```

כאן, `id: \.self` פירושו להשתמש באיבר עצמו כדי להבדיל בין כל איבר.

כאשר התוכן במערך הוא מטיפוסים פשוטים כמו `String` ו-`Int`, בדרך כלל אפשר להתחיל קודם עם צורת הכתיבה הזאת בלבד.

בהמשך, כאשר תגלשו באתרים או באפליקציות ותראו קבוצה של טקסטים חוזרים, תמונות או תוכן רשימתי אחר, תוכלו לחשוב שב-SwiftUI, התוכן הזה בדרך כלל מוכנס קודם למערך, ואז מוצג אחד אחרי השני באמצעות `ForEach`.
