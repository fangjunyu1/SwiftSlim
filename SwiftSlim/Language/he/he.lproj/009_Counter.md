# מונה

בשני השיעורים הקודמים למדנו על משתנים, קבועים, כפתורים ומתודות.

בשיעור הזה נחבר את כל הידע הזה וניצור אפליקציית מונה פשוטה.

בדוגמה הזו נלמד איך לשמור ולשנות משתנה, וגם איך לנהל אותו בעזרת `@State`.

## מימוש מונה

מונה משמש בדרך כלל לרישום כמות שצריך להמשיך לצבור. למשל, מספר קפיצות בחבל או מספר הקפות בריצה.

כעת נממש מונה פשוט: נציג מספר וכפתור, וכל לחיצה על הכפתור תגדיל את המספר.

![Num](../../RESOURCE/009_num.png)

### הצגת מספר

ראשית נשתמש ב-`Text` כדי להציג את המספר.

```swift
Text("0")
```

אבל אם המספר צריך להשתנות, אי אפשר לכתוב `"0"` ישירות באופן קשיח. במקום זה צריך לשמור את המספר בתוך משתנה.

```swift
var num = 0
```

ואז להציג את המשתנה בתוך `Text`:

```swift
Text("\(num)")
```

כאן משתמשים באינטרפולציה של מחרוזות `\()` כדי להמיר את המספר למחרוזת ולהציג אותו.

### הגדלת המספר בעזרת כפתור

כעת נוסיף כפתור.

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

כאשר ילחצו על הכפתור, נרצה שהמספר יגדל.

לכן אפשר לשנות את המשתנה בתוך הכפתור:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

הקוד הזה אומר: כאשר לוחצים על הכפתור, מחשבים `num + 1`, ואז מציבים את התוצאה בחזרה בתוך `num`.

### כתיבה מקוצרת יותר

Swift מספקת דרך קצרה יותר לכתוב את אותו הדבר:

```swift
num += 1
```

זה שקול ל:

```swift
num = num + 1
```

הכתיבה הזו נקראת “אופרטור השמה מורכב”.

אופרטורי השמה מורכבים נפוצים ב-Swift כוללים:

```
+=   השמת חיבור
-=   השמת חיסור
*=   השמת כפל
/=   השמת חילוק
%=   השמת שארית
```

למשל:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### קוד מלא

עכשיו אפשר לכתוב מונה פשוט:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

אבל בשלב הזה הקוד עדיין לא יעבוד כמו שצריך.

### הופעת שגיאה

Xcode יציג:

```
Cannot assign to property: 'self' is immutable
```

כלומר אי אפשר לשנות את התכונה, כי התצוגה היא בלתי-ניתנת לשינוי.

למה זה קורה?

## איך SwiftUI עובד

ב-SwiftUI: **הממשק מונע על ידי נתונים**. כלומר, כאשר הנתונים משתנים, הממשק מתעדכן אוטומטית.

אבל חשוב לשים לב: רק כאשר מצב (`state`) משתנה, כמו `@State` או `@Binding`, SwiftUI עוקב אחרי השינוי ומרענן את התצוגה.

אם משתמשים במשתנה רגיל, SwiftUI קורא את הערך שלו רק פעם אחת בזמן יצירת התצוגה, וגם אם אחר כך הוא ישתנה, הממשק לא יתעדכן.

למשל:

```swift
var num = 0   // 0
num = 1   // 0
```

בדוגמה למעלה, `num` הוא משתנה רגיל. הערך שלו השתנה, אבל SwiftUI לא יעדכן את התצוגה, ולכן הממשק עדיין יציג את הערך ההתחלתי `0`.

![Num](../../RESOURCE/009_state.png)

לכן רק כאשר הנתון מסומן כ-state (למשל באמצעות עטיפות תכונה כמו `@State` או `@Binding`), SwiftUI יעקוב אחריו אוטומטית ויעדכן את התצוגה.

## עטיפת התכונה `@State`

ב-SwiftUI, אם רוצים לשנות משתנה וגם לעדכן את התצוגה, חייבים להשתמש ב-`@State`.

```swift
@State var num = 0
```

`@State` גורם ל-SwiftUI לעקוב אחר המשתנה הזה.

כאשר המשתנה משתנה, SwiftUI מחשב מחדש את התצוגה ומעדכן אותה.

קוד מלא:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

עכשיו, כאשר לוחצים על הכפתור:

```swift
num += 1
```

SwiftUI יזהה ש-`num` השתנה, ואז יעדכן את התצוגה באופן אוטומטי.

![Num](../../RESOURCE/009_state1.png)

## כללי שימוש ב-`@State`

בפיתוח בפועל, `@State` בדרך כלל פועל לפי שני כללים.

### 1. בדרך כלל כותבים `@State` עם `private`

משתנה `@State` משמש בדרך כלל רק בתוך הטיפוס הנוכחי (למשל בתוך המבנה של ה-`View` הנוכחי), ולכן נהוג לכתוב:

```swift
@State private var num = 0
```

כך תצוגות אחרות לא יוכלו לגשת ישירות למשתנה הזה או לשנות אותו.

עוד לא למדנו לעומק את `private`, ולכן כרגע אפשר להבין אותו בפשטות כך:

**תוכן שמסומן כ-`private` ניתן לשימוש רק בתוך התצוגה הנוכחית, ולא מבחוץ.**

### 2. `@State` משמש רק למצב של התצוגה הנוכחית

`@State` מתאים לנתונים שהתצוגה הנוכחית מנהלת בעצמה.

למשל, תוכן של שדה קלט או מצב של מתג.

```swift
@State private var isOn = false
@State private var text = ""
```

אם הנתונים צריכים להיות משותפים בין כמה תצוגות, צריך להשתמש בסוגי state אחרים, כמו `@Binding` או `@Observable`.

נושאים אלה יילמדו בהמשך.

### כפתור להפחתה

למונה לא חייבת להיות רק אפשרות להגדיל. אפשר גם להקטין.

אפשר להוסיף כפתור `-`.

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

לאחר מכן נשתמש ב-`HStack` כדי לסדר את הכפתורים אופקית:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

התוצאה:

![Num](../../RESOURCE/009_num1.png)

כאשר לוחצים על `+`, המספר גדל ב-1; כאשר לוחצים על `-`, המספר קטן ב-1.

### כפתור איפוס

אפשר גם להוסיף כפתור איפוס שיחזיר את המספר ל-0.

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

כאשר לוחצים על כפתור האיפוס, `num` מקבל את הערך `0`.

התוצאה:

![Num](../../RESOURCE/009_num2.png)

כאשר לוחצים על כפתור `0`, המספר חוזר ל-0.

### כפתור מותאם אישית

כרגע הכפתורים שלנו יכולים להציג רק טקסט:

```swift
Button("+") {

}
```

בכתיבה הזו, הכפתור מציג רק תוכן טקסטואלי, למשל `+` או `-`.

אבל בפיתוח אמיתי בדרך כלל נרצה מראה עשיר יותר לכפתורים, כמו שימוש באייקונים או בתצוגות אחרות.

SwiftUI מאפשר להתאים אישית את התוכן שמוצג על הכפתור, ובמקרה כזה משתמשים בצורה אחרת של `Button`:

```swift
Button(action: {

}, label: {

})
```

בצורה הזו, `action` מייצג את הקוד שמתבצע בלחיצה, ו-`label` מייצג את התצוגה שמופיעה על הכפתור.

למשל, אפשר לגרום לכפתור להציג אייקון של `SF Symbols`:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

כך הכפתור כבר לא יציג טקסט, אלא אייקון.

### שיפור ממשק המונה

כדי להפוך את ממשק המונה לברור יותר, אפשר גם לשפר את סגנון התצוגה של המספר. למשל, להגדיל אותו:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

לאחר מכן נחליף גם את התוכן של הכפתורים לאייקונים, ונגדיל מעט את גודל האייקונים:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

באותה דרך אפשר להפוך גם את כפתור ההפחתה ואת כפתור האיפוס לכפתורי אייקון:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

עכשיו הכפתורים מציגים אייקונים של `SF Symbols`, והממשק נראה ברור ואינטואיטיבי יותר.

![Num](../../RESOURCE/009_num3.png)

## סיכום

בשיעור הזה, באמצעות יצירת מונה פשוט, תרגלנו יחד שימוש במשתנים, `Text` ו-`Button`. השתמשנו במשתנה כדי לשמור את המספר, ובכפתורים כדי לשלוט בהגדלה ובהקטנה שלו, ובדרך גם למדנו על אופרטורי השמה מורכבים כמו `num += 1` ו-`num -= 1`.

בתהליך המימוש, הבנו גם איך SwiftUI עובד: הממשק מונע על ידי נתונים. רק כאשר הנתונים משתנים, התצוגה מתרעננת. לכן צריך להשתמש ב-`@State` כדי לשמור מצב שניתן לשינוי. כאשר משתנה `@State` משתנה, SwiftUI יעדכן אוטומטית את התצוגה.

לבסוף גם שיפרנו מעט את הממשק: שינינו את סגנון הגופן של `Text`, והשתמשנו ב-`Button(action:label:)` כדי להתאים אישית את תוכן הכפתורים כך שיוכלו להציג אייקונים. כך ממשק המונה נעשה ברור ויפה יותר.

עכשיו כבר נחשפנו לשימוש הבסיסי בתצוגות של SwiftUI. ב-SwiftUI, רוב הממשקים בנויים סביב משתנים. כאשר המשתנים משתנים, SwiftUI מעדכן אוטומטית את התצוגה. לכן, בפיתוח אפליקציות SwiftUI, לרוב מתחילים בתכנון הנתונים שצריך לשמור, ורק אחר כך בונים בעזרתם את הממשק ואת לוגיקת האינטראקציה.

## הקוד המלא

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
