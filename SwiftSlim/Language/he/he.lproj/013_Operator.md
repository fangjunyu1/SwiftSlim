# אופרטורים

בשיעור הזה נלמד על אופרטורים (`Operators`) ב-Swift.

כאשר אנחנו מבצעים חישובים, השוואות או שיפוט לוגי, אנחנו חייבים להשתמש באופרטורים. הפעולות האלה מופיעות כמעט בכל חלק של הרצת התוכנית, ולכן מדובר באחת היכולות הבסיסיות ביותר בתכנות.

```swift
let a = 100 * 8
```

במהלך לימוד תכנות, הרבה אנשים מגלים שלמי שיש חשיבה מתמטית מסוימת, קל יותר להבין לוגיקה של תוכניות. הסיבה לכך היא שבבסיס התכנות יש תמיד “חישוב” ו”שיפוט לפי כללים”, ואלה בדיוק התפקידים המרכזיים של האופרטורים.

בפיתוח אמיתי, אופרטורים לא משמשים רק לחישובים מספריים פשוטים, אלא מופיעים גם במגוון רחב של תרחישים מעשיים.

למשל, במשחק בלוקים, כאשר המשתמש גורר בלוק למקום מסוים, צריך לחשב:

- את מיקום הקואורדינטות של הבלוק
- את המיקום שבו הבלוק יכול להתמלא
- האם יעד ההצבה זמין או לא

התהליך הזה של “חישוב + בדיקה” נשען כולו על אופרטורים.

![Block](../../Resource/013_block.png)

ללמוד אופרטורים זה לא רק ללמוד חשבון, אלא גם להניח יסוד לפריסה של ממשקים, לעיבוד נתונים וללוגיקת אינטראקציה בהמשך.

## איך החישוב מתבצע ב-SwiftUI

ב-SwiftUI, נהוג בדרך כלל להפעיל פונקציה דרך `Button`, ואז לבצע את לוגיקת החישוב בתוך הפונקציה.

למשל:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

כאשר לוחצים על הכפתור, הקונסול מציג:

```
c: 20
```

בשיעור הזה, נכתוב את לוגיקת החישוב בתוך `func`, נפעיל אותה דרך כפתור, ונשתמש ב-`print` כדי להציג את התוצאה.

בדוגמאות הבאות, כדי לפשט את הצגת הקוד, נציין ישירות את תוצאות החישוב בתוך הקוד, ולא נחזור שוב ושוב על `print`.

## ניהול הפלט של הקונסול

מכיוון שכל לחיצה מייצרת פלט חדש, אפשר ללחוץ על כפתור הניקוי בפינה הימנית התחתונה של אזור ה-Console כדי למחוק את ההיסטוריה ולבדוק בקלות את התוצאה הנוכחית.

![Console](../../Resource/013_console.png)

## דרך נוספת לכתוב `Button`

כאשר הכפתור מבצע רק פונקציה אחת, אפשר להשתמש בצורה תמציתית יותר:

```swift
Button(action: ) {
    // View
}
```

בכתיבה הזאת, `action` מציין את הפונקציה שצריכה להתבצע בלחיצה על הכפתור (פונקציה ללא פרמטרים), והסוגריים המסולסלים `{}` משמשים להגדרת התצוגה של הכפתור.

אם נחבר את זה לפונקציה `calculate` מהדוגמה הקודמת, אפשר לכתוב:

```swift
Button(action: calculate) {
    Text("Button")
}
```

כאשר לוחצים על הכפתור, הפונקציה `calculate` תתבצע ישירות.

לכן, כאשר הלוגיקה של הכפתור פשוטה והוא רק קורא לפונקציה אחת, הכתיבה באמצעות `action` נראית ברורה יותר, וגם תואמת יותר לסגנון קוד מסודר.

## אופרטורים אריתמטיים

אופרטורים אריתמטיים (`Arithmetic Operators`) משמשים לחישובים מספריים:

- `+` חיבור
- `-` חיסור
- `*` כפל
- `/` חילוק
- `%` שארית

דוגמאות:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

חשוב לשים לב ש-Swift היא שפה חזקה מבחינת טיפוסים, ולכן היא לא מערבבת אוטומטית בין סוגי נתונים שונים:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

כאשר שני הטיפוסים שונים, צריך להמיר ידנית את אחד מהם.

## אופרטורי השוואה

אופרטורי השוואה (`Comparison Operators`) משמשים לבדוק את היחס בין שני ערכים, והתוצאה שלהם היא `Bool`:

- `==` שווה
- `!=` לא שווה
- `>` גדול מ-
- `<` קטן מ-
- `>=` גדול או שווה ל-
- `<=` קטן או שווה ל-

דוגמאות:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

אופרטורי השוואה נפוצים מאוד בתוך משפטי תנאי. למשל, בקרוסלת הציטוטים, צריך לוודא שהאינדקס לא עובר את גבולות המערך:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

אם `index` קטן מ-`sayings.count - 1`, תוצאת ההשוואה היא `true`, ואז הקוד שבתוך `if` יתבצע; אחרת הוא לא יתבצע.

## אופרטורים לוגיים

אופרטורים לוגיים (`Logical Operators`) משמשים לשילוב בין כמה תנאים:

- `&&` וגם (`AND`)
- `||` או (`OR`)
- `!` לא (`NOT`)

### `&&` וגם

כאשר כמה תנאים חייבים להתקיים יחד, משתמשים ב-`&&`.

למשל:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

בקוד הזה יש שני תנאים: `age >= 18` וגם `age <= 100`.

רק אם שני התנאים אמת (`true`), אז `legalAge` יהיה `true`; מספיק שאחד מהם לא מתקיים, והתוצאה תהיה `false`.

### `||` או

כאשר מספיק שתנאי אחד מתוך כמה יתקיים, משתמשים ב-`||`.

למשל:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

הקוד הזה אומר שכל עוד `money >= 50` או `creditCard >= 50` מתקיים, `payAllowed` יהיה `true`.

רק כאשר שני התנאים אינם מתקיימים, התוצאה תהיה `false`.

למשל, בתרחיש של תשלום, אפשר לשלם במזומן או בכרטיס אשראי. אם אף אחת מהאפשרויות לא זמינה, התשלום ייכשל.

### `!` לא

`!` הוא אופרטור השלילה הלוגית (`NOT`), והוא הופך ערך בוליאני לערך ההפוך.

אפשר להבין את זה כך: `!` הופך `true` ל-`false`, ואת `false` ל-`true`.

למשל:

```swift
let allowed = true
let result = !allowed   // false
```

כאן `!allowed` פירושו להפוך את הערך של `allowed`.

חשוב לשים לב ש-`!` ו-`!=` הם שני אופרטורים שונים. `!` הופך ערך בוליאני, ואילו `!=` הוא אופרטור השוואה שמחזיר ערך בוליאני.

## אופרטורי השמה

אופרטורי השמה (`Assignment Operators`) משמשים להצבת ערכים או לעדכון של משתנים:

- `=` השמה

גם בזמן הצהרת משתנים וגם בזמן חישובים, פעולת השמה תמיד מופיעה:

```swift
let a = 5
```

הקוד הזה אומר שהערך `5` מוצב במשתנה `a`.

### אופרטורי השמה מורכבים

בפיתוח אמיתי, מלבד אופרטור ההשמה הבסיסי, משתמשים הרבה גם ב“אופרטורי השמה מורכבים”, שמאפשרים לבצע חישוב על הערך הקיים ולעדכן אותו ישירות:

- `+=`
- `-=`
- `*=`
- `/=`

בשיעור הקודם על המונה כבר פגשנו אותם בקצרה.

למשל, כאשר רוצים להגדיל משתנה בלחיצה על כפתור:

```swift
var num = 10
num += 5   // num = 15
```

הקוד הזה אומר ש-`num` מתחבר עם `5`, ואז התוצאה מוצבת שוב לתוך `num`.

לכן הוא שקול ל:

```swift
var num = 10
num = num + 5   // num = 15
```

## אופרטור תנאי משולש

האופרטור המשולש (`Ternary Operator`) ב-Swift בודק תנאי, ומחזיר אחד מתוך שני ערכים אפשריים:

```swift
condition ? value1 : value2
```

הלוגיקה שלו היא: אם התנאי הוא `true`, מוחזר “ערך 1”; ואם הוא `false`, מוחזר “ערך 2”.

למשל:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

הקוד הזה בודק את הערך של `showColor`: אם הוא `true`, מוחזר `Color.blue` והרקע יוצג בכחול; אם הוא `false`, מוחזר `Color.clear`, כלומר לא יוצג צבע רקע.

טיפ: `Color.clear` מייצג צבע שקוף (כלומר שום דבר לא מוצג).

### ההבדל מול `if-else`

מבחינה לוגית, האופרטור המשולש שקול למשפט `if-else`.

למשל, אפשר לכתוב את הקוד למעלה גם כך:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

הלוגיקה של שתי הגרסאות זהה לחלוטין: כאשר התנאי הוא `true`, משתמשים בתוצאה הראשונה; כאשר הוא `false`, משתמשים בתוצאה השנייה.

מבחינת שימוש מעשי, הכתיבה של האופרטור המשולש קצרה יותר, ומתאימה במיוחד למצבים פשוטים של “בחירה בין שני ערכים”, בעיקר כאשר צריך להחזיר ערך ישירות.

בבניית תצוגות ב-SwiftUI, האופרטור הזה נפוץ מאוד. למשל:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

כך אפשר לשנות אפקט חזותי של תצוגה לפי תנאי, בשורת קוד אחת בלבד.

## אופרטורי טווח

אופרטורי טווח (`Range Operators`) משמשים לייצוג טווחים:

- `...` טווח סגור (כולל את שני הקצוות)
- `..<` טווח חצי-פתוח (לא כולל את הקצה הימני)

למשל:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

אפשר להבין את זה כך: `1...3` פירושו מ-1 עד 3, כולל 3; ואילו `5..<8` פירושו מ-5 עד 8, אבל בלי 8.

צורות הרחבה (להיכרות בלבד):

```
5...    // >= 5
...5    // <= 5
```

הכתיבה הזאת מייצגת “טווח חד-צדדי”, והיא משמשת לעיתים בבדיקות תנאי או בהתאמת תבניות.

אופרטורי טווח ישמשו אותנו בהמשך בלימוד לולאות `for`, וכרגע אפשר להבין אותם פשוט כטווח רציף של מספרים.

## תרגול אופרטורים

כעת נתרגל את תפקידם של אופרטורים דרך כמה דוגמאות פשוטות בתוך התצוגה.

### 1. חישוב מספר כלי הרכב הכולל

למשל, נניח שבצומת מסוים עוברים מדי יום `500` כלי רכב, ונרצה לחשב כמה כלי רכב יעברו בתוך `30` ימים.

אפשר להשתמש באופרטור חישובי:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

כאן תהליך החישוב הוא: `500` כלי רכב ביום, במשך `30` ימים, כלומר `500 × 30`, והתוצאה היא `15000`.

### 2. בדיקת מנוי לרכישה בתוך האפליקציה

אפשר לבדוק אם אדם מסוים הוא מנוי בתוך האפליקציה, כל עוד יש לו אחת משתי האפשרויות: מנוי לכל החיים או מנוי חודשי.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

בקוד הזה משתמשים באופרטור `||` (או): `lifeTime = false` פירושו שאין מנוי לכל החיים; `subscription = true` פירושו שיש מנוי פעיל.

מכיוון שמספיק שאחד מהתנאים מתקיים, `isMember` הוא `true`. לכן אותו אדם נחשב לחבר במועדון הרכישה בתוך האפליקציה.

## סיכום

ב-Swift, אופרטורים הם היסוד של חישוב נתונים ושל שיפוט לוגי. בדיוק כפי ש-`Text` מציג תוכן, אופרטורים מטפלים בנתונים עצמם.

החל מחישובים מספריים פשוטים, דרך תנאים לוגיים מורכבים, ועד שליטה על מצב הממשק וחישובי פריסה, אי אפשר להסתדר בלי אופרטורים.

באמצעות תרגול בתרחישים שונים, אפשר להתרגל בהדרגה לסוגי האופרטורים שהוצגו בשיעור הזה, וזה יהפוך לבסיס חשוב להמשך הלמידה.

### הרחבה - אופרטורי סיביות

בנוסף, Swift מספקת גם קבוצה של אופרטורים ברמה נמוכה יותר — אופרטורי סיביות (`Bitwise Operators`):

- `&` (AND ביטי)
- `|` (OR ביטי)
- `^` (XOR ביטי)
- `~` (NOT ביטי)
- `>>` (הזזה ימינה)
- `<<` (הזזה שמאלה)

למשל:

```swift
let a = 6  // בינארי 110
let b = 3  // בינארי 011

print(a & b) // 2 (בינארי 010)
print(a | b) // 7 (בינארי 111)
print(a ^ b) // 5 (בינארי 101)
print(~a)    // -7 (היפוך ביטי)
```

ב-Swift, מספרים שלמים מיוצגים בבינארי בשיטת המשלים לשתיים, ולכן לאחר היפוך מתקבלת תוצאה שלילית מתאימה.

אופרטורי סיביות פועלים ישירות על הייצוג הבינארי, והם משמשים בדרך כלל לעיבוד נתונים ברמה נמוכה או לתרחישי אופטימיזציה של ביצועים.

למתחילים, השימושים של אופרטורים כאלה נדירים יחסית, וגם בפיתוח רגיל של iOS או SwiftUI הם אינם נפוצים כל כך, ולכן כאן לא ניכנס אליהם לעומק.

אם בהמשך נעסוק בלוגיקה ברמה נמוכה יותר או באופטימיזציה של ביצועים, יהיה אפשר ללמוד ולהבין אותם בצורה מעמיקה יותר.
