# התאמת מראה

בשיעור הזה נלמד איך להתאים את הממשק למצב בהיר ולמצב כהה.

ב-iPhone, ב-iPad וב-Mac, המשתמשים יכולים לבחור מצב בהיר או מצב כהה.

כאשר המערכת עוברת למצב כהה, גם צבעי הרקע, הטקסט והאייקונים של הרבה אפליקציות משתנים בהתאם.

לדוגמה, ביום הממשק יכול להשתמש ברקע בהיר, ובלילה הוא יכול להפוך לרקע כהה.

![view](../../../Resource/025_view9.png)

היכולת להתאים אוטומטית את תצוגת הממשק לפי מראה המערכת נקראת התאמת מראה.

## תצוגה מקדימה של התאמת מראה ב-Xcode

לפני שנלמד התאמת מראה, נבין קודם איך להציג ב-Xcode תצוגה מקדימה של מצב בהיר ומצב כהה.

בתחתית אזור ה-`Canvas`, לוחצים על הכפתור `Device Settings`, וכך פותחים את החלון הקופץ `Canvas Device Settings`.

![canvas](../../../Resource/025_view.png)

בחלון הקופץ אפשר לראות את ההגדרה `Color Scheme`.

![canvas1](../../../Resource/025_view1.png)

המשמעות היא:

- `Light Appearance` פירושו מצב בהיר.
- `Dark Appearance` פירושו מצב כהה.

אפשר להחליף כאן את מראה התצוגה המקדימה ולבדוק איך הממשק הנוכחי נראה במצב בהיר ובמצב כהה.

## הצגת שני המראות זה לצד זה

אם רוצים לראות מצב בהיר ומצב כהה באותו זמן, אפשר להשתמש בתכונת `Variants`.

בתחתית אזור ה-`Canvas`, לוחצים על הכפתור `Variants` ובוחרים `Color Scheme Variants`.

![canvas2](../../../Resource/025_view2.png)

לאחר הבחירה, ה-`Canvas` יציג בו-זמנית תצוגות מקדימות של מצב בהיר ומצב כהה.

![canvas3](../../../Resource/025_view3.png)

כך קל יותר להשוות את ההבדלים בממשק בין שני המראות.

במילים פשוטות:

אם רוצים רק להחליף זמנית בין מצב בהיר למצב כהה, אפשר להשתמש ב-`Canvas Device Settings`.

אם רוצים לראות את שני המראות באותו זמן, אפשר להשתמש ב-`Color Scheme Variants`.

## תצוגות מתאימות את עצמן למראה כברירת מחדל

ב-SwiftUI, הרבה תצוגות מערכת מתאימות את עצמן אוטומטית למצב בהיר ולמצב כהה כברירת מחדל.

לדוגמה:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("בהיר")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("משהו משהו משהו")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

תוצאת התצוגה:

![canvas](../../../Resource/025_view4.png)

אפשר לראות שבמצב בהיר, האייקון והכותרת בדרך כלל מוצגים בצבע כהה.

במצב כהה, האייקון והכותרת הופכים אוטומטית לצבע בהיר.

הסיבה היא שתצוגות SwiftUI כמו `Text` ו-`Image` מתאימות כברירת מחדל את הצבעים שלהן לפי מראה המערכת.

כלומר, אם לא נגדיר ידנית צבע קבוע, SwiftUI תטפל עבורנו בחלק מהתאמת המראה.

## צבעים קבועים אינם מתחלפים אוטומטית

חשוב לשים לב: אם נגדיר ידנית צבע קבוע, הוא לא יתחלף אוטומטית בין מצב בהיר למצב כהה.

לדוגמה:

```swift
Text("משהו משהו משהו")
	.foregroundStyle(Color.gray)
```

כאן הטקסט הוגדר כ-`Color.gray`, ולכן הוא תמיד יוצג באפור.

דוגמה נוספת:

```swift
Text("כותרת")
    .foregroundStyle(Color.white)
```

הקוד הזה יציג טקסט לבן גם במצב בהיר וגם במצב כהה.

אם גם הרקע בהיר, ייתכן שיהיה קשה לראות את הטקסט הלבן.

לכן, כשמבצעים התאמת מראה, כדאי להימנע מכתיבה קשיחה אקראית של צבעים קבועים כמו `Color.white` ו-`Color.black`.

במקרים רבים אפשר להעדיף סגנונות סמנטיים של המערכת, למשל:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

המשמעות היא:

- `.primary` משמש בדרך כלל לתוכן ראשי.
- `.secondary` משמש בדרך כלל לתוכן משני.

הם מתאימים את התצוגה שלהם אוטומטית לפי מצב בהיר ומצב כהה.

## הצגת תוכן שונה לפי המראה

לפעמים אנחנו לא רוצים שרק הצבעים ישתנו, אלא גם שיוצג תוכן שונה במראות שונים.

לדוגמה:

- במצב בהיר מוצג אייקון נורה רגיל.
- במצב כהה מוצג אייקון נורה מאירה.
- במצב בהיר הכותרת מציגה `בהיר`.
- במצב כהה הכותרת מציגה `כהה`.

במצב כזה צריך לבדוק אם המערכת הנוכחית נמצאת במצב בהיר או במצב כהה.

ב-SwiftUI אפשר לקבל את מצב המראה הנוכחי דרך ערך הסביבה `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

קוד מלא:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "בהיר" : "כהה"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("משהו משהו משהו")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

תוצאת התצוגה:

![view](../../../Resource/025_view5.png)

מתוצאת התצוגה המקדימה אפשר לראות שאותה תצוגה אינה מציגה בדיוק אותו תוכן במצבי מראה שונים.

במצב בהיר, `Image` מציג את אייקון הנורה הרגיל, והכותרת מציגה `בהיר`.

במצב כהה, `Image` מציג את אייקון הנורה המאירה, והכותרת מציגה `כהה`.

כלומר, השינוי כאן אינו רק שינוי צבע. גם שם האייקון וגם טקסט הכותרת עצמם השתנו.

הסיבה היא שלא כתבנו את האייקון והכותרת ישירות בתוך `body` בצורה קשיחה, אלא חישבנו תוכן שונה לפי `colorScheme`.

בקוד הזה השתמשנו בשני מאפיינים מחושבים:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "בהיר" : "כהה"
}
```

כאשר `colorScheme` הוא `.light`, המשמעות היא שהמצב הנוכחי הוא מצב בהיר.

כאשר `colorScheme` הוא `.dark`, המשמעות היא שהמצב הנוכחי הוא מצב כהה.

לכן אפשר להחזיר שמות אייקונים וטקסט כותרת שונים לפי מצבי מראה שונים.

זהו גם שימוש נפוץ במאפיינים מחושבים: לחשב לפי המצב הנוכחי את התוכן שהתצוגה צריכה להציג.

## הבנת @Environment

זו הפעם הראשונה שאנחנו פוגשים את `@Environment`.

אפשר להבין את `@Environment` כך: קריאת ערך מתוך הסביבה של SwiftUI.

כאשר אפליקציה רצה, המערכת מספקת מידע סביבתי רב, למשל:

- השפה הנוכחית
- מצב המראה הנוכחי
- כיוון הפריסה הנוכחי
- הגדרת גודל הגופן הנוכחית

כאשר אנחנו צריכים לקבל את מצב המראה הנוכחי, אפשר לקרוא את `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

כאן:

```swift
\.colorScheme
```

פירושו קריאת הערך `colorScheme` מתוך הסביבה.

```swift
private var colorScheme
```

פירושו לשמור את הערך שנקרא בתוך המשתנה `colorScheme`.

אפשר לבחור את שם המשתנה בעצמנו. לדוגמה, אפשר גם לכתוב:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

כל עוד ה-`\.colorScheme` שלפניו לא משתנה, המשמעות היא שקוראים את מצב המראה הנוכחי.

## שני הערכים הנפוצים של colorScheme

ל-`colorScheme` יש בדרך כלל שני ערכים:

```swift
.light
.dark
```

המשמעות שלהם:

- `.light`: מצב בהיר
- `.dark`: מצב כהה

לדוגמה:

```swift
private var titleName: String {
    colorScheme == .light ? "בהיר" : "כהה"
}
```

משמעות הקוד הזה היא:

אם המצב הנוכחי הוא מצב בהיר, הביטוי `colorScheme == .light` מתקיים, ולכן מוחזר `"בהיר"`.

אחרת מוחזר `"כהה"`.

לכן, במצב בהיר:

```swift
Text(titleName)
```

הטקסט המוצג הוא:

```swift
בהיר
```

במצב כהה, הטקסט המוצג הוא:

```swift
כהה
```

באמצעות דרך זו אפשר להציג תוכן שונה לפי מראות שונים.

## דוגמה להתאמת צבעים

מלבד טקסט ואייקונים, לפעמים צריך גם לטפל בעצמנו בהתאמת צבעים.

לדוגמה:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("טקסט")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

תוצאת התצוגה:

![view](../../../Resource/025_view6.png)

בקוד הזה, הטקסט לבן והרקע שחור.

במצב בהיר, הרקע השחור ברור למדי.

אבל במצב כהה, אם גם רקע כל הממשק שחור, הרקע השחור הזה יתמזג עם רקע המערכת וייראה פחות ברור.

בשלב הזה אפשר להחליף את צבע הטקסט ואת צבע הרקע לפי מצב המראה:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("טקסט")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

תוצאת התצוגה:

![view](../../../Resource/025_view7.png)

כעת במצב בהיר יש רקע שחור וטקסט לבן.

במצב כהה יש רקע לבן וטקסט שחור.

כך, לא משנה באיזה מראה משתמשים, לטקסט ולרקע יש ניגודיות ברורה, והמשתמשים יכולים לראות את התוכן בבירור.

## שימוש ב-Assets להתאמת צבעים

מלבד בדיקה ידנית באמצעות `colorScheme`, אפשר גם להשתמש בצבעי משאבים של `Assets` כדי להתאים למצב בהיר ולמצב כהה.

הדרך הזו מתאימה יותר לניהול צבעים כלליים באפליקציה.

בפרויקט Xcode, מוצאים את תיקיית המשאבים `Assets`.

![assets](../../../Resource/025_color.png)

פותחים את תיקיית `Assets`, לוחצים לחיצה ימנית באזור ריק, ובוחרים `New Color Set` כדי ליצור משאב צבע חדש.

![assets](../../../Resource/025_color1.png)

כאן אנחנו יוצרים צבע וקוראים לו `redText`.

![assets](../../../Resource/025_color2.png)

משאב צבע יכול להגדיר בנפרד צבעים למצב בהיר ולמצב כהה.

![assets](../../../Resource/025_color3.png)

לאחר שבוחרים את אזור הצבע המתאים, אפשר לשנות את הצבע במפקח שבצד ימין.

לוחצים על הכפתור `Hide or show the Inspectors` בפינה הימנית העליונה כדי לפתוח את אזור המפקחים.

לאחר מכן בוחרים `Show the Attributes inspector`, מוצאים בתחתית את `Show Color Panel`, ופותחים את לוח הצבעים.

כאן נגדיר את `redText` כך:

## ההבדל בין colorScheme לבין Assets

גם `colorScheme` וגם `Assets` יכולים לממש התאמת מראה, אבל הם מתאימים לתרחישים שונים.

אם צריך רק להחליף תוכן טקסט, שמות תמונות או שמות `SF Symbols` לפי מצב בהיר ומצב כהה, אפשר להשתמש ב-`colorScheme`.

לדוגמה:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

אם מדובר בצבע שמשתמשים בו לעיתים קרובות באפליקציה, למשל צבע נושא, צבע רקע של כרטיס או צבע טקסט, מומלץ יותר להשתמש בצבעי `Assets`.

הסיבה היא שאפשר לעשות שימוש חוזר בצבעי `Assets` במספר תצוגות, וגם קל יותר לשנות אותם בהמשך.

לדוגמה:

```swift
Color("redText")
```

כך צריך לשנות את הצבע רק בתוך `Assets`, וכל המקומות שמשתמשים בצבע הזה ישתנו יחד.

## סיכום

בשיעור הזה למדנו על התאמת מראה למצב בהיר ולמצב כהה.

ראשית, למדנו איך להציג ב-`Canvas` של Xcode תצוגה מקדימה של מראות שונים, וגם איך להשתמש ב-`Color Scheme Variants` כדי לראות מצב בהיר ומצב כהה באותו זמן.

לאחר מכן למדנו שתצוגות SwiftUI כמו `Text` ו-`Image` מתאימות כברירת מחדל את הצבעים שלהן לפי מראה המערכת.

אבל אם נגדיר ידנית צבעים קבועים כמו `Color.white` או `Color.black`, הצבעים האלה לא יתחלפו אוטומטית.

בהמשך למדנו את `@Environment` ואת `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

באמצעות `colorScheme` אפשר לבדוק אם המצב הנוכחי הוא מצב בהיר או מצב כהה, ולהציג אייקונים, טקסט או צבעים שונים לפי מראות שונים.

לבסוף למדנו על צבעי `Assets`.

צבעי `Assets` יכולים להגדיר בנפרד צבעים למצב בהיר ולמצב כהה, ולכן הם מתאימים לניהול משאבי צבע כלליים באפליקציה.

התאמת מראה יכולה לגרום לאפליקציה להיראות טוב יותר גם ביום וגם בלילה, וגם למנוע בעיות כמו טקסט שקשה לקרוא או רקע שאינו ברור.

בפיתוח בפועל, מומלץ להעדיף את יכולת ההתאמה המובנית של SwiftUI ואת צבעי `Assets`.

כאשר צריך להחליף טקסט, אייקונים, תמונות או לטפל במספר קטן של אפקטי תצוגה מיוחדים תחת מראות שונים, אז משתמשים ב-`colorScheme` לצורך הבדיקה.

## ידע נוסף: הגדרת גודל גופן

כשקודם למדנו את `font`, השתמשנו בכמה סגנונות גופן מערכתיים:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

מלבד שימוש בסגנונות גופן מערכתיים כמו `.largeTitle` ו-`.callout`, אפשר גם להגדיר ידנית את גודל הגופן.

לדוגמה:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

כאן:

```swift
.font(.system(size: 100))
```

פירושו להגדיר את גודל הגופן ל-`100 pt`.

עבור אייקוני `SF Symbols`, גם `font` משפיע על גודל האייקון.

לכן, אם רוצים שהאייקון יוצג גדול יותר, אפשר להגדיר את הגודל באמצעות `.font(.system(size:))`.

לדוגמה:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

חשוב לשים לב: אף שהגדרה ידנית של גודל גופן גמישה יותר, לא מומלץ להשתמש בהרבה גדלי גופן קבועים עבור טקסט רגיל.

עבור טקסט רגיל, עדיף להשתמש בסגנונות גופן מערכתיים כמו `.title`, `.headline`, `.body` ו-`.caption`.

כך קל יותר להתאים למכשירים שונים ולהגדרות הגופן של המשתמש.
