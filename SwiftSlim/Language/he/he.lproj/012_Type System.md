# מערכת הטיפוסים

בשיעורים הקודמים כבר למדנו על משתנים, וידוע לנו שמשתנים יכולים לשמור נתונים מסוגים שונים.

למשל:

```swift
var hello = "Hello, World"
```

כאן `hello` שומר טקסט, ולכן הסוג שלו הוא `String`.

בהמשך נלמד בצורה מסודרת את המושג “טיפוס” (`Type`), וגם נבין הצהרה מפורשת על טיפוס (`explicit type declaration`) והסקת טיפוס (`type inference`), כדי להבין לעומק את המהות של משתנים.

## מהו טיפוס

ב-Swift, לכל ערך יש טיפוס מוגדר וברור. הטיפוס קובע מהו הערך הזה ומה אפשר לעשות איתו.

למשל:

- `42` הוא `Int` (מספר שלם)
- `"Hello"` הוא `String` (מחרוזת)
- `true` הוא `Bool` (ערך בוליאני)

אלה טיפוסים שונים, וגם השימוש בהם שונה.

## טיפוסים בסיסיים נפוצים ב-Swift

בשלב ההתחלתי, הטיפוסים הנפוצים ביותר הם:

- `String`: מחרוזת (טקסט)
- `Int`: מספר שלם
- `Double`: מספר עשרוני (דיוק גבוה יותר)
- `Bool`: ערך בוליאני (`true/false`)

בנוסף יש גם `Float` (מספר עשרוני, אבל עם דיוק נמוך יותר מ-`Double`, ולכן פחות נפוץ), וגם `Character` (תו בודד), למשל `"A"` או `"!"`.

### הצהרת טיפוס (טיפוס מפורש)

ב-Swift אפשר לציין ידנית את הטיפוס בזמן הצהרת המשתנה:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

אחרי שם המשתנה מוסיפים `: Type` כדי להצהיר על הטיפוס שלו.

התחביר של טיפוס מערך הוא `: [Type]`:

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

בקוד הזה, המערך `scores` יכול לשמור רק איברים מסוג `Int`, ואילו `tags` יכול לשמור רק איברים מסוג `String`.

הצהרה מפורשת על טיפוס מבהירה את הכוונה של הקוד, ובחלק מהמקרים גם מונעת טעויות בהסקת טיפוס.

## הסקת טיפוס

בהרבה מקרים אפשר גם לא להצהיר במפורש על הטיפוס:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"    // String
var isLogin = false  // Bool
var scores= [90, 85, 78]	// [Int]
var tags = ["Swift", "iOS"]	// [String]
```

הסיבה לכך היא שהקומפיילר של Swift מסיק אוטומטית את הטיפוס לפי “הערך ההתחלתי”.

## הצהרה מפורשת לעומת הסקת טיפוס

בשיעורים הקודמים לא הצגנו הצהרה מפורשת, כמו למשל:

```swift
var age: Int = 18
```

אלא העדפנו להשתמש בהסקת טיפוס:

```swift
var age = 18
```

בדוגמה הזו, שתי הצורות שקולות, ו-`age` ייקבע כ-`Int` בשתיהן. לעומת זאת, הצורה של הסקת טיפוס קצרה ופשוטה יותר.

הסיבה שבשלב הראשון לא הדגשנו הצהרה מפורשת היא שהסקת טיפוס מפחיתה מידע עודף בקוד, וכך מורידה את העומס על מתחילים.

## למה בכלל צריך טיפוסים

Swift היא שפה חזקה מבחינת טיפוסים (`Strongly Typed Language`).

כלומר: ברגע שהטיפוס של משתנה נקבע, אי אפשר לשנות אותו באופן חופשי.

למשל:

```swift
var age: Int = 18

age = 20	// Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

ה-`20` הראשון הוא מסוג `Int`, ולכן אפשר להציב אותו ב-`age`; ה-`"20"` השני הוא מסוג `String`, והוא לא תואם ל-`Int`, ולכן מתקבלת שגיאה.

המערך `nums` הוא מטיפוס `[Int]`, ולכן הוא יכול לשמור רק מספרים שלמים, ולא מחרוזות.

טיפוסים מגבילים את אופן השימוש בנתונים, ומאפשרים לגלות חוסר התאמה כבר בשלב הקומפילציה. כך מצטמצמים באגים שנובעים מערבוב לא נכון של נתונים, והקוד נהיה יציב וקל יותר לתחזוקה.

## מתי חייבים להצהיר במפורש על טיפוס

למרות ש-Swift יכולה להסיק טיפוסים אוטומטית, יש מקרים שבהם הקומפיילר לא מצליח להסיק אותם, או אפילו מסיק אותם בצורה שגויה. במקרים כאלה חייבים לציין את הטיפוס ידנית.

**1. מערך ריק**

כאשר יוצרים מערך ריק, חייבים להצהיר על הטיפוס במפורש:

```swift
var nums: [Int] = []
```

אם לא נכתוב טיפוס:

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

הקומפיילר לא יכול להבין מתוך מערך ריק מהו טיפוס האיברים שבתוכו, ולכן מתקבלת שגיאת קומפילציה.

**2. הימנעות מהסקת טיפוס שגויה**

```swift
var value = 10   // Int
```

בדוגמה הזאת, למרות ש-`10` יכול לייצג גם מספר עשרוני (`Double`), מאחר שאין בו נקודה עשרונית הקומפיילר יסיק שהוא מסוג `Int`.

אם רוצים ש-`value` יהיה מספר עשרוני, צריך להצהיר על הטיפוס במפורש:

```swift
var value: Double = 10   // Double
```

כאשר תוצאת הסקת הטיפוס אינה תואמת לציפייה, צריך להצהיר על הטיפוס במפורש או לשנות את צורת הכתיבה של הערך, כדי להבטיח שהטיפוס יהיה נכון.

**3. טיפוסים מותאמים אישית**

בהמשך נלמד גם טיפוסים מותאמים אישית, וכאשר משתמשים בהם, לעיתים קרובות גם צריך להצהיר במפורש:

```swift
var info: GameInfo
var users: [User] = []
```

הסיבה דומה לזו של מערך ריק: הקומפיילר לא יכול להסיק את הטיפוס מערך ריק או מערך חסר מידע.

## המרת טיפוסים

ב-Swift אין המרה אוטומטית בין טיפוסים שונים, אלא צריך להמיר אותם ידנית.

למשל, חיבור של שני מספרים מטיפוסים שונים:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b	// Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

בקוד הזה, `a` הוא מסוג `Int` ו-`b` הוא מסוג `Double`, ולכן אי אפשר לבצע עליהם חיבור ישיר.

לפני החישוב צריך להמיר אחד מהם לאותו טיפוס:

```swift
var result = Double(a) + b
```

המשמעות היא שלוקחים את `a` מסוג `Int`, ממירים אותו ל-`Double`, ואז מחברים אותו ל-`b`, והתוצאה נשמרת ב-`result`.

חשוב להבין שהמרת טיפוס לא משנה את הטיפוס של המשתנה המקורי. למשל, `Double(a)` לא משנה את `a` עצמו ל-`Double`, אלא יוצר ערך חדש מסוג `Double` לצורך החישוב.

אפשר להמיר גם טיפוסים אחרים באותה צורה, כלומר באמצעות שם הטיפוס הרצוי ואחריו `()`, שבתוכו מעבירים את הערך שרוצים להמיר. למשל:

```
Int()、Double()、String()
```

דוגמאות:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

חשוב לדעת שלא כל המרה תמיד “שומרת” על הערך בצורה מלאה. למשל, בהמרה מ-`Double` ל-`Int`, החלק העשרוני פשוט נחתך, בלי עיגול.

## כינויי טיפוסים

ב-Swift אפשר גם לתת לטיפוס קיים שם נוסף, כדי להפוך את המשמעות של הקוד לברורה יותר:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

כאן `UserID` עדיין מבוסס על `Int`, והטיפוס עצמו לא השתנה. רק השם נעשה בעל משמעות ברורה יותר.

בפרויקטים גדולים או מורכבים, כינויי טיפוסים יכולים לשפר מאוד את הקריאות של הקוד.

## דוגמה מעשית לטיפוסים

כעת נראה דוגמה פשוטה שממחישה את תפקיד הטיפוסים בפיתוח אמיתי.

### יצירת משתנה

תחילה ניצור משתנה בשם `age`:

```swift
var age = 20
```

מכיוון שצריך להציג את המשתנה הזה ב-SwiftUI וגם לשנות אותו, נכריז עליו באמצעות `@State`:

```swift
@State private var age = 20
```

כבר הזכרנו קודם את `private`, ומשמעותו שהמשתנה הזה זמין רק בתוך התצוגה הנוכחית. בהמשך נלמד עליו יותר.

### הצגת נתונים ב-SwiftUI

נציג את המשתנה `age` בתוך SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

כעת התצוגה תציג:

```swift
age: 20
```

### שינוי הנתון

כעת נוסיף שני כפתורים שישנו את `age`:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### תוצאת ההרצה

כאשר לוחצים על `changeAge1`, הערך של `age` יתעדכן ל-`18`:

```swift
age = 18
```

מכיוון ש-`18` הוא מטיפוס `Int`, בדיוק כמו `age`, ההשמה תצליח.

לעומת זאת, `changeAge2` יגרום לשגיאה:

```swift
age = "18"
```

הודעת השגיאה:

```
Cannot assign value of type 'String' to type 'Int'
```

הסיבה לכך היא ש-`"18"` הוא מטיפוס `String` (מחרוזת תמיד נכתבת בתוך `""`), ואילו `age` הוא מטיפוס `Int`. מאחר שהטיפוסים לא תואמים, אי אפשר לבצע את ההשמה.

### למה מתקבלת השגיאה?

המפתח הוא השורה הזאת:

```swift
@State private var age = 20   // Int
```

למרות שלא הצהרנו במפורש על הטיפוס, מכיוון שהערך ההתחלתי `20` הוא מטיפוס `Int`, הקומפיילר מסיק אוטומטית ש-`age` הוא `Int`.

ברגע שהטיפוס נקבע, אי אפשר להציב בו ערך מטיפוס אחר.

וחשוב עוד יותר: הכפתור `changeAge2` לא “נכשל בזמן ריצה”, אלא גורם לשגיאה כבר בזמן הקומפילציה, ולכן הקוד כלל לא יעבור קומפילציה.

זו גם אחת היתרונות הגדולים של מערכת הטיפוסים של Swift: אפשר לגלות בעיות טיפוסים כבר בזמן כתיבת הקוד.

### הכתיבה הנכונה

אם מוחקים את הקוד השגוי:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

כאשר לוחצים על `changeAge1`, התצוגה תתעדכן ל:

```
age: 18
```

### הצהרת טיפוס מפורשת (אופציונלית)

אפשר גם לציין את הטיפוס במפורש בזמן ההגדרה:

```swift
@State private var age: Int = 20
```

הכתיבה הזו הופכת את הטיפוס לברור יותר, אבל בתרחישים פשוטים היא לא חובה.

## סיכום

השיעור הזה עסק בעיקר במערכת הטיפוסים של Swift ובתפקידה בפיתוח אמיתי.

ב-Swift, לכל ערך יש טיפוס ברור, והטיפוס קובע מהי המשמעות של הנתון, ואילו פעולות אפשר לבצע עליו.

Swift היא שפה חזקה מבחינת טיפוסים: ברגע שטיפוס של משתנה נקבע, אי אפשר להציב בו ערך מסוג אחר. ההגבלה הזו מאפשרת לגלות שגיאות כבר בשלב הקומפילציה, לצמצם בעיות בזמן ריצה ולהעלות את הבטיחות של הקוד.

אפשר להסיק את טיפוס המשתנה לפי הערך ההתחלתי, ואפשר גם להצהיר עליו במפורש. במקרים מסוימים (למשל מערך ריק, טעות בהסקת טיפוס או טיפוס מותאם אישית), הקומפיילר לא מצליח להסיק נכון את הטיפוס, ואז חייבים להצהיר עליו במפורש.

אין המרה אוטומטית בין טיפוסים שונים; יש לבצע המרה מפורשת בעזרת `Int()`、`Double()`、`String()` וכדומה.

הטיפוסים גם תומכים בכינויים, ובפרויקטים מורכבים זה יכול לעזור לשפר את הקריאות של הקוד.

טיפוס הוא לא רק תווית לנתון, אלא גם מנגנון מגביל, שמבטיח שהשימוש בנתונים יהיה בטוח, ברור וניתן לשליטה.

## הרחבה - טיפוסים אופציונליים

כאשר מבצעים המרת טיפוסים, יש פעולות שעלולות להיכשל.

למשל, המרה של `String` ל-`Double`:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

כאן `Double(input)` לא מחזיר `Double`, אלא:

```
Double?
```

זהו טיפוס אופציונלי (`Optional`).

### למה מוחזר טיפוס אופציונלי?

משום שהתוכן של מחרוזת אינו ודאי. אפשר להמיר `"3.14"` ל-`Double`, אבל `"Hello"` אי אפשר להמיר ל-`Double`.

כלומר, הפעולה הזו יכולה להצליח, או להיכשל.

לכן Swift משתמשת בטיפוס אופציונלי כדי לייצג את אי-הוודאות הזאת.

### מהו טיפוס אופציונלי?

טיפוס אופציונלי אומר: ייתכן שיש ערך, וייתכן שאין.

למשל:

```
Double?
```

הטיפוס הזה אומר שאם יש ערך, בעת הדפסה הוא עשוי להופיע בצורה `Optional(...)`; ואם אין ערך, תוחזר התוצאה `nil`.

חשוב להבין ש-`nil` פירושו “אין ערך”, ולא `0` או מחרוזת ריקה.

### דוגמה

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

פלט:

```
pi:Optional(3.14)
```

זה אומר שהמחרוזת הומרה בהצלחה ל-`Double`, והתקבל הערך `3.14`, אבל בגלל שהטיפוס הוא `Double?`, הוא עטוף בתוך `Optional`.

אם ההמרה תיכשל:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

פלט:

```
pi: nil
```

מכיוון שאי אפשר להמיר את `"Hello"` ל-`Double`, ההמרה נכשלת ומוחזר `nil`.

### מהו `Optional(...)`?

כאשר מדפיסים ישירות טיפוס אופציונלי:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift מציג את “תיאור הדיבוג” שלו:

```
Optional(3.14)
```

זה אינו חלק מהנתון עצמו, אלא רק סימן לכך שהערך הנוכחי הוא טיפוס אופציונלי, ושבתוכו אכן קיים ערך.

### חילוץ ערך מתוך טיפוס אופציונלי

בפיתוח אמיתי, לעיתים קרובות נצטרך להוציא את הערך האמיתי מתוך טיפוס אופציונלי. התהליך הזה נקרא `Unwrapping`.

אחת הדרכים הנפוצות לעשות זאת היא להשתמש ב-`??` כדי לספק ערך ברירת מחדל:

```
??
```

זה נקרא אופרטור איחוד ערך ריק (`Nil-Coalescing Operator`).

למשל:

```swift
name ?? "FangJunyu"
```

המשמעות של הקוד הזה היא: אם ל-`name` יש ערך, מחזירים אותו; ואם הוא `nil`, משתמשים בערך ברירת המחדל `"FangJunyu"`.

חשוב לשים לב שערך ברירת המחדל חייב להיות מאותו טיפוס כמו הערך שבתוך הטיפוס האופציונלי.

### שימוש ב-`??` כדי לספק ערך ברירת מחדל

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")	// 30
```

פלט:

```
pi: 30
```

כאשר `input` מומר בהצלחה ל-`Double`, יודפס הערך שהתקבל. אם ההמרה הייתה נכשלת, היה מודפס ערך ברירת המחדל שסופק אחרי `??`.

אם ההמרה נכשלת:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

פלט:

```
pi: 10
```

כאן `input` לא הצליח לעבור המרה ל-`Double`, ולכן `pi` הוא `nil`, ו-`??` מספק את ערך ברירת המחדל.

טיפוסים אופציונליים משמשים לייצוג מצב שבו “אולי יש ערך, ואולי לא”.

בפעולות שעלולות להיכשל, כמו המרת טיפוס, Swift מחזירה טיפוס אופציונלי כדי לשמור על בטיחות התוכנית.

כאשר צריך להשתמש בערך, אפשר להיעזר ב-`??` כדי לספק ערך ברירת מחדל, וכך גם אם מופיע `nil`, עדיין אפשר לקבל תוצאה סבירה.
