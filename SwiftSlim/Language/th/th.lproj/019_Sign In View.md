# มุมมองการลงชื่อเข้าใช้

ในบทเรียนนี้ เราจะเรียนรู้สถานการณ์ที่ใช้งานได้จริงมากอย่างหนึ่ง นั่นคือมุมมองการลงชื่อเข้าใช้

ไม่ว่าจะเป็นเว็บไซต์หรือแอป ในระหว่างการใช้งานหลายครั้งผู้ใช้จำเป็นต้องลงชื่อเข้าใช้ด้วยบัญชีและรหัสผ่าน

ตัวอย่างเช่น หน้าลงชื่อเข้าใช้ของ GitHub:

![WordPress](../../Resource/019_github.png)

ในบทเรียนนี้ เราจะสร้างมุมมองการลงชื่อเข้าใช้ที่คล้ายกัน เพื่อให้ผู้ใช้สามารถป้อนบัญชีและรหัสผ่านด้วยตนเอง และตรวจสอบได้ว่ามีปัญหาอะไรกับข้อมูลที่ป้อนหรือไม่

## เค้าโครงส่วนบน

เราจะเขียนโค้ดของมุมมองการลงชื่อเข้าใช้ในไฟล์ `ContentView`

ก่อนอื่น มาสร้างพื้นที่แสดงสัญลักษณ์ด้านบนของมุมมองการลงชื่อเข้าใช้กันก่อน คุณสามารถเตรียมรูปไอคอนที่เหมาะสมไว้ล่วงหน้า แล้วนำไปใส่ในโฟลเดอร์ `Assets`

![icon](../../Resource/019_icon1.png)

จากนั้น ใช้ `Image` และ modifier เพื่อแสดงรูปภาพ:

```swift
Image("019_icon")
    .resizable()
    .scaledToFit()
    .frame(width: 100)
```

modifier เหล่านี้มีความหมายดังนี้:

- `resizable()`: ทำให้รูปภาพสามารถปรับขนาดได้
- `scaledToFit()`: ปรับขนาดรูปภาพโดยคงสัดส่วนเดิมไว้
- `frame(width: 100)`: กำหนดความกว้างที่แสดงของรูปภาพเป็น `100`

ถัดไป ใช้ `Text` และ modifier เพื่อแสดงหัวข้อการลงชื่อเข้าใช้:

```swift
Text("Sign in to App")
    .font(.title3)
    .fontWeight(.bold)
```

จากนั้น ใช้ `VStack` เพื่อจัดเรียงรูปภาพและข้อความในแนวตั้ง:

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
}
```

ผลลัพธ์ที่ได้:

![view](../../Resource/019_view.png)

ถึงตรงนี้ เราก็ทำส่วนหัวด้านบนของมุมมองการลงชื่อเข้าใช้เสร็จแล้ว

### ปรับปรุงเค้าโครง

ตอนนี้รูปภาพและหัวข้อจะอยู่ในบริเวณกึ่งกลางของเค้าโครงทั้งหมดโดยค่าเริ่มต้น

หากเราต้องการให้มันดูคล้ายกับ “สัญลักษณ์การลงชื่อเข้าใช้ที่ด้านบนของหน้า” มากขึ้น ก็สามารถใช้ `Spacer()` เพื่อดันพื้นที่ที่เหลือออกไป ทำให้เนื้อหาแสดงใกล้กับด้านบนมากขึ้น

```swift
VStack {
    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("Sign in to App")
        .font(.title3)
        .fontWeight(.bold)
    Spacer()
}
```

ผลลัพธ์ที่ได้:

![view](../../Resource/019_view1.png)

แต่ตอนนี้ `Image` และ `Text` กลับอยู่ใกล้กับขอบบนของหน้าจอมากเกินไป และดูค่อนข้างอึดอัด

ในเวลานี้ เราสามารถใช้ `padding` เพื่อเพิ่มระยะห่างด้านบนให้กับ `VStack` ทั้งหมดได้

```swift

    Image("019_icon")
        .resizable()
        .scaledToFit()
        .frame(width: 60)
    Text("Sign in to App")
        .font(.title)
        .fontWeight(.bold)
    Spacer()
}
.padding(.top, 50)
```

![view](../../Resource/019_view2.png)

ด้วยวิธีนี้ พื้นที่หัวข้อด้านบนของหน้าจะดูเหมาะสมยิ่งขึ้น

## ชื่อผู้ใช้และรหัสผ่าน

หน้าเข้าสู่ระบบมักต้องให้ผู้ใช้ป้อนชื่อผู้ใช้และรหัสผ่าน

ใน SwiftUI เราสามารถใช้ `TextField` เพื่อรับข้อมูลที่ผู้ใช้ป้อนได้

แต่ต้องสังเกตว่า: `TextField` เองจะไม่เก็บข้อมูลที่ป้อนไว้ในระยะยาว มันเป็นเพียงตัวควบคุมการป้อนข้อมูลเท่านั้น สิ่งที่เก็บข้อมูลเหล่านี้จริง ๆ คือ ตัวแปรที่เรา bind ให้กับมัน

ดังนั้น ก่อนอื่นเราจึงต้องสร้างตัวแปร `@State` สองตัวเพื่อเก็บชื่อผู้ใช้และรหัสผ่าน:

```swift
@State private var user = ""
@State private var password = ""
```

เมื่อค่าของตัวแปร `@State` เปลี่ยน SwiftUI จะรีเฟรชมุมมองที่เกี่ยวข้องโดยอัตโนมัติ

จากนั้น ใช้ `TextField` เพื่อ bind ตัวแปรทั้งสองนี้:

```swift
TextField("input user", text: $user)
TextField("input password", text: $password)
```

ที่นี่ `$user` และ `$password` หมายถึง “binding”

กล่าวคือ เมื่อผู้ใช้พิมพ์ข้อความลงในช่องป้อน ตัวแปรก็จะเปลี่ยนตามไปพร้อมกัน และเมื่อค่าตัวแปรเปลี่ยน เนื้อหาที่แสดงในช่องป้อนก็จะเปลี่ยนตามไปพร้อมกันด้วย

ความสัมพันธ์แบบที่ “มุมมองและข้อมูลซิงก์กันไปมา” นี้ก็คือ binding

โปรดสังเกตว่า ที่นี่เราต้องใช้รูปแบบที่มี `$`:

```swift
$user
```

เพราะ `TextField` ไม่ต้องการสตริงธรรมดา แต่ต้องการค่า binding ที่ “สามารถแก้ไขข้อมูลได้สองทาง”

### แสดงช่องป้อนข้อมูล

ให้นำมันไปไว้ใน `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            TextField("input user", text: $user)
            TextField("input password", text: $password)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

ผลการแสดงผล:

![view](../../Resource/019_view3.png)

เนื่องจากตอนนี้ `user` และ `password` มีค่าเริ่มต้นเป็นสตริงว่าง:

```swift
""
```

ดังนั้น ช่องป้อนข้อมูลจะแสดงข้อความ placeholder ก่อน เช่น:

```swift
input user
```

ข้อความนี้มีไว้เพียงเพื่อบอกผู้ใช้ว่า “ควรป้อนอะไรที่นี่” ไม่ใช่เนื้อหาที่ป้อนจริง

### ปรับปรุงช่องป้อนข้อมูล

ตอนนี้ช่องป้อนข้อมูลใช้งานได้แล้ว แต่รูปแบบเริ่มต้นยังค่อนข้างเรียบง่าย

เพื่อให้หน้าจอชัดเจนขึ้น เราสามารถเพิ่มหัวข้อไว้ด้านหน้าช่องป้อนข้อมูล และปรับแต่งรูปแบบของช่องป้อนเล็กน้อย

ตัวอย่างเช่น เพิ่มหัวข้อก่อน:

```swift
Text("Username")
    .fontWeight(.bold)
```

จากนั้น ใช้ `HStack` เพื่อวางหัวข้อและช่องป้อนข้อมูลไว้ในบรรทัดเดียวกัน:

```swift
HStack {
    Text("Username")
        .fontWeight(.bold)
    TextField("input user", text: $user)
}
```

ที่นี่เราใช้ `HStack` เพราะเราต้องการ “แสดงหัวข้อทางซ้าย และช่องป้อนข้อมูลทางขวา”

สิ่งที่ต้องสังเกตคือ `TextField` จะใช้พื้นที่ว่างที่เหลืออยู่โดยค่าเริ่มต้น

![color](../../Resource/019_view6.png)

เพื่อให้ขนาดของช่องป้อนข้อมูลทั้งสองช่องดูสม่ำเสมอกันมากขึ้น เราสามารถใช้ `frame(width:)` เพื่อตั้งค่าความกว้างให้กับมัน ทำให้ช่องป้อนข้อมูลดูเป็นระเบียบมากขึ้น

```swift
TextField("input user", text: $user)
    .frame(width: 200)
```

จากนั้น เพิ่มเส้นขอบเพื่อให้ช่องป้อนข้อมูลเด่นชัดยิ่งขึ้น:

```swift
TextField("input user", text: $user)
    .frame(width: 200)
    .border(Color.gray,width: 1)
```

ตอนนี้ เรามาเพิ่มทั้งชื่อผู้ใช้และรหัสผ่านลงในมุมมอง `ContentView`:

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""
    var body: some View {
        VStack {
            Image("019_icon")
                .resizable()
                .scaledToFit()
                .frame(width: 60)
            Text("Sign in to App")
                .font(.title)
                .fontWeight(.bold)
            HStack {
                Text("Username")
                    .fontWeight(.bold)
                TextField("input user", text: $user)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Spacer()
        }
        .padding(.top, 50)
    }
}
```

ผลการแสดงผล:

![view](../../Resource/019_view4.png)

ถึงตรงนี้ ผู้ใช้ก็สามารถป้อนชื่อผู้ใช้และรหัสผ่านได้แล้ว

## ปุ่มลงชื่อเข้าใช้

ต่อไป เราจะเพิ่มปุ่มลงชื่อเข้าใช้ไว้ด้านล่างของช่องป้อนข้อมูล

```swift
Button("Sign in") {

}
```

จากนั้น ใช้ `buttonStyle` เพื่อเพิ่มรูปแบบระบบที่เด่นชัดขึ้นให้กับปุ่ม:

```swift
Button("Sign in") {

}
.buttonStyle(.borderedProminent)
```

ผลการแสดงผล:

![view](../../Resource/019_view5.png)

เพื่อยืนยันว่าปุ่มสามารถตอบสนองต่อการคลิกได้จริง เราสามารถเขียน `print` ไว้ในปุ่มก่อนเพื่อทดสอบ:

```swift
Button("Sign in") {
    print("click Sign in")
}
.buttonStyle(.borderedProminent)
```

เมื่อคลิกปุ่ม คอนโซลจะแสดงผล:

```swift
click Sign in
```

สิ่งนี้แสดงว่าปุ่มตอบสนองต่อการคลิกได้สำเร็จแล้ว

## ตรรกะการลงชื่อเข้าใช้

ตอนนี้ เราจะเพิ่มตรรกะการตรวจสอบพื้นฐานที่สุดให้กับปุ่มลงชื่อเข้าใช้: **ตรวจสอบว่าผู้ใช้ได้ป้อนชื่อผู้ใช้และรหัสผ่านหรือไม่**

เราต้องการให้รองรับสถานการณ์ต่อไปนี้:

- ถ้าผู้ใช้ไม่ได้ป้อนอะไรเลย ให้แสดง `Empty`
- ถ้าผู้ใช้ป้อนมาเพียงอย่างใดอย่างหนึ่ง ให้แจ้งว่าขาดอะไร
- ถ้าป้อนทั้งชื่อผู้ใช้และรหัสผ่านแล้ว ให้แสดง `Success`

### ไม่ได้ป้อนข้อมูลใด ๆ

เนื่องจาก `user` และ `password` เป็นสตริงว่างโดยค่าเริ่มต้น:

```swift
@State private var user = ""
@State private var password = ""
```

ดังนั้นเมื่อผู้ใช้ไม่ได้ป้อนอะไรเลย ค่าทั้งสองก็ยังคงว่างอยู่

ใน Swift ประเภทอย่าง `String` สามารถใช้ `isEmpty` เพื่อตรวจสอบได้ว่าเนื้อหาว่างหรือไม่

### คุณสมบัติ isEmpty

`isEmpty` มักใช้เพื่อตรวจสอบว่าสตริง อาร์เรย์ หรือเนื้อหาอื่น ๆ ว่างหรือไม่

ตัวอย่างเช่น:

```swift
user = ""
user.isEmpty // true


user = "123"
user.isEmpty // false
```

ถ้าเนื้อหาว่าง `isEmpty` จะคืนค่า `true`; ถ้าเนื้อหาไม่ว่าง `isEmpty` จะคืนค่า `false`

ดังนั้น เราจึงสามารถใช้มันเพื่อตรวจสอบได้ว่าชื่อผู้ใช้และรหัสผ่านมีการป้อนข้อมูลหรือไม่

### ใช้ isEmpty เพื่อตรวจสอบตัวแปร

ถ้าผู้ใช้ไม่ได้ป้อนข้อมูลใด ๆ เลย:

```swift
user.isEmpty // true
password.isEmpty    // true
```

ในกรณีนี้ เราสามารถเขียนเงื่อนไขได้แบบนี้:

```swift
if user.isEmpty && password.isEmpty {
    print("Empty")
}
```

โค้ดนี้หมายความว่า: ถ้า `user` ว่าง และ `password` ก็ว่างด้วย ให้แสดง:

```swift
Empty
```

ที่นี่ `&&` เป็นตัวดำเนินการเชิงตรรกะที่หมายถึง “และ”

นั่นคือ เมื่อเงื่อนไขทางซ้ายเป็นจริง และเงื่อนไขทางขวาก็เป็นจริงด้วย เงื่อนไขทั้งหมดจึงจะเป็นจริง

ดังนั้น โค้ดนี้จะทำงานก็ต่อเมื่อทั้งชื่อผู้ใช้และรหัสผ่านว่างพร้อมกัน

## ผู้ใช้กรอกมาเพียงบางส่วน

ต่อไป มาพิจารณาอีกสถานการณ์หนึ่ง: ผู้ใช้กรอกเพียงชื่อผู้ใช้ หรือกรอกเพียงรหัสผ่าน

ตัวอย่างเช่น:

```swift
@State private var user = "Fang Junyu"
@State private var password = ""
```

ในกรณีนี้ ชื่อผู้ใช้ไม่ว่าง แต่รหัสผ่านว่าง

เราสามารถใช้ `isEmpty` ต่อไปเพื่อตรวจสอบได้:

### กรณีที่ผู้ใช้กรอกบัญชีหรือรหัสผ่าน

ถ้าผู้ใช้กรอกเพียงชื่อผู้ใช้หรือรหัสผ่าน เราจำเป็นต้องแจ้งให้ทราบอย่างเหมาะสมว่ายังขาดอะไรอยู่

ตัวอย่างเช่น ถ้าผู้ใช้กรอกเพียงชื่อผู้ใช้:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

ตรรกะของโค้ดนี้คือ: ถ้าชื่อผู้ใช้ว่าง ให้แจ้งให้กรอกชื่อผู้ใช้ มิฉะนั้น ให้ตรวจสอบว่ารหัสผ่านว่างหรือไม่ ถ้ารหัสผ่านว่าง ให้แจ้งให้กรอกรหัสผ่าน

### ผู้ใช้กรอกข้อมูลครบทั้งหมด

ถ้าชื่อผู้ใช้และรหัสผ่านถูกป้อนครบแล้ว `isEmpty` ของทั้งสองจะคืนค่า `false`

ในกรณีนี้ เราสามารถเขียนได้แบบนี้:

```swift
if !user.isEmpty && !password.isEmpty {
    print("Success")
}
```

ที่นี่ `!` หมายถึง “ไม่” หรือการกลับค่าความจริง

ตัวอย่างเช่น:

```swift
user.isEmpty    // false
!user.isEmpty    // true
```

กล่าวคือ `user.isEmpty` ใช้เพื่อตรวจสอบว่า “ชื่อผู้ใช้ว่างหรือไม่” ส่วน `!user.isEmpty` ใช้เพื่อตรวจสอบว่า “ชื่อผู้ใช้ไม่ว่างหรือไม่”

สำหรับรหัสผ่านก็ใช้ตรรกะเดียวกัน

ดังนั้น ความหมายของโค้ดนี้คือ: ชื่อผู้ใช้ไม่ว่าง และรหัสผ่านก็ไม่ว่างเช่นกัน

เมื่อทั้งสองเงื่อนไขนี้เป็นจริง ระบบจะแสดง:

```swift
Success
```

## ตรรกะการตรวจสอบแบบสมบูรณ์

ตอนนี้ เรามารวมทั้งสามสถานการณ์นี้เข้าด้วยกัน แล้วเขียนลงใน `Button`:

```swift
Button("Sign in") {
    if user.isEmpty && password.isEmpty {
        print("Empty")
    } else if user.isEmpty || password.isEmpty {
        if user.isEmpty {
            print("Please enter your username.")
        } else if password.isEmpty {
            print("Please enter your password.")
        }
    } else {
        print("Success")
    }
}
.buttonStyle(.borderedProminent)
```

ลำดับการทำงานของโค้ดนี้สามารถเข้าใจได้ดังนี้:

ขั้นแรก ให้ตรวจสอบก่อนว่า:

```swift
if user.isEmpty && password.isEmpty
```

ถ้าทั้งชื่อผู้ใช้และรหัสผ่านว่าง ระบบจะแสดง `Empty` ทันที

ถ้าทั้งสองไม่ว่าง ระบบจะตรวจสอบบรรทัดถัดไป:

```swift
else if user.isEmpty || password.isEmpty
```

ที่นี่ `||` หมายถึง “หรือ”

กล่าวคือ ขอเพียงมีเงื่อนไขใดเงื่อนไขหนึ่งเป็นจริง นิพจน์ทั้งหมดก็จะเป็นจริง

ดังนั้น ตราบใดที่ชื่อผู้ใช้ว่าง หรือรหัสผ่านว่าง บล็อกโค้ดของ `else if` ก็จะถูกเรียกใช้งาน

เมื่อเข้าสู่บล็อกโค้ดแล้ว เราจะตรวจสอบต่อไปว่าช่องป้อนข้อมูลใดที่ยังไม่ได้กรอก:

```swift
if user.isEmpty {
    print("Please enter your username.")
} else if password.isEmpty {
    print("Please enter your password.")
}
```

ถ้าเงื่อนไขสองข้อก่อนหน้านี้ไม่เป็นจริง ก็หมายความว่าชื่อผู้ใช้ไม่ว่าง และรหัสผ่านก็ไม่ว่างเช่นกัน

จากนั้นโปรแกรมจะเข้าสู่ `else` สุดท้าย:

```swift
print("Success")
```

ด้วยวิธีนี้ เราก็ทำตรรกะการตรวจสอบข้อมูลเข้าสู่ระบบแบบพื้นฐานที่สุดเสร็จสมบูรณ์

## การตรวจสอบชื่อผู้ใช้และรหัสผ่าน

ตรรกะข้างต้นตรวจสอบเพียงว่า “มีการป้อนข้อมูลหรือไม่”

แต่ใน App จริง การป้อนข้อมูลอย่างเดียวไม่เพียงพอ โดยทั่วไปยังต้องส่งชื่อผู้ใช้และรหัสผ่านไปยังเซิร์ฟเวอร์เพื่อทำการตรวจสอบด้วย

ถ้าเซิร์ฟเวอร์ยืนยันว่ามีชื่อผู้ใช้นี้อยู่จริง และรหัสผ่านถูกต้อง ก็จะอนุญาตให้เข้าสู่ระบบ มิฉะนั้นจะแจ้งว่าการเข้าสู่ระบบล้มเหลว

เพื่อฝึกกระบวนการนี้ เราสามารถกำหนดบัญชีและรหัสผ่านชั่วคราวไว้ในโค้ดก่อน เพื่อจำลอง “ข้อมูลเข้าสู่ระบบที่ถูกต้อง”:

```swift
let userID = "FangJunyu"
let passWord = "123456"
```

เมื่อชื่อผู้ใช้และรหัสผ่านไม่ว่างทั้งคู่แล้ว เราจึงค่อยเปรียบเทียบว่าถูกต้องหรือไม่:

```swift
if user.isEmpty && password.isEmpty {
    ...
} else if user.isEmpty || password.isEmpty {
    ...
} else {
    if user == userID && password == passWord {
        print("Success")
    } else {
        print("Input error")
    }
}
```

ที่นี่ `==` หมายถึง “เท่ากันหรือไม่”

ตัวอย่างเช่น:

```swift
user == userID
```

ซึ่งหมายถึง ตรวจสอบว่าชื่อผู้ใช้ที่ผู้ใช้ป้อนตรงกับชื่อผู้ใช้ที่ถูกต้องที่เรากำหนดไว้ทุกตัวอักษรหรือไม่ และสำหรับรหัสผ่านก็ใช้ตรรกะเดียวกัน

ดังนั้น เมื่อชื่อผู้ใช้และรหัสผ่านที่ป้อนตรงกับค่าที่กำหนดไว้ ระบบจะแสดง `Success`; ถ้ามีค่าใดค่าหนึ่งไม่ตรง ระบบจะแสดง `Input error`

ด้วยวิธีนี้ เราก็ทำ “การตรวจสอบการเข้าสู่ระบบแบบจำลอง” อย่างง่ายเสร็จสมบูรณ์

แม้ว่านี่จะยังไม่ใช่ฟังก์ชันเข้าสู่ระบบจริงที่เชื่อมต่อกับเซิร์ฟเวอร์ แต่ก็เพียงพอแล้วที่จะช่วยให้เราเข้าใจว่า: **หลังจากกดปุ่มแล้ว สามารถเรียกใช้โค้ดที่แตกต่างกันได้ตามเงื่อนไขที่แตกต่างกัน**

## สรุป

ในบทเรียนนี้ เราได้สร้างมุมมองการลงชื่อเข้าใช้พื้นฐานเสร็จแล้ว และได้เรียนรู้โดยเน้นไปที่ “ตรรกะการตัดสินเนื้อหาที่ป้อนตามเงื่อนไข”

เนื้อหาหลักของบทเรียนนี้คือ: การใช้ `isEmpty` เพื่อตรวจสอบว่าสตริงว่างหรือไม่ และการใช้คำสั่ง `if` ร่วมกับตัวดำเนินการเชิงตรรกะเพื่อจัดการกับสถานการณ์ต่าง ๆ

ตัวดำเนินการเชิงตรรกะที่สำคัญมากสองตัวคือ:

- `&&`: หมายถึง “และ” ทั้งสองเงื่อนไขต้องเป็นจริง
- `||`: หมายถึง “หรือ” เพียงเงื่อนไขเดียวเป็นจริงก็เพียงพอ

เมื่อเงื่อนไขเหล่านี้ให้ผลเป็น `true` หรือ `false` ในที่สุด คำสั่ง `if` ก็จะรันโค้ดที่แตกต่างกันไปตามผลลัพธ์

ผ่านมุมมองการลงชื่อเข้าใช้นี้ เราได้เริ่มสัมผัสกับแนวทางการพัฒนาที่ผสาน “อินเทอร์เฟซ + ข้อมูล + การตัดสินเชิงตรรกะ” เข้าด้วยกันแล้ว

```swift
struct ContentView: View {
    @State private var user = ""
    @State private var password = ""

                .resizable()
                .scaledToFit()

                .font(.title)
                .fontWeight(.bold)

                    .fontWeight(.bold)

                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            HStack {
                Text("Password")
                    .fontWeight(.bold)
                TextField("input password", text: $password)
                    .frame(width: 200)
                    .border(Color.gray,width: 1)
            }
            Button("Sign in") {
                if user.isEmpty && password.isEmpty {
                    print("Empty")
                } else if user.isEmpty || password.isEmpty {
                    if user.isEmpty {
                        print("Please enter your username.")
                    } else if password.isEmpty {
                        print("Please enter your password.")
                    }
                } else {
                    if user == userID && password == passWord {
                        print("Success")
                    } else {
                        print("Input error")
                    }
                }
            }
            .buttonStyle(.borderedProminent)
            Spacer()
        }
        .padding(.top, 50)
    }
}
```
