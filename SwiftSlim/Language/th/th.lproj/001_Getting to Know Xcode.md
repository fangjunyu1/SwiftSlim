# ทำความรู้จักกับ Xcode

## Xcode คืออะไร

Xcode คือสภาพแวดล้อมการพัฒนาแบบบูรณาการ (IDE, Integrated Development Environment) ที่ Apple จัดเตรียมอย่างเป็นทางการ สำหรับใช้พัฒนาแอปบนแพลตฟอร์มของ Apple เช่น iOS, macOS, watchOS และ tvOS

![Swift](../../RESOURCE/001_xcode.png)

ตลอดกระบวนการสอนทั้งหมดของคอร์สนี้ เราจะใช้ Xcode ในการเขียนโค้ดและรันโปรแกรม

ในบทนี้ เราจะเรียนรู้วิธีดาวน์โหลด Xcode การสร้างแอป iOS ตัวแรก และเริ่มทำความรู้จักกับโครงสร้างหน้าตาของ Xcode

## ดาวน์โหลด Xcode

ปัจจุบัน Xcode มีวิธีติดตั้งอยู่ 2 แบบ:

1、ดาวน์โหลดผ่าน [Mac App Store](https://apps.apple.com/cn/app/xcode/id497799835?mt=12)

2、ดาวน์โหลดผ่าน [เว็บไซต์ทางการของ Apple Developer](https://developer.apple.com/xcode/)

ทั้งสองวิธีสามารถใช้ดาวน์โหลด Xcode ได้เหมือนกัน ความแตกต่างคือบนเว็บไซต์ทางการของ Apple Developer นอกจากเวอร์ชันทางการแล้ว ยังสามารถดาวน์โหลด [Beta](https://developer.apple.com/download/all/?q=Xcode) หรือเวอร์ชันเก่าได้ด้วย

หากเพียงแค่ใช้เพื่อเรียนการพัฒนา แนะนำให้ใช้เวอร์ชันทางการ เวอร์ชัน Beta มีไว้หลัก ๆ สำหรับทดสอบฟีเจอร์ใหม่ และอาจมีความไม่เสถียรอยู่บ้าง

## สร้างโปรเจกต์แรก

เมื่อเปิด Xcode ขึ้นมา จะเห็นหน้าจอเริ่มต้น

![Swift](../../RESOURCE/001_xcode1.png)

ด้านซ้ายจะมี 3 ตัวเลือกที่ใช้บ่อย:

- Create New Project —— สร้างโปรเจกต์ใหม่  
- Clone Git Repository —— โคลน Git repository  
- Open Existing Project —— เปิดโปรเจกต์ที่มีอยู่แล้ว

ด้านขวาจะแสดงโปรเจกต์ Xcode ที่เพิ่งเปิดล่าสุด

ให้เราเลือกตัวเลือก "Create New Project"

### เลือกเทมเพลตโปรเจกต์

![Swift](../../RESOURCE/001_xcode2.png)

เมื่อเข้าสู่หน้าสำหรับเลือกเทมเพลตโปรเจกต์ จะเห็นหลายแพลตฟอร์ม (เช่น iOS, macOS ฯลฯ) และมีเทมเพลตโปรเจกต์หลายประเภท

คำอธิบายของเทมเพลตที่พบบ่อย:
- App —— ประเภทแอปพื้นฐานที่สุด (แนะนำ)
- Document App —— แอปที่อิงโครงสร้างแบบเอกสาร
- Game —— ใช้สำหรับพัฒนาเกม
- Framework —— โมดูลที่สามารถนำกลับมาใช้ซ้ำได้

ในช่วงเริ่มต้น ให้เราเลือกเฉพาะเทมเพลต "iOS" - "App" เพราะนี่คือประเภทแอปที่มีส่วนติดต่อผู้ใช้

จากนั้นคลิก "Next"

### กรอกข้อมูลโปรเจกต์

เมื่อเข้าสู่หน้ากรอกข้อมูลโปรเจกต์ เราจำเป็นต้องกรอกตามความหมายของแต่ละช่อง

![Swift](../../RESOURCE/001_xcode3.png)

ความหมายของแต่ละช่องมีดังนี้:

- Product Name —— ชื่อโปรเจกต์ เช่น: SwiftSlimTest
- Team —— ทีมผู้พัฒนา หากยังไม่มีบัญชี Apple Developer แบบเสียเงิน สามารถไม่เลือกได้
- Organization Identifier —— ตัวระบุองค์กร โดยปกติมักใช้รูปแบบโดเมนกลับด้าน เช่น:
  - com.yourname
  - com.yourcompany
  - หากไม่มีโดเมน สามารถใช้ชื่อภาษาอังกฤษของตนเองเป็นตัวระบุได้
- Bundle Identifier —— ตัวระบุเฉพาะของแอป ซึ่งจะถูกสร้างให้อัตโนมัติตาม Organization Identifier และ Product Name
- Interface —— เทคโนโลยีของส่วนติดต่อผู้ใช้ ให้เลือก SwiftUI
- Language —— ภาษาในการพัฒนา ให้เลือก Swift
- Testing System —— โดยค่าเริ่มต้นจะสร้างเป้าหมายสำหรับการทดสอบ (Unit Tests) ซึ่งในช่วงเริ่มต้นสามารถข้ามไปก่อนได้
- Storage —— เฟรมเวิร์กสำหรับการจัดเก็บข้อมูลแบบถาวรในเครื่อง ซึ่งสามารถผสาน SwiftData หรือ Core Data ได้อัตโนมัติ ในช่วงเริ่มต้นสามารถข้ามไปก่อนได้

เมื่อกรอกเสร็จแล้ว ให้คลิก "Next"

### บันทึกโปรเจกต์

![Swift](../../RESOURCE/001_xcode4.png)

เลือกโฟลเดอร์ที่เหมาะสมสำหรับบันทึกโปรเจกต์

คลิกปุ่ม "Create" แล้ว Xcode จะสร้างโครงสร้างโปรเจกต์ให้อัตโนมัติ

## โครงสร้างโปรเจกต์ Xcode

หลังจากบันทึกโปรเจกต์ Xcode แล้ว ใน Finder จะสามารถเห็นโฟลเดอร์โปรเจกต์ที่ถูกสร้างขึ้น

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets —— โฟลเดอร์สำหรับเก็บรูปภาพ ไอคอนแอป และทรัพยากรอื่น ๆ

- ContentView.swift —— ไฟล์มุมมอง SwiftUI ที่ระบบสร้างมาให้โดยค่าเริ่มต้น หลังจากนี้เราจะเขียนโค้ดส่วนติดต่อผู้ใช้ในไฟล์นี้

- SwiftSlimTestApp.swift —— ไฟล์ทางเข้าแอป (App Entry Point) มีหน้าที่เริ่มการทำงานของแอป ซึ่งในตอนนี้เรายังไม่จำเป็นต้องทำความเข้าใจมัน

- SwiftSlimTest.xcodeproj —— ไฟล์โปรเจกต์ของ Xcode ซึ่งสามารถดับเบิลคลิกเพื่อเปิดโปรเจกต์ได้

## ทำความรู้จักกับหน้าตา Xcode

ดับเบิลคลิกไฟล์ .xcodeproj เพื่อเปิดโปรเจกต์ จากนั้นที่แถบ Navigator ทางซ้าย ให้เลือกไฟล์ ContentView.swift

หน้าตาของ Xcode โดยหลักแล้วแบ่งออกเป็น 5 พื้นที่:

- Navigator —— พื้นที่นำทาง ใช้ดูโครงสร้างไฟล์และค้นหา
- Editor —— พื้นที่แก้ไข ใช้เขียนโค้ด Swift หรือ SwiftUI
- Canvas —— พื้นที่แคนวาส ใช้สำหรับพรีวิว SwiftUI
- Inspector —— พื้นที่ตรวจสอบคุณสมบัติ ใช้ดูและแก้ไขคุณสมบัติของไฟล์
- Debug Area —— พื้นที่ดีบัก ใช้ดูบันทึกข้อความและผลลัพธ์การทำงาน

![Swift](../../RESOURCE/001_xcode6.png)

ข้อสังเกต: Canvas ใช้หลัก ๆ สำหรับพรีวิวหน้าจอ SwiftUI และหากอิงตามสถาปัตยกรรมอย่างเป็นทางการของ Xcode มันถือเป็นแผงพรีวิวเสริมของ Editor (Preview Pane) จะสามารถเปิดใช้งาน Canvas ได้เฉพาะเมื่อเปิดไฟล์มุมมอง SwiftUI เท่านั้น ไฟล์ประเภทอื่นจะไม่แสดงความสามารถในการพรีวิว

ในบทเรียนถัด ๆ ไป เราจะเขียนโค้ด Swift และ SwiftUI เป็นหลักในพื้นที่ Editor

เมื่อเปิดโปรเจกต์ Xcode ครั้งแรก บางครั้ง Inspector และ Debug Area อาจถูกซ่อนไว้โดยค่าเริ่มต้น สามารถคลิกปุ่มมุมขวาบนและขวาล่างเพื่อแสดงได้

![Swift](../../RESOURCE/001_xcode7.png)

คำแนะนำ: พื้นที่ Inspector ใช้หลัก ๆ สำหรับดูและแก้ไขคุณสมบัติของไฟล์ ในการพัฒนาจริง มักจะซ่อนพื้นที่นี้ไว้เพื่อให้มีพื้นที่สำหรับแก้ไขโค้ดมากขึ้น

## สรุป

ในบทนี้ เราได้เรียนรู้วิธีดาวน์โหลด Xcode การสร้างโปรเจกต์ Xcode แรก และการจัดวางหน้าตาพื้นฐานของ Xcode

ต่อไปนี้เป็นช่วงเวลาสำหรับทดลองด้วยตนเอง:
- คุณสามารถลองสร้างและลบไฟล์ในพื้นที่ Navigator ได้  
- ดูโค้ดในพื้นที่ Editor และสังเกตโครงสร้างของโค้ด
- ดูปุ่มต่าง ๆ ในพื้นที่ Canvas แล้วลองขยายหรือย่อหน้าพรีวิว

ในบทถัดไป เราจะเริ่มเขียนโค้ดง่าย ๆ บางส่วน และใช้มันเพื่อช่วยให้เราเข้าใจ Xcode ได้มากขึ้น
