# โปรแกรมจำลองและอุปกรณ์จริง

จนถึงตอนนี้ เราได้สร้างมุมมองพื้นฐานเสร็จเรียบร้อยแล้ว เนื้อหาหลักของบทนี้คือการเรียนรู้วิธีรันแอปบนโปรแกรมจำลอง (Simulator) และอุปกรณ์จริง (Physical Device)

ผมยังจำได้ว่าครั้งแรกที่ติดตั้งแอปลงบน iPhone ผมรู้สึกดีใจมาก เพราะสามารถเห็นผลงานการพัฒนาของตัวเองได้อย่างเป็นรูปธรรมมากขึ้น นี่เป็นแรงเสริมเชิงบวกอย่างหนึ่ง เมื่อเราได้ใช้งานแอปที่ตัวเองพัฒนาขึ้นจริง ความสนใจก็จะผลักดันให้เราอยากเรียนรู้มากขึ้นโดยอัตโนมัติ

ตอนนี้ โปรดเปิดโปรเจกต์เรซูเม่ส่วนตัวที่ทำเสร็จในบทก่อนหน้า แล้วเตรียมรันบนโปรแกรมจำลองและอุปกรณ์จริง

## โปรแกรมจำลอง

Xcode มีโปรแกรมจำลองหลายแบบในตัว ครอบคลุม iPhone, iPad และ Apple Watch หลายรุ่น โปรแกรมจำลองช่วยให้เราตรวจสอบความแตกต่างของเลย์เอาต์แอปบนอุปกรณ์หรือเวอร์ชันต่าง ๆ ได้

### เลือกโปรแกรมจำลอง

ที่แถบเครื่องมือด้านบนของ Xcode ในรายการอุปกรณ์ ให้เลือกโปรแกรมจำลองหนึ่งตัว (เช่น iPhone 16)

![Swift](../../RESOURCE/005_xcode.png)

### รันโปรแกรมจำลอง

จากนั้นคลิกปุ่มรันที่มุมซ้ายบน

![Swift](../../RESOURCE/005_xcode1.png)

Xcode จะสร้างโปรเจกต์โดยอัตโนมัติและเปิดโปรแกรมจำลองขึ้นมา

![Swift](../../RESOURCE/005_xcode2.png)

### ปิดโปรแกรมจำลอง

หากต้องการปิดโปรแกรมจำลอง ให้คลิกปุ่มหยุดในแถบเครื่องมือของ Xcode

![Swift](../../RESOURCE/005_xcode3.png)

หรือจะปิดตัวโปรแกรมจำลองโดยตรงก็ได้:

![Swift](../../RESOURCE/005_xcode17.png)

ถ้าปิดโปรแกรมจำลองโดยตรง Xcode อาจแสดงข้อความ:

``` id="us0biw"
Thread 1: signal SIGTERM
````

โดยทั่วไปนี่ไม่ใช่ข้อผิดพลาดของตรรกะโปรแกรม แต่เกิดจากกระบวนการดีบักถูกยุติโดยภายนอก จึงมักสามารถมองข้ามได้ และไม่ถือเป็นข้อผิดพลาดของตรรกะแอป

เมื่อเทียบกับ Xcode Preview แล้ว โปรแกรมจำลองใกล้เคียงกับสภาพแวดล้อมการรันจริงมากกว่า แต่ก็ยังไม่ใช่สภาพแวดล้อมของอุปกรณ์จริงทั้งหมด มันเหมาะสำหรับการตรวจสอบเลย์เอาต์และการทดสอบการรองรับหลายขนาดอุปกรณ์

## เรื่องลำดับความสำคัญของสภาพแวดล้อมการทดสอบ

ในกระบวนการพัฒนาจริง ควรยึดลำดับการทดสอบดังนี้:

อุปกรณ์จริง > โปรแกรมจำลอง > Xcode Preview

Preview เหมาะสำหรับการปรับ UI อย่างรวดเร็ว โปรแกรมจำลองเหมาะสำหรับการทดสอบเชิงโครงสร้าง และการยืนยันขั้นสุดท้ายจำเป็นต้องทำบนอุปกรณ์จริง โดยเฉพาะการทำงานที่เกี่ยวข้องกับ iCloud

## อุปกรณ์จริง

ทั้ง Xcode Preview และโปรแกรมจำลองต่างก็อาจมีกรณีที่แสดงผลไม่ตรงกับความเป็นจริง ดังนั้นอุปกรณ์จริงจึงมีความสำคัญเชิงตัดสิน ระบบสิทธิ์ ประสิทธิภาพ และกลไกการแจ้งเตือน เป็นพฤติกรรมที่สามารถตรวจสอบได้ครบถ้วนบนอุปกรณ์จริงเท่านั้น

### เปิดโหมดนักพัฒนา

บน iPhone ให้เข้าไปที่:

การตั้งค่า → ความเป็นส่วนตัวและความปลอดภัย → โหมดนักพัฒนา → เปิดใช้งาน

![Swift](../../RESOURCE/005_xcode4.png)

ระบบจะบังคับให้รีสตาร์ตอุปกรณ์ หากไม่ได้เปิดโหมดนักพัฒนา จะไม่สามารถดีบักและรันแอปได้

### ตั้งค่าทีมนักพัฒนา

Xcode จำเป็นต้องตั้งค่าทีมนักพัฒนา มิฉะนั้นจะไม่สามารถเซ็นชื่อแอปและติดตั้งได้

ขั้นตอนการตั้งค่า:

ชื่อโปรเจกต์ → TARGETS → Signing & Capabilities → Team

หากแสดง "Add Account..." แปลว่ายังไม่ได้เพิ่มบัญชี Apple

![Swift](../../RESOURCE/005_xcode5.png)

คลิกปุ่มเพิ่ม แล้วลงชื่อเข้าใช้ Apple ID

![Swift](../../RESOURCE/005_xcode6.png)

หลังจากลงชื่อเข้าใช้ Apple ID แล้ว ให้เลือกบัญชีที่เกี่ยวข้องเพื่อทำการตั้งค่าทีมนักพัฒนาให้เสร็จสมบูรณ์

![Swift](../../RESOURCE/005_xcode7.png)

### เชื่อมต่ออุปกรณ์

ใช้สายเชื่อมต่อ iPhone กับ Mac

เมื่อเชื่อมต่อครั้งแรก:

* Mac จะถามว่าต้องการเชื่อมต่อ iPhone กับ Mac เครื่องนี้หรือไม่
* iPhone จะถามว่าต้องการเชื่อถือคอมพิวเตอร์เครื่องนี้หรือไม่

จำเป็นต้องกด “เชื่อถือ” บน iPhone และกรอกรหัสปลดล็อกหน้าจอ จากนั้นให้เลือก iPhone ของคุณจากรายการอุปกรณ์ใน Xcode

![Swift](../../RESOURCE/005_xcode9.png)

คลิกปุ่มรันที่มุมซ้ายบน

![Swift](../../RESOURCE/005_xcode1.png)

Xcode จะสร้างโปรเจกต์และติดตั้งแอปลงบน iPhone

![Swift](../../RESOURCE/005_view.png)

**การดีบักแบบไร้สาย**

ตั้งแต่ iOS 11 เป็นต้นมา Xcode รองรับการดีบักแบบไร้สาย

![Swift](../../RESOURCE/005_xcode18.png)

เงื่อนไขเบื้องต้น:

1. การจับคู่ครั้งแรกระหว่าง iPhone กับ Mac ต้องทำผ่านสายข้อมูลก่อน
2. iPhone และ Mac ต้องอยู่ในเครือข่าย Wi-Fi เดียวกัน
3. อุปกรณ์ทั้งสองต้องอยู่ในสถานะปลดล็อก

หลังจากจับคู่ครั้งแรกผ่านสายสำเร็จแล้ว แม้จะถอดสายออก ภายหลังก็ยังสามารถรันได้โดยตรงตราบใดที่ทั้งสองอุปกรณ์อยู่ในเครือข่ายเดียวกัน

ข้อควรระวัง:

* ถ้าอุปกรณ์ไม่ปรากฏในรายการอุปกรณ์ สามารถเสียบสายใหม่เพื่อเชื่อมต่ออีกครั้งได้
* การดีบักผ่าน Wi-Fi อาจไม่เสถียรในบางครั้ง ถ้าเกิดปัญหาการเชื่อมต่อ ให้เสียบสายใหม่ได้เลย
* ในเครือข่ายบริษัทหรือเครือข่ายที่มีการแยกส่วน อาจมีข้อจำกัดในการใช้งาน

## ข้อผิดพลาดที่พบบ่อยและวิธีแก้ไข

เมื่อใช้ Xcode เชื่อมต่อ iPhone เพื่อดีบัก อาจพบข้อความผิดพลาดได้หลายประเภท ด้านล่างนี้คือปัญหาที่พบบ่อยบางส่วนและวิธีจัดการที่สอดคล้องกัน ซึ่งช่วยให้ตรวจสอบและแก้ปัญหาการเชื่อมต่อได้เร็วขึ้น

**1. การจับคู่ยังไม่เสร็จ**

```id="fnto6r"
Respond to the trust dialog on the device if present, and wait until pairing has completed.
```

![Swift](../../RESOURCE/005_xcode10.png)

แปลว่า Xcode และ iPhone ยังจับคู่สำหรับการดีบักไม่เสร็จสมบูรณ์

วิธีแก้: ตรวจสอบว่าได้กด "เชื่อถือ" บน iPhone แล้ว ถ้ายังมีข้อผิดพลาดอยู่ ให้รีสตาร์ต Xcode แล้วเชื่อมต่อใหม่

**2. ยังไม่ได้ตั้งค่าทีมนักพัฒนา**

```id="dd7ah5"
Signing for "SwiftSlimTest" requires a development team.
```

อาจเป็นเพราะยังไม่ได้ตั้งค่าทีมนักพัฒนา

วิธีแก้: เลือกบัญชีที่ลงชื่อเข้าใช้ไว้แล้วใน Signing & Capabilities

**3. ไม่สามารถ mount Developer Disk Image ได้**

```id="e0x4hm"
Previous preparation error: The developer disk image could not be mounted on this device.
```

มักเกิดขึ้นเมื่อกระบวนการติดตั้งถูกขัดจังหวะ (เช่น ถอดสายหรือหยุดการทำงานแบบบังคับ) ทำให้ตอนติดตั้งใหม่มีข้อจำกัดและเกิดข้อผิดพลาดนี้

วิธีแก้: รีสตาร์ต iPhone แล้วเสียบสายเชื่อมต่อใหม่

**4. การเชื่อมต่อ tunnel สำหรับดีบักล้มเหลว**

```id="hzks5r"
The tunnel connection failed while the system tried to connect to the device.
```

![Swift](../../RESOURCE/005_xcode11.png)

ข้อผิดพลาดนี้มักพบเมื่อมี VPN หรือ proxy เครือข่ายเข้ามารบกวน

วิธีแก้: ปิด VPN หรือ proxy เครือข่าย แล้วลองใหม่อีกครั้ง

**5. อุปกรณ์ยังไม่ถูกปลดล็อก**

```id="ai93ri"
Development services need to be enabled. Ensure that the device is unlocked.
```

![Swift](../../RESOURCE/005_xcode12.png)

อุปกรณ์อยู่ในสถานะล็อกหน้าจอ เพียงปลดล็อก iPhone ก็ได้

**6. อุปกรณ์ยังไม่ได้ลงทะเบียน**

```id="3497j2"
Device "iPhone" isn't registered in your developer account.
```

![Swift](../../RESOURCE/005_xcode13.png)

แปลว่าอุปกรณ์นี้ยังไม่ได้ถูกเพิ่มเข้าไปในรายการอุปกรณ์ของบัญชีนักพัฒนา ทำให้ Xcode ไม่สามารถสร้าง provisioning profile ที่ถูกต้องเพื่อเซ็นชื่อและติดตั้งแอปให้มันได้

วิธีแก้: คลิก "Register Device" เพื่อให้ระบบลงทะเบียนให้อัตโนมัติ

**7. เวอร์ชัน deployment ไม่ตรงกัน**

```id="c44g6i"
iOS 17.6.1 doesn’t match StoreKit2.app’s iOS 18.0 deployment target
```

![Swift](../../RESOURCE/005_xcode14.png)

หมายความว่าเวอร์ชันระบบของอุปกรณ์ต่ำกว่า deployment target ต่ำสุดของโปรเจกต์

ตำแหน่งที่ใช้แก้ไข:

TARGETS → General → Deployment Info → Deployment Target

![Swift](../../RESOURCE/005_xcode15.png)

ปรับเวอร์ชันให้ไม่สูงกว่าเวอร์ชันระบบของอุปกรณ์

หลังจากแก้ไขแล้ว ให้ล้างโฟลเดอร์ build:

Product → Clean Build Folder Immediately

![Swift](../../RESOURCE/005_xcode16.png)

เมื่อล้างเสร็จแล้ว ให้รันใหม่อีกครั้ง
