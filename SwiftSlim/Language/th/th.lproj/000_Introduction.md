# บทนำ

## Swift คืออะไร?

ก่อนจะเริ่มเรียน Swift เรามาทำความรู้จักกับภาษาโปรแกรม Swift แบบคร่าว ๆ กันก่อน Swift คือภาษาโปรแกรมสมัยใหม่ที่พัฒนาโดย Apple โดยหลักแล้วใช้สำหรับสร้างแอปบนแพลตฟอร์มของ Apple เช่น iOS, macOS, iPadOS และ watchOS

![Swift](../../RESOURCE/000_swift.png)

เมื่อเทียบกับภาษาโปรแกรมอื่น ๆ แล้ว Swift มีไวยากรณ์ที่กระชับกว่า และให้ความสำคัญกับความปลอดภัยและความสามารถในการอ่านมากกว่า เมื่อใช้งานร่วมกับ SwiftUI ก็จะยิ่งเรียนรู้และทำความเข้าใจได้ง่ายขึ้น แน่นอนว่าส่วนตัวผมเองก่อนหน้านี้ก็ไม่ได้เคยเรียนภาษาโปรแกรมอื่นมาก่อน จึงรู้สึกว่ามันอาจยากกว่า HTML อยู่เล็กน้อย

Swift ไม่ได้ใช้พัฒนาเฉพาะแอปบนแพลตฟอร์มของ Apple เท่านั้น ชุมชน Swift ก็กำลังพยายามย้าย Swift ไปยัง [Android](https://www.swift.org/blog/nightly-swift-sdk-for-android/) และแม้กระทั่งให้ทำงานบน[เซิร์ฟเวอร์](https://github.com/awslabs/swift-aws-lambda-runtime)หรือสภาพแวดล้อมอื่น ๆ ได้ จากมุมมองด้านการเติบโตในระยะยาว Swift ยังมีพื้นที่ให้พัฒนาได้อีกมาก และในอนาคตก็มีโอกาสที่จะกลายเป็นภาษาแบบข้ามแพลตฟอร์มคล้ายกับ Flutter หรือ React Native

## เนื้อหาการเรียน

บทเรียนชุดนี้จะเน้นพื้นฐานของ Swift และ SwiftUI เป็นหลัก

เราจะเริ่มจากเรื่องง่าย ๆ อย่างตัวแปรและชนิดข้อมูล ใช้ SwiftUI สร้างหน้าจอ ทำความเข้าใจวิธีเก็บและอ่านข้อมูล และในภายหลังก็อาจลงลึกไปถึงเนื้อหาขั้นสูงอย่าง SwiftData, การซิงก์ด้วย iCloud เป็นต้น

แต่ละบทจะดำเนินไปโดยอิงจากกรณีตัวอย่างจริง เพื่อให้เรียนรู้ความรู้ของ Swift ผ่านการใช้งานจริง

บางบทเรียนอาจเริ่มจากการอธิบายไวยากรณ์และ API จำนวนมากก่อน แล้วค่อยเข้าสู่การทำโปรเจกต์จริง สำหรับผู้เริ่มต้นแล้ว วิธีนี้ก็เหมือนกับให้เด็กท่องจำคำศัพท์จากพจนานุกรมก่อน โดยไม่มีบริบทในการใช้งาน สุดท้ายก็มีแต่จะทำให้ผู้เริ่มต้นหมดความสนใจ

บทเรียนชุดนี้จะพาคุณเรียนรู้ความรู้ผ่านสถานการณ์การพัฒนาจริง โดยใช้ความต้องการจริงเป็นตัวนำไปสู่ประเด็นความรู้ที่เกี่ยวข้อง

เมื่อเรียนจบบทเรียนชุดนี้แล้ว คุณจะสามารถพัฒนาและรันแอปพื้นฐานได้ด้วยตัวเอง ติดตั้งลงบนอุปกรณ์ iOS / macOS ของตนเองได้ และยังสามารถนำแอปขึ้นเผยแพร่บน App Store ได้ด้วย

## ผู้เริ่มต้นจากศูนย์ควรเรียนอย่างไร?

ตอนเริ่มต้น ผมเองก็เป็นคนที่ไม่มีพื้นฐานมาก่อนเหมือนกัน ช่วงแรกเพียงแค่ดู [วิดีโอสอน Swift](https://www.youtube.com/watch?v=4SaddkGm5yA&list=PLliocbKHJNwuLHJlVv6q5tKJxJltjBspC&index=3) บางตอนบน Youtube แล้วก็พบว่าการพัฒนาแอป iOS ไม่ได้ซับซ้อนอย่างที่คิด จึงเริ่มเกิดความสนใจขึ้นมา หลังจากนั้นผมก็เริ่มลองเรียนรู้ Swift ให้มากขึ้น เช่น ลองใช้เกมเขียนโปรแกรมใน [Swift Playgrounds](https://apps.apple.com/cn/app/swift-playgrounds/id1496833156?mt=12) อ่าน [เอกสารคู่มือทางการของ Swift](https://docs.swift.org/swift-book/documentation/the-swift-programming-language/) รวมถึงเรียนจาก [100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui)

มองย้อนกลับไป ตอนนั้นจริง ๆ แล้ว [100 Days Of SwiftUI](https://www.hackingwithswift.com/100/swiftui) คือสิ่งที่ช่วยวางกรอบการเรียนรู้ ทำให้ผมสามารถเรียน SwiftUI ได้อย่างเป็นระบบ แต่เพราะบทเรียนนั้นเป็นภาษาอังกฤษ และผมเองก็ไม่มีพื้นฐานด้านการเขียนโปรแกรมมาก่อน จึงทำให้เกิดอุปสรรคในการทำความเข้าใจอยู่มาก ในช่วงนั้น ผมจึงใช้ AI อย่าง [ChatGPT](https://chatgpt.com/) และ [Claude](https://claude.ai/?redirect=claude.com) เพื่อช่วยอธิบายและทำความเข้าใจเนื้อหา

ระหว่างการเรียนรู้ ย่อมมีบางเรื่องที่ทำความเข้าใจได้ยาก ผมจำได้ว่าตอนเรียนเรื่อง UserDefaults ผมรู้แค่ว่าจะบันทึกข้อมูลอย่างไร แต่ไม่รู้ว่าข้อมูลนั้นต้องอ่านกลับมาด้วยตนเอง จนถึงขั้นสงสัยว่าโค้ดของตัวเองมีปัญหาหรือไม่ ทั้งที่เป็นเพียงจุดความรู้เล็ก ๆ แบบนี้ ก็ยังทำให้ผมเสียเวลาไปหลายวัน

ยกตัวอย่างอีกเช่น closure, main thread, generic และแนวคิดอื่น ๆ ตอนช่วงต้นที่เรียน ผมยังไม่สามารถเข้าใจสิ่งเหล่านี้ได้อย่างลึกซึ้ง พอในการพัฒนาจริงกลับมาเจออีกครั้ง ก็ต้องเสียแรงและเวลาหยิบมันกลับมาเรียนใหม่อีกรอบ

ดังนั้น คำแนะนำของผมคือ:

**1、เสริมพื้นฐานให้แน่น**

ในภาษาจีนมีคำพูดว่า “ทุกอย่างยากที่สุดตอนเริ่มต้น” สำหรับผู้เริ่มต้นที่เรียน Swift จากศูนย์ ระหว่างทางย่อมต้องเจอกับเนื้อหาบางอย่างที่เข้าใจได้ยากอยู่แล้ว —— ซึ่งบ่อยครั้งไม่ได้แปลว่าเนื้อหานั้นซับซ้อนมาก แต่เป็นเพราะที่ฐานล่างของพีระมิดความรู้นี้ ยังมีแนวคิดพื้นฐานบางอย่างที่ผู้เริ่มต้นยังไม่เคยได้สัมผัสมาก่อน

เมื่อเจอสถานการณ์แบบนี้ สามารถพิจารณาใช้เครื่องมือ AI อย่าง [ChatGPT](https://chatgpt.com/) และ [Claude](https://claude.ai/?redirect=claude.com) เพื่อช่วยเติมเต็มพื้นฐานที่ขาดหายไปได้ทันที

**2、พยายามเข้าใจทุกจุดความรู้**

ระหว่างการเรียนรู้ สิ่งสำคัญที่สุดคือพยายามทำความเข้าใจทุกจุดความรู้ให้ได้ แม้ว่าจะต้องใช้เวลาหลายวันก็ตาม

ถ้าเลือกข้ามสิ่งที่ยังไม่เข้าใจไป ในการพัฒนาครั้งถัดไปก็มักมีโอกาสสูงที่จะเจอมันอีก และมันจะกลายเป็นอุปสรรคต่อความคืบหน้าในการพัฒนา สุดท้ายก็ยังต้องย้อนกลับมาใช้เวลาเรียนใหม่อยู่ดี แทนที่จะมาแก้ทีหลัง สู้ทำความเข้าใจให้ชัดเจนตั้งแต่ช่วงเรียนจะดีกว่า

ในแก่นแท้แล้ว แต่ละจุดความรู้ก็คือเครื่องมือชิ้นหนึ่ง —— ยิ่งคุณมีเครื่องมือมากเท่าไร ความสามารถในการแก้ปัญหาก็ยิ่งมากขึ้นเท่านั้น

มันก็เหมือนกับการสร้างบ้าน ถ้าในมือคุณมีเครื่องมือเพียงชิ้นเดียว แม้จะพอทำงานส่วนใหญ่ให้เสร็จได้แบบฝืน ๆ แต่เมื่อเจอกับงานที่ต้องการคุณภาพสูงขึ้น คุณก็จะเริ่มรู้สึกว่าเครื่องมือไม่เพียงพอ การมีเครื่องมือมากขึ้น จึงทำให้คุณสร้างบ้านที่มั่นคงกว่าเดิมได้

**3、รักษาการเรียนรู้อย่างต่อเนื่อง**

การเรียนรู้ต้องอาศัยความต่อเนื่องในระยะยาว มีคนไม่มากนักที่ยินดีจะใช้เวลาหลายเดือนเพื่อเรียนบทเรียนหนึ่งอย่างจริงจัง การทุ่มเวลาอย่างสม่ำเสมอสำคัญกว่าการเร่งหนักในช่วงสั้น ๆ

ถึงแม้ในแต่ละวันจะทำได้เพียงหนึ่งเปอร์เซ็นต์ แต่ถ้าทำต่อเนื่องหนึ่งร้อยวัน ก็สามารถบรรลุเป้าหมายการเรียนรู้ทั้งหมดได้เช่นกัน

## วิสัยทัศน์ด้านการศึกษา

คอร์สนี้ไม่ได้ตั้งใจจะสอนความรู้ Swift และ SwiftUI ให้มากจนเกินไป แต่ต้องการพาผู้เริ่มต้นก้าวเข้าสู่เส้นทางของการเขียนโปรแกรมด้วย Swift เปรียบเหมือนการต่อบล็อก —— บทเรียนมีหน้าที่อธิบายวิธีประกอบ ส่วนเวลาที่เหลือคือเวลาที่คุณจะได้ไปสร้างบล็อกของตัวเอง

โปรเจกต์นี้เปิดซอร์สไว้บน [GitHub](https://github.com/fangjunyu1/SwiftSlim) แล้ว สามารถดาวน์โหลดและใช้งานได้ฟรี ขณะเดียวกันก็มีเวอร์ชันบน App Store ให้ทดลองใช้งานด้วย

เราหวังว่าจะช่วยให้ผู้คนที่ไม่มีพื้นฐานด้านเทคนิค แต่ปรารถนาจะเปลี่ยนเส้นทางอาชีพ ได้มีโอกาสใหม่ผ่านการเรียนรู้การเขียนโปรแกรมและการพัฒนาแอป

## การอัปเดตเนื้อหา

เมื่อแพลตฟอร์มและเครื่องมือของ Apple มีการอัปเดต API บางส่วนก็อาจมีการเปลี่ยนแปลงได้ หากพบปัญหาหรือเห็นว่าเนื้อหาจำเป็นต้องอัปเดต สามารถอ้างอิงเอกสารทางการหรือเวอร์ชันล่าสุดของคลังโค้ดโอเพนซอร์สได้
