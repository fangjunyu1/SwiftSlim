# เริ่มต้นจาก ContentView

## การเตรียมตัวก่อนเรียน

ในบทนี้ เราจะเริ่มต้นจากไฟล์ ContentView.swift เพื่อทำความรู้จักกับโครงสร้างพื้นฐานของ SwiftUI อย่างเป็นระบบ ซึ่งประกอบด้วย:

- คอมเมนต์
- โครงสร้างของ View
- เลย์เอาต์ VStack / HStack / ZStack
- ไอคอน SF Symbols
- ตัวปรับแต่งมุมมอง (modifier)
- โค้ดพรีวิว #Preview

ก่อนอื่น ให้หาโปรเจกต์ Xcode ที่สร้างไว้ก่อนหน้านี้ แล้วดับเบิลคลิกไฟล์ .xcodeproj

จากนั้นในแถบ Navigator ทางซ้าย ให้เลือกไฟล์ ContentView.swift

หมายเหตุ: ทุกครั้งที่เปิดโปรเจกต์ Canvas อาจแสดงข้อความ "Preview paused" ให้กดปุ่มรีเฟรชเพื่อเรียกพรีวิวกลับมา

![Swift](../../RESOURCE/002_view7.png)

## ทำความรู้จักกับ ContentView

โค้ดของ ContentView:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

#Preview {
    ContentView()
}
````

แม้โค้ดจะมีไม่มาก แต่ภายในนั้นได้รวมโครงสร้างแกนหลักของ SwiftUI ไว้แล้ว

### 1. คอมเมนต์

ส่วนบนสุดของไฟล์:

```swift
//
//  ContentView.swift
//  SwiftSlimTest
//
//  Created by 方君宇 on 2026/2/23.
//
```

นี่คือคอมเมนต์ของไฟล์ ใช้สำหรับบันทึกข้อมูลของไฟล์ ซึ่งรวมถึงชื่อไฟล์ ชื่อโปรเจกต์ ผู้สร้าง และเวลาที่สร้าง

ใน Swift ใช้ // สำหรับคอมเมนต์แบบบรรทัดเดียว:

```swift
// นี่คือคอมเมนต์
```

คอมเมนต์ช่วยเพิ่มความสามารถในการอ่านโค้ด และช่วยให้นักพัฒนาเข้าใจตรรกะของโค้ดได้ง่ายขึ้น

ในการพัฒนาจริง โค้ดอาจอ่านยากและซับซ้อนได้ ถ้าไม่เขียนคอมเมนต์ ก็อาจเกิดสถานการณ์ที่ผ่านไปสามวันแล้วกลับมาดูโค้ดของตัวเองไม่เข้าใจ

ดังนั้น การใส่คอมเมนต์ระหว่างเขียนโค้ดอย่างสม่ำเสมอจึงเป็นนิสัยการพัฒนาที่ดี การใช้คอมเมนต์บันทึกตรรกะของโค้ด จะช่วยให้การดูแลรักษาโค้ดในภายหลังสะดวกขึ้น

**การปิดโค้ดชั่วคราว**

คอมเมนต์ยังสามารถใช้ปิดโค้ดชั่วคราว เพื่อช่วยตรวจสอบปัญหาได้ด้วย

ตัวอย่างเช่น:

```
A
B
C
```

A、B、C คือโค้ดสามส่วน โดยมีหนึ่งส่วนที่เกิดข้อผิดพลาด เราสามารถใช้การคอมเมนต์เพื่อไล่หาปัญหาได้

เริ่มจากใส่คอมเมนต์ให้ A:

```
// A
B
C
```

ถ้าคอมเมนต์ A แล้วโค้ดกลับมาทำงานปกติ แสดงว่าปัญหาอยู่ที่โค้ด A

ถ้าคอมเมนต์ A แล้วปัญหายังอยู่ เราก็สามารถคอมเมนต์ B ต่อไป แล้วตรวจสอบแบบเดียวกันไปเรื่อย ๆ จนกว่าจะพบโค้ดที่มีปัญหา

ระหว่างการพัฒนา เราจะเจอปัญหาหลายรูปแบบ ซึ่งส่วนใหญ่ต้องอาศัยการคอมเมนต์โค้ดเพื่อช่วยหาสาเหตุ วิธีนี้ช่วยให้เราระบุตำแหน่งของโค้ดที่มีปัญหา และค้นหา BUG ได้

ใน Xcode สามารถใช้คีย์ลัดต่อไปนี้:

```
Command ⌘ + /
```

เพื่อเพิ่มหรือลบคอมเมนต์ได้อย่างรวดเร็ว

### 2. นำเข้าเฟรมเวิร์ก SwiftUI

```swift
import SwiftUI
```

โค้ดบรรทัดนี้หมายถึงการนำเข้าเฟรมเวิร์ก SwiftUI

ชนิดต่าง ๆ ใน SwiftUI เช่น View、Text、Image、VStack ล้วนมาจากเฟรมเวิร์กนี้

ถ้าไม่นำเข้าเฟรมเวิร์ก SwiftUI, Xcode จะรายงานข้อผิดพลาดว่า:

```
Cannot find type 'View' in scope
```

ซึ่งหมายความว่าคอมไพเลอร์ไม่สามารถรู้จักชนิด View ได้

### 3. โครงสร้างของ View

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

เมื่อเห็นโครงสร้าง View นี้ครั้งแรก อาจรู้สึกแปลกตา เพราะมีคีย์เวิร์ดอย่าง struct、View、var、body、some อยู่ภายใน

ตอนนี้เรายังไม่ได้เรียนคีย์เวิร์ดเหล่านี้อย่างเป็นทางการ เพียงแค่รู้ว่าความหมายของโค้ดส่วนนี้คือ การสร้างมุมมองชื่อ ContentView ขึ้นมา

คุณสามารถมองว่า View คือกระดานวาดภาพหนึ่งแผ่น และสิ่งที่เราใช้วาดลงไปบนกระดานนั้นก็คือ SwiftUI

ตัวอย่างเช่น:

![Swift](../../RESOURCE/002_view.png)

ภาพด้านบนแสดงสามหน้า ซึ่งในความเป็นจริงก็คือ View สามตัว

```swift
struct View1: View {
    var body: some View {
        ...
    }
}
struct View2: View {
    var body: some View {
        ...
    }
}
struct View3: View {
    var body: some View {
        ...
    }
}
```

เมื่อเราใช้ SwiftUI พัฒนาแอป แต่ละหน้าก็คือ View หนึ่งตัว

### 4. โค้ด SwiftUI

ภายใน View จะมีโค้ด SwiftUI ที่ใช้แสดงผลดังนี้:

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

โค้ด SwiftUI ส่วนนี้หมายความว่า มีเลย์เอาต์แนวตั้งหนึ่งตัว ภายในแสดงไอคอนหนึ่งอันและข้อความหนึ่งบรรทัด

![Swift](../../RESOURCE/002_view1.png)

#### เลย์เอาต์ VStack

```swift
VStack { }  // เลย์เอาต์แนวตั้ง
```

VStack หมายถึงคอนเทนเนอร์จัดวางแบบแนวตั้ง มุมมองภายในจะเรียงจากบนลงล่าง

![Swift](../../RESOURCE/002_view8.png)

เลย์เอาต์ที่พบบ่อยใน SwiftUI มี 3 แบบ:

* VStack —— เรียงแนวตั้ง
* HStack —— เรียงแนวนอน
* ZStack —— ซ้อนทับกัน (แกน Z)

```swift
HStack { }  // เรียงแนวนอน
ZStack { }  // ซ้อนทับกัน
```

ภาพตัวอย่างการจัดเรียงของแต่ละเลย์เอาต์:

![Swift](../../RESOURCE/002_view2.png)

ตัวอย่าง: ใช้ HStack เพื่อจัดเรียงแนวนอน

```swift
HStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

จะเห็นว่าไอคอนรูปโลกและข้อความถูกแสดงเรียงในแนวนอน

![Swift](../../RESOURCE/002_view3.png)

เมื่อเราต้องการเรียงในแนวนอน ให้ใช้ HStack และเมื่อต้องการจัดวางแบบซ้อนทับกัน ให้ใช้ ZStack

#### Image และ SF Symbols

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

โค้ดส่วนนี้ใช้แสดงไอคอนรูปโลก ซึ่งไอคอนนี้มาจากระบบไอคอน SF Symbols ของ Apple

![Swift](../../RESOURCE/002_view9.png)

ความหมายของโค้ดนี้คือ แสดงไอคอนรูปโลก ขนาดใหญ่ และใช้สีเน้นของระบบ

นอกจากจะแสดงไอคอนรูปโลกแล้ว เรายังสามารถแสดงไอคอนอื่น ๆ ได้ด้วย

ตัวอย่างเช่น แสดงรูปกระเป๋าเป้:

```swift
Image(systemName: "backpack")
```

![Swift](../../RESOURCE/002_view4.png)

**แล้วจะแสดงไอคอนอื่นได้อย่างไร?**

เราจำเป็นต้องใช้คลังไอคอนระบบ SF Symbols ที่ Apple จัดเตรียมไว้ให้

เข้าไปที่เว็บไซต์ทางการของ Apple Developer แล้วดาวน์โหลด [SF Symbols](https://developer.apple.com/sf-symbols/)

![Swift](../../RESOURCE/002_sf.png)

จากนั้นเปิดแอป SF Symbols

![Swift](../../RESOURCE/002_sf1.png)

ด้านซ้ายคือหมวดหมู่ของสัญลักษณ์ ส่วนด้านขวาคือไอคอนที่สอดคล้องกัน

คลิกขวาที่ไอคอน แล้วเลือก "คัดลอกชื่อ 1 รายการ" เพื่อคัดลอกชื่อ ซึ่งชื่อนี้ก็คือสตริงที่ตรงกับไอคอนนั้น

ตัวอย่างเช่น:

```
"globe"
"backpack"
"heart"
```

ให้นำชื่อไอคอนที่คัดลอกมา ไปใส่ใน Image(systemName:) ก็จะแสดงไอคอนที่แตกต่างกันได้

ข้อควรระวัง: ไอคอนแต่ละตัวใน SF Symbols มีเวอร์ชันระบบขั้นต่ำที่รองรับ หากเวอร์ชันระบบต่ำเกินไป ไอคอนอาจไม่แสดง ต้องตรวจสอบข้อมูลความเข้ากันได้ในแอป SF Symbols

#### Modifier

ใน SwiftUI, modifier คือเมธอดชนิดหนึ่งที่ใช้เปลี่ยนรูปลักษณ์ภายนอกหรือพฤติกรรมของมุมมอง

คุณสามารถมอง modifier เหมือนเสื้อผ้า เมื่อสวมเสื้อผ้าต่างกัน รูปลักษณ์ที่แสดงออกมาก็จะแตกต่างกัน

```swift
Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
```

imageScale และ foregroundStyle ก็คือ modifier ของมุมมอง Image ซึ่งใช้ตกแต่งมุมมอง Image โดยไม่เปลี่ยนเนื้อหาของ Image เอง

**1. imageScale**

```swift
.imageScale(.large)
```

มันใช้ควบคุมขนาดของไอคอน SF Symbols:

* .small
* .medium
* .large

![Swift](../../RESOURCE/002_view11.png)

เราสามารถเปลี่ยนตัวเลือกต่าง ๆ เพื่อให้สัญลักษณ์ SF Symbols แสดงในขนาดที่แตกต่างกันได้

**2. foregroundStyle**

```swift
.foregroundStyle(.tint)
```

foregroundStyle ใช้ควบคุมสีของส่วนหน้า

.tint หมายถึงสีเน้นของสภาพแวดล้อมปัจจุบัน (accent color) ซึ่งโดยค่าเริ่มต้นบน iOS จะเป็นสีน้ำเงิน

ถ้าเราต้องการเปลี่ยนสีของส่วนหน้าเป็นสีแดง:

```swift
.foregroundStyle(.red)
```

![Swift](../../RESOURCE/002_view5.png)

#### มุมมอง Text

Text คือมุมมองข้อความ ใช้สำหรับแสดงสตริง

```swift
Text("Hello, world!")
```

ตัวอย่างเช่น ถ้าต้องการแสดงชื่อของฉัน:

```swift
Text("FangJunyu")
```

ข้อควรระวัง: สตริงจำเป็นต้องใส่ไว้ภายในเครื่องหมายอัญประกาศคู่ ""

คุณสามารถลองแสดงชื่อของคุณ เบอร์โทรศัพท์ หรือข้อมูลข้อความอื่น ๆ ได้

#### padding ระยะขอบด้านใน

ใน SwiftUI, padding ใช้สำหรับเพิ่มพื้นที่ว่างระหว่างเนื้อหาของมุมมองกับขอบเขตของมัน ซึ่งจัดอยู่ในประเภท “ระยะขอบด้านใน (padding / content inset)”

```swift
HStack {
    ...
}
.padding()
```

โค้ดด้านบนหมายถึงการเพิ่มระยะขอบมาตรฐานของระบบให้กับมุมมอง HStack

**padding คืออะไร?**

padding หมายถึง "พื้นที่ว่างระหว่างเนื้อหาของมุมมองกับขอบเขตของมัน"

ในภาพด้านล่าง เมื่อ HStack สีฟ้าตั้งค่า padding แล้ว พื้นที่สีน้ำเงินจะหดเข้าด้านใน ทำให้ดูเหมือนว่า "เล็กลงหนึ่งรอบ"

![Swift](../../RESOURCE/002_view6.png)

**ระยะขอบเริ่มต้น**

modifier อย่าง padding() จะใช้ระยะห่างมาตรฐานที่ระบบแนะนำโดยอัตโนมัติ

```swift
.padding()
```

บนแพลตฟอร์มหรือบริบทที่ต่างกัน ค่านี้อาจต่างกันได้ เช่น:

* บน iOS มักอยู่ที่ประมาณ 16 pt
* บน macOS หรือ watchOS ระยะมาตรฐานของระบบอาจมากหรือน้อยกว่านี้ ทั้งนี้ขึ้นอยู่กับแนวทางการออกแบบของแต่ละแพลตฟอร์ม

**กำหนดระยะขอบเอง**

เรายังสามารถตั้งค่าระยะขอบเฉพาะด้านให้กับมุมมองได้

1. กำหนดทิศทางเดียว

```swift
.padding(.top, 20)
.padding(.leading, 20)
.padding(.trailing, 20)
.padding(.bottom, 20)
```

คำอธิบายของแต่ละทิศทาง:

* .top: ระยะขอบด้านบน
* .bottom: ระยะขอบด้านล่าง
* .leading: ระยะขอบด้านหน้า
* .trailing: ระยะขอบด้านหลัง

![Swift](../../RESOURCE/002_view12.png)

ข้อควรระวัง: leading และ trailing จะปรับตามทิศทางภาษาโดยอัตโนมัติ เช่น ในสภาพแวดล้อมภาษาอาหรับแบบ RTL ทิศทางจะถูกสลับให้อัตโนมัติ

2. กำหนดหลายทิศทางพร้อมกัน

```swift
.padding([.top, .leading], 20)
```

![Swift](../../RESOURCE/002_view13.png)

เราสามารถใช้ array เพื่อระบุหลายทิศทางพร้อมกันได้ การใช้งาน array อย่างละเอียดจะอธิบายในบทเรียนถัดไป ตอนนี้เพียงแค่รู้จักรูปแบบการเขียนนี้ไว้ก่อนก็พอ

3. กำหนดแนวนอนหรือแนวตั้ง

```swift
.padding(.horizontal, 20)
.padding(.vertical, 20)
```

ซึ่งเทียบเท่ากับ:

```swift
.padding([.leading, .trailing], 20)
.padding([.top, .bottom], 20)
```

![Swift](../../RESOURCE/002_view14.png)

**ลบระยะขอบ**

ถ้าไม่ต้องการระยะขอบเลย สามารถใช้ .padding(0) เพื่อลบออกได้:

```swift
.padding(0)
```

หรือจะลบ modifier padding ออกไปตรง ๆ ก็ได้:

```swift
// .padding()
```

นี่คือบทบาทของ #Preview ที่ใช้ควบคุมการแสดงมุมมองพรีวิวใน Canvas

เมื่อเราต้องการพรีวิวมุมมอง SwiftUI ใน Xcode ให้เพิ่มโค้ด #Preview เข้าไป และถ้าไม่ต้องการพรีวิว ก็สามารถคอมเมนต์หรือลบโค้ด #Preview ออกได้

## สรุป

แม้ว่าโค้ดในไฟล์ ContentView.swift จะมีไม่มาก แต่ในความเป็นจริงแล้วมันครอบคลุมแนวคิดแกนหลักหลายอย่างของ SwiftUI สำหรับผู้เริ่มต้น โค้ดชุดนี้อาจดูแปลกตา แต่เมื่อค่อย ๆ แยกโครงสร้างของโค้ดออกมาศึกษา ก็จะช่วยสร้างความเข้าใจเบื้องต้นเกี่ยวกับ SwiftUI ได้

ทบทวนเนื้อหาในบทนี้ เราเริ่มจากการเรียนรู้คอมเมนต์ // ซึ่งสามารถใช้เพื่ออธิบายตรรกะของโค้ด หรือใช้ปิดโค้ดชั่วคราวได้

จากนั้น เราได้รู้ว่าไฟล์ SwiftUI จำเป็นต้องนำเข้าเฟรมเวิร์ก SwiftUI:

```swift
import SwiftUI
```

ถ้าไม่นำเข้าเฟรมเวิร์ก คอมไพเลอร์จะไม่สามารถรู้จักชนิดอย่าง View ได้

ต่อมา เราได้รู้จักโครงสร้างพื้นฐานของมุมมอง SwiftUI:

```swift
struct ContentView: View {
    var body: some View {
        ...
    }
}
```

โดยที่ ContentView คือชื่อของมุมมอง

เรายังได้เรียนรู้คอนเทนเนอร์เลย์เอาต์ที่พบบ่อย 3 แบบ ได้แก่ VStack (เรียงแนวตั้ง)、HStack (เรียงแนวนอน) และ ZStack (ซ้อนทับกัน)

```swift
VStack {
    Image(systemName: "globe")
        .imageScale(.large)
        .foregroundStyle(.tint)
    Text("Hello, world!")
}
.padding()
```

ในโค้ดส่วนนี้หมายความว่า แสดงคอนเทนเนอร์ที่เรียงในแนวตั้ง โดยภายในคอนเทนเนอร์แสดงไอคอนหนึ่งอันและข้อความหนึ่งบรรทัด

Image สามารถใช้แสดงไอคอน SF Symbols และใช้ modifier เพื่อควบคุมขนาดและสีของไอคอนได้

มุมมอง Text ใช้สำหรับแสดงข้อความ

padding คือระยะขอบ ซึ่งใช้เพิ่มพื้นที่โปร่งใสรอบ ๆ มุมมอง

สุดท้ายคือมุมมองพรีวิว #Preview ซึ่งใช้แสดงพรีวิวใน Canvas

### แบบฝึกหัดหลังเรียน

สำหรับผู้เริ่มต้น เนื้อหาในบทนี้อาจค่อนข้างซับซ้อน เราสามารถเสริมความเข้าใจด้วยการฝึกต่อไปนี้:

* เปลี่ยนชื่อไอคอน SF Symbols
* เปลี่ยนสีด้านหน้าของไอคอนเป็นสีดำ
* เปลี่ยน VStack เป็น HStack
* คอมเมนต์ Image หรือ Text แล้วสังเกตการเปลี่ยนแปลงของมุมมองพรีวิว

### ของแถม: การเติมโค้ดอัตโนมัติ (Code Completion)

ระหว่างพิมพ์โค้ด หากคุณสังเกตดี ๆ อาจพบว่า Xcode จะแสดงรายการตัวเลือกที่ใช้ได้ขึ้นมาโดยอัตโนมัติ

ตัวอย่างเช่น เมื่อเราแก้ไข modifier imageScale:

```swift
.imageScale(.)
```

Xcode จะแสดงตัวเลือกที่สามารถใช้ได้:

![Swift](../../RESOURCE/002_view10.png)

นี่คือกลไกการเติมโค้ดอัตโนมัติ (Code Completion) ซึ่งทำงานบนพื้นฐานของการอนุมานชนิดข้อมูลและการแนะนำสมาชิกของ enum จึงช่วยเพิ่มความเร็วในการพิมพ์และลดความผิดพลาดได้

ในบทเรียนถัด ๆ ไป เราจะอธิบายเรื่อง enum อย่างเป็นทางการ ตอนนี้เพียงแค่ทำความรู้จักแบบคร่าว ๆ ก่อน
