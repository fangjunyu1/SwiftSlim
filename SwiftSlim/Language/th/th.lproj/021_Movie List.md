# รายการภาพยนตร์

ในบทเรียนนี้ เราจะสร้างรายการภาพยนตร์

![movie](../../Resource/021_movie.png)

ในรายการนี้จะแสดงโปสเตอร์ ชื่อ ผู้กำกับ และคะแนนของภาพยนตร์แต่ละเรื่อง

ในตัวอย่างนี้ เราจะได้เรียนรู้หัวข้อที่สำคัญมากหัวข้อหนึ่งคือ โครงสร้าง `struct` ซึ่งช่วยให้เรารวมข้อมูลหลายอย่างของภาพยนตร์หนึ่งเรื่องให้กลายเป็นข้อมูลชุดเดียว นอกจากนี้เรายังจะได้รู้จัก `UUID()`, `ForEach`, เส้นคั่น `Divider` รวมถึงวิธีใช้วัตถุแบบกำหนดเองในการจัดการข้อมูล

ความรู้เหล่านี้พบได้บ่อยมากในการพัฒนา SwiftUI ในภายหลัง หลังจากเรียนบทนี้จบ คุณจะไม่เพียงทำรายการภาพยนตร์ได้เท่านั้น แต่ยังจะเริ่มเข้าใจด้วยว่า “จะแสดงข้อมูลชุดหนึ่งออกมาเป็นหน้าจอได้อย่างไร”

## ภาพยนตร์หนึ่งเรื่อง

เราสามารถเริ่มจากการสร้างหน้าจอของภาพยนตร์เพียงเรื่องเดียวก่อน

![movie](../../Resource/021_movie1.png)

เลย์เอาต์ของหน้าจอนี้ประกอบด้วยสองส่วนหลัก: ด้านซ้ายคือโปสเตอร์ภาพยนตร์ ด้านขวาคือข้อมูลแนะนำภาพยนตร์

### โปสเตอร์ภาพยนตร์

ด้านซ้ายจะแสดงโปสเตอร์ภาพยนตร์ ซึ่งเราสามารถใช้ `Image` เพื่อแสดงรูปภาพได้

ตัวอย่างเช่น:

```swift
Image("The Shawshank Redemption")
    .resizable()
    .scaledToFit()
    .frame(height: 180)
    .cornerRadius(10)
```

โค้ดนี้หมายถึงการแสดงรูปภาพที่มีชื่อว่า `"The Shawshank Redemption"`

สิ่งที่ต้องระวังคือ ชื่อรูปภาพจะต้องมีอยู่จริงในโฟลเดอร์ทรัพยากร `Assets` มิฉะนั้นหน้าจอจะไม่สามารถแสดงรูปภาพนี้ได้อย่างถูกต้อง

![movie](../../Resource/021_movie2.png)

หน้าที่ของ modifiers เหล่านี้มีดังนี้:

- `resizable()` หมายความว่ารูปภาพสามารถปรับขนาดได้
- `scaledToFit()` หมายความว่าระหว่างการย่อหรือขยายจะคงสัดส่วนเดิมไว้ เพื่อไม่ให้รูปภาพถูกยืดจนผิดรูป
- `frame(height: 180)` หมายความว่ากำหนดความสูงของรูปภาพเป็น 180
- `cornerRadius(10)` หมายความว่ากำหนดมุมโค้งของรูปภาพเป็น 10

ด้วยวิธีนี้ เราจะได้โปสเตอร์ภาพยนตร์ที่มีขนาดเหมาะสมและมีมุมโค้ง

![movie](../../Resource/021_movie3.png)

### ข้อมูลแนะนำภาพยนตร์

ด้านขวาจะแสดงข้อมูลแนะนำภาพยนตร์ ซึ่งรวมถึงชื่อภาพยนตร์ ผู้กำกับ และคะแนน

![movie](../../Resource/021_movie4.png)

เราสามารถใช้ `Text` เพื่อแสดงเนื้อหาส่วนนี้ได้:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)
	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}
	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

ในโค้ดนี้ `.font` หมายถึงขนาดตัวอักษร และ `.fontWeight` หมายถึงน้ำหนักของตัวอักษร

เพื่อให้ระยะห่างระหว่าง “ชื่อภาพยนตร์” “ข้อมูลผู้กำกับ” และ “ข้อมูลคะแนน” ชัดเจนขึ้น ที่นี่เราใช้ `VStack` ชั้นนอกสุดและตั้งค่า:

```swift
spacing: 10
```

ซึ่งหมายความว่าระยะห่างระหว่างแต่ละกลุ่มเนื้อหาภายใน `VStack` นี้จะเท่ากับ 10

เหตุผลที่ผู้กำกับและคะแนนถูกห่อด้วย `VStack` แยกกันอีกชั้นหนึ่ง ก็เพราะทั้งสองส่วนนี้อยู่ในโครงสร้างแบบ “หัวข้อ + เนื้อหา” ดังนั้นลำดับชั้นของหน้าจอจะชัดเจนขึ้น และยังสะดวกต่อการปรับสไตล์ต่อในภายหลังด้วย

นอกจากนี้ `VStack` จะจัดวางตรงกลางโดยค่าเริ่มต้น เพื่อให้ข้อความทั้งหมดชิดซ้าย เราจึงตั้งค่า:

```swift
alignment: .leading
```

ด้วยวิธีนี้ ส่วนแนะนำภาพยนตร์จะดูเป็นระเบียบมากขึ้น

### เส้นคั่น

ตอนนี้ แม้ว่าเนื้อหาในส่วนแนะนำภาพยนตร์จะถูกแยกห่างกันด้วย `spacing` แล้ว แต่เส้นแบ่งระหว่างเนื้อหาต่าง ๆ ก็ยังไม่ชัดเจนพอ

ในเวลานี้ เราสามารถเพิ่มเส้นคั่นได้:

```swift
Divider()
```

ตัวอย่างเช่น:

```swift
VStack(alignment: .leading,spacing: 10) {
	Text("The Shawshank Redemption")
	    .font(.title3)
	    .fontWeight(.bold)

	Divider()

	VStack(alignment: .leading) {
		Text("director")
		    .fontWeight(.bold)
		Text("Frank Darabont")
	}

	Divider()

	VStack(alignment: .leading) {
		Text("rating")
		    .fontWeight(.bold)
		Text("9.3 / 10")
	}
}
```

ด้วยวิธีนี้ ส่วนแนะนำภาพยนตร์จะมีเอฟเฟกต์การแบ่งที่ชัดเจนขึ้น

![movie](../../Resource/021_movie4.png)

`Divider` เป็น view ที่เรียบง่ายมากแต่ใช้งานบ่อยมาก หน้าที่ของมันคือใช้แยกเนื้อหาที่แตกต่างกันออกจากกัน

ใน `VStack`, `Divider()` จะถูกแสดงเป็นเส้นแนวนอน

ใน `HStack`, `Divider()` จะถูกแสดงเป็นเส้นแนวตั้ง

นอกจากนี้ ยังสามารถปรับสไตล์ของ `Divider` ได้ผ่าน `.frame`, `.background`, `.padding` และวิธีอื่น ๆ

ตัวอย่างเช่น:

```swift
Divider()
    .frame(height: 2)
    .background(Color.blue)
    .padding(.horizontal)
```

ซึ่งจะสร้างเส้นคั่นสีน้ำเงิน ความหนา 2 และมีระยะขอบด้านในทางซ้ายและขวา

![divider](../../Resource/021_divider.png)

### หน้าจอแบบสมบูรณ์

สุดท้าย เราสามารถใช้ `HStack` เพื่อจัดโปสเตอร์ภาพยนตร์และข้อมูลแนะนำภาพยนตร์ให้อยู่เรียงกันซ้ายขวา

```swift
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
    	.scaledToFit()
    	.frame(height: 180)
    	.cornerRadius(10)
	VStack(alignment: .leading,spacing: 10) {
		Text("The Shawshank Redemption")
		    .font(.title3)
		    .fontWeight(.bold)

		Divider()

		VStack(alignment: .leading) {
			Text("director")
			    .fontWeight(.bold)
			Text("Frank Darabont")
		}

		Divider()

		VStack(alignment: .leading) {
			Text("rating")
			    .fontWeight(.bold)
			Text("9.3 / 10")
		}
	}
	.frame(width: 200)
}
```

ที่นี่ `HStack(spacing: 20)` หมายความว่ามีระยะห่าง 20 ระหว่างส่วนซ้ายและขวา

สิ่งที่ต้องสังเกตก็คือ `Divider()` เช่นเดียวกับ `TextField` และ `Slider` ที่กล่าวถึงก่อนหน้านี้ จะพยายามใช้พื้นที่ที่มีอยู่ให้มากที่สุดโดยค่าเริ่มต้น

ดังนั้น เราจึงกำหนดความกว้างคงที่ให้กับ `VStack` ชั้นนอกของส่วนแนะนำทางด้านขวา:

```swift
.frame(width: 200)
```

วิธีนี้จะทำให้ความกว้างของเส้นคั่นและพื้นที่ข้อความคงที่เท่ากัน และภาพรวมจะดูเป็นระเบียบมากขึ้น

ถึงตรงนี้ เราก็ได้ทำหน้าจอแสดงผลของ “ภาพยนตร์หนึ่งเรื่อง” เสร็จแล้ว

![movie](../../Resource/021_movie1.png)

## เก็บภาพยนตร์ใน array

ถ้าเราต้องการแสดงภาพยนตร์หลายเรื่องต่อเนื่องกันในรูปแบบเดียวกัน ก็หมายความว่าเราต้องเขียนโค้ดลักษณะเดียวกันซ้ำสำหรับภาพยนตร์แต่ละเรื่อง

ตัวอย่างเช่น:

```swift
// The Shawshank Redemption
HStack(spacing: 20) {
	Image("The Shawshank Redemption")
    	.resizable()
		...
}
// Forrest Gump
HStack(spacing: 20) {
	Image("Forrest Gump")
    	.resizable()
		...
}
// 3 Idiots
HStack(spacing: 20) {
	Image("3 Idiots")
    	.resizable()
		...
}
// ...
```

แม้ว่าโค้ดที่เขียนแบบนี้จะสามารถทำให้เกิดผลลัพธ์ได้ แต่โค้ดจะยิ่งยาวขึ้นเรื่อย ๆ และการดูแลรักษาก็จะยุ่งยากมาก

ตัวอย่างเช่น ถ้าต้องการเปลี่ยนระยะห่างของภาพยนตร์ทั้งหมดจาก `20` เป็น `15` หรือเปลี่ยนความกว้างด้านขวาจาก `200` เป็น
`220` ก็จำเป็นต้องแก้โค้ดที่ซ้ำกันทีละส่วนด้วยตนเอง

เห็นได้ชัดว่านี่ไม่ใช่วิธีที่มีประสิทธิภาพ

ก่อนหน้านี้เราได้เรียนรู้เรื่อง array มาแล้ว และยังได้เรียนรู้การใช้ `ForEach` เพื่อแสดง view ซ้ำตาม array ด้วย

ตัวอย่างเช่น:

```swift
VStack {
    let images = ["430F9BEF","193E0C8C","FEE2AB8D","35AA4D15"]

    ForEach(images, id: \.self) {
        item in
        Image(item)
            .resizable()
            .scaledToFit()
            .frame(width: 200)
    }
}
```

วิธีนี้เหมาะมากสำหรับกรณีที่มี “ข้อมูลเพียงหนึ่งชนิด” เช่น กลุ่มของชื่อรูปภาพ

แต่ภาพยนตร์หนึ่งเรื่องไม่ได้มีเพียงค่าเดียว อย่างน้อยก็ประกอบด้วย:

1. โปสเตอร์ภาพยนตร์
2. ชื่อภาพยนตร์
3. ผู้กำกับ
4. คะแนน

กล่าวอีกอย่างคือ ภาพยนตร์หนึ่งเรื่องแท้จริงแล้วเป็นชุดของข้อมูลที่เกี่ยวข้องกัน ไม่ใช่เพียงสตริงเดียว

ถ้าเราใช้แค่ array เราก็จะเก็บข้อมูลเหล่านี้ได้เพียงแยกกัน:

```swift
let movieName = ["The Shawshank Redemption", "Forrest Gump", "3 Idiots", "The Pursuit of Happyness", "Dead Poets Society"]
let movieDirector = ["Frank Darabont", "Robert Zemeckis", "Rajkumar Hirani", "Gabriele Muccino", "Peter Weir"]
let movieRating = ["9.3 / 10", "8.8 / 10", "8.4 / 10", "8.0 / 10", "8.1 / 10"]
```

ตอนแสดงผล เราต้องพึ่งพา index เดียวกันเพื่อจับคู่กันทีละรายการ:

```swift
Text(movieName[0])
    .font(.title3)
    .fontWeight(.bold)

Divider()

VStack(alignment: .leading) {
	Text("director")
	    .fontWeight(.bold)
	Text(movieDirector[0])
}
```

ปัญหาของวิธีเขียนแบบนี้ก็คือ ต้นทุนในการดูแลรักษาสูงมาก

เพราะข้อมูลของภาพยนตร์หนึ่งเรื่องถูกแยกกระจายไปอยู่ในหลาย array แค่ array ใด array หนึ่งมีรายการเกินมา ขาดไป หรือเรียงลำดับไม่ตรงกัน ผลลัพธ์ของการแสดงผลก็จะผิดพลาด

โดยเฉพาะเวลาที่เพิ่มหรือลบภาพยนตร์ คุณต้องแก้ไขหลาย array พร้อมกัน ซึ่งพลาดได้ง่ายมาก

แล้วเราจะรวมข้อมูลอย่าง “ชื่อภาพยนตร์ ผู้กำกับ คะแนน” ให้กลายเป็นก้อนเดียวได้อย่างไร?

คำตอบก็คือ ต้องใช้ `struct`

## การกำหนดโครงสร้าง `struct`

ใน Swift, `struct` คือ “โครงสร้าง”

คุณสามารถเข้าใจมันได้ว่าเป็น “ชนิดข้อมูลแบบกำหนดเอง” ที่สามารถรวมฟิลด์ที่เกี่ยวข้องหลาย ๆ ตัวเข้าด้วยกันให้เป็นหนึ่งเดียว

การใช้งานพื้นฐาน:

```swift
struct StructName {
	let name: String
}
```

ในโค้ดนี้ `struct` เป็นคีย์เวิร์ดที่ใช้บอกว่าเรากำลังกำหนดโครงสร้างหนึ่งตัว ส่วน `StructName` คือชื่อของโครงสร้างนั้น

เนื้อหาที่อยู่ในวงเล็บปีกกา คือฟิลด์ต่าง ๆ ที่โครงสร้างนี้มีอยู่ และทุกฟิลด์จะต้องเขียนชื่อกับชนิดข้อมูลให้ชัดเจน

โดยทั่วไป ชื่อของ `struct` มักจะขึ้นต้นด้วยตัวอักษรใหญ่ เช่น `Movie`, `Student`, `UserInfo` ซึ่งเป็นรูปแบบการตั้งชื่อที่พบได้บ่อยใน Swift

คุณสามารถเข้าใจ `struct` อย่างง่าย ๆ ว่าเป็นกล่องบรรจุภัณฑ์ว่าง ๆ ซึ่งแต่ละฟิลด์เปรียบเสมือนตำแหน่งว่างที่เตรียมไว้ในกล่อง

เมื่อช่องเหล่านี้ยังไม่ได้ใส่เนื้อหา มันก็เป็นเพียงกล่องว่างเท่านั้น เมื่อใส่ค่าที่เหมาะสมลงไปครบทุกฟิลด์แล้ว มันจึงจะกลายเป็นกล่องของขวัญที่สมบูรณ์

### อินสแตนซ์ของ `struct`

ก่อนหน้านี้เราเพียงกำหนดตัวโครงสร้างขึ้นมา ซึ่งเท่ากับเป็นการเตรียมรูปแบบของกล่องบรรจุภัณฑ์ไว้

เรายังต้องใส่เนื้อหาจริงเข้าไป จึงจะได้ “อินสแตนซ์” ที่สามารถใช้งานได้จริง

เวลาเราสร้างอินสแตนซ์ มักจะใส่ `()` ต่อท้ายชื่อของโครงสร้าง:

```swift
StructName(...)
```

สิ่งที่ใส่ในวงเล็บก็คือค่าของฟิลด์ที่โครงสร้างนี้ต้องการ

ตัวอย่างเช่น:

```swift
StructName(name: "Fang Junyu")
```

โค้ดนี้หมายความว่า: สร้างอินสแตนซ์ใหม่ตามรูปแบบของ `StructName` และกำหนดค่า `"Fang Junyu"` ให้กับฟิลด์ `name`

เมื่อเราใส่ค่าที่จำเป็นครบทุกฟิลด์ ก็เท่ากับว่าเราได้เติมกล่องนี้จนเต็มแล้ว

ตอนนั้นเองเราจึงจะได้อินสแตนซ์ของโครงสร้างที่สมบูรณ์

### การเข้าถึงคุณสมบัติของ `struct`

เมื่อเราสร้างอินสแตนซ์เสร็จแล้ว เราสามารถใช้ “dot syntax” เพื่อเข้าถึงคุณสมบัติภายในของมันได้

รูปแบบพื้นฐาน:

```swift
instance.propertyName
```

ตัวอย่างเช่น:

```swift
let st = StructName(name: "Fang Junyu")
st.name    // "Fang Junyu"
```

ตรงนี้ `st` ก็คืออินสแตนซ์ของโครงสร้าง และ `st.name` หมายถึงการอ่านค่า `name` จากอินสแตนซ์นี้

คุณสามารถเข้าใจวิธีอ่านแบบนี้ได้ว่า เมื่อเราเขียน `st.name` ก็เหมือนกับการอ่านเนื้อหาที่ถูกเก็บไว้ในตำแหน่งหนึ่งของกล่อง

วิธีเขียนแบบนี้จะพบได้บ่อยมากในหน้าจอ SwiftUI ภายหลัง

ตัวอย่างเช่น ถ้าเรามีโครงสร้างของนักเรียนที่มีชื่อ อายุ และห้องเรียน เวลาแสดงบนหน้าจอ เราก็สามารถอ่านได้แยกกันว่า:

```swift
student.name
student.age
student.className
```

ข้อดีของวิธีนี้คือทำให้ข้อมูลชัดเจนขึ้น และยังสะดวกต่อการจัดการอีกด้วย

### ตำแหน่งของ `struct`

สำหรับผู้เริ่มต้น โดยทั่วไปมักจะเขียนโครงสร้างไว้นอก `ContentView`

ตัวอย่างเช่น:

```swift
import SwiftUI

struct StructName {
	let name: String
}

struct ContentView: View {
	// ...
}
```

แบบนี้จะทำให้โครงสร้างชัดเจนขึ้น และอ่านได้ง่ายขึ้นด้วย

คุณจะพบว่า `ContentView` เองก็จริง ๆ แล้วเป็น `struct` เช่นกัน

### โครงสร้าง `Movie`

เมื่อมีพื้นฐานข้างต้นแล้ว เราก็สามารถกำหนดโครงสร้างของภาพยนตร์ได้:

```swift
struct Movie {
    let name: String
    let director: String
    let rating: String
}
```

นี่คือโครงสร้างชื่อ `Movie` ซึ่งประกอบด้วยสามฟิลด์คือ `name`, `director` และ `rating` โดยแทนชื่อภาพยนตร์ ผู้กำกับ และคะแนนตามลำดับ

ด้วยวิธีนี้ ข้อมูลที่เกี่ยวข้องกับภาพยนตร์หนึ่งเรื่องก็ไม่จำเป็นต้องถูกแยกไปเก็บไว้ในหลาย array อีกต่อไป แต่สามารถรวมเข้าด้วยกันเป็นชุดเดียวได้โดยตรง

ตัวอย่างเช่น เราสามารถสร้างอินสแตนซ์ของภาพยนตร์แบบนี้ได้:

```swift
Movie(name: "The Shawshank Redemption",
      director: "Frank Darabont",
      rating: "9.3 / 10"
	  )
```

อินสแตนซ์นี้แทน “ข้อมูลภาพยนตร์ที่สมบูรณ์หนึ่งชุด”

### เก็บโครงสร้างไว้ใน array

ก่อนหน้านี้เราได้บอกไว้ว่า array เก็บข้อมูลได้เพียงชนิดเดียว

ตอนนี้เราได้มีโครงสร้าง `Movie` แล้ว ดังนั้น array ก็สามารถเก็บ `Movie` ได้หลายตัว

```swift
let lists: [Movie] = []
```

ตัวอย่างเช่น:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
	Movie(name: "Forrest Gump",
        director: "Robert Zemeckis",
        rating: "8.8 / 10"),
	Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
	Movie(name: "The Pursuit of Happyness",
        director: "Gabriele Muccino",
        rating: "8.0 / 10"),
	Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

ใน array `lists` นี้ สิ่งที่เก็บอยู่ไม่ใช่สตริง แต่เป็นอินสแตนซ์ `Movie` หลายตัว

นั่นหมายความว่าแต่ละ element ใน array คือภาพยนตร์หนึ่งเรื่องที่สมบูรณ์

ดังนั้น เมื่อต้องการแสดงรายการภาพยนตร์ เราจึงสามารถใช้ `ForEach` เพื่ออ่านแต่ละเรื่องได้ทีละเรื่อง

```swift
ForEach(lists, id: \.self) { movie in
    HStack(spacing: 20) {
        Image(movie.name)
            .resizable()
            .scaledToFit()
            .frame(height: 180)
            .cornerRadius(10)
        
        VStack(alignment: .leading,spacing: 10) {
            Text(movie.name)
                .font(.title3)
                .fontWeight(.bold)
            Divider()
            VStack(alignment: .leading) {
                Text("director")
                    .fontWeight(.bold)
                Text(movie.director)
            }
            Divider()
            VStack(alignment: .leading) {
                Text("rating")
                    .fontWeight(.bold)
                Text(movie.rating)
            }
        }
        .frame(width: 200)
    }
}
```

ตรงนี้ `movie in` หมายความว่า ในแต่ละรอบของลูป จะมีการหยิบภาพยนตร์หนึ่งเรื่องออกมาจาก array และตั้งชื่อชั่วคราวว่า `movie`

จากนั้นเราก็สามารถใช้:

```swift
movie.name
movie.director
movie.rating
```

เพื่ออ่านชื่อ ผู้กำกับ และคะแนนของภาพยนตร์เรื่องนี้ทีละส่วน และนำไปแสดงในหน้าจอ

## ข้อผิดพลาดของ `ForEach`

ถึงตรงนี้ เราได้ทำรูปแบบพื้นฐานของ array ภาพยนตร์และ `ForEach` เสร็จแล้ว

แต่ถ้ารันโค้ดต่อไปนี้โดยตรง:

```swift
ForEach(lists,id: \.self) { movie in
	// ...
}
```

จะมีข้อความผิดพลาดขึ้นว่า:

```swift
Generic struct 'ForEach' requires that 'Movie' conform to 'Hashable'
```

ความหมายของข้อผิดพลาดนี้คือ เมื่อคุณเขียน `id: \.self` นั้น SwiftUI จำเป็นต้องใช้แต่ละ element ใน array เป็น “ตัวระบุที่ไม่ซ้ำกัน” เพื่อแยกความแตกต่างของแต่ละรายการ

รูปแบบการเขียนนี้เหมาะกับชนิดข้อมูลเรียบง่ายอย่าง `String` และ `Int` เพราะตัวมันเองแยกความแตกต่างได้ค่อนข้างง่าย

แต่ `Movie` เป็นโครงสร้างแบบกำหนดเองที่เราสร้างขึ้น `ForEach` จึงไม่รู้ว่าจะใช้ตัวมันเองเป็นตัวระบุเฉพาะอย่างไร จึงเกิดข้อผิดพลาดขึ้น

วิธีแก้ปัญหาที่พบบ่อยที่สุดก็คือ ทำให้ `Movie` ปฏิบัติตามโปรโตคอล `Identifiable`

## โปรโตคอล

ใน Swift โปรโตคอลสามารถเข้าใจได้ว่าเป็นชนิดหนึ่งของ “กฎ” หรือ “ข้อกำหนด”

คุณอาจมองมันเหมือนข้อตกลงขององค์การระหว่างประเทศ: หากประเทศหนึ่งต้องการเข้าร่วมองค์การระหว่างประเทศ ก็มักจะต้องปฏิบัติตามกฎบางอย่างที่องค์การนั้นกำหนด เปิดเผยข้อมูลบางส่วน และปฏิบัติตามกฎร่วมกันบางอย่าง ก่อนจะเข้าร่วมหรือมีส่วนร่วมในกิจการบางอย่างได้

โปรโตคอลก็ทำงานด้วยหลักการคล้ายกัน

เมื่อชนิดข้อมูลหนึ่งต้องการปฏิบัติตามโปรโตคอลใดโปรโตคอลหนึ่ง ก็ต้องทำให้ตรงตามสิ่งที่โปรโตคอลนั้นกำหนดไว้ ก่อนจึงจะสามารถใช้ความสามารถที่เกี่ยวข้องได้

รูปแบบพื้นฐาน:

```swift
struct Movie: Identifiable {
    // ...
}
```

ตรงนี้ `: Identifiable` หมายความว่า `Movie` ปฏิบัติตามโปรโตคอล `Identifiable`

ถ้าชนิดข้อมูลหนึ่งปฏิบัติตามหลายโปรโตคอล ก็สามารถคั่นด้วยเครื่องหมายจุลภาคได้:

```swift
struct Movie: Identifiable, Codable {
    // ...
}
```

สำหรับ `Identifiable` แล้ว ข้อกำหนดที่สำคัญที่สุดคือ ชนิดข้อมูลนั้นต้องมี `id` ที่ใช้ระบุตัวเองได้

ตัวอย่างเช่น:

```swift
struct Movie: Identifiable {
    let id: Int
    let name: String
    let director: String
    let rating: String
}
```

ตรงนี้ `id` ก็เหมือนหมายเลขบัตรประจำตัว ที่ใช้แยกความแตกต่างของภาพยนตร์แต่ละเรื่อง

ตราบใดที่ `id` ของภาพยนตร์แต่ละเรื่องไม่ซ้ำกัน SwiftUI ก็จะสามารถแยกแต่ละ element ใน array ได้อย่างถูกต้อง

ตัวอย่างเช่น:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 2,
          name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(id: 3,
          name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(id: 4,
          name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(id: 5,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

จากนั้นใช้ใน `ForEach`:

```swift
ForEach(lists, id: \.id) { movie in
    // ...
}
```

ด้วยวิธีนี้ก็จะไม่เกิดข้อผิดพลาดแล้ว

เนื่องจาก `Movie` ได้ปฏิบัติตาม `Identifiable` อยู่แล้ว รูปแบบที่พบได้บ่อยกว่าคือการละ `id:` ไปเลย:

```swift
ForEach(lists) { movie in
    // ...
}
```

เพราะ SwiftUI รู้แล้วว่าต้องใช้ `movie.id` เพื่อแยกรายการแต่ละรายการ

## ตัวระบุแยกแยะของ `ForEach`

ต่อไป เราจะทำความเข้าใจบทบาทของ `id` ใน `ForEach` ให้ลึกขึ้น

ตัวอย่างเช่น:

```swift
ForEach(lists, id: \.self)
```

ตรงนี้หมายถึง: ใช้ตัว element เองเป็นตัวระบุแยกแยะ

ส่วน:

```swift
ForEach(lists, id: \.id)
```

หมายถึง: ใช้ฟิลด์ `id` ของ element เป็นตัวระบุแยกแยะ

หากฟิลด์ใดฟิลด์หนึ่งสามารถรับประกันความไม่ซ้ำได้ด้วยตัวเอง ก็สามารถนำมาใช้ชั่วคราวได้เช่นกัน

ตัวอย่างเช่น หากชื่อภาพยนตร์ทุกเรื่องต่างกัน รูปแบบการเขียนต่อไปนี้ก็อาจทำงานได้ปกติภายใต้ข้อมูลปัจจุบัน:

```swift
ForEach(lists, id: \.name)
```

แต่ตรงนี้มีปัญหาที่ซ่อนอยู่คือ `name` ไม่จำเป็นต้องไม่ซ้ำตลอดไป

สมมุติว่าในอนาคตคุณเพิ่มภาพยนตร์อีกเรื่องที่มีชื่อเดียวกันเข้ามา แบบนั้น `name` ก็จะไม่สามารถแยกแต่ละ element ได้อย่างแม่นยำอีกต่อไป

ในเวลานั้น แม้โค้ดจะยัง compile ได้ แต่เมื่อมีการรีเฟรช อัปเดต แทรก หรือลบ view, SwiftUI อาจจดจำผิด ทำให้เกิดการแสดงผลที่ผิดปกติ

ดังนั้น เมื่อต้องการแยกข้อมูลอย่างมั่นคง ควรใช้ฟิลด์ที่ไม่ซ้ำกันจริง ๆ ก็คือ `id`

## UUID

แม้ว่าการเขียน `id: 1`, `id: 2` ด้วยมือจะช่วยแก้ปัญหาได้ แต่ก็ยังมีความเสี่ยงอยู่: อาจเผลอเขียน `id` ซ้ำ และทำให้ตัวระบุเฉพาะใช้งานไม่ได้

ตัวอย่างเช่น:

```swift
let lists: [Movie] = [
    Movie(id: 1,
          name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(id: 1,
          name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

แบบนี้ก็จะเกิด `id` ที่ซ้ำกันสองตัว และตัวระบุเฉพาะก็จะไม่ทำงาน

เพื่อหลีกเลี่ยงความผิดพลาดจากการเขียนด้วยมือ เรามักจะใช้ `UUID()`

ใน Swift, `UUID()` จะสร้างตัวระบุขนาด 128 บิตแบบสุ่ม โดยปกติจะปรากฏเป็นข้อความยาว ๆ ที่ประกอบด้วยตัวอักษรและตัวเลข

ตัวอย่างเช่น:

```swift
let uuid = UUID()    // "3F2504E0-4F89-11D3-9A0C-0305E82C3301"
```

ตรงนี้ `UUID()` จริง ๆ แล้วคือการสร้างอินสแตนซ์ของชนิด `UUID` ซึ่งคุณสามารถเข้าใจได้ว่าเป็น “หมายเลขเฉพาะ” ที่สร้างขึ้นโดยอัตโนมัติ

ดังนั้นค่าที่ถูกสร้างขึ้นแต่ละครั้งจึงแทบจะไม่ซ้ำกันเลย จึงเหมาะมากสำหรับใช้เป็น `id`

เราสามารถแก้ `Movie` ให้เป็นแบบนี้:

```swift
struct Movie: Identifiable {
    let id = UUID()
    let name: String
    let director: String
    let rating: String
}
```

โค้ดนี้หมายความว่า ฟิลด์ `id` ในโครงสร้าง `Movie` จะได้รับค่าจากอินสแตนซ์ `UUID` ใหม่โดยค่าเริ่มต้น

นั่นคือ ทุกครั้งที่เราสร้าง `Movie` ใหม่ ระบบจะช่วยสร้าง `id` ที่ไม่ซ้ำให้โดยอัตโนมัติก่อน

เมื่อ `id` มีค่าเริ่มต้นอยู่แล้ว ตอนที่เราสร้างอินสแตนซ์ `Movie` ทีหลังก็ไม่จำเป็นต้องกรอก `id` ด้วยมืออีก

ตัวอย่างเช่น:

```swift
let lists: [Movie] = [
    Movie(name: "The Shawshank Redemption",
          director: "Frank Darabont",
          rating: "9.3 / 10"),
    Movie(name: "Forrest Gump",
          director: "Robert Zemeckis",
          rating: "8.8 / 10"),
    Movie(name: "3 Idiots",
          director: "Rajkumar Hirani",
          rating: "8.4 / 10"),
    Movie(name: "The Pursuit of Happyness",
          director: "Gabriele Muccino",
          rating: "8.0 / 10"),
    Movie(name: "Dead Poets Society",
          director: "Peter Weir",
          rating: "8.1 / 10")
]
```

แม้ตรงนี้จะไม่ได้เขียน `id` แต่จริง ๆ แล้วอินสแตนซ์ภาพยนตร์นี้ก็ยังมี `id` ของตัวเอง เพียงแต่ค่านั้นถูกสร้างขึ้นโดย `UUID()` อัตโนมัติแล้ว

เมื่อใช้วิธีนี้ เราก็ไม่จำเป็นต้องกรอก `id` ให้กับภาพยนตร์แต่ละเรื่องด้วยมืออีก ซึ่งไม่เพียงลดปริมาณโค้ด แต่ยังช่วยหลีกเลี่ยงข้อผิดพลาดจากการกรอก id ซ้ำด้วย

สุดท้าย เราเพียงเพิ่มรูป `Banner` ไว้ด้านบน และใช้ `ScrollView` เพื่อให้เนื้อหาสามารถเลื่อนได้ เท่านี้ก็จะได้หน้ารายการภาพยนตร์ที่สมบูรณ์

## สรุป

ในบทเรียนนี้ เราได้เรียนรู้หัวข้อที่สำคัญมากหัวข้อหนึ่งคือ `struct`

ผ่าน `struct` เราสามารถรวมหลายฟิลด์ที่เกี่ยวข้องของภาพยนตร์หนึ่งเรื่องให้กลายเป็นหนึ่งเดียว แทนที่จะเก็บชื่อ ผู้กำกับ และคะแนนแยกไว้ในหลาย array

เมื่อรวมข้อมูลเสร็จแล้ว เราก็สามารถใช้ `ForEach` เพื่อแสดงภาพยนตร์แต่ละเรื่องใน array ออกมาทีละเรื่องใน view ได้

ในขณะเดียวกัน เราก็ยังได้เข้าใจบทบาทของ “ตัวระบุแยกแยะ” ใน `ForEach` ด้วย SwiftUI จำเป็นต้องรู้ว่าจะแยกแต่ละ element ใน array อย่างไร จึงจะสามารถแสดงและอัปเดต view ได้อย่างถูกต้อง

ดังนั้น เราจึงทำให้ `Movie` ปฏิบัติตามโปรโตคอล `Identifiable` และให้ `id` ที่ไม่ซ้ำกับมัน

เพื่อหลีกเลี่ยงข้อผิดพลาดเวลาใส่ `id` ด้วยมือ เราจึงได้เรียนรู้ `UUID()` ต่อไป เพื่อให้ระบบสร้างตัวระบุเฉพาะให้โดยอัตโนมัติ

หลังจากเรียนบทนี้จบ คุณไม่ได้เพียงแค่สร้างรายการภาพยนตร์เสร็จเท่านั้น แต่ยังเริ่มสัมผัสแนวคิดสำคัญอย่างหนึ่งใน SwiftUI ด้วย: **จัดระเบียบข้อมูลก่อน แล้วจึงสร้าง view ตามข้อมูลนั้น**

## โค้ดเต็ม

```swift
import SwiftUI

struct Movie: Identifiable {
    var id = UUID()
    let name: String
    let director: String
    let rating: String
}

struct ContentView: View {
    let lists: [Movie] = [
        Movie(name: "The Shawshank Redemption",
              director: "Frank Darabont",
              rating: "9.3 / 10"),
        Movie(name: "Forrest Gump",
              director: "Robert Zemeckis",
              rating: "8.8 / 10"),
        Movie(name: "3 Idiots",
              director: "Rajkumar Hirani",
              rating: "8.4 / 10"),
        Movie(name: "The Pursuit of Happyness",
              director: "Gabriele Muccino",
              rating: "8.0 / 10"),
        Movie(name: "Dead Poets Society",
              director: "Peter Weir",
              rating: "8.1 / 10")
    ]
    var body: some View {
        ScrollView {
            VStack(spacing:50) {
                Image("banner")
                    .resizable()
                    .scaledToFit()
                ForEach(lists) { movie in
                    HStack(spacing: 20) {
                        Image(movie.name)
                            .resizable()
                            .scaledToFit()
                            .frame(height: 180)
                            .cornerRadius(10)
                        
                        VStack(alignment: .leading,spacing: 10) {
                            Text(movie.name)
                                .font(.title3)
                                .fontWeight(.bold)

                            Divider()

                            VStack(alignment: .leading) {
                                Text("director")
                                    .fontWeight(.bold)
                                Text(movie.director)
                            }

                            Divider()

                            VStack(alignment: .leading) {
                                Text("rating")
                                    .fontWeight(.bold)
                                Text(movie.rating)
                            }
                        }
                        .frame(width: 200)
                    }
                }
            }
        }
    }
}
```
