# ทำให้ปุ่มเริ่มทำงาน

บทเรียนนี้จะเรียนรู้เกี่ยวกับปุ่ม Button, การใช้ print เพื่อแสดงข้อมูลสำหรับการดีบัก, และฟังก์ชันของ Swift

เนื้อหาเหล่านี้เป็นพื้นฐานของการเขียนโปรแกรมแบบโต้ตอบใน SwiftUI

## ปุ่ม

ปุ่มเป็นคอนโทรล UI ที่เราพบได้บ่อยที่สุด เมื่อเราต้องการดาวน์โหลด เปิด หรือออกจากบางอย่าง โดยทั่วไปเราจะกดปุ่ม

ตัวอย่างเช่น ใน App Store เมื่อกดปุ่ม “รับ” เพื่อดาวน์โหลดแอป

![Button](../../RESOURCE/008_button.png)

ใน SwiftUI ปุ่มใช้ `Button` แทน

รูปแบบพื้นฐาน:

```swift
Button("") {
    
}
```

โครงสร้างสามารถแบ่งออกเป็น 2 ส่วน:

```swift
Button("ข้อความบนปุ่ม") {
    โค้ดที่จะทำงานเมื่อกดปุ่ม
}
```

ตัวอย่าง:

```swift
struct ContentView: View {
    var body: some View {
        Button("Start") {

        }
    }
}
```

เมื่อรันแล้ว จะเห็นปุ่มชื่อ Start

![Button](../../RESOURCE/008_button1.png)

เมื่อผู้ใช้กดปุ่ม โค้ดภายใน `{}` จะถูกเรียกใช้งาน

สิ่งที่ต้องระวังคือ: `Button` เป็นคอนโทรลหน้าจอ (View) ของ SwiftUI ดังนั้นจึงต้องเขียนไว้ภายใน `body`

### ตัวปรับแต่ง buttonStyle

ใน SwiftUI, `Button` มีรูปแบบปุ่มที่ระบบเตรียมไว้ให้ ซึ่งสามารถใช้งานได้ผ่านตัวปรับแต่ง `buttonStyle`

ตัวอย่างเช่น:

```swift
Button("Start") {
    
}
.buttonStyle(.bordered)
```

`buttonStyle` จะใช้รูปแบบปุ่มที่ระบบมีให้

ตัวเลือกที่พบบ่อยมีดังนี้:

* `.automatic`
* `.plain`
* `.bordered`
* `.borderedProminent`
* `.borderless`

![Button](../../RESOURCE/008_button7.png)

ความแตกต่างของแต่ละรูปแบบส่วนใหญ่อยู่ที่ว่า ปุ่มมีกรอบหรือไม่ มีพื้นหลังหรือไม่ และมีการเน้นทางสายตามากน้อยเพียงใด

## การแสดงผลด้วย print

ในการเขียนโปรแกรม เรามักต้องการตรวจสอบว่าโปรแกรมทำงานมาถึงโค้ดช่วงใดแล้ว

Swift มีฟังก์ชัน `print` สำหรับแสดงข้อมูลดีบักออกมา

รูปแบบพื้นฐาน:

```swift
print("Hello")
```

โค้ดบรรทัดนี้จะแสดงข้อความออกไปยัง Console (คอนโซล)

### ทดสอบว่าปุ่มใช้งานได้หรือไม่

เราสามารถเขียน `print` ไว้ภายในปุ่ม เพื่อใช้ทดสอบว่าปุ่มถูกกดหรือไม่

```swift
Button("Start") {
    print("Test 123")
}
```

เมื่อกดปุ่ม Console จะแสดงผลว่า:

```
Test 123
```

นี่แสดงว่าปุ่มถูกเรียกใช้งานสำเร็จแล้ว

![Button](../../RESOURCE/008_button3.png)

ข้อควรระวัง: `print` เป็นโค้ดของ Swift ไม่ใช่คอนโทรลหน้าจอ จึงไม่สามารถวางไว้ใน `body` โดยตรงได้ ไม่เช่นนั้นจะเกิดข้อผิดพลาด

ตัวอย่างเช่น การเขียนแบบด้านล่างนี้ถือว่าไม่ถูกต้อง:

```swift
struct ContentView: View {
    var body: some View {
        print("Test 123")  // ผิด: body ต้องคืนค่าเป็น View
    }
}
```

ใน SwiftUI, `body` ใช้สำหรับอธิบายโครงสร้างของหน้าจอ จึงสามารถใส่ได้เฉพาะคอนโทรลหน้าจอ (View) เช่น `Text`, `Button`, `Image` เป็นต้น

โค้ดทั่วไป (เช่น ตรรกะการคำนวณ, `print` และอื่น ๆ) ต้องใส่ไว้ในปุ่ม ฟังก์ชัน หรือส่วนจัดการเหตุการณ์อื่น ๆ

### Console คอนโซล

ข้อความที่แสดงด้วย `print` จะไปปรากฏใน Console (คอนโซล) ของ Xcode

หากมองไม่เห็น Console สามารถกดปุ่ม Show the Debug Area ที่มุมขวาล่างเพื่อเปิดพื้นที่ดีบัก แล้วกด Show the Console เพื่อแสดงคอนโซล

![](../../RESOURCE/008_button4.png)

ที่มุมซ้ายล่างของ Console จะมี 2 แท็บคือ: `Executable` และ `Previews`

![](../../RESOURCE/008_button5.png)

สองแท็บนี้ตรงกับสภาพแวดล้อมการรันที่ต่างกัน: เมื่อใช้งานในหน้าพรีวิว Canvas ข้อความจาก `print` จะปรากฏในแท็บ `Previews`; แต่ถ้ารันแอปใน Simulator หรือบนเครื่องจริง ข้อความจะแสดงในแท็บ `Executable`

ดังนั้น หากพบว่า `print` ไม่มีผลลัพธ์ออกมา ก่อนอื่นให้ตรวจสอบก่อนว่าเราเลือกแท็บถูกต้องหรือไม่

## ตัวอย่าง - ASCII art

ในบางโปรเจกต์โค้ด เรามักจะเห็น ASCII art

ASCII art คือภาพที่ประกอบขึ้นจากตัวอักษรธรรมดา ตัวอย่างเช่น:

![ASCII](../../RESOURCE/008_ascii.png)

เราสามารถใช้ปุ่ม + `print` เพื่อแสดง ASCII art ได้

โค้ดตัวอย่าง:

```swift
struct ContentView: View {
    var body: some View {
        Button("Show ASCII") {
            print(" /\\_/\\")
            print("( o.o )")
            print(" > ^ <")
        }
        .buttonStyle(.borderedProminent)
    }
}
```

เมื่อกดปุ่ม Console จะแสดงผลว่า:

```
 /\_/\
( o.o )
 > ^ <
```

ข้อควรระวัง: เครื่องหมาย `\` ภายในสตริง ต้องเขียนเป็น `\\` เพราะเครื่องหมาย backslash ในสตริงเป็นอักขระสำหรับ escape

## ฟังก์ชัน

ภายในปุ่ม เราสามารถเขียนโค้ดได้โดยตรง

ตัวอย่างเช่น:

```swift
Button("Start") {
    print("Hello")
}
```

ถ้าโค้ดมีไม่มาก การเขียนแบบนี้ไม่มีปัญหา แต่ในการพัฒนาจริง ตรรกะที่ถูกเรียกจากปุ่มหนึ่งปุ่มอาจมีหลายบรรทัดมาก

ตัวอย่างเช่น:

```swift
Button("Start") {
    print("Hello")
    print("World")
    print("This")
    print("is")
    // อาจยังมีโค้ดอีกมาก
}
```

หากโค้ดมากขึ้นเรื่อย ๆ การเขียนไว้ภายในปุ่มโดยตรงจะทำให้โค้ดหน้าจอดูรก และอ่านหรือดูแลรักษาได้ยาก

ดังนั้น โดยทั่วไปเราจะย้ายโค้ดเหล่านี้ไปไว้ในฟังก์ชัน แล้วค่อยเรียกใช้ฟังก์ชันนั้นจากในปุ่ม

### ฟังก์ชันคืออะไร

ฟังก์ชันคือชุดโค้ดที่สามารถนำกลับมาใช้ซ้ำได้

เมื่อเราต้องการให้โค้ดบางส่วนทำงาน เราเพียงเรียกใช้ฟังก์ชันนั้น ก็จะสามารถรันโค้ดภายในฟังก์ชันได้

วิธีนี้ช่วยให้โครงสร้างของโค้ดชัดเจนขึ้น และยังสะดวกต่อการนำกลับมาใช้ซ้ำ

### รูปแบบพื้นฐาน

ใน Swift ฟังก์ชันใช้คีย์เวิร์ด **func** ในการประกาศ:

```swift
func randomInt() {
    // code
}
```

โค้ดนี้คือการประกาศฟังก์ชันหนึ่งตัว

`randomInt` คือชื่อของฟังก์ชัน ใช้เพื่อระบุชุดโค้ดนี้

`()` คือส่วนของพารามิเตอร์ ซึ่งสามารถใช้รับข้อมูลที่ส่งมาจากภายนอกได้ หากไม่ต้องการข้อมูล ก็ปล่อยว่างไว้ได้

ภายใน `{}` คือพื้นที่ของโค้ดในฟังก์ชัน โค้ดทั้งหมดที่ต้องการให้ทำงานจะเขียนไว้ตรงนี้

ตัวอย่างเช่น:

```swift
func getName() {
    print("FangJunyu")
}
```

หน้าที่ของฟังก์ชันนี้คือแสดงข้อความหนึ่งบรรทัดออกมา

### การเรียกใช้ฟังก์ชัน

การประกาศฟังก์ชันเป็นเพียงการสร้างชุดโค้ดนี้ขึ้นมาเท่านั้น หากต้องการให้มันทำงาน ยังต้องเรียกใช้ฟังก์ชันด้วย

วิธีเรียกใช้คือเขียนชื่อฟังก์ชันตามด้วย `()`:

```swift
getName()
```

เมื่อโปรแกรมทำงานมาถึงบรรทัดนี้ ก็จะรันโค้ดภายในฟังก์ชัน `getName`

ตัวอย่างสมบูรณ์:

```swift
func getName() {
    print("FangJunyu")
}

getName()
```

เมื่อรันแล้ว Console จะแสดงผลว่า:

```
FangJunyu
```

### การใช้ฟังก์ชันใน SwiftUI

ใน SwiftUI ฟังก์ชันมักจะเขียนไว้นอก `body`

ตัวอย่างเช่น:

```swift
struct ContentView: View {

    func getName() {
        print("FangJunyu")
    }

    var body: some View {
        Button("getName") {
            getName()
        }
    }
}
```

เมื่อกดปุ่ม ก็จะเรียก `getName()`

วิธีนี้ช่วยให้โค้ดหน้าจอดูเรียบง่าย ส่วนตรรกะจริงค่อยจัดการไว้ในฟังก์ชัน

### พารามิเตอร์ของฟังก์ชัน

บางครั้ง ฟังก์ชันหนึ่งตัวจำเป็นต้องทำงานแตกต่างกันตามข้อมูลที่ได้รับ

ตัวอย่างเช่น เราต้องการให้ฟังก์ชันแสดงชื่อที่ต่างกันได้ ไม่ใช่แค่แสดง `"FangJunyu"` แบบตายตัว

ในกรณีนี้จึงสามารถใช้พารามิเตอร์ได้ โดยพารามิเตอร์สามารถเข้าใจได้ว่าเป็น: ข้อมูลที่ส่งเข้าไปตอนเรียกใช้ฟังก์ชัน

ตัวอย่างเช่น:

```swift
func getName(name: String) {
    print(name)
}
```

ฟังก์ชันนี้เพิ่มพารามิเตอร์หนึ่งตัว โดย `name` คือชื่อพารามิเตอร์ และ `String` คือชนิดข้อมูลของพารามิเตอร์

นั่นหมายความว่าฟังก์ชันนี้ต้องรับข้อมูลชนิด `String`

เนื่องจากตอนนี้ฟังก์ชันต้องใช้ข้อมูล ดังนั้นตอนเรียกใช้จึงต้องส่งข้อมูลนี้เข้าไปด้วย

```swift
getName(name: "Sam")
```

เมื่อเรียกใช้โดยส่ง `"Sam"` เข้าไป ภายในฟังก์ชันก็จะใช้ค่านี้ในการแสดงผล

ผลลัพธ์ที่ได้:

```
Sam
```

## ตัวอย่าง - ใช้ฟังก์ชันแสดง ASCII art

ในตัวอย่างก่อนหน้านี้ เราเขียน `print` ไว้ภายในปุ่มโดยตรง

แต่ถ้า ASCII art ซับซ้อนมากขึ้น โค้ดก็จะยาวขึ้นตามไปด้วย ตอนนี้เราจึงสามารถย้ายตรรกะไปไว้ในฟังก์ชัน เพื่อให้โค้ดหน้าจอดูชัดเจนขึ้น

โค้ดตัวอย่าง:

```swift
struct ContentView: View {

    func printCat() {
        print(" /\\_/\\")
        print("( o.o )")
        print(" > ^ <")
    }

    var body: some View {
        Button("Show ASCII") {
            printCat()
        }
        .buttonStyle(.borderedProminent)
    }
}
```

เมื่อรันแล้วกดปุ่ม Console จะแสดงผลว่า:

```
 /\_/\
( o.o )
 > ^ <
```

ในที่นี้มี 2 สิ่งเกิดขึ้น เมื่อผู้ใช้กดปุ่ม:

1. ปุ่มเรียกใช้ฟังก์ชัน `printCat()`
2. โค้ด `print` ภายในฟังก์ชันถูกดำเนินการ

วิธีนี้ทำให้โค้ดหน้าจอมีหน้าที่เพียงแค่เรียกเหตุการณ์ ส่วนตรรกะจริงจะถูกจัดการภายในฟังก์ชัน

ในการพัฒนาจริง การทำงานของปุ่มส่วนใหญ่ก็มักจะถูกเขียนผ่านฟังก์ชันแบบนี้

## สรุป

ในบทเรียนนี้ เราได้เรียนรู้ 3 เรื่องสำคัญ:

**1. ปุ่ม Button**

`Button` คือคอนโทรลใน SwiftUI ที่ใช้สำหรับกระตุ้นการทำงานบางอย่าง

```swift
Button("Start") {

}
```

เมื่อผู้ใช้กดปุ่ม โค้ดภายในวงเล็บปีกกาจะถูกเรียกใช้งาน

**2. การแสดงผลด้วย print**

`print` ใช้สำหรับแสดงข้อมูลดีบักออกไปยัง Console

```swift
print("Hello")
```

นักพัฒนาสามารถดูสถานะการทำงานของโปรแกรมผ่าน Console ได้

**3. ฟังก์ชัน**

ฟังก์ชันคือชุดโค้ดที่สามารถนำกลับมาใช้ซ้ำได้

```swift
func sayHello() {
    print("Hello")
}
```

การเรียกใช้ฟังก์ชัน:

```swift
sayHello()
```

หากฟังก์ชันต้องการข้อมูล สามารถใช้ พารามิเตอร์ ได้:

```swift
func sayHello(name: String) {
    print(name)
}
```

ในบทเรียนถัดไป เราจะเรียนรู้คอนโทรล SwiftUI เพิ่มเติม รวมถึงวิธีที่ข้อมูลทำงานร่วมกับหน้าจอ

## แบบฝึกหัด

ลองทำแบบฝึกหัดต่อไปนี้:

1. สร้างปุ่มหนึ่งปุ่ม โดยให้ข้อความบนปุ่มเป็น `"Hello"`
2. เมื่อกดปุ่ม ให้แสดง `Hello Swift` ใน Console
3. ย้ายโค้ดแสดงผลไปไว้ในฟังก์ชัน แล้วค่อยเรียกใช้ฟังก์ชันนั้นจากในปุ่ม
