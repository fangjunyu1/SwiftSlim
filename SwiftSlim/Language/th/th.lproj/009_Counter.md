# ตัวนับ

ในสองบทเรียนก่อนหน้านี้ เราได้เรียนรู้เรื่องตัวแปร ค่าคงที่ ปุ่ม และเมธอดไปแล้ว

ในบทเรียนนี้ เราจะนำความรู้เหล่านี้มารวมกัน เพื่อสร้างแอปตัวนับอย่างง่าย

ในตัวอย่างนี้ เราจะเรียนรู้วิธีเก็บค่าและเปลี่ยนค่าของตัวแปร รวมถึงการใช้ `@State` เพื่อจัดการตัวแปร

## สร้างตัวนับ

ตัวนับมักใช้สำหรับบันทึกจำนวนที่ต้องเพิ่มขึ้นเรื่อย ๆ เช่น จำนวนครั้งของการกระโดดเชือก จำนวนรอบการวิ่ง เป็นต้น

ต่อไปเราจะสร้างตัวนับอย่างง่าย: แสดงตัวเลขหนึ่งตัวและปุ่มหนึ่งปุ่ม เมื่อกดปุ่มแล้วตัวเลขจะเพิ่มขึ้น

![Num](../../RESOURCE/009_num.png)

### แสดงตัวเลข

ก่อนอื่น เราใช้ `Text` เพื่อแสดงตัวเลข

```swift
Text("0")
```

แต่ถ้าตัวเลขต้องมีการเปลี่ยนแปลง เราไม่ควรเขียน `"0"` แบบตายตัวไว้ตรง ๆ แต่ควรใช้ตัวแปรเก็บตัวเลขแทน

```swift
var num = 0
```

จากนั้นแสดงตัวแปรใน `Text`:

```swift
Text("\(num)")
```

ที่นี่เราใช้ string interpolation `\()` เพื่อแปลงตัวเลขเป็นสตริงแล้วนำมาแสดงผล

### ปุ่มเพิ่มตัวเลข

ถัดไป เราเพิ่มปุ่มหนึ่งปุ่ม

```swift
Button("+") {

}
.buttonStyle(.borderedProminent)
```

เมื่อปุ่มถูกกด เราต้องการให้ตัวเลขเพิ่มขึ้น

ดังนั้นเราจึงสามารถแก้ไขค่าตัวแปรภายในปุ่มได้:

```swift
Button("+") {
    num = num + 1
}
.buttonStyle(.borderedProminent)
```

โค้ดนี้หมายความว่า: เมื่อกดปุ่ม จะคำนวณ `num + 1` แล้วนำผลลัพธ์ไปกำหนดค่าให้ `num`

### วิธีเขียนที่กระชับกว่า

Swift มีรูปแบบการเขียนที่กระชับกว่านี้:

```swift
num += 1
```

ซึ่งมีความหมายเท่ากับ:

```swift
num = num + 1
```

รูปแบบนี้เรียกว่า “ตัวดำเนินการกำหนดค่าแบบผสม” (compound assignment operator)

ตัวดำเนินการกำหนดค่าแบบผสมที่พบบ่อยใน Swift มีดังนี้:

```
+=   กำหนดค่าจากการบวก
-=   กำหนดค่าจากการลบ
*=   กำหนดค่าจากการคูณ
/=   กำหนดค่าจากการหาร
%=   กำหนดค่าจากการหารเอาเศษ
```

ตัวอย่างเช่น:

```swift
num -= 2   // num = num - 2
num *= 3   // num = num * 3
```

### โค้ดทั้งหมด

ตอนนี้เราสามารถเขียนตัวนับอย่างง่ายได้แล้ว:

```swift
struct ContentView: View {
    var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

แต่ในตอนนี้ โค้ดยังไม่สามารถทำงานได้ตามปกติ

### เกิดข้อผิดพลาด

Xcode จะแจ้งว่า:

```swift
Cannot assign to property: 'self' is immutable
```

ซึ่งหมายความว่าไม่สามารถแก้ไขพร็อพเพอร์ตีได้ เพราะวิวนี้ไม่สามารถแก้ไขค่าได้

แล้วทำไมถึงเป็นแบบนี้?

## หลักการทำงานของ SwiftUI

ใน SwiftUI: **หน้าจอถูกขับเคลื่อนด้วยข้อมูล** ซึ่งหมายความว่าเมื่อข้อมูลเปลี่ยน หน้าจอก็จะรีเฟรชโดยอัตโนมัติ

แต่มีข้อสำคัญคือ SwiftUI จะคอยติดตามการเปลี่ยนแปลงและรีเฟรชมุมมอง ก็ต่อเมื่อเป็นการเปลี่ยนแปลงของสถานะ (เช่น `@State`, `@Binding`) เท่านั้น

ถ้าใช้ตัวแปรธรรมดา SwiftUI จะอ่านค่าของตัวแปรเพียงครั้งเดียวตอนที่สร้างวิว หลังจากนั้นถึงแม้ว่าค่าตัวแปรจะเปลี่ยน หน้าจอก็จะไม่อัปเดต

ตัวอย่างเช่น:

```swift
var num = 0   // 0
num = 1   // 0
```

ในตัวอย่างด้านบน `num` เป็นตัวแปรธรรมดา แม้ว่าค่าของมันจะเปลี่ยนไปแล้ว แต่ SwiftUI จะไม่อัปเดตวิว ดังนั้นหน้าจอยังคงแสดงค่าเริ่มต้นเป็น 0

![Num](../../RESOURCE/009_state.png)

ดังนั้น เฉพาะเมื่อข้อมูลถูกทำเครื่องหมายให้เป็นสถานะเท่านั้น (เช่น ใช้ property wrapper อย่าง `@State`, `@Binding`) SwiftUI จึงจะคอยตรวจจับการเปลี่ยนแปลงและอัปเดตวิวโดยอัตโนมัติ

## ตัวห่อคุณสมบัติ @State

ใน SwiftUI หากต้องการแก้ไขค่าตัวแปรและให้อัปเดตหน้าจอด้วย จะต้องใช้ `@State`

```swift
@State var num = 0
```

`@State` จะทำให้ SwiftUI คอยติดตามตัวแปรนี้

เมื่อค่าตัวแปรเปลี่ยน SwiftUI จะคำนวณวิวใหม่ และอัปเดตหน้าจอ

โค้ดทั้งหมด:

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack {
			Text("\(num)")
        	Button("+") {
        	    num += 1
        	}
        	.buttonStyle(.borderedProminent)
		}
    }
}
```

ตอนนี้เมื่อกดปุ่ม:

```swift
num += 1
```

SwiftUI จะตรวจพบว่า `num` มีการเปลี่ยนแปลง แล้วอัปเดตวิวให้อัตโนมัติ

![Num](../../RESOURCE/009_state1.png)

## กฎการใช้งาน @State

ในการพัฒนาจริง `@State` มักจะยึดตาม 2 ข้อนี้

### 1. โดยทั่วไป @State จะใช้ร่วมกับ private

ตัวแปร `@State` มักใช้ภายในประเภทปัจจุบันเท่านั้น (เช่น ภายในโครงสร้าง `View` ปัจจุบัน) ดังนั้นจึงมักเขียนเป็น:

```swift
@State private var num = 0
```

วิธีนี้จะช่วยป้องกันไม่ให้วิวอื่นเข้าถึงหรือแก้ไขตัวแปรนี้โดยตรง

ตอนนี้เรายังไม่ได้เรียน `private` อย่างเป็นทางการ สามารถเข้าใจแบบง่าย ๆ ได้ว่า:

**สิ่งที่ถูกกำกับด้วย `private` จะใช้งานได้เฉพาะภายในวิวปัจจุบันเท่านั้น วิวภายนอกจะไม่สามารถใช้งานได้**

### 2. @State ใช้สำหรับสถานะของวิวปัจจุบันเท่านั้น

`@State` เหมาะสำหรับข้อมูลที่ `View` ปัจจุบันเป็นผู้จัดการเอง

ตัวอย่างเช่น เนื้อหาในช่องกรอกข้อมูล หรือสถานะของสวิตช์

```swift
@State private var isOn = false
@State private var text = ""
```

ถ้าข้อมูลจำเป็นต้องถูกแชร์ระหว่างหลายวิว จะต้องใช้สถานะประเภทอื่น เช่น `@Binding` หรือ `@Observable`

เนื้อหาเหล่านี้จะเรียนในบทเรียนถัด ๆ ไป

### ปุ่มลดค่า

ตัวนับไม่ได้มีแค่การเพิ่มค่า แต่ยังสามารถลดค่าได้ด้วย

เราสามารถเพิ่มปุ่ม `-` ได้

```swift
Button("-") {
    num -= 1
}
.buttonStyle(.borderedProminent)
```

จากนั้นใช้ `HStack` เพื่อจัดเรียงปุ่มในแนวนอน:

```swift
HStack {
    Button("+") {
        num += 1
    }
    .buttonStyle(.borderedProminent)
    Button("-") {
        num -= 1
    }
    .buttonStyle(.borderedProminent)
}
```

ผลลัพธ์:

![Num](../../RESOURCE/009_num1.png)

เมื่อกดปุ่ม `+` ตัวเลขจะเพิ่มขึ้น 1; เมื่อกดปุ่ม `-` ตัวเลขจะลดลง 1

### ปุ่มรีเซ็ต

เรายังสามารถเพิ่มปุ่มรีเซ็ต เพื่อให้ตัวเลขกลับเป็น 0 ได้

```swift
Button("0") {
    num = 0
}
.buttonStyle(.borderedProminent)
```

เมื่อกดปุ่มรีเซ็ต `num` จะถูกกำหนดค่าเป็น 0

ผลลัพธ์:

![Num](../../RESOURCE/009_num2.png)

เมื่อกดปุ่ม `0` ตัวเลขจะกลายเป็น 0

### ปุ่มแบบกำหนดเอง

ตอนนี้ปุ่มของเรายังแสดงได้แค่ข้อความ:

```swift
Button("+") {

}
```

ปุ่มในรูปแบบนี้จะแสดงได้เฉพาะข้อความเท่านั้น เช่น `+` หรือ `-`

แต่ในการพัฒนาจริง เรามักต้องการให้ปุ่มมีหน้าตาหลากหลายมากขึ้น เช่น ใช้ไอคอนหรือวิวอื่น ๆ

SwiftUI อนุญาตให้เรากำหนดเนื้อหาที่แสดงบนปุ่มเองได้ ซึ่งในกรณีนี้สามารถใช้รูปแบบการเขียนอีกแบบของปุ่ม:

```swift
Button(action: {

}, label: {

})
```

ในรูปแบบนี้ `action` คือโค้ดที่จะทำงานเมื่อกดปุ่ม ส่วน `label` คือวิวที่ใช้แสดงบนปุ่ม

ตัวอย่างเช่น เราสามารถทำให้ปุ่มแสดงไอคอนจาก SF Symbols ได้:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
})
```

แบบนี้ปุ่มก็จะไม่แสดงข้อความอีกต่อไป แต่จะแสดงเป็นไอคอนแทน

### ปรับปรุงหน้าจอตัวนับ

เพื่อให้หน้าจอตัวนับดูชัดเจนขึ้น เราสามารถปรับรูปแบบการแสดงตัวเลขได้ เช่น ทำให้ตัวเลขใหญ่ขึ้น:

```swift
Text("\(num)")
    .font(.largeTitle)
    .fontWeight(.bold)
```

ต่อไป เราจะเปลี่ยนเนื้อหาที่แสดงบนปุ่มให้เป็นไอคอน และขยายขนาดไอคอนให้เหมาะสม:

```swift
Button(action: {
    num += 1
}, label: {
    Image(systemName: "plus.app")
        .imageScale(.large)
})
```

ในทำนองเดียวกัน เราก็สามารถเปลี่ยนปุ่มลดค่าและปุ่มรีเซ็ตให้เป็นไอคอนได้เช่นกัน:

```swift
Button(action: {
    num -= 1
}, label: {
    Image(systemName: "minus.square")
        .imageScale(.large)
})

Button(action: {
    num = 0
}, label: {
    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
        .imageScale(.large)
})
```

ตอนนี้ปุ่มจะแสดงเป็นไอคอนของ SF Symbols และหน้าจอก็จะดูเข้าใจง่ายขึ้นด้วย

![Num](../../RESOURCE/009_num3.png)

## สรุป

บทเรียนนี้เราได้ฝึกใช้ตัวแปร `Text` และ `Button` ร่วมกัน ผ่านการสร้างตัวนับอย่างง่าย เราใช้ตัวแปรเพื่อเก็บตัวเลข และใช้ปุ่มเพื่อควบคุมให้ตัวเลขเพิ่มหรือลด พร้อมทั้งได้เรียนรู้ตัวดำเนินการกำหนดค่าแบบผสม เช่น `num += 1` และ `num -= 1`

ระหว่างกระบวนการสร้าง เราได้เข้าใจวิธีทำงานของ SwiftUI: หน้าจอถูกขับเคลื่อนด้วยข้อมูล เมื่อข้อมูลเปลี่ยน หน้าจอก็จะรีเฟรช ดังนั้นเราจึงต้องใช้ `@State` เพื่อเก็บสถานะที่เปลี่ยนแปลงได้ เมื่อค่าของตัวแปร `@State` เปลี่ยน SwiftUI ก็จะอัปเดตวิวโดยอัตโนมัติ

สุดท้าย เราได้ปรับแต่งหน้าจออย่างง่าย ด้วยการเปลี่ยนรูปแบบตัวอักษรของ `Text` และใช้ `Button(action:label:)` เพื่อกำหนดเนื้อหาของปุ่มเอง ทำให้ปุ่มสามารถแสดงไอคอนได้ ส่งผลให้หน้าจอตัวนับดูชัดเจนและสวยงามมากขึ้น

ตอนนี้เราได้เริ่มสัมผัสวิธีการใช้งานพื้นฐานของ SwiftUI View แล้ว ใน SwiftUI หน้าจอส่วนใหญ่ถูกขับเคลื่อนด้วยตัวแปร เมื่อค่าตัวแปรเปลี่ยน SwiftUI จะอัปเดตวิวโดยอัตโนมัติ ดังนั้น ในการพัฒนาแอป SwiftUI เรามักต้องออกแบบก่อนว่ามีข้อมูลอะไรบ้างที่ต้องเก็บ แล้วค่อยใช้ข้อมูลเหล่านั้นมาสร้างหน้าจอและตรรกะการโต้ตอบ

## โค้ดทั้งหมด

```swift
struct ContentView: View {
    @State var num = 0
    var body: some View {
        VStack(spacing: 20) {
            Text("\(num)")
                .font(.largeTitle)
                .fontWeight(.bold)
            HStack {
                Button(action: {
                    num += 1
                }, label: {
                    Image(systemName: "plus.app")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num -= 1
                }, label: {
                    Image(systemName: "minus.square")
                        .imageScale(.large)
                })
                
                Button(action: {
                    num = 0
                }, label: {
                    Image(systemName: "arrow.trianglehead.2.clockwise.rotate.90")
                        .imageScale(.large)
                })
            }
        }
    }
}
```
