# การจัดการไฟล์ Xcode

ในบทเรียนนี้ เราจะเรียนรู้การทำงานกับไฟล์ที่พบบ่อยใน Xcode

ในบทเรียนก่อนหน้านี้ เราเขียนโค้ดส่วนใหญ่ไว้ใน `ContentView` วิธีนี้ช่วยให้สังเกตโค้ดและผลลัพธ์ที่แสดงได้ง่าย

ตัวอย่างเช่น:

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

แต่ใน App จริง โดยปกติเราจะไม่เขียนโค้ดทั้งหมดไว้ในไฟล์เดียว

เมื่อหน้า คอมโพเนนต์ และฟังก์ชันมีมากขึ้น เราจำเป็นต้องแยก View ต่าง ๆ ออกเป็นไฟล์ SwiftUI คนละไฟล์

ตัวอย่างเช่น:

```swift
ContentView.swift
CardView.swift
AuthorView.swift
```

วิธีนี้ทำให้โครงสร้างโค้ดชัดเจนขึ้น และสะดวกต่อการดูแลและแก้ไขในภายหลัง

## ทำไมต้องจัดการไฟล์

ใน SwiftUI หน้าเพจหรือคอมโพเนนต์หนึ่ง ๆ โดยปกติสามารถเขียนเป็นไฟล์ View แยกต่างหากได้

ตัวอย่างเช่น:

```swift
struct CardView: View {
    var body: some View {
        Text("Card")
    }
}
```

ในที่นี้ `CardView` คือ View ของ SwiftUI

หากเขียน View ทั้งหมดไว้ใน `ContentView` โค้ดจะยาวขึ้นเรื่อย ๆ และอ่านได้ยากขึ้น

ดังนั้น เราจึงต้องเรียนรู้การสร้างไฟล์ การเปลี่ยนชื่อไฟล์ การลบไฟล์ รวมถึงการใช้โฟลเดอร์เพื่อจัดการไฟล์โค้ดต่าง ๆ

## สร้างไฟล์ SwiftUI

ใน Xcode การสร้างไฟล์ SwiftUI โดยทั่วไปมีสองวิธีที่พบบ่อย:

วิธีหนึ่งคือสร้างไฟล์เปล่า แล้วเขียนโค้ดเอง

อีกวิธีหนึ่งคือสร้างไฟล์จากเทมเพลต SwiftUI โดยให้ Xcode สร้างโค้ดพื้นฐานให้โดยอัตโนมัติ

สำหรับผู้เริ่มต้น แนะนำให้ใช้ไฟล์เทมเพลต SwiftUI มากกว่า เพราะมันจะสร้างโครงสร้างพื้นฐานให้โดยอัตโนมัติ

### สร้างไฟล์เปล่า

ในพื้นที่ Navigator ทางด้านซ้ายของ Xcode ให้คลิกขวาที่พื้นที่ว่างหรือโฟลเดอร์ แล้วเลือก `New Empty File`

ตำแหน่งการทำงานมีดังนี้:

![xcode](../../../Resource/027_view1.png)

หลังจากสร้างไฟล์เปล่าแล้ว โดยปกติ Xcode จะสร้างไฟล์ Swift ที่ยังไม่ได้ตั้งชื่อ เช่น `Untitled.swift`

เราสามารถเปลี่ยนชื่อเป็น `TestView.swift` ได้

ในโปรเจกต์ SwiftUI หากไฟล์นี้ใช้สำหรับแสดง View โดยทั่วไปแนะนำให้เติม `View` ไว้ท้ายชื่อ

ตัวอย่างเช่น:

```swift
HomeView.swift
CardView.swift
ProfileView.swift
```

นี่ไม่ใช่กฎบังคับ แต่ช่วยให้เรามองออกได้ทันทีว่านี่คือไฟล์ View

หลังจากสร้างไฟล์เปล่าแล้ว ภายในไฟล์อาจมีเพียงคอมเมนต์เริ่มต้น หรืออาจเป็นไฟล์ว่างก็ได้ ขึ้นอยู่กับเวอร์ชันของ Xcode และวิธีการสร้าง

```swift
//
//  TestView.swift
//  SwiftSlimTest
//
//  Created by Fang Junyu on 2026/5/12.
//
```

ตอนนี้ เราต้องเขียนโค้ด SwiftUI ด้วยตนเอง:

```swift
import SwiftUI

struct TestView: View {
    var body: some View {
        Text("This is Test")
    }
}

#Preview {
    TestView()
}
```

เพียงเท่านี้ ไฟล์ View ของ SwiftUI แบบง่าย ๆ ก็ถูกสร้างเรียบร้อยแล้ว

### เปลี่ยนชื่อไฟล์

หากหลังจากสร้างไฟล์แล้วต้องการแก้ไขชื่อไฟล์ ให้เลือกไฟล์ในพื้นที่ Navigator ทางด้านซ้ายของ Xcode แล้วกดปุ่ม Return `⏎`

![xcode](../../../Resource/027_view2.png)

จากนั้นก็สามารถแก้ไขชื่อไฟล์ได้

สิ่งที่ต้องระวังคือ การเปลี่ยนชื่อไฟล์จะไม่เปลี่ยนชื่อ View ในโค้ดโดยอัตโนมัติ

ตัวอย่างเช่น หลังจากเปลี่ยนชื่อไฟล์เป็น `ProfileView.swift` แล้ว ในโค้ดอาจยังคงเป็น `struct TestView: View`

แม้ว่าสิ่งนี้ไม่จำเป็นต้องทำให้เกิด error แต่ก็ทำให้สับสนได้ง่าย

ดังนั้น แนะนำให้ตั้งชื่อไฟล์และชื่อ View ให้สอดคล้องกัน เช่น หากชื่อไฟล์คือ `ProfileView.swift` ชื่อ View ก็ควรใช้ `ProfileView` เช่นกัน

## สร้างไฟล์เทมเพลต SwiftUI

นอกจากการสร้างไฟล์เปล่าแล้ว เรายังสามารถสร้างไฟล์เทมเพลต SwiftUI ได้โดยตรง

ในพื้นที่ Navigator ทางด้านซ้ายของ Xcode ให้คลิกขวาที่พื้นที่ว่างหรือโฟลเดอร์ แล้วเลือก `New File from Template...`

ตำแหน่งการทำงานมีดังนี้:

![xcode](../../../Resource/027_view3.png)

ในหน้าต่างที่แสดงขึ้นมา ด้านบนสามารถเลือกแพลตฟอร์มต่าง ๆ ได้ เช่น `iOS`、`macOS` เป็นต้น

ตรงนี้ให้คงการเลือกแพลตฟอร์มที่ตรงกับโปรเจกต์ปัจจุบันไว้ ตัวอย่างเช่น หากเรากำลังสร้าง iOS App ก็เลือก `iOS`

![xcode](../../../Resource/027_view4.png)

จากนั้นเลือก `SwiftUI View` ที่ด้านล่าง

![xcode](../../../Resource/027_view12.png)

จากนั้นคลิก `Next` ใส่ชื่อไฟล์ แล้วบันทึก

ตัวอย่างเช่น เราสร้างไฟล์ `Test2View.swift`

Xcode จะสร้างโค้ดลักษณะคล้ายด้านล่างให้โดยอัตโนมัติ:

```swift
import SwiftUI

struct Test2View: View {
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    Test2View()
}
```

เมื่อเทียบกับไฟล์เปล่า ไฟล์เทมเพลตจะมี `import SwiftUI`、`struct`、`body` และ `#Preview` มาให้โดยอัตโนมัติ

ดังนั้นสำหรับผู้เริ่มต้น การใช้ไฟล์เทมเพลต SwiftUI จะง่ายกว่า และลดโอกาสลืมเขียนโค้ดได้มากกว่า

**คำแนะนำ**: นอกจากการคลิกขวาเพื่อสร้างไฟล์แล้ว ยังสามารถคลิกปุ่มเครื่องหมายบวกที่ด้านล่างของ Navigator ทางด้านซ้ายของ Xcode เพื่อสร้างไฟล์หรือโฟลเดอร์ได้เช่นกัน

![xcode](../../../Resource/027_view5.png)

## สร้างโฟลเดอร์

เมื่อไฟล์มีจำนวนมากขึ้น เราสามารถใช้โฟลเดอร์เพื่อจัดการได้

ในพื้นที่ Navigator ทางด้านซ้ายของ Xcode ให้คลิกขวาที่พื้นที่ว่างหรือโฟลเดอร์ แล้วเลือก `New Folder`

หลังจากสร้างโฟลเดอร์แล้ว สามารถลากไฟล์ที่เกี่ยวข้องเข้าไปในโฟลเดอร์ได้

![xcode](../../../Resource/027_view7.png)

โฟลเดอร์ใช้เพื่อจัดระเบียบโครงสร้างโปรเจกต์เป็นหลัก และจะไม่เปลี่ยนวิธีการใช้งานของ View เอง

ตราบใดที่ชื่อ View ถูกต้อง ในไฟล์อื่นก็ยังสามารถใช้งานผ่านรูปแบบเช่น `TestView()`、`Test2View()` ได้เหมือนเดิม

## ลบไฟล์หรือโฟลเดอร์

หากต้องการลบไฟล์หรือโฟลเดอร์ ให้เลือกไฟล์ที่ต้องการในพื้นที่ Navigator ทางด้านซ้ายของ Xcode จากนั้นคลิกขวาแล้วเลือก `Delete`

หรือเลือกไฟล์แล้วกดปุ่ม Delete `⌫` ก็ได้

Xcode จะแสดงหน้าต่างยืนยันการลบ

![xcode](../../../Resource/027_view10.png)

การเลือก `Move to Trash` หมายถึงย้ายไฟล์ไปยังถังขยะ

หากเห็น `Remove Reference` หมายถึงลบเฉพาะการอ้างอิงออกจากโปรเจกต์ Xcode แต่จะไม่ลบไฟล์ต้นฉบับบนดิสก์

สำหรับผู้เริ่มต้น หากเป็นเพียงการลบไฟล์ทั่วไปที่เพิ่งสร้างเอง โดยปกติเลือก `Move to Trash` ก็เพียงพอแล้ว

## แสดง View ของ SwiftUI หลายตัว

หลังจากเรียนรู้การสร้างไฟล์แล้ว เราสามารถแยก View ต่าง ๆ ออกเป็นไฟล์ต่าง ๆ ได้

ตัวอย่างเช่น เราสร้างไฟล์ SwiftUI สองไฟล์:

```swift
AuthorView.swift
CardView.swift
```

`AuthorView` ใช้สำหรับแสดงชื่อผู้เขียน:

```swift
import SwiftUI

struct AuthorView: View {
    var body: some View {
        Text("Fang Junyu")
            .font(.title3)
            .fontWeight(.bold)
            .foregroundStyle(Color.white)
    }
}
```

`CardView` ใช้สำหรับแสดงพื้นหลังของการ์ด:

```swift
import SwiftUI

struct CardView: View {
    var body: some View {
        RoundedRectangle(cornerRadius: 10)
            .fill(Color.indigo.opacity(0.8))
            .frame(width: 200, height: 50)
    }
}
```

จากนั้น เราสามารถใช้ View ทั้งสองนี้ใน `ContentView` ได้:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        ZStack {
            CardView()
            AuthorView()
        }
    }
}

#Preview {
    ContentView()
}
```

หลังจากรันแล้ว จะเห็นว่า `CardView` และ `AuthorView` ถูกนำมาประกอบแสดงร่วมกัน:

![xcode](../../../Resource/027_view11.png)

ในผลลัพธ์นี้ `CardView` ทำหน้าที่เป็นพื้นหลัง ส่วน `AuthorView` แสดงอยู่ด้านบน เพราะทั้งสองถูกวางไว้ใน `ZStack`

ใน `ContentView` เราใช้ View ทั้งสองผ่าน `CardView()` และ `AuthorView()` โดย `()` ที่อยู่หลังชื่อหมายถึงการสร้างและใช้งาน View นี้

นี่คือการซ้อน View ที่พบบ่อยใน SwiftUI: ภายใน View หนึ่ง สามารถใช้งาน View อื่นต่อได้

## ไฟล์จุดเริ่มต้น

สุดท้าย เรามาทบทวนกันว่า App เริ่มแสดงหน้าจอจากที่ใด

ในโปรเจกต์ SwiftUI โดยปกติจะมีไฟล์จุดเริ่มต้นหนึ่งไฟล์:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

ในที่นี้ `@main` หมายความว่านี่คือตำแหน่งจุดเริ่มต้นของ App

ใน `WindowGroup` เราจะเห็น:

```swift
ContentView()
```

นี่หมายความว่า หลังจาก App เริ่มทำงาน จะเริ่มแสดง `ContentView` ก่อน

จากนั้นใน `ContentView` ก็แสดงสิ่งต่อไปนี้อีก:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ดังนั้นลำดับชั้นของ View สามารถเข้าใจได้ดังนี้:

```swift
SwiftSlimTestApp
    - ContentView
        - CardView
        - AuthorView
```

หากเราเปลี่ยน View ที่แสดงในไฟล์จุดเริ่มต้นเป็น `CardView`:

```swift
import SwiftUI

@main
struct SwiftSlimTestApp: App {
    var body: some Scene {
        WindowGroup {
            CardView()
        }
    }
}
```

ดังนั้นหลังจาก App เริ่มทำงาน ก็จะแสดงเฉพาะ `CardView`

กล่าวคือ `WindowGroup` ในโค้ดจุดเริ่มต้นเป็นตัวกำหนดว่า หลังจาก App เริ่มทำงานแล้ว View ใดจะแสดงเป็นอันดับแรก

และภายใน View นี้ ก็สามารถแสดง View อื่นต่อได้อีก

## สรุป

ในบทเรียนนี้ เราได้เรียนรู้การจัดการไฟล์ที่พบบ่อยใน Xcode

เราได้รู้วิธีสร้างไฟล์ SwiftUI วิธีเปลี่ยนชื่อไฟล์ วิธีสร้างโฟลเดอร์ และวิธีลบไฟล์

ในขณะเดียวกัน เรายังได้เรียนรู้แนวคิดสำคัญอย่างหนึ่ง: View ของ SwiftUI สามารถแยกออกเป็นไฟล์ต่าง ๆ แล้วนำไปใช้ใน View อื่นได้

ตัวอย่างเช่น:

```swift
ZStack {
    CardView()
    AuthorView()
}
```

ในที่นี้ `CardView()` และ `AuthorView()` คือการใช้งาน View อื่นภายใน `ContentView`

สุดท้าย เรายังได้ทบทวนตรรกะการแสดงผลของไฟล์จุดเริ่มต้น

เมื่อ App เริ่มทำงาน จะเข้าสู่โค้ดจุดเริ่มต้นที่มีเครื่องหมาย `@main` ก่อน จากนั้นจึงแสดง View ที่อยู่ใน `WindowGroup`

สิ่งนี้ช่วยให้เราเข้าใจว่า ไฟล์ SwiftUI ต่าง ๆ ไม่ได้แยกขาดจากกัน แต่สามารถประกอบเข้าด้วยกัน และสุดท้ายกลายเป็นหน้าจอ App ที่สมบูรณ์ได้
