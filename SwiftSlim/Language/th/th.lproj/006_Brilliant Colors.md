# สีสันที่โดดเด่น

ในบทนี้ เราจะเรียนรู้ modifier ด้านภาพที่พบบ่อยใน SwiftUI ซึ่งประกอบด้วย:

- สี
- สีพื้นหน้า
- สีพื้นหลัง
- การเลื่อนตำแหน่ง
- ความโปร่งใส
- ความเบลอ

นอกจากนี้ยังจะได้เรียนรู้เรื่อง Safe Area (พื้นที่ปลอดภัย)

modifier เหล่านี้ใช้สำหรับควบคุมลักษณะการแสดงผลของมุมมอง เพื่อให้อินเทอร์เฟซชัดเจนและมีลำดับชั้นมากขึ้น

## สี

ใน SwiftUI เราสามารถกำหนดสีของข้อความได้

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

.blue หมายถึงสีน้ำเงิน ซึ่งจริง ๆ แล้วเป็นรูปแบบย่อของ Color.blue (อาศัย type inference)

สีที่พบบ่อยมีดังนี้:

```id="w7s9a1"
.black
.green
.yellow
.pink
.gray
...
```

ทั้งหมดนี้เป็น static property ของ Color

![Color](../../RESOURCE/006_color.png)

คุณสามารถมองว่า Color คือชนิดข้อมูลของสี ส่วน .blue และ .red คือสีจริงที่อยู่ภายใต้ชนิดนี้

### มุมมอง Color

ใน SwiftUI, Color ก็สามารถใช้แสดงเป็นมุมมองได้เช่นกัน

```swift
Color.red
    .frame(width: 100, height:100)
    .cornerRadius(10)
```

![Color](../../RESOURCE/006_color5.png)

โค้ดส่วนนี้จะสร้างสี่เหลี่ยมสีแดงขนาด 100×100

นอกจากนี้ยังสามารถทำให้ทั้งหน้าจอแสดงเป็นสีใดสีหนึ่งได้:

```swift
struct ContentView: View {
    var body: some View {
        Color.red
    }
}
```

![Color](../../RESOURCE/006_color1.png)

เมื่อรันแล้ว จะเห็นว่าสีแดงไม่ได้ปูเต็มทั้งหน้าจอ พื้นที่ด้านบนและด้านล่างของ iPhone ยังคงเป็นสีขาว ซึ่งตรงนี้เกี่ยวข้องกับการมีอยู่ของ Safe Area (พื้นที่ปลอดภัย)

## Safe Area (พื้นที่ปลอดภัย)

Safe Area คือพื้นที่ที่ระบบเผื่อไว้เพื่อหลีกเลี่ยงไม่ให้เนื้อหาถูกบัง ซึ่งรวมถึง:

1. แถบสถานะด้านบน (เวลา, แบตเตอรี่)

2. แถบ Home indicator ด้านล่าง

3. บริเวณรอยบาก (notch) หรือ Dynamic Island

![Color](../../RESOURCE/006_color3.png)

โดยค่าเริ่มต้น SwiftUI จะจำกัดเนื้อหาให้อยู่ภายในพื้นที่ปลอดภัย ดังนั้นมุมมองจึงไม่ยื่นไปถึงขอบหน้าจอ

### ละเลย Safe Area (พื้นที่ปลอดภัย)

ถ้าต้องการให้สีปูเต็มทั้งหน้าจอ สามารถใช้ ignoresSafeArea:

```swift
Color.red
    .ignoresSafeArea()
```

หรือใช้ edgesIgnoringSafeArea:

```swift
Color.red
    .edgesIgnoringSafeArea(.all)
```

![Color](../../RESOURCE/006_color4.png)

แบบนี้มุมมองก็จะยืดไปเต็มพื้นที่ทั้งหน้าจอ

สิ่งที่ควรสังเกตคือ edgesIgnoringSafeArea เป็นรูปแบบการเขียนที่เก่ากว่า และตั้งแต่ iOS 14 เป็นต้นมา แนะนำให้ใช้ ignoresSafeArea

## สีพื้นหน้า

### modifier foregroundStyle

ในบทก่อนหน้า เราได้เรียนรู้การใช้ foregroundStyle เพื่อกำหนดสีมาแล้ว

ตัวอย่างเช่น:

```swift
Text("Fang Junyu")
    .foregroundStyle(.blue)
```

foregroundStyle เป็นระบบ style แบบใหม่ ที่รองรับสี, ไล่ระดับสี และวัสดุ (material) เป็นต้น

![Color](../../RESOURCE/006_color2.png)

### modifier foregroundColor

foregroundColor ก็สามารถใช้กำหนดสีได้เช่นกัน:

```swift
Text("Fang Junyu")
    .foregroundColor(.blue)
```

วิธีใช้งานเหมือนกับ foregroundStyle

ใน Xcode เวอร์ชันใหม่ ๆ Xcode จะมีคำเตือนว่า foregroundColor อาจถูกเลิกใช้ใน iOS เวอร์ชันอนาคต ดังนั้นจึงแนะนำให้ใช้ foregroundStyle ก่อนเป็นลำดับแรก

## พื้นหลัง

ถ้าต้องการเพิ่มสีพื้นหลังให้มุมมอง สามารถใช้ background:

```swift
background(.red)
```

ตัวอย่างเช่น เพิ่มสีพื้นหลังให้ข้อความ:

```swift
Text("FangJunyu.com")
    .background(.orange)
```

![Color](../../RESOURCE/006_color6.png)

เวลาเราเลือกข้อความในแอปอย่าง Word หรือ Chrome ก็จะเห็นเอฟเฟกต์สีพื้นหลังลักษณะคล้ายกันนี้

![Color](../../RESOURCE/006_color16.png)

ถ้าต้องการขยายพื้นที่พื้นหลัง ต้องใช้ร่วมกับ padding:

```swift
Text("FangJunyu.com")
    .padding(10)
    .background(Color.orange)
```

![Color](../../RESOURCE/006_color7.png)

ตรงนี้ต้องจำกฎสำคัญข้อหนึ่งไว้:

modifier ของ SwiftUI จะสร้างมุมมองจากบนลงล่าง modifier ที่เขียนอยู่ด้านหลัง จะทำงานกับผลลัพธ์ที่เกิดจาก modifier ก่อนหน้า

ดังนั้น:

```swift
.padding()
.background()
```

หมายความว่าพื้นหลังจะครอบมุมมองหลังจากที่เพิ่มระยะขอบด้านในแล้ว

ถ้าสลับลำดับ:

```swift
Text("FangJunyu.com")
    .background(Color.orange)
    .padding(10)
```

![Color](../../RESOURCE/006_color6.png)

พื้นหลังจะไม่ขยาย เพราะพื้นหลังไม่ได้ครอบ padding ที่อยู่ด้านหลัง

## ตัวอย่าง - น็อตสี่เหลี่ยมสี่มุม

ตอนนี้เราจะสร้างมุมมองน็อตสี่เหลี่ยมสี่มุมแบบง่าย ๆ

![Color](../../RESOURCE/006_color8.png)

ก่อนอื่น สร้างสี่เหลี่ยมสีขาวขนาด 50 × 50:

```swift
Color.white
    .frame(width: 50, height: 50)
```

ถ้าต้องการให้กลายเป็นวงกลม สามารถใช้ cornerRadius:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
```

เมื่อรัศมีมุมโค้งเท่ากับครึ่งหนึ่งของความกว้างและความสูง มันก็จะกลายเป็นวงกลม

ตอนนี้เพิ่มพื้นหลังสีน้ำเงิน:

```swift
Color.white
    .frame(width: 50, height: 50)
    .cornerRadius(25)
    .padding(20)
    .background(.blue)
```

![Color](../../RESOURCE/006_color8.png)

padding ที่เพิ่มเข้ามาจะขยายพื้นที่ชั้นนอก และ background จะวาดสีน้ำเงินลงบนพื้นที่ชั้นนอกนั้น

ตอนนี้ก็ได้เอฟเฟกต์น็อตสี่เหลี่ยมสี่มุมเรียบร้อยแล้ว

### อีกแนวคิดหนึ่ง

นอกจากใช้สีพื้นหลังสร้างน็อตสี่เหลี่ยมสี่มุมแล้ว ยังสามารถใช้ ZStack เพื่อทำได้เช่นกัน

ก่อนหน้านี้เราได้เรียนรู้ว่า ZStack ใช้จัดวางแบบซ้อนทับ และน็อตสี่เหลี่ยมสี่มุมก็สามารถมองได้ว่าเป็นการซ้อนทับกันของวงกลมหนึ่งอันกับสี่เหลี่ยมหนึ่งอัน

```swift
ZStack {
    Color.blue
        .frame(width: 70, height: 70)
    Color.white
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color8.png)

ZStack จะซ้อนมุมมองตามลำดับที่เขียน โดยมุมมองที่เพิ่มทีหลังจะแสดงอยู่ชั้นบน

## ตัวอย่าง - วงกลมซ้อนกันสองวง

ไอคอนหลายแบบประกอบขึ้นจากการซ้อนรูปทรงง่าย ๆ เช่น วงกลมสองวงที่ซ้อนกัน

![Color](../../RESOURCE/006_color14.png)

ก่อนอื่น สร้างวงกลมสองวง:

```swift
Color.red
    .frame(width: 50, height:50)
    .cornerRadius(25)
Color.orange
    .frame(width: 50, height:50)
    .cornerRadius(25)
```

![Color](../../RESOURCE/006_color11.png)

เนื่องจากเราต้องการให้วงกลมสองวงแสดงแบบซ้อนกัน จึงใช้เลย์เอาต์ ZStack:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color12.png)

ตอนนี้วงกลมที่มีขนาดเท่ากันสองวงจะซ้อนทับกันทั้งหมด

เราต้องการให้วงกลมทั้งสองซ้อนกันเพียงบางส่วน ไม่ใช่ทับกันทั้งหมด จึงสามารถใช้ offset เพื่อสร้างเอฟเฟกต์การเลื่อนตำแหน่ง

## การเลื่อนตำแหน่ง

offset เปลี่ยนเฉพาะตำแหน่งในการวาดของมุมมอง และไม่กระทบต่อการคำนวณเลย์เอาต์ของมุมมองแม่

วิธีใช้:

```swift
.offset(x:y:)
```

x คือค่าการเลื่อนในแนวนอน ส่วน y คือค่าการเลื่อนในแนวตั้ง

ค่าบวกหมายถึงเลื่อนไปทางขวา / ลงล่าง และค่าลบหมายถึงเลื่อนไปทางซ้าย / ขึ้นบน

ใช้ offset เพื่อทำให้วงกลมสองวงซ้อนกันบางส่วน:

```swift
ZStack {
    Color.red
        .frame(width: 50, height: 50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height: 50)
        .cornerRadius(25)
}
```

![Color](../../RESOURCE/006_color13.png)

ตำแหน่งของวงกลมสีแดงในเลย์เอาต์ไม่ได้เปลี่ยนไป แต่ตำแหน่งในการวาดถูกเลื่อนไปทางซ้าย 25 จุด ดังนั้นจึงเกิดเอฟเฟกต์ที่วงกลมสองวงซ้อนกันบางส่วน

## ความโปร่งใส

ใน SwiftUI, opacity ใช้สำหรับกำหนดความโปร่งใสของมุมมอง

วิธีใช้พื้นฐาน:

```swift
.opacity(0.5)
```

ค่าพารามิเตอร์ของ opacity อยู่ในช่วง 0.0 ถึง 1.0 โดยที่:

* 0 หมายถึงโปร่งใสทั้งหมด
* 1 หมายถึงทึบทั้งหมด

เราสามารถใช้ opacity เพื่อกำหนดความโปร่งใสของวงกลมสีส้มได้:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x: -25)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
```

![Color](../../RESOURCE/006_color14.png)

เมื่อกำหนด opacity ของวงกลมสีส้มเป็น 0.8 ความทึบของมันจะเท่ากับ 80% และเมื่อวงกลมทั้งสองซ้อนกัน พื้นที่ที่ทับกันจะเกิดเอฟเฟกต์การผสมสี

## ความเบลอ

ใน SwiftUI สามารถใช้ blur เพื่อกำหนดเอฟเฟกต์ความเบลอได้:

```swift
.blur(radius:10)
```

radius คือรัศมีของความเบลอ ยิ่งค่ามาก เอฟเฟกต์เบลอก็จะยิ่งชัดเจน

เราสามารถเพิ่มเอฟเฟกต์เบลอให้วงกลมทั้งสองได้:

```swift
ZStack {
    Color.red
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .offset(x:-35)
    Color.orange
        .frame(width: 50, height:50)
        .cornerRadius(25)
        .opacity(0.8)
}
.blur(radius:10)
```

![Color](../../RESOURCE/006_color15.png)

สุดท้าย เราจะเห็นวงกลมสองวงที่เบลอมาก

## สรุป

บทนี้เน้นไปที่ modifier ด้านภาพที่พบบ่อยใน SwiftUI และเรียนรู้วิธีควบคุมสี ตำแหน่ง และเอฟเฟกต์ทางภาพของมุมมองผ่าน modifier

จากตัวอย่างที่เป็นรูปธรรม เราจะเห็นบทบาทจริงของ modifier ด้านภาพแต่ละตัวในอินเทอร์เฟซ และได้เรียนรู้เรื่องพื้นที่ปลอดภัยไปด้วย

ทั้งหมดนี้เป็น modifier พื้นฐานมาก ๆ เราสามารถฝึกใช้งานให้มากขึ้น ซึ่งจะช่วยให้ควบคุมผลลัพธ์ของอินเทอร์เฟซได้ชัดเจนยิ่งขึ้นในการพัฒนาจริง

### แบบฝึกหัดหลังเรียน

* เพิ่มเอฟเฟกต์ความโปร่งใสและความเบลอให้รูปภาพหนึ่งรูป
* ตั้งค่าวงกลมสามวงที่ซ้อนกันโดยมีค่าความโปร่งใสแตกต่างกัน
* สร้างภาพพื้นหลังที่ปูเต็มหน้าจอ และละเลยพื้นที่ปลอดภัย
* ใช้ offset ปรับตำแหน่งของมุมมองหลายตัว

จุดประสงค์ของการฝึกไม่ใช่การจำ API แต่คือการสังเกตความสัมพันธ์ระหว่างการเปลี่ยนแปลงทางภาพกับพฤติกรรมของเลย์เอาต์
