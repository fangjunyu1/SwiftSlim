# ระบบชนิดข้อมูล

ในบทเรียนก่อนหน้านี้ เราได้เรียนรู้เรื่องตัวแปรมาแล้ว และรู้ว่าตัวแปรสามารถเก็บข้อมูลได้หลายชนิด

ตัวอย่างเช่น:

```swift
var hello = "Hello, World"
```

ในที่นี้ hello เก็บข้อความหนึ่งช่วง ดังนั้นชนิดข้อมูลของมันจึงเป็น String

ต่อไป เราจะเรียนรู้แนวคิดของชนิดข้อมูล (Type) อย่างเป็นระบบ รวมถึงการประกาศชนิดข้อมูลแบบชัดเจนและการอนุมานชนิดข้อมูล เพื่อให้เข้าใจธรรมชาติของตัวแปรได้ลึกยิ่งขึ้น

## ชนิดข้อมูลคืออะไร

ใน Swift ค่าทุกค่าจะมีชนิดข้อมูลที่ชัดเจน ชนิดข้อมูลจะเป็นตัวกำหนดว่าค่านี้คืออะไร และสามารถทำอะไรได้บ้าง

ตัวอย่างเช่น:

* 42 คือ Int (จำนวนเต็ม)
* "Hello" คือ String (สตริง)
* true คือ Bool (บูลีน)

สิ่งเหล่านี้เป็นคนละชนิดข้อมูล และมีการใช้งานที่แตกต่างกัน

## ชนิดข้อมูลพื้นฐานที่พบบ่อยใน Swift

ในช่วงเริ่มต้น ชนิดข้อมูลที่พบบ่อยที่สุดมีดังนี้:

* String: สตริง (ข้อความ)
* Int: จำนวนเต็ม
* Double: จำนวนทศนิยม (มีจุดทศนิยม และมีความแม่นยำสูงกว่า)
* Bool: ค่าบูลีน (true/false)

นอกจากนี้ยังมี Float (จำนวนทศนิยม) แต่มีความแม่นยำน้อยกว่า Double จึงมักใช้น้อยกว่า รวมถึง Character (อักขระเดี่ยว) เช่น "A", "!" เป็นต้น

### การประกาศชนิดข้อมูล (Explicit Type)

ใน Swift เราสามารถระบุชนิดข้อมูลด้วยตนเองขณะประกาศตัวแปรได้:

```swift
var age: Int = 18
var price: Double = 9.9
var name: String = "FangJunyu"
var isLogin: Bool = false
```

ให้ใส่ ": ชนิดข้อมูล" ต่อท้ายชื่อตัวแปร เพื่อประกาศชนิดข้อมูลของตัวแปรนั้น

รูปแบบการเขียนชนิดข้อมูลของอาร์เรย์คือ ": [ชนิดข้อมูล]":

```swift
var scores: [Int] = [90, 85, 78]
var tags: [String] = ["Swift", "iOS"]
```

ในโค้ดนี้ อาร์เรย์ scores สามารถเก็บได้เฉพาะสมาชิกชนิด Int ส่วน tags สามารถเก็บได้เฉพาะสมาชิกชนิด String

การประกาศชนิดข้อมูลแบบชัดเจนช่วยให้เจตนาของโค้ดชัดขึ้น และในบางกรณียังช่วยหลีกเลี่ยงข้อผิดพลาดจากการอนุมานชนิดข้อมูลได้

## การอนุมานชนิดข้อมูล

ในหลายกรณี เราไม่จำเป็นต้องประกาศชนิดข้อมูลแบบชัดเจน:

```swift
var age = 18         // Int
var price = 9.9      // Double
var name = "Sam"     // String
var isLogin = false  // Bool
var scores = [90, 85, 78]    // [Int]
var tags = ["Swift", "iOS"]  // [String]
```

เหตุผลก็คือ Swift compiler จะอนุมานชนิดข้อมูลให้โดยอัตโนมัติตาม "ค่าเริ่มต้น"

## การประกาศแบบชัดเจนและการอนุมานชนิดข้อมูล

ในบทเรียนก่อนหน้านี้ เราไม่ได้เน้นการประกาศชนิดข้อมูลแบบชัดเจน เช่น:

```swift
var age: Int = 18
```

แต่เลือกใช้การอนุมานชนิดข้อมูลก่อน:

```swift
var age = 18
```

ในตัวอย่างนี้ ทั้งสองรูปแบบให้ผลเหมือนกัน กล่าวคือ age จะถูกกำหนดให้เป็นชนิด Int ทั้งคู่ เมื่อเปรียบเทียบกันแล้ว การอนุมานชนิดข้อมูลจะเขียนได้กระชับกว่า

เหตุผลที่ยังไม่เน้นการประกาศแบบชัดเจนในช่วงแรก ก็เพราะการอนุมานชนิดข้อมูลช่วยลดข้อมูลส่วนเกินในโค้ด จึงช่วยลดภาระในการทำความเข้าใจสำหรับผู้เริ่มต้น

## ทำไมจึงต้องมีชนิดข้อมูล

Swift เป็นภาษาแบบ strong type (Strongly Typed Language)

ซึ่งหมายความว่า: เมื่อตัวแปรถูกกำหนดชนิดข้อมูลแล้ว ก็จะไม่สามารถเปลี่ยนเป็นชนิดอื่นได้ตามใจ

ตัวอย่างเช่น:

```swift
var age: Int = 18

age = 20      // Success
age = "20"    // Error, Cannot assign value of type 'String' to type 'Int'

var nums: [Int] = [1, 2, 3, "Hello"]  // Error
```

20 ตัวแรกเป็นชนิด Int จึงสามารถกำหนดค่าให้ age ได้ ส่วน "20" ตัวที่สองเป็นชนิด String ซึ่งไม่ตรงกับ Int จึงทำให้เกิดข้อผิดพลาด

nums เป็นอาร์เรย์ชนิด [Int] จึงเก็บได้เฉพาะจำนวนเต็ม และไม่สามารถใส่สตริงปนเข้าไปได้

ชนิดข้อมูลทำหน้าที่จำกัดวิธีการใช้ข้อมูล และช่วยตรวจพบปัญหาการไม่ตรงกันของชนิดข้อมูลตั้งแต่ช่วง compile เพื่อลดความสับสนของชนิดข้อมูล ลด BUG และเพิ่มความเสถียรกับความสามารถในการดูแลรักษาโค้ด

## เมื่อใดที่จำเป็นต้องประกาศชนิดข้อมูลแบบชัดเจน

แม้ Swift จะอนุมานชนิดข้อมูลได้อัตโนมัติ แต่ก็มีบางกรณีที่ compiler อนุมานไม่ได้ หรืออนุมานผิด ซึ่งในกรณีเหล่านี้จำเป็นต้องระบุชนิดข้อมูลด้วยตนเอง

**1. อาร์เรย์ว่าง**

เมื่อต้องการสร้างอาร์เรย์ว่าง จำเป็นต้องประกาศชนิดข้อมูลแบบชัดเจน:

```swift
var nums: [Int] = []
```

ถ้าไม่ใส่ชนิดข้อมูล:

```swift
var nums = []   // Error，Empty collection literal requires an explicit type
```

compiler ไม่สามารถรู้ได้จากอาร์เรย์ว่างว่า สมาชิกภายในควรเป็นชนิดใด จึงเกิดข้อผิดพลาดขณะ compile

**2. ป้องกันการอนุมานผิด**

```swift
var value = 10   // Int
```

ในตัวอย่างนี้ แม้ว่า 10 จะสามารถมองเป็นจำนวนทศนิยม (Double) ได้เช่นกัน แต่เพราะไม่มีจุดทศนิยม compiler จึงอนุมานให้เป็น Int โดยอัตโนมัติ

ถ้าต้องการให้ value เป็นจำนวนทศนิยม ต้องประกาศชนิดข้อมูลแบบชัดเจน:

```swift
var value: Double = 10   // Double
```

เมื่อผลจากการอนุมานชนิดข้อมูลไม่ตรงกับที่ต้องการ ควรใช้การประกาศชนิดข้อมูลแบบชัดเจน หรือปรับรูปแบบ literal เพื่อให้ชนิดข้อมูลถูกต้อง

**3. ชนิดข้อมูลแบบกำหนดเอง**

ในภายหลังเราจะได้เรียนรู้เรื่องชนิดข้อมูลแบบกำหนดเอง และเมื่อใช้ชนิดข้อมูลเหล่านี้ ก็มักจำเป็นต้องประกาศชนิดข้อมูลแบบชัดเจนเช่นกัน:

```swift
var info: GameInfo
var users: [User] = []
```

เหตุผลเหมือนกับกรณีอาร์เรย์ว่าง คือ compiler ไม่สามารถอนุมานชนิดข้อมูลได้จากค่าที่ว่างเปล่า

## การแปลงชนิดข้อมูล

ใน Swift ชนิดข้อมูลที่ต่างกันจะไม่ถูกแปลงให้อัตโนมัติ จำเป็นต้องแปลงด้วยตนเอง

ตัวอย่างเช่น การบวกเลขสองค่าที่มีชนิดข้อมูลต่างกัน:

```swift
var a: Int = 10
var b: Double = 12.5

var result = a + b    // Error, Binary operator '+' cannot be applied to operands of type 'Int' and 'Double'
```

ในโค้ดนี้ a เป็นชนิด Int ส่วน b เป็นชนิด Double เนื่องจากชนิดข้อมูลต่างกัน จึงไม่สามารถนำมาคำนวณกันโดยตรงได้

ก่อนคำนวณ ต้องแปลงค่าตัวใดตัวหนึ่งให้เป็นชนิดเดียวกันก่อน:

```swift
var result = Double(a) + b
```

โค้ดนี้หมายถึงการแปลง a จากชนิด Int ให้เป็น Double ก่อน แล้วจึงนำไปบวกกับ b และเก็บผลลัพธ์ไว้ใน result

สิ่งที่ต้องระวังคือ การแปลงชนิดข้อมูลจะไม่เปลี่ยนชนิดของตัวแปรเดิม ตัวอย่างเช่น Double(a) ไม่ได้ทำให้ a กลายเป็น Double แต่เป็นการสร้างค่า Double ใหม่ขึ้นมาเพื่อใช้ในการคำนวณเท่านั้น

ชนิดข้อมูลอื่นก็สามารถแปลงในลักษณะเดียวกันได้ คือใช้รูปแบบ ชนิดเป้าหมาย + () แล้วส่งค่าที่ต้องการแปลงเข้าไป เช่น:

```
Int()、Double()、String()
```

ตัวอย่าง:

```swift
let score: Int = 95
let message = "Score: " + String(score)  // Int → String

let price: Double = 9.9
let intPrice = Int(price)  // Double → Int， 9
```

ควรสังเกตว่า ไม่ใช่ทุกการแปลงชนิดข้อมูลจะได้ผลลัพธ์ตามที่คาด เช่น Double → Int จะตัดส่วนทศนิยมทิ้งทันที และจะไม่ปัดเศษ

## นามแฝงของชนิดข้อมูล

ใน Swift เรายังสามารถตั้งชื่อใหม่ให้ชนิดข้อมูลเดิมได้ เพื่อให้ความหมายของโค้ดชัดเจนขึ้น:

```swift
typealias UserID = Int
typealias Score = Double

var id: UserID = 1001
var grade: Score = 98.5
```

ตรงนี้ UserID โดยพื้นฐานแล้วยังคงเป็น Int ชนิดข้อมูลจริงไม่ได้เปลี่ยนไป เพียงแต่ชื่อใหม่ทำให้ความหมายชัดเจนขึ้น

ในโปรเจกต์ที่มีโค้ดจำนวนมากหรือมีชนิดข้อมูลซับซ้อน typealias จะช่วยให้โค้ดอ่านง่ายขึ้น

## ตัวอย่างการใช้ชนิดข้อมูล

ด้านล่างนี้เป็นตัวอย่างง่าย ๆ เพื่อแสดงให้เห็นบทบาทของชนิดข้อมูลในการพัฒนาจริง

### สร้างตัวแปร

ก่อนอื่น เราสร้างตัวแปร age ขึ้นมา:

```swift
var age = 20
```

เนื่องจากเราต้องการแสดงและแก้ไขตัวแปรนี้ใน SwiftUI จึงต้องใช้ @State ในการประกาศ:

```swift
@State private var age = 20
```

เราเคยกล่าวถึง private มาก่อนแล้ว ว่าหมายถึงตัวแปรนี้สามารถใช้งานได้เฉพาะภายใน view ปัจจุบันเท่านั้น และเราจะเรียนรายละเอียดเพิ่มเติมในภายหลัง

### แสดงข้อมูลใน SwiftUI

แสดงตัวแปร age ใน SwiftUI:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
        }
    }
}
```

ตอนนี้ view จะแสดงผลเป็น:

```swift
age: 20
```

### แก้ไขข้อมูล

ต่อไป เราจะเพิ่มปุ่มสองปุ่มเพื่อแก้ไขค่า age:

```swift
Button("changeAge1") {
    age = 18
}
Button("changeAge2") {
    age = "18"
}
```

### ผลลัพธ์ที่ได้

เมื่อกด changeAge1 ค่า age จะถูกกำหนดเป็น 18:

```swift
age = 18
```

เนื่องจาก 18 เป็นชนิด Int และตรงกับชนิดของ age จึงกำหนดค่าได้สำเร็จ

ส่วน changeAge2 จะเกิดข้อผิดพลาด:

```swift
age = "18"
```

ข้อความผิดพลาดคือ:

```
Cannot assign value of type 'String' to type 'Int'
```

เหตุผลก็คือ "18" เป็นชนิด String (สตริงต้องอยู่ภายในเครื่องหมาย "") แต่ age เป็นชนิด Int ชนิดข้อมูลจึงไม่ตรงกันและไม่สามารถกำหนดค่าได้

### ทำไมจึงเกิดข้อผิดพลาด

จุดสำคัญอยู่ที่โค้ดบรรทัดนี้:

```swift
@State private var age = 20   // Int
```

แม้จะไม่ได้ประกาศชนิดข้อมูลแบบชัดเจน แต่เพราะค่าเริ่มต้น 20 เป็นชนิด Int compiler จึงอนุมานให้ age เป็น Int โดยอัตโนมัติ

เมื่อชนิดข้อมูลถูกกำหนดแล้ว ก็จะไม่สามารถกำหนดค่าเป็นชนิดอื่นได้อีก

และปุ่ม changeAge2 ไม่ได้ “ล้มเหลวตอนรันจริง” แต่จะเกิดข้อผิดพลาดตั้งแต่ช่วง compile ทำให้โค้ดไม่สามารถคอมไพล์ผ่านได้

นี่ก็เป็นหนึ่งในข้อดีของภาษา strong type อย่าง Swift: สามารถตรวจพบปัญหาชนิดข้อมูลได้ตั้งแต่ตอนเขียนโค้ด

### วิธีเขียนที่ถูกต้อง

เมื่อลบโค้ดที่ผิดออกแล้ว:

```swift
struct ContentView: View {
    @State private var age = 20
    var body: some View {
        VStack(spacing: 10) {
            Text("age: \(age)")
            Button("changeAge1") {
                age = 18
            }
        }
    }
}
```

เมื่อกดปุ่ม changeAge1 view จะอัปเดตเป็น:

```
age: 18
```

### การประกาศชนิดข้อมูลแบบชัดเจน (ไม่บังคับ)

เรายังสามารถระบุชนิดข้อมูลให้ชัดเจนขณะกำหนดค่าได้เช่นกัน:

```swift
@State private var age: Int = 20
```

รูปแบบนี้ทำให้ชนิดข้อมูลดูชัดเจนขึ้น แต่ในสถานการณ์ง่าย ๆ ก็ไม่จำเป็นเสมอไป

## สรุป

บทเรียนนี้อธิบายเรื่องระบบชนิดข้อมูลของ Swift และบทบาทของมันในการพัฒนาจริง

ใน Swift ค่าทุกค่าจะมีชนิดข้อมูลที่ชัดเจน ชนิดข้อมูลเป็นตัวกำหนดความหมายของข้อมูล และกำหนดว่าข้อมูลนั้นสามารถเข้าร่วมการทำงานแบบใดได้บ้าง

Swift เป็นภาษาแบบ strong type เมื่อตัวแปรถูกกำหนดชนิดข้อมูลแล้ว ก็จะไม่สามารถกำหนดค่าเป็นชนิดอื่นได้ ข้อจำกัดนี้ช่วยให้ตรวจพบข้อผิดพลาดได้ตั้งแต่ช่วง compile จึงลดปัญหาในช่วง runtime และเพิ่มความปลอดภัยของโค้ด

ชนิดข้อมูลของตัวแปรสามารถอนุมานได้จากค่าเริ่มต้น หรือจะประกาศแบบชัดเจนก็ได้ ในบางกรณี เช่น อาร์เรย์ว่าง การอนุมานชนิดข้อมูลผิด หรือชนิดข้อมูลแบบกำหนดเอง compiler จะไม่สามารถอนุมานได้อย่างถูกต้อง จึงจำเป็นต้องประกาศชนิดข้อมูลแบบชัดเจน

ชนิดข้อมูลที่ต่างกันจะไม่ถูกแปลงโดยอัตโนมัติ ต้องใช้ Int()、Double()、String() เป็นต้นในการแปลงแบบชัดเจน

ชนิดข้อมูลยังรองรับการตั้งชื่อนามแฝง โดยเฉพาะในโปรเจกต์ที่ซับซ้อน typealias จะช่วยเพิ่มความสามารถในการอ่านโค้ด

ชนิดข้อมูลไม่ได้เป็นเพียงป้ายกำกับของข้อมูลเท่านั้น แต่ยังเป็นกลไกในการควบคุม เพื่อให้แน่ใจว่าการใช้ข้อมูลนั้นปลอดภัย ชัดเจน และตรวจสอบได้

## ความรู้เพิ่มเติม - Optional

ระหว่างการแปลงชนิดข้อมูล บางการทำงานอาจล้มเหลวได้

ตัวอย่างเช่น การแปลง String เป็น Double:

```swift
let input = "3.14"
let pi = Double(input)  // Double?
```

ตรงนี้ Double(input) จะไม่ได้คืนค่าเป็น Double แต่เป็น:

```
Double?
```

นี่ก็คือ Optional

### ทำไมจึงคืนค่าเป็น Optional

เพราะเนื้อหาของสตริงนั้นไม่แน่นอน "3.14" สามารถแปลงเป็น Double ได้ แต่ "Hello" ไม่สามารถแปลงได้

กล่าวคือ การทำงานนี้อาจสำเร็จ หรืออาจล้มเหลวก็ได้

ดังนั้น Swift จึงใช้ Optional เพื่อแทนสถานะที่ไม่แน่นอนแบบนี้

### Optional คืออะไร

Optional หมายถึง: ค่าหนึ่งค่าอาจมีอยู่ หรืออาจไม่มีอยู่ก็ได้

ตัวอย่างเช่น:

```
Double?
```

ชนิดข้อมูลนี้หมายความว่า ถ้ามีค่า เวลาพิมพ์ออกมาจะเห็นเป็น Optional(...) แต่ถ้าไม่มีค่า ก็จะได้ nil

ควรระวังว่า nil หมายถึง "ไม่มีค่า" ไม่ได้หมายถึง 0 หรือสตริงว่าง

### ตัวอย่าง

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

ผลลัพธ์:

```
pi:Optional(3.14)
```

ซึ่งหมายความว่า สตริงถูกแปลงเป็น Double สำเร็จ ได้ค่า 3.14 แต่เพราะชนิดข้อมูลเป็น Double? จึงถูกห่อไว้ใน Optional

ถ้าแปลงไม่สำเร็จ:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi)")
```

ผลลัพธ์:

```
pi: nil
```

เพราะ "Hello" ไม่สามารถแปลงเป็นชนิด Double ได้ จึงแปลงไม่สำเร็จและคืนค่าเป็น nil

### Optional(...) คืออะไร

เมื่อเราพิมพ์ค่า Optional โดยตรง:

```swift
let input = "3.14"
let pi = Double(input)
print("pi:\(pi)")
```

Swift จะแสดงผลในรูปแบบ “คำอธิบายสำหรับการดีบัก”:

```
Optional(3.14)
```

นี่ไม่ใช่ข้อมูลจริงส่วนหนึ่งของค่า แต่เป็นการบอกว่าค่าปัจจุบันเป็น "ชนิด Optional" และภายในมีค่าถูกเก็บอยู่

### การแกะค่า Optional

ในการพัฒนาจริง เรามักต้องดึงค่าจริงออกมาจาก Optional กระบวนการนี้เรียกว่า unwrapping

วิธีที่ใช้บ่อยอย่างหนึ่งคือใช้ ?? เพื่อกำหนดค่าเริ่มต้นแทน

```
??
```

สิ่งนี้เรียกว่า: Nil-Coalescing Operator

ตัวอย่างเช่น:

```swift
name ?? "FangJunyu"
```

โค้ดนี้หมายความว่า ถ้า name มีค่า ก็ให้คืนค่านั้นออกมา แต่ถ้าเป็น nil ก็ให้ใช้ค่าเริ่มต้น "FangJunyu"

สิ่งที่ต้องระวังคือ ชนิดข้อมูลของค่าเริ่มต้นจะต้องตรงกับชนิดข้อมูลภายใน Optional

### ใช้ ?? เพื่อกำหนดค่าเริ่มต้น

```swift
let input = "30"
let pi = Double(input)
print("pi:\(pi ?? 10)")    // 30
```

ผลลัพธ์:

```
pi: 30
```

เมื่อ input ถูกแปลงเป็น Double การแปลงสำเร็จ จึงพิมพ์ค่าที่ได้ออกมา แต่ถ้าการแปลงล้มเหลว ก็จะใช้ค่าเริ่มต้นที่กำหนดไว้หลัง ??

ถ้าแปลงไม่สำเร็จ:

```swift
let input = "Hello"
let pi = Double(input)

print("pi: \(pi ?? 10)")
```

ผลลัพธ์:

```
pi: 10
```

ในที่นี้ เมื่อ input ถูกแปลงเป็น Double แล้วล้มเหลวและคืนค่าเป็น nil ตัว ?? จะให้ค่าเริ่มต้นแทนสำหรับกรณี nil

Optional ใช้เพื่อแสดงสถานะว่า "อาจมีค่า หรืออาจไม่มีค่า"

ในการแปลงชนิดข้อมูล หรือการทำงานอื่นที่อาจล้มเหลว Swift จะคืนค่าเป็น Optional เพื่อรักษาความปลอดภัยของโปรแกรม

เมื่อเราต้องการใช้งานค่าภายใน เราสามารถใช้ ?? เพื่อกำหนดค่าเริ่มต้นได้ ทำให้แม้จะเกิด nil ขึ้น ก็ยังสามารถได้ค่าที่เหมาะสมกลับมา
