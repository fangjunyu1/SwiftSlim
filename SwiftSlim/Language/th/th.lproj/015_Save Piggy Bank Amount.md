# การบันทึกจำนวนเงินในกระปุกออมสิน

ในบทเรียนก่อนหน้านี้ เราได้สร้าง “กระปุกออมสิน” แบบง่าย ๆ ขึ้นมาแล้ว ในบทเรียนนี้ เราจะเรียนรู้วิธีทำให้จำนวนเงินของ “กระปุกออมสิน” ถูกบันทึกแบบถาวร

บทเรียนนี้จะเน้นเรียนรู้เรื่อง UserDefaults และ property wrapper `@AppStorage` พร้อมทั้งขยายความเกี่ยวกับบทบาทของ `onAppear` และวิธีใช้งาน Optional (`nil`)

สิ่งที่ต้องระวังคือ บทเรียนนี้จำเป็นต้องต่อยอดจากโค้ด “กระปุกออมสิน” ที่เราพัฒนาไว้ในบทก่อนหน้า

## ปัญหาของการบันทึกแบบถาวร

ตอนนี้ หน้าตาของ “กระปุกออมสิน” เรียบง่ายและใช้งานได้สะดวกมาก

![Piggy Bank](../../Resource/014_view.png)

แต่มีปัญหาที่ชัดเจนอยู่ข้อหนึ่ง: ทุกครั้งที่รีเฟรชมุมมองหรือปิดแอป จำนวนเงินรวมของ “กระปุกออมสิน” จะกลับไปเป็น 0 และจำนวนเงินเดิมที่เคยเก็บไว้ก็จะหายไปทั้งหมด

สิ่งนี้หมายความว่าข้อมูลยังไม่ได้ถูกบันทึกจริง ๆ หรือพูดอีกแบบก็คือ ตอนนี้ “กระปุกออมสิน” ยังเก็บได้เพียงข้อมูลชั่วคราวเท่านั้น

### ทำไมจึงไม่สามารถบันทึกแบบถาวรได้

สาเหตุเป็นเพราะเราใช้ตัวแปรที่ประกาศด้วย `@State`:

```swift
@State private var amount = 0
```

ตัวแปรที่ประกาศด้วย `@State` จะมีอายุการใช้งานที่ขึ้นอยู่กับ view โดยสมบูรณ์

เมื่อ view ถูกสร้างขึ้น amount จะถูกกำหนดค่าเริ่มต้นเป็น 0 และเมื่อ view ถูกทำลาย amount ก็จะถูกทำลายไปด้วย

ข้อมูลที่เก็บอยู่ใน amount นี้จึงมีอยู่เพียงในหน่วยความจำเท่านั้น และยังไม่ได้ถูกบันทึกลงบนอุปกรณ์จริง

ถ้าเราไม่ต้องการให้ amount ผูกกับอายุการใช้งานของ view ก็จำเป็นต้องทำ “การบันทึกข้อมูลแบบถาวร” หรือก็คือบันทึกข้อมูลลงบนอุปกรณ์

### “การบันทึกข้อมูลแบบถาวร” คืออะไร

การบันทึกข้อมูลแบบถาวรสามารถเข้าใจได้ว่า เป็นการย้ายข้อมูลจาก “หน่วยความจำชั่วคราว” ไปเก็บไว้ใน “พื้นที่จัดเก็บของอุปกรณ์”

เมื่อปิด view หรือปิดแอป ข้อมูลก็จะไม่หายไป

ในการพัฒนา Swift หากเป็นการบันทึกข้อมูลแบบง่าย ๆ สามารถใช้ UserDefaults ได้ แต่ถ้าเป็นข้อมูลที่ซับซ้อนมากขึ้น จะต้องใช้ SwiftData หรือ CoreData

ในบทเรียนนี้ เราจะเริ่มจาก UserDefaults ซึ่งเป็นวิธีที่ง่ายที่สุดก่อน

## UserDefaults

UserDefaults ใช้สำหรับบันทึกข้อมูลแบบ key-value ขนาดเล็ก โดยมักใช้เก็บข้อมูลพื้นฐาน เช่น String, Int, Double, Bool เป็นต้น

### การบันทึกข้อมูล

UserDefaults ใช้เมธอด `set` ในการบันทึกข้อมูล:

```swift
UserDefaults.standard.set("FangJunyu", forKey: "name")
UserDefaults.standard.set(28, forKey: "age")
UserDefaults.standard.set(true, forKey: "isMember")
```

พารามิเตอร์ตัวแรกของ UserDefaults คือข้อมูลที่ต้องการบันทึก ซึ่งในที่นี้เป็นข้อมูลชนิด String, Int และชนิดอื่น ๆ

พารามิเตอร์ตัวที่สอง `forKey` มีลักษณะคล้ายชื่อตัวแปร ใช้เป็นตัวระบุข้อมูลที่ถูกบันทึกไว้ใน UserDefaults

### การอ่านข้อมูล

UserDefaults ใช้เมธอดที่สอดคล้องกับชนิดข้อมูลในการอ่านค่า:

```swift
let name = UserDefaults.standard.string(forKey: "name")	// FangJunyu
let age = UserDefaults.standard.integer(forKey: "age")	// 28
let isMember = UserDefaults.standard.bool(forKey: "isMember")	// true
```

ตอนอ่านข้อมูล จำเป็นต้องใช้เมธอดที่ตรงกับชนิดข้อมูล ไม่เช่นนั้นอาจเกิดข้อผิดพลาดหรือได้ค่าที่ไม่ถูกต้อง

### สิ่งที่ควรระวังเกี่ยวกับ UserDefaults

#### 1. คืนค่าเป็น Optional

เมื่อใช้ UserDefaults อ่านข้อมูล เมธอดบางตัวจะคืนค่าเป็น Optional

ตัวอย่างเช่น:

```swift
let name = UserDefaults.standard.string(forKey: "name")
```

ชนิดข้อมูลของ name ที่ได้กลับมาคือ:

```swift
String?
```

นี่หมายความว่า ค่านี้อาจมีข้อมูลอยู่ หรืออาจไม่มีข้อมูลก็ได้

**ทำไมจึงคืนค่าเป็น Optional**

เพราะ Key บางตัวใน UserDefaults อาจไม่มีข้อมูลถูกเก็บไว้ก็ได้

ตัวอย่างเช่น:

```swift
let name = UserDefaults.standard.string(forKey: "City") // nil
```

ถ้าเราไม่เคยบันทึกข้อมูลด้วย Key `"City"` มาก่อน ผลลัพธ์ที่อ่านได้ก็จะเป็น:

```swift
nil
```

สิ่งที่ต้องเน้นคือ `nil` หมายถึง “ไม่มีข้อมูล” ไม่ใช่สตริงว่าง

ในบท “ระบบชนิดข้อมูล” เราเคยพบสถานการณ์คล้ายกันมาแล้ว คือเมื่อการแปลงชนิดข้อมูลล้มเหลว ก็จะคืนค่าเป็น Optional

ดังนั้น ในกรณีนี้เราต้องใช้ `??` เพื่อกำหนดค่าเริ่มต้นสำหรับการแกะค่า:

```swift
let name = UserDefaults.standard.string(forKey: "City")	?? "Rizhao"
```

ถ้า UserDefaults อ่านข้อมูลได้ ก็จะใช้ค่าที่อ่านได้ แต่ถ้าอ่านไม่ได้ (`nil`) ก็จะใช้ค่าเริ่มต้นแทน

#### 2. คืนค่าเป็นชนิดไม่ใช่ Optional

เมื่อ UserDefaults อ่านข้อมูลชนิด Int, Double, Bool จะคืนค่าเป็นชนิดที่ไม่ใช่ Optional

ตัวอย่างเช่น:

```swift
let num = UserDefaults.standard.integer(forKey: "num")	// 0
let height = UserDefaults.standard.double(forKey: "height")	// 0.0
let isBuyCar = UserDefaults.standard.bool(forKey: "isBuyCar")	// false
```

แม้ว่า Key เหล่านี้จะไม่เคยถูกบันทึกข้อมูลมาก่อน ค่าที่ได้ก็จะไม่ใช่ `nil`

แต่จะเป็นค่าเริ่มต้นดังนี้:

```swift
Int → 0
Double → 0.0
Bool → false
```

#### 3. ข้อจำกัดด้านปริมาณข้อมูล

UserDefaults เหมาะสำหรับการเก็บข้อมูลปริมาณน้อย เช่น การตั้งค่าของแอป หรือสถานะง่าย ๆ บางอย่าง

ในการใช้งานจริง มันไม่เหมาะกับการเก็บข้อมูลจำนวนมาก หรือข้อมูลที่ต้องอ่านและเขียนบ่อยมาก

## การบันทึกจำนวนเงินของ “กระปุกออมสิน”

ตอนนี้ เราสามารถใช้ UserDefaults เพื่อเขียนตรรกะสำหรับบันทึกยอดรวมของ “กระปุกออมสิน” ได้แล้ว

เมื่อผู้ใช้กดปุ่ม ให้นำจำนวนเงินที่กรอกมาบวกกับยอดรวม แล้วบันทึกยอดรวมลงใน UserDefaults

```swift
Button("Save") {
    amount += number
    number = 0
	UserDefaults.standard.set(amount, forKey: "amount")	// UserDefaults
}
.buttonStyle(.borderedProminent)
```

ตอนนี้ เราได้ทำส่วนของการเขียนข้อมูลเรียบร้อยแล้ว

## คำอธิบายของตรรกะการบันทึกแบบถาวร

แม้ว่าเราจะเขียนส่วนสำหรับบันทึกข้อมูลแบบถาวรเสร็จแล้ว แต่เมื่อทดลองรันจริง จะพบว่า ทุกครั้งที่รีเฟรช view หรือเปิดแอปใหม่ ตัวแปร amount ก็ยังคงเป็น 0

สาเหตุคือ:

```swift
@State private var amount = 0
```

ใน SwiftUI ตัวแปร `@State` จะถูกกำหนดค่าเริ่มต้นใหม่ทุกครั้งที่ view ถูกสร้างขึ้น

แม้ว่าใน UserDefaults จะบันทึกค่า amount ไว้แล้ว แต่เมื่อ view โหลดใหม่ amount ก็ยังคงถูกรีเซ็ตกลับเป็น 0

สิ่งนี้แสดงให้เห็นว่า ตอนนี้เราทำเพียงแค่ “บันทึกข้อมูลลง UserDefaults” แต่ยังไม่ได้ “ดึงข้อมูลกลับมาใช้ใน view”

ดังนั้น ตอนที่ view โหลดขึ้นมา เรายังต้องอ่านค่าจาก UserDefaults ด้วยตนเอง แล้วนำมาใส่ให้กับ amount ใหม่อีกครั้ง จึงจะถือว่าตรรกะของการบันทึกแบบถาวรสมบูรณ์

### การเปรียบเทียบเชิงตรรกะ

เราสามารถเปรียบกระบวนการนี้กับ “กระดานดำในห้องเรียน” ได้:

ระหว่างสอน ครูจะเขียนเนื้อหาลงบนกระดาน ข้อมูลเหล่านี้เปรียบได้กับสถานะใน view ปัจจุบัน (`@State`)

เมื่อคาบเรียนจบ กระดานจะถูกลบออกเพื่อเตรียมสำหรับคาบต่อไป ซึ่งก็เหมือนกับการที่ view ถูกทำลาย และข้อมูลใน `@State` ถูกล้างทิ้ง

เพื่อป้องกันไม่ให้เนื้อหาที่สอนหายไป ครูจะบันทึกเนื้อหาไว้ในเอกสารประกอบการสอนล่วงหน้า เอกสารนี้ก็เปรียบได้กับข้อมูลที่เก็บอยู่ใน UserDefaults

เมื่อเริ่มคาบใหม่ กระดานจะว่างเปล่าอีกครั้ง (`@State` ถูกกำหนดค่าเริ่มต้นใหม่) และครูจะต้องดูจากเอกสาร แล้วเขียนความรู้เดิมกลับลงบนกระดานใหม่อีกครั้ง (อ่านค่าจาก UserDefaults)

สิ่งสำคัญที่ต้องเข้าใจคือ กระดานจะไม่ฟื้นข้อมูลเดิมกลับมาเองโดยอัตโนมัติ แต่ต้องอาศัยการ “อ่านจากเอกสาร” (อ่านจาก UserDefaults) แล้วเขียนกลับเข้าไปใหม่

## ใช้ onAppear ในการอ่านข้อมูล

เมื่อ view แสดงขึ้นมา เราจำเป็นต้องอ่านค่าจาก UserDefaults ด้วยตนเอง แล้วกำหนดค่านั้นให้กับตัวแปร amount ของ `@State` เพื่อให้เกิดการบันทึกแบบถาวรอย่างแท้จริง

ใน SwiftUI เราสามารถใช้ `onAppear` เพื่อให้โค้ดบางอย่างทำงานตอนที่ view ปรากฏขึ้นมา:

```swift
.onAppear {}
```

ให้นำโค้ดสำหรับอ่าน UserDefaults ไปไว้ใน `onAppear`:

```swift
.onAppear {
    amount = UserDefaults.standard.integer(forKey: "amount")
}
```

เมื่อ view ถูกสร้างขึ้น amount จะเริ่มต้นเป็น 0 แต่เมื่อ view ปรากฏขึ้น `onAppear` จะถูกเรียกใช้ และอ่านค่าจาก UserDefaults มาใส่ให้กับ amount ดังนั้น view จึงสามารถแสดงยอดรวมที่เคยบันทึกไว้ครั้งล่าสุดได้

ตอนนี้ ตรรกะการบันทึกข้อมูลแบบถาวรของ “กระปุกออมสิน” ก็ถือว่าเสร็จสมบูรณ์แล้ว

## โค้ดฉบับสมบูรณ์

```swift
struct ContentView: View {
    
    @State private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
                UserDefaults.standard.set(amount, forKey: "amount")    // UserDefaults
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
        .onAppear {
            amount = UserDefaults.standard.integer(forKey: "amount")
        }
    }
}
```

## `@AppStorage` Property Wrapper

SwiftUI มี property wrapper ชื่อ `@AppStorage` ที่ช่วยให้ตรรกะการบันทึกด้วย UserDefaults ง่ายขึ้น

```swift
@AppStorage("amount") private var amount = 0
```

`@AppStorage` เป็นการห่อ UserDefaults ไว้อีกชั้นหนึ่ง โดย `"amount"` คือ Key ที่สอดคล้องกับข้อมูลใน UserDefaults

เมื่อ view ถูกสร้างขึ้น มันจะอ่านข้อมูลจาก UserDefaults ให้โดยอัตโนมัติ และเมื่อค่าของตัวแปรเปลี่ยน ก็จะเขียนกลับลง UserDefaults ให้อัตโนมัติเช่นกัน

มันยังรองรับกลไกการอัปเดตสถานะของ SwiftUI อีกด้วย กล่าวคือทำงานคล้าย `@State` และจะรีเฟรช view เมื่อค่ามีการเปลี่ยนแปลง

หรือพูดอีกแบบหนึ่ง `@AppStorage` สามารถจัดการทั้งส่วนของการอ่านและการบันทึกข้อมูลได้เอง เราจึงไม่ต้องเรียก UserDefaults ด้วยตนเอง และไม่ต้องใช้ `onAppear` เพื่ออ่านข้อมูลแบบ manual อีกต่อไป

ดังนั้น เราจึงสามารถใช้ `@AppStorage` แทน UserDefaults ในโค้ดเดิมได้:

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number = 0
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```

## สรุป

บทเรียนนี้เราได้เรียนรู้ประเด็นสำคัญดังต่อไปนี้:

อย่างแรก UserDefaults ใช้สำหรับบันทึกข้อมูลขนาดเล็ก และเราได้นำมันมาใช้บันทึกยอดรวมลงบนอุปกรณ์ UserDefaults มีเมธอดสำหรับอ่านและเขียนข้อมูล และยังมีประเด็นเรื่อง Optional ที่ต้องระวังด้วย

อย่างที่สอง ตัวแปร `@State` ใช้สำหรับจัดการสถานะชั่วคราวภายใน view เท่านั้น อายุการใช้งานของมันจะเท่ากับอายุของ view และเมื่อ view ถูกสร้างใหม่ `@State` ก็จะถูกกำหนดค่าเริ่มต้นใหม่เช่นกัน

ในการดึงข้อมูลจาก UserDefaults กลับมาใช้ เราได้เรียนรู้การใช้ `onAppear` เพื่ออ่านข้อมูลตอนที่ view ปรากฏขึ้น และใช้เพื่อกู้คืนข้อมูลที่เคยบันทึกไว้

สุดท้าย เราใช้ `@AppStorage` มาแทน UserDefaults แบบเดิม ทำให้ view สามารถอ่านข้อมูลอัตโนมัติขณะโหลด และเขียนข้อมูลกลับอัตโนมัติเมื่อค่ามีการเปลี่ยนแปลง จึงสามารถแทนตรรกะที่ต้องเรียก UserDefaults และ `onAppear` เองได้

ด้วยความรู้เหล่านี้ เราจึงสามารถสร้างระบบเก็บข้อมูลพื้นฐานให้กับแอป เพื่อให้ข้อมูลถูกบันทึกแบบถาวรได้

## แบบฝึกหัดหลังเรียน

ตอนนี้ “กระปุกออมสิน” รองรับการบันทึกจำนวนเงินแล้ว แต่ฟังก์ชันยังไม่สมบูรณ์ทั้งหมด

1. หากผู้ใช้ต้องการถอนเงินออก ต้องออกแบบตรรกะสำหรับการถอนเงินให้ครบถ้วน
2. หากผู้ใช้ต้องการล้างจำนวนเงินทั้งหมด ก็ต้องเพิ่มตรรกะสำหรับการรีเซ็ตด้วย
3. หากต้องการตั้งชื่อให้กระปุกออมสิน ก็จะต้องเพิ่มการบันทึกข้อมูลชนิด String เข้ามาด้วย

คุณสามารถต่อยอดจากพื้นฐานที่มีอยู่ เพื่อทำให้ฟังก์ชันของ “กระปุกออมสิน” สมบูรณ์มากขึ้น

## ความรู้เพิ่มเติม - ปัญหาค่าเริ่มต้นของช่องกรอก

ตอนนี้ เรายังสามารถปรับปรุงประสบการณ์การกรอกข้อมูลของ TextField ให้สะดวกขึ้นได้อีก

![textfield](../../Resource/015_view.png)

ในการทำงานปัจจุบัน ช่องกรอกจะแสดงค่าเริ่มต้นเป็น 0 ซึ่งหมายความว่า ทุกครั้งที่ผู้ใช้ต้องการกรอกจำนวนเงิน จะต้องลบ 0 ออกก่อน แล้วจึงพิมพ์ตัวเลขใหม่ วิธีโต้ตอบแบบนี้ไม่ค่อยเหมาะสมเท่าไร

สิ่งที่เราควรทำคือ ให้ช่องกรอกมีค่าเริ่มต้นเป็นค่าว่าง ไม่ใช่ 0

สาเหตุเป็นเพราะ TextField กับ number มีการ bind แบบสองทาง:

```swift
@State private var number = 0
TextField("Amount", value: $number, format: .number)
```

เมื่อ number มีค่าเริ่มต้นเป็น 0 ช่องกรอกก็จะแสดง 0 ตามไปด้วย

ดังนั้น เราจึงต้องเปลี่ยนให้ number กลายเป็นค่าว่าง หรือก็คือ `nil`

```swift
@State private var number = nil
```

จุดนี้จะเกิดข้อผิดพลาดขึ้น:

```swift
Generic parameter 'Value' could not be inferred
```

นี่ก็คือสถานการณ์ที่เราอธิบายไว้ในบท “ระบบชนิดข้อมูล”: เมื่อคอมไพเลอร์ไม่สามารถอนุมานชนิดข้อมูลของตัวแปรได้ เราจะต้องประกาศชนิดข้อมูลแบบชัดเจน

เนื่องจากเรากำหนดให้ number เป็น `nil` และ `nil` เองไม่ได้มีข้อมูลเรื่องชนิดติดมาด้วย คอมไพเลอร์จึงไม่รู้ว่าตัวแปรนี้ควรเป็น String, Int หรือชนิดอื่น จึงรายงานข้อผิดพลาดออกมา

ดังนั้น เราจึงต้องประกาศชนิดข้อมูลของ number แบบชัดเจน:

```swift
@State private var number:Int? = nil
```

`Int?` ตรงนี้หมายถึงชนิด Optional ซึ่งบอกว่า number สามารถมีค่าเป็น Int หรือเป็น `nil` ก็ได้

เมื่อกำหนดค่าเริ่มต้นเป็น `nil` แบบนี้ ช่องกรอกก็จะแสดงค่าว่างแทนที่จะเป็น 0

### การแกะค่า Optional

เมื่อเราเปลี่ยน number ให้เป็น Optional แล้ว โค้ดคำนวณเดิมจะเกิดข้อผิดพลาด:

```swift
amount += number
```

ข้อความแจ้งเตือนคือ:

```swift
Value of optional type 'Int?' must be unwrapped to a value of type 'Int'
```

สาเหตุเป็นเพราะตอนนี้ number อาจเป็น `nil` ได้ และถ้าเอาไปคำนวณตรง ๆ ก็จะกลายเป็น:

```swift
amount += nil
```

ซึ่งเห็นได้ชัดว่า `nil` ไม่สามารถนำไปบวกกับ amount ได้ การกระทำนี้จึงไม่ถูกต้อง

ดังนั้น เมื่อใช้ Optional ในการคำนวณ เราจำเป็นต้องแกะค่าก่อน

ตรงนี้สามารถใช้ `??` เพื่อกำหนดค่าเริ่มต้นได้:

```swift
amount += number ?? 0
```

เมื่อ number เป็น `nil` ก็จะใช้ค่าเริ่มต้น 0 แต่ถ้า number มีค่า ก็จะใช้ค่าจริงที่ผู้ใช้กรอกไว้

วิธีนี้ช่วยให้การคำนวณยังถูกต้อง และในขณะเดียวกันก็ทำให้ช่องกรอกมีค่าเริ่มต้นเป็นค่าว่าง ผู้ใช้จึงไม่ต้องคอยลบเลข 0 ออกก่อนทุกครั้งอีกต่อไป

## โค้ดเพิ่มเติม

```swift
struct ContentView: View {
    
    @AppStorage("amount") private var amount = 0
    @State private var number:Int? = nil
    
    var body: some View {
        VStack {
            Text("\(amount)")
                .font(.largeTitle)
                .fontWeight(.bold)
            TextField("Amount", value: $number, format: .number)
                .frame(width: 100)
                .padding(5)
                .border(Color.black, width: 1)
            Button("Save") {
                amount += number ?? 0
                number = 0
            }
            .buttonStyle(.borderedProminent)
        }
        .background {
            Image("1")
        }
    }
}
```
