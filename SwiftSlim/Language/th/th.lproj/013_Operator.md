# ตัวดำเนินการ

ในบทเรียนนี้ เราจะเรียนรู้เรื่องตัวดำเนินการ (Operators) ของ Swift

เมื่อเราทำการคำนวณ การเปรียบเทียบ หรือการตัดสินใจเชิงตรรกะ เราจำเป็นต้องใช้ตัวดำเนินการ การทำงานเหล่านี้ปรากฏอยู่ตลอดทุกส่วนของการทำงานของโปรแกรม และเป็นหนึ่งในความสามารถพื้นฐานที่สุดของการเขียนโปรแกรม

```swift
let a = 100 * 8
```

ระหว่างการเรียนเขียนโปรแกรม หลายคนจะสังเกตได้ว่า คนที่มีพื้นฐานความคิดทางคณิตศาสตร์อยู่บ้าง มักจะเข้าใจตรรกะของโปรแกรมได้ง่ายกว่า นั่นเป็นเพราะโดยแก่นแท้แล้ว การเขียนโปรแกรมแยกไม่ออกจาก “การคำนวณ” และ “การตัดสินตามกฎ” ซึ่งสิ่งเหล่านี้ก็คือหน้าที่หลักของตัวดำเนินการ

ในการพัฒนาจริง ตัวดำเนินการไม่ได้ใช้แค่กับการคำนวณตัวเลขแบบง่ายเท่านั้น แต่ยังถูกใช้อย่างกว้างขวางในสถานการณ์ที่เป็นรูปธรรมอีกมากมาย

ตัวอย่างเช่น ในเกมบล็อก เมื่อผู้ใช้ลากบล็อกไปยังตำแหน่งที่กำหนด จำเป็นต้องคำนวณว่า:

* พิกัดของบล็อกอยู่ที่ใด
* บล็อกสามารถเติมลงในตำแหน่งใดได้บ้าง
* ตำแหน่งเป้าหมายสามารถใช้งานได้หรือไม่

กระบวนการเหล่านี้ที่ประกอบด้วย “การคำนวณ + การตัดสิน” แท้จริงแล้วล้วนพึ่งพาตัวดำเนินการทั้งสิ้น

![Block](../../Resource/013_block.png)

การเรียนรู้ตัวดำเนินการ จึงไม่ได้เป็นเพียงการเรียนรู้การคำนวณตัวเลขเท่านั้น แต่ยังเป็นการวางรากฐานสำหรับการจัดวางหน้าจอ การประมวลผลข้อมูล และตรรกะการโต้ตอบในบทเรียนถัดไป

## วิธีการคำนวณใน SwiftUI

ใน SwiftUI โดยทั่วไปเรามักใช้ Button เพื่อเรียกใช้ฟังก์ชัน แล้วทำตรรกะการคำนวณภายในฟังก์ชันนั้น

ตัวอย่างเช่น:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

เมื่อกดปุ่ม Console จะแสดงผลว่า:

```
c: 20
```

ในบทเรียนนี้ เราจะเขียนตรรกะการคำนวณไว้ใน func เป็นหลัก แล้วใช้ปุ่มเป็นตัวกระตุ้นการทำงาน พร้อมใช้ print เพื่อแสดงผลลัพธ์

ในตัวอย่างต่อ ๆ ไป เพื่อให้โค้ดกระชับขึ้น เราจะใส่ผลลัพธ์ของการคำนวณไว้เป็นหมายเหตุในโค้ดโดยตรง และจะไม่ใช้ print ซ้ำในทุกตัวอย่าง

## การจัดการผลลัพธ์ใน Console

เนื่องจากทุกครั้งที่กดปุ่มจะมีข้อความถูกพิมพ์ออกมา คุณสามารถกดปุ่มล้างที่มุมขวาล่างของพื้นที่ Console เพื่อเคลียร์ประวัติการแสดงผล ทำให้ดูผลลัพธ์ปัจจุบันได้สะดวกขึ้น

![Console](../../Resource/013_console.png)

## การเขียน Button อีกแบบหนึ่ง

เมื่อปุ่มมีหน้าที่เพียงเรียกใช้ฟังก์ชันเดียว สามารถใช้รูปแบบที่กระชับกว่าได้:

```swift
Button(action: ) {
    // View
}
```

ในรูปแบบนี้ action ใช้ระบุฟังก์ชันที่จะทำงานเมื่อกดปุ่ม (ฟังก์ชันที่ไม่มีพารามิเตอร์) ส่วนวงเล็บปีกกา {} ใช้กำหนดมุมมองของปุ่ม

เมื่อนำมารวมกับฟังก์ชัน calculate ก่อนหน้านี้ จะเขียนได้เป็น:

```swift
Button(action: calculate) {
    Text("Button")
}
```

เมื่อกดปุ่ม ก็จะเรียกใช้ฟังก์ชัน calculate โดยตรง

ดังนั้น เมื่อ logic ของปุ่มไม่ซับซ้อน และมีเพียงการเรียกใช้ฟังก์ชันเดียว การเขียนแบบ action จะดูชัดเจนกว่า และสอดคล้องกับรูปแบบการเขียนโค้ดที่ดีมากกว่า

## ตัวดำเนินการทางคณิตศาสตร์

ตัวดำเนินการทางคณิตศาสตร์ (Arithmetic Operators) ใช้สำหรับคำนวณตัวเลข:

* `+` การบวก
* `-` การลบ
* `*` การคูณ
* `/` การหาร
* `%` การหารเอาเศษ

ตัวอย่าง:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

สิ่งที่ต้องระวังคือ Swift เป็นภาษาแบบ strong type จึงไม่ผสมชนิดข้อมูลที่ต่างกันให้อัตโนมัติ:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

เมื่อชนิดข้อมูลของทั้งสองค่าไม่ตรงกัน จำเป็นต้องแปลงชนิดข้อมูลด้วยตนเองก่อน

## ตัวดำเนินการเปรียบเทียบ

ตัวดำเนินการเปรียบเทียบ (Comparison Operators) ใช้สำหรับตรวจสอบความสัมพันธ์ของค่าทั้งสอง โดยผลลัพธ์จะเป็น Bool:

* `==` เท่ากับ
* `!=` ไม่เท่ากับ
* `>` มากกว่า
* `<` น้อยกว่า
* `>=` มากกว่าหรือเท่ากับ
* `<=` น้อยกว่าหรือเท่ากับ

ตัวอย่าง:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

ตัวดำเนินการเปรียบเทียบมักถูกใช้ในประโยคเงื่อนไข ตัวอย่างเช่น ใน “การเลื่อนคำคม” เราต้องควบคุมไม่ให้ index เกินขอบเขตของอาร์เรย์:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

ถ้า index น้อยกว่า sayings.count - 1 ผลการเปรียบเทียบจะเป็น true และโค้ดใน if จะถูกดำเนินการ ถ้าผลเป็น false โค้ดด้านหลังก็จะไม่ถูกทำงาน

## ตัวดำเนินการเชิงตรรกะ

ตัวดำเนินการเชิงตรรกะ (Logical Operators) ใช้สำหรับรวมหลายเงื่อนไขเข้าด้วยกัน:

* `&&` และ (AND)
* `||` หรือ (OR)
* `!` ไม่ (NOT)

### && และ

เมื่อหลายเงื่อนไขจำเป็นต้องเป็นจริงพร้อมกัน ให้ใช้ &&

ตัวอย่างเช่น:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

ในโค้ดนี้มีสองเงื่อนไข: age >= 18 และ age <= 100

เฉพาะเมื่อทั้งสองเงื่อนไขเป็น true พร้อมกัน legalAge จึงจะเป็น true หากมีข้อใดข้อหนึ่งไม่เป็นจริง ผลลัพธ์ก็จะเป็น false

### || หรือ

เมื่อมีหลายเงื่อนไขและต้องการให้เป็นจริงเพียงข้อเดียว ให้ใช้ ||

ตัวอย่างเช่น:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

โค้ดนี้หมายความว่า ขอเพียง money >= 50 หรือ creditCard >= 50 อย่างใดอย่างหนึ่งเป็นจริง payAllowed ก็จะเป็น true

เฉพาะเมื่อทั้งสองเงื่อนไขเป็น false พร้อมกันเท่านั้น ผลลัพธ์จึงจะเป็น false

ตัวอย่างเช่น ในสถานการณ์การชำระเงิน เราอาจใช้เงินสดหรือบัตรเครดิตก็ได้ หากทั้งสองอย่างใช้ชำระไม่ได้ การชำระเงินก็จะล้มเหลว

### ! ไม่

! คือ ตัวดำเนินการ not เชิงตรรกะ (NOT) ใช้สำหรับกลับค่าของ Bool

สามารถเข้าใจง่าย ๆ ได้ว่า ! จะเปลี่ยน true ให้เป็น false และเปลี่ยน false ให้เป็น true

ตัวอย่างเช่น:

```swift
let allowed = true
let result = !allowed   // false
```

ตรงนี้ !allowed หมายถึงการกลับค่าของ allowed

ควรระวังว่า ! และ != เป็นตัวดำเนินการคนละแบบกัน ! ใช้กลับค่า Bool ส่วน != เป็นตัวดำเนินการเปรียบเทียบ และจะคืนค่าเป็น Bool

## ตัวดำเนินการกำหนดค่า

ตัวดำเนินการกำหนดค่า (Assignment Operators) ใช้สำหรับกำหนดค่าหรืออัปเดตค่าของตัวแปร:

* `=` กำหนดค่า

ไม่ว่าจะตอนประกาศตัวแปรหรือระหว่างการคำนวณ ก็จะมีการกำหนดค่าเข้ามาเกี่ยวข้อง:

```swift
let a = 5
```

โค้ดนี้หมายถึงการกำหนดค่า 5 ให้กับตัวแปร a

### ตัวดำเนินการกำหนดค่าแบบผสม

ในการพัฒนาจริง นอกเหนือจากตัวดำเนินการกำหนดค่าพื้นฐานแล้ว เรายังใช้ “ตัวดำเนินการกำหนดค่าแบบผสม” อยู่บ่อยครั้ง เพื่อคำนวณต่อจากค่าเดิมแล้วอัปเดตกลับเข้าไปโดยตรง:

* `+=`
* `-=`
* `*=`
* `/=`

ในบท “ตัวนับ” ก่อนหน้านี้ เราได้รู้จักตัวดำเนินการกำหนดค่าแบบผสมมาแบบคร่าว ๆ แล้ว

ตัวอย่างเช่น เมื่อกดปุ่มแล้วให้ตัวแปรเพิ่มค่าขึ้นอัตโนมัติ:

```swift
var num = 10
num += 5   // num = 15
```

โค้ดนี้หมายความว่า นำ num ไปบวกกับ 5 แล้วกำหนดผลลัพธ์กลับเข้าไปที่ num อีกครั้ง

ดังนั้น มันจึงเทียบเท่ากับ:

```swift
var num = 10
num = num + 5   // num = 15
```

## ตัวดำเนินการสามทาง

ตัวดำเนินการสามทาง (Ternary Operator) ใน Swift จะตรวจสอบเงื่อนไข แล้วคืนค่าหนึ่งในสองค่าตามผลของเงื่อนไข:

```swift
เงื่อนไข ? ค่า1 : ค่า2
```

ตรรกะการทำงานคือ ถ้าเงื่อนไขเป็น true จะคืน “ค่า1” แต่ถ้าเงื่อนไขเป็น false จะคืน “ค่า2”

ตัวอย่างเช่น:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

โค้ดนี้จะตรวจสอบค่า showColor ถ้าเป็น true จะคืน Color.blue ทำให้พื้นหลังเป็นสีน้ำเงิน ถ้าเป็น false จะคืน Color.clear ซึ่งหมายถึงไม่แสดงสีพื้นหลัง

หมายเหตุ: Color.clear หมายถึงสีโปร่งใส (ไม่แสดงอะไร)

### ความต่างจาก if-else

ในเชิงตรรกะ ตัวดำเนินการสามทางมีความหมายเทียบเท่ากับประโยค if-else

ตัวอย่างเช่น โค้ดด้านบนสามารถเขียนได้เป็น:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

ตรรกะของทั้งสองแบบเหมือนกันทั้งหมด กล่าวคือ ถ้าเงื่อนไขเป็น true จะใช้ผลลัพธ์แบบแรก ถ้าเป็น false จะใช้ผลลัพธ์แบบที่สอง

ในทางปฏิบัติ ตัวดำเนินการสามทางเขียนได้กระชับกว่า จึงเหมาะกับการแสดงเงื่อนไขแบบ “เลือกหนึ่งจากสอง” และมักใช้เมื่อเราต้องการคืนค่ากลับมาโดยตรง

ในการสร้าง View ด้วย SwiftUI ตัวดำเนินการสามทางพบได้บ่อยมาก เช่น:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

รูปแบบนี้ช่วยให้เราสามารถเปลี่ยนเอฟเฟกต์ของ view ตามเงื่อนไขได้ภายในโค้ดเพียงบรรทัดเดียว

## ตัวดำเนินการช่วงค่า

ตัวดำเนินการช่วงค่า (Range Operators) ใช้สำหรับแทนช่วงของค่า:

* `...` ช่วงปิด (รวมทั้งสองด้าน)
* `..<` ช่วงกึ่งเปิด (ไม่รวมด้านขวา)

ตัวอย่างเช่น:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

สามารถเข้าใจได้ว่า 1...3 หมายถึงตั้งแต่ 1 ถึง 3 และรวม 3 ด้วย ส่วน 5..<8 หมายถึงตั้งแต่ 5 ถึง 8 แต่ไม่รวม 8

รูปแบบขยาย (ศึกษาไว้ก่อน):

```
5...    // >= 5
...5    // <= 5
```

รูปแบบนี้หมายถึง “ช่วงด้านเดียว” ซึ่งมักใช้ในการเปรียบเทียบเงื่อนไขหรือ pattern matching

ตัวดำเนินการช่วงค่าจะถูกใช้ในบทเรียนเรื่อง for loop ภายหลัง ตอนนี้สามารถเข้าใจเบื้องต้นได้ก่อนว่า มันใช้แทนช่วงของตัวเลขที่ต่อเนื่องกัน

## ฝึกใช้ตัวดำเนินการ

ต่อไป เราจะใช้ตัวอย่างง่าย ๆ สักสองสามแบบ เพื่อฝึกดูบทบาทของตัวดำเนินการใน view

### 1. คำนวณจำนวนรถทั้งหมด

ตัวอย่างเช่น มีรถวิ่งผ่านสี่แยกหนึ่งวันละ 500 คัน ต้องการคำนวณจำนวนรถทั้งหมดใน 30 วัน

เราสามารถใช้ตัวดำเนินการคำนวณได้:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

กระบวนการคำนวณตรงนี้คือ วันละ 500 คัน เป็นเวลา 30 วัน ดังนั้นจำนวนรวมก็คือ 500 × 30 และสุดท้ายได้จำนวนรถรวม 15000 คัน

### 2. ตรวจสอบสมาชิกการซื้อภายในแอป

เราสามารถตรวจสอบได้ว่าคนคนหนึ่งเป็นสมาชิกแบบซื้อภายในแอปหรือไม่ โดยหากเขามีสมาชิกถาวร หรือสมาชิกแบบสมัครใช้งานอย่างใดอย่างหนึ่ง ก็ถือว่าเป็นสมาชิก

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

ในโค้ดนี้ใช้ตัวดำเนินการ || (หรือ): lifeTime = false หมายถึงไม่มีสมาชิกถาวร ส่วน subscription = true หมายถึงมีสมาชิกแบบสมัครใช้งาน

เพราะต้องการเพียงให้เงื่อนไขใดเงื่อนไขหนึ่งเป็นจริง isMember จึงเป็น true ดังนั้นบุคคลนี้จึงถือว่าเป็นสมาชิกการซื้อภายในแอป

## สรุป

ใน Swift ตัวดำเนินการคือพื้นฐานของการคำนวณข้อมูลและการตัดสินเชิงตรรกะ เช่นเดียวกับที่ Text ใช้แสดงเนื้อหา ตัวดำเนินการก็ใช้จัดการกับข้อมูลโดยตรง

ตั้งแต่การคำนวณตัวเลขแบบง่ายที่สุด ไปจนถึงการตัดสินเงื่อนไขที่ซับซ้อน รวมถึงการควบคุมสถานะของ view และการคำนวณ layout ล้วนไม่สามารถแยกออกจากตัวดำเนินการได้

ผ่านการฝึกฝนในสถานการณ์ที่หลากหลาย เราจะค่อย ๆ คุ้นเคยและเข้าใจตัวดำเนินการแต่ละชนิดที่กล่าวถึงในบทนี้ ซึ่งสิ่งเหล่านี้เป็นพื้นฐานสำคัญของการเรียนรู้ในลำดับถัดไป

### ความรู้เพิ่มเติม - ตัวดำเนินการระดับบิต

นอกจากนี้ Swift ยังมีตัวดำเนินการอีกกลุ่มหนึ่งที่อยู่ระดับล่างกว่า คือ ตัวดำเนินการระดับบิต (Bitwise Operators):

* `&`（AND ระดับบิต）
* `|`（OR ระดับบิต）
* `^`（XOR ระดับบิต）
* `~`（NOT ระดับบิต）
* `>>`（เลื่อนขวา）
* `<<`（เลื่อนซ้าย）

ตัวอย่างเช่น:

```swift
let a = 6  // เลขฐานสอง 110
let b = 3  // เลขฐานสอง 011

print(a & b) // 2 (เลขฐานสอง 010)
print(a | b) // 7 (เลขฐานสอง 111)
print(a ^ b) // 5 (เลขฐานสอง 101)
print(~a)    // -7 (กลับบิต)
```

ใน Swift จำนวนเต็มจะถูกเก็บในรูปแบบเลขฐานสองแบบ two’s complement ดังนั้นเมื่อกลับบิตแล้ว จะได้ผลลัพธ์เป็นจำนวนลบที่สอดคล้องกัน

ตัวดำเนินการระดับบิตจะทำงานกับข้อมูลเลขฐานสองโดยตรง และมักใช้ในงานประมวลผลระดับล่าง หรือสถานการณ์ที่ต้องการปรับแต่งประสิทธิภาพ

สำหรับผู้เริ่มต้น ตัวดำเนินการกลุ่มนี้มีโอกาสใช้งานค่อนข้างน้อย และในการพัฒนา iOS หรือ SwiftUI ทั่วไปก็มักไม่ค่อยพบ จึงยังไม่จำเป็นต้องลงลึกในตอนนี้

หากในภายหลังมีเนื้อหาเกี่ยวกับตรรกะระดับล่างหรือการเพิ่มประสิทธิภาพ ค่อยกลับมาเรียนรู้และทำความเข้าใจเพิ่มเติมได้
