# การปรับรูปลักษณ์

ในบทเรียนนี้ เราจะเรียนรู้การปรับรูปลักษณ์สำหรับโหมดสว่างและโหมดมืด

บน iPhone, iPad และ Mac ผู้ใช้สามารถเลือกโหมดสว่างหรือโหมดมืดได้

เมื่อระบบเปลี่ยนเป็นโหมดมืด สีพื้นหลัง ข้อความ และไอคอนของ App หลายตัวก็จะเปลี่ยนตามไปด้วย

ตัวอย่างเช่น อินเทอร์เฟซตอนกลางวันอาจเป็นพื้นหลังสีอ่อน ส่วนตอนกลางคืนอาจเปลี่ยนเป็นพื้นหลังสีเข้ม

![view](../../../Resource/025_view9.png)

ความสามารถในการปรับการแสดงผลของอินเทอร์เฟซโดยอัตโนมัติตามรูปลักษณ์ของระบบนี้ เรียกว่าการปรับรูปลักษณ์

## พรีวิวการปรับรูปลักษณ์ใน Xcode

ก่อนเรียนรู้การปรับรูปลักษณ์ เรามาดูก่อนว่าจะพรีวิวโหมดสว่างและโหมดมืดใน Xcode ได้อย่างไร

ที่ด้านล่างของพื้นที่ `Canvas` ให้คลิกปุ่ม `Device Settings` เพื่อเปิดหน้าต่าง `Canvas Device Settings`

![canvas](../../../Resource/025_view.png)

ในหน้าต่างนี้จะเห็นการตั้งค่า `Color Scheme`

![canvas1](../../../Resource/025_view1.png)

โดยมีความหมายดังนี้:

- `Light Appearance` หมายถึงโหมดสว่าง
- `Dark Appearance` หมายถึงโหมดมืด

เราสามารถสลับรูปลักษณ์พรีวิวจากตรงนี้ เพื่อดูผลการแสดงผลของอินเทอร์เฟซปัจจุบันในโหมดสว่างและโหมดมืด

## แสดงสองรูปลักษณ์พร้อมกัน

หากต้องการดูโหมดสว่างและโหมดมืดพร้อมกัน สามารถใช้ฟังก์ชัน `Variants`

ที่ด้านล่างของพื้นที่ `Canvas` ให้คลิกปุ่ม `Variants` แล้วเลือก `Color Scheme Variants`

![canvas2](../../../Resource/025_view2.png)

หลังจากเลือกแล้ว `Canvas` จะแสดงผลพรีวิวของโหมดสว่างและโหมดมืดพร้อมกัน

![canvas3](../../../Resource/025_view3.png)

วิธีนี้ช่วยให้เปรียบเทียบความแตกต่างของอินเทอร์เฟซภายใต้รูปลักษณ์ทั้งสองแบบได้สะดวกขึ้น

พูดอย่างง่าย:

หากต้องการสลับโหมดสว่าง/โหมดมืดชั่วคราว สามารถใช้ `Canvas Device Settings`

หากต้องการดูสองรูปลักษณ์พร้อมกัน สามารถใช้ `Color Scheme Variants`

## View จะปรับรูปลักษณ์โดยค่าเริ่มต้น

ใน SwiftUI มี system view หลายตัวที่ปรับให้เข้ากับโหมดสว่างและโหมดมืดโดยอัตโนมัติเป็นค่าเริ่มต้น

ตัวอย่างเช่น:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: "lightbulb")
                .font(.system(size: 100))

            Text("สว่าง")
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("บางอย่าง บางอย่าง บางอย่าง")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ผลการแสดงผล:

![canvas](../../../Resource/025_view4.png)

จะเห็นว่าในโหมดสว่าง ไอคอนและหัวข้อมักแสดงเป็นสีเข้ม

ในโหมดมืด ไอคอนและหัวข้อจะเปลี่ยนเป็นสีอ่อนโดยอัตโนมัติ

เพราะ view ของ SwiftUI เช่น `Text` และ `Image` จะปรับสีตามรูปลักษณ์ของระบบโดยค่าเริ่มต้น

กล่าวคือ หากเราไม่กำหนดสีคงที่ด้วยตนเอง SwiftUI จะช่วยจัดการการปรับรูปลักษณ์บางส่วนให้เรา

## สีคงที่จะไม่สลับโดยอัตโนมัติ

สิ่งที่ต้องระวังคือ หากเรากำหนดสีคงที่ด้วยตนเอง สีนั้นจะไม่สลับอัตโนมัติตามโหมดสว่าง/โหมดมืด

ตัวอย่างเช่น:

```swift
Text("บางอย่าง บางอย่าง บางอย่าง")
	.foregroundStyle(Color.gray)
```

ข้อความตรงนี้ถูกกำหนดเป็น `Color.gray` ดังนั้นจะแสดงเป็นสีเทาอยู่เสมอ

อีกตัวอย่างหนึ่ง:

```swift
Text("หัวข้อ")
    .foregroundStyle(Color.white)
```

โค้ดส่วนนี้จะแสดงข้อความสีขาวเสมอ ไม่ว่าจะอยู่ในโหมดสว่างหรือโหมดมืด

หากพื้นหลังเป็นสีอ่อนด้วย ข้อความสีขาวอาจมองเห็นได้ไม่ชัด

ดังนั้นเมื่อทำการปรับรูปลักษณ์ ควรหลีกเลี่ยงการเขียนสีคงที่ เช่น `Color.white` และ `Color.black` แบบตามใจมากเกินไป

ในหลายกรณี สามารถให้ความสำคัญกับการใช้ semantic style ของระบบก่อน เช่น:

```swift
.foregroundStyle(.primary)
.foregroundStyle(.secondary)
```

โดยมีความหมายดังนี้:

- `.primary` มักใช้สำหรับเนื้อหาหลัก
- `.secondary` มักใช้สำหรับเนื้อหารอง

ทั้งสองจะปรับผลการแสดงผลโดยอัตโนมัติตามโหมดสว่าง/โหมดมืด

## แสดงเนื้อหาที่แตกต่างกันตามรูปลักษณ์

บางครั้ง เราไม่ได้ต้องการให้แค่สีเปลี่ยน แต่ยังต้องการให้เนื้อหาที่แสดงในแต่ละรูปลักษณ์แตกต่างกันด้วย

ตัวอย่างเช่น:

- ในโหมดสว่าง แสดงไอคอนหลอดไฟแบบปกติ
- ในโหมดมืด แสดงไอคอนหลอดไฟที่สว่างขึ้น
- ในโหมดสว่าง หัวข้อแสดง `สว่าง`
- ในโหมดมืด หัวข้อแสดง `มืด`

ในกรณีนี้ เราจำเป็นต้องตรวจสอบว่าระบบปัจจุบันอยู่ในโหมดสว่างหรือโหมดมืด

ใน SwiftUI สามารถใช้ค่า environment `colorScheme` เพื่อรับรูปลักษณ์ปัจจุบันได้:

```swift
@Environment(\.colorScheme) private var colorScheme
```

โค้ดแบบเต็ม:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) private var colorScheme

    private var imageName: String {
        colorScheme == .light ? "lightbulb" : "lightbulb.max"
    }

    private var titleName: String {
        colorScheme == .light ? "สว่าง" : "มืด"
    }

    var body: some View {
        VStack(spacing: 20) {
            Image(systemName: imageName)
                .font(.system(size: 100))

            Text(titleName)
                .font(.largeTitle)
                .fontWeight(.bold)

            Text("บางอย่าง บางอย่าง บางอย่าง")
                .font(.title)
                .foregroundStyle(.gray)
        }
    }
}
```

ผลการแสดงผล:

![view](../../../Resource/025_view5.png)

จากผลพรีวิวจะเห็นว่า view เดียวกัน ภายใต้รูปลักษณ์ต่างกัน เนื้อหาที่แสดงไม่ได้เหมือนกันทั้งหมด

ในโหมดสว่าง `Image` แสดงไอคอนหลอดไฟแบบปกติ และหัวข้อแสดงเป็น `สว่าง`

ในโหมดมืด `Image` แสดงไอคอนหลอดไฟที่สว่างขึ้น และหัวข้อแสดงเป็น `มืด`

กล่าวคือ การเปลี่ยนแปลงตรงนี้ไม่ได้มีแค่สีเปลี่ยน แต่ชื่อไอคอนและข้อความหัวข้อเองก็เปลี่ยนด้วย

เพราะเราไม่ได้เขียนไอคอนและหัวข้อไว้ตายตัวใน `body` แต่คำนวณเนื้อหาที่แตกต่างกันตาม `colorScheme`

ในโค้ดส่วนนี้ เราใช้ computed property สองตัว:

```swift
private var imageName: String {
    colorScheme == .light ? "lightbulb" : "lightbulb.max"
}
```

```swift
private var titleName: String {
    colorScheme == .light ? "สว่าง" : "มืด"
}
```

เมื่อ `colorScheme` เป็น `.light` แสดงว่าปัจจุบันเป็นโหมดสว่าง

เมื่อ `colorScheme` เป็น `.dark` แสดงว่าปัจจุบันเป็นโหมดมืด

ดังนั้นเราจึงสามารถคืนชื่อไอคอนและข้อความหัวข้อที่แตกต่างกันตามรูปลักษณ์แต่ละแบบ

นี่ก็เป็นการใช้งาน computed property ที่พบบ่อยอย่างหนึ่ง: คำนวณเนื้อหาที่ view ต้องแสดงตามสถานะปัจจุบัน

## ทำความเข้าใจ @Environment

นี่เป็นครั้งแรกที่เราได้รู้จัก `@Environment`

`@Environment` สามารถเข้าใจได้ว่า: อ่านค่าหนึ่งจาก environment ของ SwiftUI

เมื่อ App ทำงาน ระบบจะให้ข้อมูล environment จำนวนมาก เช่น:

- ภาษาปัจจุบัน
- รูปลักษณ์ปัจจุบัน
- ทิศทาง layout ปัจจุบัน
- การตั้งค่าขนาดตัวอักษรปัจจุบัน

เมื่อเราต้องการรับรูปลักษณ์ปัจจุบัน ก็สามารถอ่าน `colorScheme` ได้:

```swift
@Environment(\.colorScheme) private var colorScheme
```

โดยที่

```swift
\.colorScheme
```

หมายถึงอ่านค่า `colorScheme` จาก environment

```swift
private var colorScheme
```

หมายถึงบันทึกค่าที่อ่านได้ไว้ในตัวแปร `colorScheme`

ชื่อตัวแปรสามารถกำหนดเองได้ ตัวอย่างเช่นสามารถเขียนเป็น:

```swift
@Environment(\.colorScheme) private var currentColorScheme
```

ตราบใดที่ `\.colorScheme` ด้านหน้ายังไม่เปลี่ยน ก็หมายความว่าอ่านรูปลักษณ์ปัจจุบันอยู่

## ค่าที่พบบ่อยสองค่าของ colorScheme

`colorScheme` มักมีสองค่าที่พบบ่อย:

```swift
.light
.dark
```

ค่าทั้งสองหมายถึง:

- `.light`: โหมดสว่าง
- `.dark`: โหมดมืด

ตัวอย่างเช่น:

```swift
private var titleName: String {
    colorScheme == .light ? "สว่าง" : "มืด"
}
```

ความหมายของโค้ดส่วนนี้คือ:

ถ้าปัจจุบันเป็นโหมดสว่าง เงื่อนไข `colorScheme == .light` จะเป็นจริง แล้วคืนค่า `"สว่าง"`

ไม่เช่นนั้น จะคืนค่า `"มืด"`

ดังนั้น ในโหมดสว่าง:

```swift
Text(titleName)
```

จะแสดงเป็น:

```swift
สว่าง
```

ในโหมดมืด จะแสดงเป็น:

```swift
มืด
```

ด้วยวิธีนี้ เราก็สามารถแสดงเนื้อหาที่แตกต่างกันตามรูปลักษณ์ได้

## ตัวอย่างการปรับสี

นอกจากข้อความและไอคอนแล้ว บางครั้งเรายังต้องจัดการการปรับสีด้วยตนเอง

ตัวอย่างเช่น:

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 20) {
            Text("ข้อความ")
                .font(.system(size: 100))
                .foregroundStyle(Color.white)
                .padding(50)
                .background(Color.black)
                .cornerRadius(50)
        }
    }
}
```

ผลการแสดงผล:

![view](../../../Resource/025_view6.png)

ในโค้ดส่วนนี้ ข้อความเป็นสีขาว และพื้นหลังเป็นสีดำ

ในโหมดสว่าง พื้นหลังสีดำจะเห็นได้ค่อนข้างชัด

แต่ในโหมดมืด หากพื้นหลังทั้งอินเทอร์เฟซเป็นสีดำด้วย พื้นหลังสีดำนี้จะกลืนไปกับพื้นหลังของระบบ และดูไม่ชัดเจนพอ

ตอนนี้ เราสามารถสลับสีข้อความและสีพื้นหลังตามรูปลักษณ์ได้:

```swift
struct ContentView: View {
    @Environment(\.colorScheme) var colorScheme

    var textColor: Color {
        color == .light ? Color.white : Color.black
    }

    var backgroundColor: Color {
        color == .light ? Color.black : Color.white
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("ข้อความ")
                .font(.system(size: 100))
                .foregroundStyle(textColor)
                .padding(50)
                .background(backgroundColor)
                .cornerRadius(50)
        }
    }
}
```

ผลการแสดงผล:

![view](../../../Resource/025_view7.png)

ตอนนี้ ในโหมดสว่างจะเป็นพื้นหลังสีดำและข้อความสีขาว

ในโหมดมืดจะเป็นพื้นหลังสีขาวและข้อความสีดำ

แบบนี้ไม่ว่าจะอยู่ในรูปลักษณ์ใด ข้อความและพื้นหลังก็มี contrast ที่ชัดเจน และผู้ใช้จะเห็นเนื้อหาได้ชัด

## ใช้ Assets เพื่อปรับสี

นอกจากใช้ `colorScheme` เพื่อตรวจสอบด้วยตนเองแล้ว เรายังสามารถใช้สี resource ใน `Assets` เพื่อปรับให้เข้ากับโหมดสว่าง/โหมดมืดได้

วิธีนี้เหมาะกับการจัดการสีทั่วไปใน App มากกว่า

ในโปรเจกต์ Xcode ให้หาโฟลเดอร์ resource `Assets`

![assets](../../../Resource/025_color.png)

เปิดโฟลเดอร์ `Assets` คลิกขวาบริเวณว่าง แล้วเลือก `New Color Set` เพื่อสร้างสี resource ใหม่

![assets](../../../Resource/025_color1.png)

ตรงนี้เราสร้างสีหนึ่งสี และตั้งชื่อว่า `redText`

![assets](../../../Resource/025_color2.png)

สี resource สามารถตั้งค่าสีสำหรับโหมดสว่างและโหมดมืดแยกกันได้

![assets](../../../Resource/025_color3.png)

หลังจากเลือกพื้นที่สีที่เกี่ยวข้องแล้ว สามารถแก้ไขสีใน Attributes inspector ทางด้านขวาได้

คลิกปุ่ม `Hide or show the Inspectors` ที่มุมขวาบน เพื่อเปิดพื้นที่ inspector

จากนั้นเลือก `Show the Attributes inspector` แล้วหา `Show Color Panel` ที่ด้านล่าง เพื่อเปิดแผงสี

![assets](../../../Resource/025_color4.png)

เมื่อคลิกสีในแผงสี พื้นที่สีที่สอดคล้องกับ `redText` จะเปลี่ยนตามไปด้วย

ที่นี่เราตั้งค่า `redText` เป็น:

- โหมดสว่างแสดงสีแดง
- โหมดมืดแสดงสีเขียว

![assets](../../../Resource/025_color5.png)

กลับไปที่โค้ด SwiftUI สามารถใช้สีนี้ได้แบบนี้:

```swift
struct ContentView: View {
    var body: some View {
        Color("redText")
            .frame(width: 300, height: 300)
            .cornerRadius(100)
    }
}
```

โดยที่:

```swift
Color("redText")
```

หมายถึงอ่านสี resource ที่ชื่อ `redText` จาก `Assets`

ผลการแสดงผล:

![assets](../../../Resource/025_view8.png)

จะเห็นว่า `Color("redText")` ตัวเดียวกัน จะแสดงสีที่แตกต่างกันในโหมดสว่างและโหมดมืด

นี่แสดงว่าสีใน `Assets` เองก็รองรับการปรับรูปลักษณ์เช่นกัน

## ความแตกต่างระหว่าง colorScheme และ Assets

ทั้ง `colorScheme` และ `Assets` สามารถทำการปรับรูปลักษณ์ได้ แต่เหมาะกับสถานการณ์ต่างกัน

หากเป็นเพียงการสลับเนื้อหาข้อความ ชื่อรูปภาพ หรือชื่อ `SF Symbols` ตามโหมดสว่าง/โหมดมืด สามารถใช้ `colorScheme`

ตัวอย่างเช่น:

```swift
colorScheme == .light ? "lightbulb" : "lightbulb.max"
```

หากเป็นสีที่ใช้บ่อยใน App เช่น สีธีม สีพื้นหลังของการ์ด หรือสีข้อความ แนะนำให้ใช้สีใน `Assets` มากกว่า

เพราะสีใน `Assets` สามารถนำกลับมาใช้ซ้ำในหลาย view ได้ และแก้ไขภายหลังได้สะดวกกว่า

ตัวอย่างเช่น:

```swift
Color("redText")
```

แบบนี้เพียงแก้ไขสีใน `Assets` ตำแหน่งที่ใช้สีนี้ทั้งหมดก็จะเปลี่ยนไปพร้อมกัน

## สรุป

ในบทเรียนนี้ เราได้เรียนรู้การปรับรูปลักษณ์สำหรับโหมดสว่างและโหมดมืด

ก่อนอื่น เราเรียนรู้วิธีพรีวิวรูปลักษณ์ต่าง ๆ ใน `Canvas` ของ Xcode และเรียนรู้วิธีใช้ `Color Scheme Variants` เพื่อดูโหมดสว่างและโหมดมืดพร้อมกัน

จากนั้น เราเข้าใจว่า view เช่น `Text` และ `Image` ใน SwiftUI จะปรับสีตามรูปลักษณ์ของระบบโดยอัตโนมัติเป็นค่าเริ่มต้น

แต่หากกำหนดสีคงที่ด้วยตนเอง เช่น `Color.white` หรือ `Color.black` สีเหล่านี้จะไม่สลับโดยอัตโนมัติ

ต่อมา เราเรียนรู้ `@Environment` และ `colorScheme`:

```swift
@Environment(\.colorScheme) private var colorScheme
```

ผ่าน `colorScheme` เราสามารถตรวจสอบว่าปัจจุบันเป็นโหมดสว่างหรือโหมดมืด และแสดงไอคอน ข้อความ หรือสีที่แตกต่างกันตามรูปลักษณ์

สุดท้าย เราเรียนรู้สีใน `Assets`

สีใน `Assets` สามารถตั้งค่าสีสำหรับโหมดสว่างและโหมดมืดแยกกันได้ เหมาะกับการจัดการสี resource ทั่วไปใน App

การปรับรูปลักษณ์ช่วยให้ App มีผลการแสดงผลที่ดีขึ้นทั้งตอนกลางวันและตอนกลางคืน และยังช่วยหลีกเลี่ยงปัญหาข้อความมองไม่ชัดหรือพื้นหลังไม่เด่นพอ

ในการพัฒนาจริง แนะนำให้ใช้ความสามารถในการปรับของ SwiftUI โดยค่าเริ่มต้นและสีใน `Assets` เป็นลำดับแรก

เมื่อจำเป็นต้องสลับข้อความ ไอคอน รูปภาพ หรือจัดการเอฟเฟกต์การแสดงผลพิเศษจำนวนเล็กน้อยในรูปลักษณ์ต่างกัน จึงค่อยใช้ `colorScheme` เพื่อตรวจสอบ

## ความรู้เพิ่มเติม: กำหนดขนาดตัวอักษร

ตอนเรียน `font` ก่อนหน้านี้ เราเคยใช้ system font style บางอย่าง:

```swift
Text("FangJunyu")
	.font(.largeTitle)

Text("FangJunyu")
	.font(.callout)
```

นอกจากใช้ system font style เช่น `.largeTitle` และ `.callout` แล้ว เรายังสามารถกำหนดขนาดตัวอักษรด้วยตนเองได้

ตัวอย่างเช่น:

```swift
Image(systemName: "lightbulb")
    .font(.system(size: 100))
```

ตรงนี้:

```swift
.font(.system(size: 100))
```

หมายถึงตั้งค่าขนาดตัวอักษรเป็น `100 pt`

สำหรับไอคอน `SF Symbols` แล้ว `font` ก็มีผลต่อขนาดไอคอนเช่นกัน

ดังนั้น หากต้องการให้ไอคอนแสดงใหญ่ขึ้น สามารถใช้ `.font(.system(size:))` เพื่อกำหนดขนาดได้

ตัวอย่างเช่น:

```swift
Image(systemName: "star.fill")
    .font(.system(size: 80))
```

สิ่งที่ต้องระวังคือ แม้การกำหนดขนาดตัวอักษรด้วยตนเองจะยืดหยุ่นกว่า แต่ข้อความทั่วไปไม่แนะนำให้ใช้ขนาดคงที่จำนวนมาก

สำหรับข้อความทั่วไป ควรให้ความสำคัญกับ system font style เช่น `.title`, `.headline`, `.body`, `.caption` ก่อน

แบบนี้จะปรับให้เข้ากับอุปกรณ์ต่าง ๆ และการตั้งค่าขนาดตัวอักษรของผู้ใช้ได้ดีกว่า
