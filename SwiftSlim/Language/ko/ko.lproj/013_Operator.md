# 연산자

이번 수업에서는 Swift의 연산자(Operators)를 배웁니다.

우리가 계산을 하거나, 비교를 하거나, 논리 판단을 할 때는 모두 연산자를 사용해야 합니다. 이런 작업들은 프로그램 실행의 여러 단계에 걸쳐 등장하며, 프로그래밍에서 가장 기본적인 능력 중 하나입니다.

```swift
let a = 100 * 8
```

프로그래밍을 배우다 보면, 일정한 수학적 사고를 가진 사람이 프로그램 로직을 더 쉽게 이해하는 경우가 많다는 것을 알 수 있습니다. 그 이유는 프로그래밍의 본질이 "계산"과 "규칙 판단"에서 떨어질 수 없고, 바로 이 핵심 역할을 연산자가 담당하기 때문입니다.

실제 개발에서 연산자는 단순한 숫자 계산에만 쓰이지 않고, 매우 다양한 구체적인 상황에서도 널리 사용됩니다.

예를 들어 블록 게임에서 사용자가 블록 하나를 특정 위치로 드래그했을 때는, 다음을 계산해야 합니다:

- 블록의 좌표 위치
- 블록이 채울 수 있는 위치
- 목표 위치가 사용 가능한지 여부

이러한 "계산 + 판단" 과정은 본질적으로 모두 연산자에 의존합니다.

![Block](../../Resource/013_block.png)

연산자를 배운다는 것은 단순히 숫자 계산을 배우는 것이 아니라, 이후 화면 레이아웃, 데이터 처리, 상호작용 로직을 위한 기초를 다지는 것이기도 합니다.

## SwiftUI에서 계산을 실행하는 방식

SwiftUI에서는 보통 `Button`으로 함수를 트리거하고, 함수 안에서 계산 로직을 실행합니다.

예:

```swift
struct ContentView: View {
    func calculate() {
        let a = 10
        let b = 10
        let c = a + b
        print("c: \(c)")
    }
    var body: some View {
        VStack(spacing: 10) {
            Button("Button") {
                calculate()
            }
        }
    }
}
```

버튼을 클릭하면 Console에는 다음이 출력됩니다:

```
c: 20
```

이번 수업에서는 연산 로직을 모두 `func` 안에 작성하고, 버튼으로 실행시키며, 결과는 `print`로 출력하는 방식으로 통일합니다.

이후의 예제에서는 코드 설명을 단순화하기 위해, `print` 출력을 매번 반복하지 않고 코드 안에 계산 결과를 직접 표시합니다.

## Console 출력 관리

버튼을 클릭할 때마다 출력 정보가 계속 쌓이므로, Console 영역 오른쪽 아래의 정리 버튼을 눌러 이전 로그를 지우고 현재 결과를 더 쉽게 확인할 수 있습니다.

![Console](../../Resource/013_console.png)

## Button의 또 다른 문법

버튼이 하나의 함수만 실행할 경우에는, 더 간단한 문법을 사용할 수 있습니다:

```swift
Button(action: ) {
    // View
}
```

이 문법에서 `action`은 버튼 클릭 시 실행할 함수(매개변수가 없는 함수)를 지정하고, 중괄호 `{}` 안에는 버튼의 뷰 내용을 정의합니다.

앞에서 본 `calculate` 함수와 결합하면 다음처럼 쓸 수 있습니다:

```swift
Button(action: calculate) {
    Text("Button")
}
```

버튼을 클릭하면 `calculate` 함수가 바로 실행됩니다.

따라서 버튼 로직이 단순하고 하나의 함수만 호출하는 경우에는, `action` 문법이 더 명확하고 코드 스타일에도 더 잘 맞습니다.

## 산술 연산자

산술 연산자(Arithmetic Operators)는 숫자 계산에 사용됩니다:

- `+` 덧셈
- `-` 뺄셈
- `*` 곱셈
- `/` 나눗셈
- `%` 나머지

예:

```swift
let a = 10 + 5  // 15
let b = 10 - 3  // 7
let c = 10 * 10 // 100
let d = 10 / 5  // 2
let e = 10 % 3  // 1
```

주의할 점은, Swift는 강타입 언어이기 때문에 서로 다른 데이터 타입을 자동으로 섞어 계산하지 않는다는 것입니다:

```swift
let a = 10  // Int
let b = 5.0 // Double

let c = a + b   // Error
let d = Double(a) + b   // 15.0
```

두 값의 타입이 다르면, 수동으로 타입 변환을 해야 합니다.

## 비교 연산자

비교 연산자(Comparison Operators)는 두 값의 관계를 판단하며, 결과는 `Bool`입니다:

- `==` 같다
- `!=` 같지 않다
- `>` 크다
- `<` 작다
- `>=` 크거나 같다
- `<=` 작거나 같다

예:

```swift
let a = 5 == 3  // false
let b = 2 != 4  // true
let c = 4 > 2   // true
let d = 5 < 3   // false
let e = 4 >= 4  // true
let f = 10 <= 7 // false
```

비교 연산자는 판단문에서 자주 사용됩니다. 예를 들어 "명언 캐러셀"에서는 인덱스가 배열 범위를 넘지 않도록 제어해야 했습니다:

```swift
if index < sayings.count - 1 {
    index += 1
}
```

`index`가 `sayings.count - 1`보다 작으면 비교 결과는 true가 되고, `if` 문 안의 코드가 실행됩니다. 비교 결과가 false이면 뒤의 코드는 실행되지 않습니다.

## 논리 연산자

논리 연산자(Logical Operators)는 여러 조건을 조합하는 데 사용됩니다:

- `&&` 그리고(AND)
- `||` 또는(OR)
- `!` 부정(NOT)

### && 그리고

여러 조건이 모두 동시에 만족해야 할 때는 `&&`를 사용합니다.

예:

```swift
let age = 16
let legalAge = age >= 18 && age <= 100
```

이 코드에는 두 개의 조건이 포함되어 있습니다: `age >= 18` 그리고 `age <= 100`.

두 조건이 모두 true일 때만 `legalAge`는 true가 됩니다. 둘 중 하나라도 성립하지 않으면 결과는 false입니다.

### || 또는

여러 조건 중 하나만 만족해도 될 때는 `||`를 사용합니다.

예:

```swift
let money = 1000
let creditCard = 300
let payAllowed = money >= 50 || creditCard >= 50
```

이 코드는 `money >= 50` 또는 `creditCard >= 50` 둘 중 하나만 만족하면 `payAllowed`가 true가 된다는 뜻입니다.

두 조건이 모두 만족하지 않을 때만 결과는 false가 됩니다.

예를 들어 결제 상황에서는 현금 또는 신용카드로 결제할 수 있습니다. 둘 다 불가능하다면 결제는 실패합니다.

### ! 부정

`!`는 논리 부정 연산자(NOT)로, `Bool` 값을 반대로 바꾸는 데 사용됩니다.

간단히 말하면, `!`는 true를 false로, false를 true로 바꿉니다.

예:

```swift
let allowed = true
let result = !allowed   // false
```

여기서 `!allowed`는 `allowed`의 값을 반대로 뒤집는다는 뜻입니다.

주의할 점은, `!`와 `!=`는 서로 다른 연산자라는 것입니다. `!`는 `Bool` 값을 반전시키고, `!=`는 비교 연산자로 `Bool` 값을 반환합니다.

## 대입 연산자

대입 연산자(Assignment Operators)는 값을 대입하거나 변수를 갱신하는 데 사용됩니다:

- `=` 대입

변수를 선언하거나 계산할 때는 모두 대입 연산이 포함됩니다:

```swift
let a = 5
```

이 코드는 숫자 5를 변수 `a`에 대입한다는 뜻입니다.

### 복합 대입 연산자

실제 개발에서는 기본 대입 연산자 외에도, "복합 대입 연산자"를 자주 사용합니다. 이것은 원래 값을 기준으로 바로 계산한 뒤 다시 갱신하는 연산자입니다:

- `+=`
- `-=`
- `*=`
- `/=`

앞서 "카운터" 수업에서도 복합 대입 연산자를 간단히 다룬 적이 있습니다.

예를 들어 버튼을 클릭할 때 변수가 자동으로 증가하도록 만들면:

```swift
var num = 10
num += 5   // num = 15
```

이 코드는 `num`과 5를 더한 뒤, 그 결과를 다시 `num`에 대입한다는 뜻입니다.

따라서 다음과 완전히 같습니다:

```swift
var num = 10
num = num + 5   // num = 15
```

## 삼항 연산자

Swift의 삼항 연산자(Ternary Operator)는 조건을 판단해 두 결과 중 하나를 반환합니다:

```swift
조건 ? 값1 : 값2
```

실행 로직은, 조건이 true이면 "값1"을 반환하고, 조건이 false이면 "값2"를 반환하는 것입니다.

예:

```swift
let showColor = true
let backgroundColor = showColor ? Color.blue : Color.clear
```

이 코드는 `showColor`의 값을 검사합니다. true라면 `Color.blue`를 반환해서 배경이 파란색이 되고, false라면 `Color.clear`를 반환해서 배경색이 보이지 않습니다.

팁: `Color.clear`는 투명색을 의미합니다(보이지 않음).

### if-else와의 차이

삼항 연산자는 논리적으로 `if-else` 문과 같습니다.

예를 들어 위 코드는 다음처럼도 쓸 수 있습니다:

```swift
var backgroundColor: Color

if showColor {
    backgroundColor = Color.blue
} else {
    backgroundColor = Color.clear
}
```

두 방식의 실행 로직은 완전히 같습니다. 조건이 true일 때는 첫 번째 결과를, false일 때는 두 번째 결과를 사용합니다.

실제 사용에서는 삼항 연산자가 더 간결하며, "둘 중 하나"를 표현하는 단순한 판단에 더 적합합니다. 특히 어떤 값을 직접 반환해야 하는 경우에 자주 쓰입니다.

SwiftUI의 View 구성에서도 삼항 연산자는 매우 흔합니다. 예:

```swift
Text("Hello")
    .background(showColor ? Color.blue : Color.clear)
```

이런 문법은 한 줄의 코드 안에서 조건에 따라 뷰 효과를 동적으로 바꾸게 해 줍니다.

## 범위 연산자

범위 연산자(Range Operators)는 구간을 나타내는 데 사용됩니다:

- `...` 닫힌 범위(양쪽 끝 포함)
- `..<` 반열린 범위(오른쪽 끝 미포함)

예:

```
1...3   // 1, 2, 3
5..<8   // 5, 6, 7
```

즉, `1...3`은 1부터 3까지이며 3을 포함하고, `5..<8`은 5부터 8까지지만 8은 포함하지 않습니다.

확장 형태(참고만):

```
5...    // >= 5
...5    // <= 5
```

이 문법은 "한쪽 범위"를 의미하며, 조건 판단이나 패턴 매칭에서 자주 사용됩니다.

범위 연산자는 이후 `for` 반복문을 배울 때 더 자주 쓰이게 되며, 지금은 연속된 숫자 범위를 나타내는 도구로 이해하면 충분합니다.

## 연산자 연습

이제 몇 가지 간단한 예제를 통해, 뷰 안에서 연산자가 어떤 역할을 하는지 연습해 봅시다.

### 1. 차량 총수 계산

예를 들어 어떤 교차로에서 하루에 500대의 차량이 지나간다고 할 때, 30일 동안 지나간 총 차량 수를 계산해 봅시다.

이때 산술 연산자를 사용할 수 있습니다:

```swift
let dayVehicles = 500
let totalVehicles = 500 * 30    // 15000
```

이 계산 과정은, 하루 500대 × 30일 = 총 15000대라는 뜻입니다.

### 2. 인앱 구매 회원 여부 판단

어떤 사람이 평생 회원권 또는 구독 회원권 중 하나라도 가지고 있다면, 우리는 그 사람을 회원으로 판단할 수 있습니다.

```swift
let lifeTime = false
let subscription = true
let isMember = lifeTime || subscription   // true
```

이 코드에서는 `||`(OR) 연산자를 사용합니다. `lifeTime = false`는 평생 회원권이 없다는 뜻이고, `subscription = true`는 구독 회원권이 있다는 뜻입니다.

둘 중 하나만 만족하면 되므로 `isMember`는 true입니다. 따라서 이 사람은 인앱 구매 회원입니다.

## 정리

Swift에서 연산자는 데이터 계산과 논리 판단의 기초입니다. `Text`가 내용을 표시하듯, 연산자는 데이터 자체를 처리하는 데 사용됩니다.

가장 단순한 수치 계산부터 복잡한 논리 판단, 뷰의 상태 제어와 레이아웃 계산에 이르기까지, 연산자의 역할은 빠질 수 없습니다.

여러 장면에서 연습을 반복하면, 이번 수업에서 소개한 다양한 연산자들을 점차 익히고 확실히 이해할 수 있습니다. 이것은 이후 학습을 위한 중요한 기반이 됩니다.

### 확장 지식 - 비트 연산자

또한 Swift는 더 하위 수준의 연산자 종류도 제공합니다. 바로 비트 연산자(Bitwise Operators)입니다:

- `&` (비트 AND)
- `|` (비트 OR)
- `^` (비트 XOR)
- `~` (비트 NOT)
- `>>` (오른쪽 시프트)
- `<<` (왼쪽 시프트)

예:

```swift
let a = 6  // 이진수 110
let b = 3  // 이진수 011

print(a & b) // 2 (이진수 010)
print(a | b) // 7 (이진수 111)
print(a ^ b) // 5 (이진수 101)
print(~a)    // -7 (비트 반전)
```

Swift에서 정수는 2의 보수 이진수 형태로 표현됩니다. 따라서 비트를 반전시키면 대응되는 음수 결과를 얻게 됩니다.

비트 연산자는 이진수 자체를 직접 다루며, 보통 저수준 데이터 처리나 성능 최적화 같은 장면에서 사용됩니다.

초보자 입장에서는 이런 연산자의 사용 장면이 비교적 적고, 일반적인 iOS 또는 SwiftUI 개발에서도 자주 쓰이지는 않기 때문에 여기서는 깊이 있게 설명하지 않습니다.

앞으로 저수준 로직이나 성능 최적화가 필요한 상황을 만나게 되면, 그때 더 깊이 배우고 이해해도 충분합니다。
