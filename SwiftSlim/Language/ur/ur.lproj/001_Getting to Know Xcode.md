# Xcode کو سمجھنا

## Xcode کیا ہے

Xcode ایپل کی جانب سے فراہم کردہ ایک مربوط ترقیاتی ماحول (IDE, Integrated Development Environment) ہے، جو iOS، macOS، watchOS اور tvOS جیسے ایپل پلیٹ فارمز کے لیے ایپلیکیشنز بنانے کے لیے استعمال ہوتا ہے۔

![Swift](../../RESOURCE/001_xcode.png)

اس پورے کورس کے دوران، ہم Xcode کا استعمال کرتے ہوئے کوڈ لکھیں گے اور پروگرام چلائیں گے۔

اس سبق میں، ہم سیکھیں گے کہ Xcode کیسے ڈاؤن لوڈ کیا جائے، پہلا iOS ایپ کیسے بنایا جائے، اور Xcode کے انٹرفیس کی بنیادی ساخت کو کیسے سمجھا جائے۔

## Xcode ڈاؤن لوڈ کرنا

Xcode کو انسٹال کرنے کے دو طریقے ہیں:

1、Mac App Store کے ذریعے ڈاؤن لوڈ کریں  
2、Apple Developer کی آفیشل ویب سائٹ سے ڈاؤن لوڈ کریں  

دونوں طریقوں سے Xcode ڈاؤن لوڈ کیا جا سکتا ہے۔ فرق یہ ہے کہ Apple Developer ویب سائٹ پر آپ کو Beta ورژن اور پرانے ورژنز بھی مل سکتے ہیں۔

اگر آپ صرف سیکھنے کے لیے استعمال کر رہے ہیں تو مستحکم (Stable) ورژن استعمال کرنے کی سفارش کی جاتی ہے۔ Beta ورژن عام طور پر نئے فیچرز کی جانچ کے لیے ہوتے ہیں اور غیر مستحکم ہو سکتے ہیں۔

## پہلا پروجیکٹ بنانا

Xcode کھولنے کے بعد، آپ کو ایک ابتدائی اسکرین نظر آئے گی۔

![Swift](../../RESOURCE/001_xcode1.png)

بائیں جانب تین اہم آپشنز ہوتے ہیں:

- Create New Project —— نیا پروجیکٹ بنائیں  
- Clone Git Repository —— Git ریپوزیٹری کلون کریں  
- Open Existing Project —— موجودہ پروجیکٹ کھولیں  

دائیں جانب حال ہی میں کھولے گئے پروجیکٹس دکھائے جاتے ہیں۔

ہم "Create New Project" منتخب کرتے ہیں۔

### پروجیکٹ ٹیمپلیٹ منتخب کرنا

![Swift](../../RESOURCE/001_xcode2.png)

یہاں مختلف پلیٹ فارمز (iOS، macOS وغیرہ) اور پروجیکٹ ٹیمپلیٹس نظر آئیں گے۔

عام ٹیمپلیٹس کی وضاحت:

- App —— بنیادی ایپلیکیشن (تجویز کردہ)  
- Document App —— ڈاکیومنٹ پر مبنی ایپ  
- Game —— گیم ڈیولپمنٹ کے لیے  
- Framework —— دوبارہ استعمال کے قابل ماڈیول  

ابتدائی مرحلے میں ہم "iOS" - "App" منتخب کریں گے۔

"Next" پر کلک کریں۔

### پروجیکٹ کی معلومات درج کرنا

![Swift](../../RESOURCE/001_xcode3.png)

یہاں مختلف فیلڈز بھرنے ہوں گے:

- Product Name —— پروجیکٹ کا نام، مثال: SwiftSlimTest  
- Team —— ڈیولپر ٹیم، اگر اکاؤنٹ نہیں ہے تو چھوڑ سکتے ہیں  
- Organization Identifier —— عموماً ریورس ڈومین فارمیٹ میں، مثال:  
  - com.yourname  
  - com.yourcompany  
- Bundle Identifier —— ایپ کی منفرد شناخت، خودکار طور پر بن جاتی ہے  
- Interface —— SwiftUI منتخب کریں  
- Language —— Swift منتخب کریں  
- Testing System —— ابتدائی مرحلے میں نظر انداز کریں  
- Storage —— SwiftData یا Core Data، ابتدائی طور پر نظر انداز کریں  

مکمل کرنے کے بعد "Next" پر کلک کریں۔

### پروجیکٹ محفوظ کرنا

![Swift](../../RESOURCE/001_xcode4.png)

ایک مناسب فولڈر منتخب کریں اور "Create" پر کلک کریں۔

Xcode خودکار طور پر پروجیکٹ بنا دے گا۔

## Xcode پروجیکٹ کی ساخت

پروجیکٹ محفوظ کرنے کے بعد، Finder میں فائلز نظر آئیں گی۔

![Swift](../../RESOURCE/001_xcode5.png)

- Assets.xcassets —— تصاویر اور آئیکنز کے لیے  
- ContentView.swift —— SwiftUI ویو فائل  
- SwiftSlimTestApp.swift —— ایپ کا انٹری پوائنٹ  
- SwiftSlimTest.xcodeproj —— پروجیکٹ فائل  

## Xcode انٹرفیس کو سمجھنا

.xcodeproj فائل کھولنے کے بعد، ContentView.swift منتخب کریں۔

Xcode کے پانچ بنیادی حصے ہوتے ہیں:

- Navigator —— فائلز اور ڈھانچہ دیکھنے کے لیے  
- Editor —— کوڈ لکھنے کے لیے  
- Canvas —— SwiftUI پریویو کے لیے  
- Inspector —— پراپرٹیز دیکھنے کے لیے  
- Debug Area —— لاگ اور ڈیبگ کے لیے  

![Swift](../../RESOURCE/001_xcode6.png)

Canvas صرف SwiftUI ویوز کے ساتھ کام کرتا ہے، دیگر فائلز میں یہ دستیاب نہیں ہوتا۔

ہم زیادہ تر Editor میں کام کریں گے۔

Inspector اور Debug Area شروع میں چھپے ہو سکتے ہیں، اوپر دائیں یا نیچے دائیں بٹن سے انہیں ظاہر کیا جا سکتا ہے۔

![Swift](../../RESOURCE/001_xcode7.png)

نوٹ: Inspector اکثر چھپایا جاتا ہے تاکہ زیادہ جگہ دستیاب ہو۔

## خلاصہ

اس سبق میں، ہم نے Xcode ڈاؤن لوڈ کرنا، پہلا پروجیکٹ بنانا، اور بنیادی انٹرفیس سیکھا۔

اب آپ یہ کر سکتے ہیں:
- Navigator میں فائلز بنائیں یا حذف کریں  
- Editor میں کوڈ کا جائزہ لیں  
- Canvas میں پریویو کنٹرولز آزمائیں  

اگلے سبق میں، ہم سادہ کوڈ لکھنا شروع کریں گے۔
