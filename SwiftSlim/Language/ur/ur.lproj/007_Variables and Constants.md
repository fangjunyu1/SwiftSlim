# متغیرات اور مستقلات

اس سبق میں ہم بنیادی طور پر Swift میں متغیرات (Variables) اور مستقلات (Constants) سیکھیں گے، اور ساتھ ہی کچھ عام data types اور بنیادی operators سے بھی واقف ہوں گے۔

یہ سب programming کی نہایت بنیادی معلومات ہیں، اور آگے SwiftUI سیکھنے کے لیے بھی ایک اہم بنیاد فراہم کرتی ہیں۔

## روزمرہ زندگی میں متغیرات اور مستقلات

ہم روزمرہ زندگی کی چیزوں کے ذریعے متغیرات اور مستقلات کو سمجھ سکتے ہیں۔

زندگی میں متغیرات کی مثالیں:

* ٹی وی میں ہر پروگرام کا چلنے والا مواد مختلف ہوتا ہے
* ہر دن کا موسم ایک جیسا نہیں ہوتا
* گھڑی ہر سیکنڈ بدلتی رہتی ہے

ان تمام چیزوں کی مشترک خصوصیت یہ ہے کہ: یہ بدلتی رہتی ہیں۔

اگر ٹی وی پر ہمیشہ ایک ہی تصویر ہو، موسم ہمیشہ صاف رہے، اور گھڑی کی سوئیاں کبھی حرکت نہ کریں، تو یہ چیزیں مستقلات کہلائیں گی۔

ایک چیز بدل سکتی ہے، جبکہ دوسری نہیں بدلتی۔

## متغیرات اور مستقلات کو سمجھنا

App development میں عموماً صارف سے کچھ معلومات داخل کروانا یا محفوظ کرنا ضروری ہوتا ہے۔

مثلاً:

* اکاؤنٹ کا نام
* تاریخِ پیدائش
* رابطے کی معلومات
* پتہ

یہ معلومات محفوظ کی جاتی ہیں اور پھر دکھائی جاتی ہیں۔

مثال کے طور پر، صارف App میں ایک نام داخل کرتا ہے:

```swift id="2h68p9"
FangJunyu
```

ہمیں اس نام کو محفوظ کرنا ہوتا ہے تاکہ اسے App میں دکھایا جا سکے۔

اس محفوظ کرنے کے عمل کو سادہ انداز میں یوں سمجھا جا سکتا ہے: جیسے کسی دراز میں چیز رکھی جائے۔

جب ہم نام محفوظ کرتے ہیں، تو گویا ہم اس نام کو ایک دراز میں رکھ رہے ہوتے ہیں۔

چونکہ محفوظ کرنے والی چیزیں بہت سی ہو سکتی ہیں، اس لیے درازیں بھی بہت سی ہوں گی۔ ہر دراز میں کیا رکھا ہے، یہ جاننے کے لیے ہمیں ہر دراز کا ایک نام رکھنا پڑتا ہے۔

مثلاً:

```swift id="bfj1g0"
name
```

اس مثال میں، name دراز کا نام ہے، جبکہ FangJunyu محفوظ کی گئی معلومات ہے۔

![Var](../../RESOURCE/007_var.png)

**Swift میں اگر آپ کو data محفوظ کرنا ہو، تو لازماً متغیر (`var`) یا مستقل (`let`) کا اعلان کرنا پڑتا ہے۔**

چونکہ نام عام طور پر بدلا جا سکتا ہے، اس لیے ہمیں یہاں متغیر استعمال کرنا چاہیے۔

```swift id="g6r6sd"
var name = "FangJunyu"
```

یہاں name نام کا ایک متغیر declare کیا گیا ہے، اس کی type `String` ہے، اور اس کی value `"FangJunyu"` ہے۔

### متغیر اور مستقل میں فرق

متغیر `var` کے ذریعے declare کیا جاتا ہے:

```swift id="d9qyno"
var
```

مستقل `let` کے ذریعے declare کیا جاتا ہے:

```swift id="hvlnbl"
let
```

مثلاً:

```swift id="u4kop8"
var name = "FangJunyu"
let id = 123456
```

اگر کسی value کو متغیر کے طور پر declare کیا جائے، تو اس کا مطلب ہے کہ اسے بعد میں بدلا جا سکتا ہے؛ اگر اسے مستقل کے طور پر declare کیا جائے، تو پھر اسے دوبارہ نہیں بدلا جا سکتا۔

لہٰذا، متغیر اور مستقل کے درمیان اصل فرق یہ ہے: کیا value کو تبدیل کرنے کی اجازت ہے یا نہیں۔

### مستقل کو تبدیل کرنا

اگر code کسی مستقل کی value بدلنے کی کوشش کرے:

```swift id="n26oq9"
let name = "Sam"
name = "Bob"
```

تو Swift بتائے گا کہ assignment ممکن نہیں، کیونکہ یہ ایک constant ہے۔

```swift id="wfhw3h"
Cannot assign to property: 'name' is a 'let' constant
```

یہ mechanism developer کو اس بات سے بچاتا ہے کہ کہیں کوئی اہم data غلطی سے تبدیل نہ ہو جائے۔

### SwiftUI میں دکھانا

`ContentView.swift` فائل کھولیں، اور View کے اندر ایک متغیر declare کریں:

```swift id="jlwmye"
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

یہاں name نام کا ایک متغیر declare کیا گیا ہے، اور `Text` کے ذریعے اسے interface پر دکھایا گیا ہے۔

اگر ہم متغیر کی value بدل دیں:

```swift id="g4c6gm"
struct ContentView: View {
    var name = "Sam"
    var body: some View {
        Text(name)
    }
}
```

تو `Text` میں دکھنے والا متن بھی بدل جائے گا۔

اس طریقے سے ہم متغیرات کے ذریعے interface میں دکھائی جانے والی معلومات کو کنٹرول کر سکتے ہیں، اور ہر بار `Text` کے اندر موجود متن کو براہِ راست تبدیل کرنے کی ضرورت نہیں پڑتی۔

اگر کسی اور جگہ بھی یہی معلومات درکار ہو، تو ہم اس متغیر کو وہاں بھی منتقل کر سکتے ہیں، گویا دراز کے اندر رکھی ہوئی چیز کسی اور جگہ استعمال کے لیے دے دی گئی۔

### SwiftUI میں متغیرات اور مستقلات کی جگہ

SwiftUI میں، متغیرات اور مستقلات عموماً `body` کے باہر لکھی جاتی ہیں:

```swift id="6a22lv"
struct ContentView: View {
    var name = "FangJunyu"

    var body: some View {
        Text(name)
    }
}
```

البتہ، انہیں `body` کے اندر بھی لکھا جا سکتا ہے:

```swift id="jjae5u"
struct ContentView: View {
    var body: some View {
        var name = "FangJunyu"
        Text(name)
    }
}
```

لیکن ان دونوں میں ایک اہم فرق ہے:

* اگر متغیر یا مستقل `body` کے اندر لکھی جائے، تو ہر بار view دوبارہ calculate ہونے پر یہ دوبارہ بنائی جائے گی۔
* اگر اسے `body` کے باہر لکھا جائے، تو یہ view struct کی property کے طور پر موجود رہے گی، اور code کی structure بھی زیادہ واضح ہوگی۔

اس لیے عملی development میں، متغیرات اور مستقلات کو عموماً `body` کے باہر لکھا جاتا ہے۔

## Data Types

متغیرات مختلف قسم کا data محفوظ کر سکتی ہیں، اور ان اقسام کو data types کہا جاتا ہے۔

مثلاً، ہم پہلے `Text` کے ذریعے متن دکھا چکے ہیں:

```swift id="9ijw7b"
Text("Hello, World")
```

یہاں `"Hello, World"` ایک string (`String`) type ہے۔

String متن ظاہر کرنے کے لیے استعمال ہوتی ہے، اور Swift میں اسے دوہرے quotes `""` کے اندر لکھا جاتا ہے۔

مثلاً:

```swift id="1k8wxb"
var hello = "Hello, World"
```

String کے علاوہ بھی Swift میں کئی data types موجود ہیں۔

ابتدائی مرحلے میں سب سے زیادہ عام چار types یہ ہیں:

* String
* Int
* Double
* Bool

یہ چار types بنیادی data processing کے زیادہ تر کام انجام دے سکتی ہیں۔

**String**

`String` متن کے لیے استعمال ہوتی ہے، مثلاً:

```swift id="21bhzn"
var name = "FangJunyu"
var city = "Rizhao"
```

Strings عموماً ان چیزوں کے لیے استعمال ہوتی ہیں: username، title، متن کا مواد۔

**Int**

`Int` integer یعنی صحیح عدد کو ظاہر کرتی ہے، یعنی ایسا عدد جس میں اعشاریہ نہ ہو۔

مثلاً:

```swift id="0ms3ib"
var age = 26
var count = 100
```

Integers عموماً ان چیزوں کے لیے استعمال ہوتے ہیں: عمر، مقدار، count۔

**Double**

`Double` اعشاریہ والے اعداد کے لیے استعمال ہوتی ہے۔

مثلاً:

```swift id="pl1v61"
var weight = 74.5
var height = 185.0
```

Swift میں اعشاریہ والے اعداد عموماً default طور پر `Double` ہوتے ہیں۔

ایک ملتی جلتی type `Float` بھی ہے، لیکن وہ نسبتاً کم range ظاہر کرتی ہے، اس لیے عملی development میں `Double` زیادہ عام ہے۔

**Bool**

`Bool` type دو حالتوں کو ظاہر کرنے کے لیے استعمال ہوتی ہے: `true` اور `false`۔

مثلاً:

```swift id="s9ba02"
var isShowAlert = true
var isLogin = false
```

`Bool` اکثر condition checking میں استعمال ہوتی ہے۔ مثال کے طور پر: کیا alert box دکھانا ہے یا نہیں۔

جہاں صرف دو نتائج ممکن ہوں، وہاں `Bool` بہت موزوں type ہے۔

### SwiftUI میں دکھانا

جب ہم `String` type دکھاتے ہیں، تو اسے سیدھا `Text` میں استعمال کر سکتے ہیں:

```swift id="smu9oc"
Text(name)
```

لیکن `Int` اور `Double` strings نہیں ہوتیں، اس لیے انہیں براہِ راست متن کے حصے کے طور پر استعمال نہیں کیا جا سکتا۔

اگر آپ کو `Text` میں یہ data دکھانا ہو، تو string interpolation استعمال کی جاتی ہے:

```swift id="0xpow5"
\()
```

String interpolation ہمیشہ string کے اندر لکھی جاتی ہے، اور `\()` کے اندر متغیر یا مستقل رکھی جاتی ہے۔

مثلاً:

```swift id="ug1v67"
var num = 1

Text("\(num)")
Text("DoubleNum: \(num * 2)")
```

اوپر والے code میں `""` string کو ظاہر کرتا ہے، جبکہ `\()` کا استعمال متغیر یا مستقل کو string کے اندر شامل کرنے کے لیے ہوتا ہے۔

مثلاً:

```swift id="jlwmam"
"DoubleNum: \(num * 2)"
```

اس کا دکھایا جانے والا نتیجہ ہوگا:

```swift id="7rjlwm"
DoubleNum: 2
```

اس طریقے سے ہم strings کے اندر متغیرات یا مستقلات شامل کر سکتے ہیں، اور پھر `Text` کے ذریعے انہیں interface پر دکھا سکتے ہیں۔

نوٹ: `\()` صرف string `""` کے اندر ہی استعمال ہو سکتا ہے۔

## Operators

جب ہم متغیر یا مستقل declare کرتے ہیں، تو یہ syntax نظر آتی ہے:

```swift id="8e5g50"
var num = 1
```

یہاں `=` کو assignment operator کہتے ہیں۔

اس کا کام ہے: دائیں طرف کی value کو بائیں طرف کے متغیر کو assign کرنا۔

اس مثال میں، `1` متغیر `num` کو assign کی گئی ہے۔

assignment operator کے علاوہ کچھ عام عددی operators بھی ہیں:

* `+`
* `-`
* `*`
* `/`

جب ہم اعداد پر calculation کرتے ہیں، مثلاً `Int` یا `Double` type کے data پر، تو انہی operators کا استعمال ہوتا ہے۔

مثلاً:

```swift id="s362dv"
var age = 21 + 3 // 24
var value = 30 - 2 // 28
var num = 1 * 3 // 3
var weight = 88.5 / 2 // 44.25
```

calculation مکمل ہونے کے بعد، نتیجہ متغیر میں محفوظ ہو جاتا ہے۔

## خلاصہ

متغیرات، مستقلات، اور operators programming کے سب سے بنیادی تصورات ہیں۔

متغیرات اور مستقلات کے ذریعے ہم پروگرام میں مختلف قسم کا data محفوظ کر سکتے ہیں؛ data types کے ذریعے ہم data کی نوعیت واضح کرتے ہیں؛ اور operators کے ذریعے ہم data پر calculation اور processing انجام دیتے ہیں۔

یہ تمام چیزیں programming کی دنیا کے بنیادی tools کی طرح ہیں۔ اگر آپ ان تصورات کو اچھی طرح سمجھ لیں، تو یہ آگے Swift اور SwiftUI سیکھنے کے لیے ایک مضبوط بنیاد فراہم کریں گی۔
